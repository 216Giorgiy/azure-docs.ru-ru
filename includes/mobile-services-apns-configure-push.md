После регистрации приложения в APNS и настройки проекта необходимо настроить мобильную службу для интеграции с APNS.

1. В Keychain Access щелкните правой кнопкой мыши новый сертификат приложения быстрого запуска в разделе **Ключи** или **Мои сертификаты**, щелкните **Экспорт**, назовите файл QuickstartPusher, выберите формат **.p12**, а затем нажмите кнопку **Сохранить**.

   	![](./media/mobile-services-apns-configure-push/mobile-services-ios-push-step18.png)

  Запишите имя файла и расположение экспортируемого сертификата.

>[AZURE.NOTE] В этом учебнике создается файл QuickstartPusher.p12. Имя файла и расположение могут отличаться.

2. Войдя на [портал управления Azure], щелкните элемент **Мобильные службы** и выберите нужное приложение.

   	![](./media/mobile-services-apns-configure-push/mobile-services-selection.png)

3. Перейдите на вкладку **Push** (Push-уведомления) и щелкните **Upload** (Отправить).

   	![](./media/mobile-services-apns-configure-push/mobile-push-tab-ios.png)

	Откроется диалоговое окно отправки сертификата.

4. Щелкните **Файл**, выберите файл экспортированного сертификата QuickstartPusher.p12 и введите пароль в поле **Пароль**. Убедитесь, что выбран правильный режим (либо Dev/Sandbox, либо Prod/Production) в поле **Режим**, щелкните значок галочки, а затем нажмите кнопку **Сохранить**.

   	![](./media/mobile-services-apns-configure-push/mobile-push-tab-ios-upload.png)

    > [AZURE.NOTE] В этом учебнике используются сертификаты разработки.

Ваша мобильная служба теперь настроена для работы с APNS.

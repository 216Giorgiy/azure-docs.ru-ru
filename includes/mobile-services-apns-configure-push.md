После регистрации приложения в APNS и настройки проекта необходимо настроить мобильную службу для интеграции с APNS.

1. В Keychain Access щелкните правой кнопкой мыши новый сертификат приложения быстрого запуска в разделе **Ключи** или **Мои сертификаты**, нажмите **Экспорт**, дайте файлу имя QuickstartPusher, выберите формат **.p12**, затем нажмите кнопку **Сохранить**.

   	![](./media/mobile-services-apns-configure-push/mobile-services-ios-push-step18.png)

  Запишите имя файла и расположение экспортируемого сертификата.

>[WACOM.NOTE]В этом учебнике создается файл QuickstartPusher.p12. Имя файла и расположение могут отличаться.

2. Войдите в [Портал управления Azure], щелкните **Мобильные службы**, затем щелкните свое приложение.

   	![](./media/mobile-services-apns-configure-push/mobile-services-selection.png)

3. Перейдите на вкладку **Push-уведомления** и нажмите кнопку **Загрузить**.

   	![](./media/mobile-services-apns-configure-push/mobile-push-tab-ios.png)

	Откроется диалоговое окно отправки сертификата.

4. Щелкните **Файл**, выберите файл экспортированного сертификата QuickstartPusher.p12, введите пароль в поле **Пароль**, убедитесь, что выбран правильный режим (либо Dev/Sandbox, либо Prod/Production) в поле **Режим**, щелкните значок галочки, а затем нажмите кнопку **Сохранить**.

   	![](./media/mobile-services-apns-configure-push/mobile-push-tab-ios-upload.png)

    > [WACOM.NOTE] В этом учебнике используются сертификаты разработки.

Ваша мобильная служба теперь настроена для работы с APNS.

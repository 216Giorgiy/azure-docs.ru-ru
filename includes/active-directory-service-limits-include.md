Ниже перечислены ограничения на использование и другие ограничения для службы Azure Active Directory (Azure AD).

| Категория | Ограничения |
| --- | --- |
| Каталоги | Один пользователь может принадлежать максимум к 500 каталогам Azure AD как член или гость.<br/>Один пользователь может создать максимум 20 каталогов. |
| Домены | Можно добавить не более 900 управляемых доменных имен. Если вы настраиваете все домены для федерации с локальной Active Directory, можно добавить не более чем 450 доменных имен в каждом каталоге. |
| Объекты |<ul><li>Пользователи выпуска Free службы Azure Active Directory могут создать в одном каталоге не более 500 000 объектов.</li><li>Пользователь без прав администратора может создать не более 250 объектов. Как активные объекты, так и удаленные объекты, которые доступны для восстановления этой квоты, учитываются. Только удаленные объекты, которые были удалены менее 30 дней назад доступны для восстановления. Удаленные объекты, которые больше не доступны для восстановления, учитываются эта квота в значение 1 4, в течение 30 дней. Возможно [назначить роль администратора](../articles/active-directory/users-groups-roles/directory-assign-admin-roles.md) пользователям без прав администратора, которые, скорее всего многократно превысить квоту во время их обычные обязанности.</li></ul> |
| Расширения схемы |<ul><li>Строковый тип расширения не может превышать 256 символов. </li><li>Расширения двоичного типа ограничены 256 байт.</li><li>Только 100 значений расширения, в *все* типов и *все* приложения, могут быть написаны в один объект.</li><li>Только сущности пользователя, группы, TenantDetail, устройства, приложения и субъекта-службы может быть расширена с помощью строкового типа или атрибуты двоичного типа с одним значением.</li><li>Расширения схемы доступны только в предварительной версии Graph API версии 1.21. Приложению должен быть предоставлен доступ на запись для регистрации расширения.</li></ul> |
| ПРИЛОЖЕНИЯ |Владельцами одного приложения могут быть не более 100 пользователей. |
| Группы |<ul><li>Владельцами одной группы могут быть не более 100 пользователей.</li><li>Участниками одной группы может быть любое число объектов.</li><li>Пользователь может участником любого количества групп.</li><li>Число элементов в группе, которые можно синхронизировать из локальной службы Active Directory в Azure Active Directory, с помощью Azure AD Connect ограничено до 50 000.</li></ul> |
| Панель доступа |<ul><li>Нет ограничений на число приложений, которые можно увидеть на панели доступа каждого пользователя. Это относится к пользователей с лицензиями на Azure AD Premium или Enterprise Mobility Suite.</li><li>Более 10 плиток приложения отображаются на панели доступа для каждого пользователя. Это ограничение применяется к пользователям назначаются лицензии бесплатно или Azure AD Basic выпусков Azure Active Directory. Плиток приложений примеры Box, Salesforce или Dropbox. Это ограничение не применяется к учетным записям администратора.</li></ul> |
| Отчеты | В любом отчете можно просматривать и загружать не более 1000 строк. Любые дополнительные данные будут усечены. |
| Административные единицы | Объект может входить не более чем в 30 административных единиц. |
| Роли и разрешения администратора | <li>Группу невозможно добавить в качестве владельца.<li>Группу нельзя назначить роли.<li>По умолчанию пользователя, которого нельзя изменить разрешения, за исключением клиента коммутаторах, которые являются параметры пользователя в Azure AD. |

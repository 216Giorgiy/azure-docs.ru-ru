
Во-первых, вам необходимо зарегистрировать приложение на сайте поставщика удостоверений, а затем задайте эти учетные данные в вашей мобильной службе.

1. На [портале управления Azure] перейдите к вашей мобильной службе, щелкните **Панель мониторинга** и запишите значение поля **URL-адрес мобильной службы**.

2. Зарегистрируйте приложение у одного из следующих поддерживаемых идентификаторов поставщиков.

	* [Google](mobile-services-how-to-register-google-authentication.md)
	* [Facebook](mobile-services-how-to-register-facebook-authentication.md)
	* [Twitter](mobile-services-how-to-register-twitter-authentication.md)
	* [Майкрософт](mobile-services-how-to-register-microsoft-authentication.md)
	* [Azure Active Directory](mobile-services-how-to-register-active-directory-authentication.md).  
	
    >[AZURE.IMPORTANT]Убедитесь, что URI перенаправления для мобильной службы правильно установлен на сайте разработчика поставщика удостоверений. Как описано в соответствующих инструкциях для каждого из указанных выше поставщиков, путь URL-адреса перенаправления у серверной службы .NET (`/signin-<provider>`) и серверной службы JavaScript (`/login/<provider>`) может различаться. Неправильно заданный URI перенаправления не позволяет клиентам войти в приложение. <br/>Не сообщайте никому секрет клиента.

3. В вашей мобильной службы на [портале управления Azure] щелкните вкладку **Удостоверение** и введите идентификатор и секрет приложения, полученные от поставщика удостоверений.

После настройки приложения и мобильной службы для поддержки одного поставщика удостоверений для проверки подлинности можно повторить эти действия, чтобы добавить поддержку других поставщиков удостоверений.

[портале управления Azure]: https://manage.windowsazure.com/

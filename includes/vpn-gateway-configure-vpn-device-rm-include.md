Для подключения типа "сеть — сеть" к локальной сети требуется VPN-устройство. С Azure совместимы многие VPN-устройства. Сведения о VPN-устройствах и параметрах конфигурации см. в статье [VPN-устройства и параметры IPsec/IKE для подключений типа "сеть — сеть" через VPN-шлюз](../articles/vpn-gateway/vpn-gateway-about-vpn-devices.md). Прежде чем настраивать VPN-устройство, которое вы хотите использовать, проверьте его на наличие [известных проблем совместимости устройств](../articles/vpn-gateway/vpn-gateway-about-vpn-devices.md#known). Чтобы получить сведения о конфигурации для конкретного VPN-устройства, обратитесь к его производителю.

Чтобы настроить локальное VPN-устройство, вам потребуются следующие компоненты.

- **Общедоступный IP-адрес** шлюза виртуальной сети.

    -  Чтобы найти общедоступный IP-адрес с помощью портала Azure, перейдите к разделу **Шлюзы виртуальной сети** и щелкните имя шлюза. 

    - Чтобы найти общедоступный IP-адрес шлюза виртуальной сети с помощью PowerShell, используйте следующий пример, заменив значения собственными:

            Get-AzureRmPublicIpAddress -Name GW1PublicIP -ResourceGroupName TestRG
- **Общий ключ**. Это тот же общий ключ, который указывается при создании VPN-подключения "сеть — сеть". В наших примерах мы используем очень простые общие ключи. Для практического использования следует создавать более сложные ключи.

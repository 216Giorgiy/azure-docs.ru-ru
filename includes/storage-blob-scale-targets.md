|----------|---------------|
| Максимальный размер одного большого двоичного объекта контейнера | Так же, как максимальная емкость учетной записи |
| Максимальное количество блоков в блоке BLOB-объектов или добавления больших двоичных объектов | 50 000 блоков |
| Максимальный размер блока в блочных BLOB-объектов | 100 МиБ |
| Максимальный размер блочного BLOB-объекта | 50 000 x 100 Миб (примерно 4,75 Тиб) |
| Максимальный размер блока в добавочный BLOB-объект | 4 МиБ |
| Максимальный размер добавочный BLOB-объект | 50 000 x 4 Миб (примерно 195 Гиб) |
| Максимальный размер страничного BLOB-объекта | 8 ТиБ |
| Максимальное число хранимых политик доступа на контейнер больших двоичных объектов | 5 |
|Целевая пропускная способность для одного большого двоичного объекта |Вплоть до ограничения исходящего и входящего трафика для учетной записи хранения<sup>1</sup> |

<sup>1</sup> пропускная способность одного объекта зависит от нескольких факторов, включая, но не ограничиваясь: параллелизма, размер запроса, уровень производительности, скорость источника для передачи файлов, а также место назначения для загрузки. Чтобы воспользоваться преимуществами [высокой пропускной способностью блочный BLOB-объект](https://azure.microsoft.com/blog/high-throughput-with-azure-blob-storage/) средства повышения производительности, используйте размер запроса Put Blob или Put Block > 4 MiB (> 256 MiB для производительности уровня "премиум" хранилище BLOB-объектов или Gen2 хранилища Озера данных).

<sup>1</sup>Общий предел частоты запросов для учетной записи хранения составляет 20 000 операций ввода-вывода в секунду. Если виртуальная машина выполняет максимальное количество операций ввода-вывода в секунду на один диск, то чтобы избежать возможного регулирования, убедитесь, что общее количество операций ввода-вывода в секунду для всех VHD виртуальных машин не превышает предел для учетной записи хранения (20 000 операций ввода-вывода в секунду).

Можно приблизительно вычислить количество дисков с высокой степенью использования, поддерживаемых одной учетной записью хранения на основе ограничения транзакции. Например, для виртуальной машины уровня Basic максимальное количество дисков с высокой степенью использования равно 66 (20 000/300 операций ввода-вывода в секунду на один диск), а для виртуальной машины уровня Standard этот показатель равен 40 (20 000/500 операций ввода-вывода в секунду на один диск). Обратите внимание, что учетная запись хранения может поддерживать большее количество дисков, если они не все одновременно используются с высокой степенью интенсивности.

<sup>2</sup>Под термином *входящие данные* подразумеваются все данные (запросы), отправляемые в учетную запись хранения. Под *исходящими данными* понимаются все данные (отклики), получаемые из учетной записи хранения.

<sup>3</sup>GRS — это геоизбыточное хранилище, а LRS — это локально избыточное хранилище.

В таблице ниже приводятся стандартные ограничения для службы хранилища Azure. Под ограничением на *входящие данные* подразумеваются все данные (запросы), отправляемые в учетную запись хранения. А под ограничением на *исходящие данные* — все данные (ответы), получаемые из учетной записи хранения.

| Ресурс | Ограничение по умолчанию |
| --- | --- |
| Количество учетных записей хранения на регион или подписку, включая учетные записи ценовых категорий "Стандартный" и "Премиум". | 200 |
| Максимальная емкость учетной записи хранения<sup>1</sup> | 500 ТиБ |
| Максимальное количество контейнеров BLOB-объектов, BLOB-объектов, общих папок, таблиц, очередей, сущностей или сообщений для учетной записи хранения | Без ограничений |
| Максимальная частота запросов<sup>1</sup> на учетную запись хранения | 20 000 запросов в секунду |
| Максимальная скорость входящих данных<sup>1</sup> на одну учетную запись хранения (регионы США) | 10 Гбит/с для хранилищ RA-GRS и GRS, 20 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |
| Максимальная скорость исходящих данных<sup>1</sup> на одну учетную запись хранения (регионы США) | 20 Гбит/с для хранилищ RA-GRS и GRS, 30 Гбит/с для хранилищ LRS и ZRS |
| Максимальная скорость входящих данных<sup>1</sup> на одну учетную запись хранения (регионы за пределами США) | 5 Гбит/с для хранилищ RA-GRS и GRS, 10 Гбит/с для хранилищ LRS и ZRS<sup>2</sup> |
| Максимальная скорость исходящих данных<sup>1</sup> на одну учетную запись хранения (регионы за пределами США) | 10 Гбит/с для хранилищ RA-GRS и GRS, 15 Гбит/с для хранилищ LRS и ZRS |

<sup>1</sup> Учетные записи хранения Azure поддерживают более высокие ограничения на емкость, частоту запросов, входящие и исходящие данные по запросу. Дополнительные сведения об увеличении ограничений, см. в [объявлении об учетных записях хранения большего размера и большего масштаба](https://azure.microsoft.com/blog/announcing-larger-higher-scale-storage-accounts/). Чтобы подать запрос на увеличение ограничений для учетной записи, обратитесь в [службу поддержки Azure](https://azure.microsoft.com/support/faq/).

<sup>2</sup>Параметры [репликации службы хранилища Azure](https://docs.microsoft.com/azure/storage/common/storage-redundancy):
* **RA-GRS.** Геоизбыточное хранилище с доступом для чтения. Если параметр RA-GRS включен, входящие целевые объекты для вторичного расположения идентичны целевым объектам для первичного расположения.
* **GRS.** Геоизбыточное хранилище. 
* **ZRS.** Хранилище, избыточное в пределах зоны.
* **LRS.** Локально избыточное хранилище.

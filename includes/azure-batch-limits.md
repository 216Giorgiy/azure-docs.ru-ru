| **Ресурс** | **Ограничение по умолчанию** | **Максимальное ограничение** |
| --- | --- | --- |
| Учетных записей пакетной службы на регион на подписку | 3 |50 |
| Выделенные ядра на каждую учетную запись пакетной службы (режима пакетной службы)<sup>1</sup> | 20 | Недоступно<sup>2</sup> |
| Ядра с низким приоритетом на каждую учетную запись пакетной службы (режима пакетной службы)<sup>3</sup> | 50 | Недоступно<sup>4</sup> |
| Число заданий и графиков их выполнения<sup>5</sup> на одну учетную запись пакетной службы | 20 | 5000<sup>6</sup> |
| Количество пулов на одну учетную запись пакетной службы | 20 | 2500 |

<sup>1</sup> Отображенные квоты на выделенные ядра предназначены только для учетных записей со значением **Пакетная служба** для режима выделения пула. Для учетных записей с режимом, которому задано значение **Пользовательская подписка**, квоты ядер основаны на соответствующих квотах для виртуальных машин на региональном уровне или для семейства виртуальных машин в подписке.

<sup>2</sup> Число выделенных ядер на учетную запись пакетной службы можно увеличить, при этом максимальное значение не указано. Чтобы обсудить варианты увеличения, обратитесь в службу поддержки Azure.

<sup>3</sup> Отображенные квоты ядер с низким приоритетом предназначены только для учетных записей со значением **Пакетная служба** для режима выделения пула. Ядра с низким приоритетом недоступны для учетных записей, где пул выделяется в **подписке пользователя**.

<sup>4</sup> Число ядер с низким приоритетом на учетную запись пакетной службы можно увеличить, при этом максимальное значение не указано. Чтобы обсудить варианты увеличения, обратитесь в службу поддержки Azure.

<sup>5</sup> Для завершенных заданий и графиков выполнения заданий ограничений нет.

<sup>6</sup> Свяжитесь со службой поддержки Azure, чтобы оставить запрос на повышение лимита.

## <a name="sign-in-to-the-azure-portal"></a>Вход на портал Azure

Войдите на [портал Azure](https://portal.azure.com/)

## <a name="create-a-blank-sql-database"></a>Создание пустой базы данных SQL

База данных Azure SQL существует в пределах определенного набора [вычислительных ресурсов и ресурсов хранения](../articles/sql-database/sql-database-service-tiers-dtu.md). База данных работает в пределах [группы ресурсов Azure](../articles/azure-resource-manager/resource-group-overview.md) и [логического сервера базы данных SQL Azure](../articles/sql-database/sql-database-features.md).

Чтобы создать пустую базу данных SQL, выполните приведенные ниже действия.

1. Щелкните **Создать ресурс** в верхнем левом углу окна портала Azure.

1. На странице **Создать** выберите **Базы данных** > **База данных SQL**.

   ![Создание пустой базы данных](../articles/sql-database/media/sql-database-design-first-database/create-empty-database.png)

1. В области **База данных SQL** введите или выберите следующие значения.

   | Параметр       | Рекомендуемое значение | ОПИСАНИЕ |
   | ------------ | ------------------ | ------------------------------------------------- |
   | **Database name** (Имя базы данных) | *yourDatabase* | Допустимые имена баз данных см. в статье [Идентификаторы баз данных](/sql/relational-databases/databases/database-identifiers). |
   | **Подписка** | *yourSubscription*  | Дополнительные сведения о подписках см. [здесь](https://account.windowsazure.com/Subscriptions). |
   | **Группа ресурсов** | *yourResourceGroup* | Допустимые имена групп ресурсов см. в статье о [правилах и ограничениях именования](/azure/architecture/best-practices/naming-conventions). |
   | **Выбрать источник** | Пустая база данных | Указывает, что должна быть создана пустая база данных. |

   ![создание базы данных](../articles/sql-database/media/sql-database-design-first-database/create-database.png)

   1. Выберите **Сервер**, чтобы настроить сервер для новой базы данных. Затем введите или выберите следующие значения:

      | Параметр       | Рекомендуемое значение | ОПИСАНИЕ |
      | ------------ | ------------------ | ------------------------------------------------- |
      | **Server name** (Имя сервера) | Любое глобально уникальное имя | Допустимые имена серверов см. в статье о [правилах и ограничениях именования](/azure/architecture/best-practices/naming-conventions). |
      | **Имя для входа администратора сервера** | Любое допустимое имя | Сведения о допустимых именах для входа см. в статье [Идентификаторы базы данных](/sql/relational-databases/databases/database-identifiers).|
      | **Пароль** | Любой допустимый пароль | Длина пароля должна составлять минимум 8 символов. В пароле должны использоваться символы трех категорий из перечисленных: прописные буквы, строчные буквы, цифры и специальные символы. |
      | **Местоположение.** | Любое допустимое расположение | Дополнительные сведения о регионах Azure см. [здесь](https://azure.microsoft.com/regions/). |

      Щелкните **Выбрать**.

      ![Создание базы данных — сервер](../articles/sql-database/media/sql-database-design-first-database/create-database-server.png)

   1. Выберите **Ценовая категория**, чтобы указать уровень службы, число DTU и объем хранилища. Изучите доступные ресурсы для каждого уровня служб (DTU и объем хранилища).

      Выбрав уровень сервера, число DTU и объем хранилища, нажмите кнопку **Применить**.

   1. Заполните поле **Параметры сортировки** для пустой базы данных. В этом руководстве используйте значение по умолчанию. Дополнительные сведения о параметрах сортировки см. в [этой статье](/sql/t-sql/statements/collations).

1. Заполнив форму **База данных SQL**, щелкните **Создать**, чтобы подготовить базу данных. Выполнение этого шага занимает около полутора минут.

1. На панели инструментов щелкните **Уведомления**, чтобы отслеживать процесс развертывания.

     ![уведомление](../articles/sql-database/media/sql-database-design-first-database/notification.png)

## <a name="create-a-firewall-rule"></a>Создание правила брандмауэра

Служба "База данных SQL" создает брандмауэр уровня сервера, который не позволяет внешним приложениям и инструментам подключаться к серверу или любой базе данных на сервере. Выполните приведенные ниже действия, чтобы создать [правило брандмауэра уровня сервера базы данных SQL](../articles/sql-database/sql-database-firewall-configure.md) для IP-адреса клиента. Этот процесс обеспечивает внешнее подключение через брандмауэр базы данных SQL только для IP-адреса.

> [!NOTE]
> База данных SQL обменивается данными через порт 1433. Если вы пытаетесь подключиться из корпоративной сети, исходящий трафик через порт 1433 может быть запрещен сетевым брандмауэром. В таком случае вы не сможете подключиться к серверу базы данных SQL Azure, пока ваш администратор не откроет порт 1433.

1. По завершении развертывания щелкните раздел **Базы данных SQL** в меню слева и выберите *yourDatabase* на странице **Базы данных SQL**. После этого откроется страница **обзора** базы данных, где будет указано полное имя сервера (например, *yourserver.database.windows.net*) и будут предоставлены параметры для дальнейшей настройки.

1. Скопируйте полное имя сервера. Оно понадобится вам при работе на последующих шагах для подключения к серверу и связанным базам данных.

   ![Имя сервера](../articles/sql-database/media/sql-database-design-first-database/server-name.png)

1. На панели инструментов щелкните **Настройка брандмауэра для сервера**. Откроется страница **Настройки брандмауэра** для сервера базы данных SQL.

   ![правило брандмауэра для сервера](../articles/sql-database/media/sql-database-design-first-database/server-firewall-rule.png)

   1. На панели инструментов выберите **Добавить IP-адрес клиента**, чтобы добавить текущий IP-адрес в новое правило брандмауэра. С использованием правила брандмауэра можно открыть порт 1433 для одного IP-адреса или диапазона IP-адресов.

   1. Нажмите **Сохранить**. Для текущего IP-адреса будет создано правило брандмауэра уровня сервера, с помощью которого можно открыть порт 1433 логического сервера.

   1. Щелкните **ОК**, а затем закройте страницу **Параметры брандмауэра**.

Теперь с помощью вашего IP-адреса можно проходить через брандмауэр и подключаться к серверу базы данных SQL и его базам данных с помощью SSMS или другого средства по своему усмотрению. Обязательно используйте созданную ранее учетную запись администратора сервера.

> [!IMPORTANT]
> По умолчанию доступ через брандмауэр базы данных SQL включен для всех служб Azure. На этой странице щелкните **Выкл.**, чтобы отключить доступ для всех служб Azure.

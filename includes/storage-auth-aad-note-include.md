> [!NOTE]
> - Назначения ролей RBAC может занять до двух минут для распространения.
>
> Для авторизации операций с большими двоичными объектами и очередями с использованием токена OAuth необходимо использовать протокол HTTPS.
>
> - Портал Azure теперь позволяет использовать учетные данные Azure AD для чтения и записи данных больших двоичных объектов и очередей в рамках предварительной версии. Для доступа к данным больших двоичных объектов и очереди на портале Azure, пользователю необходимо назначить Azure Resource Manager **чтения** роли, в дополнение к соответствующей роли для доступа к BLOB-объектов или очередей. Дополнительные сведения см. в разделе [предоставить доступ к контейнерам Azure и очереди с RBAC на портале Azure](../articles/storage/common/storage-auth-aad-rbac.md). 
> 
> - [Обозреватель службы хранилища Azure](https://azure.microsoft.com/features/storage-explorer/) использует ключ учетной записи хранения для доступа к данным больших двоичных объектов и очередей. Если ключ не доступен, проверки подлинности Azure AD используется для доступа к большим двоичным объектам. Авторизация Azure AD в настоящее время не поддерживается для очередей. 
>
> - Служба файлов Azure поддерживает проверку подлинности с использованием Azure AD по протоколу SMB только для присоединенных к домену виртуальных машин (предварительная версия). Общие сведения о проверке подлинности с использованием Azure AD по протоколу SMB для службы файлов Azure см. в разделе [Обзор проверки подлинности Azure Active Directory по протоколу SMB для файлов Azure (предварительная версия)](../articles/storage/files/storage-files-active-directory-overview.md).

Служба хранилища Azure предлагает три типа учетных записей хранения. Каждый тип поддерживает различные возможности, и у каждого из них своя модель ценообразования. Прежде чем создавать учетную запись хранения, рассмотрите эти различия, чтобы определить подходящий для ваших приложений тип учетной записи. Три типа учетных записей хранения:

* **Учетные записи общего назначения версии 2.** Основной тип учетной записи хранения для больших двоичных объектов, файлов, очередей и таблиц. Рекомендуется для большинства сценариев, в которых используется служба хранилища Azure.
* **Учетные записи общего назначения версии 1.** Устаревший тип учетной записи хранения для больших двоичных объектов, файлов, очередей и таблиц. Вместо этих учетных записей используйте учетные записи общего назначения версии 2, если возможно.
* **Учетные записи хранения больших двоичных объектов**. Учетные записи хранения только для больших двоичных объектов. Вместо этих учетных записей используйте учетные записи общего назначения версии 2, если возможно. 

В следующей таблице описаны типы учетных записей хранения и их возможности:

| Тип учетной записи хранения | Поддерживаемые службы                       | Поддерживаемые уровни производительности | Поддерживаемые уровни доступа               | Варианты репликации                                                | Модель развертывания<sup>1</sup>  | Шифрование<sup>2</sup> |
|----------------------|------------------------------------------|-----------------------------|--------------------------------------|--------------------------------------------------------------------|-------------------|------------|
| Общего назначения версии 2   | Большой двоичный объект, файл, очередь, таблица и диск       | "Стандартный", "Премиум"           | Горячий, холодный, архивный<sup>3</sup> | LRS, ZRS<sup>4</sup>, GRS, RA-GRS | Диспетчер ресурсов | зашифрованные;  |
| Общего назначения версии 1   | Большой двоичный объект, файл, очередь, таблица и диск       | "Стандартный", "Премиум"           | Недоступно                                  | LRS, GRS, RA-GRS                                                   | Resource Manager, классическая  | зашифрованные;  |
| Хранилище BLOB-объектов         | Большой двоичный объект (блочные и добавочные BLOB-объекты) | Стандартная                    | Горячий, холодный, архивный<sup>3</sup>                            | LRS, GRS, RA-GRS                                                   | Диспетчер ресурсов  | зашифрованные;  |

<sup>1</sup> Рекомендуется использовать модель развертывания с помощью Azure Resource Manager. Учетные записи хранения, в которых используется классическая модель развертывания, по-прежнему можно создавать в некоторых расположениях, а имеющиеся классические учетные записи по-прежнему поддерживаются. Дополнительные сведения см. в статье [Развертывание с помощью Azure Resource Manager и классическое развертывание: сведения о моделях развертывания и состоянии ресурсов](../articles/azure-resource-manager/resource-manager-deployment-model.md).

<sup>2</sup> Все учетные записи хранения шифруются с помощью Шифрования службы хранилища (SSE) для неактивных данных. Дополнительные сведения см. в статье [Шифрование службы хранилища Azure для неактивных данных (предварительная версия)](../articles/storage/common/storage-service-encryption.md).

<sup>3</sup> Архивный уровень доступен только на уровне отдельного большого двоичного объекта, а не на уровне учетной записи хранения. Заархивировать можно только блочные и добавочные BLOB-объекты. Дополнительные сведения см. в статье [Хранилище BLOB-объектов Azure: горячий, холодный, архивный уровни хранилища и уровень "Премиум" (предварительная версия)](../articles/storage/blobs/storage-blob-storage-tiers.md).

<sup>4</sup> хранилище, избыточное между зонами, (ZRS) доступно только для стандартных учетных записей хранения общего назначения версии 2. Дополнительные сведения о ZRS см. в разделе статьи [Хранилище, избыточное между зонами (ZRS). Высокодоступные приложения для службы хранилища Azure](../articles/storage/common/storage-redundancy-zrs.md). Дополнительные сведения о других вариантах репликации см. в статье [Репликация службы хранилища Azure](../articles/storage/common/storage-redundancy.md).

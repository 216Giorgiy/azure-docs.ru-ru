Шлюз виртуальной сети использует определенную подсеть с именем GatewaySubnet. Подсеть шлюза содержит IP-адреса, которые используются службами VPN-шлюза. При создании подсеть шлюза нужно назвать GatewaySubnet.  Имя подсети GatewaySubnet указывает Azure, где нужно создавать службы шлюза. Если вы используете другое имя, VPN-шлюз не будет настроен.

IP-адреса в подсети GatewaySubnet выделяются службе шлюза. При создании подсети GatewaySubnet нужно указать число IP-адресов, которое содержит подсеть. Размер указанной подсети GatewaySubnet зависит от конфигурации VPN-шлюза, которую вы хотите создать. Хотя можно создать подсеть шлюза с размером /29, рекомендуется, чтобы подсеть была большего размера. Это позволит вам включить несколько адресов, выбрав значение /27 или /28. Использование более крупной подсети для шлюза позволяет создавать достаточное число IP-адресов с учетом возможных конфигураций в будущем.

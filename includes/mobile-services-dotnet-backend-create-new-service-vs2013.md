

Выполните описанные ниже действия, чтобы создать новую мобильную службу в Azure и добавить в проект код, подключающий к ней приложение. Visual Studio 2013 подключается к Azure от вашего имени для создания мобильной службы с помощью предоставленных вами учетных данных. При создании новой мобильной службы необходимо указать базу данных SQL Azure, которую мобильная служба будет использовать для хранения данных приложения. 

1. В Visual Studio 2013 откройте обозреватель решений, щелкните правой кнопкой мыши проект приложения Магазина Windows, выберите пункт **Добавить** и щелкните **Подключенная служба...**. 

2. В диалоговом окне диспетчера служб щелкните **Создать службу...** и в разделе **Подписка** диалогового окна "Создание мобильной службы" выберите **&lt;Управление...**&lt;.  

	![create service manage subscriptions](./media/mobile-services-dotnet-backend-create-new-service-vs2013/mobile-create-service-from-vs2013.png)

3. В окне "Управление подписками Microsoft Azure" щелкните **Вход...**, чтобы выполнить вход в учетную запись Azure (если необходимо). Затем выберите доступную подписку и нажмите кнопку **Закрыть**.

	Если в вашей подписке уже есть одна или несколько существующих мобильных служб, отображаются имена служб. 

5. Вернитесь в диалоговое окно **Создание мобильной службы**, выберите свою **подписку**, серверную часть **.NET Framework** в списке **Среда выполнения** и нужный **регион** для мобильной службы, после чего введите **имя** мобильной службы.

	>[AZURE.NOTE]Имена мобильных служб должны быть уникальными. Если указанное имя не доступно, рядом с полем **Имя** отобразится красный крестик. 

6. В разделе **База данных** выберите ;Создать бесплатную базу данных SQL;**, заполните поля **Имя пользователя сервера**, **Пароль сервера** и **Подтверждение пароля сервера**, затем нажмите кнопку **Создать**.

  	![create new mobile service in VS 2013](./media/mobile-services-dotnet-backend-create-new-service-vs2013/mobile-create-service-from-vs2013-2.png)

	> [AZURE.NOTE]
	> В рамках этого учебника создается новый экземпляр бесплатной базы данных SQL и сервер. Эту новую базу данных можно потом использовать и администрировать ее, как и любой другой экземпляр базы данных SQL. Может существовать только один бесплатный экземпляр базы данных. Если в этом регионе у вас уже имеется база данных в качестве новой мобильной службы, вы можете выбрать существующую базу данных. Если вы выбираете существующую базу данных, убедитесь, что предоставлены правильные учетные данные. При вводе неверных учетных данных мобильная служба создается в неработоспособном состоянии.

7. После создания мобильной службы выберите ее из списка в диспетчере служб и нажмите кнопку **ОК**.
 
   	После завершения работы мастера проект мобильной службы добавляется в решение, устанавливаются требуемые пакеты NuGet, в проект добавляется ссылка на клиентскую библиотеку мобильных служб, а исходный код проекта обновляется.

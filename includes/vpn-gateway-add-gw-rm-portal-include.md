1. На портале последовательно выберите элементы **Создать** > **Сети** > **Шлюз виртуальной сети**. Откроется колонка **Создание шлюза виртуальной сети**.

	![Шлюз](./media/vpn-gateway-add-gw-rm-portal-include/creategw250.png)

2. В колонке **Создание шлюза виртуальной сети** в поле **Имя** укажите имя шлюза. Это имя должно отличаться от имени подсети шлюза. Это имя объекта шлюза, который вы планируете создать.

3. В поле **Расположение** укажите расположение виртуальной сети. Если этого не сделать, ваша сеть не будет отображаться в списке виртуальных сетей.
 
4. Затем выберите виртуальную сеть, в которую вы хотите добавить этот шлюз. Щелкните элемент **Виртуальная сеть**, чтобы открыть колонку **Выбор виртуальной сети**. Выберите виртуальную сеть. Чтобы виртуальная сеть появилась в списке, она должна иметь допустимую подсеть шлюза.

5. Выберите общедоступный IP-адрес. Щелкните элемент **Общедоступный IP-адрес**, чтобы открыть колонку **Выбрать общедоступный IP-адрес**. Щелкните **+Создать**, чтобы открыть колонку **Create public IP address blade** (Создание общедоступного IP-адреса). Введите имя общедоступного IP-адреса. Будет создан объект общедоступного IP-адреса, которому динамически назначается общедоступный IP-адрес. <br>Нажмите кнопку **ОК**, чтобы сохранить изменения.

5. В поле **Тип шлюза** выберите тип шлюза, который указан для конфигурации.

6. В поле **Тип VPN** выберите тип VPN, который указан для конфигурации.

7. Проследите, чтобы в поле **Подписка** отображалась правильная подписка.

8. Значение параметра **Группа ресурсов** зависит от выбранной виртуальной сети.

9. Не изменяйте сведения в поле **Расположение** после того, как будут указаны перечисленные выше параметры.

10. На этом этапе вид колонки будет таким же, как и в схеме на шаге 1. Убедитесь, что параметры соответствуют параметрам вашей конфигурации. В нижней части колонки можно выбрать параметр **Закрепить на панели мониторинга**, если необходимо, чтобы шлюз отображался на панели мониторинга.

11. Щелкните **Создать**, чтобы создать шлюз. Параметры будут проверены, и на панели мониторинга появится плитка Deploying Virtual network gateway (Развертывание шлюза виртуальной сети). Создание шлюза может занять до 45 минут. Вам понадобится обновить страницу портала, чтобы увидеть готовое состояние.

	.![Шлюз](./media/vpn-gateway-add-gw-rm-portal-include/deployvnetgw150.png)

11. После создания шлюза вы можете просмотреть назначенный ему IP-адрес в разделе сведений о виртуальной сети на портале. Шлюз будет отображаться как подключенное устройство. Щелкните подключенное устройство (шлюз виртуальной сети), чтобы получить дополнительные сведения.

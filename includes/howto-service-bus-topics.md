<h2><a name="what-are-service-bus-topics"></a>Что такое разделы и подписки Service Bus</h2>

Разделы Service Bus и подписки поддерживают модель **обмена сообщениями публикации/подписки**. При использовании разделов и подписок компоненты распределенного приложения не взаимодействуют между собой напрямую, а обмениваются сообщениями через раздел, который выступает в качестве посредника.

![TopicConcepts](. / media/howto-service-bus-topics/sb-topics-01.png)

В отличие от очередей Service Bus, где каждое сообщение обрабатывается одним потребителем, разделы и подписки предоставляют вид связи **один ко многим** с помощью шаблона публикации/подписки. Можно зарегистрировать несколько подписок на раздел. Когда сообщение отправляется в раздел, оно затем может обрабатываться независимо каждой подпиской.

Раздел подписки напоминает виртуальную очередь, которая получает копии сообщений, отправленных в раздел. Вы можете по желанию зарегистрировать правила фильтрации для раздела на основе подписки, которые позволят указать или ограничить, какие сообщения и от каких подписок могут быть получены разделом.

Разделы и подписки Service Bus позволяют масштабировать для обработки очень большого количества сообщений для очень большого количества пользователей и приложений.

<h2><a name="create-a-service-namespace"></a>Создание пространства имен службы</h2>

Чтобы начать использование разделов и подписок Service Bus в Windows Azure, необходимо сначала создать пространство имен службы. Это пространство имен службы предоставляет область ограничений для адресации ресурсов Service Bus в вашем приложении.

Создание пространства имен службы:

1.  Выполните вход на [портал управления Windows Azure.][]

2.  В левой области навигации панели управления нажмите на кнопку **Service Bus**.

3.  В нижней части портала управления нажмите на кнопку **Создать**.   
    ![][0]

4.  В диалоговом окне **Добавить новое пространство имен** введите имя пространства имен.
    Система немедленно проверяет, доступно ли оно.   
    ![][2]

5.  После того как убедитесь, что имя пространства имен доступно, выберите страну или регион, в котором будет размещаться пространство имен (убедитесь, что используется та же страна или регион, где развертываются ресурсы среды выполнения приложений).

	ВНИМАНИЕ. Выберите **тот же регион**, который вы собираетесь выбрать для развертывания приложения. Это даст лучшую производительность.

6. 	Отметьте флажок. Теперь система создает пространство имен службы и включает его. Возможно, вам придется подождать несколько минут, пока система выделит ресурсы для вашей учетной записи.

	![][6]


<h2><a name="obtain-default-credentials"></a>Получение учетных данных управления по умолчанию для пространства имен</h2>

Для выполнения операций управления, таких как создание раздела или подписки, над новым пространством имен, необходимо получить учетные данные управления для пространства имен.

1.  В левой области навигационной панели щелкните узел **Service Bus**, чтобы отобразить список доступных пространств имен:   
    ![][0]

2.  Выберите из отображенного списка пространство имен, которое вы только что создали.   
    ![][3]

3.  Нажмите кнопку **Сведения о подключении**   
    ![][4]

4.  В диалоговом окне **Сведения по доступу к подключению** найдите операции **Издатель по умолчанию** и **Ключ по умолчанию**. Запишите эти значения, так как эти сведения будут использованы ниже для выполнения операций с пространством имен.

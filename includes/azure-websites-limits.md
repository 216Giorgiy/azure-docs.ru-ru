<sup>1</sup>Квоты на веб-сайты и системы хранения приводятся для каждого плана размещения веб-сайтов, если не указано иное.

<sup>2</sup>Сайты Basic и Standard размещаются на выделенных машинах. Фактическое количество сайтов, которое может разместить клиент, определяется интенсивностью их использования, размером экземпляров виртуальных машин и соответствующим использованием аппаратных ресурсов.

<sup>3</sup>За исключением уровня Free, ограничения, имеющиеся в тарифных планах на размещение веб-сайтов, распространяются на каждую группу ресурсов. С более подробной информацией о создании и управлении группами ресурсов можно ознакомиться в разделе [Применение групп ресурсов для управления ресурсами Azure][Применение групп ресурсов для управления ресурсами Azure]. Обратите внимание, что для уровня Standard по запросу доступны дополнительные планы по размещению веб-сайтов.

<sup>4</sup>Для выделенных экземпляров предусмотрены следующие размеры: Малый, Средний и Крупный. Более подробная информация приводится в разделе [Цены на веб-сайты Azure][Цены на веб-сайты Azure]. Дополнительное количество экземпляров можно получить, отправив запрос технической поддержки.

<sup>5</sup>Для информационных материалов всех веб-сайтов используются одни и те же ресурсы системы хранения. Поэтому ограничения, касающиеся системы хранения, определяются совокупными потребностями всех веб-сайтов, а не каждым из них в отдельности. Ограничения на систему хранения можно увеличить, отправив запрос технической поддержки. 

<sup>6</sup>Хотя квоты на использование процессорного времени и оперативной памяти не распространяются на уровни Basic и Standard, эти ресурсы ограничиваются аппаратными ресурсами, доступными для выделенных экземпляров, что определяется размером этих экземпляров и их количеством.

<sup>7</sup>Предельный объем памяти для уровня Free выделяется для всех бесплатных веб-сайтов, входящих в данный план на размещение. Предельный объем памяти на уровне Shared выделяется для каждого веб-сайта в отдельности.

<sup>8</sup>Данная величина равняется количеству одновременных подключений к веб-сокету для каждого экземпляра веб-сайта. Например, если вы отмасштабируете веб-сайт уровня Basic на два экземпляра, фактически вы получите 700 одновременных подключений (350 x 2).

<sup>9</sup>Для уровня Basic действуют стандартные [расценки на SSL-подключения][расценки на SSL-подключения].

<sup>10</sup>Запуск специальных программных приложений и/или скриптов по требованию, по расписанию или на постоянной основе в качестве фоновых задач в рамках вашего экземпляра веб-сайта. Для обеспечения непрерывной работы веб-заданий требуется установка опции "Всегда включено". Для обеспечения работы веб-заданий по расписанию на уровнях Free или Standard требуется использование планировщика Azure.

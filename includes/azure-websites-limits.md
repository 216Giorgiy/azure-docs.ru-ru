<sup>1</sup>Квоты на количество веб-сайтов и на объем хранилища выделяются в соответствии с планом веб-хостинга, если не указано иное.

<sup>2</sup>Сайты уровня Basic и Standard выполняются на выделенных машинах. Фактическое количество сайтов, которое может разместить клиент, определяется интенсивностью их использования, размером экземпляров виртуальных машин и соответствующим использованием аппаратных ресурсов.

<sup>3</sup>За исключением уровня Free, ограничения на планы веб-хостинга приводятся в расчете на одну группу ресурсов. С более подробной информацией о создании и управлении группами ресурсов можно ознакомиться в разделе [Применение групп ресурсов для управления ресурсами Azure][Применение групп ресурсов для управления ресурсами Azure]. Обратите внимание, что для уровня Standard по запросу доступны дополнительные планы по размещению веб-сайтов.

<sup>4</sup>Выделенные экземпляры могут иметь малый, средний или крупный размер. Более подробная информация приводится в разделе [Цены на веб-сайты Azure][Цены на веб-сайты Azure]. Дополнительное количество экземпляров можно получить, отправив запрос технической поддержки.

<sup>5</sup>Все веб-сайты совместно используют одни и те же ресурсы хранилища для содержимого сайтов. Поэтому ограничения, касающиеся системы хранения, определяются совокупными потребностями всех веб-сайтов, а не каждым из них в отдельности. Ограничения на систему хранения можно увеличить, отправив запрос технической поддержки. 

<sup>6</sup>Для уровней Basic и Standard квота на использование ЦП и памяти не задается, но эти ресурсы ограничиваются физическими ресурсами на выделенных экземплярах (размером и количеством экземпляров).

<sup>7</sup>Предельный объем памяти на уровне Free устанавливается сразу для всех веб-сайтов в плане веб-хостинга. Предельный объем памяти на уровне Shared выделяется для каждого веб-сайта в отдельности.

<sup>8</sup>Это значение представляет собой количество параллельных подключений веб-сокетов на один экземпляр веб-сайта. Например, если вы отмасштабируете веб-сайт уровня Basic на два экземпляра, фактически вы получите 700 одновременных подключений (350 x 2).

<sup>9</sup>Для уровня Basic применяются [стандартные расценки на SSL][стандартные расценки на SSL].

<sup>10</sup>Запускает пользовательские исполняемые файлы и (или) скрипты по запросу, по расписанию или непрерывно как фоновую задачу в вашем экземпляре веб-сайтов. Для обеспечения непрерывной работы веб-заданий требуется установка опции "Всегда включено". Для обеспечения работы веб-заданий по расписанию на уровнях Free или Standard требуется использование планировщика Azure.

#Подключение к виртуальным машинам в облачной службе



При создании виртуальной машины автоматически создается облачная служба. В одной облачной службе можно создать несколько виртуальных машин, что позволит виртуальным машинам подключаться друг к другу, выполнять балансировку нагрузки между виртуальными машинами и поддерживать высокую доступность машин. 

Дополнительные сведения о балансировке нагрузки виртуальных машин см. в разделе [Балансировка нагрузки виртуальных машин](../load-balancing-vms/). Дополнительные сведения об управлении доступностью приложения см. в разделе [Управление доступностью виртуальных машин](../manage-vm-availability/). 


Во-первых, необходимо создать виртуальную машину в новой облачной службе, а затем к первой виртуальной машине можно подключить дополнительные виртуальные машины в той же облачной службе. 



1. Создание виртуальной машины с помощью действий см. в разделе [Как создать настраиваемую виртуальную машину](../howto-custom-create-vm/).


2. После создания первой настраиваемой виртуальной машины нажмите на кнопку **Создать** на панели команд [портала управления](http://manage.windowsazure.com)


	![Создание новой виртуальной машины](./media/howto-connect-vm-cloud-service/Create.png)

3. Нажмите кнопку **Виртуальная машина**, а затем нажмите **Из коллекции**.

	
	![Создание настраиваемой виртуальной машины](./media/howto-connect-vm-cloud-service/CreateNew.png)

	Появится диалоговое окно **Выберите операционную систему для виртуальной машины**. 


4. Выберите изображение на странице **Выберите образ** и нажмите кнопку со стрелкой, чтобы продолжить.


	Появится первая страница **Конфигурация виртуальной машины**.


5. В поле **Имя виртуальной машины** введите имя, которое вы хотите использовать для виртуальной машины.

6. В поле **Размер** выберите размер, который вы хотите использовать для виртуальной машины. Выбранный размер зависит от числа ядер, которые необходимы для приложения.

7. В поле **Новое имя пользователя** введите имя учетной записи администратора, которую требуется использовать для управления сервером.


8. В поле **Новый пароль** введите надежный пароль для учетной записи администратора. В поле **Подтвердите пароль** введите пароль еще раз.


9. Виртуальную машину под управлением операционной системы Linux можно защитить с помощью ключа SSH.


10. В поле **Облачный сервис** выберите облачный сервис, который будет содержать новую виртуальную машину.

11. В поле **Учетная запись хранения** выберите учетную запись хранения для хранения VHD-файла или оставьте значение поля по умолчанию для автоматического создания учетной записи хранения. Автоматически будет создана только одна учетная запись хранения. Все остальные виртуальные машины, созданные с помощью этого параметра, находятся в этой учетной записи хранения. Вы можете иметь не более 20 учетных записей хранения.


12. Чтобы использовать группу доступности, выберите ту, что была создана при создании первой виртуальной машины.

13. Проверьте конфигурацию конечной точки по умолчанию и при необходимости измените ее. 

14. Чтобы создать подключенную виртуальную машину, поставьте галочку.

#Подключение виртуальных машин с помощью виртуальной сети или облачной службы

Виртуальные машины должны находиться в облачной службе, которая выступает в качестве контейнера и предоставляет уникальное общее DNS-имя, общедоступный IP-адрес и набор конечных точек для доступа к виртуальной машине через Интернет. Облачная служба может находиться в виртуальной сети. 

Если облачная служба находится не в виртуальной сети, виртуальные машины в этой облачной службе могут взаимодействовать с другими виртуальными машинами только с помощью общих DNS-имен других виртуальных машин, а трафик проходит через Интернет. Если облачная служба находится в виртуальной сети, виртуальные машины в этой облачной службе могут взаимодействовать со всеми остальными виртуальными машинами в виртуальной сети без отправки трафика через Интернет.

Если разместить виртуальные машины в одной автономной облачной службе, можно использовать балансировку нагрузки и группы доступности. Дополнительные сведения см. в разделах [Балансировка нагрузки виртуальных машин](../../articles/load-balance-virtual-machines/) и [Управление доступностью виртуальных машин](../../articles/manage-availability-virtual-machines/). Тем не менее, нельзя упорядочить виртуальные машины в подсетях или подключить автономную облачную службу к локальной сети. Ниже приведен пример.

![Virtual machines in a standalone cloud service](./media/howto-connect-vm-cloud-service/CloudServiceExample.png)
 
Если разместить виртуальные машины в виртуальной сети, можно варьировать количество облачных служб, используемых для балансировки нагрузки и групп доступности. Кроме того, можно упорядочить виртуальные машины в подсетях таким же образом, как в локальной сети и подключить виртуальную сеть к локальной сети. Ниже приведен пример.

![Virtual machines in a virtual network](./media/howto-connect-vm-cloud-service/VirtualNetworkExample.png)

Виртуальные сети - рекомендуемый способ подключения виртуальных машин в Azure. Рекомендуется настраивать каждый уровень приложения в отдельной облачной службе. Это позволяет осуществлять расширенное делегирование прав пользователей с помощью системы управления доступом на основе ролей (RBAC). Дополнительные сведения см. в разделе [Управление доступом на основе ролей в предварительной версии портала Azure](../../articles//role-based-access-control-configure/). Вместе с тем, для соответствия максимальному количеству облачных служб для одной подписки (200 служб) может потребоваться объединить некоторые виртуальные машины из различных уровней приложения в одной облачной службе.

Для подключения виртуальных машин в виртуальной сети необходимо выполнить следующие действия.

1.	Создайте виртуальную сеть на портале управления Azure. Подробнее см. в разделе [Задачи конфигурации виртуальной сети](https://msdn.microsoft.com/library/azure/jj156206.aspx).
2.	Создайте набор облачных служб для вашего развертывания на портале управления Azure в соответствии со структурой групп доступности и балансировкой нагрузки, используя команду **Создать > Вычисления > Облачная служба > Настраиваемое создание**.
3.	При создании каждой новой виртуальной машины следует указать нужную облачную службу и виртуальную сеть. Если облачная служба была ранее связана с виртуальной сетью, ее имя уже будет выбрано.

Ниже приведен пример использования портала управления Azure.

![Selecting a cloud service for a virtual machine](./media/howto-connect-vm-cloud-service/VMConfig1.png)
 
Подключение виртуальных машин в автономной облачной службе.

1.	Создайте облачную службу для вашего развертывания на портале управления Azure с помощью команды **Создать > Вычисления > Облачная служба > Настраиваемое создание**.
2.	При создании виртуальной машины укажите имя облачной службы, созданной в шаге 1. 
Создать облачную службу для развертывания также можно в момент создания первой виртуальной машины.

1. Зарегистрируйте серверную часть мобильного приложения в клиенте Azure Active Directory, выполнив действия из статьи [Настройка мобильного приложения в Azure Active Directory].

2. Перейдите в **Active Directory** на [портале управления Azure].

   ![](./media/app-service-mobile-adal-register-app/app-service-navigate-aad.png)

3. Выберите нужный каталог и щелкните расположенную вверху вкладку **Приложения**. Чтобы зарегистрировать новое приложение, щелкните внизу **ДОБАВИТЬ**. 

4. Щелкните пункт **Добавить приложение, разрабатываемое моей организацией**.

5. В мастере добавления приложения укажите **имя** приложения и выберите тип **Собственное клиентское приложение**. Затем щелкните "Далее".

6. В поле **URI перенаправления** введите конечную точку /login/done для шлюза службы приложений. Это значение должно быть похоже на https://contoso.azurewebsites.net/login/done.

7. После добавления собственного приложения щелкните вкладку **Настройки**. Скопируйте **код клиента**. Он потребуется позднее.

8. Прокрутите страницу вниз до раздела **Разрешения для других приложений** и нажмите кнопку **Добавить приложение**.

9. Найдите ранее зарегистрированное веб-приложение и щелкните значок со знаком плюс. Затем установите флажок, чтобы закрыть диалоговое окно.

10. Для только что добавленной записи откройте раскрывающийся список **Делегированные разрешения** и выберите пункт **Доступ (имя приложения)**. Затем нажмите кнопку **Сохранить**.

   ![](./media/app-service-mobile-adal-register-app/aad-native-client-add-permissions.png)

Теперь ваше приложение настроено в AAD, и пользователи могут входить в него с помощью процедуры единого входа AAD.

### <a name="timeout-values"></a>Значения времени ожидания
#### <a name="client-to-front-door"></a>Клиент — Front Door
- Front Door использует время ожидания простоя TCP-подключения, равное 61 секунде.

#### <a name="front-door-to-application-back-end"></a>Входной двери для серверного приложения
- При ответе поблочное ответ 200 возвращается в том случае, если или при получении первого фрагмента данных.
- После HTTP-запросе пересылается к серверной части, входной двери ожидает в течение 30 секунд для первого пакета из серверной части. Затем он возвращает сообщение об ошибке 503 клиенту.
- После получения первого пакета из серверной части входной двери ожидает в течение 30 секунд на время ожидания простоя. Затем он возвращает сообщение об ошибке 503 клиенту.
- Входной двери для времени ожидания сеанса TCP серверной части — 30 минут.

### <a name="upload-and-download-data-limit"></a>Отправка и загрузка ограничение данных

|  | С помощью Поблочное кодирование (CTE) | Без фрагментации HTTP |
| ---- | ------- | ------- |
| **Загрузить** | Не ограничено на размер загружаемого файла. | Не ограничено на размер загружаемого файла. |
| **Передача** |  Не ограничено до тех пор, пока каждый отправленный обобщенное табличное Выражение является менее 2 ГБ. | Размер не может превышать 2 ГБ. |

### <a name="premium-performance-block-blob-storage"></a>Хранилище BLOB-объектов производительности уровня "премиум"

Учетную запись хранения BLOB-объектов уровня "премиум" производительности блок оптимизирован для приложений, использующих меньшего размера диапазона килобайт, объекты. Это идеальный вариант для приложений, требующих большом количестве транзакций или согласованного хранилища низкой задержкой. Хранилище BLOB-объектов производительности уровня "премиум" предназначена для масштабирования с приложениями. Если вы планируете развертывать приложения, требующие сотни тысяч запросов в секунду и петабайты пространства для емкости хранилища, обращайтесь к нам, отправив запрос на поддержку в [портала Azure](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade).

### <a name="premium-performance-filestorage-preview"></a>Производительность уровня "премиум" хранилище файлов (Предварительная версия)

Файлы "премиум" используют учетная запись хранения уникальный **хранилища (Предварительная версия)**. Этот тип учетной записи предназначена для рабочих нагрузок с высокой операций ввода-ВЫВОДА, высокой пропускной способностью согласованные низкой задержкой. Хранилище файлов уровня "премиум" масштабирование осуществляется на основе размера подготовленного общего ресурса.

|Область  |Цель  |
|---------|---------|
|Размер подготовленных max     |5 Тиб (Предварительная версия), 100 Tиб (ограниченной общедоступной предварительной версии)     |
|Общие папки   |Без ограничений  |
|ОПЕРАЦИЙ ВВОДА-ВЫВОДА     |100 000 (ограниченной общедоступной предварительной версии)    |
|Входящие|Расположенных в 4136 Миб/с     |
|Исходящие|6,204 Миб/с |

 Для уровня "премиум" файла совместно использовать целевые показатели масштабируемости, см. в разделе [файлы "премиум" масштабировать целевых объектов](../articles/storage/common/storage-scalability-targets.md#premium-files-scale-targets) раздел.

### <a name="premium-performance-page-blob-storage"></a>Хранилище BLOB-объектов страницы производительности уровня "премиум"

Производительность уровня "премиум", общего назначения версии 1 или учетные записи хранения версии 2 имеют следующие целевые показатели масштабируемости:

| Общая емкость учетной записи                            | Общая пропускная способность для учетной записи локально избыточного хранилища                     |
| ------------------------------------------------- | --------------------------------------------------------------------------- |
| Емкость диска: 35 ТБ <br>Емкость для моментальных снимков: 10 ТБ | До 50 гигабит в секунду для входящих<sup>1</sup> и исходящих подключений<sup>2</sup> |

<sup>1</sup> Все данные (запросы), отправляемые в учетную запись хранилища.

<sup>2</sup> Все данные (отклики), получаемые из учетной записи хранения.

Если вы используете учетные записи хранения производительности premium для неуправляемых дисков и потребности вашего приложения превышают целевые показатели масштабируемости для одной учетной записью, может потребоваться переход на управляемые диски. Если этот вариант вам не подходит, тогда при создании приложения настройте его на использование нескольких учетных записей хранения и распределите данные между этими учетными записями. Например, если необходимо подключить диски объемом 51 ТБ к нескольким виртуальным машинам, распределите их между двумя учетными записями хранения. Максимальный размер, поддерживаемый одной учетной записью хранилища класса Premium, составляет 35 ТБ. Убедитесь, что учетная запись хранения производительности одного уровня "премиум" никогда не имеет подготовленных дисков объемом более 35 ТБ.

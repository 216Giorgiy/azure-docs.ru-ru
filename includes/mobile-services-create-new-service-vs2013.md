С помощью следующих шагов создаются новые мобильные службы в Azure и в ваш проект добавляется код, разрешающий доступ к этой новой службе. Перед созданием мобильной службы необходимо импортировать в Visual Studio файл publishsettings из подписки Azure. Это позволяет Visual Studio подключиться к Azure от вашего имени. При создании новой мобильной службы необходимо указать базу данных SQL Azure, которую мобильная служба будет использовать для хранения данных приложения.

1.  В Visual Studio 2013 откройте обозреватель решений, щелкните правой кнопкой проект, затем выберите команду **Добавить** и **Подключенная служба...**

    ![добавление подключенной службы][]

2.  В диалоговом окне диспетчера служб нажмите **Создать службу...**, выберите **\<Импорт...\>** в разделе **Подписка** в диалоговом окне «Создание мобильной службы».

    ![создание новой мобильной службы из VS 2013][]

3.  В подписках Azure для импорта нажмите **Скачать файл подписки**, войдите в учетную запись Azure (если требуется), щелкните кнопку **Сохранить**, когда обозреватель запросит сохранить файл.

    ![загрузка файла подписки в VS][]

    <div class="dev-callout"><strong>Примечание.</strong> <p>Окно входа в систему отображается в браузере, который может находиться за окном Visual Studio. Запишите себе, где вы сохранили загруженный файл .publishsettings. Этот шаг можно пропустить, если проект уже подключен к вашей подписке Azure.</p></div>

4.  Нажмите **Обзор**, найдите расположение, где вы сохранили файл .publishsettings, выберите его, затем нажмите **Открыть** и **Импортировать**.

    ![импорт подписки в VS][]

    Visual Studio импортирует данные, необходимые для подключения к вашей подписке Azure. Если в вашей подписке уже есть одна или несколько существующих мобильных служб, отображаются имена служб.

    <div class="dev-callout"><strong>Примечание о безопасности.</strong> <p>После импорта настроек публикации удалите скачанный PUBLISHSETTINGS-файл, поскольку он содержит информацию, которая может использоваться другими пользователями для доступа к вашей учетной записи. Защитите файл, если планируете сохранить его для использования в других подключенных проектах приложений.</p></div>

5.  Вернитесь в диалоговое окно **Создание мобильной службы**, выберите свою **подписку** и нужный **регион** для вашей мобильной службы и введите **имя** мобильной службы.

    <div class="dev-callout"><strong>Примечание.</strong> <p>Имена мобильных служб должны быть уникальными. Если указанное имя не доступно, рядом с полем <strong>Имя</strong> отобразится красный крестик. </p></div>

6.  В разделе **База данных** выберите **\<Создать бесплатную базу данных SQL\>**, заполните поля **Имя пользователя сервера**, **Пароль сервера** и **Подтверждение пароля сервера**, затем нажмите **Создать**.

    ![создание службы из VS 2013 часть 2][]

    > [WACOM.NOTE]
    > В рамках данного учебника вы создадите новый бесплатный экземпляр базы данных SQL и сервер. Эту новую базу данных можно потом использовать и администрировать ее, как и любой другой экземпляр базы данных SQL. Может существовать только один бесплатный экземпляр базы данных. Если в этом регионе у вас уже имеется база данных в качестве новой мобильной службы, вы можете выбрать существующую базу данных. Если вы выбираете существующую базу данных, убедитесь, что предоставлены правильные учетные данные. При вводе неверных учетных данных мобильная служба создается в неработоспособном состоянии.

    После создания мобильной службы в клиентскую библиотеку мобильных служб добавляется ссылка на проект, исходный код проекта обновляется.

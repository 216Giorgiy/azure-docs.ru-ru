

Выполните описанные ниже действия, чтобы создать новую мобильную службу в Azure и добавить в проект код, подключающий к ней приложение. Visual Studio 2013 подключается к Azure от вашего имени для создания мобильной службы с помощью предоставленных вами учетных данных. При создании новой мобильной службы необходимо указать базу данных SQL Azure, которую мобильная служба будет использовать для хранения данных приложения. 


1. В среде Visual Studio 2013 откройте обозреватель решений, щелкните правой кнопкой мыши проект приложения магазина Windows, выберите пункт **Добавить**, а затем **Подключенная служба...**.  

2. В диалоговом окне диспетчера служб нажмите **Создать службу...**, выберите **Управление...** из **Подписки** в диалоговом окне "Создание мобильной службы".  

	![create service manage subscriptions](./media/mobile-services-create-new-service-vs2013/mobile-create-service-from-vs2013.png)

3. В окне "Управление подписками Microsoft Azure" щелкните **Вход...**, чтобы выполнить вход в учетную запись Azure (если необходимо). Затем выберите доступную подписку и нажмите кнопку **Закрыть**.

	Если в подписке уже есть одна или несколько существующих мобильных служб, то отображаются имена служб. 

5. В диалоговом окне **Создание мобильной службы** выберите **Подписка**, серверную часть **JavaScript** в **среде выполнения** и **Область** для мобильной службы, затем введите **Имя** для мобильной службы.

	>[WACOM.NOTE]Имена мобильных служб должны быть уникальными. Если указанное имя не доступно, рядом с полем **Имя** отобразится красный крестик. 

6. В разделе **База данных** выберите **Создать бесплатную базу данных SQL**, заполните поля **Имя пользователя сервера**, **Пароль сервера** и **Подтверждение пароля сервера**, затем нажмите **Создать**.

  	![create new mobile service in VS 2013](./media/mobile-services-create-new-service-vs2013/mobile-create-service-from-vs2013-2.png)


	> [WACOM.NOTE]
	>  В рамках данного учебника вы создадите новый бесплатный экземпляр базы данных SQL и сервер. Эту новую базу данных можно потом использовать и администрировать ее, как и любой другой экземпляр базы данных SQL. Может существовать только один бесплатный экземпляр базы данных. Если в этом регионе у вас уже имеется база данных в качестве новой мобильной службы, вы можете выбрать существующую базу данных. Если вы выбираете существующую базу данных, убедитесь, что предоставлены правильные учетные данные. При вводе неверных учетных данных мобильная служба создается в неработоспособном состоянии.

7. После создания мобильной службы выберите ее в списке в диспетчере служб и нажмите кнопку **OK**.

	После создания мобильной службы в клиентскую библиотеку мобильных служб добавляется ссылка на проект, исходный код проекта обновляется.

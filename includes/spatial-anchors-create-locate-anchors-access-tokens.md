### <a name="access-tokens"></a>Маркеры доступа

Маркеры доступа являются более надежным методом для проверки подлинности для привязки пространственных Azure. Особенно в том случае, когда вам подготовить приложение для развертывания в рабочей среде. Сводка этого подхода состоит в том, для настройки внутренней службы, который клиентское приложение может безопасной проверки подлинности с помощью. Интерфейсов внутренней службе с помощью AAD во время выполнения и службы пространственных маркеров привязки безопасности Azure, для запроса на токен доступа. Затем этот токен доставляется в клиентское приложение и используемые в пакете SDK для проверки подлинности Azure пространственных привязки.

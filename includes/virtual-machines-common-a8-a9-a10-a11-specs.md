
## Основные возможности

* **Высокопроизводительное оборудование**. Эти экземпляры разработаны и оптимизированы для ресурсоемких приложений, в том числе высокопроизводительных вычислительных приложений (HPC) и приложений пакетной службы, моделирования и крупномасштабных симуляторов.

    Основные характеристики, сведения о дисках и объеме памяти см. в статье [Размеры виртуальных машин в Azure](virtual-machines-linux-sizes.md). Информацию о процессорах Intel Xeon E5-2667 v3 (используемых в серии H) и Intel Xeon E5-2670 (в сериях A8–A11), включая поддерживаемые расширения набора инструкций, можно найти на веб-сайте Intel.com.

* **Предназначены для кластеров HPC**. Чтобы создать автономный кластер HPC или увеличить емкость локального кластера, в Azure можно развернуть несколько экземпляров для ресурсоемких вычислений. При желании можно развернуть средства управления кластером и планирования заданий или использовать экземпляры для ресурсоемких вычислений в другой службе Azure (например, в пакетной службе Azure).

* **Сетевое подключение RDMA для приложений MPI**. Подмножество экземпляров для ресурсоемких вычислений (H16r, H16mr, A8 и A9) представляет собой второй сетевой интерфейс для подключения с удаленным доступом к памяти (RDMA). Этот интерфейс является дополнением к стандартному сетевому интерфейсу Azure, который доступен для виртуальных машин других размеров.

    Этот интерфейс обеспечивает связь между экземплярами с поддержкой RDMA при помощи сети InfiniBand, работающей на скорости FDR для виртуальных машин H16r и H16mr и на скорости QDR для виртуальных машин A8 и A9. Поддержка технологии RDMA в этих виртуальных машинах способствует повышению масштабируемости и производительности определенных приложений интерфейса передачи сообщений (MPI) Linux и Windows. Требования см. в разделе [Доступ к сети RDMA](#access-to-the-rdma-network) этой статьи.



## Рекомендации по развертыванию

* **Подписка Azure**. Если вы хотите развернуть большое количество экземпляров для ресурсоемких вычислений, рекомендуем подписку с оплатой по мере использования или другие варианты покупки. Если вы используете [бесплатную учетную запись Azure](https://azure.microsoft.com/free/), вам доступно ограниченное количество вычислительных ядер Azure.

* **Цены и доступность**. Виртуальные машины для ресурсоемкиих вычислений предлагаются только в стандартной ценовой категории. Проверьте [доступность продуктов по регионам](https://azure.microsoft.com/regions/services/), чтобы узнать, в каких регионах Azure их можно использовать.

* **Квота на ядра**. Возможно, вам потребуется увеличить квоту на ядра в подписке Azure, которая по умолчанию составляет 20 ядер на подписку (если используется классическая модель развертывания) или 20 ядер на регион (если используется модель развертывания Resource Manager). Кроме того, количество ядер, которые можно развернуть для некоторых семейств размеров виртуальных машин (включая серию H), может быть ограничено условиями вашей подписки. Чтобы увеличить квоту, [отправьте запрос в службу поддержки](../articles/azure-supportability/how-to-create-azure-support-request.md). Это бесплатная услуга. (Ограничения по умолчанию могут быть разными в зависимости от категории подписки).

    >[AZURE.NOTE]Если вам нужны ресурсы в очень большом объеме, обратитесь в службу поддержки Azure. Квоты Azure — это ограничения по кредитам, а не гарантированная емкость. Вне зависимости от квоты с вас будет взиматься плата только за используемые ядра.

* **Виртуальная сеть**. [Виртуальная сеть](https://azure.microsoft.com/documentation/services/virtual-network/) Azure не требуется для использования ресурсоемких экземпляров. Однако для многих сценариев развертывания вам может потребоваться облачная виртуальная сеть Azure или подключение типа "сеть — сеть", если нужен доступ к локальным ресурсам, например к серверу лицензий приложений. Создайте виртуальную сеть, чтобы развернуть экземпляры. Добавление виртуальных машин для ресурсоемких вычислений в виртуальную сеть в территориальной группе не поддерживается.

* **Облачная служба или группа доступности**. Для использования сети RDMA Azure виртуальные машины с поддержкой RDMA необходимо развертывать в той же облачной службе (если используется классическая модель развертывания) или в той же группе доступности (если используется модель развертывания с помощью Azure Resource Manager). При использовании пакетной службы Azure виртуальные машины с поддержкой RDMA должны находиться в одном пуле.

* **Изменение размера**. Из-за специального оборудования, используемого в экземплярах для ресурсоемких вычислений, изменить можно только размер экземпляров для ресурсоемких вычислений одного семейства размеров (серии H или серии A для ресурсоемких вычислений). Например, можно изменить только размер виртуальной машины серии H (один размер из серии H на другой размер из этой же серии). Кроме того, не поддерживается изменение размера экземпляра, не предназначенного для ресурсоемких вычислений, на размер экземпляра, предназначенного для таких вычислений.

* **Адресное пространство сети RDMA**. Сеть RDMA в Azure резервирует адресное пространство 172.16.0.0/16. Чтобы выполнять приложения MPI в экземплярах, развернутых в виртуальной сети Azure, убедитесь, что адресное пространство виртуальной сети не пересекается с сетью RDMA.

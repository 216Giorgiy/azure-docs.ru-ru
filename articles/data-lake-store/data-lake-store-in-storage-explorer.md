# <a name="manage-azure-data-lake-store-resources-with-storage-explorer-preview"></a>Управление ресурсами Azure Data Lake Store с помощью обозревателя службы хранилища (предварительная версия)

[Azure Data Lake Store (ADLS)](https://docs.microsoft.com/en-us/azure/data-lake-store/data-lake-store-overview) представляет собой службу для хранения больших объемов неструктурированных данных, например текстовых или двоичных. Пользователи могут получить доступ к данным из любого места через протоколы HTTP или HTTPS. Через обозреватель службы хранилища пользователи могут получать доступ к данным и ресурсам ADLS и другим сущностям Azure, например большим двоичным объектам и очередям, а управлять ими. Теперь пользователи могут использовать одно средство для управления разными сущностями Azure в одном месте.

Другое преимущество заключается в том, что пользователям не нужно разрешение в подписке для управления данными ADLS. В обозревателе службы хранилища пользователи могут добавить путь ADLS к узлу Local and Attached (Локальные и присоединенные), пока действует соответствующее разрешение.

## <a name="prerequisites"></a>предварительным требованиям
Чтобы выполнить действия, описанные в этой статье, необходимо следующее.

*   Подписка Azure. См. страницу [бесплатной пробной версии Azure](https://azure.microsoft.com/pricing/free-trial).
*   Учетная запись Azure Data Lake Store. Инструкции по созданию учетной записи см. в статье [Начало работы с Azure Data Lake Store с помощью портала Azure](https://docs.microsoft.com/en-us/azure/data-lake-store/data-lake-store-get-started-portal).

## <a name="installation"></a>Установка

Скачать новейшие версии обозревателя службы хранилища Azure можно отсюда: [обозреватель службы хранилища Azure](https://azure.microsoft.com/features/storage-explorer/). Теперь мы поддерживаем версии для Windows, Linux и MAC.

## <a name="connect-to-an-azure-subscription"></a>Подключение к подписке Azure

1. После установки **Обозревателя службы хранилища Azure** щелкните значок **подключаемого модуля** слева, как показано на следующем рисунке.
       
   ![Значок подключаемого модуля](./media/data-lake-store-in-storage-explorer/plug-in-icon.png)
 
2. Выберите **Добавить учетную запись Azure**, а затем щелкните **Войти**.

   ![Подключение к подписке Azure](./media/data-lake-store-in-storage-explorer/connect-to-azure-subscription.png)

2. В диалоговом окне **Вход в Azure** нажмите кнопку **Войти**, а затем введите свои учетные данные Azure.

    ![Вход](./media/data-lake-store-in-storage-explorer/sign-in.png)

3. Выберите свою подписку и нажмите кнопку **Применить**.

    ![Применить](./media/data-lake-store-in-storage-explorer/apply-subscription.png)

    На панели Обозревателя будут показаны учетные записи в выбранной подписке.

    ![Список учетных записей](./media/data-lake-store-in-storage-explorer/account-list.png)

    Вы успешно подключились к **Azure Data Lake Store** в подписке Azure.

## <a name="connect-to-data-lake-store"></a>Подключение к хранилищу озера данных
Если вы хотите получить доступ к ресурсам, которых нет в вашей подписке, но при этом у вас есть разрешение на получение URI для ресурсов, вы можете подключиться к Data Lake Store с помощью URI после входа в систему. Выполните действия ниже.
1. Откройте обозреватель хранилищ (предварительная версия).
2. В левой области разверните узел **Local and Attached** (Локальные и присоединенные).
3. Щелкните правой кнопкой мыши **Хранилище озера данных** и в контекстном меню выберите команду **подключения к Data Lake Store**.

      ![команда контекстного меню: подключение к Data Lake Store](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-uri-attach.png)

4. Введите URI, и ds перейдите в расположение введенного URL-адреса.

      ![диалоговое окно после выбора команды контекстного меню: подключение к Data Lake Store](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-uri-attach-dialog.png)

      ![результат подключения к Data Lake Store](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-attach-finish.png)

## <a name="view-an-azure-data-lake-store-accounts-contents"></a>Просмотр содержимого учетной записи Azure Data Lake Store
В ресурсах учетной записи Azure Data Lake Store содержатся файлы и папки.

В шагах ниже описывается, как просмотреть содержимое учетной записи ADLS в обозревателе службы хранилища (предварительная версия):

1. Откройте обозреватель хранилищ (предварительная версия).
2. В левой области разверните подписку, в которой содержится нужная учетная запись Azure Data Lake Store.
3. Разверните **Data Lake Store**.
4. Щелкните правой кнопкой мыши узел учетной записи Azure Data Lake Store, который необходимо просмотреть, и в контекстном меню выберите **Открыть**. Можно также дважды щелкнуть учетную запись ADLS, чтобы открыть ее. 
5. На главной панели отобразится содержимое учетной записи ADLS.

     ![главная панель](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-toolbar-mainpane.png) 

## <a name="azure-data-lake-store-resources-management"></a>Управление ресурсами Azure Data Lake Store

Вы можете управлять ресурсами Azure Data Lake Store. Для этого доступны следующие операции:
*   навигация по ресурсам ADLS в нескольких учетных записях ADL;  
*   использование строки подключения, чтобы напрямую подключиться к ADLS и управлять им; 
*   просмотр ресурсов ADLS, которые предоставляются в общий доступ другими, с помощью списка управления доступом (ACL) в узле "Local and Attached" (Локальные и присоединенные);
*   выполнение операций CRUD в папках и файлах: поддержка рекурсивной папки и множественного выбора файлов; 
*   перетаскивание, удаление и добавление папки для быстрого доступа к последним расположениям, что соответствует возможностям обозревателя рабочего стола; 
*   копирование и открытие ссылки ADL с помощью обозревателя службы хранилища Azure в один щелчок; 
*   отображение журнала действий в нижней правой области для просмотра состояния действия;
*   отображение статистики папки и свойства файла.

## <a name="manage-resources-in-azure-storage-explorer"></a>Управление ресурсами в обозревателе службы хранилища Azure
После создания учетной записи Azure Data Lake Store можно передавать папки и файлы, а также загружать и открывать ресурсы на локальном компьютере. Вы можете закрепить ее для быстрого доступа, создавать папки, копировать URL-адреса и выбирать все ресурсы. Кроме того, есть возможность копирования, вставки, переименования, удаления, просмотра статистики папки и обновления.

С помощью элементов ниже показано, как управлять ресурсами в учетной записи Azure Data Lake Store. Следуйте приведенным ниже инструкциям в зависимости от задачи, которую необходимо выполнить.
   * **Передача файлов**

     1. На панели инструментов в основной области нажмите кнопку **Отправить**, а затем выберите **Отправить файлы...** в раскрывающемся меню.

        ![Меню для отправки файлов](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-upload-files-menu.png) 

     2. В диалоговом окне **Select files to Upload** (Выбор файлов для отправки) выберите нужные файлы.

        ![Диалоговое окно передачи папок](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-upload-files-dialog.png)

     3. Выберите **Открыть**, чтобы начать отправку.
   * **Отправка папки**

     1. На панели инструментов в основной области нажмите кнопку **Отправить**, а затем выберите **Upload Folder...** (Отправить папку...) в раскрывающемся меню.

        ![Меню для отправки папок](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-upload-folder-menu.png) 
     2. В диалоговом окне **выбора папки для передачи** выберите нужную папку.

        ![Диалоговое окно передачи папок](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-upload-folder-dialog.png)      

     3. Выберите **Выбор папки**, чтобы начать отправку папок.

        ![Диалоговое окно передачи папок](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-upload-folder-drag.png) 

      > [!NOTE] 
          > Вы можете напрямую перетащить папки и файлы на локальный компьютер, чтобы начать передачу. 
       
   * **Загрузка папок или файлов на локальный компьютер**

     1. Выберите папки или файлы, которые вы хотите загрузить.
     2. На панели инструментов в основной области нажмите кнопку **Скачать**.
     3. В диалоговом окне **Select a folder to save the downloaded files into** (Выбор папки, куда нужно сохранить загруженные файлы) укажите расположение для сохранения папок или файлов. Также присвойте папке имя.
     4. Щелкните **Сохранить**.
   * **Открытие папки или файла с локального компьютера**

     1. Выберите папку или файл, которые необходимо открыть.
     2. На панели инструментов в основной области нажмите кнопку **Открыть** или щелкните выбранную папку или файл правой кнопкой мыши и в контекстном меню выберите **Открыть**.
     3. Файл будет скачан и открыт с помощью приложения, выбор которого зависит от базового типа файла. Папка же откроется в основной области.

        ![Открытие файла](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-open.png) 

   * **Копирование файлов и папок в буфер обмена**

     1. Выберите папки или файлы, которые нужно копировать.
     2. На панели инструментов в основной области нажмите кнопку **Копировать** или щелкните выбранную папку или файл правой кнопкой мыши и в контекстном меню выберите **Копировать**.
     3. В левой области перейдите в другую учетную запись ADLS и дважды щелкните ее, чтобы она отобразилась в основной области.
     4. На панели инструментов в основной области выберите **Вставить**, чтобы создать копию. Или щелкните в контекстном меню команду **Вставить** в месте назначения.

        ![копирование и вставка папки или файла](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-copy-paste.png)

      > [!NOTE] 
          > + Копирование и вставка между разными типами хранилища **не** поддерживается. Вы можете скопировать папки и файлы ADLS и вставить их в другую учетную запись ADLS. Но **вы не можете** копировать файлы или папки ADLS и вставлять их в хранилище BLOB-объектов или наоборот. 
          > + Для выполнения операций копирования и вставки нужно загрузить папки или файлы в локальные расположения, а потом передать их оттуда в расположения назначения. В серверной части эти операции **не выполняются**. Операции копирования и вставки больших файлов могут занять много времени. Идет разработка метода оптимизации копирования и перемещения файлов.
   * **Удаление файлов или папок**

     1. Выберите папки или файлы, которые нужно удалить.
     2. На панели инструментов в основной области нажмите кнопку **Удалить** или щелкните выбранную папку или файл правой кнопкой мыши и в контекстном меню выберите **Удалить**.
     3. Щелкните **Да** в диалоговом окне подтверждения.

        ![копирование и вставка папки или файла](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-delete.png)

   * **Закрепление для быстрого доступа**

     1. Выберите папку, которую необходимо закрепить.
     2. На панели инструментов в основной области нажмите кнопку **Закрепить для быстрого доступа**.
     3. В левой области вы увидите, что выбранная папка добавлена в узел **Быстрый доступ**.

        ![закрепление для быстрого доступа](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-quick-access.png)
     4. После настройки быстрого доступа можно легко получить доступ к ресурсам.
   * **Прямые ссылки**
     1. Если доступен URL-адрес, его можно просто ввести в путь адреса в **проводнике** или браузере.
     2. Затем для перехода в расположение введенного URL-адреса автоматически запустится файл **Storage Explorer.exe**.

        ![прямая ссылка в проводнике](./media/data-lake-store-in-storage-explorer/storageexplorer-adls-deep-link.png)

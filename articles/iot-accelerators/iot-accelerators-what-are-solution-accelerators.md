# <a name="what-are-the-iot-solution-accelerators"></a>Что собой представляют акселераторы решений Интернета вещей?

_Акселераторы решений_ Интернета вещей Azure являются наборами решений, которые позволяют:

* выполнить развертывание за считаные минуты;
* быстро приступить к работе;
* настроить Azure IoT Suite в соответствии со своими потребностями.

Все акселераторы решений разрабатываются с применением одинаковых принципов и целей.

В следующем видео представлен обзор акселератора решений для удаленного мониторинга.

>[!VIDEO https://channel9.msdn.com/Shows/Internet-of-Things-Show/Meet-the-new-Remote-Monitoring-accelerator-for-Azure-IoT/Player]

## <a name="solution-accelerators-overview"></a>Общие сведения об акселераторах решений

Акселератор решений представляет собой реализацию с открытым исходным кодом для стандартного шаблона решений Интернета вещей, которое можно развернуть в Azure с помощью подписки. Каждый акселератор решений содержит настраиваемый код и службы Azure, реализующие определенный сценарий (или несколько сценариев) для Интернета вещей. Вы можете настроить любой из сценариев в соответствии со своими требованиями. Ниже приведены соответствующие сценарии.

* Визуализация данных на расширенных панелях мониторинга для получения подробных сведений и состояния решения.
* Настройка правил и оповещений через динамическую телеметрию устройства Интернета вещей.
* Планирование заданий управления для устройства, например обновления программного обеспечения и конфигурации.
* Подготовка собственных настраиваемых физических или виртуальных устройств.
* Устранение проблем и неполадок в группах устройств Интернета вещей.

Каждый акселератор решений является полнофункциональной реализацией, в которой используются имитированные или физические устройства для создания данных телеметрии. Вы можете использовать акселераторы решений для ускорения работы в следующих сценариях:

* чтобы предоставить отправную точку для собственных решений Интернета вещей;
* чтобы ознакомиться с распространенными шаблонами, которые используются при проектировании и разработке решений Интернета вещей.

В настоящее время существуют три акселератора решений:

* [удаленный мониторинг](iot-accelerators-remote-monitoring-explore.md);
* [Прогнозное обслуживание](../iot-suite/iot-suite-predictive-overview.md)
* [Подключенная фабрика](iot-accelerators-connected-factory-overview.md)

В следующей таблице показано соответствие этих решений конкретным функциям IoT.

| Решение | Прием данных | Удостоверение устройства | Управление устройствами | Пограничные вычисления | Команды и управление | Правила и действия | Прогнозная аналитика |
| ------------------------------------------------------------ | -- | -- | -- | -- | -- | -- | -- |
| [Удаленный мониторинг](iot-accelerators-remote-monitoring-explore.md)  |Yes |Yes |Yes |-   |Yes |Yes |-   |
| [Прогнозное обслуживание](../iot-suite/iot-suite-predictive-overview.md)   |Yes |Yes |-   |-   |Yes |Yes |Yes |
| [Подключенная фабрика](iot-accelerators-connected-factory-overview.md) |Yes |- |- |Yes |Yes |Yes |-   |

* *Прием данных*: прием данных в облако в нужном масштабе.
* *Удостоверения устройств*: управление уникальными удостоверениями устройств и контроль доступа устройств к решению.
* *Управление устройствами*: управление метаданными устройств и выполнение операций, таких как перезагрузка устройства и обновление встроенного ПО.
* *Команды и управление*: отправка сообщений на устройство из облака для выполнения на устройстве определенных действий.
* *Правила и действия*: правила используются в серверной части решения для операций с определенными данными, отправляемыми с устройства в облако.
* *Прогнозная аналитика*: данные, отправляемые с устройства в облако, анализируются в серверной части решения, что позволяет предсказать время выполнения определенных действий. Пример — анализ данных телеметрии, поступающих от двигателя самолета, который позволяет определить время обслуживания двигателя.

> [!NOTE]
> Чтобы развернуть акселератор решений и потренироваться настраивать его, ознакомьтесь с руководством [Акселераторы решений Интернета вещей Microsoft Azure](https://www.azureiotsuite.com/).

## <a name="azure-services"></a>Службы Azure

При развертывании акселератора решений в процессе подготовки автоматически настраивается несколько служб Azure. В следующей таблице перечислены службы, используемые в акселераторах решений:

|                      | Удаленный мониторинг  | Прогнозное обслуживание | Подключенная фабрика |
| -------------------- | ------------------ | ---------------------- | ----------------- |
| Центр Интернета вещей              | Yes                | Yes                    | Yes               |
| Концентраторы событий           |                    | Yes                    |                   |
| Аналитика временных рядов Azure |                    |                        | Yes               |
| Службы контейнеров   | Yes                |                        |                   |
| Stream Analytics     |                    | Yes                    |                   |
| Веб-приложения             | Yes                | Yes                    | Yes               |
| База данных Cosmos            | Yes                | Yes                    |                    |
| Хранилище Azure         |                    | Yes                    | Yes               |

> [!NOTE]
> Дополнительные сведения о ресурсах, развернутых в акселераторе решений для удаленного мониторинга см. в этой [статье](https://github.com/Azure/azure-iot-pcs-remote-monitoring-dotnet/blob/master/README.md#basic-vs-standard-deployments) на сайте GitHub.

* [Центр Интернета вещей Azure](../iot-hub/index.yml). Эта служба предоставляет возможности обмена сообщениями между устройствами и облаком, а также выступает в качестве шлюза для облака и других ключевых служб акселератора решений. Служба позволяет получать сообщения от устройств в реальном времени и отправлять команды на устройства. Служба также позволяет [управлять устройствами](../iot-hub/iot-hub-device-management-overview.md). Например, вы можете настроить, перезагрузить или восстановить заводские настройки на одном или нескольких устройствах, подключенных к центру.
* [Концентраторы событий Azure](../active-directory-domain-services/index.md). Эта служба позволяет получать события в крупных масштабах в облаке. Ознакомьтесь со статьей [Сравнение Центра Интернета вещей Azure и концентраторов событий Azure](../iot-hub/iot-hub-compare-event-hubs.md).
* [Аналитика временных рядов Azure](../time-series-insights/index.yml). Акселераторы решений используют эту службу для анализа и отображения данных телеметрии, полученных с устройств.
* [Служба контейнеров Azure](../container-service/index.yml). Эта служба размещает микрослужбы акселераторов решений и управляет ими.
* [Azure Cosmos DB](../cosmos-db/index.yml) и [служба хранилища Azure](../storage/index.yml) для хранения данных.
* [Azure Stream Analytics](../stream-analytics/index.yml). Предварительно настроенное решение для прогнозного обслуживания использует эту службу для обработки входящих данных телеметрии, агрегирования и обнаружения событий. Это решение также использует Stream Analytics для обработки информационных сообщений, содержащих, например, метаданные или ответы на команды от устройств.
* [Веб-приложения Azure](../app-service/index.yml) для размещения настраиваемого кода приложения в предварительно настроенных решениях.

Обзор архитектуры стандартного решения Интернета вещей см. в статье [Azure и "Интернет вещей"](iot-accelerators-what-is-azure-iot.md).

## <a name="whats-new-in-solution-accelerators"></a>Что нового появилось в акселераторах решений?

Корпорация Майкрософт переводит акселераторы решений на новую архитектуру на основе микрослужб. В таблице ниже показано текущее состояние для акселераторов решений.

| Акселератор решений | Архитектура  | Языки     |
| ---------------------- | ------------- | ------------- |
| Удаленный мониторинг      | Микрослужбы | Java и .NET |
| Прогнозное обслуживание | MVC           | .NET          |
| Подключенная фабрика      | MVC           | .NET          |

В разделах ниже описаны новые возможности акселераторов решений на основе микрослужб.

### <a name="microservices"></a>Микрослужбы

В новой версии акселератора решений для удаленного мониторинга используется архитектура микрослужб. Этот акселератор решений состоит из нескольких микрослужб, таких как *диспетчер Центра Интернета вещей* и *диспетчер хранилища*. Вы можете загрузить версии Java и .NET каждой микрослужбы, а также соответствующую документацию для разработчиков. Дополнительные сведения о микрослужбах см. в статье [Архитектура акселератора решения для удаленного мониторинга](iot-accelerators-remote-monitoring-sample-walkthrough.md).

Эта архитектура микрослужб представляет собой надежный шаблон облачных решений, который:

* является масштабируемым;
* предоставляет возможности расширения;
* несложный в эксплуатации;
* позволяет использовать альтернативные службы.

> [!TIP]
> Дополнительные сведения об архитектурах микрослужб см. на странице [архитектуры приложения для .NET](https://www.microsoft.com/net/learn/architecture), а также в записи блога, посвященной [возможностям микрослужб — мощные возможности приложения, обеспечиваемые облачной средой](https://azure.microsoft.com/blog/microservices-an-application-revolution-powered-by-the-cloud/).

При развертывании новой версии удаленного мониторинга необходимо выбрать один из следующих вариантов развертывания:

* **Базовый.** Версия с низкой стоимостью для демонстрации или тестирования развертывания. Все микрослужбы развертываются на одной виртуальной машине Azure.
* **Стандартный.** Развертывание расширенной инфраструктуры для разработки развертывания в рабочей среде. С помощью Службы контейнеров Azure можно развернуть микрослужбы на нескольких виртуальных машинах Azure. Kubernetes управляет контейнерами Docker, на которых размещаются отдельные микрослужбы.

### <a name="language-choices-java-and-net"></a>Выбор языков: Java и .NET

Вы можете развернуть все микрослужбы на Java и .NET. Как и код .NET, код Java представляет собой открытый исходный код, доступный для настройки в соответствии с вашими требованиями.

* [Репозиторий удаленного мониторинга для .NET на GitHub](https://github.com/Azure/azure-iot-pcs-remote-monitoring-dotnet)
* [Репозиторий удаленного мониторинга для Java на GitHub](https://github.com/Azure/azure-iot-pcs-remote-monitoring-java)

Запрос на реализацию на других языках можно оставить на форуме [для Azure IoT](https://feedback.azure.com/forums/321918-azure-iot).

### <a name="react-user-interface-framework"></a>Платформа пользовательского интерфейса на основе библиотеки React

Пользовательский интерфейс создается с помощью библиотеки для JavaScript [React](https://facebook.github.io/react/). Исходный код представляет собой открытый исходный код, который можно загрузить и настроить.

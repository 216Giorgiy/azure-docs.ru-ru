# <a name="what-is-azure-monitor-for-vms-preview"></a>Что такое Azure Monitor для виртуальных машин (предварительная версия)?

Azure Monitor для виртуальных машин отслеживает виртуальные машины Azure и масштабируемые наборы виртуальных машин Azure, анализируя производительность и работоспособность виртуальных машин Windows и Linux, включая различные процессы и взаимосвязанные зависимости от других ресурсов и внешних процессов. Решение включает поддержку мониторинга производительности и зависимостей приложений для виртуальных машин, размещенных в локальном или другом поставщике облачных служб. Оно включает три ключевые функции, которые предоставляют подробные аналитические сведения.

* Логические компоненты виртуальных машин Azure под управлением операционной системы Windows и Linux измеряются при оценке условий в зависимости от набора предварительно настроенных критериев работоспособности и оповещения.  
* Основные метрики производительности процессора, памяти, диска и сетевого адаптера гостевой виртуальной машины собраны и представлены в заранее определенных диаграммах производительности трендов.
* Сопоставление зависимостей показывает обнаруженные взаимосвязанные компоненты этой виртуальной машины с несколькими группами ресурсов и подписок.  

Эти возможности организованы по трем аспектам:

* Health
* Производительность
* Сопоставление

>[!NOTE]
>Сейчас функция мониторинга работоспособности предлагается только для виртуальных машин и масштабируемых наборов виртуальных машин Azure. Функции сопоставления и мониторинга производительности поддерживаются для виртуальных машин Azure и виртуальных машин, размещенных в вашей среде или у другого поставщика облачных решений.
>

Интеграция с Log Analytics обеспечивает мощную агрегацию, фильтрацию и возможность проводить анализ тенденций данных с течением времени. Комплексный мониторинг рабочих нагрузок не может осуществляться, используя только Azure Monitor, Сопоставление служб или Log Analytics.  

В контексте одной виртуальной машины можно просматривать эти данные непосредственно с виртуальной машины. Azure Monitor обеспечивает агрегированное представление виртуальных машин на основе следующей перспективы для каждой функции.

* Работоспособность. Виртуальные машины, относящиеся к группе ресурсов.
* Сопоставления и производительность. Виртуальные машины настроены на отправку отчетов в определенную рабочую область Log Analytics.

![Перспектива полезных сведений о виртуальной машине на портале](./media/vminsights-overview/vminsights-azmon-directvm-01.png)

DevOps может эффективно обеспечивать предсказуемую производительность и доступность жизненно важных приложений путем выявления критических событий операционной системы и узких мест в производительности, сетевых проблем и понимания того, связана ли проблема с другими зависимостями.  

## <a name="data-usage"></a>Использование данных 

Как только вы подключили Azure Monitor для виртуальных машин, данные, собранные с виртуальных машин, принимаются и хранятся в Azure Monitor. Служба Azure Monitor для виртуальных машин выставляет счет за принятую и сохраненную информацию, количество отслеженных временных рядов метрик критериев работоспособности, созданных правил оповещения, отправленных уведомлений по расценкам, опубликованным на [странице ценообразования](https://azure.microsoft.com/pricing/details/monitor/) Azure Monitor.

Размер журнала меняется в зависимости от длины строк счетчиков и может увеличиться с помощью числа логических дисков и сетевых адаптеров. Если у вас уже есть рабочее пространство и вы собираете эти счетчики, никаких дублирующих сборов не будет. Если вы уже используете Сопоставление служб, единственное изменение, которое вы увидите — это данные дополнительного подключения, отправляемые в Azure Monitor.

# <a name="viewing-and-analyzing-log-data-in-azure-monitor"></a>Просмотр и анализ данных журнала в Azure Monitor
Log Analytics — это основной интерфейс для работы с данными журнала и создания запросов в Azure Monitor. Откройте Log Analytics в разделе **Журналы** в меню **Azure Monitor**. Чтобы получить общие сведения об этом портале и проверить его возможности см. статью [Начало работы с Log Analytics на портале Azure](get-started-portal.md).

Для работы с запросами журнала Log Analytics предоставляет следующие возможности.

* Несколько вкладок. Создание отдельных вкладок для работы с несколькими запросами.
* Расширенная визуализация. Разнообразие параметров для работы с диаграммами.
* Улучшенная технология Intellisense и автозаполнение языка.
* Выделение синтаксиса. Повышает удобство чтения запросов. 
* Обозреватель запросов. Предоставление доступа к сохраненным запросам и функциям.
* Представление схемы. Просмотр структуры данных для помощи в написании запросов.
* Общий доступ. Создание ссылок на запросы или их закрепление на любой общей панели мониторинга Azure.
* Смарт-аналитика. Определение пиков на диаграммах и быстрый анализ их причины.
* Выбор столбцов. Сортировка и группирование столбцов в результатах запросов.

> [!NOTE]
> Log Analytics имеет те же функции, что и портал расширенной аналитики, который является внешним инструментом за пределами портала Azure. Портал расширенной аналитики по-прежнему доступен, но ссылки и другие справочные материалы к нему заменяются этой новой страницей.

### <a name="resource-logs"></a>Журналы ресурсов
Log Analytics интегрирует с различными ресурсами Azure, например Виртуальными машинами. Это означает, что вы можете открыть Log Analytics напрямую с помощью меню мониторинга ресурсов без перехода в Azure Monitor и потери контекста ресурса. Пока что **журналы** доступны не для всех ресурсов Azure, но они будут отображаться в меню портала для отдельных типов ресурсов.

Если открыть Log Analytics из меню определенного ресурса, записи журнала автоматически отобразятся только для этого ресурса.   Чтобы написать запрос на добавление других записей, вам нужно открыть его из меню Azure Monitor.

Следующие параметры еще не доступны в представлении ресурса службы Log Analytics:

- Сохранить
- задание оповещения;
- Обозреватель запросов
- переключение на другую рабочую область или ресурс (пока что не планируется).


### <a name="firewall-requirements"></a>Требования к брандмауэру
Ваш браузер запрашивает доступ к следующим адресам, чтобы получить доступ к Log Analytics.  Если браузер обращается к порталу Azure через брандмауэр, необходимо разрешить доступ для этих адресов.

| URI | IP-адрес | порты; |
|:---|:---|:---|
| portal.loganalytics.io | Динамический | 80, 443 |
| api.loganalytics.io    | Динамический | 80, 443 |
| docs.loganalytics.io   | Динамический | 80, 443 |


## <a name="log-search-classic"></a>Поиск по журналам (классическим)
Поиск по журналам является устаревшим интерфейсом портала Azure для запроса и анализа данных журнала в Azure Monitor. Сейчас он все еще доступен, но скоро будет снят с учета. Откройте поиск по журналам в разделе **Журналы (классические)** в меню Log Analytics.

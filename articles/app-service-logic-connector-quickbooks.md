#Использование соединителя QuickBooks в приложении логики#

Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока. Соединитель QuickBooks позволяет создавать и изменять различные сущности QuickBooks. Ниже приведен список сущностей QuickBooks, поддерживаемых соединителем QuickBooks.

<Table>
<TR><TD><B>Сущности</TD><TD><B>Описание</TR>
<TR>	<TD>	Account	</TD>	<TD>	Учетная запись является компонентом плана счетов и является частью книги учета. Используется для записи всего денежной суммы, выделенной для конкретного использования	</TD>	</TR>
<TR>	<TD>	Кредитное авизо	</TD>	<TD>	Кредитное авизо является финансовой операцией, представляющей возврат, оплату в кредит или часть оплаты за товары или услуги, которые были проданы.	</TD>	</TR>
<TR>	<TD>	Клиент	</TD>	<TD>	Клиент - получатель службы или продукта, предлагаемых вашим предприятием.	</TD>	</TR>
<TR>	<TD>	Оценка	</TD>	<TD>	Оценка представляет собой предложение финансовых транзакций от предприятия клиенту за товары или услуги, предлагаемые для продажи, включая предложенные цены.	</TD>	</TR>
<TR>	<TD>	Счет	</TD>	<TD>	Счет - форма продажи, где клиент оплачивает продукт или услугу позднее. Дополнительные сведения об использовании модели данных счета см. здесь.	</TD>	</TR>
<TR>	<TD>	Элемент	</TD>	<TD>	Элемент - это предмет, который приобретает, продает или перепродает ваша компания, например продукты, накладные расходы, скидки и налоги (если применимо).  Элемент отображается строкой в счете или другой форме продажи.	</TD>	</TR>
<TR>	<TD>	Товарный чек	</TD>	<TD>	Эта сущность представляет собой чек, выдаваемый клиенту.	</TD>	</TR>
</Table>

##Создание приложения API соединителя QuickBooks##
1.	Откройте Azure Marketplace с помощью кнопки "+ СОЗДАТЬ" в нижнем правом углу портала Azure.
2.	Перейдите в раздел "Мобильные и веб-приложения > Приложения API" и выполните поиск QuickBooks.
3.	Настройте соединитель QuickBooks, предоставив сведения по плану размещения и группе ресурсов, а также выбрав имя приложения API.

	![][13]
4. Настройте сущности QuickBooks для чтения или записи в разделе 'Параметры пакета'.

Теперь можно создать приложение API соединителя QuickBooks.


##Создание приложения логики##
Построим простое приложение логики, которое создает учетную запись в QuickBooks и обновляет 'Тип категории' этой учетной записи.

1.	Выполните вход на портал Azure и последовательно выберите 'Создать -> Мобильные и веб-приложения -> Приложение логики.'

	![][1]

2.	На странице 'Создание приложения логики' введите необходимые сведения, такие как имя, план обслуживания приложения и расположение.

	![][2]

3.	Нажмите 'Триггеры и действия', откроется экран редактора приложения логики.

	![][3]

4.	Установите флажок 'Запускать эту логику вручную', а это означает, что это приложение логики может быть вызвано только вручную.


5.	Разверните 'Приложения API в этой группе ресурсов' в коллекции, чтобы просмотреть все доступные приложения API. Выберите в коллекции элемент 'Соединитель QuickBooks', и 'Соединитель QuickBooks' будет добавлен в поток.


6.	Необходимо произвести проверку подлинности и авторизацию приложений логики для выполнения операций от имени пользователя, если QuickBooks находится в интерактивном режиме. Чтобы запустить авторизацию, нажмите кнопку "Авторизовать" в соединителе QuickBooks.

	![][4]

7.	После нажатия кнопки "Авторизовать" открывается диалоговое окно проверки подлинности QuickBooks. Предоставьте сведения для входа учетной записи QuickBooks, в которой вы хотите выполнять операции.

	![][5]

8. Предоставьте приложениям логики доступ к своей учетной записи для выполнения операций от вашего имени, нажав кнопку "Авторизовать" в диалоговом окне согласия.

	![][6]

9.	После выполнения проверки подлинности отображаются все действия.

	![][7]

10.	Выберите действие 'Создать или обновить учетную запись'. Отобразятся входные параметры.

	![][8]

11.	Укажите 'Имя' и 'Тип учетной записи' и нажмите кнопку ✓. 

	![][9]

12.	В разделе 'Последние использовавшиеся' в коллекции выберите 'Соединитель QuickBooks'. Будет добавлено новое действие QuickBooks.

13.	Выберите 'Обновить учетную запись' в списке действий. Отобразятся входные параметры этого действия.

	![][10]

14.	Щелкните знак '+' рядом с полем 'Идентификатор' и выберите значение идентификатора в выходных данных действия "Создать учетную запись". 

	![][11]

15.	Укажите новые значения для типа учетной записи и нажмите кнопку ✓.

	![][12]

16. Нажмите кнопку "ОК" на экране редактора приложения логики и нажмите кнопку 'Создать'. Создание займет приблизительно 30 секунд.

17. Перейдите к только что созданному приложению логики и нажмите кнопку 'Запустить', чтобы инициировать запуск.

18. Можно убедиться, что в вашей учетной записи QuickBooks создана новая учетная запись с именем 'Contoso'.

# Что такое Azure Stream Analytics?

Azure Stream Analytics (ASA) — это полностью управляемый экономичный модуль обработки событий в реальном времени, позволяющий получать ценную информацию из данных. Stream Analytics позволяет без усилий настроить аналитические вычисления потоковых данных, поступающих от устройств, датчиков, веб-сайтов, социальных сетей, приложений, систем инфраструктуры и пр., в режиме реального времени.

С помощью нескольких щелчков на портале Azure можно создать задание Stream Analytics, указав источник входных потоковых данных, приемник выходных данных для результатов задания, а также преобразование данных, выраженное на языке, подобном SQL. Вы можете отслеживать и изменять масштаб и скорость выполнения задания на портале Azure — от нескольких килобайт обрабатываемых за секунду событий до гигабайта и более.

Служба Stream Analytics основана на многолетнем опыте исследовательской работы Майкрософт по разработке высокоточных потоковых обработчиков данных, критичных ко времени их обработки, а также интеграции языков для их интуитивного определения.

## Каковы возможности использования Stream Analytics?
Сегодня по сети передается огромное количество данных на высоких скоростях. Организации, которые могут обрабатывать эту потоковую передачу данных и предпринимать в ее отношении различные действия в режиме реального времени, могут существенно повысить свою эффективность и опередить конкурентов. Сценарии аналитики потока в режиме реального времени можно найти во всех следующих отраслях: персонализированный анализ фондового рынка в режиме реального времени и оповещения от компаний, которые предоставляют финансовые услуги; выявление мошенничества в режиме реального времени; службы защиты данных и личной информации; надежный сбор и анализ данных, поступающих от датчиков и приводов, вмонтированных в физические объекты («Интернет вещей», или IoT); потоковая аналитика переходов на сайтах и приложения для управления отношениями с клиентами (CRM), отправляющие сигналы при падении уровня обслуживания клиента в течение определенного промежутка времени. Компании ищут наиболее гибкий, надежный и экономичный способ осуществления такого потокового анализа данных для событий в режиме реального времени для успешной деятельности в современном высококонкурентном мире.

## Основные возможности и преимущества
-   **Простота использования.** Stream Analytics поддерживает простую декларативную модель запросов для описания преобразований. Для облегчения работы в Stream Analytics используется вариант языка SQL. Это избавляет пользователей от необходимости использовать технически сложные системы обработки потоковых данных. Используя [язык запросов Stream Analytics](https://msdn.microsoft.com/library/azure/dn834998.aspx) в редакторе запросов на основе браузера, с помощью технологии автозавершения Intellisense вы можете быстро и легко реализовать запросы временных рядов. К ним относятся соединения с привязкой ко времени, оконные агрегаты, временные фильтры, а также другие операции общего характера, включая соединения, агрегаты, проекции и фильтры. Кроме того, поддержка проверки запросов в браузере для файла демонстрационных данных обеспечивает быструю последовательную разработку.  

-   **Масштабируемость.** Stream Analytics может обрабатывать события с высокой пропускной способностью — до 1 ГБ/с. Благодаря интеграции с [концентраторами событий Azure](http://azure.microsoft.com/services/event-hubs/) решение может принимать в секунду миллионы событий от подключенных устройств, посещений ресурсов, файлов журналов и т. п. Для этого в Stream Analytics используются возможности секционирования концентраторов событий, которые обеспечивают обработку одной секции со скоростью 1 МБ/с. Пользователи могут секционировать вычислительные операции на несколько логических шагов в рамках определения запроса. Каждый из этих шагов можно дополнительно секционировать для повышения масштабируемости.

-   **Надежность, повторяемость и быстрое восстановление.** Управляемая облачная служба Stream Analytics предотвращает потери данных при сбоях и обеспечивает непрерывность бизнес-процессов благодаря встроенным функциям восстановления. Благодаря функции внутренней поддержки состояния служба обеспечивает воспроизводимость результатов. Это дает возможность архивировать события и повторно применять обработку в будущем, всегда получая одинаковые результаты. Эта функция позволяет клиентам вернуться назад во времени и изучить вычисления в рамках анализа основных причин, анализа «что если» и т. д.

-   **Низкая стоимость.** Stream Analytics является облачной службой. Поэтому она оптимизирована таким образом, чтобы обеспечить пользователям очень низкую стоимость ввода в эксплуатацию и поддерживать функционирование аналитических решений в режиме реального времени. Служба предполагает оплату в зависимости от использования единиц потоковой передачи и объема данных, обрабатываемых в системе. Интенсивность использования определяется на основе количества обработанных событий и объема вычислительной мощности, использованного в кластере для обработки соответствующих заданий Stream Analytics.

-   **Ссылочные данные.** Stream Analytics предоставляет пользователям возможность указывать и использовать ссылочные данные. Это могут быть данные журналов или просто непотоковые данные с меньшей вариативностью по времени. Система упрощает использование ссылочных данных, которые рассматриваются как любой другой входящий поток событий для объединения с другими потоками событий, принимаемыми в режиме реального времени для выполнения преобразований.

-   **Возможности подключения.** Stream Analytics подключается непосредственно к концентраторам событий Azure для получения потока, а также к службе BLOB-объектов Azure для получения данных журналов. Результаты могут записываться из Stream Analytics в большие двоичные объекты или таблицы службы хранилища Azure, Базу данных SQL Azure, концентраторы событий, разделы или очереди сервисной шины Azure, а также в Power BI. Затем результаты можно визуализировать, дополнительно обрабатывать в рамках рабочих процессов, использовать в пакете аналитики с помощью [Azure HDInsight](http://azure.microsoft.com/services/hdinsight/) или повторно обрабатывать как последовательность событий. При использовании концентраторов событий можно объединить несколько служб Stream Analytics с другими источниками данных и модулями обработки без потери потока вычислений.

# Использование пакета аналитики «Управление журналами»

[AZURE.INCLUDE [operational-insights-note-moms](../../includes/operational-insights-note-moms.md)]

Для использования функции управления журналами в Microsoft Azure Operational Insights необходимо установить решение. Поскольку этот пакет должен быть настроен и не собирает никаких данных по умолчанию, он включен по умолчанию в каждой новой рабочей области. Если пакет был удален, его можно повторно добавить в коллекции решений. Дополнительные сведения см. в разделе [Использование коллекции решений для добавления или удаления решений](operational-insights-add-solution.md).

Можно добавить новые журналы для сбора событий и выбрать, какие уровни событий или серьезности будут собирать журналы. После настройки функция управления журналами запустит политику сбора и начнет собирать события. Получить доступ к исходным структурам событий журнала, собранных с отслеживаемых серверов, можно с помощью плитки **Управление журналами** на странице **Обзор** в Operational Insights.

![image of Log Management tile](./media/operational-insights-log-collection/overview-log-mgt.png)

Плитка открывает страницу **Управление журналами**, на которой можно просмотреть сводку событий, содержащихся в журналах. Или можно выполнить прямой поиск журналов аналогично поиску любого другого типа данных.


На странице приводятся следующие категории:

- журналы событий Windows;
- журналы IIS;
- другие добавленные журналы.

## Формат файла журнала IIS

В данный момент единственным поддерживаемым форматом журнала IIS является W3C. Беспокоиться не следует, так как это наиболее распространенный формат и формат по умолчанию в IIS 7 и IIS 8. Таким образом, при входе в собственном формате NCSA или IIS служба Operational Insights вообще не будет принимать эти журналы. Даже при использовании формата W3C вы увидите, что не все поля регистрируются по умолчанию. Дополнительные сведения об этом формате см. в статье «Выбор полей W3C для записи в журнал (IIS 7)»https://technet.microsoft.com/library/cc754702(v=WS.10).aspx).


> [AZURE.TIP]Для получения наилучших результатов поиска журналов рекомендуется выбрать все поля ведения журнала для каждого веб-сайта с помощью функции **Ведение журнала** в IIS. Также рекомендуется изменить расписание **Переход на новый файл журнала** на **Ежечасно**, что позволит отправлять в облако небольшие файлы и экономить пропускную способность.


### Сбор журналов событий Windows из Operations Manager или напрямую подключенных агентов

1. На странице **Обзор** щелкните плитку **Управление журналами**.

2. На панели мониторинга **Управление журналами** щелкните плитку **Настройка**.

3. Введите имя журнала событий, из которого нужно собрать сведения. Если вы не знаете, какое имя использовать, откройте окно свойств журнала событий Windows в **средстве просмотра событий**, скопируйте имя из поля **Полное имя** и вставьте его в поле **Собирать события из следующих журналов событий**.

4. Щелкните **+**, чтобы добавить журнал.

5. Выберите уровни событий или серьезность событий, которые нужно собирать для журнала. В этом выпуске не поддерживаются такие события, как **Аудит-успех** и **Аудит-отказ**.

6. Повторите предыдущие шаги для каждого журнала, из которого требуется собирать данные, а затем нажмите кнопку **Сохранить**.

7. События должны появиться в Operational Insights через несколько минут, после чего можно выполнять поиск данных.



### Сбор журналов IIS из Operations Manager или напрямую подключенных агентов

1. На странице **Обзор** щелкните плитку **Управление журналами**.

2. На панели мониторинга **Управление журналами** щелкните плитку **Настройка**.

3. В разделе **Журналы IIS** выберите параметр **Собирать журналы из Operations Manager**.


### Сбор журналов IIS и/или событий Windows из службы Azure Diagnostics
Эту процедуру можно настроить на портале управления Azure, но не на портале Operational Insights. В рабочей области откройте вкладку **Хранение** и включите сбор журналов из этой учетной записи хранения.


### Работа с файлами журналов

1. На странице **Обзор** щелкните плитку **Управление журналами**.

2. На панели мониторинга **Управление журналами** просмотрите журналы событий и выберите один из них для работы.

3. Щелкните любой элемент, чтобы просмотреть сведения о нем на странице **Поиск**.

4. Можно выполнить поиск других журналов в полученных первоначальных результатах, а затем проанализировать и соотнести журналы.

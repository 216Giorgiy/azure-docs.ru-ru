#Создание веб-приложения

В этой статье показано, как создать веб-приложение на основе коллекции или с помощью портала управления.

Дополнительную информацию о развертывании контента в созданном веб-приложении см. в разделе **Развертывание** статьи [Веб-приложения Azure](http://go.microsoft.com/fwlink/?LinkId=529714).

>[AZURE.NOTE] Если вы хотите начать работу со службой приложений Azure еще до создания учетной записи Azure, перейдите на страницу [ознакомительной версии службы приложений](http://go.microsoft.com/fwlink/?LinkId=523751). Там вы сможете сразу создать свое первое веб-приложение, правда срок его службы будет ограничен. Никаких кредитных карт и обязательств.

##<a name="createawebsiteportal"></a>Практическое руководство: Создание веб-приложения с помощью портала Azure

Выполните следующие действия, чтобы создать веб-приложение в Azure.

1. Войдите на [портал Azure](http://go.microsoft.com/fwlink/?LinkId=529715).

2. Щелкните значок **Создать** в нижнем левом углу портала.

3. Щелкните значок **Интернет + мобильное устройство**, щелкните значок **Веб-приложение** введите значение для URL-адреса, а затем щелкните **Создать** в нижнем правом углу колонки создания.

4. После создания веб-приложения появится текст **Развертывание группы ресурсов <имя группы ресурсов> успешно выполнено**.

5. На портале щелкните имя веб-приложения в списке веб-приложений, чтобы открыть колонку веб-приложения.

6. В колонке можно получить доступ к инструментам разработки веб-приложений, настроить публикацию веб-приложения или настроить развертывание из системы управления версиями, такой как TFS или Git. Публикация на FTP настроена для веб-приложений по умолчанию, имя FTP-сервера отображается в разделе **Основные компоненты** в колонке веб-приложения. Перед публикацией с помощью FTP или Git выберите параметр **Сбросить профиль публикации** в колонке веб-приложения, чтобы пройти аутентификацию на FTP-сервере или в репозитории Git при развертывании контента в веб-приложение.

7. Колонка **Параметры** отображает параметры приложения:

	- версия .NET, PHP, Java или Python для веб-приложения;
	- редакция в Visual Studio Online;
	- привязки SSL;
	- имена пользовательских доменов;
	- проверка подлинности и авторизация;
	- диагностика приложений и сайта;
	- конечные точки мониторинга;
	- параметры ведения журнала;
	- параметры приложения для среды Azure (например, переопределение <appSettings> в файле Web.config в вашей среде разработки);
	- строки подключения (например, переопределение <connectionStrings> в файле Web.config в вашей среде разработки);
	- обработчики сценариев для определенных расширений файлов (например, PHP).

##<a name="deleteawebsite"></a> Практическое руководство. Удаление веб-приложения
Веб-приложения удаляются с помощью значка **Удалить** на портале Azure. Значок **Удалить** находится в верхней части колонки веб-приложения.

# Службы синхронизации Azure AD Connect: рекомендации по изменению конфигурации по умолчанию

В этой статье описываются поддерживаемые и неподдерживаемые изменения в службах синхронизации Azure AD Connect.

Конфигурация, созданная при помощи Azure AD Connect, подходит для большинства сред, в которых локальные каталоги Active Directory синхронизируются с Azure AD. Однако в некоторых случаях, чтобы выполнить определенную задачу или соблюсти нужные требования, в конфигурацию приходится вносить изменения.

## Изменения в учетной записи службы
Службы синхронизации Azure AD Connect работают под учетной записью службы, созданной мастером установки. Эта учетная запись содержит ключи шифрования для базы данных, используемой в процессе синхронизации. Она создается с паролем длиной 127 символов, срок действия которого не ограничен.

- Изменение или сброс пароля учетной записи службы **не поддерживается**. Это приведет к уничтожению ключей шифрования, и служба не сможет получить доступ к базе данных и запуститься.

## Изменения в планировщике
В службах синхронизации Azure AD Connect настроена синхронизация данных удостоверения каждые 3 часа. В ходе установки создается запланированная задача, которая выполняется под учетной записью службы с разрешениями на использование сервера синхронизации.

- Внесение изменений в эту запланированную задачу **не поддерживается**. Пароль для учетной записи службы неизвестен. См. раздел [Изменения в учетной записи службы](#changes-to-the-service-account)
- Выполнение синхронизации чаще, чем через стандартные 3 часа, **не поддерживается**.

## Изменения в правилах синхронизации

Изменять конфигурацию служб синхронизации Azure AD Connect можно, но делать это следует с осторожностью, так как они должны быть максимально близко к устройству.

Ниже приведен список ожидаемых результатов изменения конфигурации.

- После обновления службы Azure AD Connect до новой версии для большинства параметров будут восстановлены значения по умолчанию.
- Изменения стандартных правил синхронизации будут потеряны после установки обновления.
- Удаленные стандартные правила создаются заново во время обновления до новой версии.
- Настраиваемые правила синхронизации, созданные вами, остаются неизменными после обновления до более новой версии.



Чтобы изменить конфигурацию по умолчанию, выполните следующие действия.

- Если нужно изменить поток атрибутов стандартного правила синхронизации, не изменяйте его. Вместо этого создайте новое правило синхронизации с более высоким приоритетом (меньшим числовым значением), содержащее требуемый поток атрибутов.
- Экспортируйте настраиваемые правила синхронизации с помощью редактора правил синхронизации. Так вы получите скрипт PowerShell, который позволяет легко создать их заново для сценария аварийного восстановления.
- Если вам необходимо изменить область или параметр соединения в стандартном правиле синхронизации, задокументируйте это и повторно внесите изменение после обновления до новой версии Azure AD Sync.

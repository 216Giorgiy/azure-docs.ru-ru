# Предоставление доступа к управлению привилегированными пользователями Azure AD

Первый пользователь, который включит управление привилегированными пользователями (PIM) в Azure AD для организации, должен быть глобальным администратором. Однако у других глобальных администраторов по умолчанию нет доступа к PIM, поэтому они не могут управлять временными назначениями. Чтобы предоставить доступ к PIM, первый пользователь может назначить другим пользователям роль администратора безопасности. Это назначение нужно выполнить в PIM, и его нельзя изменить с помощью PowerShell или на других порталах.

> [AZURE.NOTE] Для управления PIM в Azure AD требуется MFA Azure. Поскольку учетные записи Майкрософт нельзя зарегистрировать для MFA Azure, у пользователя, который входит в систему с учетной записью Майкрософт, не будет доступа к PIM в Azure AD.

Убедитесь, что в роли администратора безопасности всегда есть по крайней мере два пользователя, на случай, если один пользователь будет заблокирован или его учетная запись будет удалена.

## Предоставление другому пользователю доступа для управления PIM

1. Войдите на [портал Azure](https://portal.azure.com/) и выберите в панели мониторинга приложение **Управление привилегированными пользователями Azure AD**.
2. Выберите роль **администратора безопасности**. Появится список пользователей, которые в настоящее время находятся в этой роли.
3. Нажмите **Добавить**, и откроется колонка мастера. Роль уже будет выбрана.
4. Нажмите **Выбрать пользователей**, и откроется колонка со списком пользователей.
5. Введите имя пользователя в поле поиска. Если пользователи находятся в каталоге, их учетные записи будут отображаться по мере ввода.
6. Выберите пользователя из результатов поиска и нажмите кнопку **Готово**.
7. Нажмите кнопку **ОК**, чтобы сохранить изменение. Выбранный пользователь будет отображаться в списке, но роль по умолчанию будет временной.

  - Если вы хотите сделать роль постоянной, выберите пользователя в списке. Откроется отдельная колонка со сведениями о пользователе. Выберите в меню пользователя команду **сделать постоянной**.

8. Отправьте пользователю ссылку на документацию Azure [Приступая к работе с управлением привилегированными пользователями Azure AD](active-directory-privileged-identity-management-getting-started.md).


## Удаление прав доступа для управления PIM у другого пользователя

Прежде чем удалить кого-либо из роли администратора безопасности, всегда проверяйте, остается ли хотя бы два пользователя, которым назначена эта роль.

1. В панели мониторинга PIM щелкните роль **Администратор безопасности**. Появится список пользователей, которые в настоящее время находятся в этой роли.
2. Щелкните имя пользователя в списке пользователей.
3. Щелкните **Удалить**. Появится запрос на подтверждение.
4. Нажмите **Да**, чтобы удалить роль у пользователя.

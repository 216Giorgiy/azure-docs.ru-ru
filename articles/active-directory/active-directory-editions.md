# Выпуски Azure Active Directory

Azure Active Directory — это служба, которая предоставляет широкие возможности управления идентификацией и доступом в облаке. Она объединяет службы каталогов, расширенное управление удостоверениями, управление доступом к приложениям и платформу на основе стандартов для разработчиков. Для получения дополнительных сведений см. [это видео](http://azure.microsoft.com/documentation/videos/teched-europe-2014-cloud-identity-microsoft-azure-active-directory-explained/).

Для построения выпусков Azure Active Directory Premium и Basic были использованы многочисленные бесплатные возможности Microsoft Azure Active Directory. Эти выпуски выполняют различные расширенные функции, благодаря которым предприятия способны решать сложные задачи управления удостоверениями и доступа. Сведения о ценах на различные выпуски см. на странице [Azure Active Directory: цены](http://azure.microsoft.com/pricing/details/active-directory/). При подписке на Azure вы можете выбрать один из следующих бесплатных и платных выпусков Azure Active Directory:

- **Free** — бесплатный выпуск Azure Active Directory, который предоставляется с каждой подпиской Azure. Для него не требуется что-либо лицензировать или устанавливать. Он позволяет управлять учетными записями пользователей, синхронизацией с локальными каталогами, обеспечивать единый вход в Azure, Office 365 и тысячи популярных приложений, предоставляемых по модели SaaS, такие как Salesforce, Workday, Concur, DocuSign, Google Apps, Box, ServiceNow и Dropbox.
- **Basic** — выпуск Azure Active Directory Basic обеспечивает управление доступом к приложениям и удостоверениями в режиме самообслуживания для пользователей, которые в основном работают в облаке. Благодаря выпуску Azure Active Directory Basic вы имеете все возможности выпуска Azure Active Directory Free, можете управлять доступом на основе групп, самостоятельно сбрасывать пароль для облачных приложений, имеете прокси-сервер приложений Azure Active Directory (для публикации локальных веб-приложений с помощью Azure Active Directory), настраиваемую среду для запуска корпоративных и потребительских облачных приложений, а также SLA корпоративного уровня (время бесперебойной работы на уровне 99,9%). Администратор с лицензией Azure Active Directory Basic также может активировать пробную версию Azure Active Directory Premium.
- **Premium**. Выпуск Azure Active Directory Premium поддерживает всю функциональность выпусков Free и Basic, а также расширенные возможности по управлению удостоверениями на корпоративном уровне, которые будут описаны ниже.

Чтобы войти в Active Directory Premium и начать использовать данный выпуск, ознакомьтесь с руководством [Приступая к работе с Azure Active Directory Premium](active-directory-get-started-premium.md).

> [AZURE.NOTE]В настоящий момент выпуски Azure Active Directory Premium и Azure Active Directory Basic не поддерживаются в Китае. Для получения дополнительных сведений свяжитесь с нами через [форум, посвященный Azure Active Directory](http://feedback.azure.com/forums/169401-azure-active-directory).

## Возможности Azure Active Directory Basic

Выпуск Active Directory Basic — это платный выпуск системы Azure Active Directory, имеющий следующие возможности.

- **Фирменная символика компании** — чтобы обеспечить конечным пользователем еще более удобные условия работы, вы можете добавить на страницах входа и панели доступа своей организации логотип и цветовые схемы компании. После добавления логотипа также можно добавить локализованные версии логотипа для различных языков и локалей. Дополнительные сведения см. в разделе [Добавление фирменной символики компании на страницах входа и панели доступа](active-directory-add-company-branding.md).
- **Доступ к приложениям на основе групп** — группы позволяют выполнять подготовку пользователей и назначать в пакетном режиме пользователям доступ к тысячам различных приложений SaaS. Эти группы можно создавать только в облаке либо использовать существующие группы, которые были синхронизированы из вашей локальной системы Active Directory. Дополнительные сведения см. в разделе [Назначение группе доступа к приложению SaaS в Azure AD](active-directory-accessmanagement-group-saasapps.md).
- **Самостоятельный сброс пароля** — Azure всегда позволяет администраторам каталога сбрасывать пароли. С помощью Azure Active Directory Basic можно сократить количество обращений в службу технической поддержки, если пользователи забыли пароль, предоставляя всем пользователям в каталоге возможность сброса своего пароля благодаря применению такого же механизма входа, что и для Office 365. Дополнительные сведения см. в статье [Управление паролями из любой точки мира](active-directory-passwords.md).
- **SLA корпоративного уровня до 99,9%** — для службы Azure Active Directory Basic мы гарантируем минимальный уровень доступности 99,9 %.
- [**Предоставление безопасного удаленного доступа к локальным приложениям**](active-directory-application-proxy-get-started.md). Предоставьте вашим сотрудникам безопасный доступ к локальным приложениям, например SharePoint, Exchange и Outlook Web App из облака с помощью Azure Active Directory.

## Возможности Azure Active Directory Premium

Active Directory Premium Edition — это платный выпуск Azure Active Directory, предоставляющий всю функциональность выпусков Free и Basic, а также следующие дополнительные возможности.

- **Самостоятельное управление группами** — Azure Active Directory Premium упрощает ежедневный процесс администрирования групп, позволяя пользователям создавать группы, запрашивать доступ к другим группам, делегировать права владения группами, чтобы другие пользователи могли утверждать запросы и управлять членством в своих группах.

    Дополнительные сведения см. в разделе [Самостоятельное управление группами для пользователей в Azure AD](https://msdn.microsoft.com/library/azure/dn641267.aspx).

- **Расширенные отчеты системы безопасности и оповещения** — Обеспечение контроля и защиты доступа к вашим облачным приложениям с помощью подробных журналов, где подробно регистрируются все аномалии, а также шаблонных отчетов о попытках несогласованного доступа. Расширенные отчеты основаны на машинном самообучении и позволяют получать новую ценную информацию для повышения уровня безопасности доступа и реагирования на потенциальные угрозы.

    Дополнительные сведения см. в разделе [Просмотр отчетов о доступе и использовании](active-directory-view-access-usage-reports.md).

- **Многофакторная проверка подлинности**. Теперь функция многофакторной проверки подлинности включена в выпуск Premium и позволяет защищать доступ к локальным приложениям (VPN, RADIUS и т. д.), к Azure, к службам Microsoft Online Services, например Office 365 и Dynamics CRM Online, а также к тысячам сторонних облачных служб, которые уже интегрированы с Azure Active Directory. Просто включите многофакторную проверку подлинности для удостоверений Azure Active Directory, и система будет предлагать пользователям настроить дополнительную проверку при очередном входе.

    Дополнительную информацию см. в статье [Что такое Azure Multi-Factor Authentication?](multi-factor-authentication.md).

- **Диспетчер удостоверений Microsoft** — выпуск Премиум позволяет предоставлять права на использование сервера MIM (и клиентских лицензий) в вашей локальной сети для поддержки любого сочетания решений гибридной идентификации. Это очень удобно в тех случаях, когда имеется несколько локальных каталогов и баз данных, которые необходимо синхронизировать непосредственно в Azure Active Directory. Ограничения на количество серверов MIM, которые можно использовать, отсутствуют, однако клиентские лицензии MIM предоставляются с учетом выделения лицензии пользователя на Azure Active Directory Premium.

    Дополнительные сведения см. в статье [Диспетчер удостоверений Microsoft](http://www.microsoft.com/ru-RU/server-cloud/products/microsoft-identity-manager/default.aspx).

- **SLA корпоративного уровня до 99,9 %** — для службы Azure Active Directory Premium мы гарантируем минимальный уровень доступности 99,9 %.

    Дополнительные сведения см. в статье [Соглашение об уровне обслуживания для Active Directory Premium](http://azure.microsoft.com/support/legal/sla/).

- **Сброс пароля с обратной записью** — самостоятельно сбрасываемый пароль можно записывать обратно в локальные каталоги.

- [Azure Active Directory Connect Health](active-directory-aadconnect-health.md): наблюдение за работоспособностью локальной инфраструктуры Active Directory и получение аналитических данных об использовании.

## Текущие возможности общедоступной предварительной версии

В настоящее время в общедоступной предварительной версии доступны или будут скоро добавлены следующие возможности.

- [Административные единицы](https://msdn.microsoft.com/library/azure/dn832057.aspx): новый контейнер ресурсов Azure Active Directory, которые можно использовать для делегирования административных разрешений подмножествам пользователей и применения политик к подмножествам пользователей.
- [Добавление своих приложений SaaS](https://msdn.microsoft.com/library/azure/dn893637.aspx) в Azure Active Directory.
- Смена пароля для Facebook, Twitter и LinkedIn. Дополнительные сведения см. в [этой статье](http://blogs.technet.com/b/ad/archive/2015/02/20/azure-ad-automated-password-roll-over-for-facebook-twitter-and-linkedin-now-in-preview.aspx).
- Динамическое членство в группах. Дополнительные сведения см. в [этой статье](https://msdn.microsoft.com/library/azure/dn913807.aspx).
- [Условный доступ](https://msdn.microsoft.com/library/azure/dn906877.aspx): многофакторная проверка подлинности для каждого приложения.
- Интеграция приложения отдела кадров: Workday
- Privileged Identity Management: улучшенный контроль над управлением удостоверениями, упрощающий соблюдение условий соглашений об уровне обслуживания и обеспечение соответствия нормативным требованиям.
- Самостоятельный запрос приложений: администраторы могут предоставить список приложений SaaS, из которых пользователи могут самостоятельно выбрать те, которые хотят использовать. Приложения становятся доступными сразу или после утверждения.
- API создания и просмотра отчетов Azure: данные каждого отчета безопасности Azure Active Directory будут доступны в других инструментах мониторинга или системах SIEM.

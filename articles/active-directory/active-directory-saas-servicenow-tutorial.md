#Руководство: интеграция Azure Active Directory с ServiceNow
>[AZURE.TIP]Чтобы оставить отзыв, нажмите [здесь](http://go.microsoft.com/fwlink/?LinkId=521880).
  
Цель данного учебника — показать интеграцию Azure и ServiceNow.  
Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент в ServiceNow
  
После завершения этого руководства пользователи Azure AD, назначенные ServiceNow, получат возможность единого входа в приложение на корпоративном веб-сайте ServiceNow (вход, инициированный поставщиком услуг) или с помощью инструкций из статьи [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).
  
Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для ServiceNow
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-servicenow-tutorial/IC769496.png "Сценарий")
##Включение интеграции приложений для ServiceNow
  
В этом разделе показано, как включить интеграцию приложений для ServiceNow.

###Чтобы включить интеграцию приложений для ServiceNow, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-servicenow-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-servicenow-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-servicenow-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** щелкните **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-servicenow-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В поле **Поиск** введите **ServiceNow**.

    ![Коллекция приложений](./media/active-directory-saas-servicenow-tutorial/IC701016.png "Коллекция приложений")

7.  В области результатов выберите **ServiceNow** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![ServiceNow](./media/active-directory-saas-servicenow-tutorial/IC701017.png "ServiceNow")
##Настройка единого входа
  
В этом разделе показано, как разрешить пользователям проходить проверку подлинности в ServiceNow со своей учетной записью Azure AD, используя федерацию на основе протокола SAML.

В рамках этой процедуры вам потребуется загрузить сертификат в кодировке Base-64 в свой клиент Dropbox for Business. Если вы не знакомы с этой процедурой, посмотрите видео [Преобразование двоичного сертификата в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **ServiceNow** портала Azure AD щелкните **Настройка единого входа**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-servicenow-tutorial/IC749323.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в ServiceNow?** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-servicenow-tutorial/IC749324.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес для входа в ServiceNow** введите свой URL-адрес, используя следующий шаблон *https://<InstanceName>.servicenow.com*, а затем нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-servicenow-tutorial/IC769497.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в ServiceNow** нажмите кнопку **Загрузить сертификат**, сохраните файл сертификата на локальном компьютере, затем нажмите **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-servicenow-tutorial/IC749325.png "Настройка единого входа")

5.  В клиенте ServiceNow на панели навигации слева нажмите **Свойства**, чтобы открыть страницу **Свойства единого входа SAML 2.0**.


6. На странице **Свойства единого входа SAML 2.0** выполните следующие действия.

     6.1. В разделе **Включить внешнюю проверку подлинности** выберите **Да**.


     6.2. В текстовом поле **URL-адрес поставщика удостоверений, который выпустит маркер безопасности SAML2 с информацией о пользователе** введите **https://sts.windows.net/<GUID вашего клиента> /**.


     6.3. В текстовом поле **Базовый URL-адрес службы AuthnRequest поставщика удостоверений** введите **https://login.windows.net/<GUID вашего клиента>/saml2**.


     6.4. В текстовом поле **Базовый URL-адрес службы SingleLogoutRequest поставщика удостоверений** введите **https://login.windows.net/<GUID вашего клиента>/saml2**.


     6.5. В текстовом поле **Привязка протокола для службы SingleLogoutRequest поставщика удостоверений** наберите **urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect**.

     6.6. В поле **Sign LogoutRequest** выберите **Да**.

     6.7. В текстовом поле **При неудачном едином входе SAML 2.0 из-за того, что сеанс не прошел проверку подлинности, или из-за того, что это первый вход пользователя в систему, выполнить перенаправление на следующий URL-адрес** введите **https://login.windows.net/<GUID вашего клиента>/saml2**.

  

7. В разделе **Свойства поставщика службы (Service-Now)** выполните следующие действия.

     7.1. В текстовом поле **URL-адрес домашней страницы экземпляра ServiceNow** введите URL-адрес домашней страницы экземпляра ServiceNow. URL-адрес домашней страницы экземпляра ServiceNow представляет собой объединение **URL-адреса клиента ServiceNow** и **/navpage.do**: **https://<InstanceName>.service-now.com/navpage.do**. <br><br> ![Домашняя страница экземпляра ServiceNow](./media/active-directory-saas-servicenow-tutorial/IC700342.png "Домашняя страница экземпляра ServiceNow")


     7.2. В текстовом поле **Идентификатор сущности, или издатель** введите URL-адрес клиента.

     7.3. В текстовом поле **URI аудитории, которая принимает маркер SAML2** введите URL-адрес клиента.

     7.4. В текстовом поле **Поле таблицы пользователей, которое соответствует элементу NameID субъекта в SAMLResponse** введите **email**.

     7.5. В текстовом поле **Политика NameID для возвращения NameID субъекта в SAMLResponse** введите **urn:oasis:names:tc:SAML:1.1:nameid-format:unspecified**.

     7.6 Оставьте флажок **Создать запрос AuthnContextClass в инструкции AuthnRequest** снятым.

     7.7 В текстовом поле **метод AuthnContextClassRef, который будет включен в запрос AuthnRequest SAML 2.0 к поставщику удостоверений** введите **http://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/password**.



8. В разделе **Дополнительные параметры** выполните следующие действия.

     8.1. В текстовом поле **количество секунд перед ограничением notBefore или после ограничения notOnOrAfter, которое все еще считается действительным** введите **60**.


9. Чтобы сохранить конфигурацию, нажмите кнопку **Сохранить**.

10. На панели навигации слева щелкните **Сертификат**, чтобы открыть страницу **Сертификат**.



11. Для загрузки сертификата выполните следующие действия на странице сертификата.

     11.1. Нажмите кнопку **Создать**.

     11.2. В текстовом поле **Название** введите **SAML 2.0**.

     11.3. Выберите **Активно**.

     11.4. В поле **Формат** выберите **PEM**.

     11.5. Создайте файл в кодировке Base-64 из загруженного сертификата. >[AZURE.NOTE]Дополнительные сведения см. в разделе [Преобразование двоичного сертификата в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

     11.6. В **Блокноте** откройте файл в кодировке Base-64 и скопируйте содержимое этого файла в буфер обмена.

     11.7. Вставьте содержимое буфера обмена в текстовое поле **Сертификат PEM**.

     11.8. Нажмите кнопку **Submit** (Отправить).



12. На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку Завершить, чтобы закрыть диалоговое окно Настройка единого входа.<br><br> ![Настройка единого входа](./media/active-directory-saas-servicenow-tutorial/IC749326.png "Настройка единого входа")




##Настройка подготовки учетных записей пользователей
  
В этом разделе показано, как включить подготовку учетных записей пользователей Active Directory для ServiceNow.


### Чтобы настроить подготовку учетных записей пользователей, выполните следующие действия.

1. На странице интеграции приложения **ServiceNow** портала управления Azure нажмите **Настроить подготовку учетных записей пользователей**. <br><br> ![Подготовка пользователей](./media/active-directory-saas-servicenow-tutorial/IC769498.png "Подготовка пользователей")


2. На странице **Введите учетные данные ServiceNow, чтобы включить автоматическую подготовку пользователей** укажите следующие параметры конфигурации.

     2.1. В текстовом поле **Имя экземпляра ServiceNow** введите имя экземпляра ServiceNow.

     2.2. В текстовом поле **Имя пользователя администратора ServiceNow** введите имя учетной записи администратора ServiceNow.

     2.3. В текстовом поле **Пароль администратора ServiceNow** введите пароль для этой учетной записи.

     2.4. Нажмите **проверить** для проверки конфигурации.

     2.5. Нажмите кнопку **Далее**, чтобы открыть страницу **Дальнейшие шаги**.

     2.6. Если вы хотите подготовить всех пользователей для этого приложения, выберите "**Автоматически подготовить все учетные записи пользователей в каталоге для этого приложения**". <br><br> ![Дальнейшие действия](./media/active-directory-saas-servicenow-tutorial/IC698804.png "Дальнейшие действия")

     2.7. На странице **Дальнейшие шаги** нажмите **Завершить**, чтобы сохранить конфигурацию.











##Назначение пользователей
  
Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которые должны использовать приложение, доступ путем их назначения.

###Чтобы назначить пользователей ServiceNow, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **ServiceNow** нажмите **Назначить пользователей**.

    ![Назначить пользователей](./media/active-directory-saas-servicenow-tutorial/IC769499.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да** для подтверждения назначения.

    ![Да](./media/active-directory-saas-servicenow-tutorial/IC767830.png "Да")
  
Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).

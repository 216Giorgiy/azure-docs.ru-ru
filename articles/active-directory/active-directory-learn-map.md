# Дополнительные сведения об Azure AD

###Другие статьи на эту тему
[Что такое Azure AD?](active-directory-whatis.md)<br> [Как это работает?](active-directory-works.md)<br> [Приступая к работе](active-directory-get-started.md)<br> [Дальнейшие действия](active-directory-next-steps.md)<br> [Подробнее](active-directory-learn-map.md)

У меня уже есть подписка на Azure. Как с этим связана Azure AD? [Как подписки связаны с каталогом](active-directory-how-subscriptions-associated-directory.md)

Для выполнения административных задач Azure AD, таких как управление пользователями, управление доменом или настройка единого входа, используйте модуль Azure Active Directory для командлетов Windows PowerShell. [Управление Azure AD с помощью Windows PowerShell](https://msdn.microsoft.com/library/azure/jj151815.aspx)

Используйте Azure Active Directory Graph API для программного доступа к Azure AD через конечные точки REST API. [Azure AD Graph API](https://msdn.microsoft.com/library/azure/hh974476.aspx)

Можно добавить имя пользовательского домена в Azure Active Directory, чтобы использовать его в Microsoft Online Services. [Интернет-управление доменом](active-directory-add-domain.md)

Чтобы понять, какие ограничения в использовании и ограничения по другим службам применимы к службе Azure Active Directory, см. раздел [Лимиты и ограничения службы Azure Active Directory](active-directory-service-limits-restrictions.md)

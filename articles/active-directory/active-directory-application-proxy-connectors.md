# Публикация приложений в отдельных сетях и расположениях с помощью групп соединителей

> [AZURE.NOTE]Прокси приложения — это функция, которая доступна только после обновления до выпуска Premium или Basic службы Azure Active Directory. Дополнительные сведения см. в статье [Выпуски Azure Active Directory](active-directory-edition.md).

Группы соединителей удобно использовать в нескольких различных сценариях, включая следующие:


- сайты с несколькими взаимосвязанными центрами обработки данных, для которых требуется передавать за пределы центров обработки данных минимальный объем трафика и не использовать перекрестные каналы связи между центрами, которые обычно дороги и имеют высокую задержку. В каждом центре обработки данных вы можете развернуть соединители для обслуживания только тех приложений, которые размещаются в данном центре обработки данных, это обеспечивает экономию на перекрестных каналах связи и делает работу пользователей абсолютно прозрачной.
- Управление приложениями, которые установлены в изолированных сетях, не являющихся частью главной корпоративной сети. Группы соединителей можно использовать для установки выделенных соединителей для изолированных сетей, чтобы ограничить область действия приложений соответствующей сетью.
- Для приложений, установленных на платформе IaaS для доступа к облаку, группы соединителей предоставляют общую службу для безопасного доступа к ним без создания дополнительной зависимости от корпоративной сети или фрагментирования процедуры взаимодействия. Соединители можно установить в каждом облачном центре обработки данных для обслуживания только тех приложений, которые размещены в этой сети. Можно установить несколько соединителей для обеспечения высокой доступности.
- Поддержка сред с несколькими лесами, в которых для леса можно развернуть отдельные соединители, обрабатывающие определенные приложения.
- Группы соединителей могут применяться на сайтах аварийного восстановления для определения отработки отказа или в качестве резерва для основного сайта.
- Группы соединителей также можно использовать для обслуживания нескольких компаний из одного клиента.


## Работа с группами соединителей
Чтобы сгруппировать ваши соединители, необходимо убедиться, что вы [установили несколько соединителей](active-directory-application-proxy-enable.md), после чего их нужно назвать и сгруппировать. После этого их необходимо назначить конкретным приложениям.

## Шаг 1. Создание групп соединителей
Можно создать любое число групп соединителей. Для создания группы соединителей используется портал Azure. Выберите свой каталог и щелкните элемент **Настройка**. В разделе "Прокси-сервер приложений" щелкните элемент **Управление группами соединителей** и создайте новую группу соединителей, указав ее имя: ![](./media/active-directory-application-proxy-connectors/app_proxy_connectors_creategroup.png) ![](./media/active-directory-application-proxy-connectors/app_proxy_connectors_namegroup.png)
## Шаг 2. Назначение соединителей группам
После создания групп соединителей переместите соединители в соответствующую группу. В разделе **Прокси-сервер приложений** щелкните элемент **Управление соединителями**, а затем в разделе **Группы** выберите группу для каждого соединителя. Обратите внимание, что для того, чтобы соединители стали активными в новой группе, может потребоваться до 10 минут. ![](./media/active-directory-application-proxy-connectors/app_proxy_connectors_connectorlist.png)
## Шаг 3. Назначение приложений группам соединителей
Последний шаг заключается в назначении каждого приложения той группе соединителей, которая будет обслуживать его. В вашем каталоге на портале Azure выберите приложение, которое необходимо назначить группе, и щелкните элемент **Настройка**. В разделе **Группа соединителей** выберите группу, которую должно использовать приложение. Это изменение применяется немедленно. ![](./media/active-directory-application-proxy-connectors/app_proxy_connectors_newgroup.png) Дополнительные сведения о публикации приложений см. в разделе [Публикация приложений с помощью доменного имени](active-directory-application-proxy-custom-domains.md).

# Приступая к работе с Azure Active Directory Premium

Существуют три выпуска Azure Active Directory: Free, Basic и Premium. Выпуск Free входит в состав подписки Azure или Office 365. Выпуски Basic и Premium доступны через [Соглашение Microsoft Enterprise Agreement](https://www.microsoft.com/ru-ru/licensing/licensing-programs/enterprise.aspx) или в рамках программы [Открытая корпоративная лицензия](https://www.microsoft.com/ru-ru/licensing/licensing-programs/open-license.aspx). Подписчики Azure и Office 365 также могут купить выпуск Active Directory Premium через Интернет. [Войдите здесь](https://portal.office.com/Commerce/Catalog.aspx), чтобы купить выпуск.

> [AZURE.NOTE]Выпуски Azure Active Directory Premium и Basic доступны для клиентов в Китае, использующих всемирный экземпляр Azure Active Directory. Azure Active Directory Premium и Basic в настоящее время не поддерживаются в службе Microsoft Azure под управлением 21Vianet в Китае. Чтобы получить дополнительную информацию, свяжитесь с нами на [форуме Azure Active Directory](http://feedback.azure.com/forums/169401-azure-active-directory).

Выпуск Azure Active Directory Premium также включен в **набор корпоративных служб мобильности**. Набор корпоративных служб мобильности представляет собой экономичный способ, благодаря которому организации могут совместно использовать службы Microsoft Intune, Azure Rights Management и Active Directory Premium в рамках одного плана лицензирования. Дополнительные сведения см. на веб-сайте [Набор корпоративных служб мобильности](https://www.microsoft.com/ru-ru/server-cloud/enterprise-mobility/overview.aspx).

Чтобы начать использовать выпуск Azure Active Directory Premium, выполните следующие действия. Эти же действия относятся к выпуску Azure Active Directory Basic.

## Шаг 1. Регистрация на Active Directory Premium

Для регистрации зайдите на веб-сайт [Корпоративное лицензирование](http://www.microsoft.com/ru-ru/licensing/how-to-buy/how-to-buy.aspx).

## Шаг 2. Активирование плана лицензирования

Если вы впервые приобрели план лицензирования по программе корпоративного лицензирования корпорации Майкрософт, перед назначением лицензий из вашего каталога Azure Active Directory сначала вам необходимо активировать план лицензирования. Для этого необходимо щелкнуть ссылку входа или регистрации в электронном письме с подтверждением (см. ниже), которое вы получите после первой своей покупки плана лицензирования. При последующих покупках для этого каталога лицензии будут автоматически активироваться в этом же каталоге.

![][1]

При наличии существующего клиента щелкните ссылку **Вход** для входа с использованием вашей существующей учетной записи администратора. Для входа необходимо использовать учетные данные глобального администратора из каталога, в котором должны быть активированы лицензии.

Если вы хотите создать нового клиента Azure Active Directory для использования со своим планом лицензирования, щелкните ссылку **Регистрация**, в результате чего вы перейдете на следующий экран.

![][2]

После завершения процедуры регистрации или входа, которая была инициирована из электронного письма, вы увидите следующий экран с подтверждением, что план лицензирования активируется для вашего клиента.

![][3]

## Шаг 3. Активирование доступа к Azure Active Directory

После подготовки лицензий для вашего каталога вы получите приветственное сообщение (см. ниже), подтверждающее, что вы можете приступить к управлению своими лицензиями на Azure Active Directory Premium или набором корпоративных служб мобильности и их компонентами. Если ранее вы уже использовали Microsoft Azure, можно приступить к http://manage.windowsazure.com для назначения новых лицензий (см. шаг 4 ниже для получения дальнейших инструкций). Если вы не знакомы с Microsoft Azure, нажмите ссылку входа в электронном письме или перейдите к разделу [Доступ к странице активации Azure Active Directory](https://account.windowsazure.com/signup?offer=MS-AZR-0110P), после чего вы сможете выполнить необходимые операции для доступа к своему каталогу через портал управления Azure.

![][4]

После успешного входа будет нужно ввести данные на экране двухфакторной проверки подлинности (см. ниже) — введите номер мобильного телефона и проверьте его. После этого вы сможете активировать доступ к Azure Active Directory с помощью ссылки **Регистрация**.

![][5]

Активация может занять несколько минут, как показано ниже. Как только доступ станет активен, коричневая панель исчезнет, и вы сможете щелкнуть ссылку портала в правом верхнем углу или перейти на [портала управления Azure](http://manage.windowsazure.com).

![][6]

В этом случае доступ к Azure будет ограничен системой Azure Active Directory.

![][7]

Вы уже могли ранее получать доступ к Azure. Кроме того, доступ к Azure Active Directory можно обновить до полного доступа к Azure путем активации дополнительных подписок Azure. В этих случаях на портале управления станут доступны дополнительные возможности, как показано ниже.

![][8]

При попытке активации доступа к Azure Active Directory до получения приветственного сообщения, указанного выше, может появиться следующее сообщение об ошибке. Повторите попытку через несколько минут после получения почтового сообщения.

![][9]

Новые администраторы в вашей подписке также могут активировать свой доступ к порталу управления по этой ссылке.

## Шаг 4. Назначение лицензии учетным записям пользователей

Перед началом использования приобретенного плана необходимо вручную назначить лицензии учетным записям пользователей вашей организации, чтобы они могли использовать все возможности выпуска Premium. Для назначения лицензий пользователям, чтобы они могли использовать возможности выпуска Azure Active Directory Premium, выполните следующие действия.

Для назначения лицензий пользователям

1. Войдите на портал управления как глобальный администратор каталога, который необходимо настроить.
2. Щелкните **Active Directory**, а затем выберите каталог, где требуется назначить лицензии.
3. Перейдите на вкладку **Лицензии**, выберите **Active Directory Premium** или **Набор корпоративных служб мобильности**, а затем нажмите кнопку **Назначить**.

    ![][10]

4. В диалоговом окне выберите пользователей, которым требуется назначить лицензии, и щелкните значок галочки, чтобы сохранить изменения.

    ![][11]

## Лицензионные ограничения

Некоторые планы лицензирования являются подмножествами или супермножествами других планов лицензирования. В большинстве случаев пользователю нельзя назначить план лицензирования, который уже был ему назначен. Если вы хотите назначить план лицензирования, который является супермножеством, сначала необходимо удалить план лицензирования для подмножества.

## Требования лицензий

При назначении пользователю лицензии можно указать основное местоположение использования в свойствах его учетной записи, как показано ниже. Если место использования не указано, местоположение клиента будет автоматически назначено пользователю.

![][12]

Доступность служб и компонентов для облачной службы Майкрософт зависит от страны или региона. Служба, например голосовая связь через Интернет (VoIP), может быть доступна в одной стране или регионе и недоступна в другой. В некоторых странах или регионах компоненты в службе могут быть ограничены по юридическим причинам. Чтобы выяснить, имеются ли ограничения на службу или компонент, найдите нужную страну или регион на сайте ограничений лицензий для службы.

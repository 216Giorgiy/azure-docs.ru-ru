# <a name="azure-active-directory-deployment-plans"></a>Планы развертывания Azure Active Directory
Ищете комплексное руководство по развертыванию некоторых возможностей Azure Active Directory (Azure AD)? В следующих планах развертывания рассматриваются бизнес-преимущества, вопросы планирования, разработки и операционные процедуры, необходимые для успешного развертывания нескольких наиболее распространенных возможностей Azure AD. 

В документах вы найдете шаблоны электронной почты, схемы системной архитектуры, общие тестовые случаи и многое другое. 

Мы будем рады вашим отзывам. Пройдите этот короткий [опрос](https://aka.ms/deploymentplanfeedback), чтобы оценить пользу приведенных документов. 

|Сценарий |ОПИСАНИЕ |
|-|-|
|[Единый вход](https://aka.ms/SSODPDownload)|Единый вход позволяет вам получить доступ ко всем приложениям и ресурсам, которые необходимы для ведения бизнеса. При этом выполняется однократный вход с использованием одной учетной записи пользователя. После входа в систему вы можете перейти из Microsoft Office в SalesForce, а затем в Box без повторной проверки подлинности (например, ввода пароля).|
|[Подготовка пользователей](https://aka.ms/UserProvisioningDPDownload)|Служба Azure AD помогает автоматизировать процесс создания, обслуживания и удаления удостоверений пользователей в облачных приложениях (SaaS), таких как Dropbox, Salesforce, ServiceNow и других.|
|[Многофакторная идентификация](https://aka.ms/MFADPDownload)|Многофакторная идентификация (MFA) Azure — решение Майкрософт для двухэтапной проверки. С помощью утвержденных администратором методов проверки подлинности Azure MFA помогает защитить доступ к данным и приложениям, при этом не усложняя простой процесс входа в систему.|
|[Условный доступ](https://aka.ms/CADPDownload)|Благодаря условному доступу вы можете внедрять автоматизированные решения управления доступом, определяющие, кто может получать доступ к облачным приложениям, исходя из условий.|
|[ADFS для сквозной проверки подлинности](https://aka.ms/ADFSTOPTADPDownload)|Сквозная проверка подлинности Azure AD позволяет пользователям входить в локальные и облачные приложения, используя те же пароли. Эта функция облегчает работу пользователей, так как им не нужно запоминать на один пароль больше, и сокращает затраты на ИТ-поддержку, так как снижается вероятность того, что пользователи забудут процедуру входа. Если пользователи выполняют вход с помощью Azure AD, эта функция проверяет пароли пользователей непосредственно в локальной службе Active Directory.|
|[ADFS для синхронизации хэша паролей](https://aka.ms/ADFSTOPHSDPDownload)|При синхронизации хэша паролей хэши паролей пользователей синхронизируются из локальной службы Active Directory в Azure AD, позволяя Azure AD выполнять проверку подлинности пользователей без взаимодействия с локальной службой Active Directory.|
|[Простой единый вход](https://aka.ms/SeamlessSSODPDownload)|Простой единый вход Azure Active Directory автоматически обеспечивает пользователям вход в систему, когда они работают на корпоративных устройствах, подключенных к корпоративной сети. Если он включен, пользователям не нужно вводить пароль для входа в Azure AD, а в большинстве случаев — даже вводить имя пользователя. Эта функция предоставляет пользователям удобный доступ к облачным приложениям и не требует установки каких-либо дополнительных локальных компонентов.|
|[Самостоятельный сброс пароля](https://aka.ms/SSPRDPDownload)|Самостоятельный сброс пароля дает пользователям возможность сбрасывать пароль без вмешательства администратора, где и когда им нужно.|
|[Azure Active Directory Application Proxy](https://aka.ms/AppProxyDPDownload)|Сегодня сотрудникам нужно продуктивно работать без привязки к расположению, времени и оборудованию. Они предпочитают использовать собственные устройства — планшеты, телефоны и ноутбуки. Кроме того, они хотят иметь постоянный доступ к своим приложениям, как к приложениям SaaS в облаке, так и к приложениям в корпоративной локальной сети. Традиционно для доступа к локальным приложениям требуются виртуальные частные сети (VPN) или сети периметра. Помимо того, что такие решения довольно сложны и их непросто защитить, на их настройку и управление требуются немалые средства. Но есть более эффективный способ! Это Azure Active Directory Application Proxy.|

# Как активировать и деактивировать роли в компоненте управления привилегированными пользователями Azure AD

Компонент управления привилегированными пользователями (PIM) Azure Active Directory (AD) упрощает управление пользователями с привилегированными удостоверениями и их доступом к ресурсам в Azure AD и других веб-службах Майкрософт, таких как Office 365 или Microsoft Intune.

Эта статья предназначена для администраторов, которым требуется активировать свои роли в компоненте управления привилегированными пользователями Azure AD. В ней приводится процедура активации роли для включения необходимых разрешений и деактивации роли после завершения работы с ней.

Если вам назначена роль администратора, вы можете активировать эту роль, когда вам потребуется выполнить задачу, связанную с этой ролью. Например, если вам требуется только изредка управлять Office 365, администратор привилегированных ролей вашей организации, скорее всего, не назначит вас постоянным администратором. Вместо этого он назначит вас кандидатом для ролей "Глобальный администратор" или "Администратор Exchange Online" в Azure AD. Это означает, что при необходимости вы можете запросить назначение временной роли и получите административный доступ к Office 365 в течение заданного периода времени.


## Добавление приложения для управления привилегированными пользователями

Используйте приложение для управления привилегированными пользователями Azure AD на [портале Azure](https://portal.azure.com/) для запроса "активации" роли, даже если вы собираетесь работать на другом портале или с помощью PowerShell. Если приложение Azure AD PIM еще не установлено на портале Azure, выполните следующие действия, чтобы приступить к работе.

1. Войдите на [портал Azure](https://portal.azure.com/), если вы еще этого не сделали.
2. Если в организации имеется несколько каталогов, щелкните свое имя пользователя в правом верхнем углу портала Azure и выберите каталог, с которым вы будете работать.
3. Щелкните значок **Создать** в области навигации слева.
4. В меню "Создание" выберите пункт **Безопасность и идентификатор**.
5. Выберите **Управление привилегированными пользователями Azure AD**.
6. Установите флажок **Закрепить на панели мониторинга** и нажмите кнопку **Создать**. Откроется приложение "Управление привилегированными пользователями".

## Активация роли

Когда вам потребуется какая-либо роль, вы можете запросить ее активацию с помощью кнопки **Активация роли** в приложении для управления привилегированными пользователями Azure AD.


1. Войдите на [портал Azure](https://portal.azure.com/) и выберите элемент "Управление привилегированными пользователями Azure AD".
2. Нажмите кнопку **Активация роли**. Отобразится список назначенных вам ролей.
3. Выберите роль, которую необходимо активировать.
4. Выберите **Активировать**. Отобразится колонка **Запрос активации роли**.
5. Для активации некоторых ролей, таких как глобальный администратор, потребуется пройти многофакторную проверку подлинности (MFA). Если вы не проходили MFA при входе в систему, ее может потребоваться пройти перед активацией роли.
6. В текстовом поле введите основание для запроса активации. Возможно, администратору привилегированных ролей потребуется также указать номер запроса на устранение неисправности.
7. Нажмите кнопку **ОК**. Роль будет активирована, и сведения об изменении роли станут доступны в службах Microsoft Online Services.

## Деактивация роли

После активации роли она автоматически отключается по истечении заданного периода времени.

При необходимости роль можно отключить и раньше — вручную в приложении для управления привилегированными пользователями Azure AD. Выберите **Активация роли**, найдите роль, которая вам сейчас больше не требуется, и выберите **Деактивировать**.

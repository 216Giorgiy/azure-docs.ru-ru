# Уведомления отчетов Azure Active Directory

## Какие отчеты создают уведомления по электронной почте

На данный момент система уведомлений по электронной почте используется только в отчете об аномальных событиях при входе и отчете о пользователях с аномальными событиями при входе.

## Как происходит отправка уведомлений по электронной почте?

По умолчанию Azure Active Directory автоматически отправляет уведомления по электронной почте всем глобальным администраторам. Сообщения для каждого отчета отправляются при следующих условиях.

Для отчета об аномальных событиях при входе

- Неизвестные источники: 10 событий
- Несколько сбоев: 10 событий
- IP-адреса с подозрительными действиями: 10 событий
- Инфицированные устройства: 10 событий

Для отчета о пользователях с аномальными событиями при входе

- Неизвестные источники: 10 событий
- Несколько сбоев: 10 событий
- IP-адреса с подозрительными действиями: 10 событий
- Инфицированные устройства: 5 событий
- Отчет об аномальных операциях входа: 15 событий

Сообщение отправляется при удовлетворении любого из вышеуказанных условий в течение 30 дней или после последнего отправленного сообщения электронной почты, если этот срок еще не истек.

Аномальные операции входа — операции, которые алгоритмы машинного обучения определили как «аномальные» на основе неожиданных расположений и времени входа либо их сочетания. Это может означать, что с помощью этой учетной записи пытается войти злоумышленник. См. дополнительную информацию об этом отчете в таблице выше.

## Кто получает уведомления по электронной почте?

Сообщение отправляется всем глобальным администраторам, которым назначена лицензия Active Directory Premium. Чтобы убедиться, что сообщение доставлено, мы также отправляем его на альтернативный адрес электронной почты администраторов. Администраторы должны включить адрес aad-alerts-noreply@mail.windowsazure.com в свой список надежных отправителей, чтобы не пропустить сообщение электронной почты.

## Как часто отправляются эти сообщения?

Такое сообщение будет отправлено, только если в течение 30 дней после отправки последнего сообщения обнаружено не менее 10 новых аномальных событий входа. Как получить доступ к отчету, указанному в сообщении электронной почты

Если щелкнуть ссылку, вы перейдете на страницу отчета на портале управления Azure. Чтобы получить доступ к отчету, необходимо одновременно быть:

- администратором или соадминистратором подписки Azure;
- глобальным администратором в каталоге с назначенной лицензией Active Directory Premium. Дополнительную информацию см. в статье Выпуски Azure Active Directory.

## Можно ли отключить эти сообщения?

Да, чтобы отключить уведомления об аномальных операциях входа на портале управления Azure, щелкните **Настройка** и выберите **Отключено** в разделе **Уведомления**.

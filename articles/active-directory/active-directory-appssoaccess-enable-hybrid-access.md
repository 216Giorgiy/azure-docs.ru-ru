#Гибридный доступ с прокси-сервером приложений
С помощью прокси-сервера приложений Microsoft Azure AD можно опубликовать приложения, выполняющиеся в частной сети, для безопасного доступа к ним с любого устройства, расположенного вне сети. С помощью прокси-сервера приложений Azure AD можно опубликовать приложения, выполняющиеся в локальной или облачной частной сети. Прокси-сервер можно настроить как готовую службу в Azure AD. Для этого достаточно установить соединитель прокси-сервера приложений на сервер в вашей среде.

##Принцип работы

Следующий список шагов иллюстрирует процесс публикации приложений в гибридной среде.
 
1.	В корпоративной сети развертываются соединители. (Для обеспечения избыточности и масштабирования можно развернуть несколько соединителей.)
2.	Соединитель автоматически подключается к облачной службе.
3.	Пользователь подключаются к облачной службе, которая направляет его трафик к ресурсам через соединитель.

Дополнительные сведения о том, как работает прокси-сервер приложений, представлены на следующей схеме.

 ![Как работает прокси-сервер приложений AzureAD](./media/active-directory-appssoaccess-whatis/azureappproxxy.png)

 
1.	Пользователь обращается к приложению через прокси-сервер приложений и перенаправляется на страницу входа в Azure AD для проверки подлинности.
2.	После успешного входа в систему создается маркер и отправляется пользователю.
3.	Пользователь отправляет маркер прокси-серверу приложений, который извлекает из маркера имя субъекта пользователя (UPN) и имя субъекта безопасности (SPN) и направляет запрос соединителю.
4.	Соединитель от имени пользователя запрашивает билет Kerberos, который может использоваться для внутренней проверки подлинности (Windows). (Ограниченное делегирование Kerberos)
5.	 Active Directory выдает билет Kerberos.
6.	Полученный билет отправляется серверу приложений, на котором он проверяется.
7.	 Ответ отправляется конечному пользователю через прокси-сервер приложений. Существует только одно требование для публикации веб-приложения: веб-приложение в частной сети должно быть доступно серверу, на котором установлен соединитель. Другими словами, соединитель можно установить либо на самих серверах приложений, либо на любом другом сервере в вашей среде. Единственное условие — соединитель должен иметь доступ к веб-приложению.

##Интеграция локальных приложений с помощью прокси-сервера приложений Azure AD
С помощью панели доступа в Azure AD можно опубликовать локальные приложения в Azure AD. Для этого необходимо выполнить следующие действия.

1. Включить прокси-сервер приложений в Azure AD, установить и зарегистрировать соединитель. Подробные инструкции см. в статье [Включение прокси-сервера приложений Azure AD](active-directory-application-proxy-enable/#step-1-enable-application-proxy-in-azure-ad.md).
2. Опубликовать приложения с помощью прокси-сервера приложений Azure AD. Подробные инструкции см. в статье [Публикация приложений с помощью прокси-сервера приложений Azure AD](active-directory-application-proxy-publish.md).

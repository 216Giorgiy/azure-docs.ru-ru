# <a name="how-to-control-the-hybrid-azure-ad-join-of-your-devices"></a>Как управлять гибридным присоединением устройства к Azure AD

Гибридное присоединение к Azure AD — это процесс автоматической регистрации ваших локальных присоединенных к домену устройств в Azure AD. Бывают случаи, когда не нужно, чтобы все ваши устройства регистрировались автоматически. Например, это актуально во время начального развертывания, когда необходимо убедиться, что все работает правильно.

В этой статье предоставляются рекомендации по управлению гибридным присоединением устройств к Azure AD. 


## <a name="prerequisites"></a>Предварительные требования

В данной статье предполагается, что вы знакомы с:

-  [Общие сведения об управлении устройствами в Azure Active Directory](../device-management-introduction.md)
 
-  [Как планировать гибридную реализацию присоединения к Azure Active Directory](hybrid-azuread-join-plan.md)

-  Настройка гибридного присоединения к Azure Active Directory для [управляемых доменов](hybrid-azuread-join-managed-domains.md) или [федеративных доменов](hybrid-azuread-join-federated-domains.md).



## <a name="control-windows-current-devices"></a>Управление текущими устройствами Windows

Для устройств под управлением Windows Desktop поддерживается юбилейное обновление Windows 10 (версия 1607) или более поздняя версия. Рекомендуем выполнить обновление до последней версии Windows 10.

Все текущие устройства автоматически регистрируются в Azure AD при запуске устройства или входе пользователя в систему. Этим поведением можно управлять с помощью объекта групповой политики (GPO) или System Center Configuration Manager.

Для управления текущими устройствами Windows требуется выполнить следующее: 

1.  **все устройства**: отключите автоматическую регистрацию устройств.
2.  **выбранные устройства**: включите автоматическую регистрацию устройств.

Если вы убедились, что все работает правильно, то все готово к повторному включению автоматической регистрации для всех устройств.



## <a name="group-policy-object"></a>Объект групповой политики 

Можно управлять поведением регистрации устройств, развернув следующий объект групповой политики: **Register domain-joined computers as devices** (Регистрация присоединенных к домену компьютеров как устройств).

**Чтобы настроить этот объект групповой политики, сделайте следующее**.

1.  Откройте **диспетчер сервера** и выберите **Средства \> Управление групповой политикой**.

2.  Перейдите к узлу домена, на котором нужно включить или отключить автоматическую регистрацию.

3.  Щелкните правой кнопкой мыши **Объекты групповой политики**, а затем выберите **Создать**.

4.  Введите имя (например, *Гибридное присоединение к Azure AD*) для объекта групповой политики. 

5.  Последовательно выберите **ОК**.

6.  Щелкните новый объект групповой политики правой кнопкой мыши, а затем выберите **Изменить**.

7.  Выберите **Конфигурация компьютера \> Политики \> Административные шаблоны \> Компоненты Windows \> Регистрация устройств**. 

8.  Щелкните правой кнопкой мыши пункт **Зарегистрировать подключенные к домену компьютеры в качестве устройств** и выберите **Изменить**.

    > [!NOTE] 
    > В более ранних версиях консоли управления групповыми политиками этот шаблон групповой политики назывался по-другому. Если вы используете более раннюю версию консоли, выберите **Конфигурация компьютера \> Политики \> Административные шаблоны \> Компоненты Windows \> Присоединение к рабочей области \> Automatically workplace join client computers** (Автоматическое присоединение клиентских компьютеров к рабочей области). 

9.  Выберите один из следующих параметров и нажмите кнопку **Применить**:

    - **Отключено** — для предотвращения автоматической регистрации устройств;
    - **Включено** — для включения автоматической регистрации устройств.

10. Последовательно выберите **ОК**.

Необходимо связать объект групповой политики с расположением по своему усмотрению. Например, чтобы задать эту политику для всех текущих присоединенных к домену устройств в вашей организации, свяжите объект групповой политики с доменом. Чтобы выполнять управляемое развертывание, задайте эту политику для присоединенных к домену текущих устройств Windows, относящихся к подразделению или группе безопасности.

### <a name="configuration-manager-controlled-deployment"></a>Развертывание под управлением Configuration Manager 

Можно управлять поведением регистрации текущих устройств, настроив следующий параметр клиента: **Automatically register new Windows 10 domain joined devices with azure Active Directory** (Автоматическая регистрация новых присоединенных к домену устройств Windows 10 в Azure Active Directory).


**Чтобы задать этот параметр клиента, сделайте следующее**.

1.  Откройте **Configuration Manager**, а затем перейдите к разделу **Облачные службы**.

2.  В разделе **Параметры устройства** выберите одно из следующих значений параметра **Automatically register new Windows 10 domain joined devices with azure Active Directory** (Автоматическая регистрация новых присоединенных к домену устройств Windows 10 в Azure Active Directory):

    - **Нет** — для предотвращения автоматической регистрации устройств;
    - **Да** — для включения автоматической регистрации устройств.


3.  Последовательно выберите **ОК**.
    

Необходимо связать этот параметр клиента с расположением по своему усмотрению. Например, чтобы настроить этот параметр клиента для всех текущих устройств Windows в вашей организации, свяжите его с доменом. Чтобы выполнять управляемое развертывание, можно настроить этот параметр клиента для присоединенных к домену текущих устройств Windows, относящихся к подразделению или группе безопасности.

> [!Important]
> Хотя описанная выше настройка относится к существующим устройствам Windows 10, которые уже присоединены к домену, существует вероятность, что устройства, которые только присоединяются к домену, могут попытаться гибридно присоединиться к Azure AD. Это связано с возможной задержкой применения групповой политики или параметров Configuration Manager к устройствам Windows 10, которые только присоединились к домену. Чтобы избежать этого, рекомендуем создать новый образ Sysprep (используется в качестве примера для метода подготовки) на устройстве, которое ранее никогда не было гибридно присоединено к Azure AD и на котором уже действует требуемая групповая политика или клиентские параметры Configuration Manager. Новый образ также необходимо использовать для подготовки новых компьютеров, которые присоединяются к домену вашей организации. 

## <a name="control-windows-down-level-devices"></a>Управление устройствами Windows нижнего уровня

Чтобы регистрировать устройства Windows нижнего уровня, необходимо скачать из Центра загрузки Майкрософт и установить пакет установщика Windows (MSI-файл), опубликованный на странице [Microsoft Workplace Join for non-Windows 10 computers](https://www.microsoft.com/download/details.aspx?id=53554) (Microsoft Workplace Join для компьютеров под управлением ОС, отличной от Windows 10).

Для развертывания пакета используйте систему распространения программного обеспечения, например System Center Configuration Manager. Этот пакет поддерживает параметры стандартной автоматической установки с использованием параметра quiet. В [System Center Configuration Manager](https://www.microsoft.com/cloud-platform/system-center-configuration-manager) доступны дополнительные преимущества предыдущих версий, такие как возможность отслеживать ход выполнения регистрации.

Установщик создает в системе запланированную задачу, которая выполняется в контексте пользователя. Задача запускается в момент входа пользователя в систему Windows. Эта задача автоматически присоединяет устройство к Azure AD, используя учетные данные пользователя, после аутентификации с помощью Azure AD.


В целях контроля регистрации устройств следует развернуть пакет установщика Windows только на выбранной группе устройств Windows нижнего уровня. Если вы убедились, что все работает правильно, то все готово для развертывания пакета на всех устройствах нижнего уровня.

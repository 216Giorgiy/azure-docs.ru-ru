# <a name="control-the-hybrid-azure-ad-join-of-your-devices"></a>Управление гибридным присоединением устройств к Azure AD

Гибридное присоединение к Azure Active Directory (Azure AD) — это процесс автоматической регистрации локальных присоединенных к домену устройств в Azure AD. Иногда не нужно, чтобы все устройства регистрировались автоматически. Например, это актуально при начальном развертывании, когда необходимо убедиться, что все работает правильно.

В этой статье предоставлены рекомендации по управлению гибридным присоединением устройств к Azure AD. 


## <a name="prerequisites"></a>Предварительные требования

Для работы с этой статьей нужны знания в следующих областях:

-  [Общие сведения об управлении устройствами в Azure Active Directory](../device-management-introduction.md)
 
-  [Планирование реализации гибридного присоединения к Azure Active Directory](hybrid-azuread-join-plan.md).

-  [Настройка гибридного присоединения к Azure Active Directory для управляемых доменов](hybrid-azuread-join-managed-domains.md) или [настройка гибридного присоединения к Azure Active Directory для федеративных доменов](hybrid-azuread-join-federated-domains.md).



## <a name="control-windows-current-devices"></a>Управление текущими устройствами Windows

Для устройств под управлением Windows Desktop поддерживается юбилейное обновление Windows 10 (версия 1607) или более поздняя версия. Рекомендуем выполнить обновление до последней версии Windows 10.

Все текущие устройства Windows автоматически регистрируются в Azure AD при запуске устройства или входе пользователя в систему. Этим поведением можно управлять с помощью объекта групповой политики (GPO) или System Center Configuration Manager.

Для управления текущими устройствами Windows требуется выполнить следующее: 


1.  **Все устройства.** Отключите автоматическую регистрацию устройств.
2.  **Выбранные устройства.** Включите автоматическую регистрацию устройств.

Когда вы убедитесь, что все работает правильно, можно повторно включить автоматическую регистрацию для всех устройств.



### <a name="group-policy-object"></a>Объект групповой политики 

Можно управлять поведением регистрации устройств, развернув объект групповой политики **Register domain-joined computers as devices** (Регистрация подключенных к домену компьютеров как устройств).

Чтобы настроить этот объект групповой политики, сделайте следующее:

1.  Откройте **диспетчер серверов** и выберите **Инструменты** > **Управление групповой политикой**.

2.  Перейдите к узлу домена, на котором нужно включить или отключить автоматическую регистрацию.

3.  Щелкните правой кнопкой мыши **Объекты групповой политики**, а затем выберите **Создать**.

4.  Введите имя (например, **Гибридное присоединение к Azure AD**) для объекта групповой политики. 

5.  Нажмите кнопку **ОК**.

6.  Щелкните новый объект групповой политики правой кнопкой мыши, а затем выберите **Изменить**.

7.  Выберите **Конфигурация компьютера** > **Политики** > **Административные шаблоны** > **Компоненты Windows** > **Регистрация устройств**. 

8.  Щелкните правой кнопкой мыши пункт **Зарегистрировать подключенные к домену компьютеры в качестве устройств** и выберите **Изменить**.

    > [!NOTE] 
    > В более ранних версиях консоли управления групповыми политиками этот шаблон групповой политики назывался по-другому. Если вы используете более раннюю версию консоли, последовательно выберите **Конфигурация компьютера** > **Политики** > **Административные шаблоны** > **Компоненты Windows** > **Присоединение к рабочей области** > **Automatically workplace join client computers (Автоматическое присоединение клиентских компьютеров к рабочей области)**. 

9.  Выберите один из указанных ниже параметров и нажмите кнопку **Применить**:

    - **Выключено.** Предотвращение автоматической регистрации устройств.
    - **Включено.** Включение автоматической регистрации устройств.

10. Нажмите кнопку **ОК**.

Необходимо связать объект групповой политики с расположением по своему усмотрению. Например, чтобы задать эту политику для всех текущих присоединенных к домену устройств в вашей организации, свяжите объект групповой политики с доменом. Чтобы выполнять управляемое развертывание, задайте эту политику для присоединенных к домену текущих устройств Windows, относящихся к подразделению или группе безопасности.

### <a name="configuration-manager-controlled-deployment"></a>Развертывание под управлением Configuration Manager 

Можно управлять поведением регистрации текущих устройств, настроив объект групповой политики **Automatically register new Windows 10 domain joined devices with Azure Active Directory** (Автоматическая регистрация новых присоединенных к домену устройств Windows 10 в Azure Active Directory).

Чтобы настроить этот параметр клиента, сделайте следующее:

1.  Откройте **Configuration Manager**, а затем перейдите к разделу **Облачные службы**.

2.  В разделе **Параметры устройства** выберите одно из приведенных ниже значений параметра **Automatically register new Windows 10 domain joined devices with azure Active Directory** (Автоматическая регистрация новых присоединенных к домену устройств Windows 10 в Azure Active Directory):

    - **Нет.** Предотвращение автоматической регистрации устройств.
    - **Да.** Включение автоматической регистрации устройств.


3.  Нажмите кнопку **ОК**.
    

Необходимо связать этот параметр клиента с расположением по своему усмотрению. Например, чтобы настроить этот параметр клиента для всех текущих устройств Windows в вашей организации, свяжите его с доменом. Чтобы выполнять управляемое развертывание, можно настроить этот параметр клиента для присоединенных к домену текущих устройств Windows, относящихся к подразделению или группе безопасности.

> [!Important]
> Хотя описанная выше настройка относится к существующим устройствам Windows 10, которые уже присоединены к домену, есть вероятность, что устройства, которые только присоединяются к нему, могут попытаться выполнить гибридное присоединение к Azure AD. Это связано с возможной задержкой применения групповой политики или параметров Configuration Manager к устройствам Windows 10. 
>
> Чтобы избежать этого, рекомендуем создать новый образ Sysprep (используется в качестве примера для метода подготовки). Создайте его с устройства, для которого ранее никогда не выполнялось гибридное присоединение к Azure AD и в котором уже применен параметр групповой политики или параметр клиента Configuration Manager. Новый образ также необходимо использовать для подготовки новых компьютеров, которые присоединяются к домену вашей организации. 

## <a name="control-windows-down-level-devices"></a>Управление устройствами Windows нижнего уровня

Чтобы регистрировать устройства Windows нижнего уровня, необходимо скачать из Центра загрузки и установить пакет установщика Windows (MSI-файл), опубликованный на странице [Microsoft Workplace Join for non-Windows 10 computers](https://www.microsoft.com/download/details.aspx?id=53554) (Microsoft Workplace Join для компьютеров под управлением ОС, отличной от Windows 10).

Для развертывания пакета используйте систему распространения программного обеспечения, например [System Center Configuration Manager](https://www.microsoft.com/cloud-platform/system-center-configuration-manager). Этот пакет поддерживает параметры стандартной автоматической установки с использованием параметра quiet. В текущей ветви System Center Configuration Manager доступны дополнительные преимущества предыдущих версий, такие как возможность отслеживать ход регистрации.

Установщик создает в системе запланированную задачу, которая выполняется в контексте пользователя. Задача запускается в момент входа пользователя в систему Windows. Эта задача автоматически присоединяет устройство к Azure AD, используя учетные данные пользователя, после аутентификации с помощью Azure AD.

В целях контроля регистрации устройств следует развернуть пакет установщика Windows только на выбранной группе устройств Windows нижнего уровня. Когда вы убедитесь, что все работает правильно, можно приступать к развертыванию пакета на всех устройствах нижнего уровня.

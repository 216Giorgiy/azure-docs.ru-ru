#Управление группами безопасности в Azure Active Directory

Одна из основных возможностей Azure Active Directory (Azure AD) — это возможность управлять доступом к ресурсам. Это могут быть внутренние ресурсы каталога, например разрешения на управление объектами с помощью ролей в каталоге, или это могут быть ресурсы, внешние относительно каталога, например приложения SaaS, службы Azure, сайты SharePoint или локальные ресурсы. Владелец ресурса может назначить доступ к ресурсу целой группе. В результате доступ к ресурсу будет у каждого члена группы. После этого членством в группе может управлять владелец группы. То есть владелец ресурса фактически делегирует владельцу группы разрешение назначать пользователям доступ к ресурсу.


##Как создать группу безопасности и управлять ей

**Создание группы на портале Azure**

1. На портале Azure щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. Щелкните вкладку **Группы**.
3. На странице «Группы» щелкните **Добавить группу**.
4. В окне **Добавление группы** укажите имя и описание группы.
5. Эту задачу можно выполнить с помощью портала учетной записи Office 365, портала учетной записи Windows Intune или портала Azure, в зависимости от того, на какие службы подписана ваша организация. Дополнительную информацию об использовании порталов для управления Azure Active Directory см. в разделе [Управление каталогом Azure AD](active-directory-administer).

## Как назначить пользователей в группу безопасности или удалить их из нее

**Добавление участника в группу на портале Azure**

1. На портале Azure щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. Щелкните вкладку **Группы**.
3. На странице **Группы** щелкните имя группы, в которую требуется добавить участников. По умолчанию отобразится вкладка **Участники** выбранной группы.
4. На странице группы щелкните **Добавить участников**.
5. На странице **Добавление участников** щелкните имя пользователя или группы, которую хотите добавить в качестве участника этой группы, и убедитесь, что это имя добавлено в область «Выбранные».


**Удаление участника из группы на портале Azure**

1. На портале Azure щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. Щелкните вкладку **Группы**.
3. На странице «Группы» щелкните имя группы, из которой требуется удалить участников.
4. На странице группы щелкните вкладку **Участники**.
5. На странице группы щелкните имя участника, которого хотите удалить из этой группы, и щелкните **Удалить**.
6. Подтвердите, что хотите удалить этого участника из группы, щелкнув **Да** в ответ на запрос подтверждения действия.


## Как с помощью правила динамически управлять участниками группы безопасности

**Чтобы включить динамическое членство для определенной группы, выполните следующие действия:**

1. На портале Azure на вкладке **Группы** выберите группу, которую необходимо изменить, а затем на соответствующей вкладке **Настройка** для параметра **Включить динамическое членство** установите значение **Да**.
2. Теперь можно настроить простое правило для группы, которое будет управлять динамическим членством для функций этой группы. Убедитесь, что переключатель **Добавить пользователей** установлен, затем выберите свойство пользователя из раскрывающегося меню (например, department, jobTitle и т. д.).
3. После этого выберите условие («Не равно», «Равно», «Не начинается с», «Начинается с», «Не содержит», «Содержит», «Не соответствует», «Соответствует») и укажите значение для выбранного свойства пользователя.

Например, если назначена группа для приложения SaaS (дополнительную информацию см. в разделе «Назначение группе доступа к приложению SaaS в Azure AD»), и вы включаете динамическое членство для этой группы, задав правило, согласно которому для параметра «Добавить пользователей:» задано свойство jobTitle, которое равно (-eq) Sales Rep, то все пользователи в каталоге Azure AD, название должности которых — Sales Rep, будут иметь доступ к этому приложению SaaS.

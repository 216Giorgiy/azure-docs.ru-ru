# <a name="tutorial-azure-active-directory-integration-with-way-we-do"></a>Руководство по интеграции Azure Active Directory с Way We Do

В этом руководстве описано, как интегрировать Way We Do с Azure Active Directory (Azure AD).

Интеграция Azure AD с приложением Way We Do обеспечивает следующие преимущества.

- С помощью Azure AD вы можете контролировать доступ к Way We Do.
- Вы можете включить автоматический вход пользователей в Way We Do (единый вход) с использованием их учетных записей Azure AD.
- Вы можете управлять учетными записями централизованно — на портале Azure.

Подробнее узнать об интеграции приложений SaaS с Azure AD можно в разделе [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory](../manage-apps/what-is-single-sign-on.md).

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с Way We Do, вам потребуются:

- подписка Azure AD;
- подписка Way We Do с поддержкой единого входа.

> [!NOTE]
> Мы не рекомендуем использовать рабочую среду для проверки действий в этом учебнике.

При проверке действий в этом учебнике соблюдайте следующие рекомендации:

- Не используйте рабочую среду без необходимости.
- Если у вас нет пробной среды Azure AD, вы можете [получить пробную версию на один месяц](https://azure.microsoft.com/pricing/free-trial/).

## <a name="scenario-description"></a>Описание сценария
В рамках этого руководства проводится проверка единого входа Azure AD в тестовой среде. Сценарий, описанный в этом учебнике, состоит из двух стандартных блоков.

1. Добавление Way We Do из коллекции.
2. настройка и проверка единого входа в Azure AD.

## <a name="adding-way-we-do-from-the-gallery"></a>Добавление Way We Do из коллекции
Чтобы настроить интеграцию Way We Do с Azure AD, необходимо добавить Way We Do из коллекции в список управляемых приложений SaaS.

**Чтобы добавить Way We Do из коллекции, сделайте следующее.**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**. 

    ![Кнопка "Azure Active Directory"][1]

2. Перейдите к разделу **Корпоративные приложения**. Затем выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"][2]
    
3. Чтобы добавить новое приложение, в верхней части диалогового окна нажмите кнопку **Создать приложение**.

    ![Кнопка "Новое приложение"][3]

4. В поле поиска введите **Way We Do**, выберите **Way We Do** на панели результатов и нажмите кнопку **Добавить**, чтобы добавить это приложение.

    ![Way We Do в списке результатов](./media/waywedo-tutorial/tutorial_waywedo_addfromgallery.png)

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

В этом разделе описана настройка и проверка единого входа Azure AD в Way We Do с использованием тестового пользователя Britta Simon.

Для работы единого входа в Azure AD необходимо знать, какой пользователь в Way We Do соответствует пользователю в Azure AD. Иными словами, необходимо установить связь между пользователем Azure AD и соответствующим пользователем в Way We Do.

Чтобы настроить и проверить единый вход Azure AD в Way We Do, вам потребуется выполнить действия в следующих стандартных блоках.

1. **[Настройка единого входа Azure AD](#configure-azure-ad-single-sign-on)** необходима, чтобы пользователи могли использовать эту функцию.
2. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
3. **[Создание тестового пользователя приложения Way We Do](#create-a-way-we-do-test-user)** требуется для того, чтобы в Way We Do существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure AD.
4. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить Britta Simon использовать единый вход Azure AD.
5. **[Проверка единого входа](#test-single-sign-on)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="configure-azure-ad-single-sign-on"></a>Настройка единого входа Azure AD

В этом разделе описано, как включить единый вход Azure AD на портале Azure и настроить его в приложении Way We Do.

**Чтобы настроить единый вход Azure AD в Way We Do, выполните следующие действия.**

1. На портале Azure на странице интеграции с приложением **Way We Do** щелкните **Единый вход**.

    ![Ссылка "Настройка единого входа"][4]

2. В диалоговом окне **Единый вход** в разделе **Режим** выберите **Вход на основе SAML**, чтобы включить функцию единого входа.
 
    ![Диалоговое окно "Единый вход"](./media/waywedo-tutorial/tutorial_waywedo_samlbase.png)

3. В разделе **Домены и URL-адреса приложения Way We Do** выполните следующие действия.

    ![Сведения о домене и URL-адресах единого входа для приложения Way We Do](./media/waywedo-tutorial/tutorial_waywedo_url.png)

    a. В текстовом поле **URL-адрес для входа** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.waywedo.com/Authentication/ExternalSignIn`

    b. В текстовом поле **Идентификатор** введите URL-адрес в следующем формате: `https://<SUBDOMAIN>.waywedo.com`

    > [!NOTE] 
    > Эти значения приведены в качестве примера. Замените эти значения фактическим URL-адресом для входа и идентификатором. Чтобы получить эти значения, обратитесь к [группе поддержки клиентов Way We Do](mailto:support@waywedo.com). 
 
4. В разделе **Сертификат подписи SAML** щелкните **Certificate (Raw)** (Сертификат (необработанный)), а затем сохраните файл сертификата на компьютере.

    ![Ссылка для скачивания сертификата](./media/waywedo-tutorial/tutorial_waywedo_certificate.png) 

5. Нажмите кнопку **Сохранить** .

    ![Кнопка "Сохранить" в окне настройки единого входа](./media/waywedo-tutorial/tutorial_general_400.png)

6. В разделе **Конфигурация Way We Do** щелкните **Настроить Way We Do**, чтобы открыть окно **Настройка единого входа**. Скопируйте **идентификатор сущности SAML и URL-адрес службы единого входа SAML** из раздела **Краткий справочник**.

    ![Конфигурация Way We Do](./media/waywedo-tutorial/tutorial_waywedo_configure.png) 

7. В другом окне веб-браузера войдите в приложение Way We Do с правами администратора безопасности.

8. Нажмите щелкните **значок человека** в правом верхнем углу любой страницы в Way We Do и в раскрывающемся меню выберите **Account** (Учетная запись).

    ![Way We Do: учетная запись](./media/waywedo-tutorial/tutorial_waywedo_account.png) 

9. Щелкните **значок меню**, чтобы открыть меню навигации, и выберите **Single Sign On** (Единый вход).

    ![Way We Do: единый вход](./media/waywedo-tutorial/tutorial_waywedo_single.png)

10. На странице **Single sign-on setup** (Настройка единого входа) выполните следующие действия.

    ![Way We Do: сохранение](./media/waywedo-tutorial/tutorial_waywedo_save.png)

    a. Щелкните переключатель **Turn on single sign-on** (Включить единый вход), установив для него значение **Yes** (Да).

    b. В текстовом поле **Single sign-on name** (Имя для единого входа) введите имя.

    c. В текстовое поле **Entity ID** (Идентификатор сущности) вставьте значение **идентификатора сущности**, скопированное на портале Azure.

    d. В текстовое поле **SAML SSO URL** (URL-адрес единого входа SAML) вставьте значение **URL-адреса службы единого входа SAML**, скопированное на портале Azure.

    д. Передайте сертификат, нажав кнопку **Select** (Выбрать) рядом с полем **Certificate** (Сертификат).

    Е. **Необязательные параметры** -
    
    * "Enable Passwords" (Включить пароли). Если этот параметр отключен, для пользователей Way We Do действует обычный пароль и они могут использовать только единый вход.

    * "Enable Auto-provisioning" (Включить автоматическую подготовку). Когда этот параметр включен, адреса электронной почты, используемые для входа, автоматически сверяются со списком пользователей в Way We Do. Если адрес электронной почты не соответствует активному пользователю в Way We Do, для выполняющего вход пользователя автоматически добавляется новая учетная запись и ему предлагается ввести все недостающие сведения.

      > [!NOTE]
      > Пользователи, добавленные с помощью функции единого входа, являются обычными пользователями без каких-либо ролей в системе. Администратор может войти в систему и изменить их роль безопасности на роль редактора или администратора. Кроме того, он может назначить одну или несколько ролей для организационной диаграммы. 

    ж. Нажмите кнопку **Save** (Сохранить), чтобы сохранить настройки.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

Цель этого раздела — создать на портале Azure тестового пользователя с именем Britta Simon.

   ![Создание тестового пользователя Azure AD][100]

**Чтобы создать тестового пользователя в Azure AD, выполните следующие действия:**

1. На портале Azure в области слева нажмите кнопку **Azure Active Directory**.

    ![Кнопка "Azure Active Directory"](./media/waywedo-tutorial/create_aaduser_01.png)

2. Чтобы открыть список пользователей, перейдите в раздел **Пользователи и группы** и щелкните **Все пользователи**.

    ![Ссылки "Пользователи и группы" и "Все пользователи"](./media/waywedo-tutorial/create_aaduser_02.png)

3. Чтобы открыть диалоговое окно **Пользователь**, в верхней части диалогового окна **Все пользователи** щелкните **Добавить**.

    ![Кнопка "Добавить"](./media/waywedo-tutorial/create_aaduser_03.png)

4. В диалоговом окне **Пользователь** сделайте следующее.

    ![Диалоговое окно "Пользователь"](./media/waywedo-tutorial/create_aaduser_04.png)

    a. В поле **Имя** введите **BrittaSimon**.

    Б. В поле **Имя пользователя** введите адрес электронной почты для пользователя Britta Simon.

    c. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.

    d. Нажмите кнопку **Создать**.
 
### <a name="create-a-way-we-do-test-user"></a>Создание тестового пользователя Way We Do

Цель этого раздела — создать пользователя с именем Britta Simon в Way We Do. Приложение Way We Do поддерживает JIT-подготовку. Эта функция включена по умолчанию. В этом разделе никакие действия с вашей стороны не требуются. При попытке получить доступ к приложению Way We Do создается учетная запись пользователя (если она еще не создана).

> [!Note]
> Чтобы создать пользователя вручную, обратитесь к [группе поддержки Way We Do](mailto:support@waywedo.com).

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю Britta Simon использовать единый вход Azure, предоставив этому пользователю доступ к Way We Do.

![Назначение роли пользователя][200] 

**Чтобы назначить пользователя Britta Simon в Way We Do, выполните указанные ниже действия.**

1. На портале Azure откройте представление приложений, перейдите к представлению каталога, а затем выберите **Корпоративные приложения** и щелкните **Все приложения**.

    ![Назначение пользователя][201] 

2. Из списка приложений выберите **Way We Do**.

    ![Ссылка на Way We Do в списке "Приложения"](./media/waywedo-tutorial/tutorial_waywedo_app.png)  

3. В меню слева выберите **Пользователи и группы**.

    ![Ссылка "Пользователи и группы"][202]

4. Нажмите кнопку **Добавить**. Затем в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Область "Добавление назначения"][203]

5. В диалоговом окне **Пользователи и группы** в списке пользователей выберите **Britta Simon**.

6. В диалоговом окне **Пользователи и группы** нажмите кнопку **Выбрать**.

7. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

# <a name="tutorial-azure-active-directory-integration-with-iprova"></a>Руководство по Интеграция Azure Active Directory с iProva

В этом руководстве описано, как интегрировать iProva с Azure Active Directory (Azure AD).
Интеграция Azure AD с приложением iProva обеспечивает следующие преимущества.

* С помощью Azure AD вы можете контролировать доступ к iProva.
* Можно включить автоматический вход пользователей в iProva (единый вход) с помощью учетной записи Azure Active Directory.
* Вы можете управлять учетными записями централизованно, через портал Azure.

Дополнительные сведения об интеграции приложения SaaS с Azure Active Directory см. в статье [Единый вход в приложениях в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis).
Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с iProva, вам потребуется:

* подписка Azure AD Если у вас нет среды Azure Active Directory, вы можете получить бесплатную пробную версию на один месяц на [веб-сайте Microsoft Azure](https://azure.microsoft.com/pricing/free-trial/).
* Подписка на iProva с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства вы настроите и проверите единый вход Azure Active Directory в тестовой среде.

* iProva поддерживает единый вход, инициированный поставщиком услуг.

## <a name="add-iprova-from-the-gallery"></a>Добавление iProva из коллекции

Чтобы настроить интеграцию iProva с Azure Active Directory, необходимо добавить iProva из коллекции в список управляемых приложений SaaS.

Чтобы добавить iProva из коллекции, выполните следующие действия:

1. На [портале Azure](https://portal.azure.com) в области слева щелкните **Azure Active Directory**.

    ![Кнопка Azure Active Directory](common/select-azuread.png)

2. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в верхней части диалогового окна выберите **Новое приложение**.

    ![Кнопка "Создать приложение"](common/add-new-app.png)

4. В поле поиска введите **iProva**. Выберите **iProva** на панели результатов и нажмите кнопку **Добавить**, чтобы добавить это приложение.

     ![iProva в списке результатов](common/search-new-app.png)

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

В этом разделе описана настройка и проверка единого входа Azure Active Directory в iProva с использованием тестового пользователя Britta Simon.
Для обеспечения работы единого входа необходимо установить связь между пользователем Azure Active Directory и соответствующим пользователем в iProva.

Чтобы настроить и проверить единый вход Azure Active Directory в iProva, выполните инструкции в следующих базовых блоках:

- [Получение сведений о конфигурации из iProva](#retrieve-configuration-information-from-iprova), необходимых на следующих шагах.
- [Настройка единого входа Azure AD](#configure-azure-ad-single-sign-on) необходима, чтобы пользователи могли использовать эту функцию.
- [Настройка единого входа в iProva](#configure-iprova-single-sign-on) для настройки параметров единого входа на стороне приложения.
- [Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user) требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
- [Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user) необходимо, чтобы разрешить Britta Simon использовать единый вход Azure AD.
- [Создание тестового пользователя iProva](#create-an-iprova-test-user) с именем Britta Simon, связанного с одноименным пользователем в Azure Active Directory.
- [Проверка единого входа](#test-single-sign-on) позволяет убедиться в корректной работе конфигурации.

### <a name="retrieve-configuration-information-from-iprova"></a>Получение сведений о конфигурации из iProva

В этом разделе вы получите сведения из iProva, необходимые для настройки единого входа Azure Active Directory.

1. Откройте в браузера **страницу сведений о SAML2** на сайте iProva, используя URL-адрес следующего вида:

    | | |
    |-|-|
    | `https://SUBDOMAIN.iprova.nl/saml2info`|
    | `https://SUBDOMAIN.iprova.be/saml2info`|
    | | |

     ![Просмотр страницы сведений о SAML2 в iProva](media/iprova-tutorial/iprova-saml2-info.png)

2. Не закрывайте пока открытую вкладку браузера, а следующие шаги выполняйте в новой вкладке.

### <a name="configure-azure-ad-single-sign-on"></a>Настройка единого входа Azure AD

В этом разделе описано включение единого входа Azure AD на портале Azure.

Чтобы настроить единый вход Azure Active Directory в iProva, выполните следующие действия:

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **iProva**  выберите **Единый вход**.

    ![Ссылка "Настройка единого входа"](common/select-sso.png)

2. В диалоговом окне **Выбрать метод единого входа** выберите режим **SAML**, чтобы включить единый вход.

    ![Режим выбора единого входа](common/select-saml-option.png)

3. На странице **Настройка единого входа с помощью SAML** нажмите значок **правки**, чтобы открыть диалоговое окно **Базовая конфигурация SAML**.

    ![Значок правки в области базовой конфигурации SAML](common/edit-urls.png)

4. В разделе **Базовая конфигурация SAML** выполните следующие действия:

    a. В поле **Идентификатор** укажите значение, которое отображается под меткой **идентификатора сущности** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

    b. В поле **URL-адрес ответа** укажите значение, которое отображается под меткой **URL-адреса ответа** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

    c. В поле **URL-адрес входа** укажите значение, которое отображается под меткой **URL-адреса входа** на **странице сведений о SAML2 в iProva**. Эта страница, которую вы открывали в другой вкладке браузера.

    ![Сведения о домене и URL-адресах единого входа для приложения iProva](common/sp-identifier-reply.png)

5. Приложение iProva ожидает утверждения SAML в определенном формате. Настройте следующие утверждения для этого приложения. Значениями этих атрибутов можно управлять в разделе **Атрибуты пользователя** на странице интеграции приложения. На странице **Настройка единого входа с помощью SAML** нажмите значок **правки**, чтобы открыть диалоговое окно **Атрибуты пользователя**.

    ![Диалоговое окно "Атрибуты пользователя"](common/edit-attribute.png)

6. В разделе **Утверждения пользователя** диалогового окна **Атрибуты пользователя** настройте атрибут токена SAML, как показано на рисунке выше. Выполните следующие действия.

    | ИМЯ | Атрибут источника| Пространство имен |
    | ---------------| -------- | -----|
    | `samaccountname` | `user.onpremisessamaccountname`| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims`|
    | | |

    a. Чтобы открыть диалоговое окно **Управление утверждениями пользователя**, выберите **Добавить новое утверждение**.

    ![Утверждения пользователя](common/new-save-attribute.png)

    ![Диалоговое окно "Управление утверждениями пользователя"](common/new-attribute-details.png)

    b. В текстовом поле **Имя** введите имя атрибута, отображаемое для этой записи.

    c. В списке **Пространство имен** введите значение пространства имен, отображаемое для этой строки.

    d. Для параметра **Источник** задайте значение **Атрибут**.

    д. В списке **Атрибут источника** введите значение атрибута, отображаемое для этой строки.

    Е. Нажмите кнопку **ОК**.

    ж. Щелкните **Сохранить**.

7. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** щелкните значок **копирования**, чтобы скопировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="configure-iprova-single-sign-on"></a>Настройка единого входа в iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Выберите **Application management** (Управление приложением).

4. Выберите **General** (Общие) на панели **System settings** (Параметры системы).

5. Выберите **Изменить**

6. Прокрутите вниз до раздела **Access control** (Управление доступом).

    ![Параметры управления доступом в iProva](media/iprova-tutorial/iprova-accesscontrol.png)

7. Найдите параметр **Users are automatically logged on with their network accounts** (Пользователи автоматически входят в систему с сетевыми учетными записями) и выберите для него значение **Yes, authentication via SAML** (Да, с аутентификацией через SAML). Появятся дополнительные параметры.

8. Выберите **Set up** (Настроить).

9. Щелкните **Далее**.

10. iProva предложит скачать данные федерации по URL-адресу или загрузить их из файла. Выберите вариант **From URL** (По URL-адресу).

    ![Скачивание метаданных AAD](media/iprova-tutorial/iprova-download-metadata.png)

11. Вставьте URL-адрес метаданных, который вы сохранили на последнем шаге раздела Настройка единого входа.

12. Чтобы скачать метаданные из AAD, нажмите кнопку в форме стрелки.

13. Когда скачивание завершится, появится сообщение с подтверждением **Valid Federation Data file downloaded** (Скачан допустимый файл с данными федерации).

14. Щелкните **Далее**.

15. Пока пропустите действие **Test login** (Тестовый вход) и выберите **Next** (Далее).

16. В раскрывающемся списке **Claim to use** (Утверждение для использования) выберите **windowsaccountname**.

17. Выберите **Готово**.

18. Теперь вы снова попадете на экран **Edit general settings** (Редактирование общих параметров). Внизу страницы нажмите кнопку **ОК**, чтобы сохранить конфигурацию.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

В этом разделе описано, как создать на портале Azure тестового пользователя с именем Britta Simon.

1. На портале Azure на панели слева выберите **Azure Active Directory** > **Пользователи** > **Все пользователи**.

    ![Меню "Пользователи и группы" и "Все пользователи"](common/users.png)

2. В верхней части экрана выберите **Новый пользователь**.

    ![Кнопка "Новый пользователь"](common/new-user.png)

3. В диалоговом окне **Пользователь** выполните следующие действия:

    ![Диалоговое окно "Пользователь"](common/user-properties.png)

    a. В поле **Имя** введите значение **BrittaSimon**.
  
    b. В поле **Имя пользователя** введите *<ваше_имя>\@<домен_вашей_компании>.<доменная_зона>*. 
    Например, BrittaSimon@contoso.com.

    c. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле **Пароль**.

    d. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю Britta Simon использовать единый вход Azure, предоставив этому пользователю доступ к iProva.

1. На портале Azure выберите **Корпоративные приложения** > **Все приложения** > **iProva**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **iProva**.

    ![Ссылка на iProva в списке приложений](common/all-applications.png)

3. В меню слева выберите **Пользователи и группы**.

    ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

4. Выберите **Добавить пользователя**, а в диалоговом окне **Добавление назначения** — **Пользователи и группы**.

    ![Диалоговое окно "Добавление назначения"](common/add-assign-user.png)

5. В диалоговом окне **Пользователи и группы** выберите **Britta Simon** в списке **Пользователи**, а в нижней части экрана — **Выбрать**.

6. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор ролей** нужно выбрать соответствующую роль для пользователя из списка. В нижней части экрана нажмите кнопку **Выбрать**.

7. В диалоговом окне **Добавление назначения** выберите **Назначить**.

### <a name="create-an-iprova-test-user"></a>Создание тестового пользователя iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Выберите **Application management** (Управление приложением).

4. Выберите **Users** (Пользователи) на панели **Users and user groups** (Пользователи и группы пользователей).

5. Выберите **Добавить**.

6. В поле **Имя пользователя** введите *brittasimon\@<домен_вашей_компании>.<доменная_зона>*. 
    Например, BrittaSimon@contoso.com.

7. В поле **Full name** (Имя и фамилия) введите значение **BrittaSimon**.

8. Выберите вариант **No password (use single sign-on)** (Без пароля (использование единого входа)).

9. В поле **Адрес электронной почты** введите *<ваше_имя>\@<домен_вашей_компании>.<доменная_зона>*. 
   Например, BrittaSimon@contoso.com.

10. Внизу страницы нажмите кнопку **Finish** (Завершить).

# <a name="tutorial-azure-active-directory-integration-with-iprova"></a>Руководство. Интеграция Azure Active Directory с iProva

В этом руководстве описано, как интегрировать iProva с Azure Active Directory (Azure AD).
Интеграция Azure AD с приложением iProva обеспечивает следующие преимущества.

* С помощью Azure AD вы можете контролировать доступ к iProva.
* Можно включить автоматический вход пользователей в iProva (единый вход) с помощью учетной записи Azure AD.
* Вы можете управлять учетными записями централизованно — на портале Azure.

Подробнее узнать об интеграции приложений SaaS с Azure AD можно в статье [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](https://docs.microsoft.com/azure/active-directory/active-directory-appssoaccess-whatis).
Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

Чтобы настроить интеграцию Azure AD с iProva, вам потребуется:

* подписка Azure AD; Если у вас нет среды Azure AD, вы можете получить пробную версию на один месяц по [этой ссылке](https://azure.microsoft.com/pricing/free-trial/).
* Подписка iProva с поддержкой единого входа.

## <a name="scenario-description"></a>Описание сценария

В рамках этого руководства проводится настройка и проверка единого входа Azure AD в тестовой среде.

* iProva поддерживает единый вход, инициированный **поставщиком услуг**.

## <a name="adding-iprova-from-the-gallery"></a>Добавление iProva из коллекции

Чтобы настроить интеграцию iProva с Azure AD, необходимо добавить iProva из коллекции в список управляемых приложений SaaS.

**Чтобы добавить iProva из коллекции, выполните следующие действия:**

1. На **[портале Azure](https://portal.azure.com)** в области навигации слева щелкните значок **Azure Active Directory**.

    ![Кнопка "Azure Active Directory"](common/select-azuread.png)

2. Перейдите в колонку **Корпоративные приложения** и выберите **Все приложения**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

3. Чтобы добавить новое приложение, в верхней части диалогового окна нажмите кнопку **Создать приложение**.

    ![Кнопка "Новое приложение"](common/add-new-app.png)

4. В поле поиска введите **iProva**, выберите **iProva** на панели результатов и нажмите кнопку **Добавить**, чтобы добавить это приложение.

     ![iProva в списке результатов](common/search-new-app.png)

## <a name="configure-and-test-azure-ad-single-sign-on"></a>Настройка и проверка единого входа в Azure AD

В этом разделе описана настройка и проверка единого входа Azure AD в iProva с использованием тестового пользователя **Britta Simon**.
Для обеспечения работы единого входа необходимо установить связь между пользователем Azure AD и соответствующим пользователем в iProva.

Чтобы настроить и проверить единый вход Azure AD в iProva, вам потребуется выполнить действия в следующих стандартных блоках.

1. **[Получить сведения о конфигурации из iProva](#retrieve-configuration-information-from-iprova)** для подготовки к следующим шагам.
2. **[Настройка единого входа Azure AD](#configure-azure-ad-single-sign-on)** необходима, чтобы пользователи могли использовать эту функцию.
3. **[Настройка единого входа в iProva](#configure-iprova-single-sign-on)**, чтобы настроить параметры единого входа на стороне приложения.
4. **[Создание тестового пользователя Azure AD](#create-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
5. **[Создание тестового пользователя iProva](#create-iprova-test-user)**, чтобы в iProva существовал пользователь Britta Simon, связанный с одноименным пользователем в Azure AD.
6. **[Назначение тестового пользователя Azure AD](#assign-the-azure-ad-test-user)** необходимо, чтобы позволить Britta Simon использовать единый вход Azure AD.
7. **[Проверка единого входа](#test-single-sign-on)** необходима, чтобы убедиться в корректной работе конфигурации.

### <a name="retrieve-configuration-information-from-iprova"></a>Получение сведений о конфигурации из iProva

В этом разделе вы получите некоторые важные сведения из iProva.
Эти сведения потребуются для настройки единого входа в AAD.

1. Откройте в браузера **страницу сведений о SAML2** на сайте iProva, используя URL-адрес следующего вида:

    | | |
    |-|-|
    | `https://SUBDOMAIN.iprova.nl/saml2info`|
    | `https://SUBDOMAIN.iprova.be/saml2info`|
    | | |

     ![Просмотр страницы сведений о SAML2 в iProva](media/iprova-tutorial/iprova-saml2-info.png)

2. Не закрывайте пока открытую вкладку браузера, а следующие шаги выполняйте в новой вкладке.

### <a name="configure-azure-ad-single-sign-on"></a>Настройка единого входа Azure AD

В этом разделе описано включение единого входа Azure AD на портале Azure.

Чтобы настроить единый вход AAD в iProva, выполните следующие действия.

1. На [портале Azure](https://portal.azure.com/) на странице интеграции с приложением **iProva**  выберите **Единый вход**.

    ![Ссылка "Настройка единого входа"](common/select-sso.png)

2. В диалоговом окне **Выбрать метод единого входа** выберите режим **SAML/WS-Fed**, чтобы включить единый вход.

    ![Режим выбора единого входа](common/select-saml-option.png)

3. На странице **Настройка единого входа с помощью SAML** щелкните **Изменить**, чтобы открыть диалоговое окно **Базовая конфигурация SAML**.

    ![Правка базовой конфигурации SAML](common/edit-urls.png)

4. В разделе **Базовая конфигурация SAML** выполните приведенные ниже действия.

    a. Заполните поле **Идентификатор** значением, которое отображается под меткой **EntityID** на **странице сведений о SAML2 в iProva** (которую вы оставили открытой в другой вкладке браузера).

    b. Заполните поле **URL-адрес ответа** значением, которое отображается под меткой **Reply URL** на **странице сведений о SAML2 в iProva** (которую вы оставили открытой в другой вкладке браузера).

    c. Заполните поле **URL-адрес входа** значением, которое отображается под меткой **Sign-on URL** на **странице сведений о SAML2 в iProva** (которую вы оставили открытой в другой вкладке браузера).

    ![Сведения о домене и URL-адресах единого входа для приложения iProva](common/sp-identifier-reply.png)

5. Приложение iProva ожидает утверждения SAML в определенном формате. Настройте следующие утверждения для этого приложения. Управлять значениями этих атрибутов можно в разделе **Атрибуты пользователя** на странице интеграции приложения. На странице **Настройка единого входа с помощью SAML** нажмите кнопку **Изменить**, чтобы открыть диалоговое окно **Атрибуты пользователя**.

    ![изображение](common/edit-attribute.png)

6. В разделе **Утверждения пользователя** диалогового окна **Атрибуты пользователя** настройте атрибут токена SAML, как показано на рисунке выше, и выполните следующие действия.

    | ИМЯ | Исходный атрибут| Пространство имен |
    | ---------------| -------- | -----|
    | `samaccountname` | `user.onpremisessamaccountname`| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims`|
    | | |

    a. Щелкните **Добавить новое утверждение**, чтобы открыть диалоговое окно **Управление утверждениями пользователя**.

    ![изображение](common/new-save-attribute.png)

    ![изображение](common/new-attribute-details.png)

    b. В текстовом поле **Имя** введите имя атрибута, отображаемое для этой строки.

    c. В списке **Пространство имен** введите значение пространства имен для этой строки.

    d. В качестве источника выберите **Атрибут**.

    д. В списке **Атрибут источника** введите значение атрибута, отображаемое для этой строки.

    Е. Нажмите кнопку **ОК**.

    ж. Выберите команду **Сохранить**.

7. На странице **Настройка единого входа с помощью SAML** в разделе **Сертификат подписи SAML** нажмите кнопку "Копировать", чтобы копировать **URL-адрес метаданных федерации приложений** и сохранить его на компьютере.

    ![Ссылка для скачивания сертификата](common/copy-metadataurl.png)

### <a name="configure-iprova-single-sign-on"></a>Настройка единого входа в iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Щелкните **Application management** (Управление приложением).

4. Щелкните **General** (Общие) на панели **System settings** (Параметры системы).

5. Щелкните **Edit** (Редактировать).

6. Прокрутите вниз до раздела **Access control** (Управление доступом).

    ![Параметры управления доступом в iProva](media/iprova-tutorial/iprova-accesscontrol.png)

7. Найдите параметр **Users are automatically logged on with their network accounts** (Пользователи автоматически входят в систему с сетевыми учетными записями) и выберите для него значение **Yes, authentication via SAML** (Да, с аутентификацией через SAML). Появятся дополнительные параметры.

8. Нажмите кнопку **Setup** (Установка).

9. Нажмите кнопку **Далее**.

10. Теперь iProva предложит скачать данные федерации по URL-адресу или загрузить их из файла. Выберите параметр **from URL** (По URL-адресу).

    ![Скачивание метаданных AAD](media/iprova-tutorial/iprova-download-metadata.png)

11. Вставьте здесь URL-адрес метаданных, который вы сохранили на последнем шаге главы **Настройка единого входа**.

12. Чтобы скачать метаданные из AAD, нажмите кнопку в форме стрелки.

13. Когда скачивание завершится, появится сообщение с подтверждением **Valid Federation Data file downloaded** (Скачан допустимый файл с данными федерации).

14. Нажмите кнопку **Далее**.

15. Пока пропустите действие **Test login** (Тестовый вход) и нажмите кнопку **Next** (Далее).

16. В раскрывающемся списке с именем **Claim to use** (Утверждение для использования) выберите **windowsaccountname**.

17. Нажмите кнопку **Готово**.

18. Теперь вы снова попадете на экран **Edit general settings** (Редактирование общих параметров). **Прокрутите вниз** до самого низа страницы и нажмите кнопку **ОК**, чтобы сохранить конфигурацию.

### <a name="create-an-azure-ad-test-user"></a>Создание тестового пользователя Azure AD

Цель этого раздела — создать на портале Azure тестового пользователя с именем Britta Simon.

1. На портале Azure в области слева выберите **Azure Active Directory**, **Пользователи**, а затем — **Все пользователи**.

    ![Ссылки "Пользователи и группы" и "Все пользователи"](common/users.png)

2. В верхней части экрана выберите **Новый пользователь**.

    ![Кнопка "Новый пользователь"](common/new-user.png)

3. В разделе свойств пользователя сделайте следующее.

    ![Диалоговое окно "Пользователь"](common/user-properties.png)

    a. В поле **Имя** введите значение **BrittaSimon**.
  
    b. В поле **Имя пользователя** введите **yourname@yourcompanydomain.extension**.  
    Например, BrittaSimon@contoso.com

    c. Установите флажок **Показать пароль** и запишите значение, которое отображается в поле "Пароль".

    d. Нажмите кнопку **Создать**.

### <a name="assign-the-azure-ad-test-user"></a>Назначение тестового пользователя Azure AD

В этом разделе описано, как разрешить пользователю Britta Simon использовать единый вход Azure, предоставив этому пользователю доступ к iProva.

1. На портале Azure выберите **Корпоративные приложения**, **Все приложения**, а затем **iProva**.

    ![Колонка "Корпоративные приложения"](common/enterprise-applications.png)

2. В списке приложений выберите **iProva**.

    ![Ссылка на iProva в списке "Приложения"](common/all-applications.png)

3. В меню слева выберите **Пользователи и группы**.

    ![Ссылка "Пользователи и группы"](common/users-groups-blade.png)

4. Нажмите кнопку **Добавить пользователя**, а затем в диалоговом окне **Добавление назначения** выберите **Пользователи и группы**.

    ![Область "Добавление назначения"](common/add-assign-user.png)

5. В диалоговом окне **Пользователи и группы** из списка пользователей выберите **Britta Simon**, а затем в верхней части экрана нажмите кнопку **Выбрать**.

6. Если ожидается, что в утверждении SAML будет получено какое-либо значение роли, то в диалоговом окне **Выбор ролей** нужно выбрать соответствующую роль для пользователя из списка и затем нажать кнопку **Выбрать**, расположенную в нижней части экрана.

7. В диалоговом окне **Добавление назначения** нажмите кнопку **Назначить**.

### <a name="create-iprova-test-user"></a>Создание тестового пользователя iProva

1. Войдите в iProva с учетной записью **администратора**.

2. Откройте меню **Go to** (Перейти).

3. Щелкните **Application management** (Управление приложением).

4. Щелкните **Users** (Пользователи) на панели **Users and user groups** (Пользователи и группы).

5. Нажмите кнопку **Add** (Добавить).

6. В поле **Username** (Имя пользователя) введите значение **brittasimon@yourcompanydomain.extension**  
    Например, BrittaSimon@contoso.com

7. В поле **Full name** (Имя и фамилия) введите значение **BrittaSimon**.

8. Выберите вариант **No password (use single sign-on)** (Без пароля (использование единого входа)).

9. В поле **E-mail address** (Адрес электронной почты) введите значение **yourname@yourcompanydomain.extension**, например BrittaSimon@contoso.com.

10. Прокрутите вниз до конца страницы и нажмите кнопку **Finish** (Завершить).

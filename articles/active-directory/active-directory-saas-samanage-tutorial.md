#Учебник. Интеграция Azure Active Directory с Samanage
>[AZURE.TIP]Чтобы оставить отзыв, щелкните [здесь](http://go.microsoft.com/fwlink/?LinkId=522516).
  
Цель данного учебника — показать интеграцию Azure и Samanage. Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент Samanage.
  
После завершения этого руководства пользователи Azure AD, назначенные в Samanage, будут иметь возможность единого входа в приложение на сайте Samanage компании (вход, инициированный поставщиком услуг) или с помощью инструкций из статьи [Общие сведения о панели доступа](https://msdn.microsoft.com/library/dn308586).
  
Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для Samanage
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-samanage-tutorial/IC771705.png "Сценарий")
##Включение интеграции приложений для Samanage
  
В этом разделе показано, как включить интеграцию приложений для Samanage.

###Чтобы включить интеграцию приложений для Samanage, выполните следующие действия:

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-samanage-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-samanage-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-samanage-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** щелкните **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-samanage-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В **поле поиска** введите **Samanage**.

    ![Коллекция приложений](./media/active-directory-saas-samanage-tutorial/IC771707.png "Коллекция приложений")

7.  В области результатов выберите **Samanage** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![Samanage](./media/active-directory-saas-samanage-tutorial/IC771708.png "Samanage")
##Настройка единого входа
  
В этом разделе показано, как разрешить пользователям проходить аутентификацию в Samanage со своей учетной записью Azure AD, используя федерацию на основе протокола SAML.  
В рамках этой процедуры потребуется создать файл сертификата в кодировке Base-64.  
Если вы не знакомы с этой процедурой, посмотрите видео [Преобразование двоичного сертификата в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

###Чтобы настроить единый вход, выполните следующие действия:

1.  На странице интеграции с приложением **Samanage** портала Azure AD щелкните **Настройка единого входа**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-samanage-tutorial/IC771709.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в Samanage** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Единый вход Microsoft Azure AD](./media/active-directory-saas-samanage-tutorial/IC771710.png "Единый вход Microsoft Azure AD")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес для входа в Samanage** введите свой URL-адрес, используя шаблон *https://\<имя-клиента>.samanage.com*, а затем нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-samanage-tutorial/IC771711.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в Samanage** нажмите кнопку **Загрузить сертификат**, а затем сохраните файл сертификата на компьютере.

    ![Настройка единого входа](./media/active-directory-saas-samanage-tutorial/IC777613.png "Настройка единого входа")

5.  В другом окне веб-браузера войдите на сайт Samanage компании в качестве администратора.

6.  Щелкните **Панель мониторинга** и выберите **Настройка** в левой области навигации.

    ![Панель мониторинга](./media/active-directory-saas-samanage-tutorial/IC771712.png "Панель мониторинга")

7.  Щелкните **Единый вход**.

    ![Единый вход](./media/active-directory-saas-samanage-tutorial/IC771713.png "Единый вход")

8.  На диалоговой странице **Вход с помощью SAML** выполните указанные ниже действия, после чего нажмите кнопку **Сохранить изменения**:

    1.  Щелкните **Включить единый вход с помощью SAML**.
        ![Вход с помощью SAML](./media/active-directory-saas-samanage-tutorial/IC771719.png "Вход с помощью SAML")
    2.  На диалоговой странице **Настройка единого входа в Samanage** портала Azure скопируйте значение поля **Идентификатор поставщика удостоверений** и вставьте его в текстовое поле **URL-адрес поставщика удостоверений**.
        ![Настройка единого входа](./media/active-directory-saas-samanage-tutorial/IC771720.png "Настройка единого входа")
    3.  На странице **Настройка единого входа в Samanage** портала Azure скопируйте значение поля **URL-адрес удаленного входа** и вставьте его в текстовое поле **URL-адрес входа**.
    4.  На странице **Настройка единого входа в Samanage** портала Azure скопируйте значение поля **URL-адрес удаленного выхода** и вставьте его в текстовое поле **URL-адрес выхода**.
    5.  Создайте файл **в кодировке base-64** из скачанного сертификата.  

        >[AZURE.TIP]Дополнительные сведения можно узнать в видео [Преобразование бинарного сертификата в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

    6.  Откройте сертификат в кодировке Base-64 в блокноте, скопируйте его содержимое в буфер обмена, а затем вставьте его в текстовое поле **Сертификат X.509**.
    7.  Щелкните **Создавать пользователей, если они не существуют в Samanage**.
        ![Блокировка изменений](./media/active-directory-saas-samanage-tutorial/IC771722.png "Блокировка изменений")
    8.  Нажмите кнопку **Обновить**.

9.  На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-samanage-tutorial/IC771723.png "Настройка единого входа")
##Настройка подготовки учетных записей пользователей
  
Чтобы пользователи Azure AD могли выполнять вход в Samanage, они должны быть подготовлены для Samanage. В случае с Samanage подготовка выполняется вручную.

###Чтобы подготовить учетные записи пользователей, выполните следующие действия.

1.  Выполните вход в клиент **Samanage**.

2.  Выберите **Панель мониторинга > Настройка**.

    ![Настройка](./media/active-directory-saas-samanage-tutorial/IC771724.png "Настройка")

3.  Откройте вкладку **Пользователи**.

    ![Пользователи](./media/active-directory-saas-samanage-tutorial/IC771725.png "Пользователи")

4.  Щелкните **Новый пользователь**.

    ![Новый пользователь](./media/active-directory-saas-samanage-tutorial/IC771726.png "Новый пользователь")

5.  В соответствующих полях введите **электронный адрес** и **имя** учетной записи Azure AD, которую необходимо подготовить, а затем нажмите кнопку **Сохранить**.

    >[AZURE.NOTE]Владелец учетной записи Azure AD получит электронное сообщение со ссылкой для активации учетной записи.

    ![Создание пользователя](./media/active-directory-saas-samanage-tutorial/IC771727.png "Создание пользователя")

>[AZURE.NOTE]Вы можете использовать любые другие инструменты создания учетной записи пользователя Samanage или API, предоставляемые Samanage для подготовки учетных записей пользователя AAD.

##Назначение пользователей
  
Чтобы проверить свою конфигурацию, предоставьте доступ пользователям Azure AD, которые должны использовать приложение, назначив их.

###Чтобы назначить пользователей Samanage, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **Samanage** щелкните **Назначить пользователей**.

    ![Назначение пользователей](./media/active-directory-saas-samanage-tutorial/IC771728.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да** для подтверждения назначения.

    ![Да](./media/active-directory-saas-samanage-tutorial/IC767830.png "Да")
  
Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](https://msdn.microsoft.com/library/dn308586).

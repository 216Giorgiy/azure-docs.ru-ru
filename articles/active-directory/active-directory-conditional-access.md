# Защита доступа к Office 365 и другим приложениям, подключенным к Azure Active Directory  
  
Защита доступа к ресурсам компании важна для каждой организации. С появлением облачных служб и мобильных устройств механизмы обращения пользователей к ресурсам компании сильно изменились. Это требует внесения изменений в стратегии по защите корпоративных ресурсов.
  
## Почему необходимо использовать условный доступ?  
 Возможности управления условным доступом в Azure Active Directory предлагают компаниям простые способы защиты своих ресурсов как в облаке, так и локально. Независимо от того, нужно ли вам "запретить доступ к моим ресурсам с помощью украденного пароля" или "иметь работоспособное и управляемое устройство для доступа к корпоративному содержимому", Azure Active Directory позволит удовлетворить эти потребности.

## Как обеспечивается управление условным доступом?  
 С условным контролем доступа Azure Active Directory проверяет определенные условия проверки подлинности пользователя, которые вы можете выбрать. Если эти условия выполняются, пользователь проходит проверку подлинности, и ему дается доступ к приложению.
   
![](./media/active-directory-conditional-access/conditionalaccess-overview.png)

## Управление доступом к ресурсам на основе пользователей
  
- **Атрибуты пользователя**: на уровне атрибутов пользователя можно применять политики, которые гарантируют, что только авторизованные пользователи смогут получить доступ к ресурсам компании.   
- **Членство пользователя в группе**: уровнем доступа пользователя также можно управлять на основе их членства в группе или группах.   
- **Многофакторная проверка подлинности (MFA)**: кроме того, также можно применять политики, в которых пользователь должен проверить свою подлинность с помощью многофакторной проверки подлинности. Например, можно заставить пользователя подтвердить ПИН-код на личном мобильном телефоне, чтобы обеспечить дополнительный уровень безопасности. Многофакторная проверка подлинности защищает ресурсы от несанкционированного доступа пользователя, который получил доступ к имени пользователя и паролю действительного пользователя. 

## Условный доступ на основе устройств 

- **Зарегистрированные устройства**: на уровне устройства можно задать политики, обеспечивающие доступ только с зарегистрированных или известных устройств. Кроме того, можно использовать решение управления мобильными устройствами (MDM), такое как Microsoft Intune, для разрешения доступа только управляемым устройствам. Условный доступ на уровне устройства гарантирует, что доступ разрешен только тем устройствам, которые соответствуют политикам, например, с принудительным вводом ПИН-кода. Кроме того, с помощью решения MDM мы можем обеспечить удаленное удаление корпоративных данных на утерянном или украденном устройстве.  
- **Политики устройства**: также можно задать политики для ограничения доступа только на основе приложений. Кроме того, также можно задать уровень доступа на основе физического расположения устройства, т. е. определив, поступает ли запрос из известной корпоративной сети или извне.  

Уровень доступа, который можно задать с помощью таких политик, можно применить как к облачным, так и к локальным ресурсам. В качестве ресурсов в облаке могут выступать приложения, такие как Office 365 и сторонние приложения SaaS. Кроме того, это могут быть бизнес-приложения, размещенные в облаке.

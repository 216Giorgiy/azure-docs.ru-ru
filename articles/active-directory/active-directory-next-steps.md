# Какие дальнейшие действия можно предпринимать с помощью Azure AD?

## Управление учетными записями пользователей в Azure Active Directory
Управление учетными записями пользователей — это больше, чем просто создание или удаление пользователей и назначение разрешений. Вы можете создавать отчеты о доступе и использовании, устанавливать политики паролей, выполнять самостоятельный сброс пароля, настроить многофакторную проверку подлинности и использовать многие другие преимущества. [Подробнее об управлении учетными записями пользователей](active-directory-create-users.md).

## Синхронизация и управление паролями
Синхронизируйте пароли пользователей, чтобы пользователи могли входить в облачные службы Майкрософт, такие как Office 365, Dynamics CRM и Microsoft Intune, используя тот же пароль, что и для входа в локальную сеть. [Подробнее об управлении паролями](active-directory-manage-passwords.md).

## Управление группами в Azure AD
Используйте группы для назначения общего набора разрешений и прав многим учетных записям одновременно вместо того, чтобы назначать каждую учетную запись по отдельности. Можно также синхронизировать группы из локальной службы Windows Server Active Directory. Подробнее об управлении группами в Azure Active Directory: [Подробнее об управлении группами](active-directory-manage-groups.md).

## Принудительное соблюдение политики условного доступа для зарегистрированных мобильных устройств
Регистрация устройств Azure AD предоставляет устройствам удостоверения, которые используются для аутентификации устройства, когда пользователь выполняет вход. Аутентифицированное устройство и его атрибуты затем можно использовать для принудительного соблюдения политик условного доступа в приложениях, размещенных в облаке и локально. [Подробнее о регистрации устройств Azure AD](active-directory-conditional-access.md).

#Руководство. Интеграция Azure Active Directory с Boomi
>[AZURE.TIP]Чтобы оставить отзыв, нажмите [здесь](http://go.microsoft.com/fwlink/?LinkId=529589).

Цель данного руководства — показать интеграцию Azure и Boomi. Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Подписка с поддержкой единого входа Boomi

После завершения этого руководства пользователи Azure AD, назначенные Boomi, будут иметь возможность единого входа в приложение на корпоративном веб-сайте Boomi (вход, инициированный поставщиком услуг) или с помощью инструкций из статьи [Общие сведения о панели доступа](https://msdn.microsoft.com/library/dn308586).

Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для Boomi
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-boomi-tutorial/IC791134.png "Сценарий")
##Включение интеграции приложений для Boomi

В этом разделе показано, как включить интеграцию приложений для Boomi.

###Чтобы включить интеграцию приложений для Boomi, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-boomi-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-boomi-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-boomi-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** нажмите **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-boomi-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В **поле поиска** введите **Boomi**.

    ![Коллекция приложений](./media/active-directory-saas-boomi-tutorial/IC790822.png "Коллекция приложений")

7.  В области результатов выберите **Boomi** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![Boomi](./media/active-directory-saas-boomi-tutorial/IC790823.png "Boomi")
##Настройка единого входа

В этом разделе показано, как разрешить пользователям проходить проверку подлинности в Boomi со своей учетной записью Azure AD, используя федерацию на основе протокола SAML.

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **Boomi** портала Azure AD нажмите **Настройка единого входа**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-boomi-tutorial/IC790824.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в Boomi?** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-boomi-tutorial/IC790825.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес ответа Boomi** введите свой **URL-адрес входа в Boomi AtomSphere** (например, “*https://platform.boomi.com/sso/AccountName/saml*”)) и нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-boomi-tutorial/IC790826.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в Boomi** нажмите кнопку **Загрузить сертификат**, а затем сохраните файл сертификата локально на компьютере.

    ![Настройка единого входа](./media/active-directory-saas-boomi-tutorial/IC790827.png "Настройка единого входа")

5.  В другом окне веб-браузера войдите на свой корпоративный веб-сайт Boomi в качестве администратора.

6.  На панели инструментов в верхней части экрана нажмите название своей организации и выберите параметр **Настроить**.

    ![Настройка](./media/active-directory-saas-boomi-tutorial/IC790828.png "Настройка")

7.  Нажмите **Параметры единого входа**.

    ![Параметры единого входа](./media/active-directory-saas-boomi-tutorial/IC790829.png "Параметры единого входа")

8.  В разделе **Параметры единого входа** выполните следующие действия.

    ![Параметры единого входа](./media/active-directory-saas-boomi-tutorial/IC790830.png "Параметры единого входа")

    1.  Установите флажок **Разрешить единый вход SAML**.
    2.  Нажмите кнопку **Импорт**, чтобы передать загруженный сертификат.
    3.  На странице **Настройка единого входа в Boomi** портала Azure скопируйте значение поля **URL-адрес удаленного входа** и вставьте его в текстовое поле **URL-адрес входа поставщика удостоверений**.
    4.  Для параметра **Идентификатор расположения федерации** выберите значение **Идентификатор федерации определяется элементом NameID оператора Subject**.
    5.  Щелкните **Сохранить**.

9.  На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-boomi-tutorial/IC775560.png "Настройка единого входа")
##Настройка подготовки учетных записей пользователей

Чтобы пользователи Azure AD могли выполнять вход в Boomi, они должны быть подготовлены для Boomi. В случае с Boomi подготовка выполняется вручную.

###Чтобы настроить подготовку учетных записей пользователей, выполните следующие действия.

1.  Выполните вход на веб-сайт компании **Boomi** в качестве администратора.

2.  Последовательно выберите пункты **Управление пользователями > Пользователи**.

    ![Пользователи](./media/active-directory-saas-boomi-tutorial/IC790831.png "Пользователи")

3.  Нажмите кнопку **Добавить пользователя**.

    ![Добавить пользователя](./media/active-directory-saas-boomi-tutorial/IC790832.png "Добавить пользователя")

4.  На диалоговой странице **Добавление ролей пользователей** выполните следующие действия.

    ![Добавить пользователя](./media/active-directory-saas-boomi-tutorial/IC790833.png "Добавить пользователя")

    1.  Введите **Имя**, **Фамилию** и **Адрес электронной почты** действующей учетной записи AAD, которую хотите подготовить, в соответствующие текстовые поля.
    2.  Нажмите кнопку ОК.

>[AZURE.NOTE]Вы можете использовать любые другие средства создания учетной записи пользователя Boomi или API, предоставляемые Boomi для подготовки учетных записей пользователя AAD.

##Назначение пользователей

Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которые должны использовать приложение, доступ путем их назначения.

###Чтобы назначить пользователей Boomi, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **Boomi** нажмите **Назначить пользователей**.

    ![Назначить пользователей](./media/active-directory-saas-boomi-tutorial/IC790834.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да** для подтверждения назначения.

    ![Да](./media/active-directory-saas-boomi-tutorial/IC767830.png "Да")

Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](https://msdn.microsoft.com/library/dn308586).

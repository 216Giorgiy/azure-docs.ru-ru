# <a name="tutorial-configure-asana-for-automatic-user-provisioning"></a>Руководство по настройке Asana для автоматической подготовки пользователей

Цель этого руководства — показать, как в Asana и Azure AD настроить автоматическую подготовку и отзыв учетных записей пользователей из Azure AD в Asana.

## <a name="prerequisites"></a>предварительным требованиям

Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

*   Клиент Azure Active Directory.
*   клиент Asana с планом [Enterprise](https://www.asana.com/pricing) или выше; 
*   учетная запись пользователя в Asana с разрешениями администратора. 

> [!NOTE] 
> Интеграция подготовки Azure AD зависит от [API Asana](https://app.asana.com/api/1.0/scim/Users), доступного для Asana.

## <a name="assigning-users-to-asana"></a>Назначение пользователей в Asana

В Azure Active Directory для определения того, какие пользователи должны получать доступ к выбранным приложениям, используется концепция, называемая "назначение". В контексте автоматической подготовки учетной записи синхронизированы будут только пользователи, которые были назначены приложению в Azure AD. 

Перед настройкой и включением службы подготовки необходимо решить, какие пользователи в Azure AD представляют пользователей, которым необходим доступ к приложению Asana. После этого можно назначить этих пользователей для приложения Asana, выполнив следующие действия:

[Назначение пользователя корпоративному приложению](active-directory-coreapps-assign-user-azure-portal.md)

### <a name="important-tips-for-assigning-users-to-asana"></a>Важные рекомендации по назначению пользователей в Asana

*   Рекомендуем назначить одного пользователя Azure AD в Asana для тестирования конфигурации подготовки. Дополнительные пользователи могут быть назначены позднее.

## <a name="configuring-user-provisioning-to-asana"></a>Настройка подготовки учетных записей пользователей в Asana 

В этом разделе описывается подключение вашего каталога Azure AD к API подготовки учетных записей пользователей в Asana и настройка службы подготовки для создания, обновления и отключения назначенных учетных записей пользователей в Asana на основе назначения пользователей в Azure AD.

> [!TIP]
> Для Asana также можно включить единый вход на основе SAML. Для этого следуйте инструкциям на [портале Azure](https://portal.azure.com). Единый вход можно настроить независимо от автоматической подготовки, хотя эти две возможности дополняют друг друга.

### <a name="to-configure-automatic-user-account-provisioning-to-asana-in-azure-ad"></a>Настройка автоматической подготовки учетных записей пользователей Azure AD в Asana.

1)  На [портале Azure](https://portal.azure.com) перейдите в раздел **Azure Active Directory > Корпоративные приложения > Все приложения**.

2) Если для Asana уже настроен единый вход, найдите свой экземпляр Asana с помощью функции поиска. В противном случае щелкните **Добавить** и найдите **Asana** в коллекции приложений. Выберите **Asana** в результатах поиска и добавьте его в список приложений.

3)  Выберите экземпляр Asana и откройте вкладку **Подготовка**.

4)  Для параметра **Режим подготовки к работе** выберите значение **Automatic** (Автоматически).

![Подготовка Asana](./media/active-directory-saas-asana-provisioning-tutorial/asanaazureprovisioning.png)

5) В разделе учетных данных администратора выполните приведенные ниже инструкции по созданию токена и введите его в текстовое поле **Секретный токен**.

* Войдите в [Asana](https://app.asana.com), используя учетною запись администратора.
* Щелкните фотографию профиля на верхней панели и выберите текущие параметры для названия организации.
* Откройте вкладку учетных записей служб.
* Щелкните Add Service Account (Добавить учетную запись службы).
* При необходимости обновите имя, сведения и фотографию профиля, скопируйте **токен** и щелкните "Сохранить изменения".

6) На портале Azure щелкните **Проверить подключение**, чтобы убедиться, что Azure AD может подключиться к приложению Asana. Если подключение отсутствует, убедитесь, что учетная запись Asana предоставляет права администратора, и повторите шаг **Проверить подключение**.

7) В поле **Почтовое уведомление** введите адрес электронной почты пользователя или группы, которые должны получать уведомления об ошибках подготовки, а также установите флажок ниже.

8) Выберите команду **Сохранить**. 

9) В разделе "Сопоставления" выберите **Synchronize Azure Active Directory Users to Asana** (Синхронизировать пользователей Azure Active Directory с Asana).

10) В разделе **Сопоставления атрибутов** следует просмотреть атрибуты пользователей, которые будут синхронизированы из Azure AD в Asana. Обратите внимание, что атрибуты, которые выбраны в качестве свойств **Matching** (Соответствие), будут использоваться для сопоставления учетных записей пользователей в Asana для операций обновления. Нажмите кнопку **Сохранить**, чтобы зафиксировать все изменения. Дополнительные сведения см. в статье о [настройке сопоставлений атрибутов для подготовки пользователей](active-directory-saas-customizing-attribute-mappings.md).

11) Чтобы включить службу подготовки Azure AD для Asana, в разделе **Параметры** измените значение параметра **Состояние подготовки** на **Включено**.

12) Выберите команду **Сохранить**. 

Будет запущена начальная синхронизация всех пользователей, назначенных для Asana в разделе "Пользователи". Начальная операция синхронизации занимает больше времени, чем последующие. Последующие операции синхронизации выполняются примерно каждые 20 минут при условии, что служба запущена. В разделе **Сведения о синхронизации** можно отслеживать ход выполнения и переходить по ссылкам для просмотра отчетов по подготовке, в которых зафиксированы все действия, выполняемые службой подготовки в приложении Asana.

Дополнительные сведения о чтении журналов подготовки Azure AD см. в руководстве по [отчетам об автоматической подготовке учетных записей](active-directory-saas-provisioning-reporting.md).

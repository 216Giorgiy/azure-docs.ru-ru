# <a name="gdpr-in-the-azure-active-directory-application-proxy"></a>GDPR в прокси приложения Azure Active Directory  

Прокси приложения Azure Active Directory (Azure AD), как и все остальные службы и функции корпорации Майкрософт, соответствует требованиям GDPR. Дополнительные сведения о поддержке GDPR корпорацией Майкрософт см. в [условиях лицензионного соглашения и документации по лицензированию](http://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&DocumentTypeId=31).

Так как этот компонент включает соединители, устанавливаемые на компьютерах, для поддержания соответствия требованиям GDPR необходимо отслеживать несколько событий. Прокси приложения создает журналы следующих типов, которые могут содержать EUII:

- Журналы событий соединителя
- Журналы событий Windows

Соответствие GDPR можно обеспечить двумя способами.

- удалять и экспортировать все запросы по мере возникновения;

- отключить ведение журналов.

Сведения:

- Сведения о том, как настроить хранение данных журналов событий Windows, см. [здесь](https://technet.microsoft.com/library/cc952132.aspx). 
- Общие сведения о журнале событий Windows см. в [этой статье](https://msdn.microsoft.com/library/windows/desktop/aa385772.aspx).


Журналы событий соединителя размещены в каталоге `C:\ProgramData\Microsoft\Microsoft AAD Application Proxy Connector\Trace`. В следующих разделах описаны соответствующие действия для журналов событий соединителя. Эти действия следует выполнить на всех компьютерах с соединителями.
 

## <a name="processing-requests"></a>Обработка запросов

Вам нужно контролировать запросы трех разных типов: 

- Delete
- Просмотр
- экспорт.
 
Для обработки запросов на просмотр и экспорт необходимо просмотреть все файлы журналов и найти в них соответствующие записи. 

Так как файлы журналов имеют текстовый формат, поиск записей для конкретного пользователя можно выполнять, например, с помощью команды `findstr`. Выполните поиск следующих полей, которые могут встречаться в журналах: 

- UserId
- Типы имен пользователей, настроенные для всех приложений, использующих ограниченное делегирование Kerberos:
    - Имя участника-пользователя
    - имя локального участника-пользователя;
    - компонент, содержащий имя пользователя, в имени участника-пользователя;
    - компонент, содержащий имя пользователя, в имени локального участника-пользователя;
    - имя локальной учетной записи SAM. 

 
Все эти поля можно собирать и передавать пользователю.
Для обработки запросов на удаление нужно удалять соответствующие журналы. Чтобы создать новый файл журнала, можно перезапустить службу соединителя (соединитель Microsoft Azure AD Application Proxy). После создания нового файла журнала можно смело удалять старые файлы. Теперь вы можете выполнить тот же процесс, который описан для просмотра и экспорта, чтобы найти все соответствующие журналы и выборочно удалить поля или файлы. Если старые файлы журналов вам не нужны, можно просто удалить их все сразу.

#Руководство. Интеграция Azure Active Directory с Kudos
  
Цель данного руководства — показать интеграцию Azure и Kudos.  
Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент Kudos
  
По завершении работы с этим руководством пользователи Azure AD, назначенные в Kudos, смогут выполнять единый вход в приложение на веб-сайте Kudos компании (вход, инициированный поставщиком услуг) или следуя указаниям в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).
  
Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для Kudos
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-kudos-tutorial/IC787799.png "Сценарий")
##Включение интеграции приложений для Kudos
  
В этом разделе показано, как включить интеграцию приложений для Kudos.

###Чтобы включить интеграцию приложений для Kudos, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-kudos-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-kudos-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-kudos-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** нажмите **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-kudos-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В **поле поиска** введите **Kudos**.

    ![Коллекция приложений](./media/active-directory-saas-kudos-tutorial/IC787800.png "Коллекция приложений")

7.  В области результатов выберите **Kudos** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![Kudos](./media/active-directory-saas-kudos-tutorial/IC787801.png "Kudos")
##Настройка единого входа
  
В этом разделе показано, как разрешить пользователям проходить проверку подлинности в Kudos со своей учетной записью Azure AD, используя федерацию на основе протокола SAML.  
В рамках этой процедуры потребуется создать файл сертификата в кодировке Base-64.  
Если вы не знакомы с этой процедурой, посмотрите видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **Kudos** портала Azure AD нажмите кнопку **Настроить единый вход**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-kudos-tutorial/IC787802.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в Kudos** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-kudos-tutorial/IC787803.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес для входа в Kudos** введите свой URL-адрес, используя следующий шаблон "*https://company.kudosnow.com*", а затем нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-kudos-tutorial/IC787804.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в Kudos** нажмите кнопку **Загрузить сертификат**, а затем сохраните файл сертификата на компьютере.

    ![Настройка единого входа](./media/active-directory-saas-kudos-tutorial/IC787805.png "Настройка единого входа")

5.  В другом окне веб-браузера войдите на ваш корпоративный веб-сайт Kudos в качестве администратора.

6.  В верхнем меню нажмите пункт **Параметры**.

    ![данных](./media/active-directory-saas-kudos-tutorial/IC787806.png "данных")

7.  Выберите пункты **Интеграции > Единый вход**.

8.  В разделе **Единый вход** сделайте следующее:

    ![Единый вход](./media/active-directory-saas-kudos-tutorial/IC787807.png "Единый вход")

    1.  На странице диалогового окна **Настройка единого входа в Kudos** портала Azure скопируйте значение поля **URL-адрес службы единого входа** и вставьте его в текстовое поле **URL-адрес входа**.
	2.  Создайте файл **в кодировке Base-64** из скачанного сертификата.  

        >[AZURE.TIP]
        Дополнительные сведения можно узнать из видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

    3.  Откройте сертификат в кодировке Base-64 в Блокноте, скопируйте его содержимое в буфер обмена, а затем вставьте его в текстовое поле **Сертификат X.509**.
    4.  На странице диалогового окна **Настройка единого входа в Kudos** портала Azure скопируйте значение поля **URL-адрес службы единого выхода** и вставьте его в текстовое поле **URL-адрес выхода из системы**.
	5.  В текстовом поле **URL-адрес Kudos** введите название своей компании.
    6.  Щелкните **Сохранить**.

9.  На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-kudos-tutorial/IC787808.png "Настройка единого входа")
##Настройка подготовки учетных записей пользователей
  
Чтобы разрешить пользователям Azure AD вход в Kudos, они должны быть подготовлены для Kudos. В случае с Kudos подготовка выполняется вручную.

###Чтобы подготовить учетные записи пользователей, выполните следующие действия.

1.  Выполните вход на веб-сайт **Kudos** компании в качестве администратора.

2.  В меню в верхней части страницы щелкните **Параметры**.

    ![данных](./media/active-directory-saas-kudos-tutorial/IC787806.png "данных")

3.  Выберите **Администратор пользователей**.

4.  Откройте вкладку **Пользователи**, а затем нажмите элемент **Добавить пользователя**.

    ![Администратор пользователей](./media/active-directory-saas-kudos-tutorial/IC787809.png "Администратор пользователей")

5.  В разделе **Добавить пользователя** выполните следующие действия.

    ![Добавить пользователя](./media/active-directory-saas-kudos-tutorial/IC787810.png "Добавить пользователя")

    1.  Заполните поля **Имя**, **Фамилия**, **Адрес электронной почты** и введите другие данные действующей учетной записи Azure Active Directory, которую требуется подготовить, в соответствующих текстовых полях.
    2.  Нажмите кнопку **Создать пользователя**.

>[AZURE.NOTE]Вы можете использовать любые другие средства создания учетной записи пользователя Kudos или API, предоставляемые Kudos для подготовки учетных записей пользователя AAD.

##Назначение пользователей
  
Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которые должны использовать приложение, доступ путем их назначения.

###Чтобы назначить пользователей Kudos, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **Kudos** нажмите кнопку **Назначить пользователей**.

    ![Назначить пользователей](./media/active-directory-saas-kudos-tutorial/IC787811.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да**, чтобы подтвердить назначение.

    ![Да](./media/active-directory-saas-kudos-tutorial/IC767830.png "Да")
  
Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).

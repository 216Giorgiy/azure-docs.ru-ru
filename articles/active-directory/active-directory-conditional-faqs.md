# Часто задаваемые вопросы об условном доступе

## Какие приложения работают с политиками условного доступа?
См. статью [Conditional access- What applications are supportedу](active-directory-conditional-access-supported-apps.md) (Условный доступ. Поддерживаемые приложения).

## Являются ли политики условного доступа обязательными для службы совместной работы B2B и гостевых пользователей?
Политики являются обязательными для пользователей службы совместной работы B2B. Тем не менее, в некоторых случаях пользователь может не быть удовлетворять требованиям политики, например, если его организация не поддерживает многофакторную проверку подлинности. В настоящее время политика не является обязательной для гостевых пользователей SharePoint. В SharePoint поддерживается гостевое отношение, таким образом, учетные записи гостевых пользователей не подвергаются проверке на соответствие политикам доступа на сервере проверки подлинности. В SharePoint гостевым доступом можно управлять.

## Распространяется ли политика SharePoint Online на OneDrive для бизнеса?
Да.
 
## Почему не удается задать политику для клиентских приложений, таких как Word или Outlook?
Политика условного доступа устанавливает требования для доступа к службе и является обязательной, когда в этой службе выполняется проверка подлинности. Поэтому политика не задается непосредственно в клиентском приложении, а применяется при вызове в службу. Например, политика, заданная в SharePoint, будет применяться к клиентам, вызывающим SharePoint, а политика, заданная в Exchange, будет применяться к Outlook.

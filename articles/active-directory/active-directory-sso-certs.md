# <a name="managing-certificates-for-federated-single-sign-on-in-azure-active-directory"></a>Управление сертификатами для федеративного единого входа в Azure Active Directory
В этой статье рассматриваются распространенные вопросы и предоставляются сведения, связанные с сертификатами, создаваемыми службой Azure Active Directory для обеспечения федеративного единого входа в приложения SaaS. Эти приложения можно добавить из коллекции приложений Azure AD или с помощью шаблона приложения не из коллекции. Приложение должно быть настроено с помощью федеративного единого входа.

Эта статья относится только к тем приложениям, которые настроены для использования функции **единого входа Azure AD** с **федерацией SAML**, как показано в следующем примере:

![единого входа Azure AD](./media/active-directory-sso-certs/saml_sso.PNG)

## <a name="auto-generated-certificate-for-gallery-and-non-gallery-applications"></a>Автоматически созданный сертификат для приложений из коллекции и не из коллекции
При добавлении нового приложения из коллекции и настройке входа на основе SAML служба Azure AD создает для этого приложения сертификат со сроком действия 3 года. Этот сертификат можно скачать в разделе **Сертификат подписи SAML**. Для приложений из коллекции в этом разделе может предлагаться скачать **Сертификат** или **Метаданные федерации** в зависимости от требований приложения.

![единого входа Azure AD](./media/active-directory-sso-certs/saml_certificate_download.png)

## <a name="how-to-customize-the-expiration-date-for-your-federation-certificate-and-roll-over-the-new-certificate"></a>Настройка даты окончания срока действия сертификата федерации и смена нового сертификата
По умолчанию срок действия сертификатов истекает через три года. Вы можете выбрать другой срок действия сертификата, выполнив следующие действия. На приведенных снимках экрана для примера используется Salesforce, но эти действия можно применять для любого федеративного приложения SaaS.

1. На портале управления [Azure Active Directory](https://aad.portal.azure.com) щелкните меню навигации **Корпоративное приложение**, а затем на странице **Обзор** щелкните **Новое приложение**, как показано ниже:

   ![Открытие мастера настройки единого входа](./media/active-directory-sso-certs/enterprise_application_new_application.png)

2. Выполните поиск приложений в коллекции и выберите то приложение, которое необходимо добавить. Если не удается найти необходимое приложение, то добавьте его с помощью параметра **Приложение не из коллекции**. Эта функция доступна только для владельцев версии **Azure AD Premium (P1 и P2)**.

    ![единого входа Azure AD](./media/active-directory-sso-certs/add_gallery_application.png)

3. Щелкните ссылку **Единый вход** в левой области навигации и замените **Режим единого входа** на **Вход на основе SAML**. При этом для вашего приложения будет создан сертификат со сроком действия 3 года.

4. Чтобы создать сертификат, в разделе **Сертификат подписи SAML** щелкните ссылку **Создать сертификат**.

    ![Создание нового сертификата](./media/active-directory-sso-certs/create_new_certficate.png)

5. При нажатии ссылки "Создать сертификат" откроется календарь, в котором можно выбрать любую дату и время в интервале до трех лет. Выбранные дата и время будут датой и временем истечения срока действия нового сертификата. Чтобы сохранить сертификат, нажмите кнопку **Сохранить**.

    ![Загрузка и отправка сертификата](./media/active-directory-sso-certs/certifcate_date_selection.PNG)

6. Теперь новый сертификат доступен для скачивания. Щелкните ссылку **Сертификат**, чтобы скачать его. На этом этапе сертификат еще не активен. Когда требуется перейти на этот сертификат, установите флажок **Make new certificate active** (Сделать новый сертификат активным) и нажмите кнопку "Сохранить". С этого момента служба Azure AD начинает использовать новый сертификат для подписания ответов.

7.  Чтобы узнать, как передать сертификат в определенное приложение SaaS, щелкните на странице ссылку "View application configuration tutorial" (Просмотреть руководство по настройке приложения).

## <a name="how-to-renew-a-certificate-that-will-soon-expire"></a>Обновление сертификата с истекающим сроком действия
Описанный ниже процесс обновления в идеале не приведет к значительному простою. Приведенные в этом разделе снимки экрана описывают использование Salesforce в качестве примера. Эти действия можно применить к любому федеративному приложению SaaS.

1. На странице **единого входа** приложения **Azure Active Directory** создайте сертификат для своего приложения. Для этого в разделе **Сертификат подписи SAML** щелкните ссылку **Создать сертификат**.

    ![Создание нового сертификата](./media/active-directory-sso-certs/create_new_certficate.png)

2. Выберите дату и время истечения срока действия нового сертификата и нажмите кнопку **Сохранить**.

3. Скачайте сертификат в разделе "Сертификат подписи SAML". Загрузите новый сертификат в экран настройки единого входа приложения SaaS. Чтобы узнать, как это сделать для определенного приложения SaaS, щелкните на странице ссылку **View application configuration tutorial** (Просмотреть руководство по настройке приложения).
   
4. Чтобы активировать новый сертификат в Azure AD, установите флажок "Make new certificate active" (Сделать новый сертификат активным) и нажмите кнопку **Сохранить** в верхней части страницы. При этом на стороне Azure AD активируется новый сертификат. Состояние сертификата меняется с **Новый** на **Активный**. С этого момента служба Azure AD начинает использовать новый сертификат для подписания ответов. 
   
    ![Создание нового сертификата](./media/active-directory-sso-certs/new_certificate_download.png)

# <a name="what-is-azure-ad-b2b-collaboration-preview"></a>Что такое предварительная версия службы совместной работы Azure AD B2B?

Возможности службы совместной работы Azure AD B2B позволяют ИТ-специалистам и информационным работникам тесно сотрудничать с партнерами в любой другой организации по всему миру. Они могут предоставлять доступ к документам, ресурсам и приложениям, сохраняя при этом полный контроль над их внутренними данными. Разработчики могут использовать предоставляемые Azure AD интерфейсы API типа "бизнес — бизнес" для написания приложений, которые защищенным способом соединяют две организации и обеспечивают информационных работников непрерывным доступом и интуитивно понятной навигацией.

Возможности службы совместной работы Azure AD B2B позволяют организациям любого размера и из любой отрасли, независимо от их нормативных требований, эффективно и безопасно сотрудничать с партнерами по всему миру. Именно это является целью данного обновления общедоступной предварительной версии службы совместной работы B2B.

## <a name="how-does-it-work"></a>Как это работает?

В текущем предварительном выпуске для установления связи с организацией ИТ-специалисты и информационные работники могут добавлять пользователей из другой организации по одному или по несколько одновременно, используя портал или API диспетчера приглашений. Чтобы сделать то же самое, администраторы могут воспользоваться новыми возможностями на портале Azure (https://portal.azure.com) и PowerShell. А информационные работники могут использовать панель доступа на странице http://myapps.microsoft.com. Разработчики могут создавать приложения, используя API диспетчера приглашений Azure AD B2B для добавления пользователей службы совместной работы B2B и настройки рабочих процессов приглашения и подключения.

Как правило, пользователи службы совместной работы B2B подключаются через процедуру отправки приглашения и последующей активации. Вот как это работает.

1. Предположим, пользователь Джон Доу (John Doe) из компании WoodGrove хочет добавить пользователя Сэма Угла (Sam Oogle), используя его адрес электронной почты Gmail (gsamoogle@gmail.com).

2. Джон переходит на портал WoodGrove (portal.azure.com) или на панель доступа (myapps.microsoft.com), входит в систему и добавляет пользователя в каталог (или группу, или приложение) WoodGrove.

3. Джон указывает адрес электронной почты Сэма, на который будет отправлено приглашение.

4. После этого в WoodGrove AD создается следующий пользователь (снимок экрана взят из интерфейса администратора на сайте portal.azure.com):

  ![пользователь добавлен](media/active-directory-b2b-what-is-azure-ad-b2b/user-is-added.png)

5. Когда этот пользователь добавлен, Azure AD отправляет Сэму сообщение с приглашением:

  ![сообщение с приглашением отправлено Сэму](media/active-directory-b2b-what-is-azure-ad-b2b/invitation-mail-sent-to-sam.png)

6. Теперь Сэм должен нажать кнопку **Начало работы** и войти в систему. В этот момент Azure AD обновляет его объект пользователя в каталоге, применяя данные из его маркера (снимок экрана взят из интерфейса администратора на [портале Azure](https://portal.azure.com)):

  ![заполняется профиль пользователя](media/active-directory-b2b-what-is-azure-ad-b2b/user-profile-is-populated.png)

7. Теперь, когда приглашение Сэма активировано, он может получить доступ к ресурсам WoodGrove. И, конечно, администратор может управлять его данными, как и любого другого пользователя в каталоге (снимок экрана взят из интерфейса администратора на [портале Azure](https://portal.azure.com)):

  ![Теперь Сэм является пользователем Azure AD](media/active-directory-b2b-what-is-azure-ad-b2b/sam-now-user-in-azure-ad.png)

## <a name="public-preview-features"></a>Функции общедоступной предварительной версии
Вы уже используете возможности службы совместной работы B2B, представленные в общедоступной предварительной версии, и мы получаем от вас массу превосходных отзывов. И мы прислушиваемся к вашим пожеланиям! В данное обновление общедоступной предварительной версии включены все собранные улучшения. Ниже перечислены основные возможности, включенные в это обновление общедоступной предварительной версии службы совместной работы B2B.

1. Усовершенствованный интерфейс администратора B2B:
  - теперь администрировать можно с сайта https://portal.azure.com;
  - администраторы могут приглашать пользователей B2B в каталог, группу или приложение.

2. Возможность для информационных работников самостоятельно отправлять приглашения в службу совместной работы B2B с помощью панели доступа: https://myapps.microsoft.com. Информационные работники могут приглашать пользователей службы совместной работы B2B в любую самостоятельную группу или приложение, которыми они управляют.

3. Теперь можно пригласить пользователя, используя любой адрес электронной почты. У пользователя может иметь локальный адрес электронной почты Microsoft Exchange, адрес Office&365;, outlook.com или любой внешний адрес электронной почты (Gmail, Yahoo и т. д.), при этом он может с легкостью получить доступ к приглашающей организации, выполнив простую встроенную процедуру создания учетной записи Azure AD или учетной записи Майкрософт.

4. Преимущества профессионального сообщения с приглашением с фирменной символикой клиента.

5. Расширенные возможности настройки подключения с помощью интерфейсов API приглашений.

6. Многофакторная идентификация для пользователей службы совместной работы B2B в приглашающей организации.

7. Возможность делегирования права приглашения пользователям, не являющимся администраторами.

8. Поддержка PowerShell для службы совместной работы B2B.

9. Возможности аудита и отчетности.

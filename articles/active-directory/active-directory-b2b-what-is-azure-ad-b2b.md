# <a name="about-the-azure-ad-b2b-collaboration-preview"></a>Сведения о предварительной версии службы совместной работы Azure AD B2B
В этой статье рассматриваются цели, возможности и преимущества обновления общедоступной предварительной версии службы совместной работы Azure Active Directory (Azure AD) B2B.

Как ИТ-специалисты, так и информационные работники могут воспользоваться преимуществами возможностей службы совместной работы Azure AD B2B. Вы можете тесно сотрудничать с партнерами из других организаций по всему миру, независимо от размера и отрасли организаций или их нормативных требований. Вы можете предоставлять им доступ к документам, ресурсам и приложениям, сохраняя при этом полный контроль над внутренними данными.

С помощью API-интерфейсов Azure AD B2B разработчики могут создавать приложения, обеспечивающие безопасную совместную работу организаций. Вопрос о процессе и навигации не вызывает затруднений у информационных работников.

## <a name="how-b2b-collaboration-works"></a>Принцип совместной работы B2B

В текущем предварительном выпуске ИТ-специалисты и информационные работники устанавливают связи с партнерскими организациями, добавляя пользователей из другой организации по одному или по несколько одновременно с помощью портала Azure или API диспетчера приглашений.

Чтобы сделать то же самое, администраторы могут воспользоваться новыми возможностями на портале Azure (https://portal.azure.com) и PowerShell.

Информационные работники могут использовать панель доступа на странице http://myapps.microsoft.com.

Разработчики могут создавать приложения, используя API диспетчера приглашений Azure AD B2B для добавления пользователей службы совместной работы B2B и настройки рабочих процессов приглашения и подключения.

Как правило, пользователи службы совместной работы B2B добавляются через процедуру отправки приглашения и последующей активации. Вот как это работает.

1. Предположим, пользователь Джон Доу из компании WoodGrove хочет добавить пользователя Сэма Угла, используя его адрес электронной почты Gmail gsamoogle@gmail.com.

2. Джон Доу переходит на портал WoodGrove (portal.azure.com) или на панель доступа (myapps.microsoft.com), входит в систему и добавляет пользователя Сэма Угла в каталог (или группу, или приложение) WoodGrove.

3. Джон Доу указывает адрес электронной почты Сэма Угла, по которому будет отправлено приглашение.

4. Когда Джон Доу отправляет приглашение, в Azure AD для WoodGrove создается пользователь Сэм Угл, как показано ниже:

  ![Пользовательский интерфейс администратора на сайте portal.azure.com](media/active-directory-b2b-what-is-azure-ad-b2b/user-is-added.png)

5. После создания пользователя Azure AD отправляет приглашение по электронной почте Сэму Углу:

  ![Отправка сообщения с приглашением Сэму Углу](media/active-directory-b2b-what-is-azure-ad-b2b/invitation-mail-sent-to-sam.png)

6. В приглашении Сэм Угл выбирает **Начало работы**.  
Открывается портал Azure.

7. Сэм Угл входит на портал Azure.

8. Azure AD обновляет объект пользователя Сэма Угла в Azure AD с данными его токена, как показано ниже:

  ![Профиль пользователя Сэма Угла на портале Azure](media/active-directory-b2b-what-is-azure-ad-b2b/user-profile-is-populated.png)

9. Теперь, когда пользователь Сэм Угл активировал приглашение, он сможет получить доступ к ресурсам WoodGrove. Как и любой другой пользователь в Azure AD, пользователем Сэм Угл может управлять администратор. Ниже приведен список пользователей.

  ![Список пользователей Azure AD](media/active-directory-b2b-what-is-azure-ad-b2b/sam-now-user-in-azure-ad.png)

## <a name="public-preview-features"></a>Функции общедоступной предварительной версии
Исходя из отзывов пользователей, обновление общедоступной предварительной версии службы совместной работы B2B предоставляет такие основные возможности:

* Для администраторов: получение улучшений пользовательского интерфейса с помощью [портала Azure](https://portal.azure.com). Например, администраторы могут приглашать пользователей B2B в каталог, группу или приложение.  

* Для информационных работников: самостоятельная отправка приглашения в службу совместной работы B2B с помощью [панели доступа](https://myapps.microsoft.com). Информационные работники могут приглашать пользователей службы совместной работы B2B в любую самостоятельную группу или приложение, которыми они управляют.

* Приглашенным пользователям разрешается иметь любой адрес электронной почты. Пользователи могут получать доступ к приглашенной организации, создав учетную запись Azure AD или Майкрософт, независимо от используемой учетной записи (Office 365 или локальная учетная запись Microsoft Exchange, outlook.com или учетная запись любой социальной сети (Gmail, Yahoo! и т. д.)).

* Создание профессионального сообщения с приглашением с фирменной символикой клиента.

* Настройка ориентации пользователя с помощью интерфейсов API приглашений.

* Настройка Многофакторной идентификации для пользователей службы совместной работы B2B в приглашающей организации.

* Делегирование права приглашения пользователям, не являющимся администраторами.

* Обеспечение поддержки PowerShell для службы совместной работы B2B.

* Возможности аудита и отчетности.

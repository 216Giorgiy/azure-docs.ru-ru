# <a name="how-to-get-an-azure-active-directory-tenant"></a>Как получить клиент Azure Active Directory
В Azure Active Directory (Azure AD) [клиент](https://msdn.microsoft.com/library/azure/jj573650.aspx#Anchor_0) представляет организацию.  Это выделенный экземпляр службы Azure AD, который организация получает и которым начинает владеть после начала сотрудничества с корпорацией Майкрософт, то есть после подписки на облачную службу Майкрософт, например Azure, Microsoft Intune или Office 365.  Каждый клиент Azure AD отделен от остальных клиентов Azure AD.  

Клиент содержит пользователей в организации и информацию о них, такую как пароли, данные профилей пользователей, разрешения и т. д.  Кроме того, он содержит группы, приложения и другие данные, относящиеся к организации и ее системе безопасности.

Чтобы разрешить пользователям Azure AD вход в ваше приложение, необходимо зарегистрировать его в своем клиенте.  Вы можете создать клиент Azure AD и опубликовать приложение в нем **абсолютно бесплатно**. Но вы можете приобрести расширенные функции в клиенте.  На самом деле большинство разработчиков создают несколько клиентов и приложений для экспериментов, разработки, промежуточного хранения и тестирования.

## <a name="use-an-existing-azure-ad-tenant"></a>Использование существующего клиента Azure AD

У многих разработчиков уже есть клиенты благодаря службам или подпискам, которые связаны с клиентами Azure AD (например, подписки Office 365 или Azure).  Чтобы проверить, есть ли у вас клиент, войдите на [портал Azure](https://portal.azure.com) с помощью учетной записи, которую вы хотите использовать для управления приложением и посмотрите в правый верхний угол, где отображаются сведения об учетной записи.  Если у вас есть клиент, вы автоматически войдете в него и увидите имя клиента непосредственно под своей учетной записью.  Если ваша учетная запись связана с несколькими клиентами, щелкните имя учетной записи, чтобы открыть меню, где можно переключаться между клиентами.

Если у вас нет клиента, связанного с учетной записью, под именем учетной записи отобразится GUID и вы не сможете регистрировать приложения, пока не [создадите новый клиент](#create-a-new-azure-ad-tenant).

## <a name="create-a-new-azure-ad-tenant"></a>Создание нового клиента Azure AD

Если у вас еще нет клиента Azure AD или вы хотите создать новый клиент, это можно сделать, используя [процедуру создания каталога](https://portal.azure.com/#create/Microsoft.AzureActiveDirectory) на [портале Azure](https://portal.azure.com).  Процесс займет около минуты, после чего вам будет предложено перейти в только что созданный клиент.

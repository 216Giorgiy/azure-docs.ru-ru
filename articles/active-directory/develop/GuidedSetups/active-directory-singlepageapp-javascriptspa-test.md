
## <a name="test-your-code"></a>Тестирование кода

Если вы используете Visual Studio, то нажмите `F5`, чтобы запустить проект. Откроется браузер, и вы будете перенаправлены на страницу *http://localhost:{порт}*, где вы увидите кнопку *Call Microsoft Graph API* (Вызвать API Microsoft Graph).

Если вы не используете Visual Studio, то убедитесь, что веб-сервер запущен и что на нем настроена папка, содержащая веб-приложение JavaScript. Откройте браузер и введите *http://localhost:{port}/path*, где *port* соответствует порту, с которого веб-сервер ожидает передачи данных, а *path* — это путь к файлу index.html.

Нажмите кнопку *Call Microsoft Graph API* (Вызвать API Microsoft Graph). Если действие выполняется впервые, то отобразится всплывающее окно, предлагающее пользователю войти в систему.
 
![Пример снимка экрана](media/active-directory-singlepageapp-javascriptspa-test/javascriptspascreenshot1.png)


### <a name="consent"></a>Согласие на предоставление разрешений
При первом входе в приложение появится экран согласия, как показано ниже, где необходимо явно дать согласие:

 ![Экран согласия](media/active-directory-singlepageapp-javascriptspa-test/javascriptspaconsent.png)

Так как вы запрашиваете API Microsoft Graph, может отобразиться еще одна страница согласия. Это происходит из-за *динамического согласия*, при котором согласие требуется для каждой области, запрашиваемой приложением. В примере приложения, созданном в этом руководстве, предоставляется область *user.read*, поэтому необходимо дать согласие на чтение приложением профиля пользователя.

> [!IMPORTANT]
> В настоящее время из-за известной проблемы с *msal* javascript необходимо отключить блокирование всплывающих окон в браузерах Chrome и Firefox, чтобы окно *динамического согласия* работало корректно. Отключение блокирования всплывающих является обязательным, если пользователь впервые вызывает API Microsoft Graph с помощью `acquireTokenPopup`.

### <a name="more-information-about-scopes-and-delegated-permissions"></a>Дополнительные сведения об областях и делегированных разрешениях

API Microsoft Graph требуется область `user.read` для чтения профиля пользователя. По умолчанию эта область автоматически добавляется в каждое приложение, регистрируемое на портале регистрации. Некоторым другим API для Microsoft Graph, а также пользовательским API для вашего внутреннего сервера могут требоваться дополнительные области. Например, для Microsoft Graph требуется область `Calendars.Read`, чтобы отобразить список календарей пользователя. Чтобы получить доступ к календарю пользователя в контексте приложения, в сведения о регистрации приложения необходимо добавить делегированное разрешение `Calendars.Read`, а затем добавить область `Calendars.Read` в вызов `acquireTokenSilent`. При увеличении количества областей от пользователя могут потребоваться дополнительные согласия.

Если для API серверной части не требуется область (не рекомендуется), то вы можете использовать `clientId` в качестве области в вызове `acquireTokenSilent` и (или) вызовах `acquireTokenPopup`.

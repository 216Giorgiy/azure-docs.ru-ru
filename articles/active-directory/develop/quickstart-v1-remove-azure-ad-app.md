# <a name="quickstart-remove-an-application-from-azure-active-directory"></a>Краткое руководство. Удаление приложения в Azure Active Directory

Корпоративным разработчикам и поставщикам программного обеспечения (SaaS), которые зарегистрировали приложения с помощью Azure Active Directory (Azure AD), возможно, потребуется удалить регистрацию приложения своего клиента Azure AD.

Из этого краткого руководства вы узнаете, как выполнять следующие задачи:

* [удалять приложение, созданное организацией](#removing-an-application-authored-by-your-organization);
* [удалять мультитенантное приложение, авторизованное другой организацией](#removing-a-multi-tenant-application-authorized-by-another-organization).

## <a name="prerequisites"></a>Предварительные требования

Чтобы начать работу, вам понадобится клиент Azure AD, содержащий зарегистрированные приложения.

* Если клиента нет, [узнайте, как его получить](quickstart-create-new-tenant.md).
* Чтобы узнать, как добавлять и регистрировать приложения для своего клиента, см. раздел [Интеграция приложений с Azure Active Directory](quickstart-v1-integrate-apps-with-azure-ad.md).

## <a name="removing-an-application-authored-by-your-organization"></a>Удаление приложения, созданного организацией

Приложения, зарегистрированные организацией, отображаются в фильтре **Мои приложения** на главной странице клиента **Регистрация приложений**. Эти приложения были зарегистрированы вручную через портал Azure или программно через PowerShell или Microsoft API Graph. В частности, в вашем клиенте они представлены объектами приложения и субъекта-службы. Дополнительные сведения об этих объектах см. в статье [Объекты приложения и субъекта-службы в Azure Active Directory (Azure AD)](app-objects-and-service-principals.md).

### <a name="to-remove-a-single-tenant-application-from-your-directory"></a>Порядок удаления однотенантного приложения из каталога

1. Войдите на [портале Azure](https://portal.azure.com).
1. Если учетная запись предоставляет доступ нескольким клиентам, выберите свою учетную запись в правом верхнем углу и нужный клиент Azure AD для этого сеанса портала.
1. В области навигации слева выберите службу **Azure Active Directory**, **Регистрация приложений**, а затем найдите и выберите приложение, которое необходимо настроить.
    Вы перейдете на главную страницу регистрации приложения, в которой откроется страница **Параметры** приложения.
1. На главной странице регистрации приложения выберите **Удалить**.
1. Выберите **Да**, чтобы подтвердить удаление приложения.

### <a name="to-remove-a-multi-tenant-application-from-its-home-directory"></a>Удаление мультитенантного приложения из домашнего каталога

1. Войдите на [портале Azure](https://portal.azure.com).
1. Если учетная запись предоставляет доступ нескольким клиентам, выберите свою учетную запись в правом верхнем углу и нужный клиент Azure AD для этого сеанса портала.
1. В области навигации слева выберите службу **Azure Active Directory**, **Регистрация приложений**, а затем найдите и выберите приложение, которое необходимо настроить.
    Вы перейдете на главную страницу регистрации приложения, в которой откроется страница **Параметры** приложения.
1. На странице **Параметры** выберите **Свойства** и переключите **Мультитенантный** в положение **Нет**, чтобы сделать приложение однотенантным, а затем выберите **Сохранить**.
    Объекты субъекта-службы приложения остаются в клиентах всех организаций, которые предоставили свое согласие.
1. На главной странице регистрации приложения выберите **Удалить**.
1. Выберите **Да**, чтобы подтвердить удаление приложения.

## <a name="removing-a-multi-tenant-application-authorized-by-another-organization"></a>Удаление мультитенантного приложения, авторизованного другой организацией

Приложения, отображаемые в категории фильтра **Все приложения** (за исключением регистраций **Мои приложения**) на главной странице клиента **Регистрация приложений**, являются мультитенантными.

С технической точки зрения эти мультитенантные приложения поступили из другого клиента и были зарегистрированы в вашем клиенте во время процесса предоставления разрешения. В частности, они представлены только основным объектом службы в клиенте без соответствующего объекта приложения. Дополнительные сведения о различиях между объектами приложения и субъекта-службы см. в статье [Объекты приложения и субъекта-службы в Azure Active Directory (Azure AD)](app-objects-and-service-principals.md).

Чтобы удалить доступ к каталогу (после предоставления согласия) для мультитенантного приложения, администратору организации необходимо удалить субъект-службу приложения. У администратора должны быть права на доступ глобального администратора и возможность удалить права доступа с помощью портала Azure или [командлетов PowerShell для Azure AD](https://go.microsoft.com/fwlink/?LinkId=294151).

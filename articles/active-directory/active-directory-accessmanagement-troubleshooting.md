# Устранение неполадок динамического членства в группах

| Симптом | Действие |
|--------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Мной было настроено правило для группы, но обновление членства в группе не произошло. | Убедитесь, что на вкладке «Настройка» каталога параметр «Разрешить делегированное управление группой» имеет значение «Да». Обратите внимание, что вы увидите этот параметр, только если вошли под учетной записью с лицензией Azure Active Directory Premium. Проверьте значения атрибутов пользователей в правиле — существуют ли пользователи, удовлетворяющие правилу? |
| Мной было настроено правило, но теперь существующие участники правила удалены. | Это ожидаемо — существующие участники группы удаляются при включении или изменении правила. В группу добавляется результирующий набор пользователей на основе оценки правила. |
| Я не вижу мгновенного изменения членства, когда добавляю или изменяю правило. Почему? | Специальная оценка членства выполняется периодически в асинхронном фоновом процессе. Количество пользователей в клиенте и размер группы, созданной в результате оценки правила, влияют на длительность процесса. Обычно клиенты с небольшим количеством пользователей увидят изменения членства в группах уже через несколько минут. В случае клиентов с большим числом пользователей заполнение может занять 30 минут или больше. |

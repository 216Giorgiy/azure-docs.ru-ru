## Скачать Azure AD Connect



Для начала работы с Azure AD Connect можно скачать последнюю версию, используя следующую ссылку: [Скачать общедоступную предварительную версию Azure AD Connect](http://connect.microsoft.com/site1164/program8612)

## Перед установкой Azure AD Connect
Прежде чем установить Azure AD Connect и обновить DirSync, вам потребуется ряд элементов.

- Подписка Azure или [пробная подписка Azure](http://azure.microsoft.com/pricing/free-trial/).
- Azure AD Premium или [пробная версия Azure AD Premium](http://aka.ms/aadptrial).
- Учетная запись глобального администратора Azure AD для клиента Azure AD, с которым необходима интеграция.
- Контроллер домена AD или рядовой сервер с Windows Server 2008 или более поздней версией.
- Учетная запись администратора предприятия для локальной службы Active Directory.
- Необязательно: тестовая учетная запись пользователя для проверки синхронизации. 

Если вы планируете использовать единый вход со службами AD FS, также потребуется следующее:

- учетные данные локального администратора на требуемых серверах федерации;
- учетные данные локального администратора на всех серверах рабочей группы (не присоединенные к домену), на которых планируется развернуть роль прокси веб-приложения;
- сервер Windows Server 2012 R2 для сервера федерации;
- сервер Windows Server 2012 R2 для прокси веб-приложения;
-  PFX-файл с одним SSL-сертификатом для  предполагаемого имени службы федерации, например fs.contoso.com;
- компьютер, на котором выполняется мастер, должен подключаться к другим компьютерам, на которых требуется установить AD FS или прокси веб-приложения через удаленное управление Windows.


## Установка Azure AD Connect

После скачивания Azure AD Connect выполните следующие действия для установки этого компонента.

1. Войдите на сервер, на который требуется установить Azure AD Connect, как администратор предприятия.
2. Перейдите к файлу AzureADConnect.msi и дважды щелкните его.
3. Выполните указания мастера с использованием параметров экспресс-установки или пользовательских параметров.
4. Необязательно: используйте тестовую учетную запись пользователя для входа в облачную службу, например Office 365, для тестирования.

## Дополнительные настройки служб синхронизации
При установке служб синхронизации вы можете пропустить дополнительный раздел конфигурации, а Azure AD Connect настроит все автоматически. Так, настраиваются экземпляр SQL Server 2012 Express, создаются соответствующие группы, а им назначаются разрешения. Если вы хотите изменить параметры по умолчанию, можно использовать следующую таблицу, чтобы изучить доступные дополнительные параметры.

Дополнительная настройка | Описание 
------------- | ------------- |
Имя SQL Server |Позволяет указать имя SQL Server и имя экземпляра. Выберите этот параметр, если у вас уже есть сервер базы данных AD, который вы хотите использовать.
Учетная запись службы |По умолчанию Azure AD Connect создает учетную запись службы для служб синхронизации. Но возникает проблема — пароль формируется автоматически и неизвестен пользователю, устанавливающему Azure AD Connect. В большинстве случаев это приемлемо, но если вам нужно настроить некоторые дополнительные параметры, например область синхронизируемых подразделений, вам необходимо создать учетную запись и выбрать пароль. |
Разрешения | По умолчанию Azure AD Connect создает четыре группы после установки служб синхронизации. Это группы "Администраторы", "Операторы", "Обзор", "Сброс пароля". Если вы хотите задать собственные группы, это можно сделать здесь.
Импорт параметров |Используйте этот параметр при импорте данных конфигурации из DirSync службы синхронизации Azure AD.|



## Экспресс-установка
"Экспресс-установка" — это параметр по умолчанию и один из наиболее распространенных сценариев. При его использовании Azure AD Connect развертывает службу синхронизации с включенной синхронизацией хэша пароля. Это возможно только для одного леса и позволяет пользователям применять свой локальный пароль для входа в облако. При использовании экспресс-установки синхронизация включается автоматически после завершения установки. Если вы используете этот параметр, вам нужно всего шесть коротких щелчков для расширения вашего локального каталога в облако.

## Выборочная установка

При выборочной установке можно выбрать несколько различных вариантов. В следующей таблице описаны страницы мастера, доступные при выборе выборочной установки.

Имя страницы | Описание
-------------------    | ------------- | 
Вход пользователя|На этой странице можно выбрать, будет ли использоваться синхронизация паролей, федерации со службами AD FS или ни один из этих параметров.
Подключиться к каталогам|На этой странице можно добавить один или несколько каталогов, которые необходимо синхронизировать.
Фильтрация синхронизации| Здесь можно определить, требуется ли синхронизировать всех пользователей и все группы или выбрать одну группу для каждого каталога и синхронизировать только их.
Локальные удостоверения|Здесь можно указать, что пользователи существуют только во всех каталогах, которые были добавлены в Connect на страницы каталогов, или существуют в нескольких каталогах. Если пользователи существуют в нескольких каталогах, необходимо выбрать атрибут, который уникально идентифицирует их в каталогах. Например, атрибут почты, ObjectSID или SAMAccountName часто используются для уникальной идентификации пользователей.
Удостоверения Azure|На этой странице указывается исходная привязка, которая будет использоваться для федерации удостоверений.
Дополнительные возможности|В таблице ниже приводится краткое описание доступных дополнительных возможностей.

<center>![Express Installation](./media/active-directory-aadconnect-get-started/of.png)</center>


Дополнительные возможности | Описание
-------------------    | ------------- | 
Гибридное развертывание Exchange |Гибридное развертывание Exchange реализует сосуществование почтовых ящиков Exchange в локальной среде и в Azure за счет синхронизации набора атрибутов из Azure AD с локальным каталогом.
Фильтрации приложений и атрибутов Azure AD|Включив фильтрацию приложений и атрибутов Azure AD, вы сможете адаптировать набор синхронизированных атрибутов к определенному набору на следующей странице мастера. При этом в мастере открываются две дополнительные страницы настроек.  
Обратная запись паролей|При включении обратной записи паролей изменения, внесенные в пароли в Azure AD, будут записаны в локальный каталог.
Обратная запись пользователей|При включении обратной записи пользователей,созданные в Azure AD пользователи будут записаны в локальный каталог. При этом в мастере открывается дополнительная страница настроек.  
Синхронизация устройств|При включении синхронизации устройств конфигурация устройств может записываться в Azure AD.
Синхронизация атрибутов расширения Active Directory|При включении синхронизации атрибутов расширения каталогов заданные атрибуты будут синхронизироваться с Azure AD. При этом в мастере открывается дополнительная страница настроек.  

Описание дополнительных параметров конфигурации, например для изменения конфигурации по умолчанию, использования редактора правил синхронизации и декларативной подготовки, см. в разделе [Управление Azure AD Connect](active-directory-aadconnect-whats-next.md)

## Вспомогательные компоненты Azure AD Connect

Ниже приведен перечень необходимых и вспомогательных компонентов, которые Azure AD Connect установит на сервере, где установлен Azure AD Connect. Этот список предназначен для базовой установки Express. Если вы решили использовать другой выпуск SQL Server на странице установки службы синхронизации, перечисленные ниже компоненты SQL Server 2012 не установлены.

- Соединитель Forefront Identity Manager Azure Active Directory
- Программы командной строки Microsoft SQL Server 2012
- Microsoft SQL Server 2012 Native Client
- Microsoft SQL Server 2012 Express LocalDB
- Модуль Azure Active Directory для Windows PowerShell
- Помощник по входу в Microsoft Online Services для ИТ-специалистов
- Распространяемый пакет Microsoft Visual C++ 2013

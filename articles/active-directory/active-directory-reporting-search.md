# Поиск отчетов Azure Active Directory
*Данная документация является частью [руководства по отчетам Azure Active Directory](active-directory-reporting-guide.md).*

Azure Active Directory (Azure AD) предоставляет администраторам каталогов возможность поиска в нескольких отчетах событий, связанных с безопасностью пользователей, их деятельностью и аудитом.

Чтобы открыть панель поиска, последовательно выберите **классический портал Azure > ваша служба Azure Active Directory > Отчеты.** Панель находится в верхней части списка отчетов.

Чтобы найти действия или события аудита для конкретного пользователя, выберите диапазон дат в полях "От" и "До", введите имя участника-пользователя или отображаемое имя пользователя и нажмите кнопку с флажком (ОК).

## Отчеты, включенные в поиск
На данный момент не все отчеты включаются в результаты поиска. Эта таблица показывает, какие отчеты включены.

| Отчет | Включено |
| --- | --- |
| Попытки входа из неизвестных источников |Нет |
| "Операции входа после нескольких неудачных попыток"; |Нет |
| "Операции входа из нескольких географических регионов". |Нет |
| Попытки входа с IP-адресов с подозрительными действиями |Нет |
| Попытки входа с возможно инфицированных устройств |Нет |
| Нестандартные действия при входе |Нет |
| Пользователи с аномальными событиями при входе |Нет |
| Пользователи с утерянными учетными данными |Нет |
| Отчет аудита |Да |
| Действие сброса пароля |Да |
| Действия регистрации сброса пароля |Да |
| Действия групп по самообслуживанию |Да |
| Использование приложений |Нет |
| Действия по подготовке учетных записей |Да |
| Состояние смены пароля |Нет |
| Ошибки подготовки учетной записи |Нет |
| Использование RMS |Нет |
| Самые активные пользователи RMS |Нет |
| Использование устройства с RMS |Нет |

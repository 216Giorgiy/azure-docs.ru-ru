# <a name="manage-your-security-info-preview"></a>Управление сведениями для защиты (предварительная версия)

[!INCLUDE [preview-notice](../../../includes/active-directory-end-user-preview-notice-security-info.md)]

Сведения для защиты можно использовать для входа с рабочей или учебной учетной записью или для сброса пароля.

Во время входа, в зависимости от параметров вашей организации, на экране может появиться флажок с подписью **Больше не спрашивать X дн**. Этот флажок позволяет вам оставаться в системе устройства в течение числа дней, установленного администратором, не проходя повторные проверки.

## <a name="change-your-info"></a>Изменение сведений
Можно обновить или добавить сведения для защиты либо изменить параметры по умолчанию. Доступные действия зависят от того, что разрешено администратором и вашей организацией.

### <a name="to-change-your-info"></a>Изменение ваших сведений

1. Войдите в рабочую или учебную учетную запись.

2. Перейдите на сайт myapps.microsoft.com, щелкните ваше имя в правом верхнем углу страницы, а затем выберите **Профиль**.

3. В области **Управление учетной записью** выберите **Изменить сведения о безопасности**.

    ![Страница профиля с выделенной ссылкой "Изменить сведения о безопасности"](media/security-info/security-info-profile.png)

4. Используйте свой метод по умолчанию, чтобы подтвердить доступ и просмотреть сведения для защиты, если администратор настроил эту процедуру для вашей организации.

5. На странице **Безопасность учетной записи** можно выполнить следующее.

    - Выберите **Добавление сведений для защиты учетной записи**, чтобы добавить дополнительные методы

    - Выберите **Изменить значение по умолчанию**, чтобы изменить метод по умолчанию.

    - Щелкните значок **карандаша** рядом с настроенным методом, чтобы изменить сведения.

    ![Снимок экрана сведений для защиты с существующей, доступной для редактирования информацией](media/security-info/security-info-edit.png)

6. После внесения изменений можно уйти со страницы, и ваши изменения будут сохранены.

Если вы не видите эти параметры или вам не удается получить доступ к странице myapps.microsoft.com, возможно, ваша организация использует настраиваемые параметры или пользовательскую страницу. Вам потребуется обратиться к администратору за помощью.

## <a name="manage-your-security-info-for-a-lost-or-potentially-compromised-device"></a>Управление сведениями для защиты для утерянного или потенциально скомпрометированного устройства

Если вы потеряли устройство или оно было скомпрометировано, необходимо будет повторить процесс проверки всех ранее доверенных устройств

### <a name="to-manage-your-security-info-for-lost-or-potentially-compromised-devices"></a>Управление сведениями для защиты для утерянного или потенциально скомпрометированного устройства

1. Войдите в рабочую или учебную учетную запись.

2. Перейдите на сайт myapps.microsoft.com, щелкните свое имя в правом верхнем углу страницы, а затем выберите **Профиль**.

3. В области **Управление учетной записью** выберите **Forget MFA on remembered devices** (Удалить сведения MFA о сохраненных устройствах).
    
    Выбор этого параметра означает, что вам придется снова пройти процесс многофакторная проверка подлинности (MFA) после входа.

    ![Экран профиля с выделенной ссылкой для удаления сведений](media/security-info/security-info-forget.png)

## <a name="common-problems-and-solutions-with-your-security-info"></a>Распространенные проблемы со сведениями для защиты и их устранение

Эта статья поможет вам устранить проблемы со сведениями для защиты, включая проблемы с двухфакторной проверкой подлинности.

|Проблема|Решение|
|-------|--------|
|У меня нет с собой телефона|Возможно, ваш телефон не всегда при вас, но вам все равно нужно войти в свою рабочую или учебную учетную запись. Чтобы устранить эту проблему, можно войти с помощью другого метода аутентификации, не требующего использования личного телефона, например, с помощью рабочего номера телефона. Чтобы добавить дополнительные методы в сведения для защиты, необходимо выполнить действия, описанные в разделе [Изменение сведений](#change-your-info).|
|Мой телефон потерян или украден|К сожалению, телефон может быть потерян или украден. В этом случае настоятельно рекомендуется известить об этом вашу организацию, чтобы специалисты отдела ИТ могли сбросить ваши пароли для приложений и удалить все сохраненные устройства из вашего списка доверенных устройств. Вы также можете удалить свои доверенные устройства, выполнив действия, описанные в разделе [Управление сведениями для защиты для утерянного или потенциально скомпрометированного устройства](#manage-your-security-info-for-a-lost-or-potentially-compromised-device).|
|У меня новый номер телефона|Существуют два способа устранения этой проблемы. Вы можете войти с помощью альтернативного метода аутентификации, не требующего номера телефона, например, с использованием электронной почты. Если это невозможно, вы можете обратиться к специалистам отдела ИТ своей организации и попросить их удалить ваши настройки. Чтобы добавить дополнительные методы в сведения для защиты, необходимо выполнить действия, описанные в разделе [Изменение сведений](#change-your-info).|
|У меня установлен неправильный метод по умолчанию|Вы можете изменить метод по умолчанию с помощью параметров безопасности. Более подробно это описывается в разделе [Изменение сведений](#change-your-info).|
|Я не получаю текстовое сообщение или звонок на свое мобильное устройство|Если ранее вы получали текстовые сообщения или звонки на свое мобильное устройство, скорее всего, сейчас проблема связана с поставщиком услуг мобильной связи, а не с учетной записью. Убедитесь, что мощность сигнала мобильной связи достаточно высока и вы можете получать текстовые сообщения и принимать телефонные звонки. Для проверки вы можете попросить друга позвонить вам или отправить текстовое сообщение.<br><br>Если вы получили текстовое сообщение и смогли принять звонок, но все равно не получили уведомление, попробуйте использовать другой метод. Чтобы добавить дополнительные методы в сведения для защиты, необходимо выполнить действия, описанные в разделе [Изменение сведений](#change-your-info). Если вы не можете добавить другой метод, можно обратиться в службу поддержки вашей организации и попросить удалить ваши настройки, чтобы вы могли настроить свои методы при следующем входе в систему.<br><br>В ситуации с частыми проблемами из-за слабого сигнала мобильной связи рекомендуется использовать на мобильной устройстве приложение Microsoft Authenticator. Приложение может создавать случайные коды безопасности для входа, для использования которых не требуется подключение к мобильной сети или Интернету. Чтобы получить дополнительные сведения о приложении Microsoft Authenticator, ознакомьтесь со статьей [Начало работы с приложением Microsoft Authenticator](https://docs.microsoft.com/azure/multi-factor-authentication/end-user/microsoft-authenticator-app-how-to).|
|Ни один из вариантов в этой таблице не помог устранить проблему|Если вы выполнили эти инструкции по устранению неполадок, но проблемы по-прежнему возникают, обратитесь в службу поддержки своей организации.|

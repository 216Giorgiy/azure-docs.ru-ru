# <a name="manage-app-passwords-using-security-info-preview"></a>Управление паролями приложений с использованием сведений для защиты (предварительная версия)

[!INCLUDE[preview-notice](../../../includes/active-directory-end-user-preview-notice-security-info.md)]

Некоторые внебраузерные приложения, например Outlook 2010, не поддерживают двухфакторную проверку подлинности. Это означает, что при использовании двухфакторной проверки подлинности приложение не будет работать. Чтобы избежать этой проблемы, помимо обычного пароля, создайте автоматически сгенерированный пароль для использования с каждым внебраузерным приложением.

При использовании паролей приложений необходимо помнить следующее:

- Пароли приложений автоматически создаются и вводятся только один раз для каждого приложения.

- Максимальное количество паролей на пользователя — 40. Если вы попытаетесь создать пароль после достижения этого предела, вам будет предложено удалить один из существующих паролей, чтобы создать новый.

- Мы рекомендуем использовать один пароль приложения для каждого устройства, а не для отдельных приложений. Например, создайте один пароль для всех приложений на ноутбуке и еще один — для всех приложений на настольном компьютере.

    >[!Note]
    >Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Это означает, что после включения двухфакторной проверки подлинности отпадает потребность в использовании паролей приложений для клиентов Office 2013. Дополнительные сведения см. в статье [Современная проверка подлинности в клиентских приложениях Office 2013 и Office 2016](https://support.office.com/article/how-modern-authentication-works-for-office-2013-and-office-2016-client-apps-e4c45989-4b1a-462e-a81b-2a13191cf517).

## <a name="create-and-delete-app-passwords-using-security-info"></a>Создание и удаление паролей приложений с помощью сведений для защиты

Если вы используете двухфакторную проверку подлинности для рабочей или учебной учетной записи и ваш администратор включил функцию использования сведений для защиты, вы можете создавать и удалять пароли приложений на портале MyApps.

Если администратор не включил функцию использования сведений для защиты, необходимо выполнить инструкции в статье [Что такое пароли приложений в службе Многофакторной идентификации Azure](multi-factor-authentication-end-user-app-passwords.md).

### <a name="to-create-app-passwords-using-the-my-apps-portal"></a>Создание паролей приложений с помощью портала MyApps

1. Войдите в рабочую или учебную учетную запись.

2. Перейдите на сайт myapps.microsoft.com, щелкните ваше имя в правом верхнем углу страницы, а затем выберите **Профиль**.

3. В области **Управление учетной записью** выберите **Изменить сведения о безопасности**.

    ![Страница профиля с выделенной ссылкой "Изменить сведения о безопасности"](media/security-info/security-info-profile.png)

4. В окне **Безопасность учетной записи** выберите **Добавление сведений для защиты учетной записи**.

    ![Снимок экрана сведений для защиты с существующей, доступной для редактирования информацией](media/security-info/security-info-edit-add-info.png)

5. В окне **Добавление сведений для защиты учетной записи** выберите **Пароль приложения**.

6. В окне **создания пароля приложений** введите имя для пароля приложения и щелкните **Далее**.

    ![Снимок экрана, где присваивается имя паролю приложения](media/security-info/security-info-name-app-password.png)

7. Выберите **Копировать**, чтобы копировать пароль в буфер обмена, а затем щелкните **Далее**.

    ![Снимок экрана с паролем приложения для копирования](media/security-info/security-info-create-app-password.png)
    
8. Убедитесь, что пароль приложения присутствует в окне **Безопасность учетной записи**.

    ![Снимок экрана безопасности учетной записи, на котором присутствует пароль приложения](media/security-info/security-info-keep-secure-app-password.png)

### <a name="to-delete-app-passwords-using-the-my-apps-portal"></a>Удаление пароля приложения с помощью портала MyApps

1. В окне **Безопасность учетной записи** выберите **X** рядом с паролем приложения, чтобы удалить его.

    ![Снимок экрана безопасности учетной записи, удаление пароля приложения](media/security-info/security-info-keep-secure-delete-app-password.png)

2. В окне **Удаление пароля приложения** щелкните **Удалить**.

    ![Снимок экрана с окном удаления пароля приложения](media/security-info/security-info-keep-secure-delete-app-password2.png)

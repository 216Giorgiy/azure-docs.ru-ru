# Управление привилегированными пользователями Azure AD: как сделать использование MFA обязательным

Мы рекомендуем использовать многофакторную проверку подлинности для всех администраторов.

## Обязательная многофакторная проверка подлинности при управлении привилегированными пользователями в Azure AD

При входе в систему с учетной записью администратора PIM вы получите оповещения, предлагающие использовать многофакторную проверку подлинности для привилегированных учетных записей. Щелкните оповещение системы безопасности на панели мониторинга управления привилегированными пользователями. Отобразится новая колонка со списком учетных записей администраторов, для которых необходимо обязательно использовать MFA. Чтобы сделать использование MFA обязательным, выберите несколько ролей и нажмите кнопку **Исправить**. Вы также можете щелкнуть многоточия рядом с отдельными ролями, а затем нажать кнопку **Исправить**.

Еще можно щелкнуть конкретную роль в разделе "Роли" на панели мониторинга, а затем включить для нее использование MFA, щелкнув **Параметры** в колонке роли и выбрав пункт **Включить** рядом с обозначением многофакторной проверки подлинности.

## Как осуществляется многофакторная проверка подлинности при управлении привилегированными пользователями в Azure AD

> [AZURE.IMPORTANT] Так как учетные записи Майкрософт (например @outlook.com, @live.com или @hotmail.com) в настоящее время не поддерживаются для многофакторной проверки подлинности Azure, их нельзя использовать в качестве учетных записей временных администраторов для ролей с высоким уровнем привилегий. Если пользователи должны продолжать управлять рабочими нагрузками с помощью учетных записей Майкрософт, пожалуйста, преобразуйте эти записи в постоянных администраторов на время.

Существует два варианта многофакторной проверки подлинности при активации роли.

Самый простой способ — использовать многофакторную проверку подлинности Azure для пользователей, активирующих привилегированную роль. Для этого сначала убедитесь, что эти пользователи лицензированы (при необходимости) и что они зарегистрированы для многофакторной проверки подлинности Azure. Дополнительные сведения о том, как это сделать, см. в разделе [Приступая к работе с многофакторной проверкой подлинности Azure в облаке](../multi-factor-authentication/multi-factor-authentication-get-started-cloud.md#assigning-an-azure-mfa-azure-ad-premium-or-enterprise-mobility-license-to-users). Рекомендуется, но не обязательно, настроить принудительную многофакторную проверку подлинности в Azure AD для этих пользователей при входе. Это вызвано тем, что многофакторная проверка подлинности будет производиться самой системой управления привилегированными пользователями Azure AD.

В другом случае, если пользователи проходят проверку подлинности локально, за многофакторную проверку подлинности может отвечать поставщик удостоверений. Например, если службы федерации AD требуют проверку подлинности на основе смарт-карты для доступа к Azure AD, можете воспользоваться инструкциями по настройке AD FS для отправки утверждений в Azure AD [Защита облачных ресурсов с помощью многофакторной проверки подлинности Azure и AD FS](../multi-factor-authentication/multi-factor-authentication-get-started-adfs-cloud.md). Когда пользователь пытается активировать роль, служба управления привилегированными пользователями Azure AD примет тот факт, что многофакторная проверка подлинности уже была выполнена для этого пользователя при получении соответствующих утверждений.

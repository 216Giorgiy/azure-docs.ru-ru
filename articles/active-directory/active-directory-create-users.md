# Создание и изменение пользователей в Azure AD

Необходимо добавить в каталог клиента учетную запись для каждого пользователя, которому будет предоставлен доступ к облачной службе Майкрософт. Также можно изменять учетные записи пользователей или удалить их, когда они больше не будут нужны. По умолчанию, пользователи не имеют прав администратора, но вы можете такие права им назначать.

## Создание пользователя

1. Щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. На странице **Пользователи** щелкните **Добавить пользователя**.
3. На странице **Сведения о пользователе** выберите раскрывающееся меню **Тип пользователя** а затем выберите один из следующих пунктов.

	- **Новый пользователь в вашей организации** — если в каталоге нужно создать учетную запись пользователя.
	- **Пользователь с существующей учетной записью Майкрософт** — если в каталог нужно добавить существующую учетную запись Майкрософт (например, учетную запись Outlook).
	- **Пользователь из другого каталога Azure AD** — если требуется добавить в каталог учетную запись пользователя из другого каталога Azure AD. (Примечание. Чтобы выбрать пользователя в другом каталоге, необходимо быть членом этого каталога.)
	- **Пользователи в компаниях-партнерах** — если нужно пригласить в каталог пользователей, которые работают в компаниях-партнерах, и авторизовать их (см. статью [Совместная работа в Azure Active Directory B2B](active-directory-b2b-what-is-azure-ad-b2b.md)).


4. В зависимости от выбранного варианта введите имя пользователя или адрес его электронной почты либо отправьте CSV-файл, содержащий адреса электронный почты, с которыми пользователи будут входить в систему.
5. На странице **Профиль пользователя** введите имя, фамилию и понятное имя пользователя и выберите роль в раскрывающемся меню Роли. Дополнительные сведения о ролях пользователей и администраторов см. в разделе [Назначение ролей администратора в Azure AD](active-directory-assign-admin-roles.md). Укажите, следует ли **Включить многофакторную проверку подлинности**.
6. На странице **Получить временный пароль** нажмите кнопку **Создать**.

Если в организации используется более одного домена, при создании учетной записи пользователя необходимо учитывать следующие проблемы.

- Можно создать учетные записи пользователей с одним и тем же именем участника-пользователя (UPN) в разных доменах, если сначала создается, например, geoffgrisso@contoso.onmicrosoft.com, а затем geoffgrisso@contoso.com.
- Нельзя создать сначала geoffgrisso@contoso.com, а затем geoffgrisso@contoso.onmicrosoft.com.

## Изменение пользователя

Изменение пользователя на классическом портале Azure

1. Щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. На странице **Пользователи** щелкните отображаемое имя пользователя, которое требуется изменить.
3. Внесите необходимые изменения и нажмите кнопку **Сохранить**.

Если пользователь, которого вы пытаетесь изменить, синхронизирован с локальной службой Active Directory, появится сообщение об ошибке, и вы не сможете изменить этого пользователя с помощью данной процедуры. Для изменения такого пользователя применяйте локальные средства управления Active Directory.

## Сброс пароля пользователя

1. Щелкните **Active Directory**, а затем выберите имя каталога своей организации.
2. На странице **Пользователи** щелкните отображаемое имя пользователя, которое требуется изменить.
3. Щелкните **Сбросить пароль** в нижней части портала управления.
4. Щелкните **сбросить** в диалоговом окне сброса.
5. Установите флажок, чтобы сбросить пароль.

## Создание внешних пользователей

В Azure AD также можно добавлять пользователей в каталог Azure AD из другого каталога Azure AD, к которому вы принадлежите, используя учетную запись Майкрософт, а также пользователей компаний-партнеров, отправляя им CSV-файлы. Чтобы создать внешнего пользователя, создайте пользователя на портале, а затем в меню **Тип пользователя** выберите **Пользователь из другого каталога Azure AD** или **Пользователи в компаниях-партнерах**.

Какой бы вариант вы ни выбрали, пользователи, добавленные из других каталогов, создаются в качестве **внешних пользователей**. Внешние пользователи могут работать совместно с пользователями, которые уже существуют в каталоге, используя одну и ту же учетную запись. Им не нужно входить в новые учетные записи и указывать учетные данные. При входе внешние пользователи проходят проверку подлинности в домашнем каталоге, и эта проверка действует для всех остальных каталогов, в которые они были добавлены.

## Управление внешними пользователями и ограничениями

При добавлении в каталог пользователя из другого каталога такой пользователь будет для вашего каталога внешним. Изначально отображаемое имя и имя пользователя копируются из домашнего каталога пользователя в данные внешнего пользователя в вашем каталоге. С данного момента эти и другие свойства учетной записи внешнего пользователя никак не зависят от исходного каталога: если изменить данные пользователя в домашнем каталоге, например имя, должность и т. д., эти изменения не будут отражаться на учетной записи внешнего пользователя в другом каталоге.

Единственная связь между двумя учетными записями состоит в том, что пользователь всегда проходит проверку подлинности в домашнем каталоге или с помощью учетной записи Майкрософт. Поэтому для внешнего пользователя не отображается параметр сброса пароля или включения многофакторной проверки подлинности: при входе пользователя проверяется только политика проверки подлинности домашнего каталога или учетная запись Майкрософт.

> [AZURE.NOTE]
Вы все еще можете отключить внешнего пользователя в каталоге, и это заблокирует доступ к вашему каталогу.

Если пользователь будет удален в домашнем каталоге или удалит учетную запись Майкрософт, внешний пользователь в каталоге сохранится. Но пользователь не сможет получить доступ к ресурсам в вашем каталоге, так как не сможет пройти проверку подлинности в домашнем каталоге или по учетной записи Майкрософт.

Ниже приведены службы, которые сейчас поддерживают доступ для внешних пользователей Azure AD.

- **Классический портал Azure**. Позволяет внешнему пользователю, который является администратором нескольких каталогов, управлять каждым из этих каталогов.
- **SharePoint Online**. Позволяет внешнему пользователю обращаться к авторизованным ресурсам SharePoint Online, если включен внешний общий доступ.
- **Dynamics CRM**. Позволяет внешнему пользователю обращаться к авторизованным ресурсам Dynamics CRM, если у него есть лицензия PowerShell.

Вот некоторые известные ограничения для внешних пользователей Azure AD:

- Внешние пользователи, которые являются администраторами, не могут добавлять пользователей компаний-партнеров в другие каталоги (совместная работа B2B) за пределами своего домашнего каталога.
- Внешние пользователи не могут предоставлять согласие для мультитенантных приложений в каталогах, не принадлежащих к домашнему.
- Visual Studio Online на данный момент не поддерживает доступ для внешних пользователей*.
- PowerBI не поддерживает доступ для внешних пользователей;
- Портал Office не поддерживает лицензии для внешних пользователей.

* Visual Studio Online разрешает доступ внешним пользователям, которые проходят проверку подлинности с помощью учетных записей Майкрософт, но запрещает доступ внешним пользователям, прошедшим проверку подлинности с помощью рабочих или учебных учетных записей.

## Управление гостевыми пользователями и ограничения

**Гость** — это учетная запись пользователя в каталоге, атрибут UserType которой установлен в значение "Guest". У обычных пользователей этот атрибут имеет значение "Member", что означает, что они являются членами вашего каталога. Гости — это пользователи, которых пригласили в каталог, чтобы они использовали определенные ресурсы, такие как документы SharePoint Online, приложения или другие ресурсы Azure.

Гости обладают ограниченным набором прав в каталоге. Эти права ограничивают возможность гостей обнаруживать информацию о других пользователях в каталоге, но позволяют взаимодействовать с пользователями и группами, связанными с ресурсами, над которыми они работают. Гости могут:

- просматривать других пользователей и группы, связанные с подпиской Azure, которой они назначены;
- просматривать членов групп, к которым они принадлежат;
- искать других пользователей в каталоге, если им известны их адреса электронной почты;
- просматривать только ограниченный набор атрибутов пользователей, которых они нашли: только отображаемое имя, адрес электронной почты, имя участника-пользователя и эскиз фотографии;
- получать список проверенных доменов клиента;
- предоставлять согласие для приложений, наделяя их такими же правами доступа, как у членов каталога.

## Настройка политики доступа пользователей

На вкладке **Configure** (Настройка) каталога содержатся параметры для управления доступом внешних пользователей. Эти параметры может изменить только глобальный администратор каталога в пользовательском интерфейсе (Windows PowerShell или API для этого не подходят) на классическом портале Azure. Чтобы открыть вкладку **Конфигурация** на классическом портале Azure, щелкните **Active Directory**, а затем щелкните имя каталога.

![Вкладка "Настройка" в Azure Active Directory][1]

Теперь можно изменить параметры контроля доступа для внешних пользователей.

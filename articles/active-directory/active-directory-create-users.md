# Добавление и изменение пользователей в Azure Active Directory

Необходимо добавить в каталог клиента учетную запись каждого пользователя, которому будет предоставлен доступ к облачной службе Майкрософт. По умолчанию пользователи не имеют прав администратора, но вы можете назначать им роли в любое время.

## Добавление пользователей

1. Войдите на [классический портал Azure](https://manage.windowsazure.com) как глобальный администратор этого каталога.
2. Щелкните **Active Directory**, а затем выберите имя каталога своей организации.
3. Откройте вкладку **Пользователи**, а затем в области управления выберите **Добавить пользователя**.
4. На странице **Тип учетной записи пользователя** в раскрывающемся меню **Тип пользователя** выберите один из следующих пунктов.

	- **Новый пользователь в вашей организации** — если в каталоге нужно создать новую учетную запись пользователя.
	- **Пользователь с существующей учетной записью Майкрософт** — если в каталог нужно добавить существующую учетную запись Майкрософт (например, учетную запись Outlook).
	- **Пользователь в другом каталоге Microsoft Azure AD** — если требуется добавить в каталог учетную запись пользователя из другого каталога Azure AD (необходимо быть членом другого каталога, чтобы выбрать пользователя в нем).
	- **Пользователи в компаниях-партнерах** — если нужно пригласить в каталог пользователей, которые работают в компаниях-партнерах, и авторизовать их (см. статью [Предварительная версия службы совместной работы Azure Active Directory (Azure AD) B2B: простая и надежная облачная интеграция партнеров](active-directory-b2b-what-is-azure-ad-b2b.md)).


5. В зависимости от выбранного типа пользователя введите имя пользователя или адрес его электронной почты либо отправьте CSV-файл, содержащий адреса электронный почты, с помощью которых пользователи будут входить в систему.
6. На странице **Профиль** пользователя введите имя, фамилию и понятное имя пользователя, а затем выберите роль в списке **Роли**. Дополнительные сведения о ролях пользователей и администраторов см. в разделе [Назначение ролей администратора в Azure AD](active-directory-assign-admin-roles.md). Укажите, следует ли **Включить многофакторную проверку подлинности**.
7. На странице **Получение временного пароля** нажмите кнопку **Создать**.

> [AZURE.IMPORTANT] Если в организации используется более одного домена, при добавлении учетной записи пользователя необходимо учитывать следующие моменты:
>
> - можно добавить учетные записи пользователей с одним и тем же именем участника-пользователя (UPN) в разных доменах, если сначала добавляется, например, geoffgrisso@contoso.onmicrosoft.com, а затем geoffgrisso@contoso.com;
> - не следует добавлять geoffgrisso@contoso.com перед добавлением geoffgrisso@contoso.onmicrosoft.com. Порядок добавления очень важен; если вы допустите ошибку сейчас, потом ее может быть сложно исправить.

## Изменение сведений о пользователе

Вы можете изменить любые атрибуты пользователя, кроме идентификатора объекта пользователя.

1. Откройте каталог своей организации.
2. Перейдите на вкладку **Пользователи**, а затем выберите отображаемое имя пользователя, которое будет изменено.
3. Внесите необходимые изменения и нажмите кнопку **Сохранить**.

Если пользователь, имя которого вы пытаетесь изменить, синхронизирован с локальной службой Active Directory, появится сообщение об ошибке. Это значит, что вы не сможете изменить сведения о таком пользователе с помощью описанной процедуры. Для изменения сведений о таком пользователе применяйте локальные средства управления Active Directory.

## Сброс пароля пользователя

1. Откройте каталог своей организации.
2. Перейдите на вкладку **Пользователи**, а затем выберите отображаемое имя пользователя, которое будет изменено.
3. В области управления выберите **Сброс пароля**.
4. Щелкните **сбросить** в диалоговом окне сброса.
5. Чтобы завершить сброс пароля, установите флажок.

## Добавление внешних пользователей

В Azure AD также можно добавлять пользователей в каталог Azure AD из другого каталога Azure AD, к которому вы принадлежите, используя учетную запись Майкрософт, а также пользователей компаний-партнеров, отправляя им CSV-файлы. Чтобы добавить внешнего пользователя, создайте пользователя на портале, а затем в меню **Тип пользователя** выберите **Пользователь в другом каталоге Microsoft Azure AD** или **Пользователи в компаниях-партнерах**.

Пользователи, относящиеся к любому из этих типов, добавляются из других каталогов в качестве **внешних пользователей**. Внешние пользователи могут работать совместно с пользователями, которые уже существуют в каталоге, используя одну и ту же учетную запись. Им не нужно добавлять новые учетные записи и учетные данные. При входе внешние пользователи проходят проверку подлинности в домашнем каталоге, и эта проверка действует для всех остальных каталогов, в которые они были добавлены.

## Управление внешними пользователями и ограничениями

При добавлении в каталог пользователя из другого каталога такой пользователь будет для вашего каталога внешним. Изначально отображаемое имя и имя пользователя копируются из домашнего каталога пользователя в данные внешнего пользователя в вашем каталоге. С данного момента эти и другие свойства учетной записи внешнего пользователя никак не зависят от исходного каталога: если изменить данные пользователя в домашнем каталоге, например имя, должность и т. д., эти изменения не будут отражаться на учетной записи внешнего пользователя в другом каталоге.

Единственная связь между двумя учетными записями состоит в том, что пользователь всегда проходит проверку подлинности в домашнем каталоге или с помощью учетной записи Майкрософт. Поэтому для внешнего пользователя не отображается параметр сброса пароля или включения многофакторной проверки подлинности: при входе пользователя проверяется только политика проверки подлинности домашнего каталога или учетная запись Майкрософт.

> [AZURE.NOTE]
Вы все еще можете отключить внешнего пользователя в каталоге, и это заблокирует доступ к вашему каталогу.

Если пользователь будет удален в домашнем каталоге или удалит учетную запись Майкрософт, внешний пользователь в каталоге сохранится. Но он не сможет получить доступ к ресурсам в вашем каталоге, так как не сможет пройти проверку подлинности в корневом каталоге или учетной записи Майкрософт.

Ниже приведены службы, которые сейчас поддерживают доступ для внешних пользователей Azure AD:

- **Классический портал Azure** позволяет внешнему пользователю, который является администратором нескольких каталогов, управлять каждым из этих каталогов.
- **SharePoint Online** позволяет внешнему пользователю обращаться к авторизованным ресурсам SharePoint Online, если включен внешний общий доступ.
- **Dynamics CRM** позволяет внешнему пользователю обращаться к авторизованным ресурсам Dynamics CRM, если у него есть лицензия PowerShell.

Вот некоторые известные ограничения для внешних пользователей Azure AD:

- Внешние пользователи, которые являются администраторами, не могут добавлять пользователей компаний-партнеров в другие каталоги (совместная работа B2B) за пределами своего домашнего каталога.
- Внешние пользователи не могут предоставлять согласие для мультитенантных приложений в каталогах, не принадлежащих к домашнему.
- Visual Studio Online на данный момент не поддерживает доступ для внешних пользователей*.
- PowerBI не поддерживает доступ для внешних пользователей;
- Портал Office не поддерживает лицензии для внешних пользователей.

* Visual Studio Online разрешает доступ внешним пользователям, которые проходят проверку подлинности с помощью учетных записей Майкрософт, но запрещает доступ внешним пользователям, прошедшим проверку подлинности с помощью рабочих или учебных учетных записей.

## Управление гостевыми пользователями и ограничения

Гостевые учетные записи — это пользователи, которых пригласили в каталог для получения доступа к определенным ресурсам, включая документы SharePoint Online, приложения и другие ресурсы Azure. Гостевая учетная запись — это учетная запись, для основного атрибута UserType которой установлено значение Guest. У обычных пользователей (то есть членов вашего каталога) этот атрибут имеет значение Member.

Гости обладают ограниченным набором прав в каталоге. Эти права ограничивают возможность гостей обнаруживать информацию о других пользователях в каталоге, но позволяют взаимодействовать с пользователями и группами, связанными с ресурсами, над которыми они работают. Гости могут:

- просматривать других пользователей и группы, связанные с подпиской Azure, которой они назначены;
- просматривать участников группы, в которую они входят;
- искать других пользователей в каталоге, если им известны их адреса электронной почты;
- просматривать только ограниченный набор атрибутов пользователей, которых они нашли: только отображаемое имя, адрес электронной почты, имя участника-пользователя и эскиз фотографии;
- получать список проверенных доменов каталога клиента;
- предоставлять согласие для приложений, наделяя их такими же правами доступа, как у членов каталога.

## Настройка политик доступа пользователей

На вкладке **Configure** (Настройка) каталога содержатся параметры для управления доступом внешних пользователей. Эти параметры может изменить только глобальный администратор каталога на классическом портале Azure (это нельзя сделать с помощью Windows PowerShell или API Windows).

Чтобы открыть вкладку **Конфигурация** на классическом портале Azure, щелкните **Active Directory**, а затем щелкните имя каталога.

![Вкладка "Настройка" в Azure Active Directory][1]

Теперь можно изменить параметры контроля доступа для внешних пользователей.

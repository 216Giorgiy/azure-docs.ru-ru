# <a name="azure-active-directory-report-retention-policies"></a>Политики хранения отчетов Azure Active Directory
*Данная документация является частью [руководства по отчетам Azure Active Directory](active-directory-reporting-guide.md).*



В этой статье содержится обзор возможностей хранения данных для разных отчетов об активности в Azure Active Directory. 

Для клиентов уровней "Премиум" и "Премиум 2" сбор данных о действиях для отчетов об активности начинается, как только они получают лицензию уровня "Премиум". Для клиентов, использующих бесплатные версии, сбор данных о действиях начинается сразу после их входа на портал или после того, как они первый раз воспользуются интерфейсами API отчетов. 

Если вы уже просматривали отчеты на классическом портале Azure, то ваши данные сразу же отобразятся на (новом) портале Azure. Если вы не включили функцию отчетов на старом портале и выполняете вход в Azure Active Directory на портале Azure, то может потребоваться подождать до 2 часов, чтобы данные отчетов об активности отобразились. Сбор данных начинается сразу после выполнения входа в колонке Azure Active Directory.

Процесс сбора сигналов системы безопасности начинается после того, как вы соглашаетесь использовать центр защиты идентификации. 



**Отчеты о действиях**    

| Отчет | Azure AD уровня "Бесплатный" | Azure AD уровня "Премиум 1" | Azure AD уровня "Премиум 2" |
| :--    | :--           | :--                | :--                |
| Аудит каталога | 7 дней | 30 дней | 30 дней |
| Действия при входе |    7 дней | 30 дней | 30 дней |

**Сигналы системы безопасности**

| Отчет | Azure AD уровня "Бесплатный" | Azure AD уровня "Премиум 1" | Azure AD уровня "Премиум 2" |
| :--    | :--           | :--                | :--                |
| Вход, представляющий риск | 7 дней | 30 дней | 90 дней |

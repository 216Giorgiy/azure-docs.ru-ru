# Добавление имени личного домена для упрощения входа в Azure Active Directory

Чтобы улучшить и упростить вход в систему и другие действия пользователей в Azure Active Directory (Azure AD), можно использовать одно или несколько имен личных доменов. Например, если организации принадлежит доменное имя contoso.com, то пользователи могут входить в систему с помощью знакомых имен пользователей, таких как joe@contoso.com. Чтобы добавить имя личного домена, требуется учетная запись глобального администратора каталога Azure AD.

Сразу после получения каталога клиента пользователи используют для входа в Azure Active Directory такие имена, как alice@contoso.onmicrosoft.com. В этом примере contoso.onmicrosoft.com — встроенное доменное имя, которое можно использовать до проверки имени личного домена. Затем вам необходимо добавить имя личного домена, принадлежащего вашей организации, например contoso.com. После этого вы сможете присваивать знакомые имена пользователей, такие как alice@contoso.com.

Дополнительные сведения об использовании доменных имен в Azure AD см. в статье [Domain Management Concepts in Azure AD](active-directory-add-domain-concepts.md) (Общие сведения об управлении доменами в Azure AD). Большинство администраторов выполняют задачи по управлению доменным именем в Azure AD на классическом портале Azure. Однако для этого также можно использовать [PowerShell](https://msdn.microsoft.com/library/azure/e1ef403f-3347-4409-8f46-d72dafa116e0#BKMK_ManageDomains) или [API Graph предварительной версии](https://msdn.microsoft.com/Library/Azure/Ad/Graph/api/domains-operations).

## Добавление имени личного домена в каталог

Чтобы добавить имя личного домена в каталог, сделайте следующее:

1. Войдите на [классический портал Azure](https://manage.windowsazure.com/) с правами глобального администратора каталога Azure AD.

2. В области навигации слева выберите **Active Directory**.

3. Откройте свой каталог.

4. Перейдите на вкладку **Домены**.

5. На панели команд щелкните **Добавить**.

6. Введите имя личного домена, например contoso.com. Обязательно укажите COM, NET или другое расширение верхнего уровня.

7. Если вы планируете настроить этот домен для [федеративного входа](https://channel9.msdn.com/Series/Azure-Active-Directory-Videos-Demos/Configuring-AD-FS-for-user-sign-in-with-Azure-AD-Connect) в локальную службу Active Directory, установите соответствующий флажок.

8. Выберите **Добавить**.

Прежде чем вы сможете назначить имена пользователей, включающие имя личного домена, Azure AD необходимо проверить, принадлежит ли это доменное имя вашей организации. Для этого вам нужно обновить DNS-записи в соответствующем регистраторе доменных имен.

## Получение DNS-записей, необходимых для проверки доменного имени в Azure AD

Если при добавлении домена вы выбрали параметр настройки домена для федерации, отобразятся инструкции по скачиванию инструмента Azure AD Connect. После скачивания запустите его, чтобы [получить DNS-записи, которые необходимо добавить в регистратор доменных имен](active-directory-aadconnect-get-started-custom.md#verify-the-azure-ad-domain-selected-for-federation).

Если вы не выбрали параметр для федеративного входа с помощью Windows Server AD, DNS-записи отобразятся на второй странице мастера **добавления домена**.

## Добавление DNS-записей в файл зоны DNS соответствующего домена

Чтобы добавить требуемые Azure AD DNS-записи, сделайте следующее:

1.  Войдите в соответствующий регистратор доменных имен. Если у вас недостаточно разрешений для обновления файла зоны DNS для домена, передайте DNS-записи пользователю или группе с такими разрешениями в рамках организации и попросите их обновить эти записи.

2.  Обновите файл зоны DNS соответствующего домена путем добавления DNS-записей, предоставленных службой Azure AD. С их помощью Azure AD сможет проверить, кому принадлежит право владения доменом. Это не приведет к изменению какого-либо поведения, например маршрутизации почты или веб-хостинга.

[Инструкции по добавлению DNS-записей в популярные регистраторы DNS](https://support.office.com/article/Create-DNS-records-for-Office-365-when-you-manage-your-DNS-records-b0f3fdca-8a80-4e8e-9ef3-61e8a2a9ab23/)

## Проверка домена в Azure AD

Проверить домен в Azure AD можно сразу после добавления DNS-записей.

Если вы выбрали параметр включения имени личного домена в федерацию, инструмент Azure AD Connect выполнит проверку автоматически. Запустите этот инструмент после выполнения необходимых условий. В противном случае для проверки домена следует использовать классический портал Azure. Если мастер **добавления домена** все еще открыт, нажмите кнопку "Проверить" на третьей странице мастера. Распространение DNS-записей может занять несколько минут.

Если мастер **добавления домена** уже закрыт, можно проверить домен на [классическом портале Azure](https://manage.windowsazure.com/), сделав следующее:

1.  Войдите в каталог Azure AD с правами глобального администратора.

2.  В области навигации слева выберите **Active Directory**.

3.  Откройте свой каталог.

4.  Перейдите на вкладку **Домены**.

5.  В списке доменов выберите домен, который следует проверить.

6.  На панели команд щелкните **Проверить**.

7.  В диалоговом окне нажмите кнопку **Проверить**.

Теперь можно назначать имена пользователей, включающие имя личного домена.

## Добавление имен других личных доменов

Если ваша организация использует несколько личных доменов, например contoso.com и contosobank.com, имя каждого из них можно добавить в каталог Azure AD (не более 900). Для добавления каждого последующего доменного имени нужно выполнить шаги, перечисленные выше.

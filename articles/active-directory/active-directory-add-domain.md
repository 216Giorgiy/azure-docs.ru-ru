# Добавление имени личного домена в Azure Active Directory

В вашей организации для работы используется одно или несколько доменных имен, при этом пользователи выполняют вход в корпоративную сеть, используя имя корпоративного домена. При использовании Azure Active Directory (Azure AD) вы также можете добавить имя корпоративного домена. После этого вы сможете назначать в каталоге знакомые имена пользователей, например alice@contoso.com. Процесс прост:

- Добавьте доменное имя на классическом портале Azure в мастере **добавления домена**.

- Получите DNS-запись на классическом портале Azure AD или в средстве Azure AD Connect.

- Добавьте DNS-запись для доменного имени в файл зоны DNS на сайте для регистратора DNS.

- Проверьте доменное имя на классическом портале Azure AD или в средстве Azure AD Connect.


Пока вы не проверите имя личного домена, ваши пользователи будут входить в каталог с помощью таких имен пользователя, как alice@contoso.onmicrosoft.com, которые включают первоначальное доменное имя каталога. Если необходимо использовать несколько имен личных доменов, например contoso.com и contosobank.com, вы можете добавить не более 900. Выполните действия, описанные в этой статье, для добавления каждого доменного имени.

## Добавление имени личного домена в каталог

1. Войдите на [классический портал Azure](https://manage.windowsazure.com/) с правами глобального администратора каталога Azure AD.

2. В **Active Directory** откройте каталог и выберите вкладку **Домены**.

3. На панели команд выберите **Добавить**, а затем введите имя личного домена, например contoso.com. Обязательно укажите COM, NET или другое расширение верхнего уровня.

4. Если вы планируете настроить этот домен для [федеративного входа](https://channel9.msdn.com/Series/Azure-Active-Directory-Videos-Demos/Configuring-AD-FS-for-user-sign-in-with-Azure-AD-Connect) в локальную службу Active Directory, установите соответствующий флажок.

5. Выберите **Добавить**.

После того, как вы добавили доменное имя, Azure AD необходимо убедиться, что оно принадлежит вашей организации. Прежде чем Azure AD выполнит эту проверку, в файл зоны DNS необходимо добавить DNS-запись для доменного имени. Эту задачу можно выполнить на веб-сайте регистратора доменных имен, где вы регистрировали доменное имя.

## Получение DNS-записей для доменного имени

DNS-записи находятся на второй странице мастера **добавления домена**, если не настроена федерация с локальной службой Active Directory Windows Server.

Если вы выбрали настройку домена для федерации, вы будете перенаправлены на страницу загрузки средства Azure AD Connect. После загрузки средства запустите его, чтобы [получить DNS-записи, которые необходимо добавить в регистратор доменных имен](active-directory-aadconnect-get-started-custom.md#verify-the-azure-ad-domain-selected-for-federation). Средство Azure AD Connect также проверит доменное имя для Azure AD.

## Добавление DNS-записи в файл зоны DNS

1.  Войдите в соответствующий регистратор доменных имен. Если у вас нет достаточных разрешений для обновления DNS-записи, попросите пользователя или группу пользователей с соответствующими правами добавить DNS-запись.

2.  Обновите файл зоны DNS соответствующего домена путем добавления DNS-записи, предоставленной службой Azure AD. С помощью этой DNS-записи Azure AD сможет проверить, кому принадлежит право владения доменом. DNS-запись не влияет на поведение работающих систем, например на маршрутизацию почты или веб-хостинг. Распространение DNS-записей может длиться около часа.

[Инструкции по добавлению DNS-записи на сайтах популярных регистраторов DNS](https://support.office.com/article/Create-DNS-records-for-Office-365-when-you-manage-your-DNS-records-b0f3fdca-8a80-4e8e-9ef3-61e8a2a9ab23/)

## Проверка доменного имени в Azure AD

После добавления DNS-записи необходимо активировать проверку доменного имени в Azure AD. Это последний шаг при добавлении имени личного домена.

Если мастер **добавления домена** все еще открыт, нажмите кнопку **Проверить** на третьей странице мастера. Перед проверкой выполняется распространение DNS-записи. Этот процесс занимает около часа.

Если мастер **добавления домена** уже закрыт, вы можете проверить домен на [классическом портале Azure](https://manage.windowsazure.com/).

1.  Войдите в каталог Azure AD с правами глобального администратора.

2.  Откройте свой каталог и выберите вкладку **Домены**.

3.  Затем выберите домен, который следует проверить.

4.  На панели команд выберите **Проверить**, а затем щелкните **Проверить** в диалоговом окне.

Готово! Теперь можно [назначить имена пользователей, включающие имя личного домена](active-directory-add-domain-add-users.md). Если во время проверки доменного имени возникли какие-либо проблемы, см. раздел [Устранение неполадок](#troubleshooting) ниже.

## Устранение неполадок
Если не удается проверить имя пользовательского домена, это может быть связано с несколькими причинами. Начнем с наиболее распространенной и закончим наиболее редкой.

- Вы пытались проверить доменное имя, прежде чем была распространена DNS-запись. Подождите некоторое время и повторите попытку.

- Вы вообще не ввели DNS-запись. Проверьте DNS-запись, подождите, пока она распространится, и повторите попытку.

- Доменное имя уже проверено в другом каталоге. Найдите доменное имя, удалите его из другого каталога и повторите попытку.

- DNS-запись содержит ошибку. Исправьте ошибку и повторите попытку.

- У вас нет достаточных прав для обновления DNS-записей. Передайте DNS-запись пользователю или группе пользователей в своей организации с соответствующими правами и попросите их добавить DNS-запись.

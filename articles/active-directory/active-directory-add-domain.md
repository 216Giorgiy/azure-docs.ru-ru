# Добавление имени личного домена для упрощения входа в Azure Active Directory

Когда вы впервые помещаете каталог в Azure Active Directory (Azure AD), очень важно с самого начала проверить имя пользовательского домена, которое использует ваша организация, например "contoso.com". Выполнив эту задачу, вы сможете назначать знакомые имена пользователей, такие как alice@contoso.com. Пока вы не проверите имя пользовательского домена, ваши пользователи будут входить в каталог с помощью таких имен пользователя, как alice@contoso.onmicrosoft.com, которые включают имя исходного домена каталога.

## Добавление имени личного домена в каталог

Чтобы добавить имя личного домена в каталог, сделайте следующее:

1. Войдите на [классический портал Azure](https://manage.windowsazure.com/) с правами глобального администратора каталога Azure AD.

2. В области навигации слева выберите **Active Directory** и откройте свой каталог.

4. Перейдите на вкладку **Домены**.

5. На панели команд щелкните **Добавить**.

6. Введите имя пользовательского домена, например contoso.com. Обязательно укажите COM, NET или другое расширение верхнего уровня.

7. Если вы планируете настроить этот домен для [федеративного входа](https://channel9.msdn.com/Series/Azure-Active-Directory-Videos-Demos/Configuring-AD-FS-for-user-sign-in-with-Azure-AD-Connect) в локальную службу Active Directory, установите соответствующий флажок.

8. Выберите **Добавить**.

Прежде чем вы сможете назначить имена пользователей, включающие имя личного домена, Azure AD необходимо проверить, принадлежит ли это доменное имя вашей организации. Чтобы выполнить эту проверку, в файл зоны DNS добавьте DNS-запись для доменного имени. Эту задачу можно выполнить на веб-сайте регистратора доменных имен, на котором вы регистрировали доменное имя.

## Получение DNS-записей для доменного имени

Если вы решили настроить домен для федерации, вы будете перенаправлены на страницу загрузки средства Azure AD Connect. После загрузки средства запустите его, чтобы [получить DNS-записи, которые необходимо добавить на веб-сайте регистратора доменных имен](active-directory-aadconnect-get-started-custom.md#verify-the-azure-ad-domain-selected-for-federation).

Если вы решили не настраивать домен для федерации, вы найдете DNS-записи на второй странице мастера **добавления домена**.

## Добавление DNS-записи в файл зоны DNS

Чтобы добавить необходимую для Azure AD DNS-запись, выполните следующие действия.

1.  Войдите в соответствующий регистратор доменных имен. Если у вас нет достаточных разрешений для обновления DNS-записи, попросите пользователя или группу пользователей с соответствующими правами добавить DNS-запись.

2.  Обновите файл зоны DNS соответствующего домена путем добавления DNS-записи, предоставленной службой Azure AD. С помощью этой DNS-записи Azure AD сможет проверить, кому принадлежит право владения доменом. DNS-запись не влияет на поведение работающих систем, например на маршрутизацию почты или веб-хостинг.

[Инструкции по добавлению DNS-записи на сайтах популярных регистраторов DNS](https://support.office.com/article/Create-DNS-records-for-Office-365-when-you-manage-your-DNS-records-b0f3fdca-8a80-4e8e-9ef3-61e8a2a9ab23/)

## Проверка доменного имени в Azure AD

Вы можете проверить доменное имя в Azure AD сразу после добавления DNS-записи.

Если вы решили настроить имя пользовательского домена для федерации, средство Azure AD Connect выполнит проверку автоматически. Запустите этот инструмент после выполнения необходимых условий. В противном случае для проверки домена следует использовать классический портал Azure. Если мастер **добавления домена** все еще открыт, нажмите кнопку **Проверить** на третьей странице мастера. Распространение DNS-записей может длиться около часа.

Если мастер **добавления домена** уже закрыт, вы можете проверить домен на [классическом портале Azure](https://manage.windowsazure.com/).

1.  Войдите в каталог Azure AD с правами глобального администратора.

2.  В области навигации слева выберите **Active Directory**.

3.  Откройте свой каталог.

4.  Выберите вкладку **Домены**.

5.  В списке доменов выберите домен, который следует проверить.

6.  На панели команд щелкните **Проверить**.

7.  В диалоговом окне нажмите кнопку **Проверить**.

Теперь можно [назначить имена пользователей, включающие имя пользовательского домена](active-directory-add-domain-add-users.md).

## Добавление имен других личных доменов

Если ваша организация использует несколько имен пользовательских доменов, например contoso.com и contosobank.com, вы можете добавить их в каталог Azure AD. Вы можете добавить максимум 900 доменных имен. Для добавления каждого доменного имени выполните те же шаги.

## Устранение неполадок
Если не удается проверить имя пользовательского домена, это может быть связано с несколькими причинами. Начнем с наиболее распространенной и закончим наиболее редкой.

- Вы пытались проверить доменное имя, прежде чем была распространена DNS-запись. Подождите некоторое время и повторите попытку.

- Вы вообще не ввели DNS-запись. Проверьте DNS-запись, подождите, пока она распространится, и повторите попытку.

- Доменное имя уже проверено в другом каталоге. Найдите доменное имя, удалите его из другого каталога и повторите попытку.

- DNS-запись содержит ошибку. Исправьте ошибку и повторите попытку.

- У вас нет достаточных прав для обновления DNS-записей. Передайте DNS-запись пользователю или группе пользователей в своей организации с соответствующими правами и попросите их добавить DNS-запись.

# Управление параметрами активации ролей в компоненте управления привилегированными пользователями Azure AD

Администратор безопасности может настроить компонент управления привилегированными пользователями (PIM) Azure AD в своей организации, включая изменение среды работы для пользователей, активирующих назначения временных ролей.

## Управление параметрами активации ролей

1. Перейдите на [портал Azure](https://portal.azure.com) и выберите в панели мониторинга приложение **Управление привилегированными пользователями Azure AD**.
2. В таблице ролей выберите нужную роль.
3. Щелкните **Параметры**.
4. Задайте длительность активации по умолчанию в часах, передвинув ползунок или введя количество часов в текстовое поле. Максимальное значение — 72 часа.
5. Выберите **Включить** или **Выключить**, чтобы отправлять или, наоборот, не отправлять уведомления об активации администраторам. (Включение уведомлений может помочь обнаружить несанкционированные действия администратора.)
6. Щелкните **Включить**, чтобы разрешить администраторам вводить сведения о билетах в запросы на активацию. (Эти сведения могут быть полезными при дальнейшем аудите доступа к ролям.)
7. Щелкните **Включить** или **Выключить**, чтобы требовать или не требовать многофакторную проверку подлинности для запроса на активацию.
8. Щелкните **Сохранить**.

Нельзя отключить MFA для ролей с высоким уровнем привилегий в Azure AD и Office 365, в том числе:
- Глобальный администратор  
- администратор учетных записей;  
- редактор каталогов;  
- партнер с поддержкой 1 уровня;  
- партнер с поддержкой 2 уровня;  
- Администратор выставления счетов  
- администратор безопасности;  
- администратор Exchange;  
- администратор почтовых ящиков;  
- администратор Skype для бизнеса;  
- администратор SharePoint;  
- администратор соответствия требованиям.  

Дополнительные сведения об использовании MFA при управлении привилегированными пользователями см. в статье [Как сделать использование MFA обязательным](active-directory-privileged-identity-management-how-to-require-mfa.md).

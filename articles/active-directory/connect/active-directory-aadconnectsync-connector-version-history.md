# <a name="connector-version-release-history"></a>История выпусков версий соединителей

> [!NOTE]
> Эта статья касается только FIM и MIM. Эти соединители не поддерживаются в Azure AD Connect.

В этой статье перечисляются все выпущенные соединители.

## <a name="114430"></a>1.1.443.0

Дата выпуска: март 2017 г.

### <a name="enhancements"></a>Улучшения
* Универсальный соединитель SQL.</br>
  **Сценарий.** Для соединителя SQL ссылка разрешается только на один тип объекта, а также требуется перекрестная ссылка для участников. Это хорошо известное ограничение. </br>
  **Решение.** На этапе обработки ссылок с выбранным параметром "*" все сочетания типов объектов будут возвращены в модуль синхронизации.

>[!Important]
- Будет создано множество заполнителей.
- Убедитесь, что типы объектов имеют уникальные имена.


* Универсальный соединитель LDAP.</br>
 **Сценарий.** Если в определенном разделе выбрано только несколько контейнеров, поиск по-прежнему будет выполняться по всему разделу. Этот определенный раздел будет фильтроваться по службе синхронизации, а не по агенту управления, что может привести к снижению производительности. </br>

 **Решение.** Измените код соединителя GLDAP, чтобы просматривать все контейнеры, а также искать объекты в каждом из них вместо поиска по всему разделу.


* Lotus Domino.

  **Сценарий.** Поддержка удаления электронной почты Domino для удаления пользователя во время экспорта. </br>
  **Решение.** Настраиваемая поддержка удаления электронной почты для удаления пользователя во время экспорта.

### <a name="fixed-issues"></a>Исправленные проблемы:
* Универсальный соединитель веб-служб:
 * При изменении URL-адреса службы в проектах wsconfig SAP по умолчанию с помощью средства настройки веб-службы возникает следующая ошибка: "Не удалось найти часть пути".

      ``'C:\Users\cstpopovaz\AppData\Local\Temp\2\e2c9d9b0-0d8a-4409-b059-dceeb900a2b3\b9bedcc0-88ac-454c-8c69-7d6ea1c41d17\cfg.config\cloneconfig.xml'. ``

* Универсальный соединитель LDAP:
 * Соединитель GLDAP не распознает все атрибуты в AD LDS.
 * Если атрибуты UPN не обнаруживаются в схеме каталога LDAP, работа мастера приостанавливается.
 * Если атрибут objectclass не выбран, операции импорта изменений завершаются с ошибками обнаружения, что не происходит во время полного импорта.
 * На странице конфигурации Configure Partitions and Hierarchies (Настройка разделов и иерархий) не отображаются объекты с типами, соответствующими разделу серверов Novel в универсальном соединителе.  
Агент управления LDAP. Отображаются только объекты из раздела RootDSE.


* Универсальный соединитель SQL:
 * Исправление ошибки, когда многозначный атрибут импорта изменений универсального соединителя SQL не сообщал об ошибке.
 * Если экспортировать удаленные или добавленные значения многозначного атрибута в источник данных, они не будут в нем распознаваться.  


* Lotus Notes:
 * Определенное поле "Полное имя" отображается в метавселенной правильно. Однако при экспорте в заметки значение атрибута пусто или имеет значение null.
 * Исправление ошибки дублирования для издателя сертификатов
 * Если для соединителя Lotus Domino вместе с другими объектами выбран объект без данных, во время полного импорта отобразится сообщение об ошибке обнаружения.
 * По завершении выполнения импорта изменений для соединителя Lotus Domino служба Microsoft.IdentityManagement.MA.LotusDomino.Service.exe иногда возвращает сообщение об ошибке приложения.
 * Участие в группе работает надлежащим образом и поддерживается, за исключением случаев, когда при выполнении экспорта для удаления пользователя из членства отображается успешное обновление, хотя пользователь на самом деле не удаляется из членства в Lotus Notes.
 * В графическом интерфейсе настройки агента управления Lotus добавлена возможность выбрать режим экспорта Append Item at bottom (Добавление элемента внизу), чтобы добавлять новые элементы внизу во время экспорта многозначных атрибутов.
 * Соединитель добавляет логику, необходимую для удаления файла из почтовой папки и хранилища идентификаторов.
 * Нельзя удалить участников из адресной книги Notes.
 * Значения должны быть успешно удалены из многозначного атрибута.

## <a name="111170"></a>1.1.117.0
Дата выпуска: март 2016 г.

**Новый соединитель**  
Первоначальный выпуск [универсальному соединителю SQL](active-directory-aadconnectsync-connector-genericsql.md).

**Новые функции:**

* Универсальный соединитель LDAP:
  * Добавлена поддержка импорта изменений с помощью Isode.
* Соединитель веб-служб:
  * Обновлены действия csEntryChangeResult и setImportErrorCode, чтобы устранить ошибки уровня объекта, возвращаемые в модуль синхронизации.
  * Обновлены шаблоны SAP6 и SAP6User для использования новых функций обработки ошибок уровня объекта.
* Соединитель Lotus Domino:
  * Для экспорта требуется один издатель сертификатов на адресную книгу. Теперь можно использовать один и тот же пароль для всех издателей сертификатов, чтобы облегчить управление.

**Исправленные проблемы:**

* Универсальный соединитель LDAP:
  * IBM Tivoli DS: некоторые ссылочные атрибуты не обнаруживались должным образом.
  * Open LDAP: при импорте изменений усекались пробелы в начале и конце строки.
  * Novell и NetIQ: экспорт, при котором объект перемещался между подразделениями или контейнерами и в то же время переименовывался, приводил к сбою.
* Соединитель веб-служб:
  * Если у веб-службы было несколько конечных точек для одной привязки, то соединитель не мог правильно обнаружить эти конечные точки.
* Соединитель Lotus Domino:
  * Не работал экспорт атрибута fullName в базу данных обслуживания почты.
  * При экспорте с добавлением и удалением участника группы выполнялось только экспортирование добавленных участников.
  * Если документ Notes являлся недопустимым (атрибуту isValid задано значение false), происходил сбой соединителя.

#Руководство. Интеграция Azure Active Directory с Workday
  
Цель данного руководства — продемонстрировать интеграцию Azure и Workday. Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент в Workday
  
Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для Workday
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Настройка подготовки учетных записей пользователей

![Сценарий](./media/active-directory-saas-workday-tutorial/IC782919.png "Сценарий")

##Включение интеграции приложений для Workday
  
В этом разделе показано, как включить интеграцию приложений для Salesforce.

###Чтобы включить интеграцию приложений для Workday, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-workday-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-workday-tutorial/IC700994.png "Приложения")

4.  Чтобы открыть **коллекцию приложений**, щелкните **Добавить приложение**, а затем — **Добавить приложение для использования моей организацией**.

    ![Что необходимо сделать?](./media/active-directory-saas-workday-tutorial/IC700995.png "Что необходимо сделать?")

5.  В **поле поиска** введите **Workday**.

    ![Workday](./media/active-directory-saas-workday-tutorial/IC701021.png "Workday")

6.  В области результатов выберите **Workday** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![Workday](./media/active-directory-saas-workday-tutorial/IC701022.png "Workday")

##Настройка единого входа
  
В этом разделе показано, как разрешить пользователям проходить проверку подлинности в Workday со своей учетной записью Azure AD, используя федерацию на основе протокола SAML. В рамках этой процедуры потребуется создать сертификат в кодировке Base-64. Если вы не знакомы с этой процедурой, просмотрите видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **Workday** нажмите кнопку **Настроить единый вход**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC782920.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в Workday** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC782921.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** выполните следующие действия и нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-workday-tutorial/IC782957.png "Настройка URL-адреса приложения")

    1.  В текстовом поле **URL-адрес входа** введите URL-адрес, с помощью которого пользователи входят в Workday (например, *https://impl.workday.com/\<клиент>/login-saml2.htmld*).
    2.  В текстовом поле **URL-адрес ответа Workday** введите URL-адрес ответа Workday (например, *https://impl.workday.com/\<клиент>/login-saml.htmld*).

        >[AZURE.NOTE]URL-адрес ответа должен включать поддомен (например, www, wd2, wd3, wd3-impl, wd5, wd5-impl). Можно использовать что-то вроде **http://www.myworkday.com*, но **http://myworkday.com* нельзя.
 
4.  Для скачивания сертификата на странице **Настройка единого входа в Workday** нажмите кнопку **Скачать сертификат** и сохраните файл сертификата на своем компьютере.

    ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC782922.png "Настройка единого входа")

5.  В другом окне браузера войдите на свой корпоративный сайт Workday в качестве администратора.

6.  Последовательно выберите пункты **Меню > Workbench**.

    ![Workbench](./media/active-directory-saas-workday-tutorial/IC782923.png "Workbench")

7.  Перейдите в раздел **Администрирование учетной записи**.

    ![Администрирование учетной записи](./media/active-directory-saas-workday-tutorial/IC782924.png "Администрирование учетной записи")

8.  Выберите **Изменение настройки клиента — безопасность**.

    ![Изменение параметров безопасности клиента](./media/active-directory-saas-workday-tutorial/IC782925.png "Изменение параметров безопасности клиента")

9.  В разделе **URL-адреса перенаправления** выполните следующие действия.

    ![URL-адреса перенаправления](./media/active-directory-saas-workday-tutorial/IC7829581.png "URL-адреса перенаправления")

     9\.1. Нажмите кнопку **Добавить строку**.

     9\.2. В текстовых полях **URL-адрес перенаправления входа** и **URL-адрес перенаправления мобильных устройств** введите **URL-адрес клиента Workday**, который был введен на странице **Настройка URL-адреса приложения** на портале Azure.
    
     9\.3. На странице диалогового окна **Настройка единого входа в Workday** портала Azure скопируйте значение поля **URL-адрес службы единого выхода** и вставьте его в текстовое поле **URL-адрес перенаправления выхода**.

     9\.4. В текстовом поле **Среда** введите имя среды.


       >[AZURE.NOTE]Значение атрибута Environment привязано к значению URL-адреса клиента. Если доменное имя URL-адреса клиента Workday начинается с impl (например, *https://impl.workday.com/\<клиент>/login-saml2.htmld*), то атрибуту **Environment** нужно присвоить значение Implementation. Если доменное имя начинается с чего-то другого, свяжитесь с компанией Workday для получения соответствующего значения атрибута **Environment**.

10. В разделе **Настройка SAML** сделайте следующее:

    ![Настройка SAML](./media/active-directory-saas-workday-tutorial/IC782926.png "Настройка SAML")

     10\.1. Установите флажок **Включить проверку подлинности SAML**.

     10\.2. Нажмите кнопку **Добавить строку**.

11. В разделе Поставщики удостоверений SAML" выполните следующие действия.

    ![Поставщики удостоверений SAML](./media/active-directory-saas-workday-tutorial/IC7829271.png "Поставщики удостоверений SAML")

     11\.1. В текстовом поле «Имя поставщика удостоверений» введите имя поставщика (например, *SPInitiatedSSO*).

     11\.2. На странице диалогового окна **Настройка единого входа в Workday** портала Azure скопируйте значение поля **Идентификатор поставщика удостоверений** и вставьте его в текстовое поле **Издатель**.

     11\.3. Установите флажок **Включить выход, инициируемый Workday**.

     11\.4. На странице диалогового окна **Настройка единого входа в Workday** портала Azure скопируйте значение поля **URL-адрес службы единого выхода** и вставьте его в текстовое поле **URL-адрес запроса выхода**.


     11\.3. Щелкните **Сертификат открытого ключа поставщика удостоверений**, а затем нажмите кнопку **Создать**. <br><br> ![Создание](./media/active-directory-saas-workday-tutorial/IC782928.png "Создание")

     11\.4. Щелкните **Создать открытый ключ x509**. <br><br> ![Создание](./media/active-directory-saas-workday-tutorial/IC782929.png "Создание")

     11\.5. В разделе **Просмотр открытого ключа x509** сделайте следующее: <br><br> ![Просмотреть открытый ключ x509](./media/active-directory-saas-workday-tutorial/IC782930.png "Просмотреть открытый ключ x509") <br>

      1.  В текстовом поле **Имя** введите имя сертификата (например, *PPE\_SP*).
      2.  В текстовом поле **Действителен с** введите начальную дату действия сертификата.
      3.  В текстовом поле **Действителен до** введите конечную дату действия сертификата.
		
           >[AZURE.NOTE]Чтобы получить начальную и конечную даты, дважды щелкните скачанный сертификат. Даты указаны на вкладке **Подробности**.

      4.  Создайте файл **в кодировке Base-64** из скачанного сертификата.

		>[AZURE.TIP]Дополнительные сведения можно узнать из видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

      5.  Откройте сертификат в кодировке Base-64 в Блокноте и скопируйте его содержимое.
      6.  В текстовое поле **Сертификат** вставьте содержимое буфера обмена.
      7.  Нажмите кнопку **ОК**.

12.  Сделайте следующее: <br><br> ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC7829351111.png "Настройка единого входа")

     12\.1. Установите флажок **Пара закрытых ключей x509**.

     12\.2. В текстовом поле **Идентификатор поставщика службы** введите ****http://www.workday.com**.

     12\.3. Установите флажок **Включить проверку подлинности SAML, инициированную поставщиком услуг**.

     12\.4. На странице диалогового окна **Настройка единого входа в Workday** портала Azure скопируйте значение поля **URL-адрес службы единого входа** и вставьте его в текстовое поле **URL-адрес службы единого входа IdP**.
     
     12\.5. Установите флажок **Не отклонять запрос проверки подлинности, инициированный поставщиком службы**.

     12\.6. Для параметра **Метод подписи запроса проверки подлинности** выберите значение **SHA256**. <br><br> ![Метод подписи запроса проверки подлинности](./media/active-directory-saas-workday-tutorial/IC782932.png "Метод подписи запроса проверки подлинности") <br><br>
 
     12\.7. Нажмите кнопку **ОК**. <br><br> ![ОК](./media/active-directory-saas-workday-tutorial/IC782933.png "ОК")

12. На странице **Настройка единого входа в Workday** портала Azure AD нажмите кнопку **Далее**. <br><br>

    ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC782934.png "Настройка единого входа")

13. На странице **Подтверждение единого входа** нажмите кнопку **Завершить**. <br><br>

    ![Настройка единого входа](./media/active-directory-saas-workday-tutorial/IC782935111.png "Настройка единого входа")



##Настройка подготовки учетных записей пользователей
  
Чтобы подготовить тестового пользователя в Workday, необходимо обратитесь в службу поддержки Workday. Служба поддержки создаст пользователя для вас.

##Назначение пользователей
  
Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которым нужно разрешить работу с приложением, назначив их.

###Чтобы назначить пользователей Workday, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **Workday** нажмите кнопку **Назначить пользователей**.

    ![Назначение пользователей](./media/active-directory-saas-workday-tutorial/IC782935.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем нажмите кнопку **Да**, чтобы подтвердить назначение.

    ![Да](./media/active-directory-saas-workday-tutorial/IC767830.png "Да")
  
Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).

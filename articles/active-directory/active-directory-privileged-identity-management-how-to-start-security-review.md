# Управление привилегированными пользователями Azure: запуск проверки безопасности

## Запуск проверки безопасности
В будущем вы сможете выполнять проверку безопасности, используя разные инструменты на портале Azure. В этом документе рассматривается процедура запуска проверки безопасности из интерфейса **Управление привилегированными пользователями** (PIM).

Пользователь может сменить позицию или проект, и в новой должности ему может больше не требоваться привилегированный доступ. А о некоторых пользователях вы, возможно, и не помните. Чтобы снизить риск, связанный с «устаревшими» ролями, администраторы могут проверить назначенные пользователям роли. Этот процесс называется проверкой безопасности.

## Пути для запуска проверки безопасности
> [AZURE.NOTE]Если вы еще не создали панель мониторинга PIM на портале Azure, ознакомьтесь с инструкциями в статье [Приступая к работе с управлением привилегированными пользователями Azure](active-directory-privileged-identity-management-getting-started.md).

Открыть представление проверки из панели мониторинга PIM можно следующими способами:

- Панель мониторинга > Проверки безопасности > Проверка > кнопка «Проверить»
- Панель мониторинга > Роли > кнопка «Проверить»
- Панель мониторинга > выберите роль, которую нужно проверить, в списке ролей > кнопка «Проверить»

## Запуск проверки безопасности

После того как вы нажмете кнопку **Проверить**, появятся колонки **Начало проверки роли** и **Выбор роли для проверки**, причем колонка **Выбор роли для проверки** будет активна.

### Выбор роли для проверки

1. Выберите роль из списка ролей в колонке **Выбор роли для проверки**. За один раз можно выбрать только одну роль. Вместо колонки **Выбор роли для проверки** появится колонка **Выбор проверяющих**. Есть два варианта выбора проверяющих:
  - Я. Используйте этот вариант, если хотите просмотреть, как выполняется проверка безопасности, без привлечения других администраторов.
  - Самопроверка обладателем роли. Этот вариант позволяет пользователям самостоятельно проверять назначенные им роли.
2. Выберите один из этих вариантов, чтобы приступить к проверке. Откроется колонка **Изменение значений по умолчанию**.

### Самопроверка обладателем роли

1. Введите имя проверки в поле **Имя**. Рекомендуется присваивать уникальные и понятные имена, чтобы проверки было легче отслеживать.
2. Введите дату начала проверки в поле **Дата начала**.
3. Введите дату окончания проверки в поле **Дата окончания**. При выборе даты окончания рекомендуется учитывать следующее:
  - сколько пользователей необходимо проверить;
  - как быстро пользователи смогут добавить расширение и выполнить проверку.
4. Нажмите кнопку «ОК» в колонке **Изменение значений по умолчанию**. Колонка закроется.
5. Нажмите кнопку «ОК» в колонке **Запуск проверки роли**. Колонка закроется. В главном меню портала Azure появится уведомление. Обновите панель мониторинга, нажав кнопку **Обновить**, и проверка безопасности отобразится в разделе **Проверки безопасности**.
6. Сообщите проверяемым пользователям о том, что им требуется добавить расширение и проверить свой административный доступ. См. следующий шаг.
6. [Проверьте административный доступ.](active-directory-privileged-identity-management-how-to-perform-security-review.md)

### Проверка администратором

Если вы выбрали себя в качестве проверяющего (вариант «Я»), перейдите к проверке безопасности. Дополнительные сведения о выполнении проверки в статье [Управление привилегированными пользователями Azure: выполнение проверки безопасности](active-directory-privileged-identity-management-how-to-perform-security-review.md).

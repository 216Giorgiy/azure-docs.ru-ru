# Запуск проверки безопасности в компоненте управления привилегированными пользователями Azure AD

Назначения ролей становятся "устаревшими", когда у пользователей имеются права привилегированного доступа, которые им больше не нужны. Чтобы снизить риск, связанный с "устаревшими" назначениями ролей, администраторы безопасности должны периодически проверять назначенные пользователям роли. В этом документе рассматривается процедура запуска проверки безопасности в компоненте управления привилегированными пользователями (PIM).

## Запуск проверки безопасности
> [AZURE.NOTE] Если вы еще не добавили приложение PIM в панель мониторинга на портале Azure, ознакомьтесь с инструкциями в статье [Приступая к работе с управлением привилегированными пользователями Azure](active-directory-privileged-identity-management-getting-started.md).

Есть три способа запуска проверки безопасности с главной страницы приложения PIM:

- **Проверки безопасности** > **Проверка** > кнопка **Проверить**;
- **Роли** > кнопка **Проверить**;
- выбор проверяемой роли в списке ролей > кнопка **Проверить**.

При нажатии кнопки **Проверить** отображаются колонки **Начало проверки роли** и **Выбор проверяемой роли**. Элемент **Выбор проверяемой роли** выбран по умолчанию.

### Выбор роли для проверки

1. Выберите роль из списка ролей в колонке **Выбор проверяемой роли**. За один раз можно выбрать только одну роль. Вместо колонки **Выбор роли для проверки** появится колонка **Выбор проверяющих**. Есть два варианта выбора проверяющих:
  - "Я" — выберите этот вариант, если вы хотите запустить проверки безопасности в предварительном режиме без участия других администраторов.
  - "Самопроверка членами роли" — выберите этот вариант, чтобы пользователи могли проверить собственные назначения ролей.
2. Выберите один из этих вариантов, чтобы приступить к проверке. Откроется колонка **Изменение значений по умолчанию**.

### Проверка администратором

Если вы выбрали себя в качестве проверяющего (вариант «Я»), перейдите к проверке безопасности. Дополнительные сведения о выполнении проверки в статье [Управление привилегированными пользователями Azure: выполнение проверки безопасности](active-directory-privileged-identity-management-how-to-perform-security-review.md).

### Самопроверка членами роли

Если вы выбрали самостоятельную проверку пользователями назначенных им ролей, выполните следующие действия, чтобы настроить проверку и отправить уведомления.

1. Введите имя проверки в поле **Имя**. Задайте для проверки уникальное описательное имя, по которому ее будет удобно отслеживать.
2. Введите дату начала проверки в поле **Дата начала**.
3. Введите дату окончания проверки в поле **Дата окончания**. При выборе даты окончания рекомендуется учитывать следующее:
  - сколько пользователей необходимо проверить;
  - сколько времени занимает у пользователей процесс добавления приложения PIM на портале Azure и выполнение проверки.
4. Нажмите кнопку **ОК** в колонке **Изменить параметры по умолчанию**. Колонка закроется.
5. Нажмите кнопку **ОК** в колонке **Начало проверки роли**. Колонка закроется. В главном меню портала Azure отобразится соответствующее уведомление. Обновите панель мониторинга, нажав кнопку **Обновить**: проверка безопасности появится в разделе **Проверки безопасности**.
6. Вам потребуется уведомить пользователей с этой ролью о том, что они должны добавить приложение PIM, а затем [проверить свои права административного доступа](active-directory-privileged-identity-management-how-to-perform-security-review.md).  

## Управление проверками безопасности

Отслеживать ход выполнения проверяющими их проверок можно в панели мониторинга Azure AD PIM в разделе проверок безопасности. Права доступа не изменяются в каталоге до [завершения проверки](active-directory-privileged-identity-management-how-to-complete-review.md).

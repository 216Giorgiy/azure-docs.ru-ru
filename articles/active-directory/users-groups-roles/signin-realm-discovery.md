# <a name="home-realm-discovery-during-sign-in-for-microsoft-365-services"></a>Обнаружение домашней области при входе в систему для служб Microsoft 365

Мы изменяем наших Azure Active Directory (Azure AD) войдите поведение, чтобы освободить место для новых методов проверки подлинности и повышают удобство работы. Во время входа в систему Azure AD определяет, где требуется проверить подлинность пользователя. Azure AD делает обоснованные решения, чтение параметров организации и пользователя для выбранного имени пользователя, введенное на странице входа в систему. Это является шагом в направлении без пароля фьючерс, позволяющий дополнительных учетных данных, например FIDO 2.0.

## <a name="home-realm-discovery-behavior"></a>Поведение обнаружения домашней области

Исторически сложилось так, что обнаружение домашней области была регулируется доменом, предоставляемый при входе в систему, или с помощью политики обнаружения домашней области для некоторых устаревших приложений. Например в старом поведении обнаружения пользователя Azure Active Directory может допустить свое имя пользователя, но по-прежнему будет поступать в экран коллекции учетных данных организации. Это происходит, когда пользователь правильно предоставляет имя домена организации «contoso.com». Это поведение не позволяет гранулярности для настройки возможности для отдельного пользователя.

Поведение подстановки имени пользователя Azure Active Directory во время процесса входа в систему для поддержки более широкого круга учетные данные и повысить удобство использования, теперь обновляется. Новое поведение делает обоснованные решения, считывая клиента и пользователя уровня параметры в зависимости от имени пользователя, введенное на странице входа в систему. Для этого Azure Active Directory будет проверять, существует в их указанный домен имя пользователя, введенное на странице входа в систему и перенаправляет пользователя для предоставления учетных данных. 

Дополнительным преимуществом часть этой работы является улучшенная ошибки обмена сообщениями. Вот несколько примеров, улучшенный ошибки обмена сообщениями при входе в приложение, которое поддерживает только для пользователей Azure Active Directory.

1. Неверном вводе имени пользователя или имя пользователя не был синхронизирован с Azure AD:
  
    ![имя пользователя неправильно ввел или не найден](./media/signin-realm-discovery/typo-username.png)
  
2. Неверном вводе имя домена:
  
    ![имя домена неправильно ввел или не найден](./media/signin-realm-discovery/typo-domain.png)
  
3. Пользователь пытается выполнить вход с использованием доменом известных клиента:
  
    ![вход в систему с доменом известных клиента](./media/signin-realm-discovery/consumer-domain.png)
  
4. Является неправильно ввел пароль, но точное имя пользователя:  
  
    ![неверном вводе пароля с хорошим именем пользователя](./media/signin-realm-discovery/incorrect-password.png)
  
## <a name="additional-info"></a>Дополнительная информация

Помимо использования элементов интерфейса пользователя улучшенные входа в систему это изменение включает в себя механизмы, которые могут способствовать снижению риска о нарушении перечисления крупномасштабных имени пользователя.

Это изменение изначально предназначен для управляемых доменов и начинает развертывание в мая 2019, но не запускается, развертывать для федеративных доменов к концу 2019 г. Точные даты развертывание для федеративных доменов, зависит от отзывов клиентов.

> [!IMPORTANT]
> Эта функция окажет значительное влияние на федеративных доменов, полагаясь на старый режим домена обнаружения домашней области для принудительного федерации. В некоторых организациях обучения своих сотрудников, выполнить вход с использованием имени пользователя, который не существует в Azure Active Directory, но содержит правильное имя домена, так как имена доменов направляет пользователей в настоящее время к конечной точке домена своей организации. Новое поведение входа в систему не позволяет это сделать. Пользователь будет уведомлен о необходимости исправьте имя пользователя, и они не может выполнить вход с использованием имени пользователя, который не существует в Azure Active Directory.
>
> У вас или вашей организации и рекомендации, которые зависят от прежнего поведения, очень важно для администраторов организации для обновления документации по входа в систему и проверки подлинности сотрудников и для обучения сотрудников, чтобы использовать свое имя пользователя Azure Active Directory для входа.

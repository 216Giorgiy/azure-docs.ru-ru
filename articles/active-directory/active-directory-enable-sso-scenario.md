# Я могу положиться на Azure AD, чтобы включить единый вход для всех своих приложений
Эта статья предназначена для людей, отвечающих за принятие ИТ-решений, которые хотят узнать о той пользе, которую Azure Active Directory для позволяет извлекать из используемых в организации приложений.

## Выполнение приложений в облачной среде  

В современном мире, ориентированном на облачные технологии, можно найти тысячи приложений для выполнения любой задачи. ИТ-отделы часто даже не имеют представления обо всех тех приложениях SaaS, которые используются в их организациях. Во многих случаях о приложении известно, однако обеспечение любого уровня безопасности и управления требует слишком много времени и ресурсов. Многие приложения не поддерживают интеграцию с корпоративными системами удостоверений, в других случаях от одноразовых интеграций отказываются из-за затрат и сложности.

В результате

- многие организации часто сталкиваются с проблемами несанкционированного доступа к корпоративным данным, возможной утечки данных и другими угрозами безопасности, характерными для неуправляемых приложений. Так как они не знают, сколько и каких именно приложений используется, даже планирование работы с этими рисками кажется сложной задачей.
- Администраторам приходится по отдельности управлять всеми поставщиками услуг IAM, предоставляемыми управляемыми приложениями в вашей среде. Сюда относится создание и удаление пользователей и групп, а также предоставление и запрет доступа к этим приложениям.
- Пользователи вынуждены запоминать различные учетные данные для доступа к приложениям, с которыми они работают. Забытые пароли значительно повышают текущие расходы во многих организациях. Все эти проблемы отрицательно влияют на производительность труда пользователей и ваши текущие расходы.  
 
## Чем в такой ситуации может помочь Azure Active Directory?
Azure Active Directory помогает решить эти проблемы, позволяя легко расширить существующую инфраструктуру удостоверений в облако и добиться, таким образом, следующих результатов:

- обеспечение возможности доступа мобильных пользователей к любому приложению; 
- распространение возможностей управления доступом на любое облачное приложение; 
- обнаружение приложений, с которыми работают ваши пользователи;


### распространение действия удостоверения на любое приложение. 

Azure Active Directory предоставляет возможность единого входа в облачные приложения, размещенные в Azure и других веб-службах корпорации Майкрософт, таких как Office 365, CRM Online и Intune.

Кроме того, эта система может выступать в качестве посредника по удостоверениям для единого входа в приложения в другом общедоступном облаке, например Google, Amazon, IBM и других. <br> Если приложение является широко известным или распространенным, мы предварительно интегрируем его в коллекцию приложений Azure Active Directory. В настоящее время в коллекции приложений доступно около 2500 предварительно интегрированных приложений, и их число постоянно увеличивается. <br> Настройка единого входа для любого из предварительно интегрированных приложений в коллекции приложений осуществляется лишь несколькими щелчками мышью. Что делать, если мое приложение пока отсутствует в коллекции приложений? Вы можете интегрировать любое общедоступное приложение в Azure Active Directory с помощью мастера, а затем включить для него единый вход. Дополнительные сведения см. в разделах [Развертывание единого входа с помощью Azure Active Directory для новых приложений SaaS](active-directory-single-sign-on-newly-acquired-saas-apps.md) и [Интеграция единого входа Azure Active Directory с существующими приложениями](active-directory-sso-integrate-existing-apps.md)<br> А как быть с приложениями, разработанными вашей организацией? Мы предоставляем соответствующую документацию, которая позволяет разработчикам легко интегрировать приложения, разработанные вашей организацией, в Azure Active Directory. Если вы хотите узнать об этом подробнее, см. раздел [Azure AD и приложения: руководство для разработчиков](active-directory-applications-guiding-developers-for-lob-applications.md).

Благодаря поддержке предварительно интегрированных приложений, общедоступных приложений, которые отсутствуют в коллекции приложений, и приложений, разработанных в вашей организации, Azure Active Directory позволяет реализовать единый вход для всех облачных приложений.

Ценность Azure Active Directory не ограничивается облачными приложениями. С помощью Azure Active Directory вы также можете расширить сферу применения удостоверений Azure, предоставив безопасный удаленный доступ к локальным приложениям посредством единого входа. Это означает наличие удаленного доступа, который не требует таких технологий, как VPN, или других традиционных инфраструктур удаленного доступа, для доступа к локальным веб-приложениям. Azure Active Directory предоставляет эту функциональность посредством возможностей прокси приложения.

Традиционная модель единого входа основана на сопоставлении двух отдельных учетных записей в двух репозиториях удостоверений. С помощью Azure Active Directory можно даже сопоставить отдельную учетную запись с общей корпоративной учетной записью, такой как корпоративная учетная запись Twitter. Благодаря реализации единого входа для корпоративных общих учетных записей отсутствует необходимость в совместном использовании учетных данных учетных записей с пользователями, что значительно повышает защиту этих учетных записей. Если вы хотите узнать об этом подробнее, см. раздел [Совместное использование учетных записей в Azure AD](active-directory-sharing-accounts.md).

Благодаря расширению сферы действия удостоверений один пароль предоставляет доступ к тысячам приложений.



### Распространение возможностей управления доступом на любое облачное приложение

Управление доступом к облачным приложениям требует значительных затрат, если данную процедуру необходимо осуществлять вручную для отдельных приложений. Благодаря Azure Active Directory вы можете управлять доступом к облачным приложениям на основе групп из одного центрального расположения — портала Azure. Вы можете предоставлять доступ как отдельным пользователям, так и целым группам. Дополнительные сведения см. в разделе [Управление доступом к приложениям](active-directory-managing-access-to-apps.md).

Некоторые приложения, такие как Salesforce, Box, Google Apps и Concur, предоставляют интерфейсы автоматизации для создания и удаления (или отключения) учетных записей. Если поставщик предоставляет такой интерфейс, Azure Active Directory использует его. Другими словами, Azure Active Directory обеспечивает поддержку автоматической подготовки пользователей для приложений, предоставляющих соответствующий интерфейс автоматизации.

Автоматическая подготовка пользователей предоставляет следующие возможности:

- Когда пользователю предоставляется доступ к определенному приложению, Azure Active Directory автоматически подготавливает для этого приложения учетную запись единого входа.
- При удалении учетной записи пользователя в Azure Active Directory учетная запись в приложении отключается или удаляется. Настройка поставщиков автоматической подготовки пользователей предоставляет следующие преимущества:
- 	она сокращает текущие расходы благодаря автоматизации задач администрирования, которые в противном случае пришлось бы выполнять вашим ИТ-специалистам;
- она повышает уровень безопасности среды, так как снижает вероятность наличия ненужного доступа к приложениям.

Дополнительные сведения об автоматической подготовке пользователей см. в разделе [Автоматическая подготовка пользователей и ее отзыв для приложений SaaS в Azure Active Directory](active-directory-saas-app-provisioning.md).


### Обнаружение приложений, с которыми работают ваши пользователи

Благодаря единому входу и подготовке учетных записей Azure Active Directory предоставляет эффективные механизмы для улучшения условий работы пользователей и администраторов с теми приложениями в вашей среде, о которых вам известно. Однако на современных предприятиях ИТ-специалисты часто не знают обо всех используемых облачных приложениях. Azure Active Directory позволяет вам находить такие приложения с помощью компонента Cloud App Discovery. С помощью Cloud App Discovery вы сможете:

- Обнаруживать используемые приложения и оценивать их использование по числу пользователей, объему трафика или числу веб-запросов к приложению.
- Идентифицировать пользователей, использующих приложение.
- Экспортировать данные для дополнительного автономного анализа.
- Определять приоритетные приложения, которые необходимо взять под контроль ИТ-подразделения, и легко интегрировать приложения, что даст возможность использовать единый вход и управлять пользователями.

Дополнительные сведения о Cloud App Discovery см. в разделе [Как обнаруживать несанкционированные облачные приложения, которые используются в моей организации](active-directory-cloudappdiscovery-whatis.md).

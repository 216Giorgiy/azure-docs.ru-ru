# <a name="azure-active-directory-b2b-collaboration-current-limitations"></a>Текущие ограничения службы совместной работы Azure Active Directory B2B

1. Интерфейсы API приглашения службы совместной работы Azure Active Directory (Azure AD) B2B находятся в режиме бета-тестирования. Эта контактная зона API является шагом вперед, но как все бета-версии она доступна на условиях контракта для пространства имен бета-версии. В общедоступном выпуске API будет переведен на нумерованную версию.
2. В случае дублирования Многофакторной идентификации (MFA), если у вашего партнера уже настроена политика MFA, то проверка MFA для службы совместной работы B2B будет выполняться и управляться в приглашающей организации. Такой подход является предпочтительным, так как он охватывает все идентификаторы и позволяет контролировать надежность аутентификации пользователей, приглашенных в службу совместной работы B2B. Тем не менее, если ваш партнер уже настроил MFA и принудительно применяет ее, то, возможно, его пользователи должны будут выполнять эту проверку один раз в своей организации, а затем еще раз — в вашей организации. В будущих выпусках будет реализована политика, с помощью которой можно будет выбрать доверие к партнерской MFA во избежание двойной проверки.
3. Мгновенное включение. В рабочих потоках службы совместной работы B2B пользователи добавляются в каталог и динамически обновляются в процессе активации приглашений, при назначении приложений и т. д. Операции обновления и записи обычно выполняются в одном экземпляре каталога и должны реплицироваться по всем экземплярам. Мы заметили, что в некоторых случаях из-за ограниченного времени, которое отводится на выполнение репликации, могут возникать проблемы, проявляющиеся как ошибки авторизации. Мы работаем над тем, чтобы минимизировать влияние этих проблем или устранить их до выхода общедоступной версии. Скорее всего вы не столкнетесь с таким случаем, но если это все же произошло, то обновление или повторная попытка должны устранить проблему.

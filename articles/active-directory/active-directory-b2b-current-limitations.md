# <a name="limitations-of-azure-ad-b2b-collaboration"></a>Ограничения службы совместной работы Azure Active Directory B2B
К службе совместной работы Azure Active Directory B2B применяются ограничения, описанные в этой статье.

## <a name="invitation-apis-are-in-preview"></a>Интерфейсы API приглашения находятся в режиме предварительной версии.
Эта контактная зона API является шагом вперед. Тем не менее, как все предварительные версии, этот API доступен на условиях контракта для пространства имен предварительной версии. В общедоступном выпуске API будет переведен на нумерованную версию.

## <a name="possible-double-multi-factor-authentication"></a>Возможное дублирование Многофакторной идентификации
Если у вашего партнера уже настроена политика MFA, может возникнуть избыточность. Многофакторная идентификация для службы совместной работы B2B будет выполняться и управляться в приглашающей организации. Такой подход является предпочтительным, так как он охватывает все идентификаторы и позволяет контролировать надежность аутентификации пользователей, приглашенных в службу совместной работы B2B.

Тем не менее если ваш партнер уже настроил MFA и принудительно применяет ее, то, возможно, его пользователи должны будут выполнять эту проверку один раз в своей организации, а затем еще раз — в вашей организации.

В будущем выпуске будет реализована политика, позволяющая выбрать доверие к партнерской MFA во избежание двойной проверки подлинности.

## <a name="instant-on"></a>Мгновенное включение
В рабочих потоках службы совместной работы B2B пользователи добавляются в каталог и динамически обновляются в процессе активации приглашений, при назначении приложений и т. д. Операции обновления и записи обычно выполняются в одном экземпляре каталога и должны реплицироваться по всем экземплярам. Мы заметили, что в некоторых случаях из-за ограниченного времени, которое отводится на выполнение репликации, могут возникать проблемы, проявляющиеся как ошибки авторизации. Мы работаем над тем, чтобы минимизировать влияние этих проблем или устранить их до выхода общедоступной версии. Скорее всего, вы не столкнетесь с ними, но если это все же произойдет, обновление или повторная попытка должны устранить проблему.

# Что такое Microsoft Azure Active Directory


[Как это работает?](active-directory-works.md)<br> [Приступая к работе](active-directory-get-started.md)<br> [Дальнейшие действия](active-directory-next-steps.md)<br> [Подробнее](active-directory-learn-map.md)

Azure Active Directory (Azure AD) предоставляет простой способ увеличения преимуществ, получаемых от инвестиций в локальные ресурсы, которые используют облачные службы, со всей гибкостью и безопасностью, характерной для Active Directory. Это облачная платформа, которая устраняет проблемы, присущие эпохе работы в облаке, расширяя существующую инфраструктуру идентификации без необходимости перемещения. Вы можете продолжать получать прибыль от существующих инвестиций и локальных возможностей при поддержке Azure AD, чтобы управлять учетными записями и доступом в облаке.

Azure AD позволяет централизованно управлять доступом к приложениям и ресурсам, легко добавлять существующие ресурсы (облачные службы или локальные приложения) и интегрировать разрабатываемые приложения. Пользователи могут получить возможности самообслуживания и использовать единый вход для всех приложений, так что им не нужно будет знать, где искать каждое приложение, или запоминать отдельные пароли. В облаке сотрудники вашей организации могут совместно работать с деловыми партнерами, поставщиками и бизнес-клиентами, которых можно пригласить на сайты SharePoint (или принимать участие в работе на их сайтах SharePoint).

Благодаря Azure AD вы можете управлять всеми ресурсами из одного расположения как в облаке, так и локально. Задачи администрирования не дублируются. Azure AD начинает работать с тем, что уже есть. [Подробнее](active-directory-aadconnect.md).




> [AZURE.NOTE]Чтобы использовать Azure Active Directory, необходима учетная запись Azure. Если у вас нет учетной записи, вы можете [зарегистрироваться для получения бесплатной учетной записи Azure](http://azure.microsoft.com/pricing/free-trial/).


## Какие преимущества может предоставить Azure Active Directory?

Так как Azure AD — это комплексная служба, она предоставляет различные преимущества разным ролям в организации.

- Руководителям организаций Azure AD позволяет использовать потенциал облачных приложений и добиться мобильности сотрудников с уверенностью в том, что нормативные требования соблюдены.
- Поставщикам услуг Azure AD позволяет с легкостью решать проблемы идентификации и доступа, подключая службы к работе с существующими решениями для учетных записей клиента, а также клиентов Microsoft Azure и Office 365. Azure AD также может решить все вопросы, связанные с доступом в бэк-офис; вне зависимости от внутреннего или внешнего размещения, вы можете быть уверены, что правильные пользователи получат правильный доступ.
- ИТ-специалисты могут использовать Azure AD, чтобы увеличить контроль и видимость операций со «скоростью облака». Благодаря Azure AD вы сможете узнавать, какие ресурсы используют пользователи, и предоставлять предложения по самообслуживанию.<br> <br>

##Девять фактов, которые следует знать об Azure AD

### Управление паролями Active Directory из любой точки мира

Используйте Azure Active Directory, чтобы предоставить пользователям возможность управлять паролями Active Directory или Azure Active Directory с любого устройства и из любого расположения. Администратор может управлять политиками паролей и уведомлений, а также просматривать подробные данные аудита о действиях по операциям сброса паролей в момент их выполнения. [Дополнительная информация об управлении паролями в Azure AD](http://aka.ms/ssproverview).

### Установка правил доступа к облачным ресурсам

Вы можете задать правила и политику, чтобы контролировать, кто может получать доступ к облачным приложениям и ресурсам и при каких условиях. Например, можно требовать многофакторной проверки подлинности (MFA) и управлять доступом в зависимости от устройства или расположения. [Дополнительные сведения о многофакторной проверке подлинности Azure](../multi-factor-authentication.md).

### Единый вход для любого приложения

Azure Active Directory обеспечивает безопасный единый вход в облако и в локальные приложения, в том числе Microsoft Office 365 и тысячи приложений SaaS, таких как Salesforce, Workday, DocuSign, ServiceNow и Box. [Дополнительная информация о приложениях SaaS](http://azure.microsoft.com/marketplace/active-directory/).

### Совместимость с любыми устройствами

Пользователи могут запускать приложения с помощью настроенной веб-панели доступа, мобильного приложения, Office 365 или пользовательских корпоративных порталов, используя существующие учетные данные, и пользоваться одинаковыми функциональными возможностями вне зависимости от операционной системы устройства, iOS, Mac OS X, Android или Windows.

### Установка правил для внешнего доступа

Внешние пользователи могут безопасно использовать локальную сеть, защищенную брандмауэром, с помощью встроенного прокси приложения. Все эти установленные правила и политики, включая многофакторную проверку подлинности, могут применяться для доступа к облачным приложениям или установленным локальным приложениям с помощью службы Application Proxy — без необходимости переписывать их или предоставлять непосредственно через Интернет. [Дополнительные сведения о службе Azure AD Application Proxy](https://msdn.microsoft.com/library/azure/dn768219.aspx).

### Мониторинг доступа пользователей

Наконец, Azure AD предоставляет сведения о том, что происходит в вашей организации. Благодаря дополнительным отчетам и анализам вы получите уникальные сведения о доступе пользователей. Например, с помощью функции обнаружения приложений можно выявить, какие приложения наиболее активно используются в организации. [Дополнительные сведения об обнаружении приложений в облаке Azure AD](https://appdiscovery.azure.com/).

### Расширенная отчетность, аудит и аналитика
Azure AD предоставляет ряд ежедневных отчетов по использованию и доступу для администраторов ([Отчеты Azure AD](active-directory-view-access-usage-reports.md)). Кроме того, в API отчетов ([API отчетов Azure AD](active-directory-reporting-api-getting-started.md)) доступны пользовательские отчеты и подробные обзоры данных. В платных выпусках Azure AD доступно еще больше отчетов, в том числе отчеты по аудиту активности привилегированных пользователей ([Аудит Azure AD](active-directory-view-access-usage-reports.md)).

### Простота и безопасность работы для всех пользователей

Примеры: работа конечных пользователей облегчена за счет размещения профилей, приложений и системы управления доступом к ресурсам в одном месте, и нет необходимости проходить специальное обучение. Многофакторная проверка подлинности, приложения "ПО как услуга", гибридные средства и возможности самообслуживания готовы к использованию.

- Администраторы имеют доступ к порталу управления Azure AD и Windows PowerShell для обеспечения комплексного управления.

- Разработчики имеют согласованный набор API RESTful и простой доступ к публикации и получению данных интерфейсов приложений.

### Доступные выпуски Azure AD

Azure AD предоставляется в трех выпусках:

- выпуск Free, который представляет собой облачную службу каталогов;
- выпуск Basic, который представляет собой облачную службу каталогов, предоставляющую доступ к приложениям SaaS;
- выпуск Premium, который представляет собой комплексную службу каталогов на основе правил с возможностями самоуправления.

Чтобы испытать описанные здесь функции, активируйте [бесплатную пробную версию Azure](http://azure.microsoft.com/trial/get-started-active-directory/).

# <a name="create-an-access-review-of-group-members-or-application-access-with-azure-ad"></a>Создание проверки доступа для участников группы или приложения с помощью Azure AD

Назначения доступа становятся "устаревшими", когда у пользователей имеются права доступа, которые им больше не нужны. Чтобы уменьшить риск, связанный с устаревшими назначениями доступа, администраторы могут использовать Azure Active Directory (Azure AD), чтобы создать проверку доступа участников группы или пользователей, назначенных для приложения. Создание повторяющейся проверки доступа может стать экономией времени. Если вам необходимо регулярно проверять пользователей, имеющих доступ к приложению или являющихся участниками группы, вы можете определить частоту этих проверок. Дополнительные сведения об этих сценариях см. в разделах [Управление пользовательским доступом с помощью проверок доступа Azure AD](active-directory-azure-ad-controls-manage-user-access-with-access-reviews.md) и [Управление гостевым доступом с помощью проверок доступа Azure AD](active-directory-azure-ad-controls-manage-guest-access-with-access-reviews.md). 

## <a name="create-an-access-review"></a>Создание проверки доступа

1. В качестве глобального администратора перейдите на [страницу проверки доступа](https://portal.azure.com/#blade/Microsoft_AAD_ERM/DashboardBlade/), а затем выберите вкладку **Программы**.

2. Выберите программу, содержащую элемент управления проверки доступа, который требуется создать. Можно выбрать **программу по умолчанию**, которая всегда доступна, или создать другую программу. Например, вы можете использовать одну программу для каждой инициативы по обеспечению соответствия или бизнес-цели.

3. В программе выберите **Элементы управления**, а затем щелкните **Добавить**, чтобы добавить элемент управления.

4. Назовите проверку доступа. При необходимости укажите описание проверки. Имя и описание отображаются для рецензентов.

5. Установите дату начала. По умолчанию проверка доступа выполняется однократно, начинается в то же самое время, когда она создается, и завершается через один месяц. Можно изменить даты начала и окончания таким образом, чтобы проверка доступа началась в будущем и длилась необходимое количество дней.

6. Чтобы настроить повторение проверки доступа, измените частоту с однократного на еженедельный, ежемесячный, ежеквартальный или годовой период и используйте ползунок или текстовое поле, чтобы определить, сколько дней каждая повторяющаяся серия проверок будет открыта для выходных данных от рецензентов. Например, максимальная продолжительность, которую вы можете установить для ежемесячной проверки, составляет 27 дней, чтобы избежать дублирования. 

7.  Повторяющаяся серия проверок доступа может завершиться тремя способами: она работает непрерывно, чтобы запускать проверки неограниченно, до определенной даты или пока не будет достигнуто определенное количество вхождений. Вы или другой глобальный администратор можете остановить серию после создания, указав в параметрах нужную дату окончания.

8. Проверки доступа могут выполняться для участников группы или пользователей, которые были назначены для приложения. Можно также сузить область проверки доступа только до гостевых пользователей, которые являются участниками (или назначены для приложения), вместо того чтобы проверять всех пользователей-участников или пользователей с доступом к приложению.

9. Выберите одного или нескольких пользователей для проверки всех пользователей в области. Или можно указать, чтобы участники сами проверяли собственный доступ. Если ресурс является группой, вы можете попросить владельцев группы выполнить проверку. Можно также потребовать, чтобы рецензенты указывали причину при подтверждении доступа.

10. Если вы хотите вручную применить результаты по завершении проверки, щелкните **Начать**.  Либо перейдите к следующему разделу, где объясняется, как настроить автоматическую проверку.

### <a name="configuring-an-access-review-with-auto-apply"></a>Настройка проверки доступа с автоматическим применением

1.  Разверните меню "После заполнения параметров" и включите параметры "Auto apply results to resource" (Автоматически применять результаты к ресурсу). 

2.  В случаях, когда рецензент не проверял пользователей в течение рассматриваемого периода, проверка доступа может принять рекомендацию системы (если включено) об отклонении или одобрении постоянного доступа пользователя, оставить доступ без изменений или удалить доступ. Это не повлияет на пользователей, которые были проверены рецензентами вручную. Если окончательное решение рецензента — "Отклонить", тогда доступ пользователя будет удален.

3.  Чтобы включить параметр для принятия рекомендаций, если рецензенты не отвечают, в дополнительных настройках необходимо включить отображение рекомендаций.
 
4.  Наконец, нажмите кнопку **Запуск**.

В зависимости от заданных параметров в разделе "После заполнения параметров", автоматическое применение будет выполнено после даты окончания проверки или при ее прекращении в ручном режиме. Состояние проверки доступа изменится с Completed (Завершено) на Applying (Применение), а затем на Applied (Применено). Все пользователи, отклоненные по результатам проверки, будут лишены членства в группах или назначений в приложениях в течение нескольких минут.


## <a name="manage-the-access-review"></a>Управление проверкой доступа

По умолчанию Azure AD отправляет электронные сообщения рецензентам вскоре после запуска проверки. Если вы выбрали не отправлять сообщения через службу Azure AD, обязательно сообщите рецензентам, что им необходимо выполнить проверку доступа. Вы можете показать им инструкции по [проверке доступа](active-directory-azure-ad-controls-perform-access-review.md). Если проверка предназначена для проверки гостями собственного доступа, покажите им инструкции по [проверке собственного доступа](active-directory-azure-ad-controls-perform-access-review.md).

Если некоторые из рецензентов являются гостями, они будут уведомлены по электронной почте, только если уже приняли приглашение.

Чтобы управлять серией проверок доступа, перейдите к проверкам доступа на вкладке **Элементы управления**, найдите предстоящие вхождения в запланированных проверках и измените дату окончания либо добавьте или удалите рецензентов. 

Отслеживать ход выполнения проверяющими их проверок можно на панели мониторинга Azure AD в разделе **Проверки доступа**. Права доступа в каталоге не изменяются до [завершения проверки](active-directory-azure-ad-controls-complete-access-review.md).

## <a name="next-steps"></a>Дополнительная информация

После запуска проверки доступа Azure AD будет автоматически отправлять рецензентам электронное сообщение, предлагающее им проверить доступ. Если пользователь не получил электронное сообщение, ему можно отправить инструкции по [проверке доступа](active-directory-azure-ad-controls-perform-access-review.md). 

Если это однократная проверка, по завершении периода проверки доступа или после того, как администратор остановил проверку, следуйте инструкциям в статье [Завершение проверки доступа для участников группы или пользователей с доступом к приложению с помощью Azure AD](active-directory-azure-ad-controls-complete-access-review.md).  

Если это серия проверок, перейдите на вкладку **Журнал проверок** на странице серии проверок доступа, чтобы выбрать завершенную проверку.  Предстоящие проверки отображаются в разделе **Плановая проверка**, где вы можете изменить продолжительность и добавить или удалить рецензентов для отдельных проверок.

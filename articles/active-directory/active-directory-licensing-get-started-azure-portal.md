# <a name="license-yourself-and-your-users-in-azure-active-directory"></a>Самостоятельное лицензирование и лицензирование пользователей в Azure Active Directory

> [!div class="op_single_selector"]
> * [Самостоятельное лицензирование и лицензирование пользователей в Azure Active Directory](active-directory-licensing-get-started-azure-portal.md)
> * [Что такое лицензирование Microsoft Azure Active Directory на классическом портале Azure?](active-directory-licensing-what-is.md)
>
>

Azure Active Directory (Azure AD) — это служба идентификации (IDaaS), которая одновременно является и решением, и платформой от корпорации Майкрософт. Служба Azure AD предлагается в различных выпусках:

* Azure Active Directory Free, доступная в любой службе Майкрософт, например Office 365, Dynamics, Microsoft Intune или Azure. В этом режиме Azure AD не начисляет плату за использование.

* Платные выпуски Azure AD, такие как:
  - Enterprise Mobility + Security 
  - Azure AD Premium (P1 и P2);
  - Azure AD Basic
  - Azure Multi-Factor Authentication

Как и многие онлайн-службы Майкрософт, большинство платных версий Azure AD доставляются на правах доступа «на пользователя», как в Office 365, Microsoft Intune и Azure AD. В этих случаях служба приобретается в виде одной или нескольких подписок, каждая из которых включает несколько предварительно приобретенных лицензий для вашего клиента. Права доступа "на пользователя" предоставляются путем:

* Назначения лицензии. 
* Создания связи между пользователем и продуктом.
* Включения компонентов службы для пользователя.
* Использования одной из предоплаченных лицензий.

[Попробуйте Azure AD Premium.](https://portal.office.com/Signup/Signup.aspx?OfferId=01824d11-5ad8-447f-8523-666b0848b381&ali=1#0)

Развернутый обзор возможностей службы Azure AD см. в статье [Что такое Microsoft Azure Active Directory](active-directory-whatis.md). Дополнительные сведения см. на [странице соглашений об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/active-directory/v1_0/).

> [!NOTE]
> Подписки Azure с оплатой по мере использования включают создание ресурсов Azure, а затем сопоставляют их с методом оплаты. К таким подпискам не применяется ограничение на число лицензий. Когда вы предоставляете пользователям разрешение для работы с ресурсами Azure, сопоставленными с подпиской, они связываются с подпиской Azure и могут управлять ее ресурсами.

## <a name="how-does-azure-active-directory-licensing-work"></a>Как работает лицензирование Azure Active Directory

Службы Azure AD на основе лицензий работают, активируя подписку в клиенте службы Azure AD. После активации подписки функциями службы начинают управлять администраторы Azure AD, а лицензированные пользователи могут их использовать.

### <a name="manage-subscription-information"></a>Управление сведениями о подписке

Когда вы покупаете Enterprise Mobility + Security, Azure AD Premium или Azure AD Basic, в клиенте обновляется информация о подписке, в частности срок ее действия и предоплаченные лицензии. Сведения о подписке, включая число назначенных или доступных лицензий, можно просмотреть на портале Azure AD, в разделе **Azure Active Directory** откройте плитку **Лицензии** для выбранного каталога. С колонки **Лицензии** лучше всего управлять назначениями лицензий.

Каждая подписка состоит из одного или нескольких планов обслуживания, таких как Azure AD, Многофакторная идентификация, Intune, Exchange Online или SharePoint Online.  Для управления лицензиями Azure AD *не требуется* управление уровнями плана обслуживания. Office 365 отличается тем, что использует режим расширенной конфигурации при управлении доступом к службам, входящим в комплект поставки. В Azure AD для включения функций и управления отдельными разрешениями используется конфигурация службы.

> [!IMPORTANT]
> Подписки Azure AD Premium, Azure AD Basic и Enterprise Mobility + Security ограничены подготовленными для них каталогами и клиентами. Подписки нельзя разделять между каталогами или использовать их для предоставления прав пользователям в других каталогах. Подписки можно перемещать между каталогами, но для этого нужно отправить запрос в службу поддержки или отменить подписку и повторно купить ее, приобретя напрямую.
>
> Когда Azure AD или Enterprise Mobility + Security приобретена через подписку по программе корпоративного лицензирования и если соглашение включает другие службы Microsoft Online (например, Office 365), активация происходит автоматически. 

### <a name="assign-licenses"></a>Назначение лицензий

Несмотря на то что получение подписки — это все, что нужно для настройки платных возможностей, вам все еще необходимо распределять лицензии для платных функций Azure AD пользователям. Любому пользователю, который должен иметь доступ к платной функции Azure AD или который управляется с помощью этой платной функции, необходимо назначить лицензию. Назначение лицензий — это сопоставление пользователя с приобретенной службой, например Azure AD Premium, Basic или Enterprise Mobility + Security.


Назначить лицензии пользователям можно следующим образом: 

* Назначить лицензии группам на [портале Azure](active-directory-licensing-whatis-azure-portal.md).
* Назначить лицензии напрямую пользователям с помощью портала, PowerShell или API. 

Если назначить лицензию группе, лицензия будет назначена всем участникам группы. Если в группу добавить пользователей или удалить их из нее, соответствующая лицензия будет назначена или удалена. При назначении группы могут использоваться любые доступные функции управления группой. Такое назначение также согласуется с назначением приложений на основе групп.

[Назначение лицензий на основе групп](active-directory-licensing-whatis-azure-portal.md) можно использовать для настройки правил следующего вида:
* Все пользователи в каталоге автоматически получают лицензию.
* Все пользователи с соответствующими должностями получают лицензию.
* Вы можете делегировать принятие решений другим менеджерам в организации (с помощью [самостоятельного управления группами](active-directory-accessmanagement-self-service-group-management.md)).

Дополнительные сведения о назначении лицензий группам, включая расширенные сценарии и сценарии лицензирования Office 365, см. в статье [Назначение лицензий группе пользователей в Azure Active Directory](active-directory-licensing-group-assignment-azure-portal.md).

## <a name="get-started-with-azure-ad-licensing"></a>Начало работы с лицензированием Azure AD

Начать работу с Azure AD легко. Вы всегда можете создать свой каталог в ходе регистрации для использования [бесплатной пробной версии Azure](https://azure.microsoft.com/offers/ms-azr-0044p/).

Следующие рекомендации помогут убедиться, что клиент соответствует вашим целям для службы и другим службам Майкрософт, которые вы можете использовать:

- Если вы уже используете любую из служб организации от корпорации Майкрософт, значит у вас уже есть клиент Azure AD. В этом случае используйте этот каталог и для других служб, чтобы все службы могли использовать возможности основного управления удостоверениями, включая подготовку и гибридный единый вход. С единым входом ваши пользователи получают преимущества расширенных возможностей в разных службах. Поэтому, если вы решите приобрести платную службу Azure AD для своих сотрудников, мы рекомендуем использовать для этого тот же клиент.

- Мы советуем использовать новый клиент на портале Azure, если вы планируете:
  - Использовать Azure AD для различных наборов пользователей (например участники или клиенты).
  - Оценивать службы Azure AD отдельно от рабочих служб.
  - Настраивать изолированную среду для служб.

  Новый каталог создается с вашей учетной записью в качестве внешнего пользователя с разрешениями глобального администратора. При входе на портал Azure с этой учетной записи вы увидите этот клиент и получите доступ ко всем задачам администрирования.

> [!NOTE]
> Служба Azure AD поддерживает гостевых пользователей, которые являются учетными записями пользователей в клиенте Azure AD, которые были созданы с помощью учетной записи Майкрософт или удостоверения Azure AD из другого клиента. Портал администрирования Office 365 в настоящее время не поддерживает таких пользователей. Гостевые пользователи с учетными записями Майкрософт не имеют доступа к порталу администрирования Office 365, а гостевые пользователи из других клиентов Azure AD будут игнорироваться. В последнем случае доступна только локальная учетная запись пользователя в Azure AD или клиенте Office 365, где пользователь был изначально создан.

### <a name="select-one-or-more-license-trials"></a>Выбор одной или нескольких пробных версий

Вы можете активировать пробную подписку Azure AD Premium или Enterprise Mobility + Security в разделе **Azure Active Directory** &gt; **Быстрый запуск**.

![Выбор пробной версии лицензии](media/active-directory-licensing-get-started-azure-portal/select-a-license-trial.png)

Пробные лицензии доступны в колонке **Лицензии**.

### <a name="assign-licenses-to-users-and-groups"></a>Назначение лицензий пользователям и группам

После активации подписки следует назначить лицензию себе. Затем обновите страницу в браузере и убедитесь, что вы видите все необходимые функции. Следующим шагом является назначение лицензий пользователям, которым требуется доступ к платным функциям Azure AD. Как описано в разделе [Назначение лицензий](#assign-licenses), лучше всего определить группу, представляющую нужную аудиторию, и назначить ей лицензию. Тогда эта лицензия будет назначаться и удаляться для всех пользователей, которые добавляются в эту группу или удаляются из нее, соответственно.

> [!NOTE]
> Некоторые службы Майкрософт недоступны во всех расположениях. Прежде чем назначать лицензию, администратор должен указать для пользователя свойство **Место использования**. Это свойство можно задать в разделе **Пользователь** &gt; **Профиль** &gt; **Параметры** на портале Azure. Если лицензии назначаются группам, все пользователи, для которых не указано место использования, наследуют расположение каталога.

Чтобы назначить лицензию, в разделе **Azure Active Directory** &gt; **Лицензии** &gt; **Все продукты** выберите один или несколько продуктов, а затем щелкните **Назначить** на панели команд.

![Выбор лицензии для назначения](media/active-directory-licensing-get-started-azure-portal/select-license-to-assign.png)

Колонку **Пользователи или группы** можно использовать для выбора нескольких пользователей или групп или для выключения планов обслуживания в продукте. Используйте поле поиска вверху, чтобы найти имена пользователей и групп.

![Выбор пользователя или группы для назначения лицензии](media/active-directory-licensing-get-started-azure-portal/select-user-for-license-assignment.png)

Назначение лицензий группе может занять некоторое время в зависимости от числа пользователей, которые должны унаследовать лицензию. Состояние обработки можно проверить в колонке **Группы** на плитке **Лицензии**.

![Состояние назначения лицензии](media/active-directory-licensing-get-started-azure-portal/license-assignment-status.png)

При назначении лицензий Azure AD могут возникнуть ошибки назначения, но при управлении Azure AD и продуктами Enterprise Mobility + Security это случается относительно редко. Потенциальные ошибки представлены следующими случаями.
- Конфликт назначений. Пользователю ранее назначена лицензия, несовместимая с текущей. В этом случае перед назначением новой лицензии необходимо удалить предыдущую.
- Нехватка доступных лицензий. Когда количество пользователей в группе назначения превышает количество доступных лицензий, состояние назначения пользователей будет отображать ошибку назначения из-за отсутствия лицензий.

#### <a name="azure-ad-b2b-collaboration-licensing"></a>Лицензирование службы совместной работы Azure Active Directory B2B

Служба совместной работы B2B позволяет приглашать гостевых пользователей в клиент Azure AD, чтобы предоставить доступ к службам Azure AD и любым ресурсам Azure AD, которые были сделаны доступными.  

За приглашение пользователей B2B и их назначение приложениям в Azure AD плата не взимается. До 10 приложений на гостевого пользователя и 3 базовых отчета также доступны бесплатно для пользователей службы совместной работы B2B. Если гостевой пользователь имеет какие-либо соответствующие лицензии, назначенные в клиенте Azure AD партнеров, они будут назначены ему и в вашем клиенте.

Это не обязательно, но если вы хотите предоставить доступ к платным функциям Azure AD, этим гостевым пользователям B2B необходимо назначить соответствующую лицензию Azure AD. Приглашающий клиент с помощью платной лицензии Azure AD может назначить пользователям службы совместной работы B2B права для приглашения пяти дополнительных гостевых пользователей в клиент. Дополнительные сведения о сценарии см. в статье [Руководство по лицензированию службы совместной работы Azure Active Directory B2B](active-directory-b2b-licensing.md).

### <a name="view-assigned-licenses"></a>Просмотр назначенных лицензий

Сводное представление доступных и назначенных лицензий отображается в разделе **Azure Active Directory** &gt; **Лицензии** &gt; **Все продукты**.

![Просмотр сведений о лицензии](media/active-directory-licensing-get-started-azure-portal/view-license-summary.png)

Подробный список назначенных пользователей и групп можно открыть, выбрав указанный продукт. **Лицензированные пользователи**. Здесь отображаются все пользователи, которые используют лицензию, а также сведения о том, назначена ли лицензия непосредственно пользователю или наследуется от группы.

![Просмотр сведений о лицензии](media/active-directory-licensing-get-started-azure-portal/view-license-detail.png)

Аналогичным образом, список **Лицензированные группы** показывает все группы, которым назначены лицензии. Выберите пользователя или группу, чтобы открыть колонку **Лицензии**, где показаны все лицензии, назначенные этому объекту.

### <a name="remove-a-license"></a>Удаление лицензии

Чтобы удалить лицензию, перейдите к пользователю или группе и откройте плитку **Лицензии**. Выберите лицензию и щелкните **Удалить**.

![Удаление лицензии](media/active-directory-licensing-get-started-azure-portal/remove-license.png)

Лицензии, унаследованные пользователем от группы, невозможно удалить напрямую. Вместо этого нужно удалить пользователя из группы, от которой наследуется эта лицензия.

### <a name="extend-trials"></a>Расширение пробной версии

Расширения пробных версий для пользователей доступны в качестве самостоятельной регистрации через портал Office 365. Администратор клиента может перейти на портал Office (доступ зависит от разрешений для портала Office) и выбрать пробную версию Azure AD Premium. Ссылка **Расширить пробную версию** позволяет запустить процесс расширения. Потребуется кредитная карта, но плата не взимается.

![Расширение пробной версии на портале Azure](media/active-directory-licensing-get-started-azure-portal/extend-trial-beginning.png)

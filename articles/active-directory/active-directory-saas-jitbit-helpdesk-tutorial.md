#Руководство. Интеграция Azure Active Directory с Jitbit Helpdesk
  
Цель данного руководства — показать интеграцию Azure и Jitbit Helpdesk. Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент Jitbit Helpdesk
  
По завершении работы с этим руководством пользователи Azure AD, назначенные в Jitbit Helpdesk, смогут выполнять единый вход в приложение на веб-сайте Jitbit Helpdesk компании (вход, инициированный поставщиком услуг) или следуя указаниям в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).
  
Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для Jitbit Helpdesk
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777676.png "Сценарий")
##Включение интеграции приложений для Jitbit Helpdesk
  
В этом разделе показано, как включить интеграцию приложений для Jitbit Helpdesk.

###Чтобы включить интеграцию приложений для Jitbit Helpdesk, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** щелкните **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В **поле поиска** введите **Jitbit Helpdesk**.

    ![Коллекция приложений](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777677.png "Коллекция приложений")

7.  В области результатов выберите **Jitbit Helpdesk** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![JitBit](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC781008.png "JitBit")
##Настройка единого входа
  
В этом разделе показано, как разрешить пользователям проходить проверку подлинности в Jitbit Helpdesk со своей учетной записью Azure AD, используя федерацию на основе протокола SAML. В рамках этой процедуры потребуется создать файл сертификата в кодировке Base-64. Если вы не знакомы с этой процедурой, просмотрите видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

>[AZURE.IMPORTANT]Чтобы иметь возможность настроить единый вход для клиента Jitbit Helpdesk, необходимо сначала обратиться в службу технической поддержки Jitbit Helpdesk для включения соответствующей функции.

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **Jitbit Helpdesk** портала Azure AD нажмите кнопку **Настроить единый вход**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777678.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в Jitbit Helpdesk** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777679.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес входа в Jitbit Helpdesk** введите свой URL-адрес, используя следующий шаблон *https://\<имя-клиента>.Jitbit.com*, а затем нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777528.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в Jitbit Helpdesk** нажмите кнопку **Загрузить сертификат**, а затем сохраните файл сертификата локально в папке **c:\\Jitbit Helpdesk.cer**.

    ![Настройка единого входа](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777680.png "Настройка единого входа")

5.  В другом окне браузера войдите на свой корпоративный веб-сайт Jitbit Helpdesk в качестве администратора.

6.  На панели инструментов в верхней части страницы щелкните **Администрирование**.

    ![Администрирование](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777681.png "Администрирование")

7.  Щелкните **Общие параметры**.

    ![Пользователи, компании и разрешения](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777682.png "Пользователи, компании и разрешения")

8.  В разделе конфигурации **Параметры проверки подлинности** сделайте следующее:

    ![Параметры проверки подлинности](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777683.png "Параметры проверки подлинности")

    1.  Установите флажок **Включить единый вход SAML 2.0** для выполнения единого входа вместе с **OneLogin**.
    2.  На странице **Настройка единого входа в Jitbit Helpdesk** портала Azure скопируйте значение поля **Конечная точка, инициированная поставщиком услуг** и вставьте его в текстовое поле **URL-адрес конечной точки**.
    3.  Создайте файл **в кодировке Base-64** из скачанного сертификата.
        
		>[AZURE.TIP]Дополнительные сведения можно узнать из видео [Как преобразовать двоичный сертификат в текстовый файл](http://youtu.be/PlgrzUZ-Y1o).

    4.  Откройте сертификат в кодировке Base-64, скопируйте его содержимое в буфер обмена, а затем вставьте его в текстовое поле **Сертификат X.509**.
    5.  Нажмите кнопку **Сохранить изменения**.

9.  На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777684.png "Настройка единого входа")
##Настройка подготовки учетных записей пользователей
  
Чтобы пользователи Azure AD могли выполнять вход в Jitbit Helpdesk, они должны быть подготовлены для Jitbit Helpdesk. В случае с Jitbit Helpdesk подготовка выполняется вручную.

###Чтобы подготовить учетные записи пользователей, выполните следующие действия.

1.  Выполните вход в клиент **Jitbit Helpdesk**.

2.  В меню в верхней части страницы щелкните **Администрирование**.

    ![Администрирование](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777681.png "Администрирование")

3.  Нажмите кнопку **Пользователи, компании и разрешения**.

    ![Пользователи, компании и разрешения](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777682.png "Пользователи, компании и разрешения")

4.  Нажмите кнопку **Добавить пользователя**.

    ![Добавить пользователя](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777685.png "Добавить пользователя")

5.  В разделе «Создание» введите данные учетной записи Azure AD, которую необходимо подготовить, в следующих текстовых полях: **Имя пользователя**, **Адрес электронной почты**, **Имя**, **Фамилия**.

    ![Создание](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777686.png "Создание")

6.  Щелкните **Создать**.

>[AZURE.NOTE]Вы можете использовать любые другие средства создания учетной записи пользователя Jitbit Helpdesk или API, предоставляемые Jitbit Helpdesk для подготовки учетных записей пользователя AAD.

##Назначение пользователей
  
Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которые должны использовать приложение, доступ путем их назначения.

###Чтобы назначить пользователей Jitbit Helpdesk, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **Jitbit Helpdesk** нажмите кнопку **Назначить пользователей**.

    ![Назначить пользователей](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC777687.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да** для подтверждения назначения.

    ![Да](./media/active-directory-saas-jitbit-helpdesk-tutorial/IC767830.png "Да")
  
Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](active-directory-saas-access-panel-introduction.md).

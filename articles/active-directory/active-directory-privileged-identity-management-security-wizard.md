# Мастер защиты управления привилегированными пользователями Azure

Когда вы в первый раз запускаете управление привилегированными пользователями (PIM) Azure, открывается мастер. Он поможет вам оценить риски безопасности, связанные с привилегированными пользователями, и понять, как их снизить с помощью расширения для управления привилегированными пользователями Azure.

Мастер включает три раздела: **"Администраторы могут подвергнуть вас риску", "Управление возможными направлениями атаки со стороны администраторов"** и **"Определение параметров для временных администраторов"**. В каждом разделе представлен обзор используемых понятий и некоторые рекомендуемые действия.

Изначально все ваши глобальные администраторы будут постоянными. Когда вы нажмете кнопку **ВОЗМОЖНО, ВАШИ АДМИНИСТРАТОРЫ ПОДВЕРГАЮТ ВАС РИСКУ**, отобразится список пользователей с ролью глобального администратора и их количество.

Щелкнув **Управление возможными направлениями атаки со стороны администраторов**, вы сможете изменить постоянные роли администраторов на временные, оставить их постоянными или удалить пользователей из роли.

Щелкнув **Определение параметров для временных администраторов**, вы сможете настроить многофакторную проверку подлинности, включить уведомления и определить, как долго у временного администратора будут сохраняться привилегии.

## Изменение ролей глобального администратора на постоянные или временные

Есть три варианта управления сроком действия привилегий глобального администратора.

1.  Нажмите кнопку **Сделать все временными**, чтобы сделать всех глобальных администраторов временными.

2.  Нажмите кнопку **Сделать все постоянными**, чтобы сделать всех глобальных администраторов постоянными.

3.  Каждого глобального администратора можно **оставить постоянным**, **сделать временным** или **удалить из роли**.

## Изменение периода активации роли глобального администратора

Есть два способа задать период активации роли глобального администратора.

1.  Чтобы увеличить или уменьшить **период активации**, переместите соответствующий ползунок вправо или влево. Период активации может достигать 72 часов.

2.  Вы можете ввести количество **часов** в поле справа от ползунка.

## Включение уведомлений

Чтобы администраторы могли получать по почте уведомления в случае активации роли, включите уведомления, нажав кнопку **Включить**. Позже эту функцию можно отключить.

## Требование многофакторной проверки подлинности

Если вы хотите, чтобы администраторы использовали MFA для входа в свои учетные записи и отправки запроса на расширение своей роли, включите MFA, нажав кнопку **Включить**. Позже эту функцию можно отключить.

Выберите роли, к которым будут применены эти параметры. Нажмите кнопку **ОК**.

> [AZURE.WARNING] В данный момент важно наличие нескольких администраторов безопасности. Если у вас только один администратор безопасности, который не является постоянным, а служба MFA не настроена, по истечении срока действия роли этот пользователь вообще не сможет управлять PIM.

По завершении нажмите кнопку **ОК**.

Впоследствии мастер отображаться не будет. Однако вы можете вызвать его снова, нажав кнопку **Мастер** в разделе **Управление удостоверениями**.

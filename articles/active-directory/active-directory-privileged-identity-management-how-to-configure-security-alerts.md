# Настройка оповещений безопасности в компоненте управления привилегированными пользователями Azure AD

## Оповещения безопасности
Компонент управления привилегированными пользователями (PIM) Azure создает следующие оповещения, которые можно просматривать в разделе "Оповещения" панели управления PIM.

| Предупреждение | Триггер | Рекомендации |
| ----- | ------- | -------------- |
| **Постоянная активация** | Администратору была назначена постоянная роль за пределами PIM. | Просмотрите новое назначение роли и при необходимости измените ее на временную. |
| **Подозрительное продление активации привилегированных ролей** | Слишком много повторных активаций, выполненных для одной и той же роли в рамках разрешенного в параметрами времени. | Свяжитесь с пользователем и убедитесь, что ему удалось успешно активировать роль. |
| **Для активации роли настроена ненадежная проверка подлинности** | Есть роли, в параметрах которых не настроена проверка MFA. | Рекомендуется настроить обязательную многофакторную проверку подлинности (MFA) для активации всех ролей. |
| **Избыточное число администраторов** | Имеются временные администраторы, которые в последнее время не выполняли активацию своих ролей. | Удалите назначения ролей, которые больше не требуются. |
| **Слишком большое количество глобальных администраторов** | Количество глобальных администраторов больше рекомендуемого. | Удалите назначения ролей, которые больше не требуются, или сделайте некоторые из них временными. |

## Настройка параметров оповещений системы безопасности

### Оповещение "Подозрительное продление активации привилегированных ролей"

Настройте параметры **Период возобновления активации** и **Число возобновлений активации**, определяющие условия запуска этого оповещения.

1. В разделе **Действия** панели мониторинга выберите **Оповещения системы безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Задайте **период продления активации**, передвинув ползунок или указав значение в текстовом поле (в минутах). Максимальное значение — 100.
4. Задайте **количество продлений активации** в течение периода продления активации, передвинув ползунок или указав количество продлений в текстовом поле. Максимальное значение — 100.
5. Щелкните **Сохранить**.

### Оповещение "Избыточное число администраторов"
1. В разделе **Действия** панели мониторинга выберите **Оповещения системы безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Выберите максимальный срок, в течение которого роль может существовать без активации, передвинув ползунок или указав нужное количество дней в текстовом поле.
4. Щелкните **Сохранить**.

### Оповещение "Слишком большое количество глобальных администраторов"

Есть два параметра, которые могут запустить это оповещение.
- Параметр **Минимальное количество глобальных администраторов** запускает оповещение, если количество администраторов превышает допустимое.
- Параметр **Процентная доля глобальных администраторов** запускает оповещение, если доля администраторов, являющихся глобальными администраторами, превышает заданное значение.

1. В разделе **Действия** панели мониторинга выберите **Оповещения системы безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Задайте значение **Минимальное количество глобальных администраторов**, передвинув ползунок или указав количество в текстовом поле.
4. Задайте значение **Процентная доля глобальных администраторов**, передвинув ползунок или указав процентное значение в текстовом поле.
5. Щелкните **Сохранить**.

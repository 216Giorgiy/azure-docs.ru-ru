# Управление привилегированными пользователями Azure: настройка предупреждений безопасности
## Обзор предупреждений безопасности
В расширении для управления привилегированными пользователями Azure можно настроить следующие предупреждения. Их можно просмотреть в разделе «Предупреждения» на панели мониторинга для управления привилегированными пользователями.

| Предупреждение | Триггер |
| ------------- | ------------- |
| **Возможная атака «Активация постоянной роли»** | Администратор активировал свою временную роль вне расширения PIM. |
| **Подозрительное продление активации привилегированных ролей** | Слишком много повторных активаций, выполненных для одной и той же роли, с разрешенным в параметрах сроком. |
| **Подозрительное использование «ловушки» глобальным администратором** | Зафиксировано использование «ловушки» пользователем.|
| **Для активации роли настроена ненадежная проверка подлинности** | Есть роли, в параметрах которых не настроена проверка MFA. |
| **Избыточные роли администраторов повышают вероятность атаки** | Есть временные администраторы, которые не активировали свои роли в течение заданного в параметрах срока. |
| **Слишком большое количество глобальных администраторов повышает вероятность атаки** | Глобальных администраторов больше, чем разрешено в параметрах. |

## Настройка предупреждений безопасности

### Настройка предупреждения «Подозрительное продление активации привилегированных ролей»
1. В разделе **Действия** панели мониторинга выберите **Предупреждения безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Задайте **период продления активации**, передвинув ползунок или указав значение в текстовом поле (в минутах). Максимальное допустимое значение — 100.
4. Задайте **количество продлений активации** в течение периода продления активации, передвинув ползунок или указав количество продлений в текстовом поле. Максимальное количество продлений — 100.
5. Щелкните **Сохранить**.

### Настройка предупреждения «Избыточные роли администраторов повышают вероятность атаки»
1. В разделе **Действия** панели мониторинга выберите **Предупреждения безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Выберите максимальный срок, в течение которого роль может существовать без активации, передвинув ползунок или указав нужное количество дней в текстовом поле.
4. Нажмите кнопку **Сохранить**.

### Настройка предупреждения «Слишком большое количество глобальных администраторов повышает вероятность атаки»

Для этого предупреждения можно задать два триггера: количество глобальных администраторов превышает максимально допустимое; процент глобальных администраторов в общем количестве администраторов разных типов больше процента, заданного в параметрах.

1. В разделе **Действия** панели мониторинга выберите **Предупреждения безопасности**. Откроется колонка **Активные предупреждения безопасности**.
2. Щелкните **Параметры**.
3. Задайте значение **Минимальное количество глобальных администраторов**, передвинув ползунок или указав количество в текстовом поле.
4. Задайте значение **Процент глобальных администраторов**, передвинув ползунок или указав количество в текстовом поле.

# <a name="the-elements-of-the-b2b-collaboration-invitation-email"></a>Элементы сообщения с приглашением в службу совместной работы B2B

Сообщение с приглашением является важнейшим компонентом, необходимым для подключения партнеров в качестве пользователей службы совместной работы B2B в Azure AD. Основная его цель — повысить уровень доверия получателя, придать сообщению законность и социальную значимость, чтобы получатель был уверен, нажимая кнопку **Начало работы** и принимая приглашение. Это ключевой компонент в процессе налаживания совместной работы. И, конечно же, нам бы хотелось, чтобы это сообщение отлично выглядело!

## <a name="explaining-the-email"></a>Из чего состоит сообщение
Давайте рассмотрим основные элементы сообщения, чтобы понять, как лучше использовать эти возможности.

### <a name="subject"></a>Субъект
Тема сообщения имеет следующий формат: "Вас приглашает организация &lt;имя_клиента&gt;".

### <a name="from-address"></a>Адрес отправителя
Для адреса отправителя используется формат, аналогичный сети LinkedIn. Наша цель — четко указать, кто приглашает и из какой компании, а также пояснить, что сообщение поступило с адреса электронной почты Майкрософт. Получается следующий формат: "&lt;Отображаемое имя приглашающего&gt; из &lt;имя_клиента&gt; (с помощью Майкрософт)<invites@microsoft.com&gt;".

### <a name="reply-to"></a>Ответить
В поле обратного адреса указывается адрес электронной почты приглашающего (если доступен), таким образом, ответ на полученное сообщение будет отправлен приглашающему клиенту.

### <a name="branding"></a>Фирменная символика
В сообщениях с приглашением от клиента может использоваться фирменная символика компании, если ее настроить для клиента. Если вы хотите воспользоваться этой функцией, то ниже описано, как ее настроить. В сообщении будет отображаться баннер с логотипом. Для получения оптимальных результатов следуйте приведенным здесь инструкциям по настройке размера и качества изображения. Кроме того, в призыве к действию также указывается имя компании.

### <a name="call-to-action"></a>Призыв к действию
Призыв к действию состоит из двух частей: первая поясняет, почему получатель получил это сообщение, а во второй получателю предлагается что-то сделать.
- В первой части ("почему") можно обратиться к получателю, используя такой формат: "Организация &lt;имя_клиента&gt; приглашает Вас воспользоваться доступом к приложениям".

- А во второй части ("что предлагается сделать") расположена кнопка **Начало работы**. Если получатель уже добавлен и нет необходимости в отправке приглашения, то эта кнопка не отображается.

### <a name="inviters-information"></a>Сведения о приглашающем
Отображаемое имя приглашающего будет включено в сообщение. Если вы настроили для своей учетной записи Azure AD изображение профиля, то в сообщение с приглашением также будет включено это изображение. Эти элементы призваны повысить уровень доверия получателя к сообщению.
Если приглашающий пользователь еще не настроил изображение профиля, то служба Azure AD создает значок, используя вместо изображения инициалы приглашающего:

  ![отображение инициалов приглашающего](media/active-directory-b2b-invitation-email/inviters-initials.png)

### <a name="body"></a>Текст
Текст сообщения вводится приглашающим через пользовательский интерфейс или передается с помощью API приглашения. Это обычное текстовое поле, которое не обрабатывает HTML-теги по соображениям безопасности.

### <a name="footer-section"></a>Нижний колонтитул

В нижнем колонтитуле содержится фирменная символика корпорации Майкрософт, и получатель будет знать, если для отправки сообщения использовался неотслеживаемый псевдоним. Особые случаи:

- В приглашающем клиенте не указан адрес электронной почты приглашающего

  ![снимок экрана: в приглашающем клиенте не указан адрес электронной почты приглашающего](media/active-directory-b2b-invitation-email/inviter-no-email.png)


- Получателю не нужно активировать приглашение

  ![когда получателю не нужно активировать приглашение](media/active-directory-b2b-invitation-email/when-recipient-doesnt-redeem.png)

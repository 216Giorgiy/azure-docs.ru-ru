# Приступая к работе с Azure AD

###Другие статьи на эту тему
[Что такое Azure AD?](active-directory-whatis.md)<br> [Как это работает?](active-directory-works.md)<br> [Приступая к работе](active-directory-get-started.md)<br> [Дальнейшие действия](active-directory-next-steps.md)<br> [Подробнее](active-directory-learn-map.md)


Готовы начать работу? В Azure Active Directory мы предусмотрели действительно интересные и полезные возможности. Используйте их, чтобы усовершенствовать систему управления идентификацией и доступом в своей организации. Чтобы узнать, как настроить эти функции, ознакомьтесь с документацией. После этого попробуйте поработать с системой, назначив администраторам и пользователям пробные лицензии.


> [AZURE.NOTE]Azure AD всем доступна бесплатно. Зарегистрируйтесь, чтобы оформить пробную подписку Azure, после чего ваш каталог станет вам доступен. Каталог сохраняется за вами, даже если вы не намерены ничего покупать. [Попробуйте поработать с Azure AD](http://azure.microsoft.com/trial/get-started-active-directory/)

##Безопасный доступ в масштабах организации
Обеспечьте защиту данных в облаке и надежный локальный и удаленный доступ, используя многофакторную проверку подлинности. [Дополнительные сведения о многофакторной проверке подлинности](../multi-factor-authentication.md)

##Изменение и сброс паролей
Сократите расходы и число обращений в службу поддержки, позволив пользователям самостоятельно и с любого устройства менять и сбрасывать пароли Active Directory или Azure Active Directory, где бы они ни находились. Администратор может управлять политиками паролей и уведомлений, а также просматривать подробные данные аудита о действиях по операциям сброса паролей в момент их выполнения. [Дополнительные сведения об управлении паролями в Azure Active Directory](active-directory-manage-passwords.md)

##Создание и изменение пользователей
Создайте учетную запись для каждого пользователя, которому будет предоставлен доступ к облачной службе Майкрософт. По умолчанию, пользователи не имеют прав администратора, но вы можете такие права им назначать. [Дополнительные сведения о создании пользователей Azure AD](active-directory-create-users.md).

##Просмотр отчетов о доступе и использовании
Azure AD предоставляет администраторам различные отчеты. В платных выпусках Azure AD поддерживаются дополнительные отчеты, а для создания собственных настраиваемых отчетов предусмотрен отдельный интерфейс API. [Дополнительные сведения об отчетах Azure AD](active-directory-view-access-usage-reports.md).

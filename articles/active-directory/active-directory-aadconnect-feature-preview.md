# Дополнительная информация о функциях в предварительной версии
В этом разделе описывается, как использовать функции, которые сейчас доступны в предварительной версии.

## Обратная запись пользователей
> [AZURE.IMPORTANT]Предварительная версия функции обратной записи пользователей была временно в августовском обновлении для AAD Connect. Если вы включили ее, следует отключить эту функцию.

Обратная запись пользователей находится на этапе ранней предварительной версии. Ее можно использовать только в том случае, если Azure AD является источником для всех объектов-пользователей и локальный каталог Active Directory пуст перед включением этой функции.

> [AZURE.IMPORTANT]Эту функцию следует проверять только в тестовой среде и не следует использовать в каталоге Azure AD, используемом в рабочей среде.

## Обратная запись групп
Обратная запись группы в дополнительных функциях позволит вам выполнять обратную запись «групп Office 365» в лес с установленным Exchange. Это новый тип группы, управление которым всегда происходит в облаке. Его можно найти на сайте outlook.office365.com или myapps.microsoft.com как показано ниже.


![Фильтрация синхронизации](./media/active-directory-aadconnect-feature-preview/office365.png)

![Фильтрация синхронизации](./media/active-directory-aadconnect-feature-preview/myapps.png)

Эта группа будет представлена в виде группы рассылки в локальной службе AD DS. На локальном сервере Exchange должна быть установлена версия Exchange 2013 с накопительным пакетом обновления 8 (выпущен в марте 2015 года), чтобы он мог распознавать этот новый тип группы.

**Примечание**.

- Сейчас, в предварительной версии, атрибут адресной книги не заполняется. Для этого проще всего использовать командлет Exchange PowerShell update-recipient.
- Только леса со схемой Exchange являются допустимыми целевыми объектами для групп. Если схема Exchange не будет обнаружена, обратную запись группы не удастся включить.
- Функция обратной записи группы в настоящее время не обрабатывает группы безопасности или группы рассылки.

Дополнительную информацию о группах Office 365 см. [здесь](http://aka.ms/O365g).

## Расширения каталогов
Расширения каталогов позволяет расширять схему в Azure AD с помощью собственных атрибутов из локального каталога Active Directory.

Поддерживаются только однозначные атрибуты, значения которых не превышают 250 знаков. Метавселенная и схема Azure AD будут расширены за счет выбранных атрибутов. В Azure AD новое приложение добавляется с атрибутами.

Эти атрибуты будут доступны в Graph:

# <a name="add-company-branding-to-sign-in-and-access-panel-pages"></a>Добавление фирменной символики на страницу входа и панели доступа
Многие компании используют единый стиль оформления веб-сайтов и служб, которыми они управляют. Azure Active Directory предоставляет такую возможность, позволяя ИТ-специалистам настраивать внешний вид следующих веб-страниц, добавляя на них корпоративный логотип и изображения.

* **Страница входа.** Эту страницу будут видеть сотрудники и бизнес-гости при выполнении входа в Office 365 или другие веб-приложения, использующие Azure AD.
* **Страница панели доступа.** Панель доступа представляет собой веб-портал, который позволяет просматривать и запускать облачные приложения, к которым вам предоставил доступ администратор Azure AD. Панель доступа доступна здесь: [https://myapps.microsoft.com](https://myapps.microsoft.com).

В этом разделе объясняется, как настроить страницу входа и страницу панели доступа.

> [!NOTE]
> * Фирменная символика компании доступна только после обновления до выпуска Premium или Basic службы Azure Active Directory. Она также доступна пользователям Office 365. Чтобы получить дополнительные сведения, ознакомьтесь с выпусками Azure Active Directory.
> 
> * Выпуски Azure Active Directory Premium и Basic доступны для клиентов в Китае, использующих доступный по всему миру экземпляр Azure Active Directory. Эти выпуски в настоящее время не поддерживает служба Microsoft Azure под управлением 21Vianet в Китае. Чтобы получить дополнительные сведения, свяжитесь с нами через форум Azure Active Directory.


## <a name="customizing-the-sign-in-page"></a>Настройка страницы входа
Пользователи обычно взаимодействуют со страницей входа в Azure AD, когда пытаются получить доступ к облачным приложениям и службам, на которые подписана компания.

Когда вы меняете фирменную символику на странице входа, может потребоваться до часа, чтобы эти изменения вступили в силу.

Элементы фирменной символики компании появляются на странице входа в Azure AD, когда пользователи пытаются получить доступ к URL-адресу конкретного клиента, например https://outlook.com/contoso.com.

Когда пользователи посещают службу, используя универсальный URL-адрес, например www.office.com, на странице входа не отображается фирменная символика, так как пользователь еще не определен системой. Фирменная символика компании отобразится, когда пользователь введет свой идентификатор или выберет плитку пользователя.

> [!NOTE]
> * На классическом портале Azure в разделе **Active Directory** > **Каталог** > **Домены** (где настраивалась фирменная символика) ваше доменное имя должно быть в активном состоянии.
> * Фирменная символика страницы входа не применяется к странице входа для личных учетных записей Майкрософт. Если сотрудники или бизнес-гости выполнят вход с помощью личных учетных записей Майкрософт, фирменная символика компании на этих страницах входа отображаться не будет.
>

На снимках экрана ниже объясняется, как выполняется настройка страниц входа.

### <a name="scenario-1-contoso-employee-goes-to-a-generic-app-url-for-example-wwwofficecom"></a>Сценарий 1. Сотрудник Contoso использует универсальный URL-адрес приложения (например, www.office.com)

В этом примере пользователь Contoso выполняет вход в мобильное приложение или веб-приложение с помощью универсального URL-адреса. Рисунок слева всегда отображает приложение, а панель взаимодействия справа обновляется, чтобы отобразить элементы фирменной символики Contoso в соответствующих случаях.

![Страница входа в службу Office 365 до и после настройки][1]

### <a name="scenario-2-contoso-employee-goes-to-contoso-app-thats-restricted-to-internal-users"></a>Сценарий 2. Выполнение входа сотрудником Contoso в приложение, доступное только для внутренних пользователей

В этом примере пользователь Contoso выполняет вход во внутреннее приложение с помощью URL-адреса компании. На рисунке слева отображается фирменная символика компании (Contoso). Панель взаимодействия справа заблокирована для Contoso и содержит поля для выполнения входа.

![Страница входа в приложение с ограниченным доступом][2]

### <a name="scenario-3-contoso-employee-goes-to-a-contoso-app-thats-open-to-external-users"></a>Сценарий 3. Выполнение входа сотрудником Contoso в приложение, доступное внешним пользователям

В этом примере пользователи выполняют вход в бизнес-приложение из Contoso. Причем это могут быть как сотрудники Contoso, так и сторонние пользователи. На рисунке слева отображается владелец ресурса (Contoso), так же как и в случае \#2, описанном выше. Но в этот раз панель взаимодействия справа не заблокирована для Contoso, поэтому внешние пользователи могут выполнить вход.

![Вход в приложение с открытым доступом][3]

### <a name="scenario-4-fabrikam-business-guest-goes-to-contoso-app-thats-open-to-external-users"></a>Сценарий 4. Бизнес-гость Fabrikam выполняет вход в приложение Contoso, доступное внешним пользователям

В этом примере пользователь Contoso выполняет вход во внутреннее приложение с помощью URL-адреса компании. На рисунке слева отображается фирменная символика компании (Contoso). Панель взаимодействия справа заблокирована для Contoso и содержит поля для выполнения входа.

![Выполнение входа в качестве внешнего пользователя][4]


## <a name="what-elements-on-the-page-can-i-customize"></a>Какие элементы на странице можно настраивать?

На странице входа можно настроить следующие элементы:

![][5]

| Элемент страницы | Расположение на странице |
|:--- | --- |
| Баннер с логотипом | Отображается в правой верхней части страницы. Заменяет логотип приложения после определения организации пользователя (обычно после ввода имени пользователя). |
| Фоновый рисунок | Изображение, отображаемое слева на странице входа. Заменяет изображение приложения для клиентских сценариев входа (когда пользователи выполняют вход в приложение, опубликованное их организацией или же организацией, для которой они являются бизнес-гостями).<br>При подключениях с низкой пропускной способностью фоновый рисунок заменяется цветом фона. Рисунок не отображается на узких экранах, например на экранах телефонов.<br>Когда пользователи изменяют размер браузера, фоновый рисунок обрезается. Чтобы сохранить основные визуальные элементы в исходном состоянии, во время проектирования разместите их в верхнем левом углу. | 
| Флажок "Оставаться в системе" | Отображается под полем **Пароль**. |
| Текст страницы входа | Стандартный текст, отображаемый над нижним колонтитулом страницы. Он может содержать полезные сведения для пользователей, например номер телефона службы поддержки или юридическое заявление. |

> [!NOTE]
> Все элементы необязательные. Например, если указать баннер с логотипом, но не использовать фоновый рисунок, на странице входа будет отображаться логотип и изображение целевого сайта (например, изображение калифорнийской автострады для Office 365).
>

Если на странице входа установить флажок **Оставаться в системе**, пользователь сможет остаться в системе, даже когда он закроет и повторно откроет браузер, что никак не повлияет на время существования сеанса.

Чтобы управлять отображением флажка, вы можете использовать параметр **Скрыть флажок "Оставаться в системе"**.

![Параметр "Скрыть флажок "Оставаться в системе"][6]

Чтобы скрыть флажок, задайте для этого параметра значение **Скрыт**.

> [!NOTE]
> Некоторые функции SharePoint Online и Office 2010 зависят от возможности пользователей установить этот флажок. Если вы скроете этот параметр, пользователи могут получать дополнительные или непредвиденные запросы на вход в систему.
>
>

Вы также можете локализовать все элементы на этой странице. Настроив набор элементов по умолчанию, вы можете настроить дополнительные версии для разных языков. Вы также можете смешивать разные элементы. Вот что вы можете, к примеру, делать:

* Создайте большую иллюстрацию по умолчанию, подходящую для всех языков, а затем создайте отдельные версии для английского и русского языков. Когда вы настроите в браузере эти языки (один или оба), отобразится конкретное изображение. Для всех других языков отобразится иллюстрация по умолчанию.
* Настройте разные логотипы для организации (например, японскую и китайскую версии).

## <a name="configure-your-directory-with-company-branding"></a>Настройка каталога с фирменной символики компании
На классическом портале Azure можно настроить один набор элементов по умолчанию для каждого каталога. Когда элементы по умолчанию будут сохранены, администратор также сможет добавить локализованные версии каждого элемента для разных языков. Все настраиваемые элементы необязательные.

Например, если настроить эмблему баннера, но не использовать большое изображение, в правом верхнем углу страницы входа будет отображаться ваш логотип. При этом будет отображаться изображение сайта по умолчанию.

Представьте себе следующую конфигурацию.

* баннер с логотипом по умолчанию и текст страницы входа на английском языке;
* текст страницы входа на немецком языке.

Если в настройках выбран немецкий, у вас будет баннер с логотипом по умолчанию и текст на немецком языке.

Хотя технически вы можете настроить другой набор для каждого языка, поддерживаемого Azure AD, мы рекомендуем использовать небольшое количество вариантов в целях обслуживания и производительности.
 
**Чтобы добавить фирменную символику в каталог, сделайте следующее:**

1. Войдите на [классический портал Azure](https://manage.windowsazure.com) с учетными данными администратора того каталога, который нужно настроить.
2. Выберите нужный каталог.
3. На панели инструментов в верхней части экрана щелкните **Настройка**.
4. Щелкните **Настройка фирменной символики**.
5. Измените элементы, которые нужно настроить. Все поля здесь необязательные.
6. Щелкните **Сохранить**.

Применение всех изменений фирменного дизайна страницы входа может занять до одного часа.

**Чтобы добавить фирменную символику компании для конкретного языка, сделайте следующее:**

1. Войдите на [классический портал Azure](https://manage.windowsazure.com) с учетными данными администратора того каталога, который нужно настроить.
2. Выберите нужный каталог.
fs3. На панели инструментов в верхней части экрана щелкните **Настройка**.
4. Щелкните **Настройка фирменной символики**.
5. Щелкните **Добавить фирменную символику для определенного языка**.
6. Выберите язык, для которого хотите настроить логотип, и нажмите кнопку **Далее**.
7. Измените только те элементы, для которых требуется настроить переопределение для другого языка. Все поля здесь необязательные. Если оставить поле пустым, для этого элемента будет использовано пользовательское значение по умолчанию (или значение Майкрософт по умолчанию, если пользовательское значение не настроено).
8. Щелкните **Сохранить**.

**Чтобы удалить фирменную символику компании из каталога, сделайте следующее:**

1. Войдите на [классический портал Azure](https://manage.windowsazure.com) с учетными данными администратора того каталога, который нужно настроить.
2. Выберите нужный каталог.
3. На панели инструментов в верхней части экрана щелкните **Настройка**.
4. Щелкните **Настройка фирменной символики**.
5. На странице настройки фирменной символики выберите **Изменить существующие параметры фирменной символики** и перейдите к следующей странице.
6. В зависимости от того, какие элементы необходимо удалить, выполните одно или несколько из следующих действий.

    а. В разделе **Эмблема баннера** выберите **Удалить загруженную эмблему**.

    b. В разделе **Tile Logo** (Эмблема плитки) выберите **Удалить загруженную эмблему**.

    c. Удалите текст из всех текстовых полей.

    d. Нажмите кнопку **Далее**.

    д. Удалите текст из всех текстовых полей.
7. Нажмите кнопку **Сохранить** , чтобы удалить элементы.
8. При необходимости щелкните **Настроить фирменную символику** еще раз и повторите эти действия для всех элементов фирменной символики, которые требуется удалить.
    Если щелкнуть **Настройка фирменной символики** и просмотреть форму **Настройка фирменной символики по умолчанию** без настроенных параметров, все параметры фирменной символики будут удалены.


## <a name="customizable-elements"></a>Настраиваемые элементы
Корпоративные логотипы используются для страниц входа и панели доступа, а остальные элементы — только для страницы входа. В следующей таблице предоставлены подробные сведения о разных настраиваемых элементах.

| Имя | Описание | Ограничения | Рекомендации |
| --- | --- | --- | --- |
| Баннер с логотипом |Баннер с логотипом отображается на странице входа и панели доступа. |<p>JPG или PNG</p><p>60 x 280 пикселей</p><p>10 КБ</p> |<p>Используйте полный логотип организации (включая пиктограмму и эмблему).</p><p>Используйте изображение до 30 пикселей в высоту, чтобы на мобильных устройствах не появлялась полоса прокрутки.</p><p>Размер файла должен быть до 4 КБ.</p><p>Используйте прозрачный PNG-файл (на странице входа не всегда будет белый фон).</p> |
| Значок для плитки | В настоящее время не используется. |<p>JPG или PNG</p><p>120 x 120 пикселей</p><p>10 КБ</p> |<p>Сделайте это изображение простым (без маленького текста), так как оно может изменять размер до 50 %. |
| </p> | | | |
| Метка имени пользователя для входа | В настоящее время не используется. |<p>Текст в формате Юникод, не более 50 символов</p><p>Только обычный текст (без ссылок или HTML-тегов)</p> |<p>Текст должен быть коротким и простым.</p><p>Спросите пользователей, как они обычно называют свои рабочие или учебные учетные записи.</p> |
| Стандартный текст страницы входа |Этот стандартный текст отображается под формой страницы входа и может использоваться для предоставления дополнительных инструкций или сведений о получении справки и поддержки. |<p>Текст в формате Юникод, не более 256 символов</p><p>Только обычный текст (без ссылок или HTML-тегов)</p> |Длина должна быть не более 250 символов (примерно 3 строки текста) |
| Фоновый рисунок страницы входа | Большое изображение, отображаемое в левой области страницы входа (в правом углу для RTL-языков) при использовании пользователями URL-адресов конкретных клиентов. |<p>JPG или PNG</p><p>1420 x 1200 пикселей</p><p>500 КБ</p> |<p>1420 x 1200 пикселей</p><p>Важно: размер должен быть минимальным, оптимально — меньше 200 КБ. Слишком большое изображение, которого нет в кэше, замедлит скорость загрузки страницы входа.</p><p>Это изображение почти всегда обрезается с учетом пропорций экрана. Размещайте основные визуальные элементы в верхнем левом углу.</p> |
| Цвет фона страницы входа | При подключениях с низкой пропускной способностью этот сплошной цвет используется вместо фонового рисунка. | Цвет должен быть указан в формате RGB в виде шестнадцатеричного числа (например, \#FFFFFF). | Рекомендуется выбрать основной цвет баннера с логотипом. |

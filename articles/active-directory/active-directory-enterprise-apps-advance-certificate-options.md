# <a name="advanced-certificate-signing-options-in-the-saml-token-for-gallery-apps-in-azure-active-directory"></a>Расширенные параметры подписи сертификата в маркере SAML для приложений коллекции в Azure Active Directory
На сегодняшний день Azure Active Directory поддерживает тысячи предварительно интегрированных приложений в коллекции приложений Azure AD, включая более 500 приложений, поддерживающих единый вход по протоколу SAML 2.0. Когда пользователь проходит аутентификацию для приложения в Azure AD с помощью SAML, Azure AD отправляет токен в приложение (через запрос HTTP POST). Затем приложение проверяет и использует маркер для входа пользователя вместо запроса имени пользователя и пароля. Эти маркеры SAML подписываются с помощью уникального сертификата, который создается в Azure AD, а также определенных стандартных алгоритмов.

Azure Active Directory использует некоторые параметры по умолчанию для приложений коллекции. Значения по умолчанию настраиваются в зависимости от требований к приложению.

Azure Active Directory поддерживает расширенные параметры подписи сертификата. Чтобы выбрать эти параметры, сначала необходимо установить флажок **Показать дополнительные параметры подписывания сертификатов**, как показано ниже.

![Параметры подписи сертификата][1]

Когда флажок установлен, можно настроить **параметры подписи сертификата** и **алгоритм подписи сертификата**.

## <a name="certificate-signing-options"></a>Параметры подписи сертификата

Ниже приведены три типа параметров подписи сертификата, поддерживаемых Azure AD.

1. **Утверждение знака SAML**. Этот параметр по умолчанию задан для большинства приложений коллекции. Если выбрать этот параметр, Azure AD в качестве поставщика удостоверений подпишет утверждения SAML и сертификат, используя сертификат X509 приложения. Также этот параметр использует алгоритм подписи, который можно выбрать в раскрывающемся списке ниже.

2. **Ответ знака SAML**. Если выбрать этот параметр, Azure AD в качестве поставщика удостоверений подпишет ответ SAML, используя сертификат X509 приложения. Также этот параметр использует алгоритм подписи, который можно выбрать в раскрывающемся списке ниже.

3. **Ответ знака SAML и утверждение**. Если выбрать этот параметр, Azure AD в качестве поставщика удостоверений подпишет полный маркер SAML, используя сертификат X509 приложения. Также этот параметр использует алгоритм подписи, который можно выбрать в раскрывающемся списке ниже.

    ![Параметры подписи сертификата][4]

## <a name="certificate-signing-algorithm"></a>Алгоритм подписи сертификата

Azure Active Directory поддерживает два типа алгоритма подписи для подписи ответа SAML.

1. SHA256. Это алгоритм по умолчанию, который используется Azure Active Directory для подписи ответа SAML. Это последний разработанный алгоритм, который считается безопаснее, чем алгоритм SHA1. Большинство приложений поддерживают алгоритм SHA256. Если приложение поддерживает только алгоритм SHA1, значение по умолчанию можно изменить. В противном случае мы советуем использовать алгоритм SHA256 для подписи ответа SAML.

    ![Алгоритм подписи сертификата SHA256][3]

2. SHA1. Это устаревший алгоритм с сомнительной безопасностью. Если приложение поддерживает только этот алгоритм подписи, его можно выбрать в раскрывающемся списке. С помощью этого алгоритма SHA1 Azure AD подпишет ответ SAML.

    ![Алгоритм подписи сертификата SHA1][2]

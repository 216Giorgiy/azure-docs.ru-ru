# Как выполнить проверку доступа в управлении привилегированными пользователями Azure AD

Компонент управления привилегированными пользователями (PIM) Azure Active Directory (AD) упрощает управление пользователями с привилегированными удостоверениями и их доступом к ресурсам в Azure AD и других веб-службах Майкрософт, таких как Office 365 или Microsoft Intune.

Если вам назначена роль администратора, администратор привилегированных ролей вашей организации может попросить вас регулярно выполнять проверки и подтверждать, что эта роль по-прежнему требуется вам для работы. Вы можете получить сообщение электронной почты, содержащее ссылку для выполнения проверки, или перейти непосредственно на [портал Azure](https://portal.azure.com). Следуйте указаниям в этой статье, чтобы выполнить самостоятельную проверку назначенных вам ролей.

Если вы — администратор привилегированных ролей и вас интересуют функции проверки безопасности, дополнительные сведения см. в разделе [Как запустить проверки безопасности](active-directory-privileged-identity-management-how-to-start-security-review.md).

## Добавление приложения для управления привилегированными пользователями

Для выполнения проверки можно использовать приложение "Управление привилегированными пользователями" (PIM) Azure AD на [портале Azure](https://portal.azure.com/). Если приложение Azure AD PIM еще не установлено на портале, выполните следующие действия, чтобы приступить к работе.

1. Войдите на [портал Azure](https://portal.azure.com/).
2. Если в организации имеется несколько каталогов, щелкните свое имя пользователя в правом верхнем углу портала Azure и выберите каталог, с которым вы будете работать.
3. Последовательно выберите элементы **Создать** > **Безопасность + удостоверения** > **Управление привилегированными пользователями Azure AD**.

	![Включение управления привилегированными пользователями на портале][1]

4. Установите флажок **Закрепить на панели мониторинга** и нажмите кнопку **Создать**. Откроется панель мониторинга управления привилегированными пользователями.


## Утверждение или отклонение доступа

При утверждении или отклонении доступа вы просто сообщаете проверяющему, используете ли вы еще эту роль. Чтобы оставаться в этой роли, выберите **Утвердить**, а когда доступ вам больше не требуется, выберите **Запретить**. Состояние роли изменится лишь тогда, когда проверяющий применит результаты. Выполните следующие действия, чтобы найти проверку доступа и завершить ее.

1. В приложении PIM выберите **Просмотр привилегированного доступа**. Если у вас есть незавершенные проверки доступа, то они отображаются в колонке "Проверки доступа" в Azure AD.
2. Выберите проверку, которую необходимо завершить.
3. Если эта проверка создана не вами, то вы должны быть единственным пользователем, отображаемым в проверке. Установите флажок рядом со своим именем.
4. Выберите **Утвердить** или **Запретить**. Возможно, в текстовом поле **Указание причины** потребуется ввести причину своего решения.
5. Закройте колонку **Проверка ролей Azure AD**.

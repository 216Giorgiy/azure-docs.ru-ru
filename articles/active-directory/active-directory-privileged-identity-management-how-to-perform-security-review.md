# Выполнение проверок безопасности в компоненте управления привилегированными пользователями Azure AD

Компонент управления привилегированными пользователями (PIM) Azure Active Directory (AD) упрощает управление пользователями с привилегированными удостоверениями и их доступом к ресурсам в Azure AD и других веб-службах Майкрософт, таких как Office 365 или Microsoft Intune.

Если вам назначена роль администратора, администратор безопасности вашей организации может попросить вас регулярно выполнять проверки и подтверждать, что эта роль по-прежнему требуется вам для работы. Вы можете получить сообщение электронной почты, содержащее ссылку для выполнения проверки, или перейти непосредственно на [портал Azure](https://portal.azure.com). Следуйте указаниям в этой статье, чтобы выполнить самостоятельную проверку назначенных вам ролей.

Если вы — администратор безопасности, и вас интересуют функции проверки безопасности, дополнительные сведения см. в разделе [Запуск проверки безопасности](active-directory-privileged-identity-management-how-to-start-security-review.md).

## Добавление приложения для управления привилегированными пользователями

Для выполнения проверки можно использовать приложение "Управление привилегированными пользователями" (PIM) Azure AD на [портале Azure](https://portal.azure.com/). Если приложение Azure AD PIM еще не установлено на портале, выполните следующие действия, чтобы приступить к работе.

1. Войдите на [портал Azure](https://portal.azure.com/), если вы еще этого не сделали.
2. Если в организации имеется несколько каталогов, щелкните свое имя пользователя в правом верхнем углу портала Azure и выберите каталог, с которым вы будете работать.
3. Щелкните значок **Создать** в области навигации слева.
4. В меню выберите пункт **Безопасность + удостоверения**.
5. Выберите **Управление привилегированными пользователями Azure AD**.
6. Оставьте флажок **Закрепить на панели мониторинга** установленным и нажмите кнопку **Создать**. Откроется приложение "Управление привилегированными пользователями".


## Утверждение или отклонение доступа

Доступ не будет изменен до [завершения проверки](active-directory-privileged-identity-management-how-to-complete-review.md). В результате этого процесса будет просто создан контрольный список для тех, кто будет изменять права доступа для роли. Если выбрать хотя бы одного пользователя, будут включены кнопки **Утвердить доступ** и **Запретить доступ**.

1. В приложении PIM выберите пункт меню **Проверка административного доступа**. Отобразится список проверок безопасности.
2. В этом списке выберите **пользователей**, для которых необходимо изменить права доступа.
3. Нажмите кнопку **Утвердить доступ** или **Запретить доступ** для выбранных вами пользователей. В главном меню портала Azure появится уведомление, и выбранные имена в списке проверки исчезнут (их можно снова вывести на экран, изменив параметры фильтрации). Закройте колонку **Проверка ролей Azure AD**.

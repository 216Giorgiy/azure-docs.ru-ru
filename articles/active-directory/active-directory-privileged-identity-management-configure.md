# Azure AD Privileged Identity Management

Azure AD Privileged Identity Management позволяет осуществлять мониторинг и управление пользователями с привилегированными удостоверениями и их доступом к ресурсам в Azure AD и других веб-службах Майкрософт, таких как Office 365 или Microsoft Intune.

Чтобы пользователи могли выполнять привилегированные операции, организациям зачастую необходимо предоставлять многим из своих пользователей постоянный привилегированный доступ к Azure AD, ресурсам Azure или Office 365, или другим приложениям SaaS. При этом для многих пользователей возрастает угроза безопасности ресурсов, размещенных в облаке, так как нельзя провести мониторинг того, что делают пользователи с привилегиями администратора, на должном уровне. Кроме того, скомпрометированная учетная запись с привилегированным доступом может повлиять на общую безопасность облака. Azure AD Privileged Identity Management помогает устранить этот риск.

В этой предварительной версии Azure AD Privileged Identity Management позволяет:

- узнать, кто из пользователей являются администраторами Azure AD;
- обеспечить своевременный административный доступ по запросу к ресурсам в каталоге;
- получать отчеты по данным журнала доступа администраторов и изменениям в назначениях администраторов; 
- получать оповещения о доступе к привилегированным ролям; 

В этой предварительной версии Azure AD Privileged Identity Management позволяет управлять встроенными ролями Azure Active Directory в организации:

- глобального администратора; 
- администратора выставления счетов; 
- администратора служб;  
- администратора пользователей; 
- администратора паролей. 

## Своевременный доступ к правам администраторов 

Сложилось так, что пользователям можно назначать роль администратора на портале управления Azure или в Windows PowerShell. В результате пользователь становится **постоянным администратором**, который всегда активен в назначенной ему роли. В этой предварительной версии добавлена поддержка **временных администраторов**, т.е. пользователей, которым требуется выполнять активацию для получения назначенной им роли. В процессе активации происходит изменение неактивного назначения пользователя роли в Azure AD на активное.

## Включение Privileged Identity Management для каталога

Использование Azure AD Privileged Identity Management можно начать со входа на [портал Microsoft Azure](https://portal.azure.com/). Теперь Azure AD Privileged Identity Management отображается только на портале Microsoft Azure. Для включения Azure AD Privileged Identity Management для каталога нужно быть глобальным администратором.

![][1]

После инициализации этого расширения вы автоматически станете первым **администратором безопасности** каталога. Доступ к этому расширению для управления правами доступа других администраторов есть только у администратора безопасности. Во время инициализации на портале предварительной версии Azure на начальную панель будет добавлена плитка Azure AD Privileged Identity Management.

## Панель мониторинга Azure AD Privileged Identity Management 

Диспетчер привилегированных удостоверений Azure AD предоставляет панель мониторинга, на которой отображается такая важная информация, как:

- количество пользователей, которые назначены в каждую привилегированную роль;  
- количество временных и постоянных администраторов; 
- журнал доступа администраторов. 

![][2]

## Управление привилегированными ролями 

С помощью Azure AD Privileged Identity Management вы можете управлять учетными записями администраторов, добавляя или удаляя постоянных и временных администраторов для каждой роли.

![][3]

## Настройка параметров активации ролей 

Параметры активации ролей позволяют настраивать свойства активации временных ролей, в том числе:

- длительность периода активации роли;
- уведомления об активации роли; 
- данные, которые нужно указать пользователю во время активации роли.  

![][4]

## Активация роли  

Чтобы активировать роль, временному администратору необходимо отправить запрос на активацию роли с привязкой по времени. Запрос на активацию можно отправить, используя параметр **Активировать мою роль** в Azure AD Privileged Identity Management.

Администратору, которому нужно активировать роль, необходимо инициализировать Azure AD Privileged Identity Management на портале предварительной версии Azure.

Использовать Azure AD Privileged Identity Management для активации своей роли может администратор любого типа.
 
Активация роли привязана по времени. В параметрах активации роли можно настроить продолжительность активации, а также необходимые данные, которые администратор должен предоставить для активации роли.

![][5]

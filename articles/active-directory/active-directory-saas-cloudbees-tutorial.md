#Руководство. Интеграция Azure Active Directory с CloudBees
>[AZURE.TIP]Чтобы оставить отзыв, нажмите [здесь](http://go.microsoft.com/fwlink/?LinkId=528737).

Цель данного руководства — показать интеграцию Azure и CloudBees. Сценарий, описанный в этом учебнике, предполагает, что у вас уже имеется:

-   Действующая подписка на Azure
-   Клиент CloudBees

После завершения этого руководства пользователи Azure AD, назначенные CloudBees, будут иметь возможность единого входа в приложение на веб-сайте компании CloudBees (вход, инициированный поставщиком услуг) или с помощью инструкций из статьи [Общие сведения о панели доступа](https://msdn.microsoft.com/library/azure/dn308586.aspx).

Сценарий, описанный в этом учебнике, состоит из следующих блоков:

1.  Включение интеграции приложений для CloudBees
2.  Настройка единого входа
3.  Настройка подготовки учетных записей пользователей
4.  Назначение пользователей

![Сценарий](./media/active-directory-saas-cloudbees-tutorial/IC790415.png "Сценарий")

##Включение интеграции приложений для CloudBees

В этом разделе показано, как включить интеграцию приложений для CloudBees.

###Чтобы включить интеграцию приложений для CloudBees, выполните следующие действия.

1.  На портале управления Azure в левой области навигации нажмите **Active Directory**.

    ![Active Directory](./media/active-directory-saas-cloudbees-tutorial/IC700993.png "Active Directory")

2.  Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3.  Чтобы открыть представление приложений, в представлении каталога нажмите **Приложения** в верхнем меню.

    ![Приложения](./media/active-directory-saas-cloudbees-tutorial/IC700994.png "Приложения")

4.  В нижней части страницы нажмите кнопку **Добавить**.

    ![Добавление приложения](./media/active-directory-saas-cloudbees-tutorial/IC749321.png "Добавление приложения")

5.  В диалоговом окне **Что необходимо сделать?** нажмите **Добавить приложение из коллекции**.

    ![Добавить приложение из коллекции](./media/active-directory-saas-cloudbees-tutorial/IC749322.png "Добавить приложение из коллекции")

6.  В **поле поиска** введите **CloudBees**.

    ![Коллекция приложений](./media/active-directory-saas-cloudbees-tutorial/IC790416.png "Коллекция приложений")

7.  В области результатов выберите **CloudBees** и нажмите кнопку **Завершить**, чтобы добавить приложение.

    ![Cloudbees](./media/active-directory-saas-cloudbees-tutorial/IC790417.png "Cloudbees")

##Настройка единого входа

В этом разделе показано, как разрешить пользователям проходить проверку подлинности в CloudBees со своей учетной записью Azure AD, используя федерацию на основе протокола SAML. Чтобы настроить единый вход для CloudBees, необходимо извлечь значение отпечатка из сертификата. Если вы не знакомы с этой процедурой, посмотрите видео [Как извлечь значение отпечатка из сертификата](http://youtu.be/YKQF266SAxI).

###Чтобы настроить единый вход, выполните следующие действия.

1.  На странице интеграции с приложением **CloudBees** портала Azure AD нажмите **Настройка единого входа**, чтобы открыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-cloudbees-tutorial/IC790418.png "Настройка единого входа")

2.  На странице **Как пользователи должны входить в CloudBees?** выберите **Единый вход Microsoft Azure AD** и нажмите кнопку **Далее**.

    ![Настройка единого входа](./media/active-directory-saas-cloudbees-tutorial/IC790419.png "Настройка единого входа")

3.  На странице **Настройка URL-адреса приложения** в текстовом поле **URL-адрес входа в CloudBees** введите URL-адрес, используемый вашими пользователями для входа в CloudBees (например, **https://grandcentral.cloudbees.com/login*") и нажмите кнопку **Далее**.

    ![Настройка URL-адреса приложения](./media/active-directory-saas-cloudbees-tutorial/IC790420.png "Настройка URL-адреса приложения")

4.  На странице **Настройка единого входа в CloudBees** нажмите кнопку **Загрузить сертификат**, а затем сохраните файл сертификата локально на компьютере.

    ![Настройка единого входа](./media/active-directory-saas-cloudbees-tutorial/IC790421.png "Настройка единого входа")

5.  В другом окне браузера войдите на свой корпоративный веб-сайт CloudBees в качестве администратора.

6.  В верхнем меню нажмите пункт **Параметры**.

    ![данных](./media/active-directory-saas-cloudbees-tutorial/IC790422.png "данных")

7.  Откройте вкладку **Интеграция единого входа**.

8.  В разделе **Настройка единого входа SAML 2.0** выполните следующие действия.

    ![Настройка единого входа](./media/active-directory-saas-cloudbees-tutorial/IC790423.png "Настройка единого входа")

    1.  На странице **Настройка единого входа в CloudBees** портала Azure AD скопируйте значение поля **URL-адрес удаленного входа** и вставьте его в текстовое поле **URL-адрес удаленного входа**.
    2.  Скопируйте значение поля **Отпечаток** из экспортированного сертификата и вставьте его в текстовое поле **Отпечаток существующего сертификата**.
    
        >[AZURE.TIP]Дополнительные сведения можно найти в видео [Как получить значение отпечатка сертификата](http://youtu.be/YKQF266SAxI).

    3.  В текстовом поле **Домены проверки подлинности** введите домен вашей компании.
    4.  Щелкните **Сохранить**.

9.  На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.

    ![Настройка единого входа](./media/active-directory-saas-cloudbees-tutorial/IC790424.png "Настройка единого входа")

##Настройка подготовки учетных записей пользователей

Чтобы пользователи Azure AD могли выполнить вход в **CloudBees**, они должны быть подготовлены для **CloudBees**. В случае с **CloudBees** подготовка выполняется вручную.

###Чтобы подготовить учетные записи пользователей для CloudBees, выполните следующие действия.

1.  Выполните вход на веб-сайт компании **CloudBees** в качестве администратора.

2.  Выберите **Изменить пользователей**.

    ![Пользователи](./media/active-directory-saas-cloudbees-tutorial/IC790429.png "Пользователи")

3.  Щелкните **Добавить**.

    ![Добавить](./media/active-directory-saas-cloudbees-tutorial/IC790430.png "Добавить")

4.  В разделе **Добавить пользователя** выполните следующие действия.

    ![Добавить пользователя](./media/active-directory-saas-cloudbees-tutorial/IC790431.png "Добавить пользователя")

    1.  Введите значения **Имя**, **Фамилия**, **Адрес электронной почты** и другие атрибуты действующей учетной записи Azure Active Directory, которую вы хотите подготовить, в соответствующих текстовых полях.
    2.  Нажмите кнопку **Добавить пользователя**.

        >[AZURE.NOTE]Владельцу учетной записи направляется сообщение электронной почты с инструкциями по входу.

>[AZURE.NOTE]Вы можете использовать любые другие инструменты создания учетной записи пользователя CloudBees или API, предоставляемые CloudBees для подготовки учетных записей пользователя AAD.

##Назначение пользователей

Чтобы проверить свою конфигурацию, предоставьте пользователям Azure AD, которые должны использовать приложение, доступ путем их назначения.

###Чтобы назначить пользователей CloudBees, выполните следующие действия.

1.  На портале Azure AD создайте тестовую учетную запись.

2.  На странице интеграции с приложением **CloudBees** нажмите **Назначить пользователей**.

    ![Назначить пользователей](./media/active-directory-saas-cloudbees-tutorial/IC790432.png "Назначить пользователей")

3.  Выберите тестового пользователя, нажмите кнопку **Назначить**, а затем — **Да** для подтверждения назначения.

    ![Да](./media/active-directory-saas-cloudbees-tutorial/IC767830.png "Да")

Если вы хотите проверить параметры единого входа, откройте панель доступа. Дополнительные сведения о панели доступа см. в статье [Общие сведения о панели доступа](https://msdn.microsoft.com/library/azure/dn308586.aspx).

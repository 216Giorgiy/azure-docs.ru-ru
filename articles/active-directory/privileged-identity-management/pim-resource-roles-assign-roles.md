# <a name="assign-azure-resource-roles-in-pim"></a>Назначение ролей ресурсам Azure в PIM

Azure AD PIM может управлять встроенными ролями ресурсов Azure, а также настраиваемыми ролями, в том числе (но не ограничиваясь ими):

- Владелец.
- Администратор доступа пользователей
- участник;
- администратор безопасности;
- Диспетчер безопасности и другие.

>[!NOTE]
Пользователи или члены группы, которым назначены роли владельца или администратора доступа пользователей, и глобальные администраторы, которые поддерживают управление подпиской в ​​Azure AD, являются администраторами ресурсов. Эти администраторы могут назначать роли, настраивать параметры роли и проверять доступ с помощью PIM для ресурсов Azure. Просмотрите список [встроенных ролей ресурсов Azure](../../role-based-access-control/built-in-roles.md).

## <a name="assign-roles"></a>Назначение ролей

Чтобы назначить роли пользователям или группам во время просмотра панели **Роли**, выберите роль, а затем щелкните **Добавить пользователя**. 

![Панель "Роли" с кнопкой "Добавить пользователя"](media/azure-pim-resource-rbac/rbac-assign-roles-1.png)

Можно также нажать кнопку **Добавить пользователя** на панели **Члены**.

![Панель "Члены" с кнопкой "Добавить пользователя"](media/azure-pim-resource-rbac/rbac-assign-roles-2.png)


При добавлении пользователя или группы с панели **Члены** выполните следующие действия. 

1. Перед выбором пользователя или группы выберите роль на панели **Выбор роли**.

   ![Панель "Выбор роли"](media/azure-pim-resource-rbac/rbac-assign-roles-select-role.png)

2. Выберите пользователя или группу из каталога.

3. В раскрывающемся меню выберите соответствующий тип назначения. 

   - **JIT.** Предоставляет пользователю или членам группы допустимый, но не постоянный доступ к роли в течение определенного периода времени или неограниченный (если настроено в параметрах ролей). 
   - **Прямое.** Пользователю или членам группы не требуется активировать назначение роли (известное как постоянный доступ). Прямое назначение рекомендуется использовать для краткосрочного применения, когда при завершении задачи не потребуется предоставлять доступ. Примерами являются дежурные смены и действия, зависящие от времени.

4. Если назначение должно быть постоянным (на постоянной основе можно активировать JIT-назначение или постоянно действующее для прямого назначения), установите флажок под полем **Тип назначения**.

   ![Панель "Параметры членства" с полем "Тип назначения" и соответствующим флажком](media/azure-pim-resource-rbac/rbac-assign-roles-settings.png)

   >[!NOTE]
   >Флажок может быть неактивным, если другой администратор указал максимальную продолжительность назначения для каждого типа назначения в параметрах роли.

   Чтобы указать конкретную продолжительность назначения, снимите флажок и измените поля даты и времени начала и окончания.

   ![Панель "Параметры членства" с полями даты начала, времени начала, даты окончания и времени окончания](media/azure-pim-resource-rbac/rbac-assign-roles-duration.png)


## <a name="manage-role-assignments"></a>Управление назначением ролей

Администраторы могут управлять назначениями ролей, выбирая на левой панели либо вкладку **Роли**, либо **Члены**. На вкладке **Роли** администраторы могут назначить задачи управления определенной роли. На вкладке **Члены** отображаются все назначения ролей пользователей и групп для ресурса.

![Панель "Роли"](media/azure-pim-resource-rbac/rbac-assign-roles-roles.png)

![Панель "Члены"](media/azure-pim-resource-rbac/rbac-assign-roles-members.png)

>[!NOTE]
Если у вас есть роль, ожидающая активации, при просмотре членства в верхней части панели отображается баннер с уведомлением.


## <a name="modify-existing-assignments"></a>Изменение существующих назначений

Чтобы изменить существующие назначения из подробного представления пользователя или группы, щелкните **Изменить параметры** на панели действий. Измените тип назначения на **JIT** или **Прямое**.

![Панель "Сведения о пользователях" с кнопкой "Изменить параметры"](media/azure-pim-resource-rbac/rbac-assign-role-manage.png)

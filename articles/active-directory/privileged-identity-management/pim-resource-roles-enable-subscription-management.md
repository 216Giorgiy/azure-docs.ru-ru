# <a name="enable-subscription-management"></a>Включение управления подписками

Глобальный администратор каталога может не иметь доступа по умолчанию ко всем ресурсам подписки в клиенте. В этой статье описана процедура для получения доступа ко всем подпискам в клиенте и рекомендованный подход к обеспечению соответствия любым средствам безопасности, требуемых в организации после получения доступа.

## <a name="who-can-enable-management-of-subscriptions-in-my-directory"></a>Кто может включать управление подписками в каталоге?

Чтобы включить управление подпиской, каждый пользователь с ролью глобального администратора должен выполнить приведенные ниже шаги. Когда вы активируете управление подписками для себя, можно добавить других глобальных администраторов, которым также может требоваться доступ к ресурсам. Параметр каталога, который включает доступ для всех участников с ролью глобальный администратор, отсутствует.

## <a name="log-on-to-the-azure-portal"></a>Вход на портал Azure

Для начала войдите на портал Azure с учетной записью подходящего или активного участника роли глобального администратора. Если учетная запись принадлежит подходящему глобальному администратору, то роль надо активировать, перед тем как перейди к следующему этапу.

## <a name="access-the-azure-ad-admin-center"></a>Доступ к центру администрирования Azure AD

После входа на портал Azure в качестве глобального администратора можно изменять параметры, чтобы получить доступ к подпискам Azure. Перейдите в центр администрирования Azure AD и в левой области навигации найдите и выберите вкладку "Свойства".

![](media/azure-pim-resource-rbac/aad_properties.png)

В списке свойств переведите параметр "Глобальный администратор может управлять подписками Azure" в положение "Да".

![](media/azure-pim-resource-rbac/aad_properties_save.png)

## <a name="navigate-to-azure-ad-pim"></a>Переход к Azure AD PIM

Когда вы включите этот параметр, ваша учетная запись будет автоматически добавлена к роли "Администратор доступа пользователей" для каждого ресурса подписки в клиенте. Далее перейдите к управлению привилегированными пользователями Azure AD и выберите ресурсы Azure в разделе "Управление" в левой области навигации.

![](media/azure-pim-resource-rbac/aadpim_manage_azure_resources.png)

## <a name="manage-and-discover-resources"></a>Управление и обнаружение ресурсов

Если организация уже использует Azure AD PIM для защити администраторов в Azure Active Directory, то при загрузке колонки будет отображен список подписок.

![](media/azure-pim-resource-rbac/aadpim_manage_azure_resource_some_there.png)

> [!NOTE]
> Если подписки не отображаются, необходимо убедиться, что:
>- срок действия роли глобального администратора не истек; 
>- у организации есть подписка Azure.

![](media/azure-pim-resource-rbac/aadpim_rbac_empty_resource_list.png)

## <a name="configure-assignments"></a>Настройка назначений

Выберите подписку из списка и назначьте себя (или группу, участником которой вы являетесь) в качестве соответствующего владельца ресурса. 
См. [дополнительные сведения о назначении ролей ](pim-resource-roles-assign-roles.md).

Перед тем как перейти к следующему этапу, повторите эту процедуру для каждого ресурса.

## <a name="clean-up-standing-access"></a>Удаление постоянного доступа

Настроив доступ к самым важным подпискам в организации, постоянный доступ можно удалить путем отключения соответствующего параметра в свойствах каталога.

![](media/azure-pim-resource-rbac/aad_properties_no.png)

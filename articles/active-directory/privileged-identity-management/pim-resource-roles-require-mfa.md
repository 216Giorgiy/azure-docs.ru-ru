# <a name="enforce-azure-multi-factor-authentication-in-azure-resources-by-using-privileged-identity-management"></a>Принудительное применение многофакторной аутентификации ресурсов Azure с помощью управления привилегированными пользователями

Технология управления привилегированными пользователями для ролей ресурсов Azure позволяет администраторам ресурсов и удостоверений применять ограничение членства по времени и JIT-доступ для защиты критически важных элементов инфраструктуры Azure. Кроме того, управление привилегированными пользователями поддерживает принудительное применение Многофакторной идентификации Azure (MFA) для двух сценариев.

## <a name="require-multi-factor-authentication-to-activate"></a>Требование прохождения Многофакторной идентификации при активации

Администратор ресурса может потребовать, чтобы участники роли проходили Многофакторную идентификацию Azure для активации роли. Так они смогут гарантировать, что активацию запрашивают только допустимые пользователи. Этот параметр защитит критически важные ресурсы даже при компрометации учетной записи пользователя. 

Чтобы применить это требование, выберите нужный ресурс из списка управляемых ресурсов. На [обзорной панели мониторинга](pim-resource-roles-overview-dashboards.md) выберите роль в списке ролей в нижней правой части экрана.

Кроме того, к параметрам роли можно перейти из вкладок **Роли** или **Параметры роли** на панели слева.

>[!Note]
>Если эти параметры в области слева неактивны, а в верхней части страницы отображается баннер "You have eligible roles that can be activated" (У вас есть роли, допустимые для активации), значит, вы неактивный администратор. Это значит, вашу роль администратора необходимо [активировать](pim-resource-roles-activate-your-roles.md).

![Вкладки "Роли" и "Параметры роли" ](media/azure-pim-resource-rbac/aadpim_rbac_manage_a_role_v2.png)

Чтобы просмотреть членство роли, выберите **Параметры роли** из строки в верхней части экрана, чтобы открыть **Подробные параметры роли**.

Нажмите кнопку **Изменить** вверху, чтобы изменить параметры роли.

В разделе **Активировать** установите флажок **Require Multi-Factor Authentication on activation** (Требовать для активации Многофакторную идентификацию). Затем нажмите кнопку **Save** (Сохранить).

![Требовать для активации Многофакторную идентификацию](media/azure-pim-resource-rbac/aadpim_rbac_require_mfa.png)

## <a name="require-multi-factor-authentication-on-assignment"></a>Требование прохождения Многофакторной идентификации при присвоении

В некоторых случаях администратору ресурса нужно назначить роль определенному участнику на короткий период времени (например, один день). В этом случае им не нужно, чтобы назначенный участник запросил активацию. В таком режиме управление привилегированными пользователями не может принудительно применять Многофакторную идентификацию при использовании роли, так как этот участник является активным с момента назначения.

Тогда вы можете применить Многофакторную идентификацию к самому назначению, чтобы проверить личность администратора, выполняющего это назначение.

Для этого на том же экране подробных параметров роли установите флажок **Require Multi-Factor Authentication on direct assignment** (Требовать Многофакторную идентификацию для прямого назначения).

![Требовать Многофакторную идентификацию для прямого назначения](media/azure-pim-resource-rbac/aadpim_rbac_require_mfa_on_assignment.png)

# <a name="privileged-identity-management---resource-roles---mfa"></a>Управление привилегированными пользователями — роли ресурсов — Многофакторная идентификация

Управление привилегированными пользователями для ролей ресурсов Azure позволяет администраторам ресурсов и удостоверений применять ограничение членства по времени и JIT-доступ для защиты критически важных элементов инфраструктуры Azure. Кроме того, управление привилегированными пользователями поддерживает принудительное применение Многофакторной идентификации Azure (MFA) для двух сценариев.

## <a name="require-mfa-to-activate"></a>Применение MFA для активации

Администратор ресурса может потребовать, чтобы участники роли проходили Многофакторную идентификацию Azure для активации роли. Так они смогут гарантировать, что активацию запрашивают только допустимые пользователи. Этот параметр защитит критически важные ресурсы даже при компрометации учетной записи пользователя. 

Чтобы применить это требование, выберите нужный ресурс из списка управляемых ресурсов. На [обзорной панели мониторинга](pim-resource-roles-overview-dashboards.md) выберите роль в списке ролей в правом нижнем углу экрана.

Кроме того, к параметрам роли можно перейти из вкладок "Роли" или "Параметры роли" в меню навигации слева.

>[!Note]
>Если эти параметры в меню навигации слева неактивны, а в верхней части страницы отображается баннер "You have eligible roles that can be activated" (У вас есть роли, допустимые для активации), значит вашу роль администратора необходимо [активировать](pim-resource-roles-activate-your-roles.md), чтобы выполнить эту процедуру.

![](media/azure-pim-resource-rbac/aadpim_rbac_manage_a_role_v2.png)

В отображении членства в роли выберите команду "Параметры роли" из строки в верхней части экрана, чтобы открыть "Подробные параметры роли".

Нажмите кнопку **Изменить** вверху чтобы изменить параметры роли.

В разделе **Активировать** установите флажок **Требовать для активации Многофакторную идентификацию**, затем щелкните "Сохранить".

![](media/azure-pim-resource-rbac/aadpim_rbac_require_mfa.png)

## <a name="require-mfa-on-assignment"></a>Требование Многофакторной идентификации для назначения

В некоторых случаях администратору ресурса нужно назначить роль некоторому члену на короткий период времени (например, один день) без дополнительных запросов активации. В таком режиме управление привилегированными пользователями не может принудительно применять Многофакторную идентификацию при использовании роли, так как этот член является активным с момента назначения.

Тогда вы можете применить Многофакторную идентификацию к самому назначению, чтобы проверить личность администратора, выполняющего это назначение.

Для этого на том же экране подробных параметров роли установите флажок "Требовать Многофакторную идентификацию для назначения".

![](media/azure-pim-resource-rbac/aadpim_rbac_require_mfa_on_assignment.png)

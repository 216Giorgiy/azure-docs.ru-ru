# <a name="configure-azure-resource-role-settings-in-pim"></a>Настройка параметров роли ресурсов Azure в PIM

При настройке параметров роли вы определяете параметры по умолчанию, применяемые к назначениям, в среде управления привилегированными пользователями (PIM). Чтобы определить параметры для вашего ресурса, выберите вкладку **Параметры роли** на левой панели. Вы также можете нажать кнопку параметров роли на панели действий (в любой роли), чтобы просмотреть текущие параметры.

## <a name="overview"></a>Обзор

С помощью рабочего процесса утверждения в приложении "Управление привилегированными пользователями" (PIM) для ролей ресурсов Azure администраторы могут дополнительно защитить критически важные ресурсы или ограничить доступ к ним. С этой целью они могут затребовать утверждение для активации назначения ролей. 

Для ролей ресурсов Azure существует уникальная концепция иерархии ресурсов. Благодаря этой иерархии все нижестоящие дочерние ресурсы в родительском контейнере могут наследовать назначения ролей от объекта родительского ресурса. 

Например, администратор ресурсов Кирилл использует PIM, чтобы назначить Алису в качестве правомочного участника роли "Владелец" в подписке Contoso. После применения такого назначения Алиса становится владельцем всех контейнеров группы ресурсов в рамках подписки Contoso. Она также является владельцем всех ресурсов (например, виртуальных машин), содержащихся во всех группах ресурсов подписки. 

Предположим, что в подписке Contoso есть три группы ресурсов: Fabrikam Test, Fabrikam Dev и Fabrikam Prod. Каждая из этих групп ресурсов содержит одну виртуальную машину.

Параметры PIM настраиваются для каждой роли ресурса. В отличие от назначений, эти параметры не наследуются и применяются строго к роли ресурса. [Ознакомьтесь с дополнительными сведениями о допустимых назначениях и видимости ресурсов](pim-resource-roles-eligible-visibility.md).

В примере выше Кирилл использует PIM, чтобы потребовать от всех участников в роли "Владелец" подписки Contoso отправки запроса на утверждение активации. Чтобы защитить ресурсы в группе ресурсов Fabrikam Prod, Кирилл также требует утверждения участников роли "Владелец" этого ресурса. Роли владельца в Fabrikam Test и Fabrikam Dev не требуют утверждения для активации.

Когда Алиса отправляет запрос на активацию своей роли "Владелец" в подписке Contoso, утверждающий должен одобрить или отклонить этот запрос. Если Алиса решит [расширить область активации](pim-resource-roles-activate-your-roles.md#apply-just-enough-administration-practices) на группу ресурсов Fabrikam Prod, утверждающий также должен утвердить или отклонить этот запрос. Однако, если Алиса решит расширить область активации на группу Fabrikam Test, Fabrikam Dev или обе группы, утверждение не требуется.

Рабочий процесс утверждения может не понадобиться для всех участников роли. Предположим, ваша компания привлекла нескольких консультантов для помощи в разработке приложения, которое будет выполняться в подписке Azure. Вы как администратор ресурсов хотите, чтобы сотрудники получали доступ без обязательного утверждения, но партнеры по контракту должны отправлять запрос на утверждение. Чтобы настроить рабочий процесс утверждения только для партнеров по контракту, вы можете создать настраиваемую роль с теми же разрешениями, что и для роли, назначенной сотрудникам. Вы можете запросить утверждение для активации этой настраиваемой роли. [Дополнительные сведения о настраиваемых ролях](pim-resource-roles-custom-role-policy.md).

Чтобы настроить рабочий процесс утверждения и указать, кто может утверждать или отклонять запросы, выполните описанные ниже процедуры.

## <a name="require-approval-to-activate"></a>Настройка требования утверждения для активации

1. Перейдите в PIM на портале Azure и выберите ресурс из списка.

   ![Область "Ресурсы Azure" с выбранным ресурсом](media/azure-pim-resource-rbac/aadpim_manage_azure_resource_some_there.png)

2. В левой области выберите **Параметры роли**.

3. Найдите и выберите нужную роль, а затем нажмите кнопку **Изменить**, чтобы изменить параметры.

   ![Кнопка "Изменить" для роли оператора](media/azure-pim-resource-rbac/aadpim_rbac_role_settings_view_settings.png)

4. В разделе **Активация** установите флажок **Требовать утверждения для активации**.

   ![Раздел "Активация" параметров роли](media/azure-pim-resource-rbac/aadpim_rbac_settings_require_approval_checkbox.png)

## <a name="specify-approvers"></a>Указание утверждающих

Щелкните **Выбор утверждающих лиц**, чтобы открыть область **Выберите пользователя или группу**.

>[!NOTE]
>Вы должны выбрать хотя бы одного пользователя или группу для обновления настроек. Утверждающих по умолчанию нет.

Администраторы ресурсов могут добавлять любое сочетание пользователей и групп в список утверждающих. 

![Область "Выберите пользователя или группу" с выбранным пользователем](media/azure-pim-resource-rbac/aadpim_rbac_role_settings_select_approvers.png)

## <a name="request-approval-to-activate"></a>Запрос утверждения активации

Отправка запроса на утверждение не влияет на процедуру, которую участник должен выполнить для активации. [Просмотрите шаги, чтобы активировать роль](pim-resource-roles-activate-your-roles.md).

Если участник запросил активацию роли, требующей утверждения, и эта роль больше не требуется, участник может отменить свой запрос в PIM.

Чтобы отменить запрос, перейдите в PIM и выберите **Мои запросы**. Найдите запрос и щелкните **Отмена**.

![Область "Мои запросы"](media/azure-pim-resource-rbac/aadpim_rbac_role_approval_request_pending.png)

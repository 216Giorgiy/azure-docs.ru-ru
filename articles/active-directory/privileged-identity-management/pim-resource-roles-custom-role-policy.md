# <a name="use-custom-roles-to-target-pim-settings"></a>Использование настраиваемых ролей для настройки параметров управления привилегированными пользователями

Иногда для некоторых участников роли нужно применить довольно строгие параметры управления привилегированными пользователями (PIM), сохраняя высокий уровень автономности для других участников этой же роли. Предположим, ваша компания привлекла нескольких консультантов для помощи в разработке приложения, которое будет выполняться в подписке Azure. 

Вам как администратору ресурсов было бы удобно контролировать запросы сторонних консультантов, но чтобы при этом сотрудники организации получали доступ без дополнительных подтверждений. Выполните описанную ниже процедуру, чтобы настроить целевые параметры PIM для ролей ресурсов Azure.

## <a name="create-the-custom-role"></a>Создание настраиваемой роли

[Чтобы создать для ресурса настраиваемую роль, выполните инструкции в этом руководстве](../role-based-access-control-custom-roles.md).

Присвойте этой роли описательное имя, чтобы легко определять, какие из встроенных ролей нужно дублировать.

>[!NOTE]
>Следите за тем, чтобы дублировать правильную настраиваемую роль и применять такую же область действия, как у встроенной роли.

## <a name="apply-pim-settings"></a>Применение параметров PIM

Создав в клиенте роль, перейдите к настройке PIM и выберите ресурс, для доступа к которому предназначена эта роль.

![](media/azure-pim-resource-rbac/aadpim_manage_azure_resource_some_there.png)

[Настройте параметры PIM](pim-resource-roles-configure-role-settings.md) для выбранных участников этой роли.

Наконец, [назначьте роли](pim-resource-roles-assign-roles.md) определенной группе участников, для которых вы хотите применить настроенные параметры.

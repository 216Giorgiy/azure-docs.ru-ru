# <a name="eligible-assignments-and-resource-visibility-with-privileged-identity-management"></a>Допустимые назначения и видимость ресурсов при использовании управления привилегированными пользователями

Управление привилегированными пользователями (PIM) для ролей ресурсов Azure предоставляет расширенное управление безопасностью для организаций с критически важными ресурсами Azure. Администраторы ресурсов могут использовать PIM для назначения допустимых членов для ролей ресурса. Дополнительные сведения о типах и состояниях назначений для ролей ресурсов Azure представлены в следующих разделах. 

## <a name="assignment-types"></a>Типы назначений

Управление привилегированными пользователями для ресурсов Azure поддерживает два типа назначений:

- допустимое;
- Активна

Назначение допустимой роли означает, что член роли должен выполнить определенное действие для использования этой роли. Поддерживаются такие действия, как прохождение многофакторной идентификации, предоставление коммерческого обоснования или запрос утверждения от назначенных утверждающих.

Назначение активной роли не предусматривает дополнительных действий члена для использования роли. Члены, назначенные в качестве активных, постоянно имеют все полномочия, присвоенные роли.

## <a name="assignment-duration"></a>Длительность назначений

Администраторы ресурсов могут выбирать из двух вариантов для каждого типа назначения в процессе настройки параметров управления привилегированными пользователями для роли. Эти параметры определяют максимальную длительность по умолчанию для членов, которым назначается эта роль в управлении привилегированными пользователями. 

Администратор может выбрать один из этих типов назначения:

- Разрешить постоянную допустимую роль
- Разрешить постоянную активную роль

Или же администратор может выбрать один из этих типов назначения:

- Установить срок действия допустимой роли
- Установить срок действия активной роли

Если администратор ресурса выбрал вариант **Разрешить постоянную допустимую роль** или **Разрешить постоянную активную роль**, все администраторы, назначающие членов роли для этого ресурса, смогут присваивать постоянное членство.

Если же администратор ресурса выбрал вариант **Установить срок действия допустимой роли** или **Установить срок действия активной роли**, он сможет контролировать жизненный цикл назначения, устанавливая обязательные даты для начала и окончания назначений.

> [!NOTE] 
> Администраторы ресурса могут обновлять назначения с указанной датой окончания. Кроме того, члены могут самостоятельно создавать запросы на [продление или обновление назначений](pim-resource-roles-renew-extend.md).


## <a name="assignment-states"></a>Состояние назначений

Управление привилегированными пользователями для ресурсов Azure поддерживает два состояния назначений, которые отображаются на вкладке **Активные роли** в представлениях **Мои роли**, **Роли** и **Члены**. Возможны следующие состояния:

- назначено;
- активировано.

При просмотре списка членов в представлении **Активные роли** значение в столбце **Состояние** позволяет различать пользователей, которым назначена активная роль (**Назначено**) и которые выполнили действие активации для допустимой роли (**Активировано**).

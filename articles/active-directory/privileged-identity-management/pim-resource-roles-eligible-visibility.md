# <a name="eligible-assignments-and-resource-visibility"></a>Допустимые назначения и видимость ресурсов

Управление привилегированными пользователями для ролей ресурсов Azure предоставляет расширенное управление безопасностью для организаций с критически важными ресурсами Azure. Управление привилегированными пользователями позволяет администраторам ресурсов назначать допустимых членов для ролей ресурса. Дополнительные сведения о типах назначений и состояниях для ролей ресурсов Azure представлены ниже. 

## <a name="assignment-types"></a>Типы назначений

Управление привилегированными пользователями для ресурсов Azure поддерживает два типа назначений:

- допустимое;
- активное.

Назначение допустимой роли означает, что член роли должен выполнить определенное действие для использования этой роли. Поддерживаются такие действия, как прохождение многофакторной идентификации, предоставление обоснования и (или) запрос утверждения от назначенных утверждающих.

Назначение активной роли не предусматривает дополнительных действий члена для использования роли. Члены, назначенные в качестве активных, постоянно имеют все полномочия, предоставляемые ролью.

## <a name="assignment-duration"></a>Длительность назначений

Администраторы ресурсов могут выбрать один из двух вариантов для каждого типа назначения в процессе настройке для роли параметров управления привилегированными пользователями. Эти параметры определяют максимальную длительность по умолчанию для членов, которым назначается эта роль в управлении привилегированными пользователями.

- Разрешить постоянную допустимую роль
- Разрешить постоянную активную роль

или

- Установить срок действия допустимой роли
- Установить срок действия активной роли

Если администратор ресурса выбрал вариант "Разрешить постоянную допустимую роль" и (или) "Разрешить постоянную активную роль", все администраторы, имеющие право назначать членов роли для этого ресурса, смогут присваивать постоянное членство.

Варианты "Установить срок действия допустимой роли" и "Установить срок действия активной роли" позволяют контролировать жизненный цикл назначения, устанавливая обязательные даты для начала и окончания назначений.

>[!NOTE] 
>Администраторы ресурса могут продлевать назначения с указанной конечной датой, а сами члены могут самостоятельно создавать запросы на [продление или обновление назначений](pim-resource-roles-renew-extend.md).


## <a name="assignment-states"></a>Состояние назначений

Управление привилегированными пользователями для ресурсов Azure поддерживает два состояния назначений, которые отображаются на вкладке "Активные роли" в представлениях "Мои роли", "Роли" и "Члены". Возможны следующие состояния:

- назначено;
- активировано.

При просмотре списка членов в представлении "Активные роли" столбец "Состояние" позволяет различать пользователей, которым назначена активная роль ("Назначено") и которые выполнили действие активации для допустимой роли ("Активировано").

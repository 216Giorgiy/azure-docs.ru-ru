# <a name="privileged-identity-management---resource-roles---approve"></a>Управление привилегированными пользователями — роли ресурсов — утверждение

С рабочим процессом утверждения в PIM для ролей ресурсов Azure администраторы могут дополнительно защитить критически важные ресурсы или ограничить доступ к ним, требуя разрешения для активации назначения ролей. Для ролей ресурсов Azure существует концепция иерархии ресурсов. Благодаря этой иерархии все нижестоящие подчиненные дочерние ресурсы в родительском контейнере могут наследовать назначения ролей от объекта родительского ресурса. 

Например, администратор ресурсов Кирилл использует PIM, чтобы назначить Алису в качестве правомочного участника роли "Владелец" в подписке Contoso. При применении такого назначения Алиса также является владельцем всех контейнеров группы ресурсов в рамках подписки Contoso и всех ресурсов (например, виртуальных машин), содержащихся во всех группах ресурсов подписки. Предположим, что в подписке Contoso есть три группы ресурсов: Fabrikam Test, Fabrikam Dev и Fabrikam Prod. Каждая из этих групп ресурсов содержит одну виртуальную машину.

Параметры PIM настраиваются для каждой роли ресурса, и в отличие от назначений эти параметры не наследуются и применяются строго к роли ресурса. [Ознакомьтесь с дополнительными сведениями о допустимых назначениях и видимости ресурсов.](pim-resource-roles-eligible-visibility.md)

В примере выше Кирилл использует PIM, чтобы потребовать от всех участников в роли "Владелец" подписки Contoso отправки запроса на утверждение активации. Чтобы защитить ресурсы, содержащиеся в группе ресурсов Fabrikam Prod, Кирилл также требует утверждения участников роли "Владелец" этого ресурса. Роли владельца в Fabrikam Test и Fabrikam Dev не требуют утверждения для активации.

Когда Алиса отправляет запрос на активацию своей роли "Владелец" в подписке Contoso, утверждающий должен одобрить или отклонить этот запрос. Кроме того, если Алиса решит [расширить область активации](pim-resource-roles-activate-your-roles.md#just-enough-administration) на группу ресурсов Fabrikam Prod, утверждающий также должен утвердить или отклонить этот запрос. Однако, если Алиса решит расширить область активации на группу Fabrikam Test, Fabrikam Dev или обе группы, утверждение не требуется.

Рабочий процесс утверждения может не понадобиться для всех участников роли. Предположим, ваша компания привлекла нескольких консультантов для помощи в разработке приложения, которое будет выполняться в подписке Azure. Вам как администратору ресурсов было бы удобно, чтобы сотрудники получали доступ без требуемых утверждений, но партнеры по контракту должны отправить запрос на утверждение. Чтобы настроить рабочий процесс утверждения только для партнеров по контракту, вы можете создать настраиваемую роль с теми же разрешениями, что и для роли, назначенной сотрудникам, и запросить утверждение для активации этой настраиваемой роли. [Дополнительные сведения о настраиваемых ролях](pim-resource-roles-custom-role-policy.md).

Выполните следующие шаги, чтобы настроить рабочий процесс утверждения и указать, кто может утверждать или отклонять запросы.

## <a name="require-approval-to-activate"></a>Настройка требования утверждения для активации

Перейдите в PIM на портале Azure и выберите ресурс из списка.

![](media/azure-pim-resource-rbac/aadpim_manage_azure_resource_some_there.png)

В меню навигации слева выберите **Параметры роли**.

Найдите и выберите нужную роль, а затем щелкните **Изменить**, чтобы изменить параметры.

![](media/azure-pim-resource-rbac/aadpim_rbac_role_settings_view_settings.png)

В разделе активации щелкните параметр **Require approval to activate** (Требовать утверждения для активации).

![](media/azure-pim-resource-rbac/aadpim_rbac_settings_require_approval_checkbox.png)

## <a name="specify-approvers"></a>Указание утверждающих

Щелкните **Select approvers** (Выбор утверждающих лиц), чтобы открыть экран выбора.

>[!NOTE]
>Вы должны выбрать хотя бы одного пользователя или группу для обновления настроек. Утверждающих по умолчанию нет.

Администраторы ресурсов могут добавлять любое сочетание пользователей и групп в список утверждающих. 

![](media/azure-pim-resource-rbac/aadpim_rbac_role_settings_select_approvers.png)

## <a name="request-approval-to-activate"></a>Запрос утверждения активации

Отправка запроса на утверждение не влияет на процедуру, которую участник должен выполнить для активации. [Просмотрите шаги, чтобы активировать роль](pim-resource-roles-activate-your-roles.md).

Если участник запросил активацию роли, требующей утверждения, и эта роль больше не требуется, участник может отменить свой запрос в PIM.

Чтобы отменить запрос, перейдите в PIM и выберите "Мои запросы". Найдите запрос и щелкните "Отмена".

![](media/azure-pim-resource-rbac/aadpim_rbac_role_approval_request_pending.png)

## <a name="approve-or-deny-a-request"></a>Утверждение или отклонение запроса

Чтобы утвердить или отклонить запрос, вы должны быть участником в списке утверждающих. В PIM на вкладке в меню навигации слева выберите "Утвердить запросы" и найдите запрос.

![](media/azure-pim-resource-rbac/aadpim_rbac_approve_requests_list.png)

Выберите запрос, обоснуйте свой выбор и выберите утверждение или отклонение запроса.

![](media/azure-pim-resource-rbac/aadpim_rbac_approve_request_approved.png)

## <a name="workflow-notifications"></a>Уведомления в рамках рабочего процесса

- Все участники в списке утверждающих получают уведомления по электронной почте, когда требуется рассмотреть запрос на утверждение роли. В этих уведомлениях содержится прямая ссылка на запрос, который утверждающий может подтвердить или отклонить.
- Запросы разрешаются первым участником в списке. Он утверждает или отклоняет запрос. 
- Когда утверждающий отвечает на запрос, все участники в списке утверждающих получают уведомление об этом действии. 
- Администраторы ресурсов получают уведомление, когда роль участника, запрос которого утвержден, становится активной. 

>[!Note]
>Если же администратор ресурсов считает, что не нужно активировать роль для участника, запрос которого был утвержден, он может удалить назначение активной роли в PIM. Хотя администраторы ресурсов не получают уведомления об ожидающих запросах (если только они не являются участниками в списке утверждающих лиц), они могут просмотреть и отменить запросы в режиме ожидания для всех пользователей, просмотрев эти запросы в PIM.

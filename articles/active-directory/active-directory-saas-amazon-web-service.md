# Учебник. Интеграция Azure Active Directory с Amazon Web Service (AWS)

Цель этого учебника — показать, как интегрировать Amazon Web Service (AWS) с Azure Active Directory (Azure AD).<br>Интеграция Amazon Web Service (AWS) с Azure AD обеспечивает следующие преимущества:

- С помощью Azure AD вы можете контролировать доступ к Amazon Web Service (AWS). 
- Вы можете включить автоматический вход пользователей в Amazon Web Service (AWS) (единый вход) под учетной записью Azure AD.
- Вы можете управлять учетными записями централизованно — через портал Azure Active Directory

Подробнее узнать об интеграции приложений SaaS с Azure AD можно в статье [Что такое доступ к приложениям и единый вход с помощью Azure Active Directory?](active-directory-appssoaccess-whatis.md).

## Предварительные требования 

Чтобы настроить интеграцию Azure AD с Amazon Web Service (AWS), вам потребуется следующее:

- подписка Azure AD;
- Подписка, настроенная для единого входа в Amazon Web Service (AWS).


> [AZURE.NOTE]Мы не рекомендуем использовать рабочую среду для проверки действий в этом учебнике.


При проверке действий в этом учебнике соблюдайте следующие рекомендации:

- Не следует использовать рабочую среду при отсутствии необходимости.
- Если у вас нет пробной среды Azure AD, вы можете получить пробную версию на один месяц по [этой ссылке](https://azure.microsoft.com/pricing/free-trial/). 

 
## Описание сценария
Цель этого учебника — научить вас проверять единый вход в Azure AD в пробной среде. <br> Сценарий, описанный в этом учебнике, состоит из трех основных блоков:

1. Добавление Amazon Web Service (AWS) из коллекции 
2. Настройка и проверка единого входа в Azure AD


## Добавление Amazon Web Service (AWS) из коллекции
Чтобы настроить интеграцию Amazon Web Service (AWS) с Azure AD, вам потребуется добавить Amazon Web Service (AWS) из коллекции в список управляемых приложений SaaS.

### Чтобы добавить Amazon Web Service (AWS) из коллекции, выполните следующие действия:

1. На **портале управления Azure** в левой области навигации щелкните **Active Directory**. <br><br> ![Active Directory][1]

2. Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3. Чтобы открыть представление приложений, в представлении каталога щелкните **Приложения** в верхнем меню.<br><br> ![Приложения][2]
4. В нижней части страницы нажмите кнопку **Добавить**.<br><br> ![Приложения][3]
5. В диалоговом окне **Что необходимо сделать?** щелкните **Добавить приложение из коллекции**.<br><br> ![Приложения][4]
6. В поле поиска введите **Amazon Web Service (AWS)**.<br><br> ![Приложения][5]
7. В области результатов выберите **Amazon Web Service (AWS)** и нажмите кнопку **Завершить**, чтобы добавить приложение.<br><br> ![Приложения][6]



##  Настройка и проверка единого входа в Azure AD
Цель этого раздела — показать вам, как настроить и проверить единый вход Azure AD с Amazon Web Service (AWS) с использованием тестового пользователя Britta Simon.

Для работы единого входа Azure AD необходимо знать, какой пользователь в Amazon Web Service (AWS) соответствует пользователю в Azure AD. Иными словами, необходимо установить связь между пользователям Azure AD и соответствующим пользователем Amazon Web Service (AWS).<br> Чтобы установить эту связь, следует назначить **имя пользователя** в Azure AD в качестве значения **имени пользователя** в Amazon Web Service (AWS).
 
Чтобы настроить и проверить единый вход Azure AD в Amazon Web Service (AWS), вам потребуется выполнить действия в следующих стандартных блоках:

1. **[Настройка единого входа Azure AD](#configuring-azure-ad-single-single-sign-on)** необходима, чтобы позволить пользователям использовать эту функцию.
2. **[Создание тестового пользователя Azure AD](#creating-an-azure-ad-test-user)** требуется для проверки работы единого входа Azure AD от имени пользователя Britta Simon.
4. **[Создание тестового пользователя Amazon Web Service (AWS)](#creating-a-halogen-software-test-user)** требуется для создания соответствующего пользователя Britta Simon в Amazon Web Service (AWS), связанного с ее представлением в Azure AD.
5. **[Назначение тестового пользователя Azure AD](#assigning-the-azure-ad-test-user)** необходимо, чтобы позволить Britta Simon использовать единый вход Azure AD.
5. **[Проверка единого входа](#testing-single-sign-on)** необходима, чтобы убедиться в корректной работе конфигурации.

### Настройка единого входа в Azure AD

Цель этого раздела — включить единый вход Azure AD на портале Azure AD и настроить единый вход в приложение Amazon Web Service (AWS).<br> Приложение Amazon Web Service (AWS) ожидает проверочные утверждения SAML в определенном формате, который требует добавить настраиваемые сопоставления атрибутов в вашу конфигурацию **атрибутов токена SAML**. На следующем снимке экрана приведен пример. <br><br> ![Настройка единого входа][27]

**Чтобы настроить единый вход Azure AD в Amazon Web Service (AWS), выполните следующие действия:**

1. На странице интеграции с приложением **Amazon Web Service (AWS)** портала Azure AD щелкните **Настройка единого входа**, чтобы открыть диалоговое окно **Настройка единого входа**.<br><br> ![Настройка единого входа][7]

2. На странице **Как пользователи должны входить в Amazon Web Service (AWS)?** выберите **Единый вход Azure AD** и нажмите кнопку **Далее**.<br><br> ![Настройка единого входа][8]

3. На диалоговой странице **Настройка параметров приложения** нажмите кнопку «Далее». <br><br>![Настройка параметров приложения][9]
 
4. На странице **Настройка единого входа в Amazon Web Service (AWS)** нажмите кнопку **Загрузить метаданные**, а затем сохраните файл метаданных на локальном компьютере.<br><br>![Настройка единого входа][10]

5. В другом окне браузера войдите на сайт Amazon Web Service (AWS) компании в качестве администратора.

6. Щелкните **Домашняя страница консоли**. <br><br> ![Настройка единого входа][11]

7. Щелкните **Управление удостоверениями и доступом**. <br><br> ![Настройка единого входа][12]

8. Щелкните **Поставщики удостоверений**, затем щелкните **Создать поставщик**. <br><br> ![Настройка единого входа][13]

9. На диалоговой странице **Настройка поставщика** выполните следующие действия: <br><br> ![Настройка единого входа][14]

     9\.1. Выберите для параметра **Тип поставщика** значение **SAML**.

     9\.2. В текстовом поле **Имя поставщика** введите имя поставщика (например: *WAAD*).

     9\.3. Чтобы передать скачанный файл метаданных, нажмите кнопку **Выбрать файл**.

     9\.4. Щелкните **Следующий шаг**.


10. На диалоговой странице **Проверка сведений о поставщике** щелкните **Создать**. <br><br> ![Настройка единого входа][15]

11. Выберите **Роли > Создать новую роль**. <br><br> ![Настройка единого входа][16]

12. В диалоговом окне **Настройка имени роли** выполните следующие действия: <br><br> ![Настройка единого входа][17] 12.1 В текстовом поле **Имя роли** введите имя роли (например: *TestUser*).

     12\.2. Щелкните **Следующий шаг**.

13. В диалоговом окне **Выбор типа роли** выполните следующие действия: <br><br> ![Настройка единого входа][18]

     13\.1. Выберите **Роль для доступа поставщика удостоверений**.

     13\.2. В **Предоставление единого входа через Интернет (WebSSO) для доступа к поставщикам SAML** щелкните **Выбрать**.


14. В диалоговом окне «Установка доверия» выполните следующие действия: <br><br> ![Настройка единого входа][19] 14.1 В качестве поставщика SAML выберите поставщик SAML, созданный ранее (например: *WAAD*).



15. На портале Azure AD выберите подтверждение конфигурации единого входа, а затем нажмите кнопку **Далее**.<br><br>![Что такое Azure AD Connect?][20]

16. На странице **Подтверждение конфигурации единого входа** нажмите кнопку **Завершить**, чтобы закрыть диалоговое окно **Настройка единого входа**.<br><br>![Что такое Azure AD Connect?][22]


17. В меню в верхней части экрана щелкните **Атрибуты**, чтобы открыть диалоговое окно **Атрибуты маркера SAML**. <br><br> ![Настройка единого входа][21]

18. Щелкните **Добавить атрибут пользователя**. <br><br> ![Настройка единого входа][23]

19. В диалоговом окне «Добавление атрибута пользователя» выполните следующие действия. <br><br> ![Настройка единого входа][24]

     19\.1. В текстовом поле **Имя атрибута** введите ****https://aws.amazon.com/SAML/Attributes/Role**.

     19\.2. В текстовом поле **Значение атрибута** введите **arn:aws:iam::214510765665:role/Admin,arn:aws:iam::214510765665:saml-provider/WAAD**.

     19\.3. Щелкните **Завершить**, чтобы закрыть диалоговое окно **Добавление атрибута пользователя**.

20. Щелкните **Добавить атрибут пользователя**. <br><br> ![Настройка единого входа][23]


21. В диалоговом окне «Добавление атрибута пользователя» выполните следующие действия. <br><br> ![Настройка единого входа][25]

     21\.1. В текстовом поле **Имя атрибута** введите ****https://aws.amazon.com/SAML/Attributes/RoleSessionName**.

     21\.2. Из списка **Значение атрибута** выберите **user:mail**.

     21\.3. Щелкните **Завершить**, чтобы закрыть диалоговое окно **Добавление атрибута пользователя**.


22. Щелкните **Применить изменения**. <br><br> ![Настройка единого входа][26]





### Создание тестового пользователя Azure AD

Цель этого раздела — создать тестового пользователя с именем Britta Simon на портале Azure.<br> В списке пользователей выберите **Britta Simon**.<br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_01.png)

**Чтобы создать тестового пользователя в Azure AD, выполните следующие действия:**

1. На **портале управления Azure** в левой области навигации щелкните **Active Directory**.<br> ![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_02.png) 

2. Из списка **Каталог** выберите каталог, для которого нужно включить интеграцию каталогов.

3. Чтобы отобразить список пользователей, щелкните **Пользователи** в меню вверху.<br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_03.png)
 
4. Чтобы открыть диалоговое окно **Добавление пользователя**, нажмите кнопку **Добавить пользователя** на панели инструментов внизу. <br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_04.png)

5. На диалоговой странице **Расскажите об этом пользователе** выполните следующие действия: <br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_05.png)
  1. В поле «Тип пользователя» выберите значение «Новый пользователь в вашей организации».
  2. В текстовом поле **Имя пользователя** введите **BrittaSimon**.
  3. Нажмите кнопку Далее.

6.  На диалоговой странице **Профиль пользователя** выполните следующие действия: <br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_06.png)
  1. В текстовом поле **Имя** введите **Britta**.  
  2. В текстовом поле **Фамилия** введите **Simon**.
  3. В текстовом поле **Отображаемое имя** введите **Britta Simon**.
  4. Из списка **Роль** выберите **Пользователь**.
  5. Нажмите кнопку **Далее**.

7. На диалоговой странице **Получить временный пароль** нажмите кнопку **Создать**. <br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_07.png)
 
8. На диалоговой странице **Получить временный пароль** выполните следующие действия:<br>![Создание тестового пользователя Azure AD](./media/active-directory-saas-amazon-web-service/create_aaduser_08.png)
  1. Запишите значение поля **Новый пароль**.
  2. Нажмите **Завершено**.

Цель этого раздела — позволить пользователю Britta Simon использовать единый вход Azure, предоставив ей доступ к Amazon Web Service (AWS). <br><br>![Назначение пользователя][31]

2. На панели инструментов внизу щелкните **Назначить**. <br><br>![Назначение пользователя][29]

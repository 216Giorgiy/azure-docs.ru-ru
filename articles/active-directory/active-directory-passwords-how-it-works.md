# Как работает управление паролями
Управление паролями в Azure Active Directory подразумевает использование нескольких логических компонентов, которые описаны в этой статье. Дополнительные сведения о каждом компоненте можно получить по ссылкам ниже.

- [**Портал настройки управления паролями**](#password-management-configuration-portal). На [портале управления Azure](https://manage.windowsazure.com) на вкладке настроек каталога администраторы могут контролировать разные аспекты управления паролями в своих клиентах.
- [**Портал регистрации пользователей**](#user-registration-portal). На этом веб-портале пользователи могут самостоятельно зарегистрировать необходимые данные для сброса паролей.
- [**Портал сброса пользовательских паролей**](#user-password-reset-portal). Пользователи могут сбрасывать свои пароли, предварительно обработав запрос защиты. Эти запросы бывают разных типов и определяются в политике сброса паролей, которой управляет администратор.
- [**Портал изменения пользовательских паролей**](#user-password-change-portal). Пользователи могут изменять свои пароли в любое время. Для этого на этом веб-портале следует ввести старый пароль и выбрать новый.
- [**Отчеты об управлении паролями**](#password-management-reports). Администраторы могут просматривать и анализировать в своих клиентах действия по сбросу и регистрации паролей. Для этого нужно перейти на [портал управления Azure](https://manage.windowsazure.com) и в своем каталоге на вкладе «Отчеты» просмотреть раздел «Отчеты о действиях».
- [**Компонент обратной записи паролей Azure AD Connect**](#password-writeback-component-of-azure-ad-connect). При необходимости администраторы могут включить функцию обратной записи паролей во время установки Azure AD Connect. Она разрешает управление федеративными или синхронизированными паролями пользователей из облака.

## Портал настройки управления паролями
Вы можете настроить политики управления паролями для конкретного каталога. Для этого войдите на [портал управления Azure](https://manage.windowsazure.com), выберите вкладку **Настройки** и перейдите к разделу **Политика сброса пароля пользователя**. На этой странице можно контролировать многие аспекты управление паролями в вашей организации, в частности:

- Включать или отключать функцию сброса паролей для всех пользователей в каталоге.
- Задавать количество запросов защиты (один или два), которые должен обработать пользователь, чтобы сбросить свой пароль.
- Выбирать определенные типы запросов защиты для пользователей в организации. Доступны такие типы:
 - мобильный телефон (проверочный код через SMS или голосовой звонок);
 - рабочий телефон (голосовой звонок);
 - дополнительный электронный адрес (проверочный код по электронной почте);
 - контрольные вопросы (проверка подлинности на основе ранее предоставленных данных).
- Задавать количество вопросов, которые должен зарегистрировать пользователь; используется при проверке подлинности с помощью контрольных вопросов (отображается, если активирован тип контрольных вопросов).
- Задавать количество вопросов, на которые должен ответить пользователь во время сброса пароля; используется при проверке подлинности с помощью контрольных вопросов (отображается, если активирован тип контрольных вопросов).
- Определять другие контрольные вопросы, которые пользователь может зарегистрировать; используется при проверке подлинности с помощью контрольных вопросов (отображается, если активирован тип контрольных вопросов).
- Требовать регистрацию данных для сброса пароля при переходе на панель доступа приложения по адресу [http://myapps.microsoft.com](http://myapps.microsoft.com).
- Требовать повторно подтверждать ранее зарегистрированные данные по истечении определенного количества дней (отображается, только если включена принудительная регистрация).
- Предоставлять настраиваемый электронный адрес или URL-адрес службы технической поддержки, который будет отображаться для пользователей на случай возникновения проблем со сбросом паролей.
- Включать или отключать возможность обратной записи паролей (если компонент обратной записи паролей развернут с помощью AAD Connect).
- Следить за состоянием агента обратной записи паролей (если компонент обратной записи паролей развернут с помощью AAD Connect).
- Включать уведомления пользователей по электронной почте о сбросе их собственных паролей (на [портале управления Azure](https://manage.windowsazure.com) в разделе **Уведомления**).
- Включать уведомления администраторов по электронной почте о сбросе их паролей другим администратором (на [портале управления Azure](https://manage.windowsazure.com) в разделе **Уведомления**).
- Управлять названием и логотипом организации, в частности на портале сброса пользовательских паролей и в сообщениях электронной почты (на [портале управления Azure](https://manage.windowsazure.com) в разделе **Свойства каталога**).

Дополнительные сведения о настройке управления паролями в организации см. в статье [Приступая к работе: управление паролями AD Azure](active-directory-passwords-getting-started.md).

##Портал регистрации пользователей
До того как пользователи смогут воспользоваться сбросом пароля, их облачные учетные записи следует обновить актуальными данными проверки подлинности, чтобы они обязательно проходили заданное администратором количество проверок для сброса пароля. Администраторы также могут определять эту информацию для проверки подлинности от лица своих пользователей с помощью веб-порталов Azure или Office, DirSync, Azure AD Connect и Windows PowerShell.

Однако, если вы хотите, чтобы пользователи сами регистрировали свои данные, мы предоставляем веб-страницу, на которой они могут предоставлять эти сведения. Эта страница дает возможность пользователям указывать информацию для проверки подлинности в соответствии с политикой сброса пароля, которая включена в их организации. После проверки этих данных они сохраняются в облачной учетной записи пользователя и используются для ее восстановления в будущем. Вот как выглядит портал регистрации:

  ![][001]

Дополнительные сведения см. в статьях [Приступая к работе: управление паролями AD Azure](active-directory-passwords-getting-started.md) и [Рекомендации: управление паролями AD Azure](active-directory-passwords-best-practices.md).

##Портал сброса пользовательских паролей
Когда вы включите функцию самостоятельного сброса паролей, настроите в своей организации соответствующую политику и убедитесь, что у пользователей есть соответствующие контактные данные в каталоге, пользователи смогут автоматически сбрасывать свои пароли на любой веб-странице, для входа на которую используется рабочая или учебная учетную запись (например, [portal.microsoftonline.com](https://portal.microsoftonline.com)). На таких страницах имеется ссылка **Не можете получить доступ к своей учетной записи?**

  ![][002]

Выбрав эту ссылку, вы автоматически попадете на портал сброса паролей.

  ![][003]

Дополнительные сведения о том, как пользователи могут самостоятельно сбрасывать свои пароли, см. в статье [Приступая к работе: управление паролями AD Azure](active-directory-passwords-getting-started.md).

##Портал изменения пользовательских паролей
С помощью портала изменения паролей пользователи могут в любое время изменять свои пароли. Попасть на этот портал можно с панели доступа (на странице профиля) или по ссылке для изменения пароля (в приложениях Office 365). Когда срок действия пароля истекает, при входе в систему пользователь увидит запрос на его изменение.

  ![][004]

В обоих случаях, если включена обратная запись паролей, а пользователь включен в федерацию либо настроена синхронизация его пароля, новые пароли записываются обратно в локальную службу Active Directory. Вот как выглядит портал изменения пароля.

  ![][005]

Дополнительные сведения о том, как пользователи могут изменять свои пароли в локальной службе Active Directory, см. в статье [Приступая к работе: управление паролями AD Azure](active-directory-passwords-getting-started.md).

##Отчеты об управлении паролями
На вкладке **Отчеты** в разделе **Журналы действий** вы увидите два отчета об управлении паролями: **Действие сброса пароля** и **Действие регистрации сброса пароля**. С помощью этих двух отчетов можно узнать, кто из пользователей зарегистрировал данные для сброса пароля, а кто уже воспользовался функцией сброса. Вот как будут выглядят эти отчеты на [портале управления Azure](https://manage.windowsazure.com):

  ![][006]

Дополнительные сведения см. в статье [Аналитическая картина на основании отчетов управления паролями в Azure AD](active-directory-passwords-get-insights.md).

##Компонент обратной записи паролей Azure AD Connect
Если пароли пользователей в организации изначально берутся из локальной среды (через синхронизацию паролей или федерацию), установка последней версии Azure Connect AD позволит настроить обновление этих паролей прямо из облака. Это означает, что пользователи могут восстановить или изменить свой пароль AD непосредственно в Интернете. Где находится параметр активации обратной записи паролей в мастере установки Azure AD Connect, показано на рисунке ниже.

  ![][007]

Дополнительные сведения об Azure AD Connect см. в статье [Приступая к работе: Azure AD Connect](active-directory-aadconnect.md). Дополнительные сведения об обратной записи паролей см. в статье [Приступая к работе: управление паролями AD Azure](active-directory-passwords-getting-started.md).

# <a name="how-password-management-works-in-azure-active-directory"></a>Как работает управление паролями в Azure Active Directory
> [!IMPORTANT]
> **Вы здесь потому, что возникают проблемы при входе?** Если это так, [с помощью этих инструкций можно изменить и сбросить пароль](active-directory-passwords-update-your-own-password.md).
>
>

Управление паролями в Azure Active Directory (Azure AD) подразумевает использование нескольких логических компонентов, которые описаны в этой статье. Дополнительные сведения о каждом компоненте можно получить по ссылкам ниже.

* [**Портал настройки управления паролями.**](#password-management-configuration-portal) На [портале управления Azure](https://manage.windowsazure.com) на вкладке настроек каталога администраторы могут контролировать разные аспекты управления паролями в своих клиентах.
* [**Портал регистрации пользователей.**](#user-registration-portal) На этом веб-портале пользователи могут самостоятельно зарегистрировать необходимые данные для сброса паролей.
* [**Портал сброса пользовательских паролей.**](#user-password-reset-portal) Пользователи могут сбрасывать свои пароли, предварительно обработав запрос защиты. Эти запросы бывают разных типов и определяются в политике сброса паролей, которой управляет администратор.
* [**Портал изменения пользовательских паролей.**](#user-password-change-portal) Пользователи могут изменять свои пароли в любое время. Для этого на этом веб-портале следует ввести старый пароль и выбрать новый.
* [**Отчеты об управлении паролями.**](#password-management-reports) Администраторы могут просматривать и анализировать в своих клиентах действия по сбросу и регистрации паролей. Для этого нужно перейти на [портал управления Azure](https://manage.windowsazure.com) и в своем каталоге на вкладке "Отчеты" просмотреть раздел "Отчеты о действиях".
* [**Компонент обратной записи паролей Azure AD Connect.**](#password-writeback-component-of-azure-ad-connect) При необходимости администраторы могут включить компонент обратной записи паролей во время установки Azure AD Connect. Он позволяет управлять федеративными или синхронизированными паролями пользователей из облака.

## <a name="password-management-configuration-portal"></a>Портал настройки управления паролями
Вы можете настроить политики управления паролями для конкретного каталога. Для этого войдите на [портал управления Azure](https://manage.windowsazure.com), выберите вкладку **Настройки** и перейдите к разделу **Политика сброса пароля пользователя**.  На этой странице можно контролировать многие аспекты управление паролями в вашей организации, в частности:

* Включать или отключать функцию сброса паролей для всех пользователей в каталоге.
* Задавать количество запросов защиты (один или два), которые должен обработать пользователь, чтобы сбросить свой пароль.
* Выбирать определенные типы запросов защиты для пользователей в организации. Доступны такие типы:
  * мобильный телефон (проверочный код через SMS или голосовой звонок);
  * рабочий телефон (голосовой звонок);
  * дополнительный электронный адрес (проверочный код по электронной почте);
  * контрольные вопросы (проверка подлинности на основе ранее предоставленных данных).
* Задавать количество вопросов, которые должен зарегистрировать пользователь; используется при проверке подлинности с помощью контрольных вопросов (отображается, если активирован тип контрольных вопросов).
* Задавать количество вопросов, на которые должен ответить пользователь во время сброса пароля; используется при проверке подлинности с помощью контрольных вопросов (отображается, если активирован тип контрольных вопросов).
* Использовать предустановленные локализованные секретные вопросы, которые пользователь может выбрать для сброса пароля при регистрации (отображается только если секретные вопросы включены).
* Определить пользовательские секретные вопросы, которые пользователь может выбрать для сброса пароля при регистрации (отображается, только если секретные вопросы включены).
* Требовать регистрацию данных для сброса пароля при переходе на панель доступа приложения по адресу [http://myapps.microsoft.com](http://myapps.microsoft.com).
* Требовать повторно подтверждать ранее зарегистрированные данные по истечении определенного количества дней (отображается, только если включена принудительная регистрация).
* Предоставлять настраиваемый электронный адрес или URL-адрес службы технической поддержки, который отображается для пользователей на случай возникновения проблем со сбросом паролей.
* Включать или отключать возможность обратной записи паролей (если компонент обратной записи паролей развернут с помощью AAD Connect).
* Следить за состоянием агента обратной записи паролей (если компонент обратной записи паролей развернут с помощью AAD Connect).
* Включать уведомления пользователей по электронной почте о сбросе их собственных паролей (на **портале управления Azure** в разделе [Уведомления](https://manage.windowsazure.com)).
* Включать уведомления администраторов по электронной почте о сбросе их паролей другим администратором (на **портале управления Azure** в разделе [Уведомления](https://manage.windowsazure.com)
* Управлять названием и логотипом организации, в частности на портале сброса пользовательских паролей и в сообщениях электронной почты (на **портале управления Azure** в разделе [Уведомления](https://manage.windowsazure.com)

Дополнительные сведения о настройке управления паролями в организации см. в статье [Приступая к работе с компонентами управления паролями](active-directory-passwords-getting-started.md).

## <a name="user-registration-portal"></a>Портал регистрации пользователей
До того как пользователи смогут воспользоваться сбросом пароля, их облачные учетные записи следует обновить актуальными данными проверки подлинности, чтобы они обязательно проходили заданное администратором количество проверок для сброса пароля.  Администраторы также могут определять эту информацию для проверки подлинности от лица своих пользователей с помощью веб-порталов Azure или Office, DirSync, Azure AD Connect и Windows PowerShell.

Однако, если вы хотите, чтобы пользователи сами регистрировали свои данные, мы предоставляем веб-страницу, на которой они могут заполнить эти сведения.  Эта страница дает возможность пользователям указывать информацию для проверки подлинности в соответствии с политикой сброса пароля, которая включена в их организации.  После проверки этих данных они сохраняются в облачной учетной записи пользователя и используются для ее восстановления в будущем. Вот как выглядит портал регистрации:

  ![][001]

Дополнительные сведения см. в статьях [Приступая к работе с компонентами управления паролями](active-directory-passwords-getting-started.md) и [Рекомендации по управлению паролями Azure Active Directory](active-directory-passwords-best-practices.md).

## <a name="user-password-reset-portal"></a>Портал сброса пользовательских паролей
Когда вы включите функцию самостоятельного сброса паролей, настроите в своей организации соответствующую политику и убедитесь, что у пользователей есть соответствующие контактные данные в каталоге, пользователи могут автоматически сбрасывать свои пароли на любой веб-странице, для входа на которую используется рабочая или учебная учетная запись (например, [portal.microsoftonline.com](https://portal.microsoftonline.com)). На таких страницах имеется ссылка **Не удается получить доступ к своей учетной записи?**

  ![][002]

Выбрав эту ссылку, вы автоматически попадете на портал сброса паролей.

  ![][003]

Дополнительные сведения о том, как пользователям самостоятельно сбрасывать свои пароли, см. в статье [Приступая к работе с компонентами управления паролями](active-directory-passwords-getting-started.md).

## <a name="user-password-change-portal"></a>Портал изменения пользовательских паролей
С помощью портала изменения паролей пользователи могут в любое время изменять свои пароли.  Попасть на этот портал можно с панели доступа (на странице профиля) или по ссылке для изменения пароля (в приложениях Office 365).  Когда срок действия пароля истекает, при входе в систему пользователь увидит запрос на его изменение.

  ![][004]

В обоих случаях, если включена обратная запись паролей, а пользователь включен в федерацию либо настроена синхронизация его пароля, новые пароли записываются обратно в локальную службу Active Directory. Вот как выглядит портал изменения пароля.

  ![][005]

Дополнительные сведения о том, как пользователям изменять свои пароли в локальной службе Active Directory, см. в статье [Приступая к работе с компонентами управления паролями](active-directory-passwords-getting-started.md).

## <a name="password-management-reports"></a>Отчеты об управлении паролями
На вкладке **Отчеты** в разделе **Журналы действий** вы увидите два отчета об управлении паролями: **Действие сброса пароля** и **Действие регистрации сброса пароля**.  С помощью этих двух отчетов можно узнать, кто из пользователей зарегистрировал данные для сброса пароля, а кто уже воспользовался функцией сброса. Вот как будут выглядят эти отчеты на [портале управления Azure](https://manage.windowsazure.com):

  ![][006]

Дополнительные сведения см. в статье [Аналитическая картина на основе отчетов управления паролями в Azure Active Directory](active-directory-passwords-get-insights.md).

## <a name="password-writeback-component-of-azure-ad-connect"></a>Компонент обратной записи паролей Azure AD Connect
Если пароли пользователей в организации изначально берутся из локальной среды (через синхронизацию паролей или федерацию), установка последней версии Azure Connect AD позволит настроить обновление этих паролей прямо из облака.  Это означает, что пользователи могут восстановить или изменить свой пароль AD непосредственно в Интернете.  Где находится параметр активации обратной записи паролей в мастере установки Azure AD Connect, показано на рисунке ниже.

  ![][007]

Дополнительные сведения об Azure AD Connect см. в статье [Интеграция локальных удостоверений с Azure Active Directory](active-directory-aadconnect.md). Дополнительные сведения об обратной записи паролей см. в статье [Приступая к работе с компонентами управления паролями](active-directory-passwords-getting-started.md).

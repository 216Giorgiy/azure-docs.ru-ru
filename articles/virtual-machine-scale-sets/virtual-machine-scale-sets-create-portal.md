# <a name="create-a-virtual-machine-scale-set-in-the-azure-portal"></a>Создание набора масштабирования виртуальных машин на портале Azure
Масштабируемый набор виртуальных машин обеспечивает развертывание и администрирование набора идентичных автомасштабируемых виртуальных машин. Вы можете вручную изменить число виртуальных машин в масштабируемом наборе или определить правила для автоматического масштабирования на основе использования ЦП, объема памяти или сетевого трафика. В этой статье началу работы создайте масштабирования виртуальных машин, установите на портале Azure. Также можно создать набор с масштабом [Azure CLI 2.0](virtual-machine-scale-sets-create-cli.md) или [Azure PowerShell](virtual-machine-scale-sets-create-powershell.md).

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.


## <a name="log-in-to-azure"></a>Вход в Azure
Войдите на портал Azure по адресу http://portal.azure.com.


## <a name="create-virtual-machine-scale-set"></a>Создание набора масштабирования виртуальных машин
Можно развернуть с помощью образа Windows Server или Linux RHEL, CentOS, Ubuntu или SLES наборе масштабирования.

1. Щелкните **Создать** в верхнем левом углу портала Azure.
2. Поиск *набор масштабирования*, выберите **масштабный набор виртуальных машин**, а затем выберите **создать**.
3. Введите имя для набора масштабирования, например *myScaleSet*.
4. Выберите желаемому типу операционной системы, например *Windows Server 2016 Datacenter*.
5. Введите ваш нужное имя группы ресурсов, таких как *myResourceGroup*и расположение, например *Восток США*.
6. Введите нужное имя пользователя и выберите тип аутентификации.
    - Объект **пароль** должен быть не короче 12 символов и удовлетворять трех из четырех следующих требованиям сложности: одну строчную букву, одну прописную букву, одну цифру и один специальный символ. Дополнительные сведения см. в разделе [требования к имени пользователя и пароля](../virtual-machines/windows/faq.md#what-are-the-username-requirements-when-creating-a-vm).
    - При выборе образа диска ОС Linux, можно вместо этого выбрать **открытый ключ SSH**. Только укажите свой открытый ключ, например *~/.ssh/id_rsa.pub*. Можно использовать оболочки облако Azure на портале для [создать и использовать ключи SSH](../virtual-machines/linux/mac-create-ssh-keys.md).

7. Введите **имя открытого IP-адреса**, такие как *myPublicIP*.
8. Введите уникальный **метка доменного имени**, такие как *myuniquedns*. Эта метка DNS формирует базовый полное доменное имя для подсистемы балансировки нагрузки перед набором.
9. Чтобы подтвердить параметры шкалы, выберите **создать**.

    ![Создание масштабирования виртуальных машин, установите на портале Azure](./media/virtual-machine-scale-sets-create-portal/create-scale-set.png)


## <a name="connect-to-a-vm-in-the-scale-set"></a>Подключение к виртуальной машине в наборе масштабирования
При создании наборе на портале масштабирования создается подсистемы балансировки нагрузки. Правила сетевых адресов (NAT) используются для распределения трафика на экземпляры набор масштабирования для удаленных подключений, таких как RDP или SSH.

Просмотр этих NAT правила и сведения о соединении для масштаба набора экземпляров:

1. Выберите группу ресурсов, созданные на предыдущем шаге, таких как *myResourceGroup*.
2. Список ресурсов, выберите ваш **балансировки нагрузки**, такие как *myScaleSetLab*.
3. Выберите **правила NAT для входящего трафика** меню в левой части окна.

    ![Правила для входящих подключений NAT позволяют подключаться к экземплярам набора масштабирования виртуальных машин](./media/virtual-machine-scale-sets-create-portal/inbound-nat-rules.png)

С помощью этих правил преобразования сетевых адресов можно подключиться к каждой виртуальной машине в наборе масштабирования. Каждый экземпляр виртуальной Машины перечислены конечный IP-адрес и номер порта TCP. Например, если IP-адресом назначения *104.42.1.19* и TCP-порт — *50001*, подключитесь к экземпляру виртуальной Машины, как показано ниже:

- Для набора масштабирования Windows, подключитесь к экземпляру виртуальной Машины с помощью протокола удаленного рабочего СТОЛА на`104.42.1.19:50001`
- Для набора масштабирования Linux, подключитесь к экземпляру виртуальной Машины с SSH на`ssh azureuser@104.42.1.19 -p 50001`

При появлении запроса введите учетные данные, указанные на предыдущем шаге при создании набора масштабирования. Экземпляры набор масштабирования, регулярного виртуальные машины, которыми могут взаимодействовать в обычном режиме. Дополнительные сведения о способах развертывания и выполнения приложений на шкале набор экземпляров см. в разделе [развернуть приложение на наборы масштабирования виртуальных машин](virtual-machine-scale-sets-deploy-app.md)


## <a name="clean-up-resources"></a>Очистка ресурсов
Если больше не нужен, удалите группу ресурсов, масштабирования набора, а также все связанные ресурсы. Чтобы сделать это, выберите группу ресурсов для виртуальной машины и нажмите кнопку **Удалить**.

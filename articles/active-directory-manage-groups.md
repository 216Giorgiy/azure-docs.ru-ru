# Управление группами в Azure AD

Группа — это коллекция пользователей и групп, которыми можно управлять как единым целым. Пользователи и группы, принадлежащие конкретной группе, называются участниками группы. Использование групп может упростить администрирование за счет назначения общего набора разрешений и прав сразу многим учетным записям, вместо их назначения отдельным учетным записям.

Сейчас в Azure AD вы можете создавать только группы безопасности.

Группы — это удобный способ предоставлять пользователям доступ к приложениям в случаях, когда доступ к одному приложению нужно предоставить многим пользователям. Группы также можно использовать при настройке управления доступом к другим веб-службам, которые управляют доступом к ресурсам, например SharePoint Online.

Если настроена синхронизация каталогов, вы сможете видеть группы, синхронизированные из локальной службы Windows Server Active Directory, значение свойства «Где создано» которой — «Локальный каталог Active Directory». Этими группами по-прежнему придется управлять в локальной службе Active Directory. Ими нельзя управлять на портале управления Azure или удалять их там.

При наличии Office 365 можно видеть группы безопасности, поддерживающие почту, и группы распределения, которые создали и которыми управляют в Центре администрирования Exchange в Office 365. Значение свойства «Где создано» этих групп — Office 365, и ими придется по-прежнему управлять в Центре администрирования Exchange.

Вы также можете создавать объединенные группы через панель доступа. На вкладке «Настройка» в разделе «Управление группами» задайте для мини-приложения **Пользователи могут создавать группы Office 365** значение **Да**. При наличии объединенных групп Office 365, созданных на панели доступа или в Office 365, для этих групп значением свойства «Где создано» будет Azure Active Directory, и ими можно управлять через панель доступа.

Если для пользователей включено самостоятельное управление группами (дополнительную информацию см. в разделе «Самостоятельное управление группами для пользователей в Azure AD»), то вы также можете управлять этими группами на портале управления Azure в качестве администратора клиента. Вы можете добавлять и удалять участников группы, добавлять и удалять владельцев группы, изменять свойства групп и просматривать отчет об активности журнала группы, отображающий действия, которые выполнялись в группе, кто выполнял эти действия и в какое время.

> [AZURE.NOTE]Чтобы можно было назначить группу приложению, необходимо использовать Azure AD Premium. При наличии Azure AD Premium можно также использовать группы для предоставления доступа к приложениям SaaS, интегрированным с Azure AD. Дополнительную информацию см. в разделе «Предоставление группе доступа к приложению SaaS в Azure AD».

# Настройка портала разработчика в Azure API Management

Это руководство описывает способы изменения вида и поведения портала разработчика Azure API Management для его соответствия корпоративному дизайну.

## <a name="change-page-headers"> </a>Изменение текста и логотипа в заголовках страниц

Одним из важных аспектов настройки портала является замена текста вверху всех страниц на название или логотип компании.

Содержимое портала разработчика изменяется на портале Publisher, который доступен с портала управления Azure. Чтобы открыть консоль управления API, щелкните **Консоль управления** на портале Azure для своей службы управления API.

![Management console][api-management-management-console]

Портал разработчика основан на системе Content Management System или CMS. Содержимое специального типа, появляющееся в заголовке каждой страницы, называется мини-приложением. Чтобы изменить содержимое виджета, выберите пункт **Виджеты** в меню **Портал разработчика** слева, а затем выберите в списке виджет **Заголовок**.

![Widgets header][api-management-widgets-header]

Содержимое заголовка редактируется из поля **Тело**. Замените текст на "Портал разработчика Fabrikam" и нажмите **Сохранить** в нижней части страницы.

Теперь вы сможете увидеть новый заголовок на каждой странице на портале разработчика!

> Чтобы открыть портал разработчика на консоли управления, нажмите **Портал разработчика** на панели сверху.

## <a name="change-headers-styling"> </a>Изменение стиля заголовков

Цвета, шрифты, размеры, отступы и другие связанные со стилем элементы любой страницы портала задаются правилами стиля. Чтобы изменить стили, выберите пункт **Внешний вид** в меню **Портал разработчика** на портале издателя. Затем нажмите **Начать настройку**, чтобы вызвать редактор стилей.

Браузер переключится на скрытую страницу на портале разработчика, где содержатся примеры содержимого и примеры всех стилей, используемых на всем сайте. Для открытия редактора стилей установите курсор на тонкой серой вертикальной линии в левой крайней части страницы. Должна появиться панель инструментов редактора.

![Customization toolbar][api-management-customization-toolbar]

Существует два основных режима редактирования правил стилей: **Редактировать все правила** отображает список всех правил стилей, использованных где угодно, и **Выбрать элемент** позволяет выбрать элемент на странице, для которого будут отображены использованные в нем стили.

В этом разделе планируется изменить стили только для заголовков. Выберите пункт **Выбрать элемент** на панели инструментов редактора стилей, а затем нажмите **Выбрать элемент для настройки**. Элементы будут выделяться при проведении по ним указателем мыши, что указывает на возможность редактирования стилей элемента щелчком мыши. Наведите указатель мыши на имя компании в заголовке (это будет "Портал разработчика Fabrikam", если были выполнены инструкции из предыдущего раздела) и щелкните его. В редакторе стилей появится поименованный и упорядоченный по категориям набор правил стилей.

Каждое правило представляет свойство стиля выбранного элемента. Например, для текста выбранного выше заголовка размер текста содержится в @font-size-h1, а имя шрифта с вариантами содержится в @headings-font-family.

> Если вы знакомы с работой [bootstrap][], то эти правила фактически являются [переменными LESS][] в теме bootstrap, которая используется порталом разработчика.

Давайте изменим цвет текста заголовка. Выберите запись в поле **@headings-color** и введите #000000. Это шестнадцатеричный код черного цвета. После этого в конце текстового блока появится цветной квадратный индикатор. При щелчке по нему элемент выбора цвета позволит выбрать цвет.

![Color picker][api-management-customization-toolbar-color-picker]

Завершив вносить изменения в стили выбранного элемента, нажмите **Просмотр изменений**, чтобы увидеть результаты на экране. В этот момент они видимы только администраторам. Чтобы эти изменения были видны всем, нажмите кнопку **Опубликовать** в редакторе стилей и подтвердите изменения.

![Publish menu][api-management-customization-toolbar-publish-form]

> Чтобы изменить правила стилей для другого элемента на странице, выполните те же действия, что и с заголовком, - нажмите **Выбрать элемент** в редакторе стилей, выберите нужный элемент и начните изменять правила стилей, отображенные на экране.

## <a name="edit-page-contents"> </a>Редактирование содержимого страницы

Портал разработчика состоит из автоматически генерируемых страниц, таких как интерфейсы API, продукты, приложения, проблемы и самостоятельно написанное содержимое. Поскольку он базируется на системе управления содержимым, можно создавать такое содержимое, какое нужно.

Чтобы увидеть список всех существующих страниц содержимого, выберите пункт **Содержимое** в меню **Портал разработчика** в консоли управления.

![Manage content][api-management-customization-manage-content]

Щелкните страницу приветствия для редактирования содержимого домашней страницы портала разработчика. Внесите нужные изменения, при необходимости просмотрите их и нажмите **Опубликовать сейчас**, чтобы сделать их видимыми всем пользователям.

> Домашняя страница использует особый макет, который позволяет выводить заголовок вверху. Этот заголовок нельзя редактировать в разделе содержимого. Чтобы изменить этот баннер, выберите пункт **Виджеты** в меню **Портал разработчика** и выберите пункт **Домашняя страница** в раскрывающемся меню **Текущий слой**, а затем откройте пункт **Баннер** в разделе "Популярное". Содержимое в этом мини-приложении редактируется так же, как для других страниц.

# Пулы агентов

Служба контейнеров Azure распределяет агенты по нескольким пулам. Все виртуальные машины в пуле идентичны, но для каждого пула выделяется собственная подсеть. Поэтому каждый пул может являться общедоступным или частным, что влияет на возможность взаимодействия между машинами в службе контейнеров.

### Частные агенты

Частные узлы агентов работают в сети без глобальной маршрутизации. К такой сети можно обратиться только из зоны администратора или через пограничный маршрутизатор общедоступной зоны. По умолчанию DC/OS запускает приложения на частных узлах агента. Дополнительные сведения о сетевой безопасности можно получить в [документации по DC/OS](https://dcos.io/docs/1.7/administration/securing-your-cluster/).

### Общедоступные агенты

Общедоступные узлы агентов выполняют приложения и службы DC/OS в общедоступной сети. Дополнительные сведения о сетевой безопасности можно получить в [документации по DC/OS](https://dcos.io/docs/1.7/administration/securing-your-cluster/).

## Использование пулов агентов

По умолчанию **Marathon** развертывает все новые приложения на *частных* узлах агентов. Приложение на *общедоступном* узле необходимо развернуть явно. К сожалению, развертывание на общедоступном узле нельзя выполнить, просто установив нужный флажок. При создании нового приложения перейдите на вкладку **Дополнительно** и присвойте параметру **Accepted Resource Roles** (Допустимые роли ресурса) значение **slave\_public**. Этот процесс описан [здесь](container-service-mesos-marathon-ui.md#deploy-a-docker-formatted-container) и в [документации по DC/OS](https://dcos.io/docs/1.7/administration/installing/custom/create-public-agent/).

# Управление контейнерами через веб-интерфейс
   
Mesos — это среда для развертывания и масштабирования кластерных рабочих нагрузок, в которой используемое оборудование рассматривается абстрактно. Платформа на базе Mesos управляет планированием и выполнением вычислений. Хотя платформы доступны для многих популярных рабочих нагрузок, в этом документе мы подробно рассмотрим создание и масштабирование развертываний контейнеров с помощью Marathon. Прежде чем вы сможете работать с этими примерами, в службе контейнеров Azure (ACS) нужно настроить кластер Mesos и организовать к нему удаленный доступ. Сведения о том, как это сделать, см. в следующих статьях:

- [Развертывание кластера службы контейнеров Azure](./container-service-deployment.md). 
- [Подключение к кластеру службы контейнеров Azure](./container-service-connect.md).

## Изучение пользовательского интерфейса Mesos

Установив туннель SSH, перейдите по адресу http://localhost/Mesos. Отсюда будет загружен веб-интерфейс Mesos. На этой странице можно найти сведения о кластере Mesos, в том числе об активированных агентах, состоянии задач и доступности ресурсов.

![Создать развертывание](media/ui1.png)

## Изучение пользовательского интерфейса Marathon

Для просмотра пользовательского интерфейса Marathon перейдите по адресу http://localhost/Marathon. На этом экране можно запустить новый контейнер или другие приложения в кластере ACS Mesos, а также просмотреть информацию о работающих контейнерах и приложениях.

![Создать развертывание](media/ui2.png)

## Развертывание контейнера Docker

Чтобы запустить новый контейнер в кластере Mesos с помощью Marathon, нажмите кнопку `Create Application` (Создать приложение). Форма нового приложения используется для определения параметров приложения или контейнера. В этом примере будет развернут простой контейнер Nginx. Введите следующие сведения. По завершении нажмите кнопку Create (Создать).
 
Поле | Значение
----------------|-----------
ИД | nginx
Образ — | nginx
Сеть | Bridged (Мостовая)
Container Port (Порт контейнера) | 80
Host Port (Порт узла) | 80
Протокол | TCP

![Создать развертывание](media/ui3.png)

Вернитесь на главную страницу Marathon, чтобы просмотреть состояние развертывания контейнера.

![Создать развертывание](media/ui4.png)

При переключении обратно в приложение Mesos (http://localhost/Mesos) вы увидите, что задача — в данном случае контейнер Docker — выполняется в кластере Mesos. Также вы увидите узел кластера, на котором выполняется задача.

![Создать развертывание](media/ui5.png)

## Масштабирование контейнера Docker

С помощью веб-интерфейса Marathon также можно масштабировать число экземпляров контейнера. Для этого перейдите на страницу Marathon, выберите контейнер, который требуется масштабировать, и нажмите кнопку `scale` (Масштабирование). В окне Scale Application (Масштабирование приложения) введите нужное число экземпляров контейнера, а затем выберите `Scale Application` (Масштабировать приложение).

![Создать развертывание](media/ui6.png)

По завершении операции масштабирования вы увидите несколько экземпляров одной и той же задачи, распределенных между агентами Mesos.

![Создать развертывание](media/ui8.png)

# Управление контейнерами через веб-интерфейс

Mesos — это среда для развертывания и масштабирования кластерных рабочих нагрузок, в которой используемое оборудование рассматривается абстрактно. Платформа на базе Mesos управляет планированием и выполнением вычислительных рабочих нагрузок.

Хотя платформы доступны для многих популярных рабочих нагрузок, в этом документе мы рассмотрим создание и масштабирование развертываний контейнеров с помощью Marathon. Для выполнения этих примеров вам потребуется кластер Mesos, настроенный в службе контейнеров Azure. Необходимо также удаленное подключение к этому кластеру. Сведения о том, как это сделать, см. в следующих статьях:

- [Развертывание кластера службы контейнеров Azure](./container-service-deployment.md).
- [Подключение к кластеру службы контейнеров Azure](./container-service-connect.md).

## Изучение пользовательского интерфейса Mesos

Установив туннель Secure Shell (SSH), перейдите по адресу http://localhost/Mesos. Отсюда будет загружен веб-интерфейс Mesos. На этой странице можно найти сведения о кластере Mesos, в том числе об активированных агентах, состоянии задач и доступности ресурсов.

![Создание развертывания — пользовательский интерфейс 1](media/ui1.png)

## Изучение пользовательского интерфейса Marathon

Для просмотра пользовательского интерфейса Marathon перейдите по адресу http://localhost/Marathon. На этом экране можно запустить новый контейнер или другие приложения в кластере Mesos службы контейнеров Azure. Здесь вы также можете просмотреть информацию о работе контейнеров и приложений.

![Создание развертывания — пользовательский интерфейс 2](media/ui2.png)

## Развертывание контейнера формата Docker

Чтобы запустить новый контейнер в кластере Mesos с помощью Marathon, нажмите кнопку **Создать приложение**. Форма **нового приложения** используется для определения параметров приложения или контейнера. В этом примере вы развернете простой контейнер Nginx. Введите следующие сведения. Когда вы сделаете это, нажмите кнопку **Создать**.

Поле | Значение
----------------|-----------
ИД | nginx
Образ — | nginx
Сеть | Bridged (Мостовая)
Container Port (Порт контейнера) | 80
Host Port (Порт узла) | 80
Протокол | TCP

![Создание развертывания — пользовательский интерфейс 3](media/ui3.png)

Вернитесь на главную страницу Marathon, чтобы просмотреть состояние развертывания контейнера.

![Создание развертывания — пользовательский интерфейс 4](media/ui4.png)

При переключении обратно в приложение Mesos (http://localhost/Mesos) вы увидите, что задача — в данном случае контейнер формата Docker — выполняется в кластере Mesos. Также вы увидите узел кластера, на котором выполняется задача.

![Создание развертывания — пользовательский интерфейс 5](media/ui5.png)

## Масштабирование контейнеров

С помощью веб-интерфейса Marathon также можно масштабировать количество экземпляров контейнера. Для этого перейдите на страницу Marathon, выберите контейнер, который требуется масштабировать, и нажмите кнопку **Масштабировать**. В диалоговом окне **Масштабирование приложения** введите нужное количество экземпляров контейнера, а затем выберите **Масштабировать приложение**.

![Создание развертывания — пользовательский интерфейс 6](media/ui6.png)

Когда операция масштабирования завершится, вы увидите несколько экземпляров одной и той же задачи, распределенных между агентами Mesos.

![Создание развертывания — пользовательский интерфейс 7](media/ui8.png)

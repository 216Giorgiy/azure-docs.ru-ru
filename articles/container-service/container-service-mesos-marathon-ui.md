# Управление контейнерами через веб-интерфейс

DC/OS — это среда для развертывания и масштабирования кластерных рабочих нагрузок, в которой используемое оборудование рассматривается абстрактно. На базе DC/OS работает платформа, которая управляет планированием и выполнением вычислительных рабочих нагрузок.

Хотя платформы доступны для многих популярных рабочих нагрузок, в этом документе мы рассмотрим создание и масштабирование развертываний контейнеров с помощью Marathon. Для выполнения этих примеров вам потребуется кластер DC/OS, настроенный в службе контейнеров Azure. Необходимо также удаленное подключение к этому кластеру. Сведения о том, как это сделать, см. в следующих статьях:

- [Развертывание кластера службы контейнеров Azure](./container-service-deployment.md).
- [Подключение к кластеру службы контейнеров Azure](./container-service-connect.md).

## Изучение пользовательского интерфейса DC/OS

Установив туннель Secure Shell (SSH), перейдите по адресу http://localhost/. Будет загружен веб-интерфейс DC/OS, а затем вы увидите сведения о кластере, в том числе об используемых ресурсах, активных агентах и запущенных службах.

![](media/dcos/dcos2.png)

## Изучение пользовательского интерфейса Marathon

Для просмотра пользовательского интерфейса Marathon перейдите по адресу http://localhost/Marathon. На этом экране можно запустить новый контейнер или другие приложения в кластере DC/OS службы контейнеров Azure. Здесь вы также можете просмотреть информацию о работе контейнеров и приложений.

![](media/dcos/dcos3.png)

## Развертывание контейнера формата Docker

Чтобы развернуть новый контейнер с помощью Marathon, нажмите кнопку **Создать приложение** и заполните форму следующим образом. Когда закончите, щелкните **Создать приложение**.

Поле | Значение
----------------|-----------
ИД | nginx
Образ — | nginx
Сеть | Bridged (Мостовая)
Host Port (Порт узла) | 80
Протокол | TCP

![](media/dcos/dcos4.png)

![](media/dcos/dcos5.png)

![](media/dcos/dcos6.png)

Статически сопоставить порт контейнера с портом на агенте можно в режиме JSON. Для этого с помощью переключателя перейдите в мастере создания приложений в режим JSON, а затем введите следующее определение приложения в разделе portMappings. В этом примере порт 80 контейнера привязывается к порту 80 агента DC/OS. Когда изменение будет внесено, мастер можно перевести в другой режим.

```none
“hostPort”: 80,
```

![](media/dcos/dcos13.png)

Кластер DC/OS развертывается с использованием набора частных и общедоступных агентов. Чтобы получить доступ к приложению из Интернета, развертывание должно быть выполнено на общедоступном агенте. Для этого в мастере создания приложений перейдите на вкладку с дополнительными параметрами и присвойте параметру "Принятые роли ресурсов" значение slave\_public.

![](media/dcos/dcos14.png)

Вернитесь на главную страницу Marathon, чтобы просмотреть состояние развертывания контейнера.

![](media/dcos/dcos7.png)

При переключении обратно в приложение DC/OS (http://localhost/) вы увидите, что задача (в нашем случае это контейнер в формате Docker) выполняется в кластере DC/OS.

![](media/dcos/dcos8.png)

Также вы увидите узел кластера, на котором выполняется задача.

![](media/dcos/dcos9.png)

## Масштабирование контейнеров

С помощью интерфейса Marathon можно масштабировать количество экземпляров контейнера. Для этого перейдите на страницу Marathon, выберите контейнер, который требуется масштабировать, и нажмите кнопку **Масштабировать**. В диалоговом окне **Масштабирование приложения** введите нужное количество экземпляров контейнера, а затем выберите **Масштабировать приложение**.

![](media/dcos/dcos10.png)

Когда операция масштабирования завершится, вы увидите несколько экземпляров одной и той же задачи, распределенных между агентами DC/OS.

![](media/dcos/dcos11.png)

![](media/dcos/dcos12.png)

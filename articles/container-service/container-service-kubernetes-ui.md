# <a name="microsoft-azure-container-service-engine---using-the-kubernetes-web-ui"></a>Обработчик службы контейнеров Microsoft Azure. Использование веб-интерфейса Kubernetes

## <a name="prerequisites"></a>Предварительные требования
В этом пошаговом руководстве предполагается, что вы [создали кластер Kubernetes с помощью службы контейнеров Azure](container-service-kubernetes-walkthrough.md).

> [!NOTE]
> Поддержка Kubernetes в службе контейнеров Azure сейчас доступна в предварительной версии.
>

Также предполагается, что у вас установлен интерфейс командной строки Azure и средство kubectl.

Чтобы проверить наличие средства `az`, выполните такую команду:

```console
$ az --version
```

Если средство `az` не установлено, следуйте инструкциям, приведенным [здесь](https://github.com/azure/azure-cli#installation).

Чтобы проверить наличие средства `kubectl`, выполните такую команду:

```console
$ kubectl version
```

Если средство `kubectl` не установлено, выполните команду:

```console
$ az acs kubernetes install-cli
```

## <a name="overview"></a>Обзор

### <a name="connect-to-the-web-ui"></a>Подключение к веб-интерфейсу
Для запуска веб-интерфейса Kubernetes используйте такую команду:

```console
$ az acs kubernetes browse
```

В результате откроется веб-браузер, настроенный на подключение к защищенному прокси-серверу, который установит подключение вашего локального компьютера к веб-интерфейсу Kubernetes.

### <a name="create-and-expose-a-service"></a>Создание и предоставление службы
В правом верхнем углу веб-интерфейса Kubernetes расположена кнопка Create (Создать).

![Кнопка Create (Создать) в веб-интерфейсе Kubernetes](media/k8s/create.png)

При нажатии на эту кнопку откроется диалоговое окно, в котором можно приступить к созданию приложения.
Присвойте ему имя `hello-nginx`. Используйте [контейнер `nginx` из Docker](https://hub.docker.com/_/nginx/) и разверните три реплики этой веб-службы.

![Диалоговое окно создания модуля Kubernetes](media/k8s/nginx.png)

Затем создайте внешнюю службу Kubernetes для балансировки трафика на три созданные реплики.  Выберите вариант External (Внешняя) и укажите порт 80.

![Диалоговое окно создания службы Kubernetes](media/k8s/service.png)

Наконец нажмите кнопку Deploy (Развернуть), чтобы развернуть эти контейнеры и службы.

![Развертывание Kubernetes](media/k8s/deploy.png)

### <a name="view-your-containers"></a>Просмотр контейнеров
После нажатия кнопки Deploy (Развернуть) в пользовательском интерфейсе отобразится развертывание службы:

![Состояние Kubernetes](media/k8s/status.png)

В круге в левой части пользовательского интерфейса отобразится состояние каждого объекта Kubernetes. Если круг не замкнут, развертывание объекта еще не завершено. Когда объект будет полностью развернут, отобразится зеленая галочка:

![Объект Kubernetes развернут](media/k8s/deployed.png)

Когда все компоненты будут работать, можно щелкнуть один из модулей и просмотреть подробные сведения о запущенной веб-службе.

![Модули Kubernetes](media/k8s/pods.png)

В представлении конкретного модуля можно просмотреть информацию о контейнерах в нем, а также ресурсах ЦП и памяти, используемых этими контейнерами:

![Ресурсы Kubernetes](media/k8s/resources.png)

Если ресурсы не отображаются, возможно, следует подождать несколько минут для распространения данных мониторинга.

Кроме того, можно щелкнуть ссылку Logs (Журналы), чтобы просмотреть журналы для контейнера:

![Журналы Kubernetes](media/k8s/logs.png)

### <a name="viewing-your-service"></a>Просмотр службы
Помимо выполнения контейнеров, в пользовательском интерфейсе Kubernetes была создана внешняя служба `Service`, которая подготавливает подсистему балансировки нагрузки для направления трафика на контейнеры в кластере.

Для просмотра всех служб в левой области навигации щелкните элемент Services (Службы) (сейчас должна отображаться только одна служба).

![Службы Kubernetes](media/k8s/service-deployed.png)

В этом представлении должен отображаться выделенный для службы внешний IP-адрес.
Если щелкнуть этот IP-адрес, отобразится контейнер nginx, выполняющийся за подсистемой балансировки нагрузки.

![Представление nginx](media/k8s/nginx-page.png)

### <a name="resizing-your-service"></a>Изменение размеров службы
Кроме просмотра объектов в пользовательском интерфейсе, можно также изменять и обновлять объекты Kubernetes API.

Сначала перейдите к компоненту `Deployment` для службы, щелкнув в левой области навигации элемент Deployments (Развертывания).

В этом представлении выберите набор реплик и нажмите кнопку Edit (Изменить) на верхней панели навигации:

![Окно изменения в Kubernetes](media/k8s/edit.png)

Измените поле `spec.replicas`, указав значение 2, и нажмите кнопку Update (Обновить).

В результате один из модулей будет удален, а число реплик уменьшится до двух.

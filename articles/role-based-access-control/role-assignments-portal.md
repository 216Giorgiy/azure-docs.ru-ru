# <a name="manage-access-using-rbac-and-the-azure-portal"></a>Управление доступом с помощью RBAC и портала Azure

[Управление доступом на основе ролей (RBAC)](overview.md) — это способ управления доступом к ресурсам в Azure. В этой статье описывается, как управлять доступом пользователей, групп и приложений, используя RBAC и портал Azure.

## <a name="list-roles"></a>Вывод списка ролей

Определение роли — это коллекция разрешений, используемых для назначения ролей. В Azure насчитывается более 70 [встроенных ролей](built-in-roles.md). Выполните следующие действия, чтобы получить список ролей на портале.

1. На портале Azure щелкните **Все службы** и **Подписки**.

1. Выберите свою подписку.

1. Выберите **Управление доступом (IAM)**.

   ![Параметр "Роли"](./media/role-assignments-portal/list-subscription-access-control.png)

1. Чтобы просмотреть список всех встроенных и пользовательских ролей, выберите **Роли**.

   ![Параметр "Роли"](./media/role-assignments-portal/roles-option.png)

   Вы увидите, сколько пользователей и групп назначено каждой роли.

   ![Список ролей](./media/role-assignments-portal/roles-list.png)

## <a name="list-access"></a>Вывод списка доступа

При управлении доступом необходимо знать, кому, с какими разрешениями и на каком уровне предоставлен доступ. Чтобы узнать, кому предоставлен доступ, вам нужно получить список назначений ролей. Выполните следующие действия, чтобы получить список прав доступа пользователей и список доступа к различным областям.

### <a name="list-role-assignments-for-a-user"></a>Список назначений ролей для пользователя

1. В списке переходов выберите **Azure Active Directory**.

1. Щелкните **Пользователи**, чтобы открыть колонку **Все пользователи**.

   ![Колонка "Все пользователи" в Azure Active Directory](./media/role-assignments-portal/aad-all-users.png)

1. Выберите одного пользователя в списке.

1. В разделе **Управление** выберите **Ресурсы Azure**.

   ![Пользователь Azure Active Directory в колонке "Ресурсы Azure"](./media/role-assignments-portal/aad-user-azure-resources.png)

   В колонке "Ресурсы Azure" отображаются назначения ролей для выбранного пользователя и выбранной подписки. В этом списке содержатся только назначения ролей для ресурсов, для которых у вас есть разрешение на чтение. Например, если пользователю назначена какая-либо роль, для которой у вас нет разрешения на чтение, это назначение роли не будет отображаться в списке.

1. Если у вас несколько подписок, вы можете выбрать раскрывающийся список **Подписка**, чтобы просмотреть назначения ролей в другой подписке.

### <a name="list-role-assignments-for-a-resource-group"></a>Вывод списка назначений ролей для группы ресурсов

1. В списке переходов выберите **Группы ресурсов**.

1. Выберите группу ресурсов, а затем щелкните **Управление доступом (IAM)**.

   В колонке "Управление доступом (IAM)" (также известно под названием "Управление удостоверением и доступом") отображаются все, у кого есть доступ к этой группе ресурсов. Обратите внимание, что одни роли привязаны к **этому ресурсу**, а другие **унаследованы** из другой области. Доступ специально назначается группе ресурсов или наследуется в результате назначения родительской подписке.

   ![Группы ресурсов](./media/role-assignments-portal/resource-group-access-control.png)

### <a name="list-role-assignments-for-a-subscription"></a>Получение списка назначений ролей для подписки

1. На портале Azure щелкните **Все службы** и **Подписки**.

1. Выберите свою подписку.

1. Выберите **Управление доступом (IAM)**.

    В колонке "Управление доступом (IAM)" отображаются все, у кого есть доступ к этой подписке и пользовательской роли.

    ![Колонка "Управление доступом (IAM)" для подписки](./media/role-assignments-portal/subscription-access-control.png)

    Классические администраторы и соадминистраторы подписки считаются владельцами подписки в модели RBAC.

### <a name="list-role-assignments-for-a-management-group"></a>Список назначений ролей для группы управления

1. На портале Azure щелкните **Все службы**, **Группы управления**.

1. Выберите группу управления.

1. Выберите **(подробности)** для выбранной группы управления.

    ![Группы управления](./media/role-assignments-portal/management-groups-list.png)

1. Выберите **Управление доступом (IAM)**.

    В колонке "Управление доступом (IAM)" отображаются все, у кого есть доступ к этой группе управления и роли.

    ![Колонка "Управление доступом (IAM)" для группы управления](./media/role-assignments-portal/management-groups-access-control.png)

## <a name="grant-access"></a>Предоставление доступа

При использовании RBAC, чтобы предоставить доступ, нужно создать назначение ролей. Выполните следующие действия, чтобы разрешить доступ к различным областям.

### <a name="create-a-role-assignment-at-a-resource-group-scope"></a>Создание назначения роли в области действия группы ресурсов

1. В списке переходов выберите **Группы ресурсов**.

1. Выберите группу ресурсов.

1. Выберите **Управление доступом (IAM)**, чтобы просмотреть текущий список назначений ролей в области действия группы ресурсов.

   ![Колонка "Управление доступом (IAM)" для группы ресурсов](./media/role-assignments-portal/grant-resource-group-access-control.png)

1. Выберите **Добавить**, чтобы открыть панель **Добавление разрешений**.

   Если у вас нет разрешений на назначение ролей, параметр **Добавить** не будет отображаться.

   ![Панель "Добавление разрешений"](./media/role-assignments-portal/add-permissions.png)

1. В раскрывающемся списке **Роли** выберите роль, например **Участник виртуальных машин**.

1. В списке **Выбор** выберите пользователя, группу или приложение. Если субъект безопасности не отображается в списке, введите его имя в поле **Выбор**, чтобы найти в каталоге отображаемые имена, адреса электронной почты и идентификаторы объектов.

1. Нажмите кнопку **Сохранить**, чтобы создать назначение роли.

   Через несколько секунд субъекту безопасности будет назначена роль в области действия группы ресурсов.

### <a name="create-a-role-assignment-at-a-subscription-scope"></a>Создание назначения роли на в области действия подписки

1. На портале Azure щелкните **Все службы** и **Подписки**.

1. Выберите свою подписку.

1. Выберите **Управление доступом (IAM)**, чтобы просмотреть текущий список назначений ролей в области действия подписки.

   ![Колонка "Управление доступом (IAM)" для подписки](./media/role-assignments-portal/grant-subscription-access-control.png)

1. Выберите **Добавить**, чтобы открыть панель **Добавление разрешений**.

   Если у вас нет разрешений на назначение ролей, параметр **Добавить** не будет отображаться.

   ![Панель "Добавление разрешений"](./media/role-assignments-portal/add-permissions.png)

1. В раскрывающемся списке **Роли** выберите роль, например **Участник виртуальных машин**.

1. В списке **Выбор** выберите пользователя, группу или приложение. Если субъект безопасности не отображается в списке, введите его имя в поле **Выбор**, чтобы найти в каталоге отображаемые имена, адреса электронной почты и идентификаторы объектов.

1. Нажмите кнопку **Сохранить**, чтобы создать назначение роли.

   Через несколько секунд субъекту безопасности будет назначена роль в области действия подписки.

### <a name="create-a-role-assignment-at-a-management-group-scope"></a>Создание назначения ролей в области действия группы управления

1. На портале Azure щелкните **Все службы**, **Группы управления**.

1. Выберите группу управления.

1. Выберите **(подробности)** для выбранной группы управления.

    ![Группы управления](./media/role-assignments-portal/management-groups-list.png)

1. Выберите **Управление доступом (IAM)**, чтобы просмотреть текущий список назначений ролей в области действия подписки.

   ![Колонка "Управление доступом (IAM)" для группы управления](./media/role-assignments-portal/grant-management-groups-access-control.png)

1. Выберите **Добавить**, чтобы открыть панель **Добавление разрешений**.

   Если у вас нет разрешений на назначение ролей, параметр **Добавить** не будет отображаться.

   ![Панель "Добавление разрешений"](./media/role-assignments-portal/add-permissions-management-groups.png)

1. В раскрывающемся списке **Роли** выберите роль, например **Участник группы управления**.

    Сведения о поддерживаемых действиях с группами управления для различных ролей см. в разделе [Доступ к группе управления](../azure-resource-manager/management-groups-overview.md#management-group-access).

1. В списке **Выбор** выберите пользователя, группу или приложение. Если субъект безопасности не отображается в списке, введите его имя в поле **Выбор**, чтобы найти в каталоге отображаемые имена, адреса электронной почты и идентификаторы объектов.

1. Нажмите кнопку **Сохранить**, чтобы создать назначение роли.

   Через несколько секунд субъекту безопасности будет назначена роль в области действия группы управления.

## <a name="remove-access"></a>Запрет доступа

В RBAC, чтобы удалить доступ, нужно удалить назначение роли. Чтобы закрыть доступ, выполните следующие действия.

### <a name="remove-a-role-assignment"></a>Удаление назначения ролей

1. Откройте колонку **Управление доступом (IAM)**, чтобы получить сведения о группе управления, подписке, группе ресурсов или ресурсе, что имеют назначение ролей, которое нужно удалить.

1. В списке назначений ролей добавьте флажок рядом с именем субъекта безопасности, для которого нужно удалить назначение ролей.

   ![Сообщение об удалении назначения роли](./media/role-assignments-portal/remove-role-assignment-select.png)

1. Нажмите кнопку **Удалить**.

   ![Сообщение об удалении назначения роли](./media/role-assignments-portal/remove-role-assignment.png)

1. В появившемся сообщении об удалении назначения роли нажмите кнопку **Да**.

    Унаследованные назначения ролей удалить невозможно. Если нужно удалить наследуемое назначение роли, сделайте это в области, в которой создано это назначение. В колонке **Область** возле элемента **(Унаследовано)** есть ссылка, по которой можно перейти к области, где была назначена эта роль. Перейдите к области, которая указана в этом разделе, чтобы удалить назначение роли.

   ![Сообщение об удалении назначения роли](./media/role-assignments-portal/remove-role-assignment-inherited.png)

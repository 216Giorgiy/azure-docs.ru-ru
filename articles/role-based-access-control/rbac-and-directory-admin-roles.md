# <a name="understanding-resource-access-in-azure"></a>Основные сведения о доступе к ресурсам в Azure

Управление доступом в Azure начинается с оплаты. Владелец учетной записи Azure, доступ к которой осуществляется через [Центр управления учетной записью Azure](https://account.azure.com), является администратором учетной записи (АУ). Подписки — это контейнеры для оплаты, которые в то же время обеспечивают защиту: у каждой подписки есть администратор службы, который может добавлять, удалять и изменять ресурсы Azure в этой подписке на [портале Azure](https://portal.azure.com/). По умолчанию администратором службы является администратор учетной записи. Но администратор учетной записи может изменить администратора службы в Центре управления учетной записью Azure.

<br><br>![Учетные записи Azure][1]

С подпиской также связан каталог. Каталог определяет набор пользователей. Это могут быть пользователи рабочих или учебных учетных записей, создавшие каталог, или внешние пользователи-гости. Подписки доступны подмножеству пользователей каталога, которых назначил администратор служб (АС) или соадминистратор (СА). Единственное исключение для обеспечения обратной совместимости: учетные записи Майкрософт (ранее Windows Live ID) можно назначать как учетные записи АС или СА, даже если они отсутствуют в каталоге.

<br><br>![Управление доступом в Azure][2]

Функции портала Azure позволяют администраторам служб, вошедшим с помощью учетной записи Майкрософт, изменять каталог, с которым связана подписка. Эта операция влияет на управление доступом в соответствующей подписке.

<br><br>![Простая последовательность входа пользователя][3]

В самом простом случае организация (например, Contoso) принудительно внедряет выставление счетов и управление доступом в один и тот же набор подписок. То есть каталог привязывается к подпискам, принадлежащим одной и той же учетной записи Azure. После успешного входа на портал Azure пользователи видят две коллекции ресурсов (на предыдущей иллюстрации выделены оранжевым цветом).

* Каталоги, в которых существует их учетная запись пользователя (созданная или добавленная как внешний субъект). Обратите внимание, что каталог, используемый для входа в систему, в данном случае не имеет значения, поэтому ваши каталоги будут отображаться всегда, где бы вы ни выполняли вход.
* Ресурсы, составляющие часть подписок, связанных с каталогом, который используется для входа в систему и к которому пользователь может получить доступ (будь то АС или СА).

<br><br>![Пользователь с несколькими подписками и каталогами][4]

Пользователи с подписками в нескольких каталогах могут переключать текущий контекст портала Azure, используя для этого фильтр подписок. При этом осуществляется отдельный вход в другой каталог, хотя за счет единого входа пользователь этого не замечает.

В связи с единым видом каталога такие операции, как перемещение ресурсов между подписками, могут усложняться. Чтобы выполнить перенос ресурсов, может потребоваться впервые использовать команду **Изменить каталог** на странице "Подписки" диалогового окна **Параметры**, чтобы привязать подписки к одному и тому же каталогу.

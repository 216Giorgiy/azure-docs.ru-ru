# <a name="view-service-health-notifications-by-using-the-azure-portal"></a>Просмотреть уведомления о работоспособности службы с помощью портала Azure

Уведомления о работоспособности службы публикуются в Azure и содержит сведения о ресурсах в подписке. Эти уведомления — это вложенный класс действия журнала событий и можно также найти в журнале действий. Уведомления о работоспособности службы может быть информационное или пригодных к использованию, в зависимости от класса.

Существуют различные классы уведомления о работоспособности службы:  

- **Требуется действие:** Azure заметить, что-то необычное происходит в вашей учетной записи и сотрудничать с вами, чтобы исправить это. Azure отправляет уведомление, либо с подробным описанием действий, которые необходимо выполнить, или как связаться со службой Azure engineering или в службу поддержки.  
- **Вспомогательная восстановления:** произошло событие и инженеров подтвердили, что проблема влияние. Инженерному отделу Azure потребуется работать с вами напрямую, чтобы полностью восстановить работоспособность служб.  
- **Инцидент:** событие, которое оказывает влияние на службы в настоящее время это сказывается на один или несколько ресурсов в вашей подписке.  
- **Обслуживание:** действия планового обслуживания, которое может повлиять на один или несколько ресурсов в вашей подписке.  
- **Сведения:** потенциальные оптимизации, которые могут помочь улучшить использование ресурсов. 
- **Безопасность:** актуальные сведения, касающиеся безопасности, о решениях, работающих в Azure.

Каждое уведомление о работоспособности службы содержит сведения об области воздействия и влиянии на ресурсы. Вы сможете ознакомиться со следующими сведениями.

Имя свойства | ОПИСАНИЕ
-------- | -----------
каналов | Одно из следующих значений: **администратора** или **операции**.
correlationId | Обычно GUID в строковом формате. События, принадлежащие те же действия обычно используют тот же correlationId.
eventDataId | Уникальный идентификатор события.
eventName | Заголовок события.
level | Уровень события. Одно из следующих значений: **критическое**, **ошибка**, **предупреждение**, **Информационное**, или **Verbose** .
resourceProviderName | Имя поставщика ресурсов затронутого ресурса.
тип_ресурса| Тип затронутого ресурса.
subStatus | Обычно код состояния HTTP, соответствующие оставшейся вызова, но может также включать другие строки, описывающие подсостояния. Например: OK (код состояния HTTP: 200), созданный (код состояния HTTP: 201) приняты (код состояния HTTP: 202), нет содержимого (код состояния HTTP: 204), неправильный запрос (код состояния HTTP: 400), не найдено (код состояния HTTP: 404), конфликт (код состояния HTTP: 409), внутреннего сервера Ошибка (код состояния HTTP: 500), служба недоступна (код состояния HTTP: 503) и время ожидания шлюза (код состояния HTTP: 504).
eventTimestamp | Отметка времени создания события службой Azure при обработке запроса, соответствующий этому событию.
submissionTimestamp | Метка времени, когда событие стало доступно для запросов.
subscriptionId | Подписка Azure, в которой это событие было занесено в журнал.
status | Строка, описывающая состояние операции. Некоторые общие значения: **Started**, **выполняется**, **успешно**, **сбой**, **Active**, и **Разрешить**.
operationName | Имя операции.
category | Данное свойство всегда имеет **ServiceHealth**.
ResourceId | Идентификатор ресурса затронутого ресурса.
Properties.title | Локализованное название этого сообщения. По умолчанию используется английский язык.
Properties.communication | Локализованные сведения сообщения с разметкой HTML. По умолчанию используется английский язык.
Properties.incidentType | Одно из следующих значений: **AssistedRecovery**, **ActionRequired**, **сведения**, **инцидент**,  **Обслуживание**, или **безопасности**.
Properties.trackingId | Инцидент, с которым связано это событие. Используйте его для сопоставления событий, связанных с инцидентом.
Properties.impactedServices | Escape-JSON большой двоичный объект, описывающий службы и регионы, повлияли инцидента. Свойство содержит список служб, каждый из которых имеет **ServiceName**и список затрагиваемых областей, каждый из которых имеет **имя_области**.
Properties.defaultLanguageTitle | Взаимодействие на английском языке.
Properties.defaultLanguageContent | Взаимодействие на английском языке, как обычный текст или HTML-разметка.
Properties.stage | Возможные значения параметра **AssistedRecovery**, **ActionRequired**, **сведения**, **инцидент**, и **безопасности**  , **Active** или **разрешено**. Для **обслуживания**, они являются: **Active**, **Planned**, **InProgress**, **отменено**, **Перепланировано**, **Разрешить**, или **завершения**.
Properties.communicationId | Связи, с которым связано это событие.


## <a name="view-your-service-health-notifications-in-the-azure-portal"></a>Просмотреть уведомления по работоспособности службы на портале Azure
1.  На [портале Azure](https://portal.azure.com) выберите **Монитор**.

    ![Снимок экрана Azure меню портала с выбран монитор](./media/monitoring-service-notifications/home-monitor.png)

    В Azure Monitor отображено единое представление всех параметров мониторинга и данных. Сначала открывается раздел **Журнал действий** .

3.  Выберите **Оповещения**.

    ![Снимок экрана отслеживать активность журнала, с выбранные оповещения](./media/monitoring-service-notifications/service-health-summary.png)
4. Выберите **+ добавить действие журнала оповещение**и настроить оповещение, чтобы обеспечить уведомление для будущих уведомлений. Дополнительные сведения см. в разделе [создать действие предупреждения журнала службы уведомления о](monitoring-activity-log-alerts-on-service-notifications.md).

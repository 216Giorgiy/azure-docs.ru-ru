# <a name="create-and-manage-action-groups-in-azure-portal"></a>Создание групп действий и управление ими на портале Azure
## <a name="overview"></a>Обзор ##
В этой статье показано, как создавать группы действий и управлять ими на портале Azure.

Группы действий позволяют настроить список действий. Эти группы можно затем использовать при определении оповещений журнала действий. Таким образом определенная группа действий будет вызываться при активации оповещения журнала действий.

Группа действий может содержать до 10 действий каждого типа. Действие определяется сочетанием следующих параметров.

**Имя:** уникальный идентификатор в группе действий.  
**Тип действия:** определяет выполняемое действие. Доступные варианты — отправить SMS, отправить сообщение электронной почты или вызов Webhook.  
**Подробные сведения:** в зависимости от типа действия нужно указать контактный номер телефона, адрес электронной почты или указатель URI Webhook.

Дополнительные сведения о настройке групп действий с помощью шаблонов Azure Resource Manager см. в статье [Создание группы действий с помощью шаблона Resource Manager](monitoring-create-action-group-with-resource-manager-template.md).

## <a name="creating-an-action-group-using-the-azure-portal"></a>Создание группы действий с помощью портала Azure ##
1.  На [портале](https://portal.azure.com) перейдите на вкладку **Монитор** службы.

    ![Монитор](./media/monitoring-action-groups/home-monitor.png)
2.  Щелкните **Мониторинг**, чтобы открыть колонку "Мониторинг". В этой колонке отображено единое представление всех параметров мониторинга и данных. Сначала открывается раздел **Журнал действий** .

3.  Теперь щелкните раздел **Группы действий**.

    ![Action-Group](./media/monitoring-action-groups/action-groups-blade.png)
4.  Выберите команду **Добавить** группу действий и заполните поля.

    ![Add-Action-Group](./media/monitoring-action-groups/add-action-group.png)
5.  Укажите **Имя** и **Короткое имя** группы действий; короткое имя будет указываться в уведомлениях, отправленных в эту группу.

      ![Action-Group-Define](./media/monitoring-action-groups/action-group-define.png)

6.  Группа действий будет сохранена в поле **Подписка**. Это поле должно автоматически заполняться сведениями для текущей подписки.

7.  Выберите **группу ресурсов**, с которой это оповещение будет сопоставлено в **подписке**.

8.  Затем укажите список действий, выбрав сочетание следующих параметров.
  1. **Имя:** уникальный идентификатор в группе действий.
  2. **Тип действия:** определяет выполняемое действие. Доступные варианты — отправить SMS, отправить сообщение электронной почты или вызов Webhook.
  3. **Подробные сведения:** в зависимости от типа действия нужно указать контактный номер телефона, адрес электронной почты или указатель URI Webhook.

9.  Завершив настройку, нажмите кнопку **ОК**, чтобы создать группу действий.

## <a name="managing-your-action-groups"></a>Управление группами действий ##
После создания группа действий будет видна в разделе "Группы действий" службы мониторинга. Выберите группу действий, которой нужно управлять, чтобы получить возможность:
* добавлять, изменять или удалять действия;
-   удалить группу действий.

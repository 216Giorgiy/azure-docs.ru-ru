# <a name="list-of-azure-diagnostics-versions"></a>Список версий систем диагностики Azure
На этой странице приведен указатель версий схемы системы диагностики Azure, которые поставляются в составе пакета Microsoft Azure SDK.  

> [!NOTE]
> Система диагностики Azure — это компонент, который используется для сбора данных счетчиков производительности и других статистических данных из виртуальных машин Azure, масштабируемых наборов виртуальных машин, Service Fabric и облачных служб.  Данная страница применяется только в том случае, если вы используете одну из этих служб.
>

Система диагностики Azure используется в сочетании с другими продуктами диагностики корпорации Майкрософт, такими как Azure Monitor, Application Insights и Log Analytics.

## <a name="azure-sdk-and-diagnostics-versions-shipping-chart"></a>Диаграмма версий пакета SDK для Azure и системы диагностики  

|Версия пакета SDK для Azure | Версия системы диагностики Azure | Модель|  
|------------------|---------------------------|------|  
|1.x               |1.0                         | Подключаемый модуль|  
|2.0–2.4         |1.0                         |"|  
|2.5               |1.2                         |Расширение|  
|2.6               |1,3                         |"|  
|2.7               |1.4                         |"|  
|2.8               |1.5                         |"|  

 Система диагностики Azure версии 1.0 сначала поставлялась как подключаемый модуль, устанавливаемый вместе с пакетом SDK для Azure.  

 Начиная с пакета SDK 2.5 (версия системы диагностики 1.2) для системы диагностики Azure стала использоваться модель расширения. Инструменты, использующие новые функции, были доступны только в более новых пакетах SDK для Azure, но любая облачная служба или виртуальная машина, использующая систему диагностики, имела возможность выбрать последнюю доступную версию прямо в Azure.  

 Например, любой пользователь, все еще использующий пакет SDK 2.5, будет загружать систему диагностики версии 1.5, даже если он не применяет новые функции.

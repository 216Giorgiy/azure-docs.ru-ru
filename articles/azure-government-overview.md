# <a name="welcome-to-azure-government"></a>Приветствие Azure для государственных организаций
## <a name="overview"></a>Обзор
Microsoft Azure для государственных организаций предоставляет облачную платформу, основанную на [фундаментальных принципах обеспечения безопасности, конфиденциальности и управления, соответствия требованиям и прозрачности](http://azure.com/gov). Субъекты государственного сектора получают физически изолированный экземпляр Microsoft Azure, который использует [службы обеспечения соответствия требованиям](https://azure.microsoft.com/support/trust-center/compliance/) и безопасности мирового уровня, которые критически важны для всех систем и приложений правительства США, построенных на основе данной архитектуры. В число этих служб входят сертификаты соответствия FedRAMP и министерства обороны США, соглашения на уровне штата Информационной службы криминальной юстиции (CJIS), возможность издавать соглашения с бизнес-партнерами HIPAA, а также соблюдение положений документа IRS 1075. Операторами Azure для государственных организаций работают граждане США, прошедшие проверку. Поддерживаются несколько гибридных сценариев для построения и развертывания решений в локальной среде или в облаке. Субъекты государственного сектора могут также воспользоваться преимуществами мгновенной масштабируемости и гарантируемым временем непрерывной работы крупномасштабной облачной службы.

Azure для государственных организаций содержит основные компоненты инфраструктуры как услуги (IaaS), платформы как услуги (PaaS) и программного обеспечения как услуги (SaaS).  В их число входит инфраструктура, сеть, хранилище, управление данными, управление удостоверениями, а также другие многочисленные службы.

Azure для государственных организаций поддерживает большинство из исключительно хорошо зарекомендовавших себя возможностей, которыми пользуются клиенты общедоступной версии Azure, например, синхронную георепликация данных и автомасштабирование. 

* Посетите [страницу регионов](https://azure.microsoft.com/regions/#services), на которой размещается самая последняя информация об общедоступных службах.
* [Регистрация для получения пробной версии](https://azuregov.microsoft.com/trial/azuregovtrial)

Заинтересованные в облачных службах государственные организации США могут быть уверены в том, что платформа Azure Government обеспечивает исключительные возможности для масштабирования и поддерживает методы строгой безопасности для удовлетворения их постоянно изменяющихся потребностей.

## <a name="azure-government-documentation"></a>Документация по Azure для государственных организаций
Здесь описываются возможности служб [Microsoft Azure для государственных организаций](https://azure.microsoft.com/features/gov/) и приведены общие рекомендации для всех клиентов. Прежде чем добавлять специально контролируемые данные в подписке Azure для государственных организаций, следует ознакомиться с возможностями, предоставляемыми в рамках такой подписки. Если у вас возникнут вопросы, обратитесь в службу поддержки учетных записей.

Актуальные сведения о службах Azure для государственных организаций, соответствующих определенным нормам и нормативам, можно найти на [странице нормативных требований центра управления безопасностью Azure](http://www.microsoft.com/en-us/TrustCenter/Compliance/default.aspx) . Кроме того, могут быть доступны дополнительные службы Майкрософт, но они не входят в число служб Azure для государственных организаций и поэтому не рассматриваются в этом документе. Службы Azure для государственных организаций позволяют также использовать различные дополнительные ресурсы, приложения и службы, предоставляемые сторонними производителями или корпорацией Майкрософт на основании отдельных условий использования и политик конфиденциальности, которые не входят в рамки данного документа. Вы несете ответственность за проверку условий дополнительных предложений (например, предложений Marketplace) на предмет того, что они удовлетворяют требованиям по соответствию.

Службы Azure для государственных организаций доступны компаниям, занимающимся обработкой данных, к которым применяются государственные нормы и требования, например NIST 800.171 (DIB), ITAR, IRS 1075, DoD L4 и CJIS (если использование этих служб требуется для соблюдения норм). Клиенты Azure для государственных организаций проходят проверку на соответствие требованиям.

Если у вас есть вопросы о требованиях служб Azure для государственных организаций, вам следует обратиться к своей службе поддержки учетных записей.

## <a name="general-guidance-for-customers"></a>Общие рекомендации для клиентов
Так как большая часть доступных в настоящее время технических материалов исходит из того, что приложения разрабатываются для глобальной службы, а не для службы Microsoft Azure для государственных организаций, важно ознакомить разработчиков с основными отличиями приложений, разработанных специально для размещения в этой службе.

* Во-первых, есть различия в работе служб и функций, то есть определенные функции в некоторых регионах глобальной службы могут оказаться недоступными в Azure Government.
* Во-вторых, для функций, доступных в Azure Government, есть изменения в процессе конфигурации по отношению к глобальной службе.  Таким образом, необходимо изучить примеры кода, конфигурации и действия, необходимые для построения и выполнения приложений в среде облачных служб Azure для государственных организаций.
* В-третьих, следует ознакомиться с документацией технических служб Azure для государственных организаций, доступной на этом сайте. Она содержит информацию об области действия Azure для государственных организаций, а также инструкции и рекомендации по клиентским данным, подлежащим регулированию или контролю.

# <a name="routing-messages-with-iot-hub-node"></a>Маршрутизация сообщений с помощью Центра Интернета вещей (Node)

[!INCLUDE [iot-hub-selector-process-d2c](../../includes/iot-hub-selector-process-d2c.md)]

Это руководство создано на основе руководства [Подключение устройства к Центру Интернета вещей с помощью .NET].  Из этого руководства вы узнаете:

* как при помощи настроек системы можно использовать правила маршрутизации для перенаправления сообщений, отправляемых с устройства в облако;
* как выделять для дополнительной обработки интерактивные сообщения, которые требуют немедленных действий от серверной части решения.  Например, устройство может отправить аварийный сигнал, который активирует вставку билета в системе CRM.  При этом обычные сообщения от точек данных, такие как данные телеизмерения температуры, поступают в модуль аналитики.

С помощью этого руководства вы запустите три консольных приложения Node.js:

* **SimulatedDevice.js** — это измененная версия приложения, созданного в руководстве [Подключение устройства к Центру Интернета вещей с помощью .NET]. Приложение отправляет сообщения точек данных с устройства в облако каждую секунду, а интерактивные сообщения с устройства в облако — через произвольные интервалы. Это приложение использует протокол MQTT для обмена данными с Центром Интернета вещей.
* **ReadDeviceToCloudMessages.js** отображает данные телеметрии, отправленные приложением устройства.
* **ReadCriticalQueue.js** извлекает важные сообщения из очереди служебной шины, подключенной к Центру Интернета вещей.

> [!NOTE]
> В Центре Интернета вещей реализована поддержка для пакетов SDK для многих платформ устройств и языков (включая C, Java и Javascript). Инструкции по замене устройства физическим устройством, а также по подключению устройств к Центру Интернета вещей Azure см. в [центре разработчиков для Интернета вещей Azure].

Для работы с этим учебником требуется:

* Полная рабочая версия приведена в учебнике [Подключение устройства к Центру Интернета вещей с помощью .NET].
* Node.js 4.0.x или более поздней версии.
* Активная учетная запись Azure. Если ее нет, можно создать [бесплатную учетную запись][lnk-free-trial] всего за несколько минут.

Также рекомендуем ознакомиться с документацией по [службе хранилища Azure] и [службе "Служебная шина Azure"].

## <a name="send-interactive-messages-from-a-device-app"></a>Отправка интерактивных сообщений из приложения устройства
В этом разделе вы измените приложение устройства, созданное в руководстве [Подключение устройства к Центру Интернета вещей с помощью .NET], чтобы оно умело отправлять сообщения, требующие немедленной обработки.

1. Откройте в текстовом редакторе файл **simulateddevice\SimulatedDevice.js**. Этот файл содержит код приложения **SimulatedDevice**, созданного при изучении руководства [Подключение устройства к Центру Интернета вещей с помощью .NET].

2. Замените функцию **connectCallback** следующим кодом:

    ```nodejs
    var connectCallback = function (err) {
      if (err) {
        console.log('Could not connect: ' + err);
      } else {
        console.log('Client connected');
    
        // Create a message and send it to the IoT Hub every second
        setInterval(function(){
        var data, message;
        if (Math.random() > 0.7) {
            if (Math.random() > 0.5) {
                data = "This is a critical message.";
                message = new Message(data);
                message.properties.add("level", "critical");
            } else {
                data = "This is a storage message.";
                message = new Message(data);
                message.properties.add("level", "storage");
            }
            console.log("Sending message: " + message.getData());
        }
        else {
                var temperature = 20 + (Math.random() * 15);
                var humidity = 60 + (Math.random() * 20);            
                data = JSON.stringify({ deviceId: 'myFirstNodeDevice', temperature: temperature, humidity: humidity });
                message = new Message(data);
                message.properties.add('temperatureAlert', (temperature > 30) ? 'true' : 'false');
                console.log("Sending message: " + message.getData());
            }
        client.sendEvent(message, printResultFor('send'));
        }, 1000);
      }
    };
    ```
   
    Этот метод произвольно добавляет свойства `"level": "critical"` и `"level": "storage"` в сообщения, отправляемые устройством. Так мы имитируем сообщение, которое требует немедленных действий со стороны серверного приложения или которое необходимо долго хранить. Приложение поддерживает маршрутизацию сообщений на основе текста сообщения.
   
   > [!NOTE]
   > Свойства сообщений можно использовать для маршрутизации сообщений в разных ситуациях, включая обработку холодного пути. В нашем примере используется горячий путь.

2. Сохраните и закройте файл **simulateddevice\SimulatedDevice.js**.

    > [!NOTE]
    > Настоятельно рекомендуется реализовать политику повторных попыток (например, с использованием экспоненциальной задержки), как описано в статье MSDN об [обработке временных сбоев].

## <a name="add-service-bus-queue-to-your-iot-hub-and-route-messages-to-it"></a>Добавление очереди служебной шины в Центр Интернета вещей и маршрутизация сообщений в эту очередь

В этом разделе вы создадите очередь служебной шины, подключите ее к Центру Интернета вещей и укажете в нем настройки для отправки в эту очередь сообщений с определенным свойством. Дополнительные сведения о способах обработки сообщений в очередях служебной шины см. в руководстве [Начало работы с очередями служебной шины][lnk-sb-queues-node].

1. Создайте очередь служебной шины, как описано в статье [Начало работы с очередями служебной шины][lnk-sb-queues-node]. Запишите пространство имен и имя очереди.

    > [!NOTE]
    > Для очередей и разделов служебной шины, которые используются как конечные точки Центра Интернета вещей, **сеансы** или **поиск повторяющихся данных** должны быть выключены. Если один из этих параметров включен, конечная точка отображается как **недоступная** на портале Azure.

2. На портале Azure откройте Центр Интернета вещей и щелкните **Конечные точки**.

    ![Конечные точки в Центре Интернета вещей][30]

3. В колонке **Конечные точки** щелкните вверху **Добавить**, чтобы добавить свою очередь в Центр Интернета вещей. Присвойте конечной точке имя **CriticalQueue** и выберите в раскрывающемся списке **Очередь служебной шины** пространство имен служебной шины, в котором находится ваша очередь, а затем имя этой очереди. Когда все будет готово, в нижней части щелкните **ОК**.  

    ![Добавление конечной точки][31]

4. Теперь щелкните **Маршруты** в Центре Интернета вещей. Щелкните **Добавить** в верхней части колонки, чтобы создать правило маршрутизации, которое направляет сообщения в только что добавленную очередь. Выберите **Сообщения устройства** в качестве источника данных. Введите условие `level="critical"`, а затем выберите **CriticalQueue** в качестве пользовательской конечной точки этого правила маршрутизации. В нижней части щелкните **Сохранить**.  

    ![Добавление правила][32]

    Убедитесь, что **включено** использование резервного маршрута. Это значение является конфигурацией по умолчанию для Центра Интернета вещей.

    ![Резервный маршрут][33]

## <a name="optional-read-from-the-queue-endpoint"></a>Чтение сообщений из конечной точки очереди

В этом разделе вы создадите консольное приложение Node.js, которое считывает важные сообщения, передаваемые из служебной шины Центра Интернета вещей. Дополнительную информацию см. в документации по [началу работы с очередями][lnk-sb-queues-node]. 

1. Создайте пустую папку с именем `readcriticalqueue`. В папке `readcriticalqueue` создайте файл package.json, указав приведенную ниже команду в командной строке. Примите значения по умолчанию:

    ```cmd/sh
    npm init
    ```

2. В командной строке в папке `readcriticalqueue` выполните следующую команду, чтобы установить пакет **azure**:

    ```cmd/sh
    npm install azure --save
    ```

3. В папке `readcriticalqueue` в текстовом редакторе создайте файл **ReadCriticalQueue.js**.

4. Добавьте следующие инструкции `require` в начало файла **ReadCriticalQueue.js**:

    ```nodejs
    'use strict';

    var azure = require('azure');
    ```

5. Добавьте следующее объявление переменной и замените значения заполнителей именем очереди и строкой подключения служебной шины Центра Интернета вещей:

    ```nodejs
    var connectionString = '{iotservicebus connection string}';
    var queueName = '{iotservice bus queue name}';
    ```

6. Добавьте следующие две функции для вывода результатов на консоль:

    ```nodejs
    var serviceBusService = azure.createServiceBusService(connectionString);

    setInterval(function() {serviceBusService.receiveQueueMessage(queueName, function(error, receivedMessage) {
        if(!error){
            // Message received and deleted
            console.log(receivedMessage);
        } else { console.log(error); }
        });
    }, 3000);
    ```

7. Сохраните и закройте файл **ReadCriticalQueue.js**.

## <a name="run-the-applications"></a>Запуск приложений

Теперь все готово к запуску этих трех приложений.

1. Чтобы запустить приложение **ReadDeviceToCloudMessages.js**, в командной строке или оболочке перейдите к папке readdevicetocloudmessages и выполните следующую команду:

   ```cmd/sh
   node ReadDeviceToCloudMessages.js
   ```

   ![Запуск read-d2c-messages][readd2c]

2. Чтобы запустить приложение **ReadCriticalQueue.js**, в командной строке или оболочке перейдите к папке readcriticalqueue и выполните следующую команду:

   ```cmd/sh
   node ReadCriticalQueue.js
   ```
   
   ![Запуск read-critical-messages][readqueue]

3. Чтобы запустить приложение **SimulatedDevice.js**, в командной строке или оболочке перейдите к папке simulateddevice и выполните следующую команду:

   ```cmd/sh
   node SimulatedDevice.js
   ```
   
   ![Запуск приложения simulated-device][simulateddevice]

## <a name="optional-add-storage-container-to-your-iot-hub-and-route-messages-to-it"></a>(Необязательно.) Добавление контейнера службы хранилища в Центр Интернета вещей и маршрутизация сообщений в этот контейнер

В этом разделе вы создадите учетную запись хранения и подключите ее к Центру Интернета вещей, в котором настроите отправку сообщений с определенными свойствами в эту учетную запись. Дополнительные сведения об управлении хранилищем см. в документации по [началу работы со службой хранилища Azure][службе хранилища Azure].

 > [!NOTE]
   > Если вы можете использовать несколько **конечных точек**, кроме **CriticalQueue** можно настроить **StorageContainer**, чтобы работать с этими конечными точками одновременно.

1. Создайте учетную запись хранения, как описано в [документации по службе хранилища Azure][lnk-storage]. Запишите имя учетной записи.

2. На портале Azure откройте Центр Интернета вещей и щелкните **Конечные точки**.

3. В колонке **Конечные точки** выберите конечную точку **CriticalQueue** и щелкните **Удалить**. Щелкните **Да**, а затем — **Добавить**. Назовите конечную точку **StorageContainer** и выберите в раскрывающемся списке **Контейнер службы хранилища Azure**. Затем создайте **учетную запись хранения** и **контейнер службы хранилища**.  Запишите соответствующие имена.  Когда все будет готово, в нижней части щелкните **ОК**. 

 > [!NOTE]
   > Если вы можете использовать несколько **конечных точек**, удалять **CriticalQueue** не нужно.

4. Щелкните **Маршруты** в Центре Интернета вещей. Щелкните **Добавить** в верхней части колонки, чтобы создать правило маршрутизации, которое направляет сообщения в только что добавленную очередь. Выберите **Сообщения устройства** в качестве источника данных. Введите `level="storage"` в качестве условия и **StorageContainer** в качестве пользовательской конечной точки этого правила маршрутизации. В нижней части щелкните **Сохранить**.  

    Убедитесь, что **включено** использование резервного маршрута. Это значение является конфигурацией по умолчанию для Центра Интернета вещей.

1. Убедитесь, что предыдущее приложение **SimulatedDevice.js** все еще работает. 

1. На портале Azure войдите в свою учетную запись хранения, а затем в разделе **Службы BLOB-объектов** щелкните **Обзор BLOB-объектов**.  Выберите контейнер, перейдите к JSON-файлу и щелкните **Загрузить** для просмотра данных.

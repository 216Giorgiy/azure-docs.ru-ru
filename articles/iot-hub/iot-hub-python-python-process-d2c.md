# <a name="routing-messages-with-iot-hub-python"></a>Маршрутизация сообщений с помощью Центра Интернета вещей (Python)

[!INCLUDE [iot-hub-selector-process-d2c](../../includes/iot-hub-selector-process-d2c.md)]

Это руководство создано на основе руководства [Подключение устройства к Центру Интернета вещей с помощью .NET].  Из этого руководства вы узнаете:

* как при помощи настроек системы можно использовать правила маршрутизации для перенаправления сообщений, отправляемых с устройства в облако;
* как выделять для дополнительной обработки интерактивные сообщения, которые требуют немедленных действий от серверной части решения.  Например, устройство может отправить аварийный сигнал, который активирует вставку билета в системе CRM.  При этом обычные сообщения от точек данных, такие как данные телеизмерения температуры, поступают в модуль аналитики.

По завершении работы с этим руководством вы запустите два консольных приложения Python:

* **SimulatedDevice.py** — это измененная версия приложения, созданного в руководстве по [Подключение устройства к Центру Интернета вещей с помощью .NET]. Приложение отправляет сообщения точек данных с устройства в облако каждую секунду, а интерактивные сообщения с устройства в облако — через произвольные интервалы. Это приложение использует протокол MQTT для обмена данными с Центром Интернета вещей.
* **ReadCriticalQueue.py** извлекает важные сообщения из очереди служебной шины, подключенной к Центру Интернета вещей.

> [!NOTE]
> В Центре Интернета вещей реализована поддержка для пакетов SDK для многих платформ устройств и языков (включая C, Java и Javascript). Инструкции по замене устройства физическим устройством, а также по подключению устройств к Центру Интернета вещей Azure см. в [центре разработчиков для Интернета вещей Azure].

Для работы с этим учебником требуется:

* Полная рабочая версия приведена в учебнике [Подключение устройства к Центру Интернета вещей с помощью .NET].
* [Python 2.x или 3.x][lnk-python-download]. Обязательно используйте 32-разрядную или 64-разрядную версию установки согласно требованиям программы настройки. При появлении запроса во время установки обязательно добавьте Python в переменную среды соответствующей платформы. Если вы используете Python 2.x, может потребоваться [установка или обновление *pip* — системы управления пакетами Python][lnk-install-pip].
* При использовании ОС Windows потребуется [распространяемый пакет Visual C++][lnk-visual-c-redist], чтобы разрешить использовать встроенные библиотеки DLL из Python.
* [Node.js 4.0 или более поздней версии][lnk-node-download]. Обязательно используйте 32-разрядную или 64-разрядную версию установки согласно требованиям программы настройки. Это необходимо для установки [средства обозревателя Центра Интернета вещей][lnk-iot-hub-explorer].
* Активная учетная запись Azure. Если ее нет, можно создать [бесплатную учетную запись][lnk-free-trial] всего за несколько минут.

Также рекомендуем ознакомиться с документацией по [службе хранилища Azure] и [службе "Служебная шина Azure"].


## <a name="send-interactive-messages-from-a-device-app"></a>Отправка интерактивных сообщений из приложения устройства
В этом разделе вы измените приложение устройства, созданное в руководстве [Подключение устройства к Центру Интернета вещей с помощью .NET], чтобы оно умело отправлять сообщения, требующие немедленной обработки.

1. Откройте в текстовом редакторе файл **SimulatedDevice.py**. Этот файл содержит код приложения **SimulatedDevice**, созданного при изучении руководства [Подключение устройства к Центру Интернета вещей с помощью .NET].

2. Замените функцию **iothub_client_telemetry_sample_run** следующим кодом:

    ```python
    def iothub_client_telemetry_sample_run():

    try:
        client = iothub_client_init()
        print ( "IoT Hub device sending periodic messages, press Ctrl-C to exit" )
        message_counter = 0

        while True:
            random_seed = random.random()
            msg_txt_formatted = MSG_TXT % (AVG_WIND_SPEED + (random_seed * 4 + 2))
            # messages can be encoded as string or bytearray
            if (message_counter & 1) == 1:
                message = IoTHubMessage(bytearray(msg_txt_formatted, 'utf8'))
            else:
                message = IoTHubMessage(msg_txt_formatted)
            # optional: assign ids
            message.message_id = "message_%d" % message_counter
            message.correlation_id = "correlation_%d" % message_counter
            # optional: assign properties
            prop_map = message.properties()
            if random_seed > .5:
                if random_seed > .8:
                    prop_map.add("level", 'critical')
                else:
                    prop_map.add("level", 'storage')

            client.send_event_async(message, send_confirmation_callback, message_counter)
            print ( "IoTHubClient.send_event_async accepted message [%d] for transmission to IoT Hub." % message_counter )

            status = client.get_send_status()
            print ( "Send status: %s" % status )
            time.sleep(30)

            status = client.get_send_status()
            print ( "Send status: %s" % status )

            message_counter += 1

    except IoTHubError as iothub_error:
        print ( "Unexpected error %s from IoTHub" % iothub_error )
        return
    except KeyboardInterrupt:
        print ( "IoTHubClient sample stopped" )
    ```
   
    Этот метод произвольно добавляет свойства `"level": "critical"` и `"level": "storage"` в сообщения, отправляемые устройством. Так мы имитируем сообщение, которое требует немедленных действий со стороны серверного приложения или которое необходимо долго хранить. Приложение поддерживает маршрутизацию сообщений на основе текста сообщения.
   
   > [!NOTE]
   > Свойства сообщений можно использовать для маршрутизации сообщений в разных ситуациях, включая обработку холодного пути. В нашем примере используется горячий путь.

1. Сохраните и закройте файл **SimulatedDevice.py**.

    > [!NOTE]
    > Для упрощения в этом руководстве не реализуются какие-либо политики повтора. В рабочем коде следует реализовать политику повтора (например, экспоненциальную задержку), как предлагается в статье MSDN [Transient Fault Handling](Обработка временного сбоя).


## <a name="add-queues-to-your-iot-hub-and-route-messages-to-them"></a>Добавление очередей в Центр Интернета вещей и направление в них сообщений

В этом разделе вы создадите очередь служебной шины и учетную запись хранения, подключите их к Центру Интернета вещей и укажете в нем настройки для отправки сообщений с определенным свойством в эту очередь и всех сообщений в учетную запись хранения. Дополнительные сведения о том, как обрабатывать сообщения из очередей служебной шины, см. в руководстве [Как использовать очереди служебной шины с Python][lnk-sb-queues-node], а о том, как управлять хранилищем, — в [документации по службе хранилища Azure][службе хранилища Azure].

1. Создайте очередь служебной шины, как описано в статье [Начало работы с очередями служебной шины][lnk-sb-queues-node]. Запишите пространство имен и имя очереди.

    > [!NOTE]
    > Для очередей и разделов служебной шины, которые используются как конечные точки Центра Интернета вещей, **сеансы** или **поиск повторяющихся данных** должны быть выключены. Если один из этих параметров включен, конечная точка отображается как **недоступная** на портале Azure.

1. На портале Azure откройте Центр Интернета вещей и щелкните **Конечные точки**.

    ![Конечные точки в Центре Интернета вещей][30]

1. В колонке **Конечные точки** щелкните вверху **Добавить**, чтобы добавить свою очередь в Центр Интернета вещей. Присвойте конечной точке имя **CriticalQueue** и выберите в раскрывающемся списке **Очередь служебной шины** пространство имен служебной шины, в котором находится ваша очередь, а затем имя этой очереди. Когда все будет готово, в нижней части щелкните **ОК**.  

    ![Добавление конечной точки][31]

1. Теперь щелкните **Маршруты** в Центре Интернета вещей. Щелкните **Добавить** в верхней части колонки, чтобы создать правило маршрутизации, которое направляет сообщения в только что добавленную очередь. 

    ![Добавление правила][34]

    Введите имя и выберите **Сообщения устройства** в качестве источника данных. Выберите **CriticalQueue** в качестве пользовательской конечной точки этого правила маршрутизации и введите `level="critical"` в качестве строки запроса. В нижней части щелкните **Сохранить**.

    ![Сведения о маршруте][32]

    Убедитесь, что **включено** использование резервного маршрута. Это значение является конфигурацией по умолчанию для Центра Интернета вещей.

    ![Резервный маршрут][33]

## <a name="optional-read-from-the-queue-endpoint"></a>Чтение сообщений из конечной точки очереди

В этом разделе вы создадите консольное приложение Python, которое считывает критически важные сообщения, передаваемые из служебной шины Центра Интернета вещей. Дополнительную информацию см. в документации по [началу работы с очередями][lnk-sb-queues-node]. 

1. Откройте новое окно командной строки и установите пакет SDK устройства Центра Интернета вещей Azure для Python указанным ниже образом. После установки закройте окно командной строки.

    ```cmd/sh
    pip install azure-servicebus
    ```

    > [!NOTE]
    > Способы решения проблем с установкой пакета **azure-servicebus** и дополнительные параметры установки см. в документации по [пакету azure-servicebus для Python][lnk-python-service-bus].

1. Создайте файл с именем **ReadCriticalQueue.py**. Откройте его в редакторе или интегрированной среде разработки Python (например, IDLE).

1. Добавьте следующий код, чтобы импортировать необходимые модули из пакета SDK устройства.

    ```python
    from azure.servicebus import ServiceBusService, Message, Queue
    ```

1. Добавьте следующий код и замените заполнители данными подключения для служебной шины.

    ```python
    SERVICE_BUS_NAME = {serviceBusName}
    SHARED_ACCESS_POLICY_NAME = {sharedAccessPolicyName}
    SHARED_ACCESS_POLICY_KEY_VALUE = {sharedAccessPolicyKeyValue}
    QUEUE_NAME = {queueName}    
    ```

5. Добавьте следующий код, чтобы подключиться к служебной шине и считывать из нее данные: 

    ```python
    def setup_client():
        bus_service = ServiceBusService(
        service_namespace=SERVICE_BUS_NAME,
        shared_access_key_name=SHARED_ACCESS_POLICY_NAME,
        shared_access_key_value=SHARED_ACCESS_POLICY_KEY_VALUE)

        while True:
            msg = bus_service.receive_queue_message(QUEUE_NAME, peek_lock=False)
            print(msg.body)
    ```

1. Теперь добавьте функцию main. 

    ```python
    if __name__ == '__main__':
        setup_client()
    ```

1. Сохраните и закройте файл **ReadCriticalQueue.py**. Теперь все готово к запуску приложений.


## <a name="run-the-applications"></a>Запуск приложений

Теперь все готово к запуску приложений.

1. Откройте окно командной строки и установите обозреватель Центра Интернета вещей. 

    ```cmd/sh
    npm install -g iothub-explorer
    ```

1. В командной строке выполните следующую команду, чтобы начать мониторинг сообщений, отправляемых с устройства в облако. Используйте строку подключения Центра Интернета вещей в заполнителе после `--login`.

    ```cmd/sh
    iothub-explorer monitor-events [deviceId] --login "[IoTHub connection string]"
    ```

1. Откройте новое окно командной строки и перейдите к каталогу, содержащему файл **SimulatedDevice.py**.

1. Запустите файл **SimulatedDevice.py**, который периодически отправляет данные телеметрии в Центр Интернета вещей. 
   
    ```cmd/sh
    python SimulatedDevice.py
    ```

1. Чтобы запустить приложение **ReadCriticalQueue**, в командной строке или оболочке перейдите к файлу **ReadCriticalQueue.py** и выполните следующую команду:

   ```cmd/sh
   python ReadCriticalQueue.py
   ```

1. Просмотрите сообщения с устройства в командной строке, запущенной обозревателем Центра Интернета вещей, из предыдущего раздела. Просмотрите сообщения `critical` в приложении **ReadCriticalQueue**.

    ![Сообщения Python, отправляемые с устройства в облако][2]


## <a name="optional-add-storage-container-to-your-iot-hub-and-route-messages-to-it"></a>(Необязательно.) Добавление контейнера службы хранилища в Центр Интернета вещей и маршрутизация сообщений в этот контейнер

В этом разделе вы создадите учетную запись хранения и подключите ее к Центру Интернета вещей, в котором настроите отправку сообщений с определенными свойствами в эту учетную запись. Дополнительные сведения об управлении хранилищем см. в документации по [началу работы со службой хранилища Azure][службе хранилища Azure].

 > [!NOTE]
   > Для учетных записей Центра Интернета вещей, созданных в ценовой категории _Бесплатный F1_, доступна только одна **конечная точка**. Если вы можете использовать несколько **конечных точек**, кроме **CriticalQueue** можно настроить **StorageContainer**, чтобы работать с этими конечными точками одновременно.

1. Создайте учетную запись хранения, как описано в [документации по службе хранилища Azure][lnk-storage]. Запишите имя учетной записи.

2. На портале Azure откройте Центр Интернета вещей и щелкните **Конечные точки**.

3. В колонке **Конечные точки** выберите конечную точку **CriticalQueue** и щелкните **Удалить**. Щелкните **Да**, а затем — **Добавить**. Назовите конечную точку **StorageContainer** и выберите в раскрывающемся списке **Контейнер службы хранилища Azure**. Затем создайте **учетную запись хранения** и **контейнер службы хранилища**.  Запишите соответствующие имена.  Когда все будет готово, в нижней части щелкните **ОК**. 

 > [!NOTE]
   > Для учетных записей Центра Интернета вещей, созданных в ценовой категории _Бесплатный F1_, доступна только одна **конечная точка**. Если вы можете использовать несколько **конечных точек**, удалять **CriticalQueue** не нужно.

4. Щелкните **Маршруты** в Центре Интернета вещей. Щелкните **Добавить** в верхней части колонки, чтобы создать правило маршрутизации, которое направляет сообщения в только что добавленную очередь. Выберите **Сообщения устройства** в качестве источника данных. Введите `level="storage"` в качестве условия и **StorageContainer** в качестве пользовательской конечной точки этого правила маршрутизации. В нижней части щелкните **Сохранить**.  

    Убедитесь, что **включено** использование резервного маршрута. Это значение является конфигурацией по умолчанию для Центра Интернета вещей.

1. Убедитесь, что предыдущее приложение **SimulatedDevice.py** все еще работает. 

1. На портале Azure войдите в свою учетную запись хранения, а затем в разделе **Служба BLOB-объектов** нажмите кнопку **Обзор BLOB-объектов**.  Выберите контейнер, перейдите к JSON-файлу и щелкните **Загрузить** для просмотра данных.

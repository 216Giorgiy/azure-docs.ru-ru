# <a name="device-to-cloud-communications-guidance"></a>Руководство по обмену данными между устройством и облаком
Центр Интернета вещей предоставляет три варианта отправки сведений из приложения для устройства в серверную часть решения.

* [Сообщения, отправляемые с устройства в облако (D2C)][lnk-d2c] — используются для телеметрии и оповещений с учетом временных рядов.
* [Сообщаемые свойства][lnk-twins] — предназначены для передачи сведений о состоянии устройства, таких как доступные возможности, условия и состояние длительных рабочих процессов (например, конфигурация и обновления программного обеспечения).
* [Передача файлов][lnk-fileupload] — используется для отправки файлов мультимедиа и больших пакетов телеметрии, передаваемых периодическими подключаемыми устройствами или сжатых для экономии пропускной способности.

Ниже приведено подробное сравнение различных параметров обмена данными между устройством и облаком.

|  | Сообщения, отправляемые с устройства в облако | Сообщаемые свойства | Передача файлов |
| ---- | ------- | ---------- | ---- |
| Сценарий | Телеметрия и оповещения с учетом временных рядов, например пакеты данных датчика размером 256 КБ отправляются каждые 5 минут. | Доступные возможности и условия, например текущий режим подключения устройства (мобильная связь или Wi-Fi). Синхронизация длительных рабочих процессов, таких как конфигурация и обновления программного обеспечения. | Файлы мультимедиа. Большие пакеты телеметрии (обычно сжатые). |
| Хранение и извлечение | Временно хранятся в Центре Интернета вещей (до 7 дней). Только последовательное чтение. | Сохраняются в Центре Интернета вещей в двойнике устройства. Извлекаются с помощью [языка запросов Центра Интернета вещей][lnk-query]. | Хранятся в учетной записи хранения Azure, предоставленной пользователем. |
| Размер | Сообщения до 256 КБ. | Максимальный размер сообщаемого свойства — 8 КБ. | Максимальный размер файла, поддерживаемый хранилищем BLOB-объектов Azure. |
| Frequency | Высокий. Дополнительные сведения см. на странице [ограничений Центра Интернета вещей][lnk-quotas]. | Средний. Дополнительные сведения см. на странице [ограничений Центра Интернета вещей][lnk-quotas]. | Низкий. Дополнительные сведения см. на странице [ограничений Центра Интернета вещей][lnk-quotas]. |
| Протокол | Доступно при использовании всех протоколов. | В настоящее время доступно только при использовании MQTT. | Доступно при использовании любого протокола, но на устройстве требуется HTTP. |

> [!NOTE]
> Иногда приложению требуется отправка сведений (телеметрии временных рядов или оповещений), которые также должны быть доступны в двойнике устройства. В таких случаях приложение для устройства может отправить сообщение с устройства в облако и сообщить об изменении свойства либо серверная часть решения может сохранить данные в тегах двойника устройства при получении сообщения. Так как сообщения, отправляемые с устройства в облако, поддерживают более высокую пропускную способность, чем обновления двойника устройства, иногда желательно избегать обновления двойника устройства для каждого такого сообщения.

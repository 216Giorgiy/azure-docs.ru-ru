# Учебник: начало работы с центром IoT

## Введение

Центр IoT Azure — полностью управляемая служба, которая обеспечивает надежный и защищенный двунаправленный обмен данными между миллионами устройств IoT и серверной частью приложения. Одной из крупнейших задач, стоящих перед проектами IoT, является надежное и безопасное подключение устройств к серверной части приложения. Для упрощения этого сценария центр IoT Azure предлагает глобальный и надежный обмен сообщениями между устройством и облаком и в обратном направлении, обеспечивает безопасное подключение с использованием учетных данных и контроля доступа для каждого устройства и включает библиотеки устройств для наиболее популярных языков и платформ.

В этом учебнике показано, как создать центр IoT с помощью портала Azure. Кроме того, вы узнаете, как создать удостоверение устройства в центре IoT, а также как создать виртуальное устройство, которое отправляет сообщения из устройства в облако и получает эти сообщения от серверной части облачного приложения.

В конце этого учебника у вас будет три консольных приложения Windows:

* **CreateDeviceIdentity**, которое создает удостоверение устройства и соответствующий ключ безопасности для подключения к виртуальному устройству;
* **ReadDeviceToCloudMessages**, которое считывает сообщения, отправленные из устройства в облако, и отображает их содержимое;
* **SimulatedDevice**, которое подключается к центру IoT с идентификатором устройства, созданным ранее, и отправляет сообщения из устройства в облако с частотой один раз в секунду.

> [AZURE.NOTE]Для центра IoT существуют пакеты SDK для многих платформ устройств и языков (включая C, Java и Javascript). Эти пакеты работают на основе пакетов SDK для устройств IoT Azure. Пошаговые инструкции по подключению устройства к коду, приведенному в этом учебнике, и к центру IoT Azure в целом см. в [Центре разработчика IoT в Azure][]. Пакеты SDK службы IoT Azure для Java и Node будут выпущены в ближайшее время.

Чтобы пройти этот учебник, требуется:

+ Microsoft Visual Studio 2015;

+ Активная учетная запись Azure. <br/>Если ее нет, можно создать бесплатную пробную учетную запись всего за несколько минут. Подробные сведения см. на странице [Бесплатная пробная версия Azure](http://azure.microsoft.com/pricing/free-trial/?WT.mc_id=A0E0E5C02&amp;returnurl=http%3A%2F%2Fazure.microsoft.com%2Fru-RU%2Fdevelop%2Fiot%2Ftutorials%2Fgetstarted%2F target="\_blank").

## Создание центра IoT

1. Войдите на [портал Azure].

2. На панели переходов щелкните **Создать**, затем щелкните **Интернет вещей** и затем **Центр IoT**.

   	![][1]

3. В колонке **Новый центр IoT** задайте необходимые настройки центра IoT.

   	![][2]

    * В поле **Имя** введите имя центра IoT. После проверки **Имени** в поле **Имя** появится зеленая галочка.
    * Задайте необходимые настройки в разделе **Цены и уровень масштабирования**. Для этого руководства задавать определенный уровень не нужно.
    * В разделе **Группа ресурсов** создайте новую группу ресурсов или выберите существующую из списка. За дополнительной информацией обратитесь к разделу [Использование групп ресурсов для управления вашими ресурсами Azure](resource-group-portal.md).
    * В поле **Расположение** укажите географическое расположение, в котором будет размещен центр IoT.  


4. После задания настроек нового центра IoT щелкните **Создать**. Создание нового центра IoT может занять несколько минут. Для проверки состояния можно отслеживать прогресс на начальной панели. За ходом создания также можно наблюдать в разделе «Уведомления».

    ![][3]


5. После успешного создания центра IoT откройте колонку для нового центра IoT, запишите универсальный код ресурса (URI) и выберите значок **Ключ** вверху.

   	![][4]

6. Выберите политику общего доступа **iothubowner**, затем скопируйте и запишите строку подключения в колонке справа.

   	![][5]

Теперь ваш центр IoT создан и у вас есть универсальный код ресурса (URI) и строка подключения, необходимые для завершения работы с этим учебником.

## Запуск приложений

Теперь все готово к запуску приложений.

1.	В Visual Studio щелкните правой кнопкой мыши на вашем решении и выберите **Настроить запускаемые при включении проекты**. Выберите **Несколько запускаемых проектов**, а затем выберите действие **Запуск** для обоих приложений **ProcessDeviceToCloudMessages** и **SimulatedDevice**.

   	![][41]

2.	Нажмите клавишу **F5**, и вы должны увидеть, что оба приложения запущены и что сообщения, отправляемые виртуальным приложением, получаются обрабатывающим приложением.

   	![][42]

## Дальнейшие действия

В этом учебнике вы настроили новый центр IoT, создали удостоверение устройства в реестре идентификаторов центра и использовали это удостоверение для создания виртуального устройства, которое отправляет сообщения из устройства в облако. Изучение функций и сценариев центра IoT можно продолжить в следующих руководствах:

- [Отправка сообщений с облака на устройство в центре IoT][] — описывается, как отправлять сообщения на устройства и обрабатывать подтверждения о доставке от центра IoT.
- [Обработка сообщений, отправляемых с устройств в облако][] — описывается, как надежно обрабатывать данные телеметрии и интерактивные сообщения, поступающие от устройств.
- [Передача файлов с устройств][] — описывается модель использования сообщений, отправляемых из облака на устройства, для упрощения передачи файлов с устройств.

# Настройка и управление доступом к центру IoT

Эта статья содержит сведения, позволяющие ИТ-специалистам настроить среду, в которой устройства IoT связываются с центром IoT посредством сетевой инфраструктуры.

## Сетевое подключение

Устройства могут взаимодействовать с центром IoT в Azure с использованием различных протоколов. Как правило, выбор протокола определяется конкретными требованиями решения. В следующей таблице перечислены исходящие порты, которые должны быть открыты, чтобы устройство могло воспользоваться определенным протоколом.

| Протокол | Порты |
| -------- | ------- |
| HTTPS | 443 |
| AMQP | 5671 |
| AMQP через WebSocket | 443 |
| MQTT | 8883 |
| LWM2M (Управление устройствами) | 5684 |

После создания центра IoT в регионе Azure IP-адрес центра остается неизменным в течение всего срока службы центра. Однако для обеспечения качества обслуживания при перемещении корпорацией Майкрософт центра IoT в другую единицу масштабирования центру назначается новый IP-адрес.

## Центр IoT и безопасность

Взаимодействовать с центром IoT могут только устройства, зарегистрированные в центре IoT. Зарегистрированным устройствам должно быть предоставлено разрешение *DeviceConnect*. Устройство идентифицирует себя, включая в каждый создаваемый запрос маркер, содержащий уникальный идентификатор устройства. Центр проверяет действительность маркера и отсутствие устройства в черном списке (для устройств в черном списке разрешение *DeviceConnect* отменяется). Сведения о маркерах, поддерживаемых центром IoT, см. в статье [Использование маркеров безопасности центра IoT и сертификатов X.509][lnk-tokens].

Доступ к другим конечным точкам управления в центре IoT также контролируется с помощью набора разрешений: *iothubowner*, *service*, *registryRead* и *registryReadWrite*. Любое клиентское приложение управления, которое подключается к центру IoT, должно предоставить маркер с соответствующими разрешениями.

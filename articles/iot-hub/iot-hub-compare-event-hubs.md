# <a name="comparison-of-azure-iot-hub-and-azure-event-hubs"></a>Сравнение Центра Интернета вещей Azure и концентраторов событий Azure

Центр Интернета вещей и концентраторы событий Azure являются облачными службами, которые принимают большие объемы данных и обрабатывают или сохраняют эти данные для бизнес-аналитики. Эти две службы похожи тем, что обе поддерживают данные телеметрии и событие обработки с низкой задержкой и высокой надежностью. Однако только служба Центра Интернета вещей была разработана с определенными возможностями, необходимыми для поддержки широкомасштабных сценариев Интернета вещей. 

Центр Интернета вещей представляет собой облачный шлюз, соединяющий устройства и собирающий данные для автоматизации и бизнес-аналитики. Он упрощает потоковую передачу данных в облако и широкомасштабное управление устройствами. Важное отличие между Центром Интернета вещей и другими службами приема данных заключается в следующем. Центр Интернета вещей включает функции, которые расширяют связь устройств с серверными системами. Возможности двунаправленного обмена данными означают, что при получении данных с устройств вы также можете отправлять сообщения обратно на устройства для обновления свойств или вызова действия. Идентификатор на уровне устройства помогает защитить систему. С помощью распределенных вычислений логику облачной службы можно переместить на пограничные устройства.

[Концентраторы событий Azure][Azure Event Hubs] — это служба приема событий, которая может обрабатывать и сохранять большие объемы данных и телеметрии. Концентраторы событий предназначены для приема огромных объемов событий как в контексте обмена данными между разными центрами обработки данных, так и в пределах одного такого центра обработки, но эта служба не предоставляет расширенные возможности Центра Интернета вещей, обеспечиваемые непосредственно Центром Интернета вещей. По этой причине для решений Центра Интернета вещей мы не рекомендуем использовать концентраторы событий.

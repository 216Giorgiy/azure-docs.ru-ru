# <a name="understand-how-the-reserved-instance-discount-is-applied"></a>Сведения о применении скидки на зарезервированный экземпляр виртуальной машины
После покупки зарезервированного экземпляра виртуальной машины Azure скидка на зарезервированный экземпляр автоматически применяется к виртуальным машинам, атрибуты и количество которых совпадают с зарезервированным экземпляром. Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру виртуальных машин. В приведенной ниже таблице описаны затраты на виртуальную машину после приобретения зарезервированного экземпляра. Во всех случаях с вас взимается плата по обычным тарифам на хранение и сеть.

| Тип виртуальной машины  | Расходы на зарезервированный экземпляр |    
|-----------------------|--------------------------------------------|
|Виртуальные машины Linux без дополнительного программного обеспечения | Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру виртуальных машин.|
|Виртуальные машины Linux с платой за программное обеспечение (например, Red Hat) | Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение.|
|Виртуальные машины Windows без дополнительного программного обеспечения |Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение Windows.|
|Виртуальные машины Windows с дополнительным программным обеспечением (например, SQL Server) | Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру. Взимается плата за программное обеспечение Windows, а также за дополнительное программное обеспечение.|
|Виртуальные машины Windows с [преимуществом гибридного использования Azure](https://docs.microsoft.com/azure/virtual-machines/windows/hybrid-use-benefit-licensing) | Стоимость зарезервированного экземпляра покрывает затраты на инфраструктуру. Затраты на программное обеспечение Windows покрываются преимуществом гибридного использования Azure. Все дополнительное программное обеспечение оплачивается отдельно.| 

## <a name="application-of-reserved-instance-discount-to-non-windows-vms"></a>Применение скидки к зарезервированному экземпляру для виртуальных машин, отличных от Windows
 Скидка на зарезервированный экземпляр Azure применяется к работающим экземплярам виртуальных машин на почасовой основе. Для применения скидки приобретенные зарезервированные экземпляры должны соответствовать потреблению ресурсов работающими виртуальными машинами зарезервированного экземпляра. Для виртуальных машин, использующихся меньше часа, зарезервированный экземпляр будет заполнен из других виртуальных машин, которые его не используют, включая одновременно работающие виртуальные машины. По окончании часа приложение зарезервированного экземпляра для виртуальных машин блокируется. Если виртуальная машина работает неполный час или одновременно работающие виртуальные машины в течение часа не занимают весь час зарезервированного экземпляра, такой экземпляр считается недостаточно нагруженным за этот час. На приведенной ниже диаграмме показано применение зарезервированного экземпляра виртуальной машины с оплачиваемым использованием. На диаграмме представлен один оплаченный зарезервированный экземпляр и два соответствующих экземпляра виртуальных машин.

![Снимок экрана одного примененного зарезервированного экземпляра и двух соответствующих экземпляров виртуальной машины](media/billing-reserved-vm-instance-application/billing-reserved-vm-instance-application.png)

1.  Ресурсы, потребляемые свыше определенных для зарезервированного экземпляра, оплачиваются по мере использования. Если вы используете меньше ресурсов, чем предоставляется в рамках оплаченного зарезервированного экземпляра, дополнительная плата не взимается.
2.  В течение первого часа экземпляр 1 работает 0,75 часа, а экземпляр 2 работает 0,5 часа. Общее использование в течение 1 часа — 1,25 часа. За оставшиеся 0,25 часа с вас взимается оплата по мере использования.
3.  В течение второго и третьего часов оба экземпляра работали 1 час. Работа одного экземпляра входит в стоимость зарезервированного экземпляра, а за другой взимается плата по мере использования.
4.  В течение четвертого часа экземпляр 1 работает 0,5 часа, а экземпляр 2 работает 1 час. Использование экземпляра 1 полностью покрывается зарезервированным экземпляром. 0,5 часа работы экземпляра 2 также покрывается. За оставшиеся 0,5 часа с вас взимается оплата по мере использования.

Чтобы больше узнать о применении зарезервированных экземпляров и просмотреть отчеты о начислении платы за использование, см. статью [Общие сведения об использовании зарезервированных экземпляров Azure с соглашением о регистрации Enterprise](https://go.microsoft.com/fwlink/?linkid=862757).

## <a name="application-of-reserved-instance-discount-to-windows-vms"></a>Применение скидки к зарезервированному экземпляру для виртуальных машин Windows
Если используются экземпляры виртуальных машин Windows, зарезервированный экземпляр покрывает затраты на инфраструктуру. Зарезервированный экземпляр применяется к затратам на инфраструктуру виртуальных машин Windows так же, как и для виртуальных машин с другими ОС. За программное обеспечение Windows плата взимается отдельно для каждого виртуального процессора. См. статью о [затратах на программное обеспечение Windows с зарезервированными экземплярами](https://go.microsoft.com/fwlink/?linkid=862756). Вы можете покрыть расходы на лицензирование Windows, используя [Преимущество гибридного использования Azure для Windows Server] (https://docs.microsoft.com/azure/virtual-machines/windows/hybrid-use-benefit-licensing).

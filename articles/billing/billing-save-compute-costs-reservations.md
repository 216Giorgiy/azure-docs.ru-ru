# <a name="what-are-azure-reserved-vm-instances"></a>Основные сведения о зарезервированных экземплярах виртуальных машин Azure
[Зарезервированные виртуальные экземпляры виртуальной машины Azure](https://azure.microsoft.com/pricing/reserved-vm-instances) помогут сэкономить деньги путем предоплаты за один или три года использования вычислительной мощности, что позволяет получить скидку на использование виртуальных машин. Однолетние и трехлетние контракты на зарезервированные экземпляры Azure с предварительной оплатой позволят сэкономить существенную сумму (до 72 %) по сравнению с оплатой виртуальных машин по мере использования. Зарезервированные экземпляры предоставляют скидку при выставлении счетов и не влияют на состояние выполнения ваших виртуальных машин.

Зарезервированные экземпляры можно приобрести на [портале Azure](https://aka.ms/reservations). Дополнительные сведения см. в статье [Предоплата виртуальных машин с помощью Azure Reserved Virtual Machine Instances](https://go.microsoft.com/fwlink/?linkid=861721).

## <a name="why-should-i-buy-a-reserved-instance"></a>Зачем мне покупать зарезервированный экземпляр?
Если у вас есть виртуальные машины, которые работают длительное время, приобретение зарезервированного экземпляра дает наибольшую экономию. Например, при постоянном функционировании четырех экземпляров виртуальной машины D2 (цен. категория "Стандартный") в регионе "Западная часть США" без зарезервированных экземпляров оплата взимается по мере использования. Если вы приобретете зарезервированный экземпляр для каждой из этих четырех виртуальных машин, вы немедленно получите скидку на них при выставлении счетов. Теперь к ним не будут применяться ставки оплаты по мере использования. 

## <a name="what-charges-does-a-reserved-instance-cover"></a>Какие расходы покрывает зарезервированный экземпляр?
Зарезервированный экземпляр относится только к оплате инфраструктуры виртуальной машины Windows или Linux. Зарезервированный экземпляр не включает оплату дополнительного программного обеспечения, сетей и хранилищ. Если вы используете виртуальные машины Windows, расходы на лицензирование вам поможет сократить [Преимущество гибридного использования Azure](https://azure.microsoft.com/pricing/hybrid-benefit/).

## <a name="whos-eligible-to-purchase-a-reserved-instance"></a>Кто может приобрести зарезервированный экземпляр?
Зарезервированный экземпляр доступен для пользователей Azure со следующими типами подписок:
-   тип предложения подписки "Enterprise Agreement" (MS-AZR-0017P);
-   тип предложения подписки [с оплатой по мере использования](https://azure.microsoft.com/offers/ms-azr-0003p/) (MS-AZR-003P). Для приобретения зарезервированного экземпляра вам должна быть назначена роль владельца подписки. Для приобретения зарезервированных экземпляров в рамках корпоративной регистрации необходимо, чтобы администратор этой регистрации включил покупку зарезервированных экземпляров на портале EA. Эта функция включена по умолчанию.
-   Партнеры-поставщики облачных решений (CSP) могут использовать портал Azure или [партнерский центр](https://docs.microsoft.com/partner-center/azure-reservations) для приобретения зарезервированных экземпляров.

## <a name="how-is-a-reserved-instance-purchase-billed"></a>Как оплачивается покупка зарезервированного экземпляра?
Метод оплаты зарезервированного экземпляра зависит от используемой подписки. Если вы используете подписку Enterprise Agreement, стоимость зарезервированного экземпляра вычитается из баланса денежных обязательств. Если у вас нет достаточного баланса денежных обязательств для покрытия полной стоимости зарезервированного экземпляра, вам выставляется счет на соответствующую разницу.
Если вы используете подписку с оплатой по мере использования, нужная сумма немедленно снимается с кредитной карты, которая подключена к учетной записи. Если вы используете оплату по накладным, эта сумма будет включена в ближайшую накладную.

## <a name="how-is-the-purchased-reserved-instance-discount-applied"></a>Как применяется скидка зарезервированного экземпляра?
Скидка зарезервированного экземпляра применяется к виртуальным машинам, которые соответствуют атрибутам, выбранным при покупке зарезервированного экземпляра. Атрибуты включают в себя область, в которой выполняются виртуальные машины. Например, если вы хотите использовать зарезервированный экземпляр для получения скидки на четыре виртуальные машины D2 (цен. категория "Стандартный") в регионе "западная часть США", выберите ту подписку, в которой работают эти виртуальные машины. Если виртуальные машины работают в разных подписках в одной учетной записи, выберите вариант совместного использования. Совместное использование позволяет применить скидки зарезервированного экземпляра сразу в нескольких подписках. Область зарезервированного экземпляра можно изменить и после покупки. Соответствующая процедура описана в [документации по управлению зарезервированными экземплярами в Azure](billing-manage-reserved-vm-instance.md).

Скидка зарезервированного экземпляра применяется только для виртуальных машин в подписках Enterprise Agreement или с оплатой по мере использования. Для виртуальных машин, работающих в подписке с любым другим типом, невозможно воспользоваться скидкой зарезервированного экземпляра. Из числа корпоративных регистраций преимущества зарезервированного экземпляра недоступны для подписок "Enterprise — разработка и тестирование".

Чтобы лучше понять, как зарезервированный экземпляр влияет на выставление счетов за использование виртуальных машин, изучите статью [Сведения о применении скидки на зарезервированный экземпляр виртуальной машины](billing-understand-vm-reservation-charges.md).

## <a name="what-happens-when-the-reserved-instance-term-expires"></a>Что происходит, когда истекает срок действия зарезервированного экземпляра?
При завершении срока зарезервированного экземпляра скидка на оплату отменяется и для инфраструктуры виртуальной машины снова применяется оплата по мере использования. Зарезервированные экземпляры не возобновляются автоматически. Чтобы сохранить скидку на оплату, необходимо приобрести новый зарезервированный экземпляр. 

## <a name="sizes-and-regional-availability"></a>Доступность для размеров и регионов
Зарезервированные экземпляры доступны почти для всех размеров виртуальных машин за некоторыми исключениями.
- Виртуальные машины в предварительной версии. Любая серия виртуальных машин или размеров, которая находится в предварительной версии, недоступна для приобретения зарезервированного экземпляра.
- Облако. Зарезервированные экземпляры недоступны для приобретения в этих регионах: государственные организации США, Германия и Китай. 
- Недостаточная квота. Если зарезервированный экземпляр привязан к одной конкретной подписке, эта подписка должна иметь свободную квоту виртуальных процессоров для размещения этого зарезервированного экземпляра. Например, если целевая подписка имеет квоту на 10 виртуальных процессоров для серии D, вы не сможете приобрести зарезервированный экземпляр для 11 экземпляров Standard_D1. При проверке квоты для зарезервированных экземпляров учитываются все виртуальные машины, уже размещенные в подписке. Например, если подписка имеет квоту на 10 виртуальных процессоров для серии D и два развернутых экземпляра Standard_D1, в этой подписке можно приобрести зарезервированный экземпляр для 10 экземпляров Standard_D1. 
- Ограничения емкости. В редких случаях Azure может ограничить приобретение новых зарезервированных экземпляров для некоторых размеров виртуальных машин из-за нехватки емкости в соответствующем регионе.

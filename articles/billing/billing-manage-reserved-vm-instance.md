# <a name="manage-reserved-instances"></a>Управление службой Azure Reserved VM Instances

После покупки зарезервированного экземпляра виртуальной машины Azure может потребоваться применить зарезервированный экземпляр к другой подписке, отличной от указанной во время покупки. Кроме того, если соответствующие виртуальные машины работают в нескольких подписках, можно изменить область зарезервированного экземпляра на общую. Чтобы добиться максимальной скидки на зарезервированный экземпляр, убедитесь, что количество экземпляров, которые вы приобрели, соответствует атрибутам и количеству виртуальных машин, которые у вас работают. Чтобы узнать больше о зарезервированных экземплярах Azure, перейдите на страницу [Зарезервированные экземпляры виртуальных машин Azure](https://go.microsoft.com/fwlink/?linkid=862121).

## <a name="change-the-scope-for-a-reserved-instance"></a>Изменение области для зарезервированного экземпляра
 Скидка на зарезервированные экземпляры применяется к виртуальным машинам, которые соответствуют вашему зарезервированному экземпляру и попадают в его область. Областью зарезервированного экземпляра может быть одна или все подписки в контексте выставления счетов. Если указать одну подписку, зарезервированный экземпляр сопоставляется с работающими в ней виртуальными машинами. Если указать общую область, то Azure сопоставляет зарезервированный экземпляр с виртуальными машинами, работающими во всех подписках в контексте выставления счетов. Контекст выставления счетов зависит от подписки, использованной для покупки зарезервированного экземпляра. Чтобы узнать больше, ознакомьтесь с разделом [Предоплата за виртуальные машины с помощью зарезервированных экземпляров виртуальных машин](https://go.microsoft.com/fwlink/?linkid=861721).

Вот как можно обновить область зарезервированного экземпляра: 
1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Резервирования**.
3. Выберите зарезервированный экземпляр.
4. Выберите **Параметры** > **Конфигурация**.
5. Изменение область. Если общую область изменить на отдельную область, то можно будет выбрать только подписку, владельцем которой вы являетесь. Можно выбрать только подписки в том же контексте выставления счетов, что и зарезервированный экземпляр. Контекст выставления счетов определяется подпиской, выбранной при покупке зарезервированного экземпляра. Область применяется только подпискам с оплатой по мере использования MS-AZR-0003P и подпискам с соглашением о регистрации Enterprise MS-AZR-0017P. Для подписок с соглашением о регистрации Enterprise и подписок для разработки и тестирования скидка на зарезервированный экземпляр недоступна.

## <a name="split-a-single-reserved-instance-into-two-reserved-instances"></a>Разделение одного зарезервированного экземпляра на два
 После покупки нескольких экземпляров может потребоваться назначить экземпляры в зарезервированном экземпляре разным подпискам. По умолчанию все экземпляры (количество, указанное во время покупки) имеют одну область. Это либо одна подписка, либо общая область. Например, вы купили 10 виртуальных машин уровня "Стандартный D2" и указали в качестве области подписку А. Теперь вам нужно изменить область для семи зарезервированных экземпляров виртуальной машины на подписку A и оставшихся 3 — на подписку Б. Разделение зарезервированного экземпляра позволяет распределить зарезервированные экземпляры, чтобы более детально управлять областью. Можно упростить выделение для подписок, выбрав общую область. Но с целью управления затратами или бюджетирования можно распределить количество между определенными подписками.

 Зарезервированный экземпляр можно разделить на два с помощью PowerShell, интерфейса командной строки или API.

### <a name="split-a-reserved-instance-by-using-powershell"></a>Разделение зарезервированного экземпляра с помощью PowerShell
1. Получите идентификатор зарезервированного экземпляра, выполнив следующую команду:

    ```powershell
    # Get the Reserved Instance orders you have access to
    Get-AzureRmReservationOrder
    ```
2. Получите сведения о зарезервированном экземпляре:

    ```powershell
    Get-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a
    ```
3. Разделите зарезервированный экземпляр на две части и распределите экземпляры:

    ```powershell
    # Split the Reserved Instance. The sum of the Reserved Instances, the quantity, must equal the total number of instances in the Reserved Instance that you're splitting.
    Split-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId b8be062a-fb0a-46c1-808a-5a844714965a -Quantity 3,2
    ```
1. Чтобы изменить область, выполните следующую команду.

    ```powershell
    Update-AzureRmReservation -ReservationOrderId a08160d4-ce6b-4295-bf52-b90a5d4c96a0 -ReservationId 5257501b-d3e8-449d-a1ab-4879b1863aca -AppliedScopeType Single -AppliedScope /subscriptions/15bb3be0-76d5-491c-8078-61fe3468d414
    ```

## <a name="add-or-change-users-who-can-manage-a-reserved-instance"></a>Добавление или изменение пользователей, которые могут управлять зарезервированным экземпляром
Вы можете делегировать управление зарезервированным экземпляром, добавив пользователей в роли для зарезервированного экземпляра. По умолчанию пользователь, который купил зарезервированный экземпляр, и учетная запись администратора участвуют в роли владельца зарезервированного экземпляра. 

Доступом к зарезервированному экземпляру можно управлять независимо от подписок, получающих скидку на зарезервированный экземпляр. Когда вы предоставляете какому-либо пользователю разрешение на управление зарезервированным экземпляром, оно не дает ему права для управления подпиской. И если вы предоставляете какому-либо пользователю разрешение на управление подпиской, указанной в области зарезервированного экземпляра, оно не дает ему права для управления зарезервированным экземпляром.
 
Чтобы делегировать управление доступом к зарезервированному экземпляру, сделайте следующее: 
1.  Войдите на [портал Azure](https://portal.azure.com).
2.  Выберите **Все службы** > **Резервирование**, чтобы отобразить список доступных вам зарезервированных экземпляров.
3.  Выберите зарезервированный экземпляр, доступ к которому требуется делегировать другим пользователям.
4.  В меню выберите **Управление доступом (IAM)**.
5.  Выберите **Добавить** > **Роль** > **Владелец** (или другую роль, если вы хотите предоставить ограниченный доступ). 
6. Укажите адрес электронной почты пользователя, которого требуется добавить в роль "Владелец". 
7. Выберите пользователя, а затем щелкните **Сохранить**.

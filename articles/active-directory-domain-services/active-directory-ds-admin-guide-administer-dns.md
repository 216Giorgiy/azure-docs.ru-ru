# Администрирование DNS в управляемых доменах доменных служб Azure AD
Доменные службы Azure Active Directory включают DNS-сервер, который обеспечивает разрешение доменных имен (DNS) для управляемого домена. Иногда может потребоваться настроить DNS на управляемом домене, чтобы получить записи для компьютеров, не присоединенных к домену, виртуальных IP-адресов для подсистем балансировки нагрузки или внешних DNS-серверов пересылки. По этой причине пользователи, принадлежащие к группе "Администраторы контроллера домена AAD", получают права на администрирование DNS в управляемом домене.

## Подготовка присоединенной к домену виртуальной машины для удаленного администрирования DNS для управляемого домена
Управляемыми доменами доменных служб Azure AD можно управлять с помощью привычных средств администрирования Active Directory, таких как центр администрирования Active Directory (ADAC) или AD PowerShell. Аналогичным образом DNS для управляемого домена можно управлять удаленно с помощью средств администрирования DNS-сервера. Администраторы клиентов не имеют права подключаться к контроллерам доменов в управляемом домене с помощью удаленного рабочего стола. Таким образом, участники группы "Администраторы контроллера домена AAD" могут администрировать DNS для управляемых доменов удаленно с помощью средств администрирования DNS-сервера с сервера Windows Server или клиентского компьютера, присоединенного к управляемому домену. Средства DNS-сервера можно установить в составе необязательного компонента "Средства удаленного администрирования сервера" (RSAT) на серверы Windows Server и клиентские компьютеры, присоединенные к управляемому домену.

Первым шагом является настройка виртуальной машины Windows Server, которая присоединена к управляемому домену. Инструкции см. в статье о [присоединении виртуальной машины Windows Server к управляемому домену доменных служб Azure AD](active-directory-ds-admin-guide-join-windows-vm.md). Обратите внимание, что эти инструкции подразумевают использование виртуальной машины Windows Server для администрирования управляемого домена AAD DS. Для этого также можно выбрать клиентскую виртуальную машину Windows (например, Windows 10). В этом случае дополнительный компонент "Средства удаленного администрирования сервера" можно установить на виртуальной машине.


## Установка средств DNS-сервера на виртуальной машине
Чтобы установить средства администрирования DNS на присоединенной к домену виртуальной машине, выполните следующие действия. Для получения [дополнительных сведений об установке и использовании средств удаленного администрирования сервера](https://technet.microsoft.com/library/hh831501.aspx) см. сайт TechNet.

1. На классическом портале Azure перейдите к узлу **Виртуальные машины**. Выберите только что созданную виртуальную машину и нажмите кнопку **Подключиться** на панели команд в нижней части окна.

    ![Подключение к виртуальной машине Windows](./media/active-directory-domain-services-admin-guide/connect-windows-vm.png)

2. Портал предложит вам открыть или сохранить RDP-файл, который используется для подключения к виртуальной машине. Скачайте RDP-файл, а затем щелкните его.

3. При запросе имени пользователя и пароля укажите учетные данные пользователя, принадлежащего к группе "Администраторы контроллера домена AAD". Например, в нашем случае bob@domainservicespreview.onmicrosoft.com.

4. На начальном экране откройте **диспетчер серверов**. На центральной панели окна диспетчера серверов выберите команду **Добавить роли и компоненты**.

    ![Запуск диспетчера серверов на виртуальной машине](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager.png)

5. На странице **Перед началом работы** **мастера добавления ролей и компонентов** нажмите кнопку **Далее**.

    ![Страница "Перед началом работы"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-begin.png)

6. На странице **Тип установки** оставьте флажок **Установка ролей или компонентов** и нажмите кнопку **Далее**.

	![Страница "Тип установки"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-type.png)

7. На странице **Выбор сервера** выберите текущую виртуальную машину из пула серверов и нажмите кнопку **Далее**.

	![Страница "Выбор сервера"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-server.png)

8. На странице **Роли сервера** нажмите кнопку **Далее**. Мы пропустим эту страницу, так как не устанавливаем никакие роли на сервере.

9. На странице **Компоненты** разверните узел **Средства удаленного администрирования сервера**, а затем узел **Средства администрирования ролей**. Из списка средств администрирования ролей выберите компонент **Средства DNS-сервера**, как показано ниже.

	![Страница "Компоненты"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-dns-tools.png)

10. На странице **Подтверждение** выберите команду **Установить**, чтобы установить компонент средств DNS-сервера на виртуальной машине. По завершении установки компонента нажмите **Закрыть**, чтобы выйти из мастера **добавления ролей и компонентов**.

	![Страница подтверждения](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-dns-confirmation.png)


## Запуск консоли управления DNS
После установки компонента средств DNS-сервера на присоединенную к домену виртуальную машину с помощью этих средств можно администрировать DNS в управляемом домене.

1. На начальном экране щелкните **Администрирование**. Должна открыться консоль **DNS**, установленная на виртуальной машине.

	![Администрирование — консоль DNS](./media/active-directory-domain-services-admin-guide/install-rsat-dns-tools-installed.png)

2. Щелкните элемент **DNS**, чтобы запустить консоль управления DNS.

3. В диалоговом окне **Подключение к DNS-серверу** щелкните параметр **Следующий компьютер** и введите доменное имя DNS управляемого домена (например, contoso100.com).

    ![Консоль DNS — подключение к домену](./media/active-directory-domain-services-admin-guide/dns-console-connect-to-domain.png)

4. Консоль DNS подключится к управляемому домену. Должно отобразиться представление, аналогичное показанному ниже.

    ![Консоль DNS — администрирование домена](./media/active-directory-domain-services-admin-guide/dns-console-managed-domain.png)

5. Теперь можно использовать консоль DNS для добавления записей DNS для компьютеров в виртуальной сети, в которой вы активировали доменные службы AAD.

> [AZURE.WARNING] Будьте очень внимательны при администрировании DNS для управляемого домена с помощью средств администрирования DNS. Следите за тем, чтобы **не удалить и не изменить встроенные записи DNS, которые используются в домене службами домена**. Сюда входят записи DNS домена, записи сервера имен и другие записи, используемые для обнаружения контроллера домена. Если изменить эти записи, работа доменных служб в виртуальной сети будет нарушена.


Дополнительные сведения об управлении DNS см. в [статье о средствах DNS на сайте Technet](https://technet.microsoft.com/library/cc753579.aspx).

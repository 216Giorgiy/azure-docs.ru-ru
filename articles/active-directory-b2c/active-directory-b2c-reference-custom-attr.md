#  Предварительная версия Azure Active Directory B2C: использование настраиваемых атрибутов для сбора данных о потребителях

[AZURE.INCLUDE [active-directory-b2c-preview-note](../../includes/active-directory-b2c-preview-note.md)]

Каталог Azure Active Directory (AD) B2C поставляется с встроенным набором атрибутов: Given Name, Surname, City, Postal Code и т. д. Однако у каждого потребительского приложения особые требования к информации (атрибутам), которую им требуется собирать у потребителей. Azure AD B2C позволяет расширить каталог (в частности, расширить набор атрибутов, хранящихся в каждой учетной записи потребителя). На [портале Azure](https://portal.azure.com/) можно создать настраиваемые атрибуты и использовать их в политиках регистрации, как показано ниже. Можно также читать и записывать эти атрибуты с помощью API Graph Azure AD, как показано [здесь](active-directory-b2c-devquickstarts-graph-dotnet.md).

> [AZURE.NOTE]Настраиваемые атрибуты скрыто используют [расширения схемы каталога API Graph Azure AD](https://msdn.microsoft.com/library/azure/dn720459.aspx).

## Как создать настраиваемый атрибут

1. [Выполните следующие действия, чтобы перейти к колонке функций B2C на портале Azure](active-directory-b2c-app-registration.md#navigate-to-the-b2c-features-blade).
2. Щелкните **Атрибуты пользователя**.
3. Щелкните **+Добавить** в верхней части колонки.
4. Введите **имя** настраиваемого атрибута (например, ShoeSize) и, при необходимости, его **описание**. Щелкните **Создать**. Готово!

    > [AZURE.NOTE]Сейчас доступен только **тип данных** String. В дальнейшем будут добавлены дополнительные типы данных (DateTime, Integer и т. д.).

Теперь настраиваемый атрибут доступен в списке **Атрибуты пользователя** и может использоваться в ваших политиках регистрации.

## Как использовать настраиваемый атрибут в политике регистрации

1. [Выполните следующие действия, чтобы перейти к колонке функций B2C на портале Azure](active-directory-b2c-app-registration.md#navigate-to-the-b2c-features-blade).
2. Щелкните **Политики регистрации**.
3. Откройте политику регистрации (например, B2C\_1\_SiUp), щелкнув ее. Щелкните **Изменить** в верхней части колонки.
4. Щелкните **Атрибуты регистрации** и выберите настраиваемый атрибут (например, ShoeSize). Нажмите кнопку **ОК**.
5. Щелкните **Утверждения приложения** и выберите настраиваемый атрибут. Нажмите кнопку **ОК**. 
6. Щелкните **Сохранить** в верхней части колонки. Готово!

Чтобы проверить взаимодействие с потребителем, можно использовать функцию «Выполнить» в политике. Теперь вы увидите ShoeSize в списке атрибутов, собираемых во время регистрации потребителя, а также в токене, возвращаемом в приложение.

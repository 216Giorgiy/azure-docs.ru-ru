# Предварительная версия Azure Active Directory (AD) B2C: организация регистрации и входа для потребителей с учетными записями Microsoft

[AZURE.INCLUDE [active-directory-b2c-preview-note](../../includes/active-directory-b2c-preview-note.md)]

## Создание приложения для учетной записи Майкрософт

Чтобы использовать учетную запись Майкрософт в качестве поставщика удостоверений в Azure Active Directory (Azure AD) B2C, необходимо сначала создать приложение для учетной записи Майкрософт и задать в нем правильные параметры. Для этого требуется учетная запись Майкрософт. Если у вас ее нет, вы можете получить такую учетную запись на сайте [https://www.live.com/](https://www.live.com/).

1. Перейдите в [Центр разработчиков для учетной записи Microsoft](https://account.live.com/developers/applications) и войдите с помощью своей учетной записи Майкрософт.
2. Щелкните **Создать приложение**.

    ![Учетная запись Майкрософт: добавление нового приложения](./media/active-directory-b2c-setup-msa-app/msa-add-new-app.png)

3. Укажите **Имя приложения** и щелкните **Я принимаю условия использования**. Для создания приложения вам необходимо принять условия использования служб Майкрософт.

    ![Учетная запись Майкрософт: имя приложения](./media/active-directory-b2c-setup-msa-app/msa-app-name.png)

4. В левой области навигации щелкните **Параметры API**. Введите действительный **контактный электронный адрес**.

    ![Учетная запись Майкрософт: параметры API](./media/active-directory-b2c-setup-msa-app/msa-api-settings.png)

5. Введите `https://login.microsoftonline.com/te/{tenant}/oauth2/authresp` в поле **URL-адреса перенаправления**. Замените **{tenant}** именем своего клиента (например, contosob2c.onmicrosoft.com). В нижней части страницы нажмите кнопку **Сохранить**.

    ![Учетная запись Майкрософт: URL-адрес перенаправления](./media/active-directory-b2c-setup-msa-app/msa-redirect-url.png)

6. В левой области навигации щелкните **Параметры приложений**. Скопируйте значения **Client ID** (Идентификатор клиента) и **Client secret** (Секрет клиента). Оба значения необходимы для настройки учетной записи Майкрософт в качестве поставщика удостоверений в вашем клиенте. **Секрет клиента** — это важные учетные данные безопасности.

    ![Учетная запись Майкрософт: секрет клиента](./media/active-directory-b2c-setup-msa-app/msa-client-secret.png)

## Настройка учетной записи Майкрософт в качестве поставщика удостоверений в клиенте

1. Выполните следующие действия, чтобы [перейти к колонке функций B2C](active-directory-b2c-app-registration.md#navigate-to-the-b2c-features-blade) на портале Azure.
2. В колонке функций B2C щелкните **Поставщики удостоверений**.
3. Нажмите **+Добавить** в верхней части колонки.
4. Укажите понятное **имя** конфигурации поставщика удостоверений. Например, введите "MSA".
5. Щелкните **Тип поставщика удостоверений**, выберите **Учетная запись Майкрософт** и нажмите кнопку **ОК**.
6. Щелкните **Настроить этот поставщик удостоверений** и введите идентификатор клиента и секрет клиента ранее созданного приложения учетной записи Майкрософт.
7. Нажмите кнопку **ОК**, а затем **Создать**, чтобы сохранить конфигурацию учетной записи Майкрософт.

# <a name="register-an-application-in-azure-active-directory-b2c"></a>Зарегистрировать приложение в Azure Active Directory B2C

Чтобы создать [приложение](active-directory-b2c-apps.md) с поддержкой регистрации и входа пользователей, сначала нужно зарегистрировать его с помощью клиента Azure Active Directory B2C. В этой статье вы узнаете, как зарегистрировать приложение в клиенте Microsoft Azure Active Directory (Azure AD) B2C за несколько минут. Ознакомившись руководством, вы зарегистрируете приложение для использования в клиенте Azure AD B2C.

## <a name="prerequisites"></a>Предварительные требования

Получите собственного клиента, следуя указаниям в статье [Руководство по созданию клиента Azure Active Directory B2C](tutorial-create-tenant.md).

Выберите следующие действия в зависимости от типа приложения:

- [регистрация веб-приложения;](#register-a-web-application)
- [регистрация веб-API;](#register-a-web-api)
- [регистрация мобильного или собственного приложения.](#register-a-mobile-or-native-application)

## <a name="register-a-web-application"></a>Регистрация веб-приложения

1. Убедитесь, что используете каталог, содержащий клиент Azure AD B2C, щелкнув **Фильтр каталога и подписки** в верхнем меню и выбрав каталог, содержащий ваш клиент.
2. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Azure AD B2C**.
3. Щелкните **Приложения**, а затем выберите **Добавить**.
4. Введите имя приложения. Например, *testapp1*.
5. Для поля **Включить веб-приложение или веб-API** и **Разрешить неявный поток** выберите **Да**.
6. Для **URL-адреса ответа** введите конечные точки, куда Azure AD B2C возвращает все маркеры, запрашиваемые вашим приложением. Например, можно задать его для локального ожидания передачи данных в `https://localhost:44316`. Если вы еще не знаете номер порта, можно ввести значение заполнителя и изменить его позже.
7. Нажмите кнопку **Создать**.

### <a name="create-a-client-secret"></a>Создание секрета клиента

Необходимо создать секрет приложения, если приложение вызывает веб-API, защищенный с помощью Azure AD B2C.

1. Выберите **Ключи** и щелкните **Создать ключ**. 
2. Чтобы представить ключ, нажмите **Сохранить**. Запишите значение параметра **Ключ приложения**. Это значение используется в качестве секрета приложения в коде приложения.
3. Щелкните **Доступ через API**, **Добавить** и выберите свой веб-API и области (разрешения).

## <a name="register-a-web-api"></a>Регистрация веб-API

1. Убедитесь, что используете каталог, содержащий клиент Azure AD B2C, щелкнув **Фильтр каталога и подписки** в верхнем меню и выбрав каталог, содержащий ваш клиент.
2. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Azure AD B2C**.
3. Щелкните **Приложения**, а затем выберите **Добавить**.
4. Введите имя приложения. Например, *testapp2*.
5. Для поля **Включить веб-приложение или веб-API** и **Разрешить неявный поток** выберите **Да**.
6. Для **URL-адреса ответа** введите конечные точки, куда Azure AD B2C возвращает все маркеры, запрашиваемые вашим приложением. Например, можно задать его для локального ожидания передачи данных в `https://localhost:44316`. Если вы еще не знаете номер порта, можно ввести значение заполнителя и изменить его позже.
7. Для поля **URI идентификатора приложения** выберите идентификатор, используемый для веб-API. Полный URI код с доменом создается автоматически. Например, `https://contosotenant.onmicrosoft.com/api`.
8. Нажмите кнопку **Создать**.
9. При необходимости добавить дополнительные области щелкните **Опубликованные области**. Область `user_impersonation` определяется по умолчанию. Область `user_impersonation` предоставляет доступ другим приложениям к этому API от имени пользователя, выполнившего вход. При желании можно удалить область `user_impersonation`.

## <a name="register-a-mobile-or-native-application"></a>Регистрация мобильного или собственного приложения

1. Убедитесь, что используете каталог, содержащий клиент Azure AD B2C, щелкнув **Фильтр каталога и подписки** в верхнем меню и выбрав каталог, содержащий ваш клиент.
2. Выберите **Все службы** в левом верхнем углу окна портала Azure, а затем найдите и выберите **Azure AD B2C**.
3. Щелкните **Приложения**, а затем выберите **Добавить**.
4. Введите имя приложения. Например, *testapp3*.
5. Для поля **Включить веб-приложение или веб-интерфейс API** выберите **Нет**.
6. Для поля **Включить собственный клиент** выберите **Да**.
7. Для **URI перенаправления** введите [URI перенаправления с настраиваемой схемой](active-directory-b2c-apps.md). Убедитесь, что выбран правильный URI перенаправления, который не содержит специальных знаков, например, символов подчеркивания.
8. Нажмите кнопку **Создать**.

### <a name="create-a-client-secret"></a>Создание секрета клиента

Необходимо создать секрет приложения, если приложение вызывает веб-API, защищенный с помощью Azure AD B2C.

1. Выберите **Ключи** и щелкните **Создать ключ**. 
2. Чтобы представить ключ, нажмите **Сохранить**. Запишите значение параметра **Ключ приложения**. Это значение используется в качестве секрета приложения в коде приложения.
3. Щелкните **Доступ через API**, **Добавить** и выберите свой веб-API и области (разрешения).

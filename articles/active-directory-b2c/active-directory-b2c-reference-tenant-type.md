# Azure Active Directory B2C: сравнение предварительной и рабочей версии клиента B2C
Если вы планируете написать рабочее приложение в Azure Active Directory (Azure AD) B2C, вам понадобится клиент соответствующего типа. Сведения о **типе клиента** отображаются в колонке функций B2C на портале Azure. Чтобы [перейти к ней](active-directory-b2c-app-registration.md#navigate-to-the-b2c-features-blade), выполните следующие действия.

## Сводка
Azure AD B2C поддерживает рабочие приложения только в **рабочих** версиях клиентов B2C, расположенных в Северной Америке.

| Тип клиента | Страны и регионы | Общедоступность |
| --- | --- | --- |
| **Рабочая версия клиента** |Страны и регионы Северной Америки |Да |
| **Рабочая версия клиента** |Страны и регионы всех континентов, за исключением Северной Америки |Нет |
| **Предварительная версия клиента** |Все страны и регионы |Нет |

> [!NOTE]
> Клиенты Azure AD B2C (пользовательские) недоступны в некоторых странах и регионах, в которых доступны клиенты Azure AD (для сотрудников). Дополнительные сведения см. в следующих разделах.
> 
> 

## Рабочая версия клиента B2C в Северной Америке
Если в одном из регионов или стран Северной Америки вы создали [клиент B2C](active-directory-b2c-get-started.md) (например, в США, Канаде, Коста-Рике, Доминиканской Республике, Эль-Сальвадоре, Гватемале, Мексике, Панаме, Пуэрто-Рико, Тринидаде и Тобаго) и в пользовательском интерфейсе администратора B2C для параметра **Tenant type** (Тип клиента) задано значение **Production-scale** (Рабочий), этот клиент можно использовать для рабочих приложений.

> [!NOTE]
> Клиент рабочей версии можно расширить до сотен миллионов идентификаторов пользователей.
> 
> 

![Снимок экрана: рабочая версия клиента](./media/active-directory-b2c-reference-tenant-type/production-scale-b2c-tenant.png)

## Предварительная версия клиента B2C в любой стране и регионе
Если клиент B2C создан в течение периода действия предварительной версии Azure AD B2C, для параметра **Tenant type** (Тип клиента) задано значение **Preview tenant** (Предварительная версия клиента). В этом случае клиент НЕОБХОДИМО использовать для разработки и тестирования, а НЕ для рабочих приложений.

> [!IMPORTANT]
> Способа перехода с предварительной версии клиента B2C на рабочую не существует. Обратите внимание, что при удалении существующего клиента B2C предварительной версии и повторном создании клиента B2C рабочей версии с тем же доменным именем могут возникнуть известные проблемы. Создавайте клиент B2C рабочей версии с другим доменным именем.
> 
> 

![Снимок экрана: предварительная версия клиента](./media/active-directory-b2c-reference-tenant-type/preview-b2c-tenant.png)

## Рабочая версия клиента B2C за пределами Северной Америки
В настоящее время общедоступная версия клиента Azure AD B2C недоступна за пределами Северной Америке. Однако для разработки и тестирования можно создать и использовать рабочие версии клиентов в одной из следующих стран и регионов: Алжир, Австрия, Азербайджан, Бахрейн, Беларусь, Бельгия, Болгария, Хорватия, Кипр, Чешская Республика, Дания, Египет, Эстония, Финляндия, Франция, Германия, Греция, Венгрия, Исландия, Ирландия, Израиль, Италия, Иордания, Казахстан, Кения, Кувейт, Латвия, Ливан, Лихтенштейн, Литва, Люксембург, Македония (БЮРМ), Мальта, Черногория, Марокко, Нидерланды, Нигерия, Норвегия, Оман, Пакистан, Польша, Португалия, Катар, Румыния, Россия, Саудовская Аравия, Сербия, Словакия, Словения, Южно-Африканская Республика, Испания, Швеция, Швейцария, Тунис, Турция, Украина, Объединенные Арабские Эмираты и Соединенное Королевство.

После того как будет объявлено о общей доступности Azure AD B2C в вышеуказанных странах или регионах, вы можете продолжать использовать эти клиенты рабочей версии и запускать свои приложения без потери данных.

## Доступность клиентов B2C
В настоящее время клиенты B2C недоступны в следующих странах и регионах: Афганистан, Аргентина, Австралия, Бразилия, Чили, Колумбия, Эквадор, Гонконг, САР, Индия, Индонезия, Ирак, Япония, Корея, Малайзия, Новая Зеландия, Парагвай, Перу, Филиппины, Сингапур, Шри-Ланка, Тайвань, Таиланд, Уругвай и Венесуэла. Мы планируем обеспечить поддержку в этих регионах в будущем.

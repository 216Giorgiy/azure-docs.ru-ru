# Предварительная версия Azure Active Directory B2C: вспомогательное средство, используемое для демонстрации настройки пользовательского интерфейса страницы

Эта статья дополняет [основную](active-directory-b2c-reference-ui-customization) статью о настройке пользовательского интерфейса страницы в Azure Active Directory (AD) B2C.

Приведенные ниже шаги позволят вам поработать с функцией настройки пользовательского интерфейса, используя наш образец содержимого. Для целей этой демонстрации используется **статическое** содержимое HTML5.

1. Отправьте образец содержимого в [хранилище больших двоичных объектов Azure](http://azure.microsoft.com/documentation/services/storage/), сделайте его общедоступным через HTTPS и включите для этих URL-адресов общий доступ к ресурсам независимо от источника (CORS).
2. Измените параметры настройки пользовательского интерфейса страницы в существующей политике регистрации и укажите упомянутые выше URL-адреса.

Перед началом работы:

- Соберите одно из приложений быстрого запуска Azure AD B2C, перечисленных [здесь](active-directory-b2c-overview.md). На этом этапе вы также создадите политику регистрации, которую нужно будет изменить.
- Скачайте [вспомогательное средство](https://github.com/AzureADSamples/B2C-AzureBlobStorage-Client).

### Шаг 1. Отправка образца содержимого в надлежащие расположения

1. Войдите на [портал предварительной версии Azure](https://portal.azure.com/).
2. Щелкните **+ Создать** -> **Данные + хранилище** -> **Учетная запись хранения**. Для создания учетной записи хранения больших двоичных объектов Azure требуется подписка Azure. Бесплатную пробную версию можно получить [здесь](https://azure.microsoft.com/ru-RU/pricing/free-trial/).
3. Введите **имя** учетной записи хранения (например, contoso.core.windows.net) и выберите соответствующие значения для параметров **Ценовая категория**, **Группа ресурсов** и **Подписка**. Убедитесь, что флажок **Закрепить на начальной панели** установлен. Щелкните **Создать**.
4. Вернитесь на начальную панель и щелкните только что созданную учетную запись хранения.
5. В разделе **Сводка** щелкните **Контейнеры** и **+ Добавить**.
6. Введите **имя** контейнера (например, b2c) и для параметра **Тип доступа** выберите значение **Большой двоичный объект**. Нажмите кнопку **ОК**.
7. Созданный контейнер появится в списке в колонке **Большие двоичные объекты**. Запишите URL-адрес контейнера. Он может выглядеть так: `https://contoso.blob.core.windows.net/b2c`. Закройте колонку **Большие двоичные объекты**.
8. В колонке учетной записи хранения щелкните **Ключи** и запишите значения полей **Имя учетной записи хранения** и **Первичный ключ доступа**.

    > [AZURE.NOTE]**Первичный ключ доступа** — важная составляющая системы безопасности.

9. Запустите вспомогательное средство и укажите в нем записанные ранее значения **Имя учетной записи хранения** и **Первичный ключ доступа**. В результате образец содержимого будет отправлен в вашу учетную запись хранения.
10. Убедитесь, что содержимое отправлено правильно. Для этого попытайтесь открыть в браузере страницу `https://contoso.blob.core.windows.net/b2c/idp.html`. Кроме того, откройте [http://test-cors.org/](http://test-cors.org/) и проверьте, включен ли для содержимого CORS-доступ (вам нужен результат `XHR status: 200`).

### Шаг 2. Изменение параметров настройки пользовательского интерфейса страницы в политике регистрации

1. Войдите в свой каталог на [портале предварительной версии Azure](https://portal.azure.com) и откройте колонку функций B2C.
2. Щелкните **Политики регистрации** и выберите свою политику (например, B2C\_1\_SiIn).
3. Последовательно щелкните **Настройка пользовательского интерфейса страницы** и **Страница выбора поставщика удостоверений**.
4. Установите переключатель **Использовать настраиваемый шаблон** в положение **Да**. В поле **универсальный код ресурса (URI) настраиваемой страницы** введите соответствующий URL-адрес содержимого, отправленного в вашу учетную запись хранения. Пример: `https://contoso.blob.core.windows.net/b2c/idp.html`. Нажмите кнопку **ОК**.
5. Щелкните **Страница регистрации локальной учетной записи**. Установите переключатель **Использовать настраиваемый шаблон** в положение **Да**. В поле **универсальный код ресурса (URI) настраиваемой страницы** введите соответствующий URL-адрес содержимого, отправленного в вашу учетную запись хранения. Например, `https://contoso.blob.core.windows.net/b2c/su.html`. Нажмите **ОК** дважды.
6. Щелкните **Сохранить**.
7. Выберите команду **Выполнить сейчас** в верхней части колонки. В раскрывающемся списке **Приложения** выберите вариант «Приложение B2C», а в списке **URL-адрес ответа или универсальный код ресурса (URI) перенаправления** выберите `https://localhost:44321/`. Нажмите кнопку **Выполнить сейчас**.
8. Откроется новая вкладка браузера, и вы сможете зарегистрировать пользователя в приложении, в котором уже есть новое содержимое.

> [AZURE.NOTE]Изменения в политике вступят в силу в течение минуты.

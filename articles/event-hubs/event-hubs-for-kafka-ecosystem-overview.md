# <a name="event-hubs-for-kafka-ecosystems"></a>Концентраторы событий для экосистем Kafka

Концентраторы событий для экосистем Kafka предоставляют конечную точку Kafka, которую существующие приложения на основе Kafka могут использовать в качестве альтернативы запуска собственного кластера Kafka. Концентраторы событий для экосистемы Kafka поддерживают [Apache Kafka 1.0](https://kafka.apache.org/10/documentation.html) и более новые версии клиента и работают с существующими приложениями Kafka, в том числе с MirrorMaker. Измените строку подключения и начните потоковую передачу событий из приложений, использующих протокол Kafka, в концентраторы событий.

## <a name="what-does-event-hubs-for-kafka-ecosystems-provide"></a>Какие возможности предоставляют концентраторы событий для экосистем Kafka?

Концентраторы событий для экосистемы Kafka предоставляют заголовок протокола на основе концентраторов событий Azure, который совместим по двоичным файлам с Kafka 1.0 и более поздними версиями, для чтения из разделов Kafka и записи в них. По существу, Kafka и концентраторы событий практически идентичны: они являются секционированными журналами, созданными для потоковой передачи данных. В следующей таблице показано составление понятий Kafka и концентраторов событий.

### <a name="kafka-and-event-hub-conceptual-mapping"></a>Сопоставление понятий Kafka и концентратора событий

| Понятия Kafka | Понятия концентраторов событий|
| --- | --- |
| HDInsight | Пространство имен |
| Раздел | Концентраторы событий |
| Секция | Секция|
| Группа потребителей | Группа потребителей |
| Offset | Offset|

### <a name="key-differences-between-kafka-and-event-hubs"></a>Основные различия между Kafka и концентраторами событий

Несмотря на то, что [Apache Kafka](https://kafka.apache.org/) — это программное обеспечение, которое можно использовать в любом месте, концентраторы событий представляют собой облачную службу, схожую с хранилищем BLOB-объектов Azure. Вам не нужно управлять серверами или сетями и настраивать брокеры — эти компоненты просто отсутствуют. Вы создаете пространство имен, которое является полным доменным именем, содержащим разделы, а затем в этом пространстве имен создаете концентраторы событий или разделы. Дополнительные сведения о концентраторах событий и пространствах имен см. в статье [Что такое концентраторы событий?](event-hubs-what-is-event-hubs.md) Выступая в качестве облачной службы, концентраторы событий используют один стабильный виртуальный IP-адрес в качестве конечной точки, поэтому клиентам не нужно знать о брокерах или машинах в кластере. 

Масштаб в концентраторах событий контролируется количеством приобретаемых единиц пропускной способности, где каждая такая единица предоставляет 1 МБ в секунду или 1000 событий в секунду входящих данных. По умолчанию концентраторы событий масштабируют единицы пропускной способности по достижении предела, используя функцию [автоматического расширения](event-hubs-auto-inflate.md). Эта функция также работает с концентраторами событий для экосистем Kafka. 

### <a name="security-and-authentication"></a>Безопасность и проверка подлинности

Концентраторам событий Azure требуется протокол SSL или TLS для всех видов взаимодействия и подписанные URL-адреса для аутентификации. Это требование также применяется к конечной точке Kafka в концентраторах событий. Для обеспечения совместимости с Kafka концентраторы событий используют SASL PLAIN для аутентификации и SASL SSL для обеспечения безопасности транспорта. Дополнительные сведения о безопасности в концентраторах событий см. в статье об [аутентификации и безопасности концентраторов событий](event-hubs-authentication-and-security-model-overview.md).

## <a name="other-event-hubs-features-available-for-kafka"></a>Другие функции концентраторов событий, доступные для Kafka

Концентраторы событий для экосистемы Kafka позволяют использовать для один протокол для записи и другой — для чтения. В этом случае текущие производители Kafka могут продолжить публикацию с помощью Kafka, а вы можете добавлять средства чтения с помощью концентраторов событий, например Azure Stream Analytics или Функции Azure. Кроме того, функции концентраторов событий, такие как [Захват](event-hubs-capture-overview.md) и [Геоизбыточное аварийное восстановление](event-hubs-geo-dr.md), также работают с концентраторами событий для экосистем Kafka.

## <a name="features-that-are-not-supported-in-the-preview"></a>Функции, которые не поддерживаются в предварительной версии

В общедоступной предварительной версии интеграции концентраторов событий для экосистем Kafka не поддерживаются следующие функции Kafka:

*   идемпотентный производитель;
*   транзакция:
*   Сжатие
*   хранение на основе размера;
*   сжатие журнала;
*   добавлений секций в существующий раздел;
*   поддержка API HTTP Kafka;
*   Kafka Connect;
*   потоки Kafka.

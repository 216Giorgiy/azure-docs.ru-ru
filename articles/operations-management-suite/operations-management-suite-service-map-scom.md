# <a name="service-map-integration-with-system-center-operations-manager-integration"></a>Интеграция схемы услуги с System Center Operations Manager
Схема услуги Operations Management Suite (OMS) автоматически обнаруживает компоненты приложений в системах Windows и Linux и сопоставляет взаимодействие между службами. Это решение позволяет рассматривать серверы как взаимосвязанные системы, предоставляющие важные службы. Схема услуги отображает сведения о подключениях между серверами, процессами и портами в любой подключенной по протоколу TCP архитектуре без дополнительной настройки. Пользователям требуется только установить агент.  Дополнительные сведения см. в [документации по схеме услуги](operations-management-suite-service-map.md).

С помощью этой предварительной версии интеграции схемы услуги и System Center Operations Manager (SCOM) можно автоматически создавать схемы распределенных приложений в SCOM на основе динамического сопоставления зависимостей в схеме услуги.

# <a name="prerequisites"></a>Предварительные требования
1.    Группы управления SCOM, управляющие набором серверов.
2.    Рабочая область OMS с включенным решением схемы услуги.
3.    Набор серверов (по крайней мере один), которые находятся под управлением SCOM и отправляют данные в схему услуги.  Поддерживаются серверы Windows и Linux.
4.    Субъект-служба с доступом к подписке Azure, связанной с рабочей областью OMS.  Дополнительные сведения о субъектах-службах см. в приложении.

# <a name="installing-service-map-management-pack"></a>Установка пакета управления схемы услуги
Интеграция SCOM и схемы услуг обеспечивается путем импорта набора пакетов управления Microsoft.SystemCenter.ServiceMap (Microsoft.SystemCenter.ServiceMap.mpb).  Этот набор содержит следующие пакеты управления:
* Microsoft.ServiceMap.Application.Views.
* Microsoft.System.Center.ServiceMap.Internal.
* Microsoft.System.Center.ServiceMap.Overrides.
* Microsoft.System.Center.ServiceMap.

# <a name="configuring-the-service-map-integration"></a>Настройка интеграции схемы услуги
1. После установки пакета управления схемы услуги в области администрирования в разделе Operations Management Suite появится новый узел, "Схема услуги".
2. Щелкните "Добавить рабочую область" в области "Service Map Overview" (Обзор схемы услуги), чтобы открыть мастер настройки.

    ![Мастер настройки SCOM](media/oms-service-map/scom-configuration.png)

3. Первый шаг в мастере — настройка подключения для указания имени субъекта-службы. Введите идентификатор клиента, идентификатор приложения (либо имя пользователя или ClientID) и пароль субъекта-службы.  Сведения о создании субъекта-службы приведены в разделе "Приложение".

    ![Настройка имени субъекта-службы для SCOM](media/oms-service-map/scom-config-spn.png)

4. Следующий шаг — выбор подписки Azure, группы ресурсов Azure (которая содержит рабочую область OMS) и рабочей области OMS.

    ![Настройка рабочей области для SCOM](media/oms-service-map/scom-config-workspace.png)

5. Следующим шагом является настройка группы серверов схемы услуги для синхронизации между SCOM и схемой услуги.  Нажмите кнопку "Добавление и удаление серверов". кнопка. Обратите внимание на то, что для интеграции и успешного построения схемы распределенного приложения для сервера он должен: 1) находиться под управлением SCOM, 2) находиться под управлением схемы услуги и 3) входить в группу серверов схемы услуги.

    ![Настройка группы для SCOM](media/oms-service-map/scom-config-group.png)

6. Необязательно. Выберите пул ресурсов сервера управления для взаимодействия с OMS и щелкните "Добавить рабочую область".

    ![Настройка пула ресурсов для SCOM](media/oms-service-map/scom-config-pool.png)

7. Обратите внимание, что для настройки и регистрации рабочей области OMS потребуется около минуты. После завершения настройки SCOM начнет первую синхронизацию схемы услуги из OMS.

    ![Настройка пула ресурсов для SCOM](media/oms-service-map/scom-config-success.png)

**Примечание.** Интервал синхронизации по умолчанию составляет 60 минут. Пользователи могут переопределить этот интервал синхронизации. Пользователи могут также вручную добавить серверы в группу серверов схемы услуги с помощью области "Разработка" (щелкните область "Разработка" > "Группы", а затем найдите "Service Map Servers Group" (Группа серверов схемы услуги)). Карты серверов для этих серверов будут синхронизированы во время следующей синхронизации (в зависимости от заданного интервала синхронизации).

# <a name="monitoring-service-map"></a>Мониторинг схемы услуги
После подключения рабочей OMS в области "Мониторинг" консоли SCOM появится новая папка "Схема услуги".
![Мониторинг SCOM](media/oms-service-map/scom-monitoring.png)

В папке "Схема услуги" имеется три узла.
## <a name="all-alerts"></a>"Все оповещения"
Показывает все оповещения об обмене данными между SCOM и решением схемы услуги в OMS.

**Примечание.** Это не оповещения OMS, отображаемые в SCOM.
## <a name="servers"></a>"Серверы"
Содержит список отслеживаемых серверов, настроенных для синхронизации из схемы услуги.

![Серверы мониторинга SCOM](media/oms-service-map/scom-monitoring-servers.png)

## <a name="server-dependency-views"></a>"Server Dependency Views" (Представления зависимости серверов)
Это представление будет отображать список всех серверов, синхронизируемых из схемы услуги. Пользователи могут щелкнуть любой сервер, чтобы просмотреть его схему распределенного приложения.

![Схема распределенного приложения SCOM](media/oms-service-map/scom-dad.png)

# <a name="editdelete-workspace"></a>Изменение или удаление рабочей области
Пользователи могут изменить или удалить настроенную рабочую область с помощью области "Service Map Overview" (Обзор схемы услуги) (область "Администрирование" > "Operations Management Suite" > "Схема услуги").  Обратите внимание, что пока можно настроить только одну рабочую область OMS.

![Изменение рабочей области SCOM](media/oms-service-map/scom-edit-workspace.png)

# <a name="configuring-rules-and-overrides"></a>Настройка правил и переопределений
Правило **_Microsoft.SystemCenter.ServiceMap.Import.Rule**_ создается для периодического извлечения сведений из схемы услуги.  Пользователи могут настроить переопределения этого правила, чтобы изменить время синхронизации.
Область "Разработка" > "Правила" > "Microsoft.SystemCenter.ServiceMapImport.Rule".

![Переопределения для SCOM](media/oms-service-map/scom-overrides.png)
* **Enabled** — позволяет включить или отключить автоматическое обновление. 
* **IntervalSeconds** — время между обновлениями.  Значение по умолчанию — 1 час. Пользователи могут изменить это значение, если необходимо чаще синхронизировать карты серверов.
* **TimeoutSeconds** — время ожидания запроса. 
* **TimeWindowMinutes** — временной диапазон запроса данных.  Значение по умолчанию — 60 минут. Максимальное значение — 1 час (максимально допустимое значение для схемы услуги).

# <a name="known-issueslimitations"></a>Известные проблемы и ограничения
В текущей версии:
1. Хотя пользователи могут вручную добавить серверы в группу серверов схемы услуги с помощью области "Разработка", карты для этих серверов будут синхронизированы из схемы услуги только во время следующего цикла синхронизации. По умолчанию он составляет&60; минут. Пользователи могут переопределить время синхронизации. 
2. Пользователи могут подключиться к отдельной рабочей области OMS.

# <a name="choosing-the-right-pricing-tier-in-azure-maps"></a>Выбор подходящей ценовой категории в Azure Maps

Для Azure Maps предлагается две ценовых категории. Цель этой статьи — помочь вам выбрать правильную ценовую категорию, которая соответствует вашим потребностям. Задайте себе два вопроса, которые помогут вам выбрать правильную ценовую категорию:

## <a name="what-geospatial-capabilities-do-i-plan-to-use"></a>Какие геопространственные возможности планируется использовать?
Если вы считаете, что основные геопространственные модули API соответствуют требованиям службы, тогда вам подходит ценовая категория S0. Если вашему приложению требуются расширенные возможности, такие как аэроснимки и гибридные снимки, получение диапазона маршрута, пакетное геокодирование и т. д, попробуйте ценовую категорию S1. Таблица **возможностей ценовой категории** ниже поможет вам лучше понять потребности своего приложения, а также выбрать ценовую категорию, которая лучше всего для него подходит.

## <a name="how-many-concurrent-users-do-i-plan-to-support"></a>Сколько одновременно работающих пользователей планируется поддерживать? 
Ценовые категории S0 и S1 могут обрабатывать различные объемы пропускной способности данных. Перед выбором ценовой категории Azure Maps подумайте, сколько одновременно работающих пользователей вы хотите поддерживать? Ценовая категория S0 может обрабатывать до **50 запросов в секунду**, а ценовая категория S1 — **более 50 запросов в секунду**.

### <a name="pricing-tier-capabilities"></a>Возможности ценовых категорий

| Функция                              |        S0           |  S1      |
|-----------------------------------------|:-------------------:|:--------:|
| поиска                                  |        ✓           |     ✓    |
| Маршрутизация                                 |        ✓           |     ✓    |
| Отображение                                  |        ✓           |     ✓    |
| Трафик                                 |        ✓           |     ✓    |
| Часовые пояса                              |        ✓           |     ✓    |
| *Снимки + гибридные снимки (предварительная версия)     |        ✓           |     ✓    |
| *Диапазон маршрута (предварительная версия)                  |        ✓           |     ✓    |
| *Расположение IP-адреса 2 (предварительная версия)                |        ✓           |     ✓    |
| *Многоугольники из поиска (предварительная версия)         |        ✓           |     ✓    |
| *Пакетное геокодирование (предварительная версия)              |        ✓           |     ✓    |
| *Пакетная маршрутизация (предварительная версия)                |        ✓           |     ✓    |
| *Матричная маршрутизация (предварительная версия)               |        ✓           |     ✓    |

> [!Note]
> Доступ к этим возможностям с помощью ценовой категории S0 перестанет поддерживаться после 4 февраля 2019 года.

Вот некоторые дополнительные точки данных, которые стоит рассмотреть: какое у вас предприятие или каков уровень важности создаваемого приложения?

Ознакомьтесь с таблицей **ценовых категорий для целевых клиентов**, чтобы лучше понять смысл ценовых категорий S0 и S1. Дополнительные сведения о ценовых категориях Azure Maps см. на [странице цен Azure Maps](https://azure.microsoft.com/pricing/details/azure-maps/). 

### <a name="pricing-tier-targeted-customers"></a>Целевые клиенты ценовой категории

| Ценовой уровень  |        Целевые клиенты                                                                |
|---------------|:-----------------------------------------------------------------------------------------|
| S0            |    <p>Ценовая категория S0 предназначена для клиентов-предприятий небольшого и среднего размера. Это подходящая ценовая категория, если вы не ожидаете большое количество одновременно работающих пользователей, а вашим требованиям к службе соответствуют геопространственные модули API, как указано в таблице выше. Этот уровень находится в общем доступе и применяется для приложений на всех этапах производства, от подтверждения концепции и ранних стадий тестирования до вывода приложения в рабочую среду и развертывания.<p>|
| S1            |    <p>Ценовая категория S1 предназначена для клиентов, которым необходима поддержка крупномасштабных критически важных корпоративных приложений, большого количества одновременно работающих пользователей или требуются расширенные геопространственные службы.</p>|

# <a name="quickstart-use-your-own-notebook-server-to-get-started-with-azure-machine-learning"></a>Краткое руководство. Использование собственного сервера записных книжек для начала работы с Машинным обучением Azure

В этой статье описано, как использовать собственный сервер записных книжек для запуска кода, расположенного в [рабочей области службы "Машинное обучение Azure"](concept-azure-machine-learning-architecture.md). Эта рабочая область — основной блок в облаке для экспериментов, обучения и развертывания моделей машинного обучения с помощью Машинного обучения Azure.

В этом кратком руководстве описано, как использовать собственное окружение Python и сервер Jupyter Notebook. Сведения о запуске без установки см. в [кратком руководстве по использованию облачного сервера записных книжек для начала работы с Машинным обучением Azure](quickstart-run-cloud-notebook.md). 

Просмотрите видеоверсию этого краткого руководства:

> [!VIDEO https://www.microsoft.com/en-us/videoplayer/embed/RE2G9N6]

В этом кратком руководстве вы:

* запишите код, который будет записывать в журнал значения внутри рабочей области;
* просмотрите записанные значения в рабочей области.

Если у вас еще нет подписки Azure, создайте бесплатную учетную запись Azure, прежде чем начинать работу. Опробуйте [бесплатную или платную версию Службы машинного обучения Azure](https://aka.ms/AMLFree).

## <a name="prerequisites"></a>Предварительные требования

* Сервер записных книжек Python 3.6 с установленным пакетом SDK Машинного обучения Azure.
* Рабочая область службы машинного обучения Azure.
* Файл конфигурации рабочей области (**aml_config/config.json**).

Все эти предварительные требования описаны в руководстве по [созданию рабочей области Машинного обучения Azure](setup-create-workspace.md#portal).

# <a name="deploy-models-with-the-azure-machine-learning-service"></a>Развертывание моделей с помощью Службы машинного обучения Azure

Служба машинного обучения Azure предоставляет несколько способов развертывания обученной модели. В этом документе вы научитесь развертывать модель как веб-службу в облаке Azure и на устройствах IoT Edge.

Вы можете развертывать модели в следующих целевых средах вычислений:

- Экземпляры контейнеров Azure (ACI).
- Служба Azure Kubernetes (AKS)
- Edge Интернета вещей Azure
- Программируемые пользователем вентильные матрицы (FPGA).

Каждый из этих объектов подробно рассмотрен далее в этом документе.

## <a name="azure-container-instances"></a>Экземпляры контейнеров Azure

Используйте Экземпляры контейнеров Azure (ACI) для развертывания моделей в качестве конечной точки REST API, если выполняется одно или несколько из следующих условий:
- Вы ищете быстрое развертывание для оценки и проверки модели. Обычно развертывание в ACI занимает менее 5 минут.
- Вы хотите развернуть модель в среде разработки или тестирования. ACI позволяет развертывать 20 групп контейнеров для одной подписки. Дополнительные сведения см. в статье [Квоты и доступность по регионам для службы "Экземпляры контейнеров Azure"](https://docs.microsoft.com/azure/container-instances/container-instances-quotas).

Дополнительные сведения см. в статье [Развертывание веб-служб в Экземплярах контейнеров Azure](how-to-deploy-to-aci.md).

## <a name="azure-kubernetes-service"></a>Служба Azure Kubernetes

Для крупномасштабных рабочих сценариев можно развернуть модель в Службе Azure Kubernetes (AKS). Можно использовать существующий кластер AKS или создать его с помощью пакета SDK для службы "Машинное обучение Azure", CLI или портала Azure.

Кластер AKS для рабочей области нужно создать только один раз. Созданный кластер можно использовать для различных развертываний. Если вы удаляете кластер или группу ресурсов, в которую он входит, во время следующего развертывания нужно будет создать новый кластер.

Развертывание в AKS обеспечивает автоматическое масштабирование, ведение журналов, сбор данных модели и короткое время отклика веб-служб. 

Процесс создания кластера AKS занимает приблизительно 20 минут.

Дополнительные сведения см. в статье [Развертывание моделей из Службы машинного обучения Azure в Службе Azure Kubernetes](how-to-deploy-to-aks.md).

## <a name="azure-iot-edge"></a>Edge Интернета вещей Azure

С устройствами Интернета вещей быстрее выполнить оценку на устройстве вместо отправки данных в облако и ожидания ответа от размещенной в облаке модели. С Azure IoT Edge вы можете разместить модель на пограничных устройствах. Развертывайте модель в IoT Edge, если вам требуется одна или несколько из следующих возможностей:
- Обработка приоритетных задач локально, даже без подключения к облаку.
- Работа со сформированными данными,которые слишком велики для быстрого извлечения из облака.
- Выполнение обработки в режиме реального времени на основе аналитики в локальных устройствах или рядом с ними.
- Удовлетворение требованиям к конфиденциальности данных. 

Дополнительные сведения см. в статье [Руководство: развертывание службы "Машинное обучение Azure" в качестве модуля IoT Edge (предварительная версия)](https://docs.microsoft.com/azure/iot-edge/tutorial-deploy-machine-learning).

Дополнительные сведения о службе IoT Edge см. в [этой статье](https://docs.microsoft.com/azure/iot-edge/).


## <a name="field-programmable-gate-arrays-fpga"></a>Программируемые пользователем вентильные матрицы (FPGA)

Модели с аппаратным ускорением работают на Project Brainwave, что позволяет добиться сверхнизкой задержки для запросов в режиме реального времени. Project Brainwave ускоряет обработку глубоких нейронных сетей (DNN), развернутых в программируемые пользователем вентильные матрицы в облаке Azure. Часто используемые DNN доступны в качестве характеризаторов для переноса обучения. Кроме того, их можно настроить с использованием весовых коэффициентов, обученных на основе ваших данных.

Дополнительные сведения см. в статье [Как развернуть модель как веб-службу в FPGA с помощью Машинного обучения Azure](how-to-deploy-fpga-web-service.md).

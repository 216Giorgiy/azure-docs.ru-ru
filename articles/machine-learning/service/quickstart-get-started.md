# <a name="quickstart-use-the-azure-portal-to-get-started-with-azure-machine-learning"></a>Краткое руководство. Начало работы со Службой машинного обучения Azure с помощью портала Azure

В этом кратком руководстве вы используете портал Azure для создания рабочей области "Машинное обучение Azure". Эта рабочая область — основной блок в облаке для экспериментов, обучения и развертывания моделей машинного обучения с помощью Машинного обучения Azure. 

Изучив это руководство, вы:

* Создавать рабочую область в своей подписке Azure.
* Попробуете использовать ее с Python в записной книжке Azure и запишите значения из нескольких итераций.
* Просмотрите записанные значения в рабочей области.

Следующие ресурсы Azure добавляются автоматически в рабочую область, если они доступны в вашем регионе.

  - [Реестр контейнеров Azure](https://azure.microsoft.com/services/container-registry/)
  - [Хранилище Azure](https://azure.microsoft.com/services/storage/)
  - [Azure Application Insights](https://azure.microsoft.com/services/application-insights/) 
  - [Хранилище ключей Azure](https://azure.microsoft.com/services/key-vault/)

Созданные ресурсы могут использоваться в качестве необходимых компонентов для других руководств и статей с практическими рекомендациями по Службе машинного обучения. Как и в других службах Azure, существуют ограничения на некоторые ресурсы, связанные со Службой машинного обучения. Например, размер кластера Azure Batch AI. Информацию об ограничениях по умолчанию и порядке увеличения квоты см. в [этой статье](how-to-manage-quotas.md).

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="use-the-workspace"></a>Используйте рабочую область

Теперь познакомьтесь с тем, как рабочая область помогает управлять сценариями машинного обучения. В этом разделе выполняются следующие действия:

* Откроете записную книжку в Записных книжках Azure.
* Запустите код, который создает записанные значения.
* Просмотрите записанные значения в рабочей области.

Этот пример демонстрирует, как рабочая область может помочь вам отследить информацию, создаваемую в сценарии. 

### <a name="open-a-notebook"></a>Откройте записную книжку 

Azure Notebooks предоставляет бесплатную облачную платформу для записных книжек Jupyter, предварительно настроенную для запуска службы "Машинное обучение Azure".  

Выберите `Open Azure Notebooks`, чтобы опробовать первый эксперимент.

 ![Открытие Записных книжек Azure](./media/quickstart-get-started/explore_ws.png)

Ваша организация может требовать [согласия администратора](https://notebooks.azure.com/help/signing-up/work-or-school-account/admin-consent) для входа.

После входа откроется вкладка и отобразится подсказка `Clone Library`. Выберите `Clone`


### <a name="run-the-notebook"></a>Запустите записную книжку

Рядом с двумя записными книжками вы увидите файл `config.json`. Этот файл конфигурации содержит сведения о рабочей области, которую вы создали.  

Выберите `01.run-experiment.ipynb`, чтобы открыть записную книжку.

Чтобы запускать ячейки по очереди, используйте `Shift`+`Enter`. Вы также можете выбрать `Cells` > `Run All`, чтобы запустить всю записную книжку. Если рядом с ячейкой отображается звездочка [*], значит она выполняется. Когда код для этой ячейки будет выполнен, появится номер.

Возможно, вам будет предложено войти в систему. Скопируйте код в сообщении. Затем перейдите по ссылке и вставьте код в новое окно. Не забудьте скопировать пробел до или после кода. Войдите в систему, используя ту же учетную запись, что и на портале Azure.

 ![Вход в систему](./media/quickstart-get-started/login.png)

В записной книжке вторая ячейка считывает из `config.json` для подключения к рабочей области.
```
ws = Workspace.from_config()
```

В третьей ячейке кода запускается эксперимент под названием my-first-experiment. Используйте это имя для поиска сведений о выполнении в рабочей области.

```
experiment = Experiment(workspace_object=ws, name = "my-first-experiment")
```

В последней ячейке записной книжки содержатся значения, которые записываются в файл журнала.

```
# Log final results
run.log("Final estimate: ",pi_estimate)
run.log("Final error: ",math.pi-pi_estimate)
```

Эти значения можно просмотреть в рабочей области после выполнения кода.

## <a name="view-logged-values"></a>Просмотр зарегистрированных в журнале значений

После заполнения всех ячеек в записной книжке вернитесь на страницу портала.  

Выберите `View Experiments`.

![Просмотр экспериментов](./media/quickstart-get-started/view_exp.png)

Закройте всплывающее окно `Reports`.

Выберите `my-first-experiment`.

См. сведения о запуске, который вы только что выполнили. Прокрутите страницу вниз до таблицы выполнения. Выберите ссылку с номером выполнения.

 ![Ссылка на журнал выполнения](./media/quickstart-get-started/report.png)

Вы увидите графики, которые были автоматически созданы из записанных значений:  

   ![Просмотр истории](./media/quickstart-get-started/plots.png)

Так как код для приблизительного расчета числа Пи использует случайные значения, на графиках будут отображаться разные значения.

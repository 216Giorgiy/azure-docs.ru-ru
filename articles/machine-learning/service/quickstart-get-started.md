# <a name="quickstart-use-the-azure-portal-to-get-started-with-azure-machine-learning-service"></a>Краткое руководство. Начало работы со Службой машинного обучения Azure с помощью портала Azure

В этом кратком руководстве вы используете портал Azure для создания рабочей области Службы машинного обучения Azure. Указанная рабочая область — это основной блок облака для экспериментов, обучения и развертывания моделей машинного обучения с помощью службы машинного обучения Azure. 

Изучив данный учебник, вы научитесь:

* Создавать рабочую область в своей подписке Azure
* Попробуйте его с Python в записной книжке Azure и записывайте значения из нескольких итераций
* Просмотрите записанные значения в рабочей области

Для удобства следующие ресурсы Azure автоматически добавляются в рабочую область, если они доступны в вашем регионе: [реестр контейнеров](https://azure.microsoft.com/services/container-registry/), [хранилище](https://azure.microsoft.com/services/storage/), [аналитические сведения о приложении](https://azure.microsoft.com/services/application-insights/) и [хранилище ключей](https://azure.microsoft.com/services/key-vault/).

Созданные ресурсы могут использоваться в качестве необходимых компонентов для других руководств и статей с практическими рекомендациями по Службе машинного обучения Azure. Как и в других службах Azure, существуют ограничения на некоторые ресурсы (например, размер кластера BatchAI), связанные со службой машинного обучения Azure. Ознакомьтесь с [этой](how-to-manage-quotas.md) статьей об ограничениях по умолчанию и порядке запроса увеличения квоты.

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="use-the-workspace"></a>Используйте рабочую область

Теперь познакомьтесь с тем, как рабочая область помогает управлять сценариями машинного обучения. В этом разделе выполните следующие действия:

* Откройте записную книжку в Azure Notebooks
* Запустите код, который создает записанные значения
* Просмотрите записанные значения в рабочей области

Это один из примеров того, как рабочая область может помочь вам отследить информацию, создаваемую в сценарии. 

### <a name="open-a-notebook"></a>Откройте записную книжку 

Azure Notebooks предоставляет бесплатную облачную платформу для записных книжек Jupyter, предварительно настроенную для запуска службы "Машинное обучение Azure".  

Щелкните кнопку `Open Azure Notebooks` для первого эксперимента.

 ![Запустите Azure Notebook](./media/quickstart-get-started/explore_ws.png)

После входа откроется вкладка и отобразится подсказка `Clone Library`.  Щелкните `Clone`


### <a name="run-the-notebook"></a>Запустите записную книжку

Рядом с двумя записными книжками вы увидите файл `config.json`.  Этот файл конфигурации содержит сведения о рабочей области, которую вы только что создали.  

Щелкните `01.run-experiment.ipynb`, чтобы открыть записную книжку.

Вы можете запускать ячейки по очереди, используя `Shift`+`Enter`.  Или используйте меню `Cells` > `Run All`, чтобы запустить всю записную книжку.  Если рядом с ячейкой есть символ [*], значит, она запущена.  Когда код для этой ячейки будет выполнен, отобразится номер.

Возможно, вам будет предложено войти в систему.  Скопируйте код в сообщении, затем щелкните ссылку и вставьте код в новом окне.  Не забудьте скопировать пробел до или после кода.  Войдите в систему, используя ту же учетную запись, что и на портале Azure.

 ![вход](./media/quickstart-get-started/login.png)

В записной книжке вторая ячейка считывает из `config.json` для подключения к рабочей области.
```
ws = Workspace.from_config()
```

В третьей ячейке кода запускается эксперимент под названием "Мой первый эксперимент".  Это имя будет использоваться для поиска сведений о выполнении в рабочей области.

```
experiment = Experiment(workspace_object=ws, name = "my-first-experiment")
```

Обратите внимание, что в последней ячейке записной книжки содержатся значения, которые записываются в файл журнала.

```
# Log final results
run.log("Final estimate: ",pi_estimate)
run.log("Final error: ",math.pi-pi_estimate)
```

Эти значения можно просмотреть в рабочей области после выполнения кода.

## <a name="view-logged-values"></a>Просмотр зарегистрированных в журнале значений

После заполнения всех ячеек в записной книжке вернитесь на страницу портала.  

Щелкните `View Experiments`.

![Открытие экспериментов](./media/quickstart-get-started/view_exp.png)

Закройте всплывающее окно `Reports`.

Щелкните `my-first-experiment`.

См. сведения о запуске, который вы только что выполнили.  Прокрутите страницу вниз, чтобы найти таблицу запусков, а затем щелкните ссылку с номером запуска.

 ![ссылка с историей запусков](./media/quickstart-get-started/report.png)

Вы увидите графики, которые были автоматически созданы из записанных значений:

   ![просмотр истории](./media/quickstart-get-started/plots.png)

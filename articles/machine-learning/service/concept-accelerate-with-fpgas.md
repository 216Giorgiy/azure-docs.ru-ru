# <a name="what-is-fpga-and-project-brainwave"></a>Что такое FPGA и Project Brainwave?

В этой статье содержатся общие сведения о программируемых пользователем вентильных матрицах (FPGA) и о том, каким образом служба "Машинное обучение Azure" предоставляет механизм искусственного интеллекта в режиме реального времени при развертывании модели в Azure FPGA.

FPGA содержат массив программируемых логических блоков и иерархию настраиваемых межсоединений. Межсоединения позволяют настраивать эти блоки различными способами после производства. В отличие от других микросхем FPGA обеспечивают сочетание программируемости и производительности.

## <a name="fpgas-vs-cpu-gpu-and-asic"></a>FPGAs vs. CPU, GPU и ASIC

![Сравнение машинного обучения Azure FPGA](./media/concept-accelerate-with-fpgas/azure-machine-learning-fpga-comparison.png)

|Процессор||ОПИСАНИЕ|
|---|:-------:|------|
|Микросхемы приложений|ASIC|Пользовательские микросхемы, такие как процессор TensorFlow (TPU) от Google, предоставляют повышенную эффективность. Их нельзя настраивать повторно в случае изменения требований.|
|Программируемые пользователем вентильные матрицы|FPGA|FPGA, например доступные в Azure, обеспечивают производительность, близкую к ASIC, но и их можно настроить через некоторое время для реализации новой логики.|
|Графические процессоры|GPU|Широко применяются для вычислений с использованием искусственного интеллекта. Они обеспечивают возможности параллельной обработки, что позволяет выполнять отрисовку образов быстрее, чем ЦП.|
|Центральные процессоры|ЦП|Процессоры общего назначения, производительность которых не подходит для обработки графики и видео.|

## <a name="project-brainwave-on-azure"></a>Project Brainwave в Azure

[Project Brainwave](https://www.microsoft.com/research/project/project-brainwave/) — экономичная аппаратная архитектура корпорации Майкрософт, основанная на устройствах FPGA от Intel, которые специалисты по обработке и анализу данных и разработчики используют для ускорения вычислений с применением искусственного интеллекта в режиме реального времени.  Эта архитектура с поддержкой FPGA доступна в Azure и предоставляет **производительность**, **гибкость** и **масштабирование**.

**FPGA позволяют добиться низкой задержки для запросов в режиме реального времени.** Пакетная обработка означает сбор большого объема данных и передачу этих данных в процессор для повышения эффективности использования оборудования. Пакетная обработка может повысить пропускную способность, но вызвать задержку, так как необходимо обработать больше данных. Для реализации нейроморфных процессоров Project Brainwave не требуется пакетная обработка. Таким образом, задержка может быть намного ниже по сравнению с ЦП и GPU.

### <a name="reconfigurable-power"></a>Настраиваемое питание
**FPGA можно перенастроить для различных типов моделей машинного обучения.** Такая гибкость упрощает ускорение приложений на основе наиболее оптимальной точности числовых значений и используемой модели памяти.

Новые методы машинного обучения разрабатываются на регулярной основе, и также быстро развивается разработка оборудования Project Brainwave. Поскольку FPGA реконфигурируются, можно оставаться в курсе с требованиями быстро меняющихся алгоритмов AI.

### <a name="whats-supported-on-azure"></a>Поддерживаемые в Azure функции
**Microsoft Azure — крупнейшая в мире облачная инвестиция в FPGA.** Project Brainwave можно запускать на инфраструктуре масштаба Azure.

Сейчас Project Brainwave поддерживает:
+ Сценарии классификации и распознавания изображений.
+ Развертывание TensorFlow.
+ DNN: ResNet 50, ResNet 152, VGG-16, SSD VGG и DenseNet 121.
+ Оборудование FPGA Intel. 

Используя эту аппаратную архитектуру с поддержкой FPGA, обученные нейронные сети работают быстро и с меньшей задержкой. Технология Project Brainwave позволяет параллелизовать предварительно обученные глубокие нейронные сети (DNN) через матрицы FPGA, что обеспечивает возможности масштабирования службы. DNN можно предварительно обучить как глубокий характеризатор для переноса обучения или настроить с использованием обновленных весовых коэффициентов.

### <a name="scenarios-and-applications"></a>Сценарии и приложения

Project Brainwave интегрировано со службой "Машинное обучение Azure". Корпорация Майкрософт использует FPGA для оценки DNN, ранжирования поиска Bing и ускорения работы программно-конфигурируемой сети, чтобы уменьшить задержку, одновременно освобождая ЦП для других задач.

## <a name="deploy-to-fpgas-on-azure"></a>Развертывание в FPGA в Azure

Ниже приведен рабочий процесс создания службы распознавания изображений в Azure с использованием поддерживаемых DNN в качестве характеризатора для развертывания в FPGA Azure.

1. С помощью пакета SDK для службы "Машинное обучение Azure" для Python создайте определение службы, которое представляет собой файл, описывающий конвейер графиков (входящие данные, характеризатор и классификатор) на основе TensorFlow. Команда развертывания автоматически выполнит сжатие определения и графов в ZIP-файл и загрузит этот файл в хранилище BLOB-объектов Azure.  DNN уже развернута в Project Brainwave для запуска в FPGA.

1. Зарегистрируйте модель с помощью пакета SDK, используя ZIP-файл в хранилище BLOB-объектов Azure.

1. Разверните службу с зарегистрированной моделью с помощью пакета SDK.

Чтобы приступить к развертыванию обученных моделей DNN для FPGA в облаке Azure, ознакомьтесь со статьей **[Как развернуть модель как веб-службу в FPGA с помощью Машинного обучения Azure](how-to-deploy-fpga-web-service.md)**.

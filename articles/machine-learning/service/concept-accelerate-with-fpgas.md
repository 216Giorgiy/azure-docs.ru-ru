# <a name="what-is-fpga-and-project-brainwave"></a>Что такое FPGA и Project Brainwave?

В этой статье представлено введение в программируемые пользователем вентильные матрицы (FPGA) и то, как FPGA интегрируется с обучением в Azure для обеспечения AI в реальном времени.

## <a name="fpgas-vs-cpu-gpu-and-asic"></a>FPGAs vs. CPU, GPU и ASIC

FPGA содержат массив программируемых логических блоков и иерархию реконфигурируемых межсоединений, которые позволяют создавать блоки по-разному после производства.

В отличие от других микросхем FPGA обеспечивают сочетание программируемости и производительности:

![Сравнение машинного обучения Azure FPGA](./media/concept-accelerate-with-fpgas/azure-machine-learning-fpga-comparison.png)

- **Центральные процессоры** являются процессорами общего назначения. Производительность ЦП не идеальна для обработки графики и видео.
- **Графические процессоры** обеспечивают параллельную обработку и популярны при выборе для вычислений AI. Параллельная обработка с использованием GPU приводит к более быстрому рендерингу образов, чем с CPU.
- **Специальные интегральные схемы**, такие как Google’s TensorFlow Processor Units, представляют собой настраиваемые схемы. Хотя эти микросхемы обеспечивают максимальную эффективность, ASIC негибкие.
- **FPGA**, такие, которые доступны в Azure, обеспечивают производительность, близкую к ASIC, но предлагают гибкость, которую нужно переконфигурировать позже.

FPGA теперь есть на каждом новом сервере Azure. Microsoft уже использует FPGA для ранжирования поиска Bing, оценки глубинных нейронных сетей и ускорения работы в сети. Эти варианты реализации решения FPGA уменьшают задержку, освобождая ЦП для других задач.

## <a name="project-brainwave-on-azure"></a>Project Brainwave в Azure

Project Brainwave – это аппаратная архитектура, разработанная на основе устройств FPGA Intel и используемая для ускорения вычислений AI в реальном времени. Благодаря этой экономичной архитектуре с поддержкой FPGA обученная нейронная сеть может работать как можно быстрее и с меньшей задержкой. Технология Project Brainwave позволяет параллелизовать предварительно обученные глубокие нейронные сети через матрицы FPGA, что обеспечивает возможности масштабирования службы.

- Производительность

    FPGA позволяют добиться низкой задержки для запросов в режиме реального времени. Пакетная обработка означает разбиение запроса на более мелкие части и передачу их на процессор для повышения эффективности использования оборудования. Пакетная обработка не эффективна и может привести к задержке. Brainwave не требует пакетной обработки; поэтому время задержки в 10 раз ниже по сравнению с CPU и GPU.

- Гибкость

    FPGA можно перенастроить для различных типов моделей машинного обучения. Такая гибкость упрощает ускорение приложений на основе наиболее оптимальной точности числовых значений и используемой модели памяти.

    Новые методы машинного обучения разрабатываются на регулярной основе, и также быстро развивается разработка оборудования Project Brainwave. Поскольку FPGA реконфигурируются, можно оставаться в курсе с требованиями быстро меняющихся алгоритмов AI.

- Масштаб

    Microsoft Azure – крупнейшая в мире облачная инвестиция в FPGA. Brainwave можно запускать на инфраструктуре масштаба Azure.

В настоящее время доступен предварительный просмотр Project Brainwave, интегрированный с "Машинным обучением Azure". Кроме того, доступен ограниченный предварительный просмотр, чтобы довести Project Brainwave до границы, чтобы вы могли использовать эту скорость вычислений в своих собственных предприятиях и объектах.

В текущей предварительной версии Brainwave ограничивается развертыванием TensorFlow и нейронными сетями на основе ResNet50 на оборудовании Intel FPGA для классификации и распознавания образов. Планируется поддержка большего количества коллекций и других платформ.

Следующие сценарии используют FPGA для архитектуры Project Brainwave:

- Автоматизированная система оптического контроля. См. [AI в реальном времени: Microsoft объявляет предварительный просмотр Project Brainwave](https://blogs.microsoft.com/ai/build-2018-project-brainwave/).
- Сопоставление земного покрова. См. [Как использовать FPGA для вывода глубокого обучения, чтобы выполнить сопоставление земной поверхности на терабайтах аэрофотоснимков](https://blogs.technet.microsoft.com/machinelearning/2018/05/29/how-to-use-fpgas-for-deep-learning-inference-to-perform-land-cover-mapping-on-terabytes-of-aerial-images/).

## <a name="how-to-deploy-a-web-service-to-an-fpga"></a>Развертывание веб-службы в FPGA.

Поток высокого уровня для создания службы распознавания образов в Azure с помощью ResNet50 как характеризатора выглядит следующим образом:

1. ResNet50 уже развернут в Brainwave. Вы можете создавать другие графы (ввод данных, классификация и т. д.) с помощью TensorFlow и определять конвейер (ввод -> характеризовать -> классифицировать) с помощью json-файла определения службы. Сжатие определений и графов в ZIP-файл и загрузка ZIP-файла в хранилище Azure Blob.
2. Регистрация модели с помощью API Управления моделями Машинного обучения Azure с ZIP-файлом в хранилище BLOB-объектов.
3. Развертывание службы с зарегистрированной моделью с помощью API Управления моделями Машинного обучения Azure.

Дополнительные сведения об этом процессе см. в статье [Развертывание модели как веб-службы на FPGA с помощью "Машинного обучения Azure"](how-to-deploy-fpga-web-service.md).

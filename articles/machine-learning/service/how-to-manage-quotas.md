# <a name="manage-and-request-quotas-for-azure-resources"></a>Управление квотами для ресурсов Azure и их запрашивание

Как и в других службах Azure, существуют ограничения на некоторые ресурсы, связанные со службой "Машинное обучение Azure". Эти ограничения варьируются от лимита на количество рабочих мест, которые можно создать, до лимита на фактический объем базовых вычислений, используемых для обучения или формирования моделей. Эта статья предоставляет дополнительные сведения о предварительно настроенных ограничениях в различных ресурсах Azure для подписки, а также содержит удобные ссылки для запроса увеличения квот для каждого типа ресурсов.

Помните об этих квотах при разработке и увеличении масштаба ресурсов службы "Машинное обучение Azure". Например, если в кластере не удается достичь указанного целевого числа узлов, возможно, вы достигли ограничения ядер Batch AI для подписки. Если планируется выполнять рабочие нагрузки в производственной среде с использованием Batch AI, имеет смысл увеличить одну или несколько квот по сравнению со значениями по умолчанию. Если ограничение или квоту требуется сделать выше значения ограничения по умолчанию, бесплатно отправьте запрос в службу поддержки клиентов. Ограничения не могут быть увеличены выше значения максимального ограничения, как показано в таблицах ниже. Если столбца "Максимальное ограничение" нет, то ресурс не имеет настраиваемого ограничения. 

## <a name="special-considerations"></a>Специальные рекомендации

+ Квота — это кредитный лимит, а не гарантия наличия ресурсов. Если вам нужны ресурсы в очень большом объеме, обратитесь в службу поддержки Azure.

+ Квоты являются общими для всех служб (одной из которых является" Машинное обучение Azure") в подписке, за исключением Batch AI. При оценке потребности в емкости обязательно вычислите использование квоты во всех службах.

+ Ограничения по умолчанию зависят от типа категории, например бесплатной пробной версии, оплаты по мере использования и серии (Dv2, F, G и т. д).

Ниже представлена разбивка квот по различным типам ресурсов в подписке Azure. 

> [!Important]
> Границы могут измениться. Последнюю версию всегда можно найти в [документе](https://docs.microsoft.com/azure/azure-subscription-service-limits/) по квотам на уровне службы для всех ресурсов Azure.

### <a name="virtual-machines"></a>Виртуальные машины 
Существует ограничение на количество виртуальных машин, которые можно подготовить в подписке Azure в своих службах или изолированно. Это ограничение на уровне региона на общее количество ядер, а также на основе каждого семейства.

Важно подчеркнуть, что для ядер виртуальных машин действует региональное общее ограничение и региональное ограничение размера на серию (Dv2, F и т. д.), которое применяется отдельно. Например, рассмотрим подписку с ограничением до 30 ядер виртуальных машин с восточной части США, 30 ядер серии A и 30 ядер серии D. Для этой подписки допускается развертывание 30 виртуальных машин A1 или 30 виртуальных машин D1 или сочетания виртуальных машин обоих типов, при котором должно быть не более 30 ядер (например, 10 виртуальных машин A1 и 20 виртуальных машин D1).

### <a name="batch-ai-clusters"></a>Кластеры Batch AI
В Batch AI есть квота по умолчанию для количества ядер и кластеров, разрешенных для каждого региона в подписке. Квота Batch AI отделена от упомянутой выше квоты на ядра виртуальных машин. В настоящее время ограничения на ядра не распределяются между двумя типами ресурсов.

Доступные ресурсы:
+ Количество выделенных ядер для каждого региона по умолчанию ограничено до 10–24 ядер.  Количество выделенных ядер для каждой подписки Batch AI можно увеличить. Чтобы обсудить варианты увеличения, обратитесь в службу поддержки Azure.

+ Количество ядер с низким приоритетом для каждого региона по умолчанию ограничено до 10–24 ядер.  Количество ядер с низким приоритетом для каждой подписки Batch AI можно увеличить. Чтобы обсудить варианты увеличения, обратитесь в службу поддержки Azure.

+ Максимальное количество кластеров по умолчанию в каждом регионе составляет 20, а максимальное общее ограничение — 200 кластеров. Свяжитесь со службой поддержки Azure, чтобы оставить запрос на повышение лимита.

Более подробный и актуальный список квот см. в [статье о квотах на уровне среды Azure](https://docs.microsoft.com/azure/azure-subscription-service-limits#batch-ai-limits).

Также существует ограничение на количество экземпляров контейнера, которые можно развертывать в определенный период времени (по часам) или по всей вашей подписке.

### <a name="storage"></a>служба хранилища.
Существует ограничение на количество учетных записей хранения в регионе, а также в рамках данной подписки. Ограничение по умолчанию в 200 учетных записей включает учетные записи хранения ценовой категории "Стандартный" и "Премиум". Если требуется более 200 учетных записей хранения в заданном регионе, обратитесь с запросом в [службу поддержки Azure](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest/). Команда разработчиков службы хранилища Azure рассмотрит вашу аргументацию и может утвердить до 250 учетных записей хранения для заданного региона.


## <a name="find-your-quotas"></a>Поиск квот

На портале Azure можно легко просмотреть квоту для различных ресурсов, таких как виртуальные машины, хранилище и сеть.

1. На левой панели выберите **Все службы**, а затем в общей категории выберите **Подписки**.

1. Из списка подписок выберите ту, квоту которой нужно просмотреть.

   **Присутствует ограничение**, в частности при просмотре квоты Batch AI. Как упоминалось выше, эта квота отличается от квоты вычислений в подписке. 
   После выбора **Все службы** выполните поиск по запросу "Batch AI" и откройте службу.

1. В разделе **Параметры** выберите **Использование и квоты**, чтобы просмотреть текущие квоты и использование.

1. Выберите подписку, чтобы просмотреть квоты. Не забудьте выполнить фильтрацию по нужным службе и региону.


## <a name="request-quota-increases"></a>Запрос на увеличение квоты

Если ограничение или квоту требуется сделать выше значения ограничения по умолчанию, [бесплатно отправьте запрос в службу поддержки клиентов](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/newsupportrequest/). 

Ограничения не могут превышать значение максимального ограничения, показанное в таблицах. Если максимальное ограничение отсутствует, то ресурс не имеет настраиваемого ограничения. В [этой статье](https://docs.microsoft.com/azure/azure-resource-manager/resource-manager-quota-errors) более подробно рассматривается процесс увеличения квот.

При запросе увеличения квоты вам необходимо выбрать службу, для которой необходимо повысить квоту. Это могут быть квоты таких служб, как "Машинное обучение Azure", Batch AI или Служба хранилища.

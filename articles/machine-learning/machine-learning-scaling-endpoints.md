# Масштабирование конечных точек API
Конечные точки веб-службы в машинном обучении Azure имеют выбираемые уровни ускорения, которые соответствуют скорости, с которой будет выполнено потребление конечной точки.

Чтобы управлять регулированием конечной точки, задайте максимальное количество одновременных вызовов от 20 до 200, используя ползунок на классическом портале Azure.

Синхронные API обычно используются в ситуациях, когда требуется низкий уровень задержки. Под задержкой здесь подразумевается время, необходимое для того, чтобы API завершил один запрос, и в ней не учитывается время сетевой задержки. Предположим, что имеется API с задержкой в 50 мс. Чтобы полностью задействовать доступный объем с использованием высокого уровня ускорения и максимальным количеством одновременных вызовов = 20, необходимо вызывать этот API 20 * 1000 / 50 = 400 раз в секунду. Далее, при максимальном количестве одновременных вызовов = 200 можно вызывать API 4000 раз в секунду при задержке в 50 мс.

Если планируется вызывать этот API с большей нагрузкой, чем это поддерживается при установке 200 максимальных одновременных вызовов, потребуется создать несколько конечных точек для одной веб-службы и случайным образом распределить нагрузку между ними.

Помните, что использование высоких значений количества одновременных событий может быть неблагоприятно, если API не вызывается с соответствующей этому высокой нагрузкой. При создании достаточно низкой нагрузки на API, настроенный на высокую нагрузку, периодически случайно могут возникать периоды ожидания или резкое повышение значений задержки.

Обратите внимание, что изменение параметров влияет только на поведение синхронного API (RRS). Эти параметры должны регулироваться, только если часто возникает ошибка отклика 503 Service Unavailable в синхронном API.

Интерфейс управления позволяет указать пользовательское значение количества одновременных вызовов для масштабирования конечной точки, превышающее количество по умолчанию 20 вызовов.

1. Откройте узел manage.windowsazure.com
2. Перейдите на вкладку "Машинное обучение"
3. Щелкните рабочее пространство.
4. Перейдите к веб-службе, в которой находится конечная точка ![Перейдите к веб-службе](./media/machine-learning-scaling-endpoints/figure-1.png)
5. Щелкните конечную точку, после чего перейдите на вкладку "Настройка" ![Перейдите к конфигурации конечной точки](./media/machine-learning-scaling-webservice/machlearn-2.png)
6. Измените положение ползунка, чтобы повысить уровень параллелизма и нажмите кнопку «Сохранить».

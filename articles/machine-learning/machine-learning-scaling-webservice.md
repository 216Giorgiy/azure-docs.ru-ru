# Масштабирование веб-службы

## Увеличение параллелизма

По умолчанию каждая опубликованная веб-служба настраивается для поддержки 20 параллельных запросов. Вы можете повысить степень параллелизма до 200 одновременных запросов с помощью [классического портала Azure](https://manage.windowsazure.com/), как показано на рисунке ниже.

Перейдите на [классический портал Azure](https://manage.windowsazure.com/), щелкните значок «Машинное обучение» слева, выберите рабочее пространство, используемое для публикации веб-службы, щелкните нужную веб-службу, выберите конечную точку, для которой требуется расширить параллелизм, а затем нажмите **НАСТРОИТЬ**. С помощью ползунка выберите нужное значение и нажмите кнопку **СОХРАНИТЬ** на нижней панели.

Сведения о расширении параллелизма см. в разделе [Масштабирование конечных точек API](machine-learning-scaling-endpoints.md).

   ![Машинное обучение, масштабирование конечных точек.][1]

## Добавление новых конечных точек для той же веб-службы

Масштабирование веб-службы является распространенной задачей, решить которую нужно либо для обработки более 200 параллельных запросов, либо для повышения доступности благодаря наличию нескольких конечных точек, либо для предоставления отдельной конечной точки для разных потребителей веб-службы. Пользователь может увеличить масштаб, добавив дополнительные конечные точки для той же веб-службы. Пользователь может добавить дополнительные конечные точки с помощью [классического портала Azure](https://manage.windowsazure.com/), как показано на рисунке ниже.

Перейдите на [классический портал Azure](https://manage.windowsazure.com/), щелкните значок «Машинное обучение» слева, выберите рабочее пространство, используемое для публикации веб-службы, щелкните нужную веб-службу, нажмите **ДОБАВИТЬ КОНЕЧНУЮ ТОЧКУ** на нижней панели, а затем укажите имя, описание и желаемое значение параллелизма для новой конечной точки.

Сведения о добавлении новых конечных точек см. в разделе [Создание конечных точек](machine-learning-create-endpoint.md).

# Пошаговые руководства по процессу обработки и анализа данных

Перечисленные здесь пошаговые руководства содержат указания по процессу обработки и анализа данных группы для **конкретных сценариев**. В них также описано, как объединять облачные и локальные инструменты и службы в рабочий процесс или конвейер, чтобы создать интеллектуальное приложение.

## Использование хранилища данных SQL
В пошаговом руководстве [Процесс Cortana Analytics в действии: использование хранилища данных SQL](machine-learning-data-science-process-sqldw-walkthrough.md) показано создание и развертывание модели машинного обучения с использованием хранилища данных SQL для общедоступного набора данных поездок и платежей такси Нью-Йорка.

## Использование сервера SQL
В пошаговом руководстве [Процесс аналитики Кортаны в действии: использование SQL Server](machine-learning-data-science-process-sql-walkthrough.md) показано создание и развертывание модели машинного обучения с использованием сервера SQL и общедоступного набора данных поездок и платежей такси Нью-Йорка.


## Использование кластеров HDInsight Hadoop
В пошаговом руководстве [Процесс аналитики Кортаны в действии: использование кластеров HDInsight Hadoop](machine-learning-data-science-process-hive-walkthrough.md) мы используем [кластер Azure HDInsight Hadoop](https://azure.microsoft.com/services/hdinsight/) для хранения, изучения и определения характеристик данных из общедоступного набора данных поездок и платежей такси Нью-Йорка.


## Использование кластеров Azure HDInsight Hadoop для набора данных объемом 1 ТБ
В пошаговом руководстве [Процесс аналитики Кортаны в действии: использование кластеров Azure HDInsight Hadoop с набором данных объемом 1 ТБ](machine-learning-data-science-process-hive-criteo-walkthrough.md) мы представляем комплексный сценарий, в котором [кластер Azure HDInsight Hadoop](https://azure.microsoft.com/services/hdinsight/) используется для хранения, изучения, реконструирования характеристик и сокращения выборки данных из одного из общедоступных наборов данных [Criteo](http://labs.criteo.com/downloads/download-terabyte-click-logs/).


## Использование Spark для Azure HDInsight
В пошаговом руководстве [Общие сведения об обработке и анализе данных с помощью платформы Spark в Azure HDInsight](machine-learning-data-science-spark-overview.md) показано комплексное использование процесса обработки и анализа данных группы в сценарии, где [кластер Azure HDInsight Spark](https://azure.microsoft.com/services/hdinsight/) используется для хранения, изучения, реконструирования характеристик данных из одного из общедоступного набора данных поездок и платежей такси Нью-Йорка.


## Использование озера данных Azure для хранения и аналитики
В статье [Полное пошаговое руководство по масштабируемому анализу данных в озере данных Azure](machine-learning-data-science-process-data-lake-walkthrough.md) на примере набора данных о поездках и тарифах такси в Нью-Йорке показано, как использовать озеро данных Azure для выполнения задач по исследованию и двоичной классификации данных, чтобы спрогнозировать вероятность получения чаевых за поездку.

## Использование языка R с службами R SQL сервера
Статья [Комплексное пошаговое руководство по обработке и анализу данных с помощью R служб SQL сервера](https://msdn.microsoft.com/library/mt612857.aspx) позволяет специалистам по обработке и анализу данных использовать сочетание кода R, данных SQL сервера и настраиваемых функций SQL для создания и развертывания модели R на SQL сервере.

## Использование T-SQL с службами R SQL сервера
Пошаговое руководство [Внутренняя расширенная аналитика базы данных для SQL разработчиков](https://msdn.microsoft.com/library/mt683480.aspx) предоставляет программистам на SQL практический опыт создания решений расширенной аналитики с помощью Transact-SQL и служб R SQL сервера для эксплуатации решения R.

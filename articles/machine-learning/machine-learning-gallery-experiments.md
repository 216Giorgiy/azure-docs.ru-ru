# <a name="discover-experiments-in-the-cortana-intelligence-gallery"></a>Поиск экспериментов в коллекции Cortana Intelligence
[!INCLUDE [machine-learning-gallery-item-selector](../../includes/machine-learning-gallery-item-selector.md)]

## <a name="experiments-for-machine-learning-studio"></a>Эксперименты для Студии машинного обучения
Коллекция содержит разнообразные **[эксперименты](https://gallery.cortanaintelligence.com/experiments)**, разработанные в [Студии машинного обучения Azure](https://studio.azureml.net). К ним относятся как эксперименты, демонстрирующие конкретный метод машинного обучения, так и готовые решения для сложных задач машинного обучения.

> [!NOTE]
> ***Эксперимент*** — это холст в Студии машинного обучения, на котором можно создать модель прогнозной аналитики, связав друг с другом данные из различных аналитических модулей. Вы можете пробовать различные идеи, выполнять пробные запуски и в конечном итоге опубликовать свою модель в виде веб-службы в Azure. Пример создания простого эксперимента см. в разделе [Руководство по машинному обучению. Создание первого эксперимента в Студии машинного обучения Azure](machine-learning-create-experiment.md). Подробное пошаговое руководство по созданию решения прогнозной аналитики см. в разделе [Пошаговое руководство по разработке решения для прогнозной аналитики в службе Машинное обучение Azure для оценки кредитных рисков](machine-learning-walkthrough-develop-predictive-solution.md).
> 
> 

## <a name="discover"></a>Обнаружение
  Для поиска экспериментов в коллекции перейдите на сайт [коллекции](http://gallery.cortanaintelligence.com) и в верхней части домашней страницы щелкните раздел **Experiments**
 (Эксперименты).

 На странице **[Experiments](https://gallery.cortanaintelligence.com/experiments)**
 (Эксперименты) отобразится список недавно добавленных и самых популярных экспериментов.
Щелкните **See all** (Показать все), чтобы просмотреть все эксперименты.
На открывшейся странице можно просмотреть все эксперименты в коллекции. Здесь также можно пользоваться фильтрами в левой части страницы или строкой поиска вверху, чтобы найти определенные эксперименты.

 Щелкните любой эксперимент, чтобы открыть страницу его сведений и просмотреть данные о предназначении эксперимента. На этой странице в разделе комментариев можно оставлять замечания, отзывы и задавать вопросы. Ссылку на страницу можно опубликовать в социальных сетях LinkedIn и Twitter для друзей и коллег. Кроме того, ссылку можно отправить по электронной почте, чтобы другие пользователи также могли просмотреть найденный вами эксперимент.

![Расскажите друзьям о том, что вы нашли](media\\machine-learning-gallery-how-to-use-contribute-publish\\share-links.png)

![Добавление комментариев](media\\machine-learning-gallery-how-to-use-contribute-publish\\comments.png)

## <a name="download"></a>Загрузить
Вы можете скачивать копии любых экспериментов из коллекции в свою рабочую область Студии, а затем вносить в эту копию изменения, чтобы создавать собственные решения.
Получить копию эксперимента можно двумя способами:

* **Из коллекции**. Если вы нашли нужный эксперимент в коллекции, его копию можно легко скачать, а затем открыть в рабочей области Студии машинного обучения.
* **Из Студии машинного обучения**. В Студии можно использовать любой эксперимент коллекции в качестве шаблона для создания эксперимента.

### <a name="from-the-gallery"></a>Из коллекции
Чтобы скачать копию эксперимента из коллекции, выполните следующее:

1. Откройте страницу сведений эксперимента в коллекции.
2. Щелкните **Открыть в Studio**.
   
    ![Доступ к эксперименту из коллекции](media\\machine-learning-gallery-experiments\\open-experiment-from-gallery.png)

После нажатия кнопки **Open in Studio** (Открыть в Студии) эксперимент загружается в рабочую область Студии машинного обучения и открывается там. (Если вы еще не вошли в Студию, прежде чем эксперимент скопируется в рабочую область, будет предложено войти, используя учетную запись Майкрософт.)

### <a name="from-within-machine-learning-studio"></a>Из Студии машинного обучения Azure
Во время работы в Студии машинного обучения можно также открывать те же примеры экспериментов:

1. В студии машинного обучения щелкните **+New** (Создать).
2. Выберите вкладку **Experiment** (Эксперимент). Эксперимент можно выбрать в списке экспериментов коллекции, предоставленных корпорацией Майкрософт, или можно найти нужный эксперимент, используя поле поиска.
3. Наведите указатель мыши на эксперимент и нажмите кнопку **Открыть в Studio**. Эксперимент скопируется в рабочую область и откроется (для просмотра сведений об эксперименте щелкните **View in Gallery** (Просмотреть в коллекции), чтобы перейти на страницу данных эксперимента в коллекции).
   
    ![Доступ к эксперименту коллекции из Студии машинного обучения](media\\machine-learning-gallery-experiments\\open-experiment-from-studio.png)

Теперь можно настроить, итерировать и развернуть этот эксперимент, как и любой другой эксперимент, созданный в Студии машинного обучения.

![Эксперимент, открытый в Студии](media\\machine-learning-gallery-experiments\\experiment-open-in-studio.png)

## <a name="contribute"></a>Участие
После входа в коллекцию вы становитесь участником ее сообщества. Это позволяет вам делиться своими экспериментами, чтобы они приносили пользу и другим пользователям.

### <a name="publish-your-experiment-to-the-gallery"></a>Публикация эксперимента в коллекции
Для отправки своего эксперимента в коллекцию Cortana Intelligence выполните следующие действия.

1. Войдите в Студию машинного обучения, используя учетную запись Майкрософт.
2. Создайте свой эксперимент и запустите его.
3. Когда вы будете готовы опубликовать свой эксперимент в коллекции, нажмите кнопку **Publish to Gallery** (Опубликовать в коллекции) под холстом эксперимента.
   
    ![Нажатие кнопки Publish to Gallery (Опубликовать в коллекции)](media\\machine-learning-gallery-experiments\\publish-experiment-to-gallery.png)
4. Заполните поля заголовка и тега. Они должны быть описательными и содержать методы или реальную проблему, которую решает ваш эксперимент, например "Двоичная классификация: анализ мнений с помощью Twitter".
   
    ![Заполните поля заголовка и тега во время публикации](media\\machine-learning-gallery-experiments\\experiment-description.png)
5. Напишите сводку по содержимому вашей документации. Дайте краткое описание проблемы и подхода для ее решения.
6. Используйте поле подробного описания, чтобы описать этапы вашего эксперимента. Сюда следует включить следующее:
   
   * снимок экрана с диаграммой эксперимента;
   * источники данных и соответствующее объяснение;
   * Обработка данных
   * Проектирование признаков
   * описание модели;
   * результаты и оценку эффективности модели.

Чтобы просмотреть список своих публикаций, выберите изображение в правом верхнем углу на странице коллекции и щелкните свое имя, чтобы открыть страницу учетной записи.

### <a name="update-your-experiment"></a>Обновление эксперимента
Если необходимо внести изменения в рабочий процесс (модули, параметры и т. п.) в эксперименте, опубликованном в коллекции, перейдите обратно к эксперименту в Студии машинного обучения, внесите изменения и заново опубликуйте его. Существующий опубликованный эксперимент будет обновлен с учетом изменений.

Если необходимо изменить следующие данные своего эксперимента или удалить эксперимент из коллекции, все эти изменения можно внести в коллекции:

* имя эксперимента;
* сводка или описание текста;
* указанные теги;
* используемое изображение;
* параметр видимости (общедоступность или вне списка);
* удалить эксперимент из коллекции.

Эти изменения можно внести в коллекцию на странице профиля или сведений эксперимента.

#### <a name="from-your-experiments-details-page"></a>На странице сведений об эксперименте
На этой странице нажмите кнопку Edit (Изменить), чтобы изменить сведения эксперимента.

![Нажатие кнопки Edit (Изменить) для изменения эксперимента](media\\machine-learning-gallery-experiments\\edit-button.png)

Страница сведений перейдет в режим редактирования. Щелкните Edit (Изменить) рядом с именем эксперимента, сводкой, тегом и т. п., чтобы изменить их. Закончив вносить изменения, нажмите кнопку Done (Готово).

![Щелчок Edit (Изменить), чтобы изменить сведения, и нажатие кнопки Done (Готово) по завершении](media\\machine-learning-gallery-experiments\\edit-details-page.png)

Вы также можете щелкнуть значок настроек, чтобы изменить отображение эксперимента (общедоступное или вне списка) или удалить эксперимент из коллекции.

![Щелчок значка настроек, чтобы изменить видимость или удалить эксперимент](media\\machine-learning-gallery-experiments\\settings-button.png)

#### <a name="from-your-profile-page"></a>На странице профиля
На этой странице можно щелкнуть стрелку вниз для эксперимента и выбрать пункт Edit (Изменить). Откроется страница сведений эксперимента в режиме редактирования. Закончив вносить изменения, нажмите кнопку Done (Готово).

Можно также выбрать пункт Delete (Удалить), чтобы удалить эксперимент из коллекции.

![Щелчок пункта Edit (Изменить) или Delete (Удалить)](media\\machine-learning-gallery-experiments\\edit-delete-buttons.png)

### <a name="suggestions-for-publishing-and-for-quality-documentation"></a>Рекомендации по публикации и требования к качеству документации
* Хотя можно предположить, что у читателя есть опыт обработки и анализа данных, рекомендуется использовать простую речь и по возможности объяснить все как можно подробнее.
* Не все читатели будут знакомы с Cortana Intelligence Suite, ведь это относительно новый пакет.
  Поэтому укажите достаточно сведений и приведите поэтапное объяснение, чтобы облегчить понимание действий для читателей.
* Визуальные элементы, в том числе диаграммы или снимки экрана с данными, помогут читателям лучше понять содержимое публикации и использовать его должным образом. Дополнительные сведения о том, как добавлять изображения в документацию см. в разделе [Publishing Guidelines and Examples](https://gallery.cortanaintelligence.com/Collection/Publishing-Guidelines-and-Examples-1) (Публикация руководств и примеров).
* Если в эксперимент включен набор данных (не импортированный с помощью модуля импорта), он будет опубликован в коллекции вместе с экспериментом. Поэтому убедитесь, что у набора данных, который публикуется, есть соответствующие условия лицензирования, предусматривающие общий доступ и скачивание для любого пользователя. Публикация в коллекции осуществляется в соответствии с [условиями использования](https://azure.microsoft.com/support/legal/website-terms-of-use/)Azure.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы
**Каковы требования к изображению при отправке или редактировании изображения для моего эксперимента?**

Изображения, которые можно отправить вместе с вашим экспериментом будет использоваться для создания плитки публикуемого эксперимента. Рекомендуется использовать изображение размером до 500 Кбайт с соотношением сторон 3:2. Рекомендуется использовать разрешение 960 x 640

**Что происходит с набором данных, использованным в эксперименте? Набор данных также публикуется в коллекции?**

Если набор данных является частью вашего эксперимента и не импортирован с помощью модуля импорта данных, он будет опубликован в коллекции вместе с вашим экспериментом. Поэтому убедитесь, что у набора данных, который публикуется с экспериментом, есть соответствующие условия лицензирования, которые допускают общий доступ и скачивание для любого пользователя.

**У меня есть эксперимент, использующий модуль "Импорт данных" для извлечения данных из HDInsight или SQL. Для получения данных он использует учетные данные. Как опубликовать такой эксперимента и быть уверенным, что мои учетные данные не будут при этом переданы другим пользователям?**

В настоящее время мы не позволяем публиковать эксперименты, использующие учетные данные.

**Как указать несколько тегов?**

Если тег введен, нажмите клавишу TAB, чтобы ввести другой тег.

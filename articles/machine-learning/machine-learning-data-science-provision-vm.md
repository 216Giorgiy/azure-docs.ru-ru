# Подготовка виртуальной машины для обработки и анализа данных

## Введение

Виртуальная машина для обработки и анализа данных — это образ виртуальной машины (VM) Azure, в котором предварительно установлен и настроен ряд распространенных средств, которые обычно используются для анализа данных и машинного обучения. К этим средствам относятся:

- Revolution R Open;
- дистрибутив Anaconda Python;
- выпуск Visual Studio Community;
- Power BI Desktop;
- выпуск SQL Server Express.


Обработка и анализ данных включают в себя повторяющееся выполнение последовательности задач: поиска, загрузки и предварительной обработки данных, построения и тестирования моделей, а также развертывания моделей для использования в интеллектуальных приложениях. Для выполнения этих задач специалисты по обработке и анализу данных часто используют широкий набор средств. Поиск правильной версии программного обеспечения и его последующая загрузка и установка могут занимать довольно много времени. Виртуальная машина для обработки и анализа данных может существенно упростить выполнение этих задач.

Используйте виртуальную машину для обработки и анализа данных, чтобы упростить выполнение этих задач. Начните работу над проектом по анализу данных, создав образ, который содержит программное обеспечение, обычно используемое для задач по анализу и машинному обучению на различных языках, включая R, Python, SQL и C#. Visual Studio использует удобный интерфейс IDE для разработки и тестирования кода. Пакет SDK Azure в виртуальной машине позволяет создавать приложения с помощью различных служб на облачной платформе Майкрософт.

## Предварительные требования

Для создания виртуальной машины Azure необходимы следующие компоненты:

- **Подписку на Azure**: См. [Получение бесплатной пробной версии Azure](http://azure.microsoft.com/documentation/videos/get-azure-free-trial-for-testing-hadoop-in-hdinsight/).

*   **Учетная запись хранения Azure**. См. раздел [Создание учетной записи хранения Azure](storage-whatis-account.md). Если вы не хотите использовать существующую учетную запись хранения, ее можно создать при создании виртуальной машины.


## Создание виртуальной машины для обработки и анализа данных

Ниже перечислены действия для создания экземпляра виртуальной машины для обработки и анализа данных.

1.	Войдите на портал Azure.
2.	Последовательно выберите элементы **Создать** -> **Среда выполнения приложений** -> **Marketplace** и выполните поиск по фразе *Виртуальная машина для обработки и анализа данных*. Выберите виртуальную машину под названием «**Виртуальная машина для обработки и анализа данных**», опубликованную корпорацией Майкрософт. Отобразится панель с описанием виртуальной машины для обработки и анализа данных.
3.	В нижней части страницы нажмите кнопку **Создать**. Откроется мастер. ![provision-data-science-vm](./media/machine-learning-data-science-provision-vm/provision-data-science-virtual-machine.png)
4.	 В следующих разделах приведены данные для каждого шага мастера по созданию виртуальной машины для обработки и анализа данных.


 **1. Основные сведения**.

- **Имя**. Имя создаваемого сервера для обработки и анализа данных.
- **Имя пользователя**. Имя пользователя учетной записи администратора.
- **Пароль**. Пароль учетной записи администратора.
- **Подписка**. При наличии нескольких подписок выберите ту, в которой будет создана виртуальная машина и с которой будет взиматься плата.
- **Группа ресурсов**. Вы можете создать новую группу или использовать существующую.
- **Расположение**. Выберите наиболее подходящий центр обработки данных. Обычно это центр обработки данных, в котором размещена большая часть ваших данных, или ближайший к вашему физическому расположению центр для наиболее быстрого доступа к сети.

 **2. Размер**.

- Выберите один из типов серверов, который соответствует вашим функциональным требованиям и финансовым ограничениям. Дополнительные варианты размера виртуальной машины можно увидеть, щелкнув элемент «Просмотреть все».

 **3. Параметры**.

 **4. Тип диска**.

- Выберите «Премиум», если вы предпочитаете накопители SSD. В противном случае выберите «Стандартный».

 **5. Учетная запись хранения**.


- Вы можете создать учетную запись хранения Azure в подписке или использовать существующую учетную запись в расположении, которое было выбрано на этапе работы мастера «Основные сведения».
- В большинстве случаев для остальных параметров используются значения по умолчанию. Если нужно использовать значение, отличное от значения по умолчанию, вы можете навести указатель мыши на информационную ссылку, чтобы получить справку по конкретному полю.
- Проверьте, все ли сведения введены правильно.
- Чтобы начать подготовку, щелкните **Купить**. Отобразится ссылка на условия транзакции. За использование виртуальной машины не взимается дополнительная плата, кроме платы за размер сервера, выбранный на шаге **Размер**. 

Подготовка занимает 10–20 минут. Состояние подготовки отображается на портале Azure.

## Доступ к виртуальной машине для обработки и анализа данных

После создания виртуальной машины вы можете выполнить вход в нее с помощью удаленного рабочего стола, используя учетную запись администратора, созданную в разделе «Основные сведения» этапа 4.

После создания и подготовки виртуальной машины вы можете начать использовать установленные и настроенные на ней средства. Для многих средств есть значки на рабочем столе и плитки в меню «Пуск».

## Средства, установленные на виртуальной машине для обработки и анализа данных

### R
Если для анализа данных необходимо использовать средство R, на виртуальной машине установлено средство Revolution R Open (RRO). Это дистрибутив R с открытым кодом, полностью совместимый с CRAN-R. Он содержит последнюю версию ядра R с открытым кодом, а также библиотеку Intel Math Kernel Library. Виртуальная машина содержит также интерфейс IDE, называемый RRO RGui. Кроме того, вы можете загрузить и использовать другие интерфейсы IDE, например [RStudio](http://www.rstudio.com).

### Python
Для разработки, при которой используется язык Python, установлен дистрибутив Anaconda Python 2.7. Этот дистрибутив содержит базовую версию Python, а также примерно 300 наиболее популярных математических и инженерных пакетов и пакетов для обработки и анализа данных. Вы можете использовать интерфейсы IDE, входящие в один пакет с Anaconda, например IDLE или Spyder. Их можно запустить, выполнив поиск на панели поиска (сочетание клавиш **WINDOWS**+**S**).

### IPython Notebook
Дистрибутив Anaconda содержит также средство IPython Notebook, которое является средой для совместного использования кода и выполнения анализа. Сервер Ipython Notebook предварительно настроен. На рабочем столе есть значок, с помощью которого можно запустить браузер для доступа к серверу Notebook. Если для доступа к виртуальной машине вы используете удаленный рабочий стол, можно перейти по ссылке [https://localhost:9999 /](https://localhost:9999/), чтобы получить доступ к серверу IPython Notebook. (Примечание. При появлении предупреждения о сертификате просто щелкните «Продолжить».)

### Выпуск Visual Studio Community 2015
На виртуальной машине установлен выпуск Visual Studio Community. Это бесплатная версия популярного интерфейса IDE от Майкрософт, которую можно использовать для ознакомления, а также для небольших групп. Условия использования вы можете узнать ***здесь*** (ссылка будет добавлена позже). Откройте Visual Studio, дважды щелкнув значок на рабочем столе или выбрав соответствующий пункт в меню **Пуск**. Кроме того, вы можете выполнить поиск программ, нажав клавиши **WINDOWS**+**S** и введя запрос «Visual Studio».

Примечание. Может появиться сообщение о том, что период ознакомления истек. Чтобы получить доступ к нужному выпуску Visual Studio Community, вы можете ввести учетные данные существующей учетной записи Майкрософт или сначала создать учетную запись, а затем ввести учетные данные. Выполнив вход, вы можете создавать проекты, используя такие языки, как C# и Python.

### SQL Server Express
В выпуск Visual Studio Community входит также ограниченная версия SQL Server. Для доступа к серверу SQL Server запустите **SQL Server Management Studio**. Имя виртуальной машины будет использовано в качестве имени сервера. При входе от имени администратора в ОС Windows используйте проверку подлинности Windows. Выполнив вход в SQL Server Management Studio, вы можете создавать других пользователей и базы данных, импортировать данные и выполнять запросы SQL.

### Таблицы Azure 
На виртуальной машине установлено несколько средств Azure. На рабочем столе есть ярлык для доступа к документации по пакету SDK Azure. - **AzCopy** используется для перемещения данных из учетной записи хранения Microsoft Azure и обратно. - **Обозреватель хранилищ Azure** позволяет просматривать объекты, которые хранятся в учетной записи хранения Azure. - **Microsoft Azure Powershell** — язык сценариев, используемый для администрирования ресурсов Azure на языке сценариев. Он также установлен на виртуальной машине.

###Power BI

Для создания панелей мониторинга и широких возможностей визуализации установлено средство **Power BI Desktop**. Используйте это средство для извлечения данных из различных источников, создания панелей мониторинга и отчетов и их публикации в облаке. Дополнительные сведения см. на сайте [Power BI](http://powerbi.microsoft.com).

Примечание. Для доступа к Power BI необходима учетная запись Office 365.

# <a name="supported-data-exports-for-this-preview"></a>Поддерживаемые форматы экспорта данных в предварительной версии 
Данные можно экспортировать в нескольких разных форматах. Эти форматы позволяют сохранять промежуточные результаты подготовки данных перед интеграцией в остальную часть рабочего процесса службы машинного обучения.

## <a name="types"></a>Типы 
### <a name="csv-file"></a>CSV-файл 
Запись файла со значениями с разделителями-запятыми в хранилище.

#### <a name="options"></a>Параметры
- Line Endings (Завершение строк)
- Replace Nulls With (Заменить значения Null)
- Replace Errors with (Заменить ошибочные значения) 
- Разделитель


### <a name="parquet"></a>PARQUET ###
Запись набора данных в формате PARQUET в хранилище.

Формат PARQUET может принимать разные формы в хранилище. Наборы данных небольшого размера иногда хранятся в отдельном файле PARQUET. Разные библиотеки Python поддерживают операции чтения и записи в отдельные файлы этого формата. Сейчас AMLWB выполняет запись файла Parquet во время локального интерактивного использования с помощью библиотеки Python PyArrow. Это означает, что сейчас файл PARQUET —это единственный поддерживаемый формат выходных данных PARQUET во время локального интерактивного использования.

При масштабировании (в Spark) AMLWB полагается на возможности чтения и записи PARQUET Spark. Структура стандартного формата выходных данных PARQUET (единственный поддерживаемый формат) в Spark во многом напоминает набор данных HIVE. Это означает, что папка содержит большое количество файлов PARQUET, каждый из которых входит в состав более мелкой секции набора данных большего размера. 

#### <a name="caveats"></a>Предупреждения ####
PARQUET как формат появился относительно недавно. Он имеет некоторые несоответствия реализации в различных библиотеках. Например, в Spark применяются ограничения на символы, допустимые в именах столбцов при записи в формат PARQUET. В PyArrow эти ограничения не применяются. Имя столбца не может содержать следующие символы: ";", "{", "}", "(", ")", "\\n", "\\t", "=", ",".

>[!NOTE]
>Чтобы обеспечить совместимость со Spark, когда данные записываются в формат PARQUET, каждое вхождение этих символов заменяется на подчеркивание ("_"). **

>[!NOTE]
>Чтобы обеспечить согласованность со Spark при локальном использовании и масштабировании всех данных, записанных в формат PARQUET через приложение, Python и Spark имеют собственные соответствующие имена столбцов. Перед записью в формат PARQUET символы, на которые применяются ограничения в Sparks, необходимо удалить. Это позволит получить ожидаемые имена столбцов.

# <a name="supported-data-sources-for-this-release"></a>Поддерживаемые источники данных в этом выпуске 
В этой статье приведен список источников данных, поддерживаемых средством подготовки данных службы "Машинное обучение Azure".

Поддерживаемые в этом выпуске источники данных приведены ниже.

## <a name="types"></a>Типы 
### <a name="directory-versus-file"></a>Каталог или файл
*Файлы и каталоги.* Выберите отдельный файл и передайте его в средство подготовки данных, которое проанализирует его тип и определит параметры подключения файла по умолчанию, отображающиеся на следующей странице. Выберите каталог или набор файлов в каталоге (средство выбора файлов поддерживает множественный выбор). При этом файлы считываются в качестве отдельного потока данных и добавляются друг к другу (при необходимости с отделенными заголовками).

Ниже приведены типы файлов:
- с разделителями (.csv, .tsv, .txt и т. д.); 
- с полями фиксированного размера;
- с обычным текстом;
- Файл JSON

### <a name="csv-file"></a>CSV-файл
Чтение CSV-файла из хранилища.

#### <a name="options"></a>Параметры
- Разделитель
- Комментарий
- Заголовки
- Десятичный символ.
- Кодировка файла.
- Пропускаемые строки.

### <a name="tsv-file"></a>TSV-файл
Чтение файла значений TSV из хранилища.

#### <a name="options"></a>Параметры
- Комментарий
- Заголовки
- Кодировка файла.
- Пропускаемые строки.

### <a name="excel-xlsxlsx"></a>Excel (XLS и XLSX)
Чтение файла Excel (по одному листу) путем указания имени или номера листа.

#### <a name="options"></a>Параметры
- Номер и имя листа.
- Заголовки
- Пропускаемые строки.

### <a name="json-file"></a>Файл JSON
Чтение файла JSON из хранилища. Обратите внимание, что файл преобразовывается в плоскую структуру при чтении.

#### <a name="options"></a>Параметры
None

### <a name="parquet"></a>PARQUET
Чтение набора данных Parquet (одного файла или папки).

Формат PARQUET может принимать разные формы в хранилище. Иногда для наборов данных меньшего размера используется один PARQUET-файл. Различные библиотеки Python поддерживают чтение или запись в одном PARQUET-файле. Сейчас Azure Machine Learning Workbench выполняет чтение файла Parquet во время локального интерактивного использования с помощью библиотеки Python PyArrow. В этой библиотеке поддерживаются отдельные PARQUET-файлы (при условии, что они написаны в таком виде и не являются элементами большего набора данных) и наборы данных PARQUET. 

Набор данных PARQUET — это коллекция файлов PARQUET, каждый из которых представляет более мелкую секцию набора данных большего размера. Наборы данных обычно находятся в папке. Для распространенных платформ, таких как Spark и Hive, они имеют выходной формат по умолчанию PARQUET.

>[!NOTE]
>При чтении данных PARQUET, находящихся в папке с несколькими файлами PARQUET, безопаснее всего выбрать каталог для чтения и установить флажок **Parquet Dataset** (Набор данных PARQUET). Поэтому PyArrow считывает всю папку, а не отдельные файлы, обеспечивая тем самым поддержку чтения более сложных способов хранения PARQUET на диске (например, секционирование папки).

Выполнение масштабирования зависит от возможностей чтения PARQUET Spark и поддержки отдельных файлов и папок.

#### <a name="options"></a>Параметры
*Набор данных PARQUET.* Этот параметр определяет, использует ли Azure Machine Learning Workbench режим при снятом или установленном флажке. В режиме при снятом флажке заданный каталог раскрывается и выполняется попытка чтения каждого файла по отдельности. В режиме с активированным флажком каталог рассматривается как полный набор данных, а PyArrow определяет самый оптимальный способ интерпретации файлов.

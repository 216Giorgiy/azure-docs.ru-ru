# <a name="supported-data-sources-for-this-release"></a>Поддерживаемые источники данных в этом выпуске 
В этой статье приведены источники данных, поддерживаемые средством подготовки данных.

Ниже приведен список источников данных, поддерживаемых в этом выпуске.

## <a name="types"></a>Типы 
### <a name="directory-vs-file"></a>Каталог или файл
Файлы или каталоги. Выберите отдельный файл и передайте его в средство подготовки данных. Средство анализирует его тип и определяет параметры подключения файла по умолчанию, которые отображаются на следующей странице. Выберите каталог или набор файлов в каталоге (средство выбора файлов поддерживает множественный выбор), используя результаты в проанализированных файлах как отдельный поток данных и файлы, добавленные друг к другу (при необходимости с отделенными заголовками).

Поддерживаемые типы файлов:
- файл с разделителями (CSV, TSV, TXT и т. д.); 
- файл с полями фиксированного размера;
- файл с обычным текстом;
- JSON-файл.

### <a name="csv-file"></a>CSV-файл
Чтение файла со значениями с разделителями-запятыми из хранилища.

#### <a name="options"></a>Параметры
- Разделитель
- Комментарий
- Заголовки
- Decimal Symbol (Десятичный символ)
- File Encoding (Кодировка файла)
- Lines To Skip (Пропускаемые строки)

### <a name="tsv-file"></a>Файл TSV
Чтение файла со значениями с разделением знаками табуляции из хранилища.

#### <a name="options"></a>Параметры
- Комментарий
- Заголовки
- File Encoding (Кодировка файла)
- Lines To Skip (Пропускаемые строки)

### <a name="excel-xlsxlsx"></a>Excel (XLS или XLSX)
Чтение файла Excel (по одному листу) с помощью указания имени номера листа.

#### <a name="options"></a>Параметры
- Sheet Name/Number (Номер и имя листа)
- Заголовки
- Lines To Skip (Пропускаемые строки)

### <a name="json-file"></a>JSON-файл
Чтение JSON-файла из хранилища (при чтении файл сжимается).

#### <a name="options"></a>Параметры
- None

### <a name="parquet"></a>PARQUET
Чтение набора PARQUET (одного файла или папки).

Формат PARQUET может принимать разные формы в хранилище. Наборы данных небольшого размера иногда хранятся в отдельном файле PARQUET. Разные библиотеки Python поддерживают операции чтения и записи в отдельные файлы этого формата. Сейчас AMLWB выполняет чтение файла Parquet во время локального интерактивного использования с помощью библиотеки Python PyArrow. Эта библиотека поддерживает отдельные файлы (если они написаны в таком виде, а не входят в состав большего набора данных) и наборы данных PARQUET. Набор данных PARQUET — это коллекция файлов PARQUET, каждый из которых представляет более мелкую секцию набора данных большего размера. Наборы данных обычно содержатся в папке. Это стандартный формат выходных данных PARQUET общих платформ, таких как Spark и Hive.

>[!NOTE]
>При чтении данных PARQUET, находящихся в папке с несколькими файлами PARQUET, безопаснее всего выбрать каталог для чтения и установить флажок Parquet Dataset (Набор данных PARQUET). Это позволит PyArrow считывать всю папку, а не отдельные файлы, обеспечивая тем самым поддержку чтения более сложных способов хранения PARQUET на диске (например, секционирование папки)**.

Масштабирование зависит от возможностей чтения Parquet Spark. Этот процесс поддерживает отдельные файлы и папки (аналогично локальному интерактивному использованию).

#### <a name="options"></a>Параметры
- Parquet Dataset (Набор данных PARQUET)
  - Этот параметр определяет, будет ли AMLWB развертывать определенный каталог и пытаться считать каждый файл в нем (режим при снятом флажке) или будет рассматривать каталог как целый набор данных и позволит PyArrow спланировать оптимальный способ взаимодействия с файлами (режим при установленном флажке).

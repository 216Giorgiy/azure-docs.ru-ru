# <a name="supported-matrix-for-this-release"></a>Поддерживаемая матрица для этого выпуска 
Когда код загружает данные с помощью источников данных или подготовки данных, при получении кадра данных Spark или Pandas поддерживаются следующие сочетания экспериментальных вычислительных сред и расположений данных:

|     |Локальные файлы  |Большой двоичный объект Azure  |База данных SQL Server***  |
|---------|---------|---------|---------|---------|
|локальные приложения Python;    |     Поддерживаются    |Не поддерживается         | Не поддерживается        |         |
|Приложение Docker Python (для виртуальной машины Linux)     |Поддерживается только в файлах проекта*         | Не поддерживается        |        Не поддерживается |         |
|Приложение Docker PySpark (для виртуальной машины Linux)     |Поддерживается только в файлах проекта*     |Поддерживаются         | Поддерживается**        |         |
|Python для виртуальной машины Azure для обработки и анализа данных     |Поддерживается только в файлах проекта*         |Не поддерживается         |Не поддерживается         |         |
|PySPark для виртуальной машины Azure для обработки и анализа данных     | Поддерживается только в файлах проекта*        |Не поддерживается         |Не поддерживается         |         |
|PySpark в Azure HDInsight     | Не поддерживается        |Поддерживаются         |Поддерживается**         |         |
|Python в Azure HDInsight     | Не поддерживается        | Не поддерживается        | Не поддерживается        |         |

Azure Data Lake Store в настоящее время не поддерживается ни для одного вычислительного объекта.

* При использовании локальных путей файлов файлы в проекте копируются в вычислительную среду, где их можно считать. За пределами проекта файлы не копируются, а пути больше не могут использоваться в вычислительной среде. Вы можете использовать подстановку источника данных, чтобы код мог использовать локальный файл при локальном запуске, а затем выполнить переключение в Azure Storage Blob для использования другой конфигурации запуска. Также можно использовать поддержку выборки в источниках данных для управления запуском при больших объемах данных только в определенных конфигурациях запуска.

** Использование драйвера SQL Server Maven JDBC версии 6.2.1. Этот пакет (или другой совместимый) должен быть включен в файл spark_dependencies.yml вычислительной среды.

*** Поддержка базы данных SQL Azure, хранилища данных SQL Azure или Microsoft SQL Server, если к базе данных можно получить доступ из вычислительной среды.

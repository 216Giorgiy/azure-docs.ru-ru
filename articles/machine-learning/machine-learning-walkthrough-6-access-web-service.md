6.	**Доступ к веб-службе**

----------

Чтобы веб-служба была полезной, пользователи должны иметь возможность отправлять данные в нее и получать результаты. Данная веб-служба представляет собой веб-службу Azure, которая может получать и возвращать данные одним из двух способов:

-	**Ответ на запрос** — пользователь отправляет единичный набор данных о кредитах в службу с помощью протокола HTTP, а служба в качестве ответа возвращает одиночный набор результатов.
-	**Пакетное выполнение** — пользователь отправляет в службу URL-адрес большого двоичного объекта Azure, который содержит одну или несколько строк данных о кредитах. Служба сохраняет результаты в другом большом двоичном объекте и возвращает URL-адрес данного контейнера.  

На вкладке **ПАНЕЛЬ МОНИТОРИНГА** для веб-службы существуют две ссылки на информацию, которая поможет разработчику разрабатывать код для доступа к этой службе. Щелкните ссылку на **страницу справки API** в строке **ЗАПРОС-ОТВЕТ**. Появится страница с примером кода для использования протокола ответа на запрос службы. Аналогичным образом, если щелкнуть ссылку в строке **ПАКЕТНОЕ ВЫПОЛНЕНИЕ**, появляется страница с примером кода для отправки пакетного запроса в службу.

Страница справки API содержит примеры для языков программирования R, C# и Python.

Дополнительные сведения о доступе к веб-службе и ее использовании см. в статье [Настройка веб-службы машинного обучения Azure, опубликованной из эксперимента машинного обучения](machine-learning-consume-web-services.md).

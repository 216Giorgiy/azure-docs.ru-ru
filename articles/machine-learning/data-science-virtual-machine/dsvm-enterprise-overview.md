# <a name="data-science-virtual-machine-based-team-analytics-and-ai-environment"></a>Командная среда аналитики и искусственного интеллекта на базе виртуальной машины для обработки и анализа данных 
[Виртуальная машина для обработки и анализа данных](overview.md) (DSVM) предоставляет среду с широкими возможностями в облаке Azure с предварительно созданным программным обеспечением для задач искусственного интеллекта и аналитики. Традиционно DSVM использовалась в качестве индивидуального компьютера для аналитики, а специалисты по обработке и анализу данных повышали производительность за счет предварительно созданной аналитической среды. Когда большие аналитические команды планируют среды аналитики для специалистов по обработке и анализу данных и разработчиков искусственного интеллекта, одной из повторяющихся тем является общая аналитическая инфраструктура разработки и экспериментирования, которая управляется в соответствии с корпоративными политиками ИТ-отдела, что также способствует совместной работе и согласованности команд по обработке и анализу данных и команд аналитики. Общая инфраструктура также позволяет ИТ-отделу лучше использовать среду аналитики. Некоторые организации также называют командную инфраструктуру аналитики и обработки и анализа данных "Аналитическая песочница". Она позволяет специалистам по обработке и анализу данных быстро понять смысл данных, выполнять эксперименты, проверять гипотезы, строить прогнозирующие модели безопасным образом без влияния на рабочую среду и при наличии доступа к различным ресурсным данным. 

Так как виртуальная машина для обработки и анализа данных функционирует на уровне инфраструктуры Azure, ИТ-администраторы могут легко настроить ее для работы в соответствии с ИТ-политиками предприятия. Кроме того, она предоставляет полную гибкость в реализации различных общих архитектур с доступом к корпоративным данным контролируемым образом. 

В этом разделе рассматриваются некоторые шаблоны и рекомендации, которые могут использоваться для развертывания DSVM как командной инфраструктуры обработки и анализа данных.  Стандартные блоки для этих шаблонов непосредственно используются в Azure IaaS (инфраструктура как услуга) и таким образом применимы к любым виртуальным машинам Azure. В этой серии статей особое внимание уделяется применению этих стандартных возможностей инфраструктуры Azure к виртуальной машине для обработки и анализа данных.

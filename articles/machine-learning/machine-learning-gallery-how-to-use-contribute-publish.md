# Поиск и совместное использование решений в Коллекции аналитических данных Cortana

Коллекция аналитических данных Cortana — это управляемый сообществом сайт для поиска и совместного использования решений, построенных с использованием набора продуктов Cortana Analytics Suite. Это место для тех, кто хочет научиться чему-то у других, начать разрабатывать собственные решения или делиться плодами своего труда с большим аналитическим сообществом.

**[ПЕРЕЙТИ К КОЛЛЕКЦИИ >>](http://gallery.cortanaanalytics.com)**

[AZURE.INCLUDE [machine-learning-free-trial](../../includes/machine-learning-free-trial.md)]

## Преимущества
Присоединяйтесь к сообществу Коллекции, чтобы воспользоваться ее преимуществами:

- поиском интересных решений;
- опытом других пользователей.

Выполните вход с помощью учетной записи Майкрософт, чтобы получить следующие дополнительные преимущества:

- Загружайте эксперименты в рабочее пространство Студии машинного обучения
- Делитесь своими экспериментами и коллекциями прогнозной аналитики и демонстрируйте их

## Находите и изучайте
Находите и изучайте различные типы решений, которые были добавлены корпорацией Майкрософт и большим аналитическим сообществом. Используйте их как отправную точку в решении собственных проблем анализа данных.

В Коллекции легко найти недавно опубликованные и популярные решения. Также возможен поиск по именам, тегам, алгоритмам и другим атрибутам. Для просмотра вклада конкретного автора щелкните имя автора в любом из элементов.

В разделе «Комментарии» для каждого решения можно оставить комментарии, отзывы или задать вопросы. Можно даже поделиться интересным решением с друзьями или коллегами в LinkedIn или Twitter. Ссылку на решение легко отправить по электронной почте, чтобы другие пользователи ознакомились с ним.

## Загрузка экспериментов
В Коллекции аналитических данных Cortana легко скопировать эксперименты прогнозной аналитики в рабочее пространство Студии машинного обучения и сразу же приступить к их использованию. Если вы не вошли в систему во время просмотра, то перед копированием эксперимента в рабочую область вам будет предложено войти в систему с использованием учетной записи Майкрософт.

## Участие
При входе в систему вы становитесь членом сообщества Коллекции и можете делиться собственными экспериментами и коллекциями с другими.

### Делитесь своими экспериментами

*Эксперимент* — это холст в Студии машинного обучения, на котором можно создать модель прогнозной аналитики, связав друг с другом данные от различных аналитических модулей. Вы можете пробовать различные идеи, выполнять пробные запуски и в конечном итоге опубликовать свою модель в виде веб-службы в Azure. Пример создания простого эксперимента см. в разделе [Руководство по машинному обучению. Создание первого эксперимента в Студии машинного обучения Azure](machine-learning-create-experiment.md). Подробное пошаговое руководство по созданию решения прогнозной аналитики см. в разделе [Руководство: разработка решения прогнозной аналитики для оценки кредитных рисков в Студии машинного обучения Azure](machine-learning-walkthrough-develop-predictive-solution.md).

Для отправки своего эксперимента в Коллекцию аналитических данных Cortana выполните следующие действия:

- Войдите в Студию машинного обучения с использованием учетной записи Майкрософт
- Создайте свой эксперимент и запустите его
- Когда вы будете готовы опубликовать свой эксперимент в Коллекции, щелкните **Опубликовать в Коллекции** под холстом эксперимента

### Делитесь своими коллекциями

*Коллекция* — это список связанных элементов из Коллекции аналитических данных Cortana, которые объединены вместе для последующего обращения, использования или совместного использования. В состав вашей коллекции могут входить ваши собственные элементы или элементы, созданные сообществом.

Например, в коллекции можно объединить элементы по определенной теме или элементы многоэтапного эксперимента, который решает сложные задачи. Примеры коллекций, добавленные корпорацией Майкрософт, представляют собой шаблоны многоэтапных экспериментов машинного обучения для решения реальных проблем, таких как обнаружение онлайн-мошенничества, классификация текста, прогнозирование в розничной торговле и прогнозирование обслуживания.

Для создания и отправки своей коллекции в Коллекцию аналитических данных Cortana выполните следующие действия:

- Войдите в Коллекцию с помощью учетной записи Майкрософт.
- Щелкните по своему изображению в верхней части окна и затем щелкните по своему имени.
- Щелкните **Новая коллекция**.
- Укажите имя коллекции, краткую сводку, описание и любые теги, которые помогут пользователям найти коллекцию.
- Добавьте изображение для идентификации коллекции.
- Щелкните **Создать**.

Теперь ваша коллекция является частью Коллекции аналитических данных Cortana.

Для добавления элементов в коллекцию откройте коллекцию, щелкните **Изменить** и затем щелкните **Добавить элемент**. Также если во время просмотра коллекции вы нашли элемент, который хотите добавить в коллекцию, просто откройте этот элемент, щелкните **Добавить в коллекцию** и укажите коллекцию, в которую хотите его добавить.

Для изменения сводки, описания или тегов коллекции откройте коллекцию и щелкните **Изменить**. При изменении коллекции также можно изменить порядок элементов в коллекции. Для этого воспользуйтесь стрелками рядом с элементом для его перемещения в списке. Также можно добавить примечания к элементам в коллекции, щелкнув в правом верхнем углу элемента и выбрав **Добавить/изменить примечание**.

## Часто задаваемые вопросы
**Хотелось бы внести изменения в рабочий процесс эксперимента, отправленного в Коллекцию. Как это можно сделать?**

На данный момент обновления рабочего процесса для экспериментов, опубликованных в Коллекции, не поддерживаются. Такие изменения можно публиковать как новый эксперимент в Коллекции, удалив старый эксперимент. Мы активно работаем над тем, чтобы реализовать публикацию обновлений в рабочий процесс для экспериментов, уже добавленных в Коллекцию.

**Придется ли мне публиковать новый эксперимент, даже если мне нужно лишь изменить теги или описание?**

В уже опубликованном эксперименте можно изменить следующие параметры без необходимости его повторной публикации:

- Имя эксперимента
- Текст сводки
- Текст описания
- Теги
- Образы

Чтобы изменить эти поля, щелкните нужный эксперимент, который хотите изменить (убедитесь, что вы вошли в систему с учетной записью Майкрософт). Откроется страница сведений эксперимента, на которой можно просмотреть параметры, которые нужно изменить или удалить. Щелкните **Изменить**, чтобы изменить любое из указанных выше полей.

**Я только что опубликовал свой эксперимент в Коллекции. Я не вижу изображение профиля рядом со своим именем.**

Возможно, вы используете учетную запись, отличную от @outlook, @msn, @live или @hotmail. Если это так, то вместо изображения профиля из параметров конфигурации учетной записи Майкрософт вы увидите изображение-заполнитель. Для решения этой проблемы войдите в систему с учетной записью Майкрософт и отправьте эксперимент повторно.

**Я обновил изображение профиля в настройках учетной записи Microsoft. Почему в моих существующих публикациях экспериментов этот рисунок профиля не отображается?**

Если необходимо, чтобы последние изменения профиля (изменение изображения профиля, имени или фамилии) отображались для всех экспериментов, необходимо повторно отправить старые эксперименты в Коллекцию. При этом потребуется удалить старые копии экспериментов, в которых отражаются старые параметры профиля.

**Каковы требования к изображению при отправке или редактировании изображения для моего эксперимента?**

Изображения, которые можно отправить вместе с вашим экспериментом будет использоваться для создания плитки публикуемого эксперимента. Рекомендуется использовать изображение размером до 500 Кбайт с соотношением сторон 3:2. Рекомендуется использовать разрешение 960 x 640

**Что происходит с набором данных, который использовался в эксперименте? Набор данных также публикуется в Коллекции?**

Если набор данных является частью вашего эксперимента и не был импортирован через модуль чтения, он будет опубликован в Коллекции вместе с вашим экспериментом. Поэтому убедитесь, что у набора данных, который публикуется с экспериментом, есть соответствующие условия лицензирования, которые допускают общий доступ и загрузку для любого пользователя.

**У меня есть эксперимент, использующий модуль чтения для извлечения данных из HDInsight или SQL. Для получения данных он использует учетные данные. Как опубликовать такой эксперимента и быть уверенным, что мои учетные данные не будут при этом переданы другим пользователям?**

В настоящее время мы не позволяем публиковать эксперименты, использующие учетные данные.

**Как можно разграничивать теги?**

Для разграничения тегов можно использовать табуляцию.

**Я вижу несколько подробно проиллюстрированных экспериментов с форматированным описанием. Однако при попытке составить форматированное описание, оно отображается в виде обычного текста.**

Форматированное описание отрисовки доступно не для всех пользователей Коллекции. Мы активно работаем над тем, чтобы предоставить эту возможность, с помощью поддержки отрисовки Markdown.

# Обмен решениями и их поиск в коллекции Cortana Intelligence

Коллекция Cortana Intelligence — это управляемый сообществом сайт для поиска и обмена решениями, созданными с использованием набора продуктов Cortana Intelligence Suite. Это место для тех, кто хочет научиться чему-то у других, начать разрабатывать собственные решения или делиться плодами своего труда с большим аналитическим сообществом.

**[ПЕРЕЙТИ К КОЛЛЕКЦИИ >>](http://gallery.cortanaintelligence.com)**

## Ресурсы, содержащиеся в коллекции

Коллекция Cortana Intelligence содержит множество ресурсов, которые можно использовать для разработки собственных аналитических решений.

- **[Эксперименты](https://gallery.cortanaintelligence.com/experiments)**. Коллекция содержит разнообразные эксперименты, разработанные в Студии машинного обучения Azure. К ним относятся как эксперименты, демонстрирующие конкретный метод машинного обучения, так и готовые решения для сложных задач машинного обучения.

- **[Записные книжки Jupyter](https://gallery.cortanaintelligence.com/notebooks)**. Эти книжки содержат код, визуализации данных и документацию на отдельном интерактивном холсте. Записные книжки в коллекции предоставляют руководства и подробное описание передовых методов и решений машинного обучения.

- **[API-интерфейсы машинного обучения](https://gallery.cortanaintelligence.com/machineLearningAPIs)**. Решение, разработанное с использованием Машинного обучения Azure, можно запустить как веб-службу, чтобы другие пользователи могли получать доступ к модели анализа с помощью набора интерфейсов REST API. В коллекции доступно множество таких решений, например подсистема рекомендаций по продукту и облачные алгоритмы для распознавания человеческих лиц и речи.

-  **[Шаблоны решений](https://gallery.cortanaintelligence.com/solutionTemplates)**. Шаблон решения служит отправной точкой, позволяющей быстро создать полнофункциональное комплексное решение для задачи в конкретной отрасли. Эти шаблоны содержат подробную документацию и все средства, необходимые для создания собственного решения.

- **[Руководства](https://gallery.cortanaintelligence.com/tutorials)**. В коллекции доступен ряд руководств, описывающих технологии и основные понятия машинного обучения, а также передовые методы для решения различных задач машинного обучения.

Эти основные ресурсы коллекции можно логически сгруппировать несколькими способами:

- **[Коллекции](https://gallery.cortanaintelligence.com/collections)**. Коллекция позволяет объединить эксперименты, интерфейсы API и другие элементы коллекции, относящиеся к конкретному решению или понятию.

- **[Отраслевые решения](https://gallery.cortanaintelligence.com/industries)**. Этот раздел коллекции объединяет различные ресурсы, относящиеся к таким отраслям, как розничная торговля, производство, банковское дело и здравоохранение.

Наконец, **[конкурсы](https://gallery.cortanaintelligence.com/competitions)** позволяют конкурировать с сообществом специалистов по обработке и анализу данных для решения сложных проблем с помощью пакета Cortana Intelligence Suite.
  
## Использование коллекции

Все пользователи могут искать и просматривать решения в коллекции, а также учиться на опыте других. Если войти с использованием учетной записи Майкрософт, можно скачать эксперименты в рабочую область Студии машинного обучения и загрузить в коллекцию собственные решения.

### Находите и изучайте

Находите и изучайте различные типы решений, которые были добавлены корпорацией Майкрософт и большим аналитическим сообществом. Используйте их как отправную точку в решении собственных проблем анализа данных.

В Коллекции легко найти недавно опубликованные и популярные решения. Также возможен поиск по именам, тегам, алгоритмам и другим атрибутам. Для просмотра вклада конкретного автора щелкните имя автора в любом из элементов.

В заголовке коллекции щелкните **Просмотреть все**, выберите раздел деталей поиска в левой части страницы и введите условия поиска вверху.

В разделе «Комментарии» для каждого решения можно оставить комментарии, отзывы или задать вопросы. Можно даже поделиться интересным решением с друзьями или коллегами в LinkedIn или Twitter. Ссылку на решение легко отправить по электронной почте, чтобы другие пользователи ознакомились с ним.

### Загрузка экспериментов

С помощью коллекции Cortana Intelligence можно легко скопировать эксперименты прогнозной аналитики в рабочую область Студии машинного обучения и сразу же приступить к их настройке и использованию. Если вы не вошли в систему во время просмотра, то перед копированием эксперимента в рабочую область вам будет предложено войти в систему с использованием учетной записи Майкрософт.

### Добавление экспериментов и коллекций

Выполнив вход в систему, вы становитесь участником сообщества коллекции. Это позволяет вам делиться собственными экспериментами и коллекциями с другими.

Дополнительные сведения и советы о создании надлежащей документации см. в разделе [Рекомендации по публикации и требования к качеству документации](#suggestions-for-publishing-and-for-quality-documentation) ниже.

#### Делитесь своими экспериментами

*Эксперимент* — это холст в Студии машинного обучения, на котором можно создать модель прогнозной аналитики, связав друг с другом данные от различных аналитических модулей. Вы можете пробовать различные идеи, выполнять пробные запуски и в конечном итоге опубликовать свою модель в виде веб-службы в Azure. Пример создания простого эксперимента см. в разделе [Руководство по машинному обучению. Создание первого эксперимента в Студии машинного обучения Azure](machine-learning-create-experiment.md). Подробное пошаговое руководство по созданию решения прогнозной аналитики см. в разделе [Руководство: разработка решения прогнозной аналитики для оценки кредитных рисков в Студии машинного обучения Azure](machine-learning-walkthrough-develop-predictive-solution.md).

Для отправки своего эксперимента в коллекцию Cortana Intelligence выполните следующие действия.

- Войдите в Студию машинного обучения с использованием учетной записи Майкрософт
- Создайте свой эксперимент и запустите его
- Когда вы будете готовы опубликовать свой эксперимент в Коллекции, щелкните **Опубликовать в Коллекции** под холстом эксперимента

Дополнительные сведения и советы по публикации эксперимента см. в разделе [Процедура публикации экспериментов Машинного обучения Azure](#process-for-publishing-azure-ml-experiments) ниже.

#### Делитесь своими коллекциями

*Коллекция* — это список связанных элементов из коллекции Cortana Intelligence, которые объединены вместе для использования для справки, совместной работы или в собственных решениях. В состав вашей коллекции могут входить ваши собственные элементы или элементы, созданные сообществом.

Например, в коллекции можно объединить элементы по определенной теме или элементы многоэтапного эксперимента, который решает сложные задачи. Примеры коллекций, добавленные корпорацией Майкрософт, представляют собой шаблоны многоэтапных экспериментов машинного обучения для решения реальных проблем, таких как обнаружение онлайн-мошенничества, классификация текста, прогнозирование в розничной торговле и прогнозирование обслуживания.

Для создания и отправки своей коллекции в коллекцию Cortana Intelligence выполните следующие действия.

- Войдите в Коллекцию с помощью учетной записи Майкрософт.
- Щелкните по своему изображению в верхней части окна и затем щелкните по своему имени.
- Щелкните **Новая коллекция**.
- Укажите имя коллекции, краткую сводку, описание и любые теги, которые помогут пользователям найти коллекцию.
- Добавьте изображение для идентификации коллекции.
- Нажмите кнопку **Создать**

Теперь ваша коллекция является частью коллекции Cortana Intelligence.

Для добавления элементов в коллекцию откройте коллекцию, щелкните **Изменить** и затем щелкните **Добавить элемент**. Также если во время просмотра коллекции вы нашли элемент, который хотите добавить в коллекцию, просто откройте этот элемент, щелкните **Добавить в коллекцию** и укажите коллекцию, в которую хотите его добавить.

Для изменения сводки, описания или тегов коллекции откройте коллекцию и щелкните **Изменить**. При изменении коллекции также можно изменить порядок элементов в коллекции. Для этого воспользуйтесь стрелками рядом с элементом для его перемещения в списке. Также можно добавить примечания к элементам в коллекции, щелкнув в правом верхнем углу элемента и выбрав **Добавить/изменить примечание**.

## Рекомендации по публикации и требования к качеству документации

- Хотя можно предположить, что у читателя есть опыт обработки и анализа данных, рекомендуется использовать простую речь и по возможности объяснить все как можно подробнее.
- Не все читатели знакомы с пакетом Cortana Intelligence Suite, так как он вышел не так давно. Поэтому необходимо привести достаточно информации и добавить пошаговые инструкции, чтобы читатели могли легко выполнить все задания.
- Визуальные элементы, в том числе диаграммы или снимки экрана с данными, помогут читателям лучше понять содержимое публикации и использовать его должным образом. Дополнительные сведения о добавлении изображений в документацию см. [здесь](https://gallery.cortanaintelligence.com/Collection/Publishing-Guidelines-and-Examples-1).
- Если набор данных является частью вашего эксперимента и не был импортирован через модуль чтения, он будет опубликован в коллекции вместе с вашим экспериментом. Поэтому убедитесь, что у набора данных, который публикуется, есть соответствующие условия лицензирования, предусматривающие общий доступ и загрузку для любого пользователя. Публикация в коллекции осуществляется в соответствии с [условиями использования](https://azure.microsoft.com/support/legal/website-terms-of-use/) Azure.

## Процедура публикации экспериментов Машинного обучения Azure

Чтобы опубликовать эксперимент в коллекции, выполните следующие пять действий.

1. Заполните поля заголовка и тега. Они должны быть описательными и содержать методы или реальную проблему, которую решает ваш эксперимент, например "Двоичная классификация: анализ мнений с помощью Twitter". ![Заполните поля заголовка и тега во время публикации](media/machine-learning-gallery-how-to-use-contribute-publish/experiment-description.png)

2. Напишите сводку по содержимому вашей документации. Дайте краткое описание проблемы и подхода для ее решения.

3. Используйте поле подробного описания, чтобы описать этапы вашего эксперимента. Сюда следует включить следующее:
	- снимок экрана с диаграммой эксперимента;
	- источники данных и соответствующее объяснение;
	- обработку данных;
	- проектирование признаков;
	- описание модели;
	- результаты и оценку эффективности модели.

	При необходимости для форматирования можно использовать разметку Markdown. Щелкните значок предварительного просмотра, чтобы посмотреть, как все будет выглядеть после публикации. Примеры в этой коллекции демонстрируют, что нужно включить и как можно систематизировать данные.

	> [AZURE.TIP] Поля для редактирования разметки Markdown и предварительного просмотра очень маленькие. Мы советуем написать документацию в редакторе разметки Markdown и вставить готовый документ в текстовое поле. Чтобы внести необходимые изменения и исправления в опубликованный эксперимент, используйте стандартные веб-инструменты для редактирования и предварительного просмотра в редакторе Markdown.

4. Загрузите эскиз элемента коллекции. Он будет отображаться вверху страницы элемента, а также на плитке элемента при просмотре коллекции. Можно выбрать изображение на компьютере или одно из готовых в коллекции.

5. Выберите настройки доступа к опубликованному содержимому: без ограничений или только для тех, кому предоставлена соответствующая ссылка.

	> [AZURE.TIP] Чтобы убедиться, что документация выглядит надлежащим образом, сначала ее можно опубликовать без внесения в список, а затем сделать ее общедоступной на странице элемента.

Вот и все.

Теперь вы можете просмотреть эксперимент в коллекции и предоставить ссылку на него другим пользователям. Если эксперимент опубликован как общедоступный, он будет отображаться при просмотре и поиске в коллекции. Выполнив вход в систему, вы можете изменить вашу документацию на странице элемента.

> [AZURE.TIP] Чтобы внести изменения в опубликованный эксперимент, перейдите к эксперименту в Студии машинного обучения Azure, внесите изменения и опубликуйте его снова. По умолчанию имеющееся опубликованное содержимое будет обновлено (а не заменено новым).

## Часто задаваемые вопросы
**Хотелось бы внести изменения в рабочий процесс эксперимента, отправленного в Коллекцию. Как это можно сделать?**

На данный момент обновления рабочего процесса для экспериментов, опубликованных в Коллекции, не поддерживаются. Такие изменения можно публиковать как новый эксперимент в Коллекции, удалив старый эксперимент. Мы активно работаем над тем, чтобы реализовать публикацию обновлений в рабочий процесс для экспериментов, уже добавленных в Коллекцию.

**Придется ли мне публиковать новый эксперимент, даже если мне нужно лишь изменить теги или описание?**

В уже опубликованном эксперименте можно изменить следующие параметры без необходимости его повторной публикации:

- Имя эксперимента
- Текст сводки
- Текст описания
- Теги
- Образы

Чтобы изменить эти поля, щелкните нужный эксперимент, который хотите изменить (убедитесь, что вы вошли в систему с учетной записью Майкрософт). Откроется страница сведений эксперимента, на которой можно просмотреть параметры, которые нужно изменить или удалить. Щелкните **Изменить**, чтобы изменить любое из указанных выше полей.

**Я только что опубликовал свой эксперимент в Коллекции. Я не вижу изображение профиля рядом со своим именем.**

Возможно, вы используете учетную запись, отличную от @outlook, @msn, @live или @hotmail. Если это так, то вместо изображения профиля из параметров конфигурации учетной записи Майкрософт вы увидите изображение-заполнитель. Для решения этой проблемы войдите в систему с учетной записью Майкрософт и отправьте эксперимент повторно.

**Я обновил изображение профиля в настройках учетной записи Microsoft. Почему в моих существующих публикациях экспериментов этот рисунок профиля не отображается?**

Если необходимо, чтобы последние изменения профиля (изменение изображения профиля, имени или фамилии) отображались для всех экспериментов, необходимо повторно отправить старые эксперименты в Коллекцию. При этом потребуется удалить старые копии экспериментов, в которых отражаются старые параметры профиля.

**Каковы требования к изображению при отправке или редактировании изображения для моего эксперимента?**

Изображения, которые можно отправить вместе с вашим экспериментом будет использоваться для создания плитки публикуемого эксперимента. Рекомендуется использовать изображение размером до 500 Кбайт с соотношением сторон 3:2. Рекомендуется использовать разрешение 960 x 640

**Что происходит с набором данных, который использовался в эксперименте? Набор данных также публикуется в Коллекции?**

Если набор данных является частью вашего эксперимента и не был импортирован через модуль чтения, он будет опубликован в Коллекции вместе с вашим экспериментом. Поэтому убедитесь, что у набора данных, который публикуется с экспериментом, есть соответствующие условия лицензирования, которые допускают общий доступ и загрузку для любого пользователя.

**У меня есть эксперимент, использующий модуль "Импорт данных" для извлечения данных из HDInsight или SQL. Для получения данных он использует учетные данные. Как опубликовать такой эксперимента и быть уверенным, что мои учетные данные не будут при этом переданы другим пользователям?**

В настоящее время мы не позволяем публиковать эксперименты, использующие учетные данные.

**Как можно разграничивать теги?**

Для разграничения тегов можно использовать табуляцию.

**Я вижу несколько подробно проиллюстрированных экспериментов с форматированным описанием. Однако при попытке составить форматированное описание, оно отображается в виде обычного текста.**

Форматированное описание отрисовки доступно не для всех пользователей Коллекции. Мы активно работаем над тем, чтобы предоставить эту возможность, с помощью поддержки отрисовки Markdown.

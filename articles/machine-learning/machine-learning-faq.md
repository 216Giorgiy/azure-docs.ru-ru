#Часто задаваемые вопросы о службе машинного обучения Azure: выставление счетов, возможности, ограничения и поддержка

Эта статья содержит ответы на часто задаваемые вопросы о службе машинного обучения Azure. Это облачная служба, предназначенная для прогнозного моделирования и ввода в эксплуатацию соответствующих решений с помощью веб-служб. Здесь предоставлена информация об использовании службы, в частности о модели выставления счетов, возможностях, ограничениях и поддержке.

##Общие вопросы

**Что такое Машинное обучение Azure?**

Машинное обучение Azure — это полностью управляемая служба, которую можно использовать для создания, проверки, задействования решений для прогнозной аналитики и управления ими в облаке. С помощью одного браузера вы сможете войти, передать данные и сразу же начать эксперименты машинного обучения. Прогнозное моделирование с возможностью перетаскивания объектов, большое разнообразие модулей и библиотека начальных шаблонов позволяют легко и быстро решать распространенные задачи машинного обучения. Дополнительную информацию см. в [обзоре службы Машинного обучения Azure](/services/machine-learning/). Основные сведения о машинном обучении, в частности информацию о ключевых терминах и понятиях, см. в статье [Что такое машинное обучение](machine-learning-what-is-machine-learning.md).


[AZURE.INCLUDE [machine-learning-free-trial](../../includes/machine-learning-free-trial.md)]

**Что такое Студия машинного обучения?**

Студия машинного обучения является средой разработки, доступной через веб-браузер. Она содержит большое разнообразие модулей с интерфейсом, предоставляющим визуальный состав, который позволяет полностью создавать рабочий процесс обработки и анализа данных в виде эксперимента.

Дополнительную информацию о Студии машинного обучения см. в статье [Что такое Студия машинного обучения Azure](machine-learning-what-is-ml-studio.md).

**Что такое служба API машинного обучения?**

Служба API машинного обучения позволяет развертывать прогнозные модели, созданные в Студии машинного обучения, в виде масштабируемых, отказоустойчивых веб-служб. Веб-службы, созданные службой API машинного обучения, представляют собой REST API, которые обеспечивают интерфейс для связи между внешними приложениями и моделями прогнозной аналитики.

Дополнительную информацию см. в статье [Подключение к веб-службе Машинного обучения Azure](machine-learning-connect-to-azure-machine-learning-web-service.md).

## Вопросы о Студии машинного обучения

###Создание эксперимента
**Предусмотрена ли возможность управления версиями или интеграции Git для диаграмм экспериментов?**

Нет, но при каждом выполнении эксперимента текущая версия диаграммы сохраняется, и ее не могут изменять другие пользователи.

###Импорт и экспорт данных для машинного обучения
**Какие источники данных поддерживаются в машинном обучении?**

Данные в Студию машинного обучения можно загрузить одним из двух способов: путем передачи локальных файлов в виде набора данных или импорта данных с помощью модуля чтения. Локальные файлы можно передать путем добавления новых наборов данных в Студии машинного обучения. Дополнительную информацию о поддерживаемых форматах файлов см. в статье [Импорт обучающих данных в Студии машинного обучения Azure](machine-learning-import-data.md).


####<a id="ModuleLimit"></a>Насколько большими могут быть объемы наборов данных для модулей?

Модули в Студии машинного обучения поддерживают наборы данных объемом до 10 ГБ плотных числовых данных для распространенных случаев использования. Если модуль принимает несколько видов входных данных, то их общий объем должен составлять 10 ГБ. Можно также создать выборку больших наборов данных с помощью запросов Hive или базы данных SQL Azure или предварительной обработки модуля "Обучение на основе счетчиков" перед приемом данных.

Следующие типы данных можно развернуть в большие наборы данных при нормализации признаков. Максимальный объем этих данных — менее 10 ГБ:

- разреженные;
- категориальные;
- строк
- Двоичные данные

В следующих модулях можно использовать наборы данных объемом менее 10 ГБ:

- модули системы рекомендаций;
- модуль SMOTE;
- модули написания сценариев: R, Python, SQL;
- модули, в которых объем выходных данных может превышать объем входных данных, такие как "Слияние" или "Хэширование признаков";
- "Перекрестная проверка", "Параметры очистки", "Порядковая регрессия" и "Многоклассовая классификация «один-все»", когда количество итераций очень велико.

Для наборов данных объемом больше, чем несколько гигабайт, требуется передать данные в службу хранилища Azure или базу данных SQL Azure либо использовать HDInsight, а не отправлять данные прямо из локального файла.


####<a id="UploadLimit"></a>Какие существуют ограничения для передачи данных?
Для наборов данных объемом больше, чем несколько гигабайт, требуется передать данные в службу хранилища Azure или базу данных SQL Azure либо использовать HDInsight, а не отправлять данные прямо из локального файла.

###Использование
**Когда запускать модель прогнозирования в виде службы пакетного выполнения, а когда в виде службы типа "запрос-ответ"?**

Служба типа "запрос-ответ" (RRS) представляет собой масштабируемую веб-службу с малой задержкой, которая используется для обеспечения интерфейса моделям, не имеющим состояний, созданным и развернутым в среде экспериментов. Служба пакетного выполнения (BES) представляет собой службу для асинхронных оценок пакета записей данных. Входные данные для службы BES похожи на входные данные, используемые в службе RRS. Основная разница заключается в том, что служба BES считывает блок записей из различных источников, таких как служба BLOB-объектов и служба таблиц в Azure, база данных SQL Azure, HDInsight (запросы Hive) и HTTP. Дополнительную информацию см. в статье [Как использовать веб-службы машинного обучения](machine-learning-consume-web-services.md).

**Как обновить модель для развернутой веб-службы?**

Для обновления прогнозной модели уже развернутых служб нужно просто внести изменение и повторно запустить эксперимент, сохранить обученную модель. Если у вас есть доступная новая версия обученной модели, ML Studio запросит о возможности обновления службы в промежуточной области. После применения обновления к промежуточной веб-службе то же самое обновление станет доступно для установки на рабочую веб-службу. Дополнительную информацию об обновлении развернутой веб-службы см. в статье [Развертывание веб-службы машинного обучения](machine-learning-publish-a-machine-learning-web-service.md).

Также можно использовать API-интерфейсы для повторного обучения. Пример кода доступен [здесь](https://azuremlretrain.codeplex.com/).


**Как контролировать веб-службу, развернутую в рабочей среде?**

После того как прогнозная модель была запущена в рабочей среде, вы сможете контролировать ее через портал Azure. Каждая развернутая служба имеет свою собственную панель мониторинга, где вы сможете увидеть данные мониторинга для этой службы.

**Имеется ли место, где можно увидеть выходные данные службы RRS или BES?**

Для RRS ответ веб-службы обычно находится там, где вы видите результат. Также его можно записать в большой двоичный объект. Для BES вывод записывается в большой двоичный объект по умолчанию. Также вывод можно записать в базу данных или таблицу с помощью модуля записи.

 **** Можно ли создавать веб-службы только на основе моделей, созданных в Студии? Нет. Веб-службы можно также создавать непосредственно из записных книжек Jupyter и из RStudio.

##Безопасность и доступность

**Кто имеет доступ к конечной точке HTTP для веб-службы, развернутой в рабочей среде по умолчанию? Как ограничить доступ к конечной точке?**

После развертывания веб-службы мы создаем для нее конечную точку по умолчанию. Такая конечная точка развертывается в производственной среде и может вызываться с помощью ее ключа API. Дополнительные конечные точки с собственными ключами можно добавить на портале Azure или программным путем с использованием API-интерфейсов управления веб-службами. Ключи доступа необходимы для выполнения вызовов к веб-службам в рабочей и промежуточной областях. Дополнительную информацию см. в статье [Подключение к веб-службе Машинного обучения Azure](machine-learning-connect-to-azure-machine-learning-web-service.md).


**Что произойдет, если не удастся найти мою учетную запись хранения?**

Студия машинного обучения использует учетную запись хранения Azure, предоставляемую пользователем, для хранения промежуточных данных при выполнении рабочего процесса. Эта учетная запись хранения предоставляется Студии машинного обучения во время создания рабочей области. После создания рабочей области, если учетная запись хранения удаляется и ее больше невозможно найти, эта рабочая область перестанет действовать и все эксперименты в этой рабочей области завершатся сбоем.

Если вы случайно удалили учетную запись хранения, единственный способ ее восстановить — создать ее повторно с таким же именем и в том же регионе, что и у удаленной учетной записи хранения. После этого синхронизируйте ключ доступа еще раз.


**Что произойдет, если ключ доступа к учетной записи хранения не синхронизирован?** Студия машинного обучения использует учетную запись хранения Azure, предоставляемую пользователем, для хранения промежуточных данных при выполнении рабочего процесса. Эта учетная запись хранения предоставляется Студии машинного обучения во время создания рабочей области, с которой связаны ключи доступа. После создания рабочей области, если ключи доступа изменяются, эта рабочая область не сможет получить доступ к учетной записи хранения и перестанет действовать, а все эксперименты в этой рабочей области завершатся сбоем.

Если вы изменили ключи доступа к учетной записи хранения, обязательно синхронизируйте их повторно в параметрах рабочей области на портале Azure.

##Поддержка и обучение

**Где можно пройти обучение по Машинному обучению Azure?**

В [центре документации Машинного обучения Azure](/services/machine-learning/) размещены видеоучебники и пошаговые руководства. Эти пошаговые руководства представляют введение в службы и пошаговые пособия по жизненному циклу процесса обработки и анализа данных, а именно: по импорту данных, очистке данных, созданию прогнозных моделей и их развертыванию в рабочей среде с использованием Машинного обучения Azure.

Мы будем добавлять новый материал в учебный центр машинного обучения на постоянной основе. Вы можете отправить запросы для получения дополнительного учебного материала в учебный центр машинного обучения на [форуме обратной связи](https://windowsazure.uservoice.com/forums/257792-machine-learning).

Вы также можете пройти обучение на сайте [Microsoft Virtual Academy](http://www.microsoftvirtualacademy.com/training-courses/getting-started-with-microsoft-azure-machine-learning).

**Где можно получить поддержку по Машинному обучению Azure?**

Чтобы получить техническую поддержку по Машинному обучению Azure, перейдите на страницу [Варианты поддержки для Azure](/support/options/) и выберите **Машинное обучение**.

Машинное обучение Azure также имеет форум сообщества на MSDN, где вы можете задавать вопросы, связанные с Azure ML. Форум контролируется командой поддержки Azure ML. Посетите [форум Azure](http://social.msdn.microsoft.com/Forums/windowsazure/home?forum=MachineLearning).

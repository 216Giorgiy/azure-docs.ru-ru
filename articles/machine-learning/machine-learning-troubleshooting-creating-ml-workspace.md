# Руководство по устранению неполадок: создание рабочей области машинного обучения и подключение к ней

В этом руководстве приведены решения некоторых проблем, часто возникающих при настройке рабочих областей для Машинного обучения Azure.

[AZURE.INCLUDE [machine-learning-free-trial](../../includes/machine-learning-free-trial.md)]

## Владелец рабочей области

При создании новой рабочей области машинного обучения в поле «ВЛАДЕЛЕЦ РАБОЧЕЙ ОБЛАСТИ» вводится идентификатор, который должен быть допустимой учетной записью Майкрософт (ранее Windows Live ID), например john-contoso@live.com или john-contoso@hotmail.com. Нельзя использовать другие учетные записи, например учетную запись корпоративной электронной почты. Чтобы создать новую учетную запись Майкрософт, перейдите по ссылке [www.live.com](http://www.live.com).

Обратите внимание, что для учетной записи, используемой для входа на классический портал Azure для создания рабочей области, не предусмотрено автоматическое предоставление разрешений на *открытие* этой рабочей области, если не указать, что эта запись является ее владельцем. Чтобы открыть рабочую область в Студии машинного обучения, необходимо войти в учетную запись Майкрософт, определенную в качестве учетной записи владельца рабочей области, или получить приглашение присоединиться к рабочей области от владельца. На классическом портале Azure можно, однако, *управлять* рабочей областью, что включает возможность изменить владельца и настроить доступ.

Дополнительную информацию об управлении рабочей областью см. в разделе [Управление рабочей областью машинного обучения Azure].

[Управление рабочей областью машинного обучения Azure]: machine-learning-manage-workspace.md

## Разрешенные регионы

Сейчас машинное обучение доступно только в ограниченном числе регионов. Если в вашу подписку не входит один из таких регионов, вы можете получить сообщение об ошибке «У вас нет подписки в разрешенных регионах».

Чтобы запросить добавление этого региона в вашу подписку, выберите **Обратиться в службу поддержки Майкрософт** на классическом портале Azure, выберите **Выставление счетов** в качестве типа проблемы и следуйте инструкциям, чтобы отправить запрос.

![Обратиться в службу поддержки Майкрософт][screen1]

## Учетная запись хранения

Для хранения данных службе машинного обучения требуется учетная запись хранения. Можно использовать имеющуюся учетную запись хранения либо создать новую учетную запись хранения при создании новой рабочей области машинного обучения (если имеется квота на ее создание).

![Создание рабочей области][screen2]

После создания новой рабочей области машинного обучения вы сможете войти в Студию машинного обучения, используя учетную запись Майкрософт, которая была указана в качестве владельца рабочей области. При появлении сообщения об ошибке «Рабочая область не найдена» (как на следующем снимке экрана) выполните следующие действия, чтобы удалить файлы cookie браузера.

![Рабочая область не найдена.][screen3]

**Удаление файлов cookie браузера**

Если вы используете Internet Explorer, нажмите кнопку **Сервис** в правом верхнем углу и выберите пункт **Свойства браузера**.

![Свойства браузера][screen4]

На вкладке **Общие**, нажмите кнопку **Удалить…**

![Вкладка «Общие»][screen5]

В диалоговом окне **Удаление истории обзора** убедитесь, что флажок **Файлы cookie и данные веб-сайтов** выбран, и нажмите кнопку **Удалить**.

![Удалить файлы cookie][screen6]

После удаления файлов cookie перезапустите браузер и перейдите на страницу [Машинного обучения Microsoft Azure](https://studio.azureml.net). Когда появится запрос имени пользователя и пароля, введите данные учетной записи Майкрософт, которая была указана в качестве владельца рабочей области.

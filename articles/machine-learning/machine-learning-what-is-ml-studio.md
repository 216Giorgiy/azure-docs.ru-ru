# Что такое Студия машинного обучения Azure?

Студия машинного обучения Microsoft Azure - это среда совместной визуальной разработки, позволяющая разрабатывать, тестировать и развертывать решения прогнозной аналитики, работающие с вашими данными. Среда разработки и обслуживания машинного обучения основана на облачной технологии, обеспечивает гибкость распоряжения вычислительными ресурсами и памятью, а также исключает проблемы установки и настройки, поскольку вся работа осуществляется через веб-браузер. 

Студия машинного обучения Microsoft Azure - это место, где пересекаются обработка и анализ данных, прогнозная аналитика, облачные ресурсы и данные клиента!

## Интерактивное рабочее пространство Студии машинного обучения Microsoft Azure

Для разработки модели прогнозной аналитики обычно используются данные из одного или нескольких источников. Для получения набора результатов эти данные преобразуются и анализируются с помощью различных операций и статистических функций. Таким образом, разработка модели идет по итерациям - путем изменения разных функций и их параметров выполняется сведение результатов, пока не будет получена подготовленная и эффективная модель.

**Студия машинного обучения Microsoft Azure** предоставляет интерактивное визуальное рабочее пространство для облегчения построения, тестирования и выполнения итераций модели прогнозной аналитики. Вы перетаскиваете ***наборы данных*** и ***модели*** анализа на интерактивный ***холст*** и связываете их вместе, чтобы сформировать ***эксперимент***, который затем ***запускается*** в Студии машинного обучения Microsoft Azure. Для выполнения итераций в макете следует ***отредактировать*** эксперимент, при необходимости ***сохранить*** копию и запустить его снова. Как только все будет готово, можно ***опубликовать*** эксперимент в качестве ***веб-службы***, чтобы к модели могли обращаться другие пользователи. 

Никакое программирование не требуется, выполняется только визуальное соединение наборов данных и модулей, чтобы создать модель прогнозной аналитики.

## Начало работы со Студией машинного обучения Microsoft Azure

При первом входе в Студию машинного обучения Microsoft Azure слева появляются следующие вкладки.

- **Домашняя страница Студии** - набор ссылок на документы и другие материалы
- **ЭКСПЕРИМЕНТЫ** - эксперименты, которые создаются, запускаются или сохраняются как черновики. 
- **ВЕБ-СЛУЖБЫ** - список опубликованных вами экспериментов. 
- **НАСТРОЙКИ** - набор параметров для настройки вашей учетной записи и ресурсов. 

>[WACOM.NOTE] При создании эксперимента в левой части холста отображается рабочий список	доступных наборов данных и модулей. Это список компонентов, используемых для построения модели.

## Компоненты эксперимента

Эксперимент состоит из наборов данных, предоставляющих информацию в модули аналитики, которые соединяются по вашему усмотрению для создания модели прогнозной аналитики. В частности, правильный эксперимент имеет следующие характеристики.

- Он имеет по крайней мере один набор данных и один модуль. 
- Наборы данных могут быть связаны только с модулями. 
- Модули могут быть связаны либо с наборами данных, либо с другими модулями. 
- Все входные порты для модулей должны иметь связь с потоком данных. 
- Все необходимые параметры для модуля должны быть установлены. 

Пример создания простого эксперимента см. в разделе [Создание простого эксперимента в Студии машинного обучения Azure](http://azure.microsoft.com/documentation/articles/machine-learning-create-experiment/). 
Подробное пошаговое руководство по созданию решения прогнозной аналитики см. в разделе [Разработка прогнозирующего решения с помощью Студии машинного обучения Azure](http://azure.microsoft.com/documentation/articles/machine-learning-walkthrough-develop-predictive-solution/).

### Наборы данных

Набор данных - это данные, отправленные в Студию машинного обучения Microsoft Azure для использования в процессе моделирования.  В Студию машинного обучения Microsoft Azure включено несколько наборов данных, чтобы с ними можно было экспериментировать; кроме того, можно отправлять дополнительные наборы данных по мере необходимости.  Ниже приведены некоторые примеры включенных наборов данных.

- **Данные MPG для разных автомобилей** - данные по пробегу в милях на галлон топлива для автомобилей в зависимости от числа цилиндров, мощности в лошадиных силах и т. п. 
- **Данные по раку груди** - данные по диагностике рака груди. 
- **Данные по лесным пожарам** - объемы лесных пожаров в Северо-Восточной Португалии. 

При построении эксперимента рабочий список наборов данных доступен на холсте слева. 

### Модули

Модуль - это алгоритм обработки данных.  В Студии машинного обучения Microsoft Azure имеется ряд модулей, начиная с функций ввода данных для процессов обучения, оценки и проверки.  Ниже приведены примеры включенных модулей.

- **Преобразование в ARFF** - преобразует упорядоченный набор данных .NET в формат ARFF. 
- **Элементарная статистика** - выполняет элементарные статистические вычисления, такие как получения математического ожидания, среднеквадратичного отклонения и др. 
- **Линейная регрессия** - создает интерактивный линейно-регрессионный анализ на основе градиентного спуска. 
- **Балльная модель** - проводит балльную оценку отрабатываемой модели классификации или регрессии. 

При построении эксперимента рабочий список модулей доступен на холсте слева. 

Модуль может иметь ряд параметров, используемых для настройки внутренних алгоритмов модуля. При выборе модуля на холсте   его параметры отображаются в области справа. Подстройка модели осуществляется изменением параметров в этой области.

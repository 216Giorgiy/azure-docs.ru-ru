<a name="change-duration"></a>
<a name="change-retention"></a>

### <a name="change-run-duration-and-storage-retention"></a>Изменение длительности выполнения и хранения

Вы можете установить это ограничение в диапазоне от 7 до 90 дней. Однако если вам необходимо превысить максимальное ограничение, [свяжитесь с командой Logic Apps](mailto://logicappsemail@microsoft.com), чтобы получить справку по своим требованиям.

1. На портале Azure в меню вашего приложения логики выберите **Параметры рабочего процесса**. 

2. В разделе **Параметры среды выполнения** в списке **Срок хранения журнала в днях** выберите **Пользовательский**. 

3. Введите необходимое число дней или перетащите ползунок на это число дней.

## <a name="http-request-limits"></a>Ограничения HTTP-запросов

Ниже приведены ограничения для одного HTTP-запроса или синхронного вызова соединителя:

#### <a name="timeout"></a>Время ожидания

Некоторые операции соединителя выполняют асинхронные вызовы или прослушивают запросы веб-перехватчика, поэтому время ожидания для этих операций может превышать эти ограничения. Дополнительные сведения см. в разделе с техническими сведениями для указанного соединителя, а также в разделе [Триггеры и действия рабочих процессов](../logic-apps/logic-apps-workflow-actions-triggers.md#http-action).

| ИМЯ | Ограничение | Заметки | 
| ---- | ----- | ----- | 
| Исходящий запрос | 120 секунд | Для более длительных операций используйте [шаблон асинхронного опроса](../logic-apps/logic-apps-create-api-app.md#async-pattern) или [цикл until](../logic-apps/logic-apps-workflow-actions-triggers.md#until-action). | 
| Синхронный отклик | 120 секунд | Чтобы на исходный запрос был получен ответ, все шаги, необходимые для получения ответа, должны быть завершены в течение указанного времени ожидания. Исключение составляет вызов другого приложения логики в качестве вложенного рабочего процесса. Дополнительные сведения см. в разделе [Вызов, активация и вложение приложений логики](../logic-apps/logic-apps-http-endpoint.md). | 
|||| 

#### <a name="message-size"></a>Размер сообщения

| Name (Имя) | Ограничение | Заметки | 
| ---- | ----- | ----- | 
| Размер сообщения | 100 МБ | Чтобы обойти это ограничение, см. раздел [Обработка больших сообщений](../logic-apps/logic-apps-handle-large-messages.md). Тем не менее некоторые соединители и API-интерфейсы могут не поддерживать фрагментацию и даже ограничение по умолчанию. | 
| Размер сообщения с фрагментацией | 1 GB | Это ограничение применяется к действиям, которые изначально поддерживают фрагментацию или для которых поддержка фрагментации включена в конфигурации среды выполнения. Дополнительные сведения см. в разделе [Обработка больших сообщений с фрагментацией](../logic-apps/logic-apps-handle-large-messages.md). | 
| Оценочное ограничение выражения | 131 072 символов | `@concat()`, `@base64()` и `@string()` не могут превышать это ограничение. |

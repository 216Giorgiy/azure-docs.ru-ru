# <a name="create-a-new-logic-app-connecting-saas-services"></a>Создание нового приложения логики, подключающего службы SaaS
В этой статье показано, как всего за несколько минут создать [приложение логики Azure](logic-apps-what-are-logic-apps.md). Мы рассмотрим простой рабочий процесс, который позволяет получать интересующие вас твиты в почтовый ящик.

Для использования этого сценария вам понадобится:

* Подписка Azure
* учетная запись Twitter;
* учетная запись Outlook.com или Office 365 Outlook.

## <a name="create-a-new-logic-app-to-email-you-tweets"></a>Создание приложения логики для отправки твитов по электронной почте

1. Войдите на [портал Azure](https://portal.azure.com). 

2. В меню слева выберите **Создать** > **Интеграция Enterprise** > **Приложение логики**.

    Или выберите **Создать**, введите в поле поиска `logic app` и нажмите клавишу ВВОД. Выберите **Приложение логики** > **Создать**.

3. Введите имя для приложения логики, выберите подписку Azure, создайте или выберите группу ресурсов Azure, выберите расположение и нажмите кнопку **Создать**.

    Если выбрать параметр **Закрепить на панели мониторинга**, приложение логики автоматически откроется после развертывания.

4. При первом открытии приложения логики можно выбрать шаблон для запуска.
Чтобы создать приложение логики с нуля, щелкните **Blank Logic App** (Пустое приложение логики). 

5. Для начала необходимо создать триггер. Это событие, которое запускает приложение логики. В поле поиска введите **twitter** и выберите **Время публикации нового твита**. Войдите, используя имя пользователя и пароль своей учетной записи Twitter.

6. Теперь введите поисковый запрос, который будет активировать приложение логики.

   ![Поиск Twitter](media/logic-apps-create-a-logic-app/twittersearch.png)

    Параметры **Частота** и **Интервал** определяют частоту, с которой приложение логики будет проверять новые твиты и возвращать их в рамках определенного интервала.

7. Нажмите кнопку **Новый шаг**, а затем выберите **Добавить действие** или **Добавить условие**.

    Если выбрать вариант **Добавить действие**, можно выбрать действие в списке [доступных соединителей](../connectors/apis-list.md). 

8. В поле поиска введите **outlook** и выберите **Отправить электронное письмо** для отправки электронного письма из учетной записи Outlook на указанный адрес электронной почты.

   ![Действия](media/logic-apps-create-a-logic-app/actions.png)

9. Теперь заполните параметры электронной почты.

   ![Параметры](media/logic-apps-create-a-logic-app/parameters.png)

10. Наконец, щелкните **Сохранить** , чтобы запустить свое приложение логики.

## <a name="manage-your-logic-app-after-creation"></a>Управление приложением логики после его создания

Теперь ваше приложение логики запущено и работает. Оно будет периодически проверять наличие твитов с определенным условием поиска. После обнаружения соответствующего твита вам будет отправлено сообщение. Теперь мы покажем, как отключать приложение, и вы увидите, как это работает.

1. Перейдите на [портал Azure](https://portal.azure.com).

2. В меню слева выберите **Другие службы**. В разделе **Интеграция Enterprise** выберите **Приложения логики**. Выберите приложение логики.

    *    Чтобы просмотреть состояние, журнал выполнения и общие сведения приложения, в меню приложения логики выберите **Обзор**. Если вы не нашли ожидаемые данные, на панели команд нажмите кнопку **Обновить**.

    *    Чтобы изменить приложение, в меню приложения логики выберите **Конструктор приложений логики**.

    *    Чтобы временно отключить приложение, в меню приложения логики выберите **Обзор**. На панели команд выберите **Отключить**.

    *    Чтобы удалить приложение, в меню приложения логики выберите **Обзор**. 
    На панели команд выберите **Удалить**. Введите имя приложения логики и выберите **Удалить**.

Менее чем за 5 минут вы смогли создать простое приложение логики, работающее в облаке. См. дополнительные сведения об [использовании функций приложений логики]. См. дополнительные сведения о [создании определений приложений логики](../logic-apps/logic-apps-author-definitions.md).

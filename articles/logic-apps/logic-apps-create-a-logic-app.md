# <a name="create-your-first-logic-app-workflow-to-automate-processes-between-cloud-apps-and-cloud-services"></a>Создание первого рабочего процесса приложения логики для автоматизации процессов между облачными приложениями и облачными службами

Создавая и выполняя рабочие процессы в [Azure Logic Apps](logic-apps-what-are-logic-apps.md), вы можете ускорить и упростить автоматизацию бизнес-процессов даже без написания кода. В первом примере показано, как создать базовый рабочий процесс приложения логики, выполняющий поиск нового содержимого веб-сайта в RSS-канале. При обнаружении нового элемента в канале веб-сайта приложение логики отправляет сообщение из учетной записи Outlook или Gmail.

Чтобы создать и запустить приложение логики, вам потребуется:

* Подписка Azure. Если у вас нет подписки, вы можете [создать бесплатную пробную версию учетной записи Azure](https://azure.microsoft.com/free/). В противном случае вы можете [зарегистрироваться на получение подписки с оплатой по мере использования](https://azure.microsoft.com/pricing/purchase-options/).

  Подписка Azure используется для выставления счетов за использование приложения логики. Узнайте об [измерении использования](../logic-apps/logic-apps-pricing.md) и [модели ценообразования](https://azure.microsoft.com/pricing/details/logic-apps) Azure Logic Apps.

Кроме того, в этом примере вам потребуются следующие элементы:

* Учетная запись Outlook.com, Office 365 Outlook или Gmail.

    > [!TIP]
    > Если у вас есть личная [учетная запись Майкрософт](https://account.microsoft.com/account), у вас также есть и учетная запись Outlook.com, а если рабочая или учебная учетная запись Azure — то учетная запись **Office 365 Outlook**.

* Подключение к RSS-каналу веб-сайта. В этом примере используется RSS-канал [веб-сайта MSDN Channel 9](https://channel9.msdn.com/): `https://s.ch9.ms/Feeds/RSS`.

## <a name="add-a-trigger-that-starts-your-workflow"></a>Добавление триггера запуска рабочего процесса

[*Триггер*](./logic-apps-what-are-logic-apps.md#logic-app-concepts) — это событие, которое запускает рабочий процесс приложения логики. Это основной элемент приложения логики.

1. Войдите на [портал Azure](https://portal.azure.com "Портал Azure").

2. В меню слева выберите **Создать** > **Интеграция с предприятием** > **Приложение логики**, как показано ниже.

     ![Портал Azure, "Создать", "Интеграция с предприятием", "Приложение логики"](media/logic-apps-create-a-logic-app/azure-portal-create-logic-app.png)

   > [!TIP]
   > Или выберите **Создать**, введите в поле поиска `logic app` и нажмите клавишу ВВОД. Затем выберите **Приложение логики** > **Создать**.

3. Введите имя приложения логики и выберите подписку Azure. Теперь создайте или выберите группу ресурсов Azure, что поможет вам упорядочить связанные ресурсы Azure и управлять ими. И наконец, выберите расположение центра обработки данных для размещения приложения логики. Когда вы будете готовы, установите флажок **Закрепить на панели мониторинга** и нажмите кнопку **Создать**.

     ![Сведения о приложении логики](media/logic-apps-create-a-logic-app/logic-app-settings.png)

   > [!NOTE]
   > Когда вы устанавливаете флажок **Закрепить на панели мониторинга**, после развертывания приложение логики появляется на панели мониторинга и автоматически открывается. Если приложение логики не появилось на панели мониторинга, щелкните элемент **Все ресурсы**, выберите **Показать больше**, а затем — свое приложение логики. Или в меню слева выберите **Больше служб**. В разделе **Интеграция с предприятием** выберите **Logic Apps**, а затем — свое приложение логики.

4. При первом открытии приложения логики в конструкторе приложений логики отображаются шаблоны начала работы. Сейчас выберите **Пустое приложение логики**. Это позволит создать приложение логики с нуля.

    После этого откроется конструктор приложений логики со службами и *триггерами*, которые можно использовать в этих приложениях.

5. В поле поиска введите `RSS` и выберите этот триггер: **RSS — при публикации элемента веб-канала**. 

    ![Триггер RSS](media/logic-apps-create-a-logic-app/rss-trigger.png)

6. Введите ссылку на RSS-канал веб-сайта, который вы хотите отслеживать. 

     Вы также можете изменить **частоту** и **интервал**. 
     Эти параметры определяют частоту, с которой приложение логики будет проверять новые элементы и возвращать найденные в рамках определенного интервала.

     В этом примере мы будем каждый день искать новые элементы, опубликованные на веб-сайте MSDN Channel 9.

     ![Укажите RSS-канал, частоту и интервал триггера.](media/logic-apps-create-a-logic-app/rss-trigger-setup.png)

7. Сохраните результаты. (На панели команд конструктора щелкните **Сохранить**.)

   ![Сохранение приложения логики](media/logic-apps-create-a-logic-app/save-logic-app.png)

   После сохранения приложение будет запущено. Но сейчас оно проверяет только новые элементы в определенном RSS-канале. 
   Чтобы сделать это приложение более практичным, мы добавим действие, которое будет выполнять приложение логики после активации триггера.

## <a name="add-an-action-that-responds-to-your-trigger"></a>Добавление действия, которое будет реагировать на триггер

[*Действие*](./logic-apps-what-are-logic-apps.md#logic-app-concepts) — это задание, выполняемое рабочим процессом приложения логики. После добавления триггера в приложение логики вы можете добавить действие, выполняющее операции с данными, созданными этим триггером. Например, сейчас мы добавим действие, отправляющее сообщение при обнаружении нового элемента в RSS-канале веб-сайта.

1. В конструкторе для триггера выберите **Новый шаг** > **Добавить действие**, как показано ниже.

   ![Добавление действия](media/logic-apps-create-a-logic-app/add-new-action.png)

   В конструкторе отображаются [доступные соединители](../connectors/apis-list.md). Вы можете выбрать действие, которое будет выполняться при активации триггера.

2. В зависимости от учетной записи электронной почты выполните действия для Outlook или Gmail.

   * Чтобы отправить сообщение из учетной записи Outlook, в поле поиска введите `outlook`. В разделе **Службы** выберите **Outlook.com** для личной учетной записи Майкрософт или **Office 365 Outlook** для рабочей или учебной учетной записи Azure. 
   В разделе **Действия** выберите **Отправить электронное письмо**.

       ![Выбор действия Outlook "Отправить электронное письмо"](media/logic-apps-create-a-logic-app/actions.png)

   * Чтобы отправить сообщение из учетной записи Gmail, в поле поиска введите `gmail`. 
   В разделе **Действия** выберите **Отправить электронное письмо**.

       ![Выбор действия Gmail "Отправить электронное письмо"](media/logic-apps-create-a-logic-app/actions-gmail.png)

3. При появлении запроса на ввод учетных данных укажите имя пользователя и пароль своей учетной записи электронной почты. 

4. Укажите сведения об этом действии, например конечный адрес электронной почты, и параметры данных, которые необходимо добавить в сообщение, например:

   ![Выбор данных для включения в сообщение электронной почты](media/logic-apps-create-a-logic-app/rss-action-setup.png)

    Если вы выбрали Outlook, приложение логики может выглядеть следующим образом:

    ![Готовое приложение логики](media/logic-apps-create-a-logic-app/save-run-complete-logic-app.png)

5.    Сохраните изменения. (На панели команд конструктора щелкните **Сохранить**.)

6. Теперь приложение логики можно вручную запустить, чтобы протестировать. На панели команд конструктора щелкните **Запуск**. Кроме того, приложение логики может проверить указанный RSS-канал на основе заданного расписания.

   Обнаружив новый элемент, приложение логики отправляет сообщение электронной почты с выбранными данными. 
   Если новые элементы не найдены, приложение логики пропускает действие отправки сообщения.

7. Чтобы отследить журнал триггера и выполнения приложения логики, в меню приложения выберите **Обзор**.

   ![Мониторинг и просмотр журнала триггера и выполнения приложения логики](media/logic-apps-create-a-logic-app/logic-app-run-trigger-history.png)

   > [!TIP]
   > Если вы не нашли ожидаемые данные, на панели команд щелкните **Обновить**.

   Дополнительные сведения о состоянии, журнале триггера и выполнения, а также о диагностике приложения логики см. в статье [Диагностика сбоев приложений логики](logic-apps-diagnosing-failures.md).

      > [!NOTE]
      > Приложение логики продолжает работать до отключения приложения. Чтобы отключить приложение, в меню приложения логики выберите **Обзор**. На панели команд выберите **Отключить**.

Поздравляем, вы только что настроили и запустили свое первое базовое приложение логики. Вы также узнали, как можно легко создать рабочие процессы автоматизации и как интегрировать облачные приложения и облачные службы без написания кода.

## <a name="manage-your-logic-app"></a>Управление приложением логики

Управление приложением логики подразумевает такие действия, как проверка состояния, изменение, просмотр журнала, отключение или его удаление.

1. Войдите на [портал Azure](https://portal.azure.com "Портал Azure").

2. В меню слева выберите **Больше служб**. В разделе **Интеграция Enterprise** выберите **Приложения логики**. Выберите приложение логики. 

   В меню приложения логики доступны следующие задачи управления:

   |Задача|Действия| 
   |:---|:---| 
   | Просмотр состояния, журнала выполнения приложения, а также общих сведений о нем.| Выберите **Обзор**.| 
   | Изменение приложения | Выберите **Конструктор приложений логики**. | 
   | Просмотр определения JSON рабочего процесса приложения | Выберите **Представление кода приложения логики**. | 
   | Просмотр выполненных в приложении логики операций | Выберите **Журнал действий**. | 
   | Просмотр предыдущих версий приложения логики | Выберите **Версии**. | 
   | Временное отключение приложения | Выберите **Обзор**, а затем на панели команд щелкните **Отключить**. | 
   | Удаление приложения | Выберите **Обзор**, а затем на панели команд щелкните **Удалить**. Введите имя приложения логики и выберите **Удалить**. |

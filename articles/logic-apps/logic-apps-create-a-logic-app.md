# <a name="automate-your-first-workflow-to-process-data-with-a-logic-app"></a>Автоматизация первого рабочего процесса для обработки данных с помощью приложения логики

Чтобы быстрее интегрировать системы и службы для своей организации, можно автоматизировать рабочие процессы и бизнес-процессы с помощью [Azure Logic Apps](../logic-apps/logic-apps-what-are-logic-apps.md). Из этого краткого руководства вы узнаете, как легко можно создать и запустить автоматизированный рабочий процесс путем создания приложения логики. В примере приложения показано, как автоматизировать рабочий процесс, проверяющий RSS-канал веб-сайта на наличие новых элементов и отправляющий сообщение электронной почты по каждому такому элементу.

Этот образец приложения логики отправляет сообщение электронной почты, как в следующем примере:

![Отправленное сообщение о новом элементе веб-канала RSS](./media/logic-apps-create-a-logic-app/rss-feed-email.png)

Ниже представлен высокоуровневый рабочий процесс приложения логики, которое вы создаете:

![Обзор: пример приложения логики](./media/logic-apps-create-a-logic-app/logic-app-simple-overview.png)

Из этого краткого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание пустого приложения логики.
> * Добавление триггера для запуска рабочего процесса при появлении нового элемента в RSS-канале.
> * Добавление действия для отправки сообщения с подробными сведениями об элементе веб-канала RSS.
> * Выполнение рабочего процесса приложения логики.

Если у вас еще нет подписки Azure, [подпишитесь для получения бесплатной учетной записи](https://azure.microsoft.com/free/) Azure, прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования

* Учетная запись электронной почты любого поставщика электронной почты, поддерживаемого Azure Logic Apps для отправки уведомлений. Например, можно использовать Office 365 Outlook, Outlook.com или Gmail. Сведения о других поддерживаемых соединителях отправки электронной почты см. в [списке соединителей](https://docs.microsoft.com/connectors/). В этом кратком руководстве используется Office 365 Outlook.

  > [!TIP]
  > Если у вас есть личная [учетная запись Майкрософт](https://account.microsoft.com/account), у вас также есть и учетная запись Outlook.com, а если рабочая или учебная учетная запись Azure — то учетная запись Office 365 Outlook.

* Подключение к RSS-каналу веб-сайта. В этом примере используется [RSS-канал популярных новостей веб-сайта Reuters](http://feeds.reuters.com/reuters/topNews): `http://feeds.reuters.com/reuters/topNews`.

Для работы с этим кратким руководством не требуется писать код, но Logic Apps поддерживает другие сценарии, использующие код, например выполнение кода из приложения логики с помощью [службы "Функции Azure"](../azure-functions/functions-overview.md).

## <a name="create-a-blank-logic-app"></a>Создание пустого приложения логики 

1. Войдите на [портал Azure](https://portal.azure.com "Портал Azure"). 

2. В главном меню Azure выберите **Создать** > **Enterprise Integration** (Интеграция Enterprise) > **Приложение логики**.

   ![Портал Azure, "Создать", "Интеграция с предприятием", "Приложение логики"](./media/logic-apps-create-a-logic-app/azure-portal-create-logic-app.png)

3. Создайте приложение логики с параметрами в таблице под этим рисунком.

   ![Ввод данных приложения логики](./media/logic-apps-create-a-logic-app/logic-app-settings.png)

   | Настройка | Рекомендуемое значение | Описание | 
   | ------- | --------------- | ----------- | 
   | **Имя** | *имя_вашего_приложения_логики* | Укажите уникальное имя приложения логики. | 
   | **Подписка** | *Имя вашей подписки Azure* | Выберите подписку Azure, которую нужно использовать. | 
   | **Группа ресурсов** | *Имя вашей группы ресурсов Azure* | Создайте [группу ресурсов Azure](../azure-resource-manager/resource-group-overview.md) для этого приложения логики, а также организуйте все ресурсы, связанные с этим приложением. | 
   | **Расположение** | *Ваш регион центра обработки данных Azure* | Выберите регион центра обработки данных для развертывания приложения логики, например западная часть США. | 
   | **Служба Log Analytics** | Отключить | Включите ведение журнала диагностики для приложения логики, но для этого краткого руководства оставьте параметр **Выкл**. | 
   |||| 

4. Когда вы будете готовы, установите флажок **Закрепить на панели мониторинга**. Таким образом приложение логики автоматически отобразится на панели мониторинга Azure и откроется после развертывания. Выберите **Создать**.

   > [!NOTE]
   > Если вы не хотите закреплять свое приложение логики, необходимо вручную найти и открыть его после развертывания, чтобы продолжить.

   После развертывания приложения логики в Azure открывается конструктор Logic Apps и отображается страница с вводным видео. 
   Под видео можно найти общие шаблоны приложений логики. 
   В этом кратком руководстве вы создадите приложение логики с нуля. 

5. Прокрутите вниз после вводного видео и общих триггеров. В разделе **Шаблоны** выберите **Пустое приложение логики**.

   ![Выбор шаблона пустого приложения логики](./media/logic-apps-create-a-logic-app/choose-logic-app-template.png)

   В конструкторе Logic Apps отображаются доступные соединители и их триггеры, которые используются для запуска рабочего процесса приложения логики.

   ![Триггеры приложений логики](./media/logic-apps-create-a-logic-app/logic-app-triggers.png)

## <a name="add-a-trigger-to-detect-new-items"></a>Добавление триггера для обнаружения новых элементов

Все рабочие процессы приложений логики начинаются с [триггера](../logic-apps/logic-apps-what-are-logic-apps.md#logic-app-concepts). Триггер срабатывает, когда происходит определенное событие или когда новые данные соответствуют заданному условию. При каждом срабатывании триггера обработчик Logic Apps создает экземпляр приложения логики, запускающий и выполняющий рабочий процесс.

1. В поле поиска введите слово rss в качестве фильтра. Выберите этот триггер: **RSS - When a feed item is published** (RSS — при публикации элемента веб-канала). 

   ![Выбор триггера RSS - When a feed item is published (RSS — при публикации элемента веб-канала)](./media/logic-apps-create-a-logic-app/rss-trigger.png)

2. Укажите ссылку для RSS-канала, который требуется отслеживать, например `http://feeds.reuters.com/reuters/topNews`. Задайте интервал повторения и единицу времени. В этом примере веб-канал проверяется каждые пять минут.

   ![Укажите RSS-канал, частоту и интервал триггера.](./media/logic-apps-create-a-logic-app/rss-trigger-setup.png)

   Logic Apps подключается к RSS-каналу.

   > [!TIP]
   > Чтобы упростить просмотр в конструкторе, можно сворачивать и скрывать детали фигуры. Просто щелкните внутри ее заголовка.

3. Сохраните результаты своих действий. На панели инструментов конструктора нажмите кнопку **Сохранить**. 

   ![Сохранение приложения логики](./media/logic-apps-create-a-logic-app/save-logic-app.png)

   Теперь приложение логики активно, но оно только проверяет RSS-канал. Итак, давайте добавим действие, отвечающее при срабатывании триггера.

## <a name="add-an-action-to-send-email"></a>Добавление действия для отправки электронной почты

Теперь, когда у вас есть [триггер](../logic-apps/logic-apps-what-are-logic-apps.md#logic-app-concepts), добавьте действие, которое отправляет сообщение электронной почты, когда новый элемент появляется в веб-канале RSS. Рабочий процесс выполнит это действие после срабатывания триггера.

1. В конструкторе Logic Apps выберите для созданного триггера пункты **+Новый шаг** > **Добавить действие**.

   ![Добавление действия](./media/logic-apps-create-a-logic-app/add-new-action.png)

   В конструкторе отображаются действия, которые приложение логики может выполнить после срабатывания триггера.

   ![Выбор из списка действий](./media/logic-apps-create-a-logic-app/logic-app-actions.png)

2. В поле поиска введите "отправка сообщения" в качестве условия фильтра. Найдите и выберите соединитель отправки электронной почты, который вы хотите использовать. Затем выберите действие "отправить сообщение" для соединителя. Например: 

   * Для рабочих или учебных учетных записей Azure выберите Office 365 Outlook. 
   * Для личных учетных записей Майкрософт выберите Outlook.com. 
   * Для учетных записей Gmail выберите Gmail. 

   В этом кратком руководстве используется Office 365 Outlook. 
   Если вы используете другой поставщик электронной почты, то действия будут аналогичными, но пользовательский интерфейс может отличаться. 

   ![Выберите действие Office 365 Outlook - Send an email (Office 365 Outlook — отправка сообщения электронной почты).](./media/logic-apps-create-a-logic-app/actions.png)

3. При появлении запроса на ввод учетных данных укажите имя пользователя и пароль своей учетной записи электронной почты. 

   Logic Apps создает подключение к учетной записи электронной почты.

4. Теперь укажите данные, которые требуется включить в сообщение электронной почты. 

   1. В поле **Кому** введите адрес электронной почты получателя. 
   Для тестировании можете использовать свой собственный адрес.

   2. В поле **Тема** введите тему сообщения. 
   Для этого примера введите "Новый элемент RSS", как показано:

      ![Ввод темы сообщения](./media/logic-apps-create-a-logic-app/logic-app-add-subject.png)

      При щелчке в поле ввода откроется список **Добавить динамическое содержимое**, в котором можно выбрать доступные поля данных для включения в действие. 
      Если список динамического содержимого не открывается, под соответствующим полем редактирования выберите **Добавить динамическое содержимое**.

   3. Из списка **Добавить динамическое содержимое** выберите **название канала**, которое содержит заголовок элемента в сообщении электронной почты.

      ![Ввод темы сообщения](./media/logic-apps-create-a-logic-app/logic-app-select-field.png)

      В конечном итоге тема сообщения будет выглядеть примерно так:

      ![Добавленное название веб-канала](./media/logic-apps-create-a-logic-app/added-feed-title.png)

      > [!NOTE] 
      > Если вы выбрали поле с массивом, например **categories-item**, конструктор автоматически добавляет цикл For each для действия, которое ссылается на это поле. Таким образом приложение логики будет выполнять это действие для каждого элемента массива. 
      > 
      > Чтобы удалить цикл, нажмите кнопку с многоточием (**...**) на панели заголовка цикла, а затем выберите **Удалить**.

   4. В поле **Текст** введите текст сообщения электронной почты. 
   Для этого примера введите следующий текст и выберите следующие поля:

      ![Добавление содержимого в тело сообщения электронной почты](./media/logic-apps-create-a-logic-app/logic-app-complete.png)

      | Поле | Описание | 
      | ----- | ----------- | 
      | **Название веб-канала** | Отображение заголовка элемента. | 
      | **Веб-канал опубликован** | Отображение времени и даты публикации элемента. | 
      | **Ссылка основного веб-канала** | Отображение URL-адреса для элемента. | 
      ||| 

      > [!TIP]
      > Чтобы добавить пустые строки в поле редактирования, нажмите Shift + ВВОД. 
      
5. Сохраните результаты своих действий. На панели инструментов конструктора нажмите кнопку **Сохранить**.

   ![Готовое приложение логики](./media/logic-apps-create-a-logic-app/save-complete-logic-app.png)

## <a name="run-your-logic-app-workflow"></a>Выполнение рабочего процесса приложения логики

Чтобы вручную запустить приложение логики, на панели инструментов конструктора щелкните **Запустить**. В противном случае можно дождаться запуска приложения логики по настроенному расписанию.

![Запуск приложения логики](./media/logic-apps-create-a-logic-app/run-complete-logic-app.png)

Если в RSS-канале есть новые элементы, приложение логики отправляет сообщение электронной почты для каждого нового элемента. Например, ниже приведен образец электронного письма Outlook, которое отправляет это приложение логики:

![Отправленное сообщение о новом элементе веб-канала RSS](./media/logic-apps-create-a-logic-app/rss-feed-email.png)

Если в веб-канале нет новых элементов, приложение логики пропускает шаг, который отправляет сообщение, и ожидает следующего интервала для повторной проверки. 

> [!TIP]
> Если сообщения электронной почты не приходят, проверьте папку нежелательной почты. В противном случае, если вы не уверены, что приложение логики работает правильно, см. статью [Диагностика сбоев приложений логики](../logic-apps/logic-apps-diagnosing-failures.md).

Поздравляем, вы только что создали и запустили свое первое приложение логики. Из этого краткого руководства вы узнали, как можно легко и быстро создать автоматические рабочие процессы для интеграции системы и служб.

## <a name="clean-up-resources"></a>Очистка ресурсов

Приложение логики по-прежнему работает. С вас перестанет взиматься плата за его использование в подписке Azure только после его отключения или удаления. Кроме того, при создании подключений для приложения логики они сохраняются даже после удаления приложения логики. 

Завершив работу, обязательно отключите или удалите все ненужные ресурсы, за использование которых не имеете намерения платить. Чтобы удалить все ресурсы, созданные при работе с кратким руководством, удалите группу ресурсов Azure, созданную для этого приложения логики. 

### <a name="delete-resource-group"></a>Удалить группу ресурсов

Если вы не хотите сохранять данные, связанные с приложением логики, удалите группу ресурсов, созданную для этого краткого руководства, и все связанные ресурсы. Узнайте больше об [управлении группами ресурсов Azure](../azure-resource-manager/resource-group-portal.md#manage-resources).

1. В меню Azure выберите **Группы ресурсов**.

2. Выберите группу ресурсов, которую нужно удалить. В меню группы ресурсов выберите **Обзор**, если он еще не выбран. 

3. Просмотрите все ресурсы в группе, которые требуется удалить. Когда вы будете готовы, щелкните **Удалить группу ресурсов** на панели инструментов группы ресурсов.

### <a name="turn-off-logic-app"></a>Отключение приложения логики

Чтобы остановить работу приложения логики, не удаляя данные, отключите приложение. 

В меню приложения логики щелкните **Обзор**. На панели инструментов щелкните **Отключить**.

  ![Отключение приложения логики.](./media/logic-apps-create-a-logic-app/turn-off-disable-logic-app.png)

  > [!TIP]
  > Если меню приложения логики не отображается, попробуйте вернуться к панели мониторинга Azure и повторно откройте приложение логики.

### <a name="delete-logic-app"></a>Удаление приложения логики

Вы можете удалить только приложение логики и сохранить остальные связанные ресурсы, такие как созданные соединения.

1. В меню приложения логики щелкните **Обзор**. На панели инструментов щелкните **Отключить**. 

   ![Удаление приложения логики](./media/logic-apps-create-a-logic-app/delete-logic-app.png)

   > [!TIP]
   > Если меню приложения логики не отображается, попробуйте вернуться к панели мониторинга Azure и повторно откройте приложение логики.

2. Подтвердите удаление приложения логики и выберите **Удалить**.

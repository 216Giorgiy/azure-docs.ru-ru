# <a name="create-your-first-logic-app-for-automating-workflows-and-processes-through-the-azure-portal"></a>Создание первого приложения логики для автоматизации рабочих процессов и процессов с помощью портала Azure

Вы можете интегрировать системы и службы путем создания и выполнения автоматизированных рабочих процессов с помощью [Azure Logic Apps](../logic-apps/logic-apps-what-are-logic-apps.md). При этом совершенно не нужно прибегать к написанию кода. Чтобы показать, как легко можно автоматизировать задачи с помощью рабочего процесса, в этом руководстве создается базовое приложение логики, которое проверяет веб-канал RSS на наличие нового содержимого на веб-сайте и отправляет сообщение о каждом новом элементе в веб-канале. 

![Обзор. Первый пример приложения логики](./media/logic-apps-create-a-logic-app/logic-app-overview.png)

Из этого руководства вы узнаете, как выполнять такие задачи:

> [!div class="checklist"]
> * Создание пустого приложения логики.
> * Добавление триггера для запуска приложения логики при публикации элемента в веб-канале RSS.
> * Добавление действия для отправки сообщения с подробными сведениями об элементе веб-канала RSS.
> * Выполнение и тестирование приложения логики.

## <a name="prerequisites"></a>Предварительные требования

* Подписка Azure. Если у вас нет подписки, вы можете [создать бесплатную пробную версию учетной записи Azure](https://azure.microsoft.com/free/). В противном случае вы можете [зарегистрироваться на получение подписки с оплатой по мере использования](https://azure.microsoft.com/pricing/purchase-options/).

* Учетная запись электронной почты [любого поставщика электронной почты, поддерживаемого Azure Logic Apps](../connectors/apis-list.md) для отправки уведомлений. Например, Office 365 Outlook, Outlook.com, Gmail или любого другого поддерживаемого поставщика. В этом руководстве используется Office 365 Outlook.

  > [!TIP]
  > Если у вас есть личная [учетная запись Майкрософт](https://account.microsoft.com/account), у вас также есть и учетная запись Outlook.com, а если рабочая или учебная учетная запись Azure — то учетная запись Office 365 Outlook.

* Подключение к RSS-каналу веб-сайта. В этом примере используется [RSS-канал популярных новостей веб-сайта CNN.com](http://rss.cnn.com/rss/cnn_topstories.rss): `http://rss.cnn.com/rss/cnn_topstories.rss`.

## <a name="1-create-a-blank-logic-app"></a>1. Создание пустого приложения логики 

1. Войдите на [портал Azure](https://portal.azure.com "Портал Azure").

2. В главном меню Azure выберите **Создать** > **Enterprise Integration** (Интеграция Enterprise) > **Приложение логики**.

   ![Портал Azure, "Создать", "Интеграция с предприятием", "Приложение логики"](media/logic-apps-create-a-logic-app/azure-portal-create-logic-app.png)

3. Создайте приложение логики с параметрами, указанными в следующей таблице.

   ![Ввод данных приложения логики](./media/logic-apps-create-a-logic-app/logic-app-settings.png)

   | Настройка | Рекомендуемое значение | Описание | 
   | ------- | --------------- | ----------- | 
   | **Имя** | *имя_вашего_приложения_логики* | Укажите уникальное имя приложения логики. | 
   | **Подписка** | *ваша_подписка_Azure* | Выберите подписку Azure, которую нужно использовать. | 
   | **Группа ресурсов** | *ваша_группа_ресурсов_Azure* | Создайте или выберите группу ресурсов Azure, что поможет вам упорядочить связанные ресурсы Azure и управлять ими. | 
   | **Расположение** | *ваш_регион_Azure* | Выберите регион центра обработки данных для развертывания приложения логики. | 
   |||| 

4. По завершении установите флажок **Закрепить на панели мониторинга**, а затем нажмите кнопку **Создать**.

   Вы создали ресурс Azure для приложения логики. 
   После того, как Azure развернет приложение логики, конструктор Logic Apps отобразит несколько распространенных шаблонов, чтоб вы могли быстрее приступить к работе.

   > [!NOTE] 
   > Когда вы устанавливаете флажок **Закрепить на панели мониторинга**, после развертывания приложение логики появляется на панели мониторинга и автоматически открывается в Logic Apps Designer. В противном случае вы можете вручную найти и открыть свое приложение логики.

5. Сейчас в разделе **Шаблоны** выберите **Пустое приложение логики**. Это позволит создать приложение логики с нуля.

   ![Выбор шаблона приложения логики](./media/logic-apps-create-a-logic-app/choose-logic-app-template.png)

   В конструкторе Logic Apps теперь отображаются доступные [*соединители*](../connectors/apis-list.md) и их [*триггеры*](../logic-apps/logic-apps-what-are-logic-apps.md#logic-app-concepts), которые используются для запуска рабочего процесса приложения логики.

   ![Триггеры приложений логики](./media/logic-apps-create-a-logic-app/logic-app-triggers.png)

## <a name="2-add-a-trigger-for-starting-the-workflow"></a>2. Добавление триггера для запуска рабочего процесса

Каждое приложение логики должно запускаться с помощью [*триггера*](../logic-apps/logic-apps-what-are-logic-apps.md#logic-app-concepts). Триггер срабатывает, когда происходит определенное событие или когда новые данные соответствуют заданному условию. После этого обработчик Logic Apps создает экземпляр приложения логики для выполнения рабочего процесса. Каждый раз, когда триггер срабатывает, обработчик создает другой отдельный экземпляр, который выполняет рабочий процесс приложения логики.

1. В поле поиска введите "rss" в качестве условия фильтрации. Выберите этот триггер: **RSS - When a feed item is published** (RSS — при публикации элемента веб-канала). 

   ![Выбор триггера RSS - When a feed item is published (RSS — при публикации элемента веб-канала)](./media/logic-apps-create-a-logic-app/rss-trigger.png)

2. Введите ссылку на веб-канал RSS веб-сайта, который вы хотите отслеживать, например `http://rss.cnn.com/rss/cnn_topstories.rss`. Задайте интервал повторения и единицу времени. В этом примере задайте проверку веб-канала каждый день. 

   ![Укажите RSS-канал, частоту и интервал триггера.](./media/logic-apps-create-a-logic-app/rss-trigger-setup.png)

3. Сохраните результаты. На панели инструментов конструктора нажмите кнопку **Сохранить**.
Чтобы свернуть и скрыть сведения о триггере, выберите заголовок триггера.

   ![Сохранение приложения логики](./media/logic-apps-create-a-logic-app/save-logic-app.png)

   Теперь приложение логики активно, но не будет выполнять никаких действий, кроме проверки на наличие новых элементов в веб-канале RSS, пока они не будут добавлены в рабочий процесс. 

## <a name="3-add-an-action-that-responds-to-the-trigger"></a>3. Добавление действия, которое будет реагировать на триггер

Теперь добавьте [*действие*](../logic-apps/logic-apps-what-are-logic-apps.md#logic-app-concepts), которое представляет собой задачу, выполняемую рабочим процессом приложения логики. В этом примере добавьте действие, которое отправляет сообщение электронной почты, когда новый элемент появляется в веб-канале RSS.

1. В конструкторе Logic Apps выберите для созданного триггера пункты **+Новый шаг** > **Добавить действие**.

   ![Добавление действия](./media/logic-apps-create-a-logic-app/add-new-action.png)

   В конструкторе отображаются [доступные соединители](../connectors/apis-list.md). Вы можете выбрать действие, которое будет выполняться при активации триггера.

   ![Выбор из списка действий](./media/logic-apps-create-a-logic-app/logic-app-actions.png)

2. В поле поиска введите "отправка сообщения" в качестве условия фильтра. Найдите и выберите соединитель, соответствующий поставщику услуг электронной почты. Затем выберите действие "отправить сообщение" для соединителя. Например: 

   * Если у вас рабочая или учебная учетная запись, то выберите соединитель Office 365 Outlook. 
   * Для личных учетных записей Майкрософт выберите соединитель Outlook.com. 
   * Для учетных записей Gmail выберите соединитель Gmail. 

   Мы собираемся использовать соединитель Office 365 Outlook. 
   Если вы используете другой поставщик, то действия будут аналогичными, но пользовательский интерфейс может отличаться. 

   ![Выберите действие Office 365 Outlook - Send an email (Office 365 Outlook — отправка сообщения электронной почты).](./media/logic-apps-create-a-logic-app/actions.png)

3. При появлении запроса на ввод учетных данных укажите имя пользователя и пароль своей учетной записи электронной почты. 

4. Предоставьте сведения, указанные в таблице, и выберите поля, которые должны быть включены в сообщение электронной почты.

   | Кому | Действия | 
   | -- | ----- | 
   | Выберите поля, доступные в рабочем процессе. | Щелкните в поле ввода, чтобы открыть список **Dynamic content** (Динамическое содержимое), или выберите **Add dynamic content** (Добавить динамическое содержимое). | 
   | Просмотрите другие доступные поля. | В списке **Dynamic content** (Динамическое содержимое) выберите **Просмотреть больше** для каждого раздела.  | 
   | Добавьте пустые строки в поле ввода. | Нажмите клавиши SHIFT+ВВОД. | 
   | Закройте список **Dynamic content** (Динамическое содержимое). | Выберите **Add dynamic content** (Добавить динамическое содержимое) еще раз. | 
   ||| 

   ![Выбор данных для включения в сообщение электронной почты](./media/logic-apps-create-a-logic-app/rss-action-setup.png)

   | Настройка | Рекомендуемое значение | Описание | 
   | ------- | --------------- | ----------- | 
   | **To** | *электронный_адрес_получателя* | Введите электронный адрес получателя. Для тестировании можете использовать свой собственный адрес. | 
   | **Тема** | Новое сообщение CNN: **Название веб-канала** | Введите содержимое темы электронного сообщения. <p>В этом руководстве введите предложенный текст и выберите поле **Название веб-канала** триггера, в котором показан заголовок элемента веб-канала. | 
   | **Текст** | Заголовок: **Название веб-канала** <p>Дата публикации: **Дата публикации в веб-канале** <p>Ссылка: **Ссылка на веб-канал** | Введите содержимое текста сообщения электронной почты. <p>В рамках этого руководства введите предложенный текст и выберите такие поля триггера: <p>- **Название веб-канала** — повторяется название элемента веб-канала. </br>- **Дата публикации в веб-канале** — дата и время публикации элемента. </br>- **Ссылка на веб-канал** — URL-адрес элемента веб-канала. | 
   |||| 

   > [!NOTE] 
   > Если выбрать поле, которое содержит массив, конструктор автоматически добавит цикл For each для действия, которое ссылается на массив. Таким образом приложение логики будет выполнять это действие для каждого элемента массива.

5. Сохраните изменения, когда закончите. На панели инструментов конструктора нажмите кнопку **Сохранить**.

   ![Готовое приложение логики](./media/logic-apps-create-a-logic-app/save-complete-logic-app.png)

   Чтобы проверить приложение логики, перейдите следующему разделу.

## <a name="4-run-and-test-your-workflow"></a>4. Выполнение и тестирование рабочего процесса

1. Чтобы вручную запустить приложение логики для тестирования, на панели инструментов конструктора щелкните **Запустить**. Кроме того, приложение логики может проверить указанный RSS-канал на основе заданного расписания.

   ![Запуск приложения логики](./media/logic-apps-create-a-logic-app/run-complete-logic-app.png)

   Обнаружив новый элемент, приложение логики отправляет сообщение электронной почты с выбранными данными, например:

   ![Отправленное сообщение о новом элементе веб-канала RSS](./media/logic-apps-create-a-logic-app/rss-feed-email.png)

   Если приложение логики не находит новые элементы, оно пропускает действие, которое отправляет сообщение, и ожидает следующего интервала для повторной проверки. 

2. Чтобы просмотреть журнал триггера и выполнения приложения логики, в меню приложения выберите **Обзор**.
Чтобы просмотреть дополнительные сведения о выполнении, выберите строку для этого выполнения.

   ![Мониторинг и просмотр журнала триггера и выполнения приложения логики](./media/logic-apps-create-a-logic-app/logic-app-run-trigger-history.png)

   > [!TIP]
   > Если вы не нашли ожидаемые данные, на панели инструментов щелкните **Обновить**.

   В представлении сведений о выполнении показаны выполненные или неудачные действия независимо о того, было ли выполнение успешным. 

   ![Просмотр сведений о выполнении приложения логики](./media/logic-apps-create-a-logic-app/logic-app-run-details.png)

   Дополнительные сведения о состоянии, журнале триггера и выполнения, а также о диагностике приложения логики см. в статье [Диагностика сбоев приложений логики](../logic-apps/logic-apps-diagnosing-failures.md).

3. Чтобы просмотреть входные и выходные данные каждого действия, разверните действие, которое необходимо просмотреть. Эти сведения помогут вам диагностировать и устранить проблемы в приложении логики. Например:

   ![Просмотр сведений о действии](./media/logic-apps-create-a-logic-app/logic-app-run-details-expanded.png)

   Дополнительные сведения см. в статье [Мониторинг состояния, настройка ведения журнала диагностики и включение предупреждений для Azure Logic Apps](../logic-apps/logic-apps-monitor-your-logic-apps.md).

Поздравляем, вы только что создали и запустили свое первое базовое приложение логики. Вы также узнали, как можно легко создать рабочие процессы автоматизации для интеграции систем и служб без написания кода.

> [!NOTE]
> Приложение логики продолжает работать до отключения приложения. Чтобы временно отключить приложение, перейдите к следующему разделу.

## <a name="clean-up-resources"></a>Очистка ресурсов

В этом руководстве используются ресурсы и выполняются действия, которые могут подлежать оплате по подписке Azure. Завершив проверку и работу с руководством, обязательно отключите или удалите все ресурсы, за использование которых не имеете намерения платить.

Вы можете остановить выполнение приложения логики и отправку электронных сообщений, не удаляя приложение. В меню приложения логики щелкните **Обзор**. На панели инструментов щелкните **Отключить**.

![Отключение приложения логики.](./media/logic-apps-create-a-logic-app/turn-off-disable-logic-app.png)

## <a name="faq"></a>Часто задаваемые вопросы

**В.** Что еще можно делать с приложением логики? </br>
**О.** Вы можете выполнять и другие задачи, например изменять, просматривать определение JSON, проверять журнал действий и удалять приложение логики.

Чтобы найти другие задачи управления в приложении логики, просмотрите эти команды в меню приложения логики.

| Задача | Действия | 
| ---- | ----- | 
| Просмотр состояния, журнала триггера и выполнений приложения, а также общих сведений о нем | Выберите **Обзор**. | 
| Изменение приложения | Выберите **Конструктор приложений логики**. | 
| Просмотр определения JSON рабочего процесса приложения | Выберите **Представление кода приложения логики**. | 
| Просмотр выполненных в приложении логики операций | Выберите **Журнал действий**. | 
| Просмотр предыдущих версий приложения логики | Выберите **Версии**. | 
| Временное отключение приложения | Выберите **Обзор**, а затем на панели инструментов щелкните **Отключить**. | 
| Удаление приложения | Выберите **Обзор**, а затем на панели инструментов щелкните **Удалить**. Введите имя приложения логики и выберите **Удалить**. |

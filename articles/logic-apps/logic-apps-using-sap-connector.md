# <a name="use-the-sap-connector-in-your-logic-app"></a>Использование соединителя SAP в приложении логики 

Локальный шлюз данных позволяет управлять данными и безопасно обращаться к ресурсам, расположенным в локальной среде. В этой статье мы подключимся к локальной системе SAP, запросим IDOC по протоколу HTTP и отправим ответ на этот запрос.    

 > [!NOTE]
> Текущие ограничения:
 > - Logic Apps прерывает соединение, если время обработки запроса превышает 90 секунд. В этом случае запросы могут быть заблокированы. 
 > - Средство выбора файлов не отображает все доступные поля. В этом случае пути можно добавить вручную.

## <a name="prerequisites"></a>Предварительные требования
- Установите и настройте последнюю версию [локального шлюза данных](https://www.microsoft.com/download/details.aspx?id=53127) или версию не ниже 1.15.6150.1. Инструкция см. в статье [Подключение к локальному шлюзу данных для приложений логики](http://aka.ms/logicapps-gateway). Шлюз нужно установить на локальном компьютере, прежде чем продолжать процесс.

- Скачайте последнюю версию клиентской библиотеки SAP и установите ее на компьютере, на котором установлен шлюз данных. Можно использовать любую из следующих версий SAP: 
    - SAP Server
        - SAP ECC 6.0 Unicode
        - SAP ECC 6.0 Unicode c установленным EHP 4.0
        - SAP ECC 6.0 c установленным EHP 7.0 или EHP любых предыдущих версий
 
    - SAP Client
        - SAP RFC SDK 7.20 UNICODE
        - Соединитель SAP .NET (NCo) 3.0

## <a name="add-the-sap-connector"></a>Добавление соединителя SAP

Соединитель SAP поддерживает действия. У него нет никаких триггеров. Это означает, что для начала рабочего процесса нужно применить другой триггер. 

1. Добавьте триггер запроса и ответа, а затем выберите **Новый шаг**.
2. Выберите **Добавить действие**, а затем выберите нужный соединитель SAP. Для этого введите `SAP` в поле поиска:    
 ![Выберите сервер приложений SAP или сервер сообщений SAP](media/logic-apps-using-sap-connector/picture1.png)

3. Выберите [сервер приложений](https://wiki.scn.sap.com/wiki/display/ABAP/ABAP+Application+Server) **SAP** или [сервер сообщений](http://help.sap.com/saphelp_nw70/helpdata/en/40/c235c15ab7468bb31599cc759179ef/frameset.htm) в зависимости от конфигурации SAP. Если вы не установили подключение, вам будет предложено создать его: 
    1. Выберите **подключение через локальный шлюз данных** и введите сведения о системе SAP:   
 ![Добавление строки подключения к SAP](media/logic-apps-using-sap-connector/picture2.png)  
    2. Выберите существующий **шлюз**. Или выберите вариант **Установить шлюз**, чтобы установить новый шлюз:    
 ![Установка нового шлюза](media/logic-apps-using-sap-connector/install-gateway.png)
  
    3. Введите все нужные сведения и нажмите **Создать**. Logic Apps настроит и проверит подключение, чтобы оно правильно работало.

4. Введите имя для подключения SAP.

5. Теперь вам будут доступны параметры SAP. Используйте средство выбора файлов, чтобы найти нужную категорию IDOC. Также можно вручную ввести путь и выбрать ответ HTTP в поле **body** (текст):     
 ![Действие SAP](media/logic-apps-using-sap-connector/picture3.png)

6. Создайте ответ HTTP, добавив новое действие. Сообщение ответа должно передавать выходные данные SAP.

7. Сохраните приложение логики. Проверьте его, отправив IDOC с помощью URL-адреса триггера HTTP. После отправки IDOC дождитесь ответа от приложения логики:   

  > [!TIP]
  > Узнайте о возможностях [мониторинга приложений логики](../logic-apps/logic-apps-monitor-your-logic-apps.md).

Теперь соединитель SAP добавлен в приложение логики, и вы можете ознакомиться с другими функциями.

  - BAPI
  - RFC

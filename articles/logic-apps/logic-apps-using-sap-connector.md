# <a name="get-started-with-the-sap-connector"></a>Начало работы с соединителем SAP 

Гибридные подключения к облаку — основа Logic Apps. Локальный шлюз данных позволяет управлять данными и безопасно обращаться к ресурсам, которые являются локальными для Logic Apps. В этой статье мы покажем, как подключиться к локальной системе SAP в простом сценарии: запрос IDOC по протоколу HTTP, а затем отправка ответа.    

 > [!NOTE]
 > Этот соединитель SAP поддерживает следующие системы SAP. В данный момент действует ограничение: если время ожидания запроса в Logic Apps превышает 90 секунд, то запрос блокируется. Также действует ограничение на количество полей, отображаемых в средстве выбора файлов (пути можно добавлять вручную).
 >
 >

## <a name="prerequisites"></a>Предварительные требования
- Установите и настройте последнюю версию [локального шлюза данных](https://www.microsoft.com/en-us/download/details.aspx?id=53127).  

    Установите последнюю версию локального шлюза (не ниже версии 1.15.6150.1), если это еще не сделано. Указания см. в [этой статье](http://aka.ms/logicapps-gateway). Шлюз нужно установить на локальном компьютере, прежде чем можно будет продолжить процедуру.

- Скачайте последнюю версию клиентской библиотеки SAP и установите ее на том же компьютере, где установлен универсальный шлюз.

## <a name="use-sap-connector"></a>Использование соединителя SAP

1. Давайте создадим триггер HTTP-запросов, а затем выберем действие соединителя SAP, введя в поле поиска "SAP".    
 ![Поиск SAP](media/logic-apps-using-sap-connector/picture1.png)

2. Выберите узел SAP (ApplicationHost или MessagingHost, в зависимости от настроек SAP) и создайте для него подключение, используя универсальный шлюз.
 - Если у вас нет существующего подключения, то вам будет предложено создать его.
 - Установите флажок "Connect via on-premises data gateway" (Подключение через локальный шлюз данных). Отобразятся дополнительные поля.
 - Указание имени строки подключения
 - Выберите шлюз, установленный на предыдущем шаге, или выберите "Install Gateway" (Установить шлюз), чтобы установить новый шлюз.   
 ![Добавление строки подключения к SAP](media/logic-apps-using-sap-connector/picture2.png)   
  
  > [!NOTE]
  > Существует два различных способа подключения к SAP. Один используется для [узла приложений](https://wiki.scn.sap.com/wiki/display/ABAP/ABAP+Application+Server), а другой — для [узла обмена сообщениями](http://help.sap.com/saphelp_nw70/helpdata/en/40/c235c15ab7468bb31599cc759179ef/frameset.htm).
  >
  >

3. После ввода всех сведений щелкните "Создать". Logic Apps настроит и проверит подключение, чтобы убедиться в том, что оно работает правильно. Если все проверки пройдены, то отобразятся параметры для карты, выбранной ранее. С помощью средства выбора файлов найдите необходимую категорию IDOC или введите путь вручную, а затем выберите HTTP-ответ в поле "body" (текст).    
 ![ДЕЙСТВИЕ SAP](media/logic-apps-using-sap-connector/picture3.png)

4. Создайте HTTP-ответ, добавив новое действие. Ответное сообщение должно быть из выходных данных SAP.

5. Сохраните приложение логики и проверьте его, отправив IDOC, используя URL-адрес триггера HTTP.

6. После отправки IDOC подождите ответа от приложения логики.   

  > [!TIP]
  > Узнайте о возможностях [мониторинга приложений логики](../logic-apps/logic-apps-monitor-your-logic-apps.md).
  >
  >

7. Все готово, теперь у вас есть рабочее приложение логики, использующее соединитель SAP. Вы можете приступить к изучению других предоставляемых возможностей.
  - BAPI
  - RFC

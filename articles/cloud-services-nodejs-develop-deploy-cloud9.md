# Развертывание приложения Azure из Cloud9

В этом учебном курсе рассматривается разработка, построение
и развертывание приложения Node.js в Azure для Cloud9 IDE.

Из этого учебного курса вы узнаете следующее:

-   Создание проекта интегрированной среды разработки Cloud9
-   Развертывание проекта в Azure
-   Обновление существующего развертывания Azure
-   Перемещение проектов из развертывания промежуточного хранения в рабочее развертывание

[Cloud9 IDE][Cloud9 IDE] предоставляет межплатформенную среду разработки
на основе веб-браузера. Одна из возможностей, поддерживаемых Cloud9 для проектов
на Node.js, — развертывание Azure непосредственно в интегрированной среде разработки.
Cloud9 также интегрируется со службами репозитория GitHub и BitBucket, что позволяет легко обмениваться
данными проекта с другими пользователями.

При помощи Cloud9 можно разрабатывать и развертывать приложения
в Azure из многих современных браузеров и операционных систем
без необходимости локально устанавливать средства разработки или SDK. Приведенные ниже
действия относятся к Google Chrome на Макинтоше.

## Регистрация

Чтобы использовать Cloud9, сначала необходимо посетить их веб-сайт и [зарегистрироваться
для подписки][Cloud9 IDE]. Можно выполнить вход под любой существующей
учетной записью GitHub или BitBucket либо создать учетную запись Cloud9. Доступна
как бесплатная подписка, так и платная версия, которая отличается
дополнительными функциональными возможностями. Дополнительные сведения см. в разделе [Интегрированная среда разработки Cloud9][Cloud9 IDE].

## Создание проекта Node.js

1.  Выполните вход в Cloud9, щелкните по значку **+** рядом с узлом **Мои проекты**
    и выберите **Создать проект**.

    ![создание проекта Cloud9][создание проекта Cloud9]

2.  В диалоговом окне **Создание проекта** введите имя проекта,
    параметры доступа и тип проекта. Нажмите кнопку **Создать**, чтобы создать проект.

    ![диалоговое окно Cloud9 для создания проекта][диалоговое окно Cloud9 для создания проекта]

    <div class="dev-callout">

    **Примечание.**
    Некоторые параметры требуют платной подписки Cloud9.

    </div>

    <div class="dev-callout">

    **Примечание.**
    Имя вашего проекта Cloud9 не используется при развертывании в Azure.

    </div>

3.  После создания проекта нажмите кнопку **Начать редактирование**. Если интегрированная среда разработки Cloud9 используется в первый раз, будет предложено пройти обучение работе с этой службой. Если вы хотите пропустить этот шаг и вернуться к нему позже, выберите **Только редактор**.

    ![начало редактирования проекта Cloud9][начало редактирования проекта Cloud9]

4.  Для создания нового приложения Node выберите **Файл** и **Создать
    файл**.

    ![создание файла в проекте Cloud9][создание файла в проекте Cloud9]

5.  Будет открыта новая вкладка под названием **Untitled1**. Введите
    следующий код на вкладке **Untitled1**, чтобы создать
    приложение Node:

        var http = require('http');
        var port = process.env.PORT;
        http.createServer(function(req,res) {
            res.writeHead(200, { 'Content-Type': 'text/plain' });
            res.end('hello azure\n');
        }).listen(port);

    <div class="dev-callout">

    **Примечание.**
    Использование process.env.PORT гарантирует, что приложение выберет правильный порт как при запуске в отладчике Cloud9, так и при развертывании в Azure.

    </div>

6.  Чтобы сохранить код, выберите команды **Файл** и **Сохранить как**. В диалоговом окне
    **Сохранить как** введите в качестве имени файла **server.js**, а затем
    нажмите кнопку **Сохранить**.

    <div class="dev-callout">

    **Примечание.**
    Вы можете заметить значок предупреждения, который указывает, что переменная req не используется. Это предупреждение можно без сомнений проигнорировать.

    </div>

    ![сохранение файла server.js][сохранение файла server.js]

## Создание учетной записи Azure

Для развертывания приложения в Azure необходимо иметь учетную запись. Если
у вас еще нет учетной записи Azure, вы можете подписаться
на бесплатную ознакомительную программу, выполнив следующие шаги.

[WACOM.INCLUDE [создание-учетной-записи azure][создание-учетной-записи azure]]

## Создание развертывания

1.  Чтобы создать новое развертывание, выберите **Развернуть**, а затем нажмите кнопку **+** для создания сервера развертывания.

    [создание нового развертывания][создание нового развертывания]

2.  В диалоговом окне **Добавление целевого объекта развертывания** введите имя развертывания и выберите **Azure** в списке **Выберите тип**. Это имя развертывания будет использоваться для идентификации развертывания в Cloud9; оно не будет соответствовать имени развертывания в Azure.

3.  При первом создании развертывания Cloud9 с использованием Azure необходимо настроить параметры публикации в Azure. Чтобы загрузить и установить эти параметры в Cloud9, выполните следующие действия:

    1.  Нажмите кнопку **Загрузить параметры Azure**.

        ![загрузка параметров публикации][загрузка параметров публикации]

        Будет открыт портал управления Azure с приглашением загрузить параметры публикации Azure. Для начала работы потребуется войти в систему под учетной записью Azure.

    2.  Сохраните файл параметров публикации на локальном диске.

    3.  В диалоговом окне **Добавление целевого объекта развертывания** нажмите кнопку **Выбор файла**,
        а затем выберите файл, загруженный в предыдущем действии.

    4.  После выбора файла нажмите кнопку **Загрузить**.

4.  Щелкните **+ Создать**, чтобы создать новую размещенную службу. При развертывании приложения в Azure *размещенная служба* служит контейнером для размещения приложения. Дополнительные сведения см. в разделе [Обзор создания размещенной службы для Azure][Обзор создания размещенной службы для Azure].

    ![создание нового развертывания][создание нового развертывания]

5.  Появится приглашение ввести имя новой размещенной службы и параметры конфигурации, такие как количество экземпляров, ОС хост-компьютера и центр обработки данных. Указанное имя развертывания будет использоваться в Azure как имя размещенной службы. Имя должно быть уникальным в пределах системы Azure.

    ![создание новой размещенной службы][создание новой размещенной службы]

    <div class="dev-callout">

    **Примечание.**
    В диалоговом окне **Добавление целевого объекта развертывания** будут перечислены все существующие размещенные службы Azure в разделе **Выбор существующего развертывания**, при выборе существующей размещенной службы проект будет развернут для этой службы.

    </div>

    <div class="dev-callout">

    **Примечание.**
    При выборе параметра **Включить RDP** с указанием имени пользователя и пароля будет включен удаленный рабочий стол для развертывания.

    </div>

## Развертывание в рабочей среде Azure

1.  Выберите развертывание, созданное в предыдущих действиях. Откроется
    диалоговое окно со сведениями о данном развертывании, а также
    URL-адресом рабочей среды, который будет использоваться после развертывания в Windows
    Azure.

    ![выбор развертывания][выбор развертывания]

2.  Выберите **Развернуть в рабочей среде**.

3.  Нажмите кнопку **Развернуть**, чтобы начать развертывание.

4.  Если развертывание проекта в Azure выполняется в первый раз, будет выведена ошибка **Не найден файл web.config**. Выберите **Да**, чтобы создать этот файл. В результате в проект будет добавлен файл "Web.cloud.config".

    ![сообщение об отсутствии файла web.config][сообщение об отсутствии файла web.config]

5.  Если развертывание проекта в Azure выполняется в первый раз, будет выведена ошибка **Не найден CSDEF-файл**. Выберите **Да**, чтобы создать CSDEF-файл. В результате в проект будет добавлен файл "ServiceDefinition.csdef". Файл "ServiceDefinition.csdef" используется только в Azure для публикации приложения. Дополнительные сведения см. в разделе [Обзор создания размещенной службы для Azure][Обзор создания размещенной службы для Azure].

6.  Появится приглашение выбрать размер экземпляра для этого приложения. Выберите **Малый** и нажмите кнопку **Создать**. Дополнительные сведения о размерах виртуальных машин Azure см. в материале [Практическое руководство по настройке размеров виртуальных машин][Практическое руководство по настройке размеров виртуальных машин].

    ![определение значений csdef-файла][определение значений csdef-файла]

7.  Запись развертывания будет отображать состояние процесса развертывания. После завершения развертывания оно будет отображаться **Активное**

    ![состояние развертывания][состояние развертывания]

    <div class="dev-callout">

    **Примечание.**
    Проектам развертывания через Cloud 9 IDE присваивается идентификатор GUID в качестве имени развертывания в Azure.

    </div>

8.  Диалоговое окно развертывания содержит ссылку на URL-адрес рабочей среды. После завершения развертывания щелкните URL-адрес, чтобы перейти к приложению, выполняющемуся в Azure.

    ![URL-адрес рабочей среды Azure][URL-адрес рабочей среды Azure]

## Обновление приложения

При внесении изменений в приложение можно использовать Cloud9 для развертывания обновленного приложения в той же размещенной службе Azure.

1.  В файле server.js обновите код таким образом, чтобы на экран было выведено "hello Azure v2". Можно заменить существующий код на следующий обновленный код:

        var http = require('http');
        var port = process.env.PORT;
        http.createServer(function(req,res) {
            res.writeHead(200, { 'Content-Type': 'text/plain' });
            res.end('hello azure v2\n');
        }).listen(port);

2.  Чтобы сохранить код, выберите команды **Файл** и **Сохранить**.

## Развертывание обновления в среде промежуточного хранения Azure

1.  Выберите **Развернуть в промежуточное хранилище**.

2.  Нажмите кнопку **Развернуть**, чтобы начать развертывание.

    Каждая размещенная служба Azure поддерживает две среды: промежуточного хранения и рабочую. Промежуточная среда в точности схожа с рабочей средой за исключением того, доступ к приложению в промежуточной среде возможен только с использованием допустимого URL-адреса на основе GUID, который создается Azure. Промежуточная среда служит для тестирования приложения, после проверки изменений можно переместить промежуточную версию в рабочую среду, выполнив подмену виртуального IP (VIP), как описано далее в этом руководстве.

3.  При развертывании приложения в промежуточной среде промежуточный URL-адрес на основе GUID будет отображаться в выходных данных в консоли, как показано на следующем снимке экрана. Щелкните URL-адрес, чтобы открыть промежуточную версию приложения в браузере.

    ![вывод консоли с отображением промежуточного URL-адреса][вывод консоли с отображением промежуточного URL-адреса]

## Перемещение обновления в рабочую среду с помощью подмены VIP

При развертывании службы в рабочей или промежуточной
среде ей назначается один виртуальный IP-адрес (VIP) в данной
среде. Если нужно переместить службу из промежуточной
среды в производственную, это можно сделать без повторного
развертывания, выполнив переключение VIP, в ходе которого
эти среды меняются местами. Переключение VIP позволяет сразу перенести в рабочую
среду проверенное приложение из промежуточной среды. Дополнительные
сведения см. в разделе [Обзор управления развертываниями в Azure][Обзор управления развертываниями в Azure].

1.  В диалоговом окне развертывания щелкните по ссылке **Открыть портал**, чтобы открыть портал управления Azure.

    [Ссылка из диалогового окна развертывания на портал управления Azure][Ссылка из диалогового окна развертывания на портал управления Azure]

2.  Войдите на портал под своими учетными данными.

3.  В левой части веб-страницы выберите **Размещенные службы,
    учетные записи хранения и CDN**, а затем щелкните **Размещенные службы**.

    [Портал управления Azure][Портал управления Azure]

    В области результатов будут отображены размещенная служба с именем, указанным в Cloud9, а также два развертывания, одно из которых имеет в параметре **Среда** значение **Промежуточная**, а второе — **Рабочая**

4.  Для переключения VIP выберите размещенную службу и нажмите кнопку **Переключить VIP** на ленте.

    ![ПЕРЕКЛЮЧИТЬ VIP][ПЕРЕКЛЮЧИТЬ VIP]

5.  Нажмите кнопку **ОК** в открывшемся диалоговом окне "Переключение VIP".

6.  Перейдите в приложение рабочей среды. Вы увидите, что версия приложения, ранее развернутого в промежуточной среде, теперь находится в рабочей среде.

    ![Приложение в рабочей среде на платформе Azure][Приложение в рабочей среде на платформе Azure]

## Использование удаленного рабочего стола

Если при создании развертывания был включен RDP и указаны имя пользователя и пароль, для подключения к размещенной службе можно использовать удаленный рабочий стол, выбрав конкретный экземпляр и нажав кнопку
"Подключиться" на ленте.

![Подключение к экземпляру][Подключение к экземпляру]

При нажатии кнопки "Подключиться" будет предложено открыть или загрузить RDP-файл. Этот файл содержит сведения, необходимые для подключения к сеансу удаленного рабочего стола. При запуске этого файла в Windows система выведет запрос на ввод имени пользователя и пароля, которые были введены при создании развертывания, а затем создаст
подключение с рабочим столом выбранного экземпляра.

<div class="dev-callout">

**Примечание.**
RDP-файл для подключения к размещенному экземпляру приложения будет работать только с приложением удаленного рабочего стола в Windows.

</div>

## Остановка и удаление приложения

Azure предусматривает взимание платы за час использованного рабочего времени, а серверное время потребляется, когда приложение развернуто, даже если экземпляры не выполняются и находятся в состоянии остановки. Кроме того,
серверное время используется и в промежуточном и в рабочем развертываниях.

Cloud9 специализируется на предоставлении интегрированной среды разработки и не предоставляет прямые способы для останова или удаления приложения после его развертывания в Azure. Чтобы удалить приложение, размещенное в Azure, выполните следующие действия:

1.  В диалоговом окне развертывания щелкните по ссылке **Открыть портал**, чтобы открыть портал управления Azure.

    ![Ссылка из диалогового окна развертывания на портал управления Azure][Ссылка из диалогового окна развертывания на портал управления Azure]

2.  Войдите на портал под своими учетными данными.

3.  В левой части веб-страницы выберите **Размещенные службы,
    учетные записи хранения и CDN**, а затем щелкните **Размещенные службы**.

4.  Выберите промежуточную версию развертывания (обозначается значением параметра **Среда**). Чтобы удалить приложение, нажмите кнопку **Удалить** на ленте.

    ![удаление развертывания][удаление развертывания]

5.  Выберите развертывание в рабочей среде и нажмите кнопку **Удалить**, чтобы удалить и это приложение.

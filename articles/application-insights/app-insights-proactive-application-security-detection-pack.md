# <a name="application-security-detection-pack-preview"></a>Пакет обнаружения безопасности приложений (Предварительная версия)

Application Insights автоматически анализирует данные телеметрии, созданные приложением и обнаруживает потенциальные проблемы безопасности. Эта функция позволяет выявлять потенциальные проблемы безопасности и обрабатывать их исправления приложения или отключение меры безопасности.

Этот компонент требует без дополнительной настройки, отличного от [Настройка приложения для отправки телеметрии](https://docs.microsoft.com/en-us/azure/application-insights/app-insights-usage-overview).

## <a name="when-would-i-get-this-type-of-smart-detection-notification"></a>Когда будет получить уведомления смарт-обнаружение такого типа
Существует три типа проблем безопасности, которые обнаруживаются.
1. Небезопасных URL-адресов: URL-адрес в приложении осуществляется через HTTP и HTTPS. Как правило URL-адрес, который принимает HTTPS-запросы не следует принимать HTTP-запросов. Это может означать ошибку проблемы безопасности в приложении.
2. Незащищенной формы: формы (или другого запроса «POST») в приложении используется протокол HTTP вместо HTTPS. С помощью протокола HTTP может поставить под угрозу данные пользователя, который отправляется службой формы.
3. Подозрительные действия пользователя: приложения осуществляется из нескольких странах одним пользователем в примерно то же время. Например тот же пользователь приложения из доступны Испания и Соединенных Штатов в течение одного часа. Обнаружение указывает попытку потенциально небезопасный доступ к приложению.

## <a name="does-my-app-definitely-have-a-security-issue"></a>Мое приложение определенно имеет проблема безопасности?
Нет, уведомление не означает, что в приложении определенно существует проблема безопасности. Определение любого из указанных выше сценариев можно во многих случаях указывать на проблему безопасности. Тем не менее обнаружение могут иметь естественный коммерческое обоснование и можно пропустить.

## <a name="how-do-i-fix-the-insecure-url-access-detection"></a>Как исправить обнаружения «Доступ небезопасных URL-адрес»?
1. **"Рассмотрение".** Число пользователей, доступ к небезопасных URL-адреса и URL-адрес, который предоставляет уведомление затронутых небезопасный доступ. Это поможет назначить приоритет проблемы.
2. **Область действия.** Какой процент пользователей доступ к небезопасных URL-адреса? Сколько URL-адреса были затронуты? Эти сведения можно получить из уведомления.
3. **Выполните диагностику.** Обнаружение предоставляет список небезопасных запросов, а списки URL-адресов и пользователей, которые были затронуты, помогающие диагностировать проблему.

## <a name="how-do-i-fix-the-insecure-form-detection"></a>Как исправить обнаружения «Незащищенной формы»?
1. **"Рассмотрение".** Уведомление содержит число небезопасных форм и количество пользователей, данные которого потенциально был скомпрометирован. Это поможет назначить приоритет проблемы.
2. **Область действия.** Форму, которая была вовлечена в максимальное число передач небезопасен и что такое распределение небезопасных передач со временем? Эти сведения можно получить из уведомления.
3. **Выполните диагностику.** Обнаружение предоставляет список небезопасных форм и декомпозиция количество небезопасных передач для каждой формы, которые помогают диагностировать проблемы.

## <a name="how-do-i-fix-the-suspicious-user-activity-detection"></a>Как исправить обнаружения «подозрительные действия пользователя»?
1. **"Рассмотрение".** Уведомление содержит количество различных пользователей, которые короткая подозрительное поведение. Это поможет назначить приоритет проблемы.
2. **Область действия.** Из страны исходил подозрительные запросы? Кто из пользователей были наиболее подозрительных? Эти сведения можно получить из уведомления.
3. **Выполните диагностику.** Обнаружение предоставляет список пользователей, подозрительных и список стран, для каждого пользователя, которые помогают диагностировать проблемы.

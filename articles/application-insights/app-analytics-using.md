# Использование аналитики в Application Insights

[Аналитика](app-analytics.md) позволяет выполнять расширенные запросы к данным телеметрии приложения, собранным службой [Application Insights](app-insights-overview.md). На этих страницах описан ее язык запросов.

[AZURE.INCLUDE [app-analytics-top-index](../../includes/app-analytics-top-index.md)]

## Открытие аналитики

В домашнем ресурсе вашего приложения в Application Insights щелкните "Аналитика".

![На сайте portal.azure.com откройте ресурс Application Insights и щелкните "Аналитика".](./media/app-analytics/001.png)

Встроенный учебник подаст вам некоторые идеи о возможностях.

Смотрите [исчерпывающий обзор здесь](app-analytics-tour.md).

## Написание запросов

Напишите запрос, начиная с имен каких-либо таблиц, приведенных в левой части окна. Используйте `|` для создания конвейера [операторов](app-analytics-queries.md).


![](./media/app-analytics-using/150.png)

* Не помещайте в запрос пустые строки.
* В запросе можно использовать только однократные разрывы строк.
* Чтобы разместить несколько запросов в одном окне, разделяйте их пустыми строками.
* Для выполнения запроса **поместите курсор внутри или в конце его** и нажмите кнопку Go (Вперед).


![](./media/app-analytics-using/130.png)

* Содержимое окна запроса можно сохранять и повторно вызывать.

![](./media/app-analytics-using/140.png)

## Упорядочение результатов

Можно выбрать столбцы, которые вы хотите видеть. Разверните любой элемент, чтобы увидеть все возвращенные значения столбцов.

![](./media/app-analytics-using/030.png)

> [AZURE.NOTE] Чтобы быстро изменить порядок результатов, доступных в веб-браузере, щелкните заголовок столбца. Но имейте в виду, что для большого результирующего набора в браузер загружается ограниченное количество строк. Поэтому сортировка таким способом не всегда показывает фактические наибольшие или наименьшие элементы. Для этого следует использовать оператор [top](app-analytics-queries.md#top-operator) или [sort](app-analytics-queries.md#sort-operator).

Однако рекомендуется использовать операторы [take](app-analytics-queries.md#take-operator), [top](app-analytics-queries.md#top-operator) или [summarize](app-analytics-queries.md#summarize-operator), чтобы избежать загрузки огромных таблиц с сервера. В любом случае существует автоматическое ограничение — около 10 000 строк в запросе.


## Диаграммы

Выберите тип диаграммы, которая вам нужна:

![](./media/app-analytics-using/230.png)

Если имеется несколько столбцов правильных типов, можно выбрать оси x и y и столбец измерений, чтобы разделять по ним результаты:

![](./media/app-analytics-using/100.png)

По умолчанию результаты изначально отображаются в виде таблицы, а диаграмму вы выбираете вручную. Однако можно использовать [директиву render](app-analytics-queries.md#render-directive) в конце запроса, чтобы выбрать схему.

## Экспорт в Excel

После выполнения запроса можно скачать CSV-файл. Щелкните **Экспорт в Excel**.

## Экспорт в Power BI

1. Поместите курсор в запрос, а затем выберите **Экспорт в Power BI**.

    ![](./media/app-analytics-using/240.png)

    При этом загружается файл скрипта M.

3. Скопируйте скрипт на языке M в расширенный редактор запросов Power BI Desktop.
 * Откройте экспортированный файл.
 * В Power BI Desktop выберите **"Получить данные", "Пустой запрос", "Расширенный редактор"** и вставьте этот скрипт на языке M.

    ![](./media/app-analytics-using/250.png)

4. При необходимости измените учетные данные. Теперь можно строить отчет.

    ![](./media/app-analytics-using/260.png)

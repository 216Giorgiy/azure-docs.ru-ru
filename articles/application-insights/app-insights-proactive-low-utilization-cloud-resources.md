# <a name="low-utilization-of-cloud-resources-preview"></a>Низкое использование ресурсов облака (Предварительная версия)

Application Insights автоматически анализирует потребление ресурсов ЦП каждого экземпляра роли в приложении и выявляет экземпляры с низкой загрузкой ЦП. Такое обнаружение позволяет уменьшить ресурсами Azure и снижения затрат, за счет уменьшения числа экземпляров роли, каждая роль использует или за счет уменьшения числа ролей.

Этот компонент требует без дополнительной настройки, отличного от [Настройка счетчиков производительности](https://docs.microsoft.com/en-us/azure/application-insights/app-insights-performance-counters) для вашего приложения. Он остается активным, если приложение формирует достаточно телеметрии счетчиков производительности ЦП (% загруженности процессора).

## <a name="when-would-i-get-this-type-of-smart-detection-notification"></a>Когда будет получить уведомления смарт-обнаружение такого типа
Типичные уведомление появляется, когда многие экземпляры рабочих и веб-роли имеют низкую загрузку ЦП.

## <a name="does-my-app-definitely-consume-too-many-resources"></a>Мое приложение определенно занимает слишком много ресурсов?
Нет, уведомление не означает, что приложение определенно потребляет слишком много ресурсов. Несмотря на то, что такие шаблоны низкую загрузку ЦП обычно указывают, что может уменьшить потребление ресурсов, это может быть характерны для определенной роли, или может иметь естественный коммерческое обоснование и можно пропустить. Например возможно, потребуется несколько экземпляров, для других ресурсов, например памяти и сети, а не ЦП.

## <a name="how-do-i-fix-it"></a>Как ее исправить?
Уведомления содержат диагностических сведений для поддержки в процессе диагностики:
1. **"Рассмотрение".** Уведомление приведены роли в приложении, что имеют низкую загрузку ЦП. Это поможет назначить приоритет проблемы.
2. **Область действия.** Сколько ролей короткая низкую загрузку ЦП, и сколько экземпляров каждой роли использовать возможности ЦП низкой? Эти сведения можно получить из уведомления.
3. **Выполните диагностику.** Обнаружение содержит процент ресурсов ЦП, показывающая ЦП по каждому экземпляру со временем. Также можно использовать связанные элементы и отчеты, связывание с поддержкой сведения, такие как процентилем использования ЦП, которые помогают диагностировать проблемы.

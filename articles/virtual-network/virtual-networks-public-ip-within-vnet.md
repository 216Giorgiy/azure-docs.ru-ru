# Общедоступное пространство IP-адресов в виртуальной сети (VNet)

Теперь в виртуальные сети VNet можно добавлять общедоступное пространство IP-адресов. Ранее в сети VNet можно было добавить только блоки адресов RFC 1918 (закрытое пространство). При добавлении общедоступного диапазона IP-адресов он будет рассматриваться как часть закрытого пространства IP-адресов VNet, доступная только в пределах виртуальной сети, взаимосвязанных виртуальных сетей и из локального расположения.

Добавление общедоступного пространства IP-адресов по существу работает следующим образом:

## Как добавить общедоступный диапазон IP-адресов?

Общедоступный диапазон IP-адресов добавляется таким же образом, как и закрытый: с помощью файла *netcfg* или настройки конфигурации на портале. Общедоступный диапазон IP-адресов можно добавить во время создания виртуальной сети или можно вернуться и добавить его позднее. В приведенном ниже примере показаны и общедоступное, и закрытое пространство IP-адресов, настроенные в одной и той же виртуальной сети.

## Существуют ли какие-либо ограничения?

Существует несколько запрещенных диапазонов IP-адресов:

- 224\.0.0.0/4 (многоадресная рассылка)

- 255\.255.255.255/32 (широковещательный адрес)

- 127\.0.0.0/8 (адрес обратной связи)

- 169\.254.0.0/16 (link-local)

- 68\.63.129.16/32 (внутренний DNS)

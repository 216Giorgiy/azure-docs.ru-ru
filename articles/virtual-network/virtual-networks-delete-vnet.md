# Удаление виртуальной сети

Чтобы удалить виртуальную сеть (VNet), недостаточно просто нажать кнопку **Удалить**. Процедура удаления состоит из нескольких этапов:

1. **Сохранение параметров (необязательно)**. Если вы хотите сохранить настройки виртуальной сети в локальный файл, перед удалением виртуальной сети экспортируйте файл конфигурации. Дополнительные сведения см. в статье [Экспорт параметров виртуальной сети в файл конфигурации сети](https://msdn.microsoft.com/library/azure/dn133804.aspx). Сохранение параметров позволит в будущем повторно создать эту же виртуальную сеть, если возникнет такая необходимость.

1. **Удаление шлюза виртуальной сети**. Если для виртуальной сети настроен шлюз, перед удалением виртуальной сети его необходимо удалить. Чтобы удалить шлюз виртуальной сети, перейдите на страницу панели мониторинга виртуальной сети. В нижней части страницы щелкните **Удалить шлюз**.
						
	Удаление шлюза может занять 5–10 минут, после чего можно выполнять дальнейшие действия.

1. **Удаление облачных служб, веб-сайтов и виртуальных машин**. Если в виртуальной сети развернуты какие-либо объекты, перед удалением виртуальной сети их необходимо удалить.

1. **Удаление виртуальной сети**. Щелкните справа в строке *Имя*, чтобы выделить виртуальную сеть, которую следует удалить, и нажмите кнопку **Удалить** в нижней части страницы. Следуйте инструкциям на экране.

1. **Дополнительный этап**. После удаления виртуальной сети также можно удалить настройки локальной сети, DNS-серверы и территориальную группу.

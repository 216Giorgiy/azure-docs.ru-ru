# IP-адреса в виртуальной сети Azure
В этой статье описывается создание IP-адресов для виртуальных машин, балансировщиков нагрузки, VPN-шлюзов и шлюзов приложений.

[AZURE.INCLUDE [azure-arm-classic-important-include](../../includes/learn-about-deployment-models-rm-include.md)].

## Общедоступные IP-адреса
Общедоступные IP-адреса позволяют ресурсам Azure подключаться к Интернету и другим общедоступным службам Azure, таким как [кэш Redis для Azure](https://azure.microsoft.com/services/cache) и [концентраторы событий Azure](https://azure.microsoft.com/services/event-hubs). Общедоступный IP-адрес — это независимый ресурс, который может быть связан с различными типами ресурсов Azure, включая [виртуальные машины](virtual-machines-about.md) и [балансировщики нагрузки](load-balancer-overview.md).

### Способ выделения общедоступных IP-адресов
Общедоступные IP-адреса выделяются из пула IP-адресов, доступных в определенном расположении. Полный список диапазонов IP-адресов, используемых в центрах обработки данных Microsoft Azure, опубликован на странице [загрузки диапазонов IP-адресов центра обработки данных Azure](https://www.microsoft.com/download/details.aspx?id=41653).

Существует два способа выделения IP-адреса общедоступному IP-ресурсу: *динамический* и *статический*. По умолчанию используется *динамический* способ выделения. При использовании *динамического* способа IP-адрес **не** выделяется *ресурсу общедоступного IP-адреса* во время его создания. При создании или запуске ресурса (например, виртуальной машины или балансировщика нагрузки), связанного с *ресурсом общедоступного IP-адреса*, IP-адрес выделяется из пула IP-адресов. Когда связанный ресурс удаляется или останавливается, этот IP-адрес освобождается и выделяется доступному пулу.

При использовании *статического* способа IP-адрес выделяется *ресурсу общедоступного IP-адреса* во время его создания. В таком случае (независимо от состояния связанного ресурса) этот IP-адрес остается выделенным. IP-адрес выделяется доступному пулу, только если *ресурс общедоступного IP-адреса* удаляется или способ его выделения изменяется на *динамический*.

### DNS-разрешение
Метку имени доменного имени DNS можно указывать для ресурса общедоступного IP-адреса, который создает соответствующую запись DNS на DNS-серверах Azure. Соответствующее полное доменное имя *domainnamelabel*.*location*.cloudapp.azure.com может глобально преобразовываться в IP-адрес, выделенный для ресурса общедоступного IP-адреса.

Мы рассмотрим разные типы ресурсов, которые могут быть связаны с общедоступным IP-адресом, описанным ниже.

### Виртуальная машина
Общедоступный IP-адрес связан с [виртуальной машиной](virtual-machines-about.md) посредством сетевого адаптера. У каждой виртуальной машины может быть только один общедоступный IP-адрес, независимо от количества связанных с ней сетевых адаптеров. Только *динамически* выделенный *ресурс общедоступного IP-адреса* может быть связан с сетевым адаптером виртуальной машины. Если у виртуальной машины несколько сетевых адаптеров, *ресурс общедоступного IP-адреса* может быть связан только с основным сетевым адаптером.

### Подсистема балансировщика нагрузки Azure
Общедоступный IP-адрес может быть связан с интерфейсной конфигурацией [балансировщика нагрузки Azure](load-balancer-overview.md), который служит в качестве виртуального IP-адреса с балансировкой нагрузки, доступного из Интернета. С балансировщиком нагрузки могут быть связаны и *динамические*, и *статические* ресурсы общедоступных IP-адресов. С интерфейсной конфигурацией балансировщика нагрузки можно сопоставить несколько *ресурсов общедоступных IP-адресов*. Это позволяет использовать многопользовательские среды с несколькими веб-сайтами на основе SSL.

### Шлюз приложений
Общедоступный IP-адрес может быть связан с интерфейсной конфигурацией [шлюза приложений Azure](application-gateway-introduction.md). Это позволяет использовать виртуальный IP-адрес со сбалансированной нагрузкой, доступный из Интернета. В настоящее время только *динамически* выделенный ресурс общедоступного IP-адреса может быть связан со шлюзом приложения. Тем не менее можно связать несколько общедоступных IP-адресов.

### VPN-шлюз
Общедоступный IP-адрес необходимо связать с IP-конфигурацией [VPN-шлюза Azure](vpn-gateway-about-vpngateways.md) в процессе создания VPN-подключения между виртуальной сетью Azure и локальной сетью или другой виртуальной сетью Azure. Сейчас только *динамически* выделенный ресурс общедоступного IP-адреса может быть связан с VPN-шлюзом.

### Краткий обзор

|Ресурс|Статическое выделение|Динамическое выделение|Несколько IP-адресов|
|---|---|---|---|
|Виртуальная машина и сетевой адаптер|Да|Нет|Нет|
|Интерфейсная часть балансировщика нагрузки|Да|Да|Да|
|Интерфейсная часть шлюза приложений|Да|Нет|Да|
|VPN-шлюз|Да|Нет|Нет|

## Частные IP-адреса
Частные IP-адреса обеспечивают обмен данными между ресурсами в виртуальной сети без использования IP-адресов, доступных из Интернета. Такой IP-адрес выделяется из диапазона адресов подсети в виртуальной сети.

### Способ выделения частных IP-адресов
Существует два способа выделения IP-адресов — *динамический* и *статический*. По умолчанию используется *динамический* способ, когда IP-адрес выделяется через DHCP. Кроме того, способ выделения может быть явно указан как *статический* вместе с указанием IP-адреса. В таком случае ресурсу выделяется указанный IP-адрес из диапазона адресов подсети, если такой адрес свободен (не выделен какому-либо другому ресурсу).

Есть разные типы ресурсов, которым можно выделить частный IP-адрес. Обратите внимание, что оба способа выделения (*статический* и *динамический*) используются для всех типов ресурсов.

### Виртуальная машина
Частный IP-адрес назначается сетевому адаптеру [виртуальной машины](virtual-machines-about.md). Количество частных IP-адресов каждой виртуальной машины соответствует количеству связанных с ней сетевых адаптеров.

#### Разрешение внутренних имен узлов DNS
Все виртуальные машины Azure настроены для использования [DNS-серверов под управлением Azure](virtual-networks-name-resolution-for-vms-and-role-instances.md#azure-provided-name-resolution), если другое явно не указано в процессе создания. В случае использования DNS-серверов под управлением Azure DNS-запись создается автоматически с преобразованием имени узла виртуальной машины в частный IP-адрес. Если используется виртуальная машина с несколькими сетевыми адаптерами, имя узла преобразовывается в частный IP-адрес главного сетевого адаптера.

### Внутренний балансировщик нагрузки
Частный IP-адрес можно назначить интерфейсной части [внутреннего балансировщика нагрузки Azure](load-balancer-internal-overview.md), который служит в качестве виртуального IP-адреса с балансировкой нагрузки для ресурсов в виртуальной сети. Это позволяет выполнять балансировку нагрузки без отображения IP-адреса в Интернете.

### Шлюз приложений
Частный IP-адрес можно назначить интерфейсной части [шлюза приложений Azure](application-gateway-introduction.md). Это позволит не отображать в Интернете внутреннюю конечную точку (конечную точку внутреннего балансировщика нагрузки). Поведение и способы выделения аналогичны для внутреннего балансировщика нагрузки, описанного выше.

### Краткий обзор
|Ресурс|Статическое выделение|Динамическое выделение|Несколько IP-адресов|
|---|---|---|---|
|Виртуальная машина и сетевой адаптер|Да|Да|Да|
|Интерфейсная часть балансировщика нагрузки|Да|Да|Да|
|Интерфейсная часть шлюза приложений|Да|Да|Да|

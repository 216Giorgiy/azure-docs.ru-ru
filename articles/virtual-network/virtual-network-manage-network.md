# <a name="create-change-or-delete-virtual-networks"></a>Создание, изменение или удаление виртуальных сетей

Узнайте, как создавать и удалять виртуальные сети, а также изменять такие параметры существующих виртуальных сетей, как DNS-серверы и пространства IP-адресов. Виртуальная сеть — это представление вашей собственной сети в облаке. Это логическая изоляция облака Azure, выделенного для вашей подписки. Вот что можно сделать для каждой из них:
- Выберите назначаемое адресное пространство. Адресное пространство состоит из одного или нескольких диапазонов адресов, определенных с помощью нотации CIDR, например 10.0.0.0/16.
- Выберите для каждой виртуальной сети использование DNS-сервера, предоставляемого Azure, или собственного DNS-сервера. Всем ресурсам, подключенным к виртуальной сети, назначается этот DNS-сервер для разрешения имен в данной виртуальной сети.
- Разделите виртуальную сеть на подсети с собственными диапазонами адресов в адресном пространстве этой виртуальной сети. Узнайте, как создавать, изменять и удалять подсети, прочитав статью [Создание, изменение и удаление подсетей в виртуальных сетях](virtual-network-manage-subnet.md).

В этой статье описывается, как создавать, изменять и удалять виртуальные сети, созданные посредством модели развертывания с помощью Azure Resource Manager.
 
## <a name="before"></a>Перед началом работы

Перед выполнением действий, описанных в любом разделе этой статьи, выполните следующие задачи.

- Ознакомьтесь со статьей [Подписка Azure, границы, квоты и ограничения службы](../azure-subscription-service-limits.md?toc=%2fazure%2fvirtual-network%2ftoc.json#azure-resource-manager-virtual-networking-limits), чтобы узнать об ограничениях для виртуальных сетей.
- Войдите на портал Azure, в интерфейс командной строки Azure (CLI) или Azure PowerShell с помощью учетной записи Azure. Если у вас нет учетной записи Azure, зарегистрируйтесь для получения [бесплатной пробной учетной записи](https://azure.microsoft.com/free).
- Чтобы использовать команды Azure PowerShell для выполнения задач в этой статье, сначала необходимо [установить и настроить Azure PowerShell](/powershell/azureps-cmdlets-docs?toc=%2fazure%2fvirtual-network%2ftoc.json). Убедитесь, что у вас установлена самая последняя версия командлетов Azure PowerShell. Для получения справки по командам PowerShell с примерами введите `get-help <command> -full`.
- Чтобы использовать команды интерфейса командной строки Azure (Azure CLI) для выполнения задач в этой статье, сначала необходимо [установить и настроить Azure CLI](/cli/azure/install-azure-cli?toc=%2fazure%2fvirtual-network%2ftoc.json). Убедитесь, что у вас установлена самая последняя версия Azure CLI. Для получения справки по командам интерфейса командной строки введите `az <command> --help`.


## <a name="create-vnet"></a>Создание виртуальной сети

1. Войдите на [портал](https://portal.azure.com) с помощью учетной записи, которой назначены как минимум разрешения роли "Участник сети" для подписки. Сведения о назначении ролей и разрешений учетным записям см. в статье [Встроенные роли для управления доступом на основе ролей в Azure](../active-directory/role-based-access-built-in-roles.md?toc=%2fazure%2fvirtual-network%2ftoc.json#network-contributor).
2. На портале Azure нажмите кнопку **+ Создать**. В отобразившейся колонке **Создание** щелкните **Сети**. В отобразившейся колонке **Сети** щелкните **Виртуальная сеть**.
3. В открывшейся колонке **Виртуальная сеть** в поле **Выбор модели развертывания** оставьте выбранным пункт *Resource Manager* и нажмите кнопку **Создать**.
4. В появившейся колонке **Создать виртуальную сеть** введите или выберите значения приведенных ниже параметров и щелкните **Создать**.
    - **Имя**. Имя должно быть уникальным в пределах [группы ресурсов](../azure-glossary-cloud-terminology.md?toc=%2fazure%2fvirtual-network%2ftoc.json#resource-group), выбранной для создания виртуальной сети. Это имя невозможно изменить после создания виртуальной сети. В перспективе можно создать несколько виртуальных сетей. Прочитайте статью [Naming conventions](/azure/architecture/best-practices/naming-conventions?toc=%2fazure%2fvirtual-network%2ftoc.json#naming-rules-and-restrictions) (Соглашения об именовании), чтобы ознакомиться с рекомендациями по именованию, позволяющими упростить управление несколькими виртуальными сетями.
    - **Адресное пространство**. Укажите с помощью нотации CIDR. Определяемое адресное пространство может быть общедоступным или частным (RFC 1918). Вне зависимости от того, какого типа определяется адресное пространство (общедоступное или частное), оно доступно только в пределах виртуальной сети, взаимосвязанных виртуальных сетей и всех локальных сетей, подключенных к данной виртуальной сети. Невозможно добавить следующие адресные пространства:
        - 224.0.0.0/4 (многоадресная рассылка)
        - 255.255.255.255/32 (широковещательный адрес)
        - 127.0.0.0/8 (петлевой адрес);
        - 169.254.0.0/16 (локальный адрес канала);
        - 168.63.129.16/32 (внутренний DNS)
    
      Хотя при создании виртуальной сети вы можете определить только одно адресное пространство, можно добавить дополнительные адресные пространства после ее создания. Чтобы узнать, как это сделать, выполните инструкции в разделе [Добавление и удаление адресных пространств](#add-address-spaces) этой статьи.

      >[!WARNING]
      >Если адресные пространства одних виртуальных сетей перекрывают адресные пространства других виртуальных или локальных сетей, их будет невозможно подключить друг к другу. Примите во внимание другие виртуальные или локальные сети, которые может понадобиться подключить к данной виртуальной сети в будущем, прежде чем определять адресное пространство.
      >
      >
    
    - **Имя подсети**. Это имя должно быть уникальным в пределах виртуальной сети. Это имя невозможно изменить после создания подсети. На портале требуется определить одну подсеть при создании виртуальной сети, хотя в ней не обязательно должны быть какие-либо подсети. При создании виртуальной сети на портале можно определить только одну подсеть, но после создания виртуальной сети в нее можно добавить дополнительные подсети. Чтобы добавить подсеть в виртуальную сеть, ознакомьтесь с разделом [Создание подсети](virtual-network-manage-subnet.md#create-subnet) статьи [Создание, изменение и удаление подсетей в виртуальных сетях](virtual-network-manage-subnet.md). Можно создать виртуальную сеть с несколькими подсетями с помощью интерфейса командной строки или PowerShell.

      >[!TIP]
      >Нередко разные подсети создаются для управления маршрутизацией или фильтрации трафика между ними. Перед определением подсетей рекомендуется рассмотреть возможность фильтрации и маршрутизации трафика между ними. Дополнительные сведения о фильтрации трафика между подсетями см. в статье [Фильтрация сетевого трафика с помощью групп безопасности сети](virtual-networks-nsg.md). Azure направляет трафик между подсетями автоматически, но можно переопределить маршруты Azure по умолчанию. Чтоб узнать, как это сделать, прочитайте статью [Определяемые пользователем маршруты и IP-пересылка](virtual-networks-udr-overview.md).
      >

    - **Диапазон адресов подсети**. Этот диапазон должен находиться в пределах **адресного пространства**, указанного для виртуальной сети. Наименьший допустимый диапазон — /29. Он обеспечивает восемь IP-адресов для подсети. Azure резервирует первый и последний адреса в каждой подсети для соответствия требованиям протоколов. Еще три адреса резервируются для служб Azure. В результате виртуальная сеть с диапазоном адресов подсети /29 содержит только три пригодных для использования IP-адреса. Если планируется подключить виртуальную сеть к VPN-шлюзу, необходимо создать подсеть шлюза. Узнайте больше о [рекомендациях по диапазонам адресов для подсетей шлюза](../vpn-gateway/vpn-gateway-about-vpn-gateway-settings.md?toc=%2fazure%2fvirtual-network%2ftoc.json#a-namegwsubagateway-subnet). При определенных условиях диапазон адресов после создания подсети можно изменить. Чтобы узнать, как изменить диапазон адресов подсети, прочитайте раздел [Изменение подсети](#change-subnet) статьи [Создание, изменение и удаление подсетей в виртуальных сетях](virtual-network-manage-subnet.md).
    - **Подписка**. Выберите [подписку](../azure-glossary-cloud-terminology.md?toc=%2fazure%2fvirtual-network%2ftoc.json#subscription). Виртуальная сеть не может охватывать несколько подписок, хотя она может быть подключена к виртуальным сетям в других подписках с помощью VPN-шлюза Azure или пиринга виртуальных сетей. Ресурсы Azure, которые вы подключаете к виртуальной сети, должны находиться в той же подписке, что и виртуальная сеть.
    - **Группа ресурсов**. Создайте [группу ресурсов](../azure-resource-manager/resource-group-overview.md?toc=%2fazure%2fvirtual-network%2ftoc.json#resource-groups) или выберите имеющуюся. Ресурсы Azure, которые вы подключаете к виртуальной сети, могут существовать в одной или в разных группах ресурсов.
    - **Расположение**. Выберите [расположение](https://azure.microsoft.com/regions/) Azure (также называется регионом). Виртуальная сеть не может охватывать несколько расположений Azure, хотя виртуальную сеть в одном расположении можно подключить к виртуальной сети в другом расположении с помощью VPN-шлюза Azure. Ресурсы Azure, которые вы подключаете к виртуальной сети, должны находиться в том же расположении, что и виртуальная сеть.

3. В появившейся колонке **Виртуальные сети** щелкните виртуальную сеть, параметры которой требуется просмотреть.
4. Ниже приведены параметры в колонке, отображаемой для выбранной виртуальной сети.
    - **Обзор**. Содержит сведения о виртуальной сети, такие как адресное пространство и DNS-серверы. На следующем рисунке показаны общие параметры виртуальной сети **MyVNet**.

      С помощью этой колонки виртуальную сеть можно переместить в другую подписку или группу ресурсов. В ней указаны необходимые условия для перемещения, а также процедура перемещения с использованием портала Azure, PowerShell и Azure CLI.     - **Подключенные устройства**. Отображаются все ресурсы, подключенные к виртуальной сети. В примере на предыдущем рисунке к виртуальной сети подключены три сетевых интерфейса и одна подсистема балансировки нагрузки. В списке отображаются все создаваемые и подключаемые к виртуальной сети ресурсы. В случае удаления ресурса, подключенного к виртуальной сети, он больше не отображается в списке.

При создании виртуальной сети с помощью портала Azure для разрешения имен в виртуальной сети по умолчанию используются DNS-серверы Azure. Чтобы изменить DNS-серверы, выполните инструкции в разделе [Добавление, изменение или удаление DNS-серверов](#dns-servers) этой статьи. 
    - **Пиринги**. Если в подписке имеются пиринги, они перечислены здесь. Можно просматривать параметры существующих пирингов, а также создавать, изменять или удалять пиринги. Чтобы узнать больше о пирингах, прочитайте статью [Пиринг между виртуальными сетями](virtual-network-peering-overview.md).
    - **Свойства**. Отображаются параметры виртуальной сети, включая ее идентификатор ресурса и подписку.
    - **Схема**. Схема предоставляет собой наглядное представление всех устройств, подключенных к виртуальной сети, содержащее некоторые важные сведения о них. Можно щелкнуть любое из устройств, чтобы управлять им напрямую с помощью этого представления.
    - **Common Azure settings** (Общие параметры Azure). Дополнительные сведения об общих параметрах Azure см. в статьях о [журнале действий](../azure-resource-manager/resource-group-overview.md?toc=%2fazure%2fvirtual-network%2ftoc.json#activity-logs), [контроле доступа (IAM)](../azure-resource-manager/resource-group-overview.md?toc=%2fazure%2fvirtual-network%2ftoc.json#access-control), [тегах](../azure-resource-manager/resource-group-overview.md?toc=%2fazure%2fvirtual-network%2ftoc.json#tags), [блокировках](../azure-resource-manager/resource-group-lock-resources.md?toc=%2fazure%2fvirtual-network%2ftoc.json) и [сценарии службы автоматизации](../azure-resource-manager/resource-manager-export-template.md?toc=%2fazure%2fvirtual-network%2ftoc.json#export-the-template-from-resource-group).

## <a name="address-spaces"></a>Добавление и удаление адресных пространств

Можно добавлять и удалять адресные пространства виртуальной сети. Адресные пространства должны быть указаны в нотации CIDR. Они не должны перекрываться в пределах одной виртуальной сети. Определяемые адресные пространства могут быть общедоступными или частными (RFC 1918). Вне зависимости от того, какого типа определяются адресные пространства для виртуальной сети (общедоступное или частное), они могут быть доступны только в пределах виртуальной сети, взаимосвязанных виртуальных сетей и всех локальных сетей, подключенных к данной виртуальной сети. Невозможно добавить следующие адресные пространства:
    - 224.0.0.0/4 (многоадресная рассылка)
    - 255.255.255.255/32 (широковещательный адрес)
    - 127.0.0.0/8 (петлевой адрес);
    - 169.254.0.0/16 (локальный адрес канала);
    - 168.63.129.16/32 (внутренний DNS)

1. Войдите на [портал](https://portal.azure.com) с помощью учетной записи, которой назначены как минимум разрешения роли "Участник сети" для подписки. Сведения о назначении ролей и разрешений учетным записям см. в статье [Встроенные роли для управления доступом на основе ролей в Azure](../active-directory/role-based-access-built-in-roles.md?toc=%2fazure%2fvirtual-network%2ftoc.json#network-contributor).
2. В верхней части портала Azure в поле с текстом *Поиск ресурсов* введите *виртуальные сети*. Когда в результатах поиска появится пункт **Виртуальные машины**, щелкните его.
3. В появившейся колонке **Виртуальные сети** щелкните виртуальную сеть, в которой необходимо добавить или удалить адресное пространство.
4. В колонке, отображенной для выбранной виртуальной сети, в разделе **Параметры** щелкните **Адресное пространство**.
5. В отобразившейся колонке адресных пространств выполните одно из следующих действий:
    - **Add an address space** (Добавить адресное пространство): введите в поле новое адресное пространство. Адресное пространство не может перекрывать какое-либо существующее адресное пространство, определенное для виртуальной сети.
    - **Remove an address space** (Удалить адресное пространство): щелкните правой кнопкой мыши адресное пространство, а затем щелкните **Удалить**. Если в адресном пространстве существует подсеть, это адресное пространство невозможно удалить. Перед удалением адресного пространства нужно удалить все подсети, которые в нем существуют (и все ресурсы, подключенные к этим подсетям).
6. Щелкните **Сохранить**.

## <a name="dns-servers"></a>Добавление, изменение или удаление DNS-серверов

Все виртуальные машины, подключенные к виртуальной сети, регистрируются на указанных для нее DNS-серверах и используют эти DNS-серверы для разрешения имен. У каждого сетевого интерфейса (сетевой карты) на виртуальной машине могут быть собственные параметры DNS-сервера. Если для сетевой карты указаны собственные параметры DNS-сервера, то они переопределяют параметры DNS-сервера для виртуальной сети. Чтобы узнать больше о параметрах DNS для сетевых карт, прочитайте статью [Сетевые интерфейсы](virtual-network-network-interface.md#dns). Чтобы узнать больше о разрешение имен для виртуальных машин и экземпляров ролей облачных служб, прочитайте статью [Разрешение имен для ВМ и экземпляров ролей](virtual-networks-name-resolution-for-vms-and-role-instances.md).

1. Войдите на [портал](https://portal.azure.com) с помощью учетной записи, которой назначены как минимум разрешения роли "Участник сети" для подписки. Сведения о назначении ролей и разрешений учетным записям см. в статье [Встроенные роли для управления доступом на основе ролей в Azure](../active-directory/role-based-access-built-in-roles.md?toc=%2fazure%2fvirtual-network%2ftoc.json#network-contributor).
2. В верхней части портала Azure в поле с текстом *Поиск ресурсов* введите *виртуальные сети*. Когда в результатах поиска появится пункт **Виртуальные машины**, щелкните его. 
3. В появившейся колонке **Виртуальные сети** щелкните виртуальную сеть, параметры DNS которой требуется изменить.
4. В колонке, отображенной для выбранной виртуальной сети, в разделе **Параметры** щелкните **DNS-серверы**.
5. В отобразившейся колонке для DNS-серверов выберите один из следующих параметров:
    - **По умолчанию (предоставлен Azure)**. Все имена ресурсов и частные IP-адреса автоматически регистрируются на DNS-серверах Azure. Возможно разрешение имен между любыми ресурсами, подключенными к одной и той же виртуальной сети. Этот параметр невозможно использовать для разрешения имен между разными виртуальными сетями. Для этого необходимо применить пользовательский DNS-сервер.
    - **Пользовательский**. Можно добавить один или несколько серверов в пределах лимита Azure, установленного для виртуальной сети. Чтобы узнать больше об ограничениях DNS-серверов, прочитайте статью [Подписка Azure, границы, квоты и ограничения службы](../azure-subscription-service-limits.md?toc=%2fazure%2fvirtual-network%2ftoc.json#virtual-networking-limits-classic). Доступны следующие варианты:
        - **Add an address** (Добавить адрес). DNS-сервер будет добавлен в список DNS-серверов для виртуальной сети и зарегистрирован в Azure. Если ранее вы зарегистрировали DNS-сервер в Azure, его можно будет выбрать из отображенного списка. 
        - **Remove an address** (Удалить адрес). Щелкните **X** рядом с сервером, который требуется удалить. При этом сервер будет удален только из этого списка виртуальной сети. DNS-сервер останется зарегистрированным в Azure и может использоваться в других виртуальных сетях. 
        - **Reorder DNS server addresses** (Упорядочить адреса DNS-серверов). При указании DNS-серверов важно убедиться, что они размещены в списке в правильном порядке для вашей среды. Списки DNS-серверов не работают по принципу циклического перебора. Серверы используются в том порядке, в котором они указаны. Если первый DNS-сервер в списке доступен, клиент будет использовать его вне зависимости от того, работает этот DNS-сервер правильно или нет. Удалите все DNS-серверы в списке, а затем добавьте их обратно в требуемом порядке.
        - **Change an address** (Изменить адрес). Выделите DNS-сервер в списке и введите новое имя.
6. Щелкните **Сохранить**.
7. Перезапустите виртуальные машины, подключенные к сети, чтобы им были назначены новые параметры DNS-сервера. Виртуальные машины продолжают использовать свои текущие параметры DNS, пока они не будут перезагружены.

## <a name="delete-vnet"></a>Удаление виртуальной сети

Виртуальную сеть можно удалить только в том случае, если к ней не подключены какие-либо ресурсы. Если к какой-либо подсети в виртуальной сети подключены ресурсы, необходимо сначала удалить все ресурсы, подключенные к подсетям в этой виртуальной сети. Инструкции по удалению зависит от ресурса. Чтобы узнать, как удалить ресурсы, подключенные к подсетям, ознакомьтесь с документацией по типам ресурсов, которые требуется удалить. Чтобы удалить виртуальную сеть, выполните следующее.

1. Войдите на [портал](https://portal.azure.com) с помощью учетной записи, которой назначены как минимум разрешения роли "Участник сети" для подписки. Сведения о назначении ролей и разрешений учетным записям см. в статье [Встроенные роли для управления доступом на основе ролей в Azure](../active-directory/role-based-access-built-in-roles.md?toc=%2fazure%2fvirtual-network%2ftoc.json#network-contributor).
2. В верхней части портала Azure в поле с текстом *Поиск ресурсов* введите *виртуальные сети*. Когда в результатах поиска появится пункт **Виртуальные машины**, щелкните его.
3. В появившейся колонке **Виртуальные сети** щелкните виртуальную сеть, которую требуется удалить.
4. Убедитесь в отсутствии устройств, подключенных к сети, щелкнув **Подключенные устройства** в разделе **Параметры** колонки, отображенной для выбранной виртуальной сети. При наличии подключенных устройств необходимо сначала удалить их, прежде чем удалять виртуальную сеть.  Если подключенных устройств нет, щелкните **Обзор** в колонке.
5. Щелкните значок **Удалить** в верхней части колонки. 
6. Нажмите кнопку **Да**, чтобы подтвердить удаление виртуальной сети.

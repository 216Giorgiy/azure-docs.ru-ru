# Поддержка IPv6 в Azure

Протокол IPv6 изначально был представлен IETF (Internet Engineering Task Force) в 1998 году для устранения ограничения 32-разрядных адресов IPv4 в Интернете. С ростом числа мобильных устройств, подключенных к Интернету, и развитием "Интернета вещей" (IoT) IPv6-адреса становятся все более распространенными в Интернете.

>[AZURE.IMPORTANT] Поддержка IPv6 в Azure в настоящее время находится на этапе закрытой предварительной версии и не доступна всем клиентам. Эта страница будет обновлена, когда эта возможность станет доступной всем клиентам.

IPv6-адреса можно использовать в виртуальных машинах IaaS Azure в развернутых службах Resource Manager для исходящих и входящих подключений к Интернету. Ниже перечислены возможности протокола IPv6 в Azure.

- **Виртуальные машины IaaS в Resource Manager**
	- Виртуальные машины могут использовать конечные точки IPv4 и IPv6.
	- Виртуальные машины могут подключаться к Интернету через общедоступный IP-адрес балансировщика нагрузки для входящих (балансировщик нагрузки и NAT) подключений и исходящих подключений (SNAT).
	- Виртуальные машины могут подключаться ко всем службам Azure через их конечные точки IPv4.
	- Виртуальные машины могут использовать конечные точки IPv4 только горизонтального обмена данными (сервер-сервер).
- **Azure DNS**
	- DNS Azure поддерживает IPv6 и предоставляет записи типа AAAA, а также записи IPv4 типа A.
	- DNS Azure отвечает на запросы поиска, возвращая записи типа AAAA и A, если существуют обе записи. 
- **Office365**
	- Службы Office 365 поддерживают IPv6.
	- Из локальной сети IPv6 вы можете подключиться к ресурсам в Office 365, используя канал ExpressRoute.

## Виртуальные машины IaaS в Resource Manager

Воспользоваться преимуществами подключений IPv6 можно только с помощью модели развертывания с управлением ресурсами. Хотя у каждой виртуальной машины в развертывании Resource Manager есть конечные точки IPv4 и IPv6, конечную точку IPv6 можно использовать только для подключения к Интернету через балансировщик нагрузки, как показано ниже.

## Azure DNS

Вы можете разместить записи IPv6 типа AAAA в DNS Azure. Получив запрос на ресурс, DNS Azure вернет все доступные записи для имени узла. Например, если для заданного имени узла (т. е. www.contoso.com) имеется запись типа A и запись типа AAAA, в ответ на запрос DNS Azure отправит оба адреса. После получения ответа от DNS Azure или любого другого DNS-сервера, который поддерживает записи типа AAAA, DNS-клиент примет решение, следует ли использовать для подключения конечную точку IPv4 или IPv6.

>[AZURE.NOTE] Компьютеры под управлением Windows всегда пытается подключиться по протоколу IPv6, прежде чем использовать протокол IPv4.

## Office 365

Можно использовать подключение IPv6 между локальной сетью и Office 365, используя канал ExpressRoute и пиринг Майкрософт. Убедитесь, что вы понимаете, [как работает подключение ExpressRoute](../expressroute/expressroute-introduction.md) с пирингом Майкрософт.

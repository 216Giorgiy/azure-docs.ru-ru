# <a name="create-a-virtual-network-using-the-azure-portal"></a>Создание виртуальной сети с помощью портала Azure

В этой статье вы узнаете, как создать виртуальную сеть. После создания виртуальной сети в ней развертываются две виртуальные машины для тестирования взаимодействия между ними в частной сети.

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="log-in-to-azure"></a>Вход в Azure 

Войдите на портал Azure по адресу http://portal.azure.com.

## <a name="create-a-virtual-network"></a>Создать виртуальную сеть

1. Выберите **+ Создать** в верхнем левом углу окна портала Azure.

2. Щелкните **Сеть** и выберите **Виртуальная сеть**.

3. Как показано на изображении ниже, введите *myVirtualNetwork* в поле **Имя** и *myResourceGroup* — в поле **Группа ресурсов**, выберите **расположение** и **подписку**, оставьте остальные значения по умолчанию и выберите **Создать**. 

    ![Ввод основных сведений о виртуальной сети](./media/quick-create-portal/virtual-network.png)

    **Диапазон адресов** должен быть указан в нотации CIDR. Виртуальная сеть может содержать подсети. Подсеть по умолчанию использует **диапазон адресов** 10.0.0.0/24, это весь диапазон адресов виртуальной сети, поэтому невозможно создать другую подсеть внутри виртуальной сети, указав пространство имен и диапазон по умолчанию. Указанный диапазон адресов содержит IP-адреса 10.0.0.0–10.0.0.254. Однако из них доступны только 10.0.0.4–10.0.0.254, так как Azure резервирует первые четыре адреса (0–3) и последний адрес в каждой подсети. Доступные IP-адреса назначаются ресурсам, развертываемым в виртуальной сети.

## <a name="test-network-communication"></a>Тестирование сетевой связи

Виртуальная сеть позволяет ресурсам Azure разных типов взаимодействовать друг с другом в частном порядке. Одним из таких ресурсов, который можно развернуть в виртуальной сети, является виртуальная машина. Создайте в виртуальной сети две виртуальные машины, чтобы в дальнейшем можно было проверить частную связь между ними.

### <a name="create-virtual-machines"></a>Создание виртуальных машин

1. Нажмите кнопку **Создать** в верхнем левом углу портала Azure.

2. Выберите **Вычисления**, а затем — **Windows Server 2016 Datacenter**.

3. Введите сведения о виртуальной машине, показанные на рисунке ниже. Введенные **имя пользователя** и **пароль** понадобятся позже для входа на виртуальную машину. Пароль должен включать минимум 12 символов и соответствовать [определенным требованиям к сложности](../virtual-machines/windows/faq.md?toc=%2fazure%2fvirtual-network%2ftoc.json#what-are-the-password-requirements-when-creating-a-vm). Выберите свою **подписку** и существующую группу ресурсов *myResourceGroup*. Убедитесь, что выбрано **расположение**, в котором вы создали виртуальную сеть. Когда все будет готово, нажмите кнопку **ОК**.

    ![Ввод базовых сведений о виртуальной машине](./media/quick-create-portal/virtual-machine-basics.png)

4. Выберите размер виртуальной машины и щелкните **Выбрать**. Чтобы просмотреть дополнительные размеры, выберите **Просмотреть все** или измените фильтр **Supported disk type** (Поддерживаемые типы диска). Размеры, которые вы увидите, могут отличаться от приведенных ниже. 

    ![Выбор размера виртуальной машины](./media/quick-create-portal/virtual-machine-size.png)

5. В разделе **Параметры** для параметра **Виртуальная сеть** уже должно быть установлено значение *myVirtualNetwork*. Если это не так, последовательно выберите **Виртуальная сеть** и *myVirtualNetwork*. Оставьте значение *default* в поле **Подсеть** и нажмите кнопку **ОК**.

    ![Выбор виртуальной сети](./media/quick-create-portal/virtual-machine-network-settings.png)

6. На странице **Сводка** выберите **Создать**, чтобы начать развертывание виртуальной машины. 

7. Создание виртуальной машины занимает несколько минут. После создания виртуальная машина закрепляется на панели мониторинга на портале Azure и автоматически открывается окно сводки виртуальной машины. Выберите **Сети**.

    ![Сведения о сетевых подключениях виртуальной машины](./media/quick-create-portal/virtual-machine-networking.png)

    Вы увидите, что она использует **частный IP-адрес** *10.0.0.4*. В шаге 5, в разделе **Параметры** вы выбрали виртуальную сеть *myVirtualNetwork* и приняли подсеть с именем *default*, указанную в поле **Подсеть**. Когда вы [создали виртуальную сеть](#create-a-virtual-network), то приняли значение по умолчанию для **диапазона адресов** подсети — 10.0.0.0/24. DHCP-сервер Azure назначает виртуальной машине первый доступный адрес в выбранной подсети. Так как Azure зарезервирует первые четыре адреса (0–3) в каждой подсети, то 10.0.0.4 является первым доступным IP-адресом в подсети.

    Назначенный **общедоступный IP-адрес** отличается от адреса, назначенного виртуальной машине. По умолчанию Azure назначает общедоступный IP-адрес с маршрутизацией через Интернет каждой виртуальной машине. Виртуальной машине назначается общедоступный IP-адрес из [пула адресов, назначенных каждому региону Azure](https://www.microsoft.com/download/details.aspx?id=41653). Хотя Azure знает, какой общедоступный IP-адрес назначен виртуальной машине, операционная система на этой виртуальной машине не знает его.

8. Повторите шаги 1–7, но на шаге 3 назовите виртуальную машину *myVm2*. 

9. После создания этой виртуальной машины выберите **Сети**, как вы уже делали на шаге 7. Вы увидите, что виртуальная машина использует **частный IP-адрес** *10.0.0.5*. Так как платформа Azure уже назначила первый доступный адрес *10.0.0.4* в подсети виртуальной машине *myVm1*, она назначает виртуальной машине *myVm2* адрес *10.0.0.5*, так как он был следующим доступным адресом в подсети.

### <a name="connect-to-a-virtual-machine"></a>Подключение к виртуальной машине

1. Удаленно подключитесь к виртуальной машине *myVm1*. В верхней части портала Azure введите *myVm1*. Когда виртуальная машина **myVm1** появится в результатах поиска, выберите ее. Нажмите кнопку **Подключиться**.

    ![Обзор виртуальной машины](./media/quick-create-portal/virtual-machine-overview.png)

2. После нажатия кнопки **Подключиться** создается файл протокола удаленного рабочего стола (RDP-файл), который скачивается на ваш компьютер.  

3. Откройте этот RDP-файл. При появлении запроса выберите **Подключиться**. Введите имя пользователя и пароль, указанные при создании виртуальной машины, и нажмите кнопку **ОК**. При входе в систему может появиться предупреждение о сертификате. Чтобы продолжить процесс подключения, выберите **Да** или **Продолжить**.

### <a name="validate-communication"></a>Проверка связи

Попытка проверить связь с виртуальной машиной Windows завершится неудачей, так как проверка связи запрещена в брандмауэре Windows по умолчанию. Чтобы разрешить проверку связи с *myVm1*, введите следующую команду в командной строке.

```
netsh advfirewall firewall add rule name=Allow-ping protocol=icmpv4 dir=in action=allow
```

Чтобы проверить связь с *myVm2*, введите следующую команду в командной строке виртуальной машины *myVm1*. Укажите учетные данные, использованные при создании виртуальной машины, а затем выполните подключение.

```
mstsc /v:myVm2
```

Подключение к удаленному рабочему столу будет успешно установлено, так как обе виртуальные машины используют частные IP-адреса, назначенные из подсети *default*, и удаленный рабочий стол открыт в брандмауэре Windows по умолчанию. Вы можете подключиться к *myVm2* по имени узла, так как Azure автоматически обеспечивает разрешение DNS-имен для всех узлов в виртуальной сети. В командной строке проверьте связь с *myVm1* из *myVm2*.

```
ping myvm1
```

Проверка связи будет выполнена успешно, так как вы разрешили ее в брандмауэре Windows на виртуальной машине *myVm1* на предыдущем шаге. Чтобы подтвердить возможность исходящих подключений к Интернету, введите следующую команду.

```
ping bing.com
```

Вы получите четыре ответа с сайта bing.com. По умолчанию любая виртуальная машина в виртуальной сети может устанавливать исходящие подключения к Интернету. 

Выйдите из сеанса удаленного рабочего стола.

## <a name="clean-up-resources"></a>Очистка ресурсов

Удалите группу ресурсов и ее содержимое, когда она станет не нужна. В верхней части портала Azure введите *myResourceGroup*. Когда группа ресурсов **myResourceGroup** появится в результатах поиска, выберите ее. Нажмите кнопку **Удалить**.

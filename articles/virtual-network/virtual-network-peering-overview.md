# Пиринг виртуальных сетей

Пиринг виртуальных сетей — это механизм подключения между двумя виртуальными сетями, находящимися в одном регионе, через магистральную сеть Azure. После создания пиринга две виртуальные сети будут выглядеть как одна при любом подключении. Они по-прежнему будут управляться как отдельные ресурсы, но теперь находящиеся в них виртуальные машины могут взаимодействовать друг с другом напрямую по частному IP-адресу. Трафик между виртуальными машинами в таких виртуальных сетях будет направляться через инфраструктуру Azure, подобно трафику между виртуальными машинами в одной виртуальной сети. Пиринг виртуальных сетей предоставляет следующие преимущества:

- подключение между ресурсами в разных виртуальных сетях, характеризующееся низкой задержкой и высокой пропускной способностью;
- возможность использовать ресурсы, такие как виртуальные сетевые устройства и шлюзы VPN, в пиринговой виртуальной сети (для транзита);
- подключение виртуальной сети Resource Manager к классической виртуальной сети с обеспечением полной связи между ресурсами в этих сетях.

Требования и ключевые аспекты, касающиеся пиринга виртуальных сетей:

- Две виртуальные сети, для которых настраивается пиринг, должны находиться в одном регионе Azure.
- Для настройки пиринга пространства IP-адресов используемых виртуальных сетей не должны перекрываться.
- Пиринг устанавливается между двумя виртуальными сетями. При этом возникновение производной транзитивной связи невозможно, то есть если виртуальная сеть A является пиринговой по отношению к виртуальной сети B, которая имеет такие же отношения с виртуальной сетью C, между виртуальными сетями A и C не будет установлена пиринговая связь.
- Пиринг можно установить между виртуальными сетями в двух разных подписках, если его разрешает привилегированный пользователь обеих подписок.
- Пиринговую связь можно установить между виртуальной сетью Resource Manager и другой такой же сетью или классической виртуальной сетью, однако две классические виртуальные сети не могут иметь такую связь.
- Хотя для обмена данными между виртуальными машинами в пиринговых виртуальных сетях нет дополнительных ограничений к пропускной способности, объем передаваемого трафика зависит от размера виртуальной машины.

## Соединение 
После создания пиринга между двумя виртуальными сетями виртуальная машина (с веб-ролью или рабочей ролью) в одной из этих сетей может напрямую подключаться к другим виртуальным машинам в пиринговой виртуальной сети. Между этими сетями будет поддерживаться полная связь на уровне IP. Задержки сети при обмене данными между двумя виртуальными машинами в пиринговых виртуальных сетях будут такими же, как и в локальной виртуальной сети. Пропускная способность сети зависит от размера виртуальной машины. Дополнительные ограничения к пропускной способности не применяются. Трафик между виртуальными машинами в пиринговых виртуальных сетях передается напрямую через серверную инфраструктуру Azure, а не через шлюз.

Виртуальные машины в виртуальной сети могут получать доступ к конечным точкам внутреннего балансировщика нагрузки в пиринговой виртуальной сети. Чтобы заблокировать доступ к другой виртуальной сети или подсети, в любой виртуальной сети можно использовать группы безопасности сети. При настройке пиринга пользователь может включить или отключить использование правил группы безопасности сети между виртуальными сетями. Если пользователь разрешит полное подключение между пиринговыми виртуальными сетями (параметр по умолчанию), то можно будет использовать группы безопасности сети, чтобы заблокировать или запретить доступ в подсетях или виртуальных машинах.

Внутреннее разрешение DNS-имен Azure не поддерживается для виртуальных машин в пиринговых виртуальных сетях. Виртуальным машинам будут назначены внутренние DNS-имена, разрешаемые только в пределах локальной виртуальной сети. Тем не менее пользователи могут настроить виртуальные машины в пиринговых виртуальных сетях в качестве DNS-серверов для виртуальной сети.

## Цепочка служб
Пользователи могут настроить определяемые пользователем таблицы маршрутизации, указывающие, какие виртуальные машины в пиринговых виртуальных сетях будут использоваться для следующего прыжка (как показано на схеме ниже). Таким образом, пользователи могут создать цепочку служб для передачи трафика из одной виртуальной сети в виртуальное устройство, запущенное в пиринговой виртуальной сети. Кроме того, можно создавать среды со звездообразным подключением: в центральной виртуальной сети размещается инфраструктурный компонент, например виртуальное сетевое устройство, все остальные виртуальные сеты подключаются к этой сети по пиринговой связи и часть трафика направляется на устройства, работающие в центральной виртуальной сети. Если вкратце, то в определяемой пользователем таблице маршрутизации можно указать для следующего прыжка IP-адрес виртуальной машины, входящей в пиринговую виртуальную сеть.

## Использование шлюзов для локального подключения
Для каждой виртуальной сети, пиринговой или нет, можно настроить собственный шлюз, который будет использоваться для подключения к локальной сети. С помощью шлюза также можно настроить подключение (установить связь) между пиринговыми виртуальными сетями. Если для виртуальных сетей настроены оба типа подключения, трафик между виртуальными сетями будет проходить через пиринговую связь (т. е. через магистральную сеть Azure).

Если между виртуальными сетями установлена пиринговая связь, пользователи могут использовать шлюз в одной из этих сетей в качестве транзитного пункта при подключении к локальной сети. В этом случае для виртуальной сети, использующей удаленный шлюз, нельзя настроить локальный шлюз. Для простоты одна виртуальная сеть может использовать только один шлюз, локальный или удаленный (в пиринговой виртуальной сети), как показано на схеме ниже. Транзит через шлюз не поддерживается между классической виртуальной сетью и виртуальной сетью Resource Manager. Он поддерживается, если обе пиринговые виртуальные сети принадлежат к типу Resource Manager. Трафик между пиринговыми виртуальными сетями, которые используют один канал ER, передается по пиринговой связи (т. е. через магистральную сеть Azure). Пользователи по-прежнему могут использовать в каждой виртуальной сети локальные шлюзы для подключения к локальному каналу или настроить транзит через общий шлюз для локального подключения.

## Подготовка
Пиринг виртуальных сетей может выполнять только привилегированный пользователь. Это отдельная функция в рамках пространства имен виртуальной сети. Пользователю можно предоставить права на авторизацию пиринга. Пользователь с доступом на чтение и запись в виртуальной сети наследует эти права автоматически. Администратор и привилегированный пользователь с правами на настройку пиринга могут инициировать пиринговую связь с другой виртуальной сетью. Если на другой стороне есть такой же запрос и другие требования выполняются, пиринг будет установлен.

Дополнительные сведения об установке пиринга между двумя виртуальными сетями см. в других статьях с инструкциями.

## Ограничения
Количество пиринговых связей на одну виртуальную сеть ограничено. Дополнительные сведения см. в статье [Подписка Azure, границы, квоты и ограничения службы](../azure-subscription-service-limits.md#networking-limits).

## Цены
В течение ознакомительного периода пиринг виртуальных сетей можно использовать бесплатно. После выхода общедоступной версии за входящий и исходящий трафик по пиринговой связи будет взиматься номинальная плата. Дополнительные сведения см. на [странице цен](https://azure.microsoft.com/pricing/details/virtual-network).

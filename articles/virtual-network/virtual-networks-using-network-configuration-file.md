# Настройка виртуальной сети с помощью файла конфигурации сети

## Обзор

Чтобы настроить виртуальную сеть, можно воспользоваться мастером портала управления или создать и импортировать файл конфигурации сети. Azure использует файл конфигурации сети, чтобы определить параметры виртуальной сети.

Иногда удобно использовать мастер портала управления для начального создания конфигурации виртуальной сети, а затем вносить все последующие изменения непосредственно в файл конфигурации. Например, если выполняется настройка нескольких виртуальных сетей для отдельных подписок, может понадобиться сначала создать файл конфигурации сети с помощью мастера на портале управления. Затем этот файл экспортируется для использования в качестве шаблона, изменяется для указания различных параметров и импортируется обратно на портал управления. Иногда это может оказаться эффективным способом создания нескольких виртуальных сетей при наличии нескольких подписок.

Либо, если необходимо внести изменения настроек конфигурации сети до развертывания облачных служб или виртуальных машин в сети, можно экспортировать файл, изменить его и импортировать обратно в Azure. Также можно использовать файл конфигурации сети для резервного копирования параметров конфигурации сети, если требуется повторное создание виртуальной сети.

## Создание и изменение файла конфигурации сети 
Самый простой способ создания файла конфигурации сети — экспортировать параметры сети из существующей конфигурации виртуальной сети, изменить файл, поместив в него параметры, которые необходимо настроить для виртуальных сетей. Можно также получить учебный файл и изменить его.

Чтобы изменить файл конфигурации сети, можно просто открыть файл, внести соответствующие изменения и сохранить их. Чтобы внести изменения в файл конфигурации сети, можно использовать любой редактор *XML*.

Необходимо внимательно следовать руководству по параметрам схемы файла конфигурации сети. При создании файла конфигурации сети, параметры в файле перезапишут параметры, имеющиеся в настоящее время для этой подписки в Azure. При внесении изменений,которые несовместимы с рекомендациями для параметров, в значения в файле, виртуальная сеть не может быть настроена так, как предполагалось, и в некоторых случаях Azure не позволяет импортировать файл. Информацию о конкретных параметрах, содержащихся в файле конфигурации сети, см. в разделе [Схема конфигурации виртуальной сети Azure](https://msdn.microsoft.com/library/azure/jj157100.aspx).

Azure рассматривает подсеть, в которой что-нибудь развернуто, как используемую. Когда подсеть используется, она не может быть изменена. Перед изменением переместите все, что было развернуто в подсети, в другую подсеть, которая не изменяется. См. статью [Перемещение виртуальной машины или экземпляра роли в другую подсеть](virtual-networks-move-vm-role-to-subnet.md).



## Экспорт и импорт параметров виртуальной сети на портале управления  
Экспорт и импорт параметров конфигурации сети, содержащихся в файле конфигурации сети, может осуществляться с помощью PowerShell или портала управления. Приведенные ниже инструкции касаются экспорта и импорта с использованием портала управления.

### Экспорт параметров сети
При экспорте все параметры для виртуальных сетей в подписке будут записаны в XML-файл.

1. Выполните вход в **портал управления**.
2. На портале управления в нижней части страницы **Сети** щелкните **Экспорт**. 
3. В окне **Экспорт конфигурации сети** убедитесь, что выбрана подписка, для которой нужно экспортировать параметры сети. Затем установите флажок в правом нижнем углу. 
4. При появлении запроса сохраните файл *NetworkConfig.xml* в расположении по своему усмотрению.


### Импорт параметров сети

1. На **портале управления** на панели навигации в левой нижней части щелкните **Создать**.
2. Щелкните **Сетевые службы** -> **Виртуальная сеть** -> **Импорт конфигурации**.
3. На странице **Импорт файла конфигурации сети** перейдите к нужному файлу конфигурации сети, а затем щелкните стрелку **Далее**.
4. На странице **Создание сети** будет отображаться информация на экране, показывающая, какие разделы конфигурации сети будут изменены или созданы. Если изменения представляются вам правильными, установите флажок, чтобы продолжить обновление или создание виртуальной сети.

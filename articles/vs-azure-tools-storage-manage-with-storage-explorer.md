# Управление ресурсами хранилища Azure с помощью обозревателя хранилищ (предварительная версия)

Обозреватель хранилищ Microsoft Azure (предварительная версия) представляет собой автономное средство, которое позволяет легко управлять учетными записями хранения Azure. Это средство можно использовать для оперативного управления хранилищем за пределами портала Azure — например, во время разработки приложений в Visual Studio. Эта предварительная версия упростит вашу работу с хранилищем больших двоичных объектов. Вы можете создавать и удалять контейнеры, отправлять, загружать и удалять большие двоичные объекты, а также пользоваться поиском по всем контейнерам и большим двоичным объектам. Благодаря дополнительным функциям разработчики и операторы смогут работать с ключами и политиками SAS. Разработчики Windows также могут использовать эмулятор хранилища Azure для тестирования своего кода с помощью учетной записи хранения локальной разработки.

Для просмотра и управления ресурсами хранилища в обозревателе хранилищ необходимо иметь доступ к учетной записи хранения Azure в подписке или учетной записи внешнего хранилища. Если ее нет, можно создать учетную запись за несколько минут. Если у вас есть подписка MSDN, см. страницу [Ежемесячная сумма денег на счете в Azure для подписчиков Visual Studio](https://azure.microsoft.com/pricing/member-offers/msdn-benefits-details/). Если подписки нет, см. информацию о [бесплатной пробной версии на один месяц](https://azure.microsoft.com/pricing/free-trial/).

## Управление учетными записями и подписками Azure

Для просмотра ресурсов хранилища Azure в обозревателе хранилищ вам нужно войти в учетную запись Azure с помощью одной или нескольких активных подписок. Если у вас есть несколько учетных записей Azure, вы можете добавить их в обозреватель хранилищ и выбрать те подписки, которые нужно включить в представление ресурсов в обозревателе хранилищ. Если вы ранее не использовали Azure или не добавляли необходимые учетные записи в Visual Studio, вы получите запрос войти в учетную запись Azure.

### Добавление учетной записи Azure в обозреватель хранилищ

1.	Выберите значок **Параметры** (шестеренка) на панели инструментов обозревателя хранилищ.
1.	Щелкните ссылку **Добавить учетную запись**. Войдите в учетную запись Azure, ресурсы хранилища которой хотите просмотреть. Добавленную учетную запись нужно выбрать в раскрывающемся списке выбора учетной записи. Все подписки для этой учетной записи отображаются под информацией об учетной записи.

	![][0]

1.	Установите флажки для тех подписок учетной записи, которые вы хотите просмотреть, а затем нажмите кнопку **Применить**.

	![][1]

	Ресурсы хранилища Azure для выбранных подписок будут отображены в обозревателе хранилищ.

### Присоединение внешнего хранилища

1. Найдите имя учетной записи и ключ учетной записи хранения, которую вы хотите присоединить.
	1.	На портале предварительной версии Azure выберите учетную запись хранения, которую хотите присоединить.
	1.	В разделе **Управление** области **Параметры** на портале предварительной версии Azure нажмите кнопку **Ключи**.
	1.	Скопируйте значения параметров **Имя учетной записи хранения** и **Первичный ключ доступа**.

		![][2]

1.	В контекстном меню в узле **Учетные записи хранения** в обозревателе хранилищ выберите команду **Подключить внешнее хранилище**.

	![][3]

1. Введите имя учетной записи хранения в поле **Имя учетной записи** и первичный ключ доступа в поле **Ключ учетной записи**. Чтобы продолжить, нажмите кнопку **ОК**.

	![][4]

	Внешнее хранилище появится в обозревателе хранилищ.

	![][5]

1. Чтобы удалить внешнее хранилище, в его контекстном меню выберите команду Detach (Отсоединить).

	![][6]

## Просмотр ресурсов хранилища и навигация по ним

Чтобы перейти к ресурсу хранилища Azure и просмотреть информацию о нем в обозревателе хранилищ, разверните тип хранилища и выберите нужный ресурс. Сведения о выбранном ресурсе отображаются на вкладках **Действия** и **Свойства** в нижней части обозревателя хранилищ.

![][7]

-	На вкладке **Действия** отображаются действия, которые можно выполнять в обозревателе хранилищ для выбранного ресурса хранилища, например открытие, копирование или удаление. Действия также отображаются в контекстном меню ресурса.

-	На вкладке **Свойства** отображаются такие свойства ресурса хранилища, как тип, языковой стандарт, группа ресурсов, с которой связан ресурс, и URL-адрес.

Для всех учетных записей хранилища предусмотрено действие **Открыть на портале**. При выборе этого действия обозреватель хранилищ показывает выбранную учетную запись хранилища на портале предварительной версии Azure.

Для конкретных ресурсов могут отображаться другие действия и свойства. Например, для узлов контейнеров BLOB-объектов, очередей и таблиц предусмотрено действие **Создать**. Для отдельных элементов (например, для контейнеров BLOB-объектов) предусмотрены такие действия, как **Открыть**, **Удалить** и **Получить подписанный URL-адрес**. При выборе BLOB-объектов учетной записи хранения отображается действие открытия редактора больших двоичных объектов.

## Поиск учетных записей хранения и контейнеров больших двоичных объектов

Для поиска учетных записей хранения и контейнеров больших двоичных объектов с заданным именем в подписках учетной записи Azure введите нужное имя в поле **Поиск** в обозревателе хранилищ.

![][8]

Когда вы вводите символы в поле **поиска**, в дереве ресурсов отображаются только учетные записи хранения или контейнеры больших двоичных объектов с именами, соответствующими этим символам. Чтобы очистить условия поиска, нажмите кнопку **x** в поле **поиска**.

## Изменение учетных записей хранения

Чтобы добавить или изменить содержимое учетной записи хранения, выберите команду **Открыть редактор** для нужного типа хранения. Можно выбрать действия в контекстном меню выделенного элемента или на вкладке **Действия** в нижней части обозревателя хранилищ.

![][9]

Вы можете создать или удалить контейнеры больших двоичных объектов, очереди и таблицы. Можно также изменить BLOB-объекты в обозревателе хранилищ, выбрав действие **Открыть редактор контейнера больших двоичных объектов**.

### Изменение контейнера больших двоичных объектов

1.	Выберите действие **Открыть редактор контейнера больших двоичных объектов**. В области справа откроется редактор контейнера больших двоичных объектов.

	![][10]

1.	Нажмите кнопку **Отправить** и выберите команду **Отправка файлов**.

	![][11]

	Если же файлы, которые нужно отправить, хранятся в одной папке, вы можете вместо этого выбрать команду «Отправка файлов».

1. В диалоговом окне **Отправка файлов** нажмите кнопку с многоточием (**…**) справа от поля «Файлы», чтобы выбрать файлы, которые нужно отправить. Затем выберите тип большого двоичного объекта, который необходимо отправить (блок, страница или добавление). Если хотите, можно отправить файлы в папку в контейнере больших двоичных объектов. Укажите имя папки в поле **Отправить в папку (необязательно)**. Если такой папки не существует, она будет создана.

	![][12]

	На следующем снимке экрана показана отправка трех файлов изображений в новую папку с именем **Мои новые файлы** в контейнере больших двоичных объектов **Изображения**.

	![][13]

	Используя кнопки на панели инструментов редактора больших двоичных объектов, вы сможете выбирать, скачивать, открывать, копировать и удалять файлы, а также выполнять другие действия. Область **Действия** в нижней части диалогового окна содержит сведения об успешности выполнения операции и позволяет удалить из представления только успешные действия или полностью очистить область. Щелкните значок **+** рядом с отправленными файлами, чтобы просмотреть подробный список отправленных файлов.

## Создание подписанного URL-адреса (SAS)

Для некоторых операций может потребоваться SAS для доступа к ресурсу хранилища. Вы можете создать его с помощью обозревателя хранилищ.

1.	Выберите элемент, для которого требуется создать SAS, а затем выберите команду **Получить подписанный URL-адрес** в области **Действия** или в контекстном меню элемента.

	![][14]

1.	В диалоговом окне **Подписанный URL-адрес** выберите политику, даты начала и окончания срока, а также часовой пояс. Кроме того, установите флажки для уровней доступа, необходимого для ресурса, например только для чтения, для чтения и записи и т. д. По завершении нажмите кнопку **Создать**, чтобы создать SAS.

	![][15]

1.	В диалоговом окне **Подписанный URL-адрес** указан контейнер вместе с URL-адресом и строками запросов, которые вы можете использовать для доступа к ресурсу хранилища. Нажмите кнопку **Копировать**, чтобы скопировать строки.

	![][16]

## Управление SAS и разрешениями

Для управления доступом к контейнерам больших двоичных объектов вы можете выбрать команды **Управление списком контроля доступа** и **Настройка уровня общего доступа**.

-	Управление списком контроля доступа позволяет добавлять, изменять и удалять политики доступа (доступ пользователей для чтения, записи и так далее) в выбранном контейнере больших двоичных объектов.
-	Настройка уровня общего доступа позволяет определить объем доступа к ресурсу, который предоставляется обычному пользователю.  

-

1.	Выберите контейнер больших двоичных объектов, а затем команду **Управление списком контроля доступа** в контекстном меню или в области **Действия**.

	![][17]

1.	В диалоговом окне **Список контроля доступа** нажмите кнопку **Добавить**, чтобы добавить политики доступа. Выберите политику доступа, а затем выберите разрешения для нее. Когда все будет готово, нажмите кнопку **Сохранить**.

	![][18]

1.	Чтобы задать уровень доступа для контейнера больших двоичных объектов, выберите его в обозревателе хранилищ, а затем выберите команду **Настройка уровня общего доступа** в контекстном меню или в области **Действия**.

	![][19]

1.	В диалоговом окне **Выбор уровня открытого доступа к контейнеру** выберите уровень доступа, который необходимо предоставить пользователям услуг общедоступного поставщика, а затем нажмите кнопку **Применить**.

	![][20]

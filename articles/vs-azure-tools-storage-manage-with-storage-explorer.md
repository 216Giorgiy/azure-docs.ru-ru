# Приступая к работе с обозревателем службы хранилища (предварительная версия)

## Обзор 

Обозреватель службы хранилища Microsoft Azure (предварительная версия) — это автономное приложение, позволяющее легко работать с данными из службы хранилища Azure на платформе Windows, OSX и Linux. В этой статье рассмотрены различные способы подключения к учетным записям хранения Azure и управления ими.

## Предварительные требования

- [Скачайте и установите обозреватель службы хранилища (предварительная версия).](http://go.microsoft.com/fwlink/?LinkId=708343)

## Подключение к учетной записи хранения или к службе

Обозреватель службы хранилища (предварительная версия) предоставляет множество способов подключения к учетным записям хранения. Можно подключиться к учетным записям хранения, связанным с подписками Azure, а также к учетным записям хранения и службам, к которым предоставлен общий доступ из других подписок Azure. Кроме того, можно подключиться к локальному хранилищу и управлять им с помощью эмулятора хранения Azure.

- [Подключение к подписке Azure](#connect-to-an-azure-subscription) —управление ресурсами хранения, принадлежащими к подписке Azure.
- [Подключение к локальному хранилищу](#connect-to-local-storage) — управление локальным хранилищем с помощью эмулятора хранения Azure. 
- [Присоединение ко внешнему хранилищу](#attach-or-detach-an-external-storage-account) — управление ресурсами хранения, принадлежащими к другой подписке Azure, с использованием имени и ключа учетной записи хранения.
- [Присоединение учетной записи с помощью подписанного URL-адреса](#attach-account-using-sas) — управление ресурсами хранения, принадлежащими к другой подписке Azure, с использованием подписанного URL-адреса.
- [Присоединение службы с помощью подписанного URL-адреса](#attach-service-using-sas) — управление конкретной службой хранилища (контейнером больших двоичных объектов, очередью или таблицей), принадлежащей к другой подписке Azure, с использованием подписанного URL-адреса.

## подключение к подписке Azure

> [AZURE.NOTE] Если у вас нет учетной записи Azure, можно [подписаться на бесплатную пробную версию](https://azure.microsoft.com/pricing/free-trial/?WT.mc_id=A261C142F) или [активировать преимущества для подписчиков Visual Studio](https://azure.microsoft.com/pricing/member-offers/msdn-benefits-details/?WT.mc_id=A261C142F).

1. Запустите обозреватель службы хранилища (предварительная версия). 

1. Если вы используете обозреватель службы хранилища (предварительная версия) впервые или уже не в первый раз, но он еще не подключен к учетной записи Azure, отобразится информационная панель, позволяющая подключиться к учетной записи Azure.

	![][0]
	
1. Выберите **Подключиться к Microsoft Azure** и следуйте указаниям в диалоговых окнах, чтобы войти в учетную запись Майкрософт, связанную по крайней мере с одной подпиской Azure.

После входа в учетную запись Майкрософт в левой области обозревателя службы хранилища (предварительная версия) появятся все учетные записи хранения, связанные со всеми подписками Azure, которые сопоставлены с учетной записью Майкрософт.

### Фильтрация подписок Azure

Обозреватель службы хранилища (предварительная версия) позволяет фильтровать подписки Azure, связанные с учетными записями Майкрософт, в которые выполнен вход. Таким образом можно ограничить подписки Azure, учетные записи хранения которых будут перечислены в левой области.

1. Щелкните значок **Параметры** (в виде шестеренки).

	![][1]

1. 	В верхней части левой области появится раскрывающийся список, содержащий все учетные записи Майкрософт, в которые вы вошли.

	![][3]
	 
1.	Щелкните стрелку вниз рядом с раскрывающимся списком, чтобы просмотреть все учетные записи Майкрософт, в которые вы вошли, а также ссылку на добавление дополнительных учетных записей Майкрософт (т. е. вход в них).

	![][4]

1.	Выберите необходимую учетную запись Майкрософт из раскрывающегося списка.

1. 	В левой области отобразятся все подписки Azure, связанные с выбранной учетной записью Майкрософт. Флажок слева от каждой подписки Azure позволяет указать, следует ли обозревателю службы хранилища (предварительная версия) выводить список всех учетных записей хранения, связанных с этой подпиской. Установив флажок **Все подписки** можно выбрать все подписки Azure, а сняв его — отменить выбор всех.

	![][2]

1.	Когда вы выберете подписки Azure, которыми необходимо управлять, нажмите кнопку **Применить**. После этого в левой области отобразятся все учетные записи хранения для каждой выбранной подписки Azure, принадлежащей к текущей учетной записи Майкрософт.

В этом разделе описана процедура подключения к дополнительным учетным записям Майкрософт для просмотра связанных с ними подписок Azure и учетных записей хранения.

	![][3]
	 
1.	Щелкните стрелку вниз рядом с раскрывающимся списком, чтобы просмотреть все учетные записи Майкрософт, в которые вы вошли, а также ссылку на добавление дополнительных учетных записей Майкрософт (т. е. вход в них).

	![][4]

1.	Щелкните ссылку **Добавление учетной записи** и следуйте указаниям в диалоговых окнах, чтобы войти в учетную запись, связанную по крайней мере с одной подпиской Azure.

1.	Установите флажки для подписок Azure, которые нужно просмотреть.

	![][2]

1.	Нажмите кнопку **Применить**.

### Переключение между учетными записями Майкрософт

Несмотря на то, что вы можете подключиться к нескольким учетным записям Майкрософт, в левой области отображаются учетные записи хранения, связанные с подписками только одной (текущей) учетной записи Майкрософт. Если выполнено подключение к нескольким учетным записям Майкрософт, между ними можно переключаться, выполнив следующее:

1.	Щелкните значок **Параметры** (в виде шестеренки).

	![][1]

1. 	В верхней части левой области появится раскрывающийся список, содержащий все подключенные учетные записи Майкрософт.

	![][3]
	 
1.	Щелкните стрелку вниз рядом с раскрывающимся списком, чтобы просмотреть все учетные записи Майкрософт, в которые вы вошли, а также ссылку на добавление дополнительных учетных записей Майкрософт (т. е. вход в них).

	![][4]

1.	Выберите требуемую учетную запись Майкрософт.

1.	Установите флажки для подписок Azure, которые нужно просмотреть.

	![][2]

1.	Нажмите кнопку **Применить**.
  
## Подключение к локальному хранилищу

Обозреватель службы хранилища (предварительная версия) позволяет работать с локальным хранилищем с помощью эмулятора хранения Azure. Вы можете написать код и тестировать хранилище, при этом не требуется, чтобы в Azure была развернута учетная запись хранения (так как она эмулируется эмулятором хранения Azure).

>[AZURE.NOTE] В настоящее время эмулятор хранения Azure поддерживается только для Windows.

1. Запустите обозреватель службы хранилища (предварительная версия). 

1. В левой области разверните узел **(Разработка)**.

	![][21]

1. Если эмулятор хранения Azure еще не установлен, отобразится информационная панель со ссылкой на установку. Если информационная панель отображается, щелкните ссылку **Скачивание последней версии**, чтобы установить эмулятор.

	![][22]

1. После установки эмулятора вы сможете создавать локальные большие двоичные объекты, очереди и таблицы, а также работать с ними. Чтобы научиться работать с каждым типом учетной записи хранения, выберите необходимую ссылку из указанных ниже:

	- [Управление ресурсами хранилища BLOB-объектов Azure](./vs-azure-tools-storage-explorer-blobs.md)
	- Управление ресурсами хранилища очередей Azure — *ожидается в ближайшее время*
	- Управление ресурсами хранилища таблиц Azure — *ожидается в ближайшее время*

## Присоединение и отсоединение учетной записи внешнего хранилища

Обозреватель службы хранилища (предварительная версия) позволяет присоединять учетные записи внешних хранилищ, чтобы их можно было использовать совместно. В этом разделе объясняется, как присоединить (и отсоединить) учетные записи внешних хранилищ.

### Получение учетных данных учетной записи хранилища

Чтобы предоставить общий доступ к учетной записи внешнего хранилища, владельцу этой учетной записи необходимо сначала получить ее учетные данные (имя учетной записи и ключ), а затем предоставить эти сведения тому, кто хочет присоединиться к этой внешней учетной записи. Учетные данные учетной записи хранилища можно получить на портале Azure, выполнив следующее:

1.	Войдите на [портал Azure](https://portal.azure.com).
1.	Щелкните **Обзор**.
1.	Выберите **Учетные записи хранения**.
1.	В колонке **Учетные записи хранения** выберите необходимую учетную запись хранилища.
1.	В колонке **Параметры** для выбранной учетной записи хранилища выберите **Ключи доступа**.

	![][5]
	
1.	В колонке **Ключи доступа** скопируйте значения параметров **ИМЯ УЧЕТНОЙ ЗАПИСИ ХРАНЕНИЯ** и **КЛЮЧ 1**, которые будут использоваться при присоединении учетной записи хранилища.

	![][6]

### Присоединение учетной записи внешнего хранилища

1.	В обозревателе службы хранилища (предварительная версия) щелкните правой кнопкой мыши узел **Учетные записи хранения**, а затем в контекстном меню выберите пункт **Attach External Storage** (Присоединить внешнее хранилище).

	![][7]
	
1.	В разделе *Получение учетных данных учетной записи хранения* объясняется, как получить значения имени учетной записи хранилища и ключа 1. Эти значения будут использоваться на этом шаге. В диалоговом окне **Attach External Storage** (Присоединение внешнего хранилища) введите имя учетной записи хранилища в поле **Имя учетной записи** и значение ключа 1 в поле **Ключ учетной записи**. По окончании нажмите кнопку **ОК**.

	![][8]

	После присоединения учетная запись внешнего хранилища будет отображаться в списке, и к ее имени будет добавлено **(Внешнее)**.

	![][9]

### Отсоединение учетной записи внешнего хранилища

1. 	Щелкните правой кнопкой мыши учетную запись внешнего хранилища, которую необходимо отсоединить, и в контекстном меню выберите пункт **Отсоединить**.

	![][10]

1.	При появлении окна с подтверждающим сообщением нажмите кнопку **Да**, чтобы подтвердить отсоединение учетной записи внешнего хранилища.

	![][12]

## Присоединение учетной записи с использованием подписанного URL-адреса

Подписанный URL-адрес (SAS) позволяет администратору подписки Azure предоставлять временный доступ к учетной записи хранилища, не требуя ввода учетных данных подписки.

Чтобы продемонстрировать это, представим, что пользователь UserA является администратором подписки Azure и хочет предоставить пользователю UserB доступ к учетной записи хранилища на ограниченный период времени с определенными разрешениями:

1. Пользователь UserA создает SAS (состоящий из строки подключения для учетной записи хранилища) с определенным сроком действия и необходимыми разрешениями.
1. Затем он предоставляет этот SAS пользователю, которому нужен доступ к учетной записи хранилища. В нашем примере это UserB.  
1. Пользователь UserB использует обозреватель службы хранилища (предварительная версия), чтобы присоединить учетную запись, принадлежащую пользователю UserA, используя предоставленный SAS. 

### Получение SAS для учетной записи, к которой необходимо предоставить доступ

1.	Откройте обозреватель службы хранилища (предварительная версия).
1.	В левой области щелкните правой кнопкой мыши учетную запись хранилища, к которой необходимо предоставить доступ, а затем в контекстном меню выберите пункт **Get Shared Access Signature** (Получить подписанный URL-адрес).

	![][13]

1. В диалоговом окне **Shared Access Signature** (Подписанный URL-адрес) укажите период времени и разрешения для учетной записи, а затем нажмите кнопку **Создать**.

	![][14]
 
1. После этого откроется второе диалоговое окно **Shared Access Signature** (Подписанный URL-адрес), в котором будет отображаться SAS. Выберите команду **Копировать** рядом со **строкой подключения**, чтобы скопировать ее в буфер обмена. Нажмите кнопку **Закрыть**, чтобы закрыть диалоговое окно.

### Присоединение общей учетной записи с использованием подписанного URL-адреса

1.	Откройте обозреватель службы хранилища (предварительная версия).
1.	В левой области щелкните правой кнопкой мыши узел **Учетные записи хранения**, а затем в контекстном меню выберите пункт **Attach account using SAS** (Присоединение учетной записи с использованием SAS). ![][15]

1. В диалоговом окне **Attach account using SAS** (Присоединение учетной записи с использованием SAS) сделайте следующее:

	- **Имя учетной записи** — введите имя для учетной записи. **ПРИМЕЧАНИЕ**. Имя учетной записи не должно совпадать с именем исходной учетной записи хранилища, для которой был создан этот SAS. 
 	- **Строка подключения** — вставьте строку подключения, скопированную ранее.
 	- По окончании нажмите кнопку **ОК**.
	
	![][16]

После присоединения учетная запись хранилища будет отображаться в списке, и к ее имени будет добавлено (SAS).

![][17]

## Присоединение службы с использованием подписанного URL-адреса

В разделе [Присоединение учетной записи с использованием подписанного URL-адреса](#attach-account-using-sas) показано, как администратор подписки Azure может предоставить временный доступ к учетной записи хранилища путем создания (и предоставления доступа к) SAS для учетной записи хранилища. Аналогичным образом SAS можно создать для конкретной службы (контейнера больших двоичных объектов, очереди или таблицы) в учетной записи хранилища.

### Создание подписанного URL-адреса для службы, к которой необходимо предоставить доступ

В этом контексте служба может быть контейнером больших двоичных объектов, очередью или таблицей. Ниже приведены разделы, содержащие сведения о создании SAS для этих служб:

- [Получение SAS для контейнера больших двоичных объектов](./vs-azure-tools-storage-explorer-blobs.md#get-the-sas-for-a-blob-container)
- Получение SAS для очереди — *ожидается в ближайшее время*
- Получение SAS для таблицы — *ожидается в ближайшее время*

### Присоединение общей учетной записи с использованием подписанного URL-адреса

1.	Откройте обозреватель службы хранилища (предварительная версия).
1.	В левой области щелкните правой кнопкой мыши узел **Учетные записи хранения**, а затем в контекстном меню выберите пункт **Attach service using SAS** (Присоединение службы с использованием SAS). ![][18]

1. В диалоговом окне **Attach Account using SAS** (Присоединение учетной записи с использованием SAS) вставьте URI SAS, скопированный ранее, и нажмите кнопку **ОК**.

	![][19]

После присоединения служба будет отображаться в качестве элемента узла с именем **(SAS службы)**.

![][20]

## Поиск учетных записей хранилищ

Если у вас много учетных записей хранилищ, быстро найти определенную учетную запись можно с помощью поля поиска в верхней части левой области.

По мере ввода в поле поиска в левой области будут отображаться учетные записи хранилищ, имена которых соответствуют условию поиска. На приведенном ниже снимке экрана показан пример поиска всех учетных записей хранилищ, имена которых содержат слово tarcher.

![][11]
	
Чтобы очистить условие поиска, нажмите кнопку **x** в поле поиска.

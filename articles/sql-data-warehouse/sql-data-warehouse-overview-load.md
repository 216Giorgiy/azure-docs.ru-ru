# Загрузка данных в хранилище данных SQL
Хранилище данных SQL предоставляет множество вариантов для загрузки данных в различных сценариях. Например, вам нужно загрузить:

- большие пакеты один раз в день;
- мелкие пакеты в течение дня;
- простые обновления для таблиц измерений меньшего размера.

Потребности каждого сценария так же разнообразны, как типы данных, которые требуется загрузить в хранилище данных SQL. В этой статье перечислены некоторые варианты выбора для загрузки данных.

## Технологии
Хранилище данных SQL поддерживает следующие стандартные средства загрузки:

- фабрика данных Azure;
- служебная программа командной строки bcp;
- PolyBase;
- SQL Server Integration Services (SSIS);
- средства загрузки данных сторонних производителей.

### Фабрика данных Azure (ADF)
Фабрика данных Azure является полностью управляемой службой для создания хранилищ данных, служб обработки и перемещения данных в упрощенные, масштабируемые и надежные конвейеры производства данных. Хранилище данных SQL — [поддерживаемый источник и приемник][] для[действия копирования][] ADF.

### служебная программа командной строки bcp;
**bcp** — исполняемый файл командной строки, служебная программа Майкрософт, которая позволяет загружать данные на SQL Server, в базу данных SQL и хранилище данных SQL, а также извлекать из них данные. Чтобы начать работу, следуйте инструкциям руководства [Загрузка данных с помощью bcp][]

### PolyBase;
PolyBase — это технология Майкрософт, которая упрощает анализ данных, позволяя отправлять запрос к хранилищу больших двоичных объектов Azure и Hadoop с помощью Transact-SQL и без использования MapReduce. Технология PolyBase также может загружать данные из хранилища больших двоичных объектов Azure в хранилище данных SQL. Чтобы начать работу, следуйте инструкциям руководства [Загрузка с помощью PolyBase][]

### SQL Server Integration Services (SSIS)
[SSIS][] — это платформа для построения решений интеграции и преобразования данных масштаба предприятия. Для создания пакетов, которые подключаются к хранилищу данных SQL, используйте [стандартный адаптер загрузки данных OLE DB][] с помощью диспетчера подключений ADO.Net.

### Средства сторонних производителей
Хранилище данных SQL поддерживает ведущие отраслевые решения для загрузки данных. Дополнительные сведения см. в нашем списке [партнерских решений][].

# Средства управления для хранилища данных SQL
В этом разделе рассматриваются и сравниваются средства и параметры для управления хранилищем данных SQL, чтобы вы могли выбрать соответствующий инструмент. Выбор правильного средства зависит от количества баз данных, которыми вы управляете, выполняемых задач, а также то того, как часто вы их выполняете.

## Портал Azure
[Портал Azure][] — это веб-портал управления, где можно создавать, обновлять и удалять базы данных, а также отслеживать ресурсы базы данных. Это хороший инструмент для тех, кто только знакомится с Azure или управляет небольшим количеством баз данных хранилищ, а также для оперативного выполнения любой задачи по управлению.

Портал содержит метрики, включающие текущие и исторические настройки DWU производительности, объем используемого дискового пространства, успешные и неудачные SQL-подключения, а также набор представлений и данных, которые помогают расшифровать запросы, выполняемые на экземпляре, и сведения об этих запросах.

## SQL Server Data Tools в Visual Studio	
Средства [SQL Server Data Tools][] (SSDT) в Visual Studio представляют собой клиентский инструмент, который работает на вашем компьютере и позволяет подключаться к базам данных в облаке, разрабатывать их, а также управлять ими. Если вы не новичок в разработке приложений и уже знакомы с Visual Studio или другими интегрированными средами разработки (IDE), предлагаем обратить внимание на SSDT в составе Visual Studio.

Средства SSDT содержат обозреватель сервера SQL, который позволяет визуализировать, подключать и выполнять скрипты для баз данных хранилища данных SQL. Чтобы быстро подключиться к хранилищу данных SQL, нажмите кнопку **Открыть в Visual Studio** в командной строке во время просмотра сведений базы данных на портале.

Вы можете загрузить последнюю версию [SQL Server Data Tools][], которая включает поддержку хранилища данных SQL.

## Программы командной строки
Вы можете использовать программы командной строки PowerShell или sqlcmd для управления хранилищем данных SQL и автоматизации развертывания ресурсов Azure. Рекомендуется использовать эти средства для управления большим количеством логических серверов и развертывания изменений ресурсов в рабочей среде, так как обязательные задачи должны быть включены в скрипт, а затем автоматизированы.

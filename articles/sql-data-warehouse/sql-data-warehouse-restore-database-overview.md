# Восстановление хранилища данных SQL Azure (обзор)

> [AZURE.SELECTOR]
- [Обзор][]
- [Портал][]
- [PowerShell][]
- [REST][]

Хранилище данных SQL Azure защищает ваши данные с помощью локально избыточного хранилища и автоматического резервного копирования. Автоматическое резервное копирование помогает защитить базы данных от случайного повреждения или удаления. Для этого вам не нужны права администратора. Если пользователь непреднамеренно или случайно изменит или удалит данные, вы можете обеспечить непрерывность бизнес-процессов, восстановив базу данных до более ранней точки во времени. Хранилище данных SQL использует моментальные снимки хранилища Azure для резервного копирования вашей базы данных без простоев.

## Автоматическое резервное копирование

Для ваших **активных** баз данных как минимум каждые 8 часов будут создаваться автоматические резервные копии, которые будут храниться в течение 7 дней. Это позволяет восстановить активную базу данных до одной из нескольких точек восстановления за последние 7 дней.

Если база данных приостанавливается, новые резервные копии не создаются, а предыдущие удаляются по прошествии 7 дней. Если база данных приостанавливается дольше чем на 7 дней, сохраняется последняя резервная копия, чтобы у вас всегда была хотя бы одна резервная копия.

При удалении базы данных последняя резервная копия хранится в течение 7 дней.

Выполните этот запрос к активному хранилищу данных SQL, чтобы просмотреть дату последней архивации.

```sql
select top 1 *
from sys.pdw_loader_backup_runs 
order by run_id desc;
```

Если резервные копии необходимо хранить более 7 дней, можно просто восстановить одну из точек восстановления в новую базу данных. Эту базу данных затем при необходимости можно приостановить, чтобы вы платили только за дисковое пространство, выделенное для этой резервной копии.

## Избыточность данных

Кроме возможности резервного копирования, хранилище данных SQL также обеспечивает защиту данных с помощью [локально избыточного (LRS)][] хранилища Azure класса Premium. Несколько синхронных копий данных сохраняются в локальном центре обработки данных для обеспечения прозрачной защиты данных в случае локальных сбоев. Избыточность данных гарантирует целостность данных при проблемах с инфраструктурой, например при сбоях дисков и пр. Кроме того, избыточность данных обеспечивает непрерывность бизнес-процессов в отказоустойчивой инфраструктуре с высокой доступностью.

## Восстановление базы данных

Восстановление хранилища данных SQL — это простая операция, которую можно выполнять на портале Azure или автоматизировать с помощью PowerShell или интерфейсов REST API.

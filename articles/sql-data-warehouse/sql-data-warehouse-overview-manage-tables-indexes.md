# Управление таблицами и индексами в хранилище данных SQL Azure

Обзор вопросов, рекомендаций и задач, связанных с управлением таблицами и индексами в хранилище данных SQL.



| Категория | Задача или рекомендация | Описание |
| :-----------------------| :---------------------------------------------- | :----------- |
| Индексы columnstore | Перестройка индексов после загрузки и вставки данных | По умолчанию в хранилище данных SQL каждая таблица хранится в виде кластеризованного индекса columnstore. Это улучшает производительность и сжатие данных, особенно для больших таблиц. В зависимости от способа приема данных в индекс columnstore все данные могут сохраняться без сжатия columnstore. В этом случае может не обеспечиваться та производительность, на которую рассчитаны индексы columnstore. <br/><br/>Чтобы проверить работоспособность индексов columnstore, см. статью [Управление индексами columnstore][]. |
| Распределяемые по хэшу таблицы | Проверка равномерного распределения данных между узлами | Распределяемые по хэшу таблицы почти всегда являются лучшим способом оптимизации соединений и агрегатов в больших таблицах. Хранилище данных SQL распределяет таблицы в соответствии с указанным столбцом распределения. Некоторые столбцы хорошо подходят для распределения, некоторые — нет. Строки должны быть распределены равномерно. Некоторая доля неравномерности или неравномерного смещения данных не должна вызывать беспокойство, однако при слишком большом неравномерном смещении данных, вы потеряете преимущества вычислений с массовым параллелизмом (MPP), заложенные в хранилище данных SQL.<br/><br/>Чтобы убедиться, что у распределяемых по хэшу таблиц отсутствует слишком большое неравномерное смещение данных, см. статью [Manage distributed data skew][] \(Управление неравномерным смещением распределенных данных). |

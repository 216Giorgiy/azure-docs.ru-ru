# Восстановление базы данных в хранилище данных SQL Azure (обзор)

- [PowerShell](sql-data-warehouse-manage-database-restore-powershell.md)

В этой статье описываются варианты восстановления базы данных в хранилище данных SQL Azure. К ним относятся восстановление работающего хранилища данных и удаленного хранилища данных. Работающее и удаленное хранилища данных восстанавливаются из моментальных снимков, автоматически создаваемых для всех хранилищ данных.

## Сценарии восстановления

**Восстановление после сбоев инфраструктуры** — это сценарий восстановления после ошибок инфраструктуры, например сбоев диска. Клиенту нужно обеспечить непрерывность бизнес-процессов в отказоустойчивой инфраструктуре высокой доступности.

**Восстановление после ошибки пользователя** — это сценарий восстановления после непреднамеренного или случайного повреждения либо удаления данных. Если пользователь непреднамеренно или случайно изменяет или удаляет данные, клиенту нужно обеспечить непрерывность бизнес-процессов путем восстановления базы данных до более ранней точки во времени.

## Возможности восстановления базы данных

Рассмотрим, как хранилище данных SQL повышает надежность базы данных и обеспечивает возможность восстановления и непрерывность работы в упомянутых выше случаях.


### Избыточность данных

Хранилище данных SQL содержит все данные в [локально избыточном](../storage/storage-redundancy.md) хранилище Azure уровня "Премиум", храня 3 копии ваших данных.

### Восстановление базы данных

Эта функция предназначена для восстановления до более ранней точки во времени. Служба хранилища данных SQL Azure защищает все базы данных. Для этого она автоматически создает моментальные снимки хранилища по меньшей мере каждые 8 часов и хранит их в течение 7 дней вместе с дискретным набором точек восстановления. Функции автоматического создания моментальных снимков и восстановления помогают защитить базы данных от случайного повреждения или удаления. Чтобы больше узнать о восстановлении баз данных, ознакомьтесь с [задачами восстановления базы данных][].

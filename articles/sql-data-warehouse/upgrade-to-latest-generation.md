# <a name="upgrade-to-latest-generation-of-azure-sql-data-warehouse-in-the-azure-portal"></a>Обновление до последнего поколения хранилища данных SQL Azure на портале Azure

Используйте портал Azure, чтобы обновить хранилище данных SQL Azure до последнего поколения аппаратного обеспечения и архитектуры хранилища Azure. После этого вы сможете повысить производительность, масштабируемость и получить неограниченный объем хранилища для индексов columnstore.  

## <a name="applies-to"></a>Применяется к
Это обновление применяется к хранилищам данных для уровня производительности "Оптимизировано для эластичности".  Следуя инструкциям, можно обновить хранилище данных с уровня производительности "Оптимизировано для эластичности" до соответствующего уровня "Оптимизировано для вычислений". 

## <a name="sign-in-to-the-azure-portal"></a>Выполните вход на портал Azure.

Войдите на [портале Azure](https://portal.azure.com/).

## <a name="before-you-begin"></a>Перед началом работы

1. Если работа хранилища данных при обновлении с уровня "Оптимизировано для эластичности" приостановлена, [возобновите ее](pause-and-resume-compute-portal.md).
2. Будьте готовы к непродолжительному простою. 
3. В процессе обновления будут прерваны все сеансы и подключения. Завершите все запросы перед обновлением. Если запустить обновление с активными транзакциями, откат может быть продолжительным. 

## <a name="start-the-upgrade"></a>Начало обновления

1. На портале Azure откройте хранилище данных, а затем щелкните **Перейти на уровень, оптимизированный для вычислений**.
2. Обратите внимание на значения уровня производительности "Оптимизировано для вычислений". Выбор по умолчанию сопоставим с текущим уровнем до обновления.
3. Выберите уровень производительности. В настоящее время при использовании предварительной версии уровень производительности "Оптимизировано для вычислений" стоит в половину меньше.
4. Щелкните **Обновить**.
5. Проверьте состояние на портале Azure.
6. Подождите, пока состояние хранилища данных не изменится на "Подключен к Интернету".

## <a name="rebuild-columnstore-indexes"></a>Перестроение индексов columnstore

Когда для хранилища данных будет доступен интерактивный режим, вы сможете загружать данные и выполнять запросы. Однако сначала производительность может быть низкой, так как в фоновом процессе выполняется перемещение данных в новое аппаратное обеспечение. 

Чтобы переместить данные как можно быстрее, мы рекомендуем перестроить индексы columnstore. Сведения о том, как это сделать, см. в руководстве о [перестроении индексов columnstore для повышения качества сегментов](sql-data-warehouse-tables-index.md#rebuilding-indexes-to-improve-segment-quality). 

## <a name="next-steps"></a>Дополнительная информация

# Перенос кода SQL в хранилище данных SQL

Чтобы обеспечить совместимость кода с хранилищем данных SQL, скорее всего, необходимо будет вносить изменения в базу кода. Некоторые функции хранилища данных SQL могут значительно повысить производительность, так как они предназначены для работы непосредственно в распределенной форме. Однако для обеспечения производительности и масштабирования некоторые функции недоступны.

К счастью, большинство из этих ограничений можно обойти. Пояснения см. в соответствующих разделах по разработке, указанных выше.

### Рекурсивные обобщенные табличные выражения (CTE)

Это сложный сценарий, для которого не существует простого решения. Обобщенное табличное выражение потребуется разбить на составляющие и обрабатывать поэтапно. Обычно используется достаточно сложная цепь действий; заполнение временной таблицы по мере выполнения рекурсивных промежуточных запросов. После заполнения временной таблицы можно возвратить данные как единый результирующий набор. Похожий подход использовался для того, чтобы разрешить `GROUP BY WITH CUBE` в статье [группирование по предложениям со сверткой / куб / параметры наборов группирования][].

### Обновления с помощью обобщенных табличных выражений

Если обобщенное табличное выражение не является рекурсивным, вы можете перезаписать запрос таким образом, чтобы в нем использовались вложенные запросы. Для рекурсивных обобщенных табличных выражений вам сначала потребуется создать набор результатов, как это описано выше, затем следует объединить окончательный набор результатов с целевой таблицей и выполнить обновление.

### Системные функции

Ряд системных функций также не поддерживаются. Ниже приводится список некоторых системных функций, которые могут быть полезны в хранилище данных.

- NEWID()
- NEWSEQUENTIALID()
- @@NESTLEVEL()
- @@IDENTITY()
- @@ROWCOUNT()
- ROWCOUNT\_BIG
- ERROR\_LINE()

Опять же, многие из этих ограничений можно обойти.

Например, приведенный ниже код является альтернативным решением для выборки с помощью инструкции @@ROWCOUNT.

```
SELECT  SUM(row_count) AS row_count 
FROM    sys.dm_pdw_sql_requests 
WHERE   row_count <> -1 
AND     request_id IN 
                    (   SELECT TOP 1    request_id 
                        FROM            sys.dm_pdw_exec_requests 
                        WHERE           session_id = SESSION_ID() 
                        ORDER BY end_time DESC
                    )
;
```

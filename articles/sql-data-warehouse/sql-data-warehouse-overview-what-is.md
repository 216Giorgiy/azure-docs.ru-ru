# Общие сведения о хранилище данных SQL

Хранилище данных SQL — это распределенная база данных корпоративного класса в облаке Azure, способная обрабатывать петабайты реляционных и нереляционных данных. Это первое облачное хранилище в отрасли, в котором можно увеличивать и уменьшать количество вычислительных ресурсов, а также приостанавливать их работу в считанные секунды.

В хранилище данных SQL сочетаются качество и надежность реляционной базы данных SQL Server корпоративного класса и облачные вычисления. В этой статье приведены основные сведения о ключевых архитектурных особенностях хранилища данных SQL.

## Увеличение и уменьшение количества вычислительных ресурсов и их приостановка
Хранилище данных SQL — облачное. Вычислительные ресурсы масштабируются отдельно, поэтому вы платите только за требуемую производительность запросов. Вы можете развертывать эти ресурсы динамически, увеличивать и уменьшать их количество и даже приостанавливать вычисления. Используйте все преимущества хранения в облаке и обрабатывайте запросы с той скоростью, которая нужна вам в конкретный период времени. Если вычислительные процессы остановлены, вы платите только за хранение данных.

> [AZURE.NOTE]Единицы использования хранилища данных (DWU) — это единицы масштабирования вычислительных ресурсов в хранилище данных SQL.

- Если вы хотите получать результаты быстрее, увеличьте количество DWU и платите за их возросшее количество. Если вам не нужна слишком большая вычислительная мощность, уменьшите количество DWU, и ваши расходы снова сократятся. Вычислительная мощность увеличивается пропорционально количеству DWU. Удвоение количества DWU удваивает вычислительные ресурсы. 
- В периоды, когда вам не нужно выполнять запросы, например вечером или в выходные дни, приостановите работу вычислительных ресурсов. Все выполняемые запросы будут отменены, а все DWU, выделенные для хранилища данных, — освобождены. Хранилище данных и дальше будет работать, а вот плата за вычислительные ресурсы взиматься не будет. Когда вам снова потребуется выполнять запросы, например утром в понедельник, вы сможете возобновить работу вычислительных ресурсов. 

## Вычисления с массовым параллелизмом и индексы columnstore для рекордной производительности
Хранилище данных SQL использует архитектуру Майкрософт для вычислений с массовым параллелизмом (MPP) и индексы columnstore (технология SQL Server), что обеспечивает рекордную производительность решения.

> [AZURE.NOTE]Вычисления с параллелизмом позволяют использовать для решения проблем больших данных подход «разделяй и властвуй». Данные разделяются и распределяются между несколькими вычислительными ресурсами, и каждый вычислительный ресурс параллельно обрабатывает свою часть данных.

- Основной секретный ингредиент —это технология распределенных запросов Майкрософт. Хранилище данных SQL использует усовершенствованный оптимизатор запросов, который определяет способ оптимизации распределенных запросов с учетом оценки стоимости операций запроса. В хранилище также используются дополнительные алгоритмы и методы, которые эффективно перемещают данные между вычислительными ресурсами в зависимости от потребности выполнения запроса.
- Индексы columnstore — ключ к достижению быстрой обработки запросов к хранилищу данных. В хранилище, где принцип хранения данных основан на столбцах, индексы columnstore обеспечивают пятикратное увеличение сжатия данных (по сравнению с хранилищем с традиционным, построчным, хранением) и десятикратное увеличение производительности запросов. Запросы к хранилищу данных прекрасно работают с индексами columnstore, потому что они часто сканируют всю таблицу или весь раздел таблицы. OLTP-запросы, напротив, отлично работают с индексами двоичного дерева, так как они ищут конкретные строки в таблице.


## Взаимодействие гибридного облака с сервером SQL Server корпоративного класса
Хранилище данных SQL основано на надежной реляционной СУБД SQL Server и включает компоненты, которые обязательно должны присутствовать в хранилище данных корпоративного класса, в частности хранимые процедуры, пользовательские функции, секционирование таблиц, индексы и параметры сортировки.

> [AZURE.NOTE]Если вы уже знакомы с Transact-SQL, вы можете легко применить свои знания при работе с хранилищем данных SQL. Примеры из документации пригодятся не только новичкам, но и опытным пользователям.

- Хранилище данных SQL использует язык Transact-SQL (SQL Server), индекс columnstore, технологию PolyBase и архитектуру вычислений с массовым параллелизмом (MPP), которая относится к системе аналитических платформ. Соединенные воедино, эти компоненты формируют уникальное интегрированное хранилище данных, которое работает по принципу «платформа как служба» (PaaS).  

- Используя Transact-SQL и схожесть функций между SQL Server, хранилищем данных SQL, базой данных SQL и системой аналитических платформ, можно разрабатывать решения, которые соответствуют вашим требованиям к хранению и обработке данных. Вы можете выбрать место для хранения данных, исходя из требований к производительности, безопасности и масштабированию, а затем переносить данные между локальным и облачным хранилищами по мере необходимости.


## Запросы к реляционным и нереляционным данным
Технология Polybase позволяет отправлять запросы к нереляционным данным, которые находятся в хранилище больших двоичных объектов Azure или в файловой системе Hadoop (HDFS), как если бы они находились в обычной таблице. С помощью Polybase можно отправлять запросы к нереляционным данным или импортировать такие данные в хранилище данных SQL.

> [AZURE.NOTE]Технология Polybase проста в применении и позволяет использовать данные из различных источников, выполняя уже знакомые вам команды T-SQL. Для работы с Hadoop вам не нужно изучать HiveQL или другие языки.

- Интеграция Polybase не зависит используемых технологий. Polybase предоставляет одинаковые компоненты и функциональные возможности для всех поддерживаемых дистрибутивов Hadoop. Технология Polybase позволяет читать данные в различных форматах, включая файлы с разделителями и ORC-файлы.
- Для доступа к нереляционным данным технология PolyBase использует внешние таблицы. Определения таблиц хранятся в хранилище данных SQL, а данные — во внешнем хранилище больших двоичных объектов Azure или Hadoop.


## Безопасное развертывание инфраструктуры без затрат на обслуживание
Хранилище данных SQL легко развертывается в течение нескольких секунд. Эта служба является полностью управляемой и не требует временных затрат на обслуживание и установку программных исправлений. В хранилище данных SQL предусмотрены автоматическое резервное копирование и самостоятельное восстановление: создавая точки восстановления базы данных, служба автоматически создает резервную копию ваших данных в хранилище Azure.

### Как задать вопрос команде разработчиков службы управления API?

-	Свои вопросы можно разместить на нашем [форуме MSDN для службы управления API](https://social.msdn.microsoft.com/forums/azure/home?forum=azureapimgmt).
-	Вы также можете отправить нам электронное письмо по адресу: `apimgmt@microsoft.com`.

### Что означает, если компонент находится в предварительной версии?

Компонент, находящийся в предварительной версии, функционально завершен, но все еще доступен, так как мы активно собираем отзывы о нем. Вполне возможно, что в ответ на отзывы клиентов мы внесем критические изменения, поэтому мы рекомендуем не использовать эту версию в производственной среде. Если у вас есть отзыв о компонентах, находящихся в предварительной версии, сообщите нам, используя один из механизмов, описанных в разделе [Как задать вопрос команде разработчиков службы управления API?](#how-can-i-ask-a-question-to-the-api-management-team)

### Как можно защитить подключение шлюза службы управления API к моим службам серверной части?

Поддерживается несколько вариантов защиты.

1. Использование обычной проверки подлинности HTTP. Дополнительные сведения см. в статье [Настройка параметров API](api-management-howto-create-apis.md#configure-api-settings).
2. Использование взаимной проверки подлинности SSL, как описано в разделе [Защита служб серверной части посредством проверки подлинности с помощью сертификата клиента в службе управления API Azure](api-management-howto-mutual-certificates.md).
3. Используйте утвержденный список IP-адресов в службах серверной части. Если у вас есть экземпляр службы управления API уровня "Стандартный" или "Премиум", IP-адрес шлюза остается постоянным, и его можно добавить в утвержденный список IP-адресов. IP-адрес экземпляра службы управления API можно получить на **панели мониторинга** на классическом портале Azure.
4. Экземпляр службы управления API можно подключить к виртуальной сети \(классической\). Дополнительные сведения см. в разделе [Настройка VPN-подключений в службе управления API Azure](api-management-howto-setup-vpn.md).

### Как скопировать экземпляр службы управления API в новый экземпляр?

Существует несколько вариантов копирования экземпляра службы управления API в новый экземпляр.

-	Используйте функцию архивации и восстановления службы управления API. Дополнительные сведения см. в разделе [Реализация аварийного восстановления с помощью функций архивации и восстановления службы в службе управления API Azure](api-management-howto-disaster-recovery-backup-restore.md).
-	Создайте собственную функцию архивации и восстановления с помощью [API REST службы управления API](https://msdn.microsoft.com/library/azure/dn776326.aspx), чтобы сохранить и восстановить нужные объекты из экземпляра службы.
-	Скачайте конфигурацию службы с помощью Git и отправьте ее в новый экземпляр. Дополнительные сведения см. в разделе [Сохранение и настройка конфигурации службы управления API с помощью Git](api-management-configuration-repository-git.md).

### Можно ли управлять экземпляром службы управления API программными средствами?

Да, им можно управлять с помощью [API REST службы управления API](https://msdn.microsoft.com/library/azure/dn776326.aspx) и командлетов PowerShell для [развертывания службы](https://msdn.microsoft.com/library/mt619282.aspx) и [управления службой](https://msdn.microsoft.com/library/mt613507.aspx).

### Как добавить пользователя в группу администраторов?

В настоящее время группа администраторов ограничена пользователями, которые входят в систему с помощью классического портала Azure как администраторы или соадминистраторы для подписки Azure, содержащей экземпляр службы управления API. Пользователей, созданных на портале публикации, нельзя назначить в качестве администраторов или добавить в группу администраторов.


### Почему политика, которую я хочу добавить, не включена в редакторе политик?

Если политика, которую требуется добавить, не включена, убедитесь, что вы находитесь в правильной области действия этой политики. Каждая инструкция политики предназначена для использования в определенных областях и разделах политики. Чтобы просмотреть разделы политики и области ее действия, ознакомьтесь с разделом **Использование** для этой политики в [Справочнике по политикам](https://msdn.microsoft.com/library/azure/dn894080.aspx).

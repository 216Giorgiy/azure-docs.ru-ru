# <a name="import-an-azure-functions-app-as-an-api"></a>Импорт приложения-функции Azure в качестве API

В этой статье показано, как импортировать приложение-функцию Azure в качестве API. Также здесь показано, как протестировать программный интерфейс Управления API Azure.

В этой статье раскрываются следующие темы:

> [!div class="checklist"]
> * импорт приложения-функции в качестве API;
> * проверка API на портале Azure;
> * проверка API на портале разработчика.

## <a name="prerequisites"></a>Предварительные требования

+ Выполните задачи в кратком руководстве по [созданию экземпляра службы "Управление API Azure"](get-started-create-service-instance.md).
+ Убедитесь, что в подписке есть приложение-функция Azure. Дополнительные сведения см. в разделе [Создание приложения-функции](../azure-functions/functions-create-first-azure-function.md#create-a-function-app).
+ [Создание определения OpenAPI](../azure-functions/functions-openapi-definition.md) приложения-функции Azure.

## <a name="create-api"></a>Импорт и публикация API серверной части

1. В разделе **Управление API** выберите **Интерфейсы API**.
2. В списке **Добавление нового API** выберите **Приложение функций**.

    ![Приложение функций](./media/import-function-app-as-api/function-app-api.png)
3. Выберите **Обзор**, чтобы просмотреть список приложений функций в подписке.
4. Выберите приложение. Служба "Управление API" найдет файл Swagger, связанный с выбранным приложением, извлечет и импортирует его. 
5. Добавьте суффикс URL-адреса API. Суффиксом называется имя, которое идентифицирует конкретный API в экземпляре службы "Управление API". Суффикс должен быть уникальным в этом экземпляре службы "Управление API".
6. Чтобы опубликовать API, его нужно связать с определенным продуктом. В нашем случае используется продукт **Без ограничений**. Если планируется опубликовать API и предоставить разработчикам доступ к нему, добавьте его в продукт. Добавить API в продукт можно при его создании или позже.

    Продуктами называют ассоциации из одного или нескольких API. В них можно объединить определенное количество API и предлагать их разработчикам через портал разработчика. Чтобы получить доступ к API, разработчикам необходимо сначала подписаться на продукт. После этого они получат ключ подписки, который подходит для любого API в этом продукте. Создавая экземпляр службы "Управление API", вы автоматически становитесь его администратором. Администраторы по умолчанию подписаны на все продукты.

    По умолчанию каждый экземпляр API управления поставляется с двумя демонстрационными продуктами:

    * **Starter**
    * **Unlimited**   
7. Нажмите кнопку **Создать**.

## <a name="populate-azure-functions-keys-in-azure-api-management"></a>Заполнение ключей Функций Azure в службе управления API Azure

Если импортируемые приложения-функции Azure защищены с помощью ключей, служба "Управление API" автоматически создает *именованные значения* для них. Но служба "Управление API" не заполняет записи секретными данными. Для каждой записи выполните следующие действия.  

1. Перейдите на вкладку **Именованные значения** в экземпляре службы "Управление API".
2. Выберите запись и щелкните **Показать значение** на боковой панели.

    ![Именованные значения](./media/import-function-app-as-api/apim-named-values.png)

3. Если содержимое в поле **Значение** имеет вид **код для \<имя функции Azure\>**, перейдите в меню **Приложения функций**, а затем перейдите к **Функции**.
4. Перейдите в раздел **Управление** и скопируйте соответствующий ключ на основе метода проверки подлинности приложения.

    ![Приложение функций — ключи копирования](./media/import-function-app-as-api/azure-functions-app-keys.png)

5. Вставьте ключ в текстовое поле **Значение** и затем нажмите кнопку **Сохранить**.

    ![Приложения-функции — вставка значений ключей](./media/import-function-app-as-api/apim-named-values-2.png)

## <a name="test-the-new-api-management-api-in-the-azure-portal"></a>Проверка нового программного интерфейса службы "Управление API" на портале Azure

Операции можно вызывать непосредственно на портале Azure. Использование портала Azure — удобный способ просмотра и тестирования операций с API.  

1. Выберите API, созданный на предыдущем шаге.
2. Откройте вкладку **Тест**.
3. Выберите операцию.

    На странице отобразятся поля для параметров запроса и для заголовков. Один из заголовков, **Ocp-Apim-Subscription-Key**, содержит ключ подписки для продукта, связанного с этим API. Как создатель экземпляра службы "Управление API", вы автоматически являетесь администратором, поэтому сведения о ключе будут заполнены автоматически. 
4. Нажмите кнопку **Отправить**.

    Служба серверной части вернет ответ **200 OK** и другие данные.

## <a name="call-operation"></a>Вызов операции с портала разработчика

Также операции можно вызвать с портала разработчиков, чтобы проверить API. 

1. Выберите API, который был создан на шаге [Импорт и публикация API серверной части](#create-api).
2. Щелкните **Портал разработчика**.

    Откроется сайт портала разработчика.
3. Выберите созданный **API**.
4. Выберите операцию, которую необходимо проверить.
5. Щелкните **Тестирование**.
6. Нажмите кнопку **Отправить**.
    
    После вызова операции портал разработчика отображает **состояние ответа**, **заголовки ответа** и **содержимое ответа**.

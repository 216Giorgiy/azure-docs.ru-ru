#Создание рабочих процессов расширенной кодировки с помощью конструктора рабочих процессов

##Обзор
**Конструктор рабочих процессов** - автономное средство, которое используется для создания рабочих процессов и диаграмм при кодировании с помощью **рабочего процесса Premium Media Encoder**.

Это средство также можно использовать для изменения [существующих рабочих процессов](media-services-workflow-designer.md#existing_workflows). 

>[AZURE.NOTE]Чтобы получить копию средства конструктора рабочих процессов, обратитесь в службу mepd@microsoft.com.


После создания файла рабочего процесса его можно передать как ресурс, а затем использовать для кодирования файлов мультимедиа. Сведения о кодировании при использовании **рабочего процесса Premium Media Encoder** с помощью **.NET** см. в разделе [Дополнительное кодирование с использованием рабочего процесса Premium Media Encoder](media-services-encode-with-premium-workflow.md).

##<a id="existing_workflows"></a>Изменение существующих рабочих процессов

Файлы рабочего процесса по умолчанию можно изменить с помощью средств конструктора. Файлы рабочего процесса по умолчанию можно получить [здесь](https://github.com/Azure/azure-media-services-samples/tree/master/Encoding%20Presets/VoD/MediaEncoderPremiumWorkfows). Папка также содержит описание этих файлов.


>[AZURE.NOTE] Дополнительные сведения будут доступны в ближайшее время.

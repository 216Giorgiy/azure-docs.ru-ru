# О VPN-устройствах для подключений виртуальной сети типа «сеть-сеть»

Для настройки VPN-подключения типа «сеть-сеть» требуется VPN-устройство. Подключения типа «сеть-сеть» можно использовать для создания гибридного облака или в случае, когда требуется безопасное подключение между локальной сетью и виртуальной сетью. В этой статье рассматриваются совместимые VPN-устройства и параметры конфигурации.

Важно знать, что помимо совместимых VPN-устройств, для подключений типа «сеть-сеть» также требуется общедоступный IPv4-адрес. Кроме того, потребуется выбрать тип шлюза, который будет поддерживать ваше решение. Не все VPN-устройства поддерживает все типы шлюза. См. раздел [VPN-шлюзы](vpn-gateway-about-vpngateways.md), чтобы проверить тип шлюза, необходимого для создания вашего решения.



## VPN-устройства

Для подключения «сеть-сеть» (S2S) мы утвердили набор стандартных VPN-устройств в сотрудничестве с поставщиками устройств. Список VPN-устройств, которые совместимы с виртуальной сетью, см. в разделе [Совместимые VPN-устройства](#compatible-vpn-devices) ниже. Все устройства в семействах устройств в этом списке должны работать с VPN-шлюзами Azure. Чтобы настроить VPN-устройство, см. пример конфигурации устройства, который соответствует семейству устройств.

Спецификации высокопроизводительного VPN-шлюза и VPN-шлюза с динамической маршрутизацией одинаковы, если не указано иное. Например, проверенные VPN-устройства, совместимые с VPN-шлюзами Azure с динамической маршрутизацией, также совместимы с новыми высокопроизводительными VPN-шлюзами Azure.

### Устройства, отсутствующие в списке совместимых устройств


Если вы не видите своего устройства в списке совместимых VPN-устройств и хотите использовать его для VPN-подключения, потребуется убедиться, что это устройство соответствует минимальным требованиям, приведенным в таблице [Требования к шлюзу](vpn-gateway-about-vpngateways.md#gateway-requirements). Устройства, соответствующие минимальным требованиям, также должны работать с виртуальной сетью. Чтобы получить дополнительную информацию и инструкции по настройке, обратитесь к изготовителю устройства.


## Изменение примеров конфигурации устройств

После скачивания предоставленного примера конфигурации VPN-устройства некоторые значения необходимо заменить в соответствии с параметрами вашей среды.

**Изменение примера:**

1. Откройте пример с помощью блокнота. 
1. Найдите и замените все строки <*text*> значениями, отражающими свойства вашей среды. Не забудьте добавить < and >. Если указывается имя, оно должно быть уникальным. Если команда не работает, обратитесь к документации изготовителя устройства.

| **Пример текста** | **Изменить на** |
|----------------------------------|----------------------------------------------------------------------------------------------------------------------|
| &lt;RP\_OnPremisesNetwork&gt; | Выбранное имя для данного объекта. Пример: myOnPremisesNetwork. |
| &lt;RP\_AzureNetwork&gt; | Выбранное имя для данного объекта. Пример: myAzureNetwork. |
| &lt;RP\_AccessList&gt; | Выбранное имя для данного объекта. Пример: myAzureAccessList. |
| &lt;RP\_IPSecTransformSet&gt; | Выбранное имя для данного объекта. Пример: myIPSecTransformSet. |
| &lt;RP\_IPSecCryptoMap&gt; | Выбранное имя для данного объекта. Пример: myIPSecCryptoMap. |
| &lt;SP\_AzureNetworkIpRange&gt; | Укажите диапазон. Пример: 192.168.0.0. |
| &lt;SP\_AzureNetworkSubnetMask&gt; | Укажите маску подсети. Пример: 255.255.0.0. |
| &lt;SP\_OnPremisesNetworkIpRange&gt; | Укажите локальный диапазон. Пример: 10.2.1.0. |
| &lt;SP\_OnPremisesNetworkSubnetMask&gt; | Укажите локальную маску подсети. Пример: 255.255.255.0. |
| &lt;SP\_AzureGatewayIpAddress&gt; | Эта информация относится к виртуальной сети и находится на портале управления в поле **IP-адрес шлюза**. |
| &lt;SP\_PresharedKey&gt; | Эта информация относится к виртуальной сети и находится на портале управления в разделе «Управление ключами». |

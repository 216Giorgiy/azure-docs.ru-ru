# Управление ресурсами Azure с помощью портала предварительной версии Azure

## Введение

Исторически управление ресурсом (управляемой пользователем сущностью, например сервером баз данных, базой данных или веб-сайтом) в Microsoft Azure всегда подразумевало выполнение операций только с одним ресурсом одновременно. При использовании сложного приложения, состоящего из нескольких ресурсов, управление этим приложением становится сложной задачей. На портале предварительной версии Azure можно создать группы ресурсов для совместного управления всеми ресурсами приложения. Группа ресурсов — это новое понятие в Azure, служащее границей жизненного цикла для каждого ресурса, содержащегося в этой группе.

Группы ресурсов позволяют одновременно управлять всеми ресурсами приложения. Группы ресурсов становятся доступны благодаря новому средству управления — диспетчеру ресурсов Azure. Диспетчер ресурсов Azure позволяет сгруппировать несколько ресурсов в логическую группу, которая служит границей жизненного цикла для каждого ресурса, содержащегося в группе. Обычно группа будет содержать ресурсы, относящиеся к конкретному приложению. Например, группа может содержать ресурс веб-сайта, на котором размещен открытый веб-сайт, базу данных SQL для хранения реляционных данных, используемых сайтом, и учетную запись хранения, в которой хранятся нереляционные активы.

Вот краткий обзор использования групп ресурсов на портале Azure.

## Создание групп ресурсов

При создании ресурса на портале этот ресурс всегда создается в составе группы ресурсов. Можно создать новую группу ресурсов или использовать существующую группу ресурсов при создании потока. <br><br />

![Создать группу ресурсов](./media/resource-group-portal/1_createWebsite.png)

При создании приложения, состоящего из нескольких совместно работающих ресурсов (например, веб-сайт + база данных), оно всегда создается в собственной группе ресурсов, что позволяет управлять жизненным циклом всех связанных с приложением ресурсов через группу. По мере развития приложения можно добавлять дополнительные ресурсы или удалять их из группы ресурсов.

![Создать группу ресурсов](./media/resource-group-portal/2_createWSandDB.png)

## Просмотр групп ресурсов

Чтобы просмотреть все группы ресурсов, щелкните панель перехода в левой части экрана. Группа ресурсов содержит выноску со всеми сведениями о конкретной группе ресурсов. Выноска группы ресурсов также обеспечит единое представление информации выставления счетов и мониторинга для всех ресурсов в группе ресурсов.

Раздел сводки содержит визуальную схему всех ресурсов в группе, а также показывает ресурсы в других группах, связанных с этой. На схеме ресурсов также показывается состояние каждого ресурса. ![Сводка по группе ресурсов](./media/resource-group-portal/3_1BrowseRGs.png)

Для части схемы ресурсов можно увеличить размер отображения и показать все ресурсы, содержащиеся в группе, а также связанные ресурсы из других групп. Эту часть можно прикрепить к правой стороне, копируя в нее содержание.

![ПИН-код](./media/resource-group-portal/3_2BrowseRGs.png)

Щелкнув схему ресурсов, можно открыть список ресурсов. В этом представлении будут перечислены все ресурсы в группе ресурсов или связанные с ней. При щелчке на любом из этих ресурсов откроется соответствующий модуль.

![Посмотреть ресурсы](./media/resource-group-portal/3_3BrowseRGs.png)

## Добавление ресурсов в группы ресурсов

Ресурсы в группу ресурсов можно добавить с помощью команды **Add** в модуле группы ресурсов. Выполнение этой последовательности действий позволит добавить другие ресурсы в группу ресурсов.

![Добавить ресурсы](./media/resource-group-portal/4_AddResource.png)

Примечание. Не рекомендуется помещать командный проект в одну группу с другими ресурсами Azure. При создании командного проекта в новой учетной записи и группе создайте веб-сайт. По умолчанию группой сайта станет последняя используемая группа (VSO), и ресурсы разработки/времени выполнения будут добавляться в эту же группу.

## Удаление групп ресурсов

Так как группы ресурсов позволяют управлять жизненным циклом всех содержащихся в них ресурсов, при удалении группы будут удалены и все ее ресурсы. Из группы ресурсов можно также удалить отдельные ресурсы. При удалении группы ресурсов необходимо соблюдать осторожность, поскольку с этой группой могут быть связаны другие ресурсы. Вы можете просмотреть связанные ресурсы на схеме ресурсов и принять соответствующие меры, чтобы избежать нежелательных последствий удаления группы ресурсов.

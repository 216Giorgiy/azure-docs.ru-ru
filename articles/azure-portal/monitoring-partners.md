## AlertLogic Log Manager
AlertLogic Log Manager собирает журналы виртуальных машин, приложений и платформы Azure для анализа безопасности и хранения. Сюда входит сбор журналов аудита Azure через API Azure Insights. Эта информация используется для обнаружения несанкционированных операций и соответствия нормативным требованиям.

[Перейдите к документации.][alertlogic-doc]

## AppDynamics
Служба управления производительностью приложений (APM) AppDynamics позволяет владельцам приложений быстро устранять узкие места, снижающие производительность, и оптимизировать производительность приложений, которые работают в среде Azure. AppDynamics APM легко интегрируется с Azure Marketplace и может быть использована для мониторинга облачных служб Azure (PaaS) (включая веб-роли и рабочие роли), виртуальных машин (IaaS), средств обнаружения удаленных служб (служебная шина Microsoft Azure), удаленных служб Azure очереди Microsoft Azure (большой двоичный объект Azure), очереди Azure (служебная шина (Майкрософт)), хранилища данных и хранилища BLOB-объектов Microsoft Azure.

[Перейдите к документации.][appdynamics-doc]

## CloudMonix
CloudMonix предлагает службы мониторинга, автоматизации и самовосстановления для платформы Microsoft Azure.

[Перейдите к документации.][cloudmonix-doc]

## Cloudyn
Cloudyn осуществляет управление и оптимизацию многоплатформенных, гибридных облачных развертываний, позволяя предприятиям полностью реализовать свой облачный потенциал. Это решение SaaS позволяет следить за использованием, производительностью и затратами, а также получать полезные сведения и практические рекомендации для интеллектуальной оптимизации и управления облачной инфраструктурой. Cloudyn обеспечивает учет с помощью точного взимания средств за использование и управления иерархическим распределением затрат. Решение Cloudyn интегрировано со службой мониторинга Azure, чтобы обеспечить полезные сведения и практические рекомендации для оптимизации развертывания Azure.

[Перейдите к документации.][cloudyn-doc]

## Datadog
Datadog является ведущей мировой службой мониторинга для облачных приложений, которая объединяет данные с серверов, из баз данных, инструментов и служб, чтобы создать единое представление всего стека. Эти возможности предоставляются на платформе аналитики данных на основе SaaS, позволяющий командам специалистов по разработке и операциям работать совместно, чтобы избегать простоев, устранять проблемы производительности и обеспечивать своевременное завершение циклов разработки и развертывания. Благодаря интеграции Datadog и Azure можно собирать и просматривать метрики по инфраструктуре, сопоставлять метрики виртуальной машины с метриками уровня приложения, а также секционировать и фрагментировать метрики с помощью любой комбинации свойств и настраиваемых тегов.

[Перейдите к документации.][datadog-doc]

## Dynatrace
Dynatrace OneAgent интегрируется с виртуальными машинами и службами приложений Azure посредством соответствующих механизмов расширений Azure. Таким образом можно собирать метрики производительности узлов, сети и служб. Помимо простого отображения метрик, мы можем отобразить комплексное представление сред, показывающая транзакции со стороны клиента на уровне базы данных. Корреляция проблем на основе Application Insights и полностью интегрированный анализ первопричин, включая полезные сведения о коде и базах данных на уровне методов, намного упрощают устранение неполадок и оптимизацию производительности.

[Перейдите к документации.][dynatrace-doc]

## OpsGenie
OpsGenie выполняет роль диспетчера оповещений, создаваемых Azure. OpsGenie определяет, кого нужно поставить в известность, исходя из дежурных расписаний и правил эскалации, и отправляет уведомляет их посредством электронной почты, текстовых сообщений (SMS), телефонных звонков или push-уведомлений. Проще говоря, Azure создает оповещения об обнаруженных проблемах, а OpsGenie гарантирует, что ими займутся соответствующие сотрудники.

[Перейдите к документации.][opsgenie-doc]

## PagerDuty
PagerDuty, ведущее решение по управлению инцидентами, предоставило первоклассную поддержку оповещений на основе метрик Azure. В настоящее время, помимо уведомлений на основе метрик уровня платформы для служб Azure, PagerDuty поддерживает уведомления на основе оповещений Azure Applications Insights, уведомлений автомасштабирования и событий журнала аудита. Эти усовершенствования позволяют пользователям лучше понять, как работает основная платформа Azure, и использовать все преимущества управления инцидентами в PagerDuty, чтобы получать ответы в режиме реального времени. Расширенная интеграция Azure возможна благодаря веб-перехватчикам webhook, обеспечивающим быструю и простую установку и настройку.

[Перейдите к документации.][pagerduty-doc]

## Надстройка Splunk для Microsoft Azure
Надстройка Splunk для Microsoft Azure собирает ценные данные диагностики, производительности, безопасности и аудита для инфраструктуры и веб-сайтов, работающих в Microsoft Azure.

Информация о производительности и диагностические сведения собираются из таблиц и больших двоичных объектов службы хранилища Azure. Данные аудита собираются посредством API событий Azure Insights. Данные групп безопасности сети собираются из больших двоичных объектов службы хранилища Azure. В надстройку включено несколько готовых панелей.

[Перейдите к документации.][splunk-doc]

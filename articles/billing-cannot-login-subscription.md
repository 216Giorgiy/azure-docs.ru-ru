# Не удается выполнить вход для управления подпиской Azure

В этой статье описываются некоторые из наиболее распространенных методов устранения проблем со входом.

> [AZURE.NOTE] Если в ходе работы с этой статьей вам потребуется дополнительная помощь, [обратитесь в службу поддержки](https://portal.azure.com/?#blade/Microsoft_Azure_Support/HelpAndSupportBlade), которая поможет вам быстро устранить проблему.

## Порталы Azure

| Имя | Описание | URL-адрес |
|---|---|---|
| Портал Azure |Централизованное место, где можно подготавливать ресурсы Azure и управлять ими | [https://portal.azure.com](https://portal.azure.com) |
| Классический портал Azure | Старый портал Azure |[https://manage.windowsazure.com](https://manage.windowsazure.com) |
| Центр учетных записей Azure | Централизованное место, где можно отслеживать использование службы Azure и управлять подпиской |[https://account.windowsazure.com/Subscriptions](https://account.windowsazure.com/Subscriptions) ||

## Симптом: страница зависает в состоянии загрузки

Эта проблема может возникать из-за ошибки в работе Интернет-браузера.

Чтобы устранить эту проблему, попробуйте следующие методы, соблюдая заданный порядок. После выполнения каждого метода повторите попытку подключения к странице регистрации на портале.

-	Обновите страницу.
-	Воспользуйтесь другим браузером.
-	Если вы используете Microsoft Internet Explorer, перейдите на портал Azure с помощью режима просмотра InPrivate. Для этого выполните следующие действия.

	О. Щелкните **Сервис** ![Кнопка "Сервис"](./media/billing-cannot-login-subscription/Toolsbutton.png) > **Безопасность** > **Просмотр InPrivate**.

	B. Перейдите на [портал Azure](https://portal.azure.com) или [классический портал Azure](https://manage.windowsazure.com) и выполните вход.

## Симптом: появляется сообщение об ошибке "Подписки не найдены"

Эта проблема может возникать, если учетная запись не обладает достаточными правами пользователя. Администратор учетной записи имеет доступ только к [центру управления учетной записью](https://account.windowsazure.com/), а администраторы служб и соадминистраторы имеют доступ только к [порталу Azure](https://portal.azure.com) и [классическому порталу Azure](https://manage.windowsazure.com/).

**Сценарий 1: сообщение об ошибке получено на портале Azure или на классическом портале Azure**

Для устранения этой проблемы [добавьте для учетной записи роль соадминистратора или владельца](billing-add-change-azure-subscription-administrator.md).

**Сценарий 2: сообщение об ошибке получено в центре управления учетной записью Azure**

Проверьте, является ли используемая учетная запись учетной записью администратора. Чтобы проверить, кто является администратором учетной записи, выполните следующие действия:

1.	Выполните вход на [портал Azure](https://portal.azure.com).
2.	В главном меню выберите **Подписка**.
3.	Выберите подписку, которую требуется проверить, а затем щелкните **Параметры**.
4.	Выберите **Свойства**. Администратор учетной записи подписки отобразится в поле **Администратор учетной записи**.

## Симптом: выполняется автоматический вход от имени другого пользователя

Эта проблема может возникать, если вы используете в браузере более одной учетной записи пользователя.

Для устранения этой проблемы воспользуйтесь одним из указанных ниже способов.

- Выйдите из портала, а затем снова войдите, используя необходимую учетную запись.
-	Очистите кэш и удалите файлы cookie Интернета. Для этого в браузере Internet Explorer щелкните **Сервис** ![Кнопка "Сервис"](./media/billing-cannot-login-subscription/Toolsbutton.png) > **Свойства браузера** > **Удалить**. Убедитесь, что установлены флажки для временных файлов, файлов cookie, паролей и журнала браузера. Затем нажмите кнопку "Удалить".

-	Выполните сброс параметров Internet Explorer, чтобы отменить какие-либо персональные настройки. Для этого щелкните **Сервис** ![Кнопка "Сервис"](./media/billing-cannot-login-subscription/Toolsbutton.png)> **Свойства браузера** > **Дополнительно**, установите флажок **Удалить личные настройки** и нажмите кнопку **Сброс**.

-	Перейдите на портал Azure в режиме просмотра InPrivate. Для этого щелкните **Сервис** ![Кнопка "Сервис"](./media/billing-cannot-login-subscription/Toolsbutton.png) > **Безопасность** > **Просмотр InPrivate**.

## Учетная запись Майкрософт или учетная запись организации

Учетная запись Майкрософт представляет собой адрес электронной почты, который вы используете вместе с паролем для входа в любую программу или службу Windows Live, например Outlook, Hotmail, MSN или OneDrive. Вы можете настроить учетную запись Майкрософт с использованием любого принадлежащего вам адреса электронной почты, включая корпоративную электронную почту. Дополнительные сведения см. на странице [www.microsoft.com/account](http://www.microsoft.com/account).

Если ваша учетная запись связана с учетной записью организации, выберите подходящий вариант входа, как показано ниже. Дополнительные сведения об использовании учетной записи организации см. в статье [Регистрация организации в Azure](./active-directory/sign-up-organization.md).

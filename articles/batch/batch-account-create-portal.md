# Создание учетной записи пакетной службы Azure на портале Azure и управление ею

> [AZURE.SELECTOR]
- [Портал Azure](batch-account-create-portal.md)
- [Библиотека .NET для управления пакетной службой](batch-management-dotnet.md)

На [портале Azure][azure_portal] есть средства, с помощью которых можно создать учетную запись пакетной службы Azure и управлять ею. Эту запись можно использовать для обработки объемных параллельных рабочих нагрузок. В этой статье описывается процедура создания учетной записи пакетной службы с помощью портала, а также некоторые важные параметры и свойства этой учетной записи. Например, чтобы взаимодействовать с API пакетной службы, приложениям и службам, созданным с использованием пакетной службы, требуется URL-адрес вашей учетной записи и ключ доступа, которые находятся на портале Azure.

>[AZURE.NOTE] В настоящее время портал Azure поддерживает ряд функций пакетной службы, в частности создание учетных записей пакетной службы, управление параметрами и свойствами этих учетных записей, а также создание и мониторинг пулов и заданий. Все функции пакетной службы доступны разработчикам через API пакетных служб.

## Создание учетной записи Пакетной службы

1. Войдите на [портал Azure][azure_portal].

2. Щелкните **Создать** > **Виртуальные машины** > **Пакетная служба**.

	![Пакетная служба в Marketplace][marketplace_portal]

3. Отобразится колонка **Создать учетную запись пакетной службы**. Описания каждого элемента колонки приведены ниже в пунктах *а*–*е*.

    ![Создание учетной записи Пакетной службы][account_portal]

	а. **Имя учетной записи** — уникальное имя учетной записи пакетной службы. Это имя должно быть уникальным в пределах региона Azure, в котором создана учетная запись (см. пункт *Расположение* ниже). Оно может содержать только строчные буквы и цифры и не должно быть больше 3–24 символов в длину.

	b. **Подписка** — подписка, в которой необходимо создать учетную запись пакетной службы. Если у вас только одна подписка, она будет выбрана по умолчанию.

	c. **Группа ресурсов** — существующая группа ресурсов для новой учетной записи пакетной службы (при необходимости можно создать новую группу ресурсов).

	d. **Расположение** — регион Azure, в котором необходимо создать учетную запись пакетной службы. Будут отображаться только те регионы, которые поддерживаются для вашей подписки и группы ресурсов.

    д. **Учетная запись хранения** (необязательно) — учетная запись хранения **общего назначения**, которую вы связываете с новой учетной записью пакетной службы. [Пакеты приложений](batch-application-packages.md) (компонент пакетной службы) используют связанную учетную запись хранения для хранения и извлечения пакетов приложений. Дополнительные сведения см. в статье [Развертывание приложения с помощью пакетов приложений пакетной службы Azure](batch-application-packages.md).

     > [AZURE.IMPORTANT] При повторном создании ключей в связанной учетной записи хранения необходимо учитывать специальные рекомендации. Дополнительные сведения см. в разделе [Рекомендации, касающиеся учетных записей пакетной службы](#considerations-for-batch-accounts) ниже.

4. Щелкните **Создать**, чтобы создать учетную запись.

  На портале будет указано, что выполняется **развертывание** учетной записи, по завершении которого в области *уведомлений* появится сообщение об **успешном развертывании**.

## Просмотр свойств учетной записи пакетной службы

В колонке учетной записи пакетной службы приведено несколько свойств учетной записи. Кроме того, здесь можно открыть дополнительные параметры, например ключи доступа, квоты, параметры управления пользователями и связывания с учетной записью хранения.

* **URL-адрес учетной записи пакетной службы** — этот URL-адрес предоставляет доступ к учетной записи пакетной службы при использовании интерфейсов API, например [REST API пакетной службы][api_rest], и клиентской библиотеки [.NET для пакетной службы][api_net]. Он указывается в следующем формате:

    `https://<account_name>.<region>.batch.azure.com`

* **Ключи доступа**. Чтобы просмотреть ключи доступа учетной записи пакетной службы или управлять ими, необходимо открыть колонку **Управление ключами**. Для этого щелкните значок ключа или последовательно выберите **Все параметры** > **Ключи**. Ключ доступа требуется для взаимодействия с API пакетной службы, например [REST API пакетной службы][api_rest], и клиентской библиотекой [.NET для пакетной службы][api_net].

    ![Ключи учетной записи Пакетной службы][account_keys]

* **Все параметры**. Чтобы управлять всеми параметрами учетной записи пакетной службы или просмотреть ее свойства, необходимо открыть колонку **Параметры**. Для этого щелкните **Все параметры**. Эта колонка предоставляет доступ ко всем параметрам и свойствам учетной записи. В частности, с ее помощью можно просмотреть квоты для учетной записи, выбрать учетную запись хранилища Azure, чтобы связать ее с учетной записью пакетной службы, и управлять пользователями.

    ![Колонки параметров и свойств учетной записи пакетной службы][5]

## Рекомендации, касающиеся учетных записей пакетной службы

* Чтобы создавать учетные записи пакетной службы и управлять ими, вы можете также использовать [командлеты PowerShell пакетной службы](batch-powershell-cmdlets-get-started.md) и библиотеку [.NET для управления пакетной службой](batch-management-dotnet.md).

* За использование самой учетной записи пакетной службы плата не взимается. Плата взимается за использование вычислительных ресурсов Azure, которые используют решения пакетной службы, а также за ресурсы, используемые другими службами при выполнении рабочих нагрузок. Например, плата взимается за вычислительные узлы в пулах, а если вы используете [пакеты приложений](batch-application-packages.md), — за ресурсы службы хранилища Azure, с помощью которых хранятся версии пакета приложения. Дополнительные сведения см. на странице с [ценами на пакетную службу][batch_pricing].

* Можно запустить несколько рабочих нагрузок пакетной службы в одной учетной записи пакетной службы или распределить рабочие нагрузки между учетными записями пакетной службы в разных регионах Azure.

* Если выполняется несколько крупных рабочих нагрузок пакетной службы, не забывайте об определенных [квотах и ограничениях пакетной службы](batch-quota-limit.md), действующих в отношении вашей подписки Azure и каждой учетной записи пакетной службы. Текущие квоты для учетной записи пакетной службы отображаются на портале в свойствах учетной записи.

* Если вы связываете учетную запись хранения с учетной записью пакетной службы, необходимо учитывать некоторые рекомендации относительно повторного создания ключей доступа к учетной записи хранения. Следует повторно создавать только один ключ учетной записи хранения за раз. Для этого щелкните **Синхронизировать ключи** в колонке связанной учетной записи хранения, подождите 5 минут, чтобы ключи распространились на вычислительные узлы пулов, а затем при необходимости повторно создайте и синхронизируйте следующий ключ. При попытке повторного создания двух ключей одновременно вычислительные узлы не смогут синхронизировать ни один из ключей, и, как результат, они потеряют доступ к учетной записи хранения.

  ![Повторное создание ключей учетной записи хранения][4]

> [AZURE.IMPORTANT] Сейчас пакетная служба поддерживает *только* тип учетной записи хранения **общего назначения**, как описано в шаге 5 раздела [Создайте учетную запись хранения](../storage/storage-create-storage-account.md#create-a-storage-account) в статье [Об учетных записях хранения Azure](../storage/storage-create-storage-account.md). При связывании учетной записи хранения Azure с учетной записью пакетной службы используйте *только* учетные записи хранения **общего назначения**.

# <a name="eventstore-service-overview"></a>Общие сведения о службе EventStore

>[!NOTE]
>В версии Service Fabric 6.2 интерфейсы API EventStore находятся на этапе предварительной версии для кластеров Windows, работающих только в Azure. Мы работаем над переносом этих функциональных возможностей в Linux, а также в изолированные кластеры.

## <a name="overview"></a>Обзор

Представленная в версии 6.2 служба EventStore — это средство мониторинга в Service Fabric, позволяющее проверить состояние кластера или рабочих нагрузок в данный момент времени. Служба EventStore предоставляет события Service Fabric через API-интерфейсы, на которые можно осуществлять вызовы. Эти API-интерфейсы EventStore позволяют выполнять запросы непосредственно на кластер, чтобы получить данные диагностики по любой сущности в кластере. Их следует использовать, чтобы:
* Диагностировать проблемы при разработке, тестировании или во время использования конвейера для мониторинга.
* Проверить, правильно ли обрабатываются действия управления, предпринимаемые в кластере.

Полный список событий, доступных в EventStore, см. в [этом](service-fabric-diagnostics-event-generation-operational.md) разделе.

Службу EventStore можно запрашивать для событий, доступных для каждой сущности и типа сущности в кластере. Это означает, что вы можете запрашивать события на следующих уровнях.
* Кластер. Все события уровня кластера.
* Узлы. Все события уровня узла.
* Узел. События для одного узла на основе `nodeName`.
* Приложения. Все события уровня приложения.
* Приложение. События для одного приложения.
* Службы. События из всех служб в кластерах.
* Служба. События из определенной службы.
* Разделы. События из всех разделов.
* Раздел. События из определенного раздела.
* Реплики. События из всех реплик или экземпляров.
* Реплика. События из конкретной реплики или экземпляра.


Служба EventStore также имеет возможность корреляции событий в кластере. Просмотрев события, записанные в то же время из различных объектов, которые могут влиять друг на друга, служба EventStore может связать эти события, что позволит определить причины возникновения действий в кластере. Например, если одно из приложений становится неработоспособным без принудительных изменений, EventStore будет также рассматривать другие события, предоставляемые платформой, и сопоставлять их с событием `NodeDown`. Это позволяет быстрее обнаружить сбой и выполнить анализ первопричин.

Чтобы приступить к работе с помощью службы EventStore, ознакомьтесь со статьей [Query EventStore APIs for cluster events](service-fabric-diagnostics-eventstore-query.md) (Запрос к API EventStore для события кластера).

# Разворачивайте приложения Service Fabric везде на Windows Server или Linux
Service Fabric позволяет создавать кластеры Service Fabric на любых виртуальных машинах или компьютерах под управлением Windows Server или Linux. Это позволяет разворачивать и запускать приложения Service Fabric в любой среде, как локально, так и у любого поставщика облачных решений, на наборе соединенных друг с другом компьютеров под управлением Windows Server или Linux. Service Fabric предоставит установочный пакет для создания кластеров Service Fabric. Ключевое преимущество функции развертывания где годно — отсутствие привязки к поставщику при сборке приложения с помощью Service Fabric, так как вы сами выбираете, где будут запускаться эти приложения. Это также позволяет вам получить доступ к более широкой клиентской базе, так как у различных клиентов могут быть разные требования к средам, в которых они хотят запускать приложения, например, потребности клиентов из сферы здравоохранения и финансовой отрасли могут отличаться от потребностей клиентов из сферы автомобилестроения или туризма. Выпуск технической предварительной версии этой функции ожидается в первом квартале 2016 года.

## Поддерживаемые операционные системы
Вы сможете создавать кластеры на виртуальных машинах или компьютерах под управлением следующих операционных систем: * Windows Server 2012 R2 * Windows Server 2016 * Linux

## Поддерживаемые языки программирования
Вы сможете писать приложения Service Fabric на любом из перечисленных языков программирования: * C# * Java

## Создание и настройка кластера
Установочный пакет, предоставленный Service Fabric, можно скачать из Центра загрузки Майкрософт. После загрузки этого пакета необходимо внести изменения в файл конфигурации, указав параметры кластера. Изменив параметры кластера, вы запустите сценарий установки, который создаст кластер на компьютерах, которые были указаны в параметрах кластера. Точные сведения о процессе установки будут доступны при выпуске предварительной версии этой функции в первом квартале 2016 года.

## Облачные развертывания и локальные развертывания
Процесс создания локального кластера Service Fabric аналогичен созданию кластера в любом облаке с набором виртуальных машин. Первоначальные шаги по подготовке виртуальных машин определяются облачной или локальной средой, которую вы используете. Однако после создания набора виртуальных машин, подключенных друг к другу, дальнейшие шаги по установке пакета Service Fabric, изменению параметров кластера и запуску сценариев создания кластера и управления им будут аналогичными для всех сред. Это гарантирует, что ваши знания и опыт эксплуатации и управления кластерами Service Fabric останутся актуальными при выборе новых сред размещения.

## Преимущества функции "Разворачивайте везде"
* Отсутствие привязки к поставщику.
* После написания приложения Service Fabric можно запускать в различных средах размещения без изменений или с минимальными изменениями.
* Знания о построении приложений Service Fabric будут актуальными при переходе с одной платформы размещения на другие.
* Опыт запуска кластеров Service Fabric будет актуальным при переходе от одной среды развертывания к другим.
* Широкий охват клиентов, не ограниченный средой размещения.
* Дополнительный уровень надежности и защиты от масштабных простоев путем перемещения служб в другую среду развертывания в случае отключения центра обработки данных или поставщика облачных решений.

## Различия между "Развертыванием везде" и размещенными в Azure кластерами Service Fabric
Запуск кластеров Service Fabric с использованием размещенной службы Azure предоставляет несколько преимуществ по сравнению с "Развертыванием везде", поэтому если у вас нет конкретных требований относительно того, где вы хотите запускать кластеры, рекомендуем запускать их как кластеры, размещенные в Azure. В Azure мы обеспечиваем интеграцию с другими функциями и службами Azure, которые упрощают эксплуатацию и управление кластером.

* Портал Azure: облегчает создание кластеров и управления ими.
* Диспетчер ресурсов Azure: использование ARM позволяет легко управлять всеми ресурсами, используемыми кластером, как единым целым, и упрощает отслеживание затрат и выставление счетов. 
* Диагностика: в Azure мы предоставляем интеграцию с системой диагностики Microsoft Azure (WAD) и оперативной аналитикой для диагностики.
* Автомасштабирование: для размещенных в Azure кластеров мы предоставляем встроенную функцию автомасштабирования. В других средах, в которых используется функция "Развернуть везде", вам придется реализовать собственную функцию автоматического масштабирования или выполнить масштабирование вручную с помощью API-интерфейсов, которые предоставляются Service Fabric для масштабирования кластеров.

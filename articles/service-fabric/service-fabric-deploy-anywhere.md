# Разворачивайте приложения Service Fabric везде на Windows Server или Linux
С функцией "Развертывание где угодно" Azure Service Fabric позволяет создавать кластеры Service Fabric на любых виртуальных машинах или компьютерах под управлением Windows Server или Linux. Это означает, что вы сможете разворачивать и запускать приложения Service Fabric в любой среде с набором подключенных друг к другу компьютеров под управлением Windows Server или Linux как локально, так и у любого поставщика облачных решений.

 Service Fabric предоставляет установочный пакет для создания кластеров Service Fabric. Ключевое преимущество функции развертывания где угодно — отсутствие привязки к поставщику при сборке приложения с помощью Service Fabric, так как вы сами выбираете, где будут запускаться эти приложения. Кроме того, эта функция позволяет вам расширить круг своих клиентов, так как у клиентов могут быть разные требования для сред, в которых они хотят запускать ваши приложения. Например, требования клиентов в отрасли здравоохранения и финансовой сфере могут отличаться от требований клиентов в автомобильной сфере и сфере путешествий.

Выпуск технической предварительной версии этой функции ожидается в первом квартале 2016 года.

## Поддерживаемые операционные системы
Вы сможете создавать кластеры на виртуальных машинах или компьютерах под управлением следующих операционных систем: * Windows Server 2012 R2 * Windows Server 2016 * Linux.

## Поддерживаемые языки программирования
Вы сможете писать приложения Service Fabric на следующих языках программирования: * C# * Java.

## Создание и настройка кластера
Установочный пакет, предоставленный Service Fabric, можно скачать из Центра загрузки Майкрософт. После загрузки этого пакета необходимо внести изменения в файл конфигурации, указав параметры кластера. Изменив параметры кластера, вы запустите сценарий установки, который создаст кластер на компьютерах, которые были указаны в параметрах кластера.

Точные сведения о процессе установки будут доступны при выпуске предварительной версии этой функции в первом квартале 2016 года.

## Развертывания в облаке и локальные развертывания
Процесс создания локального кластера Service Fabric аналогичен созданию кластера в любом облаке с набором виртуальных машин. Начальные действия по подготовке виртуальных машин выполняются облаком или локальной средой, которую вы используете. После создания набора виртуальных машин с сетевым подключением между ними выполните действия по установке пакета Service Fabric, затем измените параметры кластера и запустите сценарии создания кластера и управления, которые будут похожи друг на друга. Это гарантирует, что ваши знания и опыт эксплуатации и управления кластерами Service Fabric останутся актуальными при выборе новых сред размещения.

## Преимущества функции "Развертывание где угодно"
* Так как привязка к поставщику отсутствует, вы можете выбирать, где запускать приложения.
* После написания приложения Service Fabric можно запускать в различных средах размещения без изменений или с минимальными изменениями.
* Знания о построении приложений Service Fabric будут актуальными при переходе с одной платформы размещения на другие.
* Опыт запуска кластеров Service Fabric будет актуальным при переходе от одной среды развертывания к другим.
* Широкий охват клиентов не ограничен средой размещения.
* Дополнительный уровень надежности и защиты от масштабных простоев обеспечивается за счет перемещения служб в другую среду развертывания в случае отключения центра обработки данных или поставщика облачных решений.

## Преимущества управляемых кластеров Service Fabric в Azure над кластерами, создаваемыми и управляемыми с помощью функции "Развертывание где угодно"
Запуск кластеров Service Fabric в Azure предоставляет несколько преимуществ по сравнению с функцией "Развертывание где угодно", поэтому, если у вас нет конкретных требований относительно того, где вы хотите запускать кластеры, рекомендуем запускать их в Azure. В Azure мы обеспечиваем интеграцию с другими функциями и службами Azure, которые упрощают эксплуатацию и управление кластером:

* **Портал Azure:** облегчает создание кластеров и управление ими.
* **Диспетчер ресурсов Azure:** с помощью диспетчера ресурсов Azure легко управлять всеми ресурсами, используемыми кластером, как единым целым. Кроме того, он упрощает отслеживание затрат и выставление счетов.
* **Диагностика:** в Azure мы предоставляем интеграцию с системой диагностики Azure и оперативной аналитикой.
* **Автомасштабирование:** для кластеров в Azure мы предоставляем встроенную функцию автомасштабирования. В других средах, в которых используется функция "Развертывание где угодно", вам придется реализовать собственную функцию автоматического масштабирования или выполнить масштабирование вручную с помощью API-интерфейсов, которые предоставляются Service Fabric для масштабирования кластеров.

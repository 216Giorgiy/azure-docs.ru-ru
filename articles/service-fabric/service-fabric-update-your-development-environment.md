# Обновление среды разработки Service Fabric

 Service Fabric периодически выпускает новые версии среды выполнения, пакета SDK и инструментов для локальной разработки. Установка всех доступных обновлений для локальной среды разработки позволит вам создавать и тестировать приложения в локальном режиме, используя все новейшие функции, исправления и усовершенствования.

## Очистка локального кластера

 В настоящее время Service Fabric не поддерживает обновление среды выполнения Service Fabric во время работы локального кластера. Чтобы установить обновление должным образом, необходимо предварительно очистить локальный кластер.

 >[AZURE.NOTE]При очистке локального кластера все развернутые приложения и их данные будут удалены.

 Для очистки локального кластера выполните следующие действия.


 1. Закройте все окна PowerShell и откройте новое от имени администратора.

 2. Перейдите в каталог установки кластера с помощью команды `cd "$env:ProgramW6432\Microsoft SDKs\Service Fabric\ClusterSetup"`

 3. Запустите `.\CleanCluster.ps1`


## Обновление среды выполнения, пакета SDK и инструментов

 Очистив кластер, установите доступные обновления, выполнив следующие действия.


 1. Запустите установщик веб-платформы, чтобы установить [обновление до новой версии][1].

 2. После завершения его работы откройте новое окно PowerShell от имени администратора и перейдите в каталог установки кластера с помощью команды `cd "$ENV:ProgramFiles\Microsoft SDKs\ServiceFabric\ClusterSetup"`.

 3. Запустите `.\DevClusterSetup.ps1`, чтобы настроить локальный кластер.

Вот и все! Теперь можно запустить Visual Studio и продолжить создание приложений Service Fabric.

>[AZURE.NOTE]В данной версии структура проекта по умолчанию изменилась. Уже существующие проекты должны открываться и работать в Visual Studio как обычно. Если при создании, развертывании или отладке приложений возникнут какие-либо проблемы, создайте новый проект и перенесите в него свой код.

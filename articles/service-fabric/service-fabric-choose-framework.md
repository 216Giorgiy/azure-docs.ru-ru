# Выбор платформы для службы

Структура служб предлагает две высокоуровневые платформы для создания служб: API Reliable Actors и API Reliable Services. Хотя обе построены на одном и том же ядре структуры служб, они различным образом реализуют простоту и гибкость с точки зрения одновременной работы, разделения и обмена данными. Полезно понимать обе модели, чтобы можно было выбрать подходящую для определенной службы платформу в приложении.

## Сравнение API Reliable Actors и API Reliable Services

|**API Reliable Actors**|**API Reliable Services**|
|-----------------------|--------------------------|
|Проблемное пространство включает множество небольших независимых единиц состояния и логики|Необходимо поддерживать логику в нескольких компонентах|
|Нужно работать с однопотоковыми объектами, одновременно сохраняя возможность масштабирования и обеспечения согласованности данных|Необходимо использовать надежные коллекции (такие как словарь .NET и очередь) для сохранения вашего состояния и управления им|
|Требуется, чтобы платформа управляла параллельностью и степенью детализации состояния|Необходимо управлять степенью детализации и параллельностью вашего состояния|
|Требуется, чтобы платформа управляла обменом данными самостоятельно|Необходимо управлять обменом данными и схемой секционирования для службы|

Помните, что разумно использовать различные платформы для различных служб в вашем приложении. Например, у вас может быть служба с отслеживанием состояния, которая собирает данные, формируемые определенным количеством субъектов действия.

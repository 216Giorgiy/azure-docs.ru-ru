# Выбор платформы для службы

Azure Service Fabric предлагает две высокоуровневые платформы для создания служб: API Reliable Services и API Reliable Actors. Хотя обе построены на одном и том же ядре структуры служб, они различным образом реализуют простоту и гибкость с точки зрения одновременной работы, разделения и обмена данными. Полезно изучить обе модели, чтобы выбрать ту из них, которая лучше всего подходит для разработки конкретной службы приложения.

## Сравнение API Reliable Actors и API Reliable Services

|**Когда следует использовать API Reliable Actors**|**Когда следует использовать API Reliable Services**|
|-----------------------|--------------------------|
|Проблемное пространство включает множество (более 1000) небольших независимых единиц состояния и логики.|Необходимо поддерживать логику в нескольких компонентах.|
|Необходимо работать с однопоточными объектами, которые не требуют существенного внешнего взаимодействия.|Необходимо использовать надежные коллекции (такие как надежный словарь .NET и надежная очередь) для сохранения вашего состояния и управления им.|
|Требуется, чтобы платформа управляла обменом данными самостоятельно.|Необходимо управлять обменом данными и схемой секционирования для службы.|

Помните, что разумно использовать различные платформы для различных служб в вашем приложении. Например, у вас может быть служба с отслеживанием состояния, которая собирает данные, формируемые определенным количеством субъектов действия.

# Ваше приложение Service Fabric и дальнейшие действия
Ваше приложение Service Fabric создано. В этой статье описываются состав проекта и некоторые возможные дальнейшие действия.

## Ваше приложение
Каждое новое приложение включает проект приложения. В зависимости от типа выбранной службы также могут быть один или два дополнительных проекта.

### Проект приложения
Проект приложения включает следующее: –Набор ссылок на службы, которые составляют приложение. – Два профиля публикации (локальный и облачный), которые можно использовать для сохранения настроек для работы с различными средами (к ним относятся, например, конечная точка кластера и необходимость выполнять развертывания обновления по умолчанию). – Два файла параметров приложений (локальный и облачный), которые можно использовать для сохранения конфигураций приложения для конкретной среды (например, количество секций, которые необходимо создать для службы). – Сценарий развертывания, который можно использовать для развертывания приложения из командной строки или как часть автоматизированного конвейера непрерывной интеграции. – Манифест приложения, который описывает приложение.

### Надежные службы
При добавлении новой надежной службы Visual Studio добавляет проект службы в решение. Проект службы содержит класс, который наследуется от `StatelessService` или `StatefulService` в зависимости от выбранного типа.

### Надежные субъекты
При добавлении нового надежного субъекта Visual Studio добавляет в решение два проекта: проект субъекта и проект интерфейса.

Проект субъекта определяет тип субъекта и (для субъектов с отслеживанием состояния) его состояние. Проект интерфейса предоставляет интерфейс, который используется другими службами для вызова субъекта.

Обратите внимание, что проект субъекта не содержит никакого поведения по умолчанию при запуске, так как субъекты должны активироваться другими службами. Рассмотрите возможность добавления надежной службы или проекта ASP.NET для создания своих субъектов и взаимодействия с ними.

### ASP.NET 5
Шаблоны ASP.NET 5, предоставленные для использования в приложениях Service Fabric, практически идентичны тем, которые доступны для проектов ASP.NET 5, созданных независимо друг от друга. Единственными отличиями являются следующие: – Проект содержит каталог **PackageRoot** для хранения ServiceManifest вместе с пакетами данных и конфигурации. – Проект ссылается на дополнительный пакет NuGet (Microsoft.ServiceFabric.AspNet.Hosting), который выступает в качестве моста между DNX и Service Fabric.

## Дальнейшие действия
### Добавление веб-интерфейса в приложение
Service Fabric обеспечивает интеграцию с ASP.NET 5 для создания веб-точек входа в приложение. Сведения о создании интерфейса REST на основе ASP.NET WebAPI см. в разделе [Добавление веб-интерфейса в приложение][add-web-frontend].

### Создание кластера Azure
Пакет SDK Service Fabric предоставляет локальный кластер для разработки и тестирования. Сведения о создании кластера в Azure см. в статье [Настройка кластера Service Fabric на портале Azure][create-cluster-in-portal].

### Публикация изменений в Azure
Приложение в кластер Azure можно опубликовать напрямую из Visual Studio. Чтобы узнать, как это сделать, обратитесь к разделу [Публикация приложения в Azure][publish-app-to-azure].

### Визуализация кластера с помощью обозревателя Service Fabric
Обозреватель Service Fabric предоставляет простой способ визуализации кластера, включая развернутые приложения и физическую структуру. Чтобы узнать, как это сделать, обратитесь к разделу [Визуализация кластера с помощью обозревателя Service Fabric][visualize-with-sfx].

### Версии и обновление служб
Service Fabric обеспечивает независимое управление версиями и обновление независимых служб в приложении. Дополнительные сведения см. в разделе [Управление версиями и обновление служб][app-upgrade-tutorial].

### Настройка непрерывной интеграции с Visual Studio Online
Чтобы узнать, как настроить процесс непрерывной интеграции для приложения Service Fabric, обратитесь к разделу [Настройка непрерывной интеграции с Visual Studio Online][ci-with-vso].

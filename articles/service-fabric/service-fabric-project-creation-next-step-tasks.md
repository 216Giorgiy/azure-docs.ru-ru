# Ваше приложение Service Fabric и дальнейшие действия
Ваше приложение Azure Service Fabric создано. В этой статье описываются состав проекта и некоторые возможные дальнейшие действия.

## Ваше приложение
Каждое новое приложение включает проект приложения. В зависимости от типа выбранной службы также могут быть один или два дополнительных проекта.

### Проект приложения
Проект приложения включает:

- Набор ссылок на службы, входящие в состав приложения.

- Два профиля публикации (локальный и облачный), которые можно использовать для хранения настроек для работы в различных средах, например параметров конечной точки кластера и параметра, определяющего, следует ли разворачивать обновления по умолчанию.

- Два файла с параметрами приложения (локальный и облачный), которые можно использовать для хранения конфигураций приложения для разных сред, включая число разделов, которые создаются для службы.

- Сценарий развертывания, который можно использовать для развертывания приложения из командной строки или в составе автоматизированного конвейера непрерывной интеграции.

- Манифест приложения, описывающий приложение.

### Надежные службы
При добавлении новой надежной службы Visual Studio добавляет проект службы в решение. Проект службы содержит класс, который наследуется от `StatelessService` или `StatefulService` в зависимости от выбранного типа.

### Надежные субъекты
При добавлении нового надежного субъекта Visual Studio добавляет в решение два проекта: проект субъекта и проект интерфейса.

Проект субъекта определяет тип субъекта и (для субъектов с отслеживанием состояния) его состояние. Проект интерфейса предоставляет интерфейс, который используется другими службами для вызова субъекта.

Обратите внимание, что проект субъекта не содержит никакого поведения по умолчанию при запуске, так как субъекты должны активироваться другими службами. Рассмотрите возможность добавления надежной службы или проекта ASP.NET для создания своих субъектов и взаимодействия с ними.

### ASP.NET 5
Шаблоны ASP.NET 5, предоставленные для использования в приложениях Service Fabric, практически идентичны тем, которые доступны для проектов ASP.NET 5, созданных независимо друг от друга. Единственными отличиями являются:

- Проект содержит папку **PackageRoot** для хранения ServiceManifest вместе с пакетами данных и конфигурации.

- Проект ссылается на дополнительный пакет NuGet (Microsoft.ServiceFabric.AspNet.Hosting), который выступает в качестве связующего звена между средой выполнения .NET (DNX) и Service Fabric.

## Дальнейшие действия
### Добавление веб-интерфейса в приложение
Service Fabric обеспечивает интеграцию с ASP.NET 5 для создания веб-точек входа в приложение. Сведения о создании интерфейса REST на основе ASP.NET WebAPI см. в статье [Добавление веб-интерфейса в приложение][add-web-frontend].

### Создание кластера Azure
Пакет SDK Service Fabric предоставляет локальный кластер для разработки и тестирования. Сведения о создании кластера в Azure см. в статье [Настройка кластера Service Fabric на портале Azure][create-cluster-in-portal].

### Бесплатное развертывание в Azure с использованием сторонних кластеров

Для пробного развертывания приложений и управления ими в Azure без настройки собственных кластеров можно использовать бесплатную [стороннюю службу кластеров](http://aka.ms/tryservicefabric).

### Публикация изменений в Azure
Приложение в кластер Azure можно опубликовать напрямую из Visual Studio. Чтобы узнать, как это сделать, обратитесь к статье [Публикация приложения в Azure][publish-app-to-azure].

### Визуализация кластера с помощью обозревателя Service Fabric
Обозреватель Service Fabric предоставляет простой способ визуализации кластера, включая развернутые приложения и физическую структуру. Чтобы узнать, как это сделать, обратитесь к статье [Визуализация кластера с помощью обозревателя Service Fabric][visualize-with-sfx].

### Версии и обновление служб
Service Fabric обеспечивает независимое управление версиями и обновление независимых служб в приложении. Дополнительные сведения см. в статье [Управление версиями и обновление служб][app-upgrade-tutorial].

### Настройка непрерывной интеграции с Visual Studio Team Services
Чтобы узнать, как настроить процесс непрерывной интеграции для приложения Service Fabric, см. статью [Настройка непрерывной интеграции с помощью Visual Studio Team Services][ci-with-vso].

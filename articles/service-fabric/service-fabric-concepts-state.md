# Состояние службы
**Состояние службы** — это данные, необходимые службе для функционирования. Это структуры данных и переменные, которые считываются и записываются службой в ходе выполнения работы.

Пример. Рассмотрим простую службу калькулятора. Эта служба получает два числа и возвращает их сумму. Служба используется исключительно без отслеживания состояния, и с ней не связано никаких данных.

Теперь рассмотрим такой же калькулятор, но добавим к вычислению суммы также место возврата последней вычисленной суммы. Эта служба теперь использует отслеживание состояния, в которое осуществляется запись (при вычислении новой суммы), откуда осуществляется чтение (при возврате последней вычисляемой суммы).

В структуре служб первая служба называется службой без отслеживания состояния. Вторая служба называется службой с отслеживанием состояния.

## Сохранение состояния службы
Состояние может быть выведено вовне или размещаться совместно с кодом, который манипулирует состоянием. Экстернализация состояния обычно выполняется путем использования внешней базы данных или хранилища. В нашем примере с калькулятором может использоваться база данных SQL, в которой текущий результат сохраняется в таблице. Каждый запрос на вычисление суммы выполняет обновление содержания этой строки.

Состояние может также размещаться вместе с кодом, который манипулирует этим кодом. Службы с отслеживанием состояния в структуре служб создаются на основе этой модели. Структура службы предоставляет инфраструктуру, обеспечивающую высокую доступность этого состояния в случае ошибок.

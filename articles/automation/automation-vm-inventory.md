# <a name="manage-an-azure-virtual-machine-with-inventory-collection"></a>Управление виртуальной машиной Azure с помощью сбора данных инвентаризации

Отслеживание данных для виртуальной машины Azure можно включить со страницы ресурсов компьютера. В этом способе предоставляется пользовательский интерфейс на основе браузера для настройки сбора данных инвентаризации.

Если у вас еще нет подписки Azure, создайте [бесплатную](https://azure.microsoft.com/free/) учетную запись Azure, прежде чем начинать работу.
Создайте [виртуальную машину](https://docs.microsoft.com/en-us/azure/virtual-machines/windows/quick-create-portal) до начала работы, если вы еще не сделали этого.

## <a name="log-in-to-the-azure-portal"></a>Войдите на портал Azure.

Войдите на [портал Azure](https://portal.azure.com/).

## <a name="enable-inventory-collection-from-the-virtual-machine-resource-page"></a>Включение сбора данных инвентаризации на странице ресурсов виртуальной машины

1. Слева на экране выберите **Виртуальные машины**.
1. Затем выберите виртуальную машину из списка.
1. В меню ресурсов в разделе **Операции** выберите **Inventory (Preview)** (Инвентаризация (предварительная версия)).
1. В верхней части страницы появится баннер, уведомляющей о том, что решение не включено. Щелкните баннер, чтобы включить решение.
1. Выберите рабочую область Log Analytics для хранения данных журналов. Если рабочие области для этого региона недоступны, вам будет предложено создать рабочую область по умолчанию и учетную запись службы автоматизации. Щелкните **Включить**, чтобы запустить подключение компьютера.

   ![Просмотр параметров подключения](./media/automation-vm-inventory/inventory-onboarding-options.png)  

1. Строка состояния уведомляет о включении решения. Этот процесс может занять до 15 минут. В это время можно закрыть колонку или оставить ее открытой и в ней отобразится отчет о включении решения. Состояние развертывания можно отслеживать на панели уведомлений.

   ![Просмотр решения инвентаризации сразу после подключения](./media/automation-vm-inventory/inventory-onboarded.png)

1. После завершения развертывания строка состояния исчезнет. Сейчас система по-прежнему собирает данные инвентаризации, которые еще не отображаться. На сбор всех данных может потребоваться 24 часа.

## <a name="configure-your-inventory-settings"></a>Настройка параметров инвентаризации

По умолчанию для сбора настраивается программное обеспечение, службы Windows и управляющие программы Linux. Для сбора данных реестра Windows и инвентаризации файлов настройте параметры сбора данных инвентаризации.

1. В представлении **Inventory (Preview)** (Инвентаризация (предварительная версия)) в верхней части страницы нажмите кнопку **Изменить параметры**.
2. Чтобы добавить новый параметр сбора, перейдите к категории параметра, который необходимо добавить, используя вкладки, которые считывают **реестр Windows**, **файлы Windows** и **файлы Linux**. В верхней части страницы щелкните **Добавить**.
3. Подробные сведения и описания всех свойств параметров см. на странице с [документацией по инвентаризации](https://aka.ms/configinventorydocs).

## <a name="disconnecting-your-virtual-machine-from-management"></a>Отключение виртуальной машины от управления

Чтобы удалить компьютер из управления инвентаризации, сделайте следующее:

1. В левом меню на портале Azure щелкните **Log Analytics**, а затем щелкните рабочую область, используемую при подключении виртуальной машины.
1. На странице Log Analytics в категории в меню ресурсов **Workspace Data Sources** (Источники данных рабочей области) выберите **Виртуальные машины**. 
1. Из списка выберите виртуальную машину, которую необходимо отключить. Рядом с ней будет зеленый флажок с текстом This workspace (Эта рабочая область) в столбце **Подключение OMS**. В верхней части следующей страницы щелкните **Отключить** и **Да** в диалоговом окне подтверждения, чтобы отключить компьютер от управления.

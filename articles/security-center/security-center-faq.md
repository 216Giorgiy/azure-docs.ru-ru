# Центр безопасности Azure: часто задаваемые вопросы

В этой статье приведены ответы на часто задаваемые вопросы о Центре безопасности Azure.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Общие вопросы

### Что такое Центр безопасности?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

### Как можно воспользоваться Центром безопасности?
 Центр безопасности автоматические включается при оформлении подписки Microsoft Azure. Попасть в него можно на [портале Azure](https://azure.microsoft.com/features/azure-portal/): [войдите на портал](https://portal.azure.com), выберите **Обзор** и прокрутите экран до пункта **Центр безопасности**. На панели мониторинга могут сразу появиться рекомендации по обеспечению безопасности. Это связано с тем, что служба может оценивать состояние безопасности определенных элементов управления по их конфигурации в Azure. Чтобы активировать весь набор функций мониторинга, рекомендаций и предупреждений, [включите сбор данных](#data-collection).

## Выставление счетов

## Сбор данных

### Как включить сбор данных?<a name=data-collection></a>
Включить сбор данных для подписки Azure можно в политике безопасности. Чтобы включить сбор данных, [войдите на портал Azure](https://portal.azure.com) и последовательно выберите **Обзор**, **Центр безопасности** и **Политика безопасности**. Установите для параметра **Сбор данных** значение **Вкл.** и настройте учетные записи хранения, в которых будут храниться собранные данные (см. вопрос [Где хранятся мои данные?](#where-is-my-data-stored)). Если функция **сбора данных** включена, она автоматически собирает сведения о конфигурации и событиях безопасности со всех поддерживаемых виртуальных машин в подписке.

### Что происходит при включении сбора данных?
Сбор данных включается с помощью агента мониторинга Azure и расширения "Мониторинг безопасности Azure". Расширение сканирует систему на наличие конфигураций безопасности и передает их в [службу трассировки событий Windows (ETW)](https://msdn.microsoft.com/library/windows/desktop/bb968803.aspx).

Кроме того, операционная система записывает в журнал событий сведения обо всех действиях. Агент мониторинга безопасности Azure считывает записи журнала событий и трассировки ETW и копирует их в вашу учетную запись хранения для последующего анализа. Это та учетная запись хранения, которую вы настроили в политике безопасности. Дополнительные сведения об учетной записи хранения см. в ответе на вопрос [Где хранятся мои данные?](#where-is-my-data-stored).

### Влияет ли агент мониторинга или расширение "Мониторинг безопасности" на работу серверов?
Агент и расширение потребляют номинальное количество системных ресурсов и не должны существенно влиять на производительность.

### Как отключить сбор данных, если он больше не требуется?
Отключить сбор данных в подписке можно в политике безопасности: [войдите на портал Azure](https://portal.azure.com) и последовательно выберите **Обзор**, **Центр безопасности** и **Политика безопасности**. При выборе подписки открывается новая колонка, в которой можно отключить сбор данных. В верхней ленте выберите параметр **Удалить агентов**, чтобы удалить агентов с существующих виртуальных машин.

### Где хранятся мои данные?<a name=where-is-my-data-stored></a>
Для каждого региона, в котором у вас есть виртуальные машины, выберите учетную запись хранения, где хранятся данные, собираемые с этих виртуальных машин. Это позволяет хранить данные в одной географической области, чтобы обеспечить их конфиденциальность и независимость.

Учетная запись хранения для подписки выбирается в разделе "Политика безопасности": [войдите на портал Azure](https://portal.azure.com) и последовательно выберите **Обзор**, **Центр безопасности** и **Политика безопасности**. При выборе подписки открывается новая колонка. Щелкните **Выбрать учетную запись хранения**, чтобы выбрать регион. В целях безопасности собранные данные логически изолируются от данных других клиентов.

Дополнительные сведения о службе хранилища Azure и учетных записях хранения см. в [документации по хранилищу](https://azure.microsoft.com/documentation/services/storage/) и статье [Об учетных записях хранения Azure](../storage/storage-create-storage-account.md).

## Использование Центра безопасности

### Что такое политика безопасности?
Политика безопасности определяет набор средств контроля, которые рекомендуются для ресурсов в указанной подписке. В Центре безопасности вы настраиваете политики для своих подписок Azure в соответствии с действующими в вашей компании требованиями к безопасности, типом используемых приложений и конфиденциальностью данных в каждой подписке.

Например, требования к безопасности ресурсов, используемых для разработки или тестирования, могут отличаться от требований к безопасности ресурсов, которые используются для рабочих приложений. Аналогично, приложения с контролируемыми данными, такими как идентифицирующие личность сведения, могут требовать более высокого уровня безопасности. Политики безопасности, включенные в Центре безопасности, позволяют получать рекомендации по обеспечению безопасности и осуществлять мониторинг. Дополнительные сведения о политиках безопасности см. в разделе [Наблюдение за работоспособностью системы безопасности в Центре безопасности Azure](security-center-monitoring.md).

### Кто может изменять политику безопасности?
Политики безопасности настраиваются для каждой подписки. Изменять политики безопасности могут только владельцы и участники подписок.

Дополнительные сведения о настройке политики безопасности см. в статье [Настройка политик безопасности в Центре безопасности Azure](security-center-policies.md).

### Что такое рекомендация по безопасности?
 Центр безопасности анализирует состояние безопасности ваших ресурсов Azure. Когда обнаруживаются потенциальные уязвимости, Центр безопасности создает рекомендации. Рекомендации помогают выполнить процедуру настройки необходимого средства контроля. Примеры

- Подготовка антивредоносных программ для обнаружения и устранения вредоносных программ.
- Настройка [групп безопасности сети](../virtual-network/virtual-networks-nsg.md) и правил управления трафиком для виртуальных машин.
- Подготовка брандмауэров для защиты веб-приложений от атак.
- Развертывание отсутствующих обновлений системы
- Обращение к конфигурациям операционной системы, не соответствующих рекомендуемым базовым показателям

Здесь отображаются только рекомендации, включенные в политиках безопасности.

### Как увидеть текущее состояние безопасности ресурсов Azure?
Плитка **Работоспособность ресурсов** в колонке **Центр безопасности** показывает общий уровень защищенности вашей среды с детализацией по виртуальным машинам, веб-приложениям и другим ресурсам. У каждого ресурса есть индикатор, указывающий на обнаружение потенциальных проблем безопасности. При выборе плитки **Работоспособность ресурсов** отображаются ресурсы и указывается, где необходимо вмешательство и какие проблемы могут существовать.

### Что инициирует оповещение системы безопасности?
 Центр безопасности автоматически собирает, анализирует и объединяет данные журналов из ресурсов Azure, сети и партнерских решений, таких как антивредоносные программы и брандмауэры. Когда Центр безопасности обнаруживает угрозу, создается предупреждение системы безопасности. Например, определяется следующее.

- Обращение взломанных виртуальных машин к опасным IP-адресам.
- Обнаружение сложных вредоносных программ с помощью отчетов об ошибках Windows.
- Атаки на виртуальные машины методом подбора.
- Предупреждения системы безопасности, отправленные интегрированными партнерскими решениями, в частности антивредоносными программами и брандмауэрами веб-приложений.

### Как в Центре безопасности обрабатываются разрешения?
 Центр безопасности поддерживает доступ на основе ролей. Дополнительные сведения об управлении доступом на основе ролей (RBAC) в Azure см. в [этой статье](../active-directory/role-based-access-control-configure.md).

Открыв Центр безопасности, пользователь видит только рекомендации и предупреждения для ресурсов, к которым у него есть доступ. Доступ к ресурсам есть только у пользователей, которым назначена роль владельца, участника или читателя в соответствующей подписке или группе ресурсов.

Изменять политики безопасности могут только владельцы и участники подписок.

### Какие типы виртуальных машин поддерживаются?
Поддерживаются [классические виртуальные машины и диспетчеры ресурсов](../azure-classic-rm.md), включая виртуальные машины, входящие в кластеры Azure Service Fabric.

Рекомендации в разделе "Контроль доступа" сейчас касаются только классических виртуальных машин. Правила групп безопасности сети и рекомендации по установке брандмауэров веб-приложений сейчас касаются только виртуальных машин, работающих в качестве диспетчеров ресурсов.

### Поддерживаются ли виртуальные машины Linux?
Центр безопасности Azure предлагает базовые функции мониторинга для виртуальных машин Linux (только Ubuntu версий 12.04, 14.04, 14.10 и 15.04). Впоследствии будут добавлены функции мониторинга работоспособности и сбора или анализа данных, а также поддержка других дистрибутивов Linux.

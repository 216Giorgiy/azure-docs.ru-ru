# Управление оповещениями безопасности в Центре безопасности Azure и реагирование на них
Этот документ помогает использовать возможности Центра безопасности Azure для управления и реагирования на оповещения безопасности.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Что такое оповещения системы безопасности?
Центр безопасности автоматически собирает, анализирует и объединяет данные журналов из ресурсов Azure, сети и встроенного ПО, такого как антивредоносные программы и брандмауэры, для выявления реальных угроз и сокращения количества ложных срабатываний. Список приоритетных оповещений системы безопасности, в том числе оповещений от интегрированных решений партнеров, отображается в Центре безопасности вместе с информацией, необходимой для быстрого анализа, и рекомендациями по устранению атаки.
 
Специалисты по безопасности из корпорации Майкрософт постоянно анализируют новые угрозы для своих пользовательских и корпоративных продуктов и веб-служб, возникающие по всему миру, в том числе новые шаблоны и тенденции атак. Поэтому Центр безопасности может обновлять свои алгоритмы обнаружения по мере выявления новых уязвимостей и эксплойтов, помогая пользователям своевременно защищаться от постоянно совершенствующихся угроз. Вот некоторые типы угроз, которые может выявлять Центр безопасности.

- **Обнаружение атак методом подбора через сетевые данные**. Использует модели машинного обучения, которые анализируют типичные шаблоны сетевого трафика для приложений, более эффективно выявляет попытки доступа злоумышленников вместо законных пользователей.
- **Обнаружение атак методом подбора через данные конечной точки**. Используется анализ журналов компьютера. Этот метод позволяет различать успешные и неудачные попытки.
- **Виртуальные машины, взаимодействующие с вредоносными IP-адресами**. Сравнивает сетевой трафик с помощью средств глобального анализа угроз (Майкрософт), обнаруживает скомпрометированные компьютеры, которые взаимодействуют с серверами контроля и управления (и наоборот).
- **Скомпрометированные виртуальные машины**. На основе поведенческого анализа журналов компьютера и корреляции с другими сигналами определяет аномальные события, которые, вероятно, являются результатами компрометации и эксплуатации виртуальной машины.

## Управление предупреждениями системы безопасности 

Текущие оповещения можно просматривать на плитке **Оповещения системы безопасности**. Выполните следующие действия, чтобы просмотреть дополнительные сведения о каждом оповещении.

1. На панели мониторинга в Центре безопасности вы увидите плитку **Оповещения системы безопасности**.

    ![Плитка "Оповещения системы безопасности" в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig1-new.png)

2.  Щелкните эту плитку, чтобы открыть колонку **Оповещения системы безопасности** с дополнительными сведениями об оповещениях, как показано ниже.

    ![Колонка "Оповещения системы безопасности" в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig2-new.png)

В нижней части этой колонки содержатся сведения для каждого оповещения. Чтобы отсортировать данные, щелкните соответствующий столбец. Ниже приводятся определения столбцов:

- **Оповещение** — краткое описание оповещения.
- **Число** — список всех оповещений определенного типа, обнаруженных в определенный день.
- **Обнаружено** — служба, отвечающая за активацию оповещения.
- **Дата** — дата возникновения события.
- **Состояние** — текущее состояние оповещения. Значение состояния может быть одним из трех, указанных ниже:
    - **Активно** — обнаружено оповещение системы безопасности.
    - **Отклонено** — оповещение системы безопасности отклонено пользователем. Это состояние обычно используется для оповещений, которые были исследованы, однако не были признаны фактической атакой.

- **Серьезность** — уровень серьезности, который может быть высоким, средним или низким.

Оповещения можно фильтровать по дате, состоянию и уровни серьезности. Фильтрация оповещений нужна в ситуациях, когда требуется сузить область просматриваемых оповещений системы безопасности. Например, вам может потребоваться проверить оповещения системы безопасности, возникшие за последние 24 часа, поскольку вы исследуете потенциальное нарушение безопасности в системе.

1. В колонке **Оповещения системы безопасности** щелкните **Фильтр**. Откроется колонка **Фильтр**, где можно выбрать значения даты, серьезности и состояния, которые требуется просмотреть.

	![Фильтрование оповещений в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig5-new.png)

2. 	После анализа оповещения системы безопасности может выясниться, что оно ложно для вашей среды или указывает на ожидаемое поведение определенного ресурса. Как бы то ни было, если выяснилось, что оповещение системы безопасности неприменимо, вы можете отклонить его, а затем отфильтровать из представления. Оповещение системы безопасности можно отклонить двумя способами. Щелкните оповещение правой кнопкой мыши и выберите пункт **Отклонить** или наведите указатель мыши на элемент, нажмите кнопку с тремя точками, которая появится справа, и выберите **Отклонить**. Отклоненные оповещения системы безопасности можно просмотреть, щелкнув плитку **Фильтр** и выбрав пункт **Отклоненные**.

	![Фильтрование оповещений в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig6-new.png)

### Реагирование на оповещения системы безопасности
Выберите оповещение системы безопасности, чтобы получить дополнительные сведения о событиях, вызвавших оповещение, и (при необходимости) действиях, которые следует предпринять для устранения атаки. Оповещения системы безопасности группируются по типу и дате. Если щелкнуть оповещение системы безопасности, откроется колонка, содержащая список сгруппированных оповещений.

![Реагирование на оповещения системы безопасности в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig7.png)

В этом случае запущенные оповещения относятся к подозрительной активности протокола удаленного рабочего стола (RDP). В первом столбце показано, какие ресурсы подверглись атаке, во втором — время обнаружения атаки, в третьем — состояние оповещения, в четвертом — уровень его серьезности. Просмотрев эти данные, щелкните атакованный ресурс. Откроется новая колонка с рекомендациями по дальнейшим действиям, как в приведенном ниже примере.

![Рекомендации по реагированию на оповещения системы безопасности в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig8-1.png)

В поле **Предупреждение** этой колонки вы найдете дополнительные сведения о данном событии. Эти дополнительные сведения содержат причину появления предупреждения системы безопасности, целевой ресурс, IP-адрес источника (если применимо) и рекомендации по устранению этой причины. В некоторых случаях IP-адрес источника будет пустым (недоступным), так как не все журналы событий безопасности Windows включают IP-адрес.

> [AZURE.NOTE] Исправления, предлагаемые Центром безопасности Azure, зависят от оповещения системы безопасности. В некоторых случаях для реализации рекомендуемого исправления могут потребоваться другие возможности Azure. Например, исправление для этой атаки — поместить в черный список IP-адрес, с которого исходит атака, с помощью правила [списка управления доступом к сети](../virtual-network/virtual-networks-acl.md) или [группы безопасности сети](../virtual-network/virtual-networks-nsg.md).

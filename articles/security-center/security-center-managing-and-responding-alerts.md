# Управление оповещениями безопасности в Центре безопасности Azure и реагирование на них
Узнайте, как использовать возможности Центра безопасности Azure для управления и реагирования на оповещения безопасности с помощью этого пошагового руководства.

> [AZURE.NOTE]Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Функции Центра безопасности Azure по управлению оповещениями безопасности
Центр безопасности Azure автоматически собирает, анализирует и объединяет данные журнала из ресурсов Azure, сети и решений партнеров, таких как программы защиты от вредоносных программ и брандмауэры, для выявления реальных угроз и сокращения ложных срабатываний. Список приоритетных оповещений представлен как оповещения безопасности.

Текущие оповещения можно просматривать на плитке "Оповещения системы безопасности". Выполните следующие действия, чтобы просмотреть дополнительные сведения о каждом оповещении.

1. На панели мониторинга **Центр безопасности Azure** отображается плитка **Оповещения системы безопасности**. 

    ![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig1.png)

2.  Щелкните оповещение безопасности на графике: откроется колонка **Оповещения системы безопасности**, в которой будут отображаться дополнительные сведения об этом оповещении, как показано ниже:
    
    ![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig2.png)

В нижней части этой колонки содержатся сведения для каждого оповещения. Сведения можно отсортировать для каждого столбца, щелкнув нужный столбец. Ниже приводятся определения столбцов.

- **Оповещение** — краткое описание оповещения.
- **Число** — представляет число всех оповещений определенного типа, обнаруженных в определенный день.
- **Обнаружено** — служба, отвечающая за вызов оповещения.
- **Дата** — время возникновения события.
- **Состояние** — текущее состояние для данного оповещения. Значение состояния может быть одним из трех, указанных ниже.
    - **Активно** — обнаружено оповещение системы безопасности.
    - **Отклонено** — оповещение системы безопасности отклонено пользователем. Это состояние обычно используется для оповещений, которые были исследованы, однако не были признаны фактической атакой, или были исследованы и разрешены.

- **Серьезность** — уровень серьезности, который может быть высоким, средним или низким.
  

### Реагирование на оповещение безопасности
Многие действия могут указывать на возможные атаки на среду вашей организации. Например, сбор данных о сети, законно выполняемый сетевым администратором, может выглядеть как та или иная форма атаки. В других случаях плохая настройка системы может привести к появлению ложных срабатываний системы обнаружения вторжений, что может затруднить выявление фактических нарушений безопасности. Просмотрев оповещения безопасности с помощью Центра безопасности Azure, можно предпринять некоторые действия в зависимости от серьезности оповещения.

Чтобы выполнить действие, выберите нужное оповещение. Справа откроется новая колонка с более подробными сведениями, как показано ниже:

![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig3.png)

В этом случае запущенные оповещения относятся к подозрительной активности протокола удаленного рабочего стола (RDP). В первом столбце показано, какие ресурсы подверглись атаке, затем указывается время обнаружения атаки, состояние оповещения и уровень серьезности. Проанализировав эти сведения, щелкните атакованный ресурс. Откроется новая колонка, содержащая предлагаемые дополнительные действия, как показано в приведенном ниже примере:

![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig4.png)
  
> [AZURE.NOTE]Исправления, предлагаемые Центром безопасности Azure, зависят от оповещения системы безопасности, кроме того, в некоторых случаях может потребоваться использовать другие возможности Azure для реализации рекомендуемого исправления. Например, исправление для данной атаки — поместить в черный список IP-адрес, создающий эту атаку, с помощью правила [списка управления доступом к сети](virtual-networks-acl.md) или [группы безопасности сети](virtual-networks-nsg.md).

### Дополнительные задачи
Оповещения можно фильтровать по дате, состоянию и уровни серьезности. В колонке "Оповещения системы безопасности" щелкните "Фильтр" и включите нужные параметры, как показано ниже:

![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig5.png)

Фильтрация оповещений может быть удобна в ситуациях, когда требуется сузить область просматриваемых данных на панели мониторинга. Например, может потребоваться проверить оповещения безопасности, возникшие за последние 24 часа, поскольку исследуется потенциальное нарушение безопасности в системе. Хотя большинство оповещений системы безопасности будет содержать рекомендации, которые необходимо применить, в некоторых случаях может потребоваться проигнорировать оповещение, потому что для вашей среды это ложное срабатывание или ожидаемое поведение для конкретного ресурса. В любом случае можно скрыть рекомендации для конкретного ресурса, используя пункт**Отклонить**.

Чтобы отклонить задачу, щелкните ресурс, чтобы просмотреть дополнительные сведения или применить рекомендуемую конфигурацию. Щелкните задачу правой кнопкой мыши, чтобы отклонить ее, и вы увидите пункт **Отклонить**, как на рисунке ниже:

![Включение сбора данных](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig6.png)

### Оповещения

В коллекции оповещений для каждого оповещения содержится уникальное описание и исправление. Оповещения, отображаемые в Центре безопасности Azure, основываются на сценарии атаки. Приведенные ниже сценарии атак запускаются модулем Майкрософт.

- **Обнаружение атак методом подбора через данные сети**. Обнаружение таких атак основано на моделях машинного обучения на основе данных сетевого трафика. 
- **Обнаружение атак методом подбора через данные конечной точки**. Обнаружение таких атак основано на запросах Центром безопасности Azure журналов компьютера; это позволяет различать успешные и неудачные попытки. 
- **Виртуальные машины, взаимодействующие с вредоносными IP-адресами**. Обнаружение таких атак основано на обнаружении Центром безопасности Azure компьютеров, безопасность которых нарушена программами-роботами и которые взаимодействуют с серверами контроля и управления (и наоборот).

# Управление оповещениями безопасности в Центре безопасности Azure и реагирование на них
Этот документ помогает использовать возможности Центра безопасности Azure для управления и реагирования на оповещения безопасности.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Что такое оповещения системы безопасности?
 Центр безопасности автоматически собирает, анализирует и объединяет данные журнала из ресурсов Azure, сети и решений партнеров, таких как программы защиты от вредоносных программ и брандмауэры, для выявления реальных угроз и сокращения ложных срабатываний. Список приоритетных оповещений представлен как оповещения безопасности.

Текущие оповещения можно просматривать на плитке **Оповещения системы безопасности**. Выполните следующие действия, чтобы просмотреть дополнительные сведения о каждом оповещении.

1. На панели мониторинга в центре безопасности Azure отображается плитка **Оповещения системы безопасности**.

    ![Плитка "Оповещения системы безопасности" в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig1.png)

2.  Щелкните значок оповещений системы безопасности. Откроется колонка **Оповещения системы безопасности** с дополнительными сведениями об оповещениях, пример которой показан ниже.

    ![Колонка "Оповещения системы безопасности" в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig2.png)

В нижней части этой колонки содержатся сведения для каждого оповещения. Чтобы отсортировать данные, щелкните соответствующий столбец. Ниже приводятся определения столбцов:

- **Оповещение** — краткое описание оповещения.
- **Число** — список всех оповещений определенного типа, обнаруженных в определенный день.
- **Обнаружено** — служба, отвечающая за активацию оповещения.
- **Дата** — дата и время возникновения события.
- **Состояние** — текущее состояние оповещения. Значение состояния может быть одним из трех, указанных ниже:
    - **Активно** — обнаружено оповещение системы безопасности.
    - **Отклонено** — оповещение системы безопасности отклонено пользователем. Это состояние обычно используется для оповещений, которые были исследованы, однако не были признаны фактической атакой.

- **Серьезность** — уровень серьезности, который может быть высоким, средним или низким.

### Реагирование на оповещения системы безопасности
Многие действия могут указывать на возможные атаки на среду вашей организации. Например, сбор данных о сети, законно выполняемый сетевым администратором, может выглядеть как та или иная форма атаки. В других случаях плохая настройка системы может привести к появлению ложных срабатываний системы обнаружения вторжений, что может затруднить выявление фактических нарушений безопасности. Просмотрев оповещения безопасности с помощью Центра безопасности, можно предпринять некоторые действия в зависимости от серьезности оповещения.

Чтобы выполнить действие, выберите нужное оповещение. Справа откроется новая колонка с более подробными сведениями, как показано ниже.

![Реагирование на оповещения системы безопасности в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig3.png)

В этом случае запущенные оповещения относятся к подозрительной активности протокола удаленного рабочего стола (RDP). В первом столбце показано, какие ресурсы подверглись атаке, во втором — время обнаружения атаки, в третьем — состояние оповещения, в четвертом — уровень его серьезности. Просмотрев эти данные, щелкните атакованный ресурс. Откроется новая колонка с рекомендациями по дальнейшим действиям, как в приведенном ниже примере.

![Рекомендации по реагированию на оповещения системы безопасности в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig4.png)

> [AZURE.NOTE] Исправления, предлагаемые Центром безопасности Azure, зависят от оповещения системы безопасности. В некоторых случаях для реализации рекомендуемого исправления могут потребоваться другие возможности Azure. Например, исправление для данной атаки — поместить в черный список IP-адрес, создающий эту атаку, с помощью правила [списка управления доступом к сети](virtual-networks-acl.md) или [группы безопасности сети](virtual-networks-nsg.md).

### Управление оповещениями системы безопасности
Оповещения можно фильтровать по дате, состоянию и уровни серьезности. В колонке **Оповещения системы безопасности** щелкните **Фильтр** и включите нужные параметры, как показано ниже.

![Фильтрование оповещений в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig5.png)

Фильтрация оповещений может быть удобна в ситуациях, когда требуется сузить область просматриваемых данных на панели мониторинга. Например, может потребоваться проверить оповещения безопасности, возникшие за последние 24 часа, поскольку исследуется потенциальное нарушение безопасности в системе.

Большинство оповещений системы безопасности содержат рекомендации, которые необходимо применить, однако оповещения, которые представляют собой ложные срабатывания для вашей среды или указывают на неожиданное поведение определенного ресурса, следует игнорировать. В любом случае можно скрыть рекомендации для конкретного ресурса, используя пункт**Отклонить**.

Чтобы отклонить задачу, щелкните ее правой кнопкой мыши, после чего вы увидите пункт **Отклонить**, как показано ниже.

![Параметр отклонения в Центре безопасности Azure](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig6.png)

В коллекции оповещений для каждого оповещения содержатся уникальное описание и исправление. Оповещения, отображаемые в Центре безопасности, основываются на сценарии атаки. Оповещения, связанные с приведенными ниже сценариями атак, запускаются модулем Майкрософт.

- **Обнаружение атак методом подбора через данные сети**: основано на моделях машинного обучения на основе данных сетевого трафика.
- **Обнаружение атак методом подбора через данные конечной точки**: основано на запросах Центром безопасности Azure журналов компьютера. Это позволяет различать успешные и неудачные попытки.
- **Виртуальные машины, взаимодействующие с вредоносными IP-адресами**: основано на обнаружении Центром безопасности Azure компьютеров, безопасность которых нарушена программами-роботами и которые взаимодействуют с серверами контроля и управления (и наоборот).

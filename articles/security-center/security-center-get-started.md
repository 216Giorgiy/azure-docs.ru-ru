# <a name="azure-security-center-quick-start-guide"></a>Краткое руководство по центру безопасности Azure
Эта статья поможет вам быстро начать работу с центром безопасности Azure. Она содержит инструкции по использованию функций центра безопасности для мониторинга политик безопасности и управления ими.

## <a name="prerequisites"></a>Предварительные требования
Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной учетной записи](https://azure.microsoft.com/pricing/free-trial/).

Уровень "Бесплатный" центра безопасности автоматически активирован во всех подписках Azure. В рамках этого уровня предусмотрены политики безопасности, непрерывная оценка безопасности и практические рекомендации по безопасности для защиты ресурсов Azure.

Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения о портале Azure см. в связанной [документации](https://azure.microsoft.com/documentation/services/azure-portal/).

## <a name="permissions"></a>Разрешения
В центре безопасности отображается только информация, связанная с ресурсами, для которых вам назначена роль владельца, участника или читателя в рамках подписки или группы ресурсов, к которым принадлежит ресурс. Подробные сведения о ролях и разрешенных действиях в центре безопасности см. в статье [Permissions in Azure Security Center](security-center-permissions.md) (Разрешения в центре безопасности Azure).

## <a name="data-collection"></a>Сбор данных
Центр безопасности собирает данные c виртуальных машин Azure и c компьютеров, не относящихся к Azure, чтобы отслеживать уязвимости и угрозы безопасности. Для сбора данных используется агент Microsoft Monitoring Agent, который считывает различные конфигурации, связанные с безопасностью, и журналы событий с компьютера, а также копирует данные в рабочую область для анализа. Центр безопасности подготавливает агент Microsoft Monitoring Agent к работе на всех имеющихся и создаваемых поддерживаемых виртуальных машинах Azure. Дополнительные сведения о том, как происходит сбор данных, см. в статье [Включение сбора данных](security-center-enable-data-collection.md).

Мы настоятельно рекомендуем использовать автоматическую подготовку. Она обязательна для подписок уровня "Стандартный" в центре безопасности. Если отключить ее, мониторинг безопасности ваших ресурсов будет ограничен.

Дополнительные сведения о ценовых категориях "Бесплатный" и "Стандартный" см. в статье [Цены на центр безопасности](security-center-pricing.md).

Следующие шаги описывают, как получить доступ к компонентам центра безопасности и как их использовать.

## <a name="access-security-center"></a>Доступ к Центру безопасности
Для получения доступа к Центру безопасности выполните на портале следующие действия.

1. В меню **Microsoft Azure** выберите пункт **Центр безопасности**.

   ![Меню Azure][1]
2. При первом входе в центр безопасности открывается колонка **приветствия**. Выберите **Запустить центр безопасности**, чтобы открыть **центр безопасности**.
   ![Экран приветствия][10]
3. Когда вы запустите центр безопасности из колонки приветствия или через меню Microsoft Azure, откроется колонка **Центр безопасности**. Чтобы в следующий раз вы могли быстро открыть колонку **Центр безопасности**, выберите параметр **Закрепить колонку на панели мониторинга** (вверху справа).
   ![Параметр "Закрепить колонку на панели мониторинга"][2]

## <a name="use-security-center"></a>Использование Центра безопасности
В центре безопасности можно настроить политики безопасности для подписок Azure и групп ресурсов. Давайте настроим политику безопасности для вашей подписки.

1. В главном меню центра безопасности выберите **Политика безопасности**.
2. В разделе **Центра безопасности — политика безопасности** выберите подписку.
3. В разделе **Политика безопасности — сбор данных** включена функция **Автоматическая подготовка**. Центр безопасности подготавливает агент Microsoft Monitoring Agent к работе на всех имеющихся и создаваемых поддерживаемых виртуальных машинах Azure.

    ![Политика безопасности][12]

4. В колонке **Политика безопасности** выберите компонент политики **Политика безопасности**.

     ![Политика безопасности][11]

5. В разделе **Показать рекомендации для** включите рекомендации, которые вы хотите добавить в политику безопасности. Примеры:

   * Если установить параметр **Обновления системы** в значение **Включено**, для всех поддерживаемых виртуальных машин будет выполняться проверка наличия неустановленных обновлений ОС.
   * Установка параметра **Уязвимости ОС** в значение **Включено** обеспечивает сканирование виртуальных машин для выявления конфигураций ОС, которые могут сделать виртуальную машину более уязвимой к атакам.

6. Щелкните **Сохранить**.

### <a name="view-recommendations"></a>Просмотр рекомендаций
1. Вернитесь в колонку **Центр безопасности** и выберите элемент **Рекомендации**. Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. Когда центр безопасности выявляет потенциальные уязвимости системы безопасности, он создает рекомендации, отображая их в колонке **Рекомендации**.
   ![Рекомендации в центре безопасности Azure][5]
2. Для просмотра дополнительной информации и (или) выполнения действий по устранению проблем выберите нужную рекомендацию в колонке **Рекомендации**.

### <a name="view-the-security-state-of-your-resources"></a>Просмотр состояния безопасности ресурсов
1. Вернитесь в колонку **Центр безопасности** . Раздел **Предотвращение** на панели мониторинга содержит индикаторы состояния безопасности для виртуальных машин, сети, данных и приложений.
2. Для получения дополнительных сведений выберите **Вычисления**. Откроется колонка **Вычисления** с тремя вкладками:

  - **Обзор** — содержит рекомендации по мониторингу и виртуальным машинам.
  - **Виртуальные машины и компьютеры** — содержит текущее состояние безопасности всех виртуальных машин и компьютеров.
  - **Облачные службы** — содержит список веб-ролей и рабочих ролей, отслеживаемых с помощью центра безопасности.

    ![Работоспособность системы безопасности вычислений][6]

3. На вкладке **Обзор** выберите одну из рекомендаций, чтобы просмотреть дополнительные сведения и (или) настроить необходимые элементы управления.
4. На вкладке **Виртуальные машины и компьютеры** выберите ресурс для просмотра дополнительных сведений.

### <a name="view-security-alerts"></a>Просмотр оповещений безопасности
1. Вернитесь в колонку **Центр безопасности** и щелкните плитку **Оповещения безопасности**. Откроется колонка **Оповещения безопасности**, где отображается список оповещений. Центр безопасности создает эти оповещения по итогам анализа журналов безопасности и сетевой активности. Кроме того, здесь отображаются оповещения от интегрированных партнерских решений.
   ![Оповещения безопасности в Центре безопасности Azure][7]

2. Щелкните оповещение, чтобы увидеть дополнительные сведения. В данном примере мы выберем **Modified system binary discovered in dump filter** (В фильтре дампов обнаружен измененный системный двоичный файл). Откроется колонка с дополнительными сведениями об этом оповещении.
   ![Сведения об оповещении безопасности в центре безопасности Azure][8]

### <a name="view-the-health-of-your-partner-solutions"></a>Просмотрите состояние партнерского решения. Для этого выполните следующее:
1. Вернитесь в колонку **Центр безопасности** . С помощью элемента **Решения безопасности** можно быстро оценить состояние работоспособности решений партнеров, интегрированных в подписку Azure.
2. Выберите элемент **Решения безопасности**. Откроется колонка со списком решений партнеров, подключенных к центру безопасности.
   ![Решения партнеров][9]
3. Выберите партнерское решение. Откроется колонка, в которой отображается состояние партнерского решения и связанных ресурсов. Нажмите кнопку **Консоль решений** , чтобы открыть партнерский интерфейс управления для этого решения.

   ![Решения партнеров][13]

## <a name="next-steps"></a>Дальнейшие действия
Из этой статьи вы узнали о мониторинге безопасности и управлении политиками в центре безопасности. Теперь, когда вы ознакомились с функциями центра безопасности, переходите к следующим действиям.

* Сведения о настройке политики безопасности для подписки Azure см. в разделе [Настройка политик безопасности в Центре безопасности Azure](security-center-policies.md).
* Чтобы использовать рекомендации центра безопасности для защиты ресурсов Azure, ознакомьтесь с разделом [Управление рекомендациями по безопасности в Центре безопасности Azure](security-center-recommendations.md).
* Сведения о просмотре текущих оповещений безопасности и управлении ими см. в разделе [Управление оповещениями безопасности в Центре безопасности Azure и реагирование на них](security-center-managing-and-responding-alerts.md).
- Чтобы узнать больше об интеграции с решениями партнеров для повышения общей безопасности, ознакомьтесь с разделом [Интеграция партнеров и решений в центре безопасности Azure](security-center-partner-integration.md).
- Сведения об управлении данными и их защите в центре безопасности см. в разделе [Защита данных в центре безопасности Azure](security-center-data-security.md).
* Ознакомьтесь с [возможностями расширенного обнаружения угроз](security-center-detection-capabilities.md), которые включены в [уровень "Стандартный"](security-center-pricing.md) центра безопасности. Уровень "Стандартный" предоставляется бесплатно первые 60 дней.
* Если у вас есть вопросы, см. [часто задаваемые вопросы о центре безопасности Azure](security-center-faq.md).

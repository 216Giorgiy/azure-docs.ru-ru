# Начало работы с Центром безопасности Azure

Этот документ поможет вам быстро начать работу с Центром безопасности Azure. Он содержит инструкции по работе с компонентами мониторинга политик безопасности и управления ими, а также ссылки на дальнейшие действия.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. В документе приводится обзор службы с помощью примера развертывания. Он не является пошаговым руководством.

## Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Предварительные требования

Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Центр безопасности включается в подписке. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/).

 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения см. в [документации по порталу](https://azure.microsoft.com/documentation/services/azure-portal/).


## Доступ к Центру безопасности

Для получения доступа к Центру безопасности выполните на портале следующие действия.

1. Выберите **Обзор** и найдите параметр **Центр безопасности**. ![Доступ к Центру безопасности Azure на портале][1]

2. Выберите **Центр безопасности**. Откроется колонка **Центр безопасности**.
3. Для того чтобы быстро открывать колонку **Центр безопасности** в будущем, выберите параметр **Закрепить колонку на панели мониторинга** (вверху справа). ![Параметр "Закрепить колонку на панели мониторинга"][2]

## Использование Центра безопасности

Настройка **политики** безопасности для подписки.

4. Щелкните элемент **Политика безопасности** в колонке **Центр безопасности**.
5. В колонке **Политика безопасности — определить политику для подписки** выберите подписку.
6. В колонке **Политика безопасности** включите **Сбор данных** для автоматической сборки журналов. Кроме того, включение **сбора данных** позволит подготовить расширение мониторинга для всех действующих и новых виртуальных машин в подписке.
7. Щелкните **Выбор учетной записи хранения**. Для каждого региона, в котором у вас есть виртуальные машины, выберите учетную запись хранения, где хранятся данные, собираемые с этих виртуальных машин. Если вы не выберете учетную запись хранения для каждого региона, она будет создана автоматически. В целях безопасности собранные данные логически изолируются от других данных клиентов.
8. Включите **Рекомендации**, которые хотели бы видеть в политике безопасности. Примеры:

  - Включение **обновлений системы** обеспечивает сканирование всех поддерживаемых виртуальных машин на отсутствие обновлений ОС.
  - Включение **правил базовых шаблонов** обеспечивает сканирование виртуальных машин для определения конфигураций ОС, которые могут повысить уязвимость виртуальной машины к атакам.

![Плитка "Политика безопасности" в Центре безопасности Azure][3]

Работа с **рекомендациями**.

9. Вернитесь в колонку **Центр безопасности** и выберите элемент **Рекомендации**. Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. При обнаружении потенциальной уязвимости системы безопасности здесь отображается рекомендация.
11.	Для просмотра дополнительной информации и (или) устранения соответствующих проблем щелкните нужную рекомендацию.

![Рекомендации в Центре безопасности Azure][4]

Просмотр состояния работоспособности и безопасности ресурсов в разделе **Работоспособность ресурсов**.

12.	Вернитесь в колонку **Центр безопасности**.
13.	Элемент **Работоспособность ресурсов** содержит индикаторы состояния безопасности для **виртуальных машин**, **сети**, **SQL** и **приложений**.
14.	Для просмотра дополнительных сведений выберите **Виртуальные машины**.
15.	В колонке **Виртуальные машины** отображается сводка состояния, которая показывает состояние вредоносных программ, обновлений системы, перезагрузки и правил базовых шаблонов для ваших виртуальных машин.
16.	В разделе **Меры профилактики** выберите элемент, чтобы просмотреть дополнительные сведения и (или) настроить необходимые элементы управления.
17.	Для просмотра дополнительных сведений о конкретных виртуальных машинах воспользуйтесь детализацией.

![Плитка "Работоспособность ресурсов" в Центре безопасности Azure][5]

Работа с **оповещениями безопасности**.

19.	Вернитесь в колонку **Центр безопасности** и выберите элемент **Оповещения безопасности**. В колонке **Оповещения безопасности** отображается список оповещений. Когда Центр безопасности анализирует журналы безопасности и сетевую активность, создаются оповещения. Кроме того, включены оповещения, отправляемые встроенными решениями партнеров. ![Оповещения безопасности в Центре безопасности Azure][6]

21.	Щелкните оповещение, чтобы увидеть дополнительные сведения.

  ![Сведения оповещений безопасности в Центре безопасности Azure][7]

# Начало работы с Центром безопасности Azure

Этот документ поможет вам быстро начать работу с Центром безопасности Azure. Он содержит инструкции по работе с компонентами мониторинга политик безопасности и управления ими, а также ссылки на дальнейшие действия.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. В документе приводится обзор службы с помощью примера развертывания. Он не является пошаговым руководством.

## Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Предварительные требования

Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Центр безопасности включается в подписке. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/).

 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения см. в [документации по порталу](https://azure.microsoft.com/documentation/services/azure-portal/).


## Доступ к Центру безопасности

Для получения доступа к Центру безопасности выполните на портале следующие действия.

1. Выберите **Обзор** и найдите параметр **Центр безопасности**. ![Доступ к Центру безопасности Azure на портале][1]

2. Выберите **Центр безопасности**. Откроется колонка **Центр безопасности**.
3. Для того чтобы быстро открывать колонку **Центр безопасности** в будущем, выберите параметр **Закрепить колонку на панели мониторинга** (вверху справа). ![Параметр "Закрепить колонку на панели мониторинга"][2]

## Использование Центра безопасности

В центре безопасности можно настроить политики безопасности для подписок Azure и групп ресурсов. Давайте настроим **политику** безопасности для вашей подписки:

1. Щелкните плитку **Политика** в колонке **Центр безопасности**. ![Центр безопасности][3]

2. В колонке **Политика безопасности — определить политику для подписки** выберите подписку. ![Колонка "Политика безопасности" в Центре безопасности Azure][4]

3. В колонке **Политика безопасности** включите **Сбор данных** для автоматической сборки журналов. Кроме того, включение **сбора данных** позволит подготовить расширение мониторинга для всех действующих и новых виртуальных машин в подписке.
4. Выберите **Выбрать учетную запись хранения для региона**. Для каждого региона, в котором у вас есть виртуальные машины, выберите учетную запись хранения, где хранятся данные, собираемые с этих виртуальных машин. Если вы не выберете учетную запись хранения для каждого региона, она будет создана автоматически. В целях безопасности собранные данные логически изолируются от других данных клиентов.

     > [AZURE.NOTE] Рекомендуем сначала включить сбор данных и выбрать учетную запись хранения на уровне подписки. Политики безопасности можно настроить на уровне подписки Azure и группы ресурсов, но настройка сбора данных и учетной записи хранения осуществляется только на уровне подписки.

5. Включите **Рекомендации**, которые хотели бы видеть в политике безопасности. Примеры:

 - При включении **обновлений системы** все поддерживаемые виртуальные машины будут просканированы на наличие обновлений ОС.
 - Включение **правил базовых шаблонов** обеспечивает сканирование виртуальных машин для определения конфигураций ОС, которые могут повысить уязвимость виртуальной машины к атакам.

Работа с **рекомендациями**.

1. Вернитесь в колонку **Центр безопасности** и выберите элемент **Рекомендации**. Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. При обнаружении потенциальной уязвимости системы безопасности здесь отображается рекомендация.
2.	Для просмотра дополнительной информации и устранения соответствующих проблем выберите нужную рекомендацию. ![Рекомендации в Центре безопасности Azure][5]

Просмотр состояния работоспособности и безопасности ресурсов в разделе **Работоспособность системы безопасности ресурсов**.

1.	Вернитесь в колонку **Центр безопасности**.
2.	Плитка **Работоспособность системы безопасности ресурсов** содержит индикаторы состояния безопасности для **виртуальных машин**, **сети**, **SQL** и **приложений**.
3.	Для просмотра дополнительных сведений выберите **Виртуальные машины**.
4.	В колонке **Виртуальные машины** отображается сводная информация о состоянии, которая показывает состояние антивирусных программ, обновлений системы, перезагрузок и правил базовых шаблонов для ваших виртуальных машин.
5.	В разделе **Рекомендации для виртуальной машины** выберите элемент, чтобы просмотреть дополнительные сведения о нем или настроить необходимые элементы управления.
6.	Разверните сведения для просмотра дополнительной информации о конкретной виртуальной машине. ![Плитка "Работоспособность ресурсов" в Центре безопасности Azure][6]

Работа с **оповещениями системы безопасности**.

1.	Вернитесь в колонку **Центр безопасности** и выберите элемент **Оповещения безопасности**. В колонке **Оповещения безопасности** отображается список оповещений. Когда Центр безопасности анализирует журналы безопасности и сетевую активность, создаются оповещения. Кроме того, включены оповещения, отправляемые встроенными решениями партнеров. ![Оповещения безопасности в Центре безопасности Azure][7]

2.	Щелкните оповещение, чтобы увидеть дополнительные сведения. ![Сведения оповещений безопасности в Центре безопасности Azure][8]

Просмотр состояния **партнерского решения**.

1. Вернитесь в колонку **Центр безопасности**. С помощью плитки **Решения партнеров** вы можете отслеживать состояние работоспособности партнерских решений, интегрированных в вашу подписку Azure.
2. Выберите плитку **Решения партнеров**. Откроется колонка со списком партнерских решений, подключенных к центру безопасности. ![Решения партнеров][9]

3. Выберите партнерское решение. В этом примере мы выберем решение **F5-WAF2**. Откроется колонка, в которой отображается состояние партнерского решения и связанных с ним ресурсов. Нажмите кнопку **Консоль решения**, чтобы открыть партнерский интерфейс управления для этого решения. ![Сведения о партнерском решении][10]

Параметр **состояния** партнерского решения может иметь следующие значения:

- "Исправно" (зеленый) — нет никаких проблем с работоспособностью;
- "Неисправно" (красный) — проблема работоспособности, которая требует немедленного внимания;
- "Создание отчетов остановлено" (оранжевый) — решение перестало сообщать о работоспособности;
- "Неизвестно" (оранжевый) — работоспособность решения неизвестна в данный момент, поскольку произошел сбой процесса добавления нового ресурса в существующее решение;
- "Не сообщается" (серый) — решение еще не сообщало о своем состоянии; такое состояние может отображаться для решения, если оно только что подключилось и продолжает развертывание.

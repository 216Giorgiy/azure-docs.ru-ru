#Наблюдение за работоспособностью системы безопасности в Центре безопасности Azure
Этот документ поможет использовать возможности мониторинга в Центре безопасности Azure для наблюдения за соответствием политикам.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Что такое Центр безопасности Azure?
Центр безопасности Azure помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает в себя встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

##Что такое наблюдение за работоспособностью системы безопасности?
Наблюдение часто определяется как отслеживание и ожидание возникновения события для реагирования на какую-либо ситуацию. Наблюдение за системой безопасности подразумевает наличие упреждающей стратегии аудита ресурсов для выявления систем, которые не соответствуют стандартам организации или рекомендациям.

##Наблюдение за работоспособностью системы безопасности
После включения [политик безопасности](security-center-policies.md) для ресурсов подписки Центр безопасности Azure выполняет анализ безопасности ресурсов, чтобы определить потенциальные уязвимости. Хотя сведения о конфигурации сети становятся доступными немедленно, получение сведений о конфигурации виртуальной машины, таких как состояние обновлений системы безопасности и конфигурация операционной системы, может занять не меньше часа. Сведения о состоянии безопасности ваших ресурсов, наряду с возникшими проблемами, можно просмотреть в колонках раздела **Работоспособность безопасности ресурсов**. Список таких проблем также можно просмотреть в колонке **Рекомендации**.

Дополнительные сведения о том, как применить рекомендации, см. в разделе [Реализация рекомендаций по безопасности в Центре безопасности Azure](security-center-recommendations.md).

Элемент **Работоспособность ресурса** служит для отслеживания состояния безопасности ресурсов. Вы увидите ряд проблем с высоким и средним уровнем серьезности, требующих принятия мер, как показано ниже:

![Работоспособность ресурсов](./media/security-center-monitoring/security-center-monitoring-fig1-new.png)

Включенные политики безопасности будут влиять на типы средств контроля, которые отслеживаются на этой плитке. Если Центр безопасности Azure выявит уязвимость, которую необходимо устранить (например, виртуальную машину, на которой отсутствуют обновления безопасности, или подсеть без [группы безопасности сети](virtual-networks-nsg.md)), она будет указана здесь.

###Виртуальная машина.
Если щелкнуть **Виртуальные машины** в элементе **Работоспособность ресурса**, откроется колонка **Виртуальные машины** с более подробными данными о действиях по адаптации и предотвращению, а также со списком всех виртуальных машин, отслеживаемых Центром безопасности Azure, как показано ниже.

![Отсутствует обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig2-new.png)

Открыв эту колонку, вы увидите три раздела:

- Процесс внедрения
- Рекомендации по виртуальным машинам
- Виртуальные машины

Для каждого раздела можно выбрать отдельные параметры, чтобы просмотреть дополнительные сведения о рекомендуемых действиях по устранению этой проблемы. В следующих разделах они будут рассмотрены более подробно.

####Процесс внедрения
В этом разделе указывается общее количество виртуальных машин, инициализированных для сбора данных, и их текущее состояние. Когда на виртуальной машине инициализируется коллекция данных, она будет готова к приему политик безопасности от Центра безопасности Azure. Щелкнув эту запись, вы попадете в колонку **Инициализация коллекции данных** и увидите имена виртуальных машин и текущее состояние коллекции данных в колонке **Состояние инициализации**, как показано ниже:

![Состояние инициализации](./media/security-center-monitoring/security-center-monitoring-fig3-new.png)


####Рекомендации по виртуальным машинам
В этом разделе приводится ряд рекомендаций по каждой виртуальной машине, отслеживаемой центром безопасности Azure. Первый столбец содержит описание рекомендации, второй — общее число виртуальных машин, на которые она влияет, а третий — степень серьезности, как показано ниже:

![Рекомендации по виртуальным машинам](./media/security-center-monitoring/security-center-monitoring-fig4-new.png)

Каждая рекомендация включает набор действий, которые будут выполнены при ее выборе. Например, если щелкнуть **Отсутствуют обновления системы**, откроется колонка **Отсутствуют обновления системы** со списком виртуальных машин, на которых отсутствуют исправления, и уровнем серьезности для каждого из них, как показано ниже:

![Отсутствуют обновления системы](./media/security-center-monitoring/security-center-monitoring-fig5-new.png)

В колонке "Отсутствуют обновления системы" отобразится таблица со следующими сведениями:

- **ВИРТУАЛЬНАЯ МАШИНА**: имя виртуальной машины, на которой отсутствуют обновления.
- **ОБНОВЛЕНИЯ СИСТЕМЫ**: число отсутствующих обновлений системы.
- **ВРЕМЯ ПОСЛЕДНЕГО СКАНИРОВАНИЯ**: время последней проверки виртуальной машины центром безопасности Azure на наличие обновлений.
- **СОСТОЯНИЕ** — текущее состояние рекомендации: 
	- **Открыто** — рекомендация еще не применена;
	- **Выполняется** — сейчас рекомендация применяется к ресурсам, от вас не требуется никаких действий;
	- **Разрешено** — рекомендация уже выполнена (в этом состоянии строка неактивна).
- **СЕРЬЕЗНОСТЬ** — описание уровня серьезности этой конкретной рекомендации: 
	- **Высокий** — уязвимость существует для важных ресурсов (приложение, виртуальная машина, группы безопасности сети) и требует внимания;
	- **Средний** — для завершения процесса или устранения уязвимости требуются некритические или дополнительные действия;
	- **Низкий** — уязвимость должна быть разрешена, но не требует немедленного внимания. (По умолчанию низкий уровень не представлен, однако вы можете выполнить фильтрацию по этому значению, чтобы просмотреть соответствующие рекомендации.)

Чтобы просмотреть сведения о рекомендуемых обновлениях, щелкните имя виртуальной машины. Откроется новая колонка для этой виртуальной машины со списком отсутствующих обновлений:

![Отсутствуют обновления системы на виртуальной машине](./media/security-center-monitoring/security-center-monitoring-fig6-new.png)

> [AZURE.NOTE] Приведенные здесь рекомендации по безопасности совпадают с указанными в колонке "Рекомендации". Дополнительные сведения о применении рекомендаций см. в разделе [Реализация рекомендаций по безопасности в Центре безопасности Azure](security-center-recommendations.md). Это относится не только к виртуальным машинам, но и ко всем ресурсам, доступным на плитке "Работоспособность ресурса".

####Виртуальные машины
В разделе "Виртуальные машины" содержатся общие сведения обо всех виртуальных машинах, а также рекомендации. В каждом столбце представлен один набор рекомендаций, как показано ниже:

![Виртуальные машины](./media/security-center-monitoring/security-center-monitoring-fig7-new.png)

Значок, который отображается под каждой рекомендацией, позволяет определить, какие виртуальные машины требуют внимания и что именно им нужно.

В приведенном выше примере для одной из виртуальных машин дана критическая рекомендация в отношении защиты от вредоносных программ. Чтобы получить дополнительные сведения о виртуальной машине, щелкните ее — откроется новая колонка с данными этой машины, как показано ниже:

![Сведения о безопасности виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig8-new.png)

В этой колонке вы найдете сведения о безопасности виртуальной машины. В нижней части этой колонки будут указаны рекомендуемые действия и важность каждой рекомендации.

###Сеть
Состояние предотвращения для сети относится к виртуальным сетям, отслеживаемым с помощью Центра безопасности Azure. Если щелкнуть элемент **Сеть** на плитке "Работоспособность ресурса", откроется колонка **Сеть** с дополнительными сведениями, как показано ниже:

![Сеть](./media/security-center-monitoring/security-center-monitoring-fig9-new.png)

Открыв эту колонку, вы увидите два раздела — рекомендации по сети и сеть.
 
Для каждого раздела можно выбрать отдельные параметры, чтобы просмотреть дополнительные сведения о рекомендуемых действиях по устранению этой проблемы. В следующих разделах они будут рассмотрены более подробно.

####Рекомендации по сети

Как и в разделе сведений о работоспособности ресурсов виртуальных машин, в этой колонке содержится сводный список проблем в верхней части колонки и список отслеживаемых сетей в нижней.

В разделе детализации состояния сети перечисляются действия по предотвращению, такие как [Не включены списки контроля доступа для конечных точек](virtual-machines-set-up-endpoints.md), [Не включены группы безопасности сети](virtual-networks-nsg.md), "Работоспособные подсети" и **Доступ в NSG не ограничен**. Если вы щелкнете одну из этих рекомендаций в новой колонке, откроется новая вкладка с более подробными сведениями, как показано ниже:

![Конечная точка](./media/security-center-monitoring/security-center-monitoring-fig10-new.png)

В этом примере колонка **Ограничение доступа через общедоступные внешние конечные точки** содержит список групп сетевой безопасности (NSG), которые входят в это предупреждение, подсеть и сеть, с которой связана соответствующая группа сетевой безопасности, текущее состояние рекомендации и степень ее серьезности. Если вы щелкните группу сетевой безопасности, откроется другая вкладка следующего вида:

![Ограничить конечную точку](./media/security-center-monitoring/security-center-monitoring-fig11-new.png)

Она содержит сведения о группе сетевой безопасности, ее расположение и список действующих правил для входящих подключений. В нижней части колонки указывается виртуальная машина, связанная с этой группой сетевой безопасности. Если вы хотите включить правила для входящих подключений, которые блокируют открытый в настоящее время нежелательные порт, или изменить источник текущего правила для входящих подключений, нажмите кнопку **Изменить правило для входящего подключения** в верхней части колонки.

####Сеть
В разделе **Сеть** дается иерархическое представление группы ресурсов, подсети и сетевого интерфейса, связанных с вашей виртуальной машиной:

![Дерево сети](./media/security-center-monitoring/security-center-monitoring-fig121-new.png)

В этом разделе [виртуальные машины на основе диспетчера ресурсов отделяются от классических виртуальных машин](resource-manager-deployment-model.md), что позволяет быстро определить, доступны ли для них сетевые функции управления службой Azure (ASM/классич.) или управления ресурсами Azure (ARM). Чтобы получить отсюда доступ к сетевой карте, разверните подсеть и щелкните имя виртуальной машины. Если это действие выполняется для виртуальной машины на основе диспетчера ресурсов, откроется новая вкладка следующего вида:

![Дерево сети](./media/security-center-monitoring/security-center-monitoring-fig13-new.png)

Эта колонка содержит сведения о сетевой карте и действующие в ее отношении рекомендации. Если вы выбрали классическую виртуальную машину, откроется новая колонка с другими параметрами:

![Список управления доступом](./media/security-center-monitoring/security-center-monitoring-fig14-new.png)

Здесь можно изменить общий и закрытый порт и создать ACL для этой виртуальной машины.

###SQL
Если щелкнуть **SQL** на элементе **Работоспособность ресурса**, откроется колонка "SQL", содержащая рекомендации, связанные с такими проблемами, как отключенный аудит, отключенное прозрачное шифрование данных и общее состояние работоспособности базы данных.

![Работоспособность ресурса SQL](./media/security-center-monitoring/security-center-monitoring-fig15-new.png)

Можно щелкнуть любую из этих рекомендаций и получить дополнительные сведения для принятия дальнейших мер по устранению проблемы. В приведенном ниже примере показана развернутая рекомендация **Аудит баз данных не включен**.

![Работоспособность ресурса SQL](./media/security-center-monitoring/security-center-monitoring-fig16-new.png)

В колонке **Включение аудита в базах данных SQL** вы найдете список баз данных SQL, сервер, на котором они размещены, указание на то, унаследована ли эта настройка от сервера или является уникальной для базы данных, текущее состояние проблемы и степень ее серьезности. Если вы выберете рекомендацию, щелкнув базу данных, откроется колонка **Аудит и обнаружение угроз** следующего вида:

![Работоспособность ресурса SQL](./media/security-center-monitoring/security-center-monitoring-fig17-new.png)

Чтобы включить эту возможность, выберите **ВКЛ** под параметр **Аудит** и нажмите кнопку **Сохранить**.

###Приложения
Если рабочая нагрузка Azure предусматривает приложения, размещаемые на [виртуальных машинах диспетчера ресурсов](resource-manager-deployment-model.md) с предоставляемыми веб-портами (TCP-порты 80 и 443), Центр безопасности Azure может отслеживать их для выявления потенциальных проблем безопасности и предлагать рекомендации по исправлению. Если щелкнуть элемент **Приложения**, откроется колонка **Приложения**, содержащая ряд рекомендаций, приведенных в разделе действий для предотвращения, а также сведения о приложениях с детализацией по узлу или виртуальному IP-адресу, как показано ниже.

![Индекс безопасности приложений](./media/security-center-monitoring/security-center-monitoring-fig18-new.png)

Аналогично рекомендациям для других ресурсов выше, вы можете просматривать дополнительные сведения о проблеме и способах ее устранения, щелкнув нужную рекомендацию. Пример, показанный на рисунке ниже, — это приложение, которое было определено как **небезопасное веб-приложение**. При выборе приложения, которое было признано небезопасным, открывается другая колонка со следующим доступным параметром:

![Приложения](./media/security-center-monitoring/security-center-monitoring-fig19-new.png)

Колонка **Небезопасные веб-приложения** будет содержать список всех виртуальных машин, содержащих приложения, которые не считаются безопасными. В списке отображаются имя виртуальной машины, текущее состояние проблемы и степень безопасности. Если вы щелкните веб-приложение, откроется колонка **Добавление брандмауэра веб-приложения**, где можно установить сторонний брандмауэр веб-приложения, как показано ниже:

![Добавить WAF](./media/security-center-monitoring/security-center-monitoring-fig20-new.png)

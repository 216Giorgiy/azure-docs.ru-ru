#Наблюдение за работоспособностью системы безопасности в Центре безопасности Azure
В этом пошаговом руководстве содержатся подробные сведения о возможностях мониторинга в Центре безопасности Azure.

> [AZURE.NOTE]Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

##Что такое наблюдение за работоспособностью системы безопасности?
Наблюдение часто определяется как отслеживание и ожидание возникновения события для реагирования на какую-либо ситуацию. Наблюдение за системой безопасности подразумевает наличие упреждающей стратегии аудита ресурсов для выявления систем, которые не соответствуют стандартам организации или рекомендациям.

##Мониторинг соответствия политикам
После включения [политик безопасности](security-center-policies.md) для ресурсов подписки Центр безопасности Azure выполняет анализ безопасности ресурсов, чтобы определить потенциальные уязвимости. Хотя сведения о конфигурации сети становятся доступными немедленно, получение сведений о конфигурации виртуальной машины, таких как состояние обновлений системы безопасности и конфигурация операционной системы, может занять не меньше часа. Сведения о состоянии безопасности ваших ресурсов, наряду с возникшими проблемами, можно просмотреть в колонках раздела "Работоспособность безопасности ресурсов". Список таких проблем также можно просмотреть в колонке "Рекомендации".

Дополнительные сведения о том, как применить рекомендации, см. в разделе [Применение рекомендаций по безопасности в Центре безопасности Azure](security-center-recommendations.md).

##Работоспособность ресурса
Плитка **Работоспособность ресурса** служит для отслеживания состояния безопасности ресурсов. Вы увидите ряд проблем с высоким и средним уровнем серьезности, требующих принятия мер, как показано ниже:

![Работоспособность ресурсов](./media/security-center-monitoring/security-center-monitoring-fig1.png)

Включенные политики безопасности будут влиять на типы средств контроля, которые отслеживаются на этой плитке. Если Центр безопасности Azure выявит уязвимость, которую необходимо устранить (например, виртуальную машину, на которой отсутствуют обновления безопасности, или подсеть без [группы безопасности сети](virtual-networks-nsg.md)), она будет указана здесь.

###Виртуальная машина.
Если щелкнуть элемент **Виртуальные машины** в плитке **Работоспособность ресурса**, откроется колонка**Виртуальные машины** с более подробными сведениями о действиях по адаптации и предотвращению, а также со списком всех виртуальных машин, отслеживаемых Центром безопасности Azure, как показано ниже:

![Отсутствует обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig2.png)

Открыв эту колонку, вы увидите три раздела. Для каждого раздела можно выбрать отдельные параметры, чтобы просмотреть дополнительные сведения о рекомендуемых действиях по устранению этой проблемы.

Например, если щелкнуть **Отсутствуют обновления системы** в разделе **Действия для предотвращения**, откроется колонка**Отсутствуют обновления системы** со списком виртуальных машин, на которых отсутствуют исправления, и уровнем серьезности для каждого из них, как показано ниже:

![Отсутствует обновление системы, исправление](./media/security-center-monitoring/security-center-monitoring-fig3.png)

Чтобы просмотреть сведения о рекомендуемых обновлениях, щелкните имя виртуальной машины. Откроется новая колонка для этой виртуальной машины со списком отсутствующих обновлений:

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig4.png)

> [AZURE.NOTE]Приведенные здесь рекомендации по безопасности совпадают с указанными в колонке "Рекомендации". Дополнительные сведения о применении рекомендаций см. в разделе [Применение рекомендаций по безопасности в Центре безопасности Azure](security-center-recommendations.md). Это относится не только к виртуальным машинам, но и ко всем ресурсам, доступным на плитке "Работоспособность ресурса".

###Сеть
Состояние предотвращения для сети относится к виртуальным сетям, отслеживаемым с помощью Центра безопасности Azure. Если щелкнуть элемент **Сеть** на плитке "Работоспособность ресурса", откроется колонка **Сеть** с дополнительными сведениями, как показано ниже:

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig5.png)

Как и в разделе сведений о работоспособности ресурсов виртуальных машин, в этой колонке содержится сводный список проблем в верхней части колонки и список отслеживаемых сетей в нижней.

В разделе детализации состояния сети перечисляются действия по предотвращению, такие как [Не включены списки контроля доступа для конечных точек](virtual-machines-set-up-endpoints.md), [Не включены группы безопасности сети](virtual-networks-nsg.md) и "Работоспособные подсети". Можно щелкнуть любой из этих пунктов и получить дополнительные сведения для принятия дальнейших мер по устранению проблемы.

В качестве примера приведем оповещение **Не включены группы безопасности сети в подсетях**. Если щелкнуть это предупреждение, откроется колонка **Настройка отсутствующих групп безопасности сети для подсетей**. В ней можно увидеть описание подсети, для которой не включена группа безопасности сети, и список сетевых адаптеров, принадлежащих к этой подсети. Выберите подсеть, для которой вы хотите применить группу безопасности, и выберите пункт **Настроить группу безопасности сети**. Откроется колонка **Выберите группу безопасности сети**, как показано ниже:

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig6.png)

###SQL
Если щелкнуть элемент**SQL** на плитке **Работоспособность ресурса**, откроется колонка **SQL**, содержащая рекомендации, связанные с такими проблемами, как отключенный аудит, отключенное прозрачное шифрование данных и общее состояние работоспособности базы данных. В нижней части колонки содержится детализация состояния SQL Server и указаны базы данных для каждого сервера, как показано ниже:

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig7.png)

Можно щелкнуть любую из этих рекомендаций и получить дополнительные сведения для принятия дальнейших мер по устранению проблемы. В примере ниже показан раздел, в котором настраивается включение аудита на серверах SQL Server.

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig8.png)

###Приложения
Если рабочая нагрузка Azure предусматривает приложения, размещаемые на [виртуальных машинах диспетчера ресурсов](resource-manager-deployment-model.md) с предоставляемыми веб-портами (TCP-порты 80 и 443), Центр безопасности Azure может отслеживать их для выявления потенциальных проблем безопасности и предлагать рекомендации по исправлению. Если щелкнуть плитку **Приложения**, откроется колонка **Приложения**, содержащая ряд рекомендаций, приведенных в разделе действий для предотвращения, а также сведения о приложениях с детализацией по узлу или виртуальному IP-адресу, как показано ниже:

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig9.png)

Аналогично рекомендациям для других ресурсов выше, вы можете просматривать дополнительные сведения о проблеме и способах ее устранения, щелкнув нужную рекомендацию. Пример, показанный на рисунке ниже, — это приложение, которое было определено как **небезопасное веб-приложение**. При выборе приложения, которое было признано небезопасным, открывается другая колонка с предлагаемым решением (в данном случае это включение брандмауэра веб-приложения).

![Рекомендуемое обновление системы для виртуальной машины](./media/security-center-monitoring/security-center-monitoring-fig10.png)

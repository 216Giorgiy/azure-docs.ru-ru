# <a name="monitoring-resource-security-hygiene-in-azure-security-center"></a>Мониторинг гигиены безопасности ресурсов в центре безопасности Azure
В этой статье описываются возможности мониторинга, доступные в центре безопасности Azure для наблюдения за соответствием политикам, позволяющие максимально повысить безопасность ресурсов.

## <a name="what-is-resource-security-hygiene"></a>Что понимается под гигиеной безопасности ресурсов?
Наблюдение часто определяется как отслеживание и ожидание возникновения события для реагирования на какую-либо ситуацию. Гигиена безопасности ресурсов подразумевает наличие упреждающей стратегии аудита ресурсов для выявления систем, которые не соответствуют стандартам организации или рекомендациям.

## <a name="resource-security-hygiene"></a>Гигиена безопасности ресурсов
Когда вы включаете [политики безопасности](security-center-policies.md) для ресурсов в рамках подписки, центр безопасности проверяет состояние защиты ресурсов, чтобы определить потенциальные уязвимости. Хотя сведения о конфигурации сети становятся доступными сразу же, Сбор информации о конфигурации виртуальных машин и компьютеров, включая сведения о состоянии обновления системы безопасности и конфигурации операционной системы, может длиться час или дольше. Время выполнения операции зависит от числа виртуальных машин и компьютеров, установленных на агенте. Можно просмотреть полный список проблем и способы повышения безопасности сети и немедленного уменьшения риска в плитке **Рекомендации**.

Сведения о состоянии безопасности ваших ресурсов, а также возникшие проблемы можно просмотреть для отдельных типов ресурсов.

- Для наблюдения за работоспособностью ресурсов компьютера и приложений и получения рекомендаций по повышению безопасности см. раздел [Защита ваших машин и приложений в центре безопасности Azure](security-center-virtual-machine-protection.md).
- Чтобы наблюдать за сетевыми ресурсами, такими как виртуальные машины, группы безопасности сети и конечные точки, и получать рекомендации по повышению безопасности, см. раздел [Защита сети в центре безопасности Azure](security-center-network-recommendations.md). 
- Для мониторинга ресурсов данных и хранилища, например серверов SQL Server и учетных записей хранения, и получения рекомендаций по повышению безопасности см. раздел [Защита службы SQL Azure и данных в центре безопасности Azure](security-center-sql-service-recommendations.md). 
- Чтобы отслеживать ресурсы удостоверений и доступа, включая многофакторную проверку подлинности и разрешения учетных записей, и получить рекомендации по повышению безопасности, см. раздел [Мониторинг удостоверений и доступа в центре безопасности Azure](security-center-identity-access.md). 
- Чтобы отслеживать JIT-доступ к ресурсам, см. раздел [Управление доступом к виртуальным машинам с помощью JIT](security-center-just-in-time.md). 


Дополнительные сведения о том, как применить рекомендации, см. в разделе [Внедрение рекомендаций по безопасности](security-center-recommendations.md).

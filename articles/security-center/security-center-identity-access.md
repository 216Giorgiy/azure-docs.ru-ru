# <a name="monitoring-identity-and-access-in-azure-security-center"></a>Мониторинг идентификации и доступа в центре безопасности Azure
Из этого документа вы узнаете, как с помощью центра безопасности Azure отслеживать действия, связанные с идентификацией и доступом пользователей.

## <a name="why-monitor-identity-and-access"></a>Зачем нужен мониторинг идентификации и доступа?
Для предприятия идентификация должна быть плоскостью управления, а защита удостоверений — основным приоритетом. Если в прошлом в организациях были реализованы периметры и они были одними из основных защитных границ, в наше время при перемещении все большего числа данных и приложений в облако таким "периметром" становится идентификация.

Отслеживая действия по идентификации, можно принимать упреждающие меры до возникновения инцидента или ответные меры для предотвращения попыток атак. Панель мониторинга "Удостоверения и доступ" предоставляет обзор состояния идентификации, включая количество неудачных попыток входа в систему, сведения об учетной записи пользователя, которая использовалась во время таких попыток, сведения о заблокированных учетных записях, учетных записях с измененными или сброшенными паролями, а также о текущем количестве учетных записей пользователей, вошедших в систему.

## <a name="how-to-monitor-identity-and-access-activities"></a>Как отслеживать действия, связанные с доступом и идентификацией?
Чтобы получить доступ к панели мониторинга **Удостоверения и доступ** для визуализации текущих действий, связанных с идентификацией и доступом, сделайте следующее.

1.  Откройте панель мониторинга **Центр безопасности**.
2.  В области слева в разделе **Предотвращение** щелкните **Удостоверения и доступ**. При наличии нескольких рабочих областей появится селектор рабочих областей.

    ![Выбор рабочей области](./media/security-center-identity-access\security-center-identity-access-fig1.png)

    > [!NOTE]
    > Если в последнем столбце отображается элемент **UPGRADE PLAN** (План обновления), это означает, что для соответствующей рабочей области используется бесплатная подписка. Чтобы использовать эту функцию, необходимо перейти на уровень "Стандартный". Если же отображается элемент REQUIRES UPDATE (Требуется обновление), необходимо обновить [службу Azure Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview), чтобы использовать эту функцию. Дополнительные сведения о тарифном плане см. на странице с ценами на центр безопасности Azure. 
    > 
3. Если необходимо исследовать несколько рабочих областей, вы можете назначить приоритет для исследования на основе данных в столбце **Ошибки входа**. В нем указано текущее количество неудачных попыток входа в этой рабочей области. Щелкните рабочую область, которую необходимо использовать. Появится панель мониторинга **Удостоверения и доступ**.

    ![идентификация и доступ](./media/security-center-identity-access\security-center-identity-access-fig2.png)

4. Сведения, доступные на этой панели мониторинга, могут помочь вам мгновенно определить потенциально подозрительную активность. Эта панель мониторинга состоит из трех основных областей:
    * **Состояние идентификатора.** Предоставляет сводные данные о действиях идентификации, выполненных в этой рабочей области.
    * **Ошибки входа.** Позволяет быстро определить основную причину неудачной попытки входа и отображает первые десять учетных записей с наибольшим количеством неудачных попыток входа.
    * **Входы по времени.** Позволяет быстро определить количество входов в тот или иной момент времени и отображает учетные записи компьютеров с наибольшим количеством попыток входа.
    
Независимо от выбранной плитки данные на панели мониторинга будут отображаться на основе [запроса на поиск по журналам](https://docs.microsoft.com/azure/security-center/security-center-search). Отличаться будет только тип запроса и результат. Вы по-прежнему можете выбрать элемент, например компьютер, щелкнуть его и увидеть соответствующие данные.

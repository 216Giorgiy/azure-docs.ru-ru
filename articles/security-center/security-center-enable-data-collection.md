# <a name="enable-data-collection-in-azure-security-center"></a>Включение сбора данных в центре безопасности Azure

> [!NOTE]
> С начала июня 2017 г. в центре безопасности будет использоваться Microsoft Monitoring Agent для сбора и хранения данных. Дополнительные сведения см. в статье [Миграция платформы центра безопасности Azure](security-center-platform-migration.md). В этой статье описывается, как будет работать центр безопасности после перехода к использованию Microsoft Monitoring Agent.
>
>

Центр безопасности собирает с виртуальных машин данные, на основе которых оценивает состояние их безопасности, предоставляет рекомендации по безопасности и оповещает об угрозах. Когда вы впервые обращаетесь в центр безопасности, вы можете включить сбор данных на всех виртуальных машинах, входящих в подписку. Если сбор данных не включен, центр безопасности порекомендует включить его в политике безопасности для этой подписки.

При включении сбора данных центр безопасности подготавливает агент Microsoft Monitoring Agent к работе на всех имеющихся и создаваемых поддерживаемых виртуальных машинах Azure. Агент Microsoft Monitoring Agent выполняет проверку различных конфигураций, связанных с безопасностью. Кроме того, операционная система создает события в журнале событий. Примеры таких данных — тип и версия операционной системы, журналы операционной системы (журналы событий Windows), выполняющиеся процессы, имя компьютера, IP-адреса, имя пользователя, выполнившего вход, и идентификатор клиента. Microsoft Monitoring Agent считывает записи журнала событий и конфигурации и копирует данные в рабочую область для анализа. Этот агент также копирует файлы аварийных дампов в рабочую область.

Если используется категория "Бесплатный" центра безопасности, сбор данных с виртуальных машин можно отключить в политике безопасности. При отключении сбора данных ограничиваются возможности по оценке безопасности виртуальных машин. Дополнительные сведения см. в статье [Отключение сбора данных](#disabling-data-collection). Функция создания моментальных снимков диска виртуальной машины и сбора артефактов работает даже в том случае, если сбор данных отключен. Сбор данных является обязательным для подписок стандартного уровня в центре безопасности.

> [!NOTE]
> Дополнительные сведения о [ценовых категориях](security-center-pricing.md) "Бесплатный" и "Стандартный" центра безопасности.
>
>

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций

> [!NOTE]
> В документе приводится обзор службы с помощью примера развертывания. Этот документ не является пошаговым руководством.
>
>

1. В колонке **Рекомендации** выберите **Enable data collection for subscriptions** (Включить сбор данных для подписок).  Откроется колонка **Turn on data collection** (Включение сбора данных).
   ![Колонка "Рекомендации"][2]
2. В колонке **Turn on data collection** (Включение сбора данных) выберите подписку. Откроется колонка **Политика безопасности** для этой подписки.
3. Чтобы включить автоматический сбор данных журналов, в колонке **Политика безопасности** выберите **Вкл.** для параметра **Сбор данных**. Кроме того, включение сбора данных позволяет подготовить расширение мониторинга для всех имеющихся и новых поддерживаемых виртуальных машин в подписке.
4. Щелкните **Сохранить**.
5. Нажмите кнопку **ОК**.

## <a name="disabling-data-collection"></a>Отключение сбора данных
Если используется категория "Бесплатный" центра безопасности, сбор данных с виртуальных машин можно отключить в любой момент в политике безопасности. Сбор данных является обязательным для подписок стандартного уровня в центре безопасности.

1. Вернитесь в колонку **Центр безопасности** и выберите плитку **Политика**. Откроется колонка политики безопасности **Определить политику для подписки**.
   ![Выбор плитки "Политика"][5]
2. В колонке политики безопасности **Определить политику для подписки** выберите подписку, для которой необходимо отключить сбор данных.
3. Откроется колонка **Политика безопасности** для этой подписки.  Выберите **Выкл.** для параметра "Сбор данных".
4. В верхней ленте щелкните **Сохранить** .

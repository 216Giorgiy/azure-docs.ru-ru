# Управление рекомендациями по безопасности в Центре безопасности Azure

В этом документе рассматривается использование рекомендаций в Центре безопасности Azure, которые помогают защитить ваши ресурсы Azure.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. В документе приводится обзор службы с помощью примера развертывания. Он не является пошаговым руководством.

## Что такое Центр безопасности Azure?
Центр безопасности Azure помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Что такое рекомендации по безопасности?
Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. Когда Центр безопасности выявляет потенциальные уязвимости системы безопасности, он создает рекомендации. Рекомендации помогают настраивать необходимые элементы управления.

## Внедрение рекомендаций по безопасности

### Настройка рекомендаций

Раздел [Настройка политик безопасности в Центре безопасности Azure](security-center-policies.md) содержит сведения о:

- настройке политик безопасности;
- включении сбора данных;
- выборе просматриваемых рекомендаций в рамках политики безопасности.

Текущие рекомендации относительно политик связаны с обновлениями системы, правилами базовых планов, антивредоносными программами, [списками управления доступом для конечных точек](../virtual-machines/virtual-machines-windows-classic-setup-endpoints.md), [группами безопасности сети](../virtual-network/virtual-networks-nsg.md) для подсетей и сетевых интерфейсов, аудитом базы данных SQL, прозрачным шифрованием данных базы данных SQL и брандмауэрами веб-приложений. В разделе [Настройка политик безопасности](security-center-policies.md) приводится описание всех рекомендаций.

### Мониторинг рекомендаций
После настройки политики безопасности Центр безопасности анализирует состояние безопасности ресурсов, чтобы выявить потенциальные уязвимости. Плитка **Рекомендации** в колонке **Центр безопасности** позволяет узнать общее количество рекомендаций, выявленных Центром безопасности.

![][2]

Чтобы просмотреть подробные сведения о каждой рекомендации, выполните следующие действия.

1. Щелкните элемент **Рекомендации** в колонке **Центр безопасности**. Откроется колонка **Рекомендации**.

Рекомендации отображаются в табличном формате, где каждая строка представляет одну конкретную рекомендацию. Таблица содержит следующие столбцы.

- **Описание**. Описание рекомендаций и необходимых действий по их реализации.
- **Ресурс**. Список ресурсов, к которым применяется эта рекомендация.
- **Состояние**. Описание текущего состояния рекомендации.
    - **Открыто** — рекомендация еще не выполнена.
    - **Выполняется** — сейчас рекомендация применяется к ресурсам. От вас не требуется никаких действий.
    - **Разрешено** — рекомендация уже выполнена (в этом случае строка станет неактивной).
- **Серьезность**. Описание уровня серьезности конкретной рекомендации.
    - **Высокий** — существует требующая внимания уязвимость важных ресурсов (таких, как приложение, виртуальная машина или группа безопасности сети).
    - **Средний** — существует уязвимость, и для ее устранения либо для завершения процесса требуются некритические или дополнительные действия.
    - **Низкий** — существует уязвимость, которая должна быть устранена, но не требует немедленного внимания. (По умолчанию низкий уровень не представлен, однако вы можете выполнить фильтрацию по этому значению, чтобы просмотреть соответствующие рекомендации).

В таблице ниже приведены справочные сведения по доступным рекомендациям и действиям, выполняемым при их применении.

> [AZURE.NOTE] Узнайте больше о [классической модели развертывания и модели развертывания диспетчера ресурсов](../azure-classic-rm.md) для ресурсов Azure.

|Рекомендации|Описание|
|-----|-----|
|Включение сбора данных для подписок или виртуальных машин|Рекомендует включить в политике безопасности сбор данных для каждой вашей подписки или выбранных виртуальных машин.|
|Устранение несоответствия правил базовых шаблонов|Рекомендует обеспечить соответствие конфигураций ОС рекомендуемым базовым показателям, например запретить сохранение паролей.|
|Применение обновлений системы|Рекомендует выполнить развертывание отсутствующих обновлений системы безопасности и критических обновлений на виртуальных машинах (только для виртуальных машин Windows).|
|Настройка списков управления доступом для конечных точек|Рекомендует настроить списки контроля доступа для ограничения доступа извне к виртуальным машинам (только для классических виртуальных машин).|
|[Добавление брандмауэра веб-приложения](security-center-add-web-application-firewall.md)|Рекомендует выполнить развертывание брандмауэра веб-приложения (WAF) для конечных точек веб-службы. Вы можете защитить несколько веб-приложений в центре обеспечения безопасности, добавив эти приложения в существующие развернутые службы WAF. Устройства WAF (созданные с помощью модели развертывания Resource Manager) должны быть развернуты в отдельной виртуальной сети. Устройства WAF (созданные с помощью классической модели развертывания) ограничены заданной группой безопасности сети. В будущем эта поддержка будет расширена до полностью настраиваемого развертывания устройства WAF (в классической модели).|
|Завершение настройки брандмауэра веб-приложения|Для завершения настройки брандмауэра веб-приложения трафик должен перенаправляться в модуль WAF. В соответствии с этой рекомендацией в установку будут внесены необходимые изменения.|
|[Включение защиты от вредоносных программ](security-center-enable-antimalware.md)|Рекомендует подготовить антивредоносные программы на виртуальных машинах (только для виртуальных машин Windows).|
|Включение групп безопасности сети в подсетях и сетевых интерфейсах|Рекомендует включить группы безопасности сети (NSG) в подсетях и сетевых интерфейсах (только для виртуальных машин диспетчера ресурсов).|
|Ограничение доступа через общедоступные внешние конечные точки|Рекомендует настроить правила входящего трафика для групп безопасности сети.|
|Включение аудита SQL для сервера|Рекомендует включить аудит для серверов SQL Azure (только для службы SQL Azure, не включая экземпляры SQL, работающие на виртуальных машинах).|
|Включение аудита SQL для базы данных|Рекомендует включить аудит для баз данных SQL Azure (только для службы SQL Azure, не включая экземпляры SQL, работающие на виртуальных машинах).|
|Включение прозрачного шифрования данных в базах данных SQL|Рекомендует включить шифрование для баз данных SQL (только для службы SQL Azure).|
|Развертывание агента виртуальной машины|Позволяет узнать, на какие виртуальные машины необходимо установить агент ВМ. На виртуальные машины необходимо установить агент виртуальной машины, чтобы проверить наличие необходимых исправлений, базовые показатели и подготовить антивредоносные программы. Агент виртуальной машины устанавливается по умолчанию на виртуальных машинах, развернутых из Azure Marketplace. В статье [Агент виртуальной машины и расширения. Часть 2](http://azure.microsoft.com/blog/2014/04/15/vm-agent-and-extensions-part-2/) содержатся сведения об установке агента ВМ.|
| [Применение шифрования дисков](security-center-apply-disk-encryption.md) |Рекомендует выполнить шифрование дисков виртуальной машины с помощью шифрования дисков Azure (виртуальных машин Windows и Linux). Шифровать рекомендуется как том операционной системы, так и том данных на виртуальной машине.|

Рекомендации можно фильтровать и отклонять.

1. Щелкните элемент **Фильтр** в колонке **Рекомендации**. Откроется колонка **Фильтр**, где можно выбрать значения серьезности и состояния, которые требуется просмотреть.

    ![][3]

2. Если выяснилось, что рекомендация неприменима, вы можете отклонить ее, а затем отфильтровать из представления. Отклонить рекомендацию можно двумя способами. Первый способ — щелкнуть элемент правой кнопкой мыши и выбрать **Отклонить**. Второй способ — навести указатель мыши на элемент, нажать кнопку с тремя точками, которая появится справа, и выбрать **Отклонить**. Отклоненные рекомендации можно просмотреть, щелкнув элемент **Фильтр** и выбрав пункт **Отклоненные**.

    ![][4]

### Применение рекомендаций
Просмотрев все рекомендации, решите, какую из них применить первой. Мы рекомендуем использовать оценку уровня серьезности в качестве основного параметра для выбора приоритетных рекомендаций. Давайте для примера рассмотрим рекомендацию **Включение защиты от вредоносных программ**.

1. В колонке **Рекомендации** выберите **Включение защиты от вредоносных программ**. ![][5]

2. В колонке **Установка защиты от вредоносных программ** выберите из списка виртуальные машины, на которых не включена защита от вредоносных программ, и щелкните **Установить защиту от вредоносных программ**.
3. Откроется колонка **Новый ресурс**, в которой вы можете выбрать нужное решение защиты от вредоносных программ. Выберите **Защита от вредоносных программ (Майкрософт)**.
4. На экран будут выведены дополнительные сведения о решении защиты от вредоносных программ. Нажмите кнопку **Создать**.
5. В колонке **Добавление расширения** введите необходимые параметры конфигурации и нажмите кнопку **ОК**. ![][6]

Теперь на выбранной виртуальной машине включена [защита от вредоносных программ (Майкрософт)](../azure-security-antimalware.md).

# Управление рекомендациями по безопасности в Центре безопасности Azure

В этом документе рассматривается использование рекомендаций в Центре безопасности Azure, которые помогают защитить ваши ресурсы Azure.

> [AZURE.NOTE] В документе приводится обзор службы с помощью примера развертывания. Он не является пошаговым руководством.

## Что такое рекомендации по безопасности?
Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. Когда Центр безопасности выявляет потенциальные уязвимости системы безопасности, он создает рекомендации. Рекомендации помогают настраивать необходимые элементы управления.

## Внедрение рекомендаций по безопасности

### Настройка рекомендаций

Раздел [Настройка политик безопасности в Центре безопасности Azure](security-center-policies.md) содержит сведения о:

- настройке политик безопасности;
- включении сбора данных;
- выборе просматриваемых рекомендаций в рамках политики безопасности.

Текущие рекомендации относительно политик связаны с обновлениями системы, правилами базовых планов, антивредоносными программами, [группами безопасности сети](../virtual-network/virtual-networks-nsg.md) для подсетей и сетевых интерфейсов, аудитом базы данных SQL, прозрачным шифрованием данных базы данных SQL и брандмауэрами веб-приложений. В разделе [Настройка политик безопасности](security-center-policies.md) приводится описание всех рекомендаций.

### Мониторинг рекомендаций
После настройки политики безопасности Центр безопасности анализирует состояние безопасности ресурсов, чтобы выявить потенциальные уязвимости. Элемент **Рекомендации** в колонке **Центр безопасности** позволяет узнать общее количество рекомендаций, выявленных центром обеспечения безопасности.

![Плитка рекомендаций][1]

Чтобы просмотреть подробные сведения о каждой рекомендации, выполните следующие действия.

1. Щелкните элемент **Рекомендации** в колонке **Центр безопасности**. Откроется колонка **Рекомендации**.

Рекомендации отображаются в табличном формате, где каждая строка представляет одну конкретную рекомендацию. Таблица содержит следующие столбцы.

- **Описание** — описание рекомендаций и необходимых действий по их реализации.
- **Ресурс** — список ресурсов, к которым применяется эта рекомендация.
- **Состояние** — описание текущего состояния рекомендации.
    - **Открыто** — рекомендация еще не выполнена.
    - **Выполняется** — сейчас рекомендация применяется к ресурсам. От вас не требуется никаких действий.
    - **Разрешено** — рекомендация уже выполнена (в этом случае строка станет неактивной).
- **Серьезность** — описание уровня серьезности конкретной рекомендации.
    - **Высокая** — существует требующая внимания уязвимость важных ресурсов (таких, как приложение, виртуальная машина или группа безопасности сети).
    - **Средняя** — существует уязвимость, и для ее устранения либо для завершения процесса требуются некритические или дополнительные действия.
    - **Низкая** — существует уязвимость, которая должна быть устранена, но не требует немедленного внимания. (По умолчанию низкий уровень не представлен, однако вы можете выполнить фильтрацию по этому значению, чтобы просмотреть соответствующие рекомендации).

Рекомендации можно фильтровать и отклонять.

1. Щелкните элемент **Фильтр** в колонке **Рекомендации**. Откроется колонка **Фильтр**, где можно выбрать значения серьезности и состояния для просмотра.

    ![Фильтрация рекомендаций][2]

2. Если выяснилось, что рекомендация неприменима, вы можете отклонить ее, а затем отфильтровать из представления. Отклонить рекомендацию можно двумя способами. Первый способ — щелкнуть элемент правой кнопкой мыши и выбрать **Отклонить**. Второй способ — навести указатель мыши на элемент, нажать кнопку с тремя точками, которая появится справа, и выбрать **Отклонить**. Отклоненные рекомендации можно просмотреть, щелкнув **Фильтр** и выбрав **Отклоненные**.

    ![Отмена рекомендации][3]

### Применение рекомендаций
Просмотрев все рекомендации, решите, какую из них применить первой. Мы рекомендуем использовать оценку уровня серьезности в качестве основного параметра для выбора приоритетных рекомендаций.

В приведенной выше таблице рекомендаций выберите рекомендацию и просмотрите ее в качестве примера.

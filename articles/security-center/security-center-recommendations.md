# Управление рекомендациями по безопасности в Центре безопасности Azure

В этом документе рассматриваются рекомендации в Центре безопасности Azure, которые помогают защитить ваши ресурсы Azure.

> [AZURE.NOTE]Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. Документ содержит вводные данные о службе на основе примера развертывания. Он не является пошаговым руководством.

## Что такое Центр безопасности Azure?
Центр безопасности Azure помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Что такое рекомендации по безопасности?
Центр безопасности Azure периодически анализирует состояние безопасности ресурсов Azure. При обнаружении потенциальной уязвимости системы безопасности создаются рекомендации. Рекомендации помогают выполнить процедуру настройки необходимого средства контроля.

## Внедрение рекомендаций по безопасности

### Настройка рекомендаций

Раздел [Настройка политик безопасности в Центре безопасности Azure](security-center-policies.md) содержит сведения о:

- настройке политик безопасности;
- включении сбора данных;
- выборе просматриваемых рекомендаций в рамках политики безопасности.

Текущие рекомендации политик связаны с обновлениями системы, правилами базовых шаблонов, защитой от вредоносных программ, [списками управления доступом для конечных точек](virtual-machines-set-up-endpoints.md), [группами безопасности сети](virtual-networks-nsg.md) для подсетей и сетевых интерфейсов, аудитом Базы данных SQL, прозрачным шифрованием данных Базы данных SQL и брандмауэром веб-приложения. В разделе [Настройка политик безопасности](security-center-policies.md) приводится описание всех рекомендаций.

### Мониторинг рекомендаций

После настройки политики безопасности Центр безопасности Azure анализирует состояние безопасности ресурсов, чтобы выявить потенциальные уязвимости. Элемент **Рекомендации** колонки **Центр безопасности** позволяет узнать общее число рекомендаций, выявленных Центром безопасности Azure.

![][2]

Чтобы просмотреть подробные сведения о каждой рекомендации, выполните следующие действия.

1. Щелкните элемент **Рекомендации** в колонке **Центр безопасности**. Откроется колонка **Рекомендации**.
2. Вы можете отфильтровать рекомендации в колонке по состоянию и уровню серьезности. Щелкните элемент **Фильтр** в колонке **Рекомендации**. Откроется колонка «Фильтр», где вы можете выбрать значения серьезности и состояния, которые требуется просмотреть. ![][3]

3. Если выяснилось, что рекомендация неприменима, вы можете отклонить ее, а затем отфильтровать из представления. Отклонить рекомендацию можно двумя способами. Щелкните элемент правой кнопкой мыши и выберите пункт **Отклонить** или наведите указатель мыши на элемент, нажмите кнопку с тремя точками, которая появится справа, и выберите **Отклонить**. Отклоненные рекомендации вы можете просмотреть, щелкнув элемент **Фильтр** и выбрав пункт **Отклоненные**. ![][4]

Рекомендации отображаются в табличном формате, где каждая строка представляет одну конкретную рекомендацию. Таблица содержит следующие столбцы.

- **ОПИСАНИЕ** — описание рекомендаций и необходимых действий по реализации.
- **РЕСУРСЫ** — к каким ресурсам применяется данная рекомендация.
- **СОСТОЯНИЕ** — текущее состояние рекомендации:
  - **Открыто** — рекомендация еще не применена;
  - **Выполняется** — сейчас рекомендация применяется к ресурсам, от вас не требуется никаких действий;
  - **Разрешено** — рекомендация уже выполнена (в этом состоянии строка неактивна).
- **СЕРЬЕЗНОСТЬ** — описание уровня серьезности этой конкретной рекомендации:
  - **Высокий** — уязвимость существует для важных ресурсов (приложение, виртуальная машина, группы безопасности сети) и требует внимания;
  - **Средний** — для завершения процесса или устранения уязвимости требуются некритические или дополнительные действия;
  - **Низкий** — уязвимость должна быть разрешена, но не требует немедленного внимания. (По умолчанию низкий уровень не представлен, однако вы можете выполнить фильтрацию по этому значению, чтобы просмотреть соответствующие рекомендации.)

В таблице ниже приведены справочные сведения по доступным рекомендациям и действиям, выполняемым при их применении.

| Рекомендации | Описание |
|----- |-----|
| Включение сбора данных для подписок или виртуальных машин | Рекомендует включить в политике безопасности сбор данных для выбранных виртуальных машин. |
| Устранение несоответствия правил базовых шаблонов | Рекомендует обеспечить соответствие конфигураций ОС рекомендуемым базовым показателям, например запретить сохранение паролей. |
| Применение обновлений системы | Рекомендует выполнить развертывание отсутствующих обновлений системы на виртуальных машинах (только для виртуальных машин Windows). |
| Настройка списков управления доступом для конечных точек | Рекомендует настроить список контроля доступа для ограничения входящего доступа к виртуальным машинам (только для классических виртуальных машин). |
| Добавление брандмауэра веб-приложения | Рекомендует выполнить развертывание брандмауэра веб-приложения (WAF) для конечных точек веб-службы (только для виртуальных машин диспетчера ресурсов). |
| Завершение настройки брандмауэра веб-приложения | Для завершения настройки брандмауэра веб-приложения трафик должен перенаправляться в модуль WAF. Эта рекомендация внесет необходимые изменения в установку. |
| Включение защиты от вредоносных программ | Рекомендует подготовить защиту от вредоносных программ на виртуальных машинах (только для виртуальных машин Windows). |
| Включение групп безопасности сети в подсетях и сетевых интерфейсах | Рекомендует включить группы безопасности сети (NSG) в подсетях и сетевых интерфейсах (только для виртуальных машин диспетчера ресурсов). |
| Ограничение доступа через общедоступные внешние конечные точки | Рекомендует настроить правила входящего трафика для групп безопасности сети. |
| Включение аудита SQL для сервера | Рекомендует включить аудит для серверов SQL Azure (только для службы SQL Azure, не включая экземпляры SQL, работающие на виртуальных машинах). |
| Включение аудита SQL для базы данных | Рекомендует включить аудит для баз данных SQL Azure (только для службы SQL Azure, не включая экземпляры SQL, работающие на виртуальных машинах). |
| Включение прозрачного шифрования данных в базах данных SQL | Рекомендует включить шифрование для баз данных SQL (только для службы SQL Azure). |
| Развертывание агента виртуальной машины | Позволяет узнать, на какие виртуальные машины необходимо установить агент ВМ. Агент виртуальной машины необходимо установить на виртуальные машины для осуществления подготовки таких функций, как поиск исправлений, проверка базовых показателей и защита от вредоносных программ. Агент виртуальной машины устанавливается по умолчанию для виртуальных машин, развернутых из Azure Marketplace. В статье [VM Agent and Extensions – Part 2](http://azure.microsoft.com/blog/2014/04/15/vm-agent-and-extensions-part-2/) содержатся сведения об установке агента виртуальной машины. |

### Применение рекомендаций
Просмотрев все рекомендации, вы можете решить, какую из них применить первой. Рекомендуется использовать оценку уровня серьезности в качестве основного параметра для выбора приоритетных рекомендаций. Мы рассмотрим применение рекомендаций на примере рекомендации "Защита от вредоносных программ".

1. В колонке **Рекомендации** выберите **Включение защиты от вредоносных программ**. ![][5]

2. В колонке **Установка защиты от вредоносных программ** выберите из списка виртуальные машины, на которых не включена защита от вредоносных программ, и щелкните **Установить защиту от вредоносных программ**.
3. Откроется колонка **Новый ресурс**, в которой вы можете выбрать нужное решение защиты от вредоносных программ. Выберите **Защита от вредоносных программ (Майкрософт)**.
4. На экран будут выведены дополнительные сведения о решении защиты от вредоносных программ. Нажмите кнопку **Создать**.
5. Введите необходимые параметры конфигурации в колонке **Добавление расширения** и нажмите кнопку **ОК**. ![][6]

Теперь на выбранной виртуальной машине включена [защита от вредоносных программ (Майкрософт)](azure-security-antimalware.md).

### Развертывание рекомендуемых решений партнеров

Рекомендация может подразумевать развертывание интегрированного решения безопасности от партнера корпорации Майкрософт. Рассмотрим пример применения такой рекомендации.

1. Вернитесь в колонку **Рекомендации**.
2.	Выберите рекомендацию **Защита веб-приложения с помощью брандмауэра веб-приложения**. Откроется колонка **Незащищенные веб-приложения**. ![][7]
3. Выберите веб-приложение. Откроется колонка **Добавление брандмауэра веб-приложения**.
4. Выберите пункт **Barracuda Web Application Firewall**. Откроется колонка, в которой содержатся сведения о брандмауэре **Barracuda Web Application Firewall**.
5. В колонке сведений щелкните **Создать**. Откроется колонка **Новый брандмауэр веб-приложения**, где вы можете выполнить действия по **настройке виртуальной машины** и указать **сведения о WAF**.
6. Выберите **Конфигурация виртуальной машины**. В колонке **Конфигурация виртуальной машины** введите сведения, необходимые для настройки виртуальной машины, на которой будет выполняться WAF. ![][8]
7. Вернитесь в колонку **Новый брандмауэр веб-приложения** и выберите **Сведения о WAF**. В колонке **Сведения о WAF** настраивается сам брандмауэр веб-приложения. Шаг 6 позволяет настроить виртуальную машину, на которой будет выполняться WAF, а шаг 7 — подготовить сам брандмауэр.

8. Вернитесь в колонку **Рекомендации**. После создания WAF появится новая запись, **Завершение настройки брандмауэра веб-приложения**. Она указывает, что необходимо завершить процедуру, фактически подключив брандмауэр WAF к виртуальной сети Azure, чтобы он мог защищать приложение. ![][9]

9. Выберите **Завершение настройки брандмауэра веб-приложения**. Откроется новая колонка. Вы увидите, что появилось новое веб-приложение, для которого необходимо перенаправить трафик.
10. Выберите веб-приложение. Откроется колонка, в которой приведены пошаговые инструкции для завершения настройки брандмауэра веб-приложения. Выполните эти действия и щелкните **Ограничить трафик**. Центр безопасности Azure выполнит необходимые подключения. ![][10]

Журналы WAF теперь полностью интегрированы. Центр безопасности Azure может начать автоматический сбор и анализ журналов, чтобы создавать важные оповещения безопасности.

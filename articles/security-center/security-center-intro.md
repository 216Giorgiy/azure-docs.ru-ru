# Введение в Центр безопасности Azure

Узнайте о Центре безопасности Azure, его основных возможностях и о том, как он работает.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. В документе приводится обзор службы с помощью примера развертывания.

## Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Она включает встроенные функции мониторинга безопасности и управления политиками для подписок Azure, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

##	Ключевые возможности
 Центр безопасности предоставляет простые в использовании и эффективные возможности предотвращения, обнаружения и устранения угроз, встроенные в Azure. Ключевые возможности:

| | |
|----- |-----|
| Предотвращение | Проверяет состояние безопасности ресурсов Azure. |
| | Определяет политики для подписок Azure в соответствии с требованиями вашей компании к обеспечению безопасности, а также с учетом типа приложений, которые вы используете, или конфиденциальности ваших данных. |
| | Использует рекомендации по обеспечению безопасности на основе политик, помогая владельцам служб выполнять процедуру реализации необходимых элементов управления. |
| | Быстро развертывает службы и приложения безопасности от корпорации Майкрософт и партнеров. |
| Обнаружение |Автоматически собирает и анализирует данные о безопасности из ресурсов Azure, сети и решений партнеров, таких как антивредоносные программы и брандмауэры. |
| | Использует глобальную аналитику угроз из продуктов и служб Майкрософт, Центра кибербезопасности и подразделения цифровых преступлений, а также из внешних каналов. |
| | Применяет расширенную аналитику, включая машинное обучение и анализ поведения. |
| Устранение | Направляет приоритетные инциденты или оповещения системы безопасности. |
| | Сообщает информацию об источнике атаки и пострадавших ресурсах. |
| | Предлагает способ пресечения текущей атаки и помогает предотвратить дальнейшие нападения. |

## Вводное пошаговое руководство
 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). [Войдите на портал](https://portal.azure.com), выберите **Обзор** и найдите параметр **Центр безопасности** либо выберите плитку **Центр безопасности**, закрепленную ранее на панели мониторинга портала.

![Плитка "Безопасность" на портале Azure][1]

В Центре безопасности вы можете настраивать политики безопасности, отслеживать конфигурации безопасности и просматривать оповещения безопасности.

### Политики безопасности

Вы можете определять политики для подписок Azure согласно требованиям безопасности вашей организации. Политики также можно настраивать в соответствии с типами используемых приложений или с уровнем конфиденциальности данных в каждой подписке. Например, требования к безопасности ресурсов, используемых для разработки или тестирования, могут отличаться от требований к безопасности ресурсов, которые используются для рабочих приложений. Аналогично: приложения с контролируемыми данными, такими как персональные данные, могут требовать более высокого уровня безопасности.

> [AZURE.NOTE] Чтобы изменить политику безопасности, необходимо быть владельцем или участником подписки.

Щелкните плитку **Политика безопасности**, чтобы открыть список подписок, и выберите подписку, данные которой хотите просмотреть.

![Плитка "Политика безопасности"][2]

Функция **Сбор данных** (см. выше) включает сбор данных для политики безопасности. Включение этой функции обеспечивает: — ежедневное сканирование всех поддерживаемых виртуальных машин для контроля безопасности и получения рекомендаций; — сбор событий безопасности для анализа и обнаружения угроз.

Функция **Показать рекомендации для:** (см. выше) позволяет выбрать элементы управления безопасностью для отслеживания и получения рекомендаций с учетом потребности в защите входящих в подписку ресурсов.

### Рекомендации по обеспечению безопасности

 Центр безопасности анализирует состояние безопасности ресурсов Azure, чтобы выявить потенциальные уязвимости. Список рекомендаций помогает выполнить процедуру настройки необходимого средства контроля. Примеры приведены ниже.

- Подготовка антивредоносных программ для обнаружения и устранения вредоносных программ.
- Настройка групп безопасности сети и правил управления трафиком для виртуальных машин.
- Подготовка брандмауэров веб-приложений для защиты от атак на веб-приложения.
- Развертывание отсутствующих обновлений системы
- Обращение к конфигурациям операционной системы, не соответствующих рекомендуемым базовым показателям

Щелкните элемент **Рекомендации**, чтобы открыть список рекомендаций. Для просмотра дополнительной информации или устранения соответствующих проблем щелкните нужную рекомендацию.

![Рекомендации по безопасности в Центре безопасности Azure][3]

### Работоспособность ресурса

Плитка **Работоспособность ресурсов** показывает общий уровень защищенности вашей среды с детализацией по типам ресурсов, к которым относятся виртуальные машины, веб-приложения и другие ресурсы.

Выберите тип ресурсов на плитке **Работоспособность ресурсов**, чтобы просмотреть дополнительные сведения, в том числе список всех обнаруженных потенциальных уязвимостей системы безопасности. (В примере ниже выбран тип **Виртуальные машины**.)

![Плитка "Работоспособность ресурсов"][4]

### Оповещения безопасности

 Центр безопасности автоматически собирает, анализирует и объединяет данные журналов из ресурсов Azure, сети и партнерских решений, таких как антивредоносные программы и брандмауэры. При обнаружении угроз создается оповещение системы безопасности. Например, определяется следующее.

- Обращение взломанных виртуальных машин к опасному IP-адресу
- обнаружение сложных вредоносных программ с помощью отчетов об ошибках Windows;
- Грубые атаки на виртуальные машины
- оповещения безопасности от встроенных брандмауэров и антивредоносных программ.

При выборе элемента **Оповещения безопасности** отображается список приоритетных оповещений.

![Оповещения безопасности][5]

При выборе оповещения отображаются дополнительные сведения об атаке и рекомендации по ее устранению.

![Сведения об оповещениях системы безопасности][6]

## Приступая к работе
Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Центр безопасности включается с помощью подписки Azure. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/).

 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения см. в [документации по порталу](https://azure.microsoft.com/documentation/services/azure-portal/).

Статья [Начало работы с Центром безопасности Azure](security-center-get-started.md) поможет вам быстро ознакомиться с компонентами мониторинга безопасности и управления политиками в Центре безопасности.

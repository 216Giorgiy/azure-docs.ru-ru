# <a name="introduction-to-azure-security-center"></a>Введение в Центр безопасности Azure
Узнайте о Центре безопасности Azure, его основных возможностях и о том, как он работает.

> [!NOTE]
> В документе приводится обзор службы с помощью примера развертывания.
>
>

## <a name="what-is-azure-security-center"></a>Что такое Центр безопасности Azure?
 Центр безопасности помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Она включает встроенные функции мониторинга безопасности и управления политиками для подписок Azure, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## <a name="key-capabilities"></a>Ключевые возможности
 Центр безопасности предоставляет простые в использовании и эффективные возможности предотвращения, обнаружения и устранения угроз, встроенные в Azure. Ключевые возможности:

| Этап | Функция |
| --- | --- |
| Предотвращение |Проверяет состояние безопасности ресурсов Azure. |
| Предотвращение | Определяет политики для подписок Azure и групп ресурсов в соответствии с требованиями вашей компании к обеспечению безопасности, а также с учетом типа приложений, которые вы используете, или конфиденциальности ваших данных. |
| Предотвращение | Использует рекомендации по обеспечению безопасности на основе политик, помогая владельцам служб выполнять процедуру реализации необходимых элементов управления. |
| Предотвращение | Быстро развертывает службы и приложения безопасности от корпорации Майкрософт и партнеров. |
| Обнаружение |Автоматически собирает и анализирует данные о безопасности из ресурсов Azure, сети и решений партнеров, таких как антивредоносные программы и брандмауэры. |
| Обнаружение | Использует глобальную аналитику угроз из продуктов и служб Майкрософт, Подразделения цифровых преступлений Майкрософт (DCU), Центра кибербезопасности Майкрософт (MSRC), а также из внешних каналов. |
| Обнаружение | Применяет расширенную аналитику, включая машинное обучение и анализ поведения. |
| Устранение |Направляет приоритетные инциденты или оповещения системы безопасности. |
| Устранение | Сообщает информацию об источнике атаки и пострадавших ресурсах. |
| Устранение | Предлагает способ пресечения текущей атаки и помогает предотвратить дальнейшие нападения. |

## <a name="introductory-walkthrough"></a>Вводное пошаговое руководство
 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). [Войдите на портал](https://portal.azure.com), выберите **Обзор** и найдите параметр **Центр безопасности** либо выберите плитку **Центр безопасности**, закрепленную ранее на панели мониторинга портала.

![Плитка "Безопасность" на портале Azure][1]

В Центре безопасности вы можете настраивать политики безопасности, отслеживать конфигурации безопасности и просматривать оповещения безопасности.

### <a name="security-policies"></a>Политики безопасности
Вы можете определять политики для подписок Azure и групп ресурсов согласно требованиям безопасности вашей организации. Политики также можно настраивать в соответствии с типами используемых приложений или с уровнем конфиденциальности данных в каждой подписке. Например, требования к безопасности ресурсов, используемых для разработки или тестирования, могут отличаться от требований к безопасности ресурсов, которые используются для рабочих приложений. Аналогично: приложения с контролируемыми данными, такими как персональные данные, могут требовать более высокого уровня безопасности.

> [!NOTE]
> Чтобы изменить политику безопасности на уровне подписки или группы ресурсов, вы должны быть владельцем или участником подписки.
>
>

В колонке **Центр безопасности** выберите плитку **Политика**, чтобы получить список подписок и групп ресурсов.   

![Колонка "Центр безопасности"][2]

В колонке **Политика безопасности** выберите подписку, чтобы просмотреть сведения о политике.

![Подписка в колонке "Политика безопасности"][3]

Параметр **Сбор данных** (см. выше) включает сбор данных для политики безопасности. Включение этой функции предоставляет:

* Ежедневное сканирование всех поддерживаемых виртуальных машин для отслеживания состояния безопасности и рекомендации.
* Сбор событий безопасности для обнаружения и анализа угроз.

Параметр **Выберите учетные записи для регионов** (см. выше) позволяет выбрать учетную запись хранения, в которой будут храниться данные, собираемые с виртуальных машин, для каждого региона, в котором у вас есть виртуальные машины. Если это не сделать, такая учетная запись хранения будет создана автоматически. В целях безопасности собранные данные логически изолируются от других данных клиентов.

> [!NOTE]
> Настройка сбора данных и выбор учетной записи хранения для региона осуществляются на уровне подписки.
>
>

Выберите **Политика предотвращения** (см. выше), чтобы открыть колонку **Политика предотвращения**. Параметр **Показать рекомендации для** позволяет выбрать элементы управления безопасностью для отслеживания и получения рекомендаций с учетом потребности в защите входящих в подписку ресурсов.

Затем выберите группу ресурсов, чтобы просмотреть сведения о политике.

![Группа ресурсов в колонке "Политика безопасности"][4]

Параметр **Наследование** (см. выше) позволяет определять группы ресурсов как:

* Наследуемые (по умолчанию). Это означает, что все политики безопасности для данной группы ресурсов наследуются от уровня подписки.
* Уникальные. Это означает, что у группы ресурсов будет пользовательская политика безопасности. Вам необходимо внести изменения в разделе **Показать рекомендации для**.

> [!NOTE]
> В случае конфликта между политикой уровня подписки и политикой уровня группы ресурсов приоритет получает политика уровня группы ресурсов.
>
>

### <a name="security-recommendations"></a>Рекомендации по обеспечению безопасности
 Центр безопасности анализирует состояние безопасности ресурсов Azure, чтобы выявить потенциальные уязвимости. Список рекомендаций помогает выполнить процедуру настройки необходимого средства контроля. Примеры приведены ниже.

* Подготовка антивредоносных программ для обнаружения и устранения вредоносных программ.
* Настройка групп безопасности сети и правил управления трафиком для виртуальных машин
* Подготовка брандмауэров веб-приложений для защиты от атак на веб-приложения.
* Развертывание отсутствующих обновлений системы
* Обращение к конфигурациям операционной системы, не соответствующих рекомендуемым базовым показателям

Щелкните элемент **Рекомендации** , чтобы открыть список рекомендаций. Чтобы просмотреть дополнительные сведения или устранить соответствующие проблемы, щелкните нужную рекомендацию.

![Рекомендации по безопасности в Центре безопасности Azure][5]

### <a name="resource-health"></a>Работоспособность ресурса
Плитка **Работоспособность безопасности ресурсов** показывает общий уровень защищенности вашей среды с детализацией по типам ресурсов, к которым относятся виртуальные машины, веб-приложения и другие ресурсы.   

Выберите тип ресурсов на элементе **Работоспособность безопасности ресурсов** , чтобы просмотреть дополнительные сведения, в том числе список всех обнаруженных потенциальных уязвимостей системы безопасности. (В примере ниже выбран тип**Виртуальные машины** .)

![Плитка "Работоспособность ресурсов"][6]

### <a name="security-alerts"></a>Оповещения безопасности
 Центр безопасности автоматически собирает, анализирует и объединяет данные журналов из ресурсов Azure, сети и партнерских решений, таких как антивредоносные программы и брандмауэры. При обнаружении угроз создается оповещение системы безопасности. Например, определяется следующее.

* обращение взломанных виртуальных машин к опасному IP-адресу;
* обнаружение сложных вредоносных программ с помощью отчетов об ошибках Windows;
* атаки на виртуальные машины методом подбора;
* оповещения безопасности от встроенных брандмауэров и антивредоносных программ.

При выборе элемента **Оповещения безопасности** отображается список приоритетных оповещений.

![Оповещения безопасности][7]

При выборе оповещения отображаются дополнительные сведения об атаке и рекомендации по ее устранению.

![Сведения об оповещениях системы безопасности][8]

### <a name="partner-solutions"></a>Решения партнеров
С помощью элемента **Решения партнеров** можно отслеживать состояние работоспособности решений партнеров, интегрированных в вашу подписку Azure. Центр безопасности отображает предупреждения, поступающие из решения.

Выберите элемент **Решения партнеров** . Откроется колонка со списком всех подключенных решений партнеров.

![Решения партнеров][9]

## <a name="get-started"></a>Приступая к работе
Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Центр безопасности включается с помощью подписки Azure. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/).

 Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения см. в [документации по порталу](https://azure.microsoft.com/documentation/services/azure-portal/).

[Краткое руководство по центру безопасности Azure](security-center-get-started.md) поможет вам быстро ознакомиться с компонентами мониторинга безопасности и управления политиками в центре безопасности.

# Введение в Центр безопасности Azure

Узнайте о Центре безопасности Azure, его основных возможностях и о том, как он работает.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. Документ содержит вводные данные о службе на основе примера развертывания.

## Что такое Центр безопасности Azure?
Центр безопасности Azure помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Она включает встроенные функции мониторинга безопасности и управления политиками для подписок Azure, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

##	Ключевые возможности
Центр безопасности Azure обеспечивает простое и эффективное предотвращение, обнаружение и устранение угроз. Ключевые возможности:

| | |
|----- |-----|
| Предотвращение | Проверяет состояние безопасности ресурсов Azure. |
| | Определяет политики для подписок Azure в соответствии с требованиями вашей компании к обеспечению безопасности, а также типом приложений или конфиденциальностью ваших данных. |
| | Использует рекомендации по обеспечению безопасности на основе политик, помогая владельцам служб выполнять процедуру реализации необходимых элементов управления. |
| | Быстро развертывает службы и приложения безопасности от корпорации Майкрософт и партнеров. |
| Обнаружение | Автоматически собирает и анализирует данные о безопасности из ресурсов Azure, сети и решений партнеров, таких как программы защиты от вредоносных программ и брандмауэры. |
| | Использует глобальную аналитику угроз из продуктов и служб Майкрософт, Центра кибербезопасности и подразделения цифровых преступлений, а также из внешних каналов. |
| | Применяет расширенную аналитику, включая машинное обучение и анализ поведения. |
| Устранение | Направляет приоритетные инциденты или оповещения системы безопасности. |
| | Сообщает информацию об источнике атаки и пострадавших ресурсах. |
| | Предлагает способ пресечения текущей атаки и помогает предотвратить дальнейшие нападения. |

## Вводное пошаговое руководство
Доступ к Центру безопасности Azure осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Чтобы получить доступ, [войдите в портал](https://portal.azure.com), выберите **Обзор** и найдите параметр **Центр безопасности** либо выберите элемент **Центр безопасности**, закрепленный ранее на панели мониторинга портала.

![][1]

В Центре безопасности Azure вы можете настраивать политики безопасности, отслеживать конфигурации безопасности и просматривать оповещения безопасности.

### Политики безопасности

Политики для подписок Azure вы можете настраивать в соответствии с требованиями безопасности вашей компании, типом приложений и конфиденциальностью данных в каждой подписке. Например, требования к безопасности ресурсов, используемых для разработки или тестирования, могут отличаться от тех, которые используются для рабочих приложений. Аналогично: приложения с контролируемыми данными, такими как персональные данные, могут требовать более высокого уровня безопасности.

> [AZURE.NOTE] Для изменения политики безопасности требуются права владельца или участника подписки.

Щелкните элемент **Политика безопасности**, чтобы открыть список подписок, и выберите подписку, данные которой хотите посмотреть.

![][2]

Функция **Сбор данных** (см. выше) включает сбор данных для политики безопасности. Включение этой функции обеспечивает: — ежедневное сканирование всех поддерживаемых виртуальных машин для контроля безопасности и получения рекомендаций; — сбор событий безопасности для анализа и обнаружения угроз.

Функция **Показать рекомендации для:** (см. выше) позволяет выбрать элементы управления безопасностью для отслеживания и получения рекомендаций с учетом потребности в защите входящих в подписку ресурсов.

### Рекомендации по обеспечению безопасности

Центр безопасности Azure анализирует состояние безопасности ресурсов Azure, чтобы выявить потенциальные уязвимости. Список рекомендаций помогает выполнить процедуру настройки необходимого средства контроля. Примеры приведены ниже.

- Подготовка защиты от вредоносных программ помогает обнаруживать и устранять вредоносные программы.
- Настройка групп безопасности сети и правил управления трафиком для виртуальных машин.
- Подготовка брандмауэра веб-приложения для защиты от атак на веб-приложения
- Развертывание отсутствующих обновлений системы
- Обращение к конфигурациям операционной системы, не соответствующих рекомендуемым базовым показателям

Щелкните элемент **Рекомендации**, чтобы открыть список рекомендаций. Для просмотра дополнительной информации или устранения соответствующих проблем щелкните нужную рекомендацию.

![][3]

### Работоспособность ресурсов

Элемент **Работоспособность ресурсов** показывает общий уровень защищенности вашей среды с детализацией по типам ресурсов — виртуальным машинам, веб-приложениям и другим ресурсам.

Выберите тип ресурсов в элементе **Работоспособность ресурсов**, чтобы просмотреть дополнительные сведения (в приведенном ниже примере это виртуальные машины), включая список всех обнаруженных потенциальных угроз безопасности.

![][4]

### Оповещения безопасности

Центр безопасности Azure автоматически собирает, анализирует и объединяет данные журнала из ресурсов Azure, сети и решений партнеров, таких как программы защиты от вредоносных программ и брандмауэры. При обнаружении угроз создается оповещение системы безопасности. Например, определяется следующее.

- Обращение взломанных виртуальных машин к опасному IP-адресу
- Обнаружение сложных вредоносных программ с использованием отчетов об ошибках Windows
- Грубые атаки на виртуальные машины
- Оповещения безопасности от встроенных брандмауэров и программ для защиты от вредоносного ПО.

При выборе элемента **Оповещения безопасности** отображается список приоритетных оповещений.

![][5]

При выборе оповещения отображаются дополнительные сведения об атаке и предложения по ее устранению.

![][6]

## Приступая к работе
Для начала работы с Центром безопасности Azure необходима подписка на Microsoft Azure. Центр безопасности Azure включается в подписке Azure. Если у вас нет подписки, вы можете зарегистрироваться для получения [бесплатной пробной версии](https://azure.microsoft.com/pricing/free-trial/).

Доступ к Центру безопасности Azure осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). Дополнительные сведения см. в [документации по порталу](https://azure.microsoft.com/documentation/services/azure-portal/).

Статья [Приступая к работе с Центром безопасности Azure](security-center-get-started.md) поможет вам быстро ознакомиться с компонентами мониторинга безопасности и управления политиками в Центре безопасности Azure.

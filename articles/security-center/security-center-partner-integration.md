# <a name="partner-and-solutions-integration-in-azure-security-center"></a>Интеграция партнеров и решений в центре безопасности Azure

В этой статье описывается, как можно интегрировать центр безопасности Azure с партнерами для повышения общей безопасности. Центр безопасности предоставляет расширенные возможности интеграции Azure, используя преимущества Azure Marketplace для сертификации и выставления счетов партнеров.

## <a name="why-deploy-partner-solutions-from-security-center"></a>Зачем нужно развертывать партнерские решения через центр безопасности

Ниже приведены четыре основные причины для использования интеграции партнеров в центре безопасности.

- **Простота развертывания.** Рекомендации центра безопасности существенно упрощают развертывание партнерских решений. Процесс развертывания можно полностью автоматизировать с помощью настроек по умолчанию и топологии сети. Пользователи также могут использовать вариант частичной автоматизации, который дает дополнительную гибкость и предоставляет расширенные настройки.
- **Интегрированные обнаружения.** События безопасности автоматически собираются из партнерских решений, объединяются и отображаются в списках оповещений и инцидентов центра безопасности. Эти события используются совместно с результатами сканирования других источников, что дает дополнительные возможности для обнаружения угроз.
- **Единый мониторинг работоспособности и единое управление.** Пользователи могут использовать интегрированные события работоспособности, чтобы отслеживать все партнерские решения в одном расположении. Предоставляются базовые возможности для управления и быстрый доступ к партнерским решениям для расширенной настройки.
- **Экспорт в SIEM.** Пользователи могут экспортировать все оповещения центра безопасности и партнеров в CEF-формат, передавая их в локальные системы управления сведениями о безопасности и событиями (SIEM) с использованием интеграции журналов Azure (предварительная версия).


## <a name="partners-that-integrate-with-security-center"></a>Партнеры, которые интегрируются с центром безопасности

Сейчас реализована интеграция платформенной функциональности с центром безопасности для решений партнеров, которые доступны в Azure Marketplace:

- Endpoint Protection ([Trend Micro](https://help.deepsecurity.trendmicro.com/azure-marketplace-getting-started-with-deep-security.html), Symantec и [антивредоносное ПО Майкрософт для облачных служб и службы "Виртуальные машины" Azure](https://docs.microsoft.com/azure/security/azure-security-antimalware)). 
- Брандмауэр веб-приложения ([Barracuda](https://www.barracuda.com/products/webapplicationfirewall), [F5](https://support.f5.com/kb/en-us/products/big-ip_asm/manuals/product/bigip-ve-web-application-firewall-microsoft-azure-12-0-0.html), [Imperva](https://www.imperva.com/Products/WebApplicationFirewall-WAF), [Fortinet](https://www.fortinet.com/resources.html?limit=10&search=&document-type=data-sheets) и [шлюз приложений Azure](https://azure.microsoft.com/blog/azure-web-application-firewall-waf-generally-available/)). 
- Брандмауэр следующего поколения ([Check Point](https://www.checkpoint.com/products/vsec-microsoft-azure/), [Barracuda](https://campus.barracuda.com/product/nextgenfirewallf/article/NGF/AzureDeployment/), [Fortinet](http://docs.fortinet.com/d/fortigate-fortios-handbook-the-complete-guide-to-fortios-5.2) и [Cisco](http://www.cisco.com/c/en/us/td/docs/security/firepower/quick_start/azure/ftdv-azure-qsg.html)). 
- Оценка уязвимостей ([Qualys](https://www.qualys.com/public-clouds/microsoft-azure/)).  

Постепенно для центра безопасности будет увеличено число партнеров по этим категориям, а также появятся новые категории.

## <a name="manage-partner-solutions"></a>Управление партнерскими решениями

После развертывания выберите параметр **Решения партнеров** на панели мониторинга **Центр безопасности**, чтобы просмотреть сведения о работоспособности решения и выполнить основные задачи управления.

![Интеграция решений партнеров](./media/security-center-partner-integration/security-center-partner-integration-fig8.png)

Содержимое, которое отображается при открытии решений по обеспечению безопасности, зависит от инфраструктуры. Как видно на предыдущем изображении, эта страница содержит три раздела:

- **Подключенные решения.** Здесь отображаются решения, подключенные к центру безопасности.
- **Обнаруженные решения.** Здесь отображаются решения, не подключенные к центру безопасности. Эти решения можно подключить, и они будут отображаться в списке подключенных решений.  Если центр безопасности не обнаруживает неподключенные решения, этот раздел будет скрыт.
- **Добавление источников данных.** Здесь отображаются источники данных Azure или сторонние источники, которые можно добавить в центр безопасности.

### <a name="connected-solutions"></a>Подключенные решения

В разделе **Подключенные решения** отображаются все решения по обеспечению безопасности, которые сейчас подключены к центру безопасности. 

![Подключенные решения](./media/security-center-partner-integration/security-center-partner-integration-fig4.png)

Сведения, которое будет отображаться для каждого из них, зависят от решения. На плитках могут отображаться:

- Значок компании для партнера.  Если в центре безопасности нет значка компании, отображаются один или несколько первых символов имени партнера.
- Тип решения.
- Может отображаться имя компьютера.
- Состояние работоспособности.  Если индикатор работоспособности не отправлен, в центре безопасности отображаются дата и время последнего получения сообщения о событии. Это позволяет определить, передает ли устройство отчеты. Если центр безопасности не получает индикатор работоспособности из определенного решения, плитка решения не отображается в этом разделе.

> [!NOTE]
> Проверяйте, отображаются ли в центре безопасности дата и время последнего получения события, чтобы определить, передает ли устройство отчеты. Решения, которые не отправляют индикатор работоспособности, отображаются как подключенные, если в течение последних 14 дней отправлялись оповещения или сообщения о событии.
>  

Некоторые из этих решений могут полностью интегрироваться в Azure, другие же могут быть локальными. Так как центр безопасности поддерживает [общий формат событий (CEF)](https://docs.microsoft.com/azure/operations-management-suite/oms-security-connect-products#what-is-cef), он может устанавливать подключение с помощью решений, использующих CEF, например, брандмауэра с поддержкой CEF. Добавив это решение в центр безопасности, брандмауэр отправляет журналы в формате CEF в центр безопасности, который направляет их для обработки в [Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview). Брандмауэр не является ресурсом Azure, но отправляет сообщения о событиях вместо индикатора работоспособности.  Единственные сведения о работоспособности, которые передаются в центр безопасности, — это время последней отправки сообщения о событии для этого устройства.  Для всех сторонних ресурсов (не принадлежащих Azure) в области работоспособности на плитке центра безопасности отображаются дата и время получения последнего сообщения о событии. Это указывает на то, что сторонний ресурс по-прежнему передает отчеты.

### <a name="discovered-solutions"></a>Обнаруженные решения

В разделе **Обнаруженные решения** отображаются все решения, которые были добавлены через Azure, и к которым в центре безопасности предлагается установить подключение.

![Обнаруженные решения](./media/security-center-partner-integration/security-center-partner-integration-fig5.png)

Центр безопасности можно интегрировать со встроенными решениями Azure, такими как [Защита идентификации Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-identityprotection). Если у вас есть лицензия для службы "Защита идентификации Azure AD", которая не подключена к центру безопасности, она будет внесена в список в разделе **Обнаруженные решения**. Чтобы интегрировать это решение с центром безопасности, нажмите кнопку **Подключить** на плитке **Защита идентификации Azure AD**. Откроется следующая страница:

![защиту идентификации Azure AD](./media/security-center-partner-integration/security-center-partner-integration-fig6.png)

Чтобы завершить подключение службы "Защита идентификации Azure AD", выберите рабочую область, в которой сохранены данные. Все данные службы "Защита идентификации Azure AD" будут передаваться из региона рабочей области, выбранного на этом шаге.  В выбранную с помощью селектора рабочую область начнут передаваться данные.

Для подключения к центру безопасности требуются права глобального администратора или администратора безопасности.  Если у вас нет таких разрешений, кнопка **Подключить** будет недоступна. Отобразится сообщение, объясняющее, почему она отключена.

Оповещения службы "Защита идентификации Azure AD" проходят через канал обнаружения центра безопасности. Это позволяет получать оповещения от центра безопасности и службы "Защита идентификации Azure AD". Центр безопасности объединяет все оповещения, которые определяет как релевантные для создания [инцидента безопасности](https://docs.microsoft.com/azure/security-center/security-center-incident). Описание инцидента безопасности предоставляет дополнительные сведения о подозрительных действиях.

### <a name="add-data-sources"></a>Добавление источников данных

Для интеграции с центром безопасности вы можете добавить как компьютеры Azure, так и сторонние компьютеры.  Это означает, что можно добавить локальный компьютер или устройство, которое поддерживает CEF.

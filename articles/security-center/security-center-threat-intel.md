# <a name="threat-intelligence-in-azure-security-center"></a>Анализ угроз в центре безопасности Azure
Этот документ поможет вам использовать анализ угроз в центре безопасности Azure для решения проблем с безопасностью.

## <a name="what-is-threat-intelligence"></a>Что такое анализ угроз?
С помощью функции "Анализ угроз", доступной в центре безопасности, ИТ-администраторы могут выявлять угрозы безопасности в среде. Например, определять, что конкретный компьютер является частью ботнета. Компьютеры могут стать узлами в бот-сети, если злоумышленники незаконно установят вредоносную программу, которые тайно подключает этот компьютер к среде управления и контроля. С его помощью также можно выявить потенциальные угрозы, поступающие по нелегальным каналам связи, таким как даркнет.

Для анализа угроз центр безопасности использует данные, поступающие из нескольких источников в Майкрософт. Центр безопасности использует эти данные для выявления потенциальных угроз в вашей среде. Область "Анализ угроз" включает три основных раздела.

- Типы обнаруженных угроз
- Источник угроз.
- Карта анализа угроз


## <a name="when-should-i-use-threat-intelligence"></a>Назначение анализа угроз
В рамках [реагирования на нарушение безопасности](https://docs.microsoft.com/azure/security-center/security-center-planning-and-operations-guide#incident-response) необходимо определить степень серьезности нарушения безопасности систем. На этом этапе следует выполнить следующие задачи:

- определить характер атаки;
- определить источник атаки;
- определить цель атаки (Была ли атака специально направлена на вашу организацию для получения конкретных данных или она случайна?);
- определить системы, безопасность которых была нарушена;
- определить файлы, к которым осуществлялся доступ, и уровень конфиденциальности этих файлов. Для этого можно использовать данные анализа угроз в центре безопасности. 

## <a name="how-to-access-the-threat-intelligence"></a>Получение доступа к функции анализа угроз
Чтобы визуализировать текущие результаты анализа угроз для среды, сначала выберите рабочую область, в которой находятся ваши данные. Если у вас одна рабочая область, селектор рабочих областей не отображается. Открывается непосредственно панель мониторинга **Анализ угроз**. Выполните инструкции ниже, чтобы перейти к панели мониторинга "Анализ угроз".

1.  Откройте панель мониторинга **Центр безопасности**.
2.  В области слева в разделе **Обнаружение** щелкните **Анализ угроз**. Откроется панель мониторинга **Анализ угроз**.

    ![Анализ угроз](./media/security-center-threat-intel/security-center-threat-intel-fig1.png)

    > [!NOTE]
    > Если в последнем столбце отображается элемент **UPGRADE PLAN** (План обновления), это означает, что для соответствующей рабочей области используется бесплатная подписка. Чтобы использовать эту функцию, необходимо перейти на уровень "Стандартный". Если же отображается элемент REQUIRES UPDATE (Требуется обновление), необходимо обновить [службу Azure Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview), чтобы использовать эту функцию. Дополнительные сведения о тарифном плане см. на странице с ценами на центр безопасности Azure. 
    > 
3. Чтобы исследовать несколько рабочих областей, вы можете назначить приоритет для исследования на основе данных в столбце **MALICIOUS IP** (Вредоносный IP-адрес). Здесь указано текущее количество вредоносных IP-адресов в этой рабочей области. Щелкните рабочую область, которую необходимо использовать, и появится панель мониторинга **Анализ угроз**.

    ![Анализ угроз](./media/security-center-threat-intel/security-center-threat-intel-fig5.png)

4. Эта панель мониторинга состоит из трех плиток:
    * **Типы угроз.** Здесь представлены типы угроз, которые были обнаружены в выбранной рабочей области.
    * **Страна источника.** Здесь представлен суммарный объем трафика по его источнику.
    * **Расположение угроз.** Помогает определить текущие расположения, в которых происходит обмен данными с вашей средой, по всему миру. На карте показаны оранжевые (входящий трафик) и красные (исходящий трафик) стрелки, определяющие направление трафика. Если щелкнуть одну из этих стрелок, на экран будут выведены сведения о типе угрозы и направлении трафика.
    * **Сведения об угрозах.** Здесь представлены дополнительные сведения об угрозе, выбранной на карте.

Независимо от выбранной плитки, данные на панели мониторинга будут отображаться на основе запроса [Поиск по журналам](https://docs.microsoft.com/azure/security-center/security-center-search), отличаться будет только тип запроса и результат.

### <a name="threat-types"></a>Типы угроз
Когда вы щелкнете плитку **Типы угроз**, появится панель мониторинга **Поиск по журналам**. Слева на ней отображаются параметры фильтра, справа — результаты запроса.

![Панель поиска для анализа угроз](./media/security-center-threat-intel/security-center-threat-intel-fig3.png)

В результатах запроса угрозы сгруппированы по имени. В области слева можно выбрать атрибут для фильтрации. Например, чтобы просмотреть только угрозы, которые в данный момент подключены к машинам, выберите для **SESSIONSTATE** значение **Подключено** и нажмите кнопку **Применить**.

![состояние сеанса](./media/security-center-threat-intel/security-center-threat-intel-fig4.png)

Для виртуальных машин Azure на панели мониторинга "Анализ угроз" будут отображаться только данные сети, которые проходят через агент. Кроме того, при анализе угроз используются следующие данные:

- Данные CEF (тип = CommonSecurityLog)
- WireData (тип = WireData)
- Журналы IIS (тип = W3CIISLog)
- Брандмауэр Windows (тип = WindowsFirewall)
- События DNS (тип = DnsEvents)

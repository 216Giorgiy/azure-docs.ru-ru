# Настройка политик безопасности в Центре безопасности Azure
В этом пошаговом руководстве описывается, как настроить политики безопасности в центре безопасности.

## Что такое политики безопасности?
Политика безопасности определяет набор элементов управления, которые рекомендуются для ресурсов в указанной подписке или группе ресурсов. В центре безопасности можно настраивать политики для подписок или групп ресурсов Azure в соответствии с потребностями безопасности вашей компании, типом приложений и конфиденциальностью данных в каждой подписке.

Например, требования к безопасности ресурсов, используемых при разработке и тестировании, могут отличаться от тех, которые используются в рабочих приложениях. Аналогично: приложения с контролируемыми данными, такими как персональные данные (PII), могут требовать более высокого уровня безопасности. Политики безопасности, настроенные в центре безопасности Azure, управляют отображением рекомендаций по обеспечению безопасности и мониторингом, помогая вам находить потенциальные уязвимости и устранять угрозы. Дополнительные сведения об определении наиболее подходящего для вас варианта см. в [руководстве по планированию использования центра безопасности Azure и работе в нем](security-center-planning-and-operations-guide.md).

## Настройка политик безопасности для подписок

Политики безопасности можно настроить для каждой подписки или группы ресурсов. Для изменения политики безопасности требуются права владельца или участника этой подписки. Перейдите на портал Azure и выполните следующие действия, чтобы настроить политики безопасности в центре безопасности.

1. Щелкните плитку **Политика** на панели мониторинга в центре безопасности.

2. Справа откроется колонка **Политика безопасности — Определить политику для подписки или группы ресурсов**. Выберите подписку, для которой требуется включить политику безопасности. Если вы предпочитаете включить политику безопасности для группы ресурсов, а не для всей подписки, перейдите к следующему разделу, в котором рассказывается о настройке политик безопасности для групп ресурсов.

    ![Определение политики](./media/security-center-policies/security-center-policies-fig1-ga.png)

3. Для этой подписки откроется колонка **Политика безопасности** со сходным набором параметров:

    ![Включение сбора данных](./media/security-center-policies/security-center-policies-fig2-ga.png)

	В этой колонке доступны следующие параметры.
	- **Политика предотвращения** — этот параметр позволяет настроить политики для каждой подписки или группы ресурсов.
	- **Уведомление по электронной почте** — уведомление по электронной почте отправляется только для первых за день оповещений и для оповещений с высоким уровнем серьезности. Параметры электронной почты можно настроить только для политик подписки. Чтобы настроить уведомление по электронной почте, см. инструкции в статье [Предоставление сведений о контактных лицах по вопросам безопасности в центре безопасности Azure](security-center-provide-security-contact-details.md).
	- **Ценовая категория** — используйте этот параметр для обновления ценовой категории. Дополнительные сведения о ценах см. на странице [Центр безопасности: цены](https://azure.microsoft.com/pricing/details/security-center/).

	
4.  Убедитесь, что для параметра **Сбор данных из виртуальных машин** задано значение **Вкл**. Этот параметр включает автоматический сбор данных журнала для существующих и новых ресурсов.

    >[AZURE.NOTE] Мы рекомендуем вам настроить сбор данных для каждой подписки, чтобы включить мониторинг безопасности для всех существующих и новых виртуальных машин. Включение сбора данных приводит к установке агента мониторинга. Если вы не хотите включать сбор данных прямо сейчас из этого расположения, это можно сделать позже из представлений работоспособности и рекомендаций. Сбор данных также можно включить только для подписки или только для выбранных виртуальных машин. Дополнительные сведения о поддерживаемых виртуальных машинах см. в разделе [Часто задаваемые вопросы о Центре безопасности Azure](security-center-faq.md).

5. Если учетная запись хранения еще не настроена, при открытии **политики безопасности** появится примерно такое сообщение:

    ![Выбор хранилища](./media/security-center-policies/security-center-policies-fig2.png)

6. Если вы увидите такое предупреждение, щелкните его и выберите регион:

    ![Выбор хранилища](./media/security-center-policies/security-center-policies-fig3-ga.png)

7. Для каждого региона, в котором у вас есть виртуальные машины, выберите учетную запись хранения, где хранятся данные, собираемые с этих виртуальных машин. Это позволяет хранить данные в одной географической области, чтобы обеспечить их конфиденциальность и независимость. Выберите нужный регион, а затем учетную запись хранения.

8. В колонке **Выбор учетных записей хранения** нажмите кнопку **ОК**.

    > [AZURE.NOTE] При желании можно объединить данные из виртуальных машин в разных регионах в рамках одной центральной учетной записи хранения. Дополнительные сведения см. в статье [Центр безопасности Azure: часто задаваемые вопросы](security-center-faq.md).

9. В колонке **Политика безопасности** щелкните **Вкл.**, чтобы включить рекомендации по безопасности, которые вы хотите использовать для этой подписки. Щелкните параметр **Политика предотвращения**. Откроется колонка **Политика безопасности**:

	![Выбор политик безопасности](./media/security-center-policies/security-center-policies-fig4-ga.png)

Следующая таблица содержит справочные сведения о различных параметрах:

| Политика | Если состояние "Вкл." |
|----- |-----|
| Обновление системы; | Ежедневно получает список доступных критических обновлений и обновлений для системы безопасности из Центра обновления Windows или служб WSUS (в зависимости от того, какая служба настроена для соответствующей виртуальной машины) и рекомендует установить отсутствующие обновления. Проверяет наличие последних обновлений в системах Linux с помощью системы управления пакетами (предоставляется дистрибутивами) для определения доступных пакетов обновления. Также проверяет наличие критических обновлений и обновлений системы безопасности на виртуальных машинах [облачных служб](./cloud-services/cloud-services-how-to-configure.md). |
| Уязвимости ОС | Ежедневно анализирует конфигурации операционной системы, которые могут сделать виртуальную машину более уязвимой для атак, и рекомендует изменения конфигурации для устранения таких уязвимостей. Дополнительные сведения о конкретных отслеживаемых конфигурациях см. в [списке рекомендуемых базовых шаблонов](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335). |
| Endpoint Protection | Рекомендует подготовить решения для защиты конечных точек для всех виртуальных машин Windows, чтобы обнаруживать и удалять вирусы, программы-шпионы и другие вредоносные программы.|
| Шифрование дисков | Рекомендует включить на всех виртуальных машинах шифрование дисков, чтобы усилить защиту хранящихся данных. 
| группы сетевой безопасности; | Рекомендует настроить [группы безопасности сети](../virtual-network/virtual-networks-nsg.md) для контроля входящего в виртуальные машины и исходящего из них трафика с использованием общедоступных конечных точек. Группы безопасности сети, настроенные для подсети, будут наследоваться всеми сетевыми интерфейсами виртуальных машин, если не указано иначе. В дополнение к проверке настройки группы безопасности сети этот параметр получает доступ к правилам безопасности входа, чтобы выявить правила, разрешающие любой входящий трафик. |
| Брандмауэр веб-приложения | Рекомендует подготовить брандмауэр веб-приложения на виртуальных машинах, если используется [общедоступный IP-адрес уровня экземпляра](../virtual-network/virtual-networks-instance-level-public-ip.md), а в связанных правилах безопасности для входящих подключений группы безопасности сети разрешен доступ к порту 80 или 443. Используется IP-адрес с балансировкой нагрузки (виртуальный IP-адрес), и в связанных правилах балансировки нагрузки и NAT для входящих подключений разрешен доступ к портам 80 или 443 (дополнительные сведения см. в разделе [Поддержка диспетчера ресурсов Azure для подсистемы балансировки нагрузки](../load-balancer/load-balancer-arm.md)). |
| Брандмауэр следующего поколения | Он расширяет защиту сети за пределы групп безопасности сети, встроенных в Azure. Центр обеспечения безопасности будет обнаруживать развертывания, для которых рекомендуется использовать брандмауэр следующего поколения, и позволит подготовить виртуальное устройство. |
| Аудит SQL | Рекомендует включить аудит доступа к базам данных и серверам SQL Azure в целях обеспечения соответствия, расширенного обнаружения и исследования. |
| Прозрачное шифрование данных SQL | Рекомендует включить шифрование при хранении для баз данных SQL Azure, связанных резервных копий и файлов журналов транзакций, что позволит защитить данные от считывания, даже если безопасность будет нарушена. |
	
Завершив настройку всех параметров, нажмите кнопку **ОК** в колонке **Политика безопасности**, где содержатся рекомендации, и **Сохранить** в колонке **Политика безопасности** с начальными настройками.

## Настройка политик безопасности для групп ресурсов

Если вы предпочитаете настраивать политику безопасности для каждой группы ресурсов, действия аналогичны тем, которые применяются для настройки политики безопасности для подписок. Основное различие заключается в необходимости развернуть имя подписки и выбрать группу ресурсов, для которой необходимо настроить политику безопасности:

![Выбор группы ресурсов](./media/security-center-policies/security-center-policies-fig5-ga.png)

После выбора группы ресурсов откроется колонка **Политика безопасности**. По умолчанию параметр **Наследование** включен. Это означает, что все политики безопасности для этой группы ресурсов наследуются от уровня подписки. Можно изменить данную настройку, если нужно создать пользовательскую политику безопасности для каждой группы ресурсов. Для этого необходимо выбрать параметр **Уникальная** и внести необходимые изменения в разделе **Политика предотвращения**.

![Политика безопасности для каждой группы ресурсов](./media/security-center-policies/security-center-policies-fig6-ga.png)

> [AZURE.NOTE] В случае конфликта между политикой уровня подписки и политикой уровня группы ресурсов приоритет получает политика уровня ресурсов.

# Настройка политик безопасности в Центре безопасности Azure
Этот документ поможет настроить политики безопасности в Центре безопасности Azure. В нем описаны действия, необходимые для выполнения этой задачи.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure.

## Что такое Центр безопасности Azure?
Центр безопасности Azure помогает вам выявлять и предотвращать угрозы, а также принимать ответные меры благодаря более полной информации о состоянии ресурсов Azure и контролю над их безопасностью. Он включает встроенные функции мониторинга безопасности и управления политиками для подписок, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

## Что такое политики безопасности?
Политика безопасности определяет набор средств контроля, которые рекомендуются для ресурсов в указанной подписке или группе ресурсов. В Центре безопасности Azure можно настраивать политики для подписок Azure или групп ресурсов в соответствии с потребностями безопасности вашей компании, типом приложений и конфиденциальностью данных в каждой подписке.

Например, требования к безопасности ресурсов, используемых для разработки или тестирования, могут отличаться от тех, которые используются для рабочих приложений. Аналогично: приложения с контролируемыми данными, такими как персональные данные (PII), могут требовать более высокого уровня безопасности. Политики безопасности, которые включены в Центре безопасности Azure, обеспечивают рекомендации по обеспечению безопасности и мониторинг, позволяя выявлять потенциальные уязвимости и бороться с угрозами.

## Настройка политик безопасности для подписок

Политики безопасности можно настроить для каждой подписки или группы ресурсов. Для изменения политики безопасности требуются права владельца или участника этой подписки. Выполните следующие действия, чтобы настроить политики безопасности в Центре безопасности Azure.

1. Щелкните элемент **Политики безопасности** на панели мониторинга Центра безопасности Azure.

2. В колонке **Политика безопасности — определение политики для подписки или группы ресурсов**, открывшейся справа, выберите подписку, для которой требуется включить политику безопасности. Если вы предпочитаете включить политику безопасности для группы ресурсов, а не для всей подписки, перейдите к следующему разделу, в котором рассказывается о настройке политик безопасности для групп ресурсов.

    ![Определение политики](./media/security-center-policies/security-center-policies-fig01.png)

3. Для этой подписки откроется колонка **Политика безопасности** с набором параметров, аналогичным показанному ниже.

    ![Включение сбора данных](./media/security-center-policies/security-center-policies-fig1-new.png)

4. Убедитесь, что для параметра **Сбор данных из виртуальных машин** задано значение **Вкл**. Этот параметр включает автоматический сбор данных журнала для существующих и новых ресурсов.

    >[AZURE.NOTE] Настоятельно рекомендуем включить сбор данных для каждой подписки, так как в этом случае мониторинг безопасности будет доступен для всех существующих и новых виртуальных машин. Включение сбора данных приводит к установке агента мониторинга. Если вы не хотите включать сбор данных прямо сейчас из этого расположения, это можно сделать позже из представлений работоспособности и рекомендаций. Сбор данных также можно включить только для подписки или только для выбранных виртуальных машин. Дополнительные сведения о поддерживаемых виртуальных машинах см. в разделе [Часто задаваемые вопросы о Центре безопасности Azure](security-center-faq.md).

5. Если учетная запись хранения еще не настроена, при открытии **политики безопасности** появится примерно такое же сообщение, как показано на рисунке ниже.

    ![Выбор хранилища](./media/security-center-policies/security-center-policies-fig2.png)

6. Если вы видите это предупреждение, щелкните этот пункт и выберите регион, как показано на рисунке ниже:

    ![Выбор хранилища](./media/security-center-policies/security-center-policies-fig3.png)

7. Для каждого региона, в котором у вас есть виртуальные машины, выберите учетную запись хранения, где хранятся данные, собираемые с этих виртуальных машин. Это позволяет хранить данные в одной географической области, чтобы обеспечить их конфиденциальность и независимость. Выберите нужный регион, а затем учетную запись хранения.

8. В колонке **Выбор учетных записей хранения** нажмите кнопку **ОК**.

    > [AZURE.NOTE] При желании можно объединить данные из виртуальных машин в разных регионах в рамках одной центральной учетной записи хранения. Дополнительные сведения см. в разделе [Часто задаваемые вопросы о Центре безопасности Azure](security-center-faq.md).

9. В колонке **Политика безопасности** щелкните **Вкл.**, чтобы включить рекомендации по безопасности, которые вы хотите использовать для этой подписки. Щелкните параметр **Политика предотвращения**. После этого откроется колонка **Политика безопасности**, показанная ниже.

	![Выбор политик безопасности](./media/security-center-policies/security-center-policies-fig1-1-new2.png)

Следующая таблица содержит справочные сведения о различных параметрах:

| Политика | Если состояние "Вкл." |
|----- |-----|
| Обновление системы; | Ежедневно получает список доступных критических обновлений и обновлений для системы безопасности из Центра обновления Windows или служб WSUS (в зависимости от того, какая служба настроена для соответствующей виртуальной машины) и рекомендует установить отсутствующие обновления. Кроме того, она проверяет наличие последних обновлений в системах Linux, а также критических обновлений и обновлений системы безопасности на виртуальных машинах [облачных служб](./cloud-services/cloud-services-how-to-configure.md). |
| Правила базовых шаблонов | Ежедневно анализирует конфигурации операционной системы, которые могут сделать виртуальную машину более уязвимой для атак, и рекомендует изменения конфигурации для устранения таких уязвимостей. Дополнительные сведения о конкретных отслеживаемых конфигурациях см. в [списке рекомендуемых базовых шаблонов](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335). |
| Endpoint Protection | Рекомендует подготовить решения для защиты конечных точек для всех виртуальных машин Windows, чтобы обнаруживать и удалять вирусы, программы-шпионы и другие вредоносные программы. 
| группы сетевой безопасности; | Рекомендует настроить [группы безопасности сети](../virtual-network/virtual-networks-nsg.md) (NSG) для контроля входящего и исходящего трафика к подсетям и сетевым интерфейсам. Группы безопасности сети, настроенные для подсети, будут наследоваться всеми сетевыми интерфейсами виртуальных машин, если не указано иначе. В дополнение к проверке настройки группы безопасности сети этот параметр получает доступ к правилам безопасности входа, чтобы выявить правила, разрешающие любой входящий трафик. |
| Брандмауэр веб-приложения | Рекомендует подготовить брандмауэр веб-приложения на виртуальных машинах, если используется [общедоступный IP-адрес уровня экземпляра](../virtual-network/virtual-networks-instance-level-public-ip.md), а в связанных правилах безопасности для входящих подключений группы безопасности сети разрешен доступ к порту 80 или 443. Используется IP-адрес с балансировкой нагрузки (виртуальный IP-адрес), и в связанных правилах балансировки нагрузки и NAT для входящих подключений разрешен доступ к портам 80 или 443 (дополнительные сведения см. в разделе [Поддержка диспетчера ресурсов Azure для подсистемы балансировки нагрузки](../load-balancer/load-balancer-arm.md)). |
| Брандмауэр следующего поколения | Расширяет защиту сети за пределы групп безопасности сети, встроенных в Azure. Центр обеспечения безопасности будет обнаруживать развертывания, для которых рекомендуется использовать брандмауэр следующего поколения, и позволит подготовить виртуальное устройство. |
| Аудит SQL | Рекомендует включить аудит доступа к базам данных и серверам SQL Azure в целях обеспечения соответствия, расширенного обнаружения и исследования. |
| Прозрачное шифрование данных SQL | Рекомендует включить шифрование при хранении для баз данных SQL Azure, связанных резервных копий и файлов журналов транзакций, что позволит защитить данные от считывания, даже если безопасность будет нарушена. |

11\. После завершения настройки всех параметров нажмите кнопку **ОК** в колонке **Политика безопасности**, где содержатся рекомендации, и кнопку **Сохранить** в колонке **Политика безопасности** с начальными настройками.

## Настройка политик безопасности для групп ресурсов

Если вы предпочитаете настраивать политику безопасности для каждой группы ресурсов, действия аналогичны тем, которые применяются для настройки политики безопасности для подписок. Основное различие заключается в необходимости развернуть имя подписки и выбрать группу ресурсов, для которой необходимо настроить политику безопасности:

![Выбор группы ресурсов](./media/security-center-policies/security-center-policies-fig4.png)

После выбора группы ресурсов откроется колонка **Политика безопасности**. По умолчанию параметр **Наследование** включен. Это означает, что все политики безопасности для этой группы ресурсов наследуются от уровня подписки. Можно изменить данную настройку, если нужно создать пользовательскую политику безопасности для каждой группы ресурсов. Для этого необходимо выбрать параметр **Уникальная** и внести необходимые изменения в разделе **Политика предотвращения**.

![Политика безопасности для каждой группы ресурсов](./media/security-center-policies/security-center-policies-fig5-new.png)

> [AZURE.NOTE] В случае конфликта между политикой уровня подписки и политикой уровня группы ресурсов приоритет получает политика уровня ресурсов.

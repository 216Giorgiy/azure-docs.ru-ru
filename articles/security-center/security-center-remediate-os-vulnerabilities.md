# <a name="remediate-os-vulnerabilities-in-azure-security-center"></a>Исправление уязвимостей ОС в центре безопасности Azure
Центр безопасности Azure ежедневно анализирует операционную систему виртуальных машин и компьютеров, чтобы определить конфигурацию, повышающую риск атак на них. Центр безопасности Майкрософт рекомендует устранять уязвимости, если конфигурация операционной системы не соответствует правилам рекомендуемой конфигурации, и изменять конфигурацию, чтобы устранить эти уязвимости.

> [!NOTE]
> Дополнительные сведения о конкретных отслеживаемых конфигурациях доступны в [списке рекомендуемых правил конфигурации](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335).
>
>

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций
Устранение уязвимостей ОС представлено в центре безопасности в виде рекомендации. Эта рекомендация будет отображаться в разделе **Рекомендации** и **Вычисления**.

В этом примере мы рассмотрим рекомендацию **Устранение уязвимостей ОС (от Майкрософт)** в разделе **Вычисления**.
1. В главном меню центра безопасности выберите **Вычисления**.

   ![Исправление уязвимостей ОС][1]

2. В разделе **Вычисления** выберите **Устранение уязвимостей ОС (от Майкрософт)**. Откроется панель мониторинга **Устранение уязвимостей ОС (от Майкрософт)**.

   ![Исправление уязвимостей ОС][2]

  В верхней части панели мониторинга содержатся следующие сведения:

  - Общее количество правил по уровню серьезности, не выполненных в конфигурации ОС виртуальной машины и компьютера.
  - Общее количество правил по типу, не выполненных в конфигурации ОС виртуальной машины и компьютера.
  - Общее количество правил, не выполненных в конфигурации ОС Windows и Linux.

  В нижней части панели мониторинга перечислены все не выполненные правила виртуальных машин и компьютеров, а также уровень серьезности отсутствующего обновления. Этот список содержит следующие параметры:

  - **CCIED** — уникальный идентификатор CCE правила; Центр безопасности использует перечисление общей конфигурации (CCE) для назначения уникальных идентификаторов для правил конфигурации.
  - **Имя** — имя не выполненного правила;
  - **Тип правила** — "Раздел реестра", "Политика безопасности" или "Политика аудита";
  - **Число OF VMs & COMPUTERS** (Количество виртуальных машин и компьютеров) — общее количество виртуальных машин и компьютеров, к которым применялось не выполненное правило;
  - **RULE SEVERITY** (Серьезность правила) указывает значение серьезности CCE: критическая ошибка, важное уведомление или предупреждение;
  - **Состояние**— текущее состояние рекомендации:

    - **Открыто**— рекомендация еще не выполнена;
    - **Выполняется** — сейчас рекомендация применяется к ресурсам (от вас не требуется никаких действий);
    - **Разрешено** — рекомендация уже применена (после устранения проблемы запись становится недоступной).

3. Выберите не выполненное правило в списке, чтобы просмотреть подробные сведения.

   ![Правила конфигурации, которые не были выполнены][3]

  В этой колонке предоставляется следующая информация:

  - "Имя" — имя правила;
  - "CCIED" — уникальный идентификатор CCE правила;
  - "Версия ОС" — версия ОС виртуальной машины или компьютера;
  - RULE SEVERITY (Серьезность правила) указывает значение серьезности CCE: критическая ошибка, важное уведомление или предупреждение;
  - FULL DESCRIPTION (Полное описание) — описание правила;
  - "Уязвимость" — описание уязвимости или риска в случае, если правило не применяется;
  - POTENTIAL IMPACT (Потенциальное воздействие) — влияние на работу организации в случае применения правила;
  - COUNTERMEASURE (Мера противодействия) — действия по устранению проблемы;
  - "Ожидаемое значение" — ожидаемое значение при анализе центром безопасности конфигурации ОС виртуальной машины посредством правила;
  - "Фактическое значение" — значение, возвращаемое после анализа конфигурации ОС виртуальной машины посредством правила;
  - "Операция правила" — операция правила, используемая центром безопасности во время анализа конфигурации ОС виртуальной машины посредством правила;

4. Выберите значок **поиска** на верхней ленте. Откроется окно "Поиск" со списком рабочих областей с выбранными уязвимостями ОС виртуальных машин и компьютеров. Эта колонка выбора рабочей области доступна, только если выбранное правило применяется к нескольким виртуальным машинам, подключенным к разным рабочим областям.

  ![Указанные рабочие области][4]

5. Выберите рабочую область. Откроется поисковой запрос Log Analytics, отфильтрованный по рабочей области с уязвимостью операционной системы.

  ![Рабочая область с уязвимостью операционной системы][5]

6. Выберите компьютер из списка для получения дополнительной информации. Откроется еще один результат поиска с данными только для этого компьютера.

  ![Фильтрация для компьютера][6]

Набор правил конфигурации можно просмотреть [здесь](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335).

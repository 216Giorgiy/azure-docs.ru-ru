# Добавление брандмауэра веб-приложения в Центре безопасности Azure

Центр безопасности Azure может рекомендовать добавить брандмауэр веб-приложения (WAF) от партнера корпорации Майкрософт в целях обеспечения безопасности веб-приложений. В этом документе вы найдете пример применения такой рекомендации.

> [AZURE.NOTE] Сведения в этом документе относятся к предварительной версии Центра безопасности Azure. В документе приводится обзор службы с помощью примера развертывания. Он не является пошаговым руководством.

## Выполнение рекомендаций

1. В колонке **Рекомендации** выберите рекомендацию **Защита веб-приложения с помощью брандмауэра веб-приложения**. ![][1]

2. В колонке **Защита веб-приложений с помощью брандмауэра веб-приложения** выберите веб-приложение. Откроется колонка **Добавление брандмауэра веб-приложения**. ![][2]
3. Можно использовать существующий брандмауэр веб-приложения (если он доступен) или создать новый. В этом примере нет доступных WAF, поэтому мы создадим новый WAF.

4. Чтобы создать WAF, выберите решение из списка решений партнеров по интеграции. В этом примере выберите **Barracuda Web Application Firewall**.
5. Откроется колонка **Barracuda Web Application Firewall**, в которой содержатся сведения о партнерском решении. В колонке сведений щелкните **Создать**. ![][3]

6. Откроется колонка **Новый брандмауэр веб-приложения**, где вы можете выполнить действия по **настройке виртуальной машины** и указать **сведения о WAF**. Выберите **Конфигурация виртуальной машины**.

7. В колонке **Конфигурация виртуальной машины** введите сведения, необходимые для настройки виртуальной машины, на которой будет выполняться WAF. ![][4]
8. Вернитесь в колонку **Новый брандмауэр веб-приложения** и выберите **Сведения о WAF**. В колонке **Сведения о WAF** настраивается сам брандмауэр веб-приложения. На шаге 7 выполняется настройка виртуальной машины, на которой будет выполняться WAF, а на шаге 8 осуществляется подготовка самого брандмауэра.

9. Вернитесь в колонку **Рекомендации**. После создания WAF появится новая запись с именем **Завершение настройки брандмауэра веб-приложения**. Эта запись указывает, что необходимо завершить процедуру, фактически подключив брандмауэр WAF к виртуальной сети Azure, чтобы он мог защищать приложение. ![][5]

10. Выберите **Завершение настройки брандмауэра веб-приложения**. Откроется новая панель. Вы увидите, что появилось веб-приложение, для которого нужно перенаправлять трафик.
11. Выберите веб-приложение. Откроется колонка, в которой приведены пошаговые инструкции по завершению настройки брандмауэра веб-приложения. Выполните эти действия и щелкните **Ограничить трафик**. Центр безопасности Azure выполнит необходимые подключения. ![][6]

> [AZURE.NOTE] Вы можете защитить несколько веб-приложений в центре обеспечения безопасности, добавив эти приложения в существующие развернутые службы WAF. Устройства WAF (созданные с помощью модели развертывания Resource Manager) должны быть развернуты в отдельной виртуальной сети. Устройства WAF (созданные с помощью классической модели развертывания) ограничены заданной группой безопасности сети. В будущем эта поддержка будет расширена до полностью настраиваемого развертывания устройства WAF (в классической модели). Узнайте больше о [классической модели развертывания и модели развертывания диспетчера ресурсов](../azure-classic-rm.md) для ресурсов Azure.

Журналы WAF теперь полностью интегрированы. Центр безопасности может автоматически начать сбор и анализ журналов, чтобы создавать важные оповещения системы безопасности.

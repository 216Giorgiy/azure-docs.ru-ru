# <a name="add-a-web-application-firewall-in-azure-security-center"></a>Добавление брандмауэра веб-приложения в Центре безопасности Azure
Центр безопасности Azure может рекомендовать добавить брандмауэр веб-приложения (WAF) от партнера корпорации Майкрософт в целях обеспечения безопасности веб-приложений. В этом документе вы найдете пример применения такой рекомендации.

> [!NOTE]
> В документе приводится обзор службы с помощью примера развертывания.  Он не является пошаговым руководством.
> 
> 

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций
1. В колонке **Рекомендации** выберите рекомендацию **Secure web application using web application firewall** (Защита веб-приложения с помощью брандмауэра веб-приложения).
   ![Защищенное веб-приложение][1]
2. В колонке **Защита веб-приложений с помощью брандмауэра веб-приложения** выберите веб-приложение. Откроется колонка **Добавление брандмауэра веб-приложения** .
   ![Add a web application firewall][2]
3. Можно использовать существующий брандмауэр веб-приложения (если он доступен) или создать новый. В этом примере нет доступных WAF, поэтому мы создадим новый WAF.
4. Чтобы создать WAF, выберите решение из списка решений партнеров по интеграции. В этом примере выберите **Barracuda Web Application Firewall**.
5. Откроется колонка **Barracuda Web Application Firewall** , в которой содержатся сведения о партнерском решении. В колонке сведений щелкните **Создать** .
   ![Колонка со сведениями о брандмауэре][3]
6. Откроется колонка **Новый брандмауэр веб-приложения**, где вы можете выполнить действия по **настройке виртуальной машины** и указать **сведения о WAF**. Выберите **Конфигурация виртуальной машины**.
7. В колонке **Конфигурация виртуальной машины** введите сведения, необходимые для настройки виртуальной машины, на которой будет выполняться WAF.
   ![VM configuration][4]
8. Вернитесь в колонку **Новый брандмауэр веб-приложения** и выберите **Сведения о WAF**. В колонке **Сведения о WAF** настраивается сам брандмауэр веб-приложения. На шаге 7 выполняется настройка виртуальной машины, на которой будет выполняться WAF, а на шаге 8 осуществляется подготовка самого брандмауэра.

## <a name="finalize-application-protection"></a>Завершение подготовки защиты приложений
1. Вернитесь в колонку **Рекомендации** . После создания WAF появляется новая запись с именем **Завершение подготовки защиты приложений**. Эта запись указывает, что необходимо завершить процедуру, фактически подключив брандмауэр WAF к виртуальной сети Azure, чтобы он мог защищать приложение.
   ![Завершение подготовки защиты приложений][5]
2. Выберите **Завершение подготовки защиты приложений**. Откроется новая панель. Вы увидите, что появилось веб-приложение, для которого нужно перенаправлять трафик.
3. Выберите веб-приложение. Откроется колонка, в которой приведены пошаговые инструкции по завершению настройки брандмауэра веб-приложения. Выполните эти действия и щелкните **Ограничить трафик**. Центр безопасности Azure выполнит необходимые подключения.
   ![Ограничить трафик][6]

> [!NOTE]
> Вы можете защитить несколько веб-приложений в центре обеспечения безопасности, добавив эти приложения в существующие развернутые службы WAF. Устройства WAF (созданные с помощью модели развертывания Resource Manager) должны быть развернуты в отдельной виртуальной сети. Устройства WAF (созданные с помощью классической модели развертывания) ограничены заданной группой безопасности сети. В будущем эта поддержка будет расширена до полностью настраиваемого развертывания устройства WAF (в классической модели). Узнайте больше о [классической модели развертывания и модели развертывания Resource Manager](../azure-classic-rm.md) для ресурсов Azure.
> 
> 

Журналы WAF теперь полностью интегрированы. Центр безопасности может автоматически начать сбор и анализ журналов, чтобы создавать важные оповещения системы безопасности.

# Использование соединителя Slack в приложении логики #

Приложения логики могут запускаться на основе различных источников данных и предлагают соединители для получения и обработки данных в рамках потока.

Соединитель Slack дает возможность отправлять сообщения в каналы Slack.

## Создание соединителя Slack для приложения логики ##
Для использования соединителя Slack необходимо сначала создать экземпляр приложения API соединителя Slack. Это можно сделать следующим образом.

1.	Откройте Azure Marketplace, используя параметр «+СОЗДАТЬ» в нижнем левом углу портала Azure.
2.	Перейдите к разделу «Мобильные приложение + Интернет \> Azure Marketplace» и найдите пункт «Соединитель Slack».
3.	Настройте соединитель Slack, как описано ниже.
 
	![][1]
 - **Имя** — введите имя соединителя Slack
 - **План обслуживания приложения** — выберите или создайте план службы приложений - 
 - **Ценовая категория** — выберите ценовую категорию для соединителя - 
 - **Группа ресурсов** — выберите или создайте группу ресурсов, где должен находиться соединитель - 
 - **Подписка** — выберите подписку, в которой необходимо создать соединитель - 
 - **Расположение** — выберите географическое расположение, где нужно развернуть соединитель

4. Нажмите кнопку «Создать». Будет создан новый соединитель Slack.
5. После создания экземпляра приложения API можно создать приложение логики в той же группе ресурсов для использования соединителя Slack.

## Использование соединителя Slack в приложении логики ##
После создания приложения API можно использовать соединитель Slack как действие для приложения логики. Для этого необходимо выполнить следующие действия.

1.	Создайте новое приложение логики и выберите ту же группу ресурсов, что и для соединителя Slack. Следуйте инструкциям по [созданию приложения логики].  	
	
2.	В созданном приложении логики откройте раздел «Триггеры и действия», чтобы открыть конструктор приложений логики и настроить поток.
	
3.	Соединитель Slack будет отображаться в разделе «Приложения API в этой группе ресурсов» в коллекции с правой стороны.
 
	![][2]
4.	Приложение API соединителя Slack можно переместить в редактор, щелкнув элемент «Соединитель Slack». Нажмите кнопку «Авторизовать». Укажите учетные данные Microsoft \(если вход не выполнен автоматически\). Войдите в учетную запись Slack, выполнив следующие действия. На последнем этапе создания будет предложено предоставить соединителю разрешение для доступа к учетной записи Slack. Нажмите кнопку «Authorizify» \(Разрешить\)
 
	![][3] 
	![][4] 
	![][5] 
	![][6]
	
5.	Теперь можно использовать соединитель Slack в потоке. В настоящее время в соединителе Slack триггеры недоступны. Доступно действие «Опубликовать сообщение»
 
	![][7]

6.	Давайте рассмотрим процесс «Опубликовать сообщение». Это действие можно использовать для отправки сообщения в канал Slack.
 
	![][8]

	Настройте свойства ввода для действия «Опубликовать сообщение» следующим образом.

 - **Текст** — укажите текст сообщения для публикации.
 - **Имя канала** — укажите канал Slack, в который будет отправлено это сообщение. Если данный параметр не указан, то сообщение будет опубликовано в канал \#general.

 **Дополнительные свойства** 
 - **Имя пользователя программы-робота** — имя программы-робота, которую необходимо использовать для этого сообщения. Сообщение будет опубликовано от имени «Робот», если этот параметр не указан.
 - **URL-адрес значка** — URL-адрес изображения для использования в качестве значка для данного сообщения
 - **Значок Emoji** — значок Emoji для использования в качестве значка для данного сообщения. Заменяет URL-адрес значка.
 

7. Для использования соединителя вне приложения логики можно использовать REST API, предоставляемые соединителем. Эти определения API можно просмотреть, выбрав «Обзор» -\> «Приложение API» -\> «Соединитель Slack». Теперь в разделе «Сводка» щелкните определение API для просмотра всех API, предоставляемых этим соединителем.

	![][9]

9. Сведения об API можно также найти в разделе [Определение Slack API].

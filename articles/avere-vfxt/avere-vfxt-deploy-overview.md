# <a name="avere-vfxt-for-azure---deployment-overview"></a>Общие сведения о развертывании Avere vFXT для Azure

В этой статье представлен обзор шагов, необходимых для настройки и запуска кластера Avere vFXT для Azure.

При первом развертывании системы Avere vFXT можно заметить, что для этого процесса требуется больше шагов, чем для развертывания большинства других средств Azure. Четкое понимание всего процесса поможет вам определить необходимые трудозатраты. После того как система будет настроена и запущена, ее способность ускорять задачи облачных вычислений оправдает все трудозатраты.

## <a name="deployment-steps"></a>Шаги по развертыванию

[Спланировав систему](avere-vfxt-deploy-plan.md), вы можете приступить к созданию кластера Avere vFXT. 

Начните с создания виртуальной машины контроллера кластера, которая используется для создания кластера vFXT.

Когда вы настроите и запустите кластер vFXT, вам потребуется подключить к нему клиенты и (при необходимости) переместить данные в новый контейнер хранилища больших двоичных объектов.  

Ниже приведен обзор всех шагов.

1. Настройка необходимых компонентов 

   Перед созданием виртуальной машины вам необходимо создать новую подписку для проекта Avere vFXT, настроить право владения подпиской, проверить квоты и при необходимости запросить их увеличение, а также принять условия использования программного обеспечения Avere vFXT. Подробные инструкции см. в статье [Подготовка к созданию Avere vFXT](avere-vfxt-prereqs.md).

1. Создание контроллера кластера

   *Контроллер кластера* — это простая виртуальная машина, находящаяся в той же виртуальной сети, что и кластер Avere vFXT. Контроллер создает узлы vFXT и формирует кластер, а также предоставляет интерфейс командной строки для управления кластером в течение его жизненного цикла.

   При настройке контроллера с общедоступным IP-адресом он также может служить узлом перехода для подключения к кластеру извне виртуальной сети.

   Все программное обеспечение, необходимое для создания кластера vFXT и управления его узлами, предварительно установлено в контроллере кластера.

   Дополнительные сведения см. в разделе [Создание виртуальной машины контроллера кластера](avere-vfxt-deploy.md#create-the-cluster-controller-vm).

1. Создание роли, которая будет применяться в среде выполнения, для узлов кластера 

   Azure использует [управление доступом на основе ролей](https://docs.microsoft.com/azure/role-based-access-control/) (RBAC), чтобы авторизовать виртуальные машины узла кластера для выполнения определенных задач. Например, узлы кластера должны иметь возможность назначать или переназначать IP-адреса другим узлам кластера. Перед созданием кластера необходимо определить роль, которая предоставляет им соответствующие разрешения.

   Предварительно установленное программное обеспечение контроллера кластера включает в себя роль-прототип для настройки. Инструкции см. в разделе [Создание роли доступа к узлу кластера](avere-vfxt-deploy.md#create-the-cluster-node-access-role).

1. Создание кластера Avere vFXT 

   В контроллере измените соответствующий сценарий создания кластера и выполните его, чтобы создать кластер. Подробные инструкции см. в разделе [Изменение сценария развертывания](avere-vfxt-deploy.md#edit-the-deployment-script). 

1. Настройка кластера 

   Подключитесь к интерфейсу конфигурации Avere vFXT (панель управления Avere), чтобы настроить параметры кластера. Согласитесь на получение поддержки мониторинга и добавьте свою систему хранения, если используете локальный центр обработки данных.

   * [Доступ к кластеру vFXT](avere-vfxt-cluster-gui.md)
   * [Включение поддержки](avere-vfxt-enable-support.md)
   * [Настройка хранилища](avere-vfxt-add-storage.md) (при необходимости)

1. Подключение клиентов

   Сведения о балансировке нагрузки и подключении клиентских компьютеров к кластеру см. в статье [Подключение кластера Avere vFXT](avere-vfxt-mount-clients.md).

1. Добавление данных (при необходимости)

   Так как Avere vFXT является масштабируемым кэшем с поддержкой нескольких клиентов, лучший способ переместить данные в новый контейнер хранилища — это стратегия многопотокового копирования файлов с поддержкой нескольких клиентов. Дополнительные сведения см. в статье [Moving data to the vFXT cluster - Parallel data ingest](avere-vfxt-data-ingest.md) (Перемещение данных в кластер vFXT. Прием параллельных данных).

# <a name="log-analytics-view-designer-visualization-part-reference"></a>Справка по элементам визуализации конструктора представлений Log Analytics
Конструктор представлений Log Analytics позволяет создавать в консоли OMS пользовательские представления, содержащие различные представления данных репозитория OMS. В этой статье содержится справка по параметрам для каждого элемента визуализации, доступного при создании пользовательских представлений.

Ниже перечислены другие статьи о конструкторе представлений:

* [Log Analytics View Designer](log-analytics-view-designer.md) (Конструктор представлений Log Analytics). Обзор конструктора представлений и процедур создания и редактирования пользовательских представлений.
* [Справка по плиткам](log-analytics-view-designer-tiles.md) — справочник по параметрам для каждой плитки, доступной в пользовательских представлениях. 

В приведенной ниже таблице перечислены различные типы плиток, доступных в конструкторе представлений.  В следующих разделах подробно описаны все типы плиток и их свойства.

| Тип представления | Описание |
|:--- |:--- |
| [Список запросов](#list-of-queries-part) |Здесь приведен список запросов поиска по журналу.  Пользователь может щелкнуть каждый запрос, чтобы отобразить его результаты.  Представление будет содержать один запрос по умолчанию. Щелкните **+ Query** (+ Запрос), чтобы добавить дополнительные запросы. |
| [Число и список](#number-amp-list-part) |Вверху приведено одно число, представляющее количество записей из запроса поиска по журналу.  В списке отображаются десять самых показательных результатов запросов. Горизонтальными линиями представлено относительное значение числового столбца. |
| [Два числа и список](#two-numbers-amp-list-part) |Вверху указаны два числа, представляющие количество записей из отдельных запросов поиска по журналу.  В списке отображаются десять самых показательных результатов запросов. Горизонтальными линиями представлено относительное значение числового столбца. |
| [Кольцевая диаграмма и список](#donut-amp-list-part) |Вверху показано одно число, полученное путем сложения чисел из столбца значений в запросе к журналу.  На кольцевой диаграмме отображаются результаты трех первых записей. |
| [Две временные шкалы и список](#two-timelines-amp-list-part) |Вверху показаны результаты двух запросов к журналу за определенный отрезок времени, представленные в виде гистограмм. В выноске отображается одно число, полученное путем сложения чисел из столбца значений в запросе к журналу.  В списке отображаются десять самых показательных результатов запросов. Горизонтальными линиями представлено относительное значение числового столбца. |
| [Сведения](#information-part) |Вверху содержится статический текст и необязательная ссылка.  В списке отображается один или несколько элементов со статическим текстом и заголовком. |
| [График с выноской и список](#line-chart-callout-amp-list-part) |Вверху представлен график с несколькими рядами данных из запросов к журналу за некоторый промежуток времени и выноска с итоговым значением.  В списке отображаются десять самых показательных результатов запросов. Горизонтальными линиями представлено относительное значение числового столбца. |
| [График и список](#line-chart-amp-list-part) |Вверху представлен график с несколькими рядами данных из запроса к журналу за некоторый промежуток времени.  В списке отображаются десять самых показательных результатов запросов. Горизонтальными линиями представлено относительное значение числового столбца. |
| [График и спарклайн](#line-chart-amp-sparklines-part) |Вверху представлен график с несколькими рядами данных из запроса к журналу за некоторый промежуток времени.  В списке отображаются десять самых показательных результатов запросов. Спарклайн представляет значение числового столбца за выбранный период времени. |

| Значок |Изображение, отображаемое в верхней части рядом с результатом. |

| **Список** | |
| Запрос |Запрос на выполнение для списка.  Будут показаны первые два свойства для первых десяти записей в результатах.  Первое свойство должно быть текстовым значением, а второе — числовым.  Линии создаются автоматически на основе относительного значения числового столбца.<br><br>С помощью команды сортировки в запросе можно отсортировать записи в списке.  Можно щелкнуть "Показать все", чтобы выполнить запрос и вернуть все записи. |
| Цвет |Цвет линий. |
| Разделитель имени и значения |Односимвольный разделитель, разбивающий свойства текста на несколько значений.  Задав разделитель, его можно использовать при указании имени для каждого поля в поле "Имя".<br><br>Например, рассмотрим свойство "Расположение", которое включает в себя значения, такие как *ул. Киевская-дом 41* и *ул. Днепровская-дом 12*.  Можно указать "–" в качестве разделителя свойства имени и значения и *улица-дом* для имени.  Таким образом, значение разделится на два свойства: *Улица* и *Дом*. |
| Запрос навигации |Запрос, выполняемый, когда пользователь выбирает элемент в списке.  Используйте {выбранный элемент}, чтобы включить синтаксис для элемента, выбранного пользователем. <br><br>Например, если запрос содержит столбец Computer, а запрос навигации — {выбранный элемент}, тогда при выборе пользователем компьютера будет выполнен запрос Computer="MyComputer".  Если запрос навигации имеет форму запроса Type=Event {выбранный элемент}, тогда будет выполнен запрос в форме Type=Event Computer="MyComputer". |
| **Список** |**> Имена столбцов** |
| Имя |Текст, отображаемый в верхней части первого столбца списка. |
| Значение |Текст, отображаемый в верхней части второго столбца списка. |

Горизонтальными линиями представлено относительное значение числового столбца.

| Настройка | Description (Описание) |
|:--- |:--- |
| **Общие сведения** | |
| Имя группы |Текст, отображаемый в верхней строке плитки. |

| **Первая диаграмма** | |
| Условные обозначения |Текст, отображаемый под выноской для первого ряда. |
| Цвет |Цвет, используемый для столбцов в первом ряду. |
| Запрос |Запрос на выполнение для первого ряда.  Счетчик количества записей за каждый временной интервал будет представлен в виде столбцов диаграммы. |
| Операция |Операция, которую необходимо выполнить со свойством значения, чтобы получить одно итоговое значение для выноски.<br><br>– Average: среднее значение для всех записей.<br>– Count: количество всех записей, возвращенных запросом.<br>– First Sample: значение из первого интервала, представленного на графике.<br>– Last Sample: значение из последнего интервала, представленного на графике.<br>– Max: максимальное значение среди всех интервалов, представленных на графике. |
| Запрос |Запрос на выполнение для графика.  Первое свойство должно быть текстовым значением, а второе — числовым.  Обычно это запрос, использующий ключевое слово **measure** для суммирования результатов.  Если запрос использует ключевое слово **interval**, тогда для оси Х графика будет использоваться этот временной интервал.  Если запрос не содержит ключевое слово **interval**, тогда для оси Х будут использоваться ежечасные интервалы. |
| **График** |**> Ось Y** |
| Использовать логарифмическую шкалу |Выберите для отображения логарифмической шкалы для оси Y. |
| Units |Укажите единицы измерения для значений, возвращенных запросом.

# <a name="system-update-assessment-solution-in-log-analytics"></a>Решение "Оценка обновлений системы" в Log Analytics

Решение для обновлений системы в службе Log Analytics позволяет применять отсутствующие обновления к серверам в инфраструктуре. После установки решения в OMS на странице **Обзор** с помощью плитки **Оценка обновлений системы** можно просмотреть обновления, отсутствующие на отслеживаемых серверах.

Сведения об отсутствующих обновлениях отображаются на панели мониторинга **Обновления** . На панели мониторинга **Обновления** можно работать с пропущенными обновлениями и разработать план для всех серверов, которым требуются обновления.

## <a name="installing-and-configuring-the-solution"></a>Установка и настройка решения
Для установки и настройки решений используйте указанные ниже данные.

- Решение "Оценка обновлений системы" необходимо добавить в рабочую область OMS, как описано в статье [Добавление решений Log Analytics из каталога решений](log-analytics-add-solutions.md).  Дополнительная настройка не требуется.

### <a name="to-work-with-updates"></a>Работа с обновлениями

1. На странице **Обзор** щелкните плитку **Оценка обновлений системы**.  
   Плитка  ![Оценка обновлений системы](./media/log-analytics-system-update/sys-update-tile.png)
2. На панели мониторинга **Обновления** просмотрите категории обновлений.  
    ![Панель мониторинга "Обновления"](./media/log-analytics-system-update/sys-updates02.png)
3. Прокрутите страницу вправо до колонки **Критические обновления или обновления для системы безопасности Windows**, а затем в разделе **Классификация** щелкните **Обновления для системы безопасности**.  
    ![Обновления для системы безопасности](./media/log-analytics-system-update/sys-updates03.png)
4. На странице поиска по журналам отображается различная информация об обновлениях для системы безопасности, которые не найдены на серверах в вашей инфраструктуре. Щелкните **Список**, чтобы просмотреть подробные сведения об обновлениях.  
    ![Результаты поиска, список](./media/log-analytics-system-update/sys-updates04.png)
5. На странице поиска по журналам представлена подробная информация о каждом обновлении. Щелкните **Представление** рядом с номером KBID, чтобы открыть соответствующую статью на сайте поддержки Майкрософт.  
    ![Результаты поиска по журналам, представление базы знаний](./media/log-analytics-system-update/sys-updates05.png)
6. В новой вкладке веб-браузера откроется страница поддержки Майкрософт, посвященная этому обновлению. Изучите сведения об отсутствующем обновлении.  
    ![Веб-страница службы технической поддержки Майкрософт](./media/log-analytics-system-update/sys-updates06.png)
7. Используя полученные сведения, вы можете установить отсутствующее обновление вручную или выполнить остальные шаги и автоматически применить обновление.
8. Чтобы установить обновление автоматически, вернитесь на панель мониторинга **Обновление** и в разделе **Запуски обновлений** выберите **Щелкните, чтобы запланировать запуск обновления**.  
    ![Запуски обновлений, вкладка "Запланированные"](./media/log-analytics-system-update/sys-updates07.png)
9. На странице **Запуски обновлений** на вкладке **Запланированные** щелкните **Добавить**, чтобы создать новый запуск обновления.  
    ![Вкладка "Запланированные", кнопка "Добавить"](./media/log-analytics-system-update/sys-updates08.png)
10. На странице **Новый запуск обновления** введите имя для запуска обновления, добавьте нужные компьютеры или группы компьютеров, задайте расписание и нажмите кнопку **Сохранить**.  
    ![Новый запуск обновления](./media/log-analytics-system-update/sys-updates09.png)
11. Вкладка **Запланированные** на странице **Запуски обновления** теперь содержит созданный запуск обновления.  
    ![Вкладка "Запланированные"](./media/log-analytics-system-update/sys-updates10.png)
12. Когда обновление запустится, на вкладке **Выполняются** появится соответствующая информация.  
    ![Вкладка "Выполняются"](./media/log-analytics-system-update/sys-updates11.png)
13. После завершения обновления на вкладке **Завершены** отобразятся сведения о его состоянии.
14. Если обновления были установлены в ходе запуска обновления, число обновлений в колонке **Критические обновления или обновления для системы безопасности Windows** сократится.  
    ![Колонка "Критические обновления или обновления для системы безопасности Windows", число обновлений сократилось](./media/log-analytics-system-update/sys-updates12.png)

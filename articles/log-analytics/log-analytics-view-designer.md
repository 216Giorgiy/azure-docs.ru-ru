# <a name="use-view-designer-to-create-custom-views-in-log-analytics"></a>Использование конструктора представлений для создания пользовательских представлений Log Analytics
Конструктор представлений в [Log Analytics](log-analytics-overview.md) позволяет создавать пользовательские представления на портале Azure, содержащие различные визуальные представления данных из рабочей области Log Analytics. Эта статья содержит обзор конструктора представлений и процедур по созданию и изменению пользовательских представлений.

Ниже перечислены другие статьи о конструкторе представлений:

* [Справка по плиткам](log-analytics-view-designer-tiles.md) — справочник по параметрам для каждой плитки, доступной в пользовательских представлениях.
* [Log Analytics View Designer visualization part reference](log-analytics-view-designer-parts.md) (Справка по элементам визуализации в конструкторе представлений Log Analytics). Справочник по параметрам всех плиток, которые можно использовать при создании пользовательских представлений.

>[!NOTE]
> Если ваша рабочая область переведена на [новый язык запросов Log Analytics](log-analytics-log-search-upgrade.md), запросы во всех представлениях должны создаваться на [новом языке запросов](https://go.microsoft.com/fwlink/?linkid=856078).  Все представления, созданные до обновления рабочей области, будут автоматически преобразованы.

## <a name="concepts"></a>Основные понятия
Представления отображаются на странице **Обзор** рабочей области Log Analytics на портале Azure.  Плитки для каждого пользовательского представления будут отображаться в алфавитном порядке вместе с плитками для решений, которые установлены в той же рабочей области.

![Страница "Обзор"](media/log-analytics-view-designer/overview-page.png)

В следующей таблице перечислены элементы, которые входят в представления, созданные с помощью конструктора представлений.

| Часть | ОПИСАНИЕ |
|:--- |:--- |
| Плитка |Отображается на странице "Обзор" рабочей области Log Analytics.  Содержит визуальный обзор сведений, содержащихся в пользовательском представлении.  На плитках различных типов записи визуализируются по-разному.  Щелкните плитку, чтобы открыть пользовательское представление. |
| Пользовательское представление |Отображается при нажатии плитки.  Содержит один или несколько элементов визуализации. |
| Элемент визуализации |Визуальное отображение данных из рабочей области Log Analytics на основе результатов одного или нескольких [поисков по журналам](log-analytics-log-searches.md).  Обычно элемент содержит заголовок с обобщенным визуальным представлением данных и список лучших результатов.  Различные типы элементов обеспечивают разную визуализацию записей из рабочей области Log Analytics.  Щелкните содержимое элемента, чтобы выполнить поиск по журналам и увидеть подробные результаты. |


## <a name="work-with-an-existing-view"></a>Работа с имеющимся представлением
При открытии представления, которое было создано с помощью конструктора представлений, будет доступно меню с параметрами, приведенными в таблице ниже.

![Меню "Обзор"](media/log-analytics-view-designer/overview-menu.png)


| Параметр | ОПИСАНИЕ |
|:--|:--|
| Обновить   | Обновление представления с помощью самых актуальных данных. | 
| Аналитика | Перейдите на [портал расширенной аналитики](log-analytics-log-search-portals.md#advanced-analytics-portal), чтобы проанализировать данные с помощью запросов поиска по журналам. (log-Analytics-log-Search-portals.md#Advanced-Analytics-Portal). |
| Фильтр    | Установка фильтра времени для данных, включенных в представление. |
| Редактирование      | Открытие представления в конструкторе представлений для изменения его содержимого и конфигурации.   |
| Клонировать     | Создание нового представления и его открытие в конструкторе представлений.  Для нового представления используется такое же имя, как и у исходного, с добавлением в конце слова "Copy" (Копия). |


## <a name="create-a-new-view"></a>Создание представления
Создайте представление в **конструкторе представлений**, щелкнув плитку "Конструктор представлений" на странице "Обзор" рабочей области Log Analytics на портале Azure.

![Плитка "Конструктор представлений"](media/log-analytics-view-designer/view-designer-tile.png)


## <a name="working-with-view-designer"></a>Работа с конструктором представлений
Конструктор представлений используется для создания новых представлений и редактирования существующих представлений.  

В окне конструктора есть три области.  Область **Конструктор** содержит создаваемое или изменяемое пользовательское представление.  Добавить плитки или элементы в область **Конструктор** можно из области **Элемент управления**.  В области **свойств** будут отображаться свойства плитки или выбранного элемента.

![«Просмотреть конструктор»](media/log-analytics-view-designer/view-designer-screenshot.png)

### <a name="configure-view-tile"></a>Настройка плитки представления
В пользовательском представлении может быть только одна плитка.  Выберите вкладку **Плитка** в области **управления**, чтобы просмотреть текущую плитку представления или выбрать другую.  В области **свойств** отображаются свойства для текущей плитки.  Настройте свойства плитки, используя подробные сведения из [справки по плиткам](log-analytics-view-designer-tiles.md), а затем сохраните изменения, нажав кнопку **Применить**.

### <a name="configure-visualization-parts"></a>Настройка элементов визуализации
Представление может содержать неограниченное число элементов визуализации.  Выберите вкладку **Представление**, а затем элемент визуализации, который хотите добавить в представление.  В области **свойств** будут отображаться свойства выбранного элемента.  Настройте свойства представления, используя подробные сведения из [справки по элементам визуализации](log-analytics-view-designer-parts.md), а затем сохраните изменения, нажав кнопку **Применить**.

Представления могут содержать только одну строку элементов визуализации.  Чтобы изменить порядок размещения элементов в представлении, перетащите нужные элементы на новое место.

Вы можете удалить из представления любой элемент визуализации, щелкнув кнопку **X** в правом верхнем углу этого элемента.


### <a name="menu-options"></a>Пункты меню
При работе с представлением в режиме редактирования доступны пункты меню, приведенные в следующей таблице.

![Меню "Правка"](media/log-analytics-view-designer/edit-menu.png)

| Параметр | ОПИСАНИЕ |
|:--|:--|
| Сохранить        | Сохранение изменений и закрытие представления. |
| Отмена      | Отмена изменений и закрытие представления. |
| Удалить представление | Удаление представления. |
| экспорт.      | Экспорт представления в [шаблон Resource Manager](../azure-resource-manager/resource-group-authoring-templates.md), который можно импортировать в другую рабочую область.  У файла будет такое же имя, как и у исходного представления, и расширение *omsview*. |
| Импорт      | Импорт *OMSVIEW*-файла, ранее экспортированного из другой рабочей области.  При этом перезаписывается конфигурация существующего представления. |
| Клонировать       | Создание нового представления и его открытие в конструкторе представлений.  Для нового представления используется такое же имя, как и у исходного, с добавлением в конце слова "Copy" (Копия). |
| Опубликовать     | Экспорт представления в JSON-файл, который может быть вставлен в [решение по управлению](../operations-management-suite/operations-management-suite-solutions-resources-views.md).  У файла будет такое же имя, как и у исходного представления, и расширение *JSON*. Также создается второй файл с расширением *RESJSON*, содержащий значения для ресурсов, определенных в JSON-файле.

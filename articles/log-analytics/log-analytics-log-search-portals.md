# <a name="viewing-and-analyzing-data-in-log-analytics"></a>Просмотр и анализ данных в Log Analytics
На портале Azure доступны две возможности анализа данных, которые хранятся в Log Analytics, и создания запросов на ad-hoc-анализ. Запросы, создаваемые с помощью этих порталов, могут использоваться для других функций, таких как предупреждения и панели мониторинга.

## <a name="log-analytics-page-preview"></a>Страница Log Analytics (предварительная версия)
Откройте страницу Log Analytics из раздела **Журналы (предварительная версия)** в меню Log Analytics. Это новый интерфейс для работы с данными журнала и создания запросов. В статье [Начало работы с порталом аналитики](query-language/get-started-analytics-portal.md) можно ознакомиться с общими сведениями об этом портале и проверить его возможности.

Страница Log Analytics предоставляет некоторые улучшения интерфейса [Поиска по журналам](#log-search).

* Несколько вкладок. Создание отдельных вкладок для работы с несколькими запросами.
* Расширенная визуализация. Разнообразие параметров для работы с диаграммами.
* Улучшенная технология Intellisense и автозаполнение языка.
* Выделение синтаксиса. Повышает удобство чтения запросов. 
* Обозреватель запросов. Предоставление доступа к сохраненным запросам и функциям.
* Представление схемы. Просмотр структуры данных для помощи в написании запросов.
* Общий доступ. Создание ссылок на запросы или их закрепление на любой общей панели мониторинга Azure.
* Смарт-аналитика. Определение пиков на диаграммах и быстрый анализ их причины.
* Выбор столбцов. Сортировка и группирование столбцов в результатах запросов.

> [!NOTE]
> Страница Log Analytics имеет те же функции, что и портал расширенной аналитики, который является внешним инструментом за пределами портала Azure. Портал расширенной аналитики по-прежнему доступен, но ссылки и другие справочные материалы к нему заменяются этой новой страницей.

![Портал расширенной аналитики](media/log-analytics-log-search-portals/advanced-analytics-portal.png)


### <a name="firewall-requirements"></a>Требования к брандмауэру
Ваш браузер запрашивает доступ к следующим адресам, чтобы получить доступ к странице Log Analytics и порталу расширенной аналитики.  Если браузер обращается к порталу Azure через брандмауэр, необходимо разрешить доступ для этих адресов.

| URI | IP-адрес | порты; |
|:---|:---|:---|
| portal.loganalytics.io | Динамический | 80, 443 |
| api.loganalytics.io    | Динамический | 80, 443 |
| docs.loganalytics.io   | Динамический | 80, 443 |


## <a name="log-search"></a>Поиск по журналам
Откройте страницу поиска по журналу из раздела **Журналы** в меню Log Analytics или из **Log Analytics** в меню Azure Monitor. Это подходит для анализа данных журнала с помощью простых запросов. Предоставляется много возможностей для редактирования запросов, даже если вы не полностью разбираетесь в языке запросов.  Обзор этих функций вы найдете в статье в [Создание поисков по журналам в Azure Log Analytics с помощью портала поиска по журналам](log-analytics-log-search-log-search-portal.md).

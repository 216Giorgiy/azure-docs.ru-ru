#Использование соединителя SugarCRM в приложении логики#

Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока. Соединитель SugarCRM позволяет создавать и изменять различные сущности, такие как учетные записи, интересы, контакты и т. д. Ниже приведены типичные сценарии интеграции, включающие использование SugarCRM.

- Синхронизация учетной записи между SugarCRM и системами ERP, такими как SAP

- Синхронизация учетной записи, контактов и интересов между Marketo и SugarCRM

- Поток от заказа до получения оплаты из SugarCRM в системы ERP


В рамках параметров пакета соединителя пользователь может задать сущности, которыми может управлять соединитель, и действия; входные и выходные параметры заполняются динамически. 

##Действия соединителя SugarCRM##
Ниже перечислены различные действия, доступные в соединителе SugarCRM.
 
- Создать модуль - это действие используется для создания новой записи для модуля SugarCRM, например учетных записей, интересов, контактов.

- Обновить модуль - это действие используется для обновления существующей записи для модуля SugarCRM.

- Удалить модуль - действие используется для удаления существующей записи для модуля SugarCRM.

- Список модулей - действие используется для составления перечня отфильтрованных записей. Если не указан запрос, возвращаются все записи.

- Получить модуль - действие используется для получения одной записи из указанного модуля.

- Получить число записей - действие используется для получения числа записей в модуле, который соответствует запросу. Если не указан запрос, возвращается общее количество записей в модуле. 

- Проверить модуль на повторения - действие используется для проверки повторяющихся записей в модуле.

*Примечание*: Дополнительные сведения о поддерживаемых аргументах в запросе см. в документации по API REST SugarCRM.
   
##Создание приложения API соединителя SugarCRM##
1.	Откройте Azure Marketplace с помощью кнопки "+ СОЗДАТЬ" в нижнем правом углу портала Azure.
2.	Перейдите в раздел "Мобильные и веб-приложения > Приложения API" и выполните поиск SugarCRM.

##Создание приложения логики##
Построим простое приложение логики, которое создает учетную запись в SugarCRM и обновляет сведения об адресе выставления счетов в этой учетной записи.

1.	Войдите на портал Azure и последовательно выберите 'Создать -> Мобильные и веб-приложения -> Приложение логики.'

	![][1]

2.	На странице 'Создание приложения логики' введите необходимые сведения, такие как имя, план обслуживания приложения и расположение.

	![][2]

3.	Нажмите 'Триггеры и действия', откроется экран редактора приложения логики. Установите флажок 'Запускать эту логику вручную', что означает, что это приложение логики может вызываться только вручную.


5.	Разверните 'Приложения API в этой группе ресурсов' в коллекции, чтобы просмотреть все доступные приложения API. Выберите в коллекции элемент 'SugarCRM', и 'Соединитель SugarCRM' будет добавлен в поток.

	![][3]

6.	Выберите действие 'Создать учетную запись'. Отобразятся входные параметры.

	![][4]

12.	Укажите имя как  'Учетная запись Майкрософт' и нажмите кнопку ✓. 

	![][5]

13.	В разделе 'Последние использовавшиеся' в коллекции выберите 'Соединитель SugarCRM'. Будет добавлено новое действие SugarCRM.

14.	Выберите 'Обновить учетную запись' в списке действий. Появятся входные параметры действия 'Обновить учетную запись'.

	![][6]

15.	Щелкните знак '+' рядом с полем 'Идентификатор записи' и выберите значение идентификатора в выходных данных действия 'Создать учетную запись'. 

	![][7]

16.	Укажите адрес выставления счетов и нажмите кнопку ✓.

	![][8]

17. Нажмите кнопку "ОК" на экране редактора приложения логики и щелкните  'Создать'. Создание займет приблизительно 30 секунд.

18. Найдите только что созданное приложение логики и нажмите кнопку  'Запустить', чтобы инициировать запуск.

19. Можно проверить, что новая учетная запись под именем  'Учетная запись Майкрософт' создана в учетной записи SugarCRM и в эту учетную запись также добавлен адрес выставления счетов.

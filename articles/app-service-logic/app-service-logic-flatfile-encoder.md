# Кодировщик неструктурированных файлов BizTalk

Используйте соединитель кодировщика неструктурированных файлов BizTalk для взаимодействия данных неструктурированных файлов (таких как Excel и CSV) и данных XML-файлов. Он может преобразовывать экземпляр неструктурированного файла в XML-файл и наоборот.

##Использование кодировщика неструктурированных файлов BizTalk
Чтобы использовать кодировщик BizTalk, необходимо сначала создать экземпляр приложения API кодировщика неструктурированных файлов BizTalk. Это можно сделать либо в процессе, при создании приложения логики, либо выбрав приложение API кодировщика неструктурированных файлов BizTalk на сайте Azure Marketplace. Ниже приведены действия по созданию приложения API кодировщика неструктурированных файлов BizTalk из Azure Marketplace: 1. Войдите на портал Azure (http://portal.azure.com). 2. Выберите "Создать" > "Marketplace" > "Все". 3. В поле поиска введите "кодировщик неструктурированных файлов BizTalk". 4. В списке результатов выберите кодировщик неструктурированных файлов BizTalk. 5. Нажмите кнопку "Создать", а затем укажите имя и другие необходимые сведения. 6. Нажмите кнопку "Создать". Вы будете перенаправлены на начальную страницу, где можно просмотреть ход выполнения процесса создания. Это может занять некоторое время. По завершении процесса вы получите уведомление.

###Настройка кодировщика неструктурированных файлов BizTalk
Созданное приложение API можно запустить непосредственно с начальной страницы портала Azure или из области конструктора при создании приложения логики.

Чтобы запустить приложение с начальной страницы Azure, найдите его, введя имя, которое было задано кодировщику неструктурированных файлов BizTalk при его создании. Для этого выполните следующие действия. 1. Введите имя кодировщика неструктурированных файлов BizTalk в поле поиска на портале Auzre и найдите его. 2. Затем в списке результатов выберите кодировщик неструктурированных файлов BizTalk. Откроется колонка приложения API, где можно настроить приложение API кодировщика неструктурированных файлов BizTalk. Чтобы начать настройку, необходимо добавить схему,выполнив следующие действия. 1. Выберите компонент "Схемы" ![Часть «Схемы» кодировщика неструктурированных файлов BizTalk][2]. 2. Затем выберите "Добавить новую" в колонке "Схемы", которая открывает ![Список действий кодировщика неструктурированных файлов BizTalk][7]. 3. Выберите один из трех параметров для предоставления схемы. Это параметры "ОТПРАВИТЬ НОВУЮ СХЕМУ", "СОЗДАТЬ ИЗ JSON" и "СОЗДАТЬ ИЗ НЕСТРУКТУРИРОВАННОГО ФАЙЛА" ![Список действий кодировщика неструктурированных файлов BizTalk][8]. 4. Выполните действия по указанию схемы в зависимости от выбора, сделанного на предыдущем шаге. Затем вы увидите, что схема отправлена: ![Список действий кодировщика неструктурированных файлов BizTalk][9]

###Использование кодировщика неструктурированных файлов BizTalk в рабочей области конструирования
Теперь, когда вы настроили кодировщик неструктурированных файлов Biztalk, его можно использовать в приложении логики. Чтобы начать работу, создайте новое приложение логики или запустите существующее, которое было создано заранее. Затем выполните следующие действия. 1. В карте "Запуск логики" щелкните "Выполнить эту логику вручную". 2. Выберите созданное ранее приложение API кодировщика неструктурированных файлов BizTalk в коллекции. (Созданный кодировщик неструктурированных файлов BizTalk можно найти в списке приложений API в правой части экрана.) 3. Щелкните жирную стрелку вправо. Имеются два доступных действия ("Неструктурированный файл в XML-файл" и "XML-файл в неструктурированный файл"): ![Список действий кодировщика неструктурированных файлов BizTalk][1] ![Список действий кодировщика неструктурированных файлов BizTalk][4].

В зависимости от выбранного действия выполните шаги, описанные ниже.

####Неструктурированный файл в XML-файл

![Список действий кодировщика неструктурированных файлов BizTalk][5]

Параметр|Тип|Описание параметра
---|---|---
Неструктурированный файл|string|Содержимое входного неструктурированного файла
Имя схемы|string|Имя схемы, которая представляет входной неструктурированный файл
Имя корневого узла|string|Имя корневого узла схемы неструктурированного файла


Действие возвращает выходные данные в виде строки — Output Xml. Выходной XML-файл содержит XML-представление содержимого входного неструктурированного файла.

####XML-файл в неструктурированный файл

![Список действий кодировщика неструктурированных файлов BizTalk][6]

Параметр|Тип|Описание параметра
---|---|---
Входной XML-файл|string|Содержимое входного XML-файла

Действие возвращает выходные данные в виде строки — Flat File. Выходной файл содержит представление неструктурированного файла содержимого входного XML-файла.

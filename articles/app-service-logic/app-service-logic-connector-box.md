# Использование соединителя Box в приложении логики



Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока. Возможны сценарии, в которых необходимо работать с Box, что позволяет вам безопасно обмениваться данными с любыми пользователями, даже если они находятся за пределами вашего брандмауэра.



Приложение коллекции Box предоставляет действия в качестве механизмов взаимодействия с Box.



1. **Действия.** Действия позволяют выполнять стандартные действия в учетной записи Box, указанной в конфигурации приложения логики. Ниже перечислены действия, которые могут выполняться в учетной записи Box с помощью соединителя Box.

	а. *Список файлов.* Эта операция возвращает сведения о всех файлах в папке. Ниже приведен список параметров, необходимых для этого действия.

	  </tr>

	  </tr>
	</table>

	>[AZURE.NOTE]Содержимое файлов не возвращается.



    b. *Получить файл.* Эта операция извлекает файл, включая его содержимое и свойства. Ниже приведен список параметров, необходимых для этого действия.

	  <tr>
	    <td>Путь к файлу</td>

	  </tr>
	  <tr>
	    <td>Тип файла</td>
	    <td>Указывает, является ли файл текстовым или двоичным.</td>
	    <td>Нет</td>
	  </tr>
	</table>
	>[AZURE.NOTE]Эта операция не удаляет файл после его чтения.



    c. Отправить файл. Как можно предположить из названия, это действие отправляет файл в учетную запись Box. Если файл уже существует, то он не будет перезаписан и возникнет ошибка. Ниже приведен список параметров, необходимых для этого действия.

## Создание соединителя Box для приложения логики

Чтобы использовать соединитель Box, необходимо сначала создать экземпляр приложения API соединителя Box. Вы можете сделать это в конструкторе приложения логики или за его пределами. Создать экземпляр за пределами конструктора можно следующим образом.

1. Откройте Azure Marketplace с домашней страницы портала Azure.

2. В разделе "Все" выполните поиск по запросу "Соединитель Box".

3. Настройте соединитель Box и нажмите кнопку "Создать".

	![][1]

4. После выполнения этих действий вы можете создать приложение логики в той же группе ресурсов для использования соединителя Box.


## Использование соединителя Box в приложении логики

После создания приложения API можно использовать соединитель Box как действие для приложения логики. Для этого необходимо выполнить следующие действия.


1. Создайте новое приложение логики и выберите ту же группу ресурсов, которую имеет соединитель Box.

2. Откройте раздел «Триггеры и действия», чтобы открыть конструктор приложений логики и настроить поток. Соединитель Box должен появиться в разделе «Последние использовавшиеся» в коллекции с правой стороны. Выберите его.

3. Если соединитель Box, выбранный при запуске приложения логики, действует как триггер, с помощью этого соединителя можно выполнять другие действия в учетной записи Box. Обратите внимание, что на момент написания этой статьи соединитель Box не содержит триггеров.

4. Первым шагом должна быть проверка подлинности и авторизация приложений логики для выполнения операций от вашего имени. Чтобы запустить авторизацию, нажмите кнопку «Авторизовать» в соединителе Box.

	![][2]

5. При нажатии кнопки «Авторизовать» открывается диалоговое окно проверки подлинности Box. Предоставьте сведения для входа учетной записи Box, в которой вы хотите выполнять операции.

	![][3]

6. Предоставьте приложениям логики доступ к своей учетной записи для выполнения операций от вашего имени.

	![][4]

7. Отображается список действий, в котором можно выбрать соответствующую операцию для выполнения.

	![][5]

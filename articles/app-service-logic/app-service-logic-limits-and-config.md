# <a name="logic-app-limits-and-configuration"></a>Ограничения и настройка приложений логики

Ниже приведены сведения о текущих ограничениях и сведения о конфигурации приложений логики Azure.

## <a name="limits"></a>Ограничения

### <a name="http-request-limits"></a>Ограничения HTTP-запросов

Ниже представлены ограничения для одного вызова HTTP-запроса и/или вызова соединителя.

#### <a name="timeout"></a>Время ожидания

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Истекло время ожидания запроса|90 секунд|При необходимости возможна компенсация с помощью [асинхронной схемы](app-service-logic-create-api-app.md) или [цикла Until](app-service-logic-loops-and-scopes.md).|

#### <a name="message-size"></a>Размер сообщения

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Размер сообщения|50 MB|Некоторые соединители и API могут не поддерживать размер 50 МБ. |
|Оценочное ограничение выражения|131 072 символов|`@concat()`, `@base64()`, `string` не могут быть длиннее|

#### <a name="retry-policy"></a>Политика повтора

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Количество повторных попыток.|4.|Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|
|Максимальный интервал повторения|1 час|Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|
|Минимальный интервал повторения|20 с|Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|

### <a name="run-duration-and-retention"></a>Длительность выполнения и удержания

Ниже представлены ограничения для единого выполнения приложения логики.

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Длительность выполнения|90 дней||
|Хранение периода удержания|90 дней|Считается с начала выполнения|
|Минимальный интервал повторения|1 с|| 15 секунд для приложений логики с планом службы приложений
|Максимальный интервал повторения|500 дней||


### <a name="looping-and-debatching-limits"></a>Ограничения зацикливания и индивидуальной обработки

Ниже представлены ограничения для единого выполнения приложения логики.

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Элементы ForEach|5 000|При необходимости можно использовать [действие запроса](../connectors/connectors-native-query.md) для фильтрации больших массивов|
|Максимальное количество итераций|5 000||
|Элементы SplitOn|5 000||
|Параллелизм ForEach|20|Можно настроить последовательность foreach, добавив `"operationOptions": "Sequential"` в действие `foreach`|


### <a name="throughput-limits"></a>Ограничения пропускной способности

Ниже представлены ограничения для одного экземпляра приложения логики. 

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Триггеров в секунду|100|При необходимости можно распределить рабочие процессы между несколькими приложениями|

### <a name="definition-limits"></a>Ограничение определений

Ниже представлены ограничения для единого определения приложения логики.

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Действий для каждого рабочего процесса|250|Это ограничение можно расширить, добавив вложенные рабочие процессы|
|Допустимая глубина вложенных действий|5|Это ограничение можно расширить, добавив вложенные рабочие процессы|
|Количество последовательностей в регионе на подписку|1000||
|Триггеров для каждого рабочего процесса|10||
|Максимальное число символов в выражении|8192||
|Максимальный размер `trackedProperties` в символах|16 000|
|`action`/`trigger` |80||
|`description` |256||
|`parameters` limit|50||
|`outputs` limit|10||

### <a name="integration-account-limits"></a>Ограничения учетной записи интеграции

Ниже приведены ограничения для артефактов, добавленных в учетную запись интеграции.

|Имя|Ограничение|Примечания|
|----|----|----|
|Схема|8 МБ|Вы можете использовать [универсальный код ресурса (URI) большого двоичного объекта](app-service-logic-enterprise-integration-schemas.md) для передачи файлов, размер которых превышает 2 МБ |
|Сопоставление (XSLT-файл)|2 МБ| |

### <a name="b2b-protocols-as2-x12-edifact-message-size"></a>Размер сообщения протоколов B2B (AS2, X12, EDIFACT)

Ниже приведены ограничения для протоколов B2B.

|Имя|Ограничение|Примечания|
|----|----|----|
|AS2|50 МБ|Применимо для декодирования и кодирования|
|X12|50 МБ|Применимо для декодирования и кодирования|
|EDIFACT|50 МБ|Применимо для декодирования и кодирования|

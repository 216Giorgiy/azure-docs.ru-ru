# Приступая к работе с соединителем Slack: добавление в приложение логики
Подключитесь к каналам Slack и публикуйте сообщения для своей команды. Соединители можно использовать в приложениях логики в рамках «рабочего процесса» для выполнения различных задач. При использовании соединителя Slack в рабочем процессе можно реализовать разнообразные сценарии с использованием других соединителей. Например, можно использовать в рабочем процессе [соединитель Facebook](app-service-logic-connector-facebook.md), чтобы опубликовать сообщение в канале Slack.

## Триггеры и действия
*Триггеры* — это события, которые происходят. Например, когда обновляется заказ или добавляется новый клиент. *Действие* — это результат триггера. Например, при обновлении заказа менеджеру по продажам отправляется оповещение. Либо при добавлении нового клиента ему отправляется приветственное письмо.

Соединитель Slack в приложении логики можно использовать как действие, он поддерживает данные в формате JSON и XML. В настоящее время для соединителя Slack не существует никаких триггеров.

Соединитель Slack предоставляет следующие триггеры и действия.

триггеры; | Действия
--- | ---
None | Опубликовать сообщение

## Создать соединитель Slack
Соединитель можно создать в приложении логики или непосредственно в Azure Marketplace. Создание соединителя в Marketplace

1. На начальной панели Azure выберите **Marketplace**.
2. Выберите **Приложения API** и найдите «Соединитель Slack».
3. Введите имя, план службы приложений и другие свойства: 
<br/> 
![][1] 

4. Щелкните **Создать**.

## Использование соединителя в качестве действия в приложении логики

> [AZURE.IMPORTANT]Соединители и приложения логики всегда следует создавать в одной и той же группе ресурсов.

После создания соединителя Slack его можно добавить как действие в приложение логики:

1.	В приложении логики откройте **Триггеры и действия**.  
	[Создание нового приложения логики](app-service-logic-create-a-logic-app.md)

2.	Соединитель Slack отображается в коллекции с правой стороны:
<br/>
![][2]

3.	Выберите созданный вами соединитель Slack, чтобы автоматически добавить его в приложение логики.
4.	Выберите **Авторизовать**. Войдите в учетную запись Slack. На последнем этапе создания будет предложено предоставить соединителю разрешение на доступ к учетной записи Slack. Выберите **Разрешить**: 
<br/> 
![][3]  
![][4]  
![][5]  
![][6]  
	
5.	Теперь можно использовать соединитель Slack в потоке. Действие «Опубликовать сообщение» доступно: 
<br/> 
![][7]


Давайте рассмотрим процесс «Опубликовать сообщение». Это действие можно использовать для отправки сообщения в любой канал Slack:
 
![][8]

Настройте входные свойства для действия «Опубликовать сообщение»:

Свойство | Описание
--- | ---
текст | Введите текст сообщения для публикации.
Имя канала | Введите канал Slack, в котором публикуется это сообщение. Если не ввести канал, сообщение будет отправлено в #general.
Дополнительные свойства | **Имя пользователя программы-робота**: имя программы-робота, которую необходимо использовать для этого сообщения. Сообщение будет опубликовано от имени «Программа-робот», если этот параметр не указан.<p><p>**URL-адрес значка**: URL-адрес изображения, используемого в качестве значка для этого сообщения.<p><p>**Значок Emoji**: значок Emoji, используемый в качестве значка для этого сообщения. Это свойство переопределяет свойство «URL-адрес значка».


Соединитель Slack имеет доступные интерфейсы REST API, поэтому его можно использовать за пределами приложения логики. Откройте соединитель Slack и выберите **Определение API**:  

![][9]


## Дополнительные возможности соединителя
После создания соединителя его можно добавить в рабочий бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

Справку по REST API Swagger см. в статье [Справочник по соединителям и приложениям API](http://go.microsoft.com/fwlink/p/?LinkId=529766).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. статью [Управление встроенными приложениями API и соединителями, а также их мониторинг](app-service-logic-monitor-your-connectors.md).

# Соединитель Facebook

Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока.

- Триггер соединителя Facebook будет извлекать новую запись во временной шкале пользователя или новую запись на странице. При извлечении нового твита он инициирует новый экземпляр потока и передает данные, полученные в запросе, в этот поток для обработки.
- Действия соединителя Facebook позволяют публиковать записи, публиковать фотографии и выполнять другие задачи. Эти действия получают ответ и делают его доступным для использования другими действиями в потоке.

## Триггеры и действия

триггеры; | Действия
--- | ---
<ul><li>Новая запись на временной шкале пользователя</li><li>Новая запись на странице</li></ul> | <ul><li>Опубликовать запись</li><li>Опубликовать фотографию</li></ul>



## Создание соединителя Facebook для приложения логики
Чтобы использовать соединитель Facebook, необходимо сначала создать экземпляр приложения API «Соединитель Facebook». Это можно сделать следующим образом.

1. На начальной панели Azure выберите **Marketplace**.
2. Выполните поиск по запросу «Соединитель Facebook», выберите его, а затем нажмите **Создать**.
3. Введите имя, план службы приложений и другие свойства. ![][1]
4.	Нажмите кнопку **Создать**.


## Использование соединителя Facebook в приложении логики
После создания приложения API вы можете использовать этот соединитель Facebook в качестве триггера или действия для своего приложения логики. Для этого необходимо выполнить следующие действия.

1.	В приложении логики перейдите в раздел **Триггеры и действия**, чтобы открыть конструктор приложений логики и настроить поток: ![][3]
2.	Соединитель Facebook будет указан в коллекции: ![][4]
3. Выберите соединитель Facebook, чтобы автоматически добавить его в конструктор. Выберите **Авторизовать**, введите свои учетные данные, а затем выберите **Разрешить**: ![][5] ![][6] ![][7] ![][8]
4.	Выберите триггер: ![][9]

Теперь вы можете использовать записи, извлеченные из триггера Facebook, в других действиях. В представленном ниже потоке все новые записи, опубликованные на пользовательской временной шкале Facebook, также будут опубликованы и на пользовательской временной шкале Twitter: ![][10]

Аналогичным образом можно создавать потоки с помощью действий соединителя Facebook. Представленный ниже поток будет извлекать новое сообщение, опубликованное в группе Yammer, а затем публиковать его на управляемой пользователем странице Facebook: ![][11]

> [AZURE.TIP]Чтобы получить идентификатор страницы Facebook или группы Yammer, найдите числовой код в URL-адресе.

## Дополнительные возможности соединителя
После создания соединителя его можно добавить в рабочий бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

Справку по API REST Swagger см. в статье [Справочные материалы по соединителям и приложениям API](http://go.microsoft.com/fwlink/p/?LinkId=529766).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. статью [Управление встроенными приложениями API и соединителями, а также их мониторинг](app-service-logic-monitor-your-connectors.md).

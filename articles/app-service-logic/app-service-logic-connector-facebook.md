# Использование соединителя Facebook в приложении логики #

Приложения логики могут запускаться на основе различных источников данных и предлагают соединители для получения и обработки данных в рамках потока.

Соединитель Facebook позволяет извлекать новую запись во временной шкале пользователя, новую запись на странице, публиковать запись, публиковать фотографию, а также выполнять другие действия в учетной записи Facebook.

- Триггер соединителя Facebook будет извлекать новую запись во временной шкале пользователя или новую запись на странице. При извлечении нового твита он инициирует новый экземпляр потока и передает данные, полученные в запросе, в этот поток для обработки. 
- Действия соединителя Facebook позволяют публиковать записи, публиковать фотографии и выполнять другие задачи. Эти действия получают ответ и делают его доступным для использования другими действиями в потоке.

## Создание соединителя Facebook для приложения логики ##
Чтобы использовать соединитель Facebook, необходимо сначала создать экземпляр приложения API «Соединитель Facebook». Это можно сделать следующим образом.

1.	Откройте Azure Marketplace, нажав кнопку «+ СОЗДАТЬ» в нижнем левом углу портала Azure.
2.	Перейдите в раздел «Мобильные и веб-приложения > Приложения API» и выполните поиск соединителя Facebook.
3.	Настройте соединитель Facebook следующим образом.
 
	![][1]
4.	Нажмите кнопку «ОК», чтобы создать прослушиватель.
5.	После создания экземпляра приложения API можно создать приложение логики в той же группе ресурсов, которая будет использовать соединитель Facebook. 
	- Экземпляр приложения API «Соединитель Facebook» можно также создать из приложения логики. 
	- Откройте редактор приложения логики и щелкните соединитель Facebook, доступный в коллекции с правой стороны.
	- Будет создан экземпляр приложения API «Соединитель Facebook» в той же группе ресурсов, в которой создано приложение логики.


## Использование соединителя Facebook в приложении логики ##
После создания приложения API вы можете использовать этот соединитель Facebook в качестве триггера или действия для своего приложения логики. Для этого необходимо выполнить следующие действия.

1.	Создайте новое приложение логики и выберите ту же группу ресурсов, что и для соединителя Facebook.
 
	![][2]
2.	Открыть раздел «Триггеры и действия», чтобы открыть конструктор приложения логики и настроить поток. 
 
	![][3]
3.	Соединитель Facebook должен отображаться в разделе «Последние использовавшиеся» в коллекции с правой стороны. Выберите его.
 
	![][4]
4.	Вы можете открыть приложение API «Соединитель Facebook» в редакторе, щелкнув элемент «Соединитель Facebook» в разделе «Последние использовавшиеся» коллекции с правой стороны. Нажмите кнопку «Авторизовать». Укажите свои учетные данные Facebook.
  
	![][5]
5.	Разрешите «Приложения логики Azure AppService». 

	![][6] ![][7] ![][8]     
6.	Выберите триггер.
 
	![][9]
7.	Теперь вы можете использовать записи, извлеченные из триггера Facebook, в других действиях. В потоке ниже каждый раз при публикации новой записи во временной шкале Facebook пользователя та же запись будет опубликована во временной шкале Twitter пользователя.
 
	![][10]
8.	Аналогичным образом можно создавать потоки с помощью действий соединителя Facebook. Следующий поток будет извлекать новое сообщение, опубликованное в группе Yammer, и публиковать это же сообщение на странице Facebook, управляемой пользователем.
 
	![][11]

**Совет.** Чтобы получить идентификатор страницы Facebook или Yammer найдите идентификатор группы числовой код в URL-адресе.

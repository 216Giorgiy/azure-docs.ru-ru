# Сведения о безопасности OAUTH в соединителях SaaS
Многие соединители программного обеспечения как услуги (SaaS), например Facebook, Twitter, DropBox и т. п., требуют авторизации пользователей по протоколу OAUTH. При использовании таких соединителей SaaS в приложениях логики предоставляется упрощенный пользовательский интерфейс, где следует нажать кнопку «Авторизовать» в конструкторе приложений логики. При нажатии кнопки **Авторизовать** предлагается выполнить вход (если этого еще не сделано) и предоставить согласие на подключение к службе SaaS от вашего имени. После предоставления согласия и авторизации приложения логики смогут получать доступ к таким службам SaaS.

## Создание собственного приложения SaaS
Такой упрощенный процесс взаимодействия возможен благодаря тому, что мы ранее создали и зарегистрировали наше приложение в упомянутых службах SaaS. В некоторых случаях вам может потребоваться зарегистрировать и использовать собственное приложение. Это необходимо, если соединители SaaS требуется задействовать в пользовательских приложениях, например при [развертывании приложения API с соединителем SaaS](app-service-api-connnect-your-app-to-saas-connector.md). В данном примере используется соединитель DropBox, однако процесс аналогичен для всех соединителей, зависящих от OAUTH.

Даже в контексте приложений логики вы можете использовать собственное приложение вместо того, которое мы предоставляем по умолчанию. Если не удается подключиться с помощью кнопки «Авторизовать», вы можете попробовать создать собственное приложение. Ниже перечислены необходимые шаги для соединителя Twitter.

1. Откройте соединитель Twitter на портале предварительной версии Azure. Последовательно выберите пункты **Обзор** > **Приложения API**. Выберите соединитель Twitter: ![][1]

2. Выберите **Параметры** > **Проверка подлинности**: ![][2]

3. Скопируйте значение **URI перенаправления**: ![][3]

4. Выберите [Twitter](http://apps.twitter.com), затем **Создать новое приложение**. В свойстве **URL-адрес обратного вызова** вставьте значение **URI перенаправления**, скопированное из соединителя Twitter: ![][4]
5. После того как приложение Twitter будет создано, выберите **Маркеры ключей и доступа**. Скопируйте эти значения.
6. Открыв настройки проверки подлинности соединителя Twitter, вставьте эти значения в свойства **Идентификатор клиента** и **Секрет клиента**: ![][5]  
7. Сохраните настройки соединителя.  

Теперь соединитель можно будет использовать в приложениях логики. Если вы используете этот соединитель из приложений логики, он будет использовать ваше приложение, а не установленное по умолчанию.

> [AZURE.NOTE]Если вы ранее авторизовали какое-либо приложение, возможно, потребуется авторизовать его повторно.

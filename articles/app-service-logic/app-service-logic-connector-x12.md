#Соединитель BizTalk X12
Служба Microsoft Azure X12 позволяет получать и отправлять сообщения в соответствии с протоколом Х12 при взаимодействии между компаниями. Х12 также часто называют ASC Х12 или комитетом аккредитованных стандартов Х12. Протокол широко используется в разных отраслях.

##Предварительные требования
- Модуль TPM для приложения API. Перед созданием соединителя X12 необходимо создать [соединитель BizTalk TPM][1]
- База данных SQL Azure. Каждому приложению API B2B требуется своя собственная база данных SQL Azure.
- Служебная шина Azure. Необязательный модуль, который используется только для пакетной обработки.

##Использование соединителя BizTalk X12
Чтобы использовать соединитель BizTalk X12, необходимо сначала создать соответствующий экземпляр приложения API для соединителя BizTalk X12. Это можно сделать как в процессе создания приложения логики, так и путем выбора приложения API соединителя BizTalk X12 в Azure Marketplace.

##Настройка соединителя BizTalk X12
Торговые партнеры — это сущности, участвующие во взаимодействиях между компаниями. Когда два партнера устанавливают связь, это называется соглашением. Определенное соглашение строится на основе взаимодействия, которого желают достичь эти два партнера, и зависит от конкретного протокола или транспорта.

Комментарии к действиям по созданию соглашения торговых партнеров см. [здесь][2].

##Использование соединителя X12 в области конструктора приложений логики
Соединитель X12 можно использовать как в качестве триггера, так и в качестве действия.

###Триггер
- Запустите конструктор потока для приложений логики Azure.
- Щелкните соединитель X12 в области справа.

	![Настройки триггера][3]
- Щелкните ->

	![Параметры триггера][4]
- Соединитель BizTalk X12 предоставляет один триггер. Выберите *Выпуск пакета*

	![Входные данные для выпуска пакета][5]
- Этот триггер не имеет входных данных. Щелкните ->

	![Выпуск пакета настроен][6]
- Соединитель возвращает в качестве выходных данных полезные данные X12, идентификатор соглашения, а также указатель, является ли сообщение пакетированным.

###Действие
- Щелкните соединитель X12 в области справа.

	![Настройки действия][7]
- Щелкните ->

	![Список действий][8]
- Соединитель X12 поддерживает множество действий. Выберите *Кодирование*

	![Кодирование входных данных][9]
- Введите входные данные для действия и настройте их

	![Кодирование настроено][10]

## Дополнительные возможности соединителя
После создания соединителя его можно добавить в бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. раздел [Управление приложениями API и соединителем, а также их отслеживание](../app-service-api/app-service-api-manage-in-portal.md).

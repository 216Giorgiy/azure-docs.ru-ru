# Использование соединителя Twilio в приложении логики #

Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока.

Соединитель Twilio позволяет отправлять и получать SMS в учетной записи Twilio. С его помощью вы также можете получать номера телефонов и сведения об использовании.

## Создание соединителя Twilio для приложения логики ##
Чтобы использовать соединитель Twilio, необходимо сначала создать экземпляр приложения API соединителя Twilio. Это можно сделать следующим образом.

1.	Откройте Azure Marketplace, нажав кнопку "+ СОЗДАТЬ" в нижнем левом углу портала Azure.
2.	Перейдите в раздел "Приложения API" и найдите "Соединитель Twilio".
3.	Настройте соединитель Twilio следующим образом.
 
	![][1]
	- **Расположение** - выберите географическое расположение, где нужно развернуть соединитель.
	- **Подписка** - выберите подписку, в которой необходимо создать соединитель.
	- **Группа ресурсов** - выберите или создайте группу ресурсов, где должен находиться соединитель.
	- **План веб-хостинга** - выберите или создайте план веб-хостинга.
	- **Ценовая категория** - выберите ценовую категорию для соединителя.
	- **Имя** - задайте имя для соединителя Twilio.
	- **Параметры пакета**.
		- **ИД безопасности учетной записи** 
		- уникальный идентификатор учетной записи. Идентификатор безопасности для учетной записи можно получить с помощью <https://www.twilio.com/user/account/settings> 
		-**маркера авторизации**, связанного с учетной записью. Маркер авторизации учетной записи можно получить с помощью <https://www.twilio.com/user/account/settings>.


4.	Нажмите кнопку "Создать". Будет создан новый соединитель Twilio.
5.	После создания экземпляра приложения API можно создать приложение логики в той же группе ресурсов для использования соединителя Twilio.

## Использование соединителя Twilio в приложении логики ##
После создания приложения API можно использовать соединитель Twilio как действие для приложения логики. Для этого необходимо выполнить следующие действия.

1.	Создайте новое приложение логики и выберите ту же группу ресурсов, которую имеет соединитель Twilio.

	![][2]
2.	Открыть раздел "Триггеры и действия", чтобы открыть конструктор приложения логики и настроить поток.

	![][3]
3.	Соединитель Twilio отображается в разделе "Приложения API в этой группе ресурсов" в коллекции с правой стороны.

	![][4]
4. Приложение API соединителя Twilio можно удалить в редакторе, щелкнув "Соединитель Twilio".

5.	Теперь можно использовать соединитель Twilio в потоке. Действие "Отправить сообщение" в потоке можно использовать для отправки сообщения. Настройте входные свойства для действия "Отправить сообщение" следующим образом.
	- **С номера телефона** — введите номер телефона Twilio, включенный для этого типа отправляемого сообщения. Только телефонные номера или сокращенные коды, приобретенные у Twilio, будут работать с этим соединителем.
	- **На номер телефона** — номер телефона абонента. Принятый формат: +, код страны, номер телефона, например +16175551212. Если опустить знак "+", Twilio будет использовать код страны, введенный в поле "С номера телефона".
	- **Текст** — текст сообщения, которое необходимо отправить.

	![][5]
	![][6]
## Дополнительные возможности соединителя
После создания соединителя его можно добавить в рабочий бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

Справку по API REST Swagger см. в статье [Справочные материалы по соединителям и приложениям API](http://go.microsoft.com/fwlink/p/?LinkId=529766).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. статью [Управление встроенными приложениями API и соединителями, а также их мониторинг](app-service-logic-monitor-your-connectors.md).

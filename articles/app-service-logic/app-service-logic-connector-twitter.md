# Использование соединителя Twitter в приложении логики #

Приложения логики могут запускаться на основе различных источников данных и предлагают соединители для получения и обработки данных в рамках потока.

Соединители Twitter позволяют вам публиковать твиты и получать твиты из временной шкалы, от друзей и подписчиков вашей учетной записи Twitter.

- Триггер соединителя Twitter будет извлекать новые твиты, связанные с указанным ключевым словом. При извлечении нового твита он инициирует новый экземпляр потока и передает данные, полученные в запросе, в этот поток для обработки. 
- Действия соединителя Twitter позволяют выполнять отправку твитов, поиск твитов, ретвиты, получать временную шкалу пользователя и т. д. Эти действия получают ответ и делают его доступным для использования другими действиями в потоке.

Чтобы использовать соединитель Twitter, необходимо сначала создать экземпляр приложения API «Соединитель Twitter». Это можно сделать следующим образом.

1. Откройте Azure Marketplace, нажав кнопку «+ СОЗДАТЬ» в нижнем левом углу портала Azure.
1. Перейдите в раздел «Приложения API» и выполните поиск соединителя Twitter.

## Использование соединителя Twitter в приложении логики ##
После создания приложения API вы можете использовать этот соединитель Twitter в качестве триггера или действия для своего приложения логики. Для этого необходимо выполнить следующие действия.

1.	Создайте новое приложение логики и выберите ту же группу ресурсов, которую имеет соединитель Twitter.
 	
	![][2]
2.	Открыть раздел «Триггеры и действия», чтобы открыть конструктор приложения логики и настроить поток. 
 	
	![][3]
3.	Соединитель Twitter должен отображаться в разделе «Последние использовавшиеся» в коллекции с правой стороны. Выберите его.
 
	![][4]
4.	Вы можете открыть приложение API «Соединитель Twitter» в редакторе, щелкнув элемент «Соединитель Twitter» в разделе «Последние использовавшиеся» коллекции с правой стороны. Нажмите кнопку «Авторизовать». Укажите свои учетные данные Twitter. Нажмите кнопку «Авторизовать приложение».
 
	![][5]
6.	Теперь вы можете использовать соединитель Twitter в потоке. Теперь вы можете использовать записи, извлеченные из триггера Twitter, в других действиях в потоке.
 
	![][6]
7.	Аналогичным образом вы можете использовать действия Twitter в потоке. Выберите действие Twitter и настройте входные данные для этого действия.

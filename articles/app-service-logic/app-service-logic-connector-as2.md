#Соединитель AS2
Соединитель AS2 от Microsoft Azure может получать и отправлять сообщения с использованием транспортного протокола AS2 во взаимодействиях между компаниями. AS2 расшифровывается как постановление о пределах применения инструкции 2. Данные безопасно и надежно транспортируются через Интернет. Безопасность обеспечивается с помощью цифровых сертификатов и шифрования.

##Предварительные требования
- Модуль TPM для приложения API. Перед созданием соединителя AS2 необходимо создать [соединитель BizTalk TPM][1]
- База данных SQL Azure. Каждому приложению API B2B требуется своя собственная база данных SQL Azure.
- Контейнер хранилища больших двоичных объектов Azure. Хранит свойства сообщений, если включена архивация AS2. Если архивация сообщений AS2 не требуется, то контейнер хранилища не нужен. 

##Использование соединителя AS2
Чтобы использовать соединитель AS2, сначала нужно создать экземпляр приложения API соединителя AS2. Это можно сделать как при создании приложения логики, так и путем выбора приложения API соединителя AS2 в Azure Marketplace.

##Настройка соединителя AS2
Торговые партнеры — это сущности, участвующие во взаимодействиях между компаниями. Когда два партнера устанавливают связь, это называется соглашением. Определенное соглашение строится на основе взаимодействия, которого желают достичь эти два партнера, и зависит от конкретного протокола или транспорта.

Комментарии к действиям по созданию соглашения торговых партнеров см. [здесь][2].

##Использование соединителя AS2 в области конструктора приложений логики
Соединитель AS2 можно использовать как в качестве триггера, так и в качестве действия.

###Триггер
- Запустите конструктор потока для приложений логики Azure.
- Щелкните соединитель AS2 в области справа.

	![Настройки триггера][3]
- Щелкните ->

	![Параметры триггера][4]
- Соединитель AS2 предоставляет один триггер. Выберите *Получение и декодирование*

	![Получение и декодирование входных данных][5]
- Этот триггер не имеет входных данных. Щелкните ->

	![Получение и декодирование настроены][6]
- Соединитель возвращает полезные данные AS2 и свои метаданные.

###Действие
- Щелкните соединитель AS2 в области справа.

	![Настройки действия][7]
- Щелкните ->

	![Список действий][8]
- Соединитель AS2 поддерживает только одно действие. Выберите *Кодирование и отправка*

	![Кодирование и отправка входных данных][9]
- Введите входные данные для действия и настройте их

	![Кодирование и отправка настроены][10]

## Дополнительные возможности соединителя
После создания соединителя его можно добавить в бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. раздел [Управление приложениями API и соединителем, а также их отслеживание](../app-service-api/app-service-api-manage-in-portal.md).

# Приступая к работе с соединителем Chatter: добавление в приложение логики 
Подключитесь к Chatter, чтобы опубликовать сообщение или найти канал. Например, можно выполнить поиск по каналу Chatter и при обнаружении конкретного результата опубликовать это сообщение из Chatter в группу продаж.

Соединитель Chatter можно добавить в рабочий процесс компании и обрабатывать данные в рамках этого процесса в приложении логики.

## Триггеры и действия

Триггер запускает новый экземпляр на основе определенного события, например при поступлении нового сообщения Chatter. Действие — это результат. Например, после получения нового сообщения Chatter оно публикуется в другой группе Chatter или на сайте другой социальной сети, например Facebook или Twitter.

Соединитель Chatter в приложении логики можно использовать как триггер или действие. Он поддерживает данные в форматах JSON и XML. В соединителе Chatter доступны следующие триггеры и действия.

триггеры; | Действия
--- | ---
Новое сообщение | <ul><li>Опубликовать сообщение</li><li>Поиск</li></ul>


## Создание соединителя Chatter для приложения логики
Соединитель можно создать в приложении логики или непосредственно в Azure Marketplace. Создание соединителя в Marketplace

1. На начальной панели Azure выберите **Marketplace**.
2. Выполните поиск по запросу «Соединитель Chatter», выберите его и затем выберите **Создать**.
3. Введите имя, план службы приложений и другие свойства.  
	![][1]
	- **Расположение** — выберите географическое расположение, где планируется развернуть соединитель.
	- **Подписка** — выберите подписку, в которой нужно создать соединитель.
	- **Группа ресурсов** — выберите или создайте группу ресурсов, в которой следует разместить соединитель.
	- **План размещения веб-сайтов** — выберите или создайте план размещения веб-сайтов.
	- **Ценовая категория** — выберите ценовую категорию для соединителя.
	- **Имя** — задайте имя для соединителя Chatter

4. Нажмите кнопку **Создать**.


## Использование соединителя Chatter в приложении логики
После создания приложения API соединитель Chatter можно будет использовать как триггер или действие в приложении логики. Для этого:

1. В приложении логики перейдите в раздел **Триггеры и действия**, чтобы открыть конструктор приложений логики и настроить поток.

2. Соединитель Chatter будет указан в коллекции: 
	![][4]
3. Выберите соединитель Chatter, чтобы автоматически добавить его в конструктор. Выберите **Авторизовать**, введите свои учетные данные, а затем выберите **Разрешить**:  
	![][5]

	![][6]

	![][7]

Теперь можно использовать соединитель Chatter в потоке. Теперь вы можете использовать сообщение, извлеченное из триггера Chatter ("Новое сообщение"), в других действиях в потоке. Настройте входные свойства для триггера Chatter следующим образом.

**Идентификатор группы** — введите идентификатор группы, из которой следует получить новое сообщение. Если идентификатор группы не указан, новое сообщение будет получено из канала пользователя:
	![][8]
	![][9]


Аналогично можно использовать действие Chatter в потоке для отправки сообщения, выбрав действие "Отправить сообщение". Настройте входные свойства для действия «Отправить сообщение» следующим образом:  
	- **Текст сообщения** - текстовое содержимое публикуемого сообщения.
	- **Идентификатор группы** - укажите идентификатор группы, в которой следует опубликовать новое сообщение. Если идентификатор группы не указан, сообщение будет опубликовано на канале пользователя.
	- 	**Имя файла** - имя файла, который будет прикреплен к сообщению.
	- 	**Данные содержимого** - данные содержимого вложения.
	- 	**Тип содержимого** - тип содержимого вложения.
	- 	**Кодирование при передаче содержимого** - кодирование при передаче содержимого вложения («none» или «base64»).
	- 	**Упоминания** - массив имен пользователей, которые будут отмечены тегами в данном сообщении.
	- 	**Хэш-теги** - массив хэш-тегов, которые будут опубликованы вместе с сообщением.

## Дополнительные возможности соединителя
После создания соединителя его можно добавить в рабочий бизнес-процесс с помощью приложения логики. См. раздел [Что такое приложения логики?](app-service-logic-what-are-logic-apps.md).

>[AZURE.NOTE]Если вы хотите начать работу с приложениями логики Azure до создания учетной записи Azure, перейдите на веб-сайт [пробного использования приложений логики](https://tryappservice.azure.com/?appservice=logic). На этом сайте вы сможете быстро создать кратковременное приложение логики начального уровня в службе приложений. Никаких кредитных карт и обязательств.

Справку по API REST Swagger см. в статье [Справочные материалы по соединителям и приложениям API](http://go.microsoft.com/fwlink/p/?LinkId=529766).

Можно также просматривать статистику производительности и управлять безопасностью соединителя. См. статью [Управление встроенными приложениями API и соединителями, а также их мониторинг](app-service-logic-monitor-your-connectors.md).

# Использование соединителя Chatter в приложении логики #

Приложения логики могут запускаться на основе разных источников данных и предлагать соединители для получения и обработки данных в рамках потока.

Соединитель Chatter позволяет подключаться к Chatter и выполнять различные действия, такие как отправка сообщения, поиск и триггер для получения новых сообщений.

## Создание соединителя Chatter для приложения логики ##
Чтобы использовать соединитель Chatter, необходимо сначала создать экземпляр приложения API соединителя Chatter. Это можно сделать следующим образом.

1.	Откройте Azure Marketplace, нажав кнопку "+ СОЗДАТЬ" в нижнем левом углу портала Azure.
2.	Перейдите в раздел "Мобильные и веб-приложения > Приложения API" и выполните поиск соединителя Chatter.
3.	Настройте соединитель Chatter следующим образом.

	![][1]
	- **Расположение** - выберите географическое расположение, где нужно развернуть соединитель.
	- **Подписка** - выберите подписку, в которой необходимо создать соединитель.
	- **Группа ресурсов** - выберите или создайте группу ресурсов, где должен находиться соединитель.
	- **План веб-хостинга** - выберите или создайте план веб-хостинга.
	- **Ценовая категория** - выберите ценовую категорию для соединителя.
	- **Имя** - задайте имя для этого соединителя Chatter.

4.	Нажмите кнопку "Создать". Будет создан новый соединитель Chatter.
5.	После создания экземпляра приложения API в той же группе ресурсов для использования соединителя Chatter можно будет создать приложение логики.

## Использование соединителя Chatter в приложении логики ##
После создания приложения API соединитель Chatter можно будет использовать как триггер или действие для приложения логики. Для этого необходимо выполнить следующие действия.

1.	Создайте новое приложение логики и выберите ту же группу ресурсов, которую имеет соединитель Chatter.

	![][2]
2.	Открыть раздел "Триггеры и действия", чтобы открыть конструктор приложений логики и настроить поток.

3.	Соединитель Chatter отображается в разделе "Приложения API в этой группе ресурсов" в коллекции с правой стороны.

	![][4]
4. В редакторе можно удалить приложение API соединителя Chatter, щелкнув "Соединитель Chatter". Нажмите кнопку "Авторизовать". Укажите свои учетные данные. Нажмите кнопку "Разрешить".

	![][5]
	![][6]
	![][7]
5.	Теперь можно использовать соединитель Chatter в потоке. Теперь вы можете использовать сообщение, извлеченное из триггера Chatter ("Новое сообщение"), в других действиях в потоке. Настройте входные свойства для триггера Chatter следующим образом.
	- **Идентификатор группы** — укажите идентификатор группы, из которого должно быть получено новое сообщение. Если не указан идентификатор группы, новое сообщение будет получено из канала пользователя.

  ![][8]
  ![][9]

6. Аналогично можно использовать действие Chatter в потоке для отправки сообщения, выбрав действие "Отправить сообщение". Настройте входные свойства для действия "Отправить сообщение" следующим образом.
	- **Текст сообщения** — текстовое содержимое отправляемого сообщения.
	- **Идентификатор группы** — укажите идентификатор группы, в которую следует добавлять новые сообщения. Если не указан идентификатор группы, сообщение будет отправлено в канал пользователя.
	- 	**Имя файла** — имя файла, вложенного в сообщение.
	- 	**Данные содержимого** — данные содержимого вложения.
	- 	**Тип содержимого** — тип содержимого вложения.
	- 	**Кодирование при передаче содержимого** — кодирование при передаче содержимого вложения ("нет" или Base64).
	- 	**Упоминания** — массив имен пользователей, отмеченных в этом сообщении.
	- 	**Хэш-теги** — массив хэш-тегов, отправляемых вместе с сообщением.

# <a id="managemediaservicesorigins"></a>Управление конечными точками потоковой передачи в учетной записи служб мультимедиа

Службы мультимедиа позволяют добавить множество конечных точек потоковой передачи в учетную запись и настроить их. С каждой учетной записью служб мультимедиа связана по крайней мере одна конечная точка потоковой передачи, именуемая **default** (по умолчанию).

>[AZURE.NOTE] Конечные точки потоковой передачи раньше назывались источниками. 


## Добавление и удаление конечных точек потоковой передачи 

1. На [портале управления](https://manage.windowsazure.com/) щелкните **Службы мультимедиа**. Затем щелкните имя службы мультимедиа.
2. Откройте страницу КОНЕЧНЫЕ ТОЧКИ ПОТОКОВОЙ ПЕРЕДАЧИ. 
3. В нижней части страницы нажмите кнопку ДОБАВИТЬ или УДАЛИТЬ. Обратите внимание, что конечную точку потоковой передачи по умолчанию удалить нельзя. 
4. Нажмите кнопку ЗАПУСК для запуска конечной точки. 
5. Щелкните имя конечной точки потоковой передачи, чтобы настроить ее.   

	![Origin page][origin-page]

## Масштабирование конечной точки потоковой передачи

Зарезервированные единицы потоковой передачи по требованию предоставляют как выделенные мощности исходящего трафика, которые могут быть приобретены с шагом 200 Мбит/с, так и  дополнительные функциональные возможности, которые в настоящее время включают [возможности динамической упаковки](http://go.microsoft.com/fwlink/?LinkId=276874). По умолчанию потоковая передача по требованию настроена в модели общего экземпляра, для которой серверные ресурсы (например, вычислительные ресурсы, мощности исходящего трафика и т. д.) совместно используются со всеми другими пользователями. Чтобы повысить производительность потоковой передачи по требованию, рекомендуется приобретать единицы потоковой передачи. 

Чтобы изменить число единиц потоковой передачи, выполните следующие действия:

1. Чтобы указать число единиц потоковой передачи, перейдите на вкладку "МАСШТАБ" и переместите ползунок **зарезервированной емкости**.

	![Scale page](./media/media-services-how-to-scale/media-services-origin-scale.png)

4. Нажмите кнопку СОХРАНИТЬ, чтобы сохранить изменения.

	Выделение новых единиц потоковой передачи занимает около 20 минут. 

	 
	>[AZURE.NOTE] Сейчас переход от любого положительного значения единиц потоковой передачи к нулевому может привести к отключению потоковой передачи по требованию на период до одного часа.

	>[AZURE.NOTE] Для расчета затрат используется наибольшее число единиц, указанных для 24-часового периода. Дополнительные сведения о ценах см. в разделе [Сведения о ценах для служб мультимедиа](http://go.microsoft.com/fwlink/?LinkId=275107).
	
## Настройка конечной точки потоковой передачи

На вкладке "Настройка" можно производить настройку, как показано на этом рисунке. Описание полей ниже.

>[AZURE.NOTE] Конфигурация на этой странице будет применена только к конечным точкам потоковой передачи, у которых есть хотя бы один зарезервированный модуль. Для резервирования зарезервированных единиц потоковой передачи по требованию.

![Configure origin][configure-origin]
  

1. Задать максимальный период кэширования, который будет указан в заголовке управления кэшем в ответах HTTP. Это значение не будет переопределять максимальное значение кэша, заданное явным образом в контенте BLOB-объекта.

2. Указать IP-адреса, которым будет разрешено подключение к опубликованной конечной точке потоковой передачи. Если IP-адреса не указаны, можно подключаться с любого IP-адреса.

3. Определить настройку для проверки подлинности заголовка подписи Akamai.

4. Можно указать политику междоменного доступа для клиентов Adobe Flash (дополнительные сведения см. в разделе [Спецификация файла политики междоменного доступа](http://www.adobe.com/devnet/articles/crossdomain_policy_file_spec.html). А также политики доступа клиента для клиентов Microsoft Silverlight (Дополнительные сведения см. в разделе [Доступ к службам за пределами домена](https://msdn.microsoft.com/ru-ru/library/cc197955(v=vs.95).aspx).  

5. Можно также настроить пользовательские имена узлов, нажав кнопку **настройка**. Дополнительную информацию см. в описании свойства **CustomHostNames** в разделе [Конечная точка потоковой передачи](https://msdn.microsoft.com/ru-ru/library/dn783468.aspx).

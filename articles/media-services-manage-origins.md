#<a id="managemediaservicesorigins"></a>Управление конечными точками потоковой передачи в учетной записи служб мультимедиа

Это одна из статей серии [Рабочий процесс для видео по запросу в службах мультимедиа](media-services-video-on-demand-workflow.md) и серии [Рабочий процесс для потоковой трансляции в службах мультимедиа](media-services-live-streaming-workflow.md).  


В службах мультимедиа Microsoft Azure **конечная точка потоковой передачи** - это служба потоковой передачи, которая может доставить содержимое непосредственно в клиентское приложение проигрывателя или в сеть доставки содержимого (CDN) для дальнейшего распространения. Службы мультимедиа также обеспечивают прозрачную интеграцию с Azure CDN. Исходящий поток из службы StreamingEndpoint может быть потоком трансляции или активом видео по запросу в учетной записи служб мультимедиа.  

Кроме того, чтобы справиться с растущими потребностями в пропускной способности, можно контролировать мощности службы конечной точки потоковой передачи, регулируя число единиц масштабирования (которые также называются единицами потоковой передачи). Рекомендуем выделить одну или несколько единиц масштабирования для приложения в рабочей среде. Единицы масштабирования предоставляют как выделенные мощности исходящего трафика, которые могут быть приобретены с шагом 200 Мбит/с, так и дополнительную функциональность, которая в настоящее время включает [возможности динамической упаковки](https://msdn.microsoft.com/library/azure/jj889436.aspx), интеграцию CDN и расширенную конфигурацию.

Плата взимается, только когда служба StreamingEndpoint используется. 

В этом разделе представлен обзор основных возможностей, предоставляемых конечными точками потоковой передачи. Здесь также показано, как использовать портал управления Azure для управления конечными точками потоковой передачи.


##Добавление и удаление конечных точек потоковой передачи 

Вы можете добавить или удалить конечные точки потоковой передачи с помощью пакета SDK для .NET, API-интерфейса REST или портала управления Azure.

Чтобы добавить или удалить конечную точку потоковой передачи с помощью портала, выполните следующие действия.

1. На [портале управления](https://manage.windowsazure.com/) щелкните **Службы мультимедиа**. Затем щелкните имя службы мультимедиа.
2. Откройте страницу **КОНЕЧНЫЕ ТОЧКИ ПОТОКОВОЙ ПЕРЕДАЧИ**. 
3. В нижней части страницы нажмите кнопку ДОБАВИТЬ или УДАЛИТЬ. Обратите внимание, что конечную точку потоковой передачи по умолчанию удалить нельзя. 
4. Нажмите кнопку ЗАПУСК для запуска конечной точки. 
5. Щелкните имя конечной точки потоковой передачи, чтобы настроить ее.   

	![Страница конечной точки потоковой передачи][streaming-endpoint]


По умолчанию можно использовать до двух конечных точек потоковой передачи. Если вам необходимо больше, изучите раздел [Квоты и ограничения](media-services-quotas-and-limitations/).

##<a id="scale_streaming_endpoints"></a>Масштабирование конечной точки потоковой передачи

Единицы потоковой передачи предоставляют как выделенные мощности исходящего трафика, которые могут быть приобретены с шагом 200 Мбит/с, так и дополнительную функциональность, которая в настоящее время включает [возможности динамической упаковки](http://go.microsoft.com/fwlink/?LinkId=276874). По умолчанию потоковая передача настроена в модели общего экземпляра, для которой серверные ресурсы (например, вычислительные ресурсы, мощности исходящего трафика и т. д.) совместно используются со всеми другими пользователями. Чтобы повысить пропускную способность для потоковой передачи, рекомендуем приобрести единицы потоковой передачи. 

Масштабировать конечные точки можно с помощью пакета SDK для .NET, API-интерфейса REST или портала управления Azure.

Чтобы изменить число единиц потоковой передачи с помощью портала, выполните следующие действия.

1. Чтобы указать число единиц потоковой передачи, перейдите на вкладку "МАСШТАБ" и переместите ползунок **зарезервированной емкости**.

	![Страница "Масштаб"](./media/media-services-how-to-scale/media-services-origin-scale.png)

4. Нажмите кнопку СОХРАНИТЬ, чтобы сохранить изменения.

	Выделение новых единиц потоковой передачи занимает около 20 минут. 

	
>[AZURE.NOTE] Сейчас переход от любого положительного значения единиц потоковой передачи к нулевому может привести к отключению потоковой передачи по требованию на период до одного часа.

	>[AZURE.NOTE] Для расчета затрат используется наибольшее число единиц, указанных для 24-часового периода. Сведения о ценах см. в разделе [Сведения о ценах на службы мультимедиа](http://go.microsoft.com/fwlink/?LinkId=275107).
	
##<a id="configure_streaming_endpoints"></a>Настройка конечной точки потоковой передачи

Конечная точка потоковой передачи позволяет настроить следующие свойства при наличии по крайней мере 1 единицы масштабирования: 

- управление доступом;
- настраиваемые имена узлов;
- управление кэшем;
- межсайтовые политики доступа.

Дополнительные сведения об этих свойствах см. в разделе [StreamingEndpoint](https://msdn.microsoft.com/library/azure/dn783468.aspx).

Настроить эти свойства можно с помощью пакета SDK для .NET, API-интерфейса REST или портала управления Azure.

Чтобы изменить число единиц потоковой передачи с помощью портала, выполните следующие действия.

1. Выберите конечную точку потоковой передачи, которую хотите настроить.
1. Перейдите на вкладку "НАСТРОЙКА".
  
Далее представлено краткое описание полей.

![Настройка источника][configure-origin]
  

1. Задать максимальный период кэширования, который будет указан в заголовке управления кэшем в ответах HTTP. Это значение не будет переопределять максимальное значение кэша, заданное явным образом в контенте BLOB-объекта.

2. Указать IP-адреса, которым будет разрешено подключение к опубликованной конечной точке потоковой передачи. Если IP-адреса не указаны, можно подключаться с любого IP-адреса.

3. Определить настройку для проверки подлинности заголовка подписи Akamai.

4. Можно указать политику междоменного доступа для клиентов Adobe Flash (дополнительные сведения см. в разделе [Спецификация файла политики междоменного доступа](http://www.adobe.com/devnet/articles/crossdomain_policy_file_spec.html)), а также политики доступа клиента для клиентов Microsoft Silverlight (дополнительные сведения см. в разделе [Доступ к службам за пределами домена](https://msdn.microsoft.com/library/cc197955(v=vs.95).aspx).  

5. Можно также настроить пользовательские имена узлов, нажав кнопку **Настройка**. Дополнительную информацию см. в описании свойства **CustomHostNames** в разделе [Конечная точка потоковой передачи](https://msdn.microsoft.com/library/dn783468.aspx).  


##<a id="enable_cdn"></a>Включение интеграции Azure CDN

Вы можете включить интеграцию Azure CDN для конечной точки потоковой передачи (по умолчанию она отключена).

Включение интеграции Azure CDN

- У конечной точки потоковой передачи должна быть по крайней мере одна единица потоковой передачи (масштабирования). Если позже вы захотите задать число единиц масштабирования равным 0, вначале потребуется отключить интеграцию CDN. 

- Конечная точка потоковой передачи должна быть остановлена. После включения CDN вы можете запустить конечную точку потоковой передачи. 

Для включения интеграции Azure CDN может потребоваться 90 минут.   

После этого отключаются следующие конфигурации: **настраиваемые имена узлов** и **контроль доступа**.

Не все центры обработки данных поддерживают интеграцию Azure CDN. При использовании портала управления Azure вы увидите параметр **ВКЛЮЧИТЬ CDN**, если он доступен в вашем центре обработки данных. 

![Включение CDN для конечной точки потоковой передачи][streaming-endpoint-enable-cdn]

При использовании пакета SDK .NET или API REST необходимо проверить, доступна ли интеграция Azure CDN в центре обработки данных. 

Для этого выполните следующие действия.

1. Попробуйте задать для свойства CdnEnabled значение true.
1. Проверьте, не возвращается ли код ошибки HTTP 412 (PreconditionFailed) с сообщением "Не удается задать для свойства CdnEnabled конечной точки потоковой передачи значение true, так как компонент CDN недоступен в текущем регионе". 

	При появлении этой ошибки центр обработки данных не поддерживает интеграцию. Попробуйте другой центр обработки данных.

# Заметки о выпуске, посвященные обновлению 1 для устройств серии StorSimple 8000  

## Обзор

В приведенных ниже заметках о выпуске описаны новые функции и критические открытые проблемы в обновлении 1 для устройств серии StorSimple 8000. Здесь также имеется список обновлений программного обеспечения и встроенного ПО StorSimple, включенных в этот выпуск. Это первый основной выпуск после версии выпуска для устройств серии StorSimple 8000, ставшей общедоступной в июле 2014 г.

После установки этого обновления версия программного обеспечения устройства изменится на StorSimple 8000 Series Update 1. Перед развертыванием обновления в решении StorSimple, просмотрите сведения, содержащиеся в заметках о выпуске. Дополнительные сведения см. в разделе [Установка обновления 1 на устройство StorSimple](storsimple-install-update-1.md).

Перед развертыванием обновлений в решении StorSimple просмотрите сведения, содержащиеся в заметках о выпуске.

>[AZURE.IMPORTANT]
> 
- Для установки обновления 1 используйте службу диспетчера StorSimple, а не Windows PowerShell для StorSimple.
- Кроме того, в этом выпуске содержатся обновления встроенного ПО для дисков. Их можно применить только тогда, когда устройство находится в режиме обслуживания. При установке этих обновлений устройство будет находиться в состоянии простоя. Эти обновления можно установить во время планового обслуживания.
- Установка обновления (включая обновления Windows) занимает примерно 5-10 часов. 
- Для новых выпусков обновления могут отображаться не сразу, поскольку развертывание обновлений выполняется поэтапно. Через несколько дней повторите попытку поиска обновлений, так как они станут доступными в ближайшее время.

## Новые возможности в обновлении 1

Это обновление содержит указанные ниже новые функции и улучшения.

- **Миграция с устройств серий 5000–7000 на устройства серии 8000**. В этом выпуске появилась новая функция миграции, с помощью которой пользователи устройств серий 5000–7000 могут перенести свои данные на физическое устройство серии StorSimple 8000 или виртуальное устройство серии 1100. Функция миграции имеет два указанных ниже ключевых преимущества.                                                                  

    - **Обеспечение непрерывности бизнеса** путем миграции существующих данных с устройств серий 5000–7000 на устройства серии 8000.
    - **Улучшенные функции в устройствах серии 8000**, например функция эффективного централизованного управления несколькими устройствами с помощью службы диспетчера StorSimple, оборудование более высокого класса, обновленное встроенное ПО, виртуальные устройства, мобильность данных и запланированное внедрение новых функций в будущем.

    Сведения о том, как выполнить миграцию с устройств серий StorSimple 5000–7000 на устройства серии 8000, см. в [руководстве по миграции](http://www.microsoft.com/download/details.aspx?id=47322).

- **Доступность на портале Azure для государственных учреждений**. Теперь StorSimple доступен на портале Azure для государственных учреждений. См. сведения о том, как [выполнить развертывание устройства StorSimple на портале Azure для государственных учреждений](storsimple-deployment-walkthrough-gov.md).

- **Поддержка других поставщиков облачных служб**. Поддерживаемые поставщики облачных служб: Amazon S3, Amazon S3 с RRS, HP и OpenStack (бета-версия).

- **Обновление до последних версий API службы хранилища**. В этом выпуске для устройства StorSimple используются последние версии API службы хранилища Azure. В устройствах серии StorSimple 8000, работающих под управлением GA, используются версии API службы хранилища Azure, выпущенные до 12 февраля 2012 г. Как сказано в [объявления об удалении некоторых версий службы хранилища](http://azure.microsoft.com/blog/2014/08/04/microsoft-azure-storage-service-version-removal/) от 10 декабря 2015 г., эти API будут считаться устаревшими. Необходимо установить обновление 1 для устройств серии StorSimple 8000 до 9 декабря 2015 г. Если не сделать этого, устройства StorSimple перестанут правильно работать.

- **Поддержка хранилищ, избыточных в пределах зоны (ZRS)**. После обновления до последней версии API службы хранилища, помимо локально избыточных хранилищ (LRS) и геоизбыточных хранилищ (GRS) устройства серии StorSimple 8000 будут поддерживать хранилища, избыточные в пределах зоны (ZRS). Сведения о ZRS см. в [статье о способах обеспечения избыточности в службе хранилища Azure](../storage/storage-redundancy.md).

- **Улучшенные возможности начального развертывания и обновления**. В этом выпуске улучшены процессы установки и обновления. В мастер установки добавлена функция, которая в случае использования неправильных конфигурации сети и параметров брандмауэра сообщает об этом пользователю. Добавлены дополнительные диагностические командлеты, упрощающие устранение неполадок, связанных с сетью, в устройстве. Дополнительные сведения о новых диагностических командлетах, предназначенных для устранения неполадок, см. в [статье о развертывании средств устранения неполадок](storsimple-troubleshoot-deployment.md).

| 8 | Сетевые интерфейсы | В предыдущих версиях устройство StorSimple, настроенное с использованием шлюза, не поддерживающего маршрутизацию, могло перейти в автономный режим. В этом выпуске метрике маршрутизации для интерфейса Data 0 присвоено самое маленькое значение. ## Обновления для физических устройств в обновлении 1

Если применить эти обновления к физическому устройству, версия программного обеспечения изменится на 6.3.9600.17491.

## Обновления SAS-контроллера и встроенного ПО в обновлении 1

Этот выпуск обновляет драйвер и встроенное ПО SAS-контроллера физического устройства. Он также содержит обновления встроенного ПО дисков в устройстве.
 
- Дополнительные сведения об обновлении SAS-контроллера см. в разделе [Обновление 1 для SAS-контроллеров LSI в устройстве Microsoft Azure StorSimple](https://support.microsoft.com/kb/3043005). 

- Дополнительные сведения об обновлении встроенного ПО см. в разделе [Обновление 1 для встроенного ПО в устройстве Microsoft Azure StorSimple](https://support.microsoft.com/kb/3063414).

- Дополнительные сведения об обновлении встроенного ПО см. в разделе [Обновление 1 для встроенного ПО дисков в устройстве Microsoft Azure StorSimple](https://support.microsoft.com/ru-ru/kb/3063416).
 
## Обновления для виртуальных устройств в обновлении 1

Эта версия не содержит никаких обновлений для виртуального устройства. Установка этого обновления не приведет к изменению версии программного обеспечения виртуального устройства.

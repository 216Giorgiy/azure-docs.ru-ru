# Установка обновления 1.2 на устройство StorSimple

## Обзор

В этом учебнике описывается установка обновления 1.2 на устройстве StorSimple, на котором используется программное обеспечение более ранней версии, чем обновление 1. Кроме того, здесь описываются дополнительные шаги по обновлению, которые необходимо выполнить, когда шлюз настроен для сетевого интерфейса устройства StorSimple, отличного от DATA 0.

Обновление 1.2 включает в себя обновления программного обеспечения устройства, драйвера LSI и встроенного ПО диска. Обновления программного обеспечения и драйвера LSI не нарушают работу устройства и могут устанавливаться через классический портал Azure. Обновления встроенного ПО диска нарушают работу устройства и могут устанавливаться только через интерфейс Windows PowerShell на устройстве.

В зависимости от версии, установленной на устройстве, можно определить, будет ли установлено обновление 1.2. Чтобы узнать версию программного обеспечения устройства, просмотрите **сводку** на **панели мониторинга** вашего устройства.

</br>

| Используемая версия программного обеспечения | Что происходит на портале |
|---------------------------------|--------------------------------------------------------------|
| Выпуск GA | Если у вас установлен выпуск GA, не устанавливайте это обновление. Для обновления устройства обратитесь в [службу поддержки Майкрософт](storsimple-contact-microsoft-support.md).|
| Обновление 0.1 | Портал устанавливает обновление 1.2. |
| Обновление 0.2 | Портал устанавливает обновление 1.2. |
| Обновление 0.3 | Портал устанавливает обновление 1.2. |
| Обновление 1 | Это обновление будет недоступно. |
| Обновление 1.1 | Это обновление будет недоступно. |

</br>

> [AZURE.IMPORTANT]

> -  Возможно, вы не сразу увидите обновление 1.2, потому что развертывание обновлений выполняется поэтапно. Через несколько дней повторите попытку поиска обновлений, так как это обновление станет доступным в ближайшее время.
> - В это обновление входит набор предварительных проверок (выполняются вручную и автоматически), которые позволяют определить работоспособность устройства, а именно состояние его оборудования и качество сетевого подключения. Эти проверки выполняются только в том случае, если обновления устанавливаются через классический портал Azure.
> - Рекомендуем устанавливать обновления программного обеспечения и драйвера с помощью классического портала Azure. Устанавливайте обновления через интерфейс Windows PowerShell на устройстве только в том случае, если предварительная проверка шлюза возвращает ошибку на портале. Для установки обновлений может потребоваться 5–10 часов (включая обновления Windows). Установка обновлений в режиме обслуживания должна выполняться только через интерфейс Windows PowerShell на устройстве. Так как обновления в режиме обслуживания нарушают работу устройства, устройство некоторое время будет недоступно.

## Установка обновления 1.2 с помощью классического портала Azure

Выполните следующие действия, чтобы установить [обновление 1.2](storsimple-update1-release-notes.md) на устройство. Эта процедура используется только в том случае, если шлюз настроен для сетевого интерфейса DATA 0 вашего устройства.

[AZURE.INCLUDE [storsimple-install-update2-via-portal](../../includes/storsimple-install-update2-via-portal.md)]

12. Убедитесь, что на устройстве установлено **обновление 1.2 для StorSimple серии 8000 (6.3.9600.17584)**. Также должна измениться **Дата последнего обновления**. Вы также увидите, что доступны обновления режима обслуживания (это сообщение может отображаться до 24 часов после установки обновлений).

    Обновления режима обслуживания являются критическими — они прерывают рабочие процессы устройства и могут быть реализованы только через интерфейс Windows PowerShell устройства.

    ![Страница "Обслуживание"](./media/storsimple-install-update-1/InstallUpdate12_10M.png "Страница "Обслуживание"")

13. Скачайте обновления режима обслуживания, выполнив действия, описанные в разделе [Загрузка исправления](#to-download-hotfixes), чтобы найти и скачать исправление KB3063416, которое устанавливает обновления встроенного ПО диска (другие обновления уже должны быть установлены).

13. Выполните действия, описанные в разделе [Установка и проверка исправления режима обслуживания](#to-install-and-verify-maintenance-mode-hotfixes) для установки обновлений режима обслуживания.

14. На классическом портале Azure перейдите на страницу **Обслуживание**. В нижней части страницы нажмите щелкните **Проверить обновления**, чтобы проверить наличие обновлений Windows, а затем нажмите кнопку **Установить обновления**. После успешной установки всех обновлений операция будет завершена.



## Установка обновления 1.2 на устройстве, шлюз которого не настроен для сетевого интерфейса DATA 0

Эту процедуру следует использовать только в том случае, если при попытке установить обновления через классический портал Azure проверка шлюза возвращает ошибку. Проверка завершается ошибкой, так как ваш шлюз назначен сетевому интерфейсу, отличному от DATA 0, а на устройстве используется более ранняя версия программного обеспечения (до обновления 1). Если на вашем устройстве нет шлюза на сетевом интерфейсе, отличном от DATA 0, вы можете обновить устройство непосредственно на классическом портале Azure. См. раздел [Установка обновления 1.2 с помощью классического портала Azure](#install-update-1.2-via-the-azure-classic-portal).

Данный метод подходит для обновления таких версий программного обеспечения: обновление 0.1, 0.2 и 0.3.


> [AZURE.IMPORTANT]
>
> - Если на вашем устройстве используется выпуск GA, обратитесь в [службу поддержки Майкрософт](storsimple-contact-microsoft-support.md) за помощью в установке обновления.
> - Эту процедуру требуется выполнить только один раз для установки обновления 1.2. Вы можете использовать классический портал Azure для установки всех последующих обновлений.

Если на вашем устройстве используется программное обеспечение версии, предшествующей обновлению 1, а один из шлюзов устройства настроен для сетевого интерфейса, отличного от DATA 0, вы можете установить обновление 1.2 следующими двумя способами.

- **Способ 1**. Загрузите обновление и установите его с помощью командлета `Start-HcsHotfix` в интерфейсе Windows PowerShell устройства. Рекомендуем использовать этот метод. **Не используйте этот метод для установки обновления 1.2, если на устройстве уже установлено обновление 1.0 или 1.1.**

- **Способ 2**. Удалите конфигурацию шлюза и установите обновление непосредственно с классического портала Azure.


По каждому из этих способов в следующих разделах приведены подробные инструкции.

## Способ 1. Установка обновления 1.2 в качестве исправления с помощью Windows PowerShell для StorSimple

Эту процедуру следует выбирать только в том случае, если на устройстве используется обновление 0.1, 0.2 или 0.3 и если проверка шлюза завершилась ошибкой при попытке установить обновления через классический портал Azure. Если вы используете выпуск GA, обратитесь в [службу поддержки Майкрософт](storsimple-contact-microsoft-support.md) за помощью в установке обновления.

Чтобы установить обновление 1.2 как исправление, необходимо скачать и установить следующие исправления.

| Порядок | КБ | Описание | Тип обновления |
|--------|-----------|-------------------------|------------- |
| 1 | KB3063418 | Обновление программного обеспечения | Обычный |
| 2 | KB3043005 | Обновление контроллера SAS LSI | Обычный |
| 3 | KB3063416 | Встроенное ПО диска | Обслуживание |

Перед использованием этой процедуры для установки обновления убедитесь в следующем.

- Оба контроллера устройства подключены к сети.

Выполните описанные ниже действия, чтобы установить обновление 1.2. **Обновление может занять около 2 часов (приблизительно 30 минут для программного обеспечения, 30 минут для драйвера и 45 минут для встроенного ПО диска).**

## Способ 2. Установка обновления 1.2 с помощью классического портала Azure после удаления конфигурации шлюза

Эта процедура применима только к устройствам StorSimple, на которых используется программное обеспечение более ранней версии, чем обновление 1, а один из шлюзов настроен для сетевого интерфейса, отличного от DATA 0. Перед установкой обновления нужно удалить настройки шлюза.

Процедура обновления может занять несколько часов. Если ваши узлы находятся в разных подсетях, удаление конфигурации шлюзов на интерфейсах iSCSI может привести к простою. Рекомендуем настроить интерфейс DATA 0 для трафика iSCSI, чтобы сократить время простоя.

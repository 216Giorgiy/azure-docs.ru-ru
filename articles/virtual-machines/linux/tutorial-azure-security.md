# <a name="monitor-vm-security-with-the-azure-security-center"></a>Мониторинг защиты виртуальных машин с помощью центра безопасности Azure

Центр безопасности Azure помогает следить за конфигурацией ресурсов Azure с точки зрения безопасности. Он также обеспечивает встроенный мониторинг системы безопасности, позволяющий обнаруживать угрозы, которые в противном случае могут оставаться незамеченными. В этом руководстве представлен краткий обзор центра безопасности Azure и описывается, как его использовать с виртуальными машинами Azure.   

## <a name="security-center-overview"></a>Обзор центра безопасности

Центр безопасности Azure помогает выявлять потенциальные проблемы в конфигурации виртуальных машины и целевые угрозы безопасности. Например, он выявляет виртуальные машины без групп безопасности сети, незашифрованные диски и атаки методом подбора по протоколу RDP. Эти сведения отображаются на панели мониторинга центра безопасности Azure в виде удобных диаграмм.

Панель мониторинга центра безопасности Azure можно открыть, щелкнув **Центр безопасности** в левой области навигации на портале Azure. Панель мониторинга содержит общее представление данных работоспособности ресурсов, оповещений системы безопасности и рекомендаций по конфигурации. Здесь можно просмотреть работоспособность системы безопасности среды Azure, узнать количество текущих рекомендаций и изучить текущее состояние оповещений об угрозах. Каждую из этих общих диаграмм можно развернуть, чтобы получить более подробную информацию.

![Панель мониторинга ASC](./media/tutorial-azure-security/asc-dash.png)

Возможности центра безопасности Azure выходят за пределы обнаружения данных, так как он предоставляет рекомендации для выявленных проблем. Например, если виртуальная машина была развернута без группы безопасности сети, будет создана рекомендация, содержащая необходимые действия по исправлению. Эти рекомендации также дают возможность автоматизировать исправления, не покидая контекст центра безопасности Azure.  

![Рекомендации](./media/tutorial-azure-security/recommendations.png)

## <a name="configure-data-collection"></a>Настройка сбора данных

Прежде чем вы сможете просмотреть конфигурации безопасности виртуальных машин, в центре безопасности Azure нужно настроить сбор данных. Для этого следует включить сбор данных и создать учетную запись хранения Azure, чтобы хранить в ней собранные данные. 

1. На панели мониторинга центра безопасности Azure щелкните **Политика безопасности** и выберите свою подписку. 
2. Выберите *Вкл.* для параметра **Сбор данных**.
3. Щелкните **Выберите учетную запись хранения** и создайте учетную запись хранения. По окончании нажмите кнопку **ОК** .
4. Щелкните **Сохранить** в колонке **Политика безопасности**. 

После этого на всех виртуальных машинах будет установлен агент сбора данных центра безопасности Azure, который начнет сбор данных. 

## <a name="configure-security-policy"></a>Настройка политики безопасности

Политика безопасности определяет элементы политики безопасности, для которых собираются данные и предоставляются рекомендации. По умолчанию ресурсы Azure анализируются с использованием всех элементов политики. Отдельные элементы политики можно отключать глобально для всех ресурсов Azure или для группы ресурсов. Это конфигурация дает возможность применять различные политики безопасности к разным наборам ресурсов Azure. Подробные сведения о политиках безопасности центра безопасности Azure см. в разделе [Настройка политик безопасности в центре безопасности Azure](../../security-center/security-center-policies.md). 

Вот как можно настроить политику безопасности для всех ресурсов Azure.

1. На панели мониторинга центра безопасности Azure щелкните **Политика безопасности** и выберите свою подписку. 
2. Щелкните **Prevention policy** (Политика предотвращения).
3. Включите или отключите элементы политики, которые должны применяться ко всем ресурсам Azure.
4. Когда закончите, нажмите кнопку **ОК**.
5. Щелкните **Сохранить** в колонке **Политика безопасности**. 

Чтобы настроить политику для определенной группы ресурсов, выполните те же действия, но в колонке политики безопасности вместо подписки выберите группу ресурсов. При настройке политики в разделе **Наследование** выберите *Уникальный*. Если вы хотите отключить сбор данных для определенной группы ресурсов, это можно также настроить здесь.

В следующем примере для группы ресурсов *myResoureGroup* создается уникальная политика. В этой политике отключены рекомендации для шифрования дисков и брандмауэра веб-приложения.

![Уникальная политика](./media/tutorial-azure-security/unique-policy.png)

## <a name="view-vm-configuration-health"></a>Просмотр состояния работоспособности конфигурации виртуальной машины

После включения сбора данных и настройки политики безопасности центр безопасности Azure начинает отображать оповещения и рекомендации. На развертываемые виртуальные машины устанавливается агент сбора данных, и центр безопасности Azure заполняется данными для этих новых виртуальных машин. Подробные сведения о работоспособности конфигурации виртуальных машин см. в разделе [Защита виртуальных машин в центре безопасности Azure](../../security-center/security-center-virtual-machine-recommendations.md). 

По мере сбора данных сведения о работоспособности ресурсов для каждой виртуальной машины и связанного ресурса Azure объединяются и отображаются на удобной для чтения диаграмме. Чтобы просмотреть состояние работоспособности ресурса, вернитесь на панель мониторинга центра безопасности Azure. В разделе **Resource security health** (Работоспособность системы безопасности ресурсов) щелкните **Вычислить**. Наконец, в колонке **Вычисления** щелкните **Виртуальные машины**. Это представление содержит сводку состояния конфигурации для всех виртуальных машин.

![Вычисление работоспособности](./media/tutorial-azure-security/compute-health.png)

Если выбрать определенную виртуальную машину, отобразятся все рекомендации для нее. Эти рекомендации описываются в следующем разделе данного руководства.

## <a name="remediate-configuration-issues"></a>Исправление проблем конфигурации

После того, как центр безопасности Azure начинает получать данные конфигурации, на основе настроенной политики безопасности создаются рекомендации. Например, если для виртуальной машины не была настроена связанная группа безопасности сети, создается рекомендация для ее создания. Вот как можно просмотреть список всех рекомендаций. 

1. Щелкните **Рекомендации** на панели мониторинга центра безопасности Azure.
3. Выберите конкретную рекомендацию. Откроется колонка со списком всех ресурсов, к которым относится эта рекомендация.
4. Выберите конкретный ресурс, который вас интересует.
5. Следуйте инструкциям по исправлению на экране. 

Во многих случаях центр безопасности Azure предоставляет практические инструкции, позволяющие выполнить рекомендацию, не покидая контекст центра безопасности Azure. Например, в следующем примере обнаружена NSG с правилом входящего трафика без ограничений. В этой рекомендации можно нажать кнопку **Изменить правила для входящего трафика**, чтобы отобразить пользовательский интерфейс, необходимый для изменения правила. 

![Рекомендации](./media/tutorial-azure-security/remediation.png)

После исправления проблем относящиеся к ним рекомендации помечаются как выполненные. 

## <a name="view-detected-threats"></a>Просмотр обнаруженных угроз

Помимо рекомендаций по настройке ресурсов центр безопасности Azure также предоставляет оповещения об обнаружении угроз. Функция оповещения системы безопасности объединяет данные, полученные с каждой виртуальной машины, из сетевых журналов Azure и подключенных партнерских решений, для обнаружения угроз безопасности ресурсов Azure. Подробные сведения о возможностях обнаружения угроз в центре безопасности Azure см. в статье [Возможности обнаружения центра безопасности Azure](../../security-center/security-center-detection-capabilities.md).

Для использования функции оповещения системы безопасности требуется повысить ценовую категорию центра безопасности Azure с *Free* до *Стандартный*. При предоставляется 30-дневная **бесплатная пробная версия**. Вот как можно изменить ценовую категорию.  

1. На панели мониторинга центра безопасности Azure щелкните **Политика безопасности** и выберите свою подписку.
2. Щелкните **Ценовая категория**.
3. Выберите новую ценовую категорию и нажмите кнопку **Выбрать**.
5. Щелкните **Сохранить** в колонке **Политика безопасности**. 

После включения диаграмма оповещений системы безопасности начнет заполняться выявленными угрозами безопасности.

![Оповещения системы безопасности](./media/tutorial-azure-security/security-alerts.png)

Выберите оповещение, чтобы просмотреть такие сведения об угрозе, как ее описание, время обнаружения, число попыток и рекомендуемые действия по исправлению. В этом примере была обнаружена атака методом подбора по протоколу RDP, которая безуспешно предпринималась 294 раза и для которой указано рекомендуемое решение.

![Атака по протоколу RDP](./media/tutorial-azure-security/rdp-attack.png)

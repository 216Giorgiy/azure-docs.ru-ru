#Изменение буквы временного диска Windows на виртуальной машине, созданной на основе классической модели развертывания

[AZURE.INCLUDE [learn-about-deployment-models](../../includes/learn-about-deployment-models-include.md)]В этой статье описывается процесс создания ресурсов с помощью классической модели развертывания.

Если вам нужно использовать диск D для хранения данных, следуйте приведенным указаниям, чтобы использовать другой диск в качестве временного. Никогда не используйте временный диск для хранения данных, которые нужно сохранить.

Прежде чем начать, потребуется подключить диск данных к виртуальной машине, чтобы хранить файл подкачки Windows (pagefile.sys) во время этой процедуры. Если у вас его нет, ознакомьтесь со статьей [Подключение диска данных к виртуальной машине Windows][Attach]. Сведения о том, как узнать, какие диски подключены, см. в разделе "Поиск диска" статьи [Отключение диска от виртуальной машины Windows][Detach].

Если вы хотите использовать существующий диск данных на диске D, убедитесь, что вы также передали соответствующий VHD в учетную запись хранения. Указания см. в этапах 3 и 4 раздела [Создание и передача виртуального жесткого диска Windows Server в Azure][VHD].

> [AZURE.WARNING]Если изменить размер виртуальной машины или выполнить команду "Остановить (освободить)", виртуальная машина может быть перемещена в новую низкоуровневую оболочку. Запланированное или незапланированное событие технического обслуживания также может вызвать такое перемещение. В этом случае временному диску будет назначена первая доступная буква диска. Если вашему приложению требуется именно диск D, убедитесь, что после перемещения файла подкачки создан новый постоянный диск, которому назначена буква D. Azure не будет забирать букву D.

> [AZURE.WARNING]Обратите внимание, что если изменить размер виртуальной машины после явного перемещения файла подкачки, то при загрузке может возникнуть ошибка, если на временном диске новой виртуальной машины не помещается файл подкачки того же размера, что исходная виртуальная машина. Ошибка также может возникнуть, если для временного диска не задана следующая доступная буква диска, в результате чего Windows ссылается на неправильную букву диска в конфигурации файла подкачки, в то время как Azure создает временный диск со следующей доступной буквой диска.

##Изменение буквы диска

1. Войдите в виртуальную машину. Дополнительные сведения см. в статье [Как войти в виртуальную машину под управлением Windows Server][Logon].

2. Переместите файл pagefile.sys с диска D на другой диск.

3. Перезапустите виртуальную машину.

4. Снова войдите и измените букву диска с D на E.

5. На [портале Azure](http://manage.windowsazure.com) подключите существующий или пустой диск данных.

6.	Снова войдите на виртуальную машину, инициализируйте диск и назначьте только что подключенному диску букву D.

7.	Убедитесь, что буква E назначена временному диску.

8.	Переместите файл pagefile.sys с другого диска на диск E.

9.	Перезапустите виртуальную машину.

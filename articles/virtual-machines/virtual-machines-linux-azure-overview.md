# Azure и Linux
Microsoft Azure — это расширяющийся набор интегрированных общедоступных облачных служб, которые включают в себя возможности анализа, работы с виртуальными машинами и базами данных, мобильные средства, веб-технологии, а также возможности работы в сети и хранения данных — все для размещения ваших решений. Microsoft Azure предоставляет масштабируемую платформу вычислений, которая позволяет платить только за те ресурсы, которые вы используете по требованию, без дополнительных затрат на локальное оборудование. Среда Azure готова к использованию, когда вы можете масштабировать свои решения в соответствии с потребностями клиентов.
 
## Виртуальные машины Azure
Виртуальные машины Azure позволяют выполнять гибкое развертывание разных вычислительных решений. Вы можете развернуть практически любую рабочую нагрузку, используя любой язык на практически любой ОС, включая Windows, Linux или пользовательскую (на основе решений наших многочисленных партнеров). По-прежнему не нашли подходящий образ? Не волнуйтесь, вы также можете перенести свои собственные образы из локальных сред.
 
## Начало работы с виртуальной машиной Linux в Microsoft Azure

Виртуальные машины, хранилища и сетевые службы Microsoft Azure позволяют реализовать архитектуру "инфраструктура как услуга" в соответствии с потребностями среды Linux в масштабах Интернета. Чтобы начать работу с виртуальной машиной Linux в Azure, вам понадобится:

1. Бесплатная пробная учетная запись, которую можно получить [здесь](https://azure.microsoft.com/pricing/free-trial/).
2. Интерфейс командной строки Azure для Linux, Mac и Windows. Его можно установить, следуя инструкциям [здесь](../xplat-cli-install.md).
3. Виртуальная машина Linux. Ее можно создать, следуя инструкциям [здесь](virtual-machines-linux-quick-create-cli.md).
4. Дополнительные сведения о виртуальных машинах Linux и Azure, включая информацию о [соглашении об уровне обслуживания (SLA)](https://azure.microsoft.com/support/legal/sla/virtual-machines/v1_0/). **Ознакомьтесь с этим документом, даже если вам не интересны юридические тонкости**.

## Логистика: регионы, дистрибутивы, доступность, размеры виртуальных машин и квоты на них
### регионы
Ресурсы Microsoft Azure распределяются по нескольким географическим регионам по всему миру. Под "регионом" подразумевается несколько центров обработки данных в отдельном географическом регионе. Начиная с 1 января 2016 г. существует 8 центров обработки данных в Америке, 2 в Европе, 6 в Азиатско-Тихоокеанском регионе, 2 в континентальной части Китая и 3 в Индии. Полный список всех существующих и новых регионов Azure см. **[здесь](https://azure.microsoft.com/regions/)**.

### Дистрибутивы
Microsoft Azure поддерживает различные дистрибутивы Linux, которые предоставляются и обслуживаются партнерами. В Azure Marketplace вы найдете такие дистрибутивы, как CentOS, Debian, Red Hat Enterprise, Ubuntu, FreeBSD и многие другие. Мы активно работаем с разными сообществами Linux, чтобы расширить список рекомендованных дистрибутивов. **[Ознакомьтесь с текущим списком дистрибутивов](virtual-machines-linux-endorsed-distros.md)**. Если в коллекции отсутствует необходимый дистрибутив Linux, вы всегда можете использовать свою виртуальную машину Linux, следуя указаниям **[на этой странице](virtual-machines-linux-create-upload-generic.md)**.

## Доступность и соглашение об уровне обслуживания Microsoft Azure
Чтобы обеспечить соответствие соглашению об уровне обслуживания (SLA) на уровне 99,95 %, необходимо развернуть две или больше виртуальных машин для выполнения рабочих нагрузок в рамках группы доступности. Таким образом виртуальные машины распределяются по несколькими доменам сбоя в центрах обработки данных, а также развертываются на узлах с разными периодами обслуживания. Дополнительные сведения о соглашении об уровне обслуживания см. **[здесь](https://azure.microsoft.com/support/legal/sla/virtual-machines/v1_0/)**.

## Размеры виртуальных машин и квоты на них
Размер развертываемой виртуальной машины в Azure необходимо выбрать из списка доступных размеров в соответствии с рабочей нагрузкой. От размера также зависит вычислительная мощность, объем памяти и емкость хранилища виртуальной машины. Счета за виртуальную машину выставляются на основе времени ее работы и потребляемых ресурсов, которые для нее выделены. Дополнительные сведения см. в статье [Размеры виртуальных машин в Azure](virtual-machines-linux-sizes.md).

Ниже приведены основные рекомендации по выбору подходящего размера виртуальной машины (A, D, DS, G и GS).

* Виртуальные машины серии A — это недорогие виртуальные машины начального уровня для небольших рабочих нагрузок и несложных сценариев разработки и тестирования. Они широко доступны во всех регионах и поддерживают все стандартные ресурсы для виртуальных машин (как подключение к ним, так и их использование).
* Размеры виртуальных машин A8–A11 предназначены для кластерных приложений с высокопроизводительными вычислениями.
* Виртуальные машины серии D предназначены для приложений, которым необходимы большие вычислительные мощности и высокопроизводительные временные диски. Виртуальные машины серии D отличаются более быстрыми процессорами, более высоким соотношением "память — ядро" и твердотельным накопителем (SSD) в качестве временного диска. 
* Виртуальные машины серии Dv2 (последняя версии этой серии) отличаются более мощным ЦП. Процессор серии Dv2 примерно на 35 % быстрее, чем процессор серии D. Он основан на процессоре Intel Xeon® E5-2673 v3 (Haswell) последнего поколения с тактовой частотой 2,4 ГГц, а благодаря технологии Intel Turbo Boost версии 2.0 может достигать 3,2 ГГц. Серия Dv2 имеет такие же конфигурации памяти и диска, как и серия D.
* Виртуальные машины серии G отличаются максимальным объемом памяти и работают на серверах с процессорами семейства Intel Xeon E5 V3.

Примечание. Для виртуальных машин серий DS и GS доступно высокопроизводительное хранилище класса Premium на основе SSD с малой задержкой, которое предназначено для интенсивных рабочих нагрузок ввода-вывода. Хранилище класса Premium доступно в определенных регионах. Дополнительные сведения см. в статье **[Хранилище Premium: высокопроизводительное хранилище для рабочих нагрузок виртуальных машин Azure](../storage/storage-premium-storage.md)**.

Для каждой подписки Azure предусмотрена квота по умолчанию, от которой зависит возможность развертывания большого количества виртуальных машин для проекта. Текущее ограничение для каждой подписки составляет 20 виртуальных машин на регион. Чтобы увеличить квоту, подайте запрос в службу поддержки. Дополнительные сведения см. в статье **[Подписка Azure, границы, квоты и ограничения службы](../azure-subscription-service-limits.md)**.

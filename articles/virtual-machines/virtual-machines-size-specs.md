# Размеры виртуальных машин

[AZURE.INCLUDE [learn-about-deployment-models](../../includes/learn-about-deployment-models-both-include.md)]

## Обзор

В этой статье описаны доступные размеры и разновидности вычислительных ресурсов на базе виртуальных машин, которые можно использовать для запуска приложений и управления рабочими нагрузками. Здесь также предоставлены рекомендации по развертыванию, которые нужно учитывать при планировании использования этих ресурсов.

Виртуальные машины Azure и облачные службы — это два из нескольких типов вычислительных ресурсов, которые предоставляются в среде Azure. Дополнительные разъяснения см. в статье [Варианты размещения вычислительных сред, предоставляемые Azure](http://go.microsoft.com/fwlink/p/?LinkID=311926).

>[AZURE.NOTE]Сведения о других ограничениях Azure см. в статье [Подписка Azure, границы, квоты и ограничения службы](../azure-subscription-service-limits.md).

## Рекомендации относительно планирования

Пользователям предлагаются виртуальные машины двух уровней: Basic и Standard. Для обоих типов доступны различные размеры, однако на уровне Basic отсутствует ряд возможностей, предлагаемых на уровне Standard (например, балансировка нагрузки и автоматическое масштабирование). На уровне Standard предлагаются различные серии: A, D, DS, G и GS. Ниже приведены замечания по некоторым из этих размеров:

*   Виртуальные машины серии D предназначены для приложений, которым необходимы большие вычислительные мощности и высокопроизводительные временные диски. Виртуальные машины серии D отличаются более быстрыми процессорами, более высоким соотношением «память — ядро» и твердотельным накопителем (SSD) в качестве временного диска. Дополнительные сведения см. в объявлении в блоге Azure [Новые размеры виртуальных машин серии D](http://azure.microsoft.com/blog/2014/09/22/new-d-series-virtual-machine-sizes/).

*   Серия Dv2, продолжение оригинальной серии D, отличается более мощным ЦП. Процессор серии Dv2 примерно на 35 % быстрее, чем процессор серии D. Он основан на процессоре Intel Xeon® E5-2673 v3 (Haswell) последнего поколения с тактовой частотой 2,4 ГГц, а благодаря технологии Intel Turbo Boost версии 2.0 может достигать 3,2 ГГц. Серия Dv2 имеет такие же конфигурации памяти и диска, как и серия D.

    Серия Dv2 будет доступна в регионах по следующему расписанию. Октябрь 2015 г.: восточная часть США 2, центральная часть США, северо-центральный регион США, западная часть США. Ноябрь 2015 г.: восточная часть США, Северная Европа, Западная Европа. Январь 2016 г.: юго-центральный регион США, восток Азиатско-Тихоокеанского региона, юго-восток Азиатско-Тихоокеанского региона, восточная Япония, западная Япония, восточная часть Австралии, юго-восточная часть Австралии, южная Бразилия.


*   Виртуальные машины серии G отличаются максимальным размером и производительностью и работают на серверах с процессорами семейства Intel Xeon E5 V3.

*   Для виртуальных машин серий D и GS доступно высокопроизводительное хранилище Premium с малой задержкой, предназначенное для интенсивных рабочих нагрузок ввода-вывода. В них для размещения дисков виртуальных машин используются твердотельные накопители, а также предоставляется локальный кэш SSD. Хранилище класса Premium доступно в определенных регионах. Дополнительные сведения см. в статье [Хранилище класса Premium: хранилище высокой производительности для рабочих нагрузок виртуальных машин Azure](../storage-premium-storage-preview-portal.md).

От размера виртуальной машины зависит ее цена. Размер также влияет на процессорную мощность, объем памяти и хранилища виртуальной машины. Стоимость хранилища рассчитывается отдельно в зависимости от количества страниц, используемых в учетной записи хранения. Дополнительные сведения см. в статьях [Виртуальные машины: цены](http://azure.microsoft.com/pricing/details/virtual-machines/) и [Цены на хранилища Azure](http://azure.microsoft.com/pricing/details/storage/). Дополнительные сведения о хранилище для виртуальных машин см. в статье [О дисках и виртуальных жестких дисках для виртуальных машин](virtual-machines-disks-vhds.md).

Следующие рекомендации помогут вам выбрать оптимальный размер экземпляра.

*   Размер A0\\Basic\_A0 доступен только при использовании Azure SDK версии 1.3 или более поздней.  

*   Минимальный размер, который рекомендуется использовать для производственных рабочих нагрузок — A1\\Basic\_A1.

*   При использовании выпуска SQL Server Enterprise рекомендуется выбирать виртуальные машины с 4 или 8 ядрами ЦП.

*   Некоторые физические узлы в центрах обработки данных Azure не поддерживают виртуальные машины больших размеров, таких как A5—A11. Из-за этого при попытке изменить размер виртуальной машины, создать новую машину в виртуальной сети, образованной до 16 апреля 2013 г., либо добавить новую виртуальную машину в существующую облачную службу могут возникать сообщения об ошибках **Не удалось настроить виртуальную машину<machine name>** или **Не удалось создать виртуальную машину<machine name>**. Обходные способы решения этих проблем для каждого сценария развертывания описаны в теме [Ошибка: не удалось настроить виртуальную машину](https://social.msdn.microsoft.com/Forums/ru-RU/9693f56c-fcd3-4d42-850e-5e3b56c7d6be/error-failed-to-configure-virtual-machine-with-a5-a6-or-a7-vm-size?forum=WAVirtualMachinesforWindows) на форуме поддержки.

*   Виртуальные машины размеров A8/A10 и A9/A11 обладают одинаковой производительностью. Экземпляры виртуальных машин размеров A8 и A9 имеют дополнительный сетевой адаптер, подключенный к сети удаленного доступа к памяти (RDMA). Это позволяет им быстро обмениваться между собой данными. Экземпляры A8 и A9 предназначены для высокопроизводительных вычислительных приложений, которым требуется постоянная высокоскоростная связь между узлами во время работы (например, приложений, использующих интерфейс передачи сообщений MPI). Экземпляры виртуальных машин размеров A10 и A11 не имеют дополнительного сетевого адаптера. Экземпляры A10 и A11 предназначены для высокопроизводительных вычислительных приложений, которые не требуют постоянной высокоскоростной связи между узлами. Такие приложение также называют параметрическими или приложениями с массовым параллелизмом.

*   Серии Dv2 и D идеально подходят для приложений, требующих быстрого процессора, большей производительности локального диска или большего количества памяти. Они предлагают мощную комбинацию для многих приложений корпоративного уровня.

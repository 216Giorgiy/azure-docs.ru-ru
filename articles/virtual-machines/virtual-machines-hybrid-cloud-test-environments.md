# Гибридные облачные тестовые среды Azure

[AZURE.INCLUDE [learn-about-deployment-models](../../includes/learn-about-deployment-models-classic-include.md)]Модель диспетчера ресурсов.


Гибридные облачные тестовые среды используют локальные интернет-подключения и один из общедоступных IP-адресов для разработки и тестирования или экспериментального использования. Кроме того, они предусматривают пошаговые указания по настройке функционирующей виртуальной сети Azure между организациями. После развертывания такой среды вы можете разрабатывать или тестировать приложения и проводить эксперименты с ними при сниженной рабочей нагрузке ИТ-среды, а также измерять производительность подключения виртуальной частной сети типа «сеть-сеть» (VPN) относительно расположения в Интернете.

## Базовая конфигурация гибридного облака

[Базовая конфигурации гибридного облака](../virtual-network/virtual-networks-setup-hybrid-cloud-environment-testing.md) предусматривает:

- упрощенную локальную сеть с четырьмя виртуальными машинами (контроллером домена, сервером приложений, клиентским компьютером и VPN-устройством под управлением Windows Server и службы маршрутизации и удаленного доступа);
- виртуальную сеть Azure с репликой контроллера домена;
- VPN-подключение типа «сеть-сеть».

## Использование фермы интрасети SharePoint в гибридном облаке

[Ферма интрасети SharePoint в гибридной облачной тестовой среде](../virtual-network/virtual-networks-setup-sharepoint-hybrid-cloud-testing.md) добавляет к базовой конфигурации гибридного облака серверы SQL Server 2014 и SharePoint Server 2013. В результате получается двухуровневая ферма SharePoint, к которой можно получить доступ из клиентского компьютера через упрощенную локальную сеть.

## Бизнес-приложение на основе веб-интерфейса в гибридном облаке

[Бизнес-приложение на основе веб-интерфейса в гибридной облачной тестовой среде](../virtual-network/virtual-networks-setup-lobapp-hybrid-cloud-testing.md) добавляет к базовой конфигурации гибридного облака серверы SQL Server 2014 и Internet Information Services (IIS). В результате получается инфраструктура, в которой можно развертывать и тестировать многоуровневые бизнес-приложения на основе веб-интерфейса.

## Сервер Office 365 Directory Synchronization (DirSync) в гибридном облаке

[Сервер Office 365 DirSync в гибридной облачной тестовой среде](../virtual-network/virtual-networks-setup-dirsync-hybrid-cloud-testing.md) добавляет к базовой конфигурации гибридного облака сервер DirSync и демонстрирует синхронизацию паролей Office 365 DirSync с пробной подпиской Office 365.

## Смоделированная гибридная облачная среда

[Смоделированная гибридная облачная среда](../virtual-network/virtual-networks-setup-simulated-hybrid-cloud-environment-testing.md) ориентирована на организации и отдельных пользователей, которым не удобно использовать прямое подключение к Интернету и общедоступные IP-адреса. Она предоставляет упрощенную локальную сеть в отдельной виртуальной сети Azure, а затем соединяет две виртуальные сети через подключение VPN типа «VNet-to-VNet».

# Пошаговое руководство. Активация последней версии 12 базы данных SQL

В этой статье описаны действия, необходимые для активации версии 12 Базы данных SQL Azure, впервые выпущенной корпорацией Майкрософт в декабре 2014 г.

Чтобы использовать последнюю версию 12, требуется подписка на Microsoft Azure или по крайней мере на [бесплатную пробную версию](http://azure.microsoft.com/pricing/free-trial/).

Вы можете активировать версию 12 на новом портале предварительной версии Azure по адресу [http://portal.azure.com/](http://portal.azure.com/) вместо [старого портала Azure](http://manage.windowsazure.com/). После активации версии 12 для вашей подписки на портале предварительной версии Azure станут доступными функции создания и обновления для этой версии. Затем пользователи смогут запустить рабочий процесс [создания](sql-database-create.md) в колонке сервера или базы данных.

> [AZURE.NOTE]Мы советуем сначала попрактиковаться на тестовых базах данных, копиях баз данных или пустых базах данных. Прежде чем обновлять рабочие базы данных, участвующие в бизнес-процессах, необходимо дождаться завершения периода использования предварительной версии.

Дополнительную информацию об обновлении до версии 12 см. в статье [Планирование и подготовка к обновлению Базы данных SQL Azure до версии 12](sql-database-v12-plan-prepare-upgrade.md).


## О. Авторизация безопасности

Сначала необходимо убедиться, что у вас достаточно прав для активации версии 12 в рамках вашей подписки. Поэтому при попытке активировать версию 12 выполняется авторизация, чтобы убедиться в наличии у вас достаточных прав для этой подписки.

 Чтобы использовать версию 12, у вас должно быть одно из следующих прав:

- владелец подписки;
- соадминистратор подписки.

Подробнее об учетных записях Azure см. в статье [Управление учетными записями, подписками и административными ролями](http://msdn.microsoft.com/library/hh531793.aspx).

## B. Действия в пользовательском интерфейсе портала предварительной версии Azure

В этом разделе описывается последовательность действий, которые можно единожды выполнить в пользовательском интерфейсе на портале предварительной версии Azure, чтобы активировать версию 12. После активации параметра он становится доступным.

Во всех сценариях активации используется одна и та же основная идея. При первой попытке [создать новый сервер базы данных SQL](sql-database-create.md) отобразится колонка **Последняя версия (предварительная версия)**, содержащая флажок, который можно установить, чтобы активировать право использования версии 12. После активации вы больше не увидите флажок. Вместо этого вы увидите элемент управления **Да|Нет**, который можно использовать, чтобы указать, следует ли использовать для нового сервера версию 12. Если вы выберете **Нет**, будет создан сервер версии 11 (согласно команде SELECT @@VERSION;).

### Б.1. Элемент управления "Да" ("Нет") для версии 12

После активации прав на использование версии 12 вы увидите элемент управления **Да|Нет**, который обведен на следующем снимке экрана портала предварительной версии Azure.

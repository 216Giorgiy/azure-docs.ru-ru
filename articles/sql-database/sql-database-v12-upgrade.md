# Вышло обновление до версии 12 Базы данных SQL


[Зарегистрируйтесь](https://portal.azure.com) для получения версии 12 Базы данных SQL, чтобы воспользоваться преимуществами Базы данных SQL нового поколения в Microsoft Azure. Во-первых, вам требуется подписка на Microsoft Azure. Зарегистрируйтесь, чтобы получить [бесплатную пробную версию Azure](http://azure.microsoft.com/pricing/free-trial) и просмотреть [информацию о ценах](http://azure.microsoft.com/pricing/details/sql-database).


## Шаги по обновлению до версии 12 Базы данных SQL


| Шаг обновления | Снимок экрана |
| :--- | :--- |
| 1\. Войдите на портал [http://portal.azure.com/](http://portal.azure.com/). | ![Новый портал Azure][1] |
| 2\. Щелкните **ОБЗОР**. | ![Обзор служб][2] |
| 3\. Выберите **Серверы SQL**. Отобразится список имен серверов SQL Server. | ![Выбор службы SQL Server][3] |
| 4\. Выберите сервер, который необходимо скопировать на новый сервер с включенным обновлением базы данных SQL. | ![Отображает список серверов SQL][4] |
| 5\. Щелкните **Параметры** или **Версия сервера**, чтобы обновить сервер до версии 12. | ![Последний компонент предварительной версии][5] |
| 6\. Щелкните **ОБНОВИТЬ ЭТОТ СЕРВЕР**. | ![Обновление SQL Server до предварительной версии][6] |
| 7\. Щелкните **UPGRADE THIS SERVER**, чтобы запустить процесс обновления. В разделе **Версия сервера** уведомление изменяется с **V2** на **Запланировано обновление**. Щелкните уведомление **Запланировано обновление**, чтобы открыть колонку с кнопкой **Отмена обновления** вверху. Щелкните **Отмена обновления**, чтобы отменить обновление сервера. **Обратите внимание**, что отмена операции невозможна ближе к концу процесса обновления. В этом случае обновление будет выполнено.|![Отмена обновления][9] 


> [AZURE.NOTE]После выбора параметра обновления для вашего сервера и баз данных, хранящихся на нем, будут включены функции версии 12 Базы данных SQL, которые невозможно отключить. До версии 12 Базы данных SQL можно обновить только серверы с уровнями служб Basic, Standard или Premium. Подробнее об уровнях служб см. в статье [Обновление баз данных SQL уровня Web или Business до новых уровней служб"](sql-database-upgrade-new-service-tiers.md).


> [AZURE.IMPORTANT]Георепликация не поддерживается в предварительной версии 12 Базы данных SQL. Подробнее см. в статье [Планирование и подготовка к обновлению до предварительной версии базы данных SQL Azure 12](sql-database-v12-plan-prepare-upgrade.md).


После выбора варианта **ОБНОВИТЬ ЭТОТ СЕРВЕР** отобразится колонка с сообщением о процессе проверки.


- В ходе этого процесса проверяется уровень служб вашей базы данных и состояние георепликации. В колонке отобразятся результаты выполненной проверки. 
- После завершения процесса проверки вы увидите список имен баз данных, для которых требуется выполнить действие, необходимое для обновления до версии 12 Базы данных SQL.
 - **Вам понадобиться выполнить действия для всех баз данных, чтобы обновить их до версии 12 Базы данных SQL**.
- После щелчка по имени каждой базы данных открывается нова колонка с рекомендованной ценовой категорией на основании вашего текущего использования. Кроме того, можно выбрать другую ценовую категорию, соответствующую вашей среде. Для всех обновляемых баз данных необходимо остановить георепликацию. 
- Обратите внимание: рекомендованная ценовая категория не отображается, если недостаточно соответствующих данных. 


| Действие | Снимок экрана |
| :--- | :--- |
| 7\. Завершив подготовку сервера к обновлению, введите имя сервера, который необходимо обновить, и нажмите кнопку **ОК**. | ![Подтверждение имени сервера для обновления][7] |
| 8\. Будет запущен процесс обновления. Обновление может занять до 24 часов. В течение этого времени остаются доступными все базы данных на сервере, но операции управления сервером и базой данных ограничены. После завершения процесса в колонке сервера отобразится состояние **Включено**. | ![Подтверждение включения компонентов предварительной версии][8] |


## Командлеты PowerShell


Командлеты PowerShell доступны для запуска, остановки и мониторинга обновления версии 11 или другой более ранней версии Базы данных SQL Azure до версии 12.


Справочную документацию по этим командлетам PowerShell см. в таких разделах:


- [Get-AzureSqlServerUpgrade](http://msdn.microsoft.com/library/mt143621.aspx)
- [Start-AzureSqlServerUpgrade](http://msdn.microsoft.com/library/mt143623.aspx)
- [Stop-AzureSqlServerUpgrade](http://msdn.microsoft.com/library/mt143622.aspx)


Командлет Stop- означает отмену, а не приостановку. После его выполнения возобновить обновление невозможно, можно только запустить его сначала. Командлет Stop- очищает и высвобождает все соответствующие ресурсы.

# Восстановление удаленной базы данных SQL Azure на портале Azure

> [AZURE.SELECTOR]
- [Restore deleted database - PowerShell](sql-database-restore-deleted-database-tutorial-powershell.md)
- [Restore deleted database - REST API](sql-database-restore-deleted-database-tutorial-rest.md)

## Обзор

В этом учебнике показано, как восстановить удаленную базу данных SQL Azure на [портале Azure](http://manage.windowsazure.com). Вы можете восстановить базу данных, которая была удалена в течение срока хранения, до момента времени удаления. Срок хранения определяется по уровню обслуживания базы данных.

При восстановлении удаленной базы данных SQL Azure создается новая база данных. Служба автоматически выбирает уровень обслуживания на основе резервной копии, используемой в точке восстановления. Убедитесь, что на логическом сервере есть доступная квота для создания еще одной базы данных. Если вы хотите запросить более высокую квоту, обратитесь в [службу поддержки Azure](http://azure.microsoft.com/support/options/).

## Ограничения и безопасность

* Функция восстановления удаленной базы данных SQL Azure включена для уровней Basic, Standard и Premium. 

* На уровне Basic резервные копии хранятся в течение периода в 7 дней, на уровне Standard — 14 дней, а на уровне Premium — 35 дней.

* Только администратор или соадминистратор подписки может отправлять запросы на восстановление.

* Имя для входа субъекта серверного уровня будет соответствовать владельцу восстановленной базы данных.
 
* Уровни обслуживания выпусков Web и Business не поддерживают восстановление удаленной базы данных SQL.
 
	* При наличии базы данных выпуска Web или Business вы можете использовать копию базы данных для получения такой копии базы данных, в которой будут содержаться транзакции, а затем экспортировать скопированную базу данных в учетную запись хранения Microsoft Azure. Дополнительную информацию см. в разделе [Инструкции по использованию копии базы данных (база данных SQL Azure)](http://msdn.microsoft.com/library/azure/ff951631.aspx) и [Инструкции по использованию службы импорта и экспорта в базе данных Azure SQL](http://msdn.microsoft.com/library/azure/hh335292.aspx).
	* В сентябре 2015 года поддержка выпусков Web и Business будет прекращена. Дополнительные сведения см. в статье [Вопросы и ответы по остановке Web Edition и Business Edition](http://msdn.microsoft.com/library/azure/dn741330.aspx).

## Практическое руководство. Восстановление удаленной базы данных SQL Azure на портале Azure

> [AZURE.VIDEO restore-a-deleted-sql-database]

1. Войдите на [портал Azure](http://manage.windowsazure.com) с помощью учетной записи Майкрософт.

2. В левой области навигации щелкните **БАЗЫ ДАННЫХ SQL**.

3. В верхней части страницы щелкните ссылку **УДАЛЕННЫЕ БАЗЫ ДАННЫХ**. Отобразится список **УДАЛЕННЫЕ БАЗЫ ДАННЫХ, КОТОРЫЕ МОГУТ БЫТЬ ВОССТАНОВЛЕНЫ**.

4. Щелкните по базе данных, которую хотите восстановить.

6. В нижней части страницы на панели команд щелкните **Восстановление**. Это откроет диалоговое окно **Указать параметры восстановления**.

7. Выберите имя в поле **ИМЯ БАЗЫ ДАННЫХ**. По умолчанию имя базы данных выбирается автоматически, но при желании его можно изменить.

	* Учтите, что удаленную базу данных можно восстановить только на сервер, с которого она была удалена, а также до момента времени, в который ее удалили.   

8. Установите флажок, чтобы отправить запрос на восстановление.

Восстановление может занять некоторое время. Для мониторинга состояния восстановления щелкните значок «Состояние» в нижнем правом углу страницы на панели команд и нажмите кнопку **ПОДРОБНОСТИ**.

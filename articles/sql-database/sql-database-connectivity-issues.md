# Проблемы с подключением к базе данных SQL

В этой статье приведены общие сведения об использовании различных средств устранения неполадок, связанных с подключением к базе данных SQL Azure.


## Определите, является проблема с подключением общей (вы просто не можете подключиться к базе данных) или специфичной для конкретного приложения.

Используя среду SQL Server Management Studio или программу SQLCMD.EXE, проверьте, можете ли вы подключиться к базе данных.

- Инструкции по подключению к базе данных SQL с помощью SQL Server Management Studio (SSMS) см. в статье [Подключение к базе данных SQL Azure с помощью среды SSMS](sql-database-connect-to-database.md).
- Инструкции по подключению к базе данных SQL с помощью SQLCMD см. в статье [Подключение к базе данных SQL Azure с помощью SQLCMD](https://msdn.microsoft.com/library/azure/ee336280.aspx).



## Выполняется ли приложение, которому не удается подключиться к базе данных SQL, в службе Azure (например, является ли оно облачной службой или веб-приложением)?

Убедитесь, что включено правило брандмауэра, разрешающее всем службам Azure подключаться к серверу и базе данных.

- Сведения о правилах брандмауэра и о том, как разрешить подключения из службы Azure, см. в статье [Брандмауэр базы данных SQL Azure](https://msdn.microsoft.com/library/azure/ee621782.aspx#ConnectingFromAzure).



## Приложение пытается подключиться к базе данных SQL из частной сети?

Убедитесь, что включены правила брандмауэра, разрешающие доступ к серверу и базе данных из конкретных сетей.

- Общие сведения о правилах брандмауэра см. в статье [Брандмауэр базы данных SQL Azure](https://msdn.microsoft.com/library/azure/ee621782.aspx).
- Инструкции по настройке правил брандмауэра см в статье [Настройка параметров брандмауэра (база данных SQL Azure)](https://msdn.microsoft.com/library/azure/jj553530.aspx).


## Если правила брандмауэра настроены правильно, попробуйте выполнить действия, описанные в [пошаговом руководстве по устранению неполадок с подключением к базе данных SQL Microsoft Azure](https://support2.microsoft.com/common/survey.aspx?scid=sw;en;3844&showpage=1).

В нем приводятся сведения по следующим проблемам с подключением к базе данных SQL:

- Не удается открыть соединение с сервером из-за настроек брандмауэра. 
- Сервер не найден или недоступен. 
- Не удается выполнить вход на сервер. 
- Ошибки, связанные с временем ожидания соединения. 
- Временные ошибки. 
- Соединение прервано из-за системных ограничений. 
- Ошибки подключений из среды SQL Server Management Studio (SSMS). 


## Дополнительная информация

- Дополнительные сведения о подключении к базе данных SQL см. в статье [Рекомендации по подключению к базе данных SQL Azure программными средствами](https://msdn.microsoft.com/library/azure/ee336282.aspx).   

- Сведения о конкретных ошибках подключения можно найти в разделе **Ошибки, связанные с потерей подключения** статьи [Сообщения об ошибках (база данных SQL Azure)](https://msdn.microsoft.com/library/azure/ff394106.aspx#bkmk_connection_errors).

- Чтобы просмотреть данные событий подключения, используйте представление [**sys.event_log (база данных SQL Azure)**](https://msdn.microsoft.com/library/dn270018.aspx).

- Чтобы просмотреть показатели по событиям подключений к базе данных, используйте представление [**sys.database_connection_stats (база данных SQL Azure)**](https://msdn.microsoft.com/library/dn269986.aspx).

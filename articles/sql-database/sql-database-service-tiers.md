# Уровни служб

Доступны три уровня служб [Базы данных SQL](sql-database-technical-overview.md): Basic, Standard и Premium. По условиям [SLA](http://azure.microsoft.com/support/legal/sla/), все три уровня гарантируют бесперебойную работу в течение 99,99 % времени и обеспечивают прогнозируемую производительность, различные варианты обеспечения непрерывности бизнес-процессов, функции безопасности и почасовое выставление счетов. Каждый уровень служб включает несколько уровней производительности, что позволяет вам выбрать уровень, наиболее соответствующий вашим рабочим нагрузкам. При необходимости увеличения или уменьшения масштаба вы легко можете изменить уровни базы данных на портале Azure с нулевым временем простоя приложения. Подробнее см. в разделе [Изменение уровней служб и уровней производительности базы данных](https://msdn.microsoft.com/library/azure/dn369872.aspx).

> [AZURE.IMPORTANT]Выпуски Web и Business станут недоступны. Узнайте, как [обновить выпуски Web и Business](sql-database-upgrade-new-service-tiers.md). Если вы собираетесь продолжать использовать выпуски Web и Business, ознакомьтесь с разделом [Вопросы и ответы по остановке версий Web и Business](http://azure.microsoft.com/pricing/details/sql-database/web-business/).

## Basic;

Уровень Basic предназначен для приложений с легкой транзакционной рабочей нагрузкой. Типичный случай использования: легкое приложение, для работы которого требуется небольшая база данных с одной активной операцией в любой момент времени.

**Производительность, размер и функции**


| Уровень служб | Basic; |
| :-- | :-- |
| Единицы пропускной способности базы данных (DTU) | 5 |
| Максимальный размер базы данных | 2 ГБ |
| Восстановление на момент времени (PITR) | С точностью до миллисекунды в течение последних 7 дней |
| Аварийное восстановление | Географическое восстановление любого региона Azure |
| Цели производительности | Число транзакций в час |


## Standard

Уровень Standard предназначен для небольших транзакционных рабочих нагрузок. Он включает более высокую производительность и расширенные встроенные функции обеспечения непрерывности бизнес-процессов, по сравнению с уровнем Basic. Типичный случай использования — это приложение с несколькими параллельными транзакциями.

**Производительность и размер**


| Уровень служб | Standard S0 | Standard S1 | Standard S2 | Standard S3 |
| :-- | :-- | :-- | :-- | :-- |
| Единицы пропускной способности базы данных (DTU) | 10 | 20 | 50 | 100 |
| Максимальный размер базы данных | 250 ГБ | 250 ГБ | 250 ГБ | 250 ГБ |


**Функции**


| Уровень служб | Standard (S0, S1, S2, S3) |
| :-- | :-- |
| Восстановление на момент времени (PITR) | С точностью до миллисекунды в течение последних 14 дней |
| Аварийное восстановление | Георепликация Standard, один дополнительный регион вне сети |
| Цели производительности | Число транзакций в минуту |


## Premium

Уровень Premium предназначен для критически важных приложений. Он включает наивысшую производительность и доступ к расширенным функциям обеспечения непрерывности бизнес-процессов, включая активную георепликацию в 4 регионах Azure на ваш выбор. Типичный случай использования — критически важное приложение с большим объемом транзакций и множеством параллельных пользователей.

**Производительность и размер**


| Уровень служб | Premium P1 | Premium P2 | Premium P6 (ранее P3) |
| :-- | :-- | :-- | :-- |
| Единицы пропускной способности базы данных (DTU) | 125 | 250 | 1000 |
| Максимальный размер базы данных | 500 ГБ | 500 ГБ | 500 ГБ |


**Функции**


| Уровень служб | Premium (P1, P2, P6) |
| :-- | :-- |
| Восстановление на момент времени (PITR) | С точностью до миллисекунды в течение последних 35 дней |
| Аварийное восстановление | Активная георепликация, до четырех дополнительных регионов вне сети |
| Цели производительности | Число транзакций в секунду |


Дополнительные сведения о ценах на эти уровни служб см. в разделе [База данных SQL Цены](http://azure.microsoft.com/pricing/details/sql-database/).

# Пошаговое руководство. Активация последней версии 12 базы данных SQL (предварительная версия)

В этой статье описаны действия, необходимые для активации параметра обновления версии 12 базы данных SQL Azure (предварительной версии), впервые выпущенной корпорацией Майкрософт в декабре 2014 г.

Чтобы использовать предварительную версию последней версии 12, требуется подписка на Microsoft Azure или по крайней мере подписка на [бесплатную пробную версию](http://azure.microsoft.com/pricing/free-trial/).

Вы можете активировать предварительную версию 12, используя новый портал управления Microsoft Azure предварительной версии по адресу [http://portal.azure.com/](http://portal.azure.com/). После активации предварительной версии 12 для вашей подписки на портале Azure станут доступными функции создания и обновления для этой версии. Затем пользователи смогут запустить рабочий процесс [создания](http://azure.microsoft.com/documentation/articles/sql-database-preview-create/) или [обновления](http://azure.microsoft.com/documentation/articles/sql-database-preview-create/) в колонке сервера или базы данных.

> [AZURE.NOTE]
> Мы советуем сначала попрактиковаться на тестовых базах данных, копиях баз данных или пустых базах данных. Прежде чем обновлять рабочие базы данных, участвующие в бизнес-процессах, необходимо дождаться завершения периода использования предварительной версии.

Дополнительную информацию о предварительной версии см. в статье [Планирование и подготовка к обновлению до базы данных SQL Azure 12 (предварительной версии)](http://azure.microsoft.com/documentation/articles/sql-database-preview-plan-prepare-upgrade/).


## А. Авторизация безопасности

Сначала необходимо убедиться, что у вас достаточно прав для активации предварительной версии 12 в рамках вашей подписки. Поэтому при попытке активировать параметр предварительной версии 12 выполняется авторизация, чтобы убедиться что у вас имеется достаточно прав для этой подписки.

 Чтобы использовать предварительную версию, у вас должно быть одно из следующих прав:

- владелец подписки;
- соадминистратор подписки.

Дополнительные сведения об учетных записях Azure см. в статье [Управление учетными записями, подписками и ролями администратора](http://msdn.microsoft.com/library/hh531793.aspx).

## Б. Действия в пользовательском интерфейсе портала

В этом разделе описывается последовательность щелчков, которые можно единожды выполнить в пользовательском интерфейсе на портале Azure, чтобы активировать параметр предварительной версии 12. После активации параметра он становится доступным.

Во всех сценариях активации используется одна и та же основная идея. При первой попытке [создать новый сервер базы данных SQL](http://azure.microsoft.com/documentation/articles/sql-database-preview-create/) отобразится колонка **Последняя версия (предварительная версия)**, содержащая флажок, который можно установить, чтобы активировать право использования предварительной версии 12. После активации вы больше не увидите флажок. Вместо этого вы увидите элемент управления **Да|Нет**, который можно использовать, чтобы указать, следует ли использовать для нового сервера предварительную версию 12. Если вы выберете **Нет**, будет создан сервер версии 11 (согласно команде SELECT @@VERSION;).

### Б.1. Элемент управления "Да" ("Нет") для предварительной версии 12

После активации прав на использование предварительной версии 12 вы увидите элемент управления **Да|Нет**, который обведен на следующем снимке экрана портала управления.

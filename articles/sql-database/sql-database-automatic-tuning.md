# <a name="automatic-tuning-in-azure-sql-database"></a>Автоматическая настройка в базе данных SQL Azure

База данных SQL Azure — это полностью управляемая служба данных, которая отслеживает запросы, выполняемые в базе данных и автоматически повышает производительность рабочей нагрузки. База данных SQL Azure имеет встроенный механизм [автоматической настройки](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), который может автоматически настраивать и повышать производительность запросов путем динамической адаптации базы данных к рабочей нагрузке. Автоматическая настройка в базе данных SQL Azure может быть одной из самых важных функций, которые можно включить в базе данных SQL Azure для оптимизации производительности запросов.

## <a name="automatic-tuning-options"></a>Параметры автоматической настройки

Параметры [автоматической настройки](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning), доступные в базе данных SQL Azure:
 1. **Создать индекс** — для идентификации индексов, которые могут повысить производительность рабочей нагрузки, создания индексов и проверки их влияния на производительность запросов.
 2. **Удалить индекс** — для идентификации избыточных и повторяющихся индексов, а также индексов, которые не использовались долгое время.
 3. **PLAN REGRESSION CORRECTION** (Исправление регрессии плана) — для идентификации SQL-запросов, использующих более медленный план выполнения по сравнению с предыдущим, и использования оптимального плана вместо регрессионного.

База данных SQL Azure определяет рекомендации для использования параметров **Создать индекс**, **Удалить индекс** и **PLAN REGRESSION CORRECTION** (Исправление регрессии плана), с помощью которых можно оптимизировать базу данных, и отображает их на портале Azure. Дополнительные сведения об идентификации индексов, которые необходимо изменить, см. в статье [Использование помощника по базам данных SQL на портале Azure](sql-database-advisor-portal.md). Вы можете вручную применить рекомендации с помощью портала или разрешить базе данных SQL Azure автоматически применять эти рекомендации, отслеживать рабочие нагрузки после изменения, а затем выполнять проверку производительности рабочей нагрузки.

Параметры автоматической настройки можно включить и отключить отдельно для каждой базы данных, а также настроить на логическом сервере и применить для каждой базы данных, которая наследует параметры от сервера. Мы советуем настроить параметры [автоматической настройки](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning) на сервере и задать наследование параметров от сервера, так как это упрощает управление этими параметрами для большого количества баз данных.

Инструкции о том, как включить автоматическую настройку с помощью портала Azure, см. в [этой статье](sql-database-automatic-tuning-enable.md).

# Установка компонентов службы заданий эластичной базы данных

[Эластичный пул баз данных (предварительная версия)](sql-database-elastic-pool-portal.md) — это прогнозируемая модель для развертывания большого количества баз данных. Создав пул, вы сможете выполнять административные задачи в каждой его базе с помощью **службы заданий эластичной базы данных**. Например, вы сможете развернуть новую схему, предусматривающую установку политики RLS в каждой базе данных для разрешения доступа к конфиденциальным сведениям только пользователям с определенными учетными данными. Ниже мы расскажем вам, как установить **службу заданий эластичной базы данных**.

**Предполагаемое время выполнения:** 10 минут.

## Предварительные требования
* Подписка Azure. Зарегистрироваться в пробной версии, которая доступна бесплатно в течение одного месяца, можно [здесь](http://azure.microsoft.com/pricing/free-trial/).
* Эластичный пул баз данных. Подробности см. в статье [Создание эластичного пула баз данных SQL Azure (предварительная версия)](sql-database-elastic-pool-portal.md).

## Установка компонентов службы
Войдите на [портал предварительной версии Azure](https://ms.portal.azure.com/#).


1. В представлении панели мониторинга эластичного пула баз данных выберите **Создать задание**.
2. Если вы создаете задание впервые, вам необходимо установить **службу заданий эластичной базы данных**, щелкнув **УСЛОВИЯ ИСПОЛЬЗОВАНИЯ ПРЕДВАРИТЕЛЬНОЙ ВЕРСИИ**. 
3. Примите условия, установив соответствующий флажок.
4. В представлении «Установка служб» щелкните **УЧЕТНЫЕ ДАННЫЕ ДЛЯ ЗАДАНИЯ**.

	![Установка служб][1]

5. Введите имя пользователя и пароль администратора базы данных. Если во всех базах эластичного пула баз данных для выполнения сценариев используется общая учетная запись, используйте ее. Так вам не придется создавать в каждой базе данных нового пользователя для выполнения сценариев. В процессе установки создается сервер базы данных SQL Azure. На этом сервере создается новая база данных, известная также как управляющая база данных, в которой хранятся метаданные для службы заданий эластичной базы данных. Созданные имя пользователя и пароль используются: 1) для входа в управляющую базу данных; (2) в качестве учетных данных для входа в каждую базу эластичного пула баз данных при запуске задания по выполнению сценария.
 
	![Создание имени пользователя и пароля][2]
6. Нажмите кнопку «ОК». Компоненты будут созданы через несколько минут в новой [группе ресурсов](../resource-group-portal.md). Новая группа ресурсов закрепляется на начальной доске, как показано на рисунке ниже. После создания компонентов в группе будет создана служба заданий эластичной базы данных (облачная служба, база данных SQL, служебная шина и хранилище).

	![Группа ресурсов на начальной доске][3]


7. Если во время установки службы заданий эластичной базы данных вы попытаетесь создать или изменить какое-то задание, после ввода **учетных данных** появится следующее сообщение.

	![Развертывание еще не завершено][4]

8. Если во время установки по какой-то причине произойдет ошибка, удалите группу ресурсов. См. статью [Удаление компонентов службы заданий эластичной базы данных](sql-database-elastic-jobs-uninstall.md).


## Дальнейшие действия

Если во время установки службы заданий эластичной базы данных вы указали новые учетные данные, которые еще не созданы в каждой базе эластичного пула баз данных, соответствующую учетную запись с правами на выполнение сценариев необходимо создать в каждой базе данных. Подробности см. в статье [Добавление пользователей в эластичный пул баз данных](sql-database-elastic-jobs-add-logins-to-dbs.md). Сведения о создании заданий см. в статье [Создание заданий и управление ими в эластичной базе данных](sql-database-elastic-jobs-create-and-manage.md).

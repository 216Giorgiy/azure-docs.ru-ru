# Устранение неполадки "База данных на сервере в настоящий момент недоступна. Повторите попытку подключения позже" и других ошибок подключения
"База данных <dbname> на сервере <servername> в настоящий момент недоступна..." — наиболее распространенный временный сбой подключения к базе данных SQL Azure. Временные ошибки подключения обычно происходят из-за запланированного события (например, обновление программного обеспечения) или незапланированного события (например, сбой процесса). Обычно эти ошибки длятся недолго: от нескольких секунд до одной минуты максимум. При получении другой ошибки проверьте, не содержит ли [сообщение об ошибке](sql-database-develop-error-messages.md) подсказки о причине ее возникновения. Определите, имеет ли она временный или постоянный характер, и выполните инструкции данного раздела.

## Порядок устранения временных проблем подключения
1.	Все известные сбои см. на [панели мониторинга служб Microsoft Azure](https://azure.microsoft.com/status).
2.	Убедитесь, что в приложении используется логика повторных попыток. Общие стратегии повторных попыток описаны в разделах [Проблемы подключения](sql-database-connectivity-issues.md) и [Рекомендации и советы по разработке](sql-database-connect-central-recommendations.md). Затем изучите особенности в [образцах кода](sql-database-develop-quick-start-client-code-samples.md).
3.	Если база данных близка к исчерпанию доступных ресурсов, может возникать временная проблема подключения. См. раздел [Устранение неполадок производительности](sql-database-troubleshoot-performance.md).
4.	Если проблемы с подключением не устранены, отправьте запрос в службу поддержки Azure. Для этого щелкните ссылку **Получить поддержку** на сайте [службы поддержки Azure](https://azure.microsoft.com/support/options).

## Порядок устранения постоянных проблем подключения
Если приложение не может подключиться вообще, то обычно это связано с настройкой IP-адреса и брандмауэра. Это могут быть перенастройка сети на стороне клиента (например, новый IP-адрес или прокси-сервер). Также нередко встречаются орфографические ошибки в параметрах соединения, например, в строке подключения.

1.	Настройте [правила брандмауэра](sql-database-configure-firewall-settings.md), разрешив предоставление IP-адреса клиенту.
2.	На всех брандмауэрах между клиентом и Интернетом откройте порт 1433 для входящих соединений.
3.	Проверьте строку подключения и другие параметры подключения. См. раздел "Строка подключения" в [статье о проблемах подключения](sql-database-connectivity-issues.md).
4.	Проверьте работоспособность службы на панели мониторинга. Если вы считаете, что имеет место региональный сбой, выполните инструкции по восстановлению в новый регион из раздела [Восстановление после сбоя](sql-database-disaster-recovery.md).
5.	Если проблемы с подключением не устранены, отправьте запрос в службу поддержки Azure. Для этого щелкните ссылку **Получить поддержку** на сайте [службы поддержки Azure](https://azure.microsoft.com/support/options).

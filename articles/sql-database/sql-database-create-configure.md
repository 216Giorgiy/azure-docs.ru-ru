# Создание и настройка базы данных SQL Azure

В этом разделе вы создадите и настроите новую базу данных SQL Azure с помощью параметра портала управления Azure **БЫСТРОЕ СОЗДАНИЕ**. Этот процесс демонстрирует создание базы данных SQL с использованием существующего сервера, а также создание нового сервера, если это необходимо.

> [AZURE.NOTE] При создании базы данных SQL с помощью функции **БЫСТРОЕ СОЗДАНИЕ** выполняется подготовка базы данных уровня Standard (S0). Для создания базы данных SQL на уровне службы и уровне производительности, кроме уровня Standard (S0), используйте команду **НАСТРАИВАЕМОЕ СОЗДАНИЕ**. Дополнительную информацию о создании базы данных SQL Azure с помощью команды **НАСТРАИВАЕМОЕ СОЗДАНИЕ** см. в статье [Приступая к работе с базой данных SQL Azure Microsoft](http://azure.microsoft.com/documentation/articles/sql-database-get-started/).

## Практическое руководство. Создание базы данных SQL Azure

1. Выполните вход на [Портал управления](https://portal.azure.com/).

2. В нижней части страницы щелкните **СОЗДАТЬ**.

	![Click SQL Databases][1]

3. Выберите **СЛУЖБЫ ДАННЫХ** и **БАЗА ДАННЫХ SQL**, а затем **БЫСТРОЕ СОЗДАНИЕ**.

	![Click New, Data Services, and Quick Create][2]
	 
5. В окне **БЫСТРОЕ СОЗДАНИЕ** введите имя новой базы данных, выберите подписку, а затем выберите сервер в списке **СЕРВЕР** (или перейдите к следующему шагу, чтобы создать новый сервер).

	![Create a new SQL Database in an existing server][7]

	При необходимости можно создать новый сервер, выбрав **Создать сервер базы данных SQL**.
    ![Create a new SQL Database and a new server][8]

	1. Выберите регион. Регион определяет географическое расположение сервера. Изменение региона достаточно трудоемко, поэтому выберите регион, имеющий смысл для этого сервера. Выберите местоположение, максимально близкое к вам. Хранение приложения и базы данных Azure в том же регионе снижает расходы на пропускную способность и задержку данных.
	2. Введите имя для входа одним словом, без пробелов. 

		База данных SQL использует проверку подлинности SQL с помощью шифрованного подключения. Будет создана новая учетная запись для проверки подлинности SQL Server, назначенная предопределенной роли сервера sysadmin и использующая введенное имя. 

		Именем входа не может быть адрес электронной почты, учетная запись пользователя Windows или идентификатор Windows Live ID. Для базы данных SQL не поддерживается ни проверка подлинности на основе утверждений, ни проверка подлинности Windows. 
	3. Укажите надежный пароль, который содержит более восьми знаков, в том числе символы в верхнем и нижнем регистре, а также цифры и символы.

	


9. Закончив, щелкните флажок **СОЗДАТЬ БАЗУ ДАННЫХ SQL** внизу страницы.

### Имя сервера, сформированное автоматически

Обратите внимание, что при создании нового сервера не было указано имя сервера. База данных SQL автоматически создает имя сервера, гарантируя отсутствие повторяющихся записей DNS. Имя сервера является буквенно-цифровой строкой, состоящей из 10 знаков. Невозможно изменить имя сервера баз данных SQL.

На следующем шаге будет настроен брандмауэр, чтобы разрешить подключение для приложений, работающих в вашей сети.

## Практическое руководство. Настройка брандмауэра для логического сервера

1. На [Портале управления](http://manage.windowsazure.com) щелкните **Базы данных SQL**, затем щелкните **Серверы**.

	![Click Servers][4]
2. В списке выберите только что созданный сервер.

2. Нажмите **Настроить**.

	![Click Configure][5]

3. В разделе **разрешенные IP-адреса** щелкните **ДОБАВИТЬ К РАЗРЕШЕННЫМ IP-АДРЕСАМ**. Это IP-адрес, прослушиваемый в настоящий момент вашим маршрутизатором или прокси-сервером. База данных SQL определяет IP-адрес, используемый текущим подключением, и создает правило брандмауэра для принятия запросов на подключение от этого устройства. 
	![Click Add to the allowed IP addresses][6]

	Создается имя по умолчанию для этого правила. Измените имя по своему усмотрению. 
	

4. При подключении к базе данных с другого компьютера вам необходимо создать новое правило, чтобы разрешить IP-адрес для подключения. Используйте поля "Начало" и "Окончание" для создания диапазона IP-адресов.

	Если клиентские компьютеры используют динамически назначаемые IP-адреса, то можно указать диапазон, достаточно широкий для охвата IP-адресов, назначенных компьютерам в сети. Начните с узкого диапазона и расширяйте его только в случае необходимости.

7. Чтобы завершить шаг, нажмите кнопку **Сохранить** в нижней части страницы. 

Теперь у вас есть база данных SQL, логический сервер, правило брандмауэра, разрешающее входящие подключения с вашего IP-адреса, и учетная запись администратора. 

**Примечание.** Только что созданный логический сервер является виртуальным и будет динамически размещаться на физических серверах в центре обработки данных. После удаления сервер восстановить невозможно. Не забудьте создать резервную копию баз данных, которые впоследствии нужно будет выложить на сервер.

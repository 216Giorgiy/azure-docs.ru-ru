# Создание своей первой базы данных SQL Azure

**Отдельная база данных**

> [AZURE.SELECTOR]
- [Azure portal](sql-database-get-started.md)
- [C#](sql-database-get-started-csharp.md)
- [PowerShell](sql-database-get-started-powershell.md)

В этой статье описано, как создать демонстрационную базу данных SQL Azure за несколько минут с помощью портала Azure. Вы узнаете, как:

- создать сервер для размещения создаваемой базы данных, а затем настроить для него правило брандмауэра;
- создать базу данных на основе образца AdventureWorks, с данными которого можно поэкспериментировать.

Перед началом работы вам потребуются учетная запись Azure и подписка. Если у вас ее нет, зарегистрируйтесь, чтобы [воспользоваться бесплатной пробной версией](http://azure.microsoft.com/pricing/free-trial/).

## Шаг 1. Вход и начало настройки базы данных SQL
1. Войдите на [портал Azure](http://portal.azure.com/).
2. Выберите пункты **Создать** > **Хранилище, кэш и резервное копирование** > **SQL Database**.

	![Создание новой базы данных SQL](./media/sql-database-get-started/create-db.png)
	
	Сведения о сервере и базе данных настраиваются в появившейся колонке **База данных SQL**.

	![Параметры базы данных и сервера для базы данных SQL](./media/sql-database-get-started/get-started-dbandserversettings.png)

## Шаг 2. Выбор параметров сервера
База данных SQL в Azure работает на сервере баз данных. На сервере могут размещаться несколько баз данных. При настройке базы данных можно создать и настроить также сервер, на котором она будет размещена, или использовать созданный ранее. Мы настроим новый сервер.

1. Введите **Имя** базы данных (мы будем использовать **AdventureWorks**). Позже мы вернемся к остальным параметрам базы данных.
2. В разделе **Сервер** щелкните **Настроить необходимые параметры**, затем выберите **Создать новый сервер**.

	![Присвоение имени базе данных](./media/sql-database-get-started/name-and-newserver.png)

3. В колонке **Новый сервер** введите **имя сервера**, которое является уникальным в пределах Azure и легко запоминается. Это имя нам понадобится позже при подключении и работе с базой данных.
4. Введите **имя входа администратора сервера**, которое легко запоминается (мы используем **AdventureAdmin**). Затем введите надежный **пароль** и введите его еще раз в поле **Подтверждение пароля**.

	![Новые параметры сервера базы данных](./media/sql-database-get-started/get-started-serversettings.png)

	 Оставьте для параметра **Создать сервер V12 (последнее обновление)** значение **Да** для использования новейших функций. **Расположение** определяет регион центра обработки данных, где создается сервер.

	>[AZURE.TIP]Серверы баз данных следует создавать по возможности ближе к приложениям, которые будут использовать базу данных. Если вы хотите изменить расположение, просто щелкните **Расположение**, выберите другой вариант и нажмите кнопку **ОК**.

5. Нажмите кнопку **ОК**, чтобы вернуться к колонке **База данных SQL**.

База данных и сервер еще не созданы. Это произойдет после следующего шага, когда вы выберите создание базы данных на основе образца AdventureWorks и подтвердите настройки.

## Шаг 4. Настройка и создание базы данных
1. В колонке **База данных SQL** щелкните элемент **Выбрать источник**, а затем — элемент **Образец**. 

	![Создание базы данных на основе образца](./media/sql-database-get-started/new-sample-db.png)

2. Вы вернетесь к колонке **База данных SQL**, где в области интерфейса **Выбрать образец** теперь отображается **AdventureWorks LT [V12]**. Нажмите **Создать**, чтобы запустить создание сервера и базы данных.

	![Создание образца базы данных](./media/sql-database-get-started/adworks_create.png)

	>[AZURE.NOTE]Для этого краткого руководства мы не меняли параметры **Ценовая категория**, **Параметры сортировки** и **Группа ресурсов**. Изменить ценовую категорию базы данных и масштабировать ее в любом направлении можно в любой момент без простоя. Дополнительные сведения см. в статьях [Цены баз данных SQL](http://azure.microsoft.com/pricing/details/sql-database/) и [Ценовые категории базы данных SQL](sql-database-service-tiers.md). Параметры сортировки базы данных нельзя изменить после того, как они уже настроены. Подробные сведения о параметрах сортировки см. в статье [Параметры сортировки и поддержка Юникода](https://msdn.microsoft.com/library/ms143726.aspx). Подробные сведения о группах ресурсов Azure см. в статье [Обзор диспетчера ресурсов Azure](resource-group-overview.md).

Вы вернетесь к начальной панели Azure, где на плитке будет отображаться ход выполнения до тех пор, пока база данных не будет создана и не появится в сети. Кроме того, вы можете нажать кнопку **Просмотреть все** и выбрать **Базы данных SQL**, чтобы подтвердить готовность базы данных к работе.
	
Поздравляем! Теперь у вас есть база данных, работающая в облаке. Все почти готово. Остался еще один ключевой шаг. Необходимо создать правило на сервере баз данных, позволяющее вам подключиться к базе данных.

## Шаг 5. Настройка брандмауэра

Необходимо настроить правило брандмауэра на сервере, которое будет разрешать подключения с IP-адреса вашего клиентского компьютера для работы с базой данных. Это не только позволяет убедиться в возможности подключения, но и просмотреть область размещения других сведений о ваших серверах SQL в Azure.

1. Нажмите кнопку **Просмотреть все**, прокрутите вниз и выберите **Серверы SQL Server**, а затем щелкните имя сервера, который вы создали ранее, в списке **Серверы SQL Server**.

	![Выбор сервера базы данных](./media/sql-database-get-started/browse_dbservers.png)

	
3. В колонке свойств базы данных, которая появится справа, выберите **Параметры**, а затем выберите из списка пункт **Брандмауэр**.

	![Открытие параметров брандмауэра](./media/sql-database-get-started/db_settings.png)


	В окне **Параметры брандмауэра** отображается ваш текущий **IP-адрес клиента**.

	![Текущий IP-адрес](./media/sql-database-get-started/firewall_config_client_ip.png)

4. Нажмите **Добавить IP-адрес клиента**, чтобы система Azure создала правило для этого IP-адреса, а затем нажмите кнопку **Сохранить**.

	![Добавление IP-адреса](./media/sql-database-get-started/firewall_config_new_rule.png)

	>[AZURE.IMPORTANT]Ваш IP-адрес, скорее всего, будет периодически меняться, и в таких случаях вы не сможете получить доступ к серверу, пока не создадите новое правило брандмауэра. Вы можете проверить IP-адрес с помощью службы [Bing](http://www.bing.com/search?q=my%20ip%20address), а затем добавить один IP-адрес или диапазон IP-адресов. Для получения подробных сведений см. статью [Настройка параметров брандмауэра](sql-database-configure-firewall-settings.md).

## Дальнейшие действия
Теперь, когда у вас есть база данных с образцами данных, ее можно исследовать с помощью предпочитаемых вами средств.

- Если вы знакомы с языком Transact-SQL и средой SQL Server Management Studio, узнайте о [подключении и запросах к базе данных SQL с помощью среды SSMS](sql-database-connect-query-ssms.md).

- Если вы знаете Excel, узнайте, как [подключиться к базе данных SQL с помощью Excel](sql-database-connect-excel.md).

- Если вы готовы начать писать код, см. статьи [Подключение к базе данных SQL и отправка запросов в нее с помощью языка C#](sql-database-connect-query.md) и [Использование базы данных SQL из .NET (C#)](sql-database-develop-dotnet-simple.md). В дополнение к языку C#, в статье [Простые примеры кода клиентских приложений для работы с Базой данных SQL](sql-database-develop-quick-start-client-code-samples.md) содержатся образцы кода на языках Node.js, Python, Ruby, Java, PHP и C++, а также практические руководства.

- Если вам требуется переместить локальные базы данных SQL Server в Azure, то для получения дополнительных сведений см. статью [Перенос базы данных в базу данных SQL Azure](sql-database-cloud-migrate.md).

# Создание базы данных в Базе данных SQL версии 12


<!--
True author is: authors="sonalmm" , ms.author="sonalm".
-->


Чтобы воспользоваться всеми преимуществами Базы данных SQL нового поколения в Microsoft Azure, [зарегистрируйтесь](https://portal.azure.com) и получите доступ к Базе данных SQL версии 12 [(в некоторых регионах доступна только предварительная версия)](sql-database-v12-whats-new.md#V12AzureSqlDbPreviewGaTable). Для начала работы потребуется подписка Microsoft Azure. Зарегистрируйтесь, чтобы получить [бесплатную пробную версию Azure](http://azure.microsoft.com/pricing/free-trial) и просмотреть [информацию о ценах](http://azure.microsoft.com/pricing/details/sql-database).


| Создание базы данных | Снимок экрана |
| :--- | :--- |
| 1. Войдите на портал [http://portal.azure.com/](http://portal.azure.com/). | ![Новый портал Azure][1] |
| 2. В левом нижнем углу страницы щелкните **Создать**. | ![Запуск новой службы][2]|
| 3. Выберите элемент **База данных SQL**.| ![Выбор из доступных служб][3] |
| 4. Откроется колонка **База данных SQL**. В поле **Имя** укажите имя базы данных. | ![Укажите имя базы данных][4] |
| 5. В колонке **База данных SQL** щелкните **СЕРВЕР**. Откроется колонка **Сервер** с двумя командами: создать новый сервер или использовать существующий.| ![Выберите тип сервера][4] |
|5а. Если вы выбрали параметр **Использовать существующий сервер**, выберите необходимый сервер и нажмите кнопку **Выбрать**. Затем выполните все действия шага 6 далее.|| ![Выберите сервер из списка][5]| 
|5б. При выборе параметра **Создать сервер** откроется колонка **Новый сервер**. Укажите имя сервера, имя для входа администратора сервера и пароль. Щелкните **РАСПОЛОЖЕНИЕ**, чтобы выбрать расположение сервера. | ![Задайте остальные параметры создания нового сервера][9]| 
|5в. В колонке **Новый сервер** можно указать, следует ли создать сервер с учетом обновлений в версии 12. Дополнительную информацию о возможностях серверов версии 12 см. в статье [Новые возможности Базы данных SQL версии 12](sql-database-v12-whats-new.md).| ![Выберите сервер версии 12][6]|
|5г. Задайте необходимые параметры в колонке **Новый сервер** и нажмите кнопку **ОК**. После этого снова откроется колонка **База данных SQL**, где вы сможете завершить создание базы данных. | ![Выполните оставшиеся действия в колонке «Новый сервер»][8]|
|6. Нажмите кнопку **Выбрать источник**. Чтобы создать базу данных, вы можете выбрать различные типы источников, например: пустую базу данных, пример базы данных или резервную копию базы данных.| ![Выберите источник базы данных][10]|
|7. Затем в колонке **База данных SQL** щелкните **ЦЕНОВАЯ КАТЕГОРИЯ**. Вы можете выбрать рекомендуемую ценовую категорию или **просмотреть все** доступные категории. Определившись с выбором, нажмите кнопку **Выбрать**. <p> Дополнительную информацию о ценовых категориях см. в статьях [Обновление Базы данных SQL Web или Business до новых уровней служб](./sql-database-upgrade-new-service-tiers/) и [Уровни служб и уровни производительности в Базе данных SQL Azure](http://msdn.microsoft.com/library/azure/dn741336.aspx). |![Выберите ценовую категорию][7]
| 8. Затем в колонке **База данных SQL** щелкните **ДОПОЛНИТЕЛЬНАЯ НАСТРОЙКА**, выберите необходимые значения и нажмите кнопку **ОК**. 
| 9. При выборе существующего сервера **Группа ресурсов** и **Подписка** выбираются автоматически. В колонке **База данных SQL** рядом с параметрами **Группа ресурсов** и **Подписка** вы увидите значок блокировки. При создании сервера вы можете выбрать существующую группу ресурсов или создать новую. Дополнительную информацию см. в статье [Использование групп ресурсов для управления ресурсами Azure](resource-group-overview.md).|![Укажите группу ресурсов][11]
| 10. Щелкните **Создать**. Будет создана база данных с поддержкой возможностей Базы данных SQL версии 12. |![Создание новой базы данных][12]

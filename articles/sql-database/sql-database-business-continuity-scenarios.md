# Сценарии обеспечения непрерывности бизнес-процессов для базы данных SQL Azure

В этой статье представлено несколько сценариев аварийного восстановления и несколько сценариев разработки приложений для обеспечения непрерывности бизнес-процессов.

## Восстановление после сбоя

В случае сбоя вы можете ознакомиться с разделом [Восстановление базы данных SQL Azure после сбоя](sql-database-disaster-recovery.md), в котором описывается, как использовать любое из следующих решений для обеспечения непрерывности бизнес-процессов, чтобы выполнить восстановление после сбоя.

- [Активная георепликация](sql-database-geo-replication-overview.md)
- [Геовосстановление](sql-database-recovery-using.backups.md#geo-restore)

Конкретные действия и время, необходимое для восстановления после сбоя, зависят от выбранного решения для обеспечения непрерывности бизнес-процессов. Тем не менее вне зависимости от того, какое используется решение, необходимо знать, когда начать восстановление базы данных и как его выполнить для каждого решения для обеспечения непрерывности бизнес-процессов, а также как настроить базу данных и приложение после восстановления, чтобы завершить восстановление после сбоя.

## Восстановление после ошибки

В случае ошибки пользователя или другого непреднамеренного изменения данных по ошибке вы можете ознакомиться с разделом [Восстановление базы данных SQL Azure после ошибки пользователя](sql-database-user-error-recovery.md), в котором описывается, как использовать любое из следующих решений для обеспечения непрерывности бизнес-процессов, чтобы выполнить восстановление после ошибки.

- [Восстановление до точки во времени](sql-database-recovery-using-backups.md#point-in-time-restore)
- [Восстановление удаленной базы данных](sql-database-recovery-using-backups.md#deleted-database-restore)

Конкретные действия и время, необходимое для восстановления после сбоя, зависят от выбранного решения для обеспечения непрерывности бизнес-процессов. Тем не менее вне зависимости от используемого решения необходимо знать, как выполнить восстановление после ошибки с помощью каждого решения для обеспечения непрерывности бизнес-процессов.

## Отработка аварийного восстановления для подготовки к сбою

Чтобы обеспечить эффективность решения для обеспечения непрерывности бизнес-процессов, рекомендуется периодически проверять готовность приложения к рабочему процессу восстановления. Проверка поведения приложения и возможных последствий потери данных и нарушений работы, которая включает отработку отказа, предусмотрена стандартами разработки. Эта процедура также является обязательной для большинства отраслевых стандартов в рамках сертификации непрерывности бизнес-процессов.

Этапы отработки аварийного восстановления:

- моделирование сбоя уровня данных;
- восстановление;
- проверка целостности приложения после восстановления.

В разделе [Отработка аварийного восстановления](sql-database-disaster-recovery-drills.md) описывается, как выполнить аварийное восстановление с помощью любого из следующих решений для обеспечения непрерывности бизнес-процессов.

- [Активная георепликация](sql-database-geo-replication-overview.md)
- [Геовосстановление](sql-database-recovery-using-backups.md#geo-restore)

## Управление последовательными обновлениями для облачных приложений с помощью активной георепликации

Обновление облачного приложения с базой данных SQL является операцией, нарушающей работу, поэтому этот сценарий необходимо учесть при планировании и проектировании непрерывности бизнес-процессов. В разделе [Управление последовательными обновлениями для облачных приложений с помощью активной георепликации базы данных SQL](sql-database-manage-application-rolling-upgrade.md) описывается, как использовать [георепликацию](sql-database-geo-replication-overview.md) в базе данных SQL, чтобы обеспечить установку последовательных обновлений облачного приложения. В этой статье рассматриваются два разных метода управления процессом обновления, а также описываются преимущества и недостатки каждого из них.

## Проектирование приложения для облачного аварийного восстановления с использованием активной георепликации

В разделе [Создание приложения для аварийного восстановления облака с использованием активной георепликации в базе данных SQL](sql-database-designing-cloud-solutions-for-disaster-recovery.md) описывается использование [активной георепликации](sql-database-geo-replication-overview.md) в базе данных SQL для проектирования приложений базы данных, которые устойчивы к региональным сбоям и разрушительным простоям. В этой статье рассматривается топология развертывания приложения, целевое соглашение об уровне обслуживания, задержка трафика и издержки, а затем приводятся распространенные шаблоны приложений, каждый из которых имеет преимущества и компромиссы.

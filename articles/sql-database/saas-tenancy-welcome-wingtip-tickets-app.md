# <a name="welcome-to-the-wingtip-tickets-sample-saas-azure-sql-database-tenancy-app"></a>Знакомство с SaaS-приложением Wingtip Tickets для размещения клиентов в службе "База данных SQL Azure"

Знакомство с SaaS-приложением Wingtip Tickets для размещения клиентов в службе "База данных SQL Azure" и список руководств по этому приложению. Аренда базы данных — это режим изоляции данных, который ваше приложение поддерживает для клиентов, размещаемых в приложении на платной основе. Есть две основных модели аренды: каждый клиент получает для себя отдельную базу данных или использует базу данных совместно с другими клиентами.

## <a name="wingtip-tickets-app"></a>Приложение Wingtip Tickets

Пример приложения Wingtip Tickets демонстрирует воздействие выбранной модели аренды на процессы разработки и управления для мультитенантных SaaS-приложений. Это воздействие описано в соответствующих руководствах. Приложение Wingtip Tickets создано на основе службы "База данных SQL Azure".

Wingtip Tickets поддерживает несколько сценариев разработки и управления, которые часто применяются для реальных клиентов SaaS-приложений. В этом примере учтены реальные схемы работы, которые постепенно сформировались для Wingtip Tickets.

Вы можете установить Wingtip Tickets в подписке Azure буквально за пять минут. В процессе установки вносятся тестовые данные для нескольких клиентов. Вы без проблем сможете установить приложение и скрипты управления для всех моделей работы одновременно, поскольку они никак не взаимодействуют друг с другом.

#### <a name="code-in-github"></a>Код в репозитории Github

Код приложения и скрипты управления размещены в репозитории GitHub.

- Модель **отдельного приложения**: [репозиторий WingtipTicketsSaaS-StandaloneApp](https://github.com/Microsoft/WingtipTicketsSaaS-StandaloneApp)
- Модель **с однотенантной БД**: [репозиторий WingtipTicketsSaaS-DbPerTenant](https://github.com/Microsoft/WingtipTicketsSaaS-DbPerTenant).
- Модель **с сегментированной мультитенантной БД**: [репозиторий WingtipTicketsSaaS-MultiTenantDB](https://github.com/Microsoft/WingtipTicketsSaaS-MultiTenantDB).

Для всех перечисленных выше моделей приложения Wingtip Tickets используется одна и та же база кода. Вы также можете применить код из Github для создания собственных проектов SaaS.



## <a name="major-database-tenancy-models"></a>Основные модели аренды базы данных

SaaS-приложение Wingtip Tickets предназначено для ведения списка событий и продажи билетов. Wingtip предоставляет службы для работы объектов. Для каждого из объектов справедливы следующие утверждения:

- Они оплачивают размещение в приложении.
- Они являются *клиентами* в приложении Wingtip.
- Они проводят мероприятия. Схема работы включает следующие концепции:
    - Цены на билеты.
    - Продажа билетов.
    - Пользователи, приобретающие билеты.

Приложение и прилагающиеся к нему скрипты управления и руководства демонстрируют полнофункциональный сценарий службы SaaS. Этот сценарий включает следующие действия:

- Подготовка клиентов.
- Мониторинг производительности и управление ею.
- Управление схемами.
- Отчеты и аналитика по всем клиентам.

Все эти действия можно выполнять в любом масштабе по мере необходимости.



## <a name="code-samples-for-each-tenancy-model"></a>Примеры кода для каждой модели аренды

В этих примерах специально выделены разные модели приложения. Но в реальной реализации могут сочетаться элементы из двух или нескольких моделей.

#### <a name="standalone-app-model"></a>Модель отдельного приложения

![Модель отдельного приложения][standalone-app-model-62s]

В этой модели приложение работает с одним клиентом. Это означает, что ему требуется только одна база данных, в которой хранятся данные только одного клиента. Такой клиент полностью изолирован от других.

Эту модель можно использовать, если вы продаете экземпляры приложения разным клиентам, каждый из которых самостоятельно эксплуатирует его. Соответственно, каждый из них будет единственным клиентом в своем экземпляре. В каждой базе данных размещается только один клиент, но хранится информация о множестве пользователей этого клиента.

#### <a name="database-per-tenant"></a>Однотенантная база данных

![Модель с однотенантной БД][database-per-tenant-model-35d]

В этой модели каждый экземпляр приложения работает для нескольких клиентов. При этом для каждого нового клиента выделяется новая база, которую будет использовать только этот клиент.

Такая модель обеспечивает полную изоляцию базы данных для каждого клиента. Служба "База данных SQL Azure" поддерживает все возможности, необходимые для реализации такой модели.

- Дополнительные сведения об этой модели аренды см. в статье о [примере мультитенантного SaaS-приложения для базы данных SQL][saas-dbpertenant-wingtip-app-overview-15d].

#### <a name="sharded-multi-tenant-databases-the-hybrid"></a>Гибридная модель с сегментированной мультитенантной БД

![Гибридная модель с сегментированной мультитенантной БД][sharded-multitenantdb-model-hybrid-79m]

В этой модели каждый экземпляр приложения работает для нескольких клиентов. При этом в некоторых (или всех) базах данных размещается по несколько клиентов. Эта модель удобна, если нужно поддерживать несколько уровней обслуживания и клиенты готовы оплачивать преимущества полной изоляции базы данных.

Схема каждой базы данных содержит идентификатор клиента. Идентификатор клиента есть даже в тех базах данных, в которых размещается только один клиент.

# Восстановление базы данных SQL Azure с использованием географического восстановления на портале Azure

> [AZURE.SELECTOR]
- [Geo-Restore - PowerShell](sql-database-geo-restore-tutorial-powershell.md)
- [Geo-Restore - REST API](sql-database-geo-restore-tutorial-rest.md)   

## Обзор

В этом учебнике показано, как восстановить базу данных SQL Azure с помощью географического восстановления на [портале Azure](http://manage.windowsazure.com). Географическое восстановление — это основной механизм защиты для аварийного восстановления баз данных SQL Azure на всех уровнях обслуживания — Basic, Standard и Premium.

## Ограничения и безопасность

* Функция географического восстановления включена для уровней Basic, Standard и Premium.

* На уровне Basic резервные копии хранятся в течение периода в 7 дней, на уровне Standard — 14 дней, а на уровне Premium — 35 дней.

* Только администратор или соадминистратор подписки может отправлять запросы на восстановление.

* Имя для входа субъекта серверного уровня будет соответствовать владельцу восстановленной базы данных.

* Уровни обслуживания выпусков Web и Business не поддерживают географическое восстановление.
 
	* При наличии базы данных выпуска Web или Business вы можете использовать копию базы данных для получения такой копии базы данных, в которой будут содержаться транзакции, а затем экспортировать скопированную базу данных в учетную запись хранения Microsoft Azure. Дополнительную информацию см. в разделе [Инструкции по использованию копии базы данных (база данных SQL Azure)](http://msdn.microsoft.com/library/azure/ff951631.aspx) и [Инструкции по использованию службы импорта и экспорта в базе данных Azure SQL](http://msdn.microsoft.com/library/azure/hh335292.aspx).

	* В сентябре 2015 года поддержка выпусков Web и Business будет прекращена. Дополнительные сведения см. в статье [Вопросы и ответы по остановке Web Edition и Business Edition](http://msdn.microsoft.com/library/azure/dn741330.aspx).

## Практическое руководство. Восстановление базы данных SQL Azure с использованием географического восстановления на портале Azure

> [AZURE.VIDEO restore-a-sql-database-using-geo-restore]

1. Войдите на [портал Azure](http://manage.windowsazure.com), используя учетную запись Майкрософт, и выберите **Базы данных SQL**.

2. В левой области навигации щелкните **БАЗЫ ДАННЫХ SQL**.

3. В верхней части страницы щелкните ссылку **СЕРВЕРЫ**.

4. В списке **СЕРВЕРЫ** выберите сервер **С ограниченной функциональностью**.

4. В верхней части страницы щелкните ссылку **РЕЗЕРВНЫЕ КОПИИ**.

5. Щелкните по базе данных, которую хотите восстановить.

6. В нижней части страницы на панели команд щелкните **Восстановление**. Это откроет диалоговое окно **Указать параметры восстановления**.

7. Введите значения в полях **ИМЯ БАЗЫ ДАННЫХ** и **ЦЕЛЕВОЙ СЕРВЕР** для восстановления базы данных. По умолчанию имя базы данных выбирается автоматически, но при желании его можно изменить.

9. Установите флажок, чтобы отправить запрос на восстановление.

Восстановление может занять некоторое время. Для мониторинга состояния восстановления щелкните значок «Состояние» в нижнем правом углу страницы на панели команд и нажмите кнопку **ПОДРОБНОСТИ**.

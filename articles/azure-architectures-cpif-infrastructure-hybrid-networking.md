# Гибридные сети (шаблоны архитектуры Azure)

[Инфраструктура интеграции облачных платформ (CPIF)](azure-architectures-cpif-overview.md) предоставляет инструкции по интеграции рабочей нагрузки для адаптации приложений в облачном решении корпорации Майкрософт.  

Инфраструктура интеграции облачных платформ описывает способы, которые следует использовать организациям, клиентам и партнерам для разработки и развертывания предназначенных для облака рабочих нагрузок с помощью гибридной облачной платформы и возможностей управления Azure, System Center и Windows Server. 

Шаблон **гибридных сетей**  является частью области **инфраструктуры**, подробно описанной в документе об архитектуре CPIF. 

##  Гибридные сети

Шаблон разработки гибридных сетей содержит сведения о компонентах и службах Azure, необходимых для предоставления сетевой функциональности, обеспечивающей предсказуемый уровень производительности и высокую доступность во всех географических регионах.  Полный список регионов и доступных в них служб Microsoft Azure представлен на сайте документации по Microsoft Azure.  Этот документ содержит обзор сетевых возможностей Microsoft Azure для гибридной среды. Виртуальные сети Microsoft Azure позволяют создавать логически изолированные сети в Azure и безопасно подключать их к локальному центру обработки данных по Интернету или с помощью подключения к частной сети.  Кроме того, индивидуальные компьютеры клиентов могут подключаться к изолированной сети Azure с помощью VPN-подключения IPsec.  

Шаблон архитектуры гибридных сетей включает следующие приоритетные области: 

- подключение локальных сетей к Azure 
- расширение виртуальных сетей Azure по регионам 
- расширение виртуальных сетей Azure между подписками 
- обеспечение удаленного доступа к сети для разработчиков 

## Обзор шаблона архитектуры 

Шаблон архитектуры гибридных сетей является сложным из-за возможного числа сценариев, которые можно создать. Этот шаблон архитектуры фокусируется на следующих четырех сценариях: 

- Гибридные сети типа "сеть-сеть" с маршрутизацией виртуальной сети в несколько прыжков в пределах одной подписки и региона 
- Гибридные сети типа "сеть-сеть" с маршрутизацией виртуальной сети в несколько прыжков для нескольких подписок и регионов 
- Гибридные сети ExpressRoute с использованием подключения MPLS 
- Гибридные сети ExpressRoute с использованием подключения IXP

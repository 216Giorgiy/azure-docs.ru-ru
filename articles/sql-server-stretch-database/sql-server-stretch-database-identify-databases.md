# Определение баз данных и таблиц для растяжения с помощью Помощника по растяжению баз данных

Для определения баз данных и таблиц, которые подходят для растяжения баз данных, скачайте Помощник по обновлению SQL Server 2016 и запустите Помощник по растяжению баз данных. Помощник по растяжению баз данных также определит критические препятствия.

## Скачивание и установка Помощника по обновлению
Скачайте и установите Помощник по обновлению [здесь](http://go.microsoft.com/fwlink/?LinkID=613421). Это средство не включено в состав установочного носителя SQL Server.

## Запуск Помощника по растяжению баз данных

1.  Запустите Помощник по обновлению.

2.  Выберите **Сценарии**, а затем **ЗАПУСТИТЬ ПОМОЩНИК ПО РАСТЯЖЕНИЮ БАЗ ДАННЫХ**.

3.  В колонке **Запуск Помощника по растяжению баз данных** щелкните **ВЫБРАТЬ БАЗЫ ДАННЫХ ДЛЯ АНАЛИЗА**.

4.  В колонке **Выбор баз данных** щелкните **ЭКЗЕМПЛЯР SQL SERVER**.

5.  В колонке **Подключение к экземпляру SQL Server** введите имя экземпляра SQL Server. Щелкните **Подключить**.

6.  В колонке **Выбор баз данных** выберите базы данных для анализа. Затем щелкните **Выбрать**.

7.  В колонке **Запуск Помощника по растяжению баз данных** щелкните **Запустить**. Будет запущен анализ.

## Просмотр результатов

1.  После завершения анализа в колонке **Помощник по растяжению баз данных** выберите одну из баз данных, которые были проанализированы, для отображения колонки **Результаты анализа**.

    В колонке **Результаты анализа** будут перечислены рекомендуемые таблицы в выбранной базе данных, которые соответствуют рекомендуемым критериям по умолчанию. При необходимости измените минимальный размер и количество строк, чтобы расширить или сократить список рекомендуемых таблиц.

2.  В списке рекомендуемых таблиц в колонке **Результаты анализа** выберите одну из рекомендуемых таблиц для отображения колонки **Результаты для таблицы**.

    В колонке **Результаты для таблицы** перечислены критические препятствия для выбранной таблицы. Сведения о критических препятствиях, обнаруженных Помощником по растяжению баз данных, см. в разделе [Ограничения для областей и критические препятствия для растяжения баз данных](sql-server-stretch-database-limitations.md).

3.  В списке критических препятствий в колонке **Результаты для таблицы** выберите одну из проблем, чтобы просмотреть колонку **Результат правила**.

    В колонке **Результат правила** описана выбранная проблема и предложены действия по ее устранению. Выполните действия по устранению проблемы, если хотите настроить выбранную таблицу для растяжения базы данных.

## Дальнейшие действия
Включите растяжение базы данных.

-   Чтобы включить растяжение базы данных для **базы данных**, обратитесь к разделу [Включение растяжения базы данных для базы данных](sql-server-stretch-database-enable-database.md).

-   Чтобы включить растяжение базы данных для другой **таблицы**, если оно уже включено для базы данных, обратитесь к разделу [Включение растяжения базы данных для таблицы](sql-server-stretch-database-enable-table.md).

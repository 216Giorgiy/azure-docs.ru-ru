# Ограничения контактной зоны и критические препятствия для использования базы данных Stretch

Узнайте о критических препятствиях, которые необходимо устранить перед включением базы данных Stretch.

## <a name="Limitations"></a>Критические препятствия
В текущей предварительной версии SQL Server 2016 перечисленные ниже элементы делают таблицу недоступной для использования Stretch.

**Свойства таблицы**
-   Более 1023 столбцов

-   Более 998 индексов

-   Таблицы, содержащие данные FILESTREAM

-   Таблицы файлов

-   Реплицированные таблицы

-   Таблицы, активно использующие отслеживание изменений или запись измененных данных

-   Оптимизированные для памяти таблицы

**Типы данных и свойств столбца**
-   Timestamp

-   sql\_variant

-   XML

-   geometry

-   geography

-   hierarchyid

-   Определяемые пользователем типы данных CLR (UDT)

**Типы столбцов**
-   COLUMN\_SET

-   вычисляемые столбцы;

**Ограничения**
-   проверочные ограничения;

-   Ограничения по умолчанию

-   Ограничения внешнего ключа, ссылающегося на таблицу

**Индексы**
-   Полнотекстовые индексы

-   XML-индексы

-   Пространственные индексы

-   Индексированные представления, которые ссылаются на таблицу

## <a name="Caveats"></a>Ограничения и предупреждения для таблиц, совместимых со Stretch
В текущей предварительной версии SQL Server 2016 имеются следующие ограничения или предупреждения для таблиц, совместимых со Stretch.

-   В таблице, совместимой со Stretch, не обеспечивается уникальность для ограничений UNIQUE и PRIMARY KEY.

-   В таблице, совместимой со Stretch, нельзя выполнять операции UPDATE и DELETE.

-   Нельзя удаленно выполнить операцию INSERT с таблицей, совместимой со Stretch, на связанном сервере.

-   Нельзя использовать репликацию таблицы, совместимой со Stretch.

-   Нельзя создать индекс для представления, включающего в себя таблицу, совместимую со Stretch.

-   Нельзя обновить или удалить данные представления, включающего в себя таблицу, совместимую со Stretch. Однако в представление, включающее в себя таблицу, совместимую со Stretch, можно вставить данные.

-   Фильтры индексов не распространяются на удаленную таблицу.

# Использование групп ресурсов для управления ресурсами Azure

### Введение

Исторически управление ресурсом (управляемой пользователем сущностью, например сервером баз данных, базой данных или веб-сайтом) в Microsoft Azure всегда подразумевало выполнение операций только с одним ресурсом одновременно. При использовании сложного приложения, состоящего из нескольких ресурсов, управление этим приложением становится сложной задачей. На портале Microsoft Azure (предварительная версия) можно создать группы ресурсов для совместного управления всеми ресурсами приложения. Группа ресурсов - это новое понятие в Azure, служащее границей жизненного цикла для каждого ресурса, содержащегося в этой группе. 
<br><br />

Группы ресурсов позволяют одновременно управлять всеми ресурсами приложения. Группы ресурсов становятся доступны благодаря новому средству управления - диспетчеру ресурсов Azure. Диспетчер ресурсов Azure позволяет сгруппировать несколько ресурсов в логическую группу, которая служит границей жизненного цикла для каждого ресурса, содержащегося в группе. Обычно группа будет содержать ресурсы, относящиеся к конкретному приложению. Например, в группе может содержаться ресурс веб-сайта, на котором размещены открытый веб-сайт, база данных SQL для хранения используемых сайтом реляционных данных и учетная запись хранилища, в которой хранятся нереляционные активы. 

Вот краткий обзор использования групп ресурсов на портале Microsoft Azure (предварительная версия). 
<br><br />

### Создание групп ресурсов

При создании ресурса на портале предварительной версии этот ресурс всегда создается в составе группы ресурсов. Можно создать новую группу ресурсов или использовать существующую группу ресурсов при создании потока. <br><br />

![](http://i.imgur.com/USKkQdW.png)

<br><br />
При создании приложения, состоящего из нескольких совместно работающих ресурсов (например, веб-сайт + база данных), оно всегда создается в собственной группе ресурсов, что позволяет управлять жизненным циклом всех связанных с приложением ресурсов с помощью группы ресурсов. По мере развития приложения можно добавлять дополнительные ресурсы или удалять их из группы ресурсов. 

![](http://i.imgur.com/Me0jbio.png)


<br><br />

### Просмотр групп ресурсов

Все группы ресурсов можно просмотреть, щелкнув панель перехода в левой части экрана. Группа ресурсов содержит выноску со всеми сведениями о конкретной группе ресурсов. Модуль группы ресурсов также обеспечивает единое представление информации о выставлении счетов и мониторинге для всех ресурсов в группе.

Раздел сводки содержит визуальную схему всех ресурсов в группе, а также показывает ресурсы в других группах ресурсов, связанных с этой группой ресурсов. На схеме ресурсов также показывается состояние каждого ресурса. 
![](http://i.imgur.com/PhJeLZQ.png)

<br><br />

Для части схемы ресурсов можно увеличить размер отображения и отобразить все ресурсы, содержащиеся в группе ресурсов, а также связанные ресурсы из других групп. Эту часть можно прикрепить к начальной панели, тем самым скопировав ее в панель.

![](http://i.imgur.com/5Wqv2XR.png)

<br><br />

  Щелкнув схему ресурсов, можно открыть список ресурсов. В этом представлении будут перечислены все ресурсы в группе ресурсов или связанные с ней. При щелчке на любом из этих ресурсов откроется соответствующий модуль. 

![](http://i.imgur.com/COPjNng.png)




<br><br />

### Добавление ресурсов в группы ресурсов

Ресурсы в группу ресурсов можно добавить с помощью команды "Добавить" в модуле группы ресурсов. Для добавления других ресурсов в группу необходимо выполнить такую же последовательность действий.

![](http://i.imgur.com/G79kayH.png)

Примечание. Не рекомендуется помещать командный проект в одну группу с другими ресурсами Azure. В случае создания командного проекта в новой учетной записи и группе необходимо создать веб-сайт. По умолчанию группой сайта станет последняя используемая группа (VSO), а ресурсы узлов выполнения и ресурсы разработчика будут добавлены в эту же группу. 



<br><br />

### Удаление групп ресурсов

Так как группы ресурсов позволяют управлять жизненным циклом всех содержащихся в них ресурсов, то при удалении группы будут удалены все ее ресурсы. Из группы ресурсов можно также удалить отдельные ресурсы. При удалении группы ресурсов необходимо соблюдать осторожность, поскольку с этой группой могут быть связаны другие ресурсы. Вы можете просмотреть связанные ресурсы на схеме ресурсов и принять соответствующие меры, чтобы избежать нежелательных последствий удаления группы ресурсов.

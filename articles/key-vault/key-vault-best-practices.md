# <a name="best-practices-to-use-key-vault"></a>Советы и рекомендации для использования хранилища ключей

## <a name="control-access-to-your-vault"></a>Управление доступом к хранилищу

Azure Key Vault — это облачная служба, которая обеспечивает защиту ключей шифрования и секретов (например, сертификатов, строк подключения и паролей). Так как это критически важные для бизнеса конфиденциальные данные, следует обеспечить защиту доступа к хранилищу ключей, чтобы доступ могли получить только авторизованные приложения и пользователи. Это [статье](key-vault-secure-your-key-vault.md) Обзор модели доступа к хранилищу ключей. Здесь приводится описание процессов аутентификации и авторизации, а также содержатся сведения о том, как защитить доступ к вашему хранилищу ключей.

Ниже приведены варианты при управлении доступом к хранилищу.
1. Заблокируйте доступ к подписке, группе ресурсов и хранилищ ключей (RBAC)
2. Создание политики доступа для каждого хранилища
3. Использование минимальных прав доступа участника для предоставления доступа
4. Включить брандмауэр и [конечные точки службы виртуальной сети](key-vault-overview-vnet-service-endpoints.md)

## <a name="use-separate-key-vault"></a>Использовать отдельное хранилище ключей

Мы рекомендуем использовать хранилище на приложение и на среду (разработки, предварительной и рабочей). Это помогает не общие секретные данные в средах и также уменьшает опасность в случае нарушения безопасности.

## <a name="backup-your-vault"></a>В хранилище архивации

Убедитесь, что вы приняли regular, обратно, устраняющие вашей [хранилище](https://blogs.technet.microsoft.com/kv/2018/07/20/announcing-backup-and-restore-of-keys-secrets-and-certificates/) на обновления или удаления и создания объектов в хранилище.

## <a name="turn-on-logging"></a>Включите ведение журнала

[Включите ведение журнала](key-vault-logging.md) для хранилища. Также настройте оповещения.

## <a name="turn-on-recovery-options"></a>Включите параметры восстановления

1. Включите [обратимое удаление](key-vault-ovw-soft-delete.md).
2. Включение защиты от очистки, чтобы защититься от принудительного удаления секрета / хранилище даже в том случае, после включения обратимого удаления.

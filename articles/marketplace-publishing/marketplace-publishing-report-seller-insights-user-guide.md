# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>Как использовать портал Microsoft Azure Marketplace Seller Insights
##<a name="onboarding"></a>Переход

>[!NOTE]
>Этот раздел предназначен только для администраторов.

###<a name="overview-of-roles"></a>Общие сведения о ролях

![тип учетной записи][5]

Администратор портала Seller Insights может добавлять, изменять и удалять пользователей.

Пользователь портала Seller Insights может просматривать все данные, доступные вашей организации.

Администраторы и пользователи должны проходить проверку подлинности с помощью действительной учетной записи Майкрософт (LiveID http://signup.live.com) или учетной записи своей организации. Введенный идентификатор должен быть связан с отслеживаемым почтовым ящиком.

###<a name="invitation-to-activate-your-account"></a>Приглашение для активации учетной записи

В идеале перед выполнением входа на портал Seller Insights администратору необходимо определить учетные записи тех лиц, которых необходимо сделать пользователями. Это могут быть учетные записи Майкрософт или учетные записи организации.

Для активации службы администратор должен перейти по URL-адресу, указанному в приглашении, полученному по электронной почте (https://reports.azure.com).

>[!NOTE]
>Примечание. Если в качестве учетной записи Майкрософт указана электронная почта администратора, необходимо создать учетную запись Майкрософт, связанную с адресом электронной почты, по которому вы получили приглашение. Это нужно сделать перед выполнением входа, перейдя по адресу https://signup.live.command

###<a name="sign-in-to-the-portal"></a>Выполнение входа на портал

1. Войдите на портал Seller Insights, щелкнув URL-адрес в сообщении электронной почты, которое вы получили.
2. Введите адрес электронной почты и пароль администратора.
3. Нажмите кнопку входа.
4. Начните администрирование учетной записи. Настройте разрешения, добавьте или измените пользователей.
5. Не забудьте предоставить пользователям URL-адрес портала.

![страница входа][6]

##<a name="signing-in"></a>Вход

Администратор должен отправить вам сообщение о создании учетной записи и предоставить URL-адрес портала Seller Insights (https://reports.azure.com).

>[!NOTE]
>Администратор является владельцем учетной записи, которая фактически публикует решения в Azure Marketplace.  Владелец этой учетной записи может предоставлять доступ другим пользователям.  

Вы можете войти на портал Seller Insights с помощью личной учетной записи Майкрософт или с помощью учетной записи компании в зависимости от того, какую учетную запись администратор использовал при настройке вашего профиля.  Администратор должен сообщить вам, какую учетную запись следует использовать.

![тип учетной записи][1]



***Добро пожаловать на портал Seller Insights!***

1. После входа откроется страница "Сводка".
2. Ознакомьтесь с политикой конфиденциальности.
3. Вы в любое время можете связаться с сотрудниками службы поддержки по вопросам, связанным с работой портала Seller Insights.
4. Для перемещения по порталу Seller Insights предназначены вкладки в верхней части страницы.  Вкладки в верхней части страниц в этом руководстве помогут вам разобраться с функциональностью портала Seller Insights.
5. Щелкните смайлик, чтобы в любое время отправить отзыв о портале Seller Insights.
6. Щелкните свое имя в правом верхнем углу, чтобы в любое врем выйти с портала Seller Insights.

![Главная страница по умолчанию][2]


##<a name="summary-view"></a>Представление сводки

1. Щелкните вкладку "Сводка".
2. Введите необходимую дату начала периода и щелкните "Применить". Отобразятся данные за 3 месяца.
3. Отобразятся данные о тренде новых заказов за выбранный период (3 месяца).
4. Новые заказы за выбранный период будут показаны здесь по каналу и типу предложения.
5. Выберите на этой панели представление пяти первых рынков, пяти первых выполняющихся предложений или пяти первых клиентов.
6. Здесь будет показана схема тренда для оплаченных предложений за выбранный период. При наличии только предложений с использованием собственной лицензии или предложений, предлагаемых непосредственно Майкрософт, эта панель не отображается.

![Представление сводки][7]

##<a name="orders-and-usage"></a>Заказы и использование
###<a name="overview"></a>Обзор

О. На *панели A* можно выбрать вкладку заказов и использования.  Здесь можно выбрать "Заказы" или "Использование" на вкладках "Тип отчета", а также выбрать интересующие данные с помощью ползунков.

B. На *панели B* можно выбрать ежемесячные данные о заказах или об использовании в зависимости от выбора, сделанного на вкладке "Тип отчета" на панели A. Представленные данные будут соответствовать введенным датам и фильтрам, примененным с помощью ползунков на панели A.

C. На *панели С* можно выбрать ежемесячные данные о тренде за 3 месяца о заказах или об использовании в зависимости от выбора, сделанного во вкладке "Тип отчета" на панели A. Представленные данные будут отражать введенные даты и фильтры, примененные с помощью ползунков на панели A.

D. На *панели D* представлены данные, связанные с каждой отдельной подпиской.  Данные на этой панели можно скачать в формате Excel или CSV.

###<a name="orders"></a>Заказы
####<a name="monthly-view---panel-a"></a>Представление за месяц. Панель A

![Карта панели А][10]

  1. Перейдите на вкладку заказов и использования в верхней строке для доступа к данным о заказах и использовании.
  2. Выберите "Заказы" на вкладке "Тип отчета".
  3. Используйте символ фильтра, чтобы отобразить или скрыть срезы (расширенный фильтр).  Используйте эти срезы для фильтрации данных: по типу лицензии Azure, типу лицензии Marketplace, рынку, типу предложения и состоянию предварительной версии.  Определения этих терминов содержатся на вкладке "Справка".
  4. Выберите необходимую дату начала и нажмите кнопку "Перейти".

![Представление заказов за месяц. Панель A][14]

####<a name="monthly-view---panel-b"></a>Представление за месяц. Панель В

  ![Карта панели В][11]

  1. Выберите Monthly View (Представление за месяц).
  2. Отобразится поканальная сводка по заказам: CSP, прямое соглашение Enterprise, соглашение Enterprise через посредника и Web Direct.
  3. Для представления за месяц на панели Order Summary (Сводка по заказам) доступны следующие диаграммы:
    - первые 10 заказов по рынкам;
    - первые 10 заказов по типам предложений;
    - заказы по типам лицензий Azure;
    - заказы по типам лицензий Marketplace.

![Представление заказов за месяц. Панель В][15]

####<a name="trend-view---panel-c"></a>Представление тренда. Панель С

  ![Карта панели С][12]

  1. Выберите Trend View (Представление тренда).
  2. Выберите элементы, которые требуется просмотреть, в раскрывающемся списке "Предметная область" и нажмите кнопку "Применить".  Значение по умолчанию — "Выделить все".

  3. Используйте символ фильтра для выбора всех данных или первых 10 для всех выбранных элементов.  Значение по умолчанию — "Первые 10".

  ![Символ фильтра месячного представления заказов на панели C][18]
  4. Наведите курсор на каждый столбик на выбранных диаграммах, чтобы увидеть базовые данные.
  5. Переключите символ "+", чтобы отобразить данные, на основе которых созданы диаграммы, выбранные для просмотра.

![Представление тренда заказов. Панель С][16]


####<a name="detailed-orders-data---panel-d"></a>Подробные данные о заказах. Панель D

  ![Карта панели D][13]

  1. На панели Order Detail (Сведения о заказе) представлены подробные данные о заказе.
  2. Используйте фильтры в верхней части каждого столбца, чтобы отфильтровать данные с помощью широкого набора команд и значений.

  4. Скачайте данные в формате Excel или CSV для автономного анализа.

##<a name="user-management"></a>Управление пользователями

>[!NOTE]
>Этот раздел предназначен только для администраторов.

###<a name="add-user"></a>Добавить пользователя

1. Перейдите на вкладку "Управление пользователями".
2. Чтобы добавить пользователя, выберите на этой вкладке "Добавить пользователя".
3. Введите адрес электронной почты пользователя, связанный с учетной записью Майкрософт или с учетной записью организации, и нажмите кнопку "Продолжить".
4. В качестве администратора можно назначить пользователям доступ к отчету с правами "только для чтения".
5. Введите данные пользователя (имя, фамилию и электронную почту) и нажмите кнопку "Продолжить".
6. Выберите разрешения, которые нужно назначить пользователю, и нажмите кнопку "Продолжить".
7. Проверьте введенные данные и нажмите кнопку "Сохранить".  Теперь пользователь успешно добавлен.

![Панель добавления пользователей][29]


###<a name="edit-user"></a>Изменение пользователя

1. Щелкните вкладку "Управление пользователями", а затем выберите "Изменить пользователей".
2. Найдите пользователя, разрешения которого требуется изменить.  Обратите внимание, что на этой вкладке вы также можете добавить пользователей.
3. Щелкните разрешения, которые требуется удалить для пользователя.  Обратите внимание, что на этой вкладке вы также можете удалить пользователя.
4. Чтобы продолжить, нажмите кнопку "Сохранить".  Разрешения пользователя успешно изменены.

![Панель изменения пользователей][30]

##<a name="support-requests"></a>Запросы в службу поддержки


###<a name="access-the-support-request-format"></a>Доступ к форме запроса в службу поддержки

1. Перейдите на вкладку "Справка".
2. Выберите "Поддержка и документация", а затем "Отправить запрос".
3. Также можно отправить запрос в службу поддержки с помощью ссылки "Свяжитесь с нами" внизу каждого экрана приложения.
4. Откроется новая вкладка, где вы сможете отправить запрос в службу поддержки.

![Доступ к форме запроса в службу поддержки][31]

###<a name="review-helpful-information"></a>Просмотр полезных сведений

1. Чтобы отправить в службу поддержки запрос, связанный с отчетами, выберите Reports and Insights (Отчеты и аналитика) в раскрывающемся списке "Тип проблемы".
2. Выберите категорию.  Доступные параметры: Questions related to reports (Вопросы, связанные с отчетами) и "Другое".
3. Отобразятся полезные ссылки, которые могут помочь найти ответ на ваш запрос.  Ознакомьтесь с ними перед тем, как отправить запрос в службу поддержки.
4. Если вам все еще необходимо отправить запрос в службу поддержки, выберите "Отправить запрос".

![Панель изменения пользователей][32]


###<a name="fill-out-and-submit-the-request"></a>Заполнение и отправление запроса

1. Введите контактные данные, часовой пояс, регион и языковые параметры и нажмите кнопку "Продолжить".
2. Дайте название проблеме и укажите как можно больше дополнительных сведений.  Также вы можете передать вспомогательные файлы снимков экрана и т. д., после чего нажмите кнопку "Отправить".
3. Вы получите подтверждение, что запрос был успешно отправлен.
4. Вы можете проверить состояние запроса в службу поддержки в любое время.

![Доступ к форме запроса в службу поддержки][33]


##<a name="resetting-a-password"></a>Сброс пароля

***Проверка учетной записи и пароля***

1. ***Проверьте, что учетная запись, которую вы используете, зарегистрирована и пароль не работает.*** При вводе учетной записи, зарегистрированной в Azure Active Directory, вам будет автоматически предложено выбрать способ входа в систему — с помощью рабочей или учебной учетной записи (учетной записи организации) или личной учетной записи LiveID.
2. Перейдите на веб-сайт по адресу https://reports.azure.com и выйдите из учетной записи, с помощью которой вы выполнили вход, или откройте сайт в личном окне, чтобы другие учетные записи не могли получить к нему доступ.
3. Введите адрес электронной почты издателя, не вводя пароль. Вы будете перенаправлены к ***шагу 4***, если учетная запись зарегистрирована в Azure Active Directory (учетные записи администраторов зарегистрированы так же, как и все учетные записи организации, настроенные с помощью учетной записи администратора).
4. Выберите "Рабочая или учебная учетная запись". Рабочие или учебные учетные записи (учетные записи организации) настраиваются компаниями, которые зарегистрированы в Azure. Личные учетные записи (LiveID) — это учетные записи, которые были зарегистрированы в Майкрософт.
 * Обратите внимание, что возможна ситуация, когда пользователь ранее настроил учетную запись LiveID, используя адрес электронной почты компании, через некоторое время организация зарегистрировалась в Azure и тот же адрес электронной почты был зарегистрирован в качестве учетной записи организации. Это означает, что этот пользователь может иметь учетную запись LiveID с паролем, который использовался для создания LiveID, и учетную запись организации с отдельным паролем, используемым в компании.
5. При выборе "Рабочая или учебная учетная запись" на портале Seller Insights следует использовать учетную запись и пароль организации, а не LiveID и пароль.
6. При использовании учетной записи организации вы будете перенаправлены на страницу входа издателя.
7. На этой странице проверьте действие пароля для учетной записи организации. Если пароль не работает, перейдите к разделу ***Сброс пароля***.

***Сброс паролей***

Попытайтесь сбросить пароль с помощью мастера самостоятельного сброса паролей, используя приведенные ниже ссылки.

>[!NOTE]
>Примечание. Для учетных записей организации, настроенных учетной записью администратора: если администратор включил возможность сброса пароля, вы сможете сбросить свой пароль. В противном случае обратитесь к администратору через форму сброса пароля.

****Учетная запись Майкрософт (Live ID): https://account.live.com/ResetPassword.aspx**** ![сброс пароля для учетной записи Майкрософт][3]

****Рабочая или учебная учетная запись (идентификатор организации): https://passwordreset.microsoftonline.com/ *** ![сброс пароля для рабочей или учебной учетной записи][4]

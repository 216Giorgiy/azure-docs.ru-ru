# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>Как использовать портал Microsoft Azure Marketplace Seller Insights

##<a name="signing-in"></a>Вход

Администратор должен отправить вам сообщение о создании учетной записи и предоставить URL-адрес портала Seller Insights (https://reports.azure.com).

Вы можете войти на портал Seller Insights с помощью личной учетной записи Майкрософт или с помощью учетной записи компании в зависимости от того, какую учетную запись администратор использовал при настройке вашего профиля.  Администратор должен сообщить вам, какую учетную запись следует использовать.

   ![тип учетной записи][1]



***Добро пожаловать на портал Seller Insights!***

1. После входа откроется страница "Сводка".
2. Ознакомьтесь с политикой конфиденциальности.
3. Вы в любое время можете связаться с сотрудниками службы поддержки по вопросам, связанным с работой портала Seller Insights.
4. Для перемещения по порталу Seller Insights предназначены вкладки в верхней части страницы.  Вкладки в верхней части страниц в этом руководстве помогут вам разобраться с функциональностью портала Seller Insights.
5. Щелкните смайлик, чтобы в любое время отправить отзыв о портале Seller Insights.
6. Щелкните свое имя в правом верхнем углу, чтобы в любое врем выйти с портала Seller Insights.

   ![страница входа по умолчанию][2]



##<a name="resetting-a-password"></a>Сброс пароля

***Проверка учетной записи и пароля***

1. ***Проверьте, что учетная запись, которую вы используете, зарегистрирована и пароль не работает.*** При вводе учетной записи, зарегистрированной в Azure Active Directory, вам будет автоматически предложено выбрать способ входа в систему — с помощью рабочей или учебной учетной записи (учетной записи организации) или личной учетной записи LiveID.
2. Перейдите на веб-сайт по адресу https://reports.azure.com и выйдите из учетной записи, с помощью которой вы выполнили вход, или откройте сайт в личном окне, чтобы другие учетные записи не могли получить к нему доступ.
3. Введите адрес электронной почты издателя, не вводя пароль. Вы будете перенаправлены к ***шагу 4***, если учетная запись зарегистрирована в Azure Active Directory (учетные записи администраторов зарегистрированы так же, как и все учетные записи организации, настроенные с помощью учетной записи администратора).
4. Выберите "Рабочая или учебная учетная запись". Рабочие или учебные учетные записи (учетные записи организации) настраиваются компаниями, которые зарегистрированы в Azure. Личные учетные записи (LiveID) — это учетные записи, которые были зарегистрированы в Майкрософт.
 * Обратите внимание, что возможна ситуация, когда пользователь ранее настроил учетную запись LiveID, используя адрес электронной почты компании, через некоторое время организация зарегистрировалась в Azure и тот же адрес электронной почты был зарегистрирован в качестве учетной записи организации. Это означает, что этот пользователь может иметь учетную запись LiveID с паролем, который использовался для создания LiveID, и учетную запись организации с отдельным паролем, используемым в компании.
5. При выборе "Рабочая или учебная учетная запись" на портале Seller Insights следует использовать учетную запись и пароль организации, а не LiveID и пароль.
6. При использовании учетной записи организации вы будете перенаправлены на страницу входа издателя.
7. На этой странице проверьте действие пароля для учетной записи организации. Если пароль не работает, перейдите к разделу ***Сброс пароля***.

***Сброс пароля***

Попытайтесь сбросить пароль с помощью мастера самостоятельного сброса паролей, используя приведенные ниже ссылки.
 * Примечание. Для учетных записей организации, настроенных учетной записью администратора: если администратор включил возможность сброса пароля, вы сможете сбросить свой пароль. В противном случае обратитесь к администратору через форму сброса пароля.

# Создание учетной записи продавца Майкрософт
В этой статье описаны действия, необходимые для создания и регистрации учетной записи продавца Майкрософт для Azure Marketplace.

## 1\. Создание учетной записи Майкрософт
> [AZURE.WARNING]Чтобы начать процесс публикации, вам нужно создать учетную запись Майкрософт. Эта учетная запись будет использоваться для регистрации и входа как в **Центр разработчиков Майкрософт**, так и на **портал публикации Azure**. У вас должна быть только одна учетная запись Майкрософт для публикации предложений в Azure Marketplace. Эта запись не должна быть привязана к конкретным службам или предложениям.

Адрес, который образует имя пользователя, должен находиться в вашем домене и управляться вашими ИТ-специалистами (например, publishing@example.com)). Сведения об оплате, налогообложении и отчетности будут направляться через эту учетную запись.

  >[AZURE.WARNING]При регистрации учетной записи Майкрософт нельзя использовать слова "Azure" и "Microsoft". Не используйте эти слова при создании и регистрации учетной записи.

### Указания

1. Создайте список рассылки (DL) или группу безопасности (SG) в домене вашей компании.
  - Добавьте свою команду адаптации в список рассылки.
  - Список рассылки должен поддерживать возможность получения подтверждений по электронной почте.
  - Используйте marketplace@example.com в качестве адреса электронной почты для списка рассылки.
  - Это необходимо сделать в ваших внутренних системах.
2. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.
3. Зарегистрируйте учетную запись Майкрософт, используя адрес списка рассылки.
 - Учетную запись Майкрософт можно зарегистрировать по адресу [https://signup.live.com/signup.aspx](https://signup.live.com/signup.aspx).
 - Используйте marketplace@example.com в качестве адреса электронной почты
 - Идентификатор вашей учетной записи Майкрософт — marketplace@example.com.
4. При регистрации укажите действующий номер телефона. Если потребуется проверить вашу личность, система отправит код подтверждения в виде текстового сообщения или автоматизированного звонка.
5. Проверьте адрес электронной почты списка рассылки.
6. Теперь вы готовы использовать новую учетную запись Майкрософт в Центре разработчиков.

> [AZURE.TIP]Благодаря списку рассылки несколько пользователей могут получать уведомления по электронной почте, которые важны для отчетности о выплатах. Он также гарантирует, что владение учетной записи Майкрософт может передаваться и не привязано к одному человеку.

## 2\. Создание учетной записи в Центре разработчиков Майкрософт
Центр разработчиков Майкрософт используется для однократной регистрации сведений о компании. Пользователь, выполняющий регистрацию, должен быть действующим представителем компании и должен предоставить личные сведения для проверки своей личности. Пользователь, выполняющий регистрацию, должен использовать учетную запись Майкрософт, которая является общей для компании. **Эта же учетная запись должна использоваться и на портале публикации Azure**. Перед созданием учетной записи в Центра разработчиков Майкрософт вам нужно проверить, нет ли в компании существующей учетной записи. В ходе создания учетной записи мы соберем сведения об адресе компании, банковском счете и налогах. Обычно эти сведения собираются у контактных лиц по финансам и бизнесу.

> [AZURE.IMPORTANT]Для прохождения различных этапов создания и развертывания предложения необходимы следующие компоненты профиля продавца.


| Профиль продавца | Запуск черновой версии | Промежуточная | Бесплатная публикация и шаблон решения | Коммерческая публикация |
|----|----|----|----|----|
|Регистрация компании | Необходимо | Необходимо | Необходимо | Необходимо |
|Идентификатор налогового профиля | Необязательно | Необязательно | Необязательно | Необходимо |
|Банковский счет | Необязательно | Необязательно | Необязательно | Необходимо |


> [AZURE.NOTE]Использование собственной лицензии (BYOL) поддерживается только для виртуальных машин. Это предложение считается **бесплатным**.


### Регистрация учетной записи организации
1. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.

2. Откройте страницу [http://dev.windows.com/registration?accountprogram=azure](http://dev.windows.com/registration?accountprogram=azure).

3. Выполните вход с учетной записью Майкрософт, зарегистрированной для вашей компании (например, marketplace@example.com)).

    ![рисунок][img-signin]

4. Выполните действия мастера "Помогите нам защитить вашу учетную запись", в ходе которых ваша личность будет проверена с помощью номера телефона или адреса электронной почты.

    ![рисунок][img-verify]

5. В разделе «Информация о регистрации учетной записи» в раскрывающемся меню выберите **страну или регион учетной записи**.

    > [AZURE.WARNING]**Допустимые страны продажи**. Для продажи своих услуг в Azure Marketplace ваше юридическое лицо должно быть зарегистрировано в одной из утвержденных стран продажи (см. выше). Это ограничение связано с выплатами и налогообложением. Мы активно стараемся расширить этот список стран, поэтому следите за новостями. Дополнительные сведения см. в статье [Политики участия в Azure Marketplace](http://go.microsoft.com/fwlink/?LinkID=526833).

6. Выберите свой тип учетной записи (**Физическое лицо** или **Компания**).

    <!-- Insert screenshot -->> [AZURE.IMPORTANT]Информация о типах учетных записей и выборе подходящего типа см. на странице [Типы учетных записей, расположения и налоги](https://msdn.microsoft.com/library/windows/apps/jj863494.aspx).

7. Укажите **отображаемое имя издателя**. Обычно это название вашей компании.

    > [AZURE.TIP]В настоящее время на портале публикации Azure не используется отображаемое имя издателя. Тем не менее, его необходимо указать для завершения процесса регистрации.

8. Введите контактные сведения для учетной записи.

    > [AZURE.IMPORTANT]Указывайте точные контактные сведения, так как они будут использоваться в процессе проверки и утверждения вашей компании в Центре разработчика.

9. При создании учетной записи компании вам нужно указать контактные данные **утверждающего лица в компании** — человека, который может подтвердить ваше право создавать учетную запись от имени организации. Завершив, нажмите кнопку **Далее**, чтобы перейти в **раздел платежных данных**.

10. Введите платежные данные для своей учетной записи. Если у вас есть код акции, который влияет на стоимость регистрации, вы можете указать его здесь. В противном случае укажите сведения о вашей кредитной карте (или адрес PayPal на поддерживаемых рынках). Завершив, нажмите кнопку **Далее**, чтобы перейти в **раздел проверки введенной информации**.

11. Проверьте данные учетной записи и убедитесь, что все правильно. Затем прочитайте и примите условия **соглашения с издателем Microsoft Azure Marketplace**. Установите флажок, который подтверждает, что вы ознакомились с этими условиями и принимаете их.

12. Нажмите кнопку **Готово**, чтобы подтвердить регистрацию. Мы отправим подтверждение на ваш адрес электронной почты разработчика.

13. Если вы планируете публиковать только бесплатные предложения, нажмите кнопку **Перейти на портал публикации Azure Marketplace**. Перейдите к разделу 3 этого документа, посвященному [регистрации учетной записи на портале публикации](#3-register-your-account-in-the-publishing-portal).

14. Если вы планируете публиковать коммерческие предложения, нажмите кнопку **Изменить информацию об учетной записи**. Укажите полные налоговые и банковские сведения на панели мониторинга Центра разработчиков.

> [AZURE.IMPORTANT]Без указания полной налоговой и банковской информации вы не сможете должным образом тестировать свои предложения на промежуточном уровне и публиковать их в рабочей среде.

Если вы хотите обновить налоговые и банковские сведения позже, перейдите к разделу 3 ([Регистрация учетной записи на портале публикации](#3-register-your-account-in-the-publishing-portal)). Вы сможете вернуться на портал публикации позже, перейдя по ссылкам.

### Добавление налоговой и банковской информации
 Если вы хотите публиковать коммерческие предложения для продажи, вам также нужно добавить сведения о выплатах и налогах и отправить их для проверки в Центре разработчиков. Если вы будете публиковать только бесплатные предложения (или предложения BYOL), указывать эту информацию не требуется. Эту информацию можно добавить позже, но на проверку сведений о налогах требуется некоторое время. Если вы точно знаете, что будете публиковать коммерческие предложения для продажи, рекомендуем добавить эти сведения как можно раньше.

**Банковские сведения**

1. Войдите в [Центр разработчиков Майкрософт](http://dev.windows.com/registration?accountprogram=azure), используя свою учетную запись Майкрософт.

2. Щелкните **Счет для выплаты** в меню слева, а затем в разделе **Выбор способа оплаты** щелкните **Банковский счет** или **PayPal**.

    > [AZURE.IMPORTANT]Если у вас есть коммерческие предложения, которые клиенты приобретают в Marketplace, на этот счет вы будете получать выплаты за эти покупки.

3. Введите платежные сведения и нажмите кнопку **Сохранить**, когда завершите.

    > [AZURE.IMPORTANT]Если вам нужно изменить счет для выплат, выполните описанные выше действия и обновите информацию. Изменение счета для выплат может привести к задержке выплат на один цикл. Эта задержка связана с необходимостью проверки измененного счета, которая обычно выполняется при добавлении первого счета для выплат. После проверки вам будет выплачена полная сумма. Все выплаты, не отправленные в текущем периоде, переносятся на следующий.

4. Нажмите кнопку **Далее**.

**Налоговые сведения**

1. Войдите в [Центр разработчиков Майкрософт](http://dev.windows.com/registration?accountprogram=azure), используя свою учетную запись Майкрософт (если требуется).

2. В меню слева щелкните **Налоговый профиль**.

3. На странице **Сведения о налогах** выберите страну или регион постоянного проживания, а затем выберите страну или регион основного гражданства. Нажмите кнопку **Далее**.

4. Введите сведения о налогах и нажмите кнопку **Далее**.

> [AZURE.WARNING]Без указания полных банковских и налоговых сведений в учетной записи Центра разработчиков Майкрософт размещать коммерческие предложения невозможно.

## 3\. Регистрация учетной записи на портале публикации
Портал публикации Azure используется для публикации предложений и управления ими. На портале публикации можно найти некоторые полезные сведения, которые помогут в процессе создания предложения.

> [AZURE.WARNING]Здесь необходимо использовать ту же учетную запись Майкрософт, которая была использована при регистрации в Центре разработчиков Майкрософт. После создания главной учетной записи издателя можно добавить дополнительных пользователей.

1.	Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.

2.	Перейдите на страницу [http://publish.windowsazure.com](http://publish.windowsazure.com).

3.	Выполните вход с учетной записью Майкрософт, зарегистрированной для вашей компании (например, marketplace@example.com)). При необходимости вы сможете добавить соадминистраторов.

  >[AZURE.TIP]Политики участия описаны на [веб-сайте Azure](http://azure.microsoft.com/support/legal/marketplace/participation-policies/).

  > При возникновении проблем с регистрацией в Центре разработчиков отправьте запрос в службу поддержки следующим образом. 1. Обратитесь в [службу поддержки](https://support.microsoft.com/getsupport?wf=0&tenant=ClassicCommercial&oaspworkflow=start_1.0.0.0&supportregion=ru-RU&pesid=15635&ccsid=635847950577064286). 2. Выберите **Центр разработчиков**. 3. Выберите **Профиль**. 4. Выберите способ связи.

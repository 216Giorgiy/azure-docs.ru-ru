# Создание учетной записи продавца Майкрософт
В этой статье описаны действия, необходимые для создания, регистрации и утверждения учетной записи продавца Майкрософт для Azure Marketplace.

## 1\. Создание учетной записи Microsoft (MSA)
> [AZURE.WARNING]Чтобы завершить процесс публикации, вам потребуется создать учетную запись Майкрософт. Эта учетная запись будет использоваться для входа как на портал публикации, так и на панель мониторинга продаж. У вас должна быть только одна учетная запись Майкрософт для публикации предложений в Azure Marketplace. Эта запись не должна быть привязана к конкретным службам или предложениям.

Адрес, который формирует имя пользователя, должен находиться в вашем домене и контролироваться вашими ИТ-специалистами (например, publishing@yourcompany.com)). Сведения об оплате, налогообложении и отчетности будут направляться через эту учетную запись.

  >[AZURE.WARNING]При регистрации учетной записи Майкрософт нельзя использовать слова Azure и Microsoft. Не используйте эти слова при создании и регистрации учетной записи.

1. Создайте список рассылки (DL) или группу безопасности (SG) в домене вашей компании.
  - Добавьте свою команду адаптации в список рассылки.
  - В список рассылки будут отправляться подтверждения по электронной почте.
  - Это необходимо сделать в ваших внутренних системах. Например, marketplace@yourcompany.com
2. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.
3. Зарегистрируйте учетную запись Майкрософт, используя адрес электронной почты списка рассылки.
 - Учетную запись Майкрософт можно зарегистрировать по адресу [https://signup.live.com/signup.aspx](https://signup.live.com/signup.aspx)
 - Используйте marketplace@yourcompany.com в качестве адреса электронной почты
 - Идентификатор вашей учетной записи Майкрософт — marketplace@yourcompany.com

    ![рисунок][img-msalive]

4. При регистрации укажите действующий номер телефона. Система отправит код подтверждения в виде текстового сообщения или автоматизированного звонка, если потребуется проверить вашу личность.
5. Проверьте адрес электронной почты списка рассылки.

    ![рисунок][img-email]

6. Теперь вы готовы использовать новую учетную запись Майкрософт на панели мониторинга продавца.

> [AZURE.IMPORTANT]Использование списка рассылки позволяет нескольким пользователям получать уведомления по электронной почте с важной информацией об оплате, а также гарантирует, что учетная запись Майкрософт не будет передана и привязана к конкретному лицу.

## 2\. Создание учетной записи панели мониторинга продавца
Панель мониторинга продавца Майкрософт используется для однократной регистрации сведений о компании. Пользователь, выполняющий регистрацию, должен быть действующим представителем компании и должен предоставить личные сведения для проверки своей личности. Пользователь, выполняющий регистрацию, должен использовать учетную запись Майкрософт (MSA), которая является общей для компании, и эта же учетная запись должна использоваться на портале публикации Azure. Перед созданием учетной записи панели мониторинга продаж вам нужно проверить, нет ли аналогичной существующей учетной записи. В ходе создания учетной записи мы соберем сведения о вашем банковском счете, о налогах и об адресе компании. Обычно эти сведения собираются у контактных лиц по финансам и бизнесу.

> [AZURE.IMPORTANT]Для прохождения различных фаз создания и развертывания предложения необходимы следующие компоненты профиля продавца.


| Профиль продавца | Запуск черновой версии | Промежуточная | Бесплатная публикация и шаблон решения | Коммерческая публикация |
|----|----|----|----|----|
|Регистрация компании | Необходимо | Необходимо | Необходимо | Необходимо |
|Идентификатор налогового профиля | Необязательно | Необязательно | Необязательно | Необходимо |
|Банковский счет | Необязательно | Необязательно | Необязательно | Необходимо |


> [AZURE.NOTE]BYOL поддерживается только для виртуальных машин и считается **БЕСПЛАТНЫМ** предложением.


### Регистрация учетной записи организации
1. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в свою личную учетную запись.

2. Откройте адрес [http://sellerdashboard.microsoft.com](http://sellerdashboard.microsoft.com)

    ![рисунок][img-sd-url]

3. Войдите в систему с учетной записью Майкрософт (MSA), зарегистрированной для вашей компании (например, AzureStore@yourcompany.com).

    ![рисунок][img-signin]

4. Выполните действия мастера "Помогите нам защитить вашу учетную запись", в ходе которых ваша личность будет проверена с помощью номера телефона или адреса электронной почты.

    ![рисунок][img-verify]

5. Перейдите в раздел **Сведения об учетной записи**. На этом экране необходимо ввести ПЕРСОНАЛЬНЫЕ данные, которые будут использоваться только для проверки подлинности личности. К ним относятся: ваше имя, адрес электронной почты, место жительства и личный номер телефона. Эти сведения не будут переданы другим лицам за пределами Майкрософт.

    ![рисунок][img-sd-top]

    ![рисунок][img-sd-info]

6. Зарегистрируйте от имени вашей компании, указав тип учетной записи "Организация", а не "Физическое лицо". Нажмите кнопку **Далее**.

    ![рисунок][img-sd-type]

7. Заполните сведения о вашей организации. Эти сведения должны быть точными: адрес центрального офиса, контактное лицо — все это будет проверено специалистами Майкрософт.

    ![рисунок][img-sd-mktg1]

8. Название компании используется порталом издателя, поэтому оно должно быть точным

    ![рисунок][img-sd-mktg2]

9. Используйте адрес центрального офиса вашей компании

    ![рисунок][img-sd-addr]

10. Укажите контактное лицо, которое будет доступно и будет рассматриваться как представитель компании

    ![рисунок][img-sd-legal]

11. Когда закончите, щелкните **Отправить для утверждения**.

    ![рисунок][img-sd-submit]


Если вы собираетесь публиковать только бесплатные предложения, можете сразу перейти к разделу 3 **Регистрация учетной записи на портале публикации**.

Если вы планируете публиковать коммерческие предложения, необходимо предоставить полную налоговую и банковскую информацию на панели мониторинга продавца для своей учетной записи. См. раздел 2.2.

> [AZURE.IMPORTANT]Без указания полной налоговой и банковской информации вы не сможете должным образом тестировать свои предложения на промежуточном уровне и публиковать их в рабочей среде.

Если вы хотите указать налоговую и банковскую информацию позже, можно перейти к следующему шагу 3. **Регистрация учетной записи на портале публикации** и вернуться назад позже с помощью ссылок на портале публикации.

### Добавление налоговой и банковской информации
 Если вы хотите публиковать коммерческие предложения для продажи, необходимо добавить сведения о выплатах и налогах и отправить их для проверки на панели мониторинга продавца. Если вы будете публиковать только бесплатные предложения (или предложения BYOL), указывать эту информацию не требуется. Эту информацию можно добавить позже, но на проверку сведений о налогах требуется некоторое время. Если вы точно знаете, что будете публиковать коммерческие предложения для продажи, рекомендуем добавить эти сведения как можно раньше.

1. Войдите в [панель мониторинга продавца](http://sellerdashboard.microsoft.com) со своей учетной записью Майкрософт.

2. Щелкните вкладку **УЧЕТНАЯ ЗАПИСЬ**, а затем "Выплаты и налоги".

3. Щелкните **ДОБАВИТЬ СВЕДЕНИЯ О ВЫПЛАТАХ И НАЛОГАХ**.

4. На странице **Выбор способа оплаты** в разделе "Новый способ оплаты" щелкните **Банковский счет** или **PayPal**.

> [AZURE.IMPORTANT]Если у вас есть коммерческие предложения, которые клиенты приобретают в Marketplace, на этот счет вы будете получать выплаты за эти покупки.

5. Введите сведения о банковском счете или учетной записи PayPal.

6. Нажмите кнопку **ДАЛЕЕ**.

7. На странице **Сведения о налогах** выберите страну или регион, в которых расположено место вашего постоянного жительства, затем выберите страну или регион для своего основного гражданства и щелкните "ДАЛЕЕ".

8. Введите сведения о налогах и щелкните **ДАЛЕЕ**.

9. Нажмите кнопку **Submit** (Отправить). Если вы еще не готовы отправить сведения о налогах для проверки, можно щелкнуть **Сохранить** или **Сохранить и закрыть**. Проверка сведений о налогах занимает некоторое время, поэтому рекомендуется отправить их для проверки как можно быстрее.

> [AZURE.WARNING]Без указания полных банковских и налоговых сведений на панели мониторинга продавца для своей учетной записи размещать коммерческие предложения будет невозможно.

## 3\. Регистрация учетной записи на портале публикации
Портал публикации Azure используется для публикации предложений и управления ими. На портале публикации можно найти некоторые полезные сведения, которые проведут вас по процессу создания предложения.

> [AZURE.WARNING]Здесь НЕОБХОДИМО использовать ту же учетную запись Майкрософт, которая была использована при регистрации на панели мониторинга продавца. После создания главной учетной записи издателя можно добавить дополнительных пользователей.

1.	Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в свою личную учетную запись.
2.	Перейдите на страницу [http://publish.windowsazure.com](http://publish.windowsazure.com).
3.	 Войдите в систему с учетной записью Майкрософт, зарегистрированной для вашей компании (например, AZStore@yourcompany.com), и при необходимости вы сможете добавить соадминистраторов.
4.	Прочтите и примите условия Соглашения издателя (отображается при первом входе в портал публикации).

  >[AZURE.TIP]Политики участия описаны [здесь](http://azure.microsoft.com/support/legal/marketplace/participation-policies/).

  > При возникновении проблем с регистрацией продавца отправьте запрос в службу поддержки следующим образом: 1. Выберите [Служба поддержки](http://go.microsoft.com/fwlink?LinkId=272975). 2. Выберите **Регистрация на панели мониторинга продавца и своя учетная запись**. 3. Выберите **Регистрация учетной записи разработчика**. 4. Выберите **способ** связи.

### Допустимые страны продаж

> [AZURE.WARNING]Для продажи своих услуг в Microsoft Azure Marketplace ваша учетная запись должна быть зарегистрирована для одной из "допустимых стран продаж". Это ограничение связано с выплатами и налогообложением. Мы активно стараемся расширить этот список стран, поэтому следите за новостями. Полный список см. в разделе 1b документа по ссылке [http://go.microsoft.com/fwlink/?LinkID=526833](http://go.microsoft.com/fwlink/?LinkID=526833).

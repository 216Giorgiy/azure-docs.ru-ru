# Создание учетной записи продавца Майкрософт
В этой статье описаны действия, необходимые для создания и регистрации учетной записи продавца Майкрософт для Azure Marketplace.

## 1\. Создание учетной записи Майкрософт
> [AZURE.WARNING]Чтобы завершить процесс публикации, вам потребуется создать учетную запись Майкрософт. Эта учетная запись будет использоваться для регистрации и входа как на портал публикации, так и на панель мониторинга продаж. У вас должна быть только одна учетная запись Майкрософт для публикации предложений в Azure Marketplace. Эта запись не должна быть привязана к конкретным службам или предложениям.

Адрес, который формирует имя пользователя, должен находиться в вашем домене и контролироваться вашими ИТ-специалистами (например, publishing@example.com)). Сведения об оплате, налогообложении и отчетности будут направляться через эту учетную запись.

  >[AZURE.WARNING]При регистрации учетной записи Майкрософт нельзя использовать слова "Azure" и "Microsoft". Не используйте эти слова при создании и регистрации учетной записи.

1. Создайте список рассылки (DL) или группу безопасности (SG) в домене вашей компании.
  - Добавьте свою команду адаптации в список рассылки.
  - Список рассылки должен поддерживать возможность получения подтверждений по электронной почте.
  - Используйте marketplace@example.com в качестве адреса электронной почты для списка рассылки.
  - Это необходимо сделать в ваших внутренних системах.
2. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.
3. Зарегистрируйте учетную запись Майкрософт, используя адрес списка рассылки.
 - Учетную запись Майкрософт можно зарегистрировать по адресу [https://signup.live.com/signup.aspx](https://signup.live.com/signup.aspx).
 - Используйте marketplace@example.com в качестве адреса электронной почты
 - ИД вашей учетной записи Майкрософт — marketplace@example.com.

    <!--[drawing][img-msalive]-->

4. При регистрации укажите действующий номер телефона. Если потребуется проверить вашу личность, система отправит код подтверждения в виде текстового сообщения или автоматизированного звонка.
5. Проверьте адрес электронной почты списка рассылки.

    <!--[drawing][img-email]-->

6. Теперь вы готовы использовать новую учетную запись Майкрософт на панели мониторинга продаж.

> [AZURE.IMPORTANT]Благодаря списку рассылки несколько пользователей могут получать уведомления по электронной почте, которые важны для отчетности о выплатах. Он также гарантирует, что владение учетной записи Майкрософт может передаваться и не привязано к одному человеку.

## 2\. Создание учетной записи панели мониторинга продавца
Панель мониторинга продавца Майкрософт используется для однократной регистрации сведений о компании. Пользователь, выполняющий регистрацию, должен быть действующим представителем компании и должен предоставить личные сведения для проверки своей личности. Пользователь, выполняющий регистрацию, должен использовать учетную запись Майкрософт, которая является общей для компании, и эта же учетная запись должна использоваться на портале публикации Azure. Перед созданием учетной записи панели мониторинга продаж вам нужно проверить, нет ли в компании аналогичной существующей учетной записи. В ходе создания учетной записи мы соберем сведения о вашем банковском счете, о налогах и об адресе компании. Обычно эти сведения собираются у контактных лиц по финансам и бизнесу.

> [AZURE.IMPORTANT]Для прохождения различных этапов создания и развертывания предложения необходимы следующие компоненты профиля продавца.


| Профиль продавца | Запуск черновой версии | Промежуточная | Бесплатная публикация и шаблон решения | Коммерческая публикация |
|----|----|----|----|----|
|Регистрация компании | Необходимо | Необходимо | Необходимо | Необходимо |
|Идентификатор налогового профиля | Необязательно | Необязательно | Необязательно | Необходимо |
|Банковский счет | Необязательно | Необязательно | Необязательно | Необходимо |


> [AZURE.NOTE]Использование собственный лицензии (BYOL) поддерживается только для виртуальных машин и считается **бесплатным** предложением.


### Регистрация учетной записи организации
1. Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.

2. Перейдите по адресу [http://sellerdashboard.microsoft.com](http://sellerdashboard.microsoft.com).

    ![рисунок][img-sd-url]

3. Выполните вход с учетной записью Майкрософт, зарегистрированной для вашей компании (например, marketplace@example.com)).

    ![рисунок][img-signin]

4. Выполните действия мастера "Помогите нам защитить вашу учетную запись", в ходе которых ваша личность будет проверена с помощью номера телефона или адреса электронной почты.

    ![рисунок][img-verify]

5. Перейдите в раздел **Сведения об учетной записи**. На этом экране необходимо ввести персональные данные, которые будут использоваться только для проверки подлинности личности. К ним относятся: ваше имя, адрес электронной почты, место жительства и личный номер телефона. Эти сведения не будут переданы другим лицам за пределами Майкрософт.

    ![рисунок][img-sd-top]

    <!--[drawing][img-sd-info]-->

6. Зарегистрируйтесь от имени вашей компании, указав тип учетной записи **Организация**, а не **Физическое лицо**. Нажмите кнопку **Далее**.

    ![рисунок][img-sd-type]

7. Заполните сведения о вашей организации. Эти сведения должны быть точными: адрес центрального офиса, контактное лицо — все это будет проверено специалистами Майкрософт.

    ![рисунок][img-sd-mktg1]

8. Название компании используется на портале публикации, поэтому оно должно быть точным.

    <!--[drawing][img-sd-mktg2]-->

9. Используйте адрес центрального офиса вашей компании.

    ![рисунок][img-sd-addr]

10. Укажите контактное лицо, которое будет доступно и будет рассматриваться как представитель компании.

    ![рисунок][img-sd-legal]

11. Когда закончите, щелкните **Отправить для утверждения**.

    ![рисунок][img-sd-submit]


Если вы собираетесь публиковать только бесплатные предложения, можете сразу перейти к разделу 3 "Регистрация учетной записи на портале публикации".

Если вы планируете публиковать коммерческие предложения, необходимо предоставить полную налоговую и банковскую информацию на панели мониторинга продавца для своей учетной записи.

> [AZURE.IMPORTANT]Без указания полной налоговой и банковской информации вы не сможете должным образом тестировать свои предложения на промежуточном уровне и публиковать их в рабочей среде.

Чтобы обновить налоговые и банковские сведения позднее, можно перейти к разделу 3 "Регистрация учетной записи на портале публикации" и через определенное время вернуться с помощью ссылок на портал публикации.

### Добавление налоговой и банковской информации
 Если вы хотите публиковать коммерческие предложения для продажи, необходимо добавить сведения о выплатах и налогах и отправить их для проверки на панели мониторинга продавца. Если вы будете публиковать только бесплатные предложения (или предложения BYOL), указывать эту информацию не требуется. Эту информацию можно добавить позже, но на проверку сведений о налогах требуется некоторое время. Если вы точно знаете, что будете публиковать коммерческие предложения для продажи, рекомендуем добавить эти сведения как можно раньше.

1. Войдите в [панель мониторинга продаж](http://sellerdashboard.microsoft.com) со своей учетной записью Майкрософт.

2. Щелкните вкладку **УЧЕТНАЯ ЗАПИСЬ**, а затем **Выплаты и налоги**.

3. Щелкните **ДОБАВИТЬ СВЕДЕНИЯ О ВЫПЛАТАХ И НАЛОГАХ**.

4. На странице **Выбор способа оплаты** в разделе **Новый способ оплаты** щелкните **Банковский счет** или **PayPal**.

> [AZURE.IMPORTANT]Если у вас есть коммерческие предложения, которые клиенты приобретают в Marketplace, на этот счет вы будете получать выплаты за эти покупки.

5. Введите сведения о банковском счете или учетной записи PayPal.

6. Нажмите кнопку **ДАЛЕЕ**.

7. На странице **Сведения о налогах** выберите страну или регион, в которых расположено место вашего постоянного жительства, затем выберите страну или регион для своего основного гражданства и нажмите кнопку "ДАЛЕЕ". Нажмите кнопку **ДАЛЕЕ**.

8. Введите сведения о налогах и нажмите кнопку **ДАЛЕЕ**.

9. Нажмите кнопку **Submit** (Отправить). Если вы еще не готовы отправить сведения о налогах для проверки, можно нажать кнопку **Сохранить** или **Сохранить и закрыть**. Проверка сведений о налогах занимает некоторое время, поэтому рекомендуется отправить их для проверки как можно быстрее.

> [AZURE.WARNING]Без указания полных банковских и налоговых сведений на панели мониторинга продавца для своей учетной записи размещать коммерческие предложения будет невозможно.

## 3\. Регистрация учетной записи на портале публикации
Портал публикации Azure используется для публикации предложений и управления ими. На портале публикации можно найти некоторые полезные сведения, которые помогут в процессе создания предложения.

> [AZURE.WARNING]Здесь необходимо использовать ту же учетную запись Майкрософт, которая была использована при регистрации на панели мониторинга продаж. После создания главной учетной записи издателя можно добавить дополнительных пользователей.

1.	Откройте новое окно в режиме инкогнито в браузере Chrome или запустите режим InPrivate в браузере Internet Explorer, чтобы не войти в существующую учетную запись.
2.	Перейдите на страницу [http://publish.windowsazure.com](http://publish.windowsazure.com).
3.	 Выполните вход с учетной записью Майкрософт, зарегистрированной для вашей компании (например, marketplace@example.com)), и при необходимости вы сможете добавить соадминистраторов.
4.	Прочтите и примите условия соглашения издателя (отображается при первом входе в портал публикации).

  >[AZURE.TIP]Политики участия описаны на [веб-сайте Azure](http://azure.microsoft.com/support/legal/marketplace/participation-policies/).

  При возникновении проблем в процессе регистрации продавца отправьте запрос в службу поддержки следующим образом. 1. Обратитесь в [службу поддержки](http://go.microsoft.com/fwlink?LinkId=272975). 2. Выберите **Регистрация на панели мониторинга продаж и своя учетная запись**. 3. Выберите **Регистрация учетной записи разработчика**. 4. Выберите способ связи.

### Допустимые страны продаж

> [AZURE.WARNING]Для продажи своих услуг в Azure Marketplace ваше зарегистрированное юридическое лицо должно быть из одной из утвержденных "стран продаж". Это ограничение связано с выплатами и налогообложением. Мы активно стараемся расширить этот список стран, поэтому следите за новостями. Полный список см. в разделе 1b [политик участия в Marketplace](http://go.microsoft.com/fwlink/?LinkID=526833).

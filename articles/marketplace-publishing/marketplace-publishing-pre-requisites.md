# Общие предварительные требования для создания предложения в Azure Marketplace
Эта статья содержит общие предварительные требования, относящиеся к бизнес-процессу, которые необходимы для создания предложения.

## Убедитесь, что у вас есть учетная запись продавца Майкрософт
Подробные инструкции по регистрации учетной записи продавца Майкрософт см. в статье [Создание и регистрация учетной записи](marketplace-publishing-accounts-creation-registration.md).

- Если учетная запись **уже зарегистрирована**, узнайте, кому в вашей компании она принадлежит или какие учетные данные были использованы для регистрации.
- Если вы **не являетесь владельцем учетной записи публикации**, владелец учетной записи может добавить вашу учетную запись Майкрософт в качестве соадминистратора, выбрав: [Портал публикации](https://publish.windowsazure.com) -> вкладка **Издатели** -> ссылка **Администраторы**.
- Убедитесь, что заинтересованные лица в процессе публикации Azure получают электронные письма, отправляемые на этот адрес. Для завершения процесса публикации эти письма необходимо отслеживать и отвечать на них.
- Не создавайте учетную запись, связанную с определенным человеком. Если этот человек уйдет из вашей организации, он сможет повлиять на доступ к сведениям об SKU и возможность публикации ваших SKU.

> [AZURE.IMPORTANT]**Если вы собираетесь публиковать только бесплатные предложения (или предложения, использующие собственную лицензию), заполнять банковскую и налоговую информацию о компании не нужно.**

> Для начала работы необходимо выполнить регистрацию компании. Однако пока компания работает над налоговой и банковской информацией в учетной записи на панели мониторинга продавца, разработчики могут начать работу по созданию образа виртуальной машины в [портале публикации](https://publish.windowsazure.com), а также по его сертификации и тестированию в промежуточной среде Azure. Полное утверждение учетной записи продавца необходимо только для последнего этапа публикации вашего предложения в Marketplace.

> При возникновении проблем с регистрацией продавца отправьте запрос в службу поддержки следующим образом: 1. Выберите [Служба поддержки](http://go.microsoft.com/fwlink?LinkId=272975). 2. Выберите **Регистрация на панели мониторинга продавца и своя учетная запись**. 3. Выберите **Регистрация учетной записи разработчика**. 4. Выберите **способ** связи.


## Получение подписки Azure с оплатой по мере использования
Эта подписка будет использоваться для создания образов виртуальных машин и передачи образов в [Azure Marketplace](http://azure.microsoft.com/marketplace). Если у вас нет существующей подписки, войдите в систему здесь https://account.windowsazure.com/signup?offer=ms-azr-0003p.

## Допустимые страны продаж
> [AZURE.WARNING]Для продажи своих услуг в Microsoft Azure Marketplace ваша учетная запись должна быть зарегистрирована для одной из "допустимых стран продаж". Это ограничение связано с выплатами и налогообложением. Мы активно стараемся расширить этот список стран, поэтому следите за новостями. Полный список см. в **Разделе 1b** [Политики участия в Microsoft Azure Marketplace](http://go.microsoft.com/fwlink/?LinkID=526833).

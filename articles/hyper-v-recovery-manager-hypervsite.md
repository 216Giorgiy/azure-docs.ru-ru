# Учебник. Настройка защиты между локальным сайтом VMM и Azure


<h2><a id="overview" name="overview" href="#overview"></a>Обзор</h2>
<p>Azure Site Recovery (служба восстановления сайтов) помогает реализовать стратегии непрерывности бизнеса и рабочей нагрузки, управляя процессами репликации, отработки отказа и восстановления виртуальных машин в ряде сценариев развертывания.<p>

Этот учебник содержит инструкции по развертыванию Azure Site Recovery для защиты рабочих нагрузок на сервере Hyper-V с Windows Server 2012 R2 на локальном узле. Репликация виртуальных машин в Azure на сервере Hyper-V проводится с помощью репликации Hyper-V. Такое развертывание особенно полезно, если серверы Hyper-V находятся в главном офисе или филиале, а среда System Center VMM не развернута.


<LI>По возможности в учебнике используются самый быстрый путь развертывания и параметры по умолчанию или минимальные.
Полные инструкции по развертыванию можно прочитать в руководствах по <a href="http://go.microsoft.com/fwlink/?LinkId=522087">планированию</a> и <a href=" http://go.microsoft.com/fwlink/?LinkId=522088">развертыванию</a>.</LI>
<LI>Подробные сведения о дополнительных сценариях развертывания Azure Site Recovery содержатся в разделе <a href="http://go.microsoft.com/fwlink/?LinkId=518690">Обзор Azure Site Recovery</a>.</LI>
<LI>При возникновении проблем с данным учебником см. вики-статью <a href="http://go.microsoft.com/fwlink/?LinkId=389879">Azure Site Recovery: распространенные ошибки и решения</a> или задайте свои вопросы на <a href="http://go.microsoft.com/fwlink/?LinkId=313628">форуме службы восстановления Azure</a>.</LI>
</UL>

Сервер Hyper-V должен содержать одну или несколько виртуальных машин. Серверы Hyper-V должны быть подключены к Интернету напрямую или через прокси-сервер.</LI>

<h2><a id="tutorial" name="tutorial" href="#tutorial"></a>Пошаговые указания</h2>
После проверки необходимых условий выполните следующие действия:
<UL>
<LI><a href="#vault">Шаг 1. Создание хранилища</a>- Создайте хранилище Azure Site Recovery.</LI>
<LI><a href="#site">Шаг 2. Создание узла Hyper-V</a>- Создайте узел Hyper-V как логический контейнер для всех серверов Hyper-V, содержащих виртуальные машины, которые нужно защитить.</LI>
<LI><a href="#download">Шаг 3. Подготовка серверов Hyper-V</a>- Создайте регистрационный ключ и скачайте установочный файл поставщика. Запустите файл на каждом сервере узла Hyper-V и выберите ключ для регистрации сервера в хранилище.</LI>
<LI><a href="#resources">Шаг 4. Подготовка ресурсов</a>- Создайте учетную запись хранения Azure, чтобы сохранять реплицированные виртуальные машины.</LI>
<LI><a href="#protectiongroup">Шаг 5. Создание и настройка групп защиты</a>- Создайте группу защиты и установите необходимые параметры. Параметры защиты будет применяться к каждой виртуальной машине, добавленной в группу.</LI>
<LI><a href="#enablevirtual">Шаг 6. Включение защиты для виртуальных машин</a>- Включите защиту для виртуальных машин, добавляя их в группу защиты.</LI>
<LI><a href="#recovery plans">Шаг 7. Тестирование развертывания</a>- Запустите тестовую отработку отказа для виртуальной машины.</LI>

</UL>




<a name="vault"></a> <h3>Шаг 1. Создание хранилища</h3>

1. Выполните вход на [Портал управления](https://manage.windowsazure.com).


2. Разверните элементы <b>Службы данных</b>, <b>Службы восстановления</b> и щелкните <b>Хранилище Site Recovery</b>.


3. Щелкните <b>Создать</b>, а затем - <b>Быстрое создание</b>.
	
4. В поле <b>Имя</b> введите понятное имя для идентификации хранилища.

5. В поле <b>Область</b> выберите географический регион для хранилища архивации. Чтобы проверить поддерживаемые регионы, см. географическую доступность в разделе <a href="http://go.microsoft.com/fwlink/?LinkId=389880">Информация о ценах на Azure Site Recovery</a>.

6. Щелкните элемент <b>Создать хранилище</b>. 

	![New Vault](./media/hyper-v-recovery-manager-configure-vault/SR_HvVault.png)

<P>Проверьте строку состояния, чтобы убедиться, что хранилище успешно создано. Это хранилище будет указано с состоянием <b>Активно</b> на главной странице служб восстановления.</P>

<a name="site"></a> <h3>Шаг 2. Создание узла Hyper-V</h3>

1. На странице "Службы восстановления" щелкните хранилище, чтобы открыть страницу быстрого запуска. Страницу быстрого запуска можно открыть и в любой другой момент, щелкнув этот значок.

	![Hyper-V site scenario](./media/hyper-v-recovery-manager-configure-vault/SR_QuickStartIcon.png)

2. В раскрывающемся списке выберите элемент **Между локальным узлом Hyper-V и Microsoft Azure**.

	![Hyper-V site scenario](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_SelectScenario.png)

3. В разделе **Создание узла Hyper-V** щелкните **Создать узел Hyper-V**. Укажите имя узла и сохраните.

	![Hyper-V site](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_CreateSite2.png)

<a name="download"></a> <h3>Шаг 3. Подготовка серверов Hyper-V</h3>
	

1. В разделе **Подготовка серверов Hyper-V** нажмите пункт **Загрузить регистрационный ключ**.
2. На странице **Загрузить регистрационный ключ** щелкните пункт **Загрузка** рядом с узлом. Скачайте ключ в безопасное расположение, легко доступное для сервера Hyper-V. Ключ действителен в течение пяти дней после создания.

	![Registration key](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_DownloadKey2.png)

4. Щелкните <b>Скачать поставщик</b> для получения последней версии.
5. Запустите файл на каждом сервере Hyper-V, который нужно зарегистрировать в хранилище. Файл устанавливает два компонента:
	- **Поставщик Azure Site Recovery** - обрабатывает связи и взаимодействия между сервером Hyper-V и порталом Azure Site Recovery. 
	- **Агент служб восстановления Azure** - обрабатывает передачу данных между виртуальными машинами, работающими на исходном сервере Hyper-V и в хранилище Azure. 
6. Обновления можно настроить в **Центре обновления Майкрософт**. Если этот параметр включен, то обновления поставщика и агента будут устанавливаться в соответствии с вашей политикой Центра обновления Майкрософт.

	![Microsoft Updates](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider1.png)

7. В поле **Установка** укажите место на сервере Hyper-V для установки поставщика и агента.

	![Install location](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider2.png)

8. После завершения установки продолжите настройку, чтобы зарегистрировать сервер в хранилище.

	![Installation complete](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider3.png)


9. На странице **Подключение к Интернету** укажите способ подключения поставщика к Azure Site Recovery. Выберите пункт <b>Use default system proxy settings</b> (Использовать настройки прокси-сервера по умолчанию), чтобы использовать настройки подключения к Интернету, по умолчанию заданные на сервере. 

	![Internet Settings](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider4.png)

9. На странице **Параметры хранилища** щелкните **Обзор** для выбора файла ключа. Укажите подписку Azure Site Recovery, имя хранилища и узла Hyper-V, к которому относится сервер Hyper-V.

	![Server registration](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_SelectKey.png)


11. Начнется процесс регистрации сервера в хранилище.

	![Server registration](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider6.png)

11. После завершения регистрации Azure Site Recovery извлекает метаданные из сервера Hyper-V, а сам сервер отображается на вкладке **Узлы Hyper-V** страницы **Серверы** в хранилище.

	![Server registration](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_Provider7.png)


<h3><a id="resources"></a>Шаг 4. Подготовка ресурсов</h3>


1. На панели страницы быстрого запуска **Подготовка ресурсов** выберите **Создать учетную запись хранения** для создания учетной записи хранения Azure, если таковой нет. Для учетной записи необходимо включить георепликацию. Она должна находиться в том же регионе, что и хранилище Azure Site Recovery, и быть связана с той же подпиской.

	![Create storage account](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_CreateResources1.png)



<h3><a id="protectiongroup"></a>Шаг 5. Создание и настройка групп защиты</h3>

<p>Группа защиты объединяют виртуальные машины с идентичными защитными параметрами. Параметры, установленные для группы защиты, будут применяться ко всем виртуальным машинам, добавленным в группу.</p>
1. В поле **Создание и настройка групп защиты** щелкните **Создать группу защиты**. Если выполнены не все предварительные требования, выдается сообщение об этом. Чтобы получить подробную информацию, щелкните пункт сообщения **Просмотр сведений**.

2. На вкладке **Группы защиты** добавьте группу защиты. Укажите имя, исходный узел Hyper-V, целевой ресурс **Azure**, имя подписки Azure Site Recovery и учетную запись хранилища Azure.

	![Protection group](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_ProtectionGroupCreate.png)


2. В поле **Настройки репликации** оставьте настройки по умолчанию.

	![Protection group](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_ProtectionGroup2.png)


<h3><a id="enablevirtual"></a>Шаг 6. Включение защиты виртуальной машины</h3>
<p>Включите защиту для виртуальных машин, добавляя их в группу защиты.</p>

1. На вкладке <b>Машины</b> для группы защиты щелкните <b>Add virtual machines to protection groups to enable protection</b> (Добавить виртуальные машины в группы защиты для обеспечения защиты).
2. На странице **Включить защиту виртуальной машины** выберите виртуальные машины, которые следует защитить. 

	![Enable virtual machine protection](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_AddVM1.png)

Запускается задание включения защиты. Ход выполнения операции можно отслеживать на вкладке **Задания**. После запуска задачи финализации защиты виртуальная машина готова к отработке отказа. После включения защиты и завершения начальной репликации, можно просмотреть виртуальные машины в Azure.

В Azure Site Recovery их можно просмотреть, выбрав элементы **Защищенные элементы** > **Группы защиты** > *protectiongroup_name* > **Виртуальные машины**. 


<h3><a id="recoveryplans"></a>Шаг 7. Тестирование развертывания</h3>

Проведите тестирование развертывания, чтобы имитировать механизм отработки отказа и восстановления в изолированной сети, не воздействуя на производственную среду. Для этого следует запустить тестовую отработку отказа для защищенной виртуальной машины.

1. **Защищенные элементы** > **Группы защиты** > *protectiongroup_name* > **Виртуальные машины** выберите виртуальную машину для отработки отказа и щелкните **Тестовая отработка отказа**.
2. На странице **Подтверждение тестовой отработки отказа** выберите **Нет**. 

	![Recovery plan](./media/hyper-v-recovery-manager-configure-vault/SRHVSite_TestFailoverNoNetwork.png)

3. Ход выполнения операции и состояние можно отслеживать на вкладке **Задания**.
4. После того как отработка отказа достигнет фазы **Полное тестирование**, выполните следующую проверку:
	- После отработки отказа просмотрите реплику виртуальной машины на портале Azure. Проверьте, успешно ли выполнен запуск виртуальной машины.
	- Если настройки предусматривают доступ к виртуальным машинам из локальной сети, можно инициировать подключение протокола удаленного рабочего стола к виртуальной машине.
	- Щелкните **Завершить проверку** для завершения действия.
	- Щелкните элемент **Примечания** для регистрации и сохранения любых наблюдений, связанных с тестовой отработкой отказа.
	- Щелкните пункт **Тестовая отработка отказа завершена**. Проведите очистку тестовой среды, чтобы автоматически отключить питание и удалить тестовую виртуальную машину.

<h2><a id="runtest" name="runtest" href="#runtest"></a>Мониторинг активности</h2>
<p>Можно использовать вкладку <b>Задания</b> и <b>Панель мониторинга</b>, чтобы просматривать и отслеживать выполнение основных заданий в хранилище Azure Site Recovery, в том числе настройку защиты для облака, включение и отключение защиты для виртуальной машины, выполнение отработки отказа (запланированной, незапланированной или тестовой) и фиксирование незапланированной отработки отказа.</p>

<p>На вкладке <b>Задания</b> можно просматривать задания, их подробности и ошибки, запускать запросы извлечения заданий, отвечающих конкретным критериям, экспортировать задания в Excel и перезапускать неудавшиеся задания.</p>

<p>Из <b>панели мониторинга</b> можно скачивать последние версии установочных файлов поставщика и агента, получать информацию о конфигурации хранилища, просматривать количество виртуальных машин, защитой которых управляет хранилище, просматривать последние задания, управлять сертификатом хранилища и проводить повторную синхронизацию виртуальных машин.</p>

<p>Дополнительную информацию о взаимодействии с заданиями и панелью мониторинга см. в <a href="http://go.microsoft.com/fwlink/?LinkId=398534">руководстве по эксплуатации и мониторингу</a>.</p>

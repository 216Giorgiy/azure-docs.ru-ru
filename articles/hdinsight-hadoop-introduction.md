# Hadoop в HDInsight — введение. Обработка и анализ данных большого размера в облаке.

Узнайте о компонентах HDInsight, общей терминологии, сценариях и ресурсах, которые помогут использовать Hadoop в HDInsight.

Azure HDInsight — это служба, которая позволяет развернуть и подготовить в облаке кластеры Apache Hadoop, обеспечивая программную платформу, предназначенную для управления, анализа и составления отчетов по данным большого размера. Ядро Hadoop обеспечивает надежное хранение данных в распределенной файловой системе Hadoop (HDFS), а также простую модель программирования MapReduce для параллельной обработки и анализа данных, хранящихся в этой распределенной системе.

### Что такое данные большого объема?

«Данными большого объема» называют данные, которые объединяются в большие объемы при постоянно увеличивающихся скоростях с использованием всего разнообразия неструктурированных форматов и семантических контекстов.

Данные большого размера — это любой большой массив цифровой информации: от текстов Twitter-канала, данных датчиков промышленного оборудования, до информации о просмотренных покупателем страницах и покупках в онлайн-каталоге. Данные большого размера могут быть историческими (т. е. сохраненными за прошлый период) или данными в реальном времени (т. е. транслироваться прямо из источника).

Чтобы данные большого размера оказались действительно пригодны для аналитики или выводов, важно не только подать правильный запрос и собрать нужные сведения. Такие данные должны быть доступны, отфильтрованы, проанализированы и представлены в удобном формате. Именно в этом Hadoop в HDInsight сможет вам помочь.

## <a name="overview"></a>Общая характеристика экосистемы Hadoop в HDInsight

Hadoop — это быстро развивающаяся технологическая платформа, предоставляющая готовые решения для работы с данными большого размера. HDInsight — это платформа, позволяющая реализовать Hadoop в облаке Microsoft Azure.

Кластеры Hadoop в HDInsight используют версии распределения платформы данных Hortonworks (HDP) и набор компонентов Hadoop в пределах этого распределения. Подробнее о компонентах и их версиях в кластерах HDInsight см. в статье [Что нового в версиях кластеров Hadoop, предоставленных HDInsight?][Что нового в версиях кластеров Hadoop, предоставленных HDInsight?]

Apache Hadoop представляет собой программную платформу для обработки и анализа данных большого размера. Ядро Apache Hadoop обеспечивает хранение данных в распределенной файловой системе Hadoop (HDFS), а также простую модель программирования MapReduce для параллельной обработки и анализа данных, хранящихся в этой распределенной системе. HDFS использует репликацию данных для устранения проблем, связанных со сбоем оборудования, которые возникают при развертывании подробных распределенных систем.

Ниже перечислены технологии Hadoop в HDInsight:

-   **[Ambari][Ambari].** Подготовка кластеров, отслеживание состояния кластеров и управление ими.
-   **[Avro][Avro]** (Библиотека Microsoft .NET для Avro). Сериализация данных для среды Microsoft .NET
-   **[HBase][HBase].** База данных нереляционного типа для таблиц с очень большим объемом информации
-   **[HDFS][HDFS].** Распределенная файловая система Hadoop
-   **[Hive][Hive].** Создание запросов по типу SQL
-   **[Mahout][Mahout].** Машинное обучение
-   **[MapReduce и YARN][MapReduce и YARN].** Распределенная обработка данных и управление ресурсами
-   **[Oozie][Oozie].** Управление рабочими процессами
-   **[Pig][Pig].** Более простой язык сценариев для вычислений MapReduce
-   **[Sqoop][Sqoop].** Импорт и экспорт данных
-   **[Storm][Storm].** Обработка быстрых, больших потоков данных в режиме реального времени.
-   **[Zookeeper][Zookeeper].** Координация процессов в распределенных системах.

### <a name="ambari"></a>Ambari

Apache Ambari используется для подготовки, отслеживания кластеров Apache Hadoop и управления ими. Он включает в себя набор удобных средств и надежных интерфейсов API, который упрощают работу с кластерами Hadoop. См. разделы [Отслеживание кластеров Hadoop в HDInsight с использованием интерфейсов Ambari API][Отслеживание кластеров Hadoop в HDInsight с использованием интерфейсов Ambari API] и [Информация о Apache Ambari API][Информация о Apache Ambari API].

### <a name="avro"></a>Avro (Библиотека Microsoft .NET для Avro)

В библиотеке Microsoft .NET для Avro реализуется компактный двоичный формат обмена данными Apache Avro для сериализации данных среды Microsoft.NET. В ней используется формат [JSON][JSON] для определения языково-независимой схемы, которая компенсирует несовместимость языков программирования. То есть данные, сериализированные на одном языке, могут быть прочитаны на другом языке. Подробную информацию о формате можно найти в разделах [Спецификации для Apache Avro][Спецификации для Apache Avro].
Формат Avro-файлов поддерживает модель распределенного программирования MapReduce. Файлы "разделяемы", т. е. вы сможете найти нужный фрагмент файла и начать считывание с нужного вам блока. Более подробно см. раздел [Сериализация данных с помощью библиотеки данных Microsoft .NET для Avro][Сериализация данных с помощью библиотеки данных Microsoft .NET для Avro]/

### <a name="hbase"></a>HBase

[Apache HBase][Apache HBase] — это нереляционная база данных, построенная на Hadoop и предназначенная для больших объемов неструктурированных и полуструктурированных данных, возможно, миллионов строк, умноженных на миллионы столбцов. Кластеры HBase в HDInsight сохраняют данные напрямую в хранилище BLOB-объектов Azure с низким значением задержки и повышенной эластичностью. См. раздел [Общие характеристики HBase в HDInsight][Общие характеристики HBase в HDInsight].

### <a name="hdfs"></a>HDFS

Распределенная файловая система Hadoop (HDFS) — это распределенная файловая система, которая вместе с MapReduce и YARN является сердцем экосистемы Hadoop. HDFS — это стандартная файловая система для кластеров Hadoop в HDInsight.

### <a name="hive"></a>Hive

[Apache Hive][Apache Hive] — приложение для хранилищ данных на основе Hadoop, которое позволяет выполнять запросы и управлять большими наборами данных в распределенном хранилище с помощью языка, подобного SQL, который называется HiveQL. Hive, как и Pig, является абстракцией на основе MapReduce и при выполнении преобразует запросы в последовательность заданий MapReduce. Hive концептуально ближе к системам управления реляционными базами данных, чем к Pig, и поэтому подходит для работы с более структурированными данными. Для неструктурированных данных более оптимальным выбором является Pig. См. [Использование Hive с Hadoop в HDInsight][Использование Hive с Hadoop в HDInsight]

### <a name="mahout"></a>Mahout

[Apache Mahout][Apache Mahout] — это масштабируемая библиотека алгоритмов машинного обучения, которые выполняются в Hadoop. Основываясь на статистических принципах, приложения для машинного обучения учат системы анализировать данные и использовать прошлые результаты для программирования дальнейшего поведения. См. раздел [Создание рекомендаций по фильмам с использованием Mahout в Hadoop][Создание рекомендаций по фильмам с использованием Mahout в Hadoop].

### <a name="mapreduce"></a>MapReduce и YARN

Hadoop MapReduce — это программная платформа для создания приложений, обрабатывающих большие массивы данных. Задача MapReduce разбивает данные на блоки и создает пары "ключ-значение" для последующей обработки.

Apache YARN — это следующее поколение MapReduce (MapReduce 2.0, или MRv2), которое разделяет две главные задачи JobTracker (управление ресурсами и планирование/отслеживание заданий) на разные сущности.

Более подробно о MapReduce см. [MapReduce][MapReduce] в Hadoop Wiki. Чтобы узнать больше о YARN, см. [Apache Hadoop NextGen MapReduce (YARN)][Apache Hadoop NextGen MapReduce (YARN)].

### <a name="oozie"></a>Oozie

[Apache Oozie][Apache Oozie] — это система координации рабочих процессов, которые управляют заданиями Hadoop. Это решение интегрировано со стеком Hadoop и поддерживает задания Hadoop для MapReduce, Pig, Hive и Sqoop. Его также можно использовать для планирования относящихся к системе заданий, таких как Java-программ и скриптов оболочки. См. раздел [Использование временного координатора Oozie с Hadoop в HDInsight][Использование временного координатора Oozie с Hadoop в HDInsight].

### <a name="pig"></a>Pig

[Apache Pig][Apache Pig] — это высокоуровневая платформа, которая позволяет выполнять сложные преобразования MapReduce для очень больших наборов данных с помощью простого языка скриптов Pig Latin. Pig переводит команды, написанные на языке Pig Latin, для использования в Hadoop. Вы можете создавать определяемые пользователем функции (UDFs) для расширения словаря Pig Latin. См. раздел [Использование Pig с Hadoop для анализа файла журнала Apache][Использование Pig с Hadoop для анализа файла журнала Apache].

### <a name="sqoop"></a>Sqoop

[Apache Sqoop][Apache Sqoop] — средство, которое передает массовые данные между Hadoop и реляционными базами данных, например SQL, или другими структурированными хранилищами данных с максимальной эффективностью. См. раздел [Использование Sqoop с Hadoop][Использование Sqoop с Hadoop].

### <a name="storm"></a>Storm

[Apache Storm][Apache Storm] — это распределенная вычислительная система реального времени для быстрой обработки больших потоков данных. Storm предлагается в качестве управляемого кластера в HDInsight. См. раздел [Анализ данных, передаваемых датчиками в реальном времени, с помощью Storm и Hadoop][Анализ данных, передаваемых датчиками в реальном времени, с помощью Storm и Hadoop].

### <a name="zookeeper"></a>Zookeeper

[Apache Zookeeper][Apache Zookeeper] координирует процессы в больших распределенных системах средствами общих иерархических пространств имен регистров данных (znode). Узлы znode содержат небольшие объемы метаинформации, необходимой для координации процессов: статус, расположение, конфигурацию и пр.

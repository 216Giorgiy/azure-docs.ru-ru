# Введение для Hadoop в HDInsight Обработка и анализ данных большого размера в облаке.

Узнайте о компонентах HDInsight, общей терминологии, сценариях и ресурсах, которые помогут использовать Hadoop в HDInsight.

Azure HDInsight - это служба, которая позволяет развернуть и подготовить в облаке кластеры Apache Hadoop, обеспечивая программную платформу, предназначенную для управления, анализа и составления отчетов по данным большого размера. Ядро Hadoop обеспечивает надежное хранение данных в распределенной файловой системе Hadoop (HDFS), а также простую модель программирования MapReduce для параллельной обработки и анализа данных, хранящихся в этой распределенной системе. 

### Что такое данные большого объема?
"Данными большого объема" называют данные, которые объединяются в большие объемы при постоянно увеличивающихся скоростях с использованием всего разнообразия неструктурированных форматов и семантических контекстов. 

Данные большого размера - это любой большой массив цифровой информации: от текстов Twitter-канала, данных датчиков промышленного оборудования, до информации о просмотренных покупателем страницах и покупках в онлайн-каталоге. Данные большого размера могут быть историческими (т. е. сохраненными за прошлый период) или данными в реальном времени (т. е. транслироваться прямо из источника). 

Чтобы данные большого размера оказались действительно пригодны для аналитики или выводов, важно не только подать правильный запрос и собрать нужные сведения. Такие данные должны быть доступны, отфильтрованы, проанализированы и представлены в удобном формате. Именно в этом Hadoop в HDInsight сможет вам помочь.

## <a name="overview"></a>Общая характеристика экосистемы Hadoop в HDInsight

Hadoop - это быстро развивающаяся технологическая платформа, предоставляющая готовые решения для работы с данными большого размера. HDInsight - это платформа, позволяющая реализовать Hadoop в облаке Microsoft Azure.
 
Кластеры Hadoop в HDInsight используют версии распределения платформы данных Hortonworks (HDP) и набор компонентов Hadoop в пределах этого распределения. Сведения о компонентах и их версиях в кластерах HDInsight см. в разделе [Новые возможности версий кластеров, предоставляемых HDInsight][component-versioning].

Apache Hadoop представляет собой программную платформу для обработки и анализа данных большого размера. Ядро Apache Hadoop обеспечивает хранение данных в распределенной файловой системе Hadoop (HDFS), а также простую модель программирования MapReduce для параллельной обработки и анализа данных, хранящихся в этой распределенной системе. HDFS использует репликацию данных для устранения проблем, связанных со сбоем оборудования, которые возникают при развертывании подробных распределенных систем.

Ниже перечислены технологии Hadoop в HDInsight:

* **[Ambari](#ambari):** Подготовка кластеров, отслеживание состояния кластеров и управление ими. 
* **[Avro](#avro)** (библиотека Microsoft .NET для Avro): Сериализация данных для среды Microsoft .NET 
* **[HBase](#hbase):** База данных нереляционного типа для таблиц с очень большим объемом информации
* **[HDFS](#hdfs):** Распределенная файловая система Hadoop
* **[Hive](#hive):** Создание запросов по типу SQL
* **[Mahout](#mahout):** Машинное обучение
* **[MapReduce и YARN](#mapreduce):** Распределенная обработка данных и управление ресурсами
* **[Oozie](#oozie):** Управление рабочими процессами
* **[Pig](#pig):** Более простой язык сценариев для вычислений MapReduce
* **[Sqoop](#sqoop):** Импорт и экспорт данных 
* **[Storm](#storm):** Обработка быстрых, больших потоков данных в режиме реального времени.
* **[Zookeeper](#zookeeper):** Координация процессов в распределенных системах.
 
Кроме того, узнайте о **[средствах бизнес-аналитики](#bi)**, которые можно использовать с HDInsight.

###<a name="ambari"></a>Ambari

Apache Ambari используется для подготовки, отслеживания кластеров Apache Hadoop и управления ими. Он включает в себя набор удобных средств и надежных интерфейсов API, который упрощают работу с кластерами Hadoop. См. разделы  [Отслеживание кластеров Hadoop в HDInsight с помощью интерфейса Ambari API](../hdinsight-monitor-use-ambari-api/) и <a target="_blank" href="https://github.com/apache/ambari/blob/trunk/ambari-server/docs/api/v1/index.md">Справочник по Apache Ambari API</a>.

### <a name="avro"></a>Avro (Библиотека Microsoft .NET для Avro)

В библиотеке Microsoft .NET для Avro реализуется компактный двоичный формат обмена данными Apache Avro для сериализации данных среды Microsoft.NET. В ней используется <a target="_blank" href="http://www.json.org/">JSON</a> для определения языково-независимой схемы, которая компенсирует несовместимость языков программирования. То есть данные, сериализированные на одном языке, могут быть прочитаны на другом языке. Подробную информацию об этом формате см. в <a target=_"blank" href="http://avro.apache.org/docs/current/spec.html">спецификации Apache Avro</a>. 
Формат файлов Avro поддерживает распределенную модель программирования MapReduce. Файлы "разделяемы", т. е. вы сможете найти нужный фрагмент файла и начать считывание с нужного вам блока. Более подробно см. раздел [Сериализация данных с помощью библиотеки данных Microsoft .NET для Avro](../hdinsight-dotnet-avro-serialization/).

### <a name="hbase"></a>HBase

<a target="_blank" href="http://hbase.apache.org/">Apache HBase</a> - это нереляционная база данных, построенная на Hadoop и предназначенная для больших объемов неструктурированных и полуструктурированных данных, возможно, миллионов строк, умноженных на миллионы столбцов. Кластеры HBase в HDInsight сохраняют данные напрямую в хранилище BLOB-объектов Azure с низким значением задержки и повышенной эластичностью. См. раздел [Общие характеристики HBase в HDInsight](../hdinsight-hbase-overview/).

### <a name="hdfs"></a>HDFS

Распределенная файловая система Hadoop (HDFS) - это распределенная файловая система, которая вместе с MapReduce и YARN является сердцем экосистемы Hadoop. HDFS - это стандартная файловая система для кластеров Hadoop в HDInsight.

### <a name="hive"></a>Hive

<a target="_blank" href="http://hive.apache.org/">Apache Hive</a> - это приложение для хранилищ данных на основе Hadoop, которое позволяет выполнять запросы и управлять большими наборами данных в распределенном хранилище с помощью языка, подобного SQL, который называется HiveQL. Hive, как и Pig, является абстракцией на основе MapReduce и при выполнении преобразует запросы в последовательность заданий MapReduce. Hive концептуально ближе к системам управления реляционными базами данных, чем к Pig, и поэтому подходит для работы с более структурированными данными. Для неструктурированных данных более оптимальным выбором является Pig. См. раздел [Использование Hive с Hadoop в HDInsight](../hdinsight-use-hive/).

### <a name="mahout"></a>Mahout

<a target="_blank" href="https://mahout.apache.org/">Apache Mahout</a> - это масштабируемая библиотека алгоритмов машинного обучения, которые выполняются в Hadoop. Основываясь на статистических принципах, приложения для машинного обучения учат системы анализировать данные и использовать прошлые результаты для программирования дальнейшего поведения. См. раздел [Создание рекомендаций по фильмам с использованием Mahout в Hadoop](../hdinsight-mahout/).

### <a name="mapreduce"></a>MapReduce и YARN
Hadoop MapReduce - это программная платформа для создания приложений, обрабатывающих большие массивы данных. Задача MapReduce разбивает данные на блоки и создает пары "ключ-значение" для последующей обработки. 

Apache YARN - это следующее поколение MapReduce (MapReduce 2.0, или MRv2), которое разделяет две главные задачи JobTracker (управление ресурсами и планирование/отслеживание заданий) на разные сущности.

Дополнительные сведения о Socket.IO см. в разделе <a target="_blank" href="http://wiki.apache.org/hadoop/MapReduce">MapReduce</a> на вики-сайте Hadoop. Дополнительные сведения о YARN см. в разделе <a target="_blank" href="http://hadoop.apache.org/docs/current/hadoop-yarn/hadoop-yarn-site/YARN.html">Apache Hadoop NextGen MapReduce (YARN)</a>.

### <a name="oozie"></a>Oozie
<a target="_blank" href="http://oozie.apache.org/">Apache Oozie</a> - это система координации рабочих процессов, которые управляют заданиями Hadoop. Это решение интегрировано со стеком Hadoop и поддерживает задания Hadoop для MapReduce, Pig, Hive и Sqoop. Его также можно использовать для планирования относящихся к системе заданий, таких как Java-программ и скриптов оболочки. См. раздел [Использование временного координатора Oozie с Hadoop в HDInsight](../hdinsight-use-oozie-coordinator-time/).

### <a name="pig"></a>Pig

<a  target="_blank" href="http://pig.apache.org/">Apache Pig</a> - это высокоуровневая платформа, которая позволяет выполнять сложные преобразования MapReduce для очень больших наборов данных с помощью простого языка скриптов Pig Latin. Pig переводит команды, написанные на языке Pig Latin, для использования в Hadoop. Вы можете создавать определяемые пользователем функции (UDFs) для расширения словаря Pig Latin. См. раздел [Использование Pig с Hadoop для анализа файла журнала Apache](../hdinsight-use-pig/).

### <a name="sqoop"></a>Sqoop
<a  target="_blank" href="http://sqoop.apache.org/">Apache Sqoop</a> является средством, которое позволяет наиболее оптимально обмениваться пакетными данными между Hadoop и реляционными базами данных, такими как SQL и другие структурированные хранилища данных. См. раздел [Использование Sqoop с Hadoop](../hdinsight-use-sqoop/).

### <a name="storm"></a>Storm
<a  target="_blank" href="https://storm.incubator.apache.org/">Apache Storm</a> - это распределенная вычислительная система реального времени для быстрой обработки больших потоков данных. Storm предлагается в качестве управляемого кластера в HDInsight. См. раздел [Анализ данных, передаваемых датчиками в реальном времени, с помощью Storm и Hadoop](../hdinsight-storm-sensor-data-analysis/).

### <a name="zookeeper"></a>Zookeeper
<a  target="_blank" href="http://zookeeper.apache.org/">Apache Zookeeper</a> координирует процессы в больших распределенных системах средствами общих иерархического пространства имен регистров данных (z-узел - znode). Znodes содержат небольшие фрагменты мета-данных, необходимых для координации процессов, таких как отслеживание статуса, конфигурации и т. д.

* Новейшие компоненты Hadoop. Дополнительные сведения см. в разделе [Новые возможности версий кластеров, предоставляемых HDInsight][component-versioning].

*  <a target="_blank" href="http://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html">HDFS</a>: дополнительные сведения об архитектуре и конструкции распределенной файловой системы Hadoop (HDFS), основной системы хранения, используемой приложениями Hadoop.		

* <a target="_blank" href="http://mapreduce.org/">MapReduce</a>: дополнительные сведения о платформе программирования для создания приложений Hadoop, которые позволяют быстро обрабатывать огромные объемы данных с помощью параллельных вычислений в больших кластерах компьютерных узлов.

# Введение для Hadoop в HDInsight

## Обзор

Azure HDInsight is a service that deploys and provisions Apache™ Hadoop® clusters in the cloud, providing a software framework designed to manage, analyze, and report on big data.

### Данные большого размера

Данные называют "данными большого размера" для обозначения их объединения в большие объемы на постоянно увеличивающихся скоростях и использованием всего разнообразия неструктурированных форматов и семантических контекстов. Коллекция данных большого размера сама по себе не имеет ценности для организации. Чтобы данные большого размера представляли ценность в плане аналитики, на основе которой можно принимать решения, важно не только формулировать надлежащие вопросы для сбора соответствующих данных, но и обеспечить доступность, чистку и анализ таких данных с последующим представлением в надлежащей манере, как правило, в сочетании с данными из ряда других источников, которые в совокупности обеспечат тот ракурс и контекст, который теперь называется гибридным.

### Apache Hadoop

Apache Hadoop представляет собой программную платформу, которая упрощает управление и анализ данных большого размера. Ядро Apache Hadoop обеспечивает хранение данных в распределенной файловой системе Hadoop (HDFS), а также простую модель программирования MapReduce для параллельной обработки и анализа данных, хранящихся в этой распределенной системе. HDFS использует репликацию данных для устранения проблем, связанных со сбоем оборудования, которые возникают при развертывании подробных распределенных систем.

### MapReduce и YARN

Чтобы упростить сложные, связанные с анализом неструктурированных данных из разных источников, модель программирования MapReduce предоставляет абстракцию ядра, которая гарантирует выполнение операций сопоставления и предварительного преобразования. Модель программирования MapReduce позволяет рассматривать все выполняемые задания в качестве математических операций с наборами данных, состоящих из пар "ключ – значение". Поэтому файлы на входе и выходе должны содержать наборы данных, которые состоят только из пар "ключ – значение". Основной вывод из этого ограничения: задания MapReduce являются по сути доступными для компоновки.

Другие проекты, связанные с Hadoop, например Pig и Hive, базируются на основе HDFS и платформы MapReduce. Проекты, подобные этому, используются для обеспечения более простого способа управления кластером, чем непосредственная работа с программами MapReduce. Например, Pig позволяет составлять программы на процедурном языке, который называется Pig Latin и компилируется в программы MapReduce, работающие в кластере. Он также предоставляет гибкие способы управления потоком данных. Hive – это инфраструктура хранилища данных, который обеспечивает табличное абстрагирование данных в файлах, хранящихся в кластере. Эти данные затем можно ставить в очередь с помощью инструкций, аналогичных SQL, которые в совокупности формируют непроцедурный язык, называемый HiveQL.

### HDInsight

Azure HDInsight делает Apache Hadoop доступным в качестве облачной службы. Это делает программную платформу HDFS/MapReduce и соответствующие проекты типа Pig, Hive и Oozie доступными в более простой, масштабируемой и экономичной среде.

К кластерам Hadoop, развернутым в HDInsight, добавлен второй головной узел для повышения доступности службы. Стандартные реализации кластеров Hadoop обыкновенно имеют один головной узел. HDInsight устраняет эту единственную точку отказа благодаря добавлению второго головного узла. Переход на новую конфигурацию кластера НA не изменяет цену кластера, если только клиенты не подготавливают кластеры с головным узлом очень большого размера.

Одной из основных эффективных функций HDInsight является способ управления и хранения данных. HDInsight в качестве файловой системы по умолчанию использует хранилище BLOB-объектов Windows Azure. Хранилище BLOB-объектов и HDFS являются разными файловыми системами, которые оптимизированы соответственно для хранения данных и для выполнения вычислений с этими данными.

-   Хранилище BLOB-объектов Azure предоставляет высоко масштабируемое и доступное, экономичное, долгосрочное, общедоступное решение по хранению данных, которые будут обрабатываться с помощью HDInsight.
-   Кластеры Hadoop, развернутые с помощью HDInsight в HDFS, оптимизированы для запуска задач расчетов MapReduce применительно к данным.

Кластеры HDInsight развертываются на вычислительных узлах Azure для выполнения задач MapReduce и могут удаляться пользователями после завершения этих задач. Хранение данных в кластерах HDFS после выполнения расчетов было бы дорогостоящим способом хранения данных. Хранилище BLOB-объектов является надежным решением хранилища Azure общего назначения. Поэтому хранение данных в BLOB-объектах позволяет удалять кластеры, использованные для вычислений, без потери пользовательских данных. Однако хранилище Blob-объектов является не только экономичным решением: Оно предоставляет полноценный интерфейс файловой системы HDFS, обеспечивающий клиентам удобство работы, предоставляя полноценный набор компонентов экосистемы Hadoop, который можно (по умолчанию) применять непосредственно к обрабатываемым данным.

HDInsight использует Azure PowerShell для настройки, запуска и последующей обработки заданий Hadoop. HDInsight также предоставляет соединитель Sqoop, который может использоваться для импорта данных из базы данных SQL Azure в HDFS или для экспорта данных в базу данных SQL Azure из HDFS.

HDInsight также поддерживает доступность YARN. Это новая универсальная распределенная платформа управления приложениями, которая заменила классическую платформу Apache Hadoop MapReduce для обработки данных в кластерах Hadoop. Эффективно выступает в качестве операционной системы Hadoop и преобразует однозадачную платформу работы с данными с пакетной обработкой в многозадачную платформу, которая поддерживает пакетную интерактивную потоковую обработку и обработку в режиме реального времени. Эта новая платформа управления повышает масштабируемость и эффективность использования кластера в соответствии с такими критериями, как гарантии по мощности, достоверность и соглашения об уровнях обслуживания.

Для импорта данных из Azure HDInsight или HDFS в Excel доступно решение Microsoft Power Query. Эта надстройка улучшает возможности самостоятельной бизнес аналитики в Excel, упрощая поиск данных и доступ к различным источникам данных. Помимо Power Query для интеграции таких средств бизнес-аналитики, как Excel, SQL Server Analysis Services и Reporting Services, доступен драйвер Microsoft Hive ODBC, который упрощает сквозной анализ данных.

## <a name="Ecosystem">Экосистема Hadoop в Azure </a>

### Введение

HDInsight предоставляет платформу реализации облачного решения корпорации Майкрософт по обработке данных большого размера. Эта федеративная экосистема позволяет управлять большими объемами данных, анализировать их, задействуя возможности параллельной обработки, предоставляемые моделью программирования MapReduce. В этом разделе упорядочены и кратко описаны технологии Hadoop, совместимые с Apache, которые могут использоваться в HDInsight.

HDInsight предоставляет возможности реализации Hive и Pig для интеграции обработки данных и возможностей хранения. Решение Майкрософт для данных большого размера интегрируется со средствами бизнес-аналитики Майкрософт, такими как службы SQL Server Analysis Services, службы Reporting Services, PowerPivot и Excel. Это позволяет выполнять операции бизнес-аналитики непосредственно с данными, которые хранятся и управляются HDInsight в хранилище BLOB-объектов.

Другие технологии, совместимые с Apache, и аналогичные технологии, которые являются частью экосистемы Hadoop и работают в кластерах Hadoop, также доступны для загрузки и использования вместе с HDInsight. К ним относятся технологии с открытым исходным кодом, например технология Sqoop, которая позволяет интегрировать HDFS с реляционными хранилищами данных.

### Pig

Pig — это высокоуровневый платформа для обработки данных большого размера в кластерах Hadoop. Pig состоит из языка потока данных, который называется Pig Latin. Этот язык поддерживает составление запросов к большим наборам данных и консольную среду выполнения. Программы, написанные на Pig Latin, состоят из рядов преобразований набора данных, которые конвертируются в ряды программы MapReduce. Абстракции на Pig Latin предоставляют расширенные структуры данных по сравнению с MapReduce и играют такую же роль в Hadoop, как SQL в реляционных СУБД. Pig Latin является полностью расширяемым языком. Определяемые пользователем функции, написанные на Java, Python, Ruby, C\# или JavaScript, могут быть вызваны для настройки каждого этапа пути обработки при составлении плана анализа. Дополнительные сведения см. в разделе [Добро пожаловать в Apache Pig!][Добро пожаловать в Apache Pig!]

### Hive

Hive – это распределенное хранилище данных, которое позволяет управлять данными, хранящимися в HDFS. Это модуль запросов Hadoop. Hive предназначается для аналитиков, имеющих обширные навыки работы с SQL, и предоставляет интерфейс, аналогичный SQL, и реляционную модель данных. Hive использует язык HiveQL, который является диалектом SQL. Hive, как и Pig, является абстракцией на основе MapReduce и при выполнении преобразует запросы в последовательность заданий MapReduce. Сценарии для Hive по своей концепции близко приближаются к сценариям реляционных СУБД и поэтому приемлемы для использования с более структурированными данными. Для неструктурированных данных более оптимальным выбором является Pig. Дополнительные сведения см. в разделе [Добро пожаловать в Apache Hive!][Добро пожаловать в Apache Hive!]

### Sqoop

Sqoop является средством, которое позволяет наиболее оптимально обмениваться пакетными данными между Hadoop и реляционными базами данных, такими как SQL и другие структурированные хранилища данных. Используйте Sqoop для импорта данных из внешних хранилищ структурированных данных в HDFS или реляционные системы, такие как Hive. Sqoop также позволяет извлекать данные из Hadoop и экспортировать извлеченные данные во внешние реляционные базы данных, корпоративные хранилища данных или любые другие типы хранилищ структурированных данных. Дополнительные сведения см. на веб-сайте [Apache Sqoop][Apache Sqoop].

### Oozie

Apache Oozie — это система рабочих процессов и координации, управляющая заданиями Hadoop. Это решение интегрировано со стеком Hadoop и поддерживает задания Hadoop Apache MapReduce, Apache Pig, Apache Hive и Apache Sqoop.. Его также можно использовать для планирования относящихся к системе заданий, таких как Java-программ и скриптов оболочки.

### Ambari

Apache Ambari используется для подготовки, отслеживания кластеров Apache Hadoop и управления ими. Он включает в себя набор удобных средств и надежных интерфейсов API, который упрощают работу с кластерами Hadoop. Дополнительные сведения об интерфейсах API см. в [справочнике по Ambari API][справочнике по Ambari API]. В настоящее время HDInsight поддерживает только функцию мониторинга Ambari. Ambari API 1.0 поддерживается кластерами HDInsight версии 2.1 и 3.0. Дополнительную информацию о Ambari см. на веб-сайте [Apache Ambari][Apache Ambari].

### Библиотека Microsoft Avro

В этой библиотеке реализуется система сериализации данных Apache Avro для среды Microsoft.NET. Apache Avro обеспечивает компактный формат обмена двоичными данными для сериализации. Библиотека использует [JSON][JSON] для определения не зависящей от языка схемы, которая гарантирует взаимодействие языков. Данные, сериализованные в одном языке, могут быть прочитаны в другом. В настоящее время поддерживаются C, C++, C\#, Java, PHP, Python и Ruby. Подробные сведения об этом формате см. в [Спецификации Аpache Avro][Спецификации Аpache Avro]. Обратите внимание, что текущая версия библиотеки Microsoft .NET для Avro не поддерживает часть удаленных вызовов процедур (RPC) этой спецификации.

Формат сериализации Apache Avro широко используется в Azure HDInsight и других средах Apache Hadoop. Это удобный способ для представления сложных структур данных в заданиях Hadoop MapReduce. Формат файлов Avro был разработан для поддержки распределенной модели программирования MapReduce. Основная особенность, которая обеспечивает распределение, состоит в «делимости» файлов в том смысле, что можно отыскать любую точку в файле и начать чтение с определенного блока. Дополнительную информацию см. в разделе [Сериализация данных с использованием библиотеки Microsoft Avro][Сериализация данных с использованием библиотеки Microsoft Avro].

## <a name="Scenarios"></a>Сценарии для HDInsight с использованием данных большого размера

В примерном сценарии показано использование HDInsight в качестве нерегламентированного анализа либо в пакетах, либо всего набора неструктурированных данных, который хранится на узлах Azure и не нуждается в частых обновлениях.

Эти условия применимые к различной деятельности в коммерческой, научной и правительственной сферах. Они могут включать, к примеру, мониторинг сбытовых цепочек в розничной торговле, отслеживание подозрительных торговых махинаций в финансовой сфере, спроса на коммунальные услуги, мониторинг качества воздуха и воды по целому ряду датчиков, а также мониторинг преступлений в городских агломерациях.

HDInsight (и технологии Hadoop в целом) наиболее всего подходят для обработки большого объема журнальных или заархивированных данных, которые не нуждаются в частом обновлении после записи и которые часто считываются, как правило, для выполнения полноценного анализа. Этот сценарий дополняет обработку данных в реляционных СУБД, которая работает с меньшими объемами данных (гигабайты вместо петабайтов) и которая должна постоянно обновляться или запрашиваться на предмет определенных точек данных в полном наборе данных. РСУБД лучше всего работает со структурированными данными, которые организованы и сохранены в соответствии с фиксированной схемой. MapReduce хорошо работает с неструктурированными данными без предварительно определенной схемы, так как позволяет интерпретировать обрабатываемые данные.

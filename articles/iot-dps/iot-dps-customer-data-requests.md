# <a name="summary-of-customer-data-request-features"></a>Сводка о возможностях запроса данных клиента

Служба подготовки устройств к добавлению в Центр Интернета вещей — это облачная служба на базе REST API, предназначенная для корпоративных клиентов, которая обеспечивает прозрачную автоматическую подготовку устройств к добавлению в Центр Интернета вещей с гарантией безопасности, начиная с устройства и заканчивая облаком.

[!INCLUDE [gdpr-intro-sentence](../../includes/gdpr-intro-sentence.md)]

Администратор клиента назначает отдельным устройствам идентификатор регистрации и идентификатор устройства. Данные этих устройств и об этих устройствах основаны на этих идентификаторах. Корпорация Майкрософт не сохраняет сведения и не имеет доступа к данным, которые позволяли бы сопоставить эти устройства с отдельными лицами.

Многие из устройств, управляемых службой подготовки устройств, не являются пользовательскими, например офисный термостат или фабричный робот. Однако клиенты могут считать некоторые устройства такими, которые содержат персональные данные, и по своему усмотрению использовать собственные средства или методы отслеживания инвентаризации, связывающие устройства с частными лицами. Служба подготовки устройств хранит все данные, связанные с устройствами, и управляет ими как персональными данными.

Администраторы клиентов могут использовать портал Azure или REST API службы, чтобы выполнять информационные запросы путем экспорта или удаления данных, связанных с идентификатором устройства или идентификатором регистрации.

> [!NOTE]
> Устройства, подготовленные в Центре Интернета вещей с помощью службы подготовки, имеют дополнительные данные, хранящиеся в службе Центра Интернета вещей. Чтобы выполнить полный запрос для данного устройства, ознакомьтесь со [справочной документацией Центра Интернета вещей](../iot-hub/iot-hub-customer-data-requests.md).

## <a name="deleting-customer-data"></a>Удаление данных клиента

Служба подготовки устройства хранит регистрации и записи о них. Регистрации содержат сведения об устройствах, которые можно подготовить, и записи о регистрации, показывающие, какие устройства уже прошли через процесс подготовки.

Администраторы клиентов могут удалить регистрации с портала Azure. При этом также удаляются все связанные записи о регистрации.

Дополнительные сведения см. в статье [Как управлять регистрацией устройств с помощью портала Azure](how-to-manage-enrollments.md).

Удалять регистрации и записи о них можно также с помощью REST API:

* Чтобы удалить информацию о регистрации одного устройства, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/deviceenrollment/delete).
* Чтобы удалить информацию о регистрации для группы устройств, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/deviceenrollmentgroup/delete).
* Чтобы удалить сведения о подготовленных устройствах, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/registrationstate/deleteregistrationstate).

## <a name="exporting-customer-data"></a>Экспорт данных клиента

Служба подготовки устройства хранит регистрации и записи о них. Регистрации содержат сведения об устройствах, которые можно подготовить, и записи о регистрации, показывающие, какие устройства уже прошли через процесс подготовки.

Администраторы клиентов могут просматривать регистрации и записи о них через портал Azure и экспортировать их с помощью копирования и вставки.

Дополнительные сведения о том, как управлять регистрациями, см. в статье [Как управлять регистрацией устройств с помощью портала Azure](how-to-manage-enrollments.md).

Экспортировать регистрации и записи о них можно также с помощью REST API:

* Чтобы экспортировать информацию о регистрации одного устройства, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/deviceenrollment/get).
* Чтобы экспортировать информацию о регистрации для группы устройств, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/deviceenrollmentgroup/get).
* Чтобы экспортировать сведения об устройствах, которые уже были подготовлены, перейдите к [этой статье](https://docs.microsoft.com/rest/api/iot-dps/registrationstate/getregistrationstate).

> [!NOTE]
> При использовании корпоративных служб корпорация Майкрософт создает некоторые данные, известные как создаваемые системой журналы. Создаваемые системой журналы службы подготовки устройств недоступны и не подлежат экспорту администраторами клиента. Эти журналы содержат сведения о фактических действиях, выполненных в службе, и диагностические данные, связанные с отдельными устройствами.

# <a name="how-to-revoke-device-access-to-your-provisioning-service-in-the-azure-portal"></a>Как отменять доступ устройства к инициализации службы на портале Azure

Правильного управления учетными данными устройства крайне важен для систем высокой профиля, как решения IoT. Для таких систем рекомендуется составить план очистить того, как отменить доступ для устройств в случаях, где свои учетные данные, маркер SAS или сертификат X.509, может представлять угрозу безопасности. В этой статье описывается отмены доступа к устройству, на этапе подготовки.

Дополнительные сведения об отзыве доступа к устройству для центра IoT после подготовки устройства. в разделе [отключения устройства](https://docs.microsoft.com/en-us/azure/iot-hub/iot-hub-devguide-identity-registry#disable-devices).

> [!NOTE] 
> Имейте в виду политики повтора, отменить доступ для устройств. Например устройства с помощью политик бесконечного повтора постоянно может попытаться зарегистрировать с помощью подготовки службы использует службы ресурсов и возможно, влияющие на производительность.

## <a name="blacklist-devices-with-an-individual-enrollment-entry"></a>Черный список устройств с помощью отдельных регистрации записи

Отдельных регистраций применяются к одному устройству и может использовать сертификаты X.509 или маркеры SAS (в доверенном платформенном МОДУЛЕ реальными или виртуальными) в качестве механизма аттестации. (Устройства, использующие токены SAS, как их аттестации механизм может предоставляться только через отдельные регистрации). Чтобы защитить черным списком устройство, которое имеет отдельные регистрации, можно отключить или удалить его запись регистрации: 

- Чтобы временно защитить черным списком устройства, можно отключить его запись регистрации. 

    1. Войдите на портал Azure и нажмите кнопку **все ресурсы** в левом меню.
    2. Нажмите кнопку подготовки службы, которую требуется защитить черным списком устройство в списке ресурсов.
    3. В службе подготовки выберите **Manage enrollments** (Управление регистрациями), а затем — вкладку **Individual Enrollments** (Отдельные регистрации).
    4. Щелкните запись регистрации устройства, которое вы хотите защитить черным списком, чтобы открыть его. 
    5. Нажмите кнопку **отключить** в нижней части списка запись регистрации, нажмите кнопку **Сохранить**.  

        ![Отключить запись отдельных регистрации на портале](./media/how-to-revoke-device-access-portal/disable-individual-enrollment.png)
    
- Чтобы окончательно защитить черным списком устройства, можно удалить его запись регистрации.

    1. Войдите на портал Azure и нажмите кнопку **все ресурсы** в левом меню.
    2. Нажмите кнопку подготовки службы, которую требуется защитить черным списком устройство в списке ресурсов.
    3. В службе подготовки выберите **Manage enrollments** (Управление регистрациями), а затем — вкладку **Individual Enrollments** (Отдельные регистрации).
    4. Установите флажок рядом с записью регистрации устройства, которое вы хотите защитить черным списком. 
    5. Нажмите кнопку **удаление** в верхней части окна, затем нажмите кнопку **Да** подтверждения удаления регистрации. 

        ![Удалить запись отдельных регистрации на портале](./media/how-to-revoke-device-access-portal/delete-individual-enrollment.png)
    
    6. После завершения действия вы увидите записи удаляются из списка отдельных регистраций.  

## <a name="blacklist-an-x509-intermediate-or-root-ca-certificate-using-an-enrollment-group"></a>Защитить черным списком X.509 промежуточного или с помощью регистрации групповой сертификат корневого ЦС

Сертификаты X.509, обычно располагаются в цепочке доверия сертификатов. В случае компрометации сертификата на любом этапе в цепочке становится доверия разрывается, и сертификат должен черный для предотвращения устройств работы потока в любой цепочки сертификатом из идет подготовка службы подготовки устройства. Дополнительные сведения о сертификатах X.509 и как они используются в службе подготовки см. в разделе [сертификаты X.509](./concepts-security.md#x509-certificates). 

Группа регистрации — записи для устройств, которые совместно используют общий механизм аттестации подписаны одним и тем же промежуточных сертификатов X.509 или корневой ЦС. Запись регистрации группы настраивается с помощью сертификата X.509, связанного с промежуточных или корневого ЦС, а также любой значения конфигурации, например двойных состояния и подключение концентратора IoT, которые являются общими для устройств с этим сертификатом в свой сертификат Цепочка. Чтобы защитить черным списком сертификат, можно отключить или удалить группу его регистрации:

- Чтобы временно защитить черным списком сертификат, можно отключить ее регистрации группы. 

    1. Войдите на портал Azure и нажмите кнопку **все ресурсы** в левом меню.
    2. Нажмите кнопку подготовки службы, которую требуется защитить черным списком сертификат подписи из списка ресурсов.
    3. В службе подготовки выберите **Manage enrollments** (Управление регистрациями), а затем — вкладку **Enrollment Groups** (Группы регистраций).
    4. Щелкните группу регистрации сертификатов для сертификата, который вы хотите защитить черным списком, чтобы открыть его.
    5. Нажмите кнопку **изменить группу** в левом верхнем регистрации записи группы.
    6. Чтобы отключить запись регистрации, выберите **отключить** на **включить запись** переключения, а затем нажмите кнопку **Сохранить**.  

        ![Отключить запись регистрации группы на портале](./media/how-to-revoke-device-access-portal/disable-enrollment-group.png)

    
- Чтобы окончательно защитить черным списком сертификат, можно удалить группу его регистрации.

    1. Войдите на портал Azure и нажмите кнопку **все ресурсы** в левом меню.
    2. Нажмите кнопку подготовки службы, которую требуется защитить черным списком устройство в списке ресурсов.
    3. В службе подготовки выберите **Manage enrollments** (Управление регистрациями), а затем — вкладку **Enrollment Groups** (Группы регистраций).
    4. Установите флажок рядом с группой регистрации сертификатов для сертификата, который вы хотите защитить черным списком. 
    5. Нажмите кнопку **удаление** в верхней части окна, затем нажмите кнопку **Да** подтверждение удаления группы регистрации. 

        ![Удалить запись регистрации группы на портале](./media/how-to-revoke-device-access-portal/delete-enrollment-group.png)

    6. После завершения действия вы увидите запись удалена из списка групп регистрации.  

> [!NOTE]
> При удалении группы подачи заявки на сертификат, устройствах, имеющих этот сертификат в своей цепочке сертификатов по-прежнему можно зарегистрировать, если включена регистрация группы для корневой сертификат или промежуточный сертификат другой выше в свой сертификат существует цепочки.

## <a name="blacklist-specific-devices-in-an-enrollment-group"></a>Черный список конкретных устройств в группе регистрации

Устройства, которые реализуют механизм аттестации X.509 использовать устройства цепочки сертификатов и закрытый ключ для проверки подлинности. Когда устройство подключается и выполняет проверку подлинности с помощью службы подготовки устройств, службы сначала ищет отдельных регистрации, соответствующие учетные данные устройства перед поиском регистрации групп, чтобы определить, можно ли подготовить устройства. Если служба обнаруживает отключенные отдельных регистрации устройства, его устройство не подключение, даже если существует группа включена регистрация для промежуточных или корневого ЦС в цепочке сертификатов устройства. Чтобы защитить черным списком отдельное устройство в группе регистрации, выполните следующие действия:

1. Войдите на портал Azure и выберите **Все ресурсы** в меню слева.
2. Список ресурсов щелкните службу подготовки, содержащего группу регистрации устройства, которое вы хотите защитить черным списком.
3. В службе подготовки выберите **Manage enrollments** (Управление регистрациями), а затем — вкладку **Individual Enrollments** (Отдельные регистрации).
4. Нажмите кнопку **Добавить** вверху. 
5. Выберите **X.509** в качестве механизма безопасности для устройства, а затем отправьте сертификат устройства. Это знаком конечного субъекта сертификата, установленных на устройстве, который используется для создания сертификатов для проверки подлинности.
6. Введите **идентификатор устройства центра IoT** для устройства. 
7. Чтобы отключить запись регистрации, выберите **отключить** на **включить запись** переключения. 
8. Выберите команду **Сохранить**. Для успешного создания соглашение о регистрации, вы увидите устройства отображаются в **отдельных регистраций** вкладки. 

    ![Отключить запись отдельных регистрации на портале](./media/how-to-revoke-device-access-portal/disable-individual-enrollment.png)

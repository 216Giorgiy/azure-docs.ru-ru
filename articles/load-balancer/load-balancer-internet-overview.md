# Доступная в Интернете подсистема балансировки нагрузки, используемая для нескольких виртуальных машин или служб

Один из вариантов использования конечных точек — конфигурация подсистемы балансировки нагрузки Azure для распределения определенного типа трафика между несколькими виртуальными машинами или службами. Например, можно распределить нагрузку от трафика веб-запросов на несколько веб-серверов или веб-ролей.

Подсистема балансировки нагрузки Azure сопоставляет общедоступный IP-адрес и номер порта для входящего трафика с частным IP-адресом и номером порта виртуальной машины и наоборот для ответного трафика из виртуальной машины.

>[AZURE.NOTE]При настройке балансировки нагрузки трафика среди нескольких виртуальных машин или служб, используя параметры по умолчанию, входящий трафик будет распределяться в случайном порядке. Если вам необходимо соответствие сеансу (или прикрепленные сеансы), см. статью [о режиме распределения подсистемы балансировки нагрузки](load-balancer-distribution-mode.md).

Для облачной службы, которая содержит экземпляры веб-ролей или рабочих ролей, в определении службы (CSDEF-файл) можно определить общедоступную конечную точку.
 
В файле servicedefinition.csdef будет содержаться конфигурация конечной точки. При наличии нескольких экземпляров роли для развертывания веб-ролей или рабочих ролей для конфигурации этой точки будет настроена подсистема балансировки нагрузки. Чтобы добавить экземпляры для облачного развертывания, измените количество экземпляров в CSFG-файле конфигурации службы.

На следующем рисунке показана конечная точка с балансировкой нагрузки для незашифрованного веб-трафика, которая является общей для трех виртуальных машин на общедоступном и частном TCP-портах 443. Эти три виртуальные машины находятся в наборе балансировки нагрузки.


![пример общедоступной подсистемы балансировки нагрузки](./media/load-balancer-internet-overview/IC727496.png))



Когда интернет-клиенты отправляют запросы веб-страниц на общедоступный IP-адрес облачной службы и TCP-порт 443, подсистема балансировки нагрузки Azure выполняет случайную балансировку этих запросов между тремя виртуальными машинами в наборе балансировки нагрузки.

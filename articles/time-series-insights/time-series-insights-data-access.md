# <a name="grant-data-access-to-a-time-series-insights-environment-using-azure-portal"></a>Предоставление доступа к данным для среды Time Series Insights с помощью портала Azure

В этой статье рассматриваются два типа политик доступа службы "Аналитика временных рядов".

## <a name="video"></a>Видео: 

### <a name="in-this-video-we-cover-creating-and-managing-access-policies-within-time-series-insights-br"></a>В этом видео рассматривается создание политик доступа и управление ими в службе "Аналитика временных рядов". </br>

> [!VIDEO https://www.youtube.com/embed/5zTIdyHMJW8]

Среда Time Series Insights содержит 2 типа независимых политик доступа:

* политики управления доступом;
* политики доступа к данным.

Оба типа политик предоставляют субъектам Azure Active Directory (пользователям или приложениям) различные разрешения для конкретной среды. Субъекты (пользователи и приложения) должны принадлежать к службе Active Directory (клиенту Azure), связанной с подпиской, которая содержит среду.

Политики управления доступом предоставляют разрешения, связанные с конфигурацией среды, например:
*   создание и удаление среды, источников событий и справочных наборов данных;
*   управление политиками доступа к данным.

Политики доступа к данным предоставляют разрешения для отправки запросов данных, обработки ссылочных данных в среде, а также предоставляют другим пользователям среды доступ к сохраненным запросам и перспективам.

Эти политики позволяют четко разделить доступ к управлению среды и доступ к данным в среде. Например, можно настроить среду так, чтобы владелец или создатель среды не имели доступа к данным. Пользователи и службы, которые могут считывать данные из среды, также могут не иметь доступ к конфигурации среды.

## <a name="grant-data-access"></a>Предоставление доступа к данным
Выполните следующие действия, чтобы предоставить доступ к данным для субъекта-пользователя.

1. Войдите на [портале Azure](https://portal.azure.com).

2. Найдите среду "Аналитика временных рядов". Введите **Временные ряды** в поле **поиска**. Выберите **Среда временных рядов** в результатах поиска. 

3. Выберите среду Time Series Insights в списке.
   
4. Выберите **Политики доступа к данным**, а затем нажмите кнопку **+Добавить**.
  ![Управление исходной средой "Аналитика временных рядов"](media/data-access/getstarted-grant-data-access1.png)

5. Щелкните **Выбор пользователя**.  Выполните поиск по имени пользователя или адресу электронной почты, чтобы найти пользователя, которого следует добавить. Нажмите **Выбрать**, чтобы подтвердить выбор. 

   ![Управление источником Time Series Insights — "Добавить"](media/data-access/getstarted-grant-data-access2.png)

6. Щелкните **Выбор ролей**. Выберите соответствующую роль доступа для пользователя.
   - Выберите **Участник**, если вы хотите разрешить пользователю изменять ссылочные данные и предоставить другим пользователям среды доступ к сохраненным запросам и перспективам. 
   - В противном случае выберите **Читатель**, чтобы разрешить пользователю обращаться к данным в среде и сохранять в ней личные (не общие) запросы.

   Нажмите кнопку **ОК**, чтобы подтвердить выбор роли.

   ![Управление источником Time Series Insights — "Выбор пользователя"](media/data-access/getstarted-grant-data-access3.png)

8. На странице **Выбор роли пользователя** нажмите кнопку **ОК**.

   ![Управление источником Time Series Insights — "Выбор ролей"](media/data-access/getstarted-grant-data-access4.png)

9. На странице **Политики доступа к данным** перечислены пользователи и соответствующие роли.

   ![Управление источником Time Series Insights — результаты](media/data-access/getstarted-grant-data-access5.png)

## <a name="provide-guest-access-to-a-user-from-another-aad-tenant"></a>Предоставление гостевого доступа пользователю другого клиента Azure Active Directory

Роль "Гость" не является ролью управления. Это термин, используемый для учетной записи, приглашенной из одного клиента в другой. После приглашения гостевой учетной записи в каталог клиента к ней может быть применен тот же элемент управления доступом, что и к другим учетным записям. Ей может быть предоставлен доступ к управлению в среде TSI с использованием колонки "Управление доступом (IAM)" или доступ к данным в среде с помощью колонки Data Access Policies (Политики доступа к данным). Дополнительные сведения о гостевом доступе к клиенту Azure Active Directory см. в статье [Добавление пользователей службы совместной работы B2B на портале Azure](https://docs.microsoft.com/azure/active-directory/b2b/add-users-administrator).

Выполните эти действия, чтобы предоставить гостевой доступ к среде службы "Аналитика временных рядов" пользователю Azure Active Directory из другого клиента.

1. Войдите на [портале Azure](https://portal.azure.com).

2. Найдите среду "Аналитика временных рядов". Введите **Временные ряды** в поле **поиска**. Выберите **Среда временных рядов** в результатах поиска.

3. Выберите среду Time Series Insights в списке.

4. Выберите **Data Access Policies** (Политики доступа к данным), а затем щелкните **+ Пригласить**.

    ![Управление источником службы "Аналитика временных рядов" — приглашение пользователя](media/data-access/getstarted-grant-data-access6.png)

5. Укажите адрес электронной почты пользователя, которого необходимо пригласить. Обратите внимание, что это должно быть сообщение электронной почты, связанное с Azure Active Directory. Вы также можете добавить к приглашению личное сообщение.

    ![Управление источником Time Series Insights — "Выбор пользователя"](media/data-access/getstarted-grant-data-access7.png)

6. На экране отобразится всплывающее оповещение с подтверждением.

    ![Управление источником службы "Аналитика временных рядов" — подтверждение пользователя](media/data-access/getstarted-grant-data-access8.png)

7. Щелкните **Выбор пользователя**. Найдите адрес электронной почты приглашенного гостевого пользователя, чтобы найти пользователя, которого необходимо добавить. Нажмите **Выбрать**, чтобы подтвердить выбор.
  
    ![Управление источником службы "Аналитика временных рядов" — подтверждение пользователя](media/data-access/getstarted-grant-data-access9.png)

8. Щелкните **Выбор ролей**. Выберите соответствующую роль гостевого пользователя.

    * Выберите **Участник**, если вы хотите разрешить пользователю изменять ссылочные данные и предоставить другим пользователям среды доступ к сохраненным запросам и перспективам.

    * В противном случае выберите **Читатель**, чтобы разрешить пользователю обращаться к данным в среде и сохранять в ней личные (не общие) запросы.

    Нажмите кнопку **ОК**, чтобы подтвердить выбор роли.

    ![Управление источником Time Series Insights — "Выбор ролей"](media/data-access/getstarted-grant-data-access10.png)

9. На странице **Выбор роли пользователя** нажмите кнопку **ОК**.

10. Сейчас на странице **Data Access Policies** (Политики доступа к данным) перечислены гостевые пользователи и соответствующие роли.

    ![Управление источником службы "Аналитика временных рядов" — подтверждение роли](media/data-access/getstarted-grant-data-access11.png)

11. Теперь гостевому пользователю необходимо выполнить определенные действия, чтобы получить доступ к среде, находящейся в клиенте Azure, в которую вы его пригласили. Сначала этот пользователь должен принять отправленное ему приглашение. Это приглашение отправляется по адресу электронной почты, использованной в приглашении на шаге 5. Чтобы принять приглашение, пользователь должен щелкнуть "Начало работы".

    ![Управление источником службы "Аналитика временных рядов" — приглашение пользователя](media/data-access/getstarted-grant-data-access12.png)

12. Затем гостевому пользователю потребуется принять разрешения, связанные с организацией администратора.

    ![Управление источником службы "Аналитика временных рядов" — принятие разрешений](media/data-access/getstarted-grant-data-access13.png)

13. Когда гостевой пользователь войдет в учетную запись электронной почты, куда вы отправили приглашение, и примет это приглашение, он перейдет на сайт insights.azure.com. После этого ему понадобится щелкнуть аватар рядом с его адресом электронной почты в правом верхнем углу экрана. 

    ![Управление источником службы "Аналитика временных рядов" — принятие разрешений](media/data-access/getstarted-grant-data-access14.png)

14. Затем гостевой пользователь должен выбрать клиента Azure из раскрывающегося меню каталога. Это должен быть клиент, для которого он получил приглашение. 

    ![Управление источником службы "Аналитика временных рядов" — принятие разрешений](media/data-access/getstarted-grant-data-access15.png)

15. Наконец, после выбора клиента для гостевого пользователя станет доступной среда службы "Аналитика временных рядов", к которой вы предоставили ему доступ. Теперь для него должны быть доступны все возможности, связанные с ролью, предоставленной на шаге 8.

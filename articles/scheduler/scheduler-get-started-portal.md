# Приступая к работе с планировщиком Azure на классическом портале Azure

## Приступая к работе

Создавать задания и коллекции заданий с помощью планировщика Azure очень просто. В этом руководстве показано, как создавать коллекции для хранения заданий и задания в коллекции, а также приводится обзор доступных задач мониторинга и управления заданиями. При использовании этого руководства вам потребуется опыт работы с Azure.

При первом открытии классического портала Azure автоматически открывается вкладка **ВСЕ ЭЛЕМЕНТЫ**. Столбцы на вкладке **ВСЕ ЭЛЕМЕНТЫ** можно сортировать. Чтобы просмотреть задания и коллекции заданий в планировщике, откройте вкладку **ПЛАНИРОВЩИК**.

![][1]

## Создание коллекции заданий и задания

1.  Войдите на [классический портал Azure](https://manage.windowsazure.com/).  

2.  Откройте **Службы приложений** > **Создать** > **Планировщик** и выберите **Настраиваемое создание**. <br /><br /> ![][2]

3.  В разделе **Коллекция заданий** выберите имя коллекции заданий в раскрывающемся списке **Коллекция заданий**. Если у вас еще нет коллекции заданий, нажмите кнопку **Создать** и введите название для обозначения новой коллекции заданий.<br /><br /> ![][3]

4.  В поле **Регион** выберите географический регион для коллекции заданий.

5.  Нажмите кнопку со стрелкой, чтобы создать коллекцию заданий и перейти к следующему этапу — созданию задания.

6.  Создадим задание, которое будет только открывать сайт http://www.microsoft.com/ с помощью запроса GET. На экране **Действие задания** укажите в полях формы следующие значения:

    1.  **Имя:** ` getmicrosoft`  

    2.  **Тип действия:** ` HTTP`

    3.  **Метод:** ` GET`

    4.  **URI:** ` http://www.microsoft.com`

   	![][4]

7.  После создания задания определите расписание. Задание можно определить как разовое, но в данном случае мы выберем повторяющееся. На некоторых снимках экрана в данном руководстве показано повторение с периодичностью 1 минута, но мы укажем 12 часов.

    1.  **Повторять каждые:** ` 12 Hours`  

    2.  **Запуск:** ` Now`

    3.  **Окончание в:** ` Select date 2 days after current day and any time`

   	![][5]

8.  Нажмите кнопку **ОК**. Для создания задания и коллекции заданий может потребоваться некоторое время. Для проверки состояния можно отслеживать уведомления в нижней части портала.

   	![][6]

   	После того как коллекция заданий и задание будут созданы, появится соответствующее сообщение. Задание появится в разделе "Задания" планировщика, а коллекция заданий — в разделе "Коллекции заданий". Настройка дополнительных параметров задания рассматривается в приведенном ниже разделе «Настройка задания».

   	![][7]

## Управление и мониторинг заданий и коллекций заданий

Созданная коллекция заданий отображается на экране управления планировщиком.

![][8]

При выборе коллекции заданий открывается новое окно со следующими параметрами.

1.  Панель мониторинга  

2.  Масштаб

3.  Журнал

4.  Задания

В следующих разделах эти вкладки описываются более подробно.

### Панель мониторинга

Если щелкнуть название коллекции заданий, открывается вкладка **Панель мониторинга**. На панели мониторинга отображаются следующие сведения.

![][9]

#### Обзор использования заданий и выполнения заданий

Таблица и ряд диаграмм, отображающих фиксированный список метрик. Эти метрики содержат значения, обновляемые в режиме реального времени и отражающие состояние вашей коллекции заданий, включая следующее.

1.  Текущие задания  

2.  Завершенные задания

3.  Невыполненные задания

4.  Включенные задания

5.  Отключенные задания

6.  Выполнение заданий

#### Краткий обзор

Таблица с фиксированным списком метрик статуса и параметров. Эти метрики содержат значения, которые указывают на статус и параметры вашей коллекции заданий и обновляются в режиме реального времени.

1.  Состояние  

2.  Регион

3.  Количество ошибок

4.  Количество незавершенных выполнений

5.  URI

### Масштаб

На вкладке **Масштаб** можно изменять параметры уровня обслуживания планировщика.

![][10]

#### Общие сведения

Эта вкладка отображается, если вы пользуетесь **бесплатным** или **стандартным** планом обслуживания.

#### Квоты

Планировщик Azure реализует квоты с учетом нескольких условий. Этот раздел содержит список пороговых значений квот и позволяет их изменять. По умолчанию настраивается один набор квот. Ограничения, накладываемые этими параметрами квот, определяются планом, изменение которого может отразиться на цене. Для масштабирования планировщика квоты можно изменить. Доступные параметры:

1.  Максимальное число заданий  

2.  Максимальная частота

3.  Максимальный интервал

### Журнал

На вкладке **Журнал** отображаются следующие сведения о выбранном задании.

![][11]

#### Таблица журнала

Таблица, в которой отображаются выбранные метрики по каждому случаю выполнения выбранного задания в системе. Эти метрики содержат обновляемые в реальном времени значения, которые отражают работоспособность планировщика.

#### Доступные метрики

Доступны следующие счетчики производительности и метрики:

1.  Состояние  

2.  Сведения

3.  Повторная попытка

4.  Количество выполнений (1, 2, 3 и т. д.)

5.  Метка времени выполнения

Щелкнув **Просмотр информации журнала**, можно просмотреть полный ответ для каждого выполнения. В этом диалоговом окне также можно скопировать ответ в буфер обмена.

![][12]

### Задания

На вкладке «Задания» отображаются следующие сведения для мониторинга журнала выполнения заданий.

![][13]

#### Таблица заданий

Таблица, в которой отображаются выбранные метрики каждого задания в системе. Эти метрики содержат обновляемые в реальном времени значения, которые отражают работоспособность планировщика.

#### Отключение, включение и удаление задания

Щелкнув на имя задания, можно включить, отключить и удалить это задание. Удаленные задания восстановить нельзя.

#### Доступные метрики

Доступны следующие счетчики и метрики.

1.  Имя  

2.  Последнее выполнение

3.  Следующее выполнение

4.  Состояние

5.  Частота

6.  Сбои

7.  Ошибки

8.  Выполнения

9.  Тип действия

### Настройка задания

Чтобы изменить параметры задания, выберите его на экране **Задания**. Это позволит вам настроить дополнительные параметры, которые нельзя настроить в мастере быстрого создания. Чтобы настроить задание, нажмите стрелку вправо рядом с его именем на экране **Задания**.

Страница конфигурации задания позволяет обновить его параметры. Показанная ниже страница настройки задания относится к заданиям HTTP и HTTPS. Типы заданий HTTP и HTTPS позволяют изменить метод для любого допустимого глагола HTTP. Кроме того, можно добавлять, удалять или изменять заголовки и данные обычной проверки подлинности.

![][14]

Показанная ниже страница настройки задания относится к заданиям очереди хранения. Типы заданий очереди хранения позволяют изменить учетную запись хранения, имя очереди, маркер SAS и текст. Раздел «Расписание» (не показан ниже) в данном случае будет точно такой же, как для заданий HTTP/HTTPS.

![][15]

И, наконец, для всех типов действий можно менять само расписание и его повторение. Можно изменить дату и времени начала, расписание повторения, а также дату и время окончания (если задание повторяется.) Чтобы сохранить внесенные изменения, нажмите кнопку **Сохранить**. Чтобы отменить изменения, нажмите кнопку **Удалить**.

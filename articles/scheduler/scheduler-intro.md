# Что такое планировщик?

Планировщик Azure позволяет описывать действия, которые выполняются в облаке. Затем он планирует и выполняет эти действия автоматически. Планировщик Azure делает это с помощью [портала Azure](scheduler-get-started-portal.md), кода, [REST API](https://msdn.microsoft.com/library/dn528946) или PowerShell.

Планировщик Azure обслуживает, управляет, планирует и вызывает запланированную работу. Планировщик Azure не размещает какую-либо рабочую нагрузку и не выполняет код. Он только _вызывает_ код, размещенные в другом месте. Этот код может размещаться в Azure, на локальном компьютере или у другого поставщика. Код вызывается через HTTP, HTTPS или очередь хранилища.

Планировщик Azure планирует задания, ведет журнал результатов выполнения заданий, который можно запросить, и детерминированно и безотказно планирует рабочие нагрузки. Запланированные сценарии мобильных служб Azure, веб-задания веб-приложений Azure и другие возможности планирования Azure используют планировщик Azure незаметно для пользователя. [REST API планировщика](https://msdn.microsoft.com/library/dn528946) помогает управлять обменом данными для этих действий. Таким образом, планировщик поддерживает [сложные расписания и расширенное повторение](scheduler-advanced-complexity.md).

Существует несколько вариантов использования планировщика Azure. Например:

+ _Повторение действий приложения_: периодический сбор данных из Twitter и передача данных в веб-канал.
+ _Ежедневное обслуживание_: ежедневная очистка журналов, создание резервных копий и другие задачи обслуживания. Например, администратор может настроить резервное копирование базы данных в 13:00 каждый день в течение следующих 9 месяцев.

Планировщик позволяет создавать, обновлять, удалять, просматривать [коллекции заданий и задания](scheduler-concepts-terms.md), а также управлять ими программно, с помощью сценариев или на портале.

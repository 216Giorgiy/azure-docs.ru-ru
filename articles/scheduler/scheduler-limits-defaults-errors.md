## Заголовок x-ms-идентификатор-запроса

Каждый запрос к службе планировщика возвращает заголовок ответа с именем **x-ms-идентификатор-запроса**. Этот заголовок содержит непрозрачное значение, которое однозначно определяет запрос.

Если запрос постоянно завершается неудачей, но однозначно составлен правильно, используйте это значение, чтобы сообщить об ошибке в корпорацию Майкрософт. Укажите в отчете значение ofx-ms-идентификатор-запроса, примерное время создания запроса, идентификатор подписки, облачную службу, коллекцию заданий и (или) задание и тип операции, которую пытался выполнить запрос.

## Коды состояний и ошибок планировщика

Наряду со стандартными кодами состояния HTTP API REST планировщика Azure возвращает расширенные коды ошибок и сообщения об ошибках. Расширенные коды не заменяют стандартные коды состояния HTTP, но содержат дополнительные сведения, которые можно использовать вместе со стандартными кодами состояния HTTP.

Например, ошибка HTTP 404 может возникать по различным причинам, поэтому наличие дополнительных сведений в расширенном сообщении может помочь в устранении проблемы. Дополнительные сведения о стандартных кодах HTTP, возвращаемых API REST, см. в статье [Коды состояний и ошибок управления службами](https://msdn.microsoft.com/library/windowsazure/ee460801.aspx). Операции API REST для API управления службами выдают стандартные коды состояния HTTP, описанные в статье [Определения кодов состояния HTTP/1.1](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html). В приведенной ниже таблице описаны распространенные ошибки, которые могут быть выданы службой.

|Код ошибки|HTTP status code (Код состояния HTTP)|Сообщение для пользователя|
|----|----|----|
|MissingOrIncorrectVersionHeader|Ошибочный запрос (400)|Заголовок управления версиями не указан или указан неправильно.|
|InvalidXmlRequest|Ошибочный запрос (400)|XML-код запроса недопустим или указан неправильно.|
|MissingOrInvalidRequiredQueryParameter|Ошибочный запрос (400)|Обязательный параметр запроса для этого запроса не указан или указан неправильно.|
|InvalidHttpVerb|Ошибочный запрос (400)|Указанная команда HTTP не распознана сервером или недопустима для данного ресурса.|
|AuthenticationFailed|Запрещено (403)|Сервер не смог проверить подлинность запроса. Убедитесь, что сертификат допустим и связан с данной подпиской.|
|ResourceNotFound|Не найдено (404)|Указанный ресурс не существует.|
|InternalError|Внутренняя ошибка сервера (500)|Сервер обнаружил внутреннюю ошибку. Повторите запрос.|
|OperationTimedOut|Внутренняя ошибка сервера (500)|Не удалось выполнить операцию за отведенное время.|
|ServerBusy|Служба недоступна (503)|Сервер (или внутренний компонент) в настоящее время недоступен для получения запросов. Повторите запрос.|
|SubscriptionDisabled|Запрещено (403)|Подписка отключена.|
|BadRequest|Ошибочный запрос (400)|Параметр задан неверно.|
|ConflictError|Конфликт (409)|Возник конфликт, не позволяющий завершить операцию.|
|TemporaryRedirect|Временное перенаправление (307)|Запрошенный объект недоступен. Временный URI нового расположения объекта можно получить из поля Location в ответе. Первоначальный запрос можно повторить для нового URI.|

Операции API могут также возвращать дополнительные сведения об ошибке, определенные службой управления. Дополнительные сведения об ошибке содержатся в тексте ответа. Текст ответа об ошибке имеет базовый формат, показанный ниже.

		<?xml version="1.0" encoding="utf-8"?>  
		<Error>  
			<Code>string-code</Code>  
			<Message>detailed-error-message</Message>  
		</Error>

# <a name="azure-government-intelligence--analytics"></a>Аналитика Azure для государственных организаций
В этой статье описаны службы аналитики, важные отличия и рекомендации для среды Azure для государственных организаций.

## <a name="azure-hdinsight"></a>Azure HDInsight
В Azure для государственных организаций используется общедоступная версия HDInsight.

### <a name="variations"></a>Варианты
Приведенные ниже возможности HDInsight сейчас не доступны в Azure для государственных организаций.

* HDInsight не доступен для Windows.
* Azure Data Lake Store в настоящее время не доступна в Azure для государственных организаций. Хранилище BLOB-объектов Azure сейчас является единственным доступным вариантом хранилища.
* Интеграция доменных служб Azure Active Directory пока недоступна. Чтобы создать безопасные кластеры Hadoop без доменных служб Active Directory, выберите один из следующих сценариев установки.

1. Интеграция HDInsight с Active Directory, выполняемой в Azure IaaS

  Это самый простой вариант архитектуры для интеграции HDInsight с Active Directory. Ниже приводится схема этой архитектуры. В этой архитектуре вы создаете контроллер домена Active Directory, выполняемый на одной или нескольких виртуальных машинах в Azure. Обычно эти виртуальные машины находятся в виртуальной сети. Вы можете настроить новую виртуальную сеть, в которой будет размещен кластер HDInsight. Чтобы HDInsight мог обращаться к Active Directory, потребуется настроить пиринговую связь между этими виртуальными сетями, как описано [в этой статье] (https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-create-vnetpeering-arm-portal). 
  
  Предварительные требования к настройке Active Directory
     * Должно быть создано организационное подразделение, в котором будут размещены виртуальные машины кластера HDInsight и используемые в кластере субъекты-службы. 
     * Для LDAPS должно быть настроено взаимодействие с Active Directory. Сертификат, используемый для настройки LDAPS, должен быть реальным (а не самозаверяющим).
     * В домене должны быть созданы обратные зоны DNS для диапазона IP-адресов из подсети HDI.
     * Потребуется учетная запись службы или учетная запись пользователя, с помощью которых будет создан кластер HDInsight. Эта учетная запись должна иметь следующие разрешения
        * Разрешения на создание объектов субъектов-служб и объектов компьютеров в рамках организационного подразделения.
        * Разрешения на создание правил прокси-сервера для обратных зон DNS.
        * Разрешения на присоединение компьютеров к домену Active Directory.
        
2. Интеграция HDInsight с локальной службой Active Directory через VPN-подключение
  
  Эта архитектура очень близка к архитектуре №1. Единственное различие заключается в том, что здесь используется локальная Active Directory, для доступа к которой из HDInsight используется [VPN-подключение из Azure к локальной сети] (https://docs.microsoft.com/en-us/azure/expressroute/expressroute-introduction). Ниже приведена диаграмма архитектуры для этой конфигурации. 
  
  Предварительные требования к настройке локальной Active Directory
     * Должно быть создано организационное подразделение, в котором будут размещены виртуальные машины кластера HDInsight и используемые в кластере субъекты-службы.
     * Для LDAPS должно быть настроено взаимодействие с Active Directory. Сертификат, используемый для настройки LDAPS, должен быть реальным (а не самозаверяющим).
     * В домене должны быть созданы обратные зоны DNS для диапазона IP-адресов из подсети HDI.
     * Потребуется учетная запись службы или учетная запись пользователя, с помощью которых будет создан кластер HDInsight. Эта учетная запись должна иметь следующие разрешения
        * Разрешения на создание объектов субъектов-служб и объектов компьютеров в рамках организационного подразделения.
        * Разрешения на создание правил прокси-сервера для обратных зон DNS.
        * Разрешения на присоединение компьютеров к домену Active Directory.


URL-адреса для Log Analytics отличаются в Azure для государственных организаций:

| Тип службы | Azure Public | Azure Government |
| --- | --- | --- |
| Кластер HDInsight | \*.azurehdinsight.net | \*.azurehdinsight.us |

Дополнительные сведения см. в [общедоступной документации по Azure HDInsight](../hdinsight/hdinsight-hadoop-introduction.md).

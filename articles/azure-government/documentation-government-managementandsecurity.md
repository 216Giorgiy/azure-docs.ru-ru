#  <a name="azure-government-management-and-security"></a>Управление и безопасность Azure для государственных организаций
##  <a name="key-vault"></a>хранилище ключей;
Дополнительные сведения об этой службе и ее использовании см. в <a href="https://azure.microsoft.com/documentation/services/key-vault">общедоступной документации по хранилищу ключей Azure. </a>
### <a name="data-considerations"></a>Рекомендации по работе с данными
Информация ниже определяет границу службы Azure для государственных организаций для хранилища ключей Azure.

| Данные, подлежащие регулированию и контролю, разрешены. | Данные, подлежащие регулированию и контролю, не разрешены. |
|--------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Все данные, зашифрованные с помощью хранилища ключей Azure, могут содержать данные, подлежащие регулированию и контролю. | Метаданные хранилища ключей Azure не должны содержать данные, подлежащие экспортному контролю. К метаданным относятся данные конфигурации, которые вводятся во время создания и обслуживания хранилища ключей.  Не вводите данные, подлежащие контролю или регулированию, в такие поля: "Имена группы ресурсов", "Имена хранилища ключей", "Имя подписки". |

В Azure для государственных организаций используется общедоступная версия хранилища ключей. В этой версии нет расширения, поэтому хранилище ключей доступно только через PowerShell и интерфейс командной строки.
## <a name="log-analytics"></a>Служба Log Analytics
В Azure для государственных организаций используется общедоступная версия Log Analytics. 

### <a name="differences-from-public-azure"></a>Отличия от общедоступной версии Azure

Приведенные ниже возможности Log Analytics в настоящее время недоступны в Azure для государственных организаций. Этот список обновляется в случае изменения состояния функций и решений.

+ Решения, которые находятся на этапе предварительной версии в общедоступной версии Azure, включая следующие:
  - Решение для мониторинга сетей.
  - решение для анализа сетей Azure;
  - Решение для Office 365.
  - Решение для анализа обновлений Windows 10.
  - Мониторинг зависимости приложений.
  - Application Insights
  - журналы действий Azure;
  - анализ службы автоматизации Azure;
  - анализ хранилища ключей.
+ Решения и функции, для которых нужна служба автоматизации Azure, включая следующие:
  - управление обновлениями;
  - управление изменениями;
  - оповещения, активирующие runbook службы автоматизации Azure.
+ Решения и функции, требующие обновления локального программного обеспечения, включая следующие:
  - интеграция с System Center Operations Manager 2016;
  - группы компьютеров из System Center Configuration Manager;
  - решение Surface Hub.
+ Функции, которые находятся на этапе предварительной версии в общедоступной версии Azure, включая следующие:
  - Экспорт данных в PowerBI.
+ Интеграция с порталом Azure.
  - Выбор учетных записей хранения Azure для мониторинга должно осуществляться с помощью PowerShell или шаблонов Resource Manager.
  - Выбор виртуальных машин для включения агента Log Analytics должен осуществляться с помощью PowerShell или шаблонов Resource Manager.
  - Метрики и система диагностики Azure.
+ Мобильные приложения OMS.
+ Расширение виртуальной машины "OMS Linux Agent".
+ Данные об использовании

Поведение приведенных ниже функций Log Analytics отличается в Azure для государственных организаций.

+ Агент Windows для Azure для государственных организаций нужно скачать с [портала Log Analytics](https://oms.microsoft.us).
+ Для передачи данных с помощью API сборщика данных требуется использовать URL-адреса Azure для государственных организаций: https://*ИД_рабочей_области*.ods.opinsights.azure.us, где *ИД_рабочей_области* — это идентификатор рабочей области на портале OMS. 
+ Для подключения вашего сервера управления System Center Operations Manager к Log Analytics необходимо скачать и импортировать обновленные пакеты управления.
  1. Скачайте и сохраните [обновленные пакеты управления](http://go.microsoft.com/fwlink/?LinkId=828749).
  2. Распакуйте скачанный файл.
  3. Импортируйте пакеты управления в Operations Manager. Сведения о том, как импортировать пакет управления с диска, см. в разделе [Импорт пакета управления диспетчера операций](http://technet.microsoft.com/library/hh212691.aspx) на веб-сайте Microsoft TechNet.
  4. Чтобы подключить Operations Manager к Log Analytics, выполните действия, описанные в разделе [Подключение Operations Manager к Log Analytics](../log-analytics/log-analytics-om-agents.md). 



### <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

+ Можно ли перенести данные из Log Analytics (в общедоступной версии Azure) в Azure для государственных организаций?
  - Нет. Невозможно переместить данные или рабочую область из общедоступной версии Azure в Azure для государственных организаций.
+ Можно ли переключаться между рабочими областями в общедоступной версии Azure и Azure для государственных организаций на портале OMS Log Analytics?
  - Нет. Для общедоступной версии Azure и Azure для государственных организаций используются отдельные порталы, не предусматривающие взаимный обмен данными. 

Дополнительные сведения см. в [общедоступной документации по Log Analytics](../log-analytics/log-analytics-overview.md).

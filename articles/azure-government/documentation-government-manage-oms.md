# <a name="azure-government-cybersecurity-monitoring-and-securing-your-assets-with-operations-management-suite-oms"></a>Кибербезопасность правительства в Azure. Мониторинг и защита активов с помощью Operations Management Suite (OMS)

## <a name="cybersecurity-in-the-cloud"></a>Кибербезопасность в облаке
Для наших клиентов, переходящих в облако, крайне важно сохранить средства управления и защиты активов служб Azure для государственных организаций, развернутых в облаке. Необходимо правильно настроить брандмауэры виртуальных машин. Для виртуальных сетей должны быть применены правильные группы безопасности сети. Доступ к ресурсам должен предоставляться только допустимым пользователям в допустимое время. Все это реализуется в необходимых рабочих потоках, которые должны быть продуманы, разработаны и подготовлены для создания безопасной инфраструктуры, которую сможет использовать ваше агентство.

Настройка такой среды может быть нелегкой задачей. Подключение существующего парка серверов к любой службе мониторинга плохо масштабируется и плохо обновляется. Сложно выполнять мониторинг инфраструктуры, размещенной у разных поставщиков, а также в облаке и в локальной среде. Наконец, поддержка мониторинга в актуальном состоянии и сбор данных для отслеживания, обнаружения, предупреждения и защиты от киберугроз требуют затрат времени, ресурсов и вычислительной мощности.

## <a name="microsoft-operations-management-suite-oms"></a>Microsoft Operations Management Suite (OMS)
Microsoft Operations Management Suite, который теперь доступен в Azure для государственных организаций, — это служба, которая позволяет выполнять все эти задачи с применением мощных средств Map Reduce и машинного обучения как услуги. Возможности OMS:
* развертывание агентов на отдельных виртуальных машинах (Linux и Windows), размещенных в Azure, у других поставщиков облачных служб или локально;
* подключение к существующим журналам через учетную запись хранения Azure для государственных организаций или конечную точку SCOM;
* выполнение актуальных служб машинного обучения и Map Reduce с возможностью полномасштабного поиска по журналам для выявления угроз в вашей среде без дополнительной настройки.

Давайте выясним, как интегрировать OMS в ваш парк и какие стандартные решения помогут в решении описанных выше трудностей.

## <a name="onboarding-servers-to-oms"></a>Подключение серверов к OMS
В первую очередь для интеграции облачных ресурсов в Operations Management Suite нужно установить агент OMS на все существующие источники журналов.
Для виртуальных машин это очень просто — достаточно вручную загрузить агент из рабочей области OMS.

## <a name="leveraging-intelligence-through-oms-solution-packs"></a>Использование аналитики с помощью пакетов решений OMS.
Теперь, когда у вас есть множество источников журналов, возникает новая проблема — как использовать все эти данные.
OMS по сути является службой поиска по журналам, которая позволяет создавать мощные запросы для сверхбыстрого поиска информации по нескольким тысячам или даже миллионам журналов. Но выявить проблемы, для которых нужно написать запросы, может быть очень сложно.

Используйте решения OMS — это пакеты запросов, интегрированные с встроенными в OMS технологиями Map Reduce и машинного обучения, позволяющие активно получать ценную информацию о парке компьютеров, переданных под контроль OMS.

Продолжая тему кибербезопасности, кратко обсудим три сценария, для которых OMS в стандартной конфигурации предлагает готовые решения.

###<a name="antimalware-assessment"></a>Оценка защиты от вредоносных программ
Оценка защиты от вредоносных программ предлагает готовый набор запросов, уведомлений и панелей мониторинга, которые позволят вам мгновенно оценить, насколько хорошо ваш парк защищен от вредоносных программ.

Эта панель мониторинга содержит 4 раздела:
* все серверы, для которых существуют активные или исправленные угрозы;
* обнаруженные угрозы на текущий момент;
* компьютеры, которые недостаточно хорошо защищены. Для этой оценки OMS просматривает журналы каждого компьютера и находит сайты с открытым брандмауэром или неправильно настроенными правилами в распространенных веб-браузерах;
* анализ методов защиты ваших серверов. Например, используется ли встроенная защита от вирусов ОС Windows и/или дополнительные средства вроде System Center Endpoint Protection.

### <a name="identity-and-access"></a>Удостоверение и доступ
Еще один распространенный сценарий кибербезопасности в облаке — компрометация учетных данных. Используются не только учетные данные для подписки, но и пользователи или секреты (обычно это сертификат или пароль) для каждой отдельной виртуальной машины.

OMS будет собирать и структурировать все попытки входа в вашем парке, разделяя их на категории в зависимости от типа (удаленные, локальные, использованные имена пользователей и т. д.) В представленном ниже примере мы видим огромное количество неудачных входов с использованием случайных строк в качестве имени пользователя. Это может означать, что компьютеры недостаточно хорошо защищены брандмауэрами и списками управления доступом.

### <a name="threat-intelligence"></a>Аналитика угроз
OMS обеспечивает защиту от вредоносных внутренних действий, то есть от действий пользователя-злоумышленника по захвату данных в случае компрометации периметра защиты вашей организации.

Аналитика угроз OMS изучает сетевые журналы на всех компьютерах, автоматически обнаруживает входящие и исходящие подключения для известных вредоносных IP-адресов (например, IP-адресов неиндексируемого сегмента Интернета) и извещает вас о таких обнаружениях.

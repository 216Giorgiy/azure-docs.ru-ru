# <a name="azure-government-networking"></a>Сетевые подключения для Azure для государственных организаций
## <a name="expressroute-private-connectivity"></a>ExpressRoute (возможность частных подключений)
В Azure для государственных организаций используется общедоступная версия ExpressRoute. Дополнительные сведения (в том числе сведения о партнерах и расположениях пиринга) см. в [общедоступной документации по ExpressRoute](../expressroute/index.md).

### <a name="variations"></a>Варианты
В Azure для государственных организаций используется общедоступная версия ExpressRoute. 

* Клиенты из государственных организаций подключаются к физически изолированным ресурсам через выделенное подключение ExpressRoute Azure для государственных организаций.
* Azure для государственных организаций обеспечивает повышенный уровень доступности и устойчивости, используя несколько пар регионов, находящихся друг от друга на расстоянии не менее 800 километров. 
* По умолчанию все подключения ExpressRoute Azure для государственных организаций настроены как активно-активные избыточные с поддержкой переноса и обеспечивают емкость сети до 10G (наименьшее значение — 50 МБ).
* Расположения ExpressRoute Azure для государственных организаций предоставляют оптимизированные пути обмена данными (кратчайшие прыжки, малую задержку, высокую производительность и т. д.) для клиентов и геоизбыточных регионов Azure для государственных организаций.
* ExpressRoute Azure для государственных организаций не использует частное подключение, не просматривает Интернет и не зависит от него.
* Физическая и логическая инфраструктура Azure для государственных организаций является физически выделенной и отделенной. Доступ к ней могут получить только граждане США.
* Корпорация Майкрософт является владельцем всей оптоволоконной инфраструктуры и работает с ней между регионами Azure для государственных организаций и расположениями Meet-Me ExpressRoute Azure для государственных организаций.
* ExpressRoute Azure для государственных организаций обеспечивает возможность подключения к облачным службам Microsoft Azure, Office 365 и CRM.

### <a name="considerations"></a>Рекомендации
Возможности частного подключения в службе Azure для государственных организаций обеспечивают две базовые службы: VPN (подключения типа "сеть — сеть" для стандартных организаций) и ExpressRoute.

Azure ExpressRoute позволяет создавать частные подключения между центрами обработки данных Azure для государственных организаций и локальной инфраструктурой или средой для совместного размещения данных. Подключения ExpressRoute осуществляются не через общедоступный Интернет и обеспечивают повышенную надежность и быстродействие, а также более низкую задержку по сравнению с обычными интернет-подключениями. В некоторых случаях использование подключений ExpressRoute для передачи данных между локальными системами и Azure позволяет существенно сэкономить средства.   

С помощью ExpressRoute вы можете установить подключения к Azure в расположении ExpressRoute (например, объект поставщика Exchange) или напрямую подключиться к Azure из имеющейся сети WAN (например, из сети MPLS VPN, предоставленной поставщиком сетевой службы).

![замещающий текст](./media/azure-government-capability-private-connectivity-options.PNG)  ![замещающий текст](./media/government-capability-expressroute.PNG)  

Чтобы сетевые службы поддерживали приложения и решения пользователей Azure для государственных организаций, мы настоятельно рекомендуем реализовать ExpressRoute (возможность частных подключений), чтобы подключиться к этой службе Azure. Если используются VPN-подключения, следует учитывать следующую информацию.

* Пользователям следует связаться с уполномоченным представителем или агентством, чтобы узнать, требуется ли возможность частных подключений или другой механизм безопасного подключения, а также понять, какие дополнительные ограничения следует учитывать.
* Пользователям нужно решить, следует ли направлять VPN-подключение типа "сеть — сеть" через зону частного подключения.
* Пользователям следует получить сеть MPLS или VPN у лицензированного поставщика доступа к частным подключениям.

Пользователи, использующие архитектуру частного подключения, должны проверить, установлена ли и обслуживается ли нужная реализация для подключения пользователей к разделительной точке пограничного маршрутизатора сети шлюза/Интернета (GN/I) для службы Azure для государственных организаций. Аналогично в организации нужно установить сетевое подключение между локальной средой и разделительной точкой пограничного маршрутизатора сети шлюза/пользователя (GN/C) для службы Azure для государственных организаций.

## <a name="support-for-bgp-communities"></a>Поддержка сообществ BGP
Этот раздел содержит общие сведения об использовании сообществ BGP с ExpressRoute в регионе AzureGov. Майкрософт объявляет маршруты в путях общедоступного пиринга и пиринга Microsoft с маршрутами, помеченными значениями соответствующего сообщества. Для чего это нужно и что такое значения сообществ, рассмотрим ниже. При этом Майкрософт не учитывает значения сообществ, которыми помечены маршруты, объявленные для Майкрософт.

Подключившись к корпорации Майкрософт через ExpressRoute в любом расположении пиринга в регионе AzureGov, вы получите доступ ко всем облачным службам Майкрософт во всех регионах в границах для государственных организаций. 

Например, подключившись через ExpressRoute к корпорации Майкрософт в Вашингтоне (округ Колумбия), вы получите доступ ко всем облачным службам Майкрософт, размещенным в регионе AzureGov.

На странице [общедоступной документации по ExpressRoute](../expressroute/index.md) откройте вкладку "Обзор". Она содержит сведения о расположениях и партнерах, а также подробный список расположений пиринга ExpressRoute для региона AzureGov.

Вы можете приобрести больше одного канала ExpressRoute. За счет геоизбыточности наличие нескольких подключений даст вам более высокий уровень доступности. Имея несколько каналов ExpressRoute, вы будете получать тот же набор префиксов, который объявляется из Майкрософт для путей общедоступного пиринга Azure и пиринга Майкрософт. Это означает, что из вашей сети в Майкрософт будет вести сразу несколько путей. В такой ситуации есть вероятность, что ваша сеть будет выбирать маршрутизацию не самым оптимальным образом, а значит, подключение к некоторым службам может оказаться недостаточно надежным. 

Майкрософт помечает префиксы, объявляемые через общедоступный пиринг и пиринг Майкрософт, значениями соответствующих сообществ BGP, которые указывают, в каком регионе размещаются эти префиксы. Значения сообществ помогут вам правильно сделать выбор и обеспечить своим клиентам оптимальную маршрутизацию.  Дополнительные сведения см. на странице [общедоступной документации по ExpressRoute](../expressroute/index.md) на вкладке "Начало работы" в разделе "Оптимизация маршрутизации".

#  Сетевые подключения для Azure для государственных организаций

##  ExpressRoute (возможность частных подключений)

Возможности частного подключения в службе Azure для государственных организаций обеспечивают две базовые службы: VPN (подключения типа "сеть — сеть" для стандартных организаций) и ExpressRoute.

Azure ExpressRoute позволяет создавать частные подключения между центрами обработки данных Azure для государственных организаций и локальной инфраструктурой или средой для совместного размещения данных. Подключения ExpressRoute осуществляются не через общедоступный Интернет и обеспечивают повышенную надежность и быстродействие, а также более низкую задержку по сравнению с обычными интернет-подключениями. В некоторых случаях использование подключений ExpressRoute для передачи данных между локальными системами и Azure позволяет существенно сэкономить средства.

С помощью ExpressRoute вы можете установить подключения к Azure в расположении ExpressRoute (например, объект поставщика Exchange) или напрямую подключиться к Azure из имеющейся сети WAN (например, из сети MPLS VPN, предоставленной поставщиком сетевой службы).

![замещающий текст](./media/azure-government-capability-private-connectivity-options.PNG) ![замещающий текст](./media/government-capability-expressroute.PNG)

Чтобы сетевые службы поддерживали приложения и решения пользователей Azure для государственных организаций, мы настоятельно рекомендуем реализовать ExpressRoute (возможность частных подключений), чтобы подключиться к этой службе Azure. Если используются VPN-подключения, следует учитывать следующую информацию.

• Пользователям следует связаться с уполномоченным представителем или агентством, чтобы узнать, требуется ли возможность частных подключений или другой механизм безопасного подключения, а также понять, какие дополнительные ограничения следует учитывать.

• Пользователям нужно решить, следует ли направлять VPN-подключение типа "сеть — сеть" через зону частного подключения.

• Пользователям следует получить сеть MPLS или VPN у лицензированного поставщика доступа к частным подключениям.

Пользователи, использующие архитектуру частного подключения, должны проверить, установлена ли и обслуживается ли нужная реализация для подключения пользователей к разделительной точке пограничного маршрутизатора сети шлюза/Интернета (GN/I) для службы Azure для государственных организаций. Аналогично в организации нужно установить сетевое подключение между локальной средой и разделительной точкой пограничного маршрутизатора сети шлюза/пользователя (GN/C) для службы Azure для государственных организаций.

В Azure для государственных организаций используется общедоступная версия ExpressRoute. Дополнительные сведения (в том числе сведения о партнерах и расположениях пиринга) см. в <a href="https://azure.microsoft.com/ru-RU/documentation/services/expressroute/">общедоступной документации по ExpressRoute </a>.

Чтобы получать дополнительные сведения и обновления, подпишитесь на <a href="https://blogs.msdn.microsoft.com/azuregov/">блог Microsoft Azure для государственных организаций. </a>

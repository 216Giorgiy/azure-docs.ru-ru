# <a name="azure-government-databases"></a>Базы данных Azure для государственных организаций
## <a name="sql-database"></a>База данных SQL
Дополнительные инструкции по настройке видимости метаданных и рекомендации по защите см. в статье <a href="https://msdn.microsoft.com/en-us/library/bb510589.aspx">Центр обеспечения безопасности для базы данных Azure SQL и SQL Server Database Engine</a> и в [общедоступной документации по базе данных SQL Azure](../sql-database/index.md).

### <a name="variations"></a>Варианты
В Azure для государственных организаций используется общедоступная версия базы данных SQL 12.

Адрес для серверов Azure SQL Server в Azure для государственных организаций отличается:

| Тип службы | Azure Public | Azure Government |
| --- | --- | --- |
| База данных SQL |*.database.windows.net |*.database.usgovcloudapi.net |

### <a name="considerations"></a>Рекомендации
Информация ниже определяет границу службы Azure для государственных организаций для Azure SQL:

| Данные, подлежащие регулированию и контролю, разрешены. | Данные, подлежащие регулированию и контролю, не разрешены. |
| --- | --- |
| База данных SQL Microsoft Azure может содержать подлежащие регулированию данные Azure для государственных организаций. Для передачи подлежащих регулированию данных Azure для государственных организаций нужно использовать средства управления базами данных. |Метаданные SQL Azure не должны содержать данные, подлежащие экспортному контролю. К метаданным относятся данные конфигурации, которые вводятся во время создания и обслуживания продукта службы хранилища.  Не вводите данные, подлежащие контролю или регулированию, в такие поля: "Имя базы данных", "Имя подписки", "Группы ресурсов", "Имя сервера", "Имя для входа администратора сервера", "Имена развертываний", "Имена ресурса", "Теги ресурса". |

## <a name="azure-redis-cache"></a>кэш Azure Redis
Дополнительные сведения об этой службе и ее использовании см. в [общедоступной документации по кэшу Redis для Azure](../redis-cache/index.md).

### <a name="variations"></a>Варианты
URL-адреса для доступа к кэшу Redis для Azure и управления им в Azure для государственных организаций отличаются:

| Тип службы | Azure Public | Azure Government |
| --- | --- | --- |
| Конечная точка кэша |*.redis.cache.windows.net |*.redis.cache.usgovcloudapi.net |
| Портал Azure |https://portal.azure.com |https://portal.azure.us |

> [!NOTE]
> Во всех сценариях и кодах должны учитываться соответствующие конечные точки и среды. Дополнительные сведения см. в разделе [Подключение к облаку Azure для государственных организаций](../redis-cache/cache-howto-manage-redis-cache-powershell.md#how-to-connect-to-azure-government-cloud-or-azure-china-cloud).
> 
> 

### <a name="considerations"></a>Рекомендации
Информация ниже определяет границу службы Azure для государственных организаций для кэша Redis для Azure:

| Данные, подлежащие регулированию и контролю, разрешены. | Данные, подлежащие регулированию и контролю, не разрешены. |
| --- | --- |
| Кэш Redis для Azure может содержать подлежащие регулированию данные Azure для государственных организаций. |Метаданные кэша Redis для Azure не должны содержать данные, подлежащие экспортному контролю. Не вводите контролируемые данные в следующие поля: "Имя в кэше", "Имя виртуальной сети", "Имя подписки", "Группы ресурсов", Resource tags (Теги ресурсов), Redis properties (Свойства Redis). |

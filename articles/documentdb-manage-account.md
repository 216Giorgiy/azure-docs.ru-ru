## <a id="keys"></a> Практическое руководство. Просмотр, копирование и повторное создание ключей доступа

При создании учетной записи DocumentDB генерируются два главных ключа доступа,
которые могут использоваться для проверки подлинности при доступе к
учетной записи DocumentDB. Благодаря наличию двух ключей доступа в DocumentDB
можно повторно создавать ключи без прерывания доступа к учетной записи
DocumentDB.

На [портале управления предварительной версией Azure](http://manage.windowsazure.com) есть раздел **Keys** (Ключи) в колонке учетной записи DocumentDB, в котором можно просматривать, копировать и повторно создавать ключи доступа к учетной записи DocumentDB.


![](./media/documentdb-manage-account/image002.jpg)




### Просмотр и копирование ключа доступа

1. На [портале управления предварительной версией Azure](https://portal.azure.com/) откройте учетную запись DocumentDB. 

2. В области Summary (Сводка) щелкните **Keys** (Ключи).

3. В колонке Keys нажмите кнопку **Copy** (Копировать) справа от ключа, который нужно скопировать.

4. Чтобы скопировать ключ, нажмите клавиши CTRL+C.


  ![](./media/documentdb-manage-account/image004.jpg)


### Повторное создание ключей доступа

Необходимо периодически изменять ключи доступа к учетной записи DocumentDB,
чтобы подключения были более безопасными. Так как предусмотрены два ключа доступа,
можно обслуживать подключения к учетной записи DocumentDB
с помощью одного ключа во время повторного создания другого.

**Предупреждение**

Повторное создание ключей доступа влияет на все приложения, которые
зависят от текущего ключа. Все клиенты, использующие ключ доступа
к учетной записи DocumentDB, должны быть обновлены для использования нового ключа.

Если у вас имеются приложения или облачные службы, которые используют учетную запись DocumentDB,
после повторного создания ключей подключения будут утрачены, если только ключи
не будут переключены. Вы можете придерживаться описанной ниже процедуры.

1. Обновите ключ доступа в коде приложения так, чтобы он ссылался
на вторичный ключ доступа к учетной записи DocumentDB.

2. Повторно создайте первичный ключ доступа для учетной записи DocumentDB.
На [портале управления предварительной версией Azure](https://portal.azure.com/)
откройте учетную запись DocumentDB.

3. В области Summary (Сводка) щелкните **Keys** (Ключи).

4. В колонке Keys выберите команду **Regenerate Primary** (Повторно создать первичный), а затем нажмите кнопку **ОК**, чтобы подтвердить создание нового ключа.

5. Убедившись в том, что новый ключ доступен для использования
(на это может потребоваться приблизительно 5 минут с момента повторного создания), обновите ключ доступа в
коде приложения так, чтобы он ссылался на новый первичный ключ.

6. Повторно создайте вторичный ключ доступа.

*Обратите внимание: новый ключ можно будет использовать для доступа
к учетной записи DocumentDB через несколько минут после создания.*

## <a id="consistency"></a> Практическое руководство. Управление параметрами согласованности DocumentDB

Служба DocumentDB поддерживает четыре четко определенных, настраиваемых
пользователем уровня согласованности данных. Это позволяет разработчикам приложений достигать
предсказуемого компромисса между согласованностью, доступностью и задержкой.

· **Строгая** согласованность (Strong) гарантирует, что операции чтения всегда
возвращают последнее записанное значение.

· Согласованность с **ограниченным запаздыванием** (Bounded) гарантирует, что операции чтения возвращают не слишком устаревшие данные. В частности, при этом операции чтения возвращают данные,
которые не более чем на K версий старше последней записанной версии.

· Согласованность **на уровне сеанса** (Session) обеспечивает монотонное чтение (т. е. вы никогда не будете
считывать старые данные, потом новые, а потом снова старые), монотонную запись (операции записи
упорядочены) и чтение самых последних записанных данных в каждой отдельной
точке представления клиента.

· Согласованность **в конечном счете** (Eventual) означает, что операции чтения
всегда возвращают действительное подмножество записанных данных, которые в конечном счете сходятся.

*Обратите внимание: по умолчанию учетные записи DocumentDB предоставляются с согласованностью
на уровне сеанса. Дополнительные сведения о параметрах согласованности
DocumentDB см. в разделе [Уровень согласованности](http://go.microsoft.com/fwlink/p/?LinkId=402365).*

### Установка согласованности по умолчанию для учетной записи DocumentDB

1. На [портале управления предварительной версией Azure](https://portal.azure.com/) откройте учетную запись DocumentDB. 

2. В области Configuration (Конфигурация) щелкните **Consistency** (Согласованность).

3. В колонке Default Consistency (Согласованность по умолчанию) выберите уровень
согласованности по умолчанию, необходимый для учетной записи DocumentDB.

4. Нажмите кнопку **Save** (Сохранить).

5. Ход выполнения операции можно отслеживать в концентраторе уведомлений
на портале управления предварительной версией Azure.

 ![](./media/documentdb-manage-account/image005.png)

 ![](./media/documentdb-manage-account/image006.png)

*Обратите внимание: на вступление в силу уровня согласованности по умолчанию
для учетной записи DocumentDB может потребоваться несколько минут.*

## <a id="capacity"></a> Практическое руководство. Управление параметрами емкости DocumentDB

Служба Microsoft Azure DocumentDB обеспечивает гибкое масштабирование по мере
изменения потребностей приложения на протяжении его жизненного цикла. Масштабирование службы
DocumentDB осуществляется путем увеличения емкости учетной записи базы данных
DocumentDB на портале управления предварительной версией Azure.

При создании учетной записи базы данных для нее выделяется определенное
пространство для хранения и резервируется пропускная способность. Вы в любой момент можете изменить
предоставленное пространство для хранения и пропускную способность учетной записи, увеличив
или уменьшив число единиц емкости на портале управления предварительной версией Azure.


### Увеличение или уменьшение числа единиц емкости

1. На [портале управления предварительной версией Azure](https://portal.azure.com/) откройте учетную запись DocumentDB. 

2. В области Usage (Использование) щелкните **Scale** (Масштаб).

3. В колонке Scale укажите число единиц емкости, необходимое
для учетной записи DocumentDB.

4. Нажмите кнопку **Save** (Сохранить). (Имейте в виду, что на завершение масштабирования
может потребоваться несколько минут. За ходом выполнения операции можно наблюдать в
концентраторе уведомлений на портале управления предварительной версией Azure.)

*Обратите внимание: предварительная версия DocumentDB поддерживает не более 5 единиц
емкости на каждую учетную запись DocumentDB.*

![](./media/documentdb-manage-account/image007.png)


## <a id="delete"></a> Практическое руководство. Удаление учетной записи DocumentDB

Чтобы удалить учетную запись DocumentDB, которая больше не нужна, воспользуйтесь командой
**Delete** (Удалить) в колонке DocumentDB Account (Учетная запись DocumentDB).

**Предупреждение**

*В предварительном выпуске восстановить содержимое удаленной
учетной записи DocumentDB невозможно. При удалении учетной записи DocumentDB удаляются
все ее ресурсы, включая базы данных, коллекции,
документы и вложения.*

![](./media/documentdb-manage-account/image009.png)

1. На [портале управления предварительной версией Azure](https://portal.azure.com/) откройте учетную запись DocumentDB,
которую нужно удалить. 

2. В колонке учетной записи выберите команду **Delete**.

3. В появившейся колонке подтверждения введите имя учетной записи
DocumentDB, чтобы подтвердить ее удаление.

4. Нажмите кнопку **Delete** в колонке подтверждения.

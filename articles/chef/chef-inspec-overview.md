# <a name="use-inspec-for-compliance-automation-of-your-azure-infrastructure"></a>Использование InSpec для автоматизации обеспечения соответствия инфраструктуры Azure
[InSpec](https://www.chef.io/inspec/) — это бесплатная платформа с открытым исходным кодом, предназначенная для тестирования и аудита приложений и инфраструктуры. InSpec работает путем сравнения фактического состояния системы с нужным состоянием, что выражается в удобочитаемом и простом для написания коде InSpec. InSpec обнаруживает нарушения и отображает результаты в форме отчета, но дает вам управлять исправлением. InSpec можно использовать для проверки состояния виртуальных машин, запущенных в Azure. С помощью InSpec также можно отслеживать и проверять состояние ресурсов и групп ресурсов в подписке.

В этой статье описаны преимущества использования InSpec, позволяющие упростить обеспечение безопасности и соответствия требованиям в Azure.

## <a name="make-compliance-easy-to-understand-and-assess"></a>Облегчение понимания и оценки соответствия
С помощью InSpec вы преобразуете свои требования в исполняемый и удобочитаемый код с контролем версий. Это позволяет упорядочить свои тесты в составные профили, где вы определяете и настраиваете исключения по мере необходимости.

## <a name="detect-fleet-wide-issues-and-prioritize-their-remediation"></a>Обнаружение широкомасштабных проблем и приоритезация их исправления
Режим обнаружения без агента InSpec позволяет быстро оценить уровень уязвимости (в масштабе). Благодаря встроенным метаданным для оценки влияния и серьезности можно определить, на каких областях необходимо сосредоточиться для исправления.

## <a name="inspect-machines-data-and-new-saas-apis"></a>Проверка компьютеров, данных и новых API SaaS
Возможности обеспечения соответствия облачного API InSpec позволяют делать как приблизительные, так и высокоточные утверждения о соответствии облака и непрерывно сообщать о них.

## <a name="satisfy-audits"></a>Реагирование на проверки
С помощью InSpec вы можете отвечать на вопросы аудита в любое время, а не только через заданные интервалы, например, ежеквартально или ежегодно. InSpec позволяет ввести цикл аудита, зная точную степень соответствия. Вы больше не будете удивляться результатам аудитора.

## <a name="reduce-ambiguity-and-miscommunication-regarding-rules"></a>Уменьшение неоднозначности и недоразумений в отношении правил
Документы оставляют конфигурации и процессы открытыми для интерпретации. Исполняемый код удаляет обсуждения о том, что следует оценивать, в пользу реальных тестов с явным намерением.

## <a name="keep-up-with-rapidly-changing-threat-and-compliance-landscapes"></a>Оставайтесь в курсе быстро изменяющихся ландшафтов угроз и соответствия
InSpec позволяет писать и публиковать код обнаружения в тот же день, а также писать новые правила, чтобы быстро реагировать на новые нормы. Это означает, что изменения в угрозах или правилах больше не являются аварийными ситуациями.

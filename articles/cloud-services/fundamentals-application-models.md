# Варианты размещения вычислительных сред, предоставляемые Azure

Azure предоставляет разные модели размещения для запуска приложений. Каждая из них содержит различный набор служб, и ваш выбор зависит от ваших целей. В этой статье рассматриваются доступные возможности и приводятся примеры их использования.

| Варианты вычислений | Аудитория |
| ------------------ | --------   |
| [Служба приложений] | Масштабируемые веб-приложения, мобильные приложения, приложения API и приложения логики для любого устройства |
| [Облачные службы] | Высокодоступные масштабируемые облачные приложения уровня n с большей степенью управления операционной системой |
| [Виртуальные машины] | Настраиваемые виртуальные машины Windows и Linux с полным управлением операционной системой |

## Другие варианты

Azure также предлагает другие модели размещения среды выполнения приложений для узконаправленных целей:

* [Мобильные службы](/services/mobile-services/). Оптимизированы для предоставления серверной части облака для приложений, выполняющихся на мобильных устройствах.
* [Пакетная служба](/services/batch/). Оптимизирована для обработки больших объемов схожих задач, предпочтительно рабочих нагрузок, которые подходят для выполнения в качестве параллельных задач на нескольких компьютерах.
* [HDInsight (Hadoop)](/services/hdinsight/) Оптимизирована для выполнения заданий [MapReduce](http://www.asp.net/aspnet/overview/developing-apps-with-windows-azure/building-real-world-cloud-apps-with-windows-azure/data-storage-options/#hadoop) в кластерах Hadoop. 

## Что следует использовать? Принятие решения

Все три модели размещения среды выполнения приложений Azure общего назначения позволяют вам создавать в облаке масштабируемые и надежные приложения. Какой из них вам следует отдать предпочтение, учитывая указанное сходство?

Самым оптимальным вариантом для большинства веб-приложений является служба приложений. Процессы развертывания и управления являются уже интегрированными в платформу, при этом имеется возможность оперативного масштабирования сайтов для удовлетворения высоких нагрузок по трафику, а встроенные инструменты для балансировки нагрузки и управления трафиком обеспечивают высокую доступность. Вы можете легко перенести уже существующие сайты в службу приложений с помощью [средства миграции в режиме онлайн](https://www.migratetoazure.net/), использовать приложение с открытым кодом из коллекции веб-приложений или создать новый сайт с помощью выбранной платформы и инструментов. [Веб-задания](http://go.microsoft.com/fwlink/?linkid=390226) облегчают добавление обработки фонового задания в приложение или даже выполнение вычислительной нагрузки, не являющейся веб-приложением.

Если требуется более широкий контроль над средой веб-сервера, например возможность удаленного входа на сервер или настройка заданий, запускаемых при запуске сервера, наиболее подходящими для этого будут облачные службы Azure.

Если какое-либо существующее приложение требует внесения значительных изменений, чтобы запустить его с помощью службы Веб-сайтов Azure или Облачных служб Azure, для упрощения процесса миграции в облачную среду можно использовать виртуальные машины Azure. Однако для надлежащей настройки, обеспечения безопасности и обслуживания виртуальных машин требуется гораздо больше рабочего времени, а также более высокий уровень подготовки специалистов в сфере ИТ-технологий по сравнению с использованием службы Веб-сайтов Azure и Облачных служб. При рассмотрении варианта использования виртуальных машин Azure следует оценить объем затрат на оказание надлежащего технического обслуживания при эксплуатации, требующегося для внесения исправлений, обновлений, а также для управления средой виртуальных машин.

Иногда ни один отдельный вариант не подходит. В таких ситуациях можно совершенно спокойно совмещать варианты моделей. Например, предположим, что вы создаете приложение, в котором хотите воспользоваться преимуществами управления веб-ролей облачных служб, а также из соображений совместимости или производительности использовать стандартный сервер SQL, размещенный в виртуальной машине.

# Таблица выпусков гостевых ОС Azure и совместимости пакетов SDK
В данном разделе предоставляются новейшие сведения о последних выпусках гостевых ОС Azure для облачных служб. Эта информация поможет вам спланировать обновление до того, как гостевая ОС будет отключена.

> [AZURE.IMPORTANT]Предоставленные на этой странице сведения касаются веб-ролей и рабочих ролей для облачных служб, выполняющихся на последней версии гостевой ОС. Они не затрагивают виртуальные машины IaaS. Если роли настроены на автоматическое обновление гостевой ОС, как описано в разделе [Параметры обновления гостевой ОС Azure][], ознакомление с этой страницей необязательно.

## Сведения о выпуске гостевой ОС

В этом разделе перечислены поддерживаемые версии гостевой ОС. Семейства и версии гостевой ОС имеют дату выпуска, дату отключения и дату окончания срока действия. На дату выпуска можно вручную выбрать версию гостевой ОС на классическом портале Azure. Гостевая ОС удаляется с классического портала Azure в дату отключения или позже. Затем она будет являться "переходной", но поддерживаться с ограниченной возможностью обновления развертывания. Дата окончания срока действия — это дата, на которую запланировано полное удаление версии или семейства версий из системы Azure. Облачные службы, по прежнему работающие на данной версии после истечения срока ее действия, будут остановлены, удалены или принудительно обновлены до более новой версии, как описано в разделе [Политика поддержки и прекращения использования гостевой ОС Azure][retirepolicy].

Майкрософт поддерживает по крайней мере две последние версии каждого поддерживаемого семейства версий гостевой ОС. Дата отключения существующей версии гостевой ОС может быть перенесена на более поздний срок, чтобы обеспечить доступность по крайней мере двух версий для развертывания.

> [AZURE.WARNING]Вывод из использования семейства версий 1 гостевой ОС начался 1 июня 2013 г. и планируется к завершению в ближайшее время. Не создавайте новых установок и не обновляйте старые с использованием этого семейства версий гостевой ОС. Дополнительные сведения см. в разделе [Сведения о выводе семейства версий 1 гостевой ОС Azure из использования][fam1retire]


### Пояснение терминов "семейство версий гостевой ОС", "версия" и "выпуск"
Каждое семейство версий гостевой ОС основано на выпущенной версии Microsoft Windows Server. Гостевая ОС — это базовая операционная система, в которой работают облачные службы Azure. Каждая гостевая ОС имеет семейство, версию и номер выпуска.

**Семейство версий гостевой ОС** соответствует выпуску операционной системы Windows Server, лежащей в основе гостевой ОС. Например, семейство 3 основано на Windows Server 2012.

**"Версия гостевой ОС"** — это образ ОС семейства версий, включая соответствующие исправления [Microsoft Security Response Center (MSRC)][msrc], доступные на дату создания новой версии гостевой ОС. В версию могут быть включены не все исправления. Нумерация начинается с 0 и увеличивается на 1 при каждом добавлении нового набора обновлений. Конечные нули отображаются, только если они значимы. То есть версия 2.10 — это другая версия, гораздо новее, чем 2.1.

**"Выпуск гостевой ОС"** — это перевыпуск определенной версии гостевой ОС. Перевыпуск происходит, если корпорация Майкрософт во время тестирования обнаруживает проблемы, требующие внесения изменений. Последний выпуск всегда заменяет все предыдущие выпуски, вне зависимости от того, являлись ли они общедоступными или нет. На классическом портале Azure у пользователей будет возможность выбрать только последний выпуск для данной версии. Развертывания, выполняемые на предыдущем выпуске, обычно (в зависимости от серьезности ошибки) не обновляются принудительно.

В приведенном ниже примере 2 означает семейство версий, 12 — версию и rel2 — выпуск.

**Выпуск гостевой ОС** — 2.12 rel2

**Строка конфигурации для этого выпуска** — WA-GUEST-OS-2.12\_201208-02

Строка конфигурации гостевой ОС содержит, помимо указанной выше информации, дату, показывающую, какие обновления MSRC учитывались в этом выпуске. В данном примере учитывались для включения в выпуск обновления MSRC, созданные для Windows Server 2008 R2 до августа 2012 г. включительно. Будут включены только исправления, относящиеся к этой версии Windows Server. Например, если исправление MSRC применяется к Microsoft Office, оно не будет включено, так как этот продукт не является частью базового образа Windows Server.

## Процесс системного обновления гостевой ОС
Эта страница содержит сведения о предстоящих выпусках гостевой ОС. Согласно мнениям клиентов, им необходимо знать, когда произойдет выпуск, так как роли облачных служб будут перезапущены, если они настроены на автоматическое обновление. Выпуски гостевой ОС обычно происходят спустя по крайней мере 5 дней после выпуска обновления MSRC, который проходит во второй вторник каждого месяца. Новые выпуски включают все соответствующие исправления MSRC для каждого семейства версий гостевой ОС.

Обновления Microsoft Azure выпускаются на постоянной основе. Гостевая ОС — только одно из таких обновлений в конвейере. На выпуск может повлиять целый ряд факторов, перечислять которые здесь не представляется целесообразным из-за их количества. Кроме того, Azure работает буквально на сотнях тысяч компьютеров. Это означает, что невозможно определить точную дату и время перезапуска ваших ролей. Майкрософт будет размещать на [RSS-канале обновления гостевой ОС][rss] последние сведения, которыми мы располагаем, но их следует воспринимать только в качестве приблизительных временных рамок. Мы осознаем, что это вызывает неудобства для клиентов, и работаем над планом по ограничению перезапусков или согласованию их по времени.

После публикации нового выпуска гостевой ОС может потребоваться время на его распространение в Azure. Когда для новой гостевой ОС обновляются службы, они перезапускаются соответствующими доменами обновления. Службы, настроенные на автоматическое обновление, получают выпуск в первую очередь. После обновления на классическом портале Azure отобразится новая версия гостевой ОС для вашей службы. В течение этого периода могут происходить перевыпуски. Некоторые версии могут разворачиваться в течение более длительных периодов времени, и автоматических перезапусков для обновления может не произойти в течение нескольких недель после официальной даты выпуска. После того как гостевая ОС становится доступной, на портале или в файле конфигурации можно выбрать ее версию.

Ценные сведения о перезапусках и ссылки на дополнительную техническую информацию по обновлениям гостевой и базовой ОС см. в разделе блога MSDN [Перезапуски экземпляров ролей из-за обновлений ОС][restarts].

При ручном обновлении гостевой ОС ознакомьтесь с [политикой вывода из эксплуатации гостевой ОС][retirepolicy].

## Архив новостей

###### **11 ноября 2014 г.**

Ноябрьский выпуск (4.14 3.21 и 2.33) состоялся 11 ноября. Это обновление было выпущено с опережением, так как оно включает обновление MSRC [Бюллетень по безопасности (Майкрософт) MS14-066 — критический][MS14-066]. Автоматически обновляемые веб-роли и рабочие роли должны перезапуститься в течение нескольких дней и получить это исправление.

###### **10 ноября 2014 г.**
Дата отключения октябрьского выпуска (4.13 3.20 и 2.32) была изменена согласно пожеланиям клиентов. Дата отключения всегда будет назначаться по крайней мере через два месяца от даты выпуска.

###### **4 ноября 2014 г.**
Октябрьский выпуск (4.13 3.20 и 2.32) состоялся 4 ноября 2014 г. Он включает исправление MSRC, приведшее к возникновению проблем с августовским и сентябрьским выпусками. Чтобы обойти эту проблему, октябрьский выпуск включает предварительно устанавливаемые, но отключенные платформы .NET 3.5 и 3.5.1. Сценарии, в которых происходит попытка установить .NET 3.5 или 3.5.1, будут фактически повторно включать их и вернут статус "успешно" для установки .NET, в то же время избегая проблемы полной установки, вызываемой исправлением MSRC.

**20 октября 2014 г. Обновлено 4 ноября 2014** — сентябрьский выпуск развертывался частично (4.12, 3.19, 2.31 и 1.39) из-за того же [исправления MSRC MS14-046][MS14-046], вызывающего сбой при попытке установить .NET 3.5 или 3.5.1 на семейство 3 или 4. Платформы .NET 3.5.x официально НЕ поддерживаются ни в одном из семейств, но Майкрософт реагирует на изменения в поведении, поскольку были затронуты установки отдельных клиентов, а эти изменения не были запланированными. Даты отключения предыдущих версий гостевой ОС (июньский и июльский выпуски) в связи с этим будут отложены, чтобы обеспечить поддержку и доступность по крайней мере двух полностью развертываемых версий гостевой ОС. Решение проблемы установки .NET появилось в октябрьском выпуске 2014 года.

Октябрьский выпуск включает предустановленные (но отключенные) версии .NET 3.5 и 3.5.1, а также исправления MSRC, указанные выше. Сценарии, в которых происходит попытка установить .NET 3.5 или 3.5.1, будут фактически повторно включать их и вернут статус "успешно" для установки .NET, в то же время избегая проблемы установки, вызываемой исправлением MSRC.

Из-за частичного развертывания двух последних выпусков те пользователи, у которых настроено автоматическое обновление, и те, кто выполнил развертывание новых установок, могут оказаться использующими один из этих выпусков гостевой ОС. В следующей таблице перечислены выпуски гостевой ОС, допускающих установку .NET 3.5 или 3.5.1 на семействах 3 и 4. В настоящее время, если выпуск допускает установку, это означает, что исправление MSRC MS14-046 НЕ установлено.

**20 октября 2014** — Поскольку августовский и сентябрьский выпуски развертывались лишь частично, следует иметь в виду следующее.

1. Изменения шифрования, описанные в разделе "Различия между гостевой ОС Azure и Windows Server по умолчанию", затронули не всю платформу Azure. Клиенты, не получившие августовский и сентябрьский выпуски, получат эти изменения в октябрьском выпуске. 

2. Августовский и сентябрьский выпуски гостевой ОС были отключены на классическом портале Azure. Выбрать их вручную невозможно. Это позволяет избежать проблем, которые могут возникнуть при выборе любой из этих версий гостевой ОС.

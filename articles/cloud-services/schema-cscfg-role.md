# <a name="azure-cloud-services-config-role-schema"></a>Схема Role конфигурации облачных служб Azure

Элемент `Role` файла конфигурации определяет число экземпляров роли, которые развертываются для каждой роли в службе, значения всех параметров конфигурации и отпечатки всех сертификатов, связанных с ролью.

Дополнительные сведения о схеме конфигурации служб Azure см. в статье [Схема конфигурации облачных служб (классических)](schema-cscfg-file.md). Дополнительные сведения о схеме определения служб Azure см. в статье [Схема определения облачных служб (классических)](schema-csdef-file.md).

##  <a name="Role"></a> Элемент Role
В следующем примере показан элемент `Role` и его дочерние элементы.

```xml 
<ServiceConfiguration>
  <Role name="<role-name>" vmName="<vm-name>">
    <Instances count="<number-of-instances>"/>
    <ConfigurationSettings>
      <Setting name="<setting-name>" value="<setting-value>" />
    </ConfigurationSettings>
    <Certificates>
      <Certificate name="<certificate-name>" thumbprint="<certificate-thumbprint>" thumbprintAlgorithm="<algorithm>"/>
    </Certificates>
  </Role>
</ServiceConfiguration>
```

В следующей таблице описаны атрибуты элемента `Role`.

| Атрибут | ОПИСАНИЕ |
| --------- | ----------- |
| name   | Обязательный элемент. Указывает имя роли. Имя должно соответствовать имени, указанному для роли в файле определения службы.|
| vmName | Необязательный элемент. Указывает DNS-имя виртуальной машины. Длина имени не должна превышать 10 символов.|

В таблице ниже описаны дочерние элементы элемента `Role`.

| Элемент | ОПИСАНИЕ |
| ------- | ----------- |
| Экземпляры | Обязательный элемент. Указывает число экземпляров, развертываемых для роли. Число экземпляров определяется целочисленным значением атрибута `count`.|
| Параметр   | Необязательный элемент. Указывает имя параметра и значение в коллекции параметров для роли. Имя параметра определяется строковым значением атрибута `name`, а значение параметра — строковым значением атрибута `value`.|
| Сертификат | Необязательный элемент. Определяет имя, отпечаток и алгоритм сертификата службы, который будет связан с ролью. Имя сертификата определяется строковым значением атрибута `name`. Отпечаток сертификата определяется строковым значением атрибута `thumbprint`, состоящим из шестнадцатеричных чисел без пробелов. Шестнадцатеричные числа должны быть представлены цифрами и алфавитными символами в верхнем регистре. Алгоритм сертификата определяется строковым значением атрибута `thumbprintAlgorithm`.|

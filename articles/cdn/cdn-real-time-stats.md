# Статистика в реальном времени в сети CDN Microsoft Azure

## Обзор

В этом документе содержатся сведения о статистике в реальном времени в сети CDN Microsoft Azure. Эта функция предоставляет актуальные данные о производительности сети CDN при доставке содержимого клиентам.

> [AZURE.NOTE] Возможности статистики в реальном времени для сетей CDN относятся к функциям уровня Premium. Сравнение возможностей CDN уровней Standard и Premium см. в статье [Обзор Azure CDN](cdn-overview.md).

Для просмотра статистики в реальном времени для платформ на базе HTTP доступны следующие диаграммы:

* [Пропускная способность](#bandwidth)
* [Коды состояний](#status-codes)
* [Состояния кэша](#cache-statuses)
* [Подключения](#connections)

> [AZURE.NOTE] На каждой диаграмме отображается статистика в режиме реального времени за указанный период времени. По истечении заданного периода будет выведено скользящее окно данных. Это означает, что старые данные будут удалены из диаграммы, чтобы освободить место для новых данных. Чтобы задать продолжительность для скользящего окна, можно воспользоваться параметром "Интервал времени для диаграмм".

## Доступ к статистике в реальном времени

1. В колонке профиля сети CDN нажмите кнопку **Управление**.

	![Кнопка управления колонкой профиля сети CDN](./media/cdn-real-time-stats/cdn-manage-btn.png)

	Откроется портал управления CDN.

2. Наведите указатель на вкладку **Аналитика**, а затем на всплывающее окно **Статистика в реальном времени**. Щелкните **Большая HTTP-платформа**.

	Отобразятся параметры отчета.

## Пропускная способность

На диаграмме "Пропускная способность" отображается объем пропускной способности, использовавшийся для текущей платформы за указанный период времени. На затененной части диаграммы показано использование пропускной способности. Точный объем пропускной способности, используемой в данный момент, отображается непосредственно под линейной диаграммой.

> [AZURE.NOTE] В отчете по использованию пропускной способности применяются следующие единицы: бит в секунду (бит/с), килобит в секунду (Кбит/с), мегабит в секунду (Мбит/с) или гигабит в секунду (Гбит/с).

## Коды состояний

Диаграмма "Коды состояний" состоит из цветных строк, которые указывают частоту возникновения кодов HTTP-ответов за указанный период времени. В левой части диаграммы (ось y) отображается частота возврата кода состояния для запросов, а в нижней части диаграммы (ось x) указывается ход времени.

Список кодов состояний находится непосредственно над диаграммой. В списке приводится каждый код состояния, который может быть включен в линейную диаграмму, и текущее количество вхождений в секунду этого кода состояния. По умолчанию отображается строка для каждого кода состояния в диаграмме. Однако можно отслеживать только те коды состояний, которые имеют особое значение для конфигурации сети CDN. Для этого нужно выбрать параметры требуемого кода состояния и отменить выбор других параметров. Выбрав необходимые коды состояний, которые будут выведены на диаграмме, нажмите кнопку "Обновить диаграмму". В этом случае не выбранные коды не будут включены в диаграмму.

> [AZURE.NOTE] Параметр **Обновить диаграмму** очистит диаграмму. После этого будут отображены только выбранные коды состояний.

Параметры кодов состояний описаны ниже.

Имя | Описание
-----|------------
Всего попаданий в секунду | Определяет, будет ли в диаграмме отображено общее число запросов в секунду для текущей платформы. Этот параметр можно использовать как базовый индикатор для просмотра процента общего количества попаданий для конкретного кода состояния.
2xx в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 2xx (например, 200, 201, 202 и т. д.) в секунду для текущей платформы. Этот тип кода состояния указывает, что запрос был успешно доставлен клиенту.
304 в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 304 в секунду для текущей платформы. Этот код состояния указывает, что запрошенный ресурс не изменялся с момента его последнего получения HTTP-клиентом.
3xx в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 3xx (например, 300, 301, 302 и т. д.) в секунду для текущей платформы. Этот параметр исключает вхождение кодов состояний 304. Этот тип кода состояния указывает, что результатом запроса стало перенаправление.
403 в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 403 в секунду для текущей платформы. Этот код состояния указывает, что запрос считается несанкционированным. Одна из возможных причин возникновения этого кода состояния: неавторизованный пользователь запрашивает ресурс, защищенный проверкой подлинности на основе маркеров.
404 в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 404 в секунду для текущей платформы. Этот код состояния указывает, что не удалось найти запрошенный ресурс.
4xx в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 4xx (например, 400, 401, 402, 405 и т. д.) в секунду для текущей платформы. Этот параметр исключает вхождения кодов состояний 403 и 404. Этот тип кода состояния указывает, что запрошенный ресурс не был доставлен клиенту.
5xx в секунду | Определяет, будет ли в диаграмме отображено общее число кодов состояний 5xx (например, 500, 501, 502 и т. д.) в секунду для текущей платформы.
Прочее в секунду | Определяет, будет ли в диаграмме указано общее число вхождений для всех остальных кодов состояний.

Можно временно скрыть данные журнала для определенного кода состояния. Для этого в области под диаграммой нужно отменить выбор нужного варианта кода состояния. Выбранный код состояния сразу же пропадет из диаграммы. Если выбрать этот код состояния, он снова появится в диаграмме.

> [AZURE.NOTE] Выделенные цветом параметры под диаграммой влияют только на отображение элементов на диаграмме. Они не влияют на возможность отслеживания указанного кода состояния.

## Состояния кэша

Диаграмма "Состояния кэша" состоит из цветных строк, которые указывают частоту возникновения определенных типов состояний кэша за указанный период времени. В левой части диаграммы (ось y) отображается частота возврата состояния кэша для запросов, а в нижней части диаграммы (ось x) указывается ход времени.

Список состояний кэша находится непосредственно над диаграммой. В списке приводится каждое состояние кэша, которое может быть включено в линейную диаграмму, и текущее количество вхождений в секунду этого состояния кэша. По умолчанию отображается строка для каждого состояния кэша в диаграмме. Однако можно отслеживать только те состояния кэша, которые имеют особое значение для конфигурации сети CDN. Для этого нужно выбрать параметры требуемого состояния кэша и отменить выбор других параметров. Выбрав необходимые состояния кэша, которые будут выведены на диаграмме, нажмите кнопку "Обновить диаграмму". В этом случае не выбранные коды не будут включены в диаграмму.

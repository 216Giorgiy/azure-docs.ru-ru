#Включение сети доставки содержимого (CDN) для Azure  

После включения конечной точки CDN для своего источника, такой как учетная запись хранения или облачная служба, все общедоступные объекты будут пригодны для пограничного кэширования CDN.  


##Создание новой конечной точки CDN  

1.	Выполните вход на [портал управления Azure](http://manage.windowsazure.com/).
2.	В области навигации щелкните **Параметры**.
3.	На ленте щелкните **Создать** В диалоговом окне **Создать** выберите **Служба приложений**, затем **CDN**, затем **Быстрое создание**.
4.	В раскрывающемся списке **Исходный домен** выберите требуемое назначение.
5.	Нажмите кнопку **Создать**, чтобы создать новую конечную точку.




> Примечание. Конфигурация, созданная для конечной точки, не будет доступна незамедлительно. Для распространения регистрации по сети CDN может потребоваться до 60 минут. Если пользователь попытается незамедлительно воспользоваться именем домена CDN, он может столкнуться с кодом состояния 400 (Неверный запрос), пока содержимое не станет доступно через CDN.

#Включение сети доставки содержимого (CDN) для Azure  

CDN для вашего источника можно включить на портале управления Azure. В настоящее время доступны источники таких типов: веб-приложение, хранилище, облачная служба. CDN можно также включить для конечной точки потоковой передачи служб мультимедиа Azure. После включения конечной точки CDN для вашего источника все общедоступные объекты становятся доступными для пограничного кэширования CDN.

Обратите внимание, что теперь можно создать пользовательский источник, который не обязательно должен быть размещен в Azure.

##Создание новой конечной точки CDN  

1.	Выполните вход на [портал управления Azure](http://manage.windowsazure.com/).
2.	В области навигации щелкните **CDN**.
3.	На ленте щелкните **Создать** В диалоговом окне **Создать** выберите **СЛУЖБЫ ПРИЛОЖЕНИЙ**, затем **CDN**, а затем **БЫСТРОЕ СОЗДАНИЕ**.
4.	В раскрывающемся списке **ТИП ИСТОЧНИКА** выберите тип источника в списке доступных типов источника.
	
	Список доступных URL-адресов источников будет отображаться в раскрывающемся списке **URL-АДРЕС ИСТОЧНИКА**.
	

	![createnew][createnew]

	При выборе типа **Пользовательский источник** можно ввести URL-адрес пользовательского источника. Не обязательно источника Azure.

	![customorigin][customorigin]

	>[AZURE.NOTE]В настоящее время для источника поддерживается только HTTP, и при этом нужно использовать расширение службы мультимедиа Azure, чтобы включить поддержку Azure CDN для конечной точки потоковой передачи служб мультимедиа Azure.
	
5.	Нажмите кнопку **Создать**, чтобы создать новую конечную точку.


>[AZURE.NOTE]Конфигурация, созданная для конечной точки, не будет доступна незамедлительно. Для распространения регистрации по сети CDN может потребоваться до 60 минут. Если пользователь попытается незамедлительно воспользоваться именем домена CDN, он может столкнуться с кодом состояния 400 (Неверный запрос), пока содержимое не станет доступно через CDN.

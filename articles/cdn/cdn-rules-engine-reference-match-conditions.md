# <a name="match-conditions-for-the-content-delivery-network-rules-engine"></a>Условия совпадения для этого обработчик правил сети доставки содержимого
В этой статье перечислены подробные описания условий подходящим соответствием для сети доставки содержимого Azure [обработчик правил](cdn-rules-engine.md).

Вторая часть правила — это условие соответствия. Условие соответствия определяет различные типы запросов, для которых будет выполнен ряд функций.

Например можно использовать условия соответствия для:
- Фильтрация запросов для содержимого в определенном месте.
- Фильтрация запросов создаваемый определенный IP-адрес или страны.
- Фильтрация запросов по сведения заголовка.

## <a name="always-match-condition"></a>Всегда соответствуют условию

Условие соответствия всегда применяется ряд возможностей по умолчанию для всех запросов.

ИМЯ | Назначение
-----|--------
[Всегда](#always) | Набор компонентов по умолчанию применяется ко всем запросам.

## <a name="device-match-condition"></a>Условие соответствия устройства

Условие соответствия "Устройство" определяет запросы, поступающие с мобильного устройства, на основе его свойств.  

ИМЯ | Назначение
-----|--------
[Устройство](#device) | Определяет запросы от мобильных устройств на основе его свойств.

## <a name="location-match-conditions"></a>Условия совпадения для расположения

Условия совпадения для расположения идентификации запросов, на основе расположения инициатора.

-----|--------
[КАК число](#as-number) | Определяет запросы, поступающие из конкретной сети.
[Страна](#country) | Определяет запросы, поступающие из указанных стран.

## <a name="origin-match-conditions"></a>Условия совпадения источника

Условия совпадения источника определите запросы, которые указывают сервер происхождения клиента или сеть доставки содержимого хранилища.

-----|--------
[Источник CDN](#cdn-origin) | Определяет запросы содержимого, хранящегося в хранилище, сеть доставки содержимого.
[Источник клиента](#customer-origin) | Определяет запросы к содержимому, хранящемуся на конкретном сервере-источнике клиента.

### <a name="as-number"></a>Номер AS 
Номер AS сети определяется его номер автономной системы (ASN). Чтобы указать, будет ли это условие, если клиента к сети «Соответствует» или «Does Not Match» указанного ASN предусмотрен специальный параметр.

Основные сведения
- Укажите несколько ASN, разделяя каждого из них один пробел. Например, 64514 64515 соответствует запросов, приходящих 64514 или 64515.
- Некоторые запросы могут не возвращать допустимый ASN. Знак вопроса (?) будет соответствовать запросы, для которых не удалось определить допустимым ASN.
- Необходимо указать все ASN для требуемую сеть. Частичные значения не будет совпадать.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша
  - Внутренний максимальный срок по умолчанию
  - Принудительное использование внутреннего максимального срока
  - Не учитывать источник без кэша
  - Внутреннее максимальное значение устаревания

### <a name="cdn-origin"></a>Источник CDN
Источник CDN совпадения условие выполняется, если выполняются оба следующих условия:
- Содержимое из сети доставки содержимого хранилища был запрошен.
- Запрос URI использует точку доступа к содержимому (например, /000001), определенные в данном состоянии соответствия.
  - Содержимого сети URL-адрес доставки: URI запроса должен содержать точку доступа выбранного содержимого.
  - URL-адрес CNAME Edge: Соответствующей конфигурации CNAME edge должен указывать на точку доступа выбранного содержимого.
  
Основные сведения
 - Точка доступа к содержимому находит службу, которая должна использоваться запрошенного содержимого.
 - Не используйте инструкции IF и для объединения условий соответствия. Например объединение на соответствие критерию источника CDN с условием соответствия источника клиента создать шаблон сопоставления, который никогда не будет сопоставлено. По этой причине нельзя объединять два условия соответствия источника CDN посредством инструкции IF и.

### <a name="client-ip-address"></a>IP-адрес клиента
Параметр предоставляется для указания ли условие IP-адрес клиента будет выполняться, если IP-адрес «Соответствует» или «Does Not Match» клиента указанного IP-адреса.

Основные сведения
- Убедитесь, что используют нотацию CIDR.
- Укажите несколько IP-адресов или блоки IP-адресов, разделяя каждого из них один пробел.
  - **Пример IPv4**: 1.2.3.4 10.20.30.40 соответствует все запросы, поступающие из 1.2.3.4 или 10.20.30.40.
  - **Пример IPv6**: все запросы, поступающие из 1:2:3:4:5:6:7:8 или 10:20:30:40:50:60:70:80 соответствует 1:2:3:4:5:6:7:8 10:20:30:40:50:60:70:80.
- Синтаксис для блок IP-адресов — базовый IP-адрес, за которым следует косой черты и размер префикса.
  - **Пример IPv4**: все запросы, поступающие из 5.5.5.64 через 5.5.5.127 соответствует 5.5.5.64/26.
  - **Пример IPv6**: 1:2:3: / 48 соответствует все запросы, поступающие из 1:2:3:0:0:0:0:0 через 1:2:3:ffff:ffff:ffff:ffff:ffff.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша
  - Внутренний максимальный срок по умолчанию
  - Принудительное использование внутреннего максимального срока

### <a name="cookie-parameter"></a>Параметр файла cookie
**Совпадений**/**не соответствует** определяет условия, при которых условию параметра куки-файл будет удовлетворен.
- **Соответствует**: требует запрос, содержащий указанный куки-файл со значением, которое соответствует по крайней мере одно из значений, определенных в данном состоянии соответствия.
- **Не соответствует**: требует, что запрос соответствует любому из следующих условий:
  - Он не содержит указанного файла cookie.
  - Он содержит указанного файла cookie, но его значение не соответствует любое из значений, определенных в данном состоянии соответствия.
  
Основные сведения
- Имя файла cookie: 
  - Специальные символы, включая символ звездочки не поддерживаются, если было указано имя файла cookie. Это означает, что только совпадения имени точное cookie пригодны для сравнения.
  - Для экземпляра этого условия соответствия можно указать только имя одного файла cookie.
  - Файл cookie имя сравнения выполняются без учета регистра.
- Значение cookie: 
  - Укажите значения нескольких файлов cookie, разделяя каждого из них один пробел.
  - Значение файла cookie могут воспользоваться преимуществами специальные символы. 
  - Если не был указан подстановочный знак, только при точном совпадении будет удовлетворять это условие соответствия. Например указав «Value» будет соответствовать «Значение», но не «Значение1» или «Значение2».
  - **Игнорировать регистр** параметр определяет, будет ли сравнение с учетом регистра должны выполняться значение файла cookie в запросе.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша
  - Внутренний максимальный срок по умолчанию
  - Принудительное использование внутреннего максимального срока
  - Не учитывать источник без кэша
  - Внутреннее максимальное значение устаревания

**Совпадений**/**не соответствует** параметр определяет, в котором эта условию удовлетворяет условиям.
- **Соответствует**: требует запрос, содержащий указанный куки-файл со значением, которое соответствует указанному регулярному выражению.
- **Не соответствует**: требует, что запрос соответствует любому из следующих условий:
  - Он не содержит указанного файла cookie.
  - Он содержит указанного файла cookie, но его значение не соответствует указанному регулярному выражению.
  
Основные сведения
- Имя файла cookie: 
  - Регулярные выражения и специальные символы, включая символ звездочки не поддерживаются, если было указано имя файла cookie. Это означает, что только совпадения имени точное cookie пригодны для сравнения.
  - Для экземпляра этого условия соответствия можно указать только имя одного файла cookie.
  - Файл cookie имя сравнения выполняются без учета регистра.
- Значение cookie: 
  - Значение файла cookie могут воспользоваться преимуществами регулярные выражения.
  - **Игнорировать регистр** параметр определяет, будет ли сравнение с учетом регистра должны выполняться значение файла cookie в запросе.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша

### <a name="edge-cname"></a>Граничная запись Cname
Основные сведения 
- Список доступных края записей CNAME только те, которые были настроены на странице записей CNAME Edge, соответствующий платформе настроен обработчик HTTP правил.
- Прежде чем пытаться удалить конфигурацию CNAME edge, убедитесь, что условие соответствия Edge Cname не на него ссылаться. Край CNAME конфигурации, которые были определены в правиле не удается удалить из записей CNAME края страницы. 
- Не используйте инструкции IF и для объединения условий соответствия. Например объединение Edge Cname совпадения условие источника клиента соответствия создать шаблон сопоставления, который никогда не будет сопоставлено. По этой причине нельзя объединять два условия совпадения Edge Cname посредством инструкции IF и.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша
  - Внутренний максимальный срок по умолчанию
  - Принудительное использование внутреннего максимального срока
  - Не учитывать источник без кэша
  - Внутреннее максимальное значение устаревания

### <a name="referring-domain"></a>Домен источников ссылок
Имя узла, связанный с referrer, через который содержимого была запрошена, определяет ли условие ссылаются домена. Параметр позволяет указать, будет ли это условие, если имя узла ссылающийся «Соответствует» или «Does Not Match» указанные значения.

Основные сведения
- Указывайте несколько имен узлов, разделяя каждого из них один пробел.
- Это условие соответствия поддерживает специальные символы.
- Если указанное значение не содержит звездочку, ее необходимо точное соответствие имени узла источника ссылки. Например задание «mydomain.com» не соответствовал бы «www.mydomain.com».
- **Игнорировать регистр** параметр определяет, выполняется ли сравнение с учетом регистра.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша

  - Принудительное использование внутреннего максимального срока

### <a name="request-header-literal"></a>Литерал заголовка запроса
**Совпадений**/**не соответствует** параметр определяет, в котором эта условию удовлетворяет условиям.
- **Соответствует**: требуется содержать указанный заголовок запроса. Его значение должно соответствовать той, для которой определяется в данное состояние соответствия.
- **Не соответствует**: требует, что запрос соответствует любому из следующих условий:

Основные сведения

- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша

  - Принудительное использование внутреннего максимального срока

### <a name="request-header-regex"></a>Регулярное выражение заголовка запроса
**Совпадений**/**не соответствует** определяет условия, при которых условию Regex заголовок запроса будет удовлетворен.
- **Соответствует**: требуется содержать указанный заголовок запроса. Его значение должно соответствовать шаблону, определенному в указанному регулярному выражению.
- **Не соответствует**: требует, что запрос соответствует любому из следующих условий:
  - Он не содержит указанный заголовок.
  - Он содержит указанный заголовок, но его значение не соответствует указанному регулярному выражению.

Основные сведения
- Имя заголовка: 
  - Заголовок имя сравнения выполняются без учета регистра.
  - Пробелы в имени заголовка необходимо заменить «% 20». 
- Значение заголовка: 
  - Значение заголовка можно воспользоваться преимуществами регулярного выражения.
  - **Игнорировать регистр** параметр определяет чувствительность к регистру сравнения значение заголовка.
  - Заголовок точных совпадений значение по крайней мере одно из указанных шаблонов будет удовлетворять этому условию.
- Из-за способом, в какой кэш отслеживаются параметры это условие соответствия несовместим с помощью следующих средств:
  - Завершение заполнения кэша
  - Внутренний максимальный срок по умолчанию
  - Принудительное использование внутреннего максимального срока
  - Не учитывать источник без кэша
  - Внутреннее максимальное значение устаревания

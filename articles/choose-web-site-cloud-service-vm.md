# Сравнение веб-сайтов, облачных служб и виртуальных машин Azure

## Обзор

Azure предоставляет несколько способов для размещения веб-сайтов: [Веб-сайты][Веб-сайты], [облачные службы][облачные службы] и [виртуальные машины Azure][виртуальные машины Azure]. Данная статья поможет вам понять имеющиеся возможности и сделать правильный выбор веб-приложения.

Самым оптимальным вариантом для веб-приложений является служба веб-сайтов Azure. Процессы развертывания и управления являются уже интегрированными в платформу, при этом имеется возможность оперативного масштабирования сайтов для удовлетворения высоких нагрузок по трафику, а встроенные инструменты для балансировки нагрузки и управления трафиком обеспечивают высокую доступность. Перенести уже существующие сайты в службу веб-сайтов Azure можно без каких бы то ни было проблем с помощью [средства миграции в режиме онлайн][средства миграции в режиме онлайн]; кроме этого, имеется возможность для использования Коллекции веб-приложений с открытым программным кодом, а также для создания нового сайта с помощью персонально выбранной платформы и инструментов. Функция [Веб-задания][Веб-задания] (находящаяся в настоящее время на стадии предварительной версии) позволяет просто и удобно добавлять к приложениям фоновые задания обработки.

Если требуется более широкий контроль над средой веб-сервера, например возможность удаленного входа на сервер или настройка заданий, запускаемых при запуске сервера, наиболее подходящими для этого будут облачные службы Azure.

Если какое-либо существующее приложение требует внесения значительных изменений, чтобы запустить его с помощью службы Веб-сайтов Azure или Облачных служб Azure, для упрощения процесса миграции в облачную среду можно использовать виртуальные машины Azure. Однако для надлежащей настройки, обеспечения безопасности и обслуживания виртуальных машин требуется гораздо больше рабочего времени, а также более высокий уровень подготовки специалистов в сфере ИТ-технологий по сравнению с использованием службы Веб-сайтов Azure и Облачных служб. При рассмотрении варианта использования виртуальных машин Azure следует оценить объем затрат на оказание надлежащего технического обслуживания при эксплуатации, требующегося для внесения исправлений, обновлений, а также для управления средой виртуальных машин.

На следующей диаграмме представлено соотношение между уровнем контроля и простотой использования для каждого из вариантов размещения веб-ресурсов на платформе Azure.

### <span id="onprem"></span></a> Требуется веб-интерфейс с возможностями фоновой обработки и с поддержкой базы данных для выполнения бизнес-приложений, интегрированных с локальными ресурсами.

Веб-сайты Azure являются оптимальным решением для комплексных бизнес-приложений. Этот вариант позволяет разрабатывать собственные приложения с автоматическим масштабированием на платформе с балансировкой нагрузки, обеспечивает защиту с помощью технологии Active Directory и предоставляет подключение к локальным ресурсам. Этот вариант предоставляет удобное управление такими приложениями посредством API и портала управления с возможностями, соответствующими мировым стандартам, а также позволяет собирать с помощью соответствующих инструментов аналитическую информацию относительно того, как клиенты используют эти ресурсы. Новая функция [Веб-задания][Веб-задания] позволяет запускать фоновые процессы и задания в качестве составных компонентов веб-уровня, при этом возможности для гибридного подключения и [функции VNET][функции VNET] обеспечивают удобное подключение к локальным ресурсам. При использовании службы Веб-сайтов Azure предоставляется три Соглашения об уровне обслуживания (SLA) и возможность:

-   Запускать приложения с высоким уровнем надежности на облачной платформе, где выполняется самостоятельное устранение ошибок и автоматическое внесение исправлений.
-   Выполнять автоматическое масштабирование в центрах обработки данных, расположенных в разных странах мира.
-   Выполнять резервное копирование и восстановление данных при возникновении аварийных ситуаций.
-   Обеспечивать соответствие стандартам ISO, SOC2 и PCI.
-   Интеграция с Active Directory

### <span id="corp"></span></a> Требуется надежный способ размещения корпоративного веб-сайта с возможностями масштабирования и с широким всемирным охватом.

Веб-сайты Azure являются оптимальным решением для размещения корпоративных веб-сайтов. Этот вариант позволяет осуществлять оперативное и удобное масштабирование сайтов для удовлетворения необходимых потребностей в пределах глобальной сети центров обработки данных. Этот вариант обеспечивает локальный охват, защиту от сбоев и интеллектуальное управление трафиком. Все ресурсы размещаются на платформе, предоставляющей инструменты, соответствующие мировым стандартам, что позволяет оперативно и удобно собирать аналитическую информацию о работоспособности сайта и об используемом сайтом трафике. При использовании службы Веб-сайтов Azure предоставляется три Соглашения об уровне обслуживания (SLA) и возможность:

-   Размещать веб-сайты при обеспечении высокого уровня надежности на облачной платформе, где выполняется самостоятельное устранение ошибок и автоматическое внесение исправлений.
-   Выполнять автоматическое масштабирование в центрах обработки данных, расположенных в разных странах мира.
-   Выполнять резервное копирование и восстановление данных при возникновении аварийных ситуаций.
-   Управлять журналами событий и трафиком с помощью интегрированных инструментов.
-   Обеспечивать соответствие стандартам ISO, SOC2 и PCI.
-   Интеграция с Active Directory

### <span id="iis6"></span></a> Имеется приложение IIS6, работающее на базе ОС Windows Server 2003.

Служба Веб-сайтов Azure позволяет избежать расходов на инфраструктуру, связанных с миграцией приложений, созданных на устаревшей платформе IIS6. Специалисты компании Microsoft разработали [удобные инструменты и подробные руководства по миграции][удобные инструменты и подробные руководства по миграции], которые позволяют проверить совместимость и выявить любые изменения, которые необходимо внести. Интеграция с Visual Studio, TFS и с широко распространенными инструментами CMS позволяет развертывать приложения IIS6 непосредственно в облачной среде. После развертывания на портале управления Azure клиентам предоставляются многофункциональные инструменты для управления, позволяющие оптимизировать масштабирование в соответствии с расходами и необходимыми потребностями. С помощью инструмента для миграции можно:

-   Осуществлять быструю и удобную миграцию в облачную среду веб-приложений, созданных для устаревшей платформы Windows Server 2003.
-   Выбрать возможность локального размещения соответствующей базы данных SQL для создания гибридного приложения.
-   Выполнить автоматический перенос базы данных SQL вместе с устаревшим приложением.

### <span id="smallbusiness"></span></a> Я владелец малого бизнеса и мне необходимо недорогое размещение сайта с возможностями расширения.

Веб-сайты Azure — оптимальное решение для этой ситуации, поскольку эту службу можно использовать бесплатно и при необходимости добавлять дополнительные возможности. Для каждого бесплатного веб-сайта в Azure предоставляется свой домен (имеющий вид *название\_вашей\_компании*.azurewebsites.net), к тому же на данной платформе имеются интегрированные инструменты для выполнения развертывания и управления, а также коллекция приложений, что значительным образом упрощает процесс начала работы. Существует множество других служб и параметров масштабирования, позволяющих соответствовать повышенным требованиям пользователей. С помощью веб-сайтов Azure можно:

-   Начать работу с бесплатного уровня и при необходимости увеличить масштаб.
-   Использовать коллекцию приложений для быстрой установки популярных веб-приложений, таких как WordPress.
-   Добавлять в приложение дополнительные службы Azure и функции при необходимости.
-   Защитить веб-сайты при помощи технологии HTTPS.

### <span id="designer"></span></a> Я веб-дизайнер или графический дизайнер, и мне необходимо создать веб-сайты для клиентов

Для нужд веб-разработчиков и веб-дизайнеров Веб-сайты Azure можно удобно интегрировать с различными платформами и инструментами, включая поддержку развертывания для Git и FTP, а также обеспечить тесную интеграцию с такими инструментами и службами, как Visual Studio и база данных SQL. С помощью веб-сайтов можно:

-   Использовать программы командной строки для [автоматизированных задач][автоматизированных задач].
-   Работать с популярными языками, такими как [.Net][.Net], [PHP][PHP], [Node.js][Node.js] и [Python][Python].
-   Выбирать три различных уровня масштабирования для масштабирования высокой емкости.
-   Осуществлять интеграцию с другими службами Azure, такими как [база данных SQL][база данных SQL], [Service Bus][Service Bus] и [служба хранилища][служба хранилища], или партнерскими предложениями от [Магазина Azure][Магазина Azure], такими как MySQL и MongoDB.
-   Осуществлять интеграцию с такими инструментами, как Visual Studio, Git, WebMatrix, WebDeploy, TFS и FTP.

### <span id="multitier"></span></a> Я переношу многоуровневое приложение с веб-ролью в облако

Если используется многоуровневое приложение, например веб-сервер, работающий совместно с базой данных, Веб-сайты Azure также являются оптимальным выбором. Они обеспечивают тесную интеграцию с базой данных SQL в Azure. Также имеется возможность использования новой функции Веб-задания (в предварительной версии) для запуска фоновых процессов.

Облачные службы рекомендуется выбирать для одного или нескольких уровней в случае, если имеется необходимость более строгого контроля за серверной средой, например, реализация удаленного доступа к серверам или настройка заданий, запускаемых при включении сервера.

Облачные службы рекомендуется выбирать для одного или нескольких уровней в случае, если предполагается использовать собственные образы машин или собственное серверное ПО и службы, которые невозможно настроить в Облачных службах Azure.

### <span id="custom"></span></a>Мое приложение зависит от гибко настраиваемой среды Windows или Linux, и его необходимо перенести в облако.

Если для приложения требуется сложная установка или настройка операционной системы и программного обеспечения, Виртуальные машины, возможно, являются наилучшим решением. С помощью Виртуальных машин можно:

-   Использовать коллекцию виртуальных машин для запуска операционной системы, например Windows или Linux, и настроить ОС для требований приложения.
-   Создавать и загружать настраиваемый образ существующего локального сервера для выполнения на виртуальных машинах в Azure.

### <span id="oss"></span></a>Мой веб-сайт использует программное обеспечение с открытым исходным кодом, и его необходимо размещать в Azure

Если платформа с открытым программным кодом поддерживается службой Веб-сайтов, то выполняется автоматическая настройка для соответствующих языков программирования и интегрированных сред разработки. Веб-сайты позволяют:

-   Использовать многочисленные популярные языки с открытым исходным кодом, такие как [.NET][.Net], [PHP][PHP], [Node.js][Node.js] и [Python][Python].
-   Настраивать WordPress, Drupal, Umbraco, DNN и многие другие веб-приложения сторонних производителей.
-   Переносить существующие приложения или создавать новые из коллекции приложений.

Если платформа с открытым программным кодом не поддерживается службой Веб-сайтов, можно запустить ее с использованием двух других вариантов, предлагаемых Azure для размещения веб-приложений. С помощью Облачных служб можно использовать запускаемые при включении задачи для установки и настройки необходимого программного обеспечения с открытым исходным кодом, которое работает на Windows. С помощью Виртуальных машин можно выполнять установку и настройку программного обеспечения в образе машины на базе Windows или Linux.

### <span id="lob"></span></a> У меня есть бизнес-приложение, которому необходимо подключиться к корпоративной сети

Если вы хотите создать бизнес-приложение, для веб-сайта может потребоваться прямой доступ к службам или данным в корпоративной сети. Это возможно реализовать на Веб-сайтах, в Облачных службах и на виртуальных машинах при использовании [службы виртуальных машин Azure][службы виртуальных машин Azure]. На Веб-сайтах можно использовать новую [функцию интеграции VNET][функцию интеграции VNET], которая позволяет запускать приложения Azure также, как если бы они находились в корпоративной сети.

### <span id="mobile"></span></a> Требуется разместить интерфейс API REST или веб-службу для мобильных клиентов

Веб-службы на базе HTTP позволяют поддерживать широкий спектр клиентов, включая мобильных клиентов. Платформы, такие как ASP.NET Web API, интегрируются с Visual Studio для упрощения процессов создания и использования служб REST. Эти службы предоставляются из конечной веб-точки, поэтому для этого сценария возможно любое размещение на Azure. Тем не менее, Веб-сайты отлично подходят для размещения интерфейсов API REST. С помощью веб-сайтов можно:

-   Быстро создать веб-сайт для размещения веб-службы HTTP в одном из глобально распределенных центров обработки данных Azure.
-   Перенести уже существующие службы или создать новые.
-   Обеспечить выполнение соглашения об уровне обслуживания касательно доступности с помощью одного экземпляра или увеличить масштаб до нескольких выделенных компьютеров.
-   Использовать опубликованный сайт, чтобы предоставлять API REST для любых клиентов HTTP, включая мобильные клиенты.

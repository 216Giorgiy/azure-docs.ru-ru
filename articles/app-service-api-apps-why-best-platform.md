# Что такое приложения API?

## Обзор

Служба приложений Azure - это полностью управляемая вычислительная платформа для профессиональных разработчиков, предоставляющая обширный набор возможностей для сценариев веб- и мобильной разработки, а также интеграции. Приложения API - это единая часть набора службы приложений, позволяющая техническому пользователю или разработчику обнаруживать, размещать, монетизировать соединители API и SaaS и управлять ими на современной, многофункциональной, масштабируемой и глобально доступной облачной платформе.

## Почему именно приложения API?

Приложения API в службе приложений Azure позволяют легко разрабатывать, публиковать, монетизировать API и управлять ими.

- **Использование API без изменений**. Используйте ASP.NET, Java, PHP, Node.js или Python для API. В API можно использовать преимущества функций службы приложений Azure без внесения изменений.

- **Подключение приложений к популярным платформам SaaS**. Служба приложений Azure позволяет легко подключаться к популярным платформам SaaS, включая Salesforce, Office 365, Twitter, Facebook, Dropbox и многие другие.

- **Простой контроль доступа**. API можно предоставить другим приложения в службе приложений Azure или сделать общедоступными. Кроме того, вы можете добавить проверку подлинности Azure Active Directory или сторонними службами без внесения изменений в код.

- **Общедоступная коллекция API и коллекция API организации**. Легко делитесь API с другими группами в организации, используя собственную частную коллекцию API организации. API можно также предоставлять для общего доступа сторонним разработчикам.

- **Автоматическое создание SDK**. Служба приложений Azure может автоматически создавать пакеты SDK для разных языков, включая C#, Java и Javascript, предоставляя API на многие платформы.

- **Использование лучшей среды IDE на рынке**. Интеграция [Visual
Studio](/campaigns/visual-studio-2013/) упрощает создание, отладку, упаковку и публикацию API, а также позволяет управлять полным жизненным циклом приложения.

- **Доступ к локальным данным**. Приложения API могут использовать данные из собственного центра обработки данных с помощью [гибридных подключений](integration-hybrid-connection-overview.md) и [VNET](web-sites-integrate-with-vnet.md).

- **Развертывание без конфликтов**. Настройте рабочие процессы непрерывной интеграции и развертывания в Visual Studio Online, GitHub, TeamCity, Hudson или BitBucket, что позволит вам автоматически выполнять сборку, тестировать и развертывать приложение API при каждом успешном возврате кода или тестах интеграции.

## Основные понятия приложений API ##

- **Коллекция приложений API**. Выберите шаблон приложения API из постоянно увеличивающегося списка существующих шаблонов.
- **Соединители**. Тип приложения API, позволяющий легко подключаться к платформам SaaS, например Salesforce и Office 365.
- **Шлюз**. Веб-приложение, обрабатывающее функции управления API, например проверку подлинности для всех приложений API в группе ресурсов. 
- **Автомасштабирование**. Приложения API могут выполнять автоматическое вертикальное или горизонтальное масштабирование для обработки входящей пользовательской нагрузки. Вручную выберите количество и размер виртуальных машин или настройте автомасштабирование для масштабирования серверов на основе нагрузки или по расписанию.
- **Непрерывная интеграция**. Настройте рабочие процессы непрерывной интеграции и развертывания в VSO, GitHub, TeamCity, Hudson или BitBucket, что позволит вам автоматически выполнять сборку, тестировать и развертывать веб-приложение при каждом успешном возврате кода или тестах интеграции.

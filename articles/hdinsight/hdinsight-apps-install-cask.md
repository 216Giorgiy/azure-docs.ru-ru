# <a name="install-published-application---cask-data-application-platform-cdap"></a>Установка опубликованного приложения Cask Data Application Platform (CDAP)

В этой статье описывается, как установить и запустить опубликованное приложение Hadoop [CDAP](http://cask.co/products/cdap/) в Azure HDInsight. Обзор платформы приложений HDInsight и список доступных опубликованных приложений независимых поставщиков программного обеспечения приведены в статье [Установка сторонних приложений Hadoop в Azure HDInsight](hdinsight-apps-install-applications.md). Инструкции по установке собственного приложения см. в статье [Установка пользовательских приложений HDInsight](hdinsight-apps-install-custom-applications.md).

## <a name="about-cdap"></a>Сведения о CDAP

Разработка приложений в Hadoop может оказаться сложной задачей.  Существует большое количество расширений технологии Hadoop, для интеграции которых может понадобиться некоторое время. Для отслеживания потока данных и сбора метрики необходимо создать отдельное решение.

### <a name="how-does-cdap-help"></a>Как работает CDAP?

Cask Data Application Platform (CDAP) — это платформа для интеграции больших данных. CDAP позволяет сосредоточиться на создании приложений, а не на базовой инфраструктуре.

CDAP использует концепции и абстракции высокого уровня, которые хорошо знакомы разработчикам. Эти абстракции скрывают сложности внутренних систем и способствуют многократному использованию решений.

Расширение CDAP, известное как [Cask Hydrator](http://cask.co/products/hydrator/), предоставляет пользовательский интерфейс для разработки конвейеров данных и управления ими. Конвейер данных состоит из различных *подключаемых модулей, которые выполняют такие задачи, как сбор данных, трансформация, анализ и последующие операции.

Каждый плагин CDAP имеет четко определенный интерфейс, поэтому оценка различных технологий — это просто замена одного подключаемого модуля другим без изменения остальной части приложения.

*Конвейеры* CDAP обеспечивают высокоуровневый графический поток данных в приложении. Эта визуализация показывает сквозной поток данных: прием, преобразование и анализ данных и, наконец, передача во внешнее хранилище данных.

В следующем примере конвейер данных принимает данные Twitter в режиме реального времени, а затем фильтрует некоторые твиты на основе заранее определенных критериев. Конвейер данных преобразует необработанные данные твита и проецирует их в более читаемый формат, а затем группирует твиты в соответствии с набором значений и записывает результаты в хранилище HBase.

![Конвейер CDAP](./media/hdinsight-apps-install-cask/pipeline.png)

Этот сквозной конвейер создан с помощью **пользовательского интерфейса Cask Hydrator**. Он использует интерфейс подключаемого модуля и функции перетаскивания для формирования связей между этапами. Вы можете самостоятельно изолировать и изменять функциональные возможности каждого подключаемого модуля. С помощью CDAP можно создать и проверить подобные конвейеры за несколько часов. Обычно в Hadoop создание таких решений может занять несколько дней.

CDAP также предоставляет расширение [Cask Tracker](http://cask.co/products/tracker/). С его помощью можно визуально отслеживать данные по мере их прохождения через приложение. Cask Tracker добавляет возможность *управления данными* в систему. Это помогает формально управлять ресурсами данных в пределах приложения. Вы можете отслеживать журнал происхождения каждой точки данных, собирать соответствующие метрики и проверять журнал данных в течение всего процесса.

Ниже представлен пример потока данных в конвейере.

![Средство для отслеживания CDAP](./media/hdinsight-apps-install-cask/tracker.png)

## <a name="prerequisites"></a>Предварительные требования

Для установки этого приложения на новый кластер HDInsight или на имеющийся кластер требуется следующая конфигурация:

* уровень кластера: "Стандартный";
* тип кластера: HBase;
* версия кластера: 3.4, 3.5.

## <a name="install-the-cdap-published-application"></a>Установка опубликованного приложения CDAP

Пошаговые инструкции по установке этого и других доступных приложений независимых поставщиков программного обеспечения см. в статье [Установка сторонних приложений Hadoop в Azure HDInsight](hdinsight-apps-install-applications.md).

## <a name="launch-cdap"></a>Запуск CDAP

1. После установки запустите CDAP из своего кластера на портале Azure. Для этого перейдите на панель **Параметры**, а затем выберите **Приложения** в категории **Общие**. На панели "Установленные приложения" перечислены все установленные приложения.

    ![Установленное приложение CDAP](./media/hdinsight-apps-install-cask/cdap-app.png)

2. При выборе CDAP отображается ссылка на веб-страницу, конечная точка HTTP, а также путь к конечной точке SSH. Щелкните ссылку на веб-страницу.

3. При появлении соответствующего запроса введите учетные данные администратора кластера.

    ![Authentication](./media/hdinsight-apps-install-cask/auth.png)

4. После входа в систему откроется домашняя страница графического пользовательского интерфейса Cask CDAP.

    ![Домашняя страница графического пользовательского интерфейса Cask](./media/hdinsight-apps-install-cask/gui.png)

5. Чтобы изучить интерфейс CDAP, щелкните ссылку меню **Cask Market** в верхней части страницы.

    ![Ссылка на Cask Market](./media/hdinsight-apps-install-cask/cask-market.png)

6. Из списка выберите **Access Log Sample** (Доступ к образцу журнала).

    ![Доступ к образцу журнала](./media/hdinsight-apps-install-cask/market-log-sample.png)

7. Для подтверждения нажмите кнопку **Загрузить**.

    ![Кнопка "Загрузить"](./media/hdinsight-apps-install-cask/market-load.png)

8. Откроется представление демонстрационных данных. Щелкните **Далее**.

    ![Доступ к образцу журнала — просмотр данных](./media/hdinsight-apps-install-cask/market-view-data.png)

9. Выберите **Поток** в качестве типа назначения, введите имя назначения и нажмите кнопку **Завершить**.

    ![Доступ к образцу журнала — выбор назначения](./media/hdinsight-apps-install-cask/market-destination.png)

10. После загрузки пакета данных выберите **View Stream Details** (Просмотреть сведения о потоке).

    ![Передача пакета данных](./media/hdinsight-apps-install-cask/market-view-details.png)

11. Чтобы включить метаданные для пространства имен, выберите **Включить** на вкладке "Использование" на странице сведений о журнале доступа.

    ![Доступ к образцу журнала — загрузка — включение метаданных](./media/hdinsight-apps-install-cask/log-loaded.png)

12. После включения метаданных откроется график с информацией о сообщении аудита.

    ![Доступ к образцу журнала — включение метаданных](./media/hdinsight-apps-install-cask/log-metadata.png)

13. Чтобы просмотреть данные журнала, выберите значок **Просмотр** в верхней части страницы.

    ![Доступ к образцу журнала — просмотр](./media/hdinsight-apps-install-cask/log-explore.png)

14. Откроется пример SQL-запроса. Вы можете изменить его по желанию, а затем выберите **Выполнить**.

    ![Образец доступа к журналу — просмотр набора данных с помощью запроса](./media/hdinsight-apps-install-cask/log-query.png)

15. После выполнения запроса выберите значок **Просмотреть** в столбце "Действия".

    ![Образец доступа к журналу — просмотр выполненного запроса](./media/hdinsight-apps-install-cask/log-query-view.png)

16. Отобразятся результаты выполнения запроса.

    ![Образец доступа к журналу — результаты выполнения запроса](./media/hdinsight-apps-install-cask/log-query-results.png)

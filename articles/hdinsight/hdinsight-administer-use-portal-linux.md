#Управление кластерами Hadoop в HDInsight с помощью портала Azure

[AZURE.INCLUDE [selector](../../includes/hdinsight-portal-management-selector.md)]


С помощью [портала Azure][preview-portal] можно подготовить кластеры Hadoop под управлением Linux в Azure HDInsight, а также управлять ими.

> [AZURE.NOTE] Действия, описанные в этом документе, относятся только к работе с кластерами Hadoop под управлением Linux. Информацию о работе с кластерами под управлением Windows см. в статье [Управление кластерами Hadoop в HDInsight с помощью портала Azure](hdinsight-administer-use-management-portal.md).


[AZURE.INCLUDE [preview-portal](../../includes/hdinsight-azure-preview-portal-nolink.md)]


## Другие средства для администрирования HDInsight
Помимо портала Azure имеются и другие средства для администрирования HDInsight.

- [Администрирование HDInsight с помощью Azure CLI](hdinsight-administer-use-command-line.md). Azure CLI — это кроссплатформенное средство командной строки, которое позволяет управлять службами Azure

- [Администрирование HDInsight с использованием Azure PowerShell](hdinsight-administer-use-powershell.md). Azure PowerShell предоставляет командлеты PowerShell для управления службами Azure.

Перед началом работы с этой статьей необходимо иметь следующее:

## Создание кластеров Hadoop в HDInsight

Кластеры HDInsight можно создать на портале Azure. Этот процесс подробно описывается в документе [Создание кластеров под управлением Linux в HDInsight с помощью портала Azure](hdinsight-hadoop-create-linux-clusters-portal.md).

## Управление кластером

Выберите кластер на портале Azure, чтобы отобразить следующие важные сведения о кластере: имя, группу ресурсов, операционную систему и URL-адрес для кластера (используется для доступа к Ambari Web для кластеров под управлением Linux).

![Сведения о кластере](./media/hdinsight-administer-use-portal-linux/clusterdetails.png)

Следующие элементы интерфейса помогут вам понять, что означают значки в верхней части этой колонки, а также в разделах __Основные компоненты__ и __Быстрые ссылки__.

* __Параметры__ и __Все параметры__: отображают колонку __Параметры__ кластера, которая содержит подробные сведения о конфигурации кластера.

* __Панель мониторинга__, __Панель мониторинга кластера__ и __URL-адрес__ — это способы перехода к панели мониторинга кластера. В зависимости от типа кластера вам может быть доступен список панелей мониторинга в кластере. Например, после щелчка значка __Панель мониторинга__ кластер Spark выведет список панелей мониторинга, а кластер Hadoop откроет веб-интерфейс Ambari.

* __Secure Shell__. Сведения, необходимые для доступа к головному узлу кластера по протоколу SSH.

* __Масштабировать кластер__. Позволяет изменить количество рабочих узлов для этого кластера.

* __Удалить__. Удаление кластера HDInsight.

* __Быстрый запуск__ (![значок облака и молнии = быстрый запуск](./media/hdinsight-administer-use-portal-linux/quickstart.png)). Отображает сведения, необходимые для начала работы с HDInsight.

* __Пользователи__ (![значок "пользователи"](./media/hdinsight-administer-use-portal-linux/users.png)). Здесь можно указать разрешения на доступ к _порталу управления_ этого кластера для других пользователей в своей подписке Azure.

	> [AZURE.IMPORTANT] Этот параметр влияет _только_ на права доступа к этому кластеру на портале Azure. Параметр не влияет на то, кто может подключаться к кластеру HDInsight или отправлять в него задания.

* __Теги__ (![значок "теги"](./media/hdinsight-administer-use-portal-linux/tags.png)). Теги позволяют задавать пары «ключ-значение» для определения пользовательской таксономии облачных служб. Например, можно создать ключ с именем __project__, а затем использовать общее значение для всех служб, связанных с определенным проектом.

* __Документация__ — ссылки на документацию по Azure HDInsight.

> [AZURE.IMPORTANT] Для управления службами, предоставляемыми кластером HDInsight, необходимо использовать интерфейс Ambari Web или API REST Ambari. Подробнее об использовании Ambari см. в статье [Управление кластерами HDInsight с помощью веб-интерфейса Ambari](hdinsight-hadoop-manage-ambari.md).

### данных

Щелкните значок __Параметры__ или ссылку __Все параметры__, чтобы открыть колонку "Параметры". Здесь также имеются некоторые функции, доступные в области __Essentials__, такие как масштабирование или сведения о Secure Shell. Из колонки "Параметры" можно получить доступ к перечисленным далее элементам.

* __Журналы аудита__: заносимые в журнал сведения, полезные для диагностики проблем с работоспособностью кластеров.

* __Вход в кластер__: отображение __имени пользователя для входа в кластер__ и __удаленного адреса__ для доступа к кластеру с помощью HTTPS.

* __Внешние метахранилища__: отображение сведений о внешних метахранилищах, используемых кластером (если таковые имеются). Если во время настройки кластера вы не настроили пользовательское метахранилище, никакие сведения отображаться не будут.

* __Действия сценария__: отображение сведений о действиях сценария, которые выполнялись в этом кластере. Можно также запускать новые действия сценария и сохранять или удалять выполненные ранее. Дополнительные сведения о действиях сценариев см. в статье [Настройка кластеров HDInsight с помощью действия сценария](hdinsight-hadoop-customize-cluster-linux.md).

* __Приложения__: отображение сведений о приложениях, которые были установлены в кластере, а также возможность добавления в кластер новых приложений из Azure Marketplace.

* __Ключи хранилища Azure__: отображение сведений об учетных записях службы хранилища Azure, используемых кластером. При выборе учетной записи хранения будет загружена колонка учетной записи хранения для выбранной учетной записи.

* __Удостоверение AAD кластера__: отображение сведений о субъект-службе для этого кластера HDInsight. Субъект-служба используется для доступа к хранилищу озера данных Azure. Если кластер не связан с хранилищем озера данных Azure во время создания кластера, для записей в этой колонке будет отображаться __Не настроено__. Дополнительные сведения об использовании хранилища озера данных Azure с HDInsight см. в статье [Создание кластера HDInsight с хранилищем озера данных](../data-lake-store/data-lake-store-hdinsight-hadoop-use-portal.md).

### <a name="scaling"></a>Масштабирование

Для масштабирования кластера с помощью портала выберите необходимый кластер HDInsight, а затем нажмите кнопку __Масштабировать кластер__. Введите __количество рабочих узлов__, которое вы хотите задать для кластера, и нажмите кнопку __Сохранить__.

![колонка масштабирования](./media/hdinsight-administer-use-portal-linux/scaling.png)

Подробнее об операциях масштабирования см. в статье [Сведения об использовании HDInsight в Linux](hdinsight-hadoop-linux-information.md#scaling).

## Мониторинг кластера

В колонке кластера HDInsight в разделе __Использование__ отображаются сведения о количестве ядер, доступных для HDInsight в рамках вашей подписки, о количестве ядер, выделенных для этого кластера, и их распределении по узлам кластера.

![Сведения об использовании](./media/hdinsight-administer-use-portal-linux/usage.png)

> [AZURE.IMPORTANT] Для мониторинга служб, предоставляемых кластером HDInsight, необходимо использовать интерфейс Ambari Web или API REST Ambari. Подробнее об использовании Ambari см. в статье [Управление кластерами HDInsight с помощью веб-интерфейса Ambari](hdinsight-hadoop-manage-ambari.md).

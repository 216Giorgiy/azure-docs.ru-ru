# Предварительные требования для ExpressRoute   

Чтобы подключиться к облачным службам (Майкрософт) через ExpressRoute, вам нужно выполнить следующие условия.

## Требования к учетной записи

- Наличие действительной и активной учетной записи Microsoft Azure. Она необходима для организации канала ExpressRoute. Каналы ExpressRoute являются ресурсами в подписках Azure. Подписка Azure обязательна, даже если будет выполняться только подключение к облачным службам (Майкрософт), отличным от Azure, например Office 365 и CRM Online.
- Активная подписка на Office 365 (при использовании служб Office 365). Дополнительные сведения см. в разделе [Особые требования к Office 365](#office-365-specific-requirements) этой статьи.

## Связь с поставщиком услуг подключения

- Связь с поставщиком услуг подключения из списка поддерживаемых поставщиков. У вас должны быть деловые отношения с поставщиком услуг подключения. Убедитесь, что услуги, предоставляемые вашим поставщиком услуг подключения, совместимы с ExpressRoute.
- Чтобы задействовать поставщика услуг подключения, который отсутствует в списке поддерживаемых поставщиков, вам потребуется создать подключение к облачным службам (Майкрософт) через точку обмена трафиком Ethernet.
	- Узнайте у своего поставщика услуг подключения, пользуется ли он одной из поддерживаемых точек обмена.
	- Попросите поставщика расширить вашу сеть до нужной вам точки.
	- Закажите канал ExpressRoute, указав точку обмена трафиком Ethernet в качестве поставщика услуг подключения.

## Физическая связь между вашей сетью и поставщиком услуг подключения

Подробные сведения о моделях подключения см. в соответствующем разделе. Пользователи должны обеспечить наличие физического подключения своей локальной инфраструктуры с инфраструктурой поставщика услуг подключения с помощью одной из описанных моделей.

## Требования к избыточности для подключений

Требования к избыточности физических подключений между инфраструктурой клиента и инфраструктурой поставщика услуг отсутствуют. Майкрософт требует наличие избыточности на уровне 3. Майкрософт требует, чтобы для всех сеансов пиринга между периметром Майкрософт и сетью клиента через поставщика услуг обеспечивалась избыточная маршрутизация. Если избыточность сеансов маршрутизации не обеспечивается, гарантированный уровень доступности будет недостижим.

## Рекомендации относительно IP-адресов и маршрутизации

Избыточность сеансов BGP при подключении к пограничной инфраструктуре Майкрософт — ответственность клиентов или поставщиков услуг подключения. Для клиентов, подключающихся через поставщиков услуг IP VPN, конфигурацию маршрутизации обычно выполняют поставщики услуг подключения. Пользователи, пользующиеся точкой обмена трафиком Ethernet или подключающиеся к узлам Майкрософт через поставщика связи Ethernet типа «точка — точка», должны настроить избыточные сеансы BGP для каждого сеанса пиринга. Только так они смогут получить гарантированный уровень доступности облачных служб Майкрософт. Поставщики услуг подключения могут предлагать эту услугу за дополнительную плату. Подробные сведения об ограничениях см. в таблице доменов маршрутизации в статье [Каналы ExpressRoute и домены маршрутизации](expressroute-circuit-peerings.md).

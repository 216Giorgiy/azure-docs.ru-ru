# Настройка виртуальной сети для ExpressRoute на классическом портале

В этой статье последовательно описано, как настроить виртуальную сеть и шлюз для ExpressRoute с использованием классической модели развертывания и классического портала.

Если вам нужны инструкции для модели развертывания Resource Manager, вы можете воспользоваться приведенными статьями, которые помогут [создать виртуальную сеть с помощью PowerShell](../virtual-network/virtual-networks-create-vnet-arm-ps.md) и [добавить VPN-шлюз в виртуальную сеть Resource Manager для ExpressRoute](expressroute-howto-add-gateway-resource-manager.md).

## Настройка виртуальной сети и шлюза

1. Перейдите на [классический портал Azure](http://manage.windowsazure.com).

2. В нижнем левом углу экрана нажмите кнопку **Создать**. В области навигации щелкните **Сети**, а затем — **Виртуальная сеть**. Нажмите кнопку **Настраиваемое создание** для запуска мастера настройки.

3. На странице **Сведения о виртуальной сети** укажите следующую информацию.

	- **Имя**: название виртуальной сети. Это имя виртуальной сети будет использоваться при развертывании виртуальных машин и экземпляров PaaS, поэтому не рекомендуется создавать слишком сложное имя.
	- **Расположение**: непосредственно связано с физическим расположением (регионом) ресурсов (виртуальных машин). Например, если вы хотите, чтобы виртуальные машины находились в виртуальной сети, которая физически расположена в восточной части США, выберите соответствующий регион. После создания виртуальной сети изменить связанное с ней расположение нельзя.

4. На странице **DNS-серверы и подключение VPN** укажите следующие сведения и щелкните стрелку «Далее» в правом нижнем углу.

	- **DNS-серверы**: введите имя и IP-адрес DNS-сервера или выберите ранее зарегистрированный DNS-сервер из раскрывающегося списка. Этот параметр не создает DNS-сервер, он позволяет указать DNS-серверы, которые вы хотите использовать для разрешения имен в этой виртуальной сети.
	- **Настройка VPN-подключения "сеть —сеть"**: установите флажок **Настройка VPN-подключения "сеть —сеть"**.
	- **Выберите ExpressRoute**: установите флажок **Использовать ExpressRoute**. Этот параметр отображается, только если на предыдущем шаге вы выбрали ***Настройка VPN-подключения "сеть-сеть"*** .
	- **Локальная сеть** представляет вашу физическую локальную площадку. Вы можете выбрать созданную ранее локальную сеть или создать новую.

	Если вы выбрали существующую локальную сеть, пропустите шаг 5.

5. Если вы создаете новую локальную сеть, отобразится страница **Подключение "сеть-сеть"**. Если вы выбрали уже существующую локальную сеть, эта страница не откроется в мастере, и вы сможете перейти к следующему разделу. Чтобы настроить локальную сеть, укажите следующие сведения и нажмите кнопку "Далее".

	- **Имя**: имя площадки в локальной сети.
	- **Адресное пространство**: включает в себя начальный IP-адрес и CIDR (число адресов). Можно указать любой диапазон адресов, если он не пересекается с диапазоном адресов для виртуальной сети.
	- **Добавить адресное пространство**: этот параметр не относится к ExpressRoute. **Примечание.** Необходимо создать сайт локальной сети для ExpressRoute. Префиксы адресов, указанные для сайта локальной сети, будут игнорироваться. Префиксы адресов, переданные в корпорацию Майкрософт через канал ExpressRoute, будут использоваться для целей маршрутизации.

6. На странице **Адресные пространства виртуальной сети** укажите следующие сведения и затем нажмите кнопку с галочкой в нижнем правом углу для настройки сети.

	- **Адресное пространство**: включает в себя начальный IP-адрес и число адресов. Убедитесь, что заданные адресные пространства не перекрываются ни с одним из адресных пространств, которые используются в локальной сети.
	- **Добавить подсеть** — включает в себя начальный IP-адрес и число адресов. Дополнительные подсети не обязательны, однако вам может понадобиться создать отдельную подсеть для виртуальных машин с динамическими IP-адресами (DIPS). Или может потребоваться разместить виртуальные машины в подсети отдельно от экземпляров PaaS.
	- **Добавить подсеть шлюза**: щелкните, чтобы добавить подсеть шлюза. Подсеть шлюза используется только для шлюза виртуальной сети и необходима в этой конфигурации. ***Важно:*** префикс подсети шлюза для ExpressRoute должен быть /28 или меньше (/27, /26 и т. п.).

7. Установите флажок в нижней части страницы, после чего начнется создание виртуальной сети. По завершении на странице **Сети** классического портала Azure в поле **Состояние** появится значение **Создано**.

8. На странице **Сети** выберите только что созданную виртуальную сеть и щелкните **Панель мониторинга**.
9. В нижней части панели мониторинга щелкните **СОЗДАТЬ ШЛЮЗ**, а затем **Да**.

10. В начале создания шлюза появится сообщение, информирующее о запуске шлюза. Создание шлюза может занимать до 15 минут.

11. Привяжите вашу сеть к каналу. Следуйте указаниям в статье [Как связать виртуальные сети и каналы ExpressRoute](expressroute-howto-linkvnet-classic.md).

# Что такое каталог данных Azure?

**Каталог данных Azure** (Майкрософт) является полностью управляемой облачной службой, выполняющей функции систем регистрации и обнаружения для корпоративных источников данных. Возможности **каталога данных Azure** позволяют любому пользователю — от аналитиков до специалистов по анализу данных и разработчиков — обнаруживать, анализировать и использовать источники данных и делиться знаниями для формирования и поддержки сообщества и культуры данных.

## Описание проблемы — целесообразность и обзор

Как правило, обнаружение корпоративных источников данных является неотъемлемым процессом, основанным на общей информации, известной ограниченной группе лиц внутри компании. Это создает множество проблем для компаний, желающих извлечь максимум пользы из своих информационных ресурсов.

-	Пользователи не знают о существовании источников данных, пока не столкнутся с ними в рамках другого процесса; не существует централизованного местоположения, где регистрируются эти источники данных.
-	Пока пользователь не знает, где располагается источник данных, он не может подключиться к данным с помощью клиентского приложения; для работы с данными ему необходимо знать строку или путь подключения.
-	Пока пользователь не знает, где располагается документация к источнику данных, он не может разобраться в назначении этих данных. Источники данных и документация располагаются в разных местах и используются в различных интерфейсах.
-	Если у пользователя есть вопросы по поводу информационного ресурса, он должен отыскать специалиста или их группу, ответственных за эти данные, и подключить их к работе вне сети; отсутствует явная связь между данными и стороной со специалистами, заинтересованной в их использовании.
-  Само обнаружение источника данных и документации не дает доступа к необходимым пользователю данным, пока он не поймет процесс запроса доступа к источнику этих данных.

Пока потребители данных сталкиваются с перечисленными трудностями, пользователи, ответственные за создание и обслуживание информационных ресурсов, сталкиваются с собственными проблемами.

-	Комментирование источников данных с помощью описательных метаданных зачастую оказывается напрасным; клиентские приложения обычно пропускают описания, хранящиеся в источнике данных.
-	Создание документации для источников данных часто является напрасным занятием. Ответственность за постоянную синхронизацию документации с источником данных приводит к тому, что пользователи не доверяют документации, так как она часто воспринимается устаревшей.
- Ограничение доступа к источнику данных и уверенность в том, что пользователи данных знают, как запросить к ним доступ, является краеугольным камнем.

Создание и обслуживание документации для источника данных являются сложной и трудоемкой задачей. А обеспечение доступности этой документации для всех пользователей данного источника данных — еще более сложной.

Вместе эти трудности образуют серьезный барьер для компаний, желающих способствовать и содействовать использованию и восприятию данных предприятия.

## Описание службы

**Каталог данных Azure** предназначен для решения этих проблем и позволяет предприятиям извлечь максимальную пользу из своих существующих информационных ресурсов за счет того, что пользователям проще обнаружить и понять данные, которыми они управляют.

**Каталог данных Azure** предоставляет облачную службу, в которой может быть зарегистрирован источник данных. Данные остаются на прежнем месте, однако копия метаданных и ссылка на местоположение источника данных будут добавлены в **каталог данных Azure**. Кроме того, чтобы облегчить обнаружение каждого источника данных с помощью функции поиска и сделать их доступными для пользователей, эти метаданные индексируются.

После регистрации источника данных его метаданные могут быть дополнены пользователем, выполнившим регистрацию, либо другим пользователем организации. Любой пользователь может добавить комментарий к источнику данных, предоставляя описания, теги и другие метаданные, например документацию и инструкции по запросу доступа к источнику данных. Эти описательные метаданные дополняют структурные метаданные (например, имена столбцов и типы данных), зарегистрированные из источника данных, для упрощения его обнаружения и понимания.

Основной целью регистрации источников являются обнаружение, понимание и использование источников данных. Когда корпоративным пользователям потребуются данные для работы (которой может быть бизнес-аналитика, разработка приложений, обработка и анализ данных или любая другая задача, требующая корректных данных), они могут использовать интерфейс обнаружения **каталога данных Azure**, чтобы быстро найти соответствующие требованиям данные, понять их, чтобы оценить целевую пригодность, и использовать, открыв источник данных в средстве на свой выбор. В то же время пользователи **каталога данных Azure** могут дополнять каталог, помечая, документируя и добавляя комментарии к уже зарегистрированным источникам данных. Они также могут регистрировать новые источники данных, которые сообщество пользователей каталога сможет обнаруживать, воспринимать и использовать.

## Регистрация источников данных

Регистрация источника данных выполняется с помощью средства регистрации источников данных для **каталога данных Azure**. Это приложение можно скачать с портала **каталога данных Azure**.

Процесс регистрации состоит из трех основных этапов:

1.	Подключение к источнику данных — пользователь указывает местоположение источника данных (например, экземпляра SQL Server) и учетные данные для подключения к нему.
2.	Выбор объектов для регистрации — в указанном расположении пользователь выбирает объекты, которые необходимо зарегистрировать с помощью **каталога данных Azure**. Это может быть полный набор таблиц во всех базах данных на сервере или прямо выбранное подмножество таблиц и представлений.
3.	Выполнение регистрации — пользователь завершает процесс, и средство регистрации источника данных извлекает структурные метаданные из источника данных и отправляет их в облачную службу **каталога данных Azure**.

> [AZURE.NOTE] В предварительной версии **каталога данных Azure** на данный момент поддерживаются следующие типы источников данных и ресурсов: [Источники данных, поддерживаемые каталогом данных Azure](data-catalog-dsr.md)

В предварительную версию **каталога данных Azure** будут добавляться дополнительные типы ресурсов-контейнеров и источников данных.

> [AZURE.IMPORTANT]При регистрации источника данных в **каталоге данных Azure** данные из источника не копируются. Данные копируются, когда в средстве регистрации источников данных установлен флажок "Включить предварительную версию". Копируются метаданные источника данных, а не сами данные. Примерами метаданных являются имена таблиц и другие объекты источника данных, а также имена и типы данных столбцов и другие атрибуты источника. Кроме того, метаданные включают расположение источника данных. Это позволяет пользователям подключаться к источникам данных, обнаруженных с помощью **каталога данных Azure**. Кроме того, если выбрать параметр "Включить предварительную версию", средство регистрации источника данных скопирует в **каталог данных Azure** небольшой набор записей. Эти записи будут отображаться пользователям, выполняющим поиск источника данных на портале **каталога данных Azure**.

## Дополнение метаданных источника данных

После завершения регистрации источники данных можно обнаруживать и использовать. Но главная ценность **каталога данных Azure** заключается в том, что описательные корпоративные метаданные включены в тот же интерфейс, что и структурные метаданные, полученные из источника данных. Эти дополнительные метаданные дают три существенных преимущества:

-	Зарегистрированные источники данных легче обнаружить. Пользовательские метаданные добавляются в индекс поиска **каталога данных Azure**. Это позволяет пользователям находить данные с помощью терминов и основных понятий, которые могут отсутствовать в исходном источнике данных. Например, если таблице базы данных, которая содержит данные клиента и имеет имя "tbl\_c45", предоставить понятное имя "Клиент", то пользователям будет легче найти данные клиентов. Аналогично при предоставлении описания, которое включает в себя имена отчетов, панелей мониторинга или процессов, использующих эти данные, облегчается поиск источника данных для пользователей, которые используют нижестоящие артефакты в качестве условий поиска.
-	Зарегистрированные источники данных становятся более понятными после обнаружения. Пользовательские метаданные отображаются для всех пользователей **каталога данных Azure**, просматривающих источник данных с комментариями. Это предоставляет дополнительный контекст и информацию. Большинство источников данных обычно не включают осмысленные описания или документацию, а те, что включают, часто ориентированы на технических администраторов и разработчиков баз данных. За счет дополнения источников данных в **каталоге данных Azure** с помощью описаний и тегов, соответствующих аудитории, пользователи помогают друг другу находить, понимать и использовать эти сведения.
-  Чтобы пользователи легко понимали и придерживались существующих процедур запроса доступа к данным и их источникам, каждый источник зарегистрированных данных может содержать сведения о запрошенном доступе.

> [AZURE.NOTE] Каждый пользователь **каталога данных Azure** может добавлять собственные теги и описания для атрибутов и ресурсов данных. **Каталог данных Azure** будет отслеживать значение и источник каждой заметки, отображая имя ее автора и дату добавления. Такой краудсорсинговый подход к метаданным гарантирует, что любой пользователь, заинтересованный в этих данных и их использовании, сможет поделиться своим мнением и ресурсами с сообществом пользователей в целом.

## Изучение, обнаружение и осмысление

Цель регистрации и дополнения источников данных в **каталоге данных Azure** — предоставление возможности обнаружения, понимания и использования этих данных всеми сотрудниками предприятия. Портал **каталога данных Azure** — основное средство реализации этой задачи.

Портал **каталога данных Azure** предоставляет два основных механизма просмотра и обнаружения данных: поиск и фильтрацию.

Чтобы выполнить поиск источников данных в **каталоге данных Azure**, просто введите условие поиска в соответствующее поле на портале **каталога данных Azure**. Портал отобразит плитки для каждого зарегистрированного источника данных, соответствующего условию поиска; каждая плитка будет содержать имя, описание и теги, назначенные этому источнику данных, и другую важную информацию.

Чтобы отфильтровать содержимое **каталога данных Azure**, просто выберите один или несколько фильтров, представленных на портале **каталога данных Azure**. На портале будут отображаться только те плитки, которые совпадают с указанными критериями фильтрации. Вы можете отфильтровать источники данных, не выполняя поиска, либо можете отфильтровать результаты поиска.

Чтобы просмотреть более полную информацию об источнике данных и понять, подходит ли она для поставленной задачи, просто щелкните плитку источника данных. Откроется панель свойств, на которой будут представлены все метаданные.

В верхней области панели свойств будут находиться дополнительные кнопки:

1.	Предварительный просмотр. Если во время регистрации источника данных была выбрана опция предварительного просмотра, то при нажатии на эту кнопку отобразится статический набор предварительных записей из источника данных.
2.	Нажмите эту кнопку, чтобы увидеть схему источника данных, включая имена столбцов и типы данных, а также все метаданные уровня столбца в **каталоге данных Azure**.

> [AZURE.NOTE] Помните, что интерфейс **обнаружения** может быть точкой входа в интерфейс **дополнения**, а не только **использования**. Используемый в **каталоге данных Azure** подход, опирающийся на совместную работу над метаданными, означает, что любой пользователь, который обнаружил источник зарегистрированных данных, может не только воспользоваться найденными данными, но и поделиться своим мнением об этой информации.

## Удаление метаданных источника данных

Иногда после регистрации источника данных требуется удалить ссылку на источник данных из **каталога данных Azure**. Это может быть обусловлено изменением бизнес-требований или выводом источника из эксплуатации. Независимо от причины **каталог данных Azure** позволяет легко удалять источники данных. Для этого просто выберите источники для удаления, и они больше не будут появляться при обнаружении и использовании.

> [AZURE.IMPORTANT] При удалении источника данных из **каталога данных Azure** удаляются только метаданные, хранящиеся в службе **каталога данных Azure**. Исходный источник данных никак не затрагивается.

## Использование источников данных

Конечной целью обнаружения данных является поиск необходимых сведений и их использование в средстве данных по своему усмотрению. Интерфейс использования данных в каталоге данных Azure позволяет сделать это двумя способами.

1.	Чтобы получить доступ к клиентским приложениям, которые поддерживаются непосредственно **каталогом данных Azure**, пользователи могут щелкнуть меню **Открыть в** на плитке источника данных на портале. Затем клиентское приложение запускается с подключением к выбранному источнику данных.
2.	Для всех клиентских приложений пользователи могут использовать сведения о подключении, которые отображаются на панели свойств для выбранного источника данных. Эта информация включает в себя все сведения (например, имя сервера, имя базы данных и имя объекта), необходимые для подключения к данным, и их можно скопировать в интерфейс подключения клиентского средства. Если были предоставлены сведения о запросе доступа для источника данных, эта информация отобразится рядом со сведениями о подключении.

# Что такое каталог данных Azure?

**Каталог данных Azure** (Майкрософт) является полностью управляемой облачной службой, выполняющей функции систем регистрации и обнаружения для корпоративных источников данных. **Каталог данных Azure** предоставляет возможности, позволяющие любому пользователю — от специалиста по системному анализу или обработке данных до разработчика — регистрировать, обнаруживать, воспринимать и использовать источники данных.

## Описание проблемы — целесообразность и обзор

Как правило, обнаружение корпоративных источников данных является неотъемлемым процессом, основанным на общей информации, известной ограниченной группе лиц внутри компании. Это создает множество проблем для компаний, желающих извлечь максимум пользы из своих информационных ресурсов.

-	Пользователи не знают о существовании источников данных, пока не столкнутся с ними в рамках другого процесса; не существует централизованного местоположения, где регистрируются эти источники данных.
-	Пока пользователь не знает, где располагается источник данных, он не может подключиться к данным с помощью клиентского приложения; для работы с данными ему необходимо знать строку или путь подключения.
-	Пока пользователь не знает, где располагается документация к источнику данных, он не может разобраться в назначении этих данных. Источники данных и документация располагаются в разных местах и используются в различных интерфейсах.
-	Если у пользователя есть вопросы по поводу информационного ресурса, он должен отыскать специалиста или их группу, ответственных за эти данные, и подключить их к работе вне сети; отсутствует явная связь между данными и стороной со специалистами, заинтересованной в их использовании.
 
Пока потребители данных сталкиваются с перечисленными трудностями, пользователи, ответственные за создание и обслуживание информационных ресурсов, сталкиваются с собственными проблемами.

-	Комментирование источников данных с помощью описательных метаданных зачастую оказывается напрасным; клиентские приложения обычно пропускают описания, хранящиеся в источнике данных. 
-	Создание документации для источников данных часто является напрасным занятием. Ответственность за постоянную синхронизацию документации с источником данных приводит к тому, что пользователи не доверяют документации, так как она часто воспринимается устаревшей.

Создание и обслуживание документации для источника данных являются сложной и трудоемкой задачей. А обеспечение доступности этой документации для всех пользователей данного источника данных — еще более сложной.

Вместе эти трудности образуют серьезный барьер для компаний, желающих способствовать и содействовать использованию и восприятию данных предприятия.

## Описание службы

**Каталог данных Azure** предназначен для решения этих проблем и позволяет предприятиям извлечь максимальную пользу из своих существующих информационных ресурсов за счет того, что пользователям проще обнаружить и понять данные, которыми они управляют.

**Каталог данных Azure** предоставляет облачную службу, в которой может быть зарегистрирован источник данных. Данные остаются на прежнем месте, однако копия метаданных и ссылка на местоположение источника данных будут добавлены в **каталог данных Azure**. Кроме того, для облегчения обнаружения каждого источника данных с помощью функции поиска эти метаданные индексируются.

После регистрации источника данных его метаданные могут быть дополнены пользователем, выполнившим регистрацию, либо другим пользователем организации. Любой пользователь может добавить комментарий к источнику данных, предоставляя описания, теги и другие метаданные, такие как документация. Эти описательные метаданные дополняют структурные метаданные (например, имена столбцов и типы данных), зарегистрированные из источника данных, для упрощения его обнаружения и понимания.

Основной целью регистрации источников являются обнаружение, понимание и использование источников данных. Когда корпоративным пользователям потребуются данные для работы (которой может быть бизнес-аналитика, разработка приложений, обработка и анализ данных или любая другая задача, требующая корректных данных), они могут использовать интерфейс обнаружения **каталога данных Azure**, чтобы быстро найти соответствующие требованиям данные, понять их, чтобы оценить целевую пригодность, и использовать, открыв источник данных в своем средстве выбора.

## Регистрация источников данных

Регистрация источника данных выполняется с помощью средства регистрации источников данных для **каталога данных Azure**. Это приложение можно загрузить с портала **каталога данных Azure**.

Процесс регистрации состоит из трех основных этапов:
 
1.	Подключение к источнику данных — пользователь указывает местоположение источника данных (например, экземпляра SQL Server) и учетные данные для подключения к нему.
2.	Выбор объектов для регистрации — в указанном расположении пользователь выбирает объекты, которые необходимо зарегистрировать с помощью **каталога данных Azure**. Это может быть полный набор таблиц во всех базах данных на сервере или прямо выбранное подмножество таблиц и представлений.
3.	Выполнение регистрации — пользователь завершает процесс, и средство регистрации источника данных извлекает структурные метаданные из источника данных и отправляет их в облачную службу **каталога данных Azure**.
	
> [AZURE.NOTE]В настоящее время для предварительного просмотра **каталог данных Azure** поддерживает следующие типы источников данных и ресурсов:
 
- таблица SQL Server;
- представление SQL Server;
- таблица базы данных Oracle;
- представление базы данных Oracle;
- многомерная аналитика служб SQL Server Analysis Services;
- составные измерения служб SQL Server Analysis Services;
- многомерный ключевой показатель эффективности служб SQL Server Analysis Services; 
- таблица служб SQL Server Analysis Services;
- отчет для служб SQL Server Reporting Services.

Во время предварительного просмотра **каталога данных Azure** будут добавлены дополнительные типы источников данных и ресурсов.

> [AZURE.IMPORTANT]При регистрации источника данных в **каталоге данных Azure** данные источника не копируются, за исключением случаев, когда в средстве регистрации источников данных установлен флажок "Включить предварительный просмотр". Копируются метаданные источника данных, а не сами данные. Примерами метаданных являются имена таблиц и другие объекты источника данных, а также имена и типы данных столбцов и другие атрибуты источника. Кроме того, метаданные включают местоположение источника данных, что позволяет пользователям, выполняющим поиск источников данных с помощью **каталога данных Azure**, подключаться к этим источникам. Если выбрать параметр "Включить предварительный просмотр", то средство регистрации источника данных также скопирует в **каталог данных Azure** небольшой набор записей, которые будут отображаться для пользователей, выполняющих поиск источника данных на портале **каталога данных Azure**.

## Дополнение метаданных источника данных

После завершения регистрации источники данных можно найти и использовать, но истинное значение **каталога данных Azure** поступает из описательных метаданных предприятия по тому же интерфейсу, что и структурные метаданные, полученные из источника данных. Эти дополнительные метаданные обеспечивают два существенных преимущества:
 
-	Зарегистрированные источники данных легче обнаружить. Пользовательские метаданные добавляются в индекс поиска **каталога данных Azure**. Это позволяет пользователям находить данные с помощью терминов и основных понятий, которые могут отсутствовать в исходном источнике данных. Например, если таблице базы данных, которая содержит данные клиента и имеет имя "tbl_c45", предоставить понятное имя "Клиент", то пользователям будет легче найти данные клиентов. Аналогично при предоставлении описания, которое включает в себя имена отчетов, панелей мониторинга или процессов, использующих эти данные, облегчается поиск источника данных для пользователей, которые используют нижестоящие артефакты в качестве условий поиска.
-	Зарегистрированные источники данных становятся более понятными после обнаружения. Пользовательские метаданные отображаются для всех пользователей **каталога данных Azure**, просматривающих источник данных с комментариями, что обеспечивает дополнительный контекст и информацию. Большинство источников данных обычно не включают осмысленные описания или документацию, а те, что включают, часто ориентированы на технических администраторов и разработчиков баз данных. За счет дополнения источников данных в **каталоге данных Azure** с помощью описаний и тегов, соответствующих аудитории, пользователи помогают друг другу находить, понимать и использовать эти сведения.
  
> [AZURE.NOTE]Каждый пользователь **каталога данных Azure** может добавить собственные метки и описания для ресурсов данных и столбцов. **Каталог данных Azure** будет отслеживать значение и источник каждого комментария и отображать пользователя и дату добавления комментария. Такой краудсорсинговый подход к метаданным гарантирует, что любой пользователь, заинтересованный в этих данных и их использовании, сможет поделиться своим мнением и ресурсами с сообществом пользователей в целом.

## Изучение, обнаружение и осмысление

Цель регистрации и дополнения источников данных в **каталоге данных Azure** заключается в возможности обнаружения, осмысления и использования этих данных пользователями всего предприятия. Портал **каталога данных Azure** является основным средством для выполнения этого процесса.

Портал **каталога данных Azure** предоставляет два механизма для просмотра и обнаружения данных: поиск и фильтрацию.

Для выполнения поиска источников данных в **каталоге данных Azure** просто введите условие поиска в соответствующее поле на портале **каталога данных Azure**. Портал отобразит плитки для каждого зарегистрированного источника данных, соответствующего условию поиска; каждая плитка будет содержать имя, описание и теги, назначенные этому источнику данных, и другую важную информацию.

Чтобы отфильтровать содержимое **каталога данных Azure**, просто выберите один или несколько фильтров, представленных на портале **каталога данных Azure**. На портале будут отображаться только те плитки, которые совпадают с указанными критериями фильтрации. Вы можете отфильтровать источники данных, не выполняя поиска, либо можете отфильтровать результаты поиска.

Чтобы просмотреть более полную информацию об источнике данных и понять, подходит ли она для поставленной задачи, просто щелкните плитку источника данных. Откроется панель свойств, на которой будут представлены все метаданные.

В верхней области панели свойств будут находиться дополнительные кнопки:
 
1.	Предварительный просмотр. Если во время регистрации источника данных была выбрана опция предварительного просмотра, то при нажатии на эту кнопку отобразится статический набор предварительных записей из источника данных.
2.	Схема. Выберите эту кнопку, чтобы увидеть схему источника данных, включая имена столбцов и типы данных, а также метаданные любого уровня столбца в **каталоге данных Azure**.

> [AZURE.NOTE]Помните, что интерфейс **обнаружения** может быть точкой входа в интерфейс **дополнения**, а не только **использования**. Краудсорсинговый подход, который предъявляет **каталог данных Azure**, означает, что любой пользователь, который обнаружил источник зарегистрированных данных, может поделиться своим мнением о данных, а не просто воспользоваться найденными данными.

## Удаление метаданных источника данных

Иногда после регистрации источника данных бывает необходимо удалить ссылку на источник данных из **каталога данных Azure**. Это может быть обусловлено изменением бизнес-требований или выводом источника из эксплуатации. Независимо от причины, **каталог данных Azure** позволяет с легкостью удалять источники данных. Для этого просто выберите источники для удаления, и они больше не будут появляться при поиске и использовании.

> [AZURE.IMPORTANT]При удалении источника данных из **каталога данных Azure** удаляются только метаданные, хранящиеся в службе **каталога данных Azure**. Исходный источник данных никак не затрагивается.

## Использование источников данных

Конечной целью обнаружения данных является поиск необходимых сведений и их использование в средстве данных по своему усмотрению. Интерфейс использования данных в каталоге данных Azure позволяет сделать это двумя способами.
 
1.	Для клиентских приложений, которые поддерживаются непосредственно **каталогом данных Azure**, пользователи могут щелкнуть меню **Подключение** на плитке источника данных на портале. Затем клиентское приложение запускается с подключением к выбранному источнику данных.
2.	Для всех клиентских приложений пользователи могут использовать сведения о подключении, которые отображаются на панели свойств для выбранного источника данных. Эта информация включает в себя все сведения (например, имя сервера, имя базы данных и имя объекта), необходимые для подключения к данным, и их можно скопировать в интерфейс подключения клиентского средства.
 
> [AZURE.NOTE]Для закрытого просмотра каталога данных Azure в меню **Подключение** будет непосредственно поддерживаться и доступно только приложение Microsoft Excel.

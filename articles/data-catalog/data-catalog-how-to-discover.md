# Обнаружение источников данных

## Введение
**Каталог данных Microsoft Azure** — это полностью управляемая облачная служба, выполняющая функции систем регистрации и обнаружения корпоративных источников данных. Проще говоря, **каталог данных Azure** помогает пользователям обнаруживать, оценивать и использовать источники данных, что, в свою очередь, повышает для организаций ценность их существующей информации. После того как источник данных зарегистрирован в **каталоге данных Azure**, служба индексирует его метаданные, чтобы пользователи могли легко находить необходимые данные.

## Поиск и фильтрация

Обнаружение информации в **каталоге данных Azure** обеспечивается двумя основными механизмами: поиском и фильтрацией.

Поиск должен быть одновременно интуитивно понятным и эффективным. Это значит, что условия поиска по умолчанию должны соответствовать всем свойствам в каталоге, включая пользовательские заметки.

Фильтрация дополняет поиск. Пользователи могут выбирать определенные характеристики (эксперты, тип источника данных, тип объекта и теги), чтобы просматривать только соответствующие ресурсы данных, а также ограничивать результаты поиска соответствующими ресурсами.

Используя сочетание поиска и фильтрации, пользователи могут быстро проходить по источникам данных, зарегистрированных в **каталоге данных Azure**, для поиска необходимых источников данных.

## Синтаксис поиска

Хотя поиск с использованием свободного текста по умолчанию является простым и интуитивно понятным, пользователи также могут использовать синтаксис поиска **каталога данных Azure** для уточнения результатов поиска. Функция поиска **каталога данных Azure** поддерживает следующие методы.

| Метод | Использование | Пример |
|---------------------------|-----------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------|
| Обычный поиск | Обычный поиск с использованием одного или нескольких условий поиска. Результаты представлены ресурсами, которые соответствуют любому свойству с одним или несколькими указанными терминами. | данные о продажах |
| Ограничение по свойству | Возврат только тех источников данных, в которых искомое слово совпадает с указанным свойством. | имя:финансы |
| Логические операторы | Расширение или сужение поиска с помощью логических операций | финансы NOT организации |
| Обособление с помощью скобок | Скобки используются для обособления частей запросов, позволяя добиться логической изоляции. Это особенно удобно при использовании логических операторов. | имя:финансы AND (теги:Q1 OR теги:Q2)Операторы сравнения |
| Операторы сравнения | Операторы сравнения, в отличие от операторов равенства, используются для свойств с числовым типом данных и типом данных даты. | время\_создания>"11/05/2014" |

Дополнительные сведения о поиске в **каталоге данных Azure** см. на странице [https://msdn.microsoft.com/library/azure/mt267594.aspx](https://msdn.microsoft.com/library/azure/mt267594.aspx).

## Выделение совпадений
При просмотре результатов поиска будут выделены все отображаемые свойства, соответствующие указанным условиям поиска: имя ресурса данных, описание и теги. Эти выделенные свойства помогают понять, почему среди результатов поиска появляется тот или иной ресурс данных.

> [AZURE.NOTE]При необходимости пользователи могут отключать выделение совпадений с помощью параметра «Выделение» на портале **каталога данных Azure**.

При просмотре результатов поиска (даже при включенной функции выделения совпадений) не всегда понятно, почему включен в результаты тот или иной ресурс данных. Поскольку поиск по умолчанию выполняется по всем свойствам, тот или иной ресурс данных может быть включен в результаты из-за совпадения свойств на уровне столбца. Кроме того, заметки к зарегистрированному ресурсу данных могут создаваться несколькими пользователями, использующими собственные теги и описания. Это еще одна причина, по которой не все метаданные могут отображаться в списке результатов поиска.

В представлении плиток по умолчанию каждая плитка, отображенная в результатах поиска, будет включать значок «Просмотр соответствий поисковому запросу». Это позволяет пользователю быстро просматривать все совпадения и их расположение, переходя к ним при необходимости.

## Сводка
Регистрация источников данных в **каталоге данных Azure** позволяет быстрее обнаруживать и оценивать эти источники благодаря тому, что из источника в службу каталога копируются структурные и описательные метаданные. Зарегистрированный источник данных пользователи смогут обнаружить на портале **каталога данных Azure** с помощью функций фильтрации и поиска.

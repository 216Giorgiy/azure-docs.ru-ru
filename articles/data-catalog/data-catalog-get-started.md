# Начало работы с каталогом данных Azure

Эта статья представляет собой полный учебник, включающий сценарии и возможности общедоступной предварительной версии каталога данных Azure. После регистрации для предварительной версии вы можете выполнить следующие действия для создания каталога данных, а также регистрировать, добавлять примечания и обнаруживать источники данных.

## Предварительные требования для учебника

Перед началом работы с этим учебником необходимо иметь следующее.

-	**Подписка Azure ** — если ее нет, можно создать бесплатную пробную учетную запись всего за несколько минут. Дополнительные сведения см. в статье [Бесплатная пробная версия](https://azure.microsoft.com/pricing/free-trial/).
-	**Azure Active Directory** — каталог данных Azure использует [Azure Active Directory](https://azure.microsoft.com/services/active-directory/) для управления доступом и идентификацией.
-	**Источники данных** — каталог данных Azure предоставляет возможности для обнаружения источника данных, и для прохождения данного учебника необходимо иметь доступ к одному или нескольким источникам данных. Учебник написан с использованием образцов баз данных Adventure Works, но можно использовать любой поддерживаемый источник данных, если вы предпочитаете работать с данными, которые вам знакомы и относятся к вашей роли.

## Упражнение 1. Установка образца базы данных Adventure Works

В этом упражнении мы установим образец Adventure Works для SQL Server Database Engine и SQL Server Analysis Services Multidimensional. Эти образцы используются в последующих упражнениях.

> [AZURE.NOTE] Это упражнение является необязательным. Оставшиеся упражнения в учебнике записываются для ссылки на образцы баз данных Adventure Works, но можно пропустить это упражнение и вместо этого работать с источниками данных. Ниже приведены действия по установке Adventure Works.

### Установка баз данных Adventure Works 2014 OLTP и хранилища данных

База данных Adventure Works OLTP поддерживает сценарии обработки транзакций в стандартной сети для вымышленного производителя велосипедов (компания Adventure Works Cycles), включая производство, продажи и закупки. База данных Adventure Works DW показывает, как построить хранилище данных.

Базы данных находятся на сайте [CodePlex.com](http://msftdbprodsamples.codeplex.com/). Их можно установить, выполнив действия, описанные в [файле сведений для образцов баз данных Adventure Works 2014](https://msftdbprodsamples.codeplex.com/downloads/get/880669).

В этом упражнении мы установим образцы баз данных Adventure Works, используемые в остальных упражнениях. Если вы хотите пропустить это упражнение и использовать источники данных предприятия, пожалуйста, приготовьтесь запоминать имена, теги и другие метаданные.

## Упражнение 2. Регистрация источников данных

В этом упражнении мы используем средство регистрации каталога данных Azure для регистрации источников данных в каталоге. Регистрация — это процесс извлечения ключевых метаданных структуры, таких как имена, типы и расположения, из источника данных и ресурсов, содержащихся в нем, и копирования этих метаданных в каталог. Источники данных и их данные остаются на своем месте, а каталог использует метаданные, чтобы упростить их обнаружение и понимание.

### Ниже описана процедура регистрации источников данных.

1.	Войдите на портал каталога данных Azure.

    ![register1][1]

2.	Прокрутите вниз и выберите **Опубликовать данные**.

    ![register2][2]
3.	Щелкните **Запуск приложения**.
4.	На странице **Приветствие** щелкните **Вход** и введите учетные данные.
5.	На странице **Каталог данных Microsoft Azure** щелкните **SQL Server**.

    ![register3][3]
6.	Введите ваше **Имя сервера** и нажмите кнопку **ПОДКЛЮЧЕНИЕ**.
7.	Следующая страница — страница регистрации метаданных вашего источника данных. В этом примере мы регистрируем объекты **Product** из пространства имен AdventureWorks Production. Вот как это сделать.

    а. В дереве иерархии выберите **Production**.

    b. Нажмите CTRL и щелкните Product, ProductCategory, ProductDescription и ProductPhoto.

    ![register4][4]

    c. Щелкните стрелку перемещения выбранных элементов (**>**). Этим вы переместите все объекты Product в список **для регистрации**.

    ![register5][5]

    г) **Необязательно**. Вы можете **включить предварительный просмотр** и (или) **добавить специалиста источника данных**.

    д. В поле **Добавить теги** введите описание, фото. При этом теги поиска будут добавлены в эти ресурсы данных. Теги — отличный способ помочь пользователям найти зарегистрированный источник данных.

    Е. Щелкните **Зарегистрировать**. Каталог данных Azure регистрирует выбранные объекты. В этом упражнении регистрируются выбранные объекты Adventure Works.

    ![register6][6]

    ж. Для просмотра зарегистрированных объектов источника данных щелкните **Просмотр портала**. На портале каталога данных Azure можно просмотреть объекты источника данных в представлении **Плитки** или **Список**.

    ![register7][7]

В этом упражнении вы зарегистрировали объекты из образца базы данных Adventure Works, чтобы пользователи в организации могли легко обнаружить их. В следующем упражнении вы узнаете, как находить зарегистрированные ресурсы данных.

## Упражнение 3. Обнаружение зарегистрированных ресурсов данных

В этом упражнении мы будем использовать портал каталога данных Azure для обнаружения зарегистрированных ресурсов данных и просмотра их метаданных. Каталог данных Azure предоставляет несколько средств для обнаружения ресурсов данных, включая простой поиск по ключевым словам, интерактивные фильтры и синтаксис расширенного поиска для пользователей power.

### Вот как находить зарегистрированные ресурсы данных

**Каталог данных Azure** оснащен инструментом поиска с простым, но эффективным синтаксисом, который позволяет легко создавать запросы для получения нужных пользователям данных. Дополнительные сведения о **каталоге данных Azure** см. в справочнике по синтаксису поиска.

**Каталог данных Azure** имеет следующие возможности для поиска.

- Поиск по ключевому слову
- Фильтр
- Расширенный поиск

Также можно уточнить ресурсы данных для просмотра. **Каталог данных Azure** имеет следующие параметры представления.

- Свойства
- Столбцы
- Предварительный просмотр

В этом примере производится поиск по ключевому слову. Поиск по **каталогу данных Azure** может осуществляться несколькими методами. В этом примере будет использоваться поисковой запрос с **группированием**.

В этом примере мы выполним поиск ресурсов данных **Группирование**, где имя равно продукту, и теги равны иллюстрации, или теги равны фото.

1.	Войдите на портал **каталога данных Azure**.
2.	Щелкните **Обнаружение**.
3.	В поле **Поиск** введите запрос с **группированием**: (tags:description OR tags:photo).
4.	Щелкните значок поиска или нажмите клавишу ВВОД. **Каталог данных Azure** отобразит ресурсы данных, соответствующие этому поисковому запросу.

    ![search][8]

В этом упражнении мы использовали портал **каталога данных Azure** для обнаружения и просмотра ресурсов данных, зарегистрированных в каталоге.

## Упражнение 4. Добавление примечаний к зарегистрированным источникам данных

В этом упражнении мы используем портал **каталога данных Azure** для создания примечаний к ресурсам данных, которые уже зарегистрированы в каталоге. Предоставленные вами примечания будут дополнять и улучшать структурные метаданные, полученные из источника данных во время регистрации, и упростят обнаружение и понимание данных. Каждый пользователь **каталога данных** может внести собственные примечания, что легко позволяет делиться информацией всем, кто использует эти данные.

### Вот как создавать примечания к ресурсам данных

1.	Войдите на портал **каталога данных Azure**.
2.	Щелкните **Обнаружение**.
3.	Выберите один или несколько **ресурсов данных**. Например, выберите **ProductPhoto** и введите "Фотографии товара для рекламных материалов".
4.	Введите **Описание**, которое поможет другим понять, как и для чего можно использовать выбранные ресурсы данных. Например, введите «Изображения продуктов». Также можно добавить дополнительные теги и просмотреть столбцы.
5.	Теперь можно попробовать поиск и фильтрацию, чтобы обнаружить ресурсы данных с помощью описательных метаданных, добавленных в каталог.

    ![annotate][9]

В этом упражнении мы добавили описательные сведения к зарегистрированным ресурсам данных, чтобы пользователи каталога могли обнаруживать источник данных с помощью понятных им терминов.

## Упражнение 5. Сбор метаданных

В этом упражнении мы будем работать с другим пользователем, чтобы добавить метаданные к ресурсам данных в каталоге. Коллективный подход каталога данных Azure к примечаниям позволяет всем пользователям добавлять теги, описания и другие метаданные, так что любой пользователь, у которого есть примечание о ресурсе данных и его использовании, может записать свое примечание и предоставить к нему доступ другим пользователям.

> [AZURE.NOTE] Если нет другого пользователя для работы с учебником, не волнуйтесь! Любой пользователь, который получает доступ к каталогу данных, может при желании добавить свое примечание. Такой коллективный подход к метаданным обеспечивает рост содержимого и разнообразия метаданных каталога.

### Вот как можно собрать метаданные о ресурсах данных

Попросите коллегу повторить описанное выше упражнение **Добавление примечаний к зарегистрированным источникам данных**. После того как ваш коллега добавит описание к ресурсу данных, например ProductPhoto, вы увидите несколько примечаний.


![crowdsource][13]

В этом упражнении мы исследовали возможности каталога данных Azure для сбора метаданных, где любой пользователь каталога может добавлять примечания к ресурсам данных, которые он находит.


## Упражнение 6. Подключение к источникам данных

В этом упражнении мы используем портал **каталога данных Azure** для подключения к источникам данных с помощью Microsoft Excel.


> [AZURE.NOTE] Важно помнить, что **каталог данных Azure** не предоставляет пользователям доступ к исходному источнику данных — он просто облегчает пользователям поиск и понимание информации. При подключении пользователей к источнику данных выбранное ими клиентское приложение будет использовать их учетные данные Windows или запросит учетные данные при необходимости. Если раньше у пользователя не было доступа к источнику данных, ему нужно будет предоставить доступ, прежде чем он сможет подключиться.

### Ниже описана процедура подключения к источнику данных из Excel.

1.	Войдите на портал **каталога данных Azure**.
2.	Щелкните **Обнаружение**.
3.	Выберите ресурс данных. В этом примере выберите ProductCategory.
4.	Выберите **Открыть в** > **Excel**.

    ![connect1][10]
5.	В окне **Уведомление безопасности Microsoft Excel** нажмите кнопку **Включить**.
6.	Откройте файл **ProductCategory.odc**.
7.	Источник данных открыт в Excel.

    ![connect2][11]

В этом упражнении мы подключились к источникам данных, обнаруженным с помощью каталога данных Azure. Портал **каталога данных Azure** позволяет пользователям подключаться напрямую с помощью клиентских приложений, интегрированных в меню **Открыть в**, а также подключаться с использованием любого выбранного приложения, используя сведения о расположении подключения, указанные в метаданных ресурса.

## Упражнение 7. Удаление метаданных источника данных

В этом упражнении мы используем портал **каталога данных Azure** для удаления предварительных данных из зарегистрированных ресурсов данных и удаления ресурсов данных из каталога.

> [AZURE.NOTE] Каталог по умолчанию дает любому пользователю возможность зарегистрировать любой источник данных и удалить любой зарегистрированный ресурс данных. Функции управления, включенные в **стандартный выпуск каталога данных Azure**, предоставляют дополнительные возможности для смены владельца ресурсов и ограничения круга пользователей, которые могут обнаруживать или удалять ресурсы.

В **каталоге данных Azure** вы можете удалить предварительные данные, удалить отдельный ресурс или удалить несколько ресурсов.

### Вот как можно удалить несколько ресурсов данных

1.	Войдите на портал **каталога данных Azure**.
2.	Щелкните **Обнаружение**.
3.	Выберите один или несколько ресурсов данных.
4.	Нажмите кнопку **Delete** (Удалить).

В этом упражнении мы удалили зарегистрированные ресурсы данных из каталога.

## Упражнение 8. Управление зарегистрированными источниками данных

В этом упражнении вы воспользуетесь функциями управления **каталога данных Azure**, чтобы стать владельцем ресурсов данных, а также ограничить круг пользователей, которые могут обнаруживать эти ресурсы данных или управлять ими.

Примечание. Возможности управления, описанные в этом упражнении, доступны только в стандартном выпуске каталога данных Azure, а не в бесплатном выпуске. В **каталоге данных Azure** можно стать владельцем ресурсов данных, добавить совладельцев ресурсов данных или задать ограничения видимости ресурсов данных.

### Вот как стать владельцем ресурсов данных и ограничить их видимость

1.	Войдите на портал **каталога данных Azure**.
2.	Щелкните **Обнаружение**.
3.	Выберите один или несколько ресурсов данных.
4.	На панели **Свойства** в разделе **Управление** щелкните **Стать владельцем**.
5.	Чтобы ограничить видимость, щелкните пункт **Владельцы и эти пользователи**.

    ![ownership][12]

В этом упражнении мы изучили возможности управления каталогом и ограничение видимости выбранных ресурсов данных.

## Сводка

В этом учебнике мы изучили основные возможности предварительной версии **каталога данных Azure**, такие как регистрация, добавление примечаний, обнаружение и управление источниками данных предприятия. Теперь, после завершения учебника, пора начать работу. Вы можете начать прямо сегодня, зарегистрировав источники данных, используемые вами и вашей командой, и пригласив коллег использовать каталог.

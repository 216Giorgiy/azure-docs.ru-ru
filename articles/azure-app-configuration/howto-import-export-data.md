# <a name="import-or-export-configuration-data"></a>Импорт и экспорт данных конфигурации

Azure поддерживают данные конфигурации приложения операции импорта и экспорта. Они позволяют работать с данными конфигурации в пакетном режиме, а также обмен данными между конфигурацию приложений хранения и проекту кода. Например вы можете настроить хранилище конфигураций одно приложение для тестирования, а другой — для рабочей и скопируйте параметры приложения между ними посредством файла таким образом, чтобы у вас нет дважды вводить данные.

В этой статье рекомендации по импорту и экспорту данных с конфигурацией приложения.

## <a name="import-data"></a>Импорт данных

Импорт привносит конфигурации хранилища данных в конфигурацию приложения из существующего источника, а не вручную, указав их. Можно использовать функцию импорта для переноса данных в хранилище конфигурации приложения или к объединенным данным из нескольких источников. Конфигурация приложения поддерживает импорт из файла JSON, YAML или свойства.

Можно импортировать либо с помощью [портала Azure](https://aka.ms/azconfig/portal) или [Azure CLI](./scripts/cli-import.md). На портале Azure выполните следующие действия.

1. Перейдите в хранилище конфигурации приложения и нажмите кнопку **импорта и экспорта**.

2. В **импорта** вкладке, выберите **источника службы** и **файл конфигурации**.

3. Выберите **для языка** и **тип файла**.

4. Щелкните **папку** значок и перейдите к файлу для импорта.

    ![Импорт файла](./media/import-file.png)

5. Выберите **разделителя** и при необходимости введите **префикса** для импортированных имена ключей.

6. При необходимости выберите **метка**.

7. Нажмите кнопку **применить** чтобы завершить импорт.

    ![Импорт файла завершено](./media/import-file-complete.png)

## <a name="export-data"></a>Экспорт данных

Экспорт записывает данные конфигурации, хранимые в конфигурации приложения в другое место назначения. Можно использовать функцию экспорта, например, для сохранения данных в хранилище конфигурации приложения в файл, который будет внедрен в коде приложения во время развертывания.

Вы можете экспортировать данные с помощью [портала Azure](https://aka.ms/azconfig/portal) или [Azure CLI](./scripts/cli-export.md). На портале Azure выполните следующие действия.

1. Перейдите в хранилище конфигурации приложения и нажмите кнопку **импорта и экспорта**.

2. В **Экспорт** вкладке, выберите **целевая служба** и **файл конфигурации**.

3. При необходимости введите **префикса** и выберите **метка** и в определенный момент времени для ключей, для экспорта.

4. Выберите **тип файла** и **разделителя**.

5. Нажмите кнопку **применить** для выполнения экспорта.

    ![Экспорт файла завершено](./media/export-file-complete.png)

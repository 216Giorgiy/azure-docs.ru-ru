# <a name="introduction-to-microsoft-azure-log-integration"></a>Введение в службу интеграции журналов Microsoft Azure

Служба интеграции журналов данных Azure позволяет интегрировать необработанные журналы из ресурсов Azure с локальными системами SIEM (Security Information and Event Management) в случае, если соединитель для [Azure Monitor](../monitoring-and-diagnostics/monitoring-get-started.md) еще недоступен от поставщика SIEM.

Предпочтительным методом интеграции журналов Azure является использование соединителя Azure Monitor от поставщика SIEM и выполнение следующих [инструкций](../monitoring-and-diagnostics/monitor-stream-monitoring-data-event-hubs.md). Тем не менее, если поставщик SIEM не предоставляет соединитель для Azure Monitor, в качестве временного решения можно использовать службу "Интеграция журналов данных Azure" (если она поддерживает вашу систему SIEM).

>[!IMPORTANT]
>Если вас интересует сбор данных журналов виртуальной машины, обращайтесь к поставщикам SIEM, так как большинство из них включают его в свое решение. Использование соединителя поставщика SIEM всегда должно быть предпочтительным вариантом.

Служба интеграции журналов Azure собирает события из журналов средства просмотра событий Windows, [журналов действий Azure](../monitoring-and-diagnostics/monitoring-overview-activity-logs.md), [оповещений центра безопасности Azure](../security-center/security-center-intro.md) и [журналов диагностики Azure](../monitoring-and-diagnostics/monitoring-overview-of-diagnostic-logs.md), которые поступают из ресурсов Azure. Благодаря такой интеграции решение SIEM может обеспечить доступ ко всем ресурсам, локальным или облачным, на унифицированной панели мониторинга, что позволяет выполнять статистическую обработку, сопоставление и анализ, а также оповещать о событиях безопасности.

>[!NOTE]
В настоящее время поддерживаются только облако Azure для коммерческих организаций и облако Azure для государственных организаций. Другие облака не поддерживаются.

## <a name="what-logs-can-i-integrate"></a>Какие журналы можно интегрировать?

Azure создает подробные журналы для каждой службы Azure. Эти журналы делятся на три типа.

* **Журналы управления и контроля**, которые предоставляют информацию о таких операциях [Azure Resource Manager](../azure-resource-manager/resource-group-overview.md), как CREATE, UPDATE и DELETE. К этому типу относятся, например, журналы действий Azure.
* **Журналы плоскости данных**, которые предоставляют информацию о событиях, возникающих при использовании ресурса Azure. Примерами журнала этого типа являются каналы средства просмотра событий Windows **System**, **Security** и **Application** на виртуальной машине Windows. Другой пример — ведение журнала диагностики, настраиваемое с помощью Azure Monitor.
* Журнал **Обработанные событий** предоставляет информацию о проанализированных событиях и оповещениях, обработанных от вашего имени. К событиям такого типа относятся оповещения центра безопасности Azure, генерируемые им после обработки и анализа вашей подписки для определения вашей текущей системы безопасности.

Интеграция журналов Azure поддерживает ArcSight, QRadar и Splunk. В любом случае обратитесь к поставщику SIEM, чтобы выяснить, есть ли у него собственный соединитель. Не следует использовать службу интеграции журналов данных Azure, если доступен собственный соединитель.

Если других вариантов нет, можно использовать службу интеграции журналов данных Azure. В следующей таблице приведены наши рекомендации.

|**SIEM** | **Клиент уже использует интегратор журналов** | **Клиент, изучающий варианты интеграции SIEM**|
|---------|--------------------------|-------------------------------------------|
|**SPLUNK** | Начните переход на [надстройку Azure Monitor для Splunk](https://splunkbase.splunk.com/app/3534/) | Используйте [соединитель SPLUNK](https://splunkbase.splunk.com/app/3534/) |
|**IBM QRADAR** | Сведения о переходе на соединитель QRadar и начале его использования приведены в конце статьи, расположенной по адресу http://aka.ms/azmoneventhub | Используйте соединитель QRadar, который описан в конце статьи, расположенной по адресу http://aka.ms/azmoneventhub  |
|**ARCSIGHT** | Продолжайте использовать интегратор журналов до тех пор, пока не будет доступен соединитель, а затем перейдите на решение на основе соединителя.  | В качестве альтернативы рассмотрите возможность использования Azure Log Analytics. Не подключайтесь к службе интеграции журналов данных Azure, если вы не захотите выполнить процесс миграции, когда соединитель станет доступным. |

>[!NOTE]
>Хотя служба интеграции журналов Azure — бесплатное решение, за хранение файлов журналов в службе хранилища Azure взимается плата.

Если вам нужна помощь, можно создать [запрос в службу поддержки](../azure-supportability/how-to-create-azure-support-request.md). Для этого укажите в запросе службу **Интеграции журналов**.

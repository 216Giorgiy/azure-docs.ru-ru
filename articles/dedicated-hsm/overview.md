# <a name="what-is-azure-dedicated-hsm"></a>Что такое служба выделенных устройств HSM Azure?

Служба выделенных устройств HSM — это решение Azure, которое обеспечивает хранение криптографических ключей в Azure. Служба выделенных устройств HSM соответствует самым строгим требованиям безопасности. Это идеальный вариант для клиентов, которым требуется аппаратный модуль безопасности, отвечающий требованиям стандарта FIPS 140-2 уровня 3 и обеспечивающий полный и исключительный контроль над устройством HSM. Устройства HSM глобально развертываются в нескольких регионах Azure. Модули можно без труда подготовить к работе как пару устройств и настроить для обеспечения высокого уровня доступности. Кроме того, развернув устройства HSM в нескольких регионах, можно выполнять отработку отказа на уровне региона. В корпорации Майкрософт для службы выделенных устройств HSM используются аппаратные модули [SafeNet Luna Network HSM 7 (модель A790)](https://safenet.gemalto.com/data-encryption/hardware-security-modules-hsms/safenet-network-hsm/) производства Gemalto. Это устройство обеспечивает высочайшую производительность и широкие возможности криптографической интеграции. Аппаратные модули безопасности напрямую подключаются к виртуальной сети клиента. Кроме того, к ним можно получить доступ из локальных приложений и средств управления. Для этого нужно установить VPN-подключение типа "точка-сеть" или "сеть-сеть". Клиентам предоставляется программное обеспечение и документация для настройки устройств HSM и управления ими на портале поддержки Gemalto.

## <a name="why-use-azure-dedicated-hsm"></a>Назначение выделенного устройства HSM Azure

### <a name="fips-140-2-level-3-compliance"></a>Соответствие требованиям стандарта FIPS 140-2 уровня 3

Многие организации подчиняются строгим отраслевым нормативным требованиям, в которых предписано хранить криптографические ключи в соответствии со стандартом [FIPS 140-2 уровня 3](https://csrc.nist.gov/publications/detail/fips/140/2/final). Служба Azure Key Vault (многопользовательская) сейчас обеспечивает только соответствие стандарту FIPS 140-2 уровня 2. Служба выделенных устройств HSM Azure обеспечивает соответствие текущим требованиям финансовых и правительственных учреждений, а также других организаций, в которых требуется соблюдать FIPS 140-2 уровня 3.

### <a name="single-tenant-devices"></a>Устройства с единственным арендатором

Многие из клиентов должны подчиняться требованию о единственном арендаторе хранилища криптографических ключей. Служба выделенных устройств HSM Azure позволяет подготовить физическое устройство в одном из глобально распределенных центров обработки данных корпорации Майкрософт. После того как устройство будет подготовлено для клиента, доступ к устройству будет только у этого клиента.

### <a name="full-administrative-control"></a>Полный административный контроль

Помимо устройств с единственным арендатором, многим клиентам требуется полный административный контроль над устройством и единственная точка административного доступа к нему. После того как устройство будет подготовлено для клиента, доступ к устройству будет только у этого клиента на административном уровне или уровне приложений. После того как клиент впервые подключится к устройству, корпорация Майкрософт не сможет получить к нему административный доступ. Для этого пользователю нужно заменить пароль. С этого момента клиент является по-настоящему единственным арендатором устройства с полным административным контролем и возможностями управлять приложениями. У корпорации Майкрософт есть доступ к устройству на уровне мониторинга для сбора данных телеметрии с помощью подключения через последовательный порт. Отслеживаются такие данные о состоянии оборудования, как температура, характеристики источника питания и работоспособность вентилятора. При необходимости клиент может отключить мониторинг. Но так он не будет получать упреждающие оповещения о работоспособности оборудования от корпорации Майкрософт.

### <a name="high-performance"></a>высокопроизводительная

Модуль производства Gemalto выбран для этой службы, так как он поддерживает множество криптографических алгоритмов, операционных систем и API. При развертывании выбранная модель модуля продемонстрировала превосходную производительность в 10 000 операций в секунду для RSA 2048. Модуль поддерживает 10 разделов, которые можно использовать для отдельных экземпляров приложений. Устройство обеспечивает малую задержку, а также высокую производительность и пропускную способность.

### <a name="unique-cloud-based-offering"></a>Беспрецедентное облачное решение

Руководствуясь особыми требованиями ограниченного круга клиентов, корпорация Майкрософт стала единственным поставщиком облачных служб, предлагающим новым клиентам выделенное устройство HSM, которое соответствует требованиям стандарта FIPS 140-2 уровня 3 и поддерживает интеграцию с облачными и локальными приложениями.

## <a name="is-azure-dedicated-hsm-right-for-you"></a>Требуется ли вам служба выделенных устройств HSM Azure?

Служба выделенных устройств HSM Azure — это специализированное решение для выполнения конкретных задач определенного круга крупных организаций. Поэтому предполагается, что основной части клиентов Azure эта служба не понадобится. Многим клиентам больше подойдет функциональность и стоимость Azure Key Vault. Чтобы помочь вам выбрать оптимальное решение, мы определили ряд критериев.

### <a name="best-fit"></a>Подходит

Служба в первую очередь подходит для сценариев "lift-and-shift", в которых предусмотрен прямой и единоличный доступ к аппаратному модулю безопасности. Примеры приведены ниже.

- Перенос приложений из локальной среды на виртуальные машины Azure.
- Перенос приложений из Amazon AWS EC2 на виртуальные машины Azure, использующих службу HSM Classic в AWS Cloud (Amazon не предлагает эту службу новым клиентам).
- Использование готового программного обеспечения на виртуальных машинах Azure, такого как Apache/Ngnix SSL Offload, Oracle TDE и ADCS.

### <a name="not-a-fit"></a>Не подходит

Облачные службы Майкрософт, поддерживающие шифрование с помощью управляемых пользователем ключей (Azure Information Protection, шифрование дисков Azure, Azure Data Lake Store, служба хранилища Azure, SQL Azure, ключ клиента Office 365) не интегрируются со службой выделенных устройств HSM Azure.

### <a name="it-depends"></a>В зависимости от обстоятельств

Во многих сценариях выбор решения будет зависеть от существующих требований и допустимых компромиссов. Например, требование о соответствии стандарту FIPS 140-2 уровня 3 часто является обязательным. В такой ситуации служба выделенных устройств HSM — единственное возможное решение на сегодня.  Если обязательных требований нет, тогда при выборе между Azure Key Vault и службой выделенных устройств HSM нужно руководствоваться определенным набором требований. Примеры приведены ниже.

- Новый код, выполняемый на виртуальной машине Azure клиента.
- Прозрачное шифрование данных SQL Server на виртуальной машине Azure.
- Шифрование данных в службе хранилища Azure на стороне клиента.
- Постоянное шифрование данных SQL Server и базы данных SQL Azure.

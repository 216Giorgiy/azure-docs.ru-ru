# <a name="transforms-and-jobs"></a>Преобразования и задания

## <a name="overview"></a>Обзор 

В последней версии REST API Служб мультимедиа Azure (версии 3) появился новый ресурс шаблона рабочего процесса для кодирования и/или анализа видеоматериалов, который называется **Преобразование**. **Преобразования** можно использовать для настройки общих задач кодирования или анализа видеоматериалов. Каждое **преобразование** описывает простой рабочий процесс задач для обработки видео- и аудиофайлов. 

Объект **преобразования** является набором инструкций, а **задание** — фактическим запросом к Службам мультимедиа Azure для применения этого **преобразования** к данному видео и аудио содержимому. **Задание** указывает такую информацию, как расположение входного и выходного видео. Вы можете указать расположение видео с помощью: URL-адреса протокола HTTP, URL-адреса SAS или пути к файлам, находящимся на локальном компьютере или в хранилище BLOB-объектов. Вы можете выполнять до 100 преобразований в учетной записи Служб мультимедиа Azure и отправлять задания в эти преобразования. Затем вы можете подписаться на события изменения состояния задания с помощью решения "Уведомления", поддерживающего прямую интеграцию с системой уведомлений службы "Сетка событий Azure". 

Так как API управляется с помощью Azure Resource Manager, вы можете использовать шаблоны Resource Manager для создания и развертывания преобразований в учетной записи Служб мультимедиа. Управление доступом на основе ролей можно также задать на уровне ресурсов в этом API, что позволит заблокировать доступ к определенным ресурсам (например, к преобразованиям).

## <a name="transform-definition"></a>Определение преобразования

В следующей таблице представлены свойства преобразования и их определения.

|ИМЯ|type|ОПИСАНИЕ|
|---|---|---|
|Идентификатор|строка|Полный идентификатор ресурса.|
|name|строка|Имя ресурса.|
|properties.created |строка|Дата и время создания преобразования в формате YYYY-MM-DDThh:mm:ssZ (UTC).|
|properties.description |строка|Подробное описание преобразования (необязательно).|
|properties.lastModified |строка|Дата и время последнего обновления преобразования в формате YYYY-MM-DDThh:mm:ssZ (UTC).|
|properties.outputs |TransformOutput[]|Массив из одного или нескольких наборов выходных данных, созданных преобразованием.|
|Тип|строка|Тип ресурса.|

Полное определение см. в статье [Transforms](https://docs.microsoft.com/rest/api/media/transforms) (Преобразования).

## <a name="job-definition"></a>определение задания;

В следующей таблице представлены свойства задания и их определения.

|ИМЯ|type|ОПИСАНИЕ|
|---|---|---|
|Идентификатор|строка|Полный идентификатор ресурса.|
|name|строка|Имя ресурса.|
|properties.created |строка|Дата и время создания преобразования в формате YYYY-MM-DDThh:mm:ssZ (UTC).|
|properties.description |строка|Подробное описание задания (необязательно).|
|properties.lastModified |строка|Дата и время последнего обновления преобразования в формате YYYY-MM-DDThh:mm:ssZ (UTC).|
|properties.outputs |JobOutput[]:JobOutputAsset[] |Выходные данные задания.|
|properties.priority |Приоритет |Приоритет обработки задания. Задания с более высоким приоритетом обрабатываются раньше, чем задания с низким приоритетом. Если значение не задано, по умолчанию используется значение normal.
|properties.state |JobState |Текущее состояние задания.
|Тип|строка|Тип ресурса.|

Полное определение см. в статье [Jobs](https://docs.microsoft.com/rest/api/media/jobs) (Задания).

## <a name="typical-workflow-and-example"></a>Типичный рабочий процесс и пример

1. Создайте преобразование. 
2. Отправьте задания в преобразование. 
3. Выведите список преобразований. 
4. Удалите преобразование, если не планируете использовать этот способ в будущем. 

Предположим, что требуется извлечь первый кадр всех видео в качестве эскиза. Ниже приведены действия по выполнению этого процесса. 

1. Определите правило для обработки. Например, используйте первый кадр видео в качестве эскиза. 
2. Затем для каждого видео, используемого в качестве входных данных, в службе нужно указать следующие сведения: 
    1. Расположение видео и 
    2. место для записи выходных данных (например, эскиза).

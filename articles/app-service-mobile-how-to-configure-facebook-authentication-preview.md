# Настройка приложения для использования имени для входа Facebook

В этом разделе показано, как настроить службы приложений Azure для использования Facebook в качестве поставщика проверки подлинности.
	
Чтобы выполнить процедуру, описанную в этом разделе, необходимо иметь учетную запись Facebook с проверенным электронным адресом и номером мобильного телефона. Чтобы создать новую учетную запись Facebook, перейдите по ссылке [facebook.com].

## <a name="register"> </a>Регистрация приложения с помощью Facebook

1. Перейдите на веб-сайт [разработчиков Facebook] и войдите с помощью своих данных учетной записи Facebook.

2. (Необязательно) Если вы еще не зарегистрированы, нажмите кнопку **Приложения**, затем **Зарегистрироваться в качестве разработчика**, примите политику и выполните действия по регистрации.

3. Нажмите кнопку **Приложения**, а затем щелкните **Создать новое приложение**

4. Выберите **Веб-сайт** в качестве платформы. Укажите уникальное имя приложения и щелкните **Создать новый идентификатор приложения Facebook**.

5. Выберите категорию приложения из раскрывающегося списка. Затем щелкните **Создать идентификатор приложения**.

6. На следующей странице выберите **Пропустить быстрый запуск** в правом верхнем углу. Вы перейдете на панель мониторинга разработчика приложения.

7. В поле **Секрет приложения** щелкните **Показать**, при запросе укажите пароль, а затем запишите значения полей **Идентификатор приложения** и **Секрет приложения**.

	> [AZURE.NOTE]**Примечание по безопасности**. Секрет клиента — это важные учетные данные безопасности. Не сообщайте этот секрет никому и не раскрывайте его в клиентском приложении.

8. В левой панели навигации щелкните **Параметры**. Введите URL-адрес шлюза в поле **Домены приложений** и укажите **контактный адрес электронной почты**.

    ![][0]

9. Если вы не видите раздел веб-сайта, показанный ниже, щелкните **Добавить платформу** и выберите **Веб-сайт**. Введите URL-адрес шлюза службы приложений в поле **URL-адрес сайта**, а затем нажмите **Сохранить изменения**.

10. Перейдите на вкладку **Дополнительно** и добавьте URI перенаправления в поле **Допустимые URI перенаправления OAuth**. После этого щелкните **Сохранить изменения**. URI перенаправления — это URL-адрес шлюза с добавлением пути _/signin-facebook_. Например, `https://contosogateway.azurewebsites.net/signin-facebook`. Убедитесь, что используете схему HTTPS.

11. Учетная запись Facebook, которая использовалась для регистрации приложения, является администратором приложения. На этом этапе только администраторы могут входить в приложение. Чтобы проверить подлинность других учетных записей Facebook, щелкните **Состояние и просмотр** в левой панели навигации. Затем щелкните **Yes** (Да), чтобы включить общий доступ.


## <a name="secrets"> </a>Добавление сведений Facebook в мобильное приложение


12. Войдите на [портал управления Azure] и перейдите к вашему шлюзу служб приложений.

13. В разделе **Параметры** выберите **Удостоверение**, а затем **Facebook**. Вставьте значения идентификатора и секрета приложения, полученные ранее. Нажмите кнопку **Сохранить**.

    ![][1]

Теперь вы можете использовать Facebook для проверки подлинности в приложении.

## <a name="azure-certified-program"></a>Программа "Сертифицировано для использования в Azure"  
Все виртуальные машины, опубликованные в Azure Marketplace, проверяются на соответствие условиям программы Azure Certified.  
*   Дополнительные сведения см. на странице Microsoft Azure Certified по адресу [azure.microsoft.com/marketplace/programs/certified](https://azure.microsoft.com/marketplace/programs/certified). 

Программа сертификации предоставляет описанные ниже преимущества.  
*   Совместимость вашей виртуальной машины с платформой Azure и моделью продажи в Marketplace.  
*   Тестирование виртуальной машины на соответствие требованиям безопасности сетевого образа, а также на отсутствие вирусов и вредоносного ПО.  
*   Добавление эмблем для виртуальной машины на уровне предложения. Эмблема проверенного решения способствует его продвижению среди корпоративных клиентов Майкрософт.

#### <a name="requirements-azure-marketplace-list-contact-me"></a>Требования, Azure Marketplace, список, связь со мной  
Тип предложения "Связь со мной" применяется, если использование типа приложения "Пробная версия" или "Транзакция" нецелесообразно. Преимущество типа предложения "Связь со мной" состоит в том, что вы можете мгновенно налаживать контакты с потенциальными клиентами при помощи решения на рынке. Отношения с потенциальными клиентами следует использовать для заключения основополагающих сделок и развития вашего бизнеса. Недостаток состоит в том, что в этом случае вовлеченность пользователей ограничена, если сравнивать с другими типами предложений.  

>[!IMPORTANT]
>Наибольшую вовлеченность пользователей обеспечивают типы предложений "Пробная версия" и "Транзакция". Ценность типа приложения "Связь со мной" состоит в обнаружении потенциальных клиентов. Выбрав тип приложения "Связь со мной", вам необходимо убедиться, что настроена цель для потенциального клиента и вы готовы в полной мере раскрыть его потенциал.

### <a name="trial"></a>Пробная версия  
Предоставление пробной версии повышает вовлеченность пользователей. Это приводит к росту популярности решения. Пробная версия позволяет клиентам протестировать ваше решение перед покупкой. Она повышает ваши шансы на продвижение в онлайн-магазинах. Кроме того, вы можете ожидать больше перспективных возможностей от взаимодействия с клиентами.  

Приложение или предложение с использованием типа "Пробная версия" развертывается в пробной среде и/или подписке Azure. Приложения других типов развертываются в среде или подписке Azure вашего клиента. Приложение или предложение должно находиться в распоряжении клиента без дополнительных требований к покупке. Для реализации готового варианта использования в приложении или предложении должно быть достаточно минимальной настройки. Для приложения или предложения должна предоставляться бесплатная поддержка (по крайней мере в течение пробного периода). Для достижения наилучших результатов нужно отслеживать клиентов и оказывать им поддержку согласно продуманному плану. Приветствуется использование вами контактов, полученных через Marketplace или собственные средства аналитики, встроенные в приложение. Контакты следует использовать для отслеживания клиентов и управления ими.

| Легко настраиваемое, готовое к использованию решение | Ваше приложение должно быть простым и быстрым в установке и настройке. |  
| Доступность / время непрерывной работы | Время непрерывной работы вашего приложение или платформы SaaS должно составлять не менее 99,9 %. |  
| Azure Active Directory | Ваше предложение должно предусматривать выполнение федеративного единого входа (SSO) в Azure Active Directory (Azure AD) с запросами на согласие. |  

#### <a name="requirements-azure-marketplace-trial-free-trial--saas-trial"></a>Требования, Azure Marketplace, пробная версия, бесплатная пробная версия или пробная версия SaaS  
Вы предоставляете решение или приложение с бесплатной пробной версией на основе технологии "программное обеспечение как услуга" (SaaS). Тип предложения "Пробная версия SaaS" помогает привлекать заинтересованных клиентов, стимулируя рост бизнеса. Бесплатную пробную версию можно представить в виде пробной учетной записи с ограниченными возможностями или сроком действия. Пробная версия должна содержать призывы к действию для ускорения перехода на платную версию вашего решения.  

| Преимущество | Требование |  
|:--- |:--- |  
| Позволяет клиенту опробовать ваш продукт перед покупкой и предусматривает автоматический переход на платную версию. Также дает возможность выполнить процесс подтверждения концепции и взаимодействовать с командами продаж Майкрософт. | Решение относится к типу предложения "Виртуальная машина или приложения Azure, шаблон решения".<br /><br />Ваше решение представляет собой предложение SaaS с мультитенантным продуктом SaaS.<br /><br />Для решения предусмотрен удобный интерфейс, понятный при первом запуске, что позволяет пользователю быстро приступить к работе.<br /><br />Используется один клиент, но пользователи добавляются в качестве гостей. |  

#### <a name="requirements-azure-marketplace-trial-interactive-demo"></a>Требования, Azure Marketplace, пробная версия, интерактивная демонстрация  
Вы можете познакомить пользователей с вашим решением при помощи интерактивной демонстрации. Преимущество интерактивной демонстрации заключается в том, что вы можете предоставить пробный выпуск без трудной подготовки сложных решений. Тип предложения "Интерактивная демонстрация" позволяет пользователю ознакомиться с решением. Тип предложения "Интерактивная демонстрация" предоставляет вам потенциальных клиентов, с которыми можно заключать основополагающие сделки для развития вашего бизнеса.  

| Преимущество | Требование |  
|:--- |:--- |  
| Позволяет пользователям ознакомиться с решением в действии без сложной настройки. | Ваше решение требует сложной настройки, для которой пробного периода может быть недостаточно. |  

#### <a name="requirements-azure-marketplace-trial-test-drive"></a>Требования, Azure Marketplace, пробная версия, тестовый выпуск  
Вы развертываете одну или несколько виртуальных машин через приложения "инфраструктура как услуга" (IaaS) или SaaS. Преимущество типа предложения "Тестовый выпуск" заключается в автоматической подготовке виртуальной машины или всего решения под руководством партнера. Тестовый выпуск позволяет пользователю оценить решение без лишних затрат. Клиенту не обязательно уже быть пользователем Azure, чтобы помочь в привлечении потенциальных клиентов более высокого уровня.  

| Преимущество | Требование |  
|:--- |:--- |  
| Позволяет клиенту опробовать ваш продукт перед покупкой.<br /><br />Позволяет пользователю ознакомиться с вашим решением с подсказками и предварительно настроенными параметрами.<br /><br />Ниже перечислены дополнительные преимущества при использовании тестового выпуска.<ul> <li>27 % запросов пользователей в Marketplace уточняются таким образом, чтобы отображались только предложения с тестовыми выпусками.</li> <li>Предложения с тестовыми выпусками позволяют получить на 38 % больше потенциальных клиентов, чем предложения без них.</li> <li>36 % пользователей, приобретающих решения в Marketplace, опробовали тестовый выпуск.</li> <li>Тестовые выпуски позволяют специалистам по реализации Майкрософт досконально анализировать продукт для внедрения инициатив по совместным продажам.</li> </ul> | Ваше решение представляет собой виртуальную машину, шаблон решения для приложения Azure или приложение SaaS с одним клиентом (либо его развертывание представляет сложность). <br /><br />У вас нет возможности преобразовать пробную версию в платное предложение. |

    >[!NOTE]

>[!NOTE]

#### <a name="requirements-azure-marketplace-transact-azure-apps-solution-template"></a>Требования, Azure Marketplace, транзакция, приложения Azure, шаблон решения  
Тип предложения "Приложение Azure, шаблон решения" используется, если в решении требуются дополнительные возможности автоматизации развертывания и настройки, недоступные обычной виртуальной машине. Вы можете автоматизировать подготовку одной или нескольких виртуальных машин с помощью шаблонов решения для приложений Azure. Вы также можете подготавливать ресурсы сети и хранилища. Тип предложения "Приложения Azure, шаблоны решений" предоставляет преимущества автоматизации для отдельных виртуальных машин и целых решений на основе IaaS.  

#### <a name="requirements-azure-marketplace-transact-container"></a>Требования, Azure Marketplace, транзакция, контейнер

>[!NOTE]

В течение следующих нескольких месяцев планируется реализовать дополнительные коммерческие возможности, включая способы выставления счетов на основе подписки и использования.  

| Требование | Сведения |

#### <a name="requirements-azure-marketplace-transact-saas-app"></a>Требования, Azure Marketplace, транзакция, приложение SaaS  
Используйте тип приложения SaaS, чтобы предоставить пользователю возможность приобрести ваше техническое решение на основе SaaS в виде подписки. Для приложения SaaS должны соблюдаться приведенные ниже требования.  
*   Счета за службу должны выставляться по фиксированному ежемесячному тарифу.  
*   Предоставьте способ обновления или отмены услуги в любой момент.  

Коммерческие транзакции хранятся в корпорации Майкрософт. Корпорация Майкрософт выставляет счета вашим клиентам от вашего имени. Чтобы использовать приложение SaaS в виде подписки, необходимо включить собственный API службы управления подписками. API службы управления подписками должен взаимодействовать непосредственно с интерфейсами API Azure Resource Manager. API службы управления подписками должен поддерживать подготовку, обновление и отмену услуги.  

| Требование | Сведения |  
|:--- |:--- |  
|Выставление счетов и ценообразование   |   Цена вашего предложения определяется помесячной фиксированной ставкой. Оплата по мере использования и возможность повышения уровня на основе использования сейчас не поддерживаются.      |  
|Отмена  |   Ваше предложение может быть отменено клиентом в любое время.      |  
|Целевая страница транзакции   |   Вы размещаете целевую страницу транзакции под совместным брендом в Azure, где пользователи могут создать учетную запись службы SaaS и управлять ею.      |   
|API подписки    |   Вы предоставляете службу, которая может взаимодействовать с подпиской SaaS для создания, обновления и удаления учетной записи пользователя и плана обслуживания. Критические изменения в API должны поддерживаться в течение 24 часов. Некритические изменения в API будут выпускаться периодически.      | 



#### <a name="requirements-azure-marketplace-transact-virtual-machine"></a>Требования, Azure Marketplace, транзакция, виртуальная машина  
Тип предложения "Виртуальная машина" используется при развертывании виртуального модуля в подписке, связанной с пользователем. Виртуальные машины полностью поддерживают коммерческое применение на основе моделей с оплатой по мере использования и с использованием собственной лицензии (BYOL). Корпорация Майкрософт проводит коммерческие транзакции и выставляет счета клиенту от вашего имени. Вы получаете преимущество использования предпочтительного платежного отношения между вашим клиентом и корпорацией Майкрософт, включая все Соглашения Enterprise.  

>[!NOTE]
>В настоящее время платежные обязательства, связанные с Соглашением Enterprise, могут применяться к использованию вашей виртуальной машины в Azure, но не к вашим тарифам на лицензирование программного обеспечения.  

>[!NOTE]
>Указав образ и цены как действующие в рамках частного предложения, вы можете предоставить возможности обнаружения и развертывания виртуальной машины только определенной группе клиентов. Благодаря частным предложениям вы можете создавать эксклюзивные предложения для своих ближайших клиентов и предлагать им настраиваемое программное обеспечение и индивидуальные условия. Благодаря таким условиям вам доступно множество сценариев взаимодействия, в том числе проведение заключаемых на местах сделок со специальными ценами и условиями, а также предоставление раннего доступа к ограниченному выпуску программного обеспечения. Благодаря частным предложениям вы можете предлагать определенные цены или продукты ограниченному числу клиентов, создавая новые номера SKU с этими сведениями.

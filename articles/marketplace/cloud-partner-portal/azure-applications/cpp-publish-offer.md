# <a name="publish-azure-application-offer"></a>Публикация предложения приложения Azure

Опубликовать предложение можно после создания предложения с информацией на странице **Новое предложение**. Выберите **Публиковать**, чтобы начать процесс публикации.

На следующей схеме представлены основные шаги в процессе публикации для запуска предложения.

![Шаги публикации предложения](./media/offer-publishing-steps.png)


## <a name="detailed-description-of-publishing-steps"></a>Подробное описание шагов публикации

В следующей таблице перечисляется и описывается каждый шаг публикации, а также приводится оценка времени для завершения каждого шага.  Временные оценки в "днях" определяются как рабочие дни, исключая выходные и праздничные дни.

|  **Шаг публикации**           | **Время**    | **Описание**                                                            |
|  -------------------           | --------    | ---------------                                                            |
| Проверка необходимых компонентов         | < 15 мин    | Информация и параметры предложения — проверенные.                        |
| Проверка параметров, которые повлияли на доход | < 15 мин  | Проверяется определение использования ресурсов Azure для предложения.             |
| Сертификация                  | < 1 день     | Команда сертификации Azure анализирует предложение. Это позволяет определить, соответствует ли предложение всем критериям применимости. Дополнительные сведения см. в разделе [Предварительные требования](./cpp-prerequisites.md). Отзывы предоставляются в случае обнаружения проблемы. |

Вы можете следить за процессом публикации на вкладке **Состояние** вашего предложения на Портале Cloud Partner.

![Вкладка "Состояние" для предложения приложения Azure](./media/offer-status-tab.png)

После публикации предложение будет добавлено в список в [категории приложений Microsoft Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/).



## <a name="errors-and-review-feedback"></a>Ошибки и отзывы в результате проверки

Помимо отображения состояния публикации вашего предложения на вкладке **Состояние** также показаны сообщения об ошибках и отзывы о шаге **проверки корпорацией Майкрософт**.  Как правило, проблемы проверки называются запросами на вытягивание.  Каждый запрос на вытягивание связан с подключенным элементом Visual Studio Team Services (VSTS, переименован в [Azure DevOps](https://azure.microsoft.com/services/devops/)), который содержит подробные сведения о проблеме.  На представленном ниже рисунке показан пример информации о запросе на вытягивание.  В более сложных ситуациях группа проверки и поддержки может отправить вам письмо. 

![Вкладка "Состояние", отображающая отзыв о проверке](./media/status-tab-ms-review.png)

Вы должны устранить каждую сообщенную проблему, прежде чем предложение сможет перейти на следующий шаг в процессе публикации.  На следующей схеме показано, как процесс получения обратной связи связан с процессом публикации.

![Шаги публикации с отзывом VSTS](./media/pub-flow-vsts-access.png)


### <a name="vsts-access"></a>Доступ VSTS

Для просмотра элементов VSTS, на которые есть ссылки в отзывах, издателям необходимо предоставить соответствующий тип авторизации.  В противном случае для новых издателей отобразится страница ответа `401 - Not Authorized`.  Чтобы запросить доступ к системе VSTS проверки предложений, выполните приведенные ниже действия.

1. Соберите следующие сведения:
    - Имя издателя и идентификатор.
    - Тип предложения (приложение Azure), имя предложения и номер SKU.
    - Ссылка в запросе на вытягивание, например: `https://solutiontemplates.visualstudio.com/marketplacesolutions/_git/contoso/pullrequest/<number>` Этот URL-адрес может быть получен из сообщения уведомления или из адреса страницы ответа 401.
    - Адреса электронной почты лиц, которым вы хотите предоставить доступ, относящиеся к организации, откуда выполняется публикация.  Они должны включать адрес владельца, которого вы указали при регистрации в качестве издателя на Портале Cloud Partner.
2. Создайте обращение в службу поддержки.  В заголовке окна Портала Cloud Partner нажмите кнопку **Справка**, а затем в меню выберите **Поддержка**.  Ваш веб-браузер по умолчанию должен запуститься и перейти на страницу нового обращения в службу поддержки Майкрософт.  (Возможно, потребуется сначала войти.)
3. В разделе **Тип проблемы** выберите **Marketplace Onboarding** (Подключение к Marketplace), в разделе **Категория** — **Проблема доступа**, а затем щелкните **Начало запроса**.

    ![Категория "Запрос в службу поддержки"](./media/support-incident1.png)

4. На странице **Шаг 1 из 2** укажите контактные данные и выберите **Продолжить**.
5. На странице **Шаг 2 из 2** укажите заголовок инцидента (например, `Request VSTS access`) и предоставьте сведения, собранные на первом шаге (выше).  Прочитайте и примите условия соглашения, а затем выберите **Отправить**.

Если создание инцидента прошло успешно, отобразится страница подтверждения.  Сохраните сведения о подтверждении для справки.  Служба поддержки Майкрософт должна ответить на ваш запрос доступа в течение нескольких рабочих дней.

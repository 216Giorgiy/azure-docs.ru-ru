# <a name="azure-application-offer"></a>Предложение приложения Azure

В этом разделе объясняется, как выполнить публикацию нового предложения приложения Azure в <a href="https://azuremarketplace.microsoft.com">Microsoft Azure Marketplace</a>.
|    |    |
|-----------------------------------------------------------------|------------------------------------------|
| Каждое приложение Azure содержит шаблон Azure Resource Manager, в котором определены все технические ресурсы, используемые приложением. Сюда можно отнести одну или несколько виртуальных машин и другие вспомогательные службы Azure или веб-службы. | ![Значок "Приложения Azure"](./media/azureapp-icon1.png)  |

## <a name="benefits"></a>Преимущества

Ниже перечислены некоторые преимущества приложений в Microsoft Marketplace:

* охватывают 100 млн пользователей Azure Active Directory в Office 365 и Dynamics 365;

* расширение команды продаж: распространение продаж по всему миру и получение канала продаж, который привлекает конечных пользователей, помогает привлекать потенциальных клиентов и инициирует беседы с новыми клиентами в разных отраслях;

* получение практических рекомендаций: мы будем публиковать рекомендации и нюансы по работе приложения в AppSource, и как улучшить продажи процедур.

## <a name="types-of-azure-applications"></a>Типы приложений Azure

Существует два типа приложений Azure: управляемое приложение и шаблон решения. Несмотря на некоторое сходство, между ними есть ряд важных отличий.

### <a name="solution-template"></a>Шаблон решения

Шаблоны решений — это один из основных способов публикации решения в Azure Marketplace. Этот тип предложения используется, если в решении требуются дополнительные возможности автоматизации развертывания и настройки, недоступные в обычной виртуальной машине (ВМ). Вы можете автоматизировать предоставление нескольких виртуальных машин с помощью шаблона решения. При этом предусматривается подготовка ресурсов сети и хранилища для предоставления сложных решений IaaS. Общие сведения о требованиях к шаблонам решений и модели выставления счетов см. в документе [Руководство по публикации предложения "Приложения Azure: шаблон решения Azure"](https://docs.microsoft.com/azure/marketplace/marketplace-solution-templates).

### <a name="managed-application"></a>Управляемое приложение

Управляемое приложение похоже на шаблон решения Marketplace, за исключением одного существенного отличия. В управляемом приложении ресурсы развертываются в группе ресурсов, управляемой поставщиком приложения. Группа ресурсов расположена в подписке пользователя, но идентификатор в клиенте издателя имеет доступ к группе ресурсов. Как издатель, вы указываете стоимость текущей поддержки решения. Благодаря управляемым приложениям Azure можно легко выполнять сборку и доставлять клиентам полностью управляемые готовые приложения.

Управляемые приложения можно предлагать не только в Marketplace, но и в каталоге служб. Каталог служб — это внутренний каталог утвержденных решений для пользователей в организации. С помощью каталога вы можете обеспечить соответствие стандартам организации, предоставляя эффективные решения для групп в организации. Каталог поможет сотрудникам в организации без труда найти набор приложений, рекомендуемых и утвержденных ИТ-отделом.

Дополнительные сведения о преимуществах и типах управляемых приложений см. в статье [Обзор управляемых приложений Azure](https://docs.microsoft.com/azure/managed-applications/overview).

## <a name="offer-components"></a>Компоненты предложения

В этом разделе описываются элементы процесса публикации предложения управляемого приложения. Он является руководством для издателя Azure Marketplace. Публикация состоит из следующих основных частей: 

* [Предварительные требования.](./cpp-prerequisites.md) Перечислены технические и бизнес-требования, которые должны быть выполнены перед созданием или публикацией предложения управляемого приложения. 
* [Создание предложения.](./cpp-create-offer.md) Перечислены шаги по созданию записи предложения управляемого приложения с использованием Портала Cloud Partner. 
* [Публикация предложения.](./cpp-publish-offer.md) Описана отправка предложения для публикации в Azure Marketplace.

## <a name="steps-in-the-publishing-process"></a>Этапы процесса публикации

Ниже перечислены основные этапы публикации для предложения приложения Azure.

1. Создание предложения. Укажите подробные сведения о предложении. К этим сведениям относятся: описание предложения, маркетинговые материалы, сведения о поддержке и характеристики ресурса.

2. Создание бизнес-ресурсов и технических ресурсов. Создайте бизнес-ресурсы (юридические документы и маркетинговые материалы) и технические ресурсы для связанного решения.

3. Создание номера SKU. Создайте номера SKU, связанные с предложением. Уникальный номер SKU нужен для каждого публикуемого образа.

4. Сертификация и публикация предложения. Оформив предложение и выполнив технические требования, вы можете отправить предложение на публикацию. В ходе отправки начинается процесс публикации. Во время этого процесса решение тестируется, проверяется, сертифицируется и активируется в Azure Marketplace.

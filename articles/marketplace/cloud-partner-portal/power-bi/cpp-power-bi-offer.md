# <a name="power-bi-application-offer"></a>Предложение приложения Power BI

|                  |                                |
|------------------|--------------------------------|
| В этом разделе описывается публикация приложения Power BI в Microsoft [AppSource Marketplace](https://appsource.microsoft.com/). Приложение Power BI упаковывает содержимое Power BI и использует потоки данных для подключения отчетов и панелей мониторинга к данным в общем хранилище данных. Затем приложение можно развертывать для других клиентов через AppSource. | ![Значок Power BI](./media/powerbi-icon.png) |

-   [Создание предложения приложения Power BI.](./cpp-create-offer.md) Шаги по созданию записи предложения приложения Power BI с использованием [Портала Cloud Partner](https://cloudpartner.azure.com).
-   [Публикации предложения приложения Power BI.](./cpp-publish-offer.md) Процедура отправки нового предложения для публикации в AppSource и способ обновления имеющегося предложения.


## <a name="publishing-steps"></a>Этапы публикации

На следующей схеме показаны основные действия публикации предложения приложения Power BI.

![Шаги публикации приложения Power BI](media/publishing-steps.png)

Процесс публикации приложений Power BI

1. Создайте приложение службы в Power BI (что приводит к созданию URL-адреса установки пакета).  Этот URL-адрес представляет технические средства предложения.  Дополнительные сведения см. в статье [Что такое Power BI?](https://go.microsoft.com/fwlink/?linkid=2028636).
2. Соберите или создайте бизнес-ресурсы, включая:
    - маркетинговые материалы;
    - юридические и справочные документы.
3.  Создание предложения. Используйте Портал Cloud Partner для настройки сведений и информации о предложении, включая его описание, маркетинговые материалы, юридическую информацию, сведения о поддержке и характеристики ресурса.
4.  Сертификация предложения и его публикация. В ходе отправки начинается процесс публикации.
    - Команда по подключению AppSource тестирует, проверяет и сертифицирует ваше приложение. 
    - После сертификации приложения вам следует проверить его в тестовой среде, а затем выпустить.

После выполнения всех вышеуказанных действий ваше приложение станет доступным в AppSource.

# <a name="power-bi-apps-technical-info-tab"></a>Вкладка технической информации о приложениях Power BI

На вкладке **Technical Info** (Технические сведения) страницы **Новое предложение** предоставляется URL-адрес пакета установщика Power BI и любая дополнительная информация, необходимая для проверки нового предложения.  В первом выпуске все приложения Power BI бесплатны, и их можно бесплатно скачать из AppSource. Поэтому вы не сможете определить номера SKU для этого типа предложения.

![Вкладка с техническими сведениями](./media/technical-info-tab.png)


## <a name="technical-info-fields"></a>Поля на вкладке с техническими сведениями 

На вкладке с **техническими сведениями** нужно указать следующие поля.  Знак звездочки (*) возле метки поля означает, что оно является обязательным.

|        Поле          |  ОПИСАНИЕ                                                                 |
|    ---------------    |  ----------------------------------------------------------------------------|
| **Installer URL** (URL-адрес установщика)     | Адрес, сгенерированный Power BI при публикации приложения и повышении его уровня до уровня рабочей среды.  Дополнительные сведения о том, как сгенерировать URL-адрес, см. в статье о [публикации приложений с панелями мониторинга и отчетами в Power BI](https://docs.microsoft.com/power-bi/service-create-distribute-apps).  |
|  **Validation instructions** (Инструкции по проверке)  |  Необязательные текстовые инструкции (не более 3000 знаков), которые помогут команде проверки Майкрософт настроить, подключить и протестировать ваше приложение. К этим инструкциям относятся: типовые параметры конфигурации, тестовые учетные записи или параметры, которые можно использовать для проверки параметра "Подключиться к данным", и т. д. Эта информация будет видна только команде проверки и будет использоваться только для целей проверки.  |
| **Is this app created as a Power BI content pack?** (Создается ли это приложение в виде пакета содержимого Power BI?) | В настоящее время это внутренне используемое поле. Оставьте для него значение по умолчанию (`No`). Если указать для этого поля значение `Yes`, это может помешать публикации.  |  
|  |  |

# <a name="power-bi-apps-storefront-details-tab"></a>Вкладка со сведениями об онлайн-магазине приложений Power BI

Используйте вкладку **Сведения об онлайн-магазине** на странице **Новое предложение**, чтобы предоставить потенциальным клиентам маркетинговую, коммерческую и юридическую информацию. На этой вкладке также указано, как управлять потенциальными клиентами, созданными в Marketplace. Эта длинная форма состоит из шести разделов: **Сведения о предложении**, **Сведения списка**, **Маркетинговые артефакты**, **Юридическая информация**, **Поддержка клиентов** и **Управление потенциальными клиентами**.  Символ звездочки (*) возле метки поля означает, что оно является обязательным.


## <a name="offer-details-section"></a>Раздел "Сведения о предложении"

В этом разделе вы публикуете общие сведения о предложении источника приложения.

![Раздел сведений о предложении на вкладке подробностей об онлайн-магазине](./media/offer-details-section.png)

В следующей таблице описаны имя и назначение этих полей.

|   Поле               |   ОПИСАНИЕ                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Offer Summary** (Краткие сведения о предложении)     | Краткое описание назначения приложения. Максимальная длина составляет 100 символов.                             |
| **Offer Description** (Описание предложения) | Описание приложения. Максимальная длина составляет 3000 символов. Поддерживается простое HTML-форматирование. |
|                       |                                                                                         |


## <a name="listing-details-section"></a>Раздел "Сведения списка"

Этот второй раздел предоставляет дополнительный контекст для вашего приложения: в каких отраслях оно обычно используется, какая категория лучше всего к нему относится, совместимые продукты и связанные поисковые запросы.

![Раздел сведений списка на вкладке подробностей об онлайн-магазине](./media/listing-details-section.png)

 В следующей таблице описаны имя и назначение этих полей.
 
|   Поле                                  |   ОПИСАНИЕ                                                        |
| --------------                           | ---------------------                                                |
| **Отрасли**                           | Выберите отрасли, к которым лучше всего подходит ваше приложение. Если приложение относится к нескольким отраслям, оставьте это поле пустым.      |
| **Категории**                           | Выберите категории, относящиеся к вашему приложению. Можно выбрать не более 3 значений.     |
| **Ссылка на справку для приложения**               | URL-адрес страницы, на которой есть справка в Интернете о приложении           |
| **Продукты, с которыми работает ваше приложение (не более 3)** | Укажите конкретные продукты, с которыми работает ваше приложение. Можно указать не более трех продуктов. Чтобы указать продукт, щелкните знак "плюс" рядом со словом New (Новое), после чего откроется текстовое поле для ввода имени продукта, с которым работает ваше приложение.      |
| **Ключевые слова для поиска (не более 3)**              | AppSource позволяет клиенту выполнять поиск по ключевым словам. Вы можете ввести набор ключевых слов, по которым приложение будет предлагаться клиентам. Например, если приложение является приложением электронной почты, можно использовать ключевые слова "электронная почта", "рассылки", "почтовое приложение" и т. д. Выбирайте слова, по которым пользователи скорее всего будут искать такое приложения в AppSource. |
|  |  |


## <a name="marketing-artifacts-section"></a>Раздел Marketing Artifacts (Маркетинговые артефакты)

Этот третий раздел позволяет отправлять фирменную символику и маркетинговые материалы.  Он состоит из четырех подразделов: **Логотипы**, **Видео**, **Документы** и **Снимки экрана**. Из маркетинговых артефактов обязательными являются логотипы и снимки экрана, но мы рекомендуем заполнить их все, чтобы повысить привлекательность предложения для клиентов.

![Раздел маркетинговых артефактов на вкладке подробностей об онлайн-магазине](./media/marketing-artifacts-section.png)

 
|    Поле                             |    ОПИСАНИЕ                                                    |
|   -----------                        |    -------------                                                  |
| *Логотипы*                              |                                                                   |
| **Логотип предложения (формат PNG, 48x48)**   | Отображается на AppSource в обзоре приложения или в результатах приложения при выполнении поиска. Поддерживается только формат PNG с разрешением 48\*48 пикселей.  |
| **Логотип предложения (формат PNG, 216x216)** | Отображается на странице сведений о приложении на AppSource.  Поддерживается только формат PNG с разрешением 216\*216 пикселей.  |
| *Видеоролики*                             |                                                                   |
| **Имя**                             | Имя или заголовок приложения.                                          |
| **URL-адрес**                              | URL-адрес видео, размещенного на сайте YouTube или Vimeo.                              |
| **Эскиз**                        | Эскиз изображения приложения.  Поддерживается только формат PNG с разрешением 1280\*720 пикселей.   |
| *Документы*                          | Не являются обязательными. Не более трех документов. Документы, отправляемые сюда, появятся на AppSource в разделе "Дополнительные сведения".  |
| **Имя**                             | Имя или заголовок вспомогательного документа.                              |
| **File**                             | Отправляемый документ должен быть в формате PDF.                             |
| *Снимки экрана*                        | Не являются обязательными. Не более пяти снимков экрана.                        |
| **Имя**                             | Имя или заголовок снимка экрана.                                       |
| **Образ —**                            | Отправляемое изображение снимка экрана должно быть в формате PNG с разрешением 1280\*720 пикселей.  | 
|   |   |


### <a name="logo-guidelines"></a>Рекомендации по логотипам

Все логотипы, передаваемые на [Портале Cloud Partner](https://cloudpartner.azure.com), должны соответствовать следующим рекомендациям.

- Не используйте в логотипе градиентный фон.
- Старайтесь не размещать в логотипе текст, включая название компании или торговую марку. Логотип должен быть "плоским". Также избегайте градиентов.
- Не растягивайте логотип.


## <a name="legal-section"></a>Юридический раздел

Этот четвертый раздел позволяет предоставить два юридических документа, обязательных для каждого предложения: политику конфиденциальности и условия использования.

![Раздел юридических сведений на вкладке сведений об онлайн-магазине](./media/legal-section.png)

|   Поле                |   ОПИСАНИЕ                           |
|------------------------|--------------------------------------   |
| **Privacy Policy URL** (URL-адрес политики конфиденциальности) | URL-адрес опубликованной политики конфиденциальности.       |
| **Условия использования**       | Политика в виде файла с обычным текстом или простого кода HTML.     |
|  |  |


## <a name="customer-support-section"></a>Раздел "Поддержка клиентов"

Укажите **URL-адрес службы поддержки** для страницы поддержки клиентов в Интернете.  Лучше всего, если эта страница поддержки в Интернете предоставляет клиентам несколько параметров контакта, например номер телефона, адрес электронной почты и чат в реальном времени. 


## <a name="lead-management-section"></a>Раздел Lead Management (Управление потенциальными клиентами)

Последний раздел содержит сведения о потенциальных клиентах, сгенерированных благодаря вашим предложениям AppSource. Здесь предлагаются следующие параметры хранения (из раскрывающегося списка) для этой информации о потенциальных клиентах.

|    Поле               |   Назначение потенциального клиента                               |
|------------------------|--------------------------------------            |
|  **None**              | Потенциальные клиенты не собраны (по умолчанию).  |
| **Большой двоичный объект Azure (не рекомендуется)** | [BLOB-объект Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview), указанный именем контейнера и строки подключения.  Этот выбор не рекомендуется; используйте вместо этого **Таблицу Azure**.  |
| **Таблица Azure**        | [Таблица Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview), указана строкой подключения.  |
| **Dynamics CRM Online** | Экземпляр [Microsoft Dynamics 365 Online](https://dynamics.microsoft.com/), указан URL-адресом и учетными данными для проверки подлинности. |
| **Конечная точка HTTPS**     | Указанная конечная точка HTTPS в формате полезных данных JSON.   |
| **Marketo**            | Экземпляр [Marketo](https://www.marketo.com/), указан идентификатором сервера, идентификатором Munchkin и идентификатором формы.   |
| **Salesforce**         | База данных [Salesforce](https://www.salesforce.com/), указана идентификатором объекта. |
|  |  |

После публикации предложения происходит проверка подключения для передачи сведений о потенциальных клиентах, а затем в указанное назначение автоматически отправляется тестовый потенциальный клиент. Сведениями о потенциальных клиентах необходимо постоянно управлять и своевременно обновлять их, чтобы отражалась ваша текущая архитектура управления клиентами.

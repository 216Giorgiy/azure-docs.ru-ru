# <a name="publish-power-bi-app-offer"></a>Публикация предложения приложения Power BI

Последний шаг после определения предложения на портале и создания связанных технических ресурсов — запуск публикации предложения.  Чтобы начать этот процесс, нажмите **Опубликовать** в окне вертикального меню **Новое предложение**.  Дополнительные сведения см. в статье [Публикация предложений Azure Marketplace и AppSource](../manage-offers/cpp-publish-offer.md).


## <a name="publishing-steps"></a>Этапы публикации

На следующей схеме представлены основные этапы в процессе публикации для прямого эфира.

![Этапы публикации для приложения Power BI](./media/publishing-process-steps.png)

Следующая таблица описывает эти шаги и дает максимальную оценку времени для их завершения.

|   Шаг публикации            |   Время     |   ОПИСАНИЕ                                                                  |
| --------------------         |------------| ----------------                                                               |
| Проверка необходимых компонентов       | 15 мин.     | Информация и параметры предложения — проверенные.                            |
| Сертификация                | 1–7 дн.   | Команда сертификации Power BI анализирует ваше предложение. Мы запустили ваше приложение Power BI с помощью теста проверки вручную, установив приложение с помощью указанного URL-адреса. См. далее, чтобы увидеть, как выполняются основные проверки как часть процесса сертификации приложения.         |
| Упаковка                    | \< 1 час  | Технические ресурсы предложения запаковываются для клиента.                        |
| Регистрация поколения потенциального клиента | \< 1 час  | Системы потенциального клиента настроены и развернуты.                                      |

Разрешите до восьми дней, чтобы завершить этот процесс. После выполнения этих этапов публикации ваше предложение приложения Power BI будет отображаться в разделе приложения Power BI [AppSource](https://appsource.microsoft.com/marketplace/apps?product=power-bi%20).


### <a name="app-certification-process"></a>Процесс сертификации приложения

Команда подключения Майкрософт использует следующий процесс для проверки вашего предложения Power BI.

1. Проверяются юридические документы и справочные ссылки.
2. Проверяется контактная информация службы поддержки.
3. URL-адрес установщика используется для проверки правильности установки. 
4. Приложение сканируется на наличие вредоносных программ и другого вредоносного контента. 
5. Выполняется проверка на соответствие отображаемого содержания описанию приложения.
6. Операции, связанные с приложениями, работают в Power BI должным образом. Открывайте отчеты и информационные панели с образцами данных, подключайтесь к пользовательским источникам данных, обновляйте и т. д.

Команда сертификации предоставляет обратную связь, если обнаруживает какие-либо проблемы.  Дополнительную информацию о требованиях приложения Power BI см. в статье [Что такое Power BI?](https://go.microsoft.com/fwlink/?linkid=2028636).

# <a name="offer-settings-tab"></a>Вкладка с параметрами предложения

На экране **New Offer** (Новое предложение) сначала создается идентификатор предложения. Идентификатор предложения состоит из трех частей: **Offer ID** (Идентификатор предложения), **Publisher ID** (Идентификатор издателя) и **Name** (Имя). Все они описываются в следующих разделах.

![Создание нового предложения консультационных услуг — вкладка "Параметры предложения"](media/consultingoffer-settings-tab.png)

*Идентификатор предложения*

Этот идентификатор является уникальным именем, которое создается при первой отправке предложения. Оно должно содержать только строчные буквы, цифры, дефисы и символы подчеркивания. **Идентификатор предложения** будет отображаться в URL-адресе и влиять на результаты поисковых систем. Например, *имя_компании_пример_службы*

Как показано в примере, **идентификатор предложения** добавляется к идентификатору издателя, образуя уникальный идентификатор. Он предоставляется в виде постоянной ссылки, которую можно добавить в закладки, и индексируется поисковыми системами.

*Как только предложение станет активным, его идентификатор невозможно будет обновить*

*Идентификатор издателя*

Этот идентификатор связан с вашей учетной записью. Когда вы войдете в учетную запись организации, ваш идентификатор **издателя** отобразится в раскрывающемся меню.

*Имя*

Эта строка отображается в качестве названия предложения в AppSource или Azure Marketplace. Поле *Имя* имеет ограничение на 50 символов.  Проверяющему может потребоваться изменить название, чтобы добавить в название предложения длительность и тип предложения.

В следующем примере с Edgewater Fullscope показано, как формируется имя предложения. Имя предложения выглядит следующим образом:

![Создание нового предложения консультационных услуг](media/cppsampleconsultingoffer.png)

Имя предложения состоит из четырех частей:

-   **Длительность**. Это значение следует указать на вкладке **Storefront Details** (Сведения об онлайн-магазине) редактора. Длительность может быть выражена в часах, днях или неделях.
-   **Type of service** (Тип услуги). Это значение следует указать на вкладке **Storefront Details** (Сведения об онлайн-магазине) редактора. Типы услуги бывают `Assessment`, `Briefing`, `Implementation`, `Proof of concept` и `Workshop`.
-   **Preposition** (Кем опубликовано). Указывается рецензентом.
-   **Name** (Имя). Определяется на странице **Offer Settings** (Параметры предложения).

>[!Note]
>Поле "Имя" имеет ограничение на 50 символов. Имя, которое вы отправляете, может потребоваться изменить, чтобы добавить продолжительность и тип предложения.

В следующем списке приведены несколько понятных имен предложений:

-   Необходимое для профессиональных услуг: часовой брифинг
-   Платформа для миграции в облако: часовой брифинг
-   PowerApps и Microsoft Flow: однодневный семинар
-   Службы машинного обучения Azure: подтверждение концепции за 3 недели
-   Решение для розничной торговли Brick and Click: часовой брифинг
-   Предоставить свои данные: недельный семинар
-   Облачная аналитика: трехдневный семинар
-   Обучение по Power BI: трехдневный семинар
-   Решение для управления продажами: недельная реализация
-   Быстрый запуск CRM: однодневный семинар
-   Dynamics 365 for Sales: двухдневная оценка

После заполнения вкладки **Offer Settings** (Параметры предложения) можно сохранить заявку. Теперь имя предложения будет отображаться над областью редактора, и его можно будет найти в разделе All Offers (Все предложения).

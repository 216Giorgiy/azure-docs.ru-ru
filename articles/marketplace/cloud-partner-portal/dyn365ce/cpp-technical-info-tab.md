# <a name="dynamics-365-for-customer-engagement-technical-info-tab"></a>Вкладка "Техническая информация" в Dynamics 365 for Customer Engagement

Вкладка **Техническая информация** на странице **Новое предложение** позволяет указать подробные сведения о приложении Dynamics 365 for Customer Engagement, включая такие ресурсы, как пакет CRM и маркетинговые логотипы.  Эта вкладка содержит четыре раздела: **Сведения о приложении**, **Пакет CRM**, **Доступность пакета CRM** и **Маркетинговые артефакты**. Символ звездочки (*) возле имени поля означает, что оно является обязательным. 


## <a name="application-info-section"></a>Раздел "Сведения о приложении"

В этом разделе вы предоставите сведения о приложении Dynamics 365.

![Раздел "Сведения о приложении" на вкладке "Техническая информация"](./media/dynce-technical-info-tab1.png)

Эти поля описаны в приведенной ниже таблице.

|      Поле                    |    ОПИСАНИЕ                  |
|    ---------                  |  ---------------                |
|   "Базовая модель лицензирования"          |  Модель лицензирования определяет, как назначаются клиенты вашему приложению в Центре администрирования Dynamics 365. Модель лицензирования **Ресурс** основана на количестве экземпляров, а модель **Пользователь** обозначает, что каждому клиенту назначается одна лицензия.  |
|  Исходящие подключения типа "сервер — сервер" и доступ к CRM Secure Store |  Позволяет настроить исходящий доступ CRM Secure Store или подключение типа "сервер — сервер". *Чтобы использовать эту функцию, на этапе сертификации требуется рассмотрение этого вопроса командой Dynamics 365.* Корпорация Майкрософт свяжется с вами для выполнения дополнительных действий, необходимых для включения поддержки этой возможности.  |
| Подписка на события жизненного цикла CRM | Для интеграции с событиями жизненного цикла Dynamics 365 вам следует предоставить выделенную службу, которая регистрируется через специальное соглашение с корпорацией Майкрософт. *Чтобы использовать эту функцию, на этапе сертификации требуется рассмотрение этого вопроса командой Dynamics 365.* Мы свяжемся с вами для выполнения дополнительных действий, необходимых для включения поддержки этой возможности.  |
| URL-адрес конфигурации приложения | Это URL-адрес веб-страницы, который позволяет пользователю настроить конфигурацию приложения. |
| Применимые продукты Dynamics 365  | Выберите продукты Dynamics 365, для которых действует это предложение. Предложение будет отображаться в выбранных продуктах в AppSource.  |
| Только маркетинговые изменения         | Значение "Да" для этого параметра обозначает, что в существующее предложение вносятся только изменения маркетинговой информации и (или) описаний.  Это позволяет пропустить для предложения этапы сертификации и подготовки.  |
|  |  |


## <a name="crm-package-section"></a>Раздел "Пакет CRM"

В этом разделе вы предоставите сведения о файле пакета AppSource.  Эта информация будет использоваться группами проверки и сертификации Dynamics 365.

![Раздел "Пакет CRM" на вкладке "Техническая информация"](./media/dynce-technical-info-tab2.png)

Эти поля описаны в приведенной ниже таблице.

|      Поле                    |    ОПИСАНИЕ                  |
|    ---------                  |  ---------------                |
|  Имя файла пакета     |  Имя файла пакета в формате ZIP.  Это имя *не* является открытым, а используется только командой сертификации Dynamics 365.  |
|  URL-адрес                          |  URL-адрес учетной записи службы хранилища Azure, в которую загружен файл пакета. Этот URL-адрес должен включать доступный только для чтения ключ SAS, чтобы наши сотрудники могли получить пакет для проверки.  |
| Более одного пакета CRM     | Выберите "Да" ТОЛЬКО в том случае, если вы намерены поддерживать несколько разных пакетов для разных версий CRM.  Каждой версии будет сопоставлен собственный файл пакета, и вам придется создавать их все по отдельности.  |
| Ресурс сценария и примера использования   | Позволяет передать документ с функциональной спецификацией приложения, который будет использоваться командой проверки Dynamics 365.  Для этой спецификации желательно выбрать формат [Шаблон пользовательского сценария E2E](http://download.microsoft.com/download/5/1/8/51812AC9-BCD8-489F-937C-5D439C494EC1/E2E%20User%20Scenario%20Template.docx).  |
|  |  |


## <a name="crm-package-availability-section"></a>Раздел "Доступность пакета CRM"

В этом разделе нужно выбрать географические регионы, в которых приложение будет доступно клиентам.  Для развертывания в перечисленных ниже регионах нужно *получить специальные разрешения и пройти проверки* на этапе сертификации: [Германия](https://docs.microsoft.com/azure/germany/), [Облако правительства США](https://docs.microsoft.com/azure/azure-government/documentation-government-welcome) и TIP.


## <a name="marketing-artifacts-section"></a>Раздел Marketing Artifacts (Маркетинговые артефакты)

В этом разделе вам следует передать логотип приложения, который будет обозначать этот пакет в AppSource Marketplace.  Для логотипа поддерживаются изображения в формате PNG с размером 255 x 115 пикселей.

# <a name="azure-marketplace-consulting-services"></a>Консультационные услуги в Azure Marketplace
Консультационные услуги в Azure Marketplace предоставляют партнерам корпорации Майкрософт цель: подключение клиентов с помощью служб, чтобы обеспечить их поддержку и расширить их использование Azure. Azure Marketplace поддерживает список предложений консультации службы от партнеров интеграторов системы (SI) корпорации Майкрософт. 

Эти предложения консультационных услуг строго привязаны к конкретным заказчикам. Издатели имеют возможность выбрать тип приложения, определить область действия и продолжительность, стоимость (фиксированная или бесплатно), а также описать передаваемые клиенту элементы. Партнеры получают извещения через систему CRM и обычно отвечают на все запросы в течение 48 часов.

## <a name="consulting-services-in-azure-marketplace-vs-appsource-storefront"></a>Консультационные услуги в Azure Marketplace и AppSource Storefront
Онлайн-магазины Marketplace связаны с целевой аудиторией и облачными продуктами Майкрософт, которые помогают клиентам найти то, что им нужно. Каждый онлайн-магазин предлагает специализированные варианты публикации, чтобы вы могли извлечь максимальную выгоду из инвестиций в публикации. Эти варианты обобщены в следующей таблице.

|         |Azure Marketplace  |AppSource |
|---------|---------|---------|
|Аудитория   |ИТ-специалисты и разработчики облачных решений|Бизнес-пользователи и лица, принимающие решения в сфере бизнес-приложений|
|Соответствие решений     |Технические приложения, созданные для (или в) Azure|Бизнес-приложения, которые используют или расширяют D365, Power BI, O365, Azure. (Надстройка или дополнение готового приложения)|
|Варианты публикации     |"Свяжитесь со мной", "Пробная версия", "Транзакция" или "Консультационные услуги"|"Свяжитесь со мной", "Пробная версия" или "Консультационные услуги"|
|В продукте      |Портал управления Azure|Dynamics 365, Office 365, клиентские приложения Office| 
|Другие      |Локальные публикации через Azure Stack|Консультационные услуги для Dynamics и Power BI|

>[!Note]
>Консультационные услуги в Azure Marketplace должны использовать предложения, предназначенные для технических специалистов, которые работают с *техническими решениями и службами Azure*.


Предложения о предоставлении услуг на базе Microsoft Dynamics 365, Office 365 и Power BI могут размещаться в AppSource. Дополнительные сведения см. в документе [с рекомендациями по размещению консультационных услуг в Microsoft AppSource](https://smp-cdn-prod.azureedge.net/documents/Microsoft%20AppSource%20Partner%20Listing%20Guidelines.pdf). 

>[!TIP]
>Эта информация предназначена только для служб Azure Marketplace. Для публикации в онлайн-магазине Azure Marketplace на портале Cloud Partner в раскрывающемся меню основного продукта нужно выбрать "Azure". Для публикации в AppSource на портале Cloud Partner в раскрывающемся меню основного продукта нужно выбрать "Dynamics 365", "Power BI" или "Power Apps".

## <a name="how-to-register-for-consulting-services-in-azure-marketplace"></a>Регистрация на предоставление консультационных услуг в Azure Marketplace
Первым делом вам нужно зарегистрировать свои консультационные услуги в [Azure Marketplace](https://azuremarketplace.microsoft.com/en-us/sell/signup). Полный контрольный список требований к регистрации издатели можно найти в статье [Become a Publisher in the Marketplace](https://docs.microsoft.com/en-us/azure/marketplace/become-publisher) (Как стать издателем в Marketplace). 

**Чего ожидать после регистрации**. Команда подключения в течение 1 – 3 рабочих дней отправит вам по электронной почте приветственное сообщение с информацией о входе на портал Cloud Partner. [Портал Cloud Partner](https://cloudpartner.azure.com) используется издателями для предоставления информации и заявок по своим предложениям. 

>[!Note]
>Партнеру достаточно зарегистрироваться однократно, чтобы стать издателем. Получив по электронной почте приветственное сообщение с информацией для входа на портал Cloud Partner, издателю больше не нужно регистрироваться повторно для публикации предложений других типов.

## <a name="publishing-process"></a>Процесс публикации
Процесс публикации предложения консультационных услуг в Azure Marketplace включает следующие этапы.

1. **Запрос**. Чтобы отправить запрос, используйте [форму заявки Azure Marketplace](https://azuremarketplace.microsoft.com/en-us/sell/nominate) и расскажите нам о вашей организации. Примечание. Издателям необходимо только один раз подать заявку на добавление в список в Azure Marketplace или AppSource. После завершения процесса выдвижения и подключения будет создан Портал Cloud Partner, на котором издатели смогут создавать и отправлять свои списки предложений.
2. **Проверка**. Ваш запрос будет проверен на соответствие требованиям к публикации. Затем мы отправим вам инструкции по размещению предложения консультационных услуг. 
3. **Отправка**. Предоставьте необходимую информацию в соответствии с инструкциями. Все содержимое и вспомогательные материалы должны быть на английском языке. 
4. **Просмотр**. Наша группа маркетологов и редакторов изучит предоставленные материалы и скорректирует их для повышения разборчивости, грамотности и эффективности. 
5. **Проверка**. Заявка возвращается вам на проверку и утверждение. 
6. **Публикация**. Когда все обсуждаемые материалы будут полностью приемлемы для вас и корпорации Майкрософт, предложение будет считаться готовым для публикации в Azure Marketplace. Дата публикации зависит от страны, указанной в предложении. Если каталог для выбранной вами страны уже активирован в Azure Marketplace, предложение появится в нем за несколько часов. В противном случае предложение станет доступным с момента публикации каталога.

## <a name="offer-listings-and-content-review-criteria"></a>Перечисление предложений и критерии проверки содержимого
Принимаются к рассмотрению предложения услуг следующих типов (вы можете разместить сразу несколько типов). 

* **Оценка**. Анализ среды клиента для определения применимости решения и предоставления оценки затрат и времени. 
* **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. 
* **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Мы рекомендуем использовать только такие решения, которые реализуются не дольше двух недель. 
* **Подтверждение концепции**. Ограниченная реализация, позволяющая определить применимость решения к требованиям клиента. 
* **Семинар**. Интерактивное взаимодействие на территории клиента. Сюда может входить обучение, брифинги, оценки и демонстрации, основанные на данных или рабочей среде клиента. 

Примечание. Все типы услуг, за исключением *брифингов*, можно предоставлять в личном порядке (в расположении клиента или на объекте партнера) или виртуально (с применением телеконференций, веб-конференций, удаленной реализации и т. д). Брифинги должны проводиться в личном порядке.

После того как вы настроите необходимые элементы, перечисленные выше, команда проверки содержимого проверит, чтобы ваше описание предложения было информативным и исчерпывающим для целевой аудитории. 

#### <a name="listing-title"></a>Заголовок списка

Обратите внимание, что поле заголовка имеет ограничение в 50 знаков и должно содержать срок действия и тип службы предложения. Это способствует оптимизации предложения для поисковых систем. Требуемый формат выглядит следующим образом: "НАЗВАНИЕ: ТИП ПРОДОЛЖИТЕЛЬНОСТИ".

Примеры:


|Пример заголовка  |Верный формат  |
|---------|---------|
|Приступая к работе с Центром Интернета вещей Azure в сфере производства    |   Интернет вещей в сфере производства: 2-дневная проверка      |
|Семинар по интеллектуальным тостерам        |  Интеллектуальные тостеры: 1-недельный семинар       |
|  Подтверждение концепции миграции SQL Server    | Миграция SQL: 3-недельное подтверждение концепции        |

На портале Cloud Partner в ваше предложение можно добавить форматирование, используя [Markdown](https://docs.microsoft.com/en-us/contribute/how-to-write-use-markdown). Это простой язык проектирования, который поможет применить форматирование к странице вашего предложения. 

**Примечания по разметке Markdown**
*   Добавьте символ * в начале и конце блока текста (не отделяя их пробелами), чтобы выделить этот текст курсивом (*пример текста*).
* Символ двоеточия (:) создает разрыв строки
*   Символ * с пробелом создает маркеры ненумерованного списка (*(пробел)Элемент списка)
*    ### <a name="space-for-heading-ex-spaceagendadeliverables"></a>Пробел для заголовка (###(пробел)Расписание/Результаты)
*   Разрыв строки после заголовка
*   ** = полужирный шрифт (не отделяя пробелами) (**без пробелов**) • Сводка предложения выделяется курсивом • "Расписание и Результаты" — это заголовок

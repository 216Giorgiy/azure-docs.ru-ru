# <a name="azure-marketplace-consulting-services"></a>Консультационные услуги в Azure Marketplace
Чтобы предоставить клиентам доступ к услугам, поддерживающим и дополняющим работу служб Microsoft Azure, в Azure Marketplace размещаются предложения консультационных услуг от системных интеграторов, сотрудничающих с корпорацией Майкрософт. 

Эти предложения консультационных услуг строго привязаны к конкретным заказчикам. Для них определяются область и длительность действия, фиксированная цена (в том числе нулевая) и конкретный результат. Потенциальные клиенты могут легко выйти на связь с партнерами. Партнеры получают извещения через систему CRM и обычно отвечают на все запросы в течение 48 часов.

Предложения о предоставлении услуг на базе Microsoft Dynamics 365, Office 365 и Power BI могут размещаться в AppSource. Дополнительные сведения см. в документе [с рекомендациями по размещению консультационных услуг в Microsoft AppSource](https://smp-cdn-prod.azureedge.net/documents/Microsoft%20AppSource%20Partner%20Listing%20Guidelines.pdf). 

## <a name="eligible-services-and-partners"></a>Соответствующие услуги и партнеры

Участники также должны иметь компетенцию уровня Silver или Gold в нужной области. Допустимые компетенции перечислены в таблице ниже.

|Компетенция  |Область решения  |
|---------|---------|
|Облачная платформа и инфраструктура     |   Облачная платформа, центр обработки данных      |
|Разработка приложений и независимый поставщик программного обеспечения     |  Разработка приложений, интеграция приложений, DevOps       |
|Управление данными и аналитика    |  Аналитика данных, платформа данных       |

Принимаются к рассмотрению предложения услуг следующих типов (вы можете разместить сразу несколько типов). 

* **Оценка**. Анализ среды клиента для определения применимости решения и предоставления оценки затрат и времени. 
* **Брифинг**. Ознакомление с решением или консультационной услугой для привлечения интереса клиента с использованием схем, демонстрационных материалов и историй клиентов. 
* **Реализация**. Полная установка, предоставляющая клиенту полностью рабочее решение. Мы рекомендуем использовать только такие решения, которые реализуются не дольше двух недель. 
* **Подтверждение концепции**. Ограниченная реализация, позволяющая определить применимость решения к требованиям клиента. 
* **Семинар**. Интерактивное взаимодействие на территории клиента. Сюда может входить обучение, брифинги, оценки и демонстрации, основанные на данных или рабочей среде клиента. 

Все типы услуг, за исключением *брифингов*, можно предоставлять в личном порядке (в расположении клиента или на объекте партнера) или виртуально (с применением телеконференций, веб-конференций, удаленной реализации и т. д.). Брифинги должны проводиться в личном порядке.

Узнайте больше о [предоставлении компетенций через Microsoft Partner Network](https://partner.microsoft.com/en-US/membership/competencies).

## <a name="publishing-process"></a>Процесс публикации
Процесс публикации предложения консультационных услуг в Azure Marketplace включает следующие этапы.

1. **Запрос**. Чтобы отправить запрос, используйте [форму заявки Azure Marketplace](https://azuremarketplace.microsoft.com/en-us/sell/nominate) и расскажите нам о вашей организации. 
2. **Проверка**. Ваш запрос будет проверен на соответствие требованиям к публикации. Затем мы отправим вам инструкции по размещению предложения консультационных услуг. 
3. **Отправка**. Предоставьте необходимую информацию в соответствии с инструкциями. Все содержимое и вспомогательные материалы должны быть на английском языке. 
4. **Просмотр**. Наша группа маркетологов и редакторов изучит предоставленные материалы и скорректирует их для повышения разборчивости, грамотности и эффективности. 
5. **Проверка**. Заявка возвращается вам на проверку и утверждение. 
6. **Публикация**. Когда все обсуждаемые материалы будут полностью приемлемы для вас и корпорации Майкрософт, предложение будет считаться готовым для публикации в Azure Marketplace. Дата публикации зависит от страны, указанной в предложении. Если каталог для выбранной вами страны уже активирован в Azure Marketplace, предложение появится в нем за несколько часов. В противном случае предложение станет доступным с момента публикации каталога.

## <a name="content-review-criteria"></a>Критерии проверки содержимого

После того как вы настроите необходимые элементы, перечисленные выше, команда проверки содержимого проверит, чтобы ваше описание предложения было информативным и исчерпывающим для целевой аудитории. 

Обратите внимание, что поле заголовка имеет ограничение в 50 знаков и должно содержать срок действия и тип службы предложения. Это способствует оптимизации предложения для поисковых систем. Требуемый формат выглядит следующим образом: "НАЗВАНИЕ: ТИП ПРОДОЛЖИТЕЛЬНОСТИ".

Примеры:


|Пример заголовка  |Верный формат  |
|---------|---------|
|Приступая к работе с Центром Интернета вещей Azure в сфере производства    |   Интернет вещей в сфере производства: 2-дневная проверка      |
|Семинар по интеллектуальным тостерам        |  Интеллектуальные тостеры: 1-недельный семинар       |
|  Подтверждение концепции миграции SQL Server    | Миграция SQL: 3-недельное подтверждение концепции        |

 
При рассмотрении описаний предложений и сводок по ним команда будет учитывать следующие критерии:

|Тип предложения  |Обязательно |Рекомендуется  |
|---------|---------|---------|
|Оценка    |   Включите подробный список операций для многодневных или многонедельных проверок и сформулируйте, какой результат может ожидать клиент      |    Оптимизируйте сводку предложения, включив основные ключевые слова с целью оптимизации для поисковых систем     |
|Брифинг    |   Четко сформулируйте, какой результат может ожидать клиент. Помните, что брифинги также должны проводиться в личном порядке. Если вы предоставляете виртуальную службу, выберите другой тип предложения    |  Оптимизируйте сводку предложения, включив основные ключевые слова с целью оптимизации для поисковых систем       |
|Реализация    |  Включите подробный список операций для многодневных или многонедельных реализаций и сформулируйте, какие технические изменения, технические артефакты или другие артефакты клиент может ожидать в качестве результатов участия       |    Оптимизируйте сводку предложения, включив основные ключевые слова с целью оптимизации для поисковых систем     |
|Подтверждение концепции    |    Определите, какие технические изменения, технические артефакты или другие артефакты клиент может ожидать в качестве результатов участия     | Оптимизируйте сводку предложения, включив основные ключевые слова с целью оптимизации для поисковых систем        |
|Семинар    |   Включите подробный список операций (ежедневных, еженедельных или ежемесячных) в зависимости от выбранного срока действия предложения. Укажите цели обучения или другие результаты участия в семинаре     |   Оптимизируйте сводку предложения, включив основные ключевые слова с целью оптимизации для поисковых систем      |

На портале Cloud Partner в ваше предложение можно добавить форматирование, используя [Markdown](https://docs.microsoft.com/en-us/contribute/how-to-write-use-markdown). Это простой язык проектирования, который поможет применить форматирование к странице вашего предложения.

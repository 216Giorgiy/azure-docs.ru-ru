<a name="saas-application-technical-publishing-guide"></a>Техническое руководство по публикации приложений SaaS
===========================================

Добро пожаловать в техническое руководство по публикации приложений SaaS. Это руководство предназначено для помощи потенциальным и существующим издателям при публикации приложений и служб в AppSource или Azure Marketplace с использованием предложения "Приложения SaaS".

Общие сведения о других предложениях Marketplace см. в [руководстве по публикации в Marketplace](https://aka.ms/sellerguide).


<a name="what-are-pre-requisites-for-publishing-a-saas-app"></a>Каковы предварительные требования для публикации приложения SaaS?
-------------------------------------------------

На [Портале Cloud Partner](https://cloudpartner.azure.com) реализован доступ к порталу на основе ролей. Портал позволяет посетителям коллективно работать над публикацией предложений. Дополнительные сведения см. в статье об [управлении пользователями](./cloud-partner-portal-manage-users.md). 

Прежде чем опубликовать предложение от имени учетной записи издателя, одному из пользователей с ролью *Владелец* необходимо принять [условия использования](https://azure.microsoft.com/support/legal/website-terms-of-use/), [заявление о конфиденциальности Майкрософт](https://www.microsoft.com/privacystatement/default.aspx) и [соглашение по программе сертификации Microsoft Azure](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/).

### <a name="creating-a-new-offer"></a>Создание предложения

После входа на [Портал Cloud Partner](https://cloudpartner.azure.com) выберите элемент **Новое предложение** в строке меню слева, где отображается меню доступных предложений. Пример этих предложений показан на следующем изображении.

![Новое предложение SaaS](media/cpp-creating-saas-offers/saas-new-offer.png)

Выберите из списка предложение, для которого вы получили утверждение. Откроется форма нового предложения.

![Форма нового предложения SaaS](media/cpp-creating-saas-offers/saas-new-offer-2.png)

В следующей таблице описаны поля предложения.

| **Поля предложения** | **Описание**                                                                                            |
|------------------|----------------------------------------------------------------------------------------------------------- |
| Offer ID (Идентификатор предложения)         | Уникальный идентификатор предложения в профиле издателя. Он будет отображаться в URL-адресах продукта и отчетах о выставлении счетов. Идентификатор может содержать только строчные буквы, цифры или дефисы (-). Он не может заканчиваться дефисом, а его длина не должна превышать 50 символов. Обратите внимание, что после активации предложения это поле блокируется. Например, если издатель Contoso опубликует предложение с идентификатором sample-vm, оно отобразится в Azure Marketplace как [https://azuremarketplace.microsoft.com/marketplace/apps/contoso.sample-vm?tab=Overview](https://azuremarketplace.microsoft.com/) |
| Publisher ID (Идентификатор издателя)     | Идентификатор издателя — это ваш уникальный идентификатор в Marketplace. Для всех предложений необходимо указывать свой идентификатор издателя. Идентификатор издателя нельзя изменить после того, как предложение будет сохранено.                                                                                       |
| ИМЯ             | Это отображаемое имя для предложения. Это имя будет отображаться в Azure Marketplace и на портале Azure. Его длина не должна превышать 50 символов. Рекомендуется указать для вашего продукта легко узнаваемое название марки. Не включайте в него название своей организации, если только это не требуется для продвижения продукта. Если вы продвигаете продукт через свой веб-сайт, убедитесь, что данное имя полностью совпадает с используемым на сайте.             |
|  |  |

Щелкните **Save** (Сохранить), чтобы сохранить настройки. В следующем разделе описано добавление планов в предложение.


### <a name="technical-information"></a>Технические сведения

В разделе технических сведений можно ввести следующие данные:

![Технические сведения о предложении SaaS](media/cpp-creating-saas-offers/saas-new-offer-technical-info.png)

Важнее всего определить, будете ли вы использовать листинг SaaS или следует включить функцию коммерции. Если вы выбрали листинг SaaS, необходимо выбрать между следующими вариантами:

-   Free (Бесплатно) — укажите URL-адрес приложения SaaS, по которому клиенты могут получить приложение.
-   Free Trial (Бесплатная пробная версия) — укажите URL-адрес приложения SaaS, по которому клиенты могут запустить пробную версию, прежде чем приобретут предложение.
-   Contact me (Связаться со мной) — применимо только при наличии подключенной системы управления потенциальными клиентами. Этот параметр позволяет клиентам запрашивать связь с ними, а вам соответственно предоставляются сведения о потенциальном клиенте.

Если у вас есть приложение SaaS, размещенное в Azure Marketplace, для которого планируется включить функцию коммерции через транзакции Майкрософт, выберите **Sell through Azure** (Продавать через Azure).  
Дополнительные сведения о подключении приложения SaaS см. в статье [SaaS - Sell through Azure](./cloud-partner-portal-saas-offer-subscriptions.md) (SaaS: продажа через Azure).


### <a name="test-drive"></a>Тестовый выпуск

Создание пробной версии для клиентов — оптимальный способ обеспечить их уверенностью при покупке. Из доступных вариантов пробной версии тестовый выпуск наиболее эффективен для генерирования высококачественных потенциальных клиентов и их конверсии в реальных.

При его использовании клиенты самостоятельно протестируют пробную версию продукта, ознакомятся с его ключевыми функциями и преимуществами на практике.

![Тестовый выпуск предложения SaaS](media/cpp-creating-saas-offers/saas-new-offer-test-drive.png)


#### <a name="how-a-test-drive-works"></a>Как работает тестовый выпуск

Потенциальный клиент выполняет поиск и обнаруживает ваше приложение в Marketplace. Клиент выполняет вход и принимает условия использования. На этом этапе клиент получает предварительно настроенную среду, где он может работать с вашим предложением в течение фиксированного количества часов, а вы получаете сведения о высококачественном потенциальном клиенте для дальнейших действий.

![Тестовый выпуск предложения SaaS 2](media/cpp-creating-saas-offers/saas-new-offer-test-drive-2.png)

Независимо от того, насколько сложным является ваше приложение, тестовый выпуск Майкрософт позволяет потенциальным клиентам самостоятельно ознакомиться с продуктом. Доступны три различных типа тестовых выпусков в зависимости от типа продукта, сценария и используемой площадки marketplace.

-   **Azure Resource Manager**. Тестовый выпуск Azure ARM — шаблон развертывания, который содержит все ресурсы Azure, составляющие решение, создаваемое издателем. Для этого типа тестового выпуска подходят приложения, использующие только ресурсы Azure.
-   **Приложение логики**. Тестовый выпуск приложения логики — это шаблон развертывания, который должен охватывать все сложные архитектуры решений. Для всех приложений Dynamics или пользовательских продуктов следует использовать этот тип тестового выпуска.
-   **Power BI**. Тестовый выпуск Power BI состоит из внедренной ссылки на настраиваемую панель мониторинга. Этот тип тестового выпуска следует использовать для любого продукта, в котором нужна демонстрация интерактивной визуализации Power BI. Достаточно отправить внедренный URL-адрес Power BI.

Основные этапы публикации для добавления тестового выпуска:

1.  Определение сценария тестового выпуска.
2.  Создание и (или) изменение шаблона Resource Manager.
3.  Создание пошагового руководства для тестового выпуска.
4.  Повторная публикация предложения.

Дополнительные сведения см. в статье о [тестовом выпуске](./what-is-test-drive.md).


### <a name="storefront-details"></a>Подробные сведения об онлайн-магазинах

Для приложений SaaS необходимо включить два первых раздела сводки и описания.

![Сведения об онлайн-магазине предложения SaaS](media/cpp-creating-saas-offers/saas-new-offer-storefront-details.png)

В следующей таблице описаны сведения об онлайн-магазине предложения.

| **Поля предложения**        | **Описание**                                                                                          |
|-------------------------| ---------------------------------------------------------------------------------------------------------|
| Offer Summary (Сводка предложения)           | Краткая сводка по ценности вашего предложения. Она будет отображаться на странице поиска вашего предложения. Длина не должна превышать 100 символов.   |
| Описание предложения       | Описание, которое будет отображаться на странице сведений о приложении. Максимально допустимое количество — 1300 символов. *Примечание*. В это поле можно вводить HTML-текст с тегами. например &ltp\>, &lth1\>, &lth2\>, &ltli\> и т. д., чтобы сделать содержимое более презентабельным. Команда портала публикации работает над добавлением функции предварительного просмотра сведений об онлайн-магазине, чтобы можно было делать содержимое более привлекательным в пошаговом режиме. Тем временем вы можете воспользоваться любым онлайн-средством просмотра HTML в реальном времени, например http://htmledit.squarefree.com, чтобы увидеть, как будет выглядеть ваше описание. |
| Отрасли              | Выберите отрасли, к которым лучше всего подходит ваше предложение. Если приложение относится к нескольким отраслям, можно выбрать не более двух. |
| Suggested Categories (Предлагаемые категории)    | Выберите категории, к которым лучше всего подходит ваше предложение. Можно выбрать не более трех категорий.     |
| Версия приложения     | Введите номер версии приложения.                                                                |
| Ключевые слова для поиска (не более трех) | Введите не более трех ключевых слов для поиска, с помощью которых клиенты смогут найти ваше приложение на веб-сайте онлайн-магазина Marketplace. |
|  |  |

### <a name="contact-information"></a>Контактные данные

В этом разделе следует ввести контакты службы поддержки вашей компании, которая будет отвечать за поддержку клиентов, пользующихся предложением.
В разделе предусмотрены три основные области: Engineering Contact (Контактные данные технической службы), Support Contact (Контактные данные службы поддержки) и Support URLs (URL-адреса службы поддержки):

![Маркетинговые контактные данные предложения SaaS](media/cpp-creating-saas-offers/saas-new-offer-contact-info.png)


| **Контакт**         | **Описание**                                                                                                                          |
|---------------------|------------------------------------------------------------------------------------------------------------------------------------------|
| Контактные данные технической службы | Укажите имя, адрес электронной почты и номер телефона контактного лица из технической службы, с которым корпорация Майкрософт может связаться по вопросам поддержки и бизнеса. |
| Контактные данные службы поддержки     | Укажите имя, адрес электронной почты, номер телефона и URL-адрес службы поддержки, с которой могут связаться ваши клиенты при наличии запросов о поддержке.                  |
|  |  |


Когда ваше предложение будет готово и вы нажмете кнопку публикации, предложение должно пройти сертификацию. Мы тестируем приложения SaaS, проверяя URL-адреса вручную, если вы используете листинг, или конечные точки — если выбран параметр Sell through Azure (Продавать через Azure). В процессе утверждения вручную мы также определим онлайн-магазин, в котором должно отображаться приложение (AppSource, Azure Marketplace или и то и другое).

<a name="updating-the-offer"></a>Обновление предложения
------------------

Существуют различные типы обновлений, которые может потребоваться применить к предложению после его публикации и активации. Любое изменение, внесенное в новую версию вашего предложения, нужно сохранить и опубликовать повторно для его отображения в Marketplace.

<a name="deleting-an-existing-offer"></a>Удаление существующего предложения
--------------------------

По своему усмотрению вы можете удалить предложение из Marketplace. Удалив предложение, вы гарантируете, что новые клиенты больше не смогут приобрести или развернуть ваше предложение (однако это никак не повлияет на существующих клиентов). Завершение предложения — это процесс прекращения действия соглашения об обслуживании и/или лицензировании между вами и существующими клиентами.

Руководства и политики, связанные с удалением и прекращением доступности предложения, регулируются соглашением с издателем Microsoft Marketplace (см. раздел 7) и политиками участия (см. раздел 6.2). В этом разделе содержатся сведения о разных поддерживаемых сценариях удаления и шагах, необходимых для их выполнения.

### <a name="delete-the-live-offer"></a>Удаление действующего предложения

При запросе на удаление действующего предложения нужно учитывать ряд аспектов. Выполните приведенные ниже шаги, чтобы получить от службы поддержки инструкции по удалению действующего предложения из Azure Marketplace.

1.  Создайте запрос в службу поддержки, используя эту ссылку.
2.  В списке Problem Type (Тип проблемы) выберите **Managing offers** (Управление предложениями), а в списке Category (Категория) — **Modifying an offer and/or SKU already in production** (Изменение предложения и (или) SKU, которые уже находятся в рабочей среде).
3.  Отправьте запрос.

Отдел службы поддержки поможет вам в ходе удаления предложения.

> [!NOTE] 
> Удаление предложения не влияет на текущий процесс покупки этого предложения. Процесс покупки для клиентов будет продолжать работать, как и раньше.
Однако предложение не будет доступно для новых покупок по завершении удаления.

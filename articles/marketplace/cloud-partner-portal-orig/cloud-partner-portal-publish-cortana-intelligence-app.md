# <a name="publish-a-cortana-intelligence-offer-using-the-cloud-partner-portal"></a>Публикация предложения Cortana Intelligence с помощью портала Cloud Partner

В этой статье описано, как опубликовать предложение Cortana Intelligence с помощью портала Cloud Partner.

## <a name="prerequisites"></a>Предварительные требования

К порталу Cloud Partner поддерживается доступ на основе ролей, что позволяет участникам совместно работать над публикацией предложения. Дополнительные сведения см. в статье [Managing users on cloud partner portal](./cloud-partner-portal-manage-users.md) (Управление пользователями на портале Cloud Partner).

Прежде чем опубликовать предложение от имени учетной записи издателя, одному из пользователей с ролью \"владельца\" необходимо согласиться с [условиями использования](https://azure.microsoft.com/support/legal/website-terms-of-use/), [заявлением о конфиденциальности Майкрософт](https://www.microsoft.com/privacystatement/default.aspx) и [соглашением по программе сертификации Microsoft Azure](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/).

## <a name="to-start-creating-a-cortana-inteligence-offer"></a>Начало создания предложения Cortana Inteligence

Выполнив все предварительные требования, вы можете приступить к созданию предложения Cortana Inteligence.

1.  Выполните вход на [портал Cloud Partner](http://cloudpartner.azure.com/).
2.  На панели навигации слева выберите **+ New offer** (+ Создать предложение).
3. Выберите **Cortana Intelligence**.
4. Выберите вкладку **Editor** (Редактор) в представлении **New Offer** (Новое предложение), чтобы настроить следующие параметры:
    -   Параметры предложения
    -   Технические сведения
    -   Подробные сведения об онлайн-магазинах
    -   Контакты

    Для каждого из перечисленных выше параметров отображается форма, которую необходимо заполнить.  Обязательные для заполнения поля в каждой форме отмечены красной звездочкой (\*).

## <a name="to-configure-offer-settings"></a>Настройка параметров предложения

В разделе Offer Settings (Параметры предложения) содержатся основные сведения о предложении, такие как идентификатор предложения, идентификатор издателя и название предложения.

### <a name="offer-id"></a>Offer ID (Идентификатор предложения)

Это уникальный идентификатор для предложения в профиле издателя.
Этот идентификатор отображается в URL-адресах продукта. Идентификатор может содержать только строчные буквы, цифры или дефисы (-). Он не может заканчиваться дефисом, а его длина не должна превышать 50 символов. 
>[!Note]
>После активации предложения это поле блокируется.

**Пример.**

Если издатель **contoso** создаст предложение с идентификатором *sample-Cortana Intelligence*, оно будет отображаться в AppSource как https://appsource.microsoft.com/marketplace/apps/**contoso**.*sample-Cortana Intelligence*?tab=Overview.

### <a name="publisher-id"></a>Publisher ID (Идентификатор издателя)

Это раскрывающийся список профилей издателей. В нем следует выбрать профиль издателя, от имени которого будет опубликовано это предложения.

>[!Note]
>После активации предложения это поле блокируется.

### <a name="name"></a>ИМЯ

Отображаемое имя для предложения. Это имя отображается в [AppSource](https://appsource.microsoft.com). Его длина не должна превышать 50 символов.

Когда введете необходимые сведения в разделе параметров предложения, выберите **Save** (Сохранить), чтобы перейти в раздел Technical Info (Технические сведения) предложения.

## <a name="to-configure-technical-info"></a>Указание технических сведений

В этом представлении указываются технические сведения, которые будут отображаться на странице предложения. К этому представлению применяются следующие поля.

### <a name="partner-mpn-id"></a>Partner MPN Id (Идентификатор MPN партнера)

Если вы являетесь зарегистрированным партнером Майкрософт, перейдите на [веб-сайт партнеров](https://partners.microsoft.com/) и выполните вход, чтобы получить свой идентификатор организации-партнера. Введите этот идентификатор в поле **Partner MPN Id** (Идентификатор MPN партнера).

### <a name="analytics-components-used"></a>Analytics components used (Используемые компоненты аналитики)

Выберите все компоненты, которые использует ваше предложение. Следует выбрать хотя бы один компонент. **Microsoft R** следует выбирать только в том случае, если вы используете одну из лицензий MRS служб SQL 2016 R, HDInsight Premium или MRS, запущенных на виртуальных машинах.

### <a name="additional-components-used"></a>Additional components used (Используемые дополнительные компоненты)

Выберите все прочие компоненты, используемые в решении.

### <a name="customer-name-using-solution"></a>Customer Name using solution (Имя клиента, использующего решение)

Укажите имя клиента, который использует это решение в рабочей среде. 

>[!Note]
>Эти сведения не публикуются в AppSource. Они используются только для оценки решения.

### <a name="azure-consumption-requirement-met"></a>Azure consumption requirement met? (Соблюдено требование к использованию Azure?)

Укажите, начисляется ли за это решение как минимум 1000 долл. США в месяц на каждого клиента за компоненты Azure Suite, или если в нем используется Microsoft R.

>[!Note]
>Эти сведения не публикуются в AppSource. Они используются только для оценки решения.

### <a name="demo-video-url"></a>Demo Video URL (URL демонстрационного видео)

Укажите URL-адрес демонстрационного видео для решения или приложения, которое может показывать клиентам ваша команда по вопросам предпродаж. 

>[!Note]
>Эти сведения не публикуются в AppSource. Они используются только для оценки решения.

#### <a name="demo-video-guidelines"></a>Рекомендации по созданию демонстрационного видео

- Продолжительность видео должна быть менее 15 минут.
- Видео должно быть минимально отредактированной записью функциональных возможностей решения. 
- Видео подчеркивает ключевые аспекты функциональности для пользователя и сосредоточено на интеграции расширенной аналитики. 
- Демонстрационные видеоролики **не** выкладываются для общего доступа, а служат для демонстрации того, как *можно было бы* представить решение потенциальному клиенту. Поэтому они должны иметь субтитры и описывать воспроизводимые действия.
- Для создания видео можно использовать любое средство записи с экрана, экспортирующее видео в стандартном формате (например, MPEG). 

Ниже показано, как создать видео с помощью Skype для бизнеса. 

1. [Начните собрание](https://support.office.com/article/Start-a-Skype-for-Business-conference-call-8dc8ac52-91ac-4db9-8672-11551fdaf997).
2. [Осуществите совместный доступ к рабочему столу](https://support.office.com/article/Share-your-screen-in-Skype-for-Business-2d436dc9-d092-4ef1-83f1-dd9f7a7cd3fc).
3. [Начните запись](https://support.office.com/article/Share-sites-or-documents-with-people-outside-your-organization-80e49744-e30f-44db-8d51-16661b1d4232).
4. После остановки записи [используйте диспетчер записи для публикации](https://support.office.com/article/Recording-Manager-save-and-publish-59a3beb7-c700-40cf-ab21-bc82a2b06351).

Отправьте записанное видео в службу, создающую общий URL-адрес. Например, это может быть [гостевая ссылка в OneDrive или SharePoint](https://support.office.com/article/Share-sites-or-documents-with-people-outside-your-organization-80e49744-e30f-44db-8d51-16661b1d4232).

### <a name="supported-regions"></a>Поддерживаемые регионы

Выберите все регионы, которые поддерживает решение. Необходимо выбрать хотя бы один регион.

### <a name="power-bi-desktop-file-pbix"></a>Power BI desktop file (.pbix) (Файл Power BI Desktop (.pbix))

Если применимо, отправьте PBIX-файл. Убедитесь, что данные импортированы в файл, а не открываются по внешней ссылке. Мы создадим для вас внедренный отчет.

### <a name="solution-architecture"></a>Архитектура решения

Отправьте документ с подробным описанием архитектуры решения. Рекомендации по отправке схемы архитектуры решения см. в документе [Advanced Analytics Solution Workflow Template](https://partnersprofilesint.blob.core.windows.net/partner-assets/documents/AppSource%20Solution%20Publishing%20Guide%20Docs/Advanced%20Analytics%20Solution%20Workflow%20Template%20V.2017.3.23.pptx) (Шаблон рабочего процесса для решения углубленной аналитики).

>[!Note]
>Эти сведения не публикуются в AppSource. Они используются только для оценки решения.

### <a name="select-segments"></a>Select segments (Выбор сегментов)

Выберите все связанные отраслевые сегменты. Если приложение предназначено для сегмента, отсутствующего в списке, введите имя сегмента в поле **Others** (Другие). Название сегмента ограничивается тремя словами.

### <a name="select-business-processes"></a>Select business processes (Выбор бизнес-процессов)

Выберите бизнес-процессы, которые наиболее точно описывают ваше решение. Если приложение связано с процессами, отсутствующими в списке, введите имя такого процесса в поле **Others** (Другие). Название процесса ограничивается тремя словами.

### <a name="trial-info"></a>Trial Info (Сведения о пробной версии)

-   **SaaS Trial URL** (URL-адрес пробной версии SaaS): введите URL-адрес пробной версии вашего приложения.
-   **Test Drive Trial URL** (URL-адрес пробной версии тестового выпуска): введите URL-адрес тестового выпуска вашего приложения.

Дополнительные сведения о пробных версиях см. в описании параметра **App type** (Тип приложения) в следующем разделе этой статьи.

После завершения ввода необходимых сведений в разделе Technical Info (Технические сведения) выберите **Save** (Сохранить), чтобы перейти в раздел Storefront Details (Сведения об онлайн-магазине) предложения. 

## <a name="to-configure-storefront-details"></a>Настройка сведений об онлайн-магазине

### <a name="offer-summary"></a>Offer Summary (Сводка предложения)

Краткая сводка по ценности вашего предложения. Она будет отображаться на странице поиска вашего предложения. Максимальная длина сводки — 100 символов.

### <a name="offer-description"></a>Описание предложения

Это описание, которое будет отображаться на странице сведений о приложении.
Максимальная длина описания — 1300 символов.

Обратите внимание, что это поле принимает HTML-содержимое с тегами \<p\>, \<h1\>, \<h2\>, \<li\> и т. д., что позволяет сделать содержимое гораздо более презентабельным. Команда портала публикации работает над добавлением функции предварительного просмотра сведений об онлайн-магазине, чтобы можно было делать содержимое более привлекательным в пошаговом режиме. Тем временем вы можете воспользоваться любым онлайн-средством просмотра HTML в режиме реального времени, например [http://htmledit.squarefree.com](http://htmledit.squarefree.com/), чтобы увидеть, как будет выглядеть ваше описание.

Рекомендуется разбить текст описания на подразделы в зависимости от предлагаемых преимуществ и выделить каждый из них подзаголовком. Клиенты обычно кратко просматривают поле сводки приложения и подзаголовки, чтобы получить представление о его предназначении и решить, стоит ли им на него взглянуть. Таким образом, важно привлечь внимание пользователя и дать ему причину продолжить чтение для получения подробных сведений.

### <a name="industries"></a>Отрасли

Выберите отрасли, к которым лучше всего подходит ваше приложение. Если приложение относится к нескольким отраслям, оставьте это поле пустым.

### <a name="categories"></a>Категории

Выберите категории, относящиеся к вашему приложению. Можно выбрать максимум две категории.

### <a name="app-type"></a>Тип решения

Выберите тип пробной версии, которую будет поддерживать приложение в AppSource. Доступны следующие типы пробных версий:
- **Free** (Бесплатная) означает, что приложение доступно бесплатно.
- **Trial** (Пробная) означает, что клиенты могут попробовать приложение в течение указанного периода.
- **Request for trial** (Запрос пробной версии) означает, что клиенты могут запросить пробную версию приложения.

Партнеры могут предоставить на AppSource два типа пробных версий.

- **Пробная** версия, известная также как **пробное использование под руководством клиента (Customer Led Trials (CLT))**, может быть одного из следующих типов: 
    - Пробная версия SaaS
        - Клиенты могут перейти по URL-адресу, предоставленному вами или вашим партнером. Клиент выполняет федеративный единый вход в AAD, чтобы получить ограниченный по времени доступ к пробной версии.
        - Это приложение SaaS с поддержкой мультитенантности, которое изолирует конфигурацию и данные пользователя от других пользователей. Или же в этом интерфейсе предоставляется только часть функций с операциями только чтения.

        **Примеры**

        - [AFS POP Retail Execution](https://appsource.microsoft.com/product/web-apps/afs.fa9fc926-3bc3-43dd-becd-3ef41b52c10b?tab=Overview)
        - [AvePoint Citizen Services](https://appsource.microsoft.com/product/web-apps/avepoint.7738ac97-fd40-4ed3-aaab-327c3e0fe0b3?tab=Overview) (Это приложение предоставляет возможность курируемой работы с понятными путями действий для выбранного набора пользователей.)

     - Тестовый выпуск
        - Ваше решение (или решение вашего партнера) либо его часть можно упаковать с помощью шаблонов Azure Resource Manager, экземпляры которых может создать AppSource. AppSource может управлять приложением в партнерской подписке с ограничением по времени и с поддержкой пула холодных и горячих экземпляров и т. п.
        - Если вы выберете этот способ, мы можем предоставить подходящие шаблоны и примеры кода.

        **Примеры**

        - [Neal Analytics Inventory Optimization](https://appsource.microsoft.com/product/web-apps/neal_analytics.8066ad01-1e61-40cd-bd33-9b86c65fa73a?tab=Overview&tag=CISHome)

- Для варианта **Request for trial** (Запрос пробной версии), известный также как **пробное использование под руководством партнера (Partner Led Trials / PLT)**, клиенты должны заполнить форму контактных сведений для последующего взаимодействия с партнером. Партнер отвечает и предоставляет демонстрационную или пробную версию приложения. Дополнительные сведения см. в видео [AppSource trial experience walkthrough](https://aka.ms/trialexperienceforwebapps) (Обзор возможностей пробного использования AppSource) с общим обзором.

>[!Note]
>Согласно результатам, **пробное использование под руководством клиента** имеет более высокий потенциал привлечения потенциальных клиентов, чем **пробное использование под руководством партнера**.

### <a name="document"></a>Документ

Можно отправить не более трех документов. Каждый документ должен быть в формате файла PDF. Чтобы добавить новый документ, выполните следующие действия:

- Выберите **+ New** (+ Создать).

<a name="publishing-a-container-image-in-the-cloud-partner-portal"></a>Публикация образа контейнера на Портале Cloud Partner
========================================================

В этой статье описывается, как опубликовать новый образ контейнера на Портале Cloud Partner.

Выполните указанные ниже действия, чтобы опубликовать новый образ контейнера.

1. Выберите **Новое предложение**, а затем выберите **Контейнеры**.

    ![Выбор параметра контейнеров для нового предложения](media/cpp-containers-guide/azure-container-offer.png)

2. На вкладке "Параметры предложения" в разделе Offer Identity (Идентификатор предложения) заполните поля **Идентификатор предложения**, **Идентификатор издателя** и **Имя**.

    ![Идентификатор предложения](media/cpp-containers-guide/containers-offer-settings.png)

3. На вкладке "Номера SKU" выберите **New SKU** (Новый номер SKU).
    >[!NOTE]
    >Вы можете добавить несколько номеров SKU.

    ![Запрос нового номера SKU](media/cpp-containers-guide/containers-sku-settings.png)

4. Укажите номер SKU и сведения о контейнере. Каждый номер SKU соответствует образу контейнера. Номер SKU состоит из двух частей:

    -   метаданные номера SKU;
    -   метаданные контейнера.

    Метаданные номера SKU содержат отображаемые сведения для образов контейнера.

    ![Метаданные номера SKU](media/cpp-containers-guide/containers-sku-details.png)

    В метаданных контейнера содержится справочная информация о репозитории образа внутри Реестра контейнеров Azure (ACR). Затем Azure Marketplace копирует этот образ в общедоступный реестр Marketplace и делает его доступным для клиентов после сертификации. Все запросы от пользователя Azure на использование образа контейнера обслуживаются из реестра контейнеров Marketplace.

   ![Метаданные контейнера](media/cpp-containers-guide/containers-image-repository.png)

    В сведениях о репозитории образа на предыдущем снимке экрана содержатся следующие поля:

    -   **Идентификатор подписки** — идентификатор подписки Azure, в котором содержится реестр ACR.
    -   **Имя группы ресурсов** — имя группы ресурсов реестра ACR.
    -   **Имя реестра** — имя реестра ACR.
    -   **Имя репозитория** — имя репозитория. Уже заданное значение невозможно изменить. Необходимо присвоить уникальное имя, чтобы оно не совпадало с именем какого-либо другого предложения в учетной записи.
    -   **Имя пользователя** — имя пользователя, связанное с ACR (имя администратора).
    -   **Пароль** — пароль, связанный с ACR.

    >[!NOTE]
    >Имя пользователя и пароль необходимы для того, чтобы убедиться, что участники имеют доступ к реестру ACR, указанному в процессе публикации.

    При публикации образа контейнера можно также предоставить один или несколько тегов образа. В дополнение к тегу образа участники также могут предоставлять хэш-код агента работоспособности системы. Обязательно добавьте **тег тестирования** к образу, чтобы образ можно было легко определить во время тестирования.

5. На вкладке Marketplace добавьте маркетинговое содержимое.

    ![Сведения о Marketplace](media/cpp-containers-guide/containers-marketplace-tab.png)

6. На вкладке "Поддержка" введите контакты специалистов технического отдела и сведения о поддержке для клиентов.

   ![Сведения о поддержке](media/cpp-containers-guide/containers-support-tab.png)

7. Выберите **Опубликовать**, чтобы опубликовать предложение. После того как вы выберите "Опубликовать", появится временная шкала, где показаны шаги, связанные с публикацией образа контейнера.

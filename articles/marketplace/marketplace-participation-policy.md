# <a name="azure-marketplace-participation-policies"></a>Политики участия в Azure Marketplace

Данные политики участия в Microsoft Azure Marketplace действуют для всех издателей и предложений в магазине Microsoft Azure Marketplace. Эти политики являются дополнением к условиям и положениям соглашения с издателем для Microsoft Marketplace. Для участия в Azure Marketplace издатели всегда должны придерживаться политик, которые описаны или упомянуты в данном документе. Если издатель не соответствует всем условиям и положениям в определенный момент времени, то корпорация Майкрософт вправе удалить предложение издателя из Azure Marketplace. Этот документ время от времени обновляется.

## <a name="base-criteria"></a>Базовые критерии

    * *Выполнение в Microsoft Azure*. Основная функция ПО или службы должна выполняться в Microsoft Azure.
    * *Развертывание в Microsoft Azure*. Издатель должен указать в объявлении о своем предложении сведения о развертывании ПО или службы в Microsoft Azure.
    * *Интеграция со службой Microsoft Azure или ее расширение*. В объявлении о предложении издатели должны указать, с какой службой Azure интегрируется программное обеспечение или служба или какую службу расширяет и как оно это делает.

1. Предложения издателя в Azure Marketplace должны иметь ограниченную или общую доступность и устоявшуюся клиентскую базу.

1. Предложения в Azure Marketplace не могут использовать какой-либо продукт или компонент, который не поддерживается или более не является коммерчески доступным, или зависеть от него.

1. Издатели должны предоставить подробную техническую документацию.  В документации должно быть описано, как использовать их предложения в Microsoft Azure.  Каждое предложение должно предоставлять или ссылаться на такую документацию в своем объявлении.

1. Издатель должен сообщить о доступности своего предложения в Azure Marketplace на своем публичном веб-сайте и включить гиперссылку на страницах объявлений с предложением на сайте [https://azuremarketplace.microsoft.com/marketplace/](https://azuremarketplace.microsoft.com/marketplace/).

## <a name="publishing-offers"></a>Публикация предложений

1. Издатель должен опубликовать по меньшей мере одно предложение в Azure Marketplace в течение 60 дней с момента принятия соглашения с издателем для Microsoft Marketplace.

1. Издатель должен следовать техническим требованиям Azure Marketplace для принятия предложения, как описано в руководстве по публикации в Marketplace и дополнительно может быть обозначено на портале публикации.

## <a name="offer-listings"></a>Списки предложений

1. Издатель должен включить подробное описание предложения в содержание страниц с объявлениями, которое должно быть точным и актуальным. Подобная информация должна включать следующие сведения (по мере необходимости):
    * *Описание предложения*
        * Краткое описание предложения
            * Сведения о номере SKU
            * Предлагаемые преимущества
        * Рекомендованное описание предложения
            * Подробные сведения о номере SKU
            * Подробное описание предлагаемых преимуществ
            * Функции: 3–5 фактических утверждений о предложении
            * Преимущества: 3–5 результатов работы функций предложения
    * *Модель ценообразования*. Предложение должно быть совместимым с [моделями ценообразования](./marketplace-faq-publisher-guide.md), поддерживаемыми в Azure Marketplace, как описано в разделе **Модели ценообразования** данного документа.
    * *Ссылка на сведения о службе поддержки клиентов*. Издатели должны предоставлять коммерчески оправданную поддержку клиентов для своих предложений в Azure Marketplace.  Она может оплачиваться как часть функциональности предложения или как предложение поддержки, приобретенное отдельно.
    * *Ресурсы предложения*. К ресурсам относятся (но не ограничиваются ими): демонстрационные видеоролики, снимки экрана, технические документы, анализ примеров из практики, рекомендации и подробная техническая документация о том, как использовать предложение издателя в Microsoft Azure.
    * *Порядок возврата средств клиентам*.
    * *Условия использования*.
    * *Политика конфиденциальности*.

1. Издатель не может со своей страницы с объявлением о предложении производить перенаправление на другую страницу или дополнительно продавать клиентам Azure другие ПО и службы, кроме тех, которые доступны в Azure Marketplace. Это ограничение не относится к службам поддержки, которые издатели предлагают за пределами Azure Marketplace.

1. Издатели не могут рекламировать в магазине Azure Marketplace свои предложения на других облачных платформах.

1. Корпорация Майкрософт оставляет за собой право проверять и изменять сведения на страницах объявлений о предложении для контроля качества. Если корпорация Майкрософт вносит какие-либо изменения в объявление о предложении, то она должна уведомить об этом издателя перед публикацией объявления о его предложении в Azure Marketplace.

## <a name="offering-classification"></a>Классификация предложений

|     |    |
| :------------------- | :-------------------|
| **Образ виртуальной машины** | Предварительно настроенный образ виртуальной машины (ВМ) с полностью установленной операционной системой и с одним или несколькими приложениями. Предложения образов виртуальных машин могут включать один или несколько образов ВМ, объединенных шаблоном Resource Manager. <br> <br> Образ виртуальной машины ("Образ") предоставляет информацию, необходимую для создания и развертывания виртуальных машин в службе "Виртуальные машины Azure". Образ состоит из операционной системы, виртуального жесткого диска и нуля или большего количества виртуальных жестких дисков данных. Клиенты могут развертывать любое число виртуальных машин из одного образа. |
| **Расширение виртуальной машины** | Агенты ВМ, которые могут быть добавлены к новым ВМ несколькими способами, включая добавление через REST API, портал Azure или командлеты Azure PowerShell. Расширения ВМ также можно установить вручную на имеющиеся ВМ и настроить для работы с ВМ на базе Windows Server или Linux. <br> <br> Расширение виртуальной машины ("**Расширение ВМ**") — это механизм для установки программного приложения или пакета приложений на виртуальные машины Azure. Расширение ВМ может включать в себя приложения ПО.  После установки внутри виртуальной машины или выполнения на ней оно может скачивать и устанавливать приложения ПО из внешнего расположения. Во избежание неясных толкований любое программное обеспечение или данные, устанавливаемые вашим Расширением ВМ, считаются содержимым Предложения в рамках данного Соглашения даже при их получении из внешнего местоположения. Вы ответственны и должны осуществлять поддержку Клиентов для всех обработчиков Расширения ВМ, связанных с вашими Предложениями Расширения ВМ. |
| **Службы** | Полностью управляемые службы для информационных работников, бизнес-аналитиков, разработчиков и ИТ-профессионалов для использования при разработке приложений или управлении системами. Marketplace поддерживает три типа служб. <br> <br> **Службы приложений** предоставляют клиентам возможность быстро разрабатывать в Azure масштабируемые облачные приложения. <br> <br> Для приобретения Служб приложений клиент должен иметь подписку Azure. Издатель отвечает за экспозамер использования клиентами Служб приложений и должен передавать эти сведения об использовании в корпорацию Майкрософт, как описано в соглашении для издателя в Microsoft Marketplace.|
| **Веб-приложение** | Пакет приложений, который можно использовать для установки и развертывания веб-сайтов с открытым кодом или защищенным правом собственности и платформ управления в службе веб-сайтов Azure. Веб-приложения должны соответствовать [принципам коллекции веб-приложений](https://www.iis.net/learn/develop/windows-web-application-gallery/windows-web-application-gallery-principles) Майкрософт. <br> <br> В рамках данного Соглашения "**Веб-приложение**" — это пакет приложений, который Клиенты используют для установки и развертывания веб-приложений (c открытым кодом или защищенным правом собственности) в службе веб-сайтов Azure. |
| **Объявление каталога** | Предложения, которые недоступны для клиентов Azure непосредственно в Marketplace, но на Marketplace отображается ссылка, значок и объявление программного обеспечения или услуги.  Клиенты направляются на веб-сайт издателя или получают инструкции о том, как получить и использовать предложение в Azure. "**Объявление только для каталога**" — это Предложение, которое недоступно для Клиентов непосредственно в Marketplace. Вместо этого в Marketplace отображается ссылка, значок или описание, которое помогает попасть клиентам на ваш сайт. Кроме того, предоставляются инструкции о том, как клиенты могут получить Предложение в Azure и использовать его. Во избежание разночтений любое программное обеспечение или данные, указанные в объявлении только для каталога, в рамках данного Соглашения считаются содержимым Предложения. |
| **Шаблон Azure Resource Manager (Resource Manager)** | Шаблон Resource Manager может ссылаться на несколько уникальных предложений, включая предложения, опубликованные другими издателями.  Это позволяет клиентам Azure развертывать одно или несколько предложений одним согласованным образом. <br> <br> "**Шаблон Azure Resource Manager (Azure Resource Manager)**" — это структура данных, которая ссылается на одно или несколько Предложений и содержит метаданные о Предложениях.  Структура данных связана с объявлением о предложении. Шаблон Resource Manager используются службой Marketplace для показа и развертывания клиентами определенных категорий Предложений. Издатели могут публиковать шаблоны ARM в Marketplace, которые ссылаются на несколько различных Предложений, включая предложения, опубликованные другими издателями. |

## <a name="pricing-models"></a>Модели ценообразования

В следующей таблице приведены модели ценообразования, поддерживаемые в Azure Marketplace в данный момент. Предложение может содержать различные номера SKU c разными моделями ценообразования.

| **Модель ценообразования** | **Описание** | **Применимо к** |
| :---------------- | :---------------| :-----------------|
| **Бесплатный** | Номер SKU уровня "Бесплатный". С клиентов не взимается плата Azure Marketplace за использование предложения. Цены для бесплатных номеров SKU нельзя установить больше нуля. | <ul><li>Образы виртуальных машин</li><li>Расширения виртуальных машин</li><li>Службы</li><li>Шаблоны диспетчера ресурсов</li></ul> |
| **бесплатной пробной версии** <br> **(Попробуйте сейчас)** | Рекламные бесплатные номера SKU на ограниченный период времени. С клиентов не взимается плата Azure Marketplace за использование предложения в течение пробного периода. После истечения пробного периода с клиентов автоматически будет взиматься стандартная плата за пользование предложением. На данный момент в Marketplace нет возможности предотвратить создание клиентом нескольких подписок для предложений с бесплатным пробным периодом. Издатель, желающий ограничить число подписок на бесплатный пробный период для клиентов, должен включить соответствующие ограничения в свои условия использования. | <ul><li>Образы виртуальных машин</li><li>Службы</li></ul> |
**BYOL** | Номер SKU типа BYOL (с использованием собственной лицензии). Если они получили доступ или используют предложение за пределами Azure Marketplace, с клиентов не взимается плата Azure Marketplace. | Образы виртуальных машин |
| **Ежемесячная подписка** | С клиента взимается фиксированная ежемесячная плата за подписку на предложение. Ежемесячные подписки начинаются с даты покупки, за исключением случаев, описанных ниже. Ежемесячная оплата не распределяется пропорционально при отмене клиентом подписки в середине месяца или если службу не использовать. Месячная плата может распределяться, если в соответствии с условиями лицензии клиента счета должны выставляться за календарный месяц или если клиент обновит подписку или изменит ее уровень на более низкий в середине месяца. Обновление и изменение уровня на более низкий поддерживается только при соответствующей настройке решения издателем. | Службы |
| **На основе использования** | С клиента взимается плата по факту использования предложения. Для образов виртуальных машин с клиентов взимается почасовая плата Azure Marketplace, устанавливаемая издателями, за использование виртуальных машин, развернутых из образов. Почасовая плата может варьироваться в зависимости от размера виртуальной машины или же может быть унифицированной. Издатель может задать несколько единиц измерения для одного плана службы приложений. Издатели несут ответственность за отслеживание отдельных клиентов, каждая единица измерения определяется предложением. Корпорация Майкрософт взимает плату с клиентов на основе сведений об использовании, переданных издателем за определенный период выставления счетов. | <ul><li>Службы</li><li>Образы виртуальных машин</li></ul> |

Издатели теперь могут снизить плату пользователей за уже опубликованные предложения виртуальных машин. Все другие типы изменений цен для имеющихся предложений в настоящее время не поддерживаются Marketplace. Издатели, которые хотят изменить плату пользователей за предложение, должны сначала удалить его из Marketplace.  Удаление следует выполнить в соответствии с требованиями соглашения с издателем для Microsoft Marketplace и этим документом.  Затем издатель может опубликовать новое предложение, которое содержит новую пользовательскую плату.

Опубликовав предложение служб в Marketplace, издатель должен вести журналы данных с минимальным периодом в 2 календарных года. Журналы данных содержат подготовку Предложений для клиентов. При несовпадении данных в журналах данных издателя и журналах Майкрософт приоритет имеют последние.

## <a name="offering-suspension-and-removal"></a>Приостановка и удаление предложения

1. Корпорация Майкрософт оставляет за собой право приостановить или удалить предложение из Azure Marketplace по любой причине. Причины возможного удаления предложения приведены ниже (но не ограничиваются только ими).
    * Предложение не подготавливалось ни одним клиентом в течение шести месяцев и более.
    * Предложение имеет высокую частоту отмены для своих платных номеров SKU.
    * Предложение постоянно получает негативные отзывы клиентов.
    * Предложение постоянно получает много запросов в службу поддержки.
    * Издатель не соответствует положениям и условиям соглашения для издателя в Microsoft Marketplace, руководства по публикации в Marketplace или настоящего документа.

1. По различным причинам вам может понадобиться удалить предложение из Marketplace. Удалив предложение вы гарантируете, что новые клиенты больше не смогут приобрести или развернуть ваше предложение (однако это никак не повлияет на существующих клиентов). Завершение предложения — это процесс прекращения действия соглашения об обслуживании и/или лицензировании между вами и существующими клиентами. Рекомендации и политики относительно удаления или прекращения действия решения регулируются соглашением с издателем для Microsoft Marketplace (см. раздел 7). Чтобы запросить удаление или прекращение действия, отправьте запрос в службу поддержки.

## <a name="taxes"></a>Налоги

1. **Страны с ремитированием (Майкрософт).**

    1. Следующие страны являются странами с ремитированием (Майкрософт), как определено в соглашении с издателем для Microsoft Marketplace: страны ЕС, Канада, Индия, Ирландия, Лихтенштейн, Монако, Новая Зеландия, Норвегия, Пуэрто-Рико, Южная Корея, Швейцария, США. Австралия, Саудовская Аравия, Тайвань и Объединенные Арабские Эмираты включены в список стран с ремитированием (Майкрософт) только для продаж Web Direct.

    1. Клиенты из Ирландии совершают выплаты в корпорацию Майкрософт в соответствии со ставкой НДС Ирландии (если не предоставлены сведения о действующей справке о присвоении номера НДС 56Б). При предоставлении действительного номера плательщика НДС бизнес-клиенты из стран ЕС и Норвегии (за пределами Ирландии) освобождаются от выплаты НДС. В противном случае клиенты облагаются налогом по ставке, действующей в стране клиента.

    1. Корпорация Майкрософт взимает швейцарский НДС со всех клиентов в Швейцарии и Лихтенштейне.

1. **Страны-независимые поставщики программного обеспечения с ремитированием.**

    1. К странам-независимым поставщикам программного обеспечения с ремитированием относятся все страны (на территории которых можно продавать товары Marketplace), не являющиеся странами с ремитированием Майкрософт, как указано в разделе 1 (3) этой политики. Кроме того, продажи в Австралии, Саудовской Аравии, Тайване и Объединенных Арабских Эмиратах по корпоративному каналу считаются продажами независимым поставщикам программного обеспечения с ремитированием. Тем не менее корпорация Майкрософт не может подтвердить, что тайваньские издатели собирают и перечисляют применимые налоги.

    1. Если издатели решили сделать свое предложение доступным в Стране для денежных переводов независимого разработчика программного обеспечения, то на них возлагается ответственность за определение своих налоговых обязательств в Стране для денежных переводов независимого разработчика программного обеспечения (если таковые имеются).  Они должны признать, что корпорация Майкрософт в настоящее время не может предоставить подробности транзакции, необходимые издателям для предоставления клиентам налоговых накладных.

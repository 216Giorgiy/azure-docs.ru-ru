# <a name="azure-marketplace-sellers-guide"></a>Руководство по Azure Marketplace для продавцов

Вас приветствует руководство по Microsoft Azure Marketplace для продавцов. Это руководство предназначено для бизнес-пользователей и менеджеров по продуктам, независимых поставщиков программного обеспечения (ISV), заинтересованных в продаже образов виртуальных машин c сертификацией Azure ИТ-специалистам и разработчикам. Благодаря клиентам Azure по всему миру [Azure Marketplace](https://azuremarketplace.microsoft.com/) предоставляет широкий охват аудитории для распространения ваших продуктов.


> [!NOTE]
> Если вы заинтересованы в продаже готового программного обеспечения как услуги (SaaS) для бизнес-пользователей, рассмотрите возможность размещения вашего ПО в [AppSource](https://appsource.microsoft.com).

- Как создавать совместимые с Azure виртуальные жесткие диски (VHD)?
- Как подать заявку и зарегистрироваться в качестве продавца?
- Как создать и опубликовать предложение?
- Как выйти на рынок и найти доступные ресурсы?
- Какие отчеты и аналитические сводки предоставляет Azure Marketplace?
- Где можно получить справку и поддержку?

Приступим.

## <a name="whats-the-azure-marketplace"></a>Что такое Azure Marketplace?

Azure Marketplace — это интернет-магазин приложений и служб, который позволяет независимым поставщикам программного обеспечения (ISV), начиная от разработчиков, чьи проекты находятся на начальном этапе развития, до целых предприятий, предлагать свои решения клиентам Azure по всему миру. Azure Marketplace позволяет издателям Azure предлагать и продавать свои инновационные образы виртуальных машин другим разработчикам, независимым продавцам ПО и ИТ-специалистам, которые хотят быстро разрабатывать облачные приложения и мобильные решения в Azure. Azure Marketplace поддерживает широкий спектр предложений, например комплексные приложения для анализа данных с обработкой данных, хранилищем данных и уровнями аналитики, а также многоуровневые приложения для электронной торговли (данные, службы и Интернет).

Облачные клиенты сталкиваются с рядом трудностей при поиске решения в соответствии с их потребностями. Azure Marketplace предоставляет механизм для решения этих проблем и подключения клиентов с помощью инновационных решений независимых поставщиков программных продуктов, как описано в следующей таблице.

| Потребность клиента | Решение Azure Marketplace |
| --- | --- |
| Необходимы дополнительные функции облачной платформы для удовлетворения потребностей бизнеса и технических требований | Предлагает постоянно расширяющийся портфель дополнительных приложений и служб в Azure |
| Сталкивается с трудностями поиска необходимого приложения или службы | Предоставляет единый центр служб для поиска и покупки приложений и служб |
| Требуется механизм масштабируемого развертывания для сторонних приложений и служб | Позволяет создавать и настраивать масштабируемые развертывания для сторонних приложений и служб |
| Требуется интеграция новых приложений и служб для работы с существующими решениями | Легко интегрирует сторонние приложения и службы с существующими решениями на платформе Azure |

Azure Marketplace обеспечивает качество, широкий выбор и надежность партнерской экосистемы Azure для клиентов по всему миру и дает такие преимущества:

- единое расположение для предложений на базе Azure от корпорации Майкрософт и партнеров;
- более 5000 предложений;
- возможности интегрированной платформы;
- упрощенную настройку, развертывание и управление.

## <a name="is-the-azure-marketplace-right-for-my-business"></a>Подходит ли Azure Marketplace для моего бизнеса?

Если Azure Marketplace является для вас верным выбором, что в результате вы сможете получить? Azure Marketplace обеспечивает новые возможности продаж, предлагая клиентам ваше программное обеспечение и службы, доступные в Azure.

- **Продавайте на новых клиентских рынках с помощью широкого портфеля решений в Azure.** Это возможность увеличить суммы покупок и перекрестные продажи предложений Marketplace с помощью служб Azure, доступных для клиентов Microsoft Online Subscription Program (MOSP) и Microsoft Enterprise Agreement. Предложения Marketplace можно легко включить в решение клиента и в сценарий презентации Azure.
- **Повышайте ценность бизнеса и увеличивайте суммы сделок для существующих и новых учетных записей клиентов.** Azure Marketplace может помочь увеличить суммы сделок, обнаружить проблемные вопросы клиента при перемещении рабочих нагрузок в облако и повысить рентабельность сделок. Иными словами, вы можете повысить ценность бизнеса путем продажи готовых решений и устранения критических упущений облачной платформы Azure в соответствии с требованиями заказчика.
- **Привлекайте более широкий круг потенциальных клиентов посредством продажи приложений и служб в Marketplace.** В Azure Marketplace проще искать и удерживать новых клиентов. Сегодня многим компаниям необходимо переносить рабочие нагрузки в облако и адаптироваться к постоянно меняющимся инфраструктурным средам. Вы сможете предоставить им приложения и службы, помогающие повысить эффективность.
- **Дополняйте и расширяйте функциональность Azure, объединив предложения Marketplace со службами Azure.** Azure Marketplace поможет выработать стратегию взаимоотношений с клиентами и эффективно налаживать контакты. Вы также можете устранить недостатки определенных платформ и удовлетворить потребности клиентов, продвигая важные комплексные решения. Продавая пакеты решений, вы максимально эффективно реализовываете потенциал экосистемы платформы Azure, что позволяет решить практически любую проблему клиента и увеличить объем продаж.

## <a name="what39s-the-customer-base-for-the-azure-marketplace"></a>Кто входит в клиентскую базу Azure Marketplace?

Услугами Azure Marketplace пользуется множество различных людей. Так как клиентская база Azure является одной из самых быстрорастущих среди всех поставщиков облачных услуг, вы получаете доступ к бесчисленному множеству ИТ-специалистов и разработчиков — от начинающих компаний до предприятий, как в государственном, так и в частном секторе.

## <a name="how-does-the-azure-marketplace-work"></a>Как работает Azure Marketplace?

Довольно просто. После получения доступа создайте образ виртуальной машины, сертифицированный Azure, и опубликуйте его в Azure Marketplace, где клиенты Azure смогут быстро его найти, купить и развернуть. Более того, клиенты развернут решение, зная, что среда адаптирована для работы в Azure и инфраструктура будет готова к работе в течение нескольких минут.

Облачный портал для партнеров (CPP) выступает в качестве центра для создания ваших предложений в Azure Marketplace. Образы виртуальных машин предварительно настроены с полностью установленной операционной системой и одним или несколькими приложениями. Чтобы сертифицировать образ для публикации, он должен соответствовать определенным требованиям, которые мы обсудим в следующем разделе данного руководства.

### <a name="technical-prerequisites"></a>Технические условия

Внимательно ознакомьтесь со всеми техническими условиями и выполните их перед публикацией своего предложения. Вам потребуется доступ к Windows или Linux, а также совместимые с Azure виртуальные жесткие диски (VHD), связанные со средствами для тестирования.

Хотя виртуальные жесткие диски Azure можно создавать и в облачной, и в локальной среде, мы советуем разрабатывать их непосредственно в облаке, используя протокол удаленного рабочего стола. Однако при необходимости вы можете скачать образ VHD и выполнить разработку в [локальной инфраструктуре](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-vm-image-creation-on-premise).

Более подробные сведения см. в статье [Технические компоненты, необходимые для создания образа виртуальной машины для Azure Marketplace](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-vm-image-creation-prerequisites).

### <a name="non-technical-prerequisites"></a>Нетехнические предварительные требования

Чтобы стать участником Azure Marketplace, потребуется выполнить ряд нетехнических предварительных требований. Во-первых, просмотрите и примите условия [политик участия в Azure Marketplace](https://azure.microsoft.com/support/legal/marketplace/participation-policies/). Программное обеспечение и службы, предлагаемые в магазине Azure Marketplace, должны соответствовать по меньшей мере одному из следующих критериев.

- **Выполнение в Microsoft Azure:** основная функция ПО или службы должна выполняться в Microsoft Azure.
- **Развертывание в Microsoft Azure:** необходимо указать в объявлении о своем предложении сведения о развертывании ПО или службы в Microsoft Azure.
- **Интеграция со службой Microsoft Azure или ее расширение:** необходимо указать в объявлении о своем предложении, с _какой_ службой Azure интегрируется его ПО или служба либо _как_ происходит интеграция или расширение ПО или службы в службе Azure.

Необходимо также соответствовать бизнес-требованиям, описанным в политиках участия в Azure Marketplace.

- Ваша компания (или ее филиал) должна находиться в стране, на территории которой могут осуществляться продажи в Azure Marketplace.
- Продукт должен иметь лицензию, совместимую с моделями выставления счетов, которые поддерживает Azure Marketplace.
- Вы должны предоставить клиентам техническую поддержку экономически обоснованным образом (на платной или бесплатной основе либо посредством оказания поддержки ресурсами сообщества).
- Требуется лицензировать программное обеспечение, а также любые зависимости от стороннего программного обеспечения.
- Предоставьте материалы, отвечающие критериям для вашего предложения, соблюдение которых требуется для представления приложения на веб-сайте [azure.microsoft.com](../../C:/Users/Lisa.Rosenberger/Desktop/azure.microsoft.com) и на портале управления Azure.

Наконец, необходимо принять [условия использования](https://azure.microsoft.com/support/legal/website-terms-of-use/), [Заявление о конфиденциальности Майкрософт](http://www.microsoft.com/privacystatement/default.aspx) и [соглашение по программе сертификации Microsoft Azure](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/).

### <a name="virtual-machine-image-preparation"></a>Подготовка образа виртуальной машины

Теперь, когда вы ознакомились со списком обязательных компонентов и выполнили необходимые задачи, можете переходить к созданию образа виртуальной машины для предложения, как описано в [руководстве по публикации образа виртуальной машины](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-vm-image-creation).

Это руководство поможет вам подготовить VHD для развертывания в Azure Marketplace, который станет основой вашего номера SKU. 

> [!NOTE]
> Процесс подготовки будет отличаться в зависимости от типа номера SKU (на основе Linux или Windows).

> [!NOTE]
>  Эту процедуру можно выполнять параллельно с [созданием учетной записи разработчика Майкрософт](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-accounts-creation-registration), как описано ниже.

### <a name="azure-certification"></a>Сертификация Azure

Получение статуса _Сертифицировано для Azure_ означает, что вы успешно прошли процесс адаптации. Этот статус заверяет клиентов, что их ИТ-специалисты и разработчики получат качественные решения, работающие или созданные на базе технологии Azure от доверенного партнера. Сертификация решения в Azure означает:

- тщательную проверку;
- совместимость с платформой Azure;
- соответствие стандартам безопасности сетевого образа;
- отсутствие вирусов и вредоносных программ;
- поддержку модели выставления счетов.

## <a name="how-do-i-nominate-my-product-and-get-approved"></a>Как предложить свой продукт и получить одобрение?

Теперь пришло время получить одобрение на продажу продуктов в Azure Marketplace (рис. 3). Корпорация Майкрософт позволяет легко предлагать продукты, выполнять публикацию и регистрироваться в качестве продавца.

![Получение одобрения на продажу в Azure Marketplace](./media/cloud-partner-portal-seller-guide/gettingapprovedsteps.png)

Рис. 3. Получение одобрения на продажу в Azure Marketplace

Первый этап получения одобрения — [предложить](https://createopportunity.azurewebsites.net/) продукт до регистрации и публикации. Имейте в виду, что получение одобрения может занять _до трех рабочих дней_.

После получения одобрения вы получите следующее:

- Квитанцию с промокодом на оплату 99 долл. США регистрационного сбора для центра разработки по электронной почте и профиль CPP.
- Предварительное техническое одобрение статуса сертификации в Azure, а также возможность создать предложение и сертифицировать ваш VHD. (Ваше приложение для центра разработки должно быть одобрено, прежде чем вы сможете создать предложение.)
- Инструкции по публикации, сведения об учетной записи портала публикации и обзор процесса публикации.
- Возможность связаться с командой адаптации Майкрософт, чтобы получить пошаговые инструкции и задать вопросы.
- Возможность публикации следующего предложения. Следующим предложениям не требуется проходить процедуру одобрения. Они могут непосредственно размещаться на портале CPP, но виртуальные машины по-прежнему должны быть сертифицированы в процессе публикации.
- Руководство по получению помощи при публикации. (Вопросы следует направлять издателю Marketplace по [ссылке](https://support.microsoft.com/getsupport?wf=0&amp;tenant=ClassicCommercial&amp;oaspworkflow=start_1.0.0.0&amp;locale=en-us&amp;supportregion=en-us&amp;pesid=15635&amp;ccsid=636233723471685249).)

Наконец, [зарегистрируйте свою учетную запись](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-accounts-creation-registration) продавца Майкрософт. Процесс одобрения и проверка могут занять _до двух недель_, поэтому используйте это время для создания вашего предложения для Azure Marketplace на портале CPP.

[Тестовые выпуски в Azure](https://azure.microsoft.com/marketplace/test-drives/) — это отличный способ продемонстрировать продукт потенциальным клиентам и предоставить им возможность попробовать его перед покупкой. Это поможет повысить вероятность превращения потенциального покупателя в реального и создаст высокоэффективные интересы.

После предоставления своих контактных данных клиенты получат доступ к предварительно созданному тестовому выпуску: самостоятельно протестируют пробную версию продукта, ознакомятся с ключевыми функциями и преимуществами на практике. Таким образом, тестовые выпуски в Azure позволяют потенциальным клиентам самостоятельно ознакомиться с продуктом.

В настоящее время публикация тестовых выпусков доступна только на классическом портале публикации. Документацию о том, как это сделать, можно найти [здесь](https://github.com/Azure/AzureTestDrive/wiki).__

Дополнительные сведения о тестовых выпусках в Azure см. [здесь](https://azuremarketplace.azureedge.net/documents/azure-marketplace-test-drive-program.pdf).

## <a name="go-to-market-checklist"></a>Контрольный список для выхода на рынок

Узнайте о наших [программах выхода на рынок](https://partner.microsoft.com/go-to-market/), которые позволят расширить глобальный охват вашей организации. Можно также использовать ресурсы в [Partner Marketing Center](http://smartpartnermarketing.microsoft.com/isv).

Перед началом работы мы советуем выполнить несколько действий, чтобы продвинуть ваше предложение в Azure Marketplace. Узнайте, готовы ли вы выйти на рынок с предложением на основе краткого контрольного списка ниже.

- **Вы объявили о том, что ваше предложение доступно в Azure Marketplace** посредством блога, сообщения электронной почты или публикации пресс-релиза.
- **Вы рекламируете предложение на своем собственном веб-сайте**, перенаправляя клиентов к предложению в Azure Marketplace, где можно его изучить, протестировать, приобрести и развернуть.
- **Вы опубликовали тестовый выпуск**, чтобы пользователи могли протестировать предложение в реальном времени в Azure во время перерыва на кофе.
- **Вы включили создание интересов по запросу**, чтобы каждый раз, когда клиенты развертывают приложение, получать их контактные данные.
- **Вы связались со своим менеджером по работе с партнерами** в корпорации Майкрософт (если он есть), чтобы узнать о дополнительных возможностях.

## <a name="what-about-reports"></a>Сведения об отчетах

Azure Marketplace предоставляет отчеты о заказах, об использовании и о клиентах, которые доступны на [портале отчетов для издателя](https://reports.azure.com) Azure Marketplace (рис. 5). Необработанные данные предоставляются в таблицах с возможностью перемещения и могут быть загружены в формате CSV или XLS.

# <a name="compare-azure-iot-central-and-azure-iot-options"></a>Сравнение вариантов Azure IoT Central и Azure IoT

Microsoft Azure IoT Central и Azure IoT предоставляют несколько вариантов реализации решения Интернета вещей. Каждый вариант соответствует разным наборам требований клиентов:

* [Azure IoT Central](overview-iot-central.md) — это программное обеспечение как услуга (SaaS), которое использует подход на основе модели для создания решений Интернета вещей корпоративного уровня, не требуя опыта в разработке облачных решений.

* [Azure IoT Suite](https://docs.microsoft.com/azure/iot-suite/) — это коллекция [предварительно настроенных решений](https://docs.microsoft.com/azure/iot-suite/iot-suite-what-are-preconfigured-solutions) корпоративного уровня, созданных на базе платформы как услуги (PaaS) Azure, которые позволяют ускорить разработку пользовательских решений Интернета вещей.

## <a name="azure-iot-hub"></a>Центр Интернета вещей Azure

Центр Интернета вещей Azure — это базовая платформа Azure PaaS, которую используют Azure IoT Central и Azure IoT Suite. Центр Интернета вещей обеспечивает надежную и безопасную двунаправленную связь между миллионами устройств Интернета вещей и облачным решением. Он помогает решить трудности реализации Интернета вещей, такие как:

* взаимодействие устройств большого объема и управление ими;
* прием больших объемов данных телеметрии;
* контроль и управление устройствами;
* обеспечение безопасности устройства.

## <a name="compare-azure-iot-central-and-azure-iot-suite"></a>Сравнение Azure IoT Central и Azure IoT Suite

Выбор продукта Интернета вещей Azure является важной частью планирования решения Интернета вещей. Центр Интернета вещей — это отдельная служба Azure, которая сама по себе не предоставляет комплексное решение Интернета вещей. Центр Интернета вещей можно использовать в качестве отправной точки для любого решения Интернета вещей. Для его применения вам не требуется Azure IoT Suite или Azure IoT Central. IoT Suite и Azure IoT Central используют Центр Интернета вещей наряду с другими службами Azure. В приведенной ниже таблице перечислены основные различия между IoT Suite и Azure IoT Central. Эти сведения помогут выбрать оптимальный вариант в соответствии с вашими требованиями:

|     | Azure IoT Central | Azure IoT Suite |
| --- | ----------- | --------- |
| Основное использование                      | Чтобы ускорить выход на рынок простых решений Интернета вещей, которым не требуется глубокая настройка службы.                                                    | Чтобы ускорить разработку пользовательского решения Интернета вещей, которому необходима максимальная гибкость.                                                                                                                             |
| Доступ к базовым службам PaaS | Программное обеспечение как услуга. Полностью управляемое решение, базовые службы не предоставляются.                                                                                            | У вас есть доступ к базовым службам Azure, можно управлять ими или заменить их при необходимости.                                                                                                                    |
| Гибкость                        | Средний. Для настройки модели решения и аспектов пользовательского интерфейса можно использовать встроенный в браузер пользовательский интерфейс. Инфраструктура не настраивается, так как различные компоненты не предоставляются. | Высокий. Код для микрослужб — это открытый исходный код, который можно изменять по своему усмотрению. Кроме того, можно настроить инфраструктуру развертывания.                                               |
| Уровень навыков                        | Низкий. Для настройки решения нужны навыки моделирования. Навыки программирования не требуются.                                                                          | Умеренно высокий. Требуются навыки работы с Java или .NET для настройки серверной части решения. Необходимы навыки JavaScript для настройки визуализации.                                                                       |
| Начало работы             | Шаблоны приложений и устройств содержат готовые модели. Может быть развернуто за несколько минут.                                                                                                  | Предварительно настроенные решения реализуют распространенные сценарии Интернета вещей. Может быть развернуто за несколько минут.                                                                                                                            |
| Цены                            | Простая и предсказуемая структура ценообразования.                                                                                                                           | Можно точнее настроить службы для управления затратами.                                                                                                                                                            |

В конечном итоге решение о том, какой продукт использовать для создания решения Интернета вещей, зависит от:

* бизнес-требований;
* типа создаваемого решения;
* набора навыков вашей команды по созданию и обслуживанию решения в долгосрочной перспективе.

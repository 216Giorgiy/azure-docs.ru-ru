# <a name="how-to-use-analytics-to-analyze-your-device-data"></a>Использование аналитики для анализа данных устройства

Microsoft Azure IoT Central предоставляет широкие возможности анализа для получения сведений о больших объемах данных, поступающих с устройств. Средства аналитики можно использовать для просмотра и анализа данных для [набора устройств](howto-use-device-sets.md) в приложении. Набор устройств — это определяемая пользователем группа устройств. Вы можете выполнить анализ для небольшого набора устройств или для одного устройства.

С правами оператора выберите **Аналитика** в меню навигации слева, щелкните набор устройств и выберите измерения, которые необходимо отобразить на графике. Ниже приведены несколько средств, которые можно использовать для дальнейшего среза данных.

* **Measurements** (Измерения). Выберите измерения для отображения, например данные температуры и влажности.
* **Агрегирование.** Выберите функцию агрегирования для измерений. Например, **Минимум** или **Среднее**.
* **Split by** (Разделить по). Выполняйте детализацию, разделив данные по свойствам или имени устройства. Например, можно разделить данные по расположению устройства:

     ![Главная страница аналитики](media\howto-create-analytics\analytics-main.png)

* **Диапазон времени.** Вы можете выбрать один из предварительно определенных диапазонов времени или создать настраиваемый. ![Диапазон времени для отображения аналитики](media\howto-create-analytics\analytics-time-range.png)

## <a name="change-the-visualizations"></a>Изменение визуализации

Вы можете изменить графики в соответствии со своими требованиями к визуализации, выбрав один из трех режимов:

* **Stacked** (С накоплением). Для каждого измерения выводится график с накоплением с собственной осью Y. Графики с накоплением удобно использовать при наличии нескольких выбранных измерений, а также при необходимости получить отдельное представление этих измерений.
* **Unstacked** (Без накопления). Для каждого измерения выводится график по оси Y, но значения для этой оси изменяются в зависимости от выделенного измерения. Графики без накопления полезны в тех случаях, когда необходимо наложение нескольких измерений, а также требуется просмотреть тенденции в этих измерениях в рамках одного диапазона времени.
* **Shared Y-axis** (Общая ось Y). Во всех графиках используется одна ось Y, и значения этой оси не изменяются. Графики с общей осью Y полезны в тех случаях, когда нужно просмотреть одно измерение, когда данные срезаны с помощью функции разделения.

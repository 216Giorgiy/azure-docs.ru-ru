# <a name="how-to-administer-your-application"></a>Администрирование приложения

После создания приложения Microsoft Azure IoT Central для его администрирования вы можете использовать раздел **Администрирование** пользовательского интерфейса Azure IoT Central. Чтобы перейти к разделу **Администрирование**, выберите **его** в левом навигационном меню.

В разделе **Администрирование** можно:

- Управление пользователями

- управлять ролями;

- просматривать сведения о выставлении счетов;

- управлять параметрами приложения;

- продлевать бесплатную пробную версию.

В разделе **Администрирование** есть дополнительное меню навигации со ссылками на различные задачи администрирования.

Чтобы получить доступ и использовать раздел **Администрирование**, вы должны быть в роли **Администратор** в приложении Azure IoT Central. Если вы создаете приложение Azure IoT Central, вам автоматически назначается роль **Администратор** для этого приложения. В разделе *Управление пользователями* в этой статье объясняется, как назначить роль **Администратор** другим пользователям.

## <a name="change-application-name"></a>Изменение имени приложения

Чтобы изменить имя приложения, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

На странице **Параметры приложения** введите имя на ваш выбор в поле **Имя приложения**, а затем нажмите кнопку **Сохранить**.

## <a name="change-the-application-url"></a>Изменение URL-адреса приложения

Чтобы изменить URL-адрес приложения, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

![Страница параметров приложения](media\howto-administer\image0-a.png)

На странице **Параметры приложения** введите URL-адрес на ваш выбор в поле **URL-адрес**, а затем нажмите кнопку **Сохранить**. Длина значения — не более 200 символов. Если URL-адрес недоступен, появится ошибка проверки.

> [!Note]
> Если URL-адрес будет изменен, старый URL-адрес может быть занят другим клиентом Azure IoT Central. В этом случае он больше недоступен для использования. При изменении URL-адреса старый URL-адрес больше не работает, поэтому вам необходимо уведомить пользователей о новом URL-адресе.

## <a name="change-the-application-image"></a>Изменение образа приложения

Дополнительные сведения об использовании образов в приложении Azure IoT Central см. в статье [Подготовка и отправка образов в приложение Azure IoT Central](howto-prepare-images.md).

## <a name="delete-an-application"></a>Удаление приложения

Чтобы удалить приложение, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

Щелкните **Удалить**.

> [!Note]
> При удалении приложения необратимо удаляются все связанные с ним данные. Чтобы удалить приложение, вы также должны иметь права на удаление ресурсов в подписке Azure, которую вы выбрали при создании приложения. Дополнительные сведения см. в статье [Использование управления доступом на основе ролей для контроля доступа к ресурсам в подписке Azure](https://docs.microsoft.com/azure/active-directory/role-based-access-control-configure).

## <a name="roles-in-azure-iot-central"></a>Роли в Azure IoT Central

Роли позволяют вам контролировать, кто в вашей организации может выполнять различные задачи в Azure IoT Central. Azure IoT Central имеет три роли, которые вы можете назначить пользователям вашего приложения. Роли назначаются для каждого приложения. Тот же пользователь может иметь разные роли в различных приложениях. Вы можете назначить одному и тому же пользователю несколько ролей в приложении.

### <a name="administrator"></a>Администратор

Пользователи в роли **администратора** имеют доступ ко всем возможностям в приложении Azure IoT Central.

Пользователю, создающему приложение, автоматически назначается роль **администратора**. В роли **администратора** должен быть хотя бы один пользователь.

### <a name="application-builder"></a>Конструктор приложений

Пользователи в роли **Конструктор приложений** могут выполнять в приложении Azure IoT Central все задачи, кроме его администрирования.

### <a name="application-operator"></a>Оператор приложения

Пользователи в роли **оператора приложения** не имеют доступа к странице **Конструктор приложений**. Они не могут администрировать приложения.

## <a name="manage-users"></a>Управление пользователями

Администраторы приложений могут назначать пользователям роли в приложении.

### <a name="add-users"></a>Добавление пользователей

Каждый пользователь должен иметь учетную запись, чтобы войти в систему и получить доступ к приложению Azure IoT Central. Учетные записи MSA и Azure Active Directory (AAD) поддерживаются в Azure IoT Central. Группы Azure Active Directory в настоящее время не поддерживаются в Azure IoT Central.

Дополнительные сведения см. на странице [справки по учетной записи Майкрософт](https://support.microsoft.com/products/microsoft-account?category=manage-account) и в статье [Краткое руководство по добавлению новых пользователей в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/add-users-azure-active-directory).

1. Чтобы добавить учетную запись пользователя в приложение Azure IoT Central, в дополнительном навигационном меню перейдите на страницу **Пользователи** в разделе **Администрирование**.

    ![Список пользователей](media\howto-administer\image1.png)

1. На странице **Пользователи** выберите **+ Добавить пользователя**, чтобы добавить пользователя.

    ![Добавить пользователя](media\howto-administer\image2.png)

1. Добавляя пользователя в приложение Azure IoT Central, выберите для него роль в раскрывающемся списке **Роль**. Дополнительные сведения о ролях см. в разделе *Роли в Azure IoT Central* этой статьи.

    ![Выбор ролей](media\howto-administer\image3.png)

    > [!NOTE]
    >  Чтобы добавить пользователей массово, введите идентификаторы всех пользователей, которых вы хотите добавить, разделенные точками с запятой. Выберите роль в раскрывающемся списке **Роль** и нажмите кнопку **Сохранить**.

1. После добавления пользователя отображается запись этого пользователя на странице **Пользователи**.

    ![Список пользователей](media\howto-administer\image4.png)

### <a name="edit-the-roles-assigned-to-users"></a>Изменение ролей, назначенных пользователям

Нельзя изменить уже заданную роль. Чтобы изменить роль, назначенную пользователю, удалите учетную запись и добавьте пользователя еще раз с другой ролью.

### <a name="delete-users"></a>Удаление пользователей

Чтобы удалить пользователей, установите один или несколько флажков на странице **Пользователи**, а затем нажмите кнопку **Удалить**.

## <a name="view-your-bill"></a>Просмотр счетов

Чтобы просмотреть свой счет, перейдите на страницу **Выставление счетов** в разделе **Администрирование** и выберите **Выставление счетов**. На новой вкладке откроется страница выставления счетов Azure, и вы сможете увидеть счет для каждого из ваших приложений Azure IoT Central.

## <a name="convert-your-trial-to-a-paid-application"></a>Преобразование пробной версии в платное приложение

После того как вы оценили IoT Central, вы можете конвертировать пробную версию в платное приложение. Для завершения этого процесса самообслуживания выполните следующие действия:

1. Используйте дополнительное меню навигации, чтобы перейти на страницу **Выставление счетов** в разделе **Администрирование**. Если вы еще не изменили пробную версию, страница выглядит следующим образом:

    ![Состояние бесплатной пробной версии](media/howto-administer/freetrial.png)

2. Нажмите кнопку **Convert to Paid** (Преобразовать в платную). Если вы еще не изменили пробную версию, всплывающее окно выглядит следующим образом:

    Во всплывающем окне выберите подходящий клиент Azure Active Directory, а затем подписку Azure, которую вы хотите использовать для приложения IoT Central.

    ![Продление бесплатной пробной версии](media/howto-administer/extend.png)

3. После нажатия кнопки **Преобразовать** пробная версия преобразуется в платное приложение и вам будет выставляться счет.

## <a name="extend-your-free-trial"></a>Продление бесплатной пробной версии

По умолчанию все бесплатные пробные версии доступны в течение 7 дней. Если вы хотите увеличить срок действия пробной версии до 30 дней, выполните следующие действия:

1. Используйте дополнительное меню навигации, чтобы перейти на страницу **Выставление счетов** в разделе **Администрирование**.

1. Нажмите кнопку **Продлить пробный период**. Во всплывающем окне выберите подходящий клиент Azure Active Directory, а затем подписку Azure, которую вы хотите использовать для приложения IoT Central.

1. Нажмите кнопку **Продлить**. Пробная версия действительна в течение 30 дней.

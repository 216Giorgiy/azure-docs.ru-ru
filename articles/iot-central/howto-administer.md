# <a name="administer-your-iot-central-application"></a>Администрирование приложения Azure IoT Central

После создания приложения Microsoft Azure IoT Central для его администрирования вы можете использовать раздел **Администрирование** пользовательского интерфейса Azure IoT Central. Чтобы перейти к разделу **Администрирование**, выберите пункт **Администрирование** в меню навигации слева.

В разделе **Администрирование** можно:

- Управление пользователями

- управлять ролями;

- просматривать сведения о выставлении счетов;

- управлять параметрами приложения;

- воспользоваться бесплатной пробной версией.

В разделе **Администрирование** есть дополнительное меню навигации со ссылками на различные задачи администрирования.

Чтобы получить доступ к разделу **Администрирование**, вам должна быть назначена роль **Администратор** приложения Azure IoT Central. Если вы создаете приложение Azure IoT Central, вам автоматически назначается роль **Администратор** для этого приложения. В разделе *Управление пользователями* в этой статье объясняется, как назначить роль **Администратор** другим пользователям.

## <a name="change-application-name"></a>Изменение имени приложения

Чтобы изменить имя приложения, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

На странице **Параметры приложения** введите требуемое имя в поле **Имя приложения**. Затем нажмите кнопку **Save** (Сохранить).

## <a name="change-the-application-url"></a>Изменение URL-адреса приложения

Чтобы изменить URL-адрес приложения, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

![Страница "Параметры приложения"](media\howto-administer\image0-a.png)

На странице **Параметры приложения** введите требуемый URL-адрес в поле **URL-адрес**, а затем нажмите кнопку **Сохранить**. Длина значения — не более 200 символов. Если URL-адрес недоступен, возникнет ошибка проверки.

> [!Note]
> Если URL-адрес будет изменен, старый URL-адрес может быть занят другим клиентом Azure IoT Central. В этом случае он будет недоступен для использования. При изменении URL-адреса старый URL-адрес больше нельзя будет использовать, поэтому вам нужно уведомить пользователей о новом URL-адресе.

## <a name="change-the-application-image"></a>Изменение образа приложения

Дополнительные сведения об использовании образов в приложении Azure IoT Central см. в статье [Подготовка и отправка образов в приложение Azure IoT Central](howto-prepare-images.md).

## <a name="copy-an-application"></a>Копирование приложения

Можно создать копию любого приложения без экземпляров устройства, журнала данных устройства и данных пользователя. Копия будет платным приложением, за которое выставляется счет. Нельзя создать приложение в пробной версии, скопировав другое приложение.

Чтобы скопировать приложение, перейдите на страницу **Параметры приложения**. Затем нажмите кнопку **Копировать**.

![Страница "Параметры приложения"](media\howto-administer\appCopy1.png)

При нажатии кнопки **Копировать** откроется диалоговое окно, в котором можно выбрать имя, URL-адрес, каталог Azure Active Directory, подписку и регион Azure для нового приложения, создаваемого путем копирования вашего приложения. Выберите значения для каждого из этих полей. Затем нажмите кнопку **Копировать**, чтобы подтвердить, что вы хотите продолжить. Дополнительные сведения о значениях, которые нужно вводить в эти поля, см. в статье [Создание приложения Azure IoT Central](howto-create-application.md).

![Страница "Параметры приложения"](media\howto-administer\appCopy2.png)

Когда приложение будет скопировано, можно перейти к нему. Для этого выберите ссылку, которая появится на странице **Параметры приложения**.

![Страница "Параметры приложения"](media\howto-administer\appCopy3.png)

> [!Note]
> При копировании приложения также копируются определение правил или действий. Но так как пользователи с правом доступа к вашему исходному приложению не копируются в скопированное приложение, вам необходимо вручную разрешить пользователям выполнять такие действия, как отправка сообщений по электронной почте, для чего назначение пользователей является необходимым.

## <a name="delete-an-application"></a>Удаление приложения

Чтобы удалить приложение, воспользуйтесь дополнительным меню навигации и перейдите на страницу **​​Параметры приложения** в разделе **Администрирование**.

Щелкните **Удалить**.

> [!Note]
> При удалении приложения необратимо удаляются все связанные с ним данные.  Чтобы удалить приложение, у вас также должны быть разрешения на удаление ресурсов в подписке Azure, которую вы выбрали при создании приложения. Дополнительные сведения см. в статье [Использование управления доступом на основе ролей для контроля доступа к ресурсам в подписке Azure](https://docs.microsoft.com/azure/active-directory/role-based-access-control-configure).

## <a name="roles-in-azure-iot-central"></a>Роли в Azure IoT Central

Роли позволяют вам контролировать, кто в вашей организации может выполнять различные задачи Azure IoT Central. Azure IoT Central имеет три роли, которые вы можете назначить пользователям вашего приложения. Роли назначаются для каждого приложения. Тот же пользователь может иметь разные роли в различных приложениях. Вы можете назначить несколько ролей одному и тому же пользователю приложения.

### <a name="administrator"></a>Администратор

Пользователи в роли **администратора** имеют доступ ко всем возможностям в приложении Azure IoT Central.

Пользователю, создавшему приложение, автоматически назначается роль **администратора**. В роли **администратора** должен быть хотя бы один пользователь.

### <a name="application-builder"></a>Конструктор приложений

Пользователи в роли **Конструктор приложений** могут выполнять в приложении Azure IoT Central все задачи, кроме его администрирования.

### <a name="application-operator"></a>Оператор приложения

Пользователи в роли **оператора приложения** не имеют доступа к странице **Конструктор приложений**. Они не могут администрировать приложения.

## <a name="manage-users"></a>Управление пользователями

Администраторы приложений могут назначать пользователям роли в приложении.

### <a name="add-users"></a>Добавление пользователей

Каждый пользователь должен иметь учетную запись, чтобы войти в систему и получить доступ к приложению Azure IoT Central. Учетные записи MSA и Azure Active Directory поддерживаются в Azure IoT Central. Группы Azure Active Directory сейчас не поддерживаются в Azure IoT Central.

Дополнительные сведения см. на странице [справки по учетной записи Майкрософт](https://support.microsoft.com/products/microsoft-account?category=manage-account) и в статье [Краткое руководство по добавлению новых пользователей в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/add-users-azure-active-directory).

1. Чтобы добавить учетную запись пользователя в приложение Azure IoT Central, в дополнительном навигационном меню перейдите на страницу **Пользователи** в разделе **Администрирование**.

    ![Список пользователей](media\howto-administer\image1.png)

1. На странице **Пользователи** выберите **+Добавить пользователя**.

    ![Добавление пользователя](media\howto-administer\image2.png)

1. Выберите роль в раскрывающемся списке **Роль**. Дополнительные сведения о ролях см. в разделе *Роли в Azure IoT Central* этой статьи.

    ![Выбор ролей](media\howto-administer\image3.png)

    > [!NOTE]
    >  Чтобы добавить пользователей массово, введите идентификаторы соответствующих пользователей, разделив их точками с запятой. Выберите роль в раскрывающемся списке **Роль**. Затем нажмите кнопку **Save** (Сохранить).

1. После добавления пользователя отображается запись этого пользователя на странице **Пользователи**.

    ![Список пользователей](media\howto-administer\image4.png)

### <a name="edit-the-roles-that-are-assigned-to-users"></a>Изменение ролей, назначенных пользователям

Роли нельзя изменить после назначения. Чтобы изменить роль, назначенную пользователю, удалите учетную запись и повторно добавьте пользователя с другой ролью.

### <a name="delete-users"></a>Удаление пользователей

Чтобы удалить пользователей, установите один или несколько флажков на странице **Пользователи**. Теперь щелкните **Удалить**.

## <a name="view-your-bill"></a>Просмотр счетов

Чтобы просмотреть свой счет, перейдите на страницу **Выставление счетов** в разделе **Администрирование**. Выберите **Выставление счетов**. В новой вкладке откроется страница выставления счетов Azure, где вы сможете просмотреть счет по каждому приложению Azure IoT Central.

## <a name="convert-your-trial-to-a-paid-application"></a>Преобразование пробной версии в платное приложение

По истечении срока действия пробной версии IoT Central вы сможете перейти на платную версию приложения. Для завершения этого процесса самообслуживания выполните следующие действия:

1. Используйте дополнительное меню навигации, чтобы перейти на страницу **Выставление счетов** в разделе **Администрирование**. Если вы не продлевали срок действия пробной версии, страница будет выглядеть так:

    ![Состояние бесплатной пробной версии](media/howto-administer/freetrial.png)

2. Выберите **Преобразовать в платную**. Если вы не продлевали срок действия пробной версии, всплывающее окно будет выглядеть так:

    ![Продление бесплатной пробной версии](media/howto-administer/extend.png)

3. Во всплывающем окне выберите требуемый клиент Azure Active Directory, а затем подписку Azure, которую вы хотите использовать для приложения IoT Central.

3. После нажатия кнопки **Преобразовать** вы перейдете на платную версию и вам будут выставляться счета за использование приложения.

## <a name="extend-your-free-trial"></a>Продление бесплатной пробной версии

По умолчанию срок действия всех бесплатных пробных версий составляет семь дней. Если вы хотите продлить срок действия пробной версии до 30 дней, выполните следующие действия:

1. Используйте дополнительное меню навигации, чтобы перейти на страницу **Выставление счетов** в разделе **Администрирование**.

1. Нажмите кнопку **Продлить пробный период**. Во всплывающем окне выберите требуемый клиент Azure Active Directory, а затем подписку Azure, которую вы хотите использовать для приложения IoT Central.

1. Затем выберите **Продлить**. Пробная версия действительна в течение 30 дней.

## <a name="use-the-azure-sdks-for-control-plane-operations"></a>Используйте пакеты SDK Azure для операций уровня управления

Пакеты SDK IoT Central Azure Resource Manager доступны для Node, Python, C#, Ruby, Java и Go. Эти библиотеки поддерживают операции уровня управления для Azure IoT Central, позволяя создавать, отображать, обновлять или удалять приложения Azure IoT Central. Они также включают вспомогательные методы для проверки подлинности и обработки ошибок в соответствии с выбранным языком. 

Примеры использования пакетов SDK для Azure Resource Manager см. здесь: [https://github.com/emgarten/iotcentral-arm-sdk-examples](https://github.com/emgarten/iotcentral-arm-sdk-examples).

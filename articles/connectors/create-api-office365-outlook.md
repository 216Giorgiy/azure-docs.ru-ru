# Начало работы с API-интерфейсом Office 365 Outlook 

Подключившись к Office 365 Outlook, вы сможете получать электронную почту, отвечать на сообщения, обновлять календари и контакты и многое другое. API Office 365 Outlook можно использовать в:

- PowerApps; 
- приложениях логики. 

>[AZURE.NOTE] Эта версия статьи предназначена для приложений логики со схемой версии 2015-08-01-preview. Чтобы получить версию схемы 2014-12-01-preview, щелкните [API Office 365](../app-service-logic/app-service-logic-connector-office365.md).

С помощью Office 365 Outlook можно:

- формировать бизнес-процессы на основе данных, получаемых из Office 365 Outlook; 
- использовать триггер при получении нового сообщения электронной почты, при создании нового контакта и т. д.;
- использовать действия для ответа на сообщения электронной почты, создания событий календаря и т. д. Эти действия получают ответ и делают выходные данные доступными для использования другими действиями. Например, если в Salesforce появляется новый объект, вы можете принять этот объект и обновить контакты в Office 365 Outlook. 
- Добавьте API Office 365 Outlook в службу PowerApps Enterprise. После этого пользователи смогут использовать этот API в своих приложениях. 

Сведения о добавлении API в PowerApps Enterprise см. в статье [Регистрация API в PowerApps](../power-apps/powerapps-register-from-available-apis.md).

Сведения о добавлении операции в приложения логики см. в статье [Создание приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

## Создание подключения к Office 365

### Добавление дополнительных настроек в PowerApps
При добавлении этого API в PowerApps Enterprise нужно ввести значения **ключа приложения** и **секрета приложения** для приложения Office 365 Azure Active Directory (AAD). В приложении Office 365 также используется **URL-адрес перенаправления**. Если у вас нет приложения Office 365, создайте его, выполнив приведенные ниже действия.

1. На [портале Azure][5] откройте **Active Directory** и выберите имя клиента вашей организации.
2. Перейдите на вкладку **Приложения** и выберите **Добавить**:  
![Клиентские приложения AAD][7]

3. На экране **Добавление приложения**:

	1. Введите **имя** для приложения.  
	2. Оставьте тип приложения **Веб**.  
	3. Щелкните **Далее**.  

	![Добавление приложения AAD: сведения о приложении][8]

6. На экране **Свойства приложения**:

	1. Введите **URL-АДРЕС ВХОДА** для своего приложения. Поскольку для PowerApps будет использоваться проверка подлинности AAD, в качестве URL-адреса входа укажите _https://login.windows.net_.
	2. Введите допустимый **универсальный код ресурса URI кода приложения** для своего приложения.  
	3. Нажмите кнопку **ОК**.  

	![Добавление приложения AAD: свойства приложения][9]

7. По завершении открывается новое приложение AAD. Перейдите на вкладку **Настройка**:  
![Приложение AAD Contoso][10]

8. В разделе _OAuth 2_ укажите в поле **URL-адрес ответа** URL-адрес, полученный при добавлении нового API Office 365 Outlook на портале Azure. Выберите **Добавить приложение**:  
![Настройка приложения AAD Contoso][11]

9. В окне **Разрешения для других приложений** выберите **Office 365 Exchange Online** и нажмите **ОК**:  
![Делегат приложения Contoso][12]

	Обратите внимание, что на странице настройки _Office 365 Exchange Online_ добавляется в список _Разрешений для других приложений_.

10. На странице **Office 365 Exchange Online** выберите **Делегированные разрешения** и установите следующие разрешения:

	- Чтение и запись контактов пользователя
	- Чтение контактов пользователя
	- Чтение и запись календарей пользователя
	- Чтение календарей пользователя
	- отправка сообщений в качестве пользователя.
	- Чтение и запись почты пользователя
	- Чтение почты пользователя

	![Разрешения для делегата приложения Contoso][13]

Будет создано новое приложение Azure Active Directory. Вы можете скопировать и вставить значения **ключа приложения** и **секрета приложения** в конфигурацию API Office 365 Outlook на портале Azure.

Полезные сведения о приложениях AAD см. в статье [Как и почему приложения добавляются в Azure AD](../active-directory/active-directory-how-applications-are-added.md).

### Добавление дополнительных настроек в приложения логики
При добавлении этого API в приложения логики необходимо войти в учетную запись Office 365 Outlook и разрешить приложениям логики подключаться к вашей учетной записи.

1. Войдите свою учетную запись Office 365 Outlook.
2. Разрешите приложениям логики подключаться к учетной записи Office 365 и использовать ее. 

После создания подключения укажите свойства Office 365 Outlook, такие как путь к папке входящих писем или сообщение электронной почты. Эти свойства описаны далее в **справочнике по REST API**.

>[AZURE.TIP] Подключение к Office 365 Outlook можно также использовать в других приложениях логики.

# Начало работы с соединителем Office 365 Outlook 

Подключившись к Office 365 Outlook, вы сможете получать электронную почту, отвечать на сообщения, обновлять календари и контакты и многое другое. Соединитель Office 365 Outlook можно использовать из таких компонентов, как:

- приложений логики. 
- PowerApps.

> [AZURE.SELECTOR]
- [Приложения логики](../articles/connectors/connectors-create-api-office365-outlook.md)
- [PowerApps Enterprise](../articles/power-apps/powerapps-create-api-office365-outlook.md)

&nbsp;

>[AZURE.NOTE] Эта версия статьи предназначена для приложений логики со схемой версии 2015-08-01-preview.

С помощью Office 365 Outlook можно:

- формировать бизнес-процессы на основе данных, получаемых из Office 365 Outlook; 
- использовать триггер при получении нового сообщения электронной почты, при создании нового контакта и т. д.;
- использовать действия для ответа на сообщения электронной почты, создания событий календаря и т. д. Эти действия получают ответ и делают выходные данные доступными для использования другими действиями. Например, если в Salesforce появляется новый объект, вы можете принять этот объект и обновить контакты в Office 365 Outlook. 
- Добавьте соединитель Office 365 Outlook в службу PowerApps Enterprise. После этого пользователи смогут использовать его в своих приложениях. 

Сведения о добавлении соединителя в PowerApps Enterprise см. в статье [Регистрация соединителя в PowerApps](../power-apps/powerapps-register-from-available-apis.md).

Сведения о добавлении операции в приложения логики см. в статье [Создание приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

## Триггеры и действия

Соединитель Office 365 Outlook предоставляет следующие триггеры и действия.

| --- | --- |
|<ul><li>При запуске предстоящего события</li><li>При получении нового электронного сообщения</li><li>При создании элементов</li><li>При обновлении элементов</li></ul>| <ul><li>Создание контакта</li><li>Создание события</li><li>Отправка электронного сообщения с утверждением</li><li>Отправка электронного сообщения</li><li>Удаление контакта</li><li>Удаление сообщения электронной почты</li><li>Удаление события</li><li>Получение вложения</li><li>Получение календарей</li><li>Получение контакта</li><li>Получение папки контактов</li><li>Получение контактов</li><li>Получение электронных сообщений</li><li>Получение события</li><li>Получение событий</li><li>Пометить как прочитанное</li><li>При запуске предстоящего события</li><li>При получении нового электронного сообщения</li><li>При создании элементов</li><li>При обновлении элементов</li><li>Ответ на сообщение</li><li>Отправка электронного сообщения с параметрами</li><li>Обновление контакта</li><li>Обновление события</li></ul> |

Все соединители поддерживают данные в форматах JSON и XML.


## Создание подключения к Office 365

При добавлении этого соединителя в приложения логики необходимо войти в учетную запись Office 365 Outlook и разрешить приложениям логики подключаться к вашей учетной записи.

1. Войдите свою учетную запись Office 365 Outlook.
2. Разрешите приложениям логики подключаться к учетной записи Office 365 и использовать ее. 

После создания подключения укажите свойства Office 365 Outlook, такие как путь к папке входящих писем или сообщение электронной почты. Эти свойства описаны далее в **справочнике по REST API**.

>[AZURE.TIP] Подключение к Office 365 Outlook можно также использовать в других приложениях логики.

| Имя| Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|таблица|строка|Да|path|Нет|Уникальный идентификатор извлекаемой папки контактов|
|$skip|целое число|Нет|запрос|Нет|Количество пропускаемых записей (значение по умолчанию — 0)|
|$top|целое число|Нет|запрос|Нет|Максимальное количество извлекаемых записей (значение по умолчанию — 256)|
|$filter|string|Нет|запрос|Нет|Запрос фильтра ODATA для ограничения количества записей|
|$orderby|строка|Нет|запрос|Нет|Запрос orderBy ODATA для указания порядка записей|

#### Ответ

|Имя|Описание|
|---|---|

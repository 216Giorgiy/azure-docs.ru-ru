# Начало работы с API хранилища BLOB-объектов Azure
Подключение к хранилищу BLOB-объектов Azure позволяет управлять файлами в контейнере BLOB-объектов, включая создание файлов, их удаление и многое другое. API хранилища BLOB-объектов Azure можно использовать из:

- приложений логики. 

>[AZURE.NOTE] Эта версия статьи предназначена для приложений логики со схемой версии 2015-08-01-preview. Чтобы получить версию схемы 2014-12-01-preview, щелкните [соединитель хранилища BLOB-объектов Azure](../app-service-logic/app-service-logic-connector-azurestorageblob.md).

С помощью хранилища BLOB-объектов Azure можно выполнять следующие действия:

- формировать бизнес-процессы на основе данных, получаемых из BLOB-объекта; 
- использовать действия, позволяющие создавать файлы, получить содержимое файлов и многое другое. Эти действия получают ответ и делают выходные данные доступными для использования другими действиями. Например, можно задать поиск слова "срочно" по файлам в BLOB-объекте, а затем отправить все файлы со словом "срочно" по электронной почте с помощью Office 365. 

Сведения о добавлении операции в приложения логики см. в статье [Создание приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

## Триггеры и действия
BLOB-объект Azure включает следующие действия. Триггеры отсутствуют.

| триггеры; | Действия|
| --- | --- |
| Нет. | <ul><li>Создание файла</li><li>Копирование файла</li><li>Удаление файла</li><li>Извлечение архива в папку</li><li>Получение содержимого файла</li><li>Получение содержимого файла с помощью пути</li><li>Получение метаданных файла</li><li>Получение метаданных файла с помощью пути</li><li>Обновление файла</li></ul> |

Все API-интерфейсы поддерживают данные в форматах JSON и XML.

## Создание подключения к BLOB-объекту Azure
При добавлении этого API в приложения логики введите следующие значения учетной записи хранилища:

|Свойство| Обязательно|Описание|
| ---|---|---|
|Имя учетной записи хранения Azure | Да | Имя учетной записи хранилища BLOB-объектов|
|Ключ доступа к учетной записи хранилища BLOB-объектов Azure | Да | Ключ доступа к учетной записи хранилища BLOB-объектов|

После создания подключения укажите свойства BLOB-объекта, такие как путь к папке или имя файла. Эти свойства описаны далее в **справочнике по REST API**.

>[AZURE.TIP] Это подключение можно использовать в других приложениях логики.
 

## Справочник по REST API Swagger
Относится к версии 1.0.

### Создание файла
Передает файл в хранилище BLOB-объектов Azure. ```POST: /datasets/default/files```

| Имя|Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|folderPath|строка|Да|запрос| Нет |Путь к папке для отправки файла в хранилище BLOB-объектов Azure|
|name|строка|Да|запрос|Нет |Имя файла, который нужно создать в хранилище BLOB-объектов Azure|
|body|string(binary) |Да|текст|Нет|Содержимое файла, отправляемого в хранилище BLOB-объектов Azure|

#### Ответ
|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|

### Копирование файла
Копирует файл в хранилище BLOB-объектов Azure. ```POST: /datasets/default/copyFile```

| Имя|Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|источник|string|Да|запрос|Нет |URL-адрес исходного файла|
|ресурс destination|строка|Да|запрос| Нет|Путь к конечному файлу в хранилище BLOB-объектов Azure, включая имя конечного файла|
|перезаписать|Логическое|Нет|запрос|Нет |Перезаписывает конечный файл, если задано значение "true"|

#### Ответ
|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|

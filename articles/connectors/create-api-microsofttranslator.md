# Начало работы с API-интерфейсом Microsoft Translator
Подключитесь к Microsoft Translator, чтобы переводить тексты, определять языки и выполнять многие другие действия. API Microsoft Translator можно использовать из:

- PowerApps; 
- приложений логики. 

С помощью Microsoft Translator вы можете:

- формировать бизнес-процессы на основе данных, получаемых из Microsoft Translator; 
- использовать действия для перевода текста, определения языка и т. д. Эти действия получают ответ и делают выходные данные доступными для использования другими действиями. Например, когда в Dropbox создается файл, с помощью Microsoft Translator можно перевести его текст на другой язык.
- Добавлять API Microsoft Translator в PowerApps Enterprise. После этого пользователи смогут использовать этот API в своих приложениях. 

Сведения о добавлении API в PowerApps Enterprise см. в статье [Регистрация API в PowerApps](../power-apps/powerapps-register-from-available-apis.md).

Сведения о добавлении операции в приложения логики см. в статье [Создание приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

## Триггеры и действия
Microsoft Translator позволяет выполнять следующие действия. Триггеры отсутствуют.

триггеры; | Действия
--- | ---
None | <ul><li>Определение языка</li><li>Преобразование текста в речь</li><li>Перевод текста</li><li>Получение языков</li><li>Получение языков для синтеза речи</li></ul>

Все API-интерфейсы поддерживают данные в форматах JSON и XML.

## Создание подключения к Microsoft Translator

### Добавление дополнительных настроек в PowerApps
При добавлении Microsoft Translator в PowerApps Enterprise нужно ввести **идентификатор клиента** и **секрет клиента** для приложения Microsoft Translator. Если у вас нет приложения Translator, создайте его, выполнив приведенные ниже действия.

1. Перейдите на страницу [разработчика Azure Data Market][5] и выполните вход со своей учетной записью Майкрософт. 

2. Выберите **Зарегистрировать приложение**.

	1. Введите значение в поле **Идентификатор клиента**.
	2. Укажите **имя** приложения.
	3. Введите фиктивное значение для параметра **URL-адрес перенаправления**. Например, введите *https://contosoredirecturl*.
	4. Введите **описание**.
	5. Нажмите кнопку **Создать**.  

	![Регистрация приложения][6]

Теперь скопируйте и вставьте значения **идентификатора клиента** и **секрет клиента** в конфигурацию Translator на портале Azure.


## Справочник по REST API Swagger
Относится к версии 1.0.

### Определение языка    
Определяет исходный язык заданного текста. ```GET: /Detect```

| Имя| Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|запрос|string|Да|запрос|Нет |Текст, язык которого будет определен|

#### Ответ
|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|


### Преобразование текста в речь    
Преобразует заданный текст в речь в виде звукового потока в формате WAVE. ```GET: /Speak```

| Имя| Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|запрос|string|Да|запрос|Нет |Преобразуемый текст|
|язык|строка|Да|запрос|Нет |Код языка для создания речи (например "ru-RU")|

#### Ответ
|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|


### Перевод текста    
Переводит текст на указанный язык с помощью Microsoft Translator. ```GET: /Translate```

| Имя| Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|запрос|string|Да|запрос|Нет |Текст для перевода|
|languageTo|строка|Да|запрос| Нет|Код целевого языка (например "f")|
|languageFrom|строка|Нет|запрос|Нет |Исходный язык. Если не указан, Microsoft Translator попытается определить язык автоматически (например en).|
|категория|строка|Нет|запрос|общие |Категория перевода (по умолчанию — "Общие")|

#### Ответ
|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|

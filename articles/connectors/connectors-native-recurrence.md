# Начало работы с триггером повторения

Благодаря триггеру повторения можно создавать такие эффективные рабочие процессы в облаке:

- планирование рабочего процесса для ежедневного запуска хранимой процедуры SQL;
- отправка по почте сводки всех твитов с определенным хэштегом за последнюю неделю.

Сведения о начале работы с триггером повторения в приложении логики см. в статье о [создании приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

---

## Использование триггера повторения

Триггер — это событие, которое можно использовать для запуска рабочего процесса, определенного в приложении логики. Дополнительные сведения о триггерах см. [здесь](connectors-overview.md).

Ниже приведен пример последовательности настройки триггера повторения в приложении логики.

1. Сначала добавьте триггер **повторения** в приложении логики.
1. Укажите параметры интервала повторения.
1. Теперь приложение логики будет запускаться с определенным интервалом.

## Технические сведения

Ниже приведены подробные сведения о триггере повторения и настраиваемых параметрах.

### Сведения о триггере

Ниже приведены свойства, которые можно настроить для триггера повторения.

#### Периодичность
Задайте интервал запуска приложения логики. Знак "*" обозначает обязательное поле.

|Отображаемое имя|Имя свойства|Описание|
|---|---|---|
|Частота*|frequency|Единица времени: `Second`, `Minute`, `Hour`, `Day` или `Year`|
|Интервал*|interval|Интервал повторения в указанной единице времени|
|Часовой пояс|timeZone|Используется, если для свойства startTime указано значение без смещения от UTC|
|Время начала|startTime|Время начала в [формате ISO 8601](https://en.wikipedia.org/wiki/ISO_8601#Combined_date_and_time_representations). Используется, если для свойства startTime указано значение без смещения от UTC|

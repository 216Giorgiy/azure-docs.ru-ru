# Начало работы с соединителем SQL



Соединитель SQL может использоваться из следующих компонентов:

- [Приложения логики](../app-service-logic/app-service-logic-what-are-logic-apps.md)  
- [PowerApps.](http://powerapps.microsoft.com)  
- [Поток](http://flows.microsoft.com)  

>[AZURE.NOTE] Эта версия статьи предназначена для приложений логики со схемой версии 2015-08-01-preview.

Для начала можно создать приложение логики, как указано в соответствующей [статье](../app-service-logic/app-service-logic-create-a-logic-app.md).

## Триггеры и действия

Соединитель SQL можно использовать как действие. Кроме того, он имеет триггеры. Все соединители поддерживают данные в форматах JSON и XML.

 Соединитель SQL предоставляет следующие триггеры и действия:

### Действия соединителя SQL
Вы можете выполнять перечисленные ниже действия:

|Действие|Описание|
|--- | ---|
|[GetTables](connectors-create-api-sqlconnector.md#gettables)|Получает таблицы из базы данных SQL|
|[PostItem](connectors-create-api-sqlconnector.md#postitem)|Вставляет новую строку в таблицу SQL|
|[GetItem](connectors-create-api-sqlconnector.md#getitem)|Получает одну строку из таблицы SQL|
|[DeleteItem](connectors-create-api-sqlconnector.md#deleteitem)|Удаляет строку из таблицы SQL|
|[PatchItem](connectors-create-api-sqlconnector.md#patchitem)|Изменяет существующую строку в таблице SQL|
### Триггеры соединителя SQL
Можно прослушивать указанные ниже события:

|--- | ---|


## Создание подключения к соединителю SQL
Для создания приложений логики с помощью соединителя SQL необходимо создать **подключение**, а затем указать данные для следующих свойств:

|Свойство| Обязательно|Описание|
| ---|---|---|
|SqlConnectionString|Да|Укажите строку подключения SQL|
Созданное подключение можно использовать для выполнения действий и прослушивания триггеров, описанных в этой статье.

>[AZURE.INCLUDE [Шаги по созданию подключения к Mashup SQL](../../includes/connectors-create-api-mashupsql.md)]

>[AZURE.TIP] Это подключение можно использовать в других приложениях логики.

## Справочник по соединителю SQL
Относится к версии 1.0.

## GetTables
Получение таблиц: получает таблицы из базы данных SQL

```GET: /datasets/default/tables```

Для этого вызова параметры отсутствуют
#### Ответ

|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|


## PostItem
Вставка строки: вставляет новую строку в таблицу SQL

```POST: /datasets/default/tables/{table}/items```

| Имя| Тип данных|Обязательно|Местонахождение|Значение по умолчанию|Описание|
| ---|---|---|---|---|---|
|таблица|строка|Да|path|Нет|Имя таблицы SQL|
|item| |Да|текст|Нет|Строка, вставляемая в указанную таблицу в SQL|

#### Ответ

|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|

| ---|---|---|---|---|---|

|id|строка|Да|path|Нет|Уникальный идентификатор извлекаемой строки|

#### Ответ

|Имя|Описание|
|---|---|
|200|ОК|
|по умолчанию|Операция завершилась ошибкой.|

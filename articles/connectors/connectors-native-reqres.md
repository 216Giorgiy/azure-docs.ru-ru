# Начало работы с компонентами запросов и ответов

Благодаря использованию компонентов запросов и ответов в приложении логики можно в режиме реального времени ответить на такие события:

- ответ на HTTP-запрос, использующий данные из локальной базы данных, через приложение логики;
- запуск приложения логики из внешнего события webhook;
- вызов приложения логики с помощью действия запроса и ответа из другого приложения логики.

Сведения о начале работы с действиями запросов и ответов в приложении логики см. в статье о [создании приложения логики](../app-service-logic/app-service-logic-create-a-logic-app.md).

---

## Использование триггера HTTP-запросов

Триггер — это событие, которое можно использовать для запуска рабочего процесса, определенного в приложении логики. Дополнительные сведения о триггерах см. [здесь](connectors-overview.md).

Ниже приведен пример последовательности настройки HTTP-запроса в конструкторе приложений логики.

1. Добавьте триггер **Request - When an HTTP request is received** (Запрос: при получении HTTP-запроса) в свое приложение логики.
	- При необходимости можно предоставить схему JSON (с помощью такого средства, как [jsonschema.net](http://jsonschema.net)) для текста запроса. Это позволит конструктору создать токены свойств в HTTP-запросе.
1. Добавьте еще одно действие, чтобы сохранить логику приложения.
1. После сохранения вы можете получить URL-адрес HTTP-запроса из карточки запроса.
1. Запрос HTTP POST (можно использовать средство [Postman](https://www.getpostman.com/)) на URL-адрес запустит приложение логики.

>[AZURE.NOTE] Если не определить действие ответа, вызывающему немедленно будет возвращен ответ `202 ACCEPTED`. Действие ответа можно использовать для настройки ответа.

![Триггер ответа](./media/connectors-native-reqres/using-trigger.png)

## Использование действия ответа HTTP
	
Действие ответа HTTP действительно только при использовании в рабочем процессе, запускаемом HTTP-запросом. Если не определить действие ответа, вызывающему немедленно будет возвращен ответ `202 ACCEPTED`. Действие ответа можно добавить на любом шаге рабочего процесса. Приложение логики не будет закрывать входящий запрос для ответа только в течение 1 минуты. Если ответ не был отправлен из рабочего процесса (а в определении есть действие ответа) в течение 1 минуты, вызывающему возвратится ответ `504 GATEWAY TIMEOUT`. Вот как можно добавить действие ответа HTTP:

1. Нажмите кнопку **Новый шаг**.
1. Выберите **Добавить действие**.
1. Чтобы открыть список с действиями ответа, в поле поиска действий введите "Ответ".

	![Выбор действия ответа](./media/connectors-native-reqres/using-action-1.png)

1. Добавьте параметры, необходимые для сообщения ответа HTTP.

	![Выполнение действия ответа](./media/connectors-native-reqres/using-action-2.png)

1. Щелкните "Сохранить" в левом верхнем углу панели инструментов. После этого приложение логики будет сохранено и опубликовано (активировано).

## Технические сведения

Ниже приведены подробные сведения о триггерах и действиях, поддерживаемых этим соединителем.

## Триггер запросов

Триггер — это событие, которое можно использовать для запуска рабочего процесса, определенного в приложении логики. Дополнительные сведения о триггерах см. [здесь](connectors-overview.md). Здесь приведен триггер одного запроса.

|Триггер|Описание|
|---|---|
|Запрос|При получении HTTP-запроса|

## Действие ответа

Действие — это операция, выполняемая рабочим процессом, определенным в приложении логики. Дополнительные сведения о действиях см. [здесь](connectors-overview.md). Здесь приведено действие одного ответа, которое может использоваться только вместе с триггером запроса.

|Действие|Описание|
|---|---|
|Ответ|Возврат ответа на связанный HTTP-запрос|

### Сведения о действиях

Ниже приведены сведения о всех действиях, их обязательных и необязательных полях ввода, а также соответствующие выходные данные, связанные с их использованием.

#### Триггер запросов
Триггер из входящего HTTP-запроса. Знак "*" обозначает обязательное поле.

|Отображаемое имя|Имя свойства|Описание|
|---|---|---|
|Схема JSON|schema|Схема JSON текста HTTP-запроса|
<br>

**Сведения о выходных данных**

Запрос

|Имя свойства|Тип данных|Описание|
|---|---|---|
|Заголовки|object|Заголовки запросов|
|Текст|object|Объект запроса|

#### Действие ответа

Ответ HTTP. Знак "*" обозначает обязательное поле.

|Отображаемое имя|Имя свойства|Описание|
|---|---|---|
|Код состояния*|statusCode|Код состояния HTTP|
|Заголовки|headers|Объект JSON заголовков ответов, которые нужно включить|
|Текст|текст|Текст ответа|

# <a name="workflow-common-parameters-for-azure-stack-validation-as-a-service"></a>Распространенные параметры рабочего процесса для проверки как услуги Azure Stack

[!INCLUDE[Azure_Stack_Partner](./includes/azure-stack-partner-appliesto.md)]

Общие параметры включают такие значения, как переменные среды и учетные данные пользователя, необходимые для всех тестов для проверки как услуги (VaaS). Эти значения определяются на уровне рабочего процесса. При создании или изменении рабочего процесса эти значения нужно сохранить. В запланированное время рабочий процесс загружает значения для теста. 

## <a name="environment-parameters"></a>Параметры среды

Параметры среды описывают тестируемую среду Azure Stack. Чтобы предоставить эти значения, нужно создать и отправить файл конфигурации метки `&lt;link&gt;. [How to get the stamp info link].`

| Имя параметра | Обязательно | type | ОПИСАНИЕ |
|----------------------------------|----------|------|---------------------------------------------------------------------------------------------------------------------------------|
| Сборка Azure Stack | Обязательно |  | Номер сборки развертывания Azure Stack (например, 1.0.170330.9). |
| Версия OEM | Yes |  | Номер версии пакета OEM, который используется во время развертывания Azure Stack. |
| Подпись OEM | Yes |  | Подпись пакета OEM, который используется во время развертывания Azure Stack. |
| Идентификатор клиента AAD | Обязательно |  | Уникальный идентификатор клиента Azure Active Directory, указанный во время развертывания Azure Stack.|
| Регион | Обязательно |  | Регион развертывания Azure Stack. |
| Конечная точка Resource Manager для клиента | Обязательно |  | Конечная точка для операций клиента с Azure Resource Manager (например, https://management.<ExternalFqdn>). |
| Конечная точка Resource Manager для администратора | Yes |  | Конечная точка для операций клиента с Azure Resource Manager (например, https://adminmanagement.<ExternalFqdn>). |
| Внешнее полное доменное имя | Yes |  | Внешнее полное доменное имя, используемое в качестве суффикса для конечных точек (например, local.azurestack.external или redmond.contoso.com). |
| Количество узлов | Yes |  | Количество узлов в развертывании. |

## <a name="test-parameters"></a>Параметры тестирования

Общие параметры теста включают конфиденциальную информацию, которую нельзя хранить в файлах конфигурации и нужно предоставлять вручную.

| Имя параметра | Обязательно | type | ОПИСАНИЕ |
|--------------------------------|------------------------------------------------------------------------------|------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Имя пользователя для клиента | Обязательно |  | Администратор клиента Azure Active Directory, который либо уже был подготовлен, либо должен быть подготовлен администратором службы в каталоге Azure AD Directory. Дополнительные сведения о подготовке учетной записи клиента см. в статье, посвященной [началу работы с Azure AD](https://docs.microsoft.com/azure/active-directory/get-started-azure-ad). Это значение используется в тесте для выполнения операций на уровне клиента, таких как развертывание шаблонов для подготовки ресурсов (виртуальных машин, учетных записей хранения и т. д.) и выполнения рабочих нагрузок. Это значение используется в тесте для выполнения операций на уровне клиента, таких как развертывание шаблонов для подготовки ресурсов (виртуальных машин, учетных записей хранения и т. д.) и выполнения рабочих нагрузок. |

| Пароль администратора службы | Требуется: проверка решения, проверка пакета.<br>Необязательно: проход теста |  | Пароль администратора службы. |

Инструкции по созданию универсального кода ресурса (URI) SAS доступны в статье, посвященной [настройке учетной записи хранения больших двоичных объектов](azure-stack-vaas-set-up-account.md). |

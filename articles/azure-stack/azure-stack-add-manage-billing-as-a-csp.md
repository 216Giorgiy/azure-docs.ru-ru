# <a name="manage-usage-and-billing-for-azure-stack-as-a-cloud-service-provider"></a>Управление потреблением и оплатой для Azure Stack в роли поставщика облачных служб 

*Область применения: интегрированные системы Azure Stack*

В этой статье подробно описаны процессы регистрации Azure Stack в качестве поставщика облачных служб (CSP) и добавления клиентов.

Если вы являетесь поставщиком облачных служб, вашу систему Azure Stack, скорее всего, используют многие клиенты с разными потребностями. У каждого клиента есть своя подписка CSP в Azure, и в них вам нужно передавать данные о потреблении Azure Stack.

Следующая схема демонстрирует шаги, позволяющие выбрать совместно используемые учетные записи служб и зарегистрировать их в учетной записи Azure. Завершив этот процесс, вы сможете подключать своих клиентов.

## <a name="create-a-csp-or-cspss-subscription"></a>Создание подписки CSP или CSPSS

### <a name="cloud-service-provider-subscription-types"></a>Типы подписки для поставщика облачных услуг

Вам нужно выбрать тип учетной записи для общих служб, которую вы будете использовать для Azure Stack. Ниже перечислены типы подписок, для которых можно зарегистрировать мультитенантную систему Azure Stack.

 - Поставщик облачных служб 
 - Подписка общих служб партнера 

#### <a name="csp-shared-services"></a>Общие службы CSP

Если Azure Stack находится под управлением Прямого партнера CSP или распространителя CSP, оптимальным решением будут подписки CSPSS (общие службы поставщика общих служб).

Подписки CSPSS связываются с клиентом общих служб. При регистрации Azure Stack нужно предоставить учетные данные той учетной записи, которая является владельцем подписки. Для регистрации Azure Stack вы можете использовать другую учетную запись, а не учетную запись администратора, с помощью которой вы развертывали Azure Stack. Они даже *не должны* принадлежать одному домену. Другими словами, для развертывания можно применить уже существующий клиент. Например, вы можете использовать ContosoCSP.onmicrosoft.com, а регистрацию выполнить от имени другого клиента IURContosoCSP.onmicrosoft.com. Но при этом не забывайте, что для рутинных задач администрирования Azure Stack нужно входить с учетной записью ContosoCSP.onmicrosoft.com. Учетная запись IURContosoCSP.onmicrosoft.com вам потребуется только для операций регистрации.

Описание подписок CSPSS и инструкции по созданию подписки приводятся в статье [Добавление служб Azure Partner Shared Services](https://msdn.microsoft.com/partner-center/shared-services).

#### <a name="csp-subscriptions"></a>Подписки CSP

Если Azure Stack находится под управлением клиента, лучше использовать подписки CSP (поставщика общих служб).

## <a name="charge-the-right-subscriptions"></a>Выбор правильной подписки для выставления счетов

Azure Stack использует специальный компонент, именуемый регистрацией. Это сохраненный в Azure объект, в котором хранятся данные о подписках Azure, используемые для снятие оплаты за потребление в конкретной инфраструктуре Azure Stack. В этом разделе мы расскажем, для чего нужна регистрация.

Благодаря регистрации Azure Stack получает следующие возможности.
 - Передавать данные о потреблении Azure Stack в Azure Commerce и выставлять счета в подписках Azure.
 - Передавать данные о потреблении ресурсов в другие подписки клиента, если используется мультитенантное развертывание Azure Stack. Мультитенантность позволяет Azure Stack поддерживать работу нескольких организаций в одном экземпляре Azure Stack.

Для каждого экземпляра Azure Stack применяется одна подписка по умолчанию и любое число клиентских подписок. Подписка Azure, указанная как подписка по умолчанию, будет использоваться для оплаты, если нет ни одной клиентской подписки. Ее необходимо зарегистрировать первой. Чтобы информирование о потреблении правильно работало для нескольких клиентов, необходимо использовать подписку CSP или CSPSS.

После этого в регистрацию вносятся данные о подписках Azure для каждого клиента, который будет использовать Azure Stack. Для клиентов необходимо использовать подписки типа CSP, принадлежащие тому же партнеру, который владеет подпиской по умолчанию. Другими словами, вы не можете включить в регистрацию чужих клиентов.

Когда Azure Stack передает в глобальную платформу Azure сведения о потреблении, служба в Azure сопоставляет каждого клиента с соответствующей подпиской, используя сведения регистрации. Если клиент не зарегистрирован, плата за потребление относится к подписке по умолчанию, указанной для соответствующего экземпляра Azure Stack.

Поскольку подписки клиентов имеют тип CSP, все счета направляются партнеру CSP, и клиентам не доступны сведения о потреблении.

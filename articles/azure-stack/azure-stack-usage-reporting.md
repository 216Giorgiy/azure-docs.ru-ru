# <a name="report-azure-stack-usage-data-to-azure"></a>Передача в Azure данных об использовании из Azure Stack 

Данные об использовании, также называемые данными о потреблении, представляют собой объем используемых ресурсов. Интегрированные системы Azure Stack, применяющие модель выставления счетов на основе потребления ресурсов, должны отправлять данные об использовании в Azure. Эти данные необходимы для выставления счетов. За настройку передачи данных об использовании в среде Azure Stack отвечает оператор Azure Stack.


> [!NOTE]
> Отправка данных об использовании обязательна для пользователей интегрированных систем Azure Stack, применяющих модель оплаты по мере использования. При этом она необязательна для пользователей, использующих модель на основе емкости. При использовании комплекта разработки Azure Stack операторы облака могут передавать данные о потреблении и тестировать эту возможность. Тем не менее с них не взимается плата за использование. Сведения о ценах на Azure Stack см. на [этой странице](https://azure.microsoft.com/overview/azure-stack/how-to-buy/).

## <a name="usage-data-reporting-flow"></a>Поток передачи отчетов по данным об использовании

Данные об использовании отправляются из Azure Stack в Azure через мост Azure. В Azure коммерческая система обрабатывает данные об использовании и создает счет. После создания счета владелец подписки Azure может просмотреть и загрузить его в [Центре управления учетной записью Azure](https://account.windowsazure.com/Subscriptions). Чтобы узнать, как лицензируется Azure Stack, см. [документ с информацией о пакетах и ценах на Azure Stack](https://go.microsoft.com/fwlink/?LinkId=842847&clcid=0x409).

## <a name="set-up-usage-data-reporting"></a>Настройка передачи данных об использовании

Чтобы настроить передачу данных об использовании в Azure Stack, необходимо [зарегистрировать экземпляр Azure Stack в Azure](azure-stack-register.md). В рамках процесса регистрации компонент моста Azure Stack настраивается для подключения Azure Stack к Azure. После настройки мост Azure отправляет следующие данные об использовании в Azure: 

* **Идентификатор единицы измерения** — уникальный идентификатор использованного ресурса.
* **Количество** — объем данных об использовании ресурса.
* **Расположение** — расположение развертывания текущего ресурса Azure Stack.
* **URI ресурса** — полный универсальный код ресурса, об использовании которого передаются данные. 
* **Идентификатор подписки** — идентификатор подписки пользователя Azure Stack.
* **Время** — время начала и окончания передачи данных об использовании. Между потреблением ресурсов в Azure Stack и передачей данных об использовании в коммерческую систему происходит некоторая задержка. Azure Stack вычисляет данные об использовании каждые 24 часа, а передача данных об использовании в конвейер коммерческой системы в Azure занимает еще несколько часов. Таким образом использование, происходящее около полуночи, может отобразиться в Azure на следующий день.

## <a name="test-usage-data-reporting"></a>Тестирование передачи данных об использовании 

1. Чтобы протестировать передачу данных об использовании, создайте несколько ресурсов в Azure Stack. Например, можно создать [учетную запись хранения](azure-stack-provision-storage-account.md), [виртуальную машину Windows Server](azure-stack-provision-vm.md) и Linux с базовым и стандартным номером SKU, чтобы увидеть, каким образом передаются данные об использовании ядер. Данные об использовании различных типов ресурсов передаются в различных единицах измерения.  

2. Запустите ресурсы и дайте им поработать несколько часов. Сведения об использовании собираются примерно один раз в час. После сбора эти данные передаются в Azure и обрабатываются в коммерческой системе Azure. Этот процесс может занять до нескольких часов.  

3. Войдите в [Центр управления учетной записью Azure](https://account.windowsazure.com/Subscriptions) с правами администратора учетной записи Azure и выберите подписку Azure, которую необходимо использовать для регистрации Azure Stack. Вы можете просмотреть данные об использовании в Azure Stack, сумму к оплате за каждый из используемых ресурсов, как показано на следующем рисунке:  

   ![поток выставления счетов](media/azure-stack-usage-reporting/pricing-details.png)

При использовании комплекта разработки Azure Stack плата за ресурсы не взимается, поэтому отображается цена 0,00 долл. При использовании интегрированных систем Azure Stack отображается фактическая стоимость каждого из этих ресурсов.

## <a name="are-users-charged-for-the-infrastructure-virtual-machines"></a>Взимается ли с пользователей плата за виртуальные машины инфраструктуры?
Нет, данные об использовании для некоторых виртуальных машин поставщика ресурсов Azure Stack и виртуальных машин инфраструктуры, созданные во время развертывания, передаются в Azure, но плата за них не взимается. 

С пользователей взимается плата только за виртуальные машины, созданные в подписках пользователей. Таким образом все рабочие нагрузки необходимо развернуть в подписках пользователей в соответствии с условиями лицензирования Azure Stack.

## <a name="how-do-i-use-my-existing-windows-server-licenses-in-azure-stack"></a>Как использовать имеющиеся лицензии Windows Server в Azure Stack? 
Имеющиеся лицензии Windows Server можно использовать в Azure Stack, эта модель называется BYOL (с использованием собственной лицензии). Применение имеющейся лицензии позволяет избежать создания дополнительных единиц измерения использования. Чтобы использовать имеющиеся лицензии, необходимо развернуть виртуальные машины Windows Server, как описано в статье [Льгота на гибридное использование Azure для Windows Server](../virtual-machines/windows/hybrid-use-benefit-licensing.md). 

## <a name="what-azure-meters-are-used-when-reporting-usage-data-in-integrated-systems"></a>Какие единицы измерения Azure используются при передаче данных об использовании в интегрированных системах?
* **Единицы измерения полной цены** — используются для ресурсов, связанных с рабочими нагрузками пользователя.  
* **Единицы измерения администратора** — используются для ресурсов инфраструктуры. Для этих единиц измерения установлена цена в ноль долларов.

## <a name="which-subscription-is-charged-for-the-resources-consumed"></a>В каких подписках взимается плата за использованные ресурсы?
Плата взимается в подписке, предоставляемой при [регистрации Azure Stack в Azure](azure-stack-register.md).

## <a name="what-types-of-subscriptions-are-supported-for-usage-data-reporting"></a>В каких типах подписок поддерживается передача данных об использовании?
При использовании интегрированных систем Azure Stack поддерживаются подписки Enterprise Agreement (EA) и CSP. 

При использовании комплекта разработки Azure Stack передача данных об использовании поддерживается в подписках Enterprise Agreement (EA), подписке с оплатой по мере использования, CSP и MSDN.

## <a name="which-sovereign-clouds-support-usage-data-reporting"></a>Какие независимые облака поддерживают передачу данных об использовании?
Для передачи данных об использовании в комплекте разработки Azure Stack необходимы подписки, созданные в глобальной системе Azure. Подписки, созданные в одном из независимых облаков (Azure для государственных организаций, Azure для Германии или Azure для Китая), невозможно зарегистрировать в Azure, поэтому они не поддерживают передачу данных об использовании. 

## <a name="how-can-users-identify-azure-stack-usage-data-in-the-azure-billing-portal"></a>Как пользователи могут определить данные об использовании Azure Stack на портале управления счетами и подписками Azure?
Пользователи могут просмотреть данные об использовании Azure Stack в файле со сведениями о потреблении. Чтобы узнать о том, как получить этот файл, см. раздел [Скачивание данных об использовании в Центре управления учетной записью (в формате CSV)](../billing/billing-download-azure-invoice-daily-usage-date.md#download-usage-from-the-account-center-csv). Файл со сведениями об использовании содержит единицы измерения Azure Stack, определяющие хранилище и виртуальные машины Azure Stack. Данные о всех ресурсах, используемых в Azure Stack, отправляются в области с именем "Azure Stack".

## <a name="why-doesnt-the-usage-reported-in-azure-stack-match-the-report-generated-from-azure-account-center"></a>Почему данные об использовании в Azure Stack не соответствуют данным, созданным в Центре управления учетной записью Azure?
Между данными об использовании, передаваемыми API-интерфейсами использования Azure Stack, и данными об использовании, передаваемыми Центром управления учетной записью Azure, возникает задержка. Эта задержка обуславливается временем, необходимым для отправки данных об использовании из Azure Stack в коммерческую систему Azure. Из-за этой задержки использование, происходящее около полуночи, может отобразиться в Azure на следующий день. Вы можете увидеть различие, если используете [API-интерфейсы использования Azure Stack](azure-stack-provider-resource-api.md) и сравните результаты с данными о потреблении на портале управления счетами и подписками Azure.

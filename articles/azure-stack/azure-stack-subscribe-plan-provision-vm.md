# <a name="create-subscriptions-to-offers-in-azure-stack"></a>Создание подписок для предложений в Azure Stack

*Область применения: интегрированные системы Azure Stack и Пакет средств разработки Azure Stack*

После [создания предложения](azure-stack-create-offer.md) пользователям необходимо создать подписку на него, чтобы они могли его использовать.   
- Как оператор облака вы можете создать подписку для пользователя на портале администратора.  Подписки можно создать для общедоступных и частных предложений.
- Как пользователь клиента вы можете подписаться на общедоступное предложение, используя пользовательский портал.  

В разделах ниже содержатся сведения о том, как создать пользовательские подписки в качестве операторов облака, а также как подписаться на предложение как пользователь.

## <a name="create-a-subscription-as-a-cloud-operator"></a>Создание подписки в качестве оператора облака
Операторы облака могут создать подписку на предложение для пользователя на портале администрирования.  Вы можете создать подписки для участников своего клиента каталога.  Если включена [мультитенантность](azure-stack-enable-multitenancy.md), можно также создать подписки для пользователей в дополнительных клиентах каталога.

Подписки можно создать для общедоступных и частных предложений.  Если вы не хотите, чтобы клиенты создавали собственные подписки, сделайте все предложения частными, а затем создайте подписки от имени клиентов. Этот подход широко используется при интеграции Azure Stack с внешними системами выставления счетов или каталогов службы.

Пользователь, для которого создали подписку, войдет на портал и узнает о том, что он подписан на предложение.  

### <a name="to-create-the-subscription-for-a-user"></a>Создание подписки для пользователя
1.  На портале администрирования перейдите в раздел **пользовательских подписок**.
2.  Выберите **Добавить**, чтобы открыть панель **Новая пользовательская подписка**. Укажите здесь следующие сведения.  

  a. **Отображаемое имя** — понятное имя для идентификации подписки, которая отображается в виде *имени подписки пользователя*.

  Б. **Пользователь** — укажите пользователя из доступного клиента каталога для этой подписки. Этот пользователь будет отображаться как *владелец*.  Формат имени пользователя зависит от решения для идентификации. Например:    

     -  **Azure AD:** *&lt;user1>@&lt;contoso.onmicrosoft.com>*

     -   **AD FS:** *&lt;user1>@&lt;azurestack.local>*     

  c.    **Клиент каталога** — выберите клиент каталога, которому принадлежит учетная запись пользователя. Если мультитенантность отключена, доступен будет только локальный клиент каталога.

3.  Выберите **Предложение**, чтобы открыть панель **предложений**, а затем выберите **предложение** для этой подписки. Так как вы создаете подписку для пользователя, можно выбрать общедоступное или частное предложение.

4.  Выберите **Создать**, чтобы создать подписку. Теперь на панели **пользовательских подписок** появится новая подписка.  Позже, когда пользователь выполнит вход на пользовательский портал, он сможет просмотреть подробные сведения об этой подписке.

### <a name="to-make-an-add-on-plan-available"></a>Как сделать доступным план надстройки  
Оператор облака может добавить план надстройки в ранее созданную подписку в любое время:   
1.  На портале администрирования выберите **Больше служб** > **User subscriptions** (Пользовательские подписки) и выберите подписку, которую нужно изменить.   

2.  Выберите **Надстройки** > **Добавить**, чтобы открыть панель **Добавить план**.  

3.  Выберите план, который нужно добавить как надстройку.  Затем на консоли появятся планы, связанные с подпиской.




## <a name="create-a-subscription-as-a-user"></a>Создание подписки в качестве пользователя
От имени пользователя войдите на пользовательский портал, чтобы из клиента каталога (организации) найти общедоступные предложения и планы надстройки и подписаться на них. Когда в среде Azure Stack станет доступна поддержка [мультитенантности](azure-stack-enable-multitenancy.md), вы сможете подписаться на предложения из удаленного клиента каталога.

### <a name="to-subscribe-to-an-offer"></a>Оформление подписки на предложение
1. [Войдите](azure-stack-connect-azure-stack.md) на портал пользователя Azure Stack (https://portal.local.azurestack.external) и щелкните **Получить подписку**.

   ![Получение подписки](media/azure-stack-subscribe-plan-provision-vm/image01.png)
2. В поле **Отображаемое имя** введите имя подписки, щелкните **Предложение**, выберите одно из предложений на панели **Выбор предложения**, а затем нажмите кнопку **Создать**.

   ![Создание предложения](media/azure-stack-subscribe-plan-provision-vm/image02.png)
3. Чтобы просмотреть созданную подписку, выберите **More services** (Больше служб), щелкните **Подписки** и выберите новую подписку.  

Подписавшись на предложение, обновите страницу портала, чтобы узнать, какие службы входят в новую подписку.

### <a name="to-subscribe-to-an-add-on-plan"></a>Оформление подписки на план надстройки
Если предложение содержит план надстройки, вы можете добавить его в свою подписку в любое время.  

1. На портале пользователя выберите **Больше служб** > **Подписки**, а затем выберите подписку, которую нужно изменить.

2. Нажмите кнопку **Добавить план**, а затем выберите нужный план надстройки. Если кнопка **Добавить план** неактивна, значит планы надстройки для предложения, связанные с этой подпиской, отсутствуют.

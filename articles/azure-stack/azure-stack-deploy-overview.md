# <a name="quickstart-evaluate-the-azure-stack-development-kit"></a>Краткое руководство по оценке пакета средств разработки Azure Stack
[Пакет средств разработки Azure Stack (ASDK)](.\asdk\asdk-what-is.md) — это среда тестирования и разработки, которую можно развернуть для оценки и демонстрации функций и служб Azure Stack. Чтобы приступить к работе с ASDK, необходимо подготовить оборудование компьютера для размещения пакета, а затем запустить несколько скриптов (установка займет несколько часов). После этого вы сможете войти на портал администратора и портал пользователя, чтобы приступить к работе с Azure Stack.

## <a name="prerequisites"></a>предварительным требованиям 
Прежде чем устанавливать ASDK, необходимо подготовить компьютер, на котором будет размещен пакет средств разработки (узел для пакета средств разработки). Компьютер, на котором размещается пакет средств разработки, должен соответствовать требованиям к оборудованию, программному обеспечению и сети. 

Также вам нужно выбрать поставщик удостоверений для развертывания: Azure Active Directory (Azure AD) или службы федерации Active Directory (AD FS). 

Узел для пакета средств разработки должен соответствовать минимальным требованиям к оборудованию. Кроме того, нужно выбрать решение для удостоверений перед началом развертывания, чтобы процесс установки прошел без сбоев. 

**[Ознакомьтесь с рекомендациями по планированию развертывания ASDK](.\asdk\asdk-deploy-considerations.md)**

> [!TIP]
> Чтобы убедиться, что оборудование соответствует всем требованиям, можно использовать [средство проверки требований к развертыванию Azure Stack](https://gallery.technet.microsoft.com/Deployment-Checker-for-50e0f51b) после установки операционной системы на компьютере, на котором будет размещен пакет средств разработки.

## <a name="download-and-extract-the-deployment-package"></a>Скачивание пакета развертывания и извлечение его содержимого
Когда вы удостоверитесь, что компьютер, на котором будет размещен пакет средств разработки, соответствует базовым требованиям для установки ASDK, скачайте и пакет ASDK и извлеките его содержимое. Пакет развертывания содержит файл Cloudbuilder.vhdx, который является виртуальным жестким диском с загружаемой операционной системой и файлами установки Azure Stack.

Пакет развертывания можно скачать на компьютер, где размещается пакет средств разработки, или на другой компьютер. Извлекаемые файлы развертывания занимают 60 ГБ дискового пространства, поэтому использование другого компьютера помогает снизить требования к оборудованию для узла пакета средств разработки.

**[Скачивание Пакета средств разработки Azure Stack (ASDK) и извлечение его содержимого](.\asdk\asdk-download.md)**

## <a name="prepare-the-development-kit-host-computer"></a>Подготовка компьютера, на котором будет размещен пакет средств разработки
Перед установкой ASDK на компьютере для его размещения необходимо подготовить среду и настроить загрузку системы с виртуального жесткого диска. После подготовки компьютера, на котором будет размещен пакета средств разработки, он загружается с виртуального жесткого диска CloudBuilder.vhdx, чтобы можно было начать развертывание ASDK.

**[Подготовка компьютера для размещения ASDK](.\asdk\asdk-prepare-host.md)**

## <a name="install-the-asdk-on-the-host-computer"></a>Установка ASDK на компьютере для размещения пакета
После подготовки компьютера, на котором будет размещен пакета средств разработки ASDK, можно развернуть его в образ CloudBuilder.vhdx. ASDK можно развернуть с помощью графического интерфейса пользователя (GUI). Для этого загрузите и запустите скрипт PowerShell asdk-installer.ps1 или разверните ASDK полностью из [командной строки](.\asdk\asdk-deploy-powershell.md). 

> [!NOTE]
> При необходимости после загрузки компьютера для размещения пакета на диск CloudBuilder.vhdx можно настроить [параметры телеметрии Azure Stack](.\asdk\asdk-telemetry.md#set-telemetry-level-in-the-windows-registry) *до* установки ASDK.


**[Установка Пакета средств разработки Azure Stack (ASDK)](.\asdk\asdk-install.md)**

## <a name="perform-post-deployment-configurations"></a>Настройка после развертывания
После установки ASDK нужно выполнить несколько рекомендуемых проверок и внести некоторые изменения в конфигурацию. Чтобы проверить правильность установки, используйте командлет test-AzureStack и установите средства PowerShell и GitHub для Azure Stack. 

После развертывания с использованием Azure Active Directory необходимо активировать порталы администрирования и клиента Azure Stack. Эта активация заключается в предоставлении порталу Azure Stack и Azure Resource Manager требуемых разрешений (перечисленные на странице принятия условий) для всех пользователей каталога.

Чтобы срок действия пароля на вход в главный компьютер со средствами разработки не завершился до окончания периода ознакомления, сбросьте политику истечения срока действия пароля.

> [!NOTE]
> Кроме того, при необходимости можно настроить [параметры телеметрии Azure Stack](.\asdk\asdk-telemetry.md#enable-or-disable-telemetry-after-deployment) *после* установки ASDK.

**[Задачи, выполняемые после развертывания ASDK](.\asdk\asdk-post-deploy.md)**

# <a name="quickstart-evaluate-the-azure-stack-development-kit"></a>Краткое руководство по оценке пакета средств разработки Azure Stack

[Пакет средств разработки Azure Stack (ASDK)](./asdk/asdk-what-is.md) — это среда тестирования и разработки, которую можно развернуть для оценки и демонстрации функций и служб Azure Stack. Чтобы приступить к работе с ASDK, необходимо подготовить оборудование компьютера для размещения пакета, а затем запустить несколько скриптов (установка займет несколько часов). После этого вы сможете войти на портал администратора или портал пользователя, чтобы начать использовать Azure Stack.

## <a name="prerequisites"></a>Предварительные требования

### <a name="asdk-host-computer-requirements"></a>Требования к компьютеру узла ASDK

Прежде чем устанавливать ASDK, необходимо подготовить компьютер, на котором будет размещен пакет средств разработки. Узел для размещения пакета средств разработки должен соответствовать требованиям к оборудованию, программному обеспечению и сети, которые описаны в статье **[Рекомендации по планированию развертывания Azure Stack](./asdk/asdk-deploy-considerations.md)**.

> [!TIP]
> Чтобы убедиться, что оборудование соответствует всем требованиям, можно использовать [средство проверки требований к развертыванию Azure Stack](https://gallery.technet.microsoft.com/Deployment-Checker-for-50e0f51b) после установки операционной системы на компьютере, на котором будет размещен пакет средств разработки.

### <a name="account-requirements"></a>Требования к учетной записи

Также вам нужно выбрать поставщик удостоверений для развертывания: Azure Active Directory (Azure AD) или службы федерации Active Directory (AD FS). Проверьте требования к учетной записи в разделе **[Требования к учетной записи](./asdk/asdk-deploy-considerations.md#account-requirements)**.

## <a name="download-and-extract-the-deployment-package"></a>Скачивание пакета развертывания и извлечение его содержимого

Завершив подготовку компьютера в качестве узла для пакета средств разработки, скачайте и извлеките пакет развертывания ASDK. Пакет развертывания содержит файл Cloudbuilder.vhdx с виртуальным жестким диском (VHD), на котором размещены загружаемая операционная система и файлы установки Azure Stack.

Пакет развертывания можно скачать на компьютер, где размещается пакет средств разработки, или на другой компьютер. Извлекаемые файлы развертывания занимают 60 ГБ дискового пространства, поэтому использование другого компьютера помогает снизить требования к пространству для хранения на узле пакета средств разработки.

**[Скачивание Пакета средств разработки Azure Stack (ASDK) и извлечение его содержимого](./asdk/asdk-download.md)**

## <a name="prepare-the-host-computer"></a>Подготовка компьютера для размещения

Перед установкой ASDK необходимо подготовить среду узла и настроить загрузку системы с виртуального жесткого диска. После этого перезапустите узел, чтобы он загрузился из файла CloudBuilder.vhdx, и приступайте к развертыванию ASDK.

**[Подготовка компьютера для размещения ASDK](./asdk/asdk-prepare-host.md)**

## <a name="install-the-asdk-on-the-host-computer"></a>установка ASDK на компьютере для размещения пакета;

Когда компьютер узла загрузится с виртуального жесткого диска, вы можете развернуть пакет средств разработки в виртуальном окружении Cloudbuilder. Вы можете развернуть ASDK через графический интерфейс пользователя (GUI), который можно вызвать с помощью скрипта PowerShell asdk-installer.ps1, или из [командной строки PowerShell](./asdk/asdk-deploy-powershell.md).

> [!NOTE]
> После загрузки узла из образа CloudBuilder.vhdx вы можете настроить [параметры телеметрии Azure Stack](./asdk/asdk-telemetry.md#set-telemetry-level-in-the-windows-registry) *перед* установкой ASDK (необязательно).

**[Установка Пакета средств разработки Azure Stack (ASDK)](./asdk/asdk-install.md)**

## <a name="perform-post-deployment-configurations"></a>Настройка после развертывания

После установки ASDK нужно выполнить несколько рекомендуемых проверок и внести некоторые изменения в конфигурацию.

**Инструменты**

Установите средства PowerShell и GitHub для Azure Stack и проверьте правильность установки, используя командлет test-AzureStack.

**Решение для идентификации**

Для развертываний, которые используют Azure Active Directory, необходимо включить оба портала Azure Stack (для администратора и клиента). Эта активация заключается в предоставлении порталу Azure Stack и Azure Resource Manager требуемых разрешений (перечисленные на странице принятия условий) для всех пользователей каталога.

**Срок действия пароля**

Сбросьте политику истечения срока действия пароля, чтобы срок действия пароля для входа на главный компьютер со средствами разработки не завершился до окончания периода ознакомления.

> [!NOTE]
> Вы также можете настроить [параметры телеметрии Azure Stack](./asdk/asdk-telemetry.md#enable-or-disable-telemetry-after-deployment) *после* установки ASDK.

**[Задачи, выполняемые после развертывания ASDK](./asdk/asdk-post-deploy.md)**

# <a name="tutorial-update-offers-and-plans"></a>Руководство по обновлению планов и предложений
Как оператор Azure Stack, вы создаете планы, содержащие перечень требуемых служб и применимых квот, на которые пользователи могут подписаться. *Базовые планы* включают основные службы, предлагаемые пользователям. При этом у вас может быть только один базовый план по предлагаемым службам. Если вам нужно изменить предложение, можно использовать *дополнительные планы*, позволяющие наращивать вычислительные мощности, хранилище или выделенные сетевые квоты, первоначально включенные в базовый план. 

Несмотря на то, что объединение всех требований в одном плане и может быть оптимальным в некоторых случаях, тем не менее, в других ситуациях у вас может возникнуть потребность в наличии как базового плана, так и предложений по дополнительным службам в дополнительных планах. К примеру, можно предложить службы IaaS в составе базового плана, а все службы PaaS – через дополнительные планы. Планы также можно использовать для управления потреблением ограниченных ресурсов среды ASDK. Например, если вы хотите, чтобы ваши пользователи экономно относились к расходуемым ресурсам, можно создать относительно небольшой базовый план (в зависимости от необходимого набора служб), и как только пользователи будут достигать оговоренных в планах пределов выделенных ресурсов, они будут получать соответствующие оповещения. После этого они смогут воспользоваться возможностями дополнительного плана для получения необходимых добавочных ресурсов. 

> [!NOTE]
> Если пользователь воспользуется дополнительным планом к основной подписке, тогда соответствующие запрошенные ресурсы появятся в его распоряжении не позднее, чем через час.

## <a name="create-an-add-on-plan"></a>Создание дополнительного плана
**Дополнительные планы** создаются путем изменения имеющегося предложения.

1. Войдите на [портал Azure Stack](https://adminportal.local.azurestack.external) с учетной записью администратора облака.
2. Для создания служб предложения плана, не предлагавшихся ранее, выполните те же действия, что и ранее, по [созданию базового плана](asdk-offer-services.md). В этом примере службы Key Vault (Microsoft.KeyVault) будут включены в план.
3. На портале администратора щелкните **Предложения**, а затем выберите вариант обновления предложения с использованием дополнительного плана.

   ![](media/asdk-update-offers/1.PNG)

4.  Прокрутите свойства предложения вниз и выберите **Дополнительные планы**. Щелкните **Добавить**.
   
    ![](media/asdk-update-offers/2.PNG)

5. Выберите план для добавления. В этом примере запрашиваем план под названием **План хранилища ключей** и нажимаем кнопку **Выбор** для добавления этого плана в предложение. Вы должны получить уведомление о том, что план успешно добавлен в предложение.
   
    ![](media/asdk-update-offers/3.PNG)

6. Просмотрите список дополнительных планов, включенных в предложение, чтобы убедиться, что новый дополнительный план находится в списке.
   
    ![](media/asdk-update-offers/4.PNG)

## <a name="subscribe-to-the-add-on-plan"></a>Подписка на дополнительный план
Необходимо выполнить вход на портал пользователя Azure Stack для расширения имеющейся подписки Azure Stack с помощью дополнительного плана.

Выполните эти действия для изучения дополнительных ресурсов, предлагаемых оператором Azure Stack, и добавления дополнительного плана в предложение, на которое вы подписались ранее.

1. Войдите на [портал пользователя](https://portal.local.azurestack.external).
2. Чтобы найти подписку, возможности которой необходимо расширить с помощью дополнительного плана, нажимаем кнопку **Дополнительные службы**, далее кнопку **Подписки**, а затем выбираем свою подписку.
   
    ![](media/asdk-update-offers/5.PNG)

3.  В обзоре возможностей подписки щелкните **Добавить план**.
   
    ![](media/asdk-update-offers/6.PNG)

4. В колонке добавления плана выберите дополнительный план, чтобы добавить его в подписку. В этом примере выбран вариант **План хранилища ключей**. Затем вы должны получить уведомление о том, что дополнительный план получен и что необходимо обновить портал для доступа к обновленной подписке.
   
    ![](media/asdk-update-offers/7.PNG)

5. И в завершение просмотрите дополнительные планы, связанные с той же подпиской, чтобы убедиться, что дополнительный план был успешно добавлен.
   
    ![](media/asdk-update-offers/8.PNG)

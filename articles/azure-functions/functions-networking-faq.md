# <a name="frequently-asked-questions-about-networking-in-azure-functions"></a>Часто задаваемые вопросы о работе в сети в функциях Azure

Ниже приведен список часто задаваемых вопросов по сети. Более подробный обзор. в статье [функции сетевые параметры документа](functions-networking-options.md)

## <a name="how-do-i-set-a-static-ip-in-functions"></a>Как задать статический IP-адрес в функции?

Развертывание функции в среде службы приложений (ASE) — в настоящее время единственным способом статический входящий и исходящий IP-адрес для своей функции. Дополнительные сведения об использовании ASE начните со статьи здесь: [Создание и использование ASE с ILB](../app-service/environment/create-ilb-ase.md).

## <a name="how-do-i-restrict-internet-access-to-my-function"></a>Как ограничить доступ к Интернету для Моя функция?

Можно ограничить доступ к Интернету в ряд способов, перечисленных ниже.

* [Ограничения IP-адресов](../app-service/app-service-ip-restrictions.md): ограничить входящий трафик для приложения-функции, диапазон IP-адресов.
* Удалите все триггеры HTTP. Для некоторых приложений достаточно просто избегать триггеров HTTP и использовать любые другие источники событий для активации функции.

Наиболее важным аспектом при этом — Имейте в виду, требующий прямой доступ на выполнение функции для использования в редакторе портала Azure. Любые изменения кода, на портале Azure требуется устройство, которое используется для просмотра на портале, чтобы его список разрешенных IP-адрес. Тем не менее, тем не менее, воспользуйтесь все вкладка функций платформы с ограничениями сети на месте.

## <a name="how-do-i-restrict-my-function-app-to-a-vnet"></a>Как ограничить Мои приложения-функции к виртуальной сети?

Единственный способ полностью ограничить функции таким образом, что весь трафик идет через виртуальную сеть является использование внутренне с балансировкой нагрузки (ILB) среда службы приложений (ASE). Этот параметр развертывает сайта в выделенной инфраструктуре внутри виртуальной сети и отправляет все триггеры и трафика через виртуальную сеть. Дополнительные сведения об использовании ASE начните со статьи здесь: [Создание и использование ASE с ILB](../app-service/environment/create-ilb-ase.md).

## <a name="how-can-i-access-resources-in-a-vnet-from-a-function-app"></a>Как получить доступ к ресурсам в виртуальной сети из приложения-функции?

Доступ к ресурсам в виртуальной сети можно из работающей функции с использованием интеграции виртуальной сети. Дополнительные сведения см. в разделе [интеграции с виртуальной СЕТЬЮ](functions-networking-options.md#vnet-integration)

## <a name="how-do-i-access-resources-protected-by-service-endpoints"></a>Как получить доступ к ресурсам, защищенным конечные точки службы?

С помощью новой интеграции виртуальной сети (в настоящее время в предварительной версии), можно получить доступ к защищенным ресурсам из функции, работающей конечной точки службы. Дополнительные сведения см. в разделе [предварительную интеграцию виртуальной сети](functions-networking-options.md#preview-vnet-integration).

## <a name="how-can-i-trigger-a-function-from-a-resource-in-a-vnet"></a>Как можно вызвать функцию из ресурса в виртуальной сети?

Функция из ресурса в виртуальной сети может активировать только путем развертывания приложения-функции в среде службы приложений. Дополнительные сведения об использовании ASE см. в разделе [Создание и использование ASE с ILB](../app-service/environment/create-ilb-ase.md).


## <a name="how-can-i-deploy-my-function-app-in-a-vnet"></a>Как можно развернуть Мои приложения-функции в виртуальной сети?

Развертывание в среде службы приложений является единственным способом для создания приложения-функции, полностью находящийся внутри виртуальной сети для сведения об использовании ILB ASE, начните со статьи здесь: [Создание и использование ASE с ILB](https://docs.microsoft.com/azure/app-service/environment/create-ilb-ase).

Для сценариев, где требуется только односторонний доступ к ресурсам виртуальной сети, меньше полный сетевой изоляции, см. в разделе [сети Обзор функций](functions-networking-options.md).

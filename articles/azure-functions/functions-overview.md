# Обзор функций Azure

## Использовать функции — просто

Вы можете написать любую функцию менее чем за минуту, например, если вам нужно запустить простое задание очистки базы данных или создать функцию, которая обрабатывает миллионы сообщений от подключенных устройств. При этом вы можете использовать любой привычный вам язык программирования (C#, Node.JS, Python и др.). Оплата взимается только один раз при выполнении кода, а масштабирование в Azure выполняется при необходимости.

Функции Azure — это решение для быстрого запуска фрагментов кода (функций) в облаке. Вы можете быстро написать код для решения своих задач, не беспокоясь о наличии приложения или инфраструктуры для его выполнения. Это делает разработку более продуктивной. [Начать работу с функциями](functions-create-first-azure-function.md) можно буквально за несколько минут.

## Что можно сделать с помощью функций

Функции Azure — это идеальное решение для обработки данных, интеграции систем, работы с Интернетом вещей (IoT) и создания простых API-интерфейсов и микрослужб.

Функции включают коллекцию шаблонов для распространенных сценариев, таких как:

* реагирование на запрос веб-перехватчика GitHub;
* изменение размера изображения, отправленного в хранилище Azure;
* работа с порядком обработки очередей;
* и многое другое. 

Дополнительные сведения о работе функций с примерами конфигураций см. в [справочнике разработчика](functions-reference.md).

## Функции

Функции Azure — это полнофункциональная, готовая к использованию в организации платформа с компонентами, которые упрощают выполнение задач интеграции и обмена данными. Используя базовые возможности функций Azure, разработчики могут работать еще продуктивнее, сосредотачиваясь только на своих целях, а не на объединении частей инфраструктуры.

Ниже представлены характеристики функций Azure.
    
* **Выбор языка** — вы можете писать функции с помощью C#, Node.js, Python, F #, PHP, пакетной службы, bash, Java или любого исполняемого файла.  
* **Модель ценообразования с оплатой за использование** — вы платите только за время, в течение которого выполняется код. См. сведения о [динамическом плане службы приложений](#pricing) ниже.  
* **Использование собственных зависимостей** — функции поддерживают NuGet и NPM, поэтому вы можете использовать предпочитаемые библиотеки.  
* **Встроенная система безопасности** — вы можете защитить функции с HTTP-активацией, которые используют поставщики OAuth (Azure Active Directory, Facebook, Google, Twitter и учетная запись Майкрософт).  
* **Интеграция без использования кода** — вы можете пользоваться службами Azure и предложениями SaaS (программное обеспечение как услуга). Примеры см. ниже в разделе, посвященном [типам интеграции](#integrations).  
* **Гибкая разработка** — вы можете вносить изменения в функции с помощью редактора на портале, а также настраивать непрерывную интеграцию и развертывание кода с помощью GitHub, Visual Studio Team Services и других средств.  
* **Открытый код** — функции имеют открытый исходный код, [доступный на GitHub](https://github.com/azure/azure-webjobs-sdk-script).

## <a name="pricing"></a>Расценки на использование функций

Есть два способа запуска функций Azure: с помощью динамического и классического планов службы приложений.

При использовании **динамического плана службы приложений** вам не нужно беспокоиться об управлении ресурсами. При запуске функции среда Azure предоставляет все необходимые вычислительные ресурсы. Вы платите только за время, в течение которого выполняется код. Подробные сведения о ценообразовании см. на странице с информацией о [ценах](/pricing/details/functions).

**Классический план службы приложений** позволяет запускать функции как веб-приложения, мобильные приложения и приложения API. Это отличное решение, если вы уже используете службу приложений для других приложений — функции можно запускать в том же плане без дополнительной оплаты. Подробные сведения см. на странице с информацией о [ценах на службу приложений](/pricing/details/app-service/).

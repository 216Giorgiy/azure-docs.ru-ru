# <a name="task-hubs-in-durable-functions-azure-functions"></a>Центры задач в устойчивых функциях (Функции Azure)

*Центр задач* для [устойчивых функций](durable-functions-overview.md) — это логический контейнер для оркестраций и действий в контексте отдельной учетной записи хранилища Azure. В одном центре задач может существовать множество функций и даже приложений-функций. Центр задач зачастую служит контейнером приложений.

Явно создавать центры задач нет необходимости. Они автоматически инициализируются средой выполнения с использованием имени, объявленного в файле *host.json*. Для каждого центра задач имеется собственный набор очередей хранилища, таблиц и больших двоичных объектов в пределах одной учетной записи хранения. Все приложения-функции, работающие в определенном центре задач, используют общие ресурсы хранилища. Функции оркестраторов и действий могут взаимодействовать друг с другом, только когда они принадлежат к одному центру задач.

## <a name="configuring-a-task-hub-in-hostjson"></a>Настройка центра задач в файле host.json

Имя центра задач можно настроить в файле *host.json* приложения-функции.

```json
{
  "durableTask": {
    "HubName": "MyTaskHub"
  }
}
```

Имена центров задач должны начинаться с буквы и содержать только буквы и цифры. Если имя не указано, центру задач в приложении-функции назначается имя по умолчанию **DurableFunctionsHub**.

> [!NOTE]
> При наличии нескольких приложений-функций, совместно использующих учетную запись хранения, рекомендуется настроить по отдельному имени центра задач для каждого приложения-функции. Это обеспечит надлежащую изоляцию приложений-функций друг от друга.

## <a name="azure-storage-resources"></a>Ресурсы хранилища Azure

Центр задач состоит из нескольких ресурсов хранилища Azure:

* Одна или несколько очередей элементов управления.
* Одна очередь рабочих элементов.
* Одна таблица журнала.
* Один контейнер хранилища, содержащий один или несколько арендуемых больших двоичных объектов.

Все эти ресурсы создаются автоматически в учетной записи хранилища Azure по умолчанию, когда функции оркестраторов и действий выполняются или запланированы к выполнению. В статье [Производительность и масштабирование](durable-functions-perf-and-scale.md) объясняется, как используются эти ресурсы.

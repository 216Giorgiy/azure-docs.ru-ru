# Как управлять содержимым в службах мультимедиа
Представление контента служб мультимедиа Azure позволяет управлять контентом мультимедиа для вашей учетной записи служб мультимедиа.

В настоящее время непосредственно на портале можно выполнять следующие операции с контентом:

- Просмотр таких сведений о контенте, как состояние публикации, опубликованный URL-адрес, размер, а также дата и время последнего обновления.
- Загрузка нового контента
- Кодирование контента
- Воспроизведение видеоконтента
- Публикация и отмена публикации контента
- Удаление контента


## Практическое руководство: Передача содержимого


1. На [портале управления](http://go.microsoft.com/fwlink/?LinkID=256666&clcid=0x409) выберите **Службы мультимедиа**, а затем щелкните имя учетной записи служб мультимедиа.
2. Выберите страницу "КОНТЕНТ". 
3. На странице или в нижней части портала нажмите кнопку **Загрузка**. 
4. В диалоговом окне **Загрузить контент** перейдите к нужному файлу ресурса. Щелкните файл и выберите **Открыть** или нажмите клавишу **ВВОД**.

	![UploadContentDialog][uploadcontent]

5. В диалоговом окне "Передача контента" нажмите кнопку с галочкой, чтобы принять значения полей "Файл" и "Имя контента".
6. Начнется передача, ход ее выполнения можно отслеживать в нижней части портала.  

	![JobStatus][status]

После завершения загрузки в списке "Контент" появится новый актив. По соглашению в конец имени будет добавлено **-Source** для отслеживания нового содержимого как источника содержимого для задач кодирования.

Если значение размера файла не обновляется после остановки процесса передачи, нажмите кнопку **Синхронизировать метаданные**. Это синхронизирует размер файла актива с фактическим размером файла в хранилище и обновляет значение на странице "Контент".	


## Практическое руководство: Кодирование контента

1. На [портале управления](http://go.microsoft.com/fwlink/?LinkID=256666&clcid=0x409) выберите **Службы мультимедиа**, а затем щелкните имя учетной записи служб мультимедиа. 
2. Щелкните страницу КОНТЕНТ в верхней части страницы.
3. Щелкните нужный источник видео для задания кодирования, а затем в нижней части страницы нажмите кнопку **Кодировать**.
4. В диалоговом окне "Кодировщик мультимедиа Azure" выберите одну из общих или дополнительных предустановок кодирования.

	**Общие предустановки**

	+ **Воспроизведение на ПК и компьютерах Mac (с помощью Flash или Silverlight)**. Эта предустановка позволяет получить актив с бесперебойной потоковой передачей, который имеет следующие характеристики: стереозвук CBR 44,1 кГц в разрядности 16 бит/образец с кодировкой 96 Кбит/с в AAC, а также видео CBR с разрешением 720 пикселей и 6 скоростями - от 3400 Кбит/с до 400 Кбит/с - с использованием основного профиля H.264 и двумя вторичными группами GOP.
	+ **Воспроизведение в HTML5 (IE, Chrome, Safari)**. Эта предустановка позволяет получить отдельный MP4-файл, который имеет следующие характеристики: стереозвук CBR 44,1 кГц в разрядности 16 бит/сэмпл с кодировкой 128 Кбит/с в AAC, а также видео CBR с разрешением 720 пикселей и скоростью 4500 Кбит/с с использованием основного профиля H.264.
	+ **Воспроизведение на устройствах с ОС iOS, ПК и компьютерах Mac**. Эта предустановка позволяет получить актив с теми же характеристиками, что и у актива с бесперебойной потоковой передачей (описан выше), но в формате, который может использоваться для доставки потоков Apple HLS на устройства iOS. 
	+ **Кодирование с защитой содержимого PlayReady**. Эта предустановка позволяет получить кодированный актив с защитой контента PlayReady.  Для использования этой предустановки необходимо включить службу доставки лицензий PlayReady. Для этого перейдите на вкладку **ЗАЩИТА СОДЕРЖИМОГО** и добавьте строку в таблицу "Отчеты о фирменной символике". Служба лицензий PlayReady служб мультимедиа будет включена в течение нескольких минут после нажатия кнопки СОХРАНИТЬ.   
	
	
		Служба лицензий PlayReady служб мультимедиа включена по умолчанию. Чтобы указать другую службу, из которой клиенты могут получить лицензию на воспроизведение зашифрованного контента PlayReady, используйте API-интерфейсы REST или пакета SDK .NET для служб мультимедиа. Дополнительную информацию см. в статье [Использование статического шифрования для защиты содержимого] и установите свойство **licenseAcquisitionUrl** в предустановке инструмента шифрования мультимедиа. Также можно использовать динамическое шифрование и установить свойство **PlayReadyLicenseAcquisitionUrl**, как описано в разделе [Использование динамического шифрования и службы доставки лицензий PlayReady](http://go.microsoft.com/fwlink/?LinkId=507720 ). 
		
		Обратите внимание, что этот вариант будет доступен только в том случае, если вы подписаны на функцию предварительного просмотра для защиты контента PlayReady. Чтобы подписаться на функции предварительного просмотра, выполните процесс, описанный на следующей странице: [Предварительные версии компонентов Microsoft Azure](http://azure.microsoft.com/ru-ru/services/preview/).  
	
		
	**Дополнительные предустановки**
	
	+ В разделе [Строки предустановок задачи для кодировщика мультимедиа Azure](http://go.microsoft.com/fwlink/?LinkId=270865) объясняется значение каждой строки предустановки в списке "Дополнительные предустановки". 


	![EncoderDialog][encoder]

	В настоящее время портал не поддерживает все форматы кодировки, поддерживаемые кодировщиком мультимедиа. Он также не поддерживает шифрование и дешифрование мультимедийных активов. Вы можете выполнить эти задачи с помощью программных средств. Дополнительную информацию см. в разделах [Создание приложений с помощью пакета SDK служб мультимедиа для .NET](http://go.microsoft.com/fwlink/?LinkId=270866)  и [Строки предустановок задачи для кодировщика мультимедиа Azure](http://go.microsoft.com/fwlink/?LinkId=270865).


5. В диалоговом окне кодировщика мультимедиа Azure введите нужное понятное имя выводимого контента или примите значение по умолчанию. Нажмите кнопку проверки, чтобы начать операцию кодирования, ход выполнения которой можно отслеживать в нижней части портала.

	После завершения кодирования представление должно иметь вид, схожий с приведенным ниже снимком экрана. 

	![PortalViewUploadCompleted][portaloverview]


	Если значение размера файла не обновляется после завершения кодирования, нажмите кнопку **Синхронизировать метаданные**. Это синхронизирует размер файла выводимого актива с фактическим размером файла в хранилище и обновляет значение на странице "Контент".	

## Практическое руководство: Публикация контента

1. На [портале управления](http://go.microsoft.com/fwlink/?LinkID=256666&clcid=0x409) выберите **Службы мультимедиа**, а затем щелкните имя учетной записи служб мультимедиа.
2. Щелкните страницу КОНТЕНТ в верхней части страницы.
3. Щелкните актив, который не был опубликован. Нажмите кнопку "Опубликовать", чтобы выполнить публикацию по открытому URL-адресу. После публикации контента на URL-адрес этот URL-адрес может быть открыт проигрывателем клиента, если тот может отображать контент в данной кодировке.

 ![PublishedContent][publishedcontent]

## Практическое руководство: Воспроизведение контента на портале


1. На [портале управления](http://go.microsoft.com/fwlink/?LinkID=256666&clcid=0x409) выберите **Службы мультимедиа**, а затем щелкните имя учетной записи служб мультимедиа.
2. Щелкните страницу КОНТЕНТ в верхней части страницы.
3. Выберите нужное видеосодержимое и нажмите кнопку **Воспроизвести** в нижней части портала. На портале можно воспроизводить только опубликованный контент. Кроме того, данная кодировка должна поддерживаться вашим веб-браузером.

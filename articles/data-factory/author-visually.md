# <a name="visually-author-data-factories"></a>Визуальная разработка фабрик данных
С пользовательским интерфейсом фабрики данных Azure пользователи могут визуально разрабатывать и развертывать ресурсы в своей фабрике, не написав не единой строки кода. Этот не требующий навыков программирования интерфейс позволяет вам перетаскивать действия на панель холста конвейера, выполнять тестовые запуски, выполнять последовательную отладку, а также развертывать и выполнять мониторинг запуска конвейера. Инструмент пользовательского интерфейса файла определения приложения можно использовать двумя способами:

1. Работать напрямую со службой фабрики данных.
2. Настроить интеграцию Git VSTS для совместной работы, контроля и управления версиями.

## <a name="authoring-with-data-factory"></a>Разработка с помощью фабрики данных
Первый вариант — это разработка непосредственно с режимом фабрики данных. Этот подход отличается от разработки через репозиторий кода VSTS отсутствием репозитория, хранящего сущности JSON изменений, и отсутствием оптимизации для совместной работы или контроля версий.

![Настройка фабрики данных](media/author-visually/configure-data-factory.png)

В режиме фабрики данных имеется только режим публикации. Любые внесенные изменения публикуются непосредственно в службе фабрики данных.

![Публикация фабрики данных](media/author-visually/data-factory-publish.png)

## <a name="authoring-with-vsts-git-integration"></a>Разработка с помощью интеграции Git VSTS
Авторизация с помощью интеграции Git VSTS позволяет управлять исходным кодом и совместно работать во время разработки конвейеров фабрики данных. Пользователи имеют возможность сопоставить фабрику данных с репозиторием учетной записи Git VSTS для управления исходным кодом, совместной работы, управления версиями и т. д. В одной учетной записи GIT VSTS может быть несколько репозиториев. Однако репозиторий GIT VSTS можно связать только с одной фабрикой данных. Если у вас еще нет учетной записи VSTS и репозитория, создайте их [здесь](https://docs.microsoft.com/en-us/vsts/accounts/create-account-msa-or-work-student).

### <a name="configure-vsts-git-repo-with-azure-data-factory"></a>Настройка репозитория GIT VSTS помощью фабрики данных Azure
Пользователи могут настроить репозиторий GIT VSTS с помощью фабрики данных двумя способами.

#### <a name="method-1-lets-get-started-page"></a>Метод 1. Страница "Начало работы"

Перейдите на страницу "Начало работы" и щелкните "Настроить репозиторий кода"

![Настройка репозитория исходного кода](media/author-visually/configure-repo.png)

После этого появится боковая панель для настройки параметров репозитория.

![Настройка параметров репозитория](media/author-visually/repo-settings.png)
* **Тип репозитория.** Git Visual Studio Team Services (в настоящее время GitHub не поддерживается).
* **Учетная запись Visual Studio Team Services.** Имя учетной записи можно найти по адресу https://{имя учетной записи}.visualstudio.com. Войдите в свою учетную запись VSTS [здесь](https://www.visualstudio.com/team-services/git/) и получите доступ к профилю Visual Studio, чтобы просмотреть свои репозитории и проекты.
* **Имя проекта.** Имя проекта можно найти по адресу https://{имя учетной записи}.visualstudio.com/{имя проекта}.
* **.** Имя репозитория. Проекты VSTS содержат репозитории Git, которые управляют исходным кодом по мере роста проекта. Создайте репозиторий либо используйте репозиторий, который уже имеется в проекте.
* **Import existing Data Factory resources to repository** (Импорт существующих ресурсов фабрики данных в репозиторий). Установив этот флажок, вы можете импортировать свои текущие ресурсы фабрики данных, созданные на холсте пользовательского интерфейса, в соответствующий репозиторий GST VSTS в формате JSON. Это действие экспортирует каждый ресурс по отдельности (то есть, связанные службы и наборы данных будут экспортироваться в отдельных файлах JSON).    Если снять этот флажок, имеющиеся ресурсы не будут импортироваться в репозиторий Git.

#### <a name="method-2-from-authoring-canvas"></a>Метод 2. Холст разработки

В разделе "Работа над холстом" выберите раскрывающееся меню "Фабрика данных" под именем фабрики данных. Затем щелкните 'Configure Code Repository (Настроить репозиторий кода). Как и в **методе 1** появится боковая панель для настройки параметров репозитория. Сведения о настройках см. в предыдущих разделах.

![Настройка репозитория кода 2](media/author-visually/configure-repo-2.png)

### <a name="version-control"></a>Управление версиями
Управление версиями, также называемое системой контроля версий, позволяет разработчикам взаимодействовать с кодом и отслеживать изменения, внесенные в базу кода. Система управления версиями является важным инструментом для проектов с несколькими разработчиками.

Каждый репозиторий Git VSTS, который связан с фабрикой данных, имеет главную ветвь. Каждый пользователь, который имеет доступ к репозиторию Git VSTS, может внести изменения двумя способами: синхронизация и публикация.

![Синхронизация и публикация](media/author-visually/sync-publish.png)

#### <a name="sync"></a>Sync

После нажатия кнопки "Синхронизация" можно извлечь изменения из главной ветви в локальную или отправить изменения из локальной ветви в главную.

![Синхронизация изменений](media/author-visually/sync-change.png)

#### <a name="publish"></a>Опубликовать
 Опубликуйте изменения в главной ветви в службе фабрики данных.

> [!NOTE]
> **Главная ветвь может не соответствовать тому, что развернуто в службе фабрики данных.** Главную ветвь *нужно* опубликовать вручную в службе фабрики данных.

![Язык выражений](media/author-visually/expression-language.png)

![Динамическое содержимое](media/author-visually/dynamic-content.png)

# <a name="mapping-data-flow-datasets"></a>Наборы данных Потока данных для сопоставления

[!INCLUDE [notes](../../includes/data-factory-data-flow-preview.md)]

Наборы данных — это конструкции Фабрики данных, определяющие форму данных, с которыми вы работаете в конвейере. В Потоке данных для данных на уровне строки и столбца необходимо указать подробное определение набора данных. Наборы данных, используемые в конвейерах потока управления, не требуют такого глубокого понимания данных.

Наборы данных в поток данных используются в преобразованиях источника и приемника. Они используются для определения схемы базовых данных. Если у вас отсутствует схема данных, вы можете задать смещение схемы для своего источника и приемника. Схема, определенная из набора данных, предоставит вам связанные типы данных, форматы данных, расположение файлов и информацию о подключениях от сопоставленной связанной службы. Метаданные из набора данных появится в преобразование исходного источник «Проекция». Схемы в наборе данных представляет тип физических данных и форму, а проекции в преобразовании «источник» представляет представление потока данных с определенными именами и типами данных.

## <a name="dataset-types"></a>Типы наборов данных

Сейчас в Потоке данных доступны четыре типа наборов данных:

* База данных Azure SQL
* Хранилище SQL
* Parquet (из ADLS и больших двоичных объектов)
* Текст с разделителями (из ADLS и больших двоичных объектов)

Отдельные наборы данных поток данных *типа источника* из *тип подключения связанной службы*. Обычно в Фабрике данных вы выбираете тип подключения (BLOB-объект, ADLS и т. д.) и затем определяете типа файла в наборе данных. В Потоке данных вы выбираете типы источников, которые можно связать с различными типами подключений связанной службы.

![Параметры преобразования источников](media/data-flow/dataset1.png "источники")

## <a name="data-flow-compatible-datasets"></a>Наборы данных, совместимые с Потоком данных

При создании набора данных в верхней правой части панели отображается флажок "Совместим с Потоком данных". Нажав эту кнопку, вы можете отфильтровать только те наборы данных, которые можно использовать с Потоком данных. 

## <a name="import-schemas"></a>Схемы импорта

При импорте схемы для наборов данных Потока данных отобразится кнопка "Импорт схемы". Нажмите эту кнопку, чтобы отобразить два варианта: импорт из источника или импорт из локального файла. В большинстве случаев вы импортируете схему непосредственно из источника. Но если у вас есть существующий файл схемы (файл PARQUET или CSV-файл с заголовками), вы можете выбрать его, и Фабрика данных определит схему на его основе.

## <a name="create-new-table"></a>Создание новой таблицы

В поток данных вы можете попросить ADF, чтобы создать новое определение таблицы в целевой базе данных, задав набор данных в приемник преобразование, которое имеет имя новой таблицы. В наборе данных SQL щелкните «Изменить» под именем таблицы и введите новое имя таблицы. После этого в преобразовании приемника, включите «Разрешить смещение схемы». Сет параметр «Импорт схемы» значение None.

![Исходная схема преобразования](media/data-flow/dataset2.png "SQL схемы")

## <a name="delimited-text-dataset"></a>Набор данных с разделителями

В наборе данных с разделителями, задаст разделитель для обработки либо единый разделители («\t «для TSV,»,» для CSV, "|" …) или использовать несколько символов в качестве разделителя. Установите переключатель строки заголовка, а затем перейдите в исходный преобразование автоматически определить типы данных.

## <a name="next-steps"></a>Дальнейшие действия

Начните с [Создание нового потока данных](data-flow-create.md) и добавьте преобразование «источник». Затем настройте набора данных для источника.

Используйте [действия копирования](copy-activity-overview.md) для импорта данных из любого ADF источника данных и поместите его в ADLS или большого двоичного объекта для доступа потока данных.

# <a name="copy-or-clone-a-data-factory-in-azure-data-factory"></a>Копирование или клонирование фабрики данных в службе "Фабрика данных Azure"

В этой статье описывается, как выполнять копирование или клонирование фабрики данных в службе "Фабрика данных Azure".

## <a name="use-cases-for-cloning-a-data-factory"></a>Варианты использования клонирования фабрики данных

Ниже описаны некоторые обстоятельства, в которых вам может оказаться полезным копирование или клонирование фабрики данных:

-   **Переименование ресурсов**. Переименование ресурсов не поддерживается в Azure. Если вы хотите переименовать фабрику данных, вы можете клонировать ее, указав другое имя, а затем удалить существующую.

-   **Отладка изменений** при недостатке функций отладки. Иногда перед применением изменений в главной фабрике может потребоваться проверить их в другой фабрике. В большинстве случаев можно использовать отладку. Однако изменения в триггерах, например, поведение изменений при автоматическом вызове триггера или через определенный период времени, не легко проверить без записи их в фабрику. В таких случаях можно клонировать фабрику и применить в ней изменения. Так как плата за Фабрику данных Azure взимается главным образом за число запусков, дополнительная плата за вторую фабрику не взымается.

## <a name="how-to-clone-a-data-factory"></a>Клонирование фабрики данных

1. Пользовательский интерфейс Фабрики данных на портале Azure позволяет вам экспортировать все полезные данные фабрики данных в шаблон Resource Manager, а также файл параметров, позволяющий изменить любые значения при клонировании фабрики.

1. Предварительно необходимо создать целевую фабрику данных на портале Azure.

1. При работе в режиме GIT каждый раз, когда вы выполняете публикацию на портале, шаблон Resource Manager фабрики сохраняется в GIT в ветви adf_publish репозитория.

1. В других сценариях шаблон Resource Manager можно скачать, нажав кнопку **Export Resource Manager template** (Экспорт шаблона Resource Manager) на портале.

1. После скачивания шаблона Resource Manager вы можете развернуть его с помощью стандартных методов развертывания такого шаблона.

1. По соображениям безопасности созданный шаблон Resource Manager не содержит каких-либо секретных сведений, например паролей связанных служб. Таким образом, вы должны предоставить эти пароли в качестве параметров развертывания. Если параметры нежелательны, необходимо получить строки подключения и пароли связанных служб из Azure Key Vault.

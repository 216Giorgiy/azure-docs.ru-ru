# Вариант использования: профилирование клиентов

Фабрика данных Azure — одна из многочисленных служб, используемых для внедрения набора аналитических инструментов Cortana Analytics Suite для ускорителей решений. Дополнительные сведения о наборе см. на сайте [Cortana Analytics Suite](http://www.microsoft.com/cortanaanalytics). В этом документе описан простой вариант использования, позволяющий понять, как фабрика данных Azure помогает решать типичные проблемы аналитики.

Для реализации этого сценария потребуется только [подписка Azure](https://azure.microsoft.com/pricing/free-trial/). Вы можете развернуть готовый образец, выполнив действия, описанные в статье [Образцы](data-factory-samples.md).

## Сценарий

Contoso — это компания, которая создает игры для нескольких платформ: игровых консолей, карманных устройств и персональных компьютеров (ПК). По мере того как геймеры играют в них, накапливаются большие объемы данных о закономерностях взаимодействия с игрой, стиле игры и персональных предпочтениях игроков. Анализируя их вместе с демографическими, географическими данными и данными о продукте, Contoso может делать выводы о том, как улучшить игру для каждого игрока и какие обновления и дополнительные платные опции предложить.

Цель компании Contoso — используя профили игроков и их историю игр, выявить возможности дополнительных и перекрестных продаж и разработать новые интересные функции для расширения бизнеса и повышения степени удовлетворенности клиентов. В нашем сценарии мы используем производителя игр в качестве примера компании, которая стремится оптимизировать бизнес с учетом поведения игроков, однако принципы, рассмотренные здесь, можно применить к любой другой компании, которая хочет повысить уровень заинтересованности и степень удовлетворенности клиентов и потребителей своих товаров и услуг.

## Сложности

При попытке реализовать рассматриваемый сценарий производители игр сталкиваются с рядом проблем. Во-первых, необходимо получать данные разных типов и объемов из различных источников данных, как локальных, так и облачных. Среди них данные о продукте, ретроспективные данные о поведении игроков, а также данные об использовании разных платформ. Во-вторых, необходимо точно определить закономерности взаимодействия с игрой. В-третьих, разработчикам игр требуется отслеживать метрики эффективности, например динамику дополнительных и перекрестных продаж игрокам с подходящими профилями, и корректировать планируемые маркетинговые кампании.

## Обзор решения

Этот простой сценарий может служить примером получения, подготовки, преобразования, анализа и публикации данных с помощью фабрики данных Azure.

На рисунке ![Комплексный рабочий процесс](./media/data-factory-customer-profiling-usecase/EndToEndWorkflow.png) показано, как развернутые конвейеры данных отображаются в интерфейсе на классическом портале Azure.

1.	Конвейер **PartitionGameLogsPipeline** считывает необработанные события игры из хранилища больших двоичных объектов и создает разделы на основе года, месяца и дня.
2.	Конвейер **EnrichGameLogsPipeline** объединяет секционированные события игры с данными геокода и дополняет их путем сопоставления IP-адресов с соответствующими географическими расположениями.
3.	Конвейер **AnalyzeMarketingCampaignPipeline** использует обогащенные данные и обрабатывает их вместе с рекламными данными для создания конечного результата, содержащего оценку эффективности маркетинговой кампании.

В этом сценарии мы используем фабрику данных Azure для управления действиями, которые копируют входные данные, преобразуют и обрабатывают их с помощью действий HDInsight (преобразований Hive и Pig), и для записи готовых данных в базу данных SQL Azure. Кроме того, используя тот же интерфейс, вы можете визуализировать сеть конвейеров данных, управлять ими и отслеживать их состояние.

## Преимущества

Благодаря анализу профилей пользователей в соответствии с бизнес-целями организации разработчики игр могут быстро собирать статистические показатели использования и анализировать эффективность маркетинговых кампаний по разным продуктам.

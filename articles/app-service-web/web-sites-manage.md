# Управление веб-приложениями с помощью портала Azure

На [портале Azure](https://portal.azure.com) вы можете управлять веб-приложениями с помощью ряда *колонок*, в которых содержатся различные компоненты. (Общие сведения о портале Azure см. в руководстве по изменениям [портала](http://go.microsoft.com/fwlink/?LinkId=529715).)

Чтобы просмотреть колонку веб-приложения, щелкните **Главная** и выберите имя веб-приложения или нажмите кнопку **Обзор** для просмотра всех ресурсов Azure.

![](./media/web-sites-manage/manage01.png)

В верхней части колонки расположены элементы управления для некоторых типовых действий.

- **Параметры** для просмотра списка всех параметров управления.

- **Обзор** для открытия веб-приложения в окне браузера.

- Другие кнопки для запуска или остановки приложения, переключения слотов развернутого приложения, удаления приложения и т. д.

**Основные компоненты** — перечислены основные сведения о приложении, включая URL-адрес, расположение, план [службы приложений](http://go.microsoft.com/fwlink/?LinkId=529714) и текущее состояние (запущено, остановлено и т. п.).

Ниже кнопки **Основные компоненты** размещены различные части, которые можно использовать для мониторинга и развертывания вашего приложения, а также управления им. Чтобы настроить макет колонки, щелкните правой кнопкой мыши и выберите **Настройка** или **Добавить части**. В остальной части этого раздела описываются части макета по умолчанию.

## Мониторинг

В разделе **Мониторинг** расположена диаграмма, отображающая показатели веб-приложения. Чтобы настроить диаграмму, щелкните **Изменить**. Вы можете выбрать диапазон времени и показатели для отображения. Доступны следующие показатели: число запросов, среднее время ответа, ошибки сервера и время ЦП.

![](./media/web-sites-manage/manage02.png)

Чтобы добавить правило оповещения, щелкните диаграмму, а затем **Добавить оповещение**. Правило оповещения уведомляет о достижении определенным показателем некоторого порогового значения. Например, можно получать оповещения, когда количество ошибок сервера превышает некоторое значение в течение 5 минут.

**Мониторинг** также включает элементы для настройки аналитики, мониторинга приложений и создания проверок доступности. Дополнительные сведения см. в разделе [Основы мониторинга для веб-приложений в службе приложений Azure](web-sites-monitor.md).

## Использование

![](./media/web-sites-manage/manage03.png)

- **Хранилище файловой системы** отображает, какой объем хранилища файлов использует ваше приложение.
- **Квоты** отображают, как приложение использует свою квоту на использование ресурсов. Щелкните эту часть, чтобы просмотреть подробные сведения.
- **Масштаб** позволяет масштабировать число экземпляров и настраивать автоматическое масштабирование. См. раздел [Масштабирование веб-приложений в службе приложений Azure](../how-to-scale-websites.md).  
- **Оценка затрат** показывает оценку затрат для приложения.
- **Ценовая категория** позволяет изменить ценовую категорию.

## Операции

![](./media/web-sites-manage/manage04.png)

- **События**. Щелкните, чтобы просмотреть журналы событий.  
- **Правила оповещений**. Щелкните, чтобы просмотреть правила оповещений и добавить новые.
- **Журналы потоковой передачи** (А). Щелкните, чтобы просмотреть журналы приложений. Чтобы включить журналы, щелкните **Параметры** и откройте колонку **Журналы диагностики**. 
- **Консоль**. Щелкните, чтобы открыть командную строку, выполняющуюся на портале. Ее можно использовать для выполнения таких команд, как `mkdir` и `dir`.  
- **Обозреватель процессов** (B). Щелкните, чтобы просмотреть процессы, запущенные в приложении, включая рабочее множество и число потоков.

## Развертывание
 
![](./media/web-sites-manage/manage05.png)

- Настройка непрерывного развертывания. См. статью [Использование Git для развертывания веб-приложений в службе приложений Azure](web-sites-publish-source-control.md).
- Слоты развертывания. См. статью [Развертывание в промежуточную среду для веб-приложений в службе приложений Azure](web-sites-staged-publishing.md).
- Задайте учетные данные развертывания для Git или FTP.

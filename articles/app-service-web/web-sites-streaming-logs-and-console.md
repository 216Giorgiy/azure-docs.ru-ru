#Журналы потоковой передачи и консоль

### Журналы потоковой передачи ###

[Портал предварительной версии Azure](http://go.microsoft.com/fwlink/?LinkId=529715) обеспечивает интегрированный просмотр журналов потоковой передачи. Это позволяет просматривать события трассировки из веб-приложений [службы приложений Azure](http://go.microsoft.com/fwlink/?LinkId=529714) в режиме реального времени.

Для настройки этой возможности необходимо выполнить несколько простых действий:

- Записать трассировки в код.
- Включить диагностику приложений на портале предварительной версии Azure.
- Щелкнуть плитку журналов потоковой передачи в колонке веб-приложения.

### Порядок записи трассировок в код ###

Запись трассировок в код не представляет никаких сложностей. В C# это так же просто, как написание следующего кода:

`````````````````````````
Trace.TraceInformation("My trace statement");
`````````````````````````

`````````````````````````
Trace.TraceWarning("My warning statement");
`````````````````````````

`````````````````````````
Trace.TraceError("My error statement");
`````````````````````````

Класс Trace находится в пространстве имен System.Diagnostics.

В приложении node.js для достижения того же результата можно написать следующий код:

`````````````````````````
console.log("My trace statement").
`````````````````````````

### Включение и просмотр журналов потоковой передачи ###

Диагностика включается отдельно для каждого веб-приложения.

1. На [портале предварительной версии Azure](https://portal.azure.com) щелкните **Обзор** > **Веб-приложения**, чтобы отобразить список всех веб-приложений.  

2. Щелкните имя веб-приложения, которое нужно настроить.

3. Щелкните **Все параметры** > **Журналы диагностики** и задайте для параметра **Ведение журнала приложения (файловая система)** значение **ВКЛ.**.

4. Отображается параметр **Уровень**, позволяющий изменять уровень серьезности регистрируемых трассировок. Здесь следует задать значение **Подробно**, если вы хотите ознакомиться с данным компонентом, поскольку это обеспечит регистрацию всех инструкций трассировки.

5. Нажмите **Сохранить** в верхней части колонки. Теперь все готово для просмотра журналов.

6. Чтобы просмотреть журналы потоковой передачи на портале, щелкните плитку **Журналы потоковой передачи** в колонке веб-приложения. Если приложение активно записывает трассировочные операторы, то они будут отображаться в появившемся окне практически в режиме реального времени.

## Доступ к консоли ##

На портале предварительной версии Azure можно получить доступ к веб-приложению из консоли. Вы можете просматривать файловую систему веб-приложения и выполнять скрипты powershell или командной строки. Вам доступен тот же набор разрешений, что и для запуска кода веб-приложения при выполнении команд консоли. Вы не можете получить доступ к защищенным каталогам или запускать скрипты, требующие повышенного уровня разрешений.

1. Перейдите в колонку веб-приложения, следуя указаниям в предыдущем разделе.

2. В колонке веб-приложения щелкните **Консоль**, чтобы открыть ее.

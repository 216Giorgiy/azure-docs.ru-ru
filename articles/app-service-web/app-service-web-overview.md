# Обзор веб-приложений

*Веб-приложения службы приложений* — это полностью управляемая вычислительная платформа, оптимизированная для размещения веб-сайтов и веб-приложений. Это предложение Microsoft Azure [платформа как услуга](https://en.wikipedia.org/wiki/Platform_as_a_service) (PaaS) позволяет сосредоточиться на бизнес-логике, так как инфраструктуру для запуска и масштабирования приложений предоставляет среда Azure.

Просмотрите 5-минутный видеоролик о [веб-приложениях службы приложений Azure с Йочай Кириати (Yochay Kiriaty)](https://azure.microsoft.com/documentation/videos/azure-app-service-web-apps-with-yochay-kiriaty/).

## Что такое веб-приложение в службе приложений?

В службе приложений *веб-приложение* — это вычислительные ресурсы, предоставляемые средой Azure для размещения веб-сайта или веб-приложения.

В зависимости от выбранной ценовой категории вычислительные ресурсы могут находиться на общих или выделенных виртуальных машинах. Код приложения выполняется на управляемой виртуальной машине, которая изолирована от других клиентов.

Этот код может быть создан на любом языке или платформе, поддерживаемой [службой приложений Azure](../app-service/app-service-value-prop-what-is.md), включая ASP.NET, Node.js, Java, PHP или Python. В веб-приложениях также можно запускать скрипты, которые используют [PowerShell и другие языки сценариев](web-sites-create-web-jobs.md#acceptablefiles).

Примеры распространенных сценариев приложений, для реализации которых вы можете использовать веб-приложения, см. в статье [Сценарии веб-приложений](https://azure.microsoft.com/documentation/scenarios/web-app/).

## Зачем использовать веб-приложения?

Вот несколько ключевых функций веб-приложений.

- **Привычность и скорость.** Используйте имеющиеся навыки для работы со знакомыми языками, платформами и интегрированными средами разработки. Подготавливайте новые веб-приложения и развертывайте в них код за считанные минуты.

- **Корпоративный уровень.** Веб-приложения предназначены для создания и размещения безопасных и критически важных приложений. Создавайте интегрированные с Active Directory бизнес-приложения, которые безопасно подключаются к локальным ресурсам, и размещайте их на безопасной облачной платформе, которая совместима со стандартами [ISO](https://www.microsoft.com/TrustCenter/Compliance/ISO-IEC-27001), [SOC](https://www.microsoft.com/TrustCenter/Compliance/SOC) и [PCI](https://www.microsoft.com/TrustCenter/Compliance/pci). Все это возможно в рамках [соглашения об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/app-service/) для предприятий.

- **Глобальное масштабирование.** Веб-приложения позволяют быстро [увеличивать](../app-service/app-service-scale.md) и [уменьшать](../azure-portal/insights-how-to-scale.md) масштаб для обработки любой входящей пользовательской нагрузки. Вручную выбирайте количество и размер виртуальных машин или настраивайте автомасштабирование на основе нагрузки или по расписанию. Глобальная инфраструктура центров обработки данных Майкрософт позволяет размещать веб-приложения, упрощая репликацию данных и размещение служб в нескольких расположениях.

- **Azure Marketplace.** Оцените постоянно пополняемый [список шаблонов веб-приложений](https://azure.microsoft.com/marketplace/). Используйте лучшие решения сообщества по приложениям OSS, одним щелчком мышью устанавливая такие пакеты, как Wordpress, Joomla и Drupal.

- **Непрерывная интеграция.** Настройте [непрерывную интеграцию и развертывание](app-service-continous-deployment.md) рабочих процессов в Visual Studio Team Services, GitHub или BitBucket. Автоматически создавайте, тестируйте и развертывайте веб-приложения при каждой успешной проверке интеграции или применения кода.

- **Промежуточные и тестовые среды.** Реализуйте [промежуточное развертывание](web-sites-staged-publishing.md) для проверки кода в подготовительной среде, которая идентична рабочей среде. А затем выпускайте новые версии приложения с нулевым временем простоя, выполняя операции замены.

- **Тестирование в рабочей среде.** Используйте промежуточное развертывание более эффективно, [выполняя A/B-тестирование](app-service-web-test-in-production-get-start.md) нового кода с помощью настраиваемого объема реального трафика.

- **Веб-задания.** [Запускайте любые программы или скрипты](web-sites-create-web-jobs.md) на виртуальных машинах веб-приложений. Выполнять задания можно по расписанию, непрерывно или с активацией событиями. [Пакет SDK для веб-заданий](websites-dotnet-webjobs-sdk-get-started.md) Azure упрощает код, предназначенный для интеграции с другими службами Azure, включая очереди хранилища, большие двоичные объекты и очереди, а также таблицы и разделы служебной шины.

- **Гибридные подключения.** Получайте доступ к локальным данным с помощью [гибридных подключений](../biztalk-services/integration-hybrid-connection-overview.md) и [виртуальной сети](../app-service-web/web-sites-integrate-with-vnet.md).

- **Интеграция с Visual Studio.** Специальные средства Visual Studio упрощают работу по созданию, развертыванию и отладке приложений API, а также управлению ими. Дополнительную информацию см. в разделе [Представляем пакет Azure SDK 2.8.1 для .NET](https://azure.microsoft.com/blog/announcing-azure-sdk-2-8-1-for-net/).

Кроме того, в веб-приложениях доступны функции, реализованные в [приложениях API](../app-service-api/app-service-api-apps-why-best-platform.md) (включая поддержку CORS) и [мобильных приложениях](../app-service-mobile/app-service-mobile-value-prop.md) (включая push-уведомления). И наоборот: можно использовать приложение API или мобильное приложение для размещения веб-приложения, используя такие возможности веб-приложений, как автоматическое масштабирование и промежуточное развертывание. Единственное различие между тремя типами приложений (приложениями API, веб-приложениями и мобильными приложениями) — это имя и значок на портале Azure. Дополнительные сведения о планах службы приложений см. в статье [Что такое служба приложений Azure?](../app-service/app-service-value-prop-what-is.md)

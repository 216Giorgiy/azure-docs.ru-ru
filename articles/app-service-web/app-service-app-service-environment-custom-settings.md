# Настраиваемые параметры конфигурации для сред службы приложений

## Обзор ##
Так как среды службы приложений изолированы для одного клиента, определенные параметры конфигурации можно применить только к одной такой среде. В этой статье описаны доступные настраиваемые параметры конфигурации для среды службы приложений.

Изменения в настройки среды службы приложений можно внести с помощью массива в новом атрибуте clusterSettings, который размещен в словаре Properties объекта ARM *hostingEnvironments*.

Сокращенный фрагмент шаблона ARM ниже содержит атрибут clusterSettings.


    "resources": [
    {
       "apiVersion": "2015-08-01",
       "type": "Microsoft.Web/hostingEnvironments",
       "name": ...,
       "location": ...,
       "properties": {
          "clusterSettings": [
             {
                 "name": "nameOfCustomSetting",
                 "value": "valueOfCustomSetting"
             }
          ],
          "workerPools": [ ...],
          etc...
       }
    }

Включив атрибут clusterSettings в шаблон ARM, можно обновить среду службы приложений.

Значение этого атрибута можно обновить через [обозреватель ресурсов Azure](https://resources.azure.com). В обозревателе ресурсов Azure перейдите к узлу для среды службы приложений \("Подписки" --\> "Группы ресурсов" --\> "Поставщики" --\> Micrososft.Web --\> hostingEnvironments\) и выберите среду, которую нужно обновить.

В окне браузера справа в верхней части панели инструментов щелкните "Чтение и запись", чтобы разрешить интерактивное редактирование в обозревателе ресурсов. Затем нажмите синюю кнопку "Изменить", чтобы активировать редактирование шаблона ARM. Перейдите к нижней части окна браузера. Атрибут clusterSettings будет отображаться в самом низу. Теперь можно ввести или обновить его значение.

Введите \(или скопируйте и вставьте\) необходимый массив значений конфигурации в атрибут clusterSettings. Затем в верхней правой части окна браузера нажмите зеленую кнопку PUT, чтобы применить изменения для среды службы приложений.

Независимо от того, каким способом обновлена среда службы приложений, понадобится приблизительно 30 минут, чтобы изменения вступили в силу. Этот период необходим для каждого внешнего интерфейса в среде службы приложений. Например, если среда службы приложений имеет четыре внешних интерфейса, на применение обновлений настроек понадобится около двух часов. Пока изменение конфигурации не вступит в силу, в среде службы приложений невозможно выполнять другие операции масштабирования или изменения конфигурации.

## Отключение протокола TLS 1.0 ##
Некоторые клиенты, в частности те, которые сталкиваются с проверками на соответствие требованиям стандарта PCI, часто задают вопрос, можно ли отключить для своих приложений протокол TLS 1.0.

Протокол TLS 1.0 можно отключить с помощью следующей записи *clusterSettings*:

        "clusterSettings": [
            {
                "name": "DisableTls1.0",
                "value": "1"
            }
        ],



## Приступая к работе
На сайте шаблонов быстрого запуска Azure на базе ARM можно найти шаблон с базовым определением, позволяющим [создать среду службы приложений](https://azure.microsoft.com/documentation/templates/201-web-app-ase-create/).

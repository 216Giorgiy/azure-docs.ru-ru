# Создание веб-приложения Java в службе приложений Azure

В этом учебнике показано, как создать веб-приложение в Microsoft Azure с помощью Java, используя Azure Marketplace или пользовательский интерфейс настройки [веб-приложений в службе приложений Azure][].

Если вы не хотите использовать ни один из этих методов, например если требуется настроить контейнер приложения, см. статью [Отправка настраиваемого веб-приложения Java в Azure](web-sites-java-custom-upload.md).

> [AZURE.NOTE]Для работы с этим учебником необходимо использовать учетную запись Microsoft Azure. Если у вас нет учетной записи, можно [активировать преимущества для подписчиков MSDN][] или [подписаться на бесплатную пробную версию][].

Если вы хотите приступить к работе со службой приложений Azure до регистрации для получения учетной записи Azure, перейдите на страницу [Пробное использование службы приложений][]. Там вы сможете немедленно создать кратковременное начальное веб-приложение в службе приложений. Для этого не потребуется ни кредитная карта, ни какие-либо обязательства.

## Создание веб-приложения Java с использованием Azure Marketplace

Здесь описано, как использовать Azure Marketplace для выбора контейнера приложения Java — Apache Tomcat или Jetty — для вашего веб-приложения.

Ниже показано, как будет выглядеть веб-приложение, созданное с помощью Tomcat из Azure Marketplace.

<!--todo:![Web app using Apache Tomcat](./media/web-sites-java-get-started/tomcat.png)-->

Ниже показано, как будет выглядеть веб-приложение, созданное с помощью Jetty из Azure Marketplace.

<!--todo:![Web app using Jetty](./media/web-sites-java-get-started/jetty.png)-->

1. Войдите на [портал Azure](http://go.microsoft.com/fwlink/?LinkId=529715).
2. Щелкните **Создать** в нижнем левом углу страницы.
3. Выберите колонку **Веб + мобильный**.
4. Щелкните **Azure Marketplace** в нижней части колонки **Веб + мобильный**.
5. Выберите **Веб**.
6. В верхней части страницы **Веб** расположено текстовое поле поиска. В этом текстовом поле укажите нужный сервер приложений Java, например **Apache Tomcat** или **Jetty**.
4. Щелкните нужный сервер приложений Java.
5. Щелкните **Создать**.
6. Задайте URL-имя.
6. Выберите регион. Например, выберите **Запад США**.
7. Щелкните **Создать**.

Через несколько секунд веб-приложение будет создано. Чтобы просмотреть веб-приложение, в колонке **Веб-приложения** на портале Azure выберите нужное веб-приложение, а затем щелкните его URL-адрес.

Теперь, когда вы создали веб-приложение с использованием контейнера приложения, см. раздел **Дальнейшие действия** для получения сведений об отправке вашего приложения в веб-приложение.

## Создание веб-приложения Java с использованием интерфейса настройки Azure

Здесь описано, как использовать пользовательский интерфейс настройки Azure для выбора контейнера приложения Java — Apache Tomcat или Jetty — для веб-приложения.

1. Войдите на портал Azure.
2. Щелкните **Создать** в нижнем левом углу страницы.
3. Выберите колонку **Веб + мобильный**.
4. Щелкните **Azure Marketplace** в нижней части колонки **Веб + мобильный**.
5. Выберите **Веб**.
6. Щелкните **Веб-приложение**.
7. Щелкните **Создать**.
8. Задайте URL-имя.
9. Выберите регион. Например, выберите **Запад США**.
10. Щелкните **Создать**.
11. После создания веб-приложения щелкните **Все параметры**.
12. Щелкните **Параметры приложения**.
13. Выберите нужную версию Java.
14. Отображаются параметры для веб-контейнера, например Tomcat и Jetty. Выберите нужный **веб-контейнер**.
15. Щелкните **Сохранить**.

Веб-приложение на базе Java будет доступно через несколько секунд. Чтобы подтвердить, что приложение написано на языке Java, щелкните его URL-адрес. Обратите внимание, что на странице будет присутствовать текст о том, что новое веб-приложение создано на основе Java.

Теперь, когда вы создали веб-приложение с использованием контейнера приложения, см. раздел **Дальнейшие действия** для получения сведений об отправке вашего приложения в веб-приложение.

# Создание кампании цифрового маркетинга на основании веб-приложений службы приложений Azure
Веб-приложения [службы приложений Azure](http://go.microsoft.com/fwlink/?LinkId=529714) отлично подходят для создания кампаний цифрового маркетинга. Цифровые маркетинговые кампании обычно являются краткосрочными и предназначены для достижения кратковременных маркетинговых целей. Необходимо рассмотреть два основных сценария. В первом сценарии сторонняя маркетинговая фирма создает кампанию для своего клиента и управляет этой кампанией в течение рекламной акции. Второй сценарий задействует маркетинговую фирму, создающую ресурсы цифровой маркетинговой кампании и затем передающую своему клиенту право владения этими ресурсами. Далее клиент запускает цифровую маркетинговую кампанию и управляет этой кампанией на свое усмотрение. Веб-сайты Azure отлично подходят для обоих сценариев. 

>[AZURE.NOTE] Если вы хотите начать работу со службой приложений Azure еще до создания учетной записи Azure, перейдите на страницу [ознакомительной версии службы приложений](http://go.microsoft.com/fwlink/?LinkId=523751). Там вы сможете сразу создать свое первое веб-приложение, правда срок его службы будет ограничен. Никаких кредитных карт и обязательств.

Ниже приведен пример глобальной, многоканальной цифровой маркетинговой кампании с использованием веб-приложений службы приложений Azure. В нем демонстрируются возможности, которые обеспечивает объединение веб-приложений службы приложений с другими службами при минимальных инвестициях в технологии. **Щелкните элемент в топографии, чтобы узнать о нем больше.**

## Создание с нуля или перенос существующих ресурсов

Быстро создавайте новые веб-приложения на основе популярных систем управления содержимым (CMS) из коллекции или переносите существующие веб-ресурсы в веб-приложения службы приложений благодаря поддержке различных языков и платформ.

На сайте Azure Marketplace есть шаблоны, позволяющие использовать существующие системы управления содержимым, такие как [Orchard], [Umbraco], [Drupal] и [WordPress]. Так что вы сможете создать веб-приложение с помощью понравившейся версии CMS. Широкий выбор внутренних серверов баз данных, в том числе для [базы данных SQL Azure] и [MySQL], позволяет подобрать сервер в соответствии со своими потребностями.

Существующие веб-ресурсы можно запускать в веб-приложениях, независимо от языка и платформы (.NET, PHP, Java, Node.js или Python). Их можно перемещать в веб-приложения с помощью известных инструментов [FTP]. Если вы часто создаете цифровые маркетинговые кампании, возможно, что у вас уже есть веб-ресурсы в системе управления версиями. Веб-приложения можно развернуть непосредственно из популярных систем управления версиями, таких как [Visual Studio], [Visual Studio Online] и [Git](локальная система, GitHub, BitBucket, DropBox, Mercurial и т. д.).

## Гибкость

Будьте гибкими - постоянно публикуйте непосредственно из существующей системы управления версиями и запускайте тестирование A/B в веб-приложениях службы приложений. 

Во время планирования, создания прототипов и на ранних этапах разработки веб-приложения вы и ваш клиент можете просматривать реальные рабочие версии приложения кампании перед его запуском путем [развертывания в промежуточном слоте] веб-приложения. При интеграции системы управления версиями с веб-приложениями службы приложений вы можете [постоянно выполнять публикации] в промежуточном слоте и переключиться без простоев в рабочую среду, когда она будет готова. 

Кроме того, при планировании изменений действующего веб-приложения можно легко [запускать тестирование A/B] предложенных обновлений с помощью функции тестирования в рабочей среде и анализировать поведение реальных пользователей, чтобы принимать обоснованные решения по разработке приложения.


## Использование социальных сетей

Цифровую маркетинговую кампанию в веб-приложениях службы приложений можно интегрировать с социальными сетями путем аутентификации у таких известных поставщиков, как Facebook и Twitter. Пример использования такого подхода с приложением ASP.NET см. в статье [Создание приложения ASP.NET MVC с аутентификацией и базой данных SQL и развертывание службы приложений Azure]. 

Кроме того, каждый сайт социальной сети обычно предоставляет информацию о других способах интеграции с .NET и прочими платформами.

## Использование мультимедиа и поддержка всех устройств

Улучшите цифровую маркетинговую кампанию с помощью других служб Azure, таких как:

-  скачивание и потоковая передача видео по глобальным каналам с помощью [служб мультимедиа Azure];
-  отправка почтовых сообщений пользователям с помощью [службы SendGrid в Azure Marketplace];
-  установка состояния присутствия на устройствах с ОС Windows, iOS и Android с помощью [мобильных служб];
-  отправка push-уведомлений на большое количество устройств с помощью [центра уведомлений].

## Глобализация

Перейдите на глобальный уровень, обслуживая региональные сайты с помощью диспетчера Azure Traffic Manager и мгновенной доставки содержимого с помощью Azure CDN.

Чтобы обслуживать глобальных клиентов в соответствующих областях, используйте [диспетчер Azure Traffic Manager], чтобы направлять посетителей на региональный сайт для оптимальной производительности. Кроме того, можно равномерно распределить нагрузку размещенного в нескольких регионах сайта среди нескольких копий веб-приложений.

Мгновенно доставляйте статическое содержимое пользователям по всему миру благодаря [интеграции веб-приложения с Azure CDN]. Azure CDN кэширует статическое содержимое в ближайшем к пользователю [узле CDN], что сводит к минимуму задержку и количество подключений к веб-приложению.

## Оптимизация

Функция автоматического масштабирования, кэширование с помощью кэша Redis для Azure, запуск фоновых задач с помощью веб-заданий и высокая доступность благодаря диспетчеру трафика Azure помогут оптимизировать веб-приложение.

Возможность [масштабирования и развертывания] веб-приложений службы приложений идеально подходит для непрогнозируемых рабочих нагрузок, в частности цифровых маркетинговых кампаний. Разверните веб-приложение вручную с помощью [портала Azure](http://go.microsoft.com/fwlink/?LinkId=529715), программным путем с помощью [API управления службами] или [сценариев PowerShell] либо автоматически с помощью функции автомасштабирования. В рамках уровня **Standard** автомасштабирование позволяет развернуть веб-приложение автоматически, исходя из использования ЦП. Эта функция позволяет максимально увеличить гибкость и свести к минимуму затраты за счет развертывания веб-приложения по мере необходимости (в зависимости от активности пользователей). Рекомендации см. в статье [Троя Ханта](Troy Hunt) [10 принципов быстрого масштабирования веб-приложений с помощью Azure].

Увеличьте скорость отклика веб-приложения с помощью [кэша Redis для Azure]. Используйте его, чтобы кэшировать данные из серверных баз данных и другие элементы, например [состояние сеанса ASP.NET] и [кэш вывода].

Поддерживайте высокую доступность веб-приложения с помощью [диспетчера трафика Azure]. Если на основном сайте возникает проблема, при использовании метода **Failover** диспетчер Traffic Manager автоматически направляет трафик на дополнительный сайт.

## Мониторинг и анализ

Средства Azure и сторонних разработчиков позволяют следить за производительностью веб-приложения. Вы сможете получать оповещения о критически важных событиях в веб-приложении. Благодаря Application Insights и средствам аналитики веб-журналов HDInsight у вас будет полное представление о пользователях. 

Ознакомьтесь [вкратце] с текущими метриками производительности веб-приложения и квотами ресурсов в колонке веб-приложения на [портале Azure](http://go.microsoft.com/fwlink/?LinkId=529715). Чтобы получить полное представление о доступности, производительности и использовании приложений, можно воспользоваться [Azure Application Insights]. Это средство позволяет быстро и эффективно устранять неполадки, проводить диагностику и получать данные об использовании. Вы также можете обеспечить расширенный мониторинг данных веб-приложений, применяя средства сторонних разработчиков, например [New Relic].

В рамках уровня **Standard** существует возможность мониторинга отклика: вы получаете почтовые уведомления, если ваше веб-приложение перестает отвечать. Дополнительную информацию см. в разделах [Практическое руководство. Получение уведомлений об оповещениях и управление правилами оповещения в Azure].

# <a name="azure-app-service-web-apps-on-linux-faq"></a>Вопросы и ответы о веб-приложениях службы приложений Azure на платформе Linux #

С выпуском службы приложений Azure на платформе Linux (сейчас в предварительной версии) мы работаем над добавлением функций и усовершенствованиями нашей платформы. Ниже приведены часто задаваемые вопросы, которые заказчики задавали нам за последние месяцы.
Если у вас есть вопрос, оставьте комментарий к статье и мы ответим в кратчайшие сроки.

## <a name="built-in-images"></a>Встроенные образы ##

**В.** Мне необходимо выполнить разветвление встроенных контейнеров Docker, предоставляемых этой платформой. Где находятся эти файлы?

**О.** Все файлы Docker можно найти здесь: https://github.com/azure-app-service. Все контейнеры Docker можно найти здесь: https://hub.docker.com/u/appsvc/.

## <a name="management"></a>управления ##

**В.** При нажатии кнопки "Перезапуск" на портале не удается перезапустить веб-приложение. Что не так?

**О.** Мы прилагаем все усилия, чтобы включить кнопку "Сброс" в ближайшем будущем. Сейчас у вас есть два варианта.
1. Добавьте или измените пустой параметр приложения, в результате этого приложение перезапустится. 
2. Остановите, а затем заново запустите свое веб-приложение.

**В.** Могу ли я подключиться к виртуальной машине по протоколу SSH?

**О.** Нет. Мы предоставим способ подключения к контейнеру приложения по протоколу SSH в будущем выпуске.

## <a name="continuous-integration--deployment"></a>Непрерывные интеграция и развертывание ##

**В.** Мое веб-приложение по-прежнему использует старый образ контейнера Docker после обновления образа в DockerHub? Поддерживается ли непрерывная интеграция и развертывание пользовательских контейнеров?

**О.** Чтобы обновить контейнер, остановите и снова запустите свое веб-приложение либо обновите или измените (либо добавьте) пустой параметр приложения, чтобы принудительно обновить контейнер приложения. Функция непрерывных интеграции и развертывания для пользовательских контейнеров появится в будущем выпуске.

## <a name="language-support"></a>Поддержка языков ##

**В.** Поддерживаются ли некомпилированные приложения .NET Core?

**О.** Нет. Вам необходимо развернуть скомпилированное приложение .NET Core со всеми зависимостями. Возможности полного развертывания и сборки будут реализованы в будущем выпуске.

## <a name="built-in-images"></a>Встроенные образы ##

**В.** Какие значения будут доступны в разделе "Загрузочный файл" при настройке стека времени выполнения?

**О.** Для Node.Js можно будет указать файл конфигурации PM2 или свой файла сценария. Для .Net Core нужно будет указать имя скомпилированной библиотеки DLL. Для Ruby можно будет указать сценарий Ruby, с помощью вы хотите инициализировать приложение.

## <a name="custom-containers"></a>Пользовательские контейнеры ##

**В.** Я использую собственный пользовательский контейнер. Мое приложение находится в каталоге \home\, но мне не удается найти файлы при просмотре содержимого с сайта SCM или FTP-клиента. Где находятся мои файлы?

**О.** Мы подключили общую папку SMB к каталогу \home\. Поэтому все содержимое переопределено.

**В.** Я хочу предоставить несколько портов в образе настраиваемого контейнера. Это возможно?

**О.** Сейчас такая возможность не поддерживается.

**В.** Я могу использовать собственное хранилище?

**О.** Сейчас такая возможность не поддерживается.

**В.** Мне не удается просматривать файловую систему или выполняющиеся процессы своего пользовательского контейнера на сайте SCM. Почему?

**О.** Сайт SCM выполняется в отдельном контейнере, поэтому вы не можете проверить файловую систему или выполняющиеся процессы контейнера приложения.

**В.** Мой настраиваемый контейнер ожидает передачи данных через порт, отличный от порта 80. Как настроить приложение для перенаправления запросов в этот порт?

**О.** Можно указать параметр приложения **PORT** и присвоить ему номер нужного порта.

## <a name="pricing-and-sla"></a>Цены и соглашение об уровне обслуживания ##

**В.** Какая модель ценообразования используется в общедоступной предварительной версии?

**О.** При использовании обычной модели ценообразования службы приложений Azure с вас будет взиматься плата за половину количества часов работы вашего приложения. Фактически это означает, что вы получаете 50-процентную скидку на цены службы приложений Azure.

## <a name="other"></a>Другие ##

**В.** Какие знаки поддерживаются в именах параметров приложения?

**О.** Для параметров приложения вы можете использовать только знаки A–Z, a–z, 0–9 и знак подчеркивания.

**В.** Где можно отправить запрос на новые функции?

**О.** Вы можете отправить свои идеи по этому адресу: https://aka.ms/webapps-uservoice. Добавьте [Linux] в заголовок своей идеи.

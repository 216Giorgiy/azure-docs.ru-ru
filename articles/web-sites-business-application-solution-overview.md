# Создание бизнес-приложения на веб-сайтах Azure

[Веб-сайты Azure] отлично подходят для создания бизнес-приложений. Эти приложения являются приложениями интрасети, которые должны быть защищены для внутреннего использования. Как правило, они требуют аутентификации, обычно с использованием каталога организации. Кроме того, для их использования обычно требуется определенный уровень доступа и интеграции с локальными службами и данными. 

Существуют ряд преимуществ переноса бизнес-приложений на веб-сайты Azure:

-  Возможность масштабирования в сторону увеличения и уменьшения в соответствии с динамической нагрузкой, например для приложения, обрабатывающего данные ежегодных аттестаций. В период аттестаций трафик для большой компании резко возрастает. Azure предоставляет варианты масштабирования, позволяющие компании выполнить горизонтальное масштабирование, чтобы справиться с нагрузкой в период большого трафика, связанного с аттестациями, и экономить деньги в течение всего остального года благодаря обратному масштабированию. 
-  Возможность уделять больше внимания разработке приложений и меньше - приобретению инфраструктуры и управлению ею.
-  Предоставление сотрудникам и партнерам больше возможностей для использования приложений из любой точки мира. В этом случае пользователям не потребуется подключаться к корпоративной сети для использования приложения, а ИТ-подразделение будет избавлено от необходимости реализовывать сложные решения с обратным прокси. Существует несколько вариантов проверки подлинности, обеспечивающих защиту доступа к корпоративным приложениям.

Ниже приведен пример бизнес-приложения, работающего на веб-сайтах Azure. В нем демонстрируются возможности, которые обеспечивает интеграция веб-сайтов Azure с другими службами при минимальных инвестициях в технические средства. **Щелкните элемент в топографии, чтобы узнать о нем больше.**

### Перенос имеющихся ресурсов

Переносите имеющиеся веб-ресурсы, написанные на разных языках и расположенные на разных платформах, на веб-сайты Azure.

Их можно запускать на веб-сайтах Azure, независимо от языка и платформы (.NET, PHP, Java, Node.js или Python). Их можно перемещать на веб-сайты Azure с помощью привычных инструментов [FTP] или системы управления версиями. Веб-сайты Azure поддерживают прямую публикацию из популярных систем управления версиями, таких как [Visual Studio], [Visual Studio Online] и [Git] (локальная система, GitHub, BitBucket, DropBox, Mercurial и т. д.).

### Защита ресурсов

Защищайте ресурсы с помощью шифрования, выполняйте аутентификацию корпоративных пользователей локально и удаленно и авторизуйте использование ими различных ресурсов. 

Защищайте внутренние ресурсы от нарушителей с помощью [HTTPS]. Домен с именем **\*.аzurewebsites.net** предоставляется сразу с SSL-сертификатом, а при использовании личного домена его SSL-сертификат можно перенести на веб-сайты Azure. За использование каждого SSL-сертификата ежемесячно взимается почасовая плата. Дополнительную информацию см. на странице [информации о ценах для веб-сайтов].

[Выполняйте аутентификацию пользователей] по каталогу организации. Веб-сайты Azure могут выполнять аутентификацию пользователей с помощью локальных поставщиков удостоверений, например служб федерации Active Directory (AD FS), или клиента Azure Active Directory, синхронизированного с корпоративным развертыванием службы Active Directory. Пользователи могут получать доступ к веб-свойствам на веб-сайтах Azure с помощью единого входа локально или удаленно. Существующие службы, такие как Office 365 или Windows Intune, уже используют Azure Active Directory. Благодаря [простой авторизации] включить аутентификацию на том же клиенте Azure Active Directory для веб-сайта очень легко. 

[Авторизуйте пользователей] для использования веб-свойств. Ту же локальную схему кода ASP.NET можно перенести на веб-сайты Azure, добавив минимум дополнительного кода и используя, например, оформление [Authorize]. При этом сохраняется такая же гибкость для мелкоструктурного управления доступом, как и для приложений, управление которыми осуществляется локально.

### Подключение к локальным ресурсам ###

Подключайтесь к данным или ресурсам веб-сайтов, где бы они ни находились: в облаке для повышения производительности или в локальном расположении для обеспечения соответствия требованиям. Дополнительную информацию о хранении данных в Azure см. в статье [Центр управления безопасностью Azure]. 

В Azure предлагается широкий выбор внутренних серверов баз данных, в том числе для [базы данных SQL Azure] и [MySQL], которые можно подобрать в соответствии с потребностям веб-сайта. Защита данных в Azure позволяет хранить данные рядом с местом размещения веб-сайта и оптимизировать его производительность.

Тем не менее, иногда компании должны хранить свои данные локально. Веб-сайты Azure позволяют с легкостью установить [гибридное подключение] к локальному ресурсу, например ко внутреннему серверу базы данных. Если вам необходимо единое решение по управлению локальными подключениями, можно интегрировать несколько веб-сайтов Azure с одной [виртуальной сетью Azure] с VPN типа "сеть-сеть". После этого у вас будет доступ к локальным ресурсам, если ваши веб-сайты Azure расположены локально. [Enterprise Pizza - подключение веб-сайтов к локальным ресурсам с помощью Service Bus][enterprisepizza]

### Оптимизация

Автоматическое масштабирование с помощью Autoscale, кэширование с помощью кэша Redis для Azure, запуск фоновых задач с помощью заданий WebJob и высокая доступность благодаря Traffic Manager Azure помогут оптимизировать бизнес-приложения.

Возможность [вертикального и горизонтального масштабирования] веб-сайтов Azure позволяет удовлетворить требования бизнес-приложения, независимо от размера рабочей нагрузки. Разверните веб-сайт вручную с помощью [портала управления Azure], программным путем с помощью [API управления службами] или [сценариев PowerShell] либо автоматически с помощью функции автомасштабирования. В рамках **стандартного** плана размещения автомасштабирование позволяет развернуть веб-сайт автоматически, исходя из использования ЦП. Рекомендации см. в статье [Троя Ханта] (Troy Hunt) [10 принципов быстрого масштабирования веб-сайтов с помощью Azure].

Увеличьте скорость отклика веб-сайта с помощью [кэша Redis для Azure]. Используйте его, чтобы кэшировать данные из серверных баз данных и другие элементы, например [состояние сеанса ASP.NET] и [кэш вывода].

Поддерживайте высокую доступность веб-сайта с помощью [Azure Traffic Manager]. Если на основном сайте возникает проблема, при использовании метода **Failover** диспетчер Traffic Manager автоматически направляет трафик на дополнительный сайт.

### Мониторинг и анализ

Средства Azure и сторонние средства позволяют быть в курсе производительности веб-сайта. Вы сможете получать оповещения о критически важных событиях на веб-сайте. Благодаря Application Insights и средствам аналитики веб-журналов HDInsight у вас будет полное представление о пользователях. 

Вы также можете получать [сводку] по текущим метрикам производительности веб-сайта и квотам на ресурсы на панели мониторинга веб-сайтов Azure. Чтобы получить полное представление о доступности, производительности и использовании приложений, можно воспользоваться [Azure Application Insights]. Это средство позволяет быстро и эффективно устранять неполадки, проводить диагностику и получать данные об использовании. Вы также можете обеспечить расширенный мониторинг данных веб-сайтов, применяя средства сторонних производителей, например [New Relic].

В рамках **стандартного** плана размещения на сайт мониторинга приходят почтовые уведомления, даже если ваш сайт перестает отвечать. Дополнительные сведения см. в разделе [Практическое руководство. Получение уведомлений об оповещениях и управление правилами оповещения в Azure].

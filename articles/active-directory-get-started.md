# Приступая к работе с Azure AD

[Что такое Azure AD?](active-directory-whatis.md)<br> [Как это работает?](active-directory-works.md)<br> [Дальнейшие действия](active-directory-next-steps.md)<br> [Подробнее](active-directory-learn-map.md)


Готовы начать работу? Azure AD всем доступна бесплатно. Зарегистрируйтесь, чтобы оформить пробную подписку Azure, после чего ваш каталог станет вам доступен. Каталог сохраняется за вами, даже если вы не намерены ничего покупать. [Попробуйте поработать с Azure AD](http://azure.microsoft.com/trial/get-started-active-directory/)

## Как мне определить, какой каталог следует использовать для тестирования, промежуточного хранения, производства?

При наличии существующей подписки на облачные службы Майкрософт, например Office 365, у вас уже есть каталог в Azure AD, и его можно использовать. Этот каталог уже выполняет проверку подлинности пользователей, которые входят в Office 365. Вам нужно просто зарегистрироваться в Azure с помощью той же самой рабочей учетной записи, которую вы используете для входа в Office 365. Затем можно управлять каталогом на портале управления Azure и добавлять каталоги для тестирования и так далее при необходимости. [Дополнительные сведения об управлении каталогом для подписки Office 365 в Azure](active-directory-how-subscriptions-associated-directory.md#manage-the-directory-for-your-office-365-subscription-in-azure).

Если у вас еще нет подписки на облачную службу Майкрософт, новый каталог будет подготовлен для вашей рабочей учетной записи при регистрации в Azure, и этот каталог можно будет использовать в качестве основного каталога.


## Как добавить собственное зарегистрированное доменное имя?
Хотите добавить пользовательское доменное имя или использовать собственное зарегистрированное доменное имя? [Подробнее о добавлении пользовательского домена](active-directory-add-domain.md).

## Как добавить мою фирменную символику компании?
С помощью Azure AD Premium вы можете добавить логотип своей компании и другую фирменную символику на страницу входа в Azure и страницы панели доступа. [Добавление фирменной символики компании на страницах  входа и панели доступа](active-directory-add-company-branding.md)

## Как интегрировать Azure AD с другим каталогом?
Вы можете настроить синхронизацию каталогов между локальным каталогом и Azure AD, включая синхронизацию паролей, и реализовать единый вход для доступа к приложениям как в локальной среде, так и в облаке. [Дополнительные сведения об интеграции каталогов](active-directory-aadconnect.md).

## Как можно создавать и изменять пользователей?
Создайте учетную запись для каждого пользователя, которому будет предоставлен доступ к облачной службе Майкрософт. По умолчанию, пользователи не имеют прав администратора, но вы можете такие права им назначать. [Дополнительные сведения о создании пользователей Azure AD](active-directory-create-users.md).

## Просмотр отчетов о доступе и использовании
Azure AD предоставляет администраторам различные отчеты. В платных выпусках Azure AD поддерживаются дополнительные отчеты, а для создания собственных настраиваемых отчетов имеется отдельный API отчетов. [Дополнительные сведения об отчетах Azure AD](active-directory-view-access-usage-reports.md).

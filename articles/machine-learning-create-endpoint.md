# Создание конечных точек

Машинное обучение Azure позволяет создать несколько конечных точек для опубликованной веб-службы. Обращение к каждой конечной точке, ее регулирование и управление ею происходит отдельно и независимо от других конечных точек этой веб-службы. У каждой конечной точки есть свой уникальный URL-адрес и ключ авторизации.

Это позволяет пользователям Машинного обучения Azure создавать веб-службы, которые они могут продавать на срок своим клиентам. Каждую конечную точку можно настроить отдельно с помощью ее обученных моделей, при этом точки остаются связанными с экспериментом, в котором создана эта веб-служба. Кроме того, все обновления эксперимента можно выборочно применять к конечной точке без перезаписи настроек.

## Пошаговое создание конечной точки
- Откройте страницу по адресу manage.windowsazure.com и щелкните "Машинное обучение" в левом столбце. Щелкните рабочую область с веб-службой, которая вас интересует.
![Navigate to workspace](./media/machine-learning-create-endpoint/figure-1.png)


- Щелкните вкладку "Веб-службы".
![Navigate to web services](./media/machine-learning-create-endpoint/figure-2.png)


- Щелкните веб-службу, которая вас интересует, чтобы увидеть список доступных конечных точек.
![Navigate to endpoint](./media/machine-learning-create-endpoint/figure-3.png)


- Нажмите кнопку "Добавить конечную точку" в нижней части страницы. Введите имя и описание, а также убедитесь, что в этой веб-службе нет других конечных точек с таким же именем. Оставьте значение по умолчанию для уровня регулирования, если у вас нет конкретных требований.
Дополнительную информацию о регулировании см. на странице [Масштабирование конечных точек API](machine-learning-scaling-endpoints.md).
![Create endpoint](./media/machine-learning-create-endpoint/figure-4.png)


После создания конечную точку можно использовать в синхронных API, API пакетов и на листах Excel.

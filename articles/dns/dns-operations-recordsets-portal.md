# Управление записями и наборами записей DNS с помощью портала Azure


> [AZURE.SELECTOR]
- [Портал Azure](dns-operations-recordsets-portal.md)
- [Интерфейс командной строки Azure](dns-operations-recordsets-cli.md)
- [PowerShell](dns-operations-recordsets.md)


В этой статье описывается, как управлять наборами записей и записями для зоны DNS с помощью портала Azure.

Важно понимать различие между наборами записей DNS и отдельными записями DNS. Набор записей — это коллекция записей в зоне с одним именем и типом. Дополнительные сведения см. в статье [Общие сведения о наборах записей и записях](dns-getstarted-create-recordset-portal.md).

## Создание нового набора записей и записи

Чтобы создать набор записей на портале Azure, см. статью [Создание записей DNS с помощью портала Azure](dns-getstarted-create-recordset-portal.md).


## Просмотр набора записей

1. На портале Azure перейдите к колонке "Зона DNS".

2. Можно выполнить поиск набора записей и выбрать его в списке результатов. Щелкните набор записей. Открываются свойства набора записей.

	![поиск набора записей](./media/dns-operations-recordsets-portal/searchset500.png)


## Добавление новой записи в набор записей

В любой набор записей можно добавить до 20 записей. Набор записей не может содержать две одинаковые записи. Пустые наборы записей (с нулевым числом записей) могут быть созданы, но они не отображаются в серверами имен Azure DNS. Наборы записей типа CNAME могут содержать не более одной записи.


1. В колонке **Свойства набора записей** для зоны DNS щелкните набор, куда требуется добавить запись.

	![выбор набора](./media/dns-operations-recordsets-portal/selectset500.png)

2. Укажите параметры записи, заполнив поля.

	![добавление записи](./media/dns-operations-recordsets-portal/addrecord500.png)

2. Щелкните элемент **Сохранить** в верхней части колонки, чтобы сохранить параметры перед ее закрытием.

3. В углу отображается уведомление о сохранении записи.

	![сохранение записи](./media/dns-operations-recordsets-portal/saving150.png)

4. После сохранения записи значения для набора записей в колонке DNS будут соответствовать новой записи.


## Изменение записи

При изменении записи в существующем наборе доступные для изменения поля зависят от типа используемой записи.

1. Выполните поиск записи в колонке **Свойства набора записей** для своего набора.

2. Измените запись. При изменении записи можно также изменить доступные для нее параметры. В приведенном ниже примере выбрано поле IP-адреса и изменяется его значение.

	![изменение записи](./media/dns-operations-recordsets-portal/modifyrecord500.png)

3. Щелкните элемент **Сохранить** в верхней части колонки, чтобы сохранить параметры. В правом верхнем углу появляется уведомление о сохранении записи.
	
	![сохранение записи](./media/dns-operations-recordsets-portal/saved150.png)


3. После сохранения записи значения для набора записей в колонке DNS будут соответствовать измененной записи.


## Удаление записи из набора записей

Для удаления записей из набора можно использовать портал Azure. При удалении последней записи из набора сам набор не удаляется.

1. Выполните поиск записи в колонке **Свойства набора записей** для своего набора.

2. Щелкните удаляемую запись. После этого щелкните элемент **Удалить**.

	![удаление записи](./media/dns-operations-recordsets-portal/removerecord500.png)

3. Щелкните элемент **Сохранить** в верхней части колонки, чтобы сохранить параметры.

3. После удаления записи значения это будут отражать значения для набора записей в колонке DNS.


## <a name="delete"></a>Удаление набора записей

1. Щелкните элемент **Удалить** в колонке **Свойства набора записей** для своего набора. 

	![удаление набора записей](./media/dns-operations-recordsets-portal/deleterecordset500.png)

2. Появится запрос подтверждения на удаление набора записей с определенным именем.

3. Убедитесь, что имя соответствует удаляемому набору записей, а затем нажмите кнопку **Да**.

4. В колонке "Зона DNS" можно убедиться, что набор записей больше не отображается.


## Работа с записями NS и SOA

Управление автоматически создаваемыми записями NS и SOA осуществляется иначе, чем другими типами записей.

### Изменение записей SOA

Невозможно добавить или удалить записи из автоматически созданного набора записей типа SOA (имя = "@"), но можно изменить любой параметр в записи SOA (кроме "Host") и значение TTL набора записей.

### Изменение записей NS на вершине зоны

Невозможно добавить, удалить или изменить записи в автоматически созданном наборе записей NS на вершине зоны (имя = "@"). Разрешено только одно изменение — изменить TTL набора записей.

### Удаление наборов записей SOA и NS

Невозможно удалить наборы записей SOA и NS на вершине зоны (имя = "@"), которые создаются автоматически вместе с зоной. Они будут удалены автоматически при удалении зоны.

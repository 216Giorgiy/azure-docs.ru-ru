# <a name="tutorial-configure-an-alias-record-to-support-apex-domain-names-with-traffic-manager"></a>Руководство. Настройка записи псевдонима для поддержки вершинных доменных имен с помощью диспетчера трафика 

Вы можете создать запись псевдонима для вашего вершинного имени домена (например, contoso.com) для ссылки на профиль диспетчера трафика. Таким образом, вместо использования в этих целях службы перенаправления можно настроить Azure DNS для ссылки на профиль диспетчера трафика непосредственно из вашей зоны. 


Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание виртуальной машины узла и сетевой инфраструктуры
> * Создание профиля диспетчера трафика
> * Создание записи псевдонима
> * Тестирование записи псевдонима


Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free/?WT.mc_id=A261C142F), прежде чем начинать работу.

## <a name="prerequisites"></a>Предварительные требования
Необходимо иметь доступное имя домена, который можно будет разместить в Azure DNS и использовать для тестирования. У вас должен быть полный контроль над этим доменом, включая возможность определять записи сервера имен (NS) для домена.

Инструкции по размещению домена в Azure DNS см. в [руководстве по размещению домена в Azure DNS](dns-delegate-domain-azure-dns.md).

В качестве примера в этом руководстве используется домен contoso.com — замените его на собственное доменное имя.

## <a name="create-the-network-infrastructure"></a>Создание сетевой инфраструктуры
Сначала создайте виртуальную сеть и подсеть для размещения веб-серверов.
1. Войдите на портал Azure по адресу http://portal.azure.com.
2. В левом верхнем углу портала щелкните **Создать ресурс**, введите в поле поиска *группа ресурсов* и создайте группу ресурсов с именем **RG-DNS-Alias-TM**.
3. Щелкните **Создать ресурс** и выберите **Сеть**, а затем параметр **Виртуальная сеть**.
4. Создайте виртуальную сеть с именем **VNet-Servers**, поместите ее в группу ресурсов **RG-DNS-Alias-TM** и назначьте этой подсети имя **SN-Web**.

## <a name="create-two-web-server-virtual-machines"></a>Создание двух виртуальных машин для веб-серверов
1. Щелкните **Создать ресурс** и выберите параметр **Виртуальная машина Windows Server 2016**.
2. Введите **Web-01** в качестве имени и поместите эту виртуальную машину в группу ресурсов **RG-DNS-Alias-TM**. Введите имя пользователя и пароль и нажмите кнопку **ОК**.
3. В разделе **Размер** выберите номер SKU c 8 ГБ ОЗУ.
4. В разделе **Параметры** выберите виртуальную сеть **VNet-Servers**, подсеть **SN-Web**.
5. Щелкните **Общедоступный IP-адрес**, в разделе **Назначение** выберите **Статический** и нажмите кнопку **ОК**.
6. В качестве общедоступных входящих портов выберите **HTTP**, **HTTPS** и **RDP (3389)**, а затем нажмите кнопку **ОК**.
7. На странице сводки нажмите кнопку **Создать**.

   Операция займет всего несколько минут.
6. Повторите этот процесс, чтобы создать еще одну виртуальную машину с именем **Web-02**.

### <a name="add-a-dns-label"></a>Добавление метки DNS
Общедоступным IP-адресам требуется метка DNS для работы с диспетчером трафика.
1. В группе ресурсов **RG-DNS-Alias-TM** щелкните общедоступный IP-адрес **Web-01-ip**.
2. В разделе **Параметры** выберите пункт **Конфигурация**.
3. В текстовом поле метки DNS-имени введите **web01pip**.
4. Выберите команду **Сохранить**.

Повторите эту процедуру для общедоступного IP-адреса **Web-02-ip**, используя **web02pip** в качестве метки DNS-имени.

### <a name="install-iis"></a>Установка служб IIS

Установите службы IIS на обе виртуальные машины, **Web-01** и **Web-02**.

1. Подключитесь к виртуальной машине **Web-01** и войдите в систему.
2. На панели мониторинга диспетчера серверов нажмите **Добавить роли и компоненты**.
3. Нажмите кнопку **Далее** три раза, а затем на странице **Роли сервера** выберите **Веб-сервер (IIS)**.
4. Щелкните **Добавить компоненты** и нажмите кнопку **Далее**.
5. Нажмите кнопку **Далее** четыре раза, а затем щелкните **Установить**.

   Ее выполнение может занять несколько минут.
6. По завершении установки щелкните **Закрыть**.
7. В веб-браузере откройте **localhost** и убедитесь, что открывается веб-страница IIS по умолчанию.

Повторите эту процедуру для установки служб IIS на виртуальную машину **Web-02**.


## <a name="create-a-traffic-manager-profile"></a>Создание профиля диспетчера трафика

Due 

1. Откройте группу ресурсов **RG-DNS-Alias-TM** и щелкните общедоступный IP-адрес **Web-01-ip**. Запишите этот IP-адрес для последующего использования. Повторите эти действия для общедоступного IP-адреса **Web-02-ip**.
1. Щелкните **Создать ресурс** и выберите **Сеть**, а затем выберите **Профиль диспетчера трафика**.
2. Введите **TM-alias-test** в качестве имени этого профиля и поместите его в группу ресурсов **RG-DNS-Alias-TM**.
3. Нажмите кнопку **Создать**.
4. После завершения развертывания щелкните **Перейти к ресурсу**.
5. На странице профиля диспетчера трафика в разделе **Параметры** щелкните **Конечные точки**.
6. Щелкните **Добавить**.
7. В качестве **типа**выберите **Внешняя конечная точка**, в качестве **имени** введите **EP-Web01**.
8. В текстовом поле **Полное доменное имя (FQDN) или IP-адрес** введите IP-адрес для **Web-01-ip**, который вы записали ранее.
9. Выберите то же **расположение**, что и для других ресурсов, и нажмите кнопку **ОК**.

Повторите эту процедуру, чтобы добавить конечную точку **Web-02**, используя IP-адрес, записанный ранее для **Web-02-ip**.

## <a name="create-an-alias-record"></a>Создание записи псевдонима

Создайте запись псевдонима, которая указывает на профиль диспетчера трафика.

1. Щелкните зону Azure DNS, чтобы открыть ее.
2. Щелкните **Набор записей**.
3. Оставьте текстовое поле **Имя** пустым для представления вершинного доменного имени (например, contoso.com).
4. Оставьте в поле **Тип** запись **A**.
5. Установите флажок **Набор записей псевдонимов**.
6. Щелкните **Выбрать службу Azure** и выберите профиль диспетчера трафика **TM-alias-test**.

## <a name="test-the-alias-record"></a>Тестирование записи псевдонима

1. В веб-браузере перейдите к вашему имени домена apex (например, contoso.com). Должна открыться веб-страница служб IIS по умолчанию. Закройте веб-браузер.
2. Завершите работу виртуальной машины **Web-01** и подождите несколько минут, пока она полностью завершит работу.
3. Откройте новый веб-браузер и снова перейдите к вашему вершинному доменному имени.
4. Снова должна появиться страница служб IIS по умолчанию, так как диспетчер трафика обработал ситуацию и направил трафик на виртуальную машину **Web-02**.

## <a name="clean-up-resources"></a>Очистка ресурсов

Удалите группу ресурсов **RG-DNS-Alias-TM**, если ресурсы, созданные для этого руководства, больше не нужны.

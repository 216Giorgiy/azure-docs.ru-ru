# Обзор Azure DNS

За каждым веб-сайтом или службой в Интернете есть IP-адрес. Например, www.microsoft.com использует IP-адрес 134.170.185.46. Служба доменных имен, или DNS, отвечает за преобразование (или разрешение) имени веб-сайта или службы в IP-адрес.

Azure DNS является службой размещения для доменов DNS, предоставляющей разрешение имен с помощью инфраструктуры Microsoft Azure. Размещая домены в Azure, вы можете управлять своими записями DNS с помощью тех же учетных данных, API и инструментов и оплачивать, как и другие службы Azure.

Домены DNS в Azure DNS размещаются в глобальной сети DNS-серверов Azure. Мы используем адресацию любому устройству сети, чтобы на каждый запрос DNS отвечал ближайший доступный DNS-сервер. Это обеспечивает высокую производительность и высокий уровень доступности для вашего домена.

Эта служба основана на диспетчере ресурсов Azure (ARM). Таким образом он использует преимущества ARM, такие как управление доступом на основе ролей, журналы аудита и блокировка ресурсов.

Доменами и записями можно управлять с помощью портала Azure, командлетов Azure PowerShell и кроссплатформенного Azure CLI. Приложения, для которых необходимо автоматическое управление DNS, можно интегрировать со службой с помощью REST API и пакетов SDK.

Azure DNS в настоящее время не поддерживает приобретение доменных имен. Чтобы приобрести домены, вы должны использовать стороннего регистратора доменных имен, который обычно взимает небольшую годовую плату. Затем эти домены можно разместить в Azure DNS для управления записями DNS — подробности см. в разделе [Делегирование домена в Azure DNS](dns-domain-delegation.md).

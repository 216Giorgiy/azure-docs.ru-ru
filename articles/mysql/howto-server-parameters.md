# <a name="how-to-configure-server-parameters-in-azure-database-for-mysql-using-the-azure-portal"></a>Как настроить параметры сервера в базе данных Azure для MySQL с помощью портала Azure

База данных Azure для MySQL поддерживает настройку некоторых параметров сервера. В этой статье описывается, как настроить эти параметры с помощью портала Azure, а также приводится список поддерживаемых параметров, их значения по умолчанию и диапазон допустимых значений. Не все параметры сервера можно настроить. Это возможно только для параметров, приведенных ниже.

## <a name="navigate-to-server-parameters-blade-on-azure-portal"></a>Переход к колонке параметров сервера на портале Azure

Войдите на портал Azure, а затем щелкните имя сервера базы данных Azure для MySQL. В разделе **Параметры** щелкните **Параметры сервера**, чтобы открыть колонку параметров сервера для базы данных Azure для MySQL.

![Колонка "Параметры сервера" на портале Azure](./media/howto-server-parameters/auzre-portal-server-parameters.png)

## <a name="list-of-configurable-server-parameters"></a>Список доступных для настройки параметров сервера

В следующей таблице перечислены параметры сервера, поддерживаемые в настоящее время. Эти параметры можно настроить в соответствии с требованиями приложения.

> [!div class="mx-tableFixed"]
|Имя параметра|По умолчанию|Диапазонный индекс|Описание|
|---|---|---|---|
|binlog_group_commit_sync_delay|1000|0, 11–1 000 000|Задает время ожидания (мкс) фиксации журнала в двоичном формате перед синхронизацией двоичного файла журнала на диск.|
|binlog_group_commit_sync_no_delay_count|0|0–1 000 000|Максимальное количество ожидаемых транзакций перед прекращением отсчета текущей задержки в соответствии со значением binlog-group-commit-sync-delay.|
|character_set_server|LATIN1|BIG5, UTF8MB4 и т. д.|Используйте charset_name в качестве кодировки по умолчанию сервера.|
|div_precision_increment|4|0–30|Число знаков, на которое следует увеличивать размерность результата операций деления.|
|group_concat_max_len|1024|4–16 777 216|Максимально допустимая длина результата GROUP_CONCAT() в байтах.|
|innodb_adaptive_hash_index|ВКЛ|ON, OFF|Указывает, включены ли адаптивные хэш-индексы InnoDB.|
|innodb_lock_wait_timeout|50|1–3600|Время ожидания (с) транзакцией InnoDB блокировки строки.|
|interactive_timeout|1800|10–1800|Время ожидания (с) сервером действия с интерактивным подключением перед его закрытием.|
|log_bin_trust_function_creators|ВЫКЛ.|ON, OFF|Эта переменная применяется, когда включено ведение журнала в двоичном формате. Она указывает, можно ли доверять создателям хранимых функций создание хранимых функций, которые порождают небезопасные события, регистрируемые в журнале в двоичном формате.|
|log_queries_not_using_indexes|ВЫКЛ.|ON, OFF|Регистрирует запросы, которым требуется получить все строки, в журнале медленных запросов.|
|log_slow_admin_statements|ВЫКЛ.|ON, OFF|Добавляет медленные административные инструкции к инструкциям, записываемым в журнал медленных запросов.|
|log_throttle_queries_not_using_indexes|0|0–4 294 967 295|Ограничивает число запросов в минуту, которые могут быть записаны в журнал медленных запросов.|
|long_query_time|10|1–10^100|Если запрос выполняется больше времени, чем указано в этом параметре (с), то сервер увеличивает значение переменной состояния Slow_queries.|
|max_allowed_packet|536 870 912|1024–1 073 741 824|Максимальный размер отдельного пакета, любой созданной или промежуточной строки либо какого-либо параметра, отправляемого функцией API C mysql_stmt_send_long_data().|
|min_examined_row_limit|0|0-18 446 744 073 709 551 615|Регистрирует запросы, которые содержат больше заданного количества строк, в журнале медленных запросов. |
|server_id|3 293 747 068|1000–4 294 967 295|Идентификатор сервера, используемый в репликации для уникальной идентификации каждого главного и ведомого элементов.|
|slave_net_timeout|60|30–3600|Время ожидания (с) следующих данных из главного элемента, по истечении которого ведомый элемент считает, что подключение нарушено, прерывает чтение и пытается восстановить подключение.|
|slow_query_log|ВЫКЛ.|ON, OFF|Включает или отключает ведение журнала медленных запросов.|
|sql_mode|Выбрано значение 0.|ALLOW_INVALID_DATES, IGNORE_SPACE и т. д.|Текущий режим сервера SQL Server.|
|time_zone|SYSTEM|Примеры: -8:00, +05:30.|Часовой пояс сервера.|
|wait_timeout|120|60–240|Время ожидания (с) сервером действия с неинтерактивным подключением перед его закрытием.|

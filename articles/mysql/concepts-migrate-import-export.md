# <a name="migrate-your-mysql-database-using-import-and-export"></a>Перенос базы данных MySQL с помощью импорта и экспорта
В этом руководстве показан наиболее распространенный способ импорта данных в базу данных Azure MySQL и экспорта данных из нее с помощью MySQL Workbench. 

## <a name="before-you-begin"></a>Перед началом работы
Прежде чем приступить к выполнению этого руководства, необходимы следующие компоненты:
- сервер базы данных Azure для MySQL ([создайте свою первую базу данных Azure MySQL](quickstart-create-mysql-server-database-using-azure-portal.md));
- MySQL Workbench ([скачать MySQL Workbench](https://dev.mysql.com/downloads/workbench/)).

## <a name="use-common-tools"></a>Использование распространенных инструментов
Используйте распространенные инструменты, такие как MySQL Workbench, Toad или Navicat, для удаленного подключения и импорта или экспорта данных в базу данных Azure для MySQL. Используйте эти инструменты на своем клиентском компьютере, подключенном к Интернету, чтобы подключиться к базе данных Azure для MySQL. Для обеспечения безопасности используйте подключение с SSL-шифрованием, а также см. статью [SSL-соединения в базе данных Azure для MySQL](concepts-ssl-connection-security.md). При переносе в базу данных Azure для MySQL не нужно перемещать файлы импорта и экспорта в особое облачное расположение. 

## <a name="create-a-database-on-the-target-azure-database-for-mysql-service"></a>Создание базы данных в целевой службе базы данных Azure для MySQL
Создайте пустую базу данных на целевом сервере базы данных Azure для MySQL, на который необходимо перенести данные. Используйте для этого MySQL Workbench, Toad, Navicat или любой сторонний инструмент MySQL. База данных может иметь то же имя, что и база данных, содержащая данные дампа. Вы также можете создать базу данных с другим именем.

## <a name="import-and-export-using-mysql-workbench"></a>Импорт и экспорт с помощью MySQL Workbench
Существует два способа экспортировать и импортировать данные в MySQL Workbench, каждый из которых служит для определенной цели. 

## <a name="table-data-export-and-import-wizard-using-object-browser-context-menu"></a>Мастер экспорта и импорта данных таблиц в контекстном меню обозревателя объектов
![Экспорт и импорта в MySQL Workbench с помощью контекстного меню обозревателя объектов](./media/concepts-migrate-import-export/p1.png)

Этот мастер поддерживает операции импорта и экспорта, используя файлы формата CSV и JSON, и включает несколько вариантов настройки (разделители, выбор столбцов, выбор кодировки и др.). Операции мастера можно выполнять на локальных или удаленно подключенных серверах MySQL. Операция импорта включает сопоставление таблиц, столбцов и типов. Мастер можно запустить из контекстного меню обозревателя объектов, щелкнув таблицу правой кнопкой мыши и выбрав **Table Data Export Wizard** (Мастер экспорта данных таблиц) или **Table Data Import Wizard** (Мастер импорта данных таблиц). 

## <a name="table-data-export-wizard"></a>Мастер экспорта данных таблиц
В следующем примере данные таблицы экспортируются в CSV-файл. 
- Щелкните правой кнопкой мыши таблицу базы данных, которую необходимо экспортировать. 
- Выберите **Table Data Export Wizard** (Мастер экспорта данных таблиц). Выберите столбцы, которые необходимо экспортировать, укажите значение "Row Offset" (Смещение строки) (при необходимости) и "Count" (Количество) (при необходимости). 
- В окне "Select data for export" (Выбор данных для экспорта) нажмите кнопку **Next** (Далее). Выберите путь к файлу, тип файла (CSV или JSON), разделитель строк, вложенные строки и разделитель полей. 
- В окне "Select output file location" (Выбор расположения выходного файла) нажмите кнопку **Next** (Далее). В окне "Export Data" (Экспорт данных) также нажмите кнопку "Next" (Далее).


## <a name="table-data-import-wizard"></a>Мастер импорта данных таблиц
В следующем примере данные таблицы импортируются из CSV-файла.
- Щелкните правой кнопкой мыши таблицу базы данных, которую необходимо импортировать. 
- Найдите CSV-файл, который необходимо импортировать, выберите его, а затем нажмите кнопку "Next" (Далее). 
- Выберите целевую таблицу (новую или существующую), установите или снимите флажок "Truncate table before import" (Усечение таблицы перед импортом) и нажмите кнопку "Next" (Далее).
- Выберите кодировку и столбцы, которые необходимо импортировать, и нажмите кнопку "Next" (Далее). 
- В окне "Import data" (Импорт данных) также нажмите кнопку "Next" (Далее). После этого данные будут импортированы.

## <a name="sql-data-export-and-import-wizard-from-management-navigator"></a>Мастер экспорта и импорта данных SQL в Management Navigator
Используйте этот мастер для экспорта или импорта данных SQL, созданных в MySQL Workbench или с помощью команды mysqldump. Для доступа к этому мастеру используйте панель "Navigator" (Навигатор) или выберите в главном меню "Server" (Сервер), а затем выберите "Data Import" (Импорт данных) или "Data Export" (Экспорт данных). 

## <a name="data-export"></a>Экспорт данных
![Экспорт данных в MySQL Workbench с помощью Management Navigator](./media/concepts-migrate-import-export/p2.png)

Эта вкладка позволяет экспортировать данные MySQL. 
- Выберите все схемы, которые необходимо экспортировать. При необходимости выберите из каждой схемы конкретные объекты или таблицы и выполните экспорт. Параметры конфигурации позволяют выполнить экспорт в папку проекта или самодостаточным файл SQL. При необходимости можно также выполнить дамп сохраненных процедур и событий или пропустить данные таблицы. 
- В качестве альтернативы можно использовать команду **Export a Result Set** (Экспортировать результирующий набор), чтобы выполнить экспорт конкретного результирующего набора из редактора SQL в другой формат, например CSV, JSON, HTML или XML. 
- Выберите объекты базы данных, которые необходимо экспортировать, и настройте связанные параметры. 
- Щелкните **Refresh** (Обновить), чтобы загрузить текущие объекты. 
- При необходимости откройте вкладку "Advanced Options" (Дополнительные параметры), которая позволяет точнее настроить операцию экспорта. Например, можно добавить блокировки таблиц, использовать инструкцию replace вместо insert, заключить идентификаторы в кавычки в виде обратного апострофа, а также выполнить другие настройки. 
- Чтобы начать процесс экспорта, нажмите кнопку **Start Export** (Начать экспорт). 


## <a name="data-import"></a>Импорт данных
![Импорт данных в MySQL Workbench с помощью Management Navigator](./media/concepts-migrate-import-export/p3.png)

Эта вкладка позволяет импортировать или восстановить экспортированные данные из операции экспорта данных. Также для восстановления можно использовать другие экспортированные данные из команды mysqldump. 
- Выберите папку проекта или самодостаточный файл SQL, выберите схему, в которую будут импортированы данные, или выберите "New" (Создать), чтобы определить новую схему. 
- Нажмите кнопку **Start Import** (Начать импорт), чтобы начать процесс. Данные будут импортированы соответствующим образом.

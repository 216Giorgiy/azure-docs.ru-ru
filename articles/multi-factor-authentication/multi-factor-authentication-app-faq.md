# <a name="microsoft-authenticator-application-faq"></a>Часто задаваемые вопросы о приложении Microsoft Authenticator
Приложение Microsoft Authenticator пришло на смену приложению Azure Authenticator. Это приложение, рекомендуемое при использовании Azure Multi-Factor Authentication. Приложение Microsoft Authenticator доступно для Windows Phone, Android и iOS.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы
### <a name="what-happened-to-the-azure-authenticator-multi-factor-auth-and-microsoft-account-apps"></a>Что случилось с Azure Authenticator, Multi-Factor Auth и приложением учетной записи Майкрософт?
Теперь вместо них используется приложение Microsoft Authenticator. Приложение Azure Authenticator обновлено до Microsoft Authenticator. Если вы используете Multi-Factor Auth или приложение учетной записи Майкрософт, установите Microsoft Authenticator и снова добавьте учетные записи. При этом важно сначала добавить учетные записи в новое приложение и лишь затем удалять старые приложения.

### <a name="im-already-using-the-microsoft-authenticator-application-for-verification-codes-how-do-i-switch-to-one-click-push-notifications"></a>Я уже использую приложение Microsoft Authenticator для кодов безопасности. Как перейти к использованию push-уведомлений одним щелчком?
Подтверждение входа через push-уведомления доступно только для учетных записей Майкрософт, но не для сторонних учетных записей, таких как Google или Facebook. Но для рабочих или учебных учетных записей Майкрософт вы можете отключить эту функцию в вашей организации.

Если вы используете учетную запись Майкрософт в качестве личной учетной записи и хотите переключить ее на использование push-уведомлений, необходимо повторно добавить эту учетную запись. Повторно зарегистрируйте устройство с помощью своей учетной записи и настройте push-уведомления.  

Если вы используете Microsoft Authenticator для учебной или рабочей учетной записи, политику использования push-уведомлений одним щелчком определяет организация, предоставившая эту учетную запись.

### <a name="when-will-i-be-able-to-use-one-click-push-notifications-on-iphone-or-ipad"></a>Когда я смогу использовать push-уведомления одним щелчком на iPhone или iPad?
Эта функция находится на этапе бета-версии до конца августа, когда станет широко доступной для учетных записей Майкрософт. Если вы хотите присоединиться к программе тестирования бета-версии, отправьте электронное сообщение по адресу msauthenticator@microsoft.com. В сообщении укажите свое имя, фамилию и Apple ID.  

### <a name="do-one-click-push-notifications-work-for-non-microsoft-accounts"></a>Совместимы ли push-уведомления одним щелчком с учетными записями, не относящимися к корпорации Майкрософт?
Нет, push-уведомления работают только с учетными записями Майкрософт и Azure Active Directory. Если компания или учебное заведение использует учетные записи Azure AD, то они могут отключить эту функцию.  

### <a name="i-restored-my-device-from-a-backup-and-my-account-codes-are-missing-or-not-working-what-happened"></a>Устройство восстановлено из резервной копии, и теперь коды учетной записи отсутствуют или не работают. Что произошло?
В целях безопасности мы не восстанавливаем учетные записи из резервных копий приложений. При восстановлении приложения iOS из архивной копии ваши учетные записи по-прежнему отображаются, но они не выполняют проверку при входе и не создают коды безопасности. После восстановления приложения удалите учетные записи и добавьте их снова.

### <a name="i-got-a-new-device-how-do-i-remove-the-microsoft-authenticator-app-from-my-old-device-and-move-to-the-new-one"></a>У меня есть новое устройство. Как удалить приложение Microsoft Authenticator со старого устройства и перенести на новое?
Когда вы добавите Microsoft Authenticator на новое устройство, это приложение не будет автоматически удалено с других устройств. Чтобы выбрать, какие устройства будут использоваться для учетной записи, перейдите на веб-сайт, на котором вы управляете двухфакторной проверкой подлинности, и удалите старые приложения.

Для личных учетных записей Майкрософт это будет страница [Безопасность учетной записи](https://account.microsoft.com/security). Для учебных или рабочих учетных записей Майкрософт это может быть страница [Мои приложения](https://myapps.microsoft.com) или пользовательский портал, настроенный в вашей организации.

### <a name="how-do-i-remove-an-account-from-the-app"></a>Как удалить учетную запись из приложения?
* Для iOS: на главном экране проведите справа налево по плитке учетной записи. Нажмите кнопку **Удалить**.
* Для Windows Phone: на главном экране нажмите кнопку меню, а затем выберите пункт **Изменить учетные записи**. Выберите элемент **X** рядом с именем учетной записи.
* Для Android: на главном экране нажмите кнопку меню, а затем выберите пункт **Edit accounts** (Изменить учетные записи). Выберите элемент **X** рядом с именем учетной записи.

Если ваше устройство Android зарегистрировано в организации, для удаления учетной записи может потребоваться дополнительное действие. На этих устройствах приложение Microsoft Authenticator автоматически регистрируется как администратор устройства. Если требуется полностью удалить приложение, сначала нужно отменить эту регистрацию в параметрах приложения.

### <a name="why-does-the-app-request-so-many-permissions"></a>Почему приложение запрашивает так много разрешений?
Вот полный список разрешений, которые мы запрашиваем, и способ их использования в приложении.

* **Камера**. Используется для сканирования QR-кодов при добавлении учебной, рабочей или сторонней учетной записи.
* **Контакты и телефон**. Мы стараемся упростить процесс входа с помощью личной учетной записи Майкрософт и для этого находим все существующие учетные записи, которые используются на телефоне.
* **SMS**. Когда вы впервые входите в систему с помощью личной учетной записи Майкрософт, мы должны убедиться, что сохраненный у нас номер телефона совпадает с тем, который вы используете. Мы отправляем текстовое сообщение на тот телефон, на который вы скачали приложение. Сообщение содержит код проверки из 6–8 цифр. Мы не просим вас найти этот код и ввести его в приложении, а просто сами отслеживаем текстовое сообщение.
* **Отображение поверх других приложений**. При получении уведомления для подтверждения идентификации мы отображаем его поверх любого приложения, которое работает в этот момент.
* **Получение данных из Интернета**. Это разрешение требуется для отправки уведомлений.
* **Запрет на переход телефона в спящий режим**. Если вы зарегистрируете устройство в организации, она сможет изменить эту политику на телефоне.
* **Управление вибрацией**. Вы можете указать, нужно ли включать вибрацию, когда приходит уведомление для проверки личности.
* **Использование оборудования для отпечатков пальцев**. При использовании некоторых рабочих и учебных учетных записей требуется вводить дополнительный ПИН-код при каждой проверке личности. Чтобы упростить этот процесс, мы позволяем использовать вместо ПИН-кода отпечаток пальца.
* **Просмотр сетевых подключений**. Приложению нужно подключение к сети или Интернету для добавления учетной записи Майкрософт.
* **Чтение содержимого хранилища**. Это разрешение используется, только когда вы сообщаете о технических проблемах через параметры приложения. Некоторые сведения из хранилища собираются для диагностики проблемы.
* **Полный доступ к сети**. Это разрешение требуется для отправки уведомлений для проверки личности.
* **Запуск при загрузке**. Это разрешение гарантирует, что после перезапуска телефона вы будете получать уведомления для проверки личности.

## <a name="contact-us"></a>Свяжитесь с нами
Если ответ на ваш вопрос отсутствует в списке, оставьте комментарий в нижней части страницы. Или [обратитесь в службу поддержки](https://support.microsoft.com/contactus), и мы постараемся решить вашу проблему как можно скорее.

При обращении в службу поддержки укажите всю возможную информацию из следующего списка.

* **Идентификатор пользователя** — адрес электронной почты, с помощью которого вы пытались войти в систему.
* **Общее описание ошибки** — точный текст полученного сообщения об ошибке.  Если сообщения об ошибке не было, подробно опишите непредвиденное поведение, которое вы заметили.
* **Страница** — на какой странице вы находились, когда увидели ошибку (включая URL-адрес)?
* **ErrorCode** — выдаваемый код возникающей ошибки.
* **SessionId** — выдаваемый идентификатор сеанса.
* **Correlation ID** — код идентификатора корреляции, создаваемый при просмотре ошибки пользователем.
* **Timestamp** — точные дата и время, когда вы заметили ошибку (включая часовой пояс).

Почти всю эту информацию вы найдете на странице входа. Если вы не успели вовремя подтвердить вход, щелкните **Просмотр подробностей**.

![Сведения об ошибке при входе](./media/multi-factor-authentication-end-user-troubleshoot/view_details.png)

Если вы укажете эти сведения, мы сможем устранить неполадку максимально быстро.

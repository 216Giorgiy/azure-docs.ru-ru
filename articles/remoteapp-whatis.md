#Что такое Azure RemoteApp?

Azure RemoteApp привносит в Azure функциональные возможности локальной программы Microsoft RemoteApp, поддерживаемой службами удаленных рабочих столов. Azure RemoteApp позволяет открывать безопасный удаленный доступ к приложениям с различных пользовательских устройств.

При перемещении RemoteApp в Azure вы сможете воспользоваться возможностями хранилища, масштабируемостью и глобальным доступом Azure, не имея при этом дела со сложной локальной конфигурацией. Корпорация Майкрософт обеспечивает техническое обслуживание Azure и гарантирует надежность системы. При этом вы можете сосредоточиться на более важных задачах, например на создании лучших приложений для вашего бизнеса. Еще одним преимуществом Azure RemoteApp является доступность: ваши пользователи получат доступ к программам RemoteApp с устройств, работающих на Windows, iOS, Mac OS X и Android. Они смогут использовать ваши приложения в удобной для них среде, а вы в это время сможете управлять приложениями из портала управления Azure. 

Ознакомьтесь с более подробной информацией о RemoteApp или, если вы уже заинтересованы, [опробуйте RemoteApp прямо сейчас](http://azure.microsoft.com/ru-ru/services/remoteapp/).

Azure RemoteApp - это часть [инфраструктуры Microsoft Virtual Desktop](http://www.microsoft.com/ru-ru/server-cloud/products/virtual-desktop-infrastructure/explore.aspx).

##Параметры развертывания RemoteApp
Существует два типа развертывания RemoteApp.


- **Облачное развертывание** размещается и сохраняет все данные для программ в облаке Azure. Пользователи могут получать доступ к приложениям через свои учетные записи Microsoft или корпоративные учетные данные, синхронизированные или прошедшие федеративную проверку подлинности Azure Active Directory.
- **Гибридное развертывание** размещается и сохраняет все данные в облаке Azure, но при этом открывает пользователям доступ к данным и ресурсам, сохраненным в вашей локальной сети. Пользователи могут получать доступ к приложениям через корпоративные учетные данные, синхронизированные или прошедшие федеративную проверку подлинности Azure Active Directory.

###Облачное развертывание

[Облачное развертывание RemoteApp](http://azure.microsoft.com/ru-ru/documentation/articles/remoteapp-create-cloud-deployment/) предоставляет автономный метод размещения приложений в облаке. Облачное развертывание работает только в облаке Azure в отличие от подключения к локальной сети.

В рамках предварительной версии RemoteApp мы предоставляем вам приложения Office 2013, предварительно установленные и готовые к совместной работе с пользователями. При использовании доступного программного обеспечения вы сможете быстро подготовить свои службы.

Дополнительное преимущество облачного развертывания с приложениями Office 2013 заключается в том, что приложения и операционная система, на которой основываются ваши службы, будут регулярно обновляться, а серверная защита Microsoft от вредоносных программ Майкрософт гарантирует полную безопасность. Конечные пользователи смогут получать доступ к приложениям через учетные записи Microsoft или корпоративные учетные данные. Вам, как администратору, все это позволит не беспокоиться о том, кому к какому приложению следует открывать доступ.

Вы можете также создать облачное развертывание для совместной работы ваших пользователей с приложением или набором приложений. Для этого необходимо [создать настраиваемый образ шаблона](http://azure.microsoft.com/ru-ru/documentation/articles/remoteapp-create-custom-image/) (с помощью которого мы публикуем приложения в RemoteApp) и при создании развертывания просто выбрать нужный образ (например, вместо образа Office 2013). 

###Гибридное развертывание
[Гибридное развертывание RemoteApp](http://azure.microsoft.com/ru-ru/documentation/articles/remoteapp-create-hybrid-deployment/) позволяет предоставлять пользователям настраиваемый набор приложений и доступ к данным и ресурсам из локальной сети. В отличие от настраиваемого образа, используемого в облачном развертывании, созданный образ для гибридного развертывания запускает приложение в доменной среде. Это гарантирует полный доступ ко всем данным и локальной сети.

Благодаря интеграции Active Directory в Azure Active Directory (с помощью DirSync) пользователи смогут получать доступ к данным и приложениям через корпоративные учетные данные. При использовании рабочей учетной записи Active Directory можно применить корпоративную политику к облаку и управлять приложениями, которые вы предоставляете через RemoteApp.

Пока вы не создадите образ шаблона на Windows Server 2012 R2 с ролью узла сеансов удаленных рабочих столов, приложения, которые вы публикуете для своих пользователей, будут несколько ограниченны. Если приложения исправно работают в этой среде образа шаблона, ваши конечные пользователи получат к ним доступ через RemoteApp. 

###Обновление развертывания
Одним из ключевых различий между гибридным и облачным развертываниями является способ обработки обновлений программного обеспечения. При использовании облачного развертывания с предустановленным образом Office 2013 вам не нужно беспокоиться об обновлениях. Система будет обслуживать себя сама и разворачивать обновления на постоянной основе как для приложений, так и для операционной системы.

При работе с гибридными развертываниями, а также с облачными развертываниями с настраиваемым образом шаблона за обслуживание образа и приложений отвечаете вы. При использовании доменных образов вы можете управлять обновлениями с помощью таких средств, как Центр обновления Windows, групповая политика или системный центр.

После обновления настраиваемого образа шаблона следует загрузить новый образ в облако Azure, а затем обновить развертывание для использования нового образа. (Это можно сделать на странице быстрого запуска RemoteApp или с панели мониторинга.)

##Поддерживаемые клиенты RemoteApp
В рамках предварительной версии Azure RemoteApp мы выпустили новое клиентское приложение Microsoft RemoteApp для Windows и Windows RT, а также обновления для приложений удаленных рабочих столов Microsoft для iOS и Android. С помощью этих приложений ваши пользователи смогут получать доступ к новым программам RemoteApp со своих мобильных устройств или компьютеров.

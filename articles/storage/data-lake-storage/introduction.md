# <a name="introduction-to-azure-data-lake-storage-gen2-preview"></a>Общие сведения о хранилище Azure Data Lake Storage Gen2 (предварительная версия)

Azure Data Lake Storage Gen2 (предварительная версия) — это набор возможностей аналитики больших данных, созданных на основе [хранилища BLOB-объектов Azure](../blobs/storage-blobs-introduction.md). Это хранилище позволяет работать с данными с использованием принципов файловой системы и хранилища объектов. Благодаря этому Data Lake Storage Gen2 — это единственная облачная мультимодальная служба хранения, позволяющая выполнять аналитику всех данных.

Хранилище Data Lake Storage Gen2 имеет все качества, необходимые на каждом из этапов жизненного цикла данных аналитики. Этого удалось достичь за счет объединения возможностей двух имеющихся служб хранения. Это хранилище предоставляет функции [Azure Data Lake Storage Gen1](../../data-lake-store/index.md), например семантика файловой системы, защита на уровне файлов и масштабирование, а также экономичность, многоуровневость, возможности высокой доступности и аварийного восстановления, большую экосистему средств и пакетов SDK [хранилища BLOB-объектов Azure](../blobs/storage-blobs-introduction.md). В Data Lake Storage Gen2 сохранились все качества хранилища объектов, а также появились возможности работы с файловой системой, что позволило оптимизировать рабочие нагрузки аналитики.

## <a name="designed-for-enterprise-big-data-analytics"></a>Возможности аналитики больших данных организаций

Data Lake Storage Gen2 — это основная служба хранения, позволяющая создавать корпоративные хранилища данных в Azure. Разработанное з целью обработки нескольких петабайт информации и с поддержкой сотен гигабит пропускной способности, хранилище Data Lake Storage Gen2 предоставляет простой способ управления огромным количеством данных.

Основополагающая функция Data Lake Storage Gen2 — это добавление [иерархических пространств имен](./namespace.md) в службу хранилища BLOB-объектов, что позволяет упорядочивать объекты и файлы в иерархии каталогов, обеспечивая тем самым удобный доступ к данным. Иерархическое пространство имен также позволяет Data Lake Storage Gen2 одновременно поддерживать принципы файловой системы и хранилища объектов. Например, согласно общему соглашению об именовании имитация иерархической структуры папок в хранилище объектов выполняется за счет косой черты в имени. Эта структура реализована в Data Lake Storage Gen2. Задачи, такие как переименование или удаление каталога, стали атомарными операциями с метаданными в каталоге. Больше не нужно перечислять или обрабатывать все объекты с общим префиксом имени каталога.

Раньше облачная аналитика влияла на производительность, возможности управления и безопасность. В Data Lake Storage Gen2 все эти проблемы устранены:

- **Производительность** оптимизирована, так как больше не нужно копировать или передавать данные, чтобы проанализировать их. Иерархическое пространство имен значительно ускоряет выполнение операций управления каталогом, что повышает общую производительность задания.

- **Управление** стало проще, так как вы можете упорядочивать файлы в каталогах и подкаталогах и так с ними работать.

- **Экономичность** достигается за счет того, что хранилище Data Lake Storage Gen2 создано на основе недорогого [хранилища BLOB-объектов Azure](../blobs/storage-blobs-introduction.md). Дополнительные функции еще более снижают совокупную стоимость владения при выполнении аналитики больших данных в Azure.

## <a name="key-features-of-data-lake-storage-gen2"></a>Основные функции Data Lake Storage Gen2

> [!NOTE]
> В общедоступной предварительной версии хранилища Data Lake Gen2 некоторые из перечисленных ниже функций могут отличаться в зависимости от их доступности. В рамках действия программы по ознакомлению с предварительной версией информация о выпуске новых функций и поддержке регионов будет поступать.
> [Зарегистрируйтесь](https://aka.ms/adlsgen2signup) на получение общедоступной предварительной версии Data Lake Storage Gen2.  

- **Доступ, совместимый с Hadoop.** Хранилище Data Lake Storage Gen2 позволяет получать доступ к данным и управлять ими так же, как и в [распределенной файловой системе Hadoop (HDFS)](http://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html). Во всех средах Apache Hadoop, в том числе [Azure HDInsight](../../hdinsight/index.yml) и [Azure Databricks](../../azure-databricks/index.yml), доступен новый [драйвер ABFS](./abfs-driver.md), который позволяет получать доступ к данным в Data Lake Storage Gen2.

- **Мультимодальный доступ к данным по нескольким протоколам.** Data Lake Storage Gen2 — это **мультимодальная** служба хранения, которая **одновременно** предоставляет доступ к данным хранилища объектов и файловой системы. Это достигается через несколько конечных точек протоколов, имеющих доступ к тем же данным. 

    В отличие от других решений данные, хранящиеся в Data Lake Storage Gen2, не нужно перемещать или преобразовывать перед запуском разнообразных средств аналитики. Доступ к данным можно получить через традиционные [API-интерфейсы хранилища BLOB-объектов](../blobs/storage-blobs-introduction.md) (например, прием данных с помощью [функции сбора данных в концентраторах событий](../../event-hubs/event-hubs-capture-enable-through-portal.md)). Затем эти данные можно одновременно обрабатывать с помощью HDInsight или Azure Databricks. 

- **Экономичность.** Data Lake Storage Gen2 отличается низкой стоимостью приобретения емкости хранилища и выполнения транзакций. За счет таких встроенных функций, как [Жизненный цикл хранилища BLOB-объектов Azure](../common/storage-lifecycle-managment-concepts.md), в ходе жизненного цикла данных тарифные ставки максимально снижаются.

- **Поддержка средств, платформ и приложений хранилища BLOB-объектов.** Хранилище Data Lake Storage Gen2 поддерживает большое количество средств, платформ и приложений хранилища BLOB-объектов.

- **Оптимизированный драйвер.** Драйвер `abfs` [оптимизирован специально](./abfs-driver.md) под аналитику больших данных. Соответствующие интерфейсы REST API подключены через конечную точку `dfs`, `dfs.core.windows.net`.

## <a name="scalability"></a>Масштабируемость

Служба хранилища Azure поддерживает возможности масштабирования по умолчанию вне зависимости от того, получаете вы доступ к данным через интерфейсы Data Lake Storage Gen2 или хранилища BLOB-объектов. Вы можете хранить и обрабатывать *множество эксабайтов данных*. Этот объем хранилища предоставляется с высокой пропускной способностью (Гбит/с) и большим количеством операций ввода-вывода в секунду (IOPS). Помимо простой сохраняемости обработка выполняется с практически постоянными задержками запроса, которые измеряются на уровне службы, учетной записи и файла.

## <a name="cost-effectiveness"></a>Экономичность

Одно из многих преимуществ создания Data Lake Storage Gen2 на основе хранилища BLOB-объектов Azure — это [низкая](https://azure.microsoft.com/pricing/details/storage) стоимость приобретения емкости хранилища и выполнения транзакций. В отличие от других облачных служб хранилище Data Lake Storage Gen2 снижает затраты, так как данные не нужно перемещать или преобразовывать перед выполнением анализа.

Кроме того, такие возможности, как [иерархическое пространство имен](./namespace.md), значительно повышают общую производительность большинства аналитических заданий. Это улучшение означает, что для обработки такого же объема данных требуется меньше вычислительной мощности, что снижает совокупную стоимость владения комплексных заданий аналитики.

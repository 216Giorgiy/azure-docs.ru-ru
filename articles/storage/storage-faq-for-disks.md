# <a name="frequently-asked-questions-about-azure-iaas-vm-disks-and-managed-and-unmanaged-premium-disks"></a>Часто задаваемые вопросы о дисках виртуальных машин Azure IaaS, а также об управляемых и неуправляемых дисках класса "Премиум"

В этой статье мы рассмотрим некоторые часто задаваемые вопросы об управляемых дисках и хранилище класса Premium.

## <a name="managed-disks"></a>Управляемые диски

**Что такое управляемые диски Azure?**

Управляемые диски — это функция автоматического управления учетными записями хранения, которая упрощает управление дисками виртуальных машин Azure IaaS. Дополнительные сведения см. в [обзоре управляемых дисков](storage-managed-disks-overview.md).

**Сколько будет стоить создание управляемого диска класса "Стандартный" из существующего виртуального жесткого диска объемом 80 ГБ?**

Управляемый диск класса "Стандартный", созданный на основе виртуального жесткого диска объемом 80 ГБ, будет считаться диском класса "Премиум" следующего доступного размера, то есть S10. С вас будет взиматься плата, соответствующая цене диска S10. Подробные сведения см. на [странице с ценами](https://azure.microsoft.com/pricing/details/storage).

**Взимается ли плата за транзакции для управляемых дисков класса "Стандартный"?**

Да, плата начисляется за каждую транзакцию. Подробности можно найти на [странице цен] (https://azure.microsoft.com/pricing/details/storage).

**При использовании управляемого диска класса "Стандартный" я буду платить за фактический объем данных на диске или за подготовленную емкость диска?**

Плата взимается за подготовленную емкость диска. Подробные сведения см. на [странице с ценами](https://azure.microsoft.com/pricing/details/storage).

**Насколько отличается цена управляемых дисков класса "Премиум" от цены неуправляемых дисков?**

Цены на управляемые и неуправляемые диски класса "Премиум" не отличаются.

**Могу ли я изменить тип учетной записи хранения ("Стандартный" или "Премиум") для управляемых дисков?**

Да. Тип учетной записи хранения для управляемых дисков можно изменить с помощью портала Azure, PowerShell или интерфейса командной строки Azure.

**Можно ли скопировать или экспортировать управляемый диск в личную учетную запись хранения?**

Да, вы можете экспортировать управляемые диски с помощью портала Azure, PowerShell или интерфейса командной строки Azure.

**Могу ли я использовать файл виртуального жесткого диска, размещенный в учетной записи хранения Azure, для создания управляемого диска в другой подписке?**

Нет.

**Могу ли я использовать файл виртуального жесткого диска, размещенный в учетной записи хранения Azure, для создания управляемого диска в другом регионе?**

Нет.

**Будут ли использоваться ограничения масштабирования для клиентов, использующих управляемые диски?**

Управляемые диски устраняют ограничения, связанные с учетными записями хранения. Но для одной подписки по умолчанию существует ограничение в 2000 управляемых дисков. Это значение можно увеличить, обратившись в службу поддержки.

**Могу ли я создать добавочный моментальный снимок управляемого диска?**

Нет. В текущей реализации функция моментальных снимков создает полную копию управляемого диска. В будущем мы планируем реализовать поддержку добавочных моментальных снимков.

**Можно ли использовать для виртуальных машин в группе доступности сочетание управляемых и неуправляемых дисков?**

Нет, все виртуальные машины в одной группе доступности должны использовать только управляемые или только неуправляемые диски. Тип дисков, которые вы хотите использовать, можно выбрать при создании группы доступности.

**Являются ли управляемые диски вариантом по умолчанию на портале Azure?**

Пока нет, но в будущем они будут использоваться по умолчанию.

**Могу ли я создать пустой управляемый диск?**

Да, пустой диск можно создать. Управляемый диск можно создать независимо от виртуальной машины, т. е. без подключения к ней.

**Сколько доменов сбоя поддерживается для группы доступности, использующей управляемые диски?**

Поддерживаемое количество доменов сбоя для группы доступности, использующей управляемые диски, — 2 или 3, в зависимости от региона размещения группы.

**Как настроить учетную запись хранения класса "Стандартный" для диагностики?**

Для диагностики виртуальной машины следует настроить частную учетную запись хранения. В будущем мы планируем также реализовать диагностику для управляемых дисков.

**Какого рода поддержка RBAC предоставляется для управляемых дисков?**

Для управляемых дисков поддерживаются три основные роли по умолчанию:

1.  Владелец — может управлять всем, включая права доступа.

2.  Участник — может управлять всем, кроме прав доступа.

3.  Читатель — может все просматривать, но не может вносить изменения.

**Можно ли скопировать или экспортировать управляемый диск в личную учетную запись хранения?**

Для управляемого диска можно получить подписанный URL-адрес (SAS) с доступом только для чтения, с помощью которого содержимое диска можно скопировать в личную учетную запись хранения или в локальное хранилище.

**Могу ли я создать копию управляемого диска?**

Клиенты могут создать моментальный снимок управляемого диска, а затем использовать этот снимок для создания нового управляемого диска.

**Поддерживаются ли еще неуправляемые диски?**

Да, сейчас мы поддерживаем и управляемые диски, и неуправляемые. Но при этом мы рекомендуем для всех новых рабочих нагрузок создавать управляемые диски и постепенно переносить на них все существующие рабочие нагрузки.

**Если я создам диск размером 128 ГБ, а затем увеличу размер до 130 ГБ, будет ли взиматься плата за следующий размер диска (512 ГБ)?**

Да.

**Могу ли я создать управляемые диски LRS, GRS или ZRS?**

Управляемые диски Azure сейчас поддерживают только локально избыточное хранилище (LRS).

## <a name="managed-disks-and-port-8443"></a>Управляемые диски и порт 8443

**Зачем клиентам нужно разрешать исходящий трафик через порт 8443 для виртуальных машин, использующих Управляемые диски Azure?**

Агент виртуальной машины использует порт 8443 для уведомления платформы Azure о состоянии каждого расширения виртуальной машины. Если этот порт не открыть, агент виртуальной машины не сможет сообщить о состоянии какого-либо расширения виртуальной машины. Дополнительные сведения об агенте виртуальной машины см. в разделе [Обзор агента виртуальной машины Azure](../virtual-machines/virtual-machines-windows-agent-user-guide.md).

**Что произойдет, если развернуть виртуальную машину с расширениями, но не открыть порт?**

Это приведет к ошибке развертывания. 

**Что произойдет, если развернуть виртуальную машину без расширений и не открыть порт?**

Это никак не повлияет на развертывание. 

**Что произойдет, если расширение устанавливается на виртуальную машину, которая уже подготовлена и работает, но на ней не открыт порт 8443?**

Расширение не будет успешно развернуто. Состояние расширения будет неизвестным. 

**Что произойдет, если используется шаблон ARM для подготовки нескольких виртуальных машин с закрытым портом 8443: одной виртуальной машины с расширениями и второй виртуальной машины, которая зависит от первой?**

Развертывание первой виртуальной машины завершится сбоем, так как развернуть расширения не удастся. Вторая виртуальная машина не будет развернута. 

**Требование открыть порт относится ко всем расширениям виртуальной машины?**

Да.

**Нужно ли разрешить входящие и исходящие подключения через порт 8443?**

Нет. Нужно разрешить только исходящие подключения через порт 8443. 

**Разрешить исходящие подключения через порт 8443 необходимо на все времени существования виртуальной машины?**

Да.

**Открытие этого порта повлияет на производительность виртуальной машины?**

Нет.

**Существует ли предполагаемая дата устранения этой проблемы, после чего мне больше не нужно будет открывать порт 8443?**

Да, это будет сделано к концу мая 2017 года.

## <a name="premium-disks--both-managed-and-unmanaged"></a>Управляемые и неуправляемые диски класса "Премиум"

**Если для виртуальной машины используется размер, который поддерживает хранилище класса Premium (например, DSv2), можно ли подключить к ней одновременно диски данных класса "Стандартный" и "Премиум"?** 

Да.

**Можно ли подключить диски данных класса "Стандартный" и "Премиум" к виртуальной машине, размер которой не поддерживает хранилище класса Premium (например, серии D, Dv2, G или F)?**

Нет. К виртуальным машинам, размер которых не поддерживает хранилище класса Premium, можно подключить только диски класса "Стандартный".

**Сколько будет стоить создание диска данных класса Premium из существующего виртуального жесткого диска, который имел размер 80 ГБ?**

Диск данных класса "Премиум", созданный на основе виртуального жесткого диска объемом 80 ГБ, будет считаться диском класса "Премиум" следующего доступного размера — P10. С вас будет взиматься плата, соответствующая цене диска P10.

**Применяется ли плата за транзакции при использовании хранилища класса Premium?**

Для каждого размера диска установлена фиксированная стоимость и определенные ограничения по пропускной способности и числу операций ввода-вывода в секунду. Дополнительная плата взимается только за пропускную способность для исходящих данных и емкость моментальных снимков (если применимо). Подробные сведения см. на [странице с ценами](https://azure.microsoft.com/pricing/details/storage).

**Какие ограничения установлены для дискового кэша по пропускной способности и операциям ввода-вывода в секунду?**

Совокупные ограничения для кэша и локального SSD-накопителя у серии DS составляют 4000 операций ввода-вывода в секунду на ядро и 33 МБ в секунду на ядро, а у серии GS — 5000 операций ввода-вывода в секунду на ядро и 50 МБ в секунду на ядро.

**Поддерживается ли локальный SSD-накопитель для виртуальных машин, использующих управляемые диски?**

Локальный SSD-накопитель представляет собой временное хранилище, которое входит в состав виртуальной машины, использующей управляемые диски. Никакая дополнительная плата за него не взимается. Мы рекомендуем не использовать локальный SSD-накопитель для хранения данных приложения, так как данные на нем не сохраняются в хранилище BLOB-объектов Azure.

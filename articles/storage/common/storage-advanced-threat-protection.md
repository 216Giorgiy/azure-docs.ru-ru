# <a name="azure-storage-advanced-threat-protection"></a>Расширенная защита от угроз для службы хранилища Azure

Расширенная защита от угроз для службы хранилища Azure позволяет выявлять аномалии в операциях учетной записи и получать уведомления о потенциально вредоносных попытках доступа к учетной записи. Этот уровень защиты позволяет устранять угрозы без необходимости в экспертных знаниях по безопасности или в управлении системами мониторинга безопасности.

Для выявления угроз определяются оповещения системы безопасности, которые активируются при возникновении аномальных действий. Эти оповещения интегрируются с [центром безопасности Azure](https://azure.microsoft.com/services/security-center/), который предоставляет сведения о подозрительных операциях и дает рекомендации о том, как определить причину угроз и устранить их. 

> [!NOTE]
> Расширенная защита от угроз для службы хранилища Azure в настоящее время доступна только для службы BLOB-объектов. Оповещения системы безопасности интегрированы с центром безопасности Azure и отправляются по электронной почте администраторам подписок.

Расширенная защита от угроз для службы хранилища Azure принимает журналы диагностики запросов на чтение, запись и удаление к службе BLOB-объектов, чтобы выявлять угрозы. Чтобы изучить оповещения от Расширенной защита от угроз, вам потребуется [настроить журналы диагностики](storage-monitor-storage-account.md#configure-logging) и включить все уровни журналов для службы BLOB-объектов.

## <a name="set-up-advanced-threat-protection-in-the-portal"></a>Настройка Расширенной защиты от угроз на портале

1. Запустите портал Azure: [https://portal.azure.com](https://portal.azure.com/).

2. Перейдите на страницу настройки учетной записи хранения Azure, которую требуется защитить. На странице **Параметры** выберите **Advanced Threat Protection**.

3. В колонке настройки **Advanced Threat Protection** сделайте следующее.
    * **Включите** функцию *Advanced Threat Protection*.
    * Щелкните **Сохранить**, чтобы сохранить новую или обновленную политику Расширенной защиты от угроз.

![Включение Расширенной защиты от угроз для службы хранилища Azure](./media/storage-advanced-threat-protection/storage-advanced-threat-protection-turn-on.png)

## <a name="explore-anomalies"></a>Изучение аномальных операций

При возникновении аномальных операций с хранилищем вы получите уведомление по электронной почте с информацией о подозрительном событии безопасности. Вы получите следующие сведения о событии:

* характер аномалии;
* имя учетной записи хранения;
* тип хранилища;
* время события.

Кроме того, в электронном сообщении приводится информация о возможных причинах, а также рекомендуемые действия по поиску и устранению потенциальной угрозы.

![Расширенная защита от угроз для службы хранилища Azure: оповещение по электронной почте](./media/storage-advanced-threat-protection/storage-advanced-threat-protection-alert-email.png)

Чтобы просматривать текущие оповещения системы безопасности из центра безопасности Azure и управлять ими, можно использовать колонку [Оповещения системы безопасности](../../security-center/security-center-managing-and-responding-alerts.md#managing-security-alerts). Если щелкнуть какое-либо оповещение, отобразятся сведения и действия, позволяющие изучить текущую угрозу и устранить будущие угрозы.

![Расширенная защита от угроз для службы хранилища Azure: оповещение по электронной почте](./media/storage-advanced-threat-protection/storage-advanced-threat-protection-alert.png)

## <a name="protection-alerts"></a>Оповещения о защите

Оповещения создаются при нестандартных и потенциально вредоносных попытках получить доступ к учетным записям хранения или использовать их. Эти события могут активировать создание следующих оповещений.

* **Доступ из непривычного расположения**. Это оповещение активируется при изменении шаблона доступа к учетной записи хранения. Например, если кто-то получил доступ к учетной записи хранения из непривычного географического расположения. Иногда предупреждение определяет допустимые действия (новое приложение или операцию обслуживания разработчиком). В других случаях оповещение указывает на угрозу (бывшего сотрудника, внешнего злоумышленника).

* **Необычное извлечение данных**. Это оповещение активируется при изменении шаблона извлечения данных из учетной записи хранения. Например, если кто-то получил доступ к необычному объему данных в учетной записи хранения. Иногда это оповещение указывает на допустимое действие (операцию обслуживания). В других случаях оно указывает на вредоносное действие (эксфильтрацию данных, брешь в системе безопасности или несанкционированную передачу данных).

* **Необычный анонимный доступ**. Это оповещение активируется при изменении шаблона доступа к учетной записи хранения. Например, если кто-то анонимно получил доступ к учетной записи хранения. В некоторых случаях это оповещение указывает на допустимый доступ с помощью общего доступа на чтение. В других случаях оно указывает на несанкционированный доступ, осуществляемый с помощью общего доступа на чтение к контейнеру и его большим двоичным объектам.

* **Непредвиденное удаление**. Это оповещение активируется в том случае, если в учетной записи хранения была выполнена одна или несколько непредвиденных операций, что определяется на основании анализа журнала этой учетной записи. Например, если кто-то выполнил операцию *DeleteBlob* с помощью нового приложения с нового IP- адреса. Иногда оповещение указывает на допустимые действие (администратор использовал другой браузер во время деловой поездки). В других случаях оповещения указывают на вредоносное действие (злоумышленника, удаляющего данные). 
 
* **Изменение прав доступа**. Это оповещение активируется при неожиданном изменении прав доступа к учетной записи хранения. Например, если кто-то изменил права доступа к учетной записи хранения с помощью нового приложения и с нового IP-адреса. Иногда оповещение указывает на допустимые действие (администратор использовал другой браузер во время деловой поездки). В других случаях оповещение указывает на вредоносное действие (например, злоумышленник повышает привилегии учетной записи, доступ к которой он получил). 

* **Передача пакета облачной службы Azure**. Это оповещение активируется при непредвиденной передаче пакета облачной службы Azure (*CSPKG*-файла) в учетную запись хранения. Например, если *CSPKG*-файл был передан с нового IP-адреса. В некоторых случаях оповещение указывает на допустимое действие. В других случаях оно указывает на вредоносное действие (например, пакет облачной службы был передан для подготовки к развертыванию вредоносной службы).

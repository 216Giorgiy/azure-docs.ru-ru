# <a name="azure-storage-service-encryption-for-data-at-rest"></a>Шифрование службы хранилища Azure для неактивных данных (предварительная версия)

Шифрование службы хранилища Azure для неактивных данных помогает защитить данные в соответствии с предложениями по безопасности и соответствию вашей организации. С помощью этой функции служба хранилища Azure автоматически шифрует данные перед их сохранением и расшифровывает данные перед извлечением. Процесс шифрования, шифрования при хранении, расшифровки и управления ключами в шифровании службы хранения является прозрачным для пользователей. При записи всех данных в службу хранилища Azure используется 256-битное [шифрование AES](https://en.wikipedia.org/wiki/Advanced_Encryption_Standard), самая надежная технология блочного шифрования на сегодняшний день.

Шифрование службы хранилища включено для всех новых и имеющихся учетных записей хранения. Его нельзя отключить. Так как данные защищены по умолчанию, вам не нужно изменять код и приложения, чтобы воспользоваться преимуществами шифрования службы хранилища.

Компонент автоматически шифрует данные в следующих расположениях:

- На обоих уровнях производительности ("Стандартный" и "Премиум").
- В обоих моделях развертывания (Azure Resource Manager и классической).
- Во всех службах хранилища Azure (хранилище BLOB-объектов, хранилище очередей, таблиц и файлов Azure). 

Шифрование службы хранения не влияет на производительность службы хранилища Azure.

Вы можете использовать ключи шифрования с шифрованием службы хранилища, управляемые корпорацией Майкрософт, или собственные ключи шифрования. Дополнительные сведения об использовании собственных ключей см. в статье [Шифрование службы хранилища с помощью управляемых клиентом ключей в Azure Key Vault](storage-service-encryption-customer-managed-keys.md).

## <a name="view-encryption-settings-in-the-azure-portal"></a>Просмотр параметров шифрования на портале Azure

Чтобы просмотреть настройки для шифрования службы хранения, войдите на [портал Azure](https://portal.azure.com) и выберите нужную учетную запись. В области **Параметры** выберите параметр **Шифрование**.

## <a name="faq-for-storage-service-encryption"></a>Часто задаваемые вопросы о шифровании службы хранилища

**Вопрос. У меня есть классическая учетная запись хранения. Могу ли я включить в ней шифрование службы хранилища?**

Ответ. Шифрование службы хранилища по умолчанию включено для всех учетных записей хранения (классических и учетных записей Resource Manager).

**Вопрос. Каким образом можно зашифровать данные в классической учетной записи хранения?**

Ответ. Если шифрование по умолчанию включено, служба хранилища Azure автоматически шифрует новые данные. 

**Вопрос. У меня есть учетная запись Resource Manager. Могу ли я включить в ней шифрование службы хранилища?**

Ответ. Шифрование службы хранилища по умолчанию включено для всех имеющихся учетных записей хранения Resource Manager. Данная возможность поддерживается для хранилищ BLOB-объектов, таблиц, очередей и файлов Azure. 

**Вопрос. Как зашифровать данные в учетной записи хранения Resource Manager?**

Ответ. Шифрование службы хранения включено по умолчанию для всех учетных записей хранения (классических и Resource Manager). Все файлы в учетной записи хранения, созданные до включения шифрования, будут зашифрованы ретроспективно с помощью фонового процесса шифрования.

**Вопрос. Можно ли создавать учетные записи хранения с включенной функцией шифрования службы хранилища с помощью Azure PowerShell и Azure CLI?**

Ответ. Шифрование службы хранилища по умолчанию включено во время создания любой учетной записи хранения (классической или учетной записи Resource Manager). Вы можете проверить свойства учетной записи с помощью Azure PowerShell и Azure CLI.

**Вопрос. Насколько дороже обходится служба хранилища Azure с включенным шифрованием службы хранилища?**

Ответ. Дополнительные затраты не требуются.

**Вопрос. Можно ли использовать собственные ключи шифрования?**

Ответ. Да, вы можете использовать собственные ключи шифрования. Дополнительные сведения см. в статье [Шифрование службы хранилища с помощью управляемых клиентом ключей в Azure Key Vault](storage-service-encryption-customer-managed-keys.md).

**Вопрос. Можно ли отменить доступ к ключам шифрования?**

Ответ. Да, если вы [используете собственные ключи шифрования](storage-service-encryption-customer-managed-keys.md) в Azure Key Vault.

**Вопрос. Включено ли шифрование службы хранилища Azure по умолчанию при создании учетной записи хранения?**

Ответ. Да, шифрование службы хранилища с использованием управляемых ключей Майкрософт включено по умолчанию для всех учетных записей хранения — классических учетных записей хранения и учетных записей Azure Resource Manager. Оно включено для всех служб, а также для хранилища BLOB-объектов, хранилища очередей, таблиц и файлов Azure.

**Вопрос. Чем это отличается от шифрования дисков Azure?**

Ответ. Шифрование дисков Azure применяется для шифрования дисков операционной системы и данных в виртуальных машинах IaaS. Дополнительные сведения см. в [руководстве по безопасности службы хранилища](../storage-security-guide.md).

**Вопрос. Что будет, если включить шифрование дисков Azure на дисках данных?**

Ответ. Это поддерживаемый режим работы. Данные будут шифроваться с помощью обоих методов.

**Вопрос. Для моей учетной записи хранения настроена геоизбыточная репликация. Если включить шифрование службы хранилища, будет зашифрована и избыточная копия?**

Ответ. Да, шифрованию подлежат все копии учетной записи хранения. Поддерживаются все варианты избыточности — локально избыточное хранилище, хранилище, избыточное в пределах зоны, геоизбыточное хранилище и геоизбыточное хранилище с доступом для чтения.

**Вопрос. Могу ли я отключить шифрование для учетной записи хранения?**

Ответ. Шифрование включено по умолчанию, и возможность выключения шифрования для вашей учетной записи не предусмотрена. 

**Вопрос. Шифрование службы хранилища разрешено только в определенных регионах?**

Ответ. Функция шифрования службы хранилища доступна для всех служб во всех регионах. 

**Вопрос. Куда обратиться, чтобы задать вопрос или оставить отзыв?**

Ответ. По любым вопросам, связанным с шифрованием службы хранилища, или чтобы оставить отзыв обращайтесь по адресу [ssediscussions@microsoft.com](mailto:ssediscussions@microsoft.com).

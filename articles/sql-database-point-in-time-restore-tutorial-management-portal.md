# Восстановление базы данных SQL Azure с использованием восстановления на момент времени на портале Azure

> [AZURE.SELECTOR]
- [Восстановление на момент времени - PowerShell](http://azure.microsoft.com/documentation/articles/sql-database-point-in-time-restore-tutorial-powershell/)
- [Восстановление на момент времени - API REST](http://azure.microsoft.com/documentation/articles/sql-database-point-in-time-restore-tutorial-rest/) 

## Обзор

В этом учебнике показано, как восстановить базу данных SQL Azure с помощью восстановления на момент времени на [портале Azure](http://manage.windowsazure.com/). В базе данных SQL Azure есть встроенные функции резервного копирования для поддержки самостоятельного восстановления на определенный момент времени для уровней обслуживания Basic, Standard и Premium.

При восстановлении на момент времени создается новая база данных. Служба автоматически выбирает уровень обслуживания на основе резервной копии, используемой в точке восстановления. Убедитесь, что на логическом сервере есть доступная квота для создания еще одной базы данных. Если вы хотите запросить более высокую квоту, обратитесь в [службу поддержки Azure](http://azure.microsoft.com/support/options/).

## Ограничения и безопасность

* Функция восстановления на момент времени включена для уровней Basic, Standard и Premium.

* На уровне Basic резервные копии хранятся в течение периода в 7 дней, на уровне Standard - 14 дней, а на уровне Premium - 35 дней.
 
* Только администратор или соадминистратор подписки может отправлять запросы на восстановление.

* Имя для входа субъекта серверного уровня будет соответствовать владельцу восстановленной базы данных.

* Уровни обслуживания выпусков Web и Business не поддерживают восстановление на момент времени.
 
	* При наличии базы данных выпуска Web или Business вы можете использовать копию базы данных для получения такой копии базы данных, в которой будут содержаться транзакции, а затем экспортировать скопированную базу данных в учетную запись хранения Microsoft Azure. Дополнительные сведения см. в разделе [Практическое руководство. Использование копии базы данных (базы данных SQL Azure)](http://msdn.microsoft.com/library/azure/ff951631.aspx) и [Инструкции по использованию службы импорта и экспорта в базе данных Azure SQL](http://msdn.microsoft.com/library/azure/hh335292.aspx).

	* В сентябре 2015 года поддержка выпусков Web и Business будет прекращена. Дополнительную информацию см. в разделе [Вопросы и ответы по остановке версий Web и Business](http://msdn.microsoft.com/library/azure/dn741330.aspx).

## Практическое руководство. Восстановление базы данных SQL Azure с использованием восстановления на момент времени на портале Azure

<iframe src="http://channel9.msdn.com/Blogs/Windows-Azure/Restore-a-SQL-Database-Using-Point-in-Time-Restore/player" width="960" height="540" allowFullScreen frameBorder="0"></iframe>

1. Войдите на портал Azure с помощью учетной записи Майкрософт.

2. В левой области навигации щелкните **Базы данных SQL**.
  
3. В списке **БАЗЫ ДАННЫХ** выберите базу данных, которую нужно восстановить. 

4. В нижней части страницы на панели команд щелкните **Восстановление**. Это откроет диалоговое окно **Указать параметры восстановления**.

5. Выберите имя в поле **ИМЯ БАЗЫ ДАННЫХ**. По умолчанию имя базы данных выбирается автоматически, но при желании его можно изменить.

6. Выберите момент времени, к которому нужно восстановить базу данных. Момент времени должен быть в пределах срока хранения базы данных.
	
7. Установите флажок, чтобы отправить запрос на восстановление.

Восстановление может занять некоторое время. Для мониторинга состояния восстановления щелкните значок "Состояние" в нижнем правом углу страницы на панели команд и нажмите кнопку **ПОДРОБНОСТИ**.

# Заметки о выпуске фабрики данных Azure

## Заметки о выпуске 10/04/2015 фабрики данных
Вы увидите **недавно обновлены фрагменты** и **недавно произошел сбой фрагменты** перечислены на **таблицы** теперь выноски. Эти списки сортируются по времени обновления фрагмента. Время обновления фрагмента изменяется в следующих ситуациях.

-  Обновить состояние срез вручную, например, с помощью **набора AzureDataFactorySliceStatus** (или), щелкнув **ЗАПУСКА** на **СРЕЗ** выноски для диаграммы.
-  Срез изменяет состояние из-за выполнения (например выполнения работы, запуска завершается, а не удалось, запуска завершено и выполнена успешно, и т. д).

Щелкните заголовок списки или **... (многоточие)** Чтобы просмотреть список больших фрагментов. Щелкните **фильтра** на панели инструментов для фильтрации фрагментов.
 
Можно просматривать фрагменты, отсортированных по времени срез, щелкнув **срезы данных (по времени срез)** плитки. Фрагменты в этих коллекциях упорядочиваются по времени фрагмента. Например, если ежечасное расписание срезы будет: - 4/4/2015 17: 00, выполняется - 4, 4, 4 и 2015 pm успешно - 3, 4, 4 и 2015 pm сбой

Но если старые срез запускается повторно, он бы не отображаются поверх этого списка, несмотря на то, что это, возможно, что пользователь является более заинтересованы в.

	-  [Oracle локальной связанные службы](https://msdn.microsoft.com/library/dn948537.aspx)

## Заметки о выпуске 27/2/2015 фабрики данных

### Улучшения
- **Azure фабрики редактора данных**. Фабрика редактора данных, которая является частью предварительной версии портала Azure, дает возможность создания, изменения и развертывания файлов JSON, которые определяют связанные службы, наборы данных и конвейеры. Основной целью редактора является предоставляют быстрый и легкий пользовательского интерфейса (UI) для создания артефактов фабрика данных Azure без необходимости установки Azure PowerShell и увеличения вверх на командлетов PowerShell. См. [Azure данных фабрики редактора - свет вес Web][adf-editor-blog] блога краткий обзор и видео на данных фабрики редактора. Подробный обзор редактора в разделе [данных фабрики редактора][adf-editor] статьи.

## Заметки о выпуске 26/1/2015 фабрики данных ##

### изменения
- Обновление **Шлюз управления данными** отправляется пакет установки [центра загрузки Майкрософт][adf-gateway-download]. Начиная с этого выпуска можно найти последнюю шлюз управления данными для использования с фабрикой данных Azure в этом расположении загрузки. Этот установочный пакет служит фабрика данных Azure и Power BI для Office 365 служб. При использовании и службы Обратите внимание, что шлюзов для фабрики данных и Power BI должен быть установлены на разных компьютерах настроены по-разному согласно рекомендации по документации фабрика данных или Power BI.
-  **Действие копирования** теперь поддерживает копирование данных между локальной базы данных SQL Server и базы данных Azure SQL. В разделе [Действие копирования][adf-copy-activity] подробные сведения и [GitHub][adf-github-samples] образцы JSON.
- **SqlSink** поддерживает новое свойство: **WriteBatchTimeout**. Это свойство обеспечивает гибкость при настройке время ожидания для операции вставки пакета для завершения операции время ожидания. Для копирования гибридное (операции копирования, которая включает в себя на локальный источник данных и источник данных облака) необходимо иметь шлюза версии 1.4 или более поздней версии, с помощью этого свойства. 
- **SQL Server связанные службы** теперь поддерживает **Проверка подлинности Windows**. 
	- При создании SQL Server связанные службы на портале, теперь можно использовать проверку подлинности Windows и задайте соответствующие учетные данные. Это требуется шлюз 1.4 или более поздней версии. 
	- При создании SQL Server связанные службы с помощью Azure PowerShell, можно указать сведения о соединении в виде простого текста или зашифровать сведения соединения с помощью обновления [командлет New-AzureDataFactoryEncryptValue][adf-encrypt-value-cmdlet] и затем использовать зашифрованную строку для свойства строки подключения в полезных данных JSON связанной службы. В разделе [связанные службы][adf-msdn-linked-services] подробные сведения об определении связанного службы в формате JSON. Средства шифрования не поддерживается с помощью командлета New-AzureDataFactoryEncryptValue еще. 

## Заметки о выпуске 12/11/2014 фабрики данных ##

### Улучшения

- Интеграция машинного обучения Azure
	- Этой версии службы фабрики данных Azure позволяет интегрировать с Azure машины обучения (ML) фабрика данных Azure с помощью **AzureMLLinkedService** и **AzureMLBatchScoringActivity**. В разделе [создания прогнозирующего конвейеры, с помощью фабрики данных и Azure машинного обучения][adf-azure-ml] подробные сведения. 
- Предоставляется состояние версии шлюза
	- На портале предварительной версии Azure будет отображаться состояние NewVersionAvailable, а в выходных данных командлета Get-AzureDataFactoryGateway будут сведения о наличии более новой версии шлюза. Вы можете продолжить изучение портала, загрузить новый установочный файл (.msi) и запустить его для установки последнего шлюза. Дополнительные настройки для этого не требуются.

### изменения

- Удален контейнер JobsContainer из службы HdInsightOnDemandLinkedService.
	- В определении JSON для HDInsightOnDemandLinkedService не необходимо указать **jobsContainer** свойство больше. Если для связанной службы по запросу уже задано это свойство, оно будет игнорироваться. Вы можете удалить свойство из определения JSON для связанной службы и обновить определение связанной службы с помощью командлета New-AzureDataFactoryLinkedService.
- Дополнительные параметры конфигурации для HDInsightOnDemandLinkedService
	- В этом выпуске мы добавлена поддержка нескольких дополнительных параметров конфигурации для HDInsightOnDemandLinked (кластера HDInsight по запросу). В разделе [Свойства ClusterCreateParameters][on-demand-hdi-parameters] подробные сведения.
- Удалено расположение шлюза
	- При создании шлюза фабрики данных Azure через портал или с помощью PowerShell (New-AzureDataFactoryGateway) теперь не требуется указывать расположение шлюза. Будет наследоваться область фабрики данных. Таким же образом, больше не требуется настраивать связанную службу сервера SQL Server с помощью свойства JSON "gatewayLocation". Пакет .NET SDK фабрики данных также обновлен для отражения этих изменениями.
	- При использовании более ранней версии SDK и Azure PowerShell вам по-прежнему придется выполнять настройку расположения.
 
     

#### Критические изменения
	
- Действие CustomActivity переименовано в DotNetActivity
	- **ICustomActivity** интерфейс переименовывается в **IDotNetActivity**. Вам потребуется обновить пакеты NuGet для Data Factory и заменить строку ICustomActivity на IDotNetActivity в исходном коде пользовательского действия.  
	- Тип настраиваемого действия в определении JSON для настраиваемого действия должно быть изменено с **CustomActivity** для **DotNetActivity**. 
	-  **CustomActivity** и **CustomActivityProperties** классы были переименованы в **DotNetActivity** и **DotNetActivityProperties** с тем же набором свойств.

		В более ранней версии SDK и Azure PowerShell можно по-прежнему использовать CustomActivity вместо DotNetActivity.
    
  		В разделе [использовать настраиваемые действия в конвейер данных фабрика Azure][adf-custom-activities] Пошаговое руководство о том, как создать пользовательское действие и использовать его в конвейер фабрика данных Azure.

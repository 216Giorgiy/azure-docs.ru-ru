# Что такое Azure Application Insights?

Visual Studio Application Insights позволяет отслеживать производительность и использование активного веб-приложения или приложения на устройстве.

* [Обнаруживайте, рассматривайте и диагностируйте][detect] проблемы производительности и устраняйте их, прежде чем большинство пользователей выявит их.
 *  Предупреждения об изменениях производительности или сбоях.
 *  Метрики, которые помогут диагностировать проблемы производительности, например время отклика, использование ЦП, отслеживание зависимостей.
 *  Тесты доступности для веб-приложений.
 *  Отчеты и уведомления о сбоях и исключениях.
 *  Эффективный поиск журналов диагностики (включая трассировки журналов на предпочитаемой платформе ведения журналов).
* [Постоянно совершенствуйте свое приложение][knowUsers], зная о том, что делают пользователи в этом приложении. 
 * Счетчики количества просмотров страниц, новые пользователи и пользователи, вернувшиеся на страницу, и другие основные аналитические данные по использованию.
 * Отслеживание событий для оценки закономерностей использования и успешности выполнения каждой функции.

## Как это работает?

Вы устанавливаете небольшой пакет SDK в приложение и настраиваете учетную запись на портале Application Insights. Пакет SDK отслеживает приложение и отправляет данные телеметрии на портал. Портал отображает статистические диаграммы и предоставляет мощные средства, чтобы помочь вам диагностировать любые проблемы.

![Пакет SDK Application Insights в приложении отправляет телеметрию в ресурс Application Insights на портале Azure.](./media/app-insights-overview/01-scheme.png)

Пакет SDK содержит несколько модулей, собирающих телеметрию, например для подсчета пользователей, сеансов и производительности. Вы также можете написать собственный код для отправки данных телеметрии на портал. Пользовательская телеметрия особенно полезна для трассировки пользовательской Истории. Вы можете вести учет событий, таких как число нажатий кнопки, достижение конкретных целей или ошибки пользователя.

Для серверов ASP.NET и веб-приложений Azure можно также установить [монитор состояний][redfield], который используется в двух целях. Он позволяет:

* выполнять мониторинг веб-приложения без необходимости повторного создания или повторной установки;
* отслеживать вызовы зависимых модулей.

## С какими приложениями работает служба?

В настоящее время существуют пакеты SDK для:

* веб-приложений:
 * [ASP.NET][greenbrown] на Azure или сервере IIS;
 * [Java][java] на JRE; 
 * [веб-страницы][client] — HTML и JavaScript.
* приложений на устройствах:
 * [Windows][windows]
 * [iOS][ios]
 * [Android][android]
 * Cordova;
 * [другие платформы][platforms].

## Что мне нужно для использования службы?

* Подписка Microsoft Azure. Application Insights — одна из многих облачных служб Azure, которые также включают веб-сайты, базы данных, виртуальные машины и т. д. (Но Application Insights можно использовать для мониторинга любых приложений. Приложение не должно выполняться в Azure.) 

 * Организация может иметь учетную запись.


## Как начать работу?

Выберите свою платформу в меню «Начало работы» слева.

Во всех случаях основная процедура выполняется следующим образом:

1. Создайте ресурс Application Insights в [Azure][portal] (и получите ключ инструментирования для него).
2. Добавьте в приложение возможность работы с соответствующим пакетом SDK (и настройте его с помощью ключа инструментирования).
3. Запустите приложение в режиме отладки или в динамическом режиме.
4. Просмотрите результаты в ресурсе в [Azure][portal].

В некоторых случаях для IDE доступен подключаемый модуль (такой как Visual Studio или Eclipse), выполняющий первые два шага автоматически. Но вы всегда можете выполнить процедуру вручную.

Если приложение является веб-узлом или службой, базовая процедура для него немного отличается:

* Добавьте пакет SDK в серверное приложение и клиентское [устройство][windows] или на [веб-страницу][client]. Данные телеметрии объединяются на портале, чтобы можно было сопоставлять события на двух концах.
* Настройте веб-тесты для отслеживания доступности сайта из разных точек по всему миру.
* Инструментируйте активное серверное приложение без повторного создания или развертывания. Это можно выполнить для [серверов IIS][redfield] и [веб-приложений Azure][azure].
* Отслеживайте вызовы зависимостей приложением из других компонентов, таких как базы данных, или с помощью интерфейсов REST API. Это можно выполнить для [серверов IIS][redfield] и [веб-приложений Azure][azure].

# <a name="tutorial-create-and-manage-exported-data"></a>Руководство. Создание экспортированных данных и управление ими

Если вы изучили учебник по анализу затрат, вы знакомы с процедурой загрузки данных из системы управления затратами вручную. Тем не менее можно создать ежедневно повторяющуюся задачу, которая будет каждый день автоматически экспортировать ваши данные из системы управления затратами в хранилище Azure. Экспортированные данные представлены в формате CSV и содержат все сведения, собираемые системой управления затратами. Затем можно использовать экспортированные данные в хранилище Azure в работе с внешними системами и объединять их с собственными пользовательскими данными. Кроме того, можно использовать экспортированные данные во внешней системе, например на панели мониторинга или в другой финансовой системе.

Примеры в этом руководстве по шагам демонстрируют экспорт ваших данных из системы управления затратами, а затем проверку успешности экспорта данных.

Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> * Создание ежедневного экспорта
> * Проверка сбора данных

## <a name="prerequisites"></a>Предварительные требования

- Учетная запись Azure EA
- Учетная запись хранилища Azure, настроенная для хранилища BLOB или файлового хранилища

## <a name="sign-in-to-azure"></a>Вход в Azure
Войдите на портал Azure по адресу [https://portal.azure.com](https://portal.azure.com/).

## <a name="create-a-daily-export"></a>Создание ежедневного экспорта

Управление затратами + выставление счетов&gt; выберите подписку или группу ресурсов в подписке &gt; Экспорт &gt; **Добавить**.

Введите имя для экспорта и укажите подписку, учетную запись хранилища Azure, контейнер и каталог хранения файлов или BLOB-контейнер, затем нажмите кнопку **Создать**.

![Новый экспорт](./media/tutorial-export-acm-data/new-export01.png)

Новый экспорт отобразится в списке экспорта. По умолчанию новые экспорты включены, затем они выполняются ежедневно. Если требуется отключить или удалить запланированный экспорт, щелкните любой пункт в списке и выберите **Выключить** или **Удалить**.

Изначально до начала экспорта может потребоваться 1–2 часа. Однако прежде чем данные отобразятся в экспортированных файлах, может пройти до четырех часов.

## <a name="verify-that-data-is-collected"></a>Проверка сбора данных

Вы легко можете убедиться, что ваши данные из системы управления расходами были собраны, и просмотреть экспортированный CSV-файл с помощью обозревателя службы хранилища Azure.

В списке экспорта щелкните имя учетной записи хранения. На странице учетной записи хранения в проводнике нажмите "Открыть". Если появится поле подтверждения, нажмите кнопку **Да**, чтобы открыть файл в обозревателе службы хранилища Azure.

![Страница учетной записи хранения](./media/tutorial-export-acm-data/storage-account-page.png)

В обозревателе службы хранилища перейдите в контейнер, который требуется открыть, и выберите папку, соответствующую текущему месяцу. Отобразится список CSV-файлов. Выберите файл и нажмите **Открыть**.

![Обозреватель хранилищ](./media/tutorial-export-acm-data/storage-explorer.png)

Файл открывается с помощью программы или приложения, которое задано для открытия файлов с расширением CSV. Ниже приведен пример в Excel.

![Пример экспорта данных](./media/tutorial-export-acm-data/example-export-data.png)

## <a name="access-exported-data-from-other-systems"></a>Доступ к экспортированным данным из других систем

Одной из целей экспорта данных системы управления затратами является возможность доступа к данным из внешних систем. Можно использовать систему панелей мониторинга или другую финансовую систему. Такие системы значительно варьируются, поэтому приводить примеры нецелесообразно.  Однако для начала можно осуществлять доступ к данным из ваших приложений, следуя инструкциям в разделе [Вводные сведения о службе хранилища Azure](../storage/common/storage-introduction.md).

# Смена клиента Azure Active Directory в RemoteApp

RemoteApp использует Azure Active Directory (Azure AD), чтобы разрешить доступ пользователям. Единственный клиент Azure AD, которого можно использовать, - тот, который связан с подпиской Azure. Вы можете просмотреть связанную подписку на странице "Параметры" на портале. Откройте столбец "Каталог" на вкладке "Подписки". 

Если вы хотите использовать другого клиента, выполните следующие действия, чтобы изменить связь с подпиской.

1. На портале удалите всех пользователей Azure AD, которым предоставили доступ к службам RemoteApp.


2. Укажите учетную запись Майкрософт (прежнее название - Live ID) администратора служб. (См. **Параметры -> Администраторы**.)
	1. Выберите выполнившего вход пользователя в правом верхнем углу и щелкните **Просмотреть мой счет**.
	2. Выберите свою подписку и щелкните **Изменить сведения о подписке**.
	3. Внесите нужные изменения.



3. Выйдите из портала и снова войдите с использованием учетной записи Майкрософт, указанной в предыдущем шаге.


4. Последовательно щелкните **Создать -> Службы приложений -> Active Directory -> Настраиваемое создание -> Использовать существующий каталог**. Добавьте клиента Azure AD, которого следует связать с этой подпиской.


5. В меню **Параметры -> Подписки**, выберите свою подписку и щелкните **Изменить каталог**. Выберите клиента Azure AD, которого хотите использовать.



Теперь вы можете использовать нового клиента Azure AD для контроля доступа к подписке Azure и настройки доступа пользователей в RemoteApp.

<h1 id="">Безопасность Operational Insights</h1>


Корпорация Майкрософт заботится о вашей конфиденциальности и безопасности ваших данных, выпуская программное обеспечение и службы, которые помогают вам управлять ИТ-инфраструктурой своей организации. Мы понимаем, что когда вы доверяете свои данные сторонним организациям, они требуют надежной защиты. Корпорация Майкрософт следует строгим нормативным требованиям и указаниям по безопасности - от создания кода до эксплуатации служб.

Защита данных - главный приоритет корпорации Майкрософт. Отправляйте свои вопросы, предложения и сообщения о неполадках, связанные с представленными ниже сведениями, включая наши политики защиты, по следующему адресу: <a href="mailto:scdata@microsoft.com" target="_blank">scdata@microsoft.com</a>.

В этой статье описываются сбор, обработка и защита данных в службе Microsoft Azure Operational Insights. Вы можете либо использовать агенты для непосредственного подключения к веб-службе, либо использовать службу System Center Operations Manager, чтобы собирать рабочие данные для службы Operational Insights. Собранные данные отправляются по Интернету в службу Operational Insights, которая размещается в Microsoft Azure.

Служба Operational Insights безопасно управляет данными с использованием следующих методов:

**Разделение данных.** Данные клиентов логическим образом отделяются для каждого компонента службы Operational Insights. Все данные отмечаются по организациям. Эти метки существуют в течение всего жизненного цикла данных и используются на каждом уровне службы. 

У каждого клиента есть выделенный большой двоичный объект Azure для долгосрочного хранения данных. Этот объект шифруется уникальными ключами клиентов, которые меняются каждые 90 дней.

**Хранение данных** Собранные метрики для каждого информационного пакета хранятся в базе данных SQL, которая размещается в Microsoft Azure. Эти данные хранятся в течение 390 дней. Индексированные данные поиска хранятся в течение примерно 10 дней, после чего очищаются. Если верхний предел (20 миллионов для каждого типа данный) достигается раньше, служба Operational Insights очищает данные, которые хранятся дольше 10 дней. Если предел данных не достигается в течение 10 дней, служба Operational Insights не очищает данные, пока предел не будет достигнут.

**Физическая защита** Службой Operational Insights управляет персонал корпорации Майкрософт, а все действия записываются в журнал и доступны для аудита. Служба Operational Insights полностью выполняется в Azure и соответствует общим условиям проектирования Azure. Вы можете просмотреть сведения о физической защите активов Azure на странице 18  <a href="http://download.microsoft.com/download/6/0/2/6028B1AE-4AEE-46CE-9187-641DA97FC1EE/Windows%20Azure%20Security%20Overview%20v1.01.pdf" target="_blank">обзора безопасности Microsoft Azure</a>.

**Нормативные требования и сертификация** Группа разработки и обслуживания Operational Insights активно сотрудничает с юридическим отделом и группой обеспечения соответствия требованиям Майкрософт, включая стандарт ISO, перед выпуском службы Operational Insights в свободный доступ.

В настоящий момент соблюдаются следующие стандарты безопасности:

- Общие условия проектирования для Windows
- Сертификация защиты информационных систем Майкрософт

# Глобальный веб-уровень с балансировкой нагрузки (шаблоны архитектуры Azure)

[Инфраструктура интеграции облачных платформ (CPIF)](azure-architectures-cpif-overview.md) предоставляет инструкции по интеграции рабочей нагрузки для адаптации приложений в облачном решении корпорации Майкрософт.

Инфраструктура интеграции облачных платформ описывает способы, которые следует использовать организациям, клиентам и партнерам для разработки и развертывания предназначенных для облака рабочих нагрузок с помощью гибридной облачной платформы и возможностей управления Azure, System Center и Windows Server.

Шаблон **глобального веб-уровня с балансировкой нагрузки** является частью области **основания**, подробно описанной в документе об архитектуре CPIF.

##  Глобальный веб-уровень с балансировкой нагрузки

Шаблон разработки глобального веб-уровня с балансировкой нагрузки содержит сведения о компонентах и службах Azure, необходимых для предоставления услуг веб-уровня, обеспечивающих предсказуемый уровень производительности и высокую доступность во всех географических регионах.

В этом шаблоне разработки веб-уровень определен как уровень обслуживания, предоставляющий традиционное содержимое HTTP/HTTPS или службы приложений изолированным образом или в виде части многоуровневого веб-приложения. В шаблоне балансировка нагрузки веб-уровня выполняется и локально внутри региона, и между регионами. С точки зрения вычислений эти службы можно предоставлять через виртуальные машины Microsoft Azure, веб-сайты или обоими способами. Виртуальные машины, предоставляющие веб-службы, могут размещать контент с помощью любого поддерживаемого дистрибутива гостевой операционной системы Microsoft Windows или Linux в рамках Microsoft Azure.


## Обзор шаблона архитектуры 

В этом документе описывается шаблон для предоставления доступа к веб-службам или содержимому веб-сервера в разных географических регионах для обеспечения доступности и избыточности. Критически важные службы приведены ниже без описания ограничений веб-платформы или методологии разработки в рамках самой веб-службы. Существует два варианта для этого шаблона: первый размещает веб-содержимое или службы на виртуальных машинах (с помощью поддерживаемых Azure операционных систем и семейств), а второй использует веб-сайты Azure. В приведенной ниже схеме наглядно показаны значимые службы и способы их использования в качестве части шаблона на примере виртуальных машин.

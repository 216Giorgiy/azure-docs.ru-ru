# Мониторинг учетной записи DocumentDB

Мониторинг учетных записей DocumentDB можно осуществлять на [портале предварительной версии Azure][портале предварительной версии Azure]. Для каждой учетной записи DocumentDB доступны метрики производительности (такие как запросы и ошибки сервера) и использования (такие как потребление памяти).

## <span id="metrics"></span></a>Практическое руководство. Просмотр метрик производительности для учетной записи DocumentDB

1.  На [портале предварительной версии Azure][портале предварительной версии Azure] нажмите кнопку **Обзор**, выберите пункт **Учетные записи DocumentDB**, а затем щелкните имя учетной записи DocumentDB, для которой требуется просмотреть метрики производительности.
2.  В области **мониторинга** по умолчанию можно просмотреть следующее:

    -   общее число запросов для текущего дня;
    -   среднее число запросов в секунду для текущего дня.

    ![][0]

3.  Если щелкнуть раздел **общего числа запросов или среднего числа запросов в секунду**, откроется подробная выноска **Метрика**.
4.  Выноска "Метрика" содержит сведения о выбранных метриках. В верхней части выноски находится график, а под ним — таблица статистических показателей для этих метрик, таких как среднее, минимальное и максимальное значения. На выноске "Метрика" также отображается список заданных оповещений, отфильтрованных по метрикам, отображающимся в текущей выноске "Метрика" (таким образом при наличии большого числа оповещений вы увидите только те, которые относятся к представленным здесь метрикам).

    ![][1]

## <span id="custom"></span></a>Практическое руководство. Настройка представлений метрик производительности для учетной записи DocumentDB

1.  Чтобы настроить метрики, отображающиеся в конкретном разделе, щелкните правой кнопкой мыши диаграмму метрики и выберите пункт **Изменить диаграмму**.
    ![][2]

2.  На выноске **Изменение диаграммы** можно изменить метрики, которые отображаются в соответствующем разделе, а также их временные диапазоны.
    ![][3]

3.  Чтобы изменить отображаемые метрики, просто установите или снимите флажки доступных метрик производительности и нажмите кнопку **Сохранить** внизу выноски.
4.  Чтобы изменить временной диапазон, выберите другой диапазон (например, **Последний час**) и нажмите кнопку **Сохранить** внизу выноски.
    ![][4]

5.  Для настраиваемого временного диапазона можно выбрать любой период времени в рамках последних 2 недель.
6.  При нажатии кнопки **Сохранить** изменения сохраняются до момента выхода из выноски "Учетная запись DocumentDB". При последующем открытии этой выноски будут отображаться первоначальные метрики и диапазон времени.

## <span id="create"></span></a>Практическое руководство. Создание параллельных диаграмм

На портале предварительной версии Azure можно создавать параллельные диаграммы метрик.

1.  Сначала щелкните правой кнопкой мыши диаграмму, с которой планируется начать, и выберите пункт **Настроить**.
    ![][5]

2.  Выберите в меню пункт **Клонировать**, чтобы скопировать раздел.

    ![][6]

3.  Наконец, нажмите кнопку **Готово** в панели инструментов вверху экрана. Теперь этот раздел можно использовать как любой другой раздел метрик, настраивая отображаемые в этом разделе метрики и диапазон времени. Сделав это, можно просматривать две разные диаграммы метрик параллельно в одно и то же время.
    ![][7]

> Обратите внимание, что диапазон времени и выбранные метрики диаграммы сбрасываются в значения по умолчанию при выходе из портала предварительной версии Azure.

## <span id="view"></span></a>Практическое руководство. Просмотр метрик использования для учетной записи DocumentDB

1.  На [портале предварительной версии Azure][портале предварительной версии Azure] нажмите кнопку **Обзор**, выберите пункт **Учетные записи DocumentDB**, а затем щелкните имя учетной записи DocumentDB, для которой требуется просмотреть метрики использования.
2.  В области **использования** по умолчанию можно просмотреть следующее:

    -   объем хранения, потребленный в учетной записи;
    -   максимально доступный объем хранения для учетной записи;
    -   использование вложений;
    -   использование пользователей и разрешений;
    -   выделение единиц мощности;
    -   использование вложений в учетной записи.
        ![][8]

## <span id="setup"></span></a>Практическое руководство. Настройка оповещений метрик производительности для учетной записи DocumentDB

1.  На [портале предварительной версии Azure][портале предварительной версии Azure] нажмите кнопку **Обзор**, выберите пункт **Учетные записи DocumentDB**, а затем щелкните имя учетной записи DocumentDB, для которой требуется просмотреть оповещения метрик производительности.
2.  В области **операций** щелкните раздел **Правила оповещений**.
    ![][9]

3.  На выноске правил оповещений нажмите кнопку **Добавить оповещение**.
    ![][10]

4.  На выноске **добавления правила оповещения** укажите следующее:

    -   имя настраиваемого правила оповещения;
    -   описание нового правила оповещения;
    -   метрику для правила оповещения;
    -   условие, пороговое значение и период для активации оповещения (например, более 5 ошибок сервера за последние 15 минут);
    -   следует ли оповещать администратора службы и соадминистраторов по электронной почте при срабатывании оповещения;
    -   дополнительные адреса электронной почты для уведомлений об оповещениях.
        ![][11]

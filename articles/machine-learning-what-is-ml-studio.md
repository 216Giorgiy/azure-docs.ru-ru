# Что такое Студия машинного обучения Azure?

Студия машинного обучения Microsoft Azure — это среда совместной визуальной разработки, позволяющая разрабатывать, тестировать и развертывать решения прогнозной аналитики, работающие с вашими данными. Среда разработки и обслуживания машинного обучения является облачной, обеспечивает гибкость вычислительных ресурсов и памяти, а также исключает проблемы установки и настройки, поскольку работа осуществляется через веб-браузер.

Студия машинного обучения Microsoft Azure — это место, где пересекаются обработка и анализ данных, прогнозная аналитика, облачные ресурсы и данные клиента!

## Интерактивное рабочее пространство Студии машинного обучения Microsoft Azure

Для разработки модели прогнозной аналитики обычно используются данные из одного из нескольких источников, которые преобразуются и анализируются с помощью разных манипуляций и статистических функций, чтобы получить ряд результатов. Разработка такой модели является итерационным процессом — путем изменения разных функций и их параметров выполняется сведение результатов, пока не будет получена подготовленная эффективная модель.

**Студия машинного обучения Microsoft Azure** предоставляет интерактивное визуальное рабочее пространство для облегчения построения, тестирования и выполнения итераций модели прогнозной аналитики. Вы перетаскиваете ***наборы данных*** и ***модели***
анализа на интерактивный ***холст*** и связываете их вместе, чтобы сформировать ***эксперимент***, который затем ***отправляется*** в Студию машинного обучения Microsoft Azure для выполнения. Для выполнения итераций в макете вы ***редактируете*** эксперимент, при необходимости ***сохраняете*** копию и отправляете его снова.

Никакое программирование не требуется, выполняется только визуальное соединение наборов данных и модулей, чтобы создать модель прогнозной аналитики.

## Начало работы со Студией машинного обучения Microsoft Azure

При первом входе в Студию машинного обучения Microsoft Azure слева появляются следующие вкладки.

-   **ЭКСПЕРИМЕНТЫ** — эксперименты, которые создаются, запускаются или сохраняются как черновики.
-   **ВЕБ-СЛУЖБЫ** — список опубликованных экспериментов.
-   **МОДУЛИ** — список ссылок на модули аналитики, доступные в Студии машинного обучения Microsoft Azure.
-   **НАБОРЫ ДАННЫХ** — список ссылок на наборы данных, доступные в Студии машинного обучения Microsoft Azure.
-   **ПАРАМЕТРЫ** — набор параметров, которые можно использовать для настройки учетной записи и ресурсов.

> [WACOM.NOTE] Списки **модулей** и **наборов данных**, которые отображаются на этих вкладках, служат только для справки. Эти списки включают дополнительные сведения о наборах данных и модулях, включая время создания и номера версий.
> При создании эксперимента рабочий список доступных наборов данных и модулей отображается на холсте слева. Это список компонентов, которые используются для построения модели.

## Компоненты эксперимента

Эксперимент состоит из наборов данных, предоставляющих данные для модулей аналитики, которые соединяются вместе для создания модели прогнозной аналитики. В частности, правильный эксперимент имеет следующие характеристики.

-   Он имеет по крайней мере один набор данных и один модуль.
-   Наборы данных могут быть связаны только с модулями.
-   Модули могут быть связаны либо с наборами данных, либо с другими модулями.
-   Все входные порты для модулей должны иметь связь с потоком данных.
-   Все необходимые параметры для модуля должны быть установлены.

Пример создания простого эксперимента см. в статье **Создание примера эксперимента**.
Более полное пошаговое руководство по созданию решения прогнозной аналитики см. в разделе **Создание эксперимента машинного обучения и выполнение итераций в нем**.

### Наборы данных

Набор данных — это данные, отправленные в Студию машинного обучения Microsoft Azure, чтобы их можно было использовать в процессе моделирования. В Студию машинного обучения Microsoft Azure включено несколько наборов данных, чтобы с ними можно было экспериментировать; кроме того, можно отправлять дополнительные наборы данных по мере необходимости. Ниже приведены примеры включенных наборов данных.

-   **Данные MPG для разных автомобилей** — данные по пробегу в милях на галлон топлива для автомобилей в зависимости от числа цилиндров, мощности в лошадиных силах и т.п.
-   **Данные по раку груди** — данные по диагностике рака груди.
-   **Данные по лесным пожарам** — объемы лесных пожаров в северо-восточной Португалии.

При построении эксперимента рабочий список наборов данных доступен на холсте слева. Можно также просмотреть список отправленных наборов данных, перейдя на вкладку **НАБОРЫ ДАННЫХ** при запуске Студии машинного обучения Microsoft Azure.

### Модули

Модуль — это алгоритм, который можно выполнять с данными. В Студии машинного обучения Microsoft Azure имеется ряд модулей, начиная с функций ввода данных для процессов обучения, оценки и проверки. Ниже приведены примеры включенных модулей.

-   **Преобразование в ARFF** — преобразует упорядоченный набор данных .NET в формат ARFF.
-   **Элементарная статистика** — выполняет элементарные статистические вычисления, такие как математическое ожидание, среднеквадратичное отклонение и др.
-   **Линейная регрессия** — создает интерактивный линейно-регрессионный анализ на основе градиентного спуска.
-   **Балльная модель** — проводит балльную оценку отрабатываемой модели классификации или регрессии.

При построении эксперимента рабочий список модулей доступен на холсте слева. Можно также просмотреть список включенных модулей, перейдя на вкладку **МОДУЛИ** при запуске Студии машинного обучения Microsoft Azure.

Модуль может иметь ряд параметров, которые можно использовать для настройки внутренних алгоритмов модуля. При выборе модуля на холсте его параметры отображаются в области справа. Можно изменять параметры в этой области для настройки модуля.

# Ограничения служб RemoteApp и значения по умолчанию

В этом разделе описаны ограничения служб для Azure RemoteApp.

  
## Ограничения коллекций
Все коллекции:

- Коллекций на пользователя: 1.
- Опубликованных приложений на коллекцию: 100 

Пробные коллекции:

- Длительность пробного периода: 30 дней.
- Пробные коллекции: по 2 на подписку.
- Пробных пользователей на коллекцию: 10. 
- Пробных шаблонных изображений: 25.
 
Активированные (платные) коллекции:

- Платных коллекций: 3 (можно запросить увеличение этого лимита).
- Платных шаблонных изображений: 25.

 
## Ограничения пользователей

Документированные ограничения коллекций являются мягкими (то есть могут быть изменены). Максимальное число пользователей в каждой коллекции является жестким ограничением — изменить его нельзя. Однако число возможных пользователей определяется уровнем коллекции следующим образом:


- Basic — 800 пользователей.
- Standard — 500 пользователей.

(Число пользователей, которое можно поддерживать, определяется количеством виртуальных машин, используемых для коллекции. Для уровня Basic поддерживаются 16 пользователей на виртуальную машину, а для уровня Standard — 10.)

Чтобы определить максимальное число назначенных пользователей, которое можно поддерживать, умножьте число платных пользователей в одной коллекции на число имеющихся у вас коллекций.
  
В учетной записи можно создать несколько коллекций с указанными ограничениями и установить до 5000 одновременных пользовательских подключений по всем коллекциям. Если требуется большее количество, можно запросить его увеличение.

## Другие значения и ограничения:

- Хранение пользовательских данных на пользователя на коллекцию: 50 ГБ.
- Время ожидания при простое: 4 часа.
- Время ожидания при отключении: 4 часа.

## Как запросить увеличение лимита
Можно запросить увеличение лимитов на коллекции и количества одновременных пользователей. Для этого откройте запрос в службу поддержки и объясните, что вам нужно.


Чтобы открыть запрос в службу поддержки, выполните следующее:

1.	На портале управления щелкните [Получить поддержку](https://manage.windowsazure.com/?getsupport=true). Если вы не выполнили вход, будет предложено ввести учетные данные.
2.	Выберите свою подписку.
3.	В разделе типа поддержки выберите **Техническая**.
4.	Щелкните **Создать билет**. 
5.	Выберите **Azure RemoteApp** в списке продуктов, представленном на следующей странице.
6.	Выберите соответствующее значение в поле **Тип проблемы**.
7.	Нажмите кнопку **Продолжить**.
8.	Выполните инструкции, указанные на следующей странице, а затем введите сведения о необходимом увеличении лимита. Например, если требуется продлить пробный период, введите: «Пожалуйста, продлите мой пробный период на X дней». 
9.	Нажмите кнопку **Отправить**, чтобы открыть запрос.

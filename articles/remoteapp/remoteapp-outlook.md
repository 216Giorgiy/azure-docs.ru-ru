# Использование Microsoft Outlook в Azure RemoteApp

Azure RemoteApp поддерживает Microsoft Outlook O365. Дополнительные сведения см. в статье [Использование Office в Azure RemoteApp](remoteapp-officesubscription.md). Есть несколько рекомендаций, касающихся настроек Outlook при использовании в Azure RemoteApp.

## Режим кэширования данных
Режим кэширования данных является рекомендуемой конфигурацией при использовании Outlook в Azure RemoteApp. При настройке учетной записи Outlook 2013 для работы в режиме кэширования Exchange Outlook 2013 работает из локальной копии пользовательского почтового ящика Microsoft Exchange, которая хранится в автономном файле данных (OST) на компьютере пользователя вместе с автономной адресной книгой (OAB). Кэшированный почтовый ящик и автономная адресная книга периодически обновляются при помощи службы O365. См. сведения о [различиях между режимом кэширования данных и оперативным режимом](https://technet.microsoft.com/library/jj683103.aspx).

Пользователь может выбрать **режим кэширования Exchange** или **оперативный режим** во время настройки учетной записи или путем изменения настроек учетной записи. Тот или другой режим также можно развернуть с помощью центра развертывания Office (OCT) или групповой политики.

См. [пошаговые инструкции по включению режима кэширования данных](https://technet.microsoft.com/library/c6f4cad9-c918-420e-bab3-8b49e1885034#proc).

## Поиск
Использование поиска в Outlook имеет определенные ограничения в Azure RemoteApp. Azure RemoteApp использует виртуальные машины пула для размещения пользовательских сеансов. Индексирование поиска зависит от идентификатора компьютера, который отличается у различных виртуальных машин. Возможно, что каждый раз, когда пользователь входит в Azure RemoteApp, его направляют на другую виртуальную машину. Это означает, что при включении локального поиска индексатор будет запускаться каждый раз, когда изменяется индикатор виртуальной машины (когда пользователь переходит на другую виртуальную машину). В зависимости от размера файла OST, индексатору может понадобиться длительное время для завершения работы и использования ресурсов, необходимых для других приложений. Поиск не только будет медленным, но и может оказаться безрезультатным. Эту проблему можно решить, включив поиск в Интернете по умолчанию. К сожалению, в Outlook 2013 индексированный или локальный поиск нельзя отключить, а поиск в Интернете нельзя включить по умолчанию.

В Outlook 2016 эту проблему можно решить благодаря новым возможностям поиска в Интернете для почтовых ящиков, которые размещены в Exchange 2016 (или в Office 365). Для этого используются результаты поиска на сервере, которые сопоставляются с локальным кэшем (OST). В некоторых сценариях Outlook может переключиться на использование индексатора поиска, но в большинстве случаев для поиска будет использоваться оперативный режим. Если поиск в почтовом ящике крайне важен, Azure RemoteApp рекомендует использовать Outlook 2016.

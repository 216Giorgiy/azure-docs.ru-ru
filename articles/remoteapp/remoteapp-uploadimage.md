# <a name="upload-a-custom-image-for-azure-remoteapp"></a>Отправка пользовательского образа для Azure RemoteApp
> [!IMPORTANT]
> Мы выводим службу Azure RemoteApp из эксплуатации 31 августа 2017 года. Дополнительные сведения см. в [объявлении](https://go.microsoft.com/fwlink/?linkid=821148).
> 
> 

После создания пользовательского образа шаблона или внесения в него изменений необходимо отправить этот образ в библиотеку образов Azure RemoteApp. Выполните следующие действия.

## <a name="before-you-start"></a>Перед началом работы
1. Убедитесь, что пользовательский образ соответствует [требованиям к образу](remoteapp-imagereqs.md) и [к приложению](remoteapp-appreqs.md).
2. Установите [модуль Azure PowerShell](/powershell/azure/overview).

## <a name="step-by-step-on-how-to-upload-custom-image"></a>Пошаговые инструкции для отправки пользовательского образа
1. Откройте портал управления Azure и перейдите на страницу RemoteApp.
2. На вкладке **Образы шаблона** в нижней части страницы щелкните **Отправить**.
3. Введите понятное имя образа и укажите расположение учетной записи хранения. Расположение должно быть тем же, в котором находится коллекция RemoteApp или в котором вы хотите ее создать.
4. При запросе скачайте скрипт на локальный ПК.
5. Скопируйте параметры команды в текстовом поле в буфер обмена.
6. Откройте окно Windows PowerShell с повышенными привилегиями.
7. В окне Windows PowerShell с повышенными привилегиями перейдите в тот же каталог, в котором находится скачанный скрипт.
8. Вставьте скопированную команду и нажмите клавишу **ВВОД**.
   
   Начнется процесс отправки, длительность которого может зависеть от многих факторов, в том числе от скорости сети и размера образа.
9. Если отправку не удается выполнить из-за перебоев в работе сети или аналогичных событий, вы всегда можете продолжить процесс отправки. Чтобы продолжить отправку, снова выполните скрипт, используя ту же командную строку.

> [!WARNING]
> Никогда не изменяйте скрипт отправки. Соответствие требованиям к образу и приложению обеспечивается с помощью специальных проверок.
> 
> 

## <a name="common-problems"></a>Распространенные проблемы
* Убедитесь, что вы используете Windows PowerShell, а не Azure PowerShell. Следует установить модуль Azure PowerShell, так как некоторые модули нужны во время процесса отправки.
* Никогда не изменяйте скрипт; для вашего удобства существуют проверки.
* Если VHD-файл будет заблокирован во время отправки, скопируйте файл или переместите его в новое расположение и повторите попытку отправки. Возможно, какой-то процесс Windows препятствует отправке.

# <a name="create-a-volume-for-azure-netapp-files"></a>Создание тома для Azure NetApp Files

Потребление емкости тома зависит от подготовленной емкости пула.  В пуле емкости можно создать несколько томов. Главным условием является то, что общий размер томов не должен превышать размер пула. 

## <a name="before-you-begin"></a>Перед началом работы 
Перед началом необходимо настроить пул емкости.  

[Настройка пула емкости](azure-netapp-files-set-up-capacity-pool.md)

## <a name="steps"></a>Действия 
1.  В колонке "Управление пулами емкости" щелкните колонку **Тома**. 
2.  Чтобы создать том щелкните **+ Добавить том**.  
    Появится окно нового тома.

3.  В окне нового тома щелкните **Создать**, а затем заполните следующие поля.   
    * **Имя**      
        Укажите имя создаваемого тома.   
        Оно должно быть уникальным в пределах группы ресурсов. Длина имени должна быть не менее 3 символов.  Имя может состоять из любых алфавитно-цифровых символов.

    * **Путь к файлу**  
        Укажите путь к файлу, который будет использоваться для создания пути экспорта для нового тома. Путь экспорта используется для подключения и получения доступа к тому.   
        Подключенный объект является конечной точкой IP-адреса службы NFS. Он создается автоматически.    
        Имя пути к файлу может содержать только буквы, цифры и дефисы. Длина имени находится в диапазоне от 16 до 40 знаков.  

    * **Квота**  
        Укажите объем логического хранилища, выделенный для тома.  
        В поле **Доступная квота** отображается объем неиспользуемого пространства выбранного пула емкости, которое можно использовать для создания нового тома. Размер нового тома не должен превышать доступную квоту.  

    * **Виртуальная сеть**  
        Укажите Виртуальную сеть Azure, из которой будет осуществляться доступ к тому. В указанной виртуальной сети должен быть настроен Azure NetApp Files. Доступ к службе Azure NetApp Files можно получить только из виртуальной сети, которая находится в том же расположении что и том.   

    ![Новый том](../media/azure-netapp-files/azure-netapp-files-new-volume.png)

4.  Последовательно выберите **ОК**. 
 
От пула емкости том наследует атрибуты подписки, группы ресурсов и расположения. Состояние развертывания можно отслеживать на вкладке уведомлений.

# Размеры для облачных служб

## Обзор 

В этом разделе описаны доступные размеры и параметры для экземпляров ролей облачной службы (веб-ролей и рабочих ролей). Здесь также предоставлены рекомендации по развертыванию, которые нужно учитывать при планировании использования этих ресурсов.

Виртуальные машины Azure и облачные службы – это два из нескольких типов вычислительных ресурсов, предоставляемых Azure. Объяснение см. в разделе [Варианты размещения вычислительных сред, предоставляемые Azure](http://go.microsoft.com/fwlink/p/?LinkID=311926).

>[AZURE.NOTE]Сведения о других ограничениях Azure см. в разделе [Лимиты, квоты и ограничения подписки и обслуживания Azure](azure-subscription-service-limits.md).

###Размеры для экземпляров рабочих и веб-ролей

Следующие рекомендации помогут вам принять решение относительно размеров:

*   Теперь экземпляры можно настроить для использования виртуальной машины серии D. Они предназначены для запуска приложений, требующих высокий вычислительной мощности и производительности временного диска. Виртуальные машины серии D отличаются более быстрыми процессорами, более высоким коэффициентом «память-ядро» и твердотельным накопителем (SSD) для временного диска. Дополнительные сведения см. в объявлениях в блоге Azure [новые размеры виртуальных машин серии D](http://azure.microsoft.com/blog/2014/09/22/new-d-series-virtual-machine-sizes/).  

*   Веб-роли и рабочие роли требуют больше места на временном диске, чем виртуальные машины Azure, из-за требований к системе. Системные файлы резервируют 4 ГБ места для файла подкачки Windows и 2 ГБ пространства для файла дампа Windows.

*   На диске ОС содержится гостевая ОС Windows, папка «Program Files» (с установками, выполненными с помощью задач запуска, если не указан другой диск), изменения в реестре, папка «System32» и .NET Framework.

*   Локальный диск ресурсов содержит журналы Azure и файлы конфигурации, систему диагностики Azure (включая резервные копии журналов IIS) и все определенные вами ресурсы локального хранилища.

*   Диск приложений (приложение), где извлекается ваш файл .cspkg и содержатся ваш веб-сайт, двоичные файлы, процесс размещения роли, задачи запуска, файл web.config и т. п.

*   Виртуальные машины размеров A8/A10 и A9/A11 имеют одинаковые производственные мощности. Экземпляры A8 и A9 виртуальной машины включают дополнительный сетевой адаптер, подключенный к сети удаленного  доступа к памяти (RDMA), для быстрого обмена данными между виртуальными машинами. Экземпляры A8 и A9 предназначены для высокопроизводительных вычислительных приложений, которые требуют непрерывной связи с низкой задержкой между узлами во время выполнения заданий, например, для приложений, использующих интерфейс передачи сообщений (MPI). Экземпляры виртуальной машины A10 и A11 не содержат дополнительный сетевой адаптер. Экземпляры a10 и A11 предназначены для высокопроизводительных вычислительных приложений, которые не требуют непрерывной связи с низкой задержкой между узлами; они также известны как параметрические или неудобно параллельные приложения.

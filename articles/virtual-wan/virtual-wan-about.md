# <a name="what-is-azure-virtual-wan-preview"></a>Что такое Виртуальная глобальная сеть Azure? (предварительная версия)

Виртуальная глобальная сеть Azure — это сетевая служба, которая обеспечивает оптимизированное и автоматизированное подключение типа "ветвь — ветвь" через Azure. Виртуальная глобальная сеть позволяет подключать и настраивать устройства ветви для взаимодействия с Azure. Это можно сделать вручную или с помощью устройств провайдеров через участника Виртуальной глобальной сети. Использование нужных устройств провайдеров позволяет облегчить использование, подключение и управление конфигурацией. Встроенная панель управления глобальной виртуальной сетью Azure обеспечивает мгновенное устранение неполадок, что экономит время и предоставляет простой способ просмотра широкомасштабных подключений типа "сайт — сайт".

> [!IMPORTANT]
> Сейчас Виртуальная глобальная сеть Azure находится на этапе управляемой общедоступной предварительной версии. Для использования Виртуальной глобальной сети необходимо [зарегистрироваться для работы с предварительной версией](#enroll).
>
> Общедоступная предварительная версия предоставляется без соглашения об уровне обслуживания и не должна использоваться для производственных рабочих нагрузок. Некоторые функции могут не поддерживаться, иметь ограничения и быть доступными не во всех расположениях Azure. См. [дополнительные условия использования для предварительных версий Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

В этой статье представлен быстрый просмотр сетевых подключений рабочих нагрузок, которые относятся и не относятся к Azure. Виртуальная глобальна сеть обеспечивает следующие преимущества.

* **Интегрированные решения для подключения в звездообразной архитектуре** . Автоматизация конфигурации "сайт — сайт" и связи между локальными сайтами и концентратором событий Azure из различных источников, включая решения участников Виртуальной глобальной сети.
* **Автоматическая настройка и конфигурация периферийных зон**. Непрерывно подключайте виртуальные сети и рабочие нагрузки к концентратору Azure.
* **Интуитивное устранение неполадок**. Вы можете посмотреть поток "узел — узел" в Azure и использовать эту информацию для принятия необходимых действий.

## <a name="vendor"></a>Работа с участниками Виртуальной глобальной сети

1. Контроллер устройства ветви (VPN/SDWAN) аутентифицируется для экспорта информации, ориентированной на веб-сайт в Azure с помощью субъект-службы Azure.
2. Контроллер устройства ветви (VPN/SDWAN) получает конфигурацию подключения Azure и обновляет локальное устройство. Это автоматизирует конфигурацию загрузки, редактирования и обновления локального устройства VPN.
3. Как только устройство будет иметь нужную конфигурацию Azure, устанавливается соединение типа "сайт — сайт" (два активных туннеля) с глобальной виртуальной сетью Azure. В Azure необходимо, чтобы контроллер ветви (VPN/SDWAN) поддерживал протокол IKEv2. Узел BGP является необязательным.

## <a name="resources"></a>Ресурсы Виртуальной глобальной сети

Чтобы настроить виртуальную глобальную сеть "узел — узел", необходимо создать следующие ресурсы.

* **virtualWAN**. Ресурс virtualWAN представляет собой виртуальное наложение сети Azure и представляет собой набор из нескольких ресурсов. Он содержит ссылки на все виртуальные концентраторы, которые необходимо иметь в глобальной виртуальной сети. Ресурсы Виртуальной глобальной сети изолированы друг от друга и не могут содержать общий концентратор. В Виртуальной глобальной сети концентраторы не взаимодействуют друг с другом.

* **Веб-айт**. Ресурс сайта, известный как VPN сайт, представляет собой локальное устройство VPN и его настройки. При работе с участником Виртуальной глобальной сети экспорт информации в Azure осуществляется автоматически.

* **Концентратор**. Виртуальный концентратор — это виртуальная сеть, управляемая Microsoft. Концентратор содержит разные конечные точки службы, чтобы обеспечить подключения с локальной сетью (vpnsite). Концентратор является основой сети в регионе. Может существовать только один концентратор в каждом регионе Azure. При создании концентратора на портале Azure автоматически создается виртуальный концентратор VNet и виртуальный концентратор VPN-шлюз.

  Шлюз концентратора отличается от виртуального сетевого шлюза, который используется для служб ExpressRoute и VPN-шлюз. Например, при использовании Виртуальной глобальной сети, не нужно создавать подключение типа "сайт — сайт" от локального компьютера непосредственно к VNet. Вместо этого создается подключение типа "сайт-сайт" к концентратору. Трафик всегда направляется через шлюз концентратора. Это значит, что виртуальным сетям не требуется их собственные шлюзы виртуальных сетей. Использование Виртуальной глобальной сети позволяет легко использовать масштабирование виртуальных сетей через виртуальный концентратор и виртуальный шлюз концентратора. 

* **Виртуальное сетевое подключение концентратора**. Ресурс виртуального сетевого подключения концентратора используется для беспрепятственного подключения концентратора к виртуальной сети. На сегодняшний день можно подключаться только к виртуальным сетям, которые находятся в одном и том же регионе концентратора.

##<a name="enroll"></a>Регистрация для использования предварительной версии

Прежде чем настраивать Виртуальную глобальную сеть, вы должны зарегистрировать свою подписку для использования предварительной версии. В противном случае вы не сможете работать с Виртуальной глобальной сетью на портале. Для регистрации отправьте электронное сообщение с идентификатором подписки по адресу <azurevirtualwan@microsoft.com>. После активации подписки на электронную почту будет отправлено письмо.

# Уровень данных нескольких сайтов(шаблоны архитектуры Azure)

[Инфраструктура интеграции облачных платформ (CPIF)](azure-architectures-cpif-overview.md) предоставляет инструкции по интеграции рабочей нагрузки для адаптации приложений в облачном решении корпорации Майкрософт. 

Инфраструктура интеграции облачных платформ описывает способы, которые следует использовать организациям, клиентам и партнерам для разработки и развертывания предназначенных для облака рабочих нагрузок с помощью гибридной облачной платформы и возможностей управления Azure, System Center и Windows Server. 

Шаблон **уровня данных нескольких сайтов** является частью области **основания**, подробно описанной в документе об архитектуре CPIF. 

## Уровень данных нескольких сайтов

Шаблон разработки уровня данных нескольких сайтов содержит сведения о компонентах и службах Azure, необходимых для предоставления услуг веб-уровня, обеспечивающих предсказуемый уровень производительности и высокую доступность во всех географических регионах. В этом шаблоне разработки уровень данных определен как уровень обслуживания, предоставляющий традиционные службы платформ данных изолированным образом или в виде части многоуровнего приложения.  В шаблоне "балансировка нагрузки уровня данных" выполняется и локально внутри региона, и между регионами.   

Представленные в SQL Server 2012 группы готовности AlwaysOn являются высокодоступным компонентом аварийного восстановления, который полностью поддерживается службами инфраструктуры Azure.  Подробные сведения и объявление официальной поддержки для групп доступности AlwaysOn в службах инфраструктуры Microsoft Azure можно найти в статье "Группы доступности AlwaysOn".   

В этом документе описывается архитектура уровня данных нескольких сайтов в Azure с использованием группы доступности AlwaysOn SQL. С необязательным вспомогательным узлом с доступом только для чтения в дополнительном центре обработки данных Azure для улучшенной функциональности и аварийного восстановления. С помощью SQL AlwaysOn в Azure предоставляется высокодоступный уровень данных, который может использоваться веб-уровнем или уровнем приложений.  

Хотя этот документ посвящен шаблонам и методикам архитектуры, полное руководство по развертыванию можно найти в официальных учебниках, в которых описывается настройка групп доступности AlwaysOn в Azure и настройка прослушивателя группы доступности AlwaysOn.

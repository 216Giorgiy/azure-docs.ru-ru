# <a name="deploy-the-azure-blueprints-iso-27001-shared-services-blueprint-sample"></a>Развертывание образца схемы Azure чертежей ISO 27001 общих служб

Развертывание образца Azure чертежей ISO 27001 общих служб в рамках схемы, необходимо выполнить следующие действия:

> [!div class="checklist"]
> - Создание новой схемы из примера
> - Пометьте свою копию образце в качестве **опубликовано**
> - Назначить копию схемы в существующую подписку

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/free), прежде чем начинать работу.

## <a name="create-blueprint-from-sample"></a>Создать план из примера

Во-первых реализации образца схемы путем создания новой схемы в вашей среде, с помощью примера в качестве начального.

1. Выберите **все службы** и найдите и выберите **политики** в левой области. На **политики** выберите **чертежей**.

1. Из **Приступая к работе** страницы слева, выберите **создать** под кнопкой _Создание схемы_.

1. Найти **ISO 27001: Общие службы** примера схемы в _другие примеры_ и выберите **использовать этот пример**.

1. Введите _основы_ образца схемы:

   - **Имя схемы**: Укажите имя для вашей копии образца схемы ISO 27001 общих служб.
   - **Расположение определения**: Используйте кнопку с многоточием и выберите группу управления, чтобы сохранить копию образца для.

1. Выберите _артефакты_ вкладки в верхней части страницы или **далее: Артефакты** в нижней части страницы.

1. Просмотрите список артефактов, входящих в состав образца схемы. Многие артефакты имеют параметры, которые мы определим более поздней версии. Выберите **сохранить черновик** после завершения просмотра в образце схемы.

## <a name="publish-the-sample-copy"></a>Пример копирования публикации

Копия примера схемы теперь была создана в вашей среде. Он создается в **черновик** режим и должен быть **опубликовано** прежде, чем его можно назначить и развертывания. Копия примера в рамках схемы можно настроить для своей среды и требований, но это изменение может отойти от его стандарте ISO 27001.

1. Выберите **все службы** и найдите и выберите **политики** в левой области. На **политики** выберите **чертежей**.

1. Выберите **схема определения** страницы слева. Используйте фильтры, чтобы найти копию образца схемы, а затем выберите его.

1. Выберите **опубликовать план** в верхней части страницы. В новую страницу в правой части, предоставляют **версии** для вашей копии образца схемы. Это свойство полезно для при внесении изменения в более поздней версии. Укажите **изменить заметки** такие, как «первая версия публикации из образца ISO 27001 схемы». Затем выберите **публикации** в нижней части страницы.

## <a name="assign-the-sample-copy"></a>Назначение копирования образца

После успешного копирования образца в рамках схемы **опубликовано**, его можно назначить подписку из данной группы управления, он сохранен. Этот шаг является, где указаны параметры чтобы сделать уникальным каждое развертывание копии примера схемы.

1. Выберите **все службы** и найдите и выберите **политики** в левой области. На **политики** выберите **чертежей**.

1. Выберите **схема определения** страницы слева. Используйте фильтры, чтобы найти копию образца схемы, а затем выберите его.

1. Выберите **назначение схемы** в верхней части на странице определения схемы.

1. Укажите значения параметров для назначения схемы:

   - Основы

     - **Подписки**. Выберите одну или несколько подписок, которые находятся в группе управления сохранить свою копию образца схемы для. Если выбрать более одной подписки, назначение создается для каждого использования с параметрами, введенными.
     - **Имя назначения**: Имя предварительно заполняется автоматически на основе имени схемы.
       При необходимости измените или оставьте значение по умолчанию.
     - **Расположение.** Выберите регион для управляемого удостоверения должен быть создан в. Azure Blueprint использует это управляемое удостоверение для развертывания всех артефактов в назначенную схему. Дополнительные сведения см. в статье [Управляемые удостоверения для ресурсов Azure](../../../../active-directory/managed-identities-azure-resources/overview.md).
     - **Определение версии схемы**: Выбрать **опубликовано** версии и скопируйте образца схемы.

   - Блокировка назначения

     Выбор блокировки схемы, задание для вашей среды. Дополнительные сведения см. в разделе [Блокировка ресурсов схем](../../concepts/resource-locking.md).

   - Управляемое удостоверение

     Оставьте значение по умолчанию _назначенного системой_ управляемых параметр identity.

   - Параметры схемы

     Параметры, определенные в этом разделе используются многие из артефактов в определение схемы для обеспечения согласованности.

     - **Название организации**: Введите короткое имя для вашей организации. Это свойство в основном используется для именования ресурсов.
     - **Префикс адреса подсети общих служб**: Укажите значение нотации CIDR для сети развернутых ресурсов друг с другом.
     - **Общее расположение службы**: Определяет места развертывания артефактов. Не все службы доступны во всех расположениях. Артефакты, развертывание таких служб предоставляют режим параметра для расположения, чтобы развернуть этот артефакт для.
     - **Допустимое расположение (политика: Схема инициативы для ISO 27001)**: Значение, указывающее допустимые расположения для ресурсов и групп ресурсов.
     - **Рабочая область log Analytics для агентов виртуальных Машин (политика: Схема инициативы для ISO 27001)**: Указывает идентификатор ресурса рабочей области. Этот параметр использует `concat` функция для конструирования идентификатором ресурса.

   - Параметры артефакта

     Параметры, определенные в этом разделе применяются к артефакта, под которой он определен. Эти параметры являются [динамических параметров](../../concepts/parameters.md#dynamic-parameters) так, как они определяются во время назначения чертежа. Полный список или параметры артефактов и их описание см. в разделе [таблицы параметров артефакта](#artifact-parameters-table).

1. После ввода всех параметров выберите **назначить** в нижней части страницы. Создается назначение плана и начинается развертывание артефакта. Это займет около часа. Чтобы проверить состояние развертывания, откройте назначения схемы.

> [!WARNING]
> Служба Azure чертежей и примеры встроенные схемы являются **взимается**. Ресурсы Azure, [стоимость определяется продукта](https://azure.microsoft.com/en-us/pricing/). Используйте [калькулятором цен](https://azure.microsoft.com/pricing/calculator/) оценить расходы на ресурсы, развертываемые в этом примере схема.

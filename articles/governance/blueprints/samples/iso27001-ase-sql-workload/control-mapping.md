# <a name="control-mapping-of-the-iso-27001-asesql-workload-blueprint-sample"></a>Сопоставление элемента управления образца рабочей нагрузки в рамках схемы ISO 27001 ASE/SQL

Ниже статье подробно описано, как в образце рабочей нагрузки ASE/SQL Azure чертежей ISO 27001 в рамках схемы сопоставляются с ISO 27001 элементов управления. Дополнительные сведения об элементах управления см. в разделе [ISO 27001](https://www.iso.org/isoiec-27001-information-security.html).

Следующие сопоставления должны **ISO 27001: 2013** элементов управления. Используйте панели навигации справа для перехода непосредственно к сопоставление конкретного элемента управления. Многие сопоставленные элементы управления реализуются с помощью ["Политика Azure"](../../../policy/overview.md) инициативы. Чтобы просмотреть полный инициативы, откройте **политики** в на портале Azure и выберите **определения** страницы. Затем найдите и выберите  **[Предварительный просмотр] аудита ISO 27001: 2013 управляет и развертывать специальные расширения виртуальной Машины для поддержки требований аудита** инициатива встроенные политики.

И наоборот наличие слишком много владельцев подписки Azure может увеличить вероятность нарушения безопасности через учетную запись скомпрометированных владельца. В рамках этой схемы позволяет поддерживать на соответствующее число владельцев подписки Azure, назначив два ["Политика Azure"](../../../policy/overview.md) определений, которые аудита количество владельцев для подписок Azure. Управление разрешениями владельца подписки могут помочь реализовать подходящих для разделения обязанностей.

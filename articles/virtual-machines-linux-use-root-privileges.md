# Использование прав корневой учетной записи на виртуальных машинах Linux в Azure

По умолчанию пользователь  `root` отключен в виртуальных машинах Linux в Azure. Пользователи могут выполнять команды с повышенными привилегиями, используя команду  `sudo`. При этом в зависимости от способа подготовки системы процедура может отличаться.

1. **SSH ключ и пароль или только пароль** - виртуальная машина подготовлена либо с использованием сертификата (`CER`-файл), либо с использованием ключа SSH и пароля или просто имени пользователя и пароля. В этом случае  `sudo` запросит пароль пользователя перед выполнением команды.

2. **Только ключ SSH** - виртуальная машина подготовлена с использованием сертификата (`CER`- или  `.pem`-файл) или ключа SSH, но без пароля.  В этом случае  `sudo` **не будет** запрашивать пароль пользователя перед выполнением команды.


## Ключ и пароль или только пароль SSH

Войдите в виртуальную машину Linux с использованием ключа или пароля SSH, а затем выполните команды с использованием  `sudo`, например:

	# sudo <command>
	[sudo] password for azureuser:

В этом случае пользователю будет предложено ввести пароль. После ввода пароля  `sudo` выполнит команду с привилегиями  `root` учетной записи.


## Только ключ SSH

Войдите в виртуальную машину Linux с использованием ключа SSH, а затем выполните команды с использованием  `sudo`, например:

	# sudo <command>

В этом случае пользователю **не будет** предложено ввести пароль. После нажатия кнопки `<enter>`  `sudo` выполнит команду с привилегиями  `root`.

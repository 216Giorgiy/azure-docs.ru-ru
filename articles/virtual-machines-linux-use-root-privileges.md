# Использование прав корневой учетной записи на виртуальных машинах Linux в Azure

По умолчанию привилегированный пользователь отключен в виртуальных машинах Linux в Azure. Пользователи могут выполнять команды с повышенными привилегиями, используя команду sudo. При этом в зависимости от способа подготовки системы процедура может отличаться.

1. **Ключ SSH и пароль или только пароль** - виртуальная машина была подготовлена либо с использованием сертификата (CER-файл), либо с использованием ключа SSH и пароля или просто имени пользователя и пароля. В этом случае перед выполнением команды sudo запросит пароль пользователя.

2. **Только ключ SSH** - виртуальная машина была подготовлена с использованием сертификата (CER- или PEM-файл) или ключа SSH, но без пароля.  В этом случае sudo **не будет** запрашивать пароль пользователя перед выполнением команды.


## Ключ и пароль или только пароль SSH

Войдите в виртуальную машину Linux с использованием ключа SSH или пароля, а затем выполните команды с использованием sudo, например:

	# sudo <command>
	[sudo] password for azureuser:

В этом случае пользователю будет предложено ввести пароль. После ввода пароля sudo выполнит команду с привилегиями корневой учетной записи.


## Только ключ SSH

Войдите в виртуальную машину Linux с использованием ключа SSH, а затем выполните команды с использованием sudo, например:

	# sudo <command>

В этом случае пользователю **не будет** предложено ввести пароль. После нажатия клавиши ВВОД sudo выполнит команду с привилегиями корневой учетной записи.

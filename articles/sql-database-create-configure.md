<h1><a id="configLogical"></a>Создание и настройка базы данных Azure SQL</h1>

В этом разделе вы создадите и настроите базу данных Azure SQL с помощью портала управления Azure. В этом процессе сначала нужно создать сервер. Этот подход может быть предпочтительнее при наличии существующих баз данных SQL Server, которые планируется отправить в Azure.

<h2><a id="createLogical"></a>Практическое руководство. Создание логического сервера</h2>

1. Выполните вход на [Портал управления](http://manage.windowsazure.com).

2. В нижней части страницы щелкните **СОЗДАТЬ**.

	![Click SQL Databases][1]

3. Нажмите **Data Services** и **SQL Database**, затем **Quick Create**.

	![Click New, Data Services, and Quick Create][2]
	 
5. В **Параметрах базы данных SQL сервера** выберите подписку.

	![Select a subscription][3]

6. В **Параметрах базы данных SQL сервера** введите имя входа в одно слово, без пробелов. 

 База данных SQL использует проверку подлинности SQL с помощью шифрованного подключения. Будет создана новая учетная запись для проверки подлинности SQL Server, назначенная предопределенной роли сервера sysadmin и использующая введенное имя. 

	В качестве логина нельзя использовать адрес электронной почты, учетную запись пользователя Windows или Windows Live ID. База данных SQL не поддерживает утверждения и проверку подлинности Windows.
	
7. Укажите надежный пароль, который содержит более восьми знаков, в том числе символы в верхнем и нижнем регистре, а также цифры и символы.

7. Выберите регион. Регион определяет географическое расположение сервера. Изменение региона достаточно трудоемко, поэтому выберите регион, имеющий смысл для этого сервера. Выберите местоположение, максимально близкое к вам. Хранение приложения и базы данных Azure в том же регионе снижает расходы на пропускную способность и задержку данных.

8. Убедитесь, что установлен флажок **Разрешить службам Azure доступ к серверу**, чтобы можно было подключиться к этой базе данных, используя портал управления базой данных SQL, службы хранилища и другие службы Azure. 

9. Закончив, щелкните флажок внизу страницы.

### Имя сервера, сформированное автоматически

Обратите внимание, что имя сервера не было задано. База данных SQL автоматически создает имя сервера, гарантируя отсутствие повторяющихся записей DNS. Имя сервера является буквенно-цифровой строкой, состоящей из 10 знаков. Невозможно изменить имя сервера баз данных SQL.

На следующем шаге будет настроен брандмауэр, чтобы разрешить подключение для приложений, работающих в вашей сети.

## Практическое руководство. Настройка брандмауэра для логического сервера

1. На [Портале управления](http://manage.windowsazure.com) нажмите **Базы данных SQL**, затем нажмите **Серверы**

	![Click Servers][4]
2. В списке выберите только что созданный сервер.

2. Щелкните **Настроить**.

	![Click Configure][5]

3. В разделе **разрешенные IP-адреса** нажмите **ДОБАВИТЬ К РАЗРЕШЕННЫМ IP-АДРЕСАМ**. Это IP-адрес, прослушиваемый в настоящий момент вашим маршрутизатором или прокси-сервером. База данных SQL определяет IP-адрес, используемый текущим подключением, и создает правило брандмауэра для принятия запросов на подключение от этого устройства. 
	![Click Add to the allowed IP addresses][6]

	Создается имя по умолчанию для этого правила. Измените имя по своему усмотрению. 
	

4. При подключении к базе данных с другого компьютера вам необходимо создать новое правило, чтобы разрешить IP-адрес для подключения. Используйте поля "Начало" и "Окончание" для создания диапазона IP-адресов.

	Если клиентские компьютеры используют динамически назначаемые IP-адреса, то можно указать диапазон, достаточно широкий для охвата IP-адресов, назначенных компьютерам в сети. Начните с узкого диапазона и расширяйте его только в случае необходимости.

7. Чтобы завершить этот шаг, нажмите кнопку **Сохранить**. 

Теперь у вас есть логический сервер, правило брандмауэра, разрешающее входящие подключения с вашего IP-адреса, и учетная запись администратора. 

**Примечание.** Только что созданный логический сервер является виртуальным и будет динамически размещаться на физических серверах в центре обработки данных. Удаление сервера является действием без возможности его отмены. Не забудьте создать резервную копию всех баз данных, которые впоследствии понадобится загрузить на сервер.

#Глоссарий гибкого масштабирования
Следующие термины определяются для возможности гибкого масштабирования базы данных SQL Azure.

![Elastic Scale terms][1]

**База данных**: База данных SQL Azure. 

**Маршрутизация, зависящая от данных**. Функциональность, которая позволяет подключать приложение к сегменту, имеющему специальный ключ сегментирования. Сравните с **запросом нескольких сегментов**.

**Глобальная карта сегментов**: Набор сопоставлений между ключами сегментирования и их соответствующими сегментами в пределах **набора сегментов**. GSM хранится в **диспетчере карты сегментов**. Сравните с **локальной картой сегментов**.

**Карта списка сегментов**: Карта сегментов, в которой ключи сегментирования сопоставляются по отдельности. Сравните с **картой диапазона сегментов**.   

**Карта локальных сегментов**: Карта локальных сегментов хранится на сегменте и содержит сопоставления для шардлетов, которые находятся на сегменте.

**Запрос нескольких сегментов**: Возможность выдавать запрос к нескольким сегментам; результирующие наборы возвращаются с помощью UNION ALL семантики (также именуется "развертываемым запросом"). Сравните с **Маршрутизацией, зависящей от данных**.

**Карта диапазона сегментов**: Карта сегментов, в которой стратегия распределения сегментов основана на нескольких диапазонах непрерывных значений. 

**Ссылочные таблицы**: Таблицы, которые не сегментированы, но реплицированы по сегментам. 

**Сегмент**: База данных SQL Azure, в которой хранятся данные из набора сегментированных данных. 

**Гибкость сегментов** (SE): Возможность выполнять **горизонтальное масштабирование** и **вертикальное масштабирование**.

**Сегментированные таблицы**: Таблицы, которые сегментированы, т. е., данные которых распространяются по сегментам на основе их значений ключей сегментирования. 

**Ключ сегментирования**: Значение столбца, которое определяет распределение данных по сегментам.  Типы значений могут быть следующими: int, bigint, varbinary или uniqueidentifier. 

**Набор сегментов**: Коллекция сегментов, которые связаны с одной картой сегментов в диспетчере карты сегментов.  

**Шардлет**: Все данные, связанные с одним значением ключа сегментирования на сегменте. Шардлет является наименьшей единицей перемещения данных, возможного при распространении сегментированных таблиц. 

**Карта сегментов**: Набор сопоставлений между ключами сегментирования и их соответствующими сегментами.

**Диспетчер карты сегментов**. Хранилище объектов управления и данных, в котором содержатся карты сегментов, расположения сегментов и сопоставления для одного или нескольких наборов сегментов.


##Команды

**Горизонтальное масштабирование**: Операция масштабирования вне (или в) коллекции сегментов путем добавления или удаления сегментов в карте сегментов.

**Слияние**: Процесс перемещения шардлетов из двух сегментов в один сегмент и соответствующее обновление карты сегментов.

**Перемещение шардлета**: Процесс перемещения одного шардлета в другой сегмент. 

**Сегмент**: Процесс горизонтального секционирования идентично структурированных данных между несколькими базами данных на основе ключа сегментирования.

**Разбиение**: Процесс перемещения нескольких шардлетов из одного сегмента в другой (обычно новый) сегмент. Ключ сегментирования предоставляется пользователем, как точка разбиения.

**Вертикальное сегментирование**: Повышение (или понижение) уровня производительности отдельного сегмента. Например, изменение сегмента от Standard до Premium (если необходимо для повышения производительности).

# Краткий справочник по навигации на портале Azure

Служба Веб-сайты Azure теперь называется [Веб-приложения службы приложений](http://go.microsoft.com/fwlink/?LinkId=529714). Мы обновляем всю свою документацию, чтобы отразить в ней такое изменение имени, а также предоставить инструкции для нового портала Azure. До завершения этого процесса вы можете использовать настоящий документ в качестве руководства по работе с веб-приложениями на новом портале Azure.
 
## Будущее старого портала

Хотя вы заметите изменение фирменной символики на старом портале, этот портал находится в процессе замены на новый портал Azure. По мере вывода старого портала из обращения вся последующая деятельность по разработке постепенно перемещается на новый портал. Все новые возможности для веб-приложений будут выходить на новом портале. Начните работу с новым порталом, чтобы воспользоваться преимуществами самых новых и лучших возможностей веб-приложений.

## Отличия в структуре старого портала и портала Azure

На старом портале все службы Azure перечислены слева. Навигация по старому порталу напоминает древовидную структуру, где вы начинаете со службы и переходите к каждому следующему элементу. Эту структуру удобно использовать при управлении независимыми компонентами. Однако приложения, построенные на платформе Azure, представляют собой коллекцию взаимосвязанных служб, а древовидная структура не слишком хорошо подходит для работы с коллекциями служб. 

Новый портал облегчает процесс сквозного построения приложений с использованием компонентов из нескольких служб. Новый портал Azure организован по принципу *journeys*.  *journey* - это последовательность *blades*, которые являются контейнерами для различных компонентов. Например, настройка автоматического масштабирования для веб-приложения является *journey*, предоставляющим вам несколько выносок, как показано в следующем примере: выноска **веб-сайт** (заголовок этой выноски еще не был обновлен до новой терминологии), выноска **Параметры** и выноска **Масштаб**. В этом примере автомасштаб сделан зависимым от уровня загрузки ЦП, поэтому также используется выноска **Процент ЦП**. Компоненты внутри *blades* называются *parts*, которые похожи на плитки. 

![](./media/app-service-web-app-preview-portal/AutoScaling.png)

## Пример навигации: создание веб-приложения

Создание нового веб-приложения осуществляется просто, как 1-2-3. На следующем рисунке рядом показаны старый портал и новый портал, чтобы продемонстрировать, что число шагов, необходимых для создания и запуска веб-приложения, осталось примерно на прежнем уровне. 

![](./media/app-service-web-app-preview-portal/CreateWebApp.png)

на новом портале можно выбрать один из наиболее распространенных типов веб-приложений, включая популярные приложения из коллекции, такие как WordPress. Полный список доступных приложений см. в [Azure Marketplace].

При создании веб-приложения необходимо указать URL-адрес, план службы приложений и расположение на новом портале - все точно так же, как и на старом портале. 

![](./media/app-service-web-app-preview-portal/CreateWebAppSettings.png)

Кроме того, новый портал позволяет определять другие общие параметры. Например, [группы ресурсов](azure-preview-portal-using-resource-groups.md) упрощают просмотр связанных ресурсов Azure и управление ими. 

## Пример навигации: параметры и возможности

Все параметры и возможности теперь логически сгруппированы в одной выноске, с которой вы можете осуществлять навигацию.

![](./media/app-service-web-app-preview-portal/WebAppSettings.png)

Например, можно создать личные домены, нажав кнопку **Custom domains and SSL** (Личные домены и SSL) на выноске **Settings** (Параметры).

![](./media/app-service-web-app-preview-portal/ConfigureWebApp.png)

Чтобы настроить оповещение мониторинга, щелкните **Requests and errors** (Запросы и ошибки) и затем **Add Alert** (Добавить оповещение).

![](./media/app-service-web-app-preview-portal/Monitoring.png)

Чтобы включить диагностику, щелкните **Diagnostics logs** (Журналы диагностики) на выноске **Settings** (Параметры).

![](./media/app-service-web-app-preview-portal/Diagnostics.png)
 
Чтобы настроить параметры приложения, щелкните **Application settings** (Параметры приложения) на выноске **Settings** (Параметры). 

![](./media/app-service-web-app-preview-portal/AppSettingsPreview.png)

За исключением имени торговой марки, на портале имеется довольно мало компонентов, которые были переименованы или иначе сгруппированы, что позволяет упростить их поиск. Например, ниже приведен снимок экрана с соответствующей страницей параметров приложения (**Configure** (Настройка)) на старом портале.

![](./media/app-service-web-app-preview-portal/AppSettings.png)

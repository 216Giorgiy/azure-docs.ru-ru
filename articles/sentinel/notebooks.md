# <a name="use-notebooks-to-hunt-for-anomalies"></a>Использование записных книжек значительных аномалии

> [!IMPORTANT]
> Azure Sentinel сейчас находится в общедоступной предварительной версии.
> Эта предварительная версия предоставляется без соглашения об уровне обслуживания и не рекомендована для использования рабочей среде. Некоторые функции могут не поддерживаться или их возможности могут быть ограничены. Дополнительные сведения см. в статье [Дополнительные условия использования предварительных выпусков Microsoft Azure](https://azure.microsoft.com/support/legal/preview-supplemental-terms/).

Azure Sentinel использует возможности интерактивных записных книжек Jupyter для обеспечения открытости проекта для действий, чтобы проанализировать или слежения нарушений в системе безопасности в вашей среде. Azure Sentinel поставляется с записными книжками, разработанных с аналитиков по вопросам безопасности корпорации Майкрософт. Каждая записная книжка, специально созданная с автономным рабочий процесс для конкретного сценария использования. Визуализации включаются в каждой записной книжки для ускорения просмотра данных и поиск угроз. Настраивать встроенные записные книжки в соответствии с потребностями, создать новые записные книжки с нуля или импортировать записные книжки из Azure, Sentinel "сообществу GitHub. Записные книжки Jupyter, импортируются как проекта на странице записных книжек Azure – это позволит пользователю получить доступ ко всем ноутбуков Azure Sentinel в одном месте. Записные книжки можно запустить с помощью нажатия кнопки, или можно настроить пользователем в соответствии со своей средой.

Полностью интегрированный интерфейс включает записные книжки для запуска в облачных вычислений и хранения с отсутствие базового накладных расходов на обслуживание. Возможность использования существующих экосистемы записные книжки Jupyter позволяет рекордное исходной модели, без предоставления общего доступа данные клиентов. 


Каждая записная книжка размещается в Azure Notebooks (сейчас в предварительной версии), интерактивная среда разработки в облаке Azure. Блокноты всегда представляют собой доступную и всегда доступен из любого браузера, в любой точке мира.  Azure, Sentinel "встроенные записные книжки для исследования и охота клонируются в проект, принадлежит вам, а какие можно изменять и адаптироваться к вашей среде. Ниже приведены некоторые из самых популярных записных книжек встроенные.

- **Предупреждения об изменении расследования и поможет при покупке**: Этой интерактивной записной книжки обеспечивает быстрое рассмотрение различных классов оповещений, извлечение связанного действия и привлечению новых специалистов оповещения с помощью связанного действия и данных, из которого было выдано оповещение.

- **Узел конечной точки интерактивной поможет при покупке**: Позволяет слежения предмет брешей в системе безопасности путем детализации соответствующие действия безопасности, связанные с узлом конечной точки.  

- **Войдите в Office интерактивной охота**: Позволяет слежения для подозрительных входов визуализации географических расположениях suspect журналов, а также отображение вход необычного поведения на основе данных в Office 365.

## <a name="run-a-notebook"></a>Запуск записной книжки
В следующем примере мы запустите записную книжку встроенный поиск глубокого углубляться в расположение аномалии, чтобы увидеть, если любой пользователь в непредвиденное расположение делает что-то в сети.

1. На портале Azure Sentinel щелкните **записных книжек** и затем выберите любой из встроенных записных книжек.
  
   ![Выберите записной книжки](./media/notebooks/select-notebook.png)

3. Нажмите кнопку **импорта** клонировать репозиторий GitHub в проект записных книжек Azure.
   ![Импорт записной книжки](./media/notebooks/import1.png)
4. Каждая записная книжка поможет выполнить все действия по выполнению слежения или исследования. Чтобы включить выполнение одним щелчком автоматически импортируется моделей, библиотеки и другие зависимости и конфигурация для подключения к Azure Sentinel. Весь код и библиотеки, необходимые для выполнения записной книжки предварительно загружены. Вы можете сразу выполнения записную книжку в рабочей области Log Analytics без дополнительной настройки.

   ![Импорт репозитория](./media/notebooks/import2.png)

5. Изучите, измените и запустите все примеры записных книжек предоставляются. Их можно использовать в качестве стандартных блоков для различных сценариев.

   ![Выберите записной книжки](./media/notebooks/import3.png)

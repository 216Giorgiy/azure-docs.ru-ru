# <a name="batch-ai-service-quotas-and-limits"></a>Квоты и ограничения Azure Batch AI

[!INCLUDE [batch-ai-retiring](../../includes/batch-ai-retiring.md)]

Как и в других службах Azure, существуют ограничения на некоторые ресурсы, связанные с Azure Batch AI. В Batch AI эти ограничения являются квотами по умолчанию, применяемыми на уровне подписки для каждого региона, в котором [доступна](https://azure.microsoft.com/global-infrastructure/services/) служба. В данной статье рассматриваются эти значения по умолчанию и то, как запросить увеличение квот.

Помните об этих квотах при разработке и увеличении масштаба ресурсов службы Batch AI. Например, если в кластере не удается достичь указанного целевого числа узлов, возможно, вы достигли ограничения ядер Batch AI для подписки.

Если планируется выполнять рабочие нагрузки в производственной среде с использованием Batch AI, имеет смысл увеличить одну или несколько квот по сравнению со значениями по умолчанию.

> [!NOTE]
> Квота — это кредитный лимит, а не гарантия наличия ресурсов. Если вам нужны ресурсы в очень большом объеме, обратитесь в службу поддержки Azure.
> 
>

## <a name="other-limits"></a>Другие ограничения

Ниже приведены строгие ограничения, которые нельзя превышать.

| **Ресурс** | **Максимальное ограничение** |
| --- | --- |
| Максимальное количество рабочих областей на группу ресурсов | 800 |
| Максимальный размер кластера | 100 узлов |
| Максимальное количество процессов MPI в GPU на узел | 1–4 |
| Максимальное количество рабочих ролей GPU на узел | 1–4 |
| Максимальное время существования задания | 7 дней<sup>1</sup> |
| Максимальное количество серверов параметров на узел | 1 |

<sup>1</sup> Максимальное время существования задания означает время от начала запуска задания и до его завершения. Завершенные задания сохраняются в течение неограниченного времени; данные для заданий, которые не были завершены в течение максимального времени существования, недоступны.

## <a name="view-batch-ai-quotas"></a>Просмотр квот Batch AI

Просмотрите текущие квоты для подписки Batch AI на [портале Azure][portal].

1. На панели слева щелкните **Все службы**. Выполните поиск **Batch AI** и щелкните, чтобы открыть службу.
2. В меню Batch AI нажмите кнопку **Использование и квоты**.
3. Выберите подписку, чтобы просмотреть квоты.

## <a name="increase-a-batch-ai-cores-quota"></a>Увеличение квоты на ядра в Batch AI

Выполните следующие действия, чтобы запросить увеличение квоты для своей подписки Batch AI на [портале Azure][portal]. 

1. На панели слева щелкните **Все службы**. Выполните поиск **Batch AI** и щелкните, чтобы открыть службу.
2. В меню Batch AI щелкните **Новый запрос в службу поддержки**.
3. В разделе **Основные сведения**:
   
    a. **Тип проблемы** > **Квота**.
   
    2. **Подписка**, выберите свою подписку.
   
    c. **Тип квоты** > **Batch AI**.
   
    d. **План поддержки**, выберите свой план поддержки.

    Нажмите кнопку **Далее**.
4. В разделе **Проблема**:
   
    a. Выберите **Серьезность** в соответствии с [последствиями для организации][support_sev].
   
    2. На вкладке **Сведения о квотах** укажите расположение, тип квоты и тип ресурса. Укажите новое ограничение, которое нужно запросить. Щелкните **Сохранить и продолжить**.

    c. (Необязательно.) Отправьте любые важные файлы с дополнительными сведениями о причине увеличения лимита.
   
    Нажмите кнопку **Далее**.
5. В разделе **Контактные данные**:
   
    a. Выберите **Preferred contact method**(Предпочтительный способ связи).
   
    2. Проверьте и введите необходимые контактные данные.
   
    Щелкните **Создать** , чтобы отправить запрос в службу поддержки.

После отправки запроса служба поддержки Azure свяжется с вами. Обработка запроса может занять до 2 рабочих дней.

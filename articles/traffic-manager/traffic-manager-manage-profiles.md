# <a name="manage-an-azure-traffic-manager-profile"></a>Управление профилем диспетчера трафика Azure

Профили диспетчера трафика используют методы маршрутизации трафика, чтобы управлять распределением трафика для облачных служб и конечных точек веб-сайта. В статье объясняется, как создавать эти профили и управлять ими.

## <a name="create-a-traffic-manager-profile-using-quick-create"></a>Создание профиля диспетчера трафика с помощью быстрого создания

Используя функцию быстрого создания на классическом портале Azure, вы можете легко создать профиль диспетчера трафика. Эта функция дает возможность создавать профили с параметрами основной конфигурации. Однако ее нельзя использовать, чтобы настроить набор конечных точек (облачных служб и веб-сайтов), порядок отработки отказа для метода отказоустойчивой маршрутизации трафика, параметры мониторинга и т. д. Эти параметры можно настроить на классическом портале Azure уже после создания профиля. Диспетчер трафика поддерживает до 200 конечных точек на профиль. Но для большинства сценариев использования требуется небольшое количество конечных точек.

### <a name="to-create-a-traffic-manager-profile"></a>Создание профиля диспетчера трафика

1. **Разверните свои облачные службы и веб-сайты в рабочей среде.** Дополнительные сведения об облачных службах см. в [этой статье](http://go.microsoft.com/fwlink/p/?LinkId=314074). Дополнительные сведения о веб-сайтах см. в [этой статье](http://go.microsoft.com/fwlink/p/?LinkId=393327).
2. **Войдите на классический портал Azure.** Щелкните **Создать** в левом нижнем углу портала, а затем, чтобы начать настройку профиля, щелкните **Сетевые службы > Диспетчер трафика** и **Быстрое создание**.
3. **Настройте префикс DNS.**  Присвойте профилю диспетчера трафика уникальное имя префикса DNS. Можно указать только префикс для доменного имени диспетчера трафика.
4. **Выберите подписку.**  Выберите соответствующую подписку Azure. Каждый профиль связан с одной подпиской. Если у вас только одна подписка, этот параметр не отображается.
5. **Выберите метод маршрутизации трафика.** Выберите метод маршрутизации трафика в разделе **Traffic routing Policy** (Политика маршрутизации трафика). Дополнительные сведения о методах маршрутизации трафика см. в статье [Методы маршрутизации трафика диспетчером трафика](traffic-manager-routing-methods.md).
6. **Нажмите кнопку "Создать", чтобы создать профиль.** После завершения настройки профиля его можно найти на панели диспетчера трафика на классическом портале Azure.
7. **Настройте конечные точки, мониторинг и дополнительные параметры на классическом портале Azure.** С помощью кнопки "Быстрое создание" можно настроить только основные параметры. Вам потребуется настроить дополнительные параметры: список конечных точек и порядок отработки отказа конечных точек.

## <a name="disable-enable-or-delete-a-profile"></a>Включение, отключение или удаление профиля

Вы можете отключить существующий профиль диспетчера трафика, чтобы он не указывал запросам пользователей на настроенные конечные точки. При отключении профиля диспетчера трафика сам профиль и содержащиеся в нем сведения останутся нетронутыми и могут быть отредактированы в интерфейсе диспетчера трафика.  При повторном включении профиля перенаправление возобновляется. При создании профиля диспетчера трафика на классическом портале Azure он включается автоматически. Если вы решили, что профиль больше не нужен, его можно удалить.

### <a name="to-disable-a-profile"></a>Отключение профиля

1. При использовании имени пользовательского домена измените запись CNAME на DNS-сервере в Интернете, чтобы она больше не указывала на профиль диспетчера трафика.
2. Трафик больше не будет направляться на конечные точки с помощью параметров профиля диспетчера трафика.
3. Выберите профиль, который требуется отключить. На странице диспетчера трафика выделите профиль, щелкнув столбец рядом с его именем. Обратите внимание, если щелкнуть имя профиля или стрелку рядом с именем, откроется страница параметров для профиля.
4. После выбора профиля нажмите кнопку **Отключить** в нижней части страницы.

### <a name="to-enable-a-profile"></a>Включение профиля

1. Выберите профиль, который требуется отключить. На странице диспетчера трафика выделите профиль, щелкнув столбец рядом с его именем. Обратите внимание, если щелкнуть имя профиля или стрелку рядом с именем, откроется страница параметров для профиля.
2. После выбора профиля нажмите кнопку **Включить** в нижней части страницы.
3. При использовании имени пользовательского домена создайте запись ресурса CNAME на DNS-сервере в Интернете, указывающую на доменное имя вашего профиля диспетчера трафика.
4. Трафик снова будет направляться на конечные точки.

### <a name="to-delete-a-profile"></a>Порядок удаления профиля

1. Убедитесь, что запись ресурса DNS на DNS-сервере Интернета больше не использует запись ресурса CNAME, указывающую на доменное имя вашего профиля диспетчера трафика.
2. Выберите профиль, который требуется отключить. На странице диспетчера трафика выделите профиль, щелкнув столбец рядом с его именем. Обратите внимание, если щелкнуть имя профиля или стрелку рядом с именем, откроется страница параметров для профиля.
3. После выбора профиля нажмите кнопку **Удалить** в нижней части страницы.

## <a name="view-traffic-manager-profile-change-history"></a>Просмотр журнала изменений профиля диспетчера трафика

Журнал изменений профиля диспетчера трафика можно просмотреть на классическом портале Azure в разделе «Службы управления».

### <a name="to-view-your-traffic-manager-change-history"></a>Для этого выполните следующее:

1. В левой области классического портала Azure откройте раздел **Службы управления**.
2. На странице служб управления щелкните **Журналы операций**.
3. На странице «Журналы операций» вы можете с помощью фильтра выбрать журнал изменений для просмотра. Выбрав параметры фильтрации, установите флажок, чтобы просмотреть результаты.

   * Чтобы просмотреть изменения всех профилей, выберите свою подписку и временной диапазон, а затем — пункт **Диспетчер трафика** в раскрывающемся списке **Тип**.
   * Чтобы выполнить фильтрацию по имени профиля, введите имя профиля в поле **Имя службы** или выберите его из раскрывающегося списка.
   * Чтобы просмотреть информацию о каждом отдельном изменении, выберите строку с изменением, с которым вы хотите ознакомиться, и в нижней части страницы нажмите кнопку **Сведения** . В окне **Сведения об операции** можно просмотреть XML-представление объекта API, который был создан или обновлен в рамках операции.

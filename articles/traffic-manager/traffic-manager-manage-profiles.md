# Управление профилем диспетчера трафика

Профиль диспетчера трафика нужен, чтобы определять, какие конечные точки облачных служб и веб-сайтов будет отслеживать диспетчер трафика и какой метод балансировки нагрузки будет использоваться для распределения соединений между этими конечными точками. 

## Создание профиля диспетчера трафика с помощью функции "Быстрое создание"

Используя функцию **Быстрое создание** на портале управления, вы можете легко создать профиль диспетчера трафика. Эта функция дает возможность создавать профили с параметрами основной конфигурации. Однако ее нельзя использовать, чтобы настроить набор конечных точек (облачных служб и веб-сайтов), порядок отработки отказа для метода отказоустойчивой балансировки нагрузки, параметры мониторинга и т. д. Эти параметры можно настроить на портале управления уже после создания профиля. Список процедур настройки см. в статье [Задачи настройки диспетчера трафика](https://msdn.microsoft.com/library/azure/hh744830.aspx).

### Создание профиля диспетчера трафика

1. **Разверните облачные службы и веб-сайты в рабочей среде.** Дополнительные сведения об облачных службах см. в статье [Облачные службы](http://go.microsoft.com/fwlink/p/?LinkId=314074). Сведения об облачных службах см. в разделе [Рекомендации](https://msdn.microsoft.com/library/azure/5229dd1c-5a91-4869-8522-bed8597d9cf5#bkmk_TrafficManagerBestPracticesProfile). Дополнительные сведения о веб-сайтах см. в статье [Веб-сайты](http://go.microsoft.com/fwlink/p/?LinkId=393327).

2. **Войдите на портал управления.** Чтобы создать новый профиль диспетчера трафика, нажмите кнопку **Создать** в левом нижнем углу портала, а затем, чтобы начать настройку профиля, последовательно выберите элементы **Сетевые службы**, **Диспетчер трафика** и **Быстрое создание**.
3. **Укажите префикс DNS.** Задайте для профиля диспетчера трафика уникальное имя префикса DNS. Вы можете указать префикс только для доменного имени диспетчера трафика.
4. **Выберите подписку.** Выберите соответствующую подписку Azure. Каждый профиль связан с одной подпиской. Если имеется только одна подписка, этот параметр не отображается.
5. **Выберите метод балансировки нагрузки.** В списке **Политика балансировки нагрузки** выберите метод балансировки нагрузки. Дополнительные сведения о методах балансировки нагрузки см. в статье [О методах балансировки нагрузки диспетчера трафика](traffic-manager-load-balancing-methods.md).
6. **Нажмите кнопку "Создать", чтобы создать новый профиль**. После завершения настройки профиля его можно найти на панели диспетчера трафика на портале управления.
7. **Настройте на портале управления конечные точки, мониторинг и дополнительные параметры.** Так как с помощью функции быстрого создания можно настроить только основные параметры, для требуемой конфигурации необходимо настроить дополнительные параметры, например список конечных точек и порядок отработки отказа конечных точек. Список процедур настройки см. в статье [Задачи настройки диспетчера трафика](https://msdn.microsoft.com/library/azure/hh744830.aspx).

## Включение, отключение и удаление профиля

Существующий профиль диспетчера трафика можно отключить, чтобы при запросах пользователей он не ссылался на настроенные в нем конечные точки. Если отключить профиль диспетчера трафика, то профиль и сведения в нем не исчезнут и их можно будет изменять в интерфейсе диспетчера трафика. Если необходимо снова включить профиль, вы легко это можете сделать на портале управления. После этого работа ссылок возобновится. При создании профиля диспетчера трафика на портале управления он включается автоматически. Если вы решили, что профиль больше не нужен, его можно удалить. 

### Отключение профиля

1. Измените запись ресурса DNS на DNS-сервере в Интернете, используя соответствующий тип записи и указатель на другое имя или IP-адрес определенного расположения в Интернете. Другими словами, следует изменить запись ресурса DNS на DNS-сервере в Интернете, чтобы он больше не использовал запись ресурса CNAME, указывающую на доменное имя вашего профиля диспетчера трафика
2. Трафик перестанет направляться на конечные точки через параметры профиля диспетчера трафика.
3. Выберите профиль, который необходимо отключить. Для этого выделите его на странице диспетчера трафика, щелкнув столбец рядом с именем профиля. Не щелкайте имя профиля и стрелку рядом с этим именем, чтобы не переходить на страницу параметров профиля.
4. Выбрав профиль, в нижней части страницы нажмите кнопку **Отключить**.

### Включение профиля

1. Выберите профиль, который необходимо включить. Для этого выделите его на странице диспетчера трафика, щелкнув столбец рядом с именем профиля. Не щелкайте имя профиля и стрелку рядом с этим именем, чтобы не переходить на страницу параметров профиля.
2. Выбрав профиль, в нижней части страницы нажмите кнопку **Включить**.
3. Измените запись ресурса DNS на DNS-сервере в Интернете на тип записи CNAME, который сопоставляет доменное имя вашей организации с доменным именем вашего профиля диспетчера трафика. Дополнительные сведения см. в статье [Направление интернет-домена компании на домен диспетчера трафика](traffic-manager-point-internet-domain.md).
4. Трафик снова пойдет на конечные точки.

### Удаление профиля

1. Убедитесь, что запись ресурса DNS на DNS-сервере в Интернете больше не использует запись ресурса CNAME, указывающую на доменное имя вашего профиля диспетчера трафика.
2. Выберите профиль, который необходимо удалить. Для этого выделите его на странице диспетчера трафика, щелкнув столбец рядом с именем профиля. Не щелкайте имя профиля и стрелку рядом с этим именем, чтобы не переходить на страницу параметров профиля.
4. Выбрав профиль, в нижней части страницы нажмите кнопку **Удалить**.

## Просмотр журнала изменений профиля диспетчера трафика

Журнал изменений профиля диспетчера трафика можно просмотреть на портале управления в разделе "Службы управления".

### Чтобы просмотреть журнал изменений диспетчера трафика, выполните следующее:

1. В левой панели портала управления откройте раздел **Службы управления**.
2. На странице служб управления щелкните **Журналы операций**.
3. На странице "Журналы операций" вы можете с помощью фильтра выбрать журнал изменений для просмотра. Выбрав параметры фильтрации, установите флажок, чтобы просмотреть результаты.
   - Чтобы просмотреть изменения всех профилей, выберите свою подписку и временной диапазон, а затем - пункт **Диспетчер трафика** в раскрывающемся списке **Тип**.
   - Чтобы фильтровать по имени профиля, введите имя профиля в поле **Имя службы** или выберите его из раскрывающегося списка.
   - Для просмотра сведений о каждом отдельном изменении выберите строку с изменением, которое необходимо просмотреть, и в нижней части страницы нажмите кнопку **Сведения**. В окне **Сведения об операции** можно просмотреть XML-представление объекта API, который был создан или обновлен в рамках операции, а также скопировать код XML в буфер обмена.

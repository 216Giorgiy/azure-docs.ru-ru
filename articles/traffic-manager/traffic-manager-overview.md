# Что такое диспетчер трафика

Диспетчер трафика Microsoft Azure позволяет управлять распределением пользовательского трафика между конечными точками службы в разных центрах обработки данных по всему миру.

К конечным точкам службы, поддерживаемым диспетчером трафика Azure, относятся виртуальные машины, веб-приложения и облачные службы Azure. Можно также использовать диспетчер трафика Azure для внешних конечных точек, не относящихся к среде Azure.

Диспетчер трафика использует систему доменных имен (DNS) для перенаправления запросов пользователей к наиболее подходящей конечной точке на основе настроенного метода маршрутизации трафика и текущего представления работоспособности конечных точек. Затем клиенты подключаются непосредственно к подходящей конечной точке.

Диспетчер трафика поддерживает [ряд методов маршрутизации трафика](traffic-manager-routing-methods.md), подходящих для приложений с различными потребностями. Диспетчер трафика обеспечивает [проверки работоспособности и автоматическую отработку отказа конечных точек](traffic-manager-monitoring.md), позволяя создавать высокодоступные приложения, которые устойчивы к сбоям, включая сбой всего региона Azure.

## Преимущества диспетчера трафика

Диспетчер трафика позволяет решать указанные ниже задачи.

- **Повышение доступности критически важных приложений**. Диспетчер трафика позволяет обеспечить высокий уровень доступности критически важных приложений за счет мониторинга конечных точек, работающих в Azure, и автоматической отработки отказа в случае их сбоя.
- **Повышение скорости реагирования высокопроизводительных приложений**. Платформа Azure дает возможность запускать облачные службы и веб-сайты в распределенных по всему миру центрах обработки данных. Диспетчер трафика может уменьшить время отклика приложений, направляя их к конечной точке с наименьшим значением задержки сети для данного клиента.
- **Обновление и техническое обслуживание без простоя**. Можно легко выполнять обновление и другие запланированные операции обслуживания приложений без простоев для пользователей, используя диспетчер трафика для перенаправления трафика в альтернативные конечные точки в периоды технического обслуживания.
- **Объединение локальных и облачных приложений**. Диспетчер трафика поддерживает внешние конечные точки, не относящиеся к Azure, что позволяет использовать его в гибридных облачных и локальных развертываниях, включая сценарии пакетной передачи данных в облако, миграции в облако и отработки отказа в облако.
- **Распределение трафика в крупных и сложных развернутых службах**. Методы маршрутизации трафика можно комбинировать с помощью [вложенных профилей диспетчера трафика](traffic-manager-nested-profiles.md). Это позволяет создавать усложненные гибкие конфигурации маршрутизации трафика, соответствующие потребностям более крупных и сложных развернутых служб.

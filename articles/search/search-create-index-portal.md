# Создание индекса службы поиска Azure с помощью портала Azure
> [AZURE.SELECTOR]
- [Обзор](search-what-is-an-index.md)
- [Портал](search-create-index-portal.md)
- [.NET](search-create-index-dotnet.md)
- [REST](search-create-index-rest-api.md)

Эта статья поможет вам создать [индекс](search-what-is-an-index.md) службы поиска Azure с помощью портала Azure.

Перед выполнением инструкций, приведенных в этом руководстве, и созданием индекса следует [создать службу поиска Azure](search-create-service-portal.md).


## 1\. Переход к колонке службы поиска Azure
1. Щелкните "Все ресурсы" в меню слева на [портале Azure](https://portal.azure.com/#blade/HubsExtension/BrowseResourceBlade/resourceType/Microsoft.Search%2FsearchServices).
2. Выберите службу поиска Azure.

## 2\. Добавление индекса и указание его имени
1. Нажмите кнопку "Добавить индекс".
2. Укажите имя для индекса службы поиска Azure. Поскольку в этом руководстве мы создаем индекс для поиска гостиниц, мы назовем свой индекс "hotels".
  * Имя индекса должно начинаться с буквы и содержать только строчные буквы, цифры или дефисы ("-").
  * Как и имя службы, выбранное имя индекса также будет частью URL-адреса конечной точки, на которую будут отправляться HTTP-запросы для API-интерфейса службы поиска Azure.
3. Щелкните элемент "Поля", чтобы открыть новую колонку.

![](./media/search-create-index-portal/add-index.png)


## 3\. Создание и определение полей индекса
1. После выбора элемента "Поля" откроется новая колонка с формой для ввода определения индекса.
2. Добавьте поля в индекс с помощью этой формы.

  * Поле *key* типа Edm.String является обязательным для каждого индекса службы поиска Azure. Это поле ключа создается по умолчанию с именем поля "id". В нашем индексе мы изменили его на "hotelId".
  * Некоторые свойства схемы индекса можно задать только один раз и нельзя обновлять в будущем. Из-за этого любые обновления схемы, которые требуют переиндексирования (например изменение типов полей), пока невозможны после первоначальной настройки.
  * Мы тщательно выбрали значения свойств для каждого поля в зависимости от того, как мы планируем использовать их в приложении. При проектировании индекса учитывайте удобство работы с поиском и бизнес-потребности, поэтому каждому полю необходимо назначить [подходящие свойства](https://msdn.microsoft.com/library/azure/dn798941.aspx). Эти свойства управляют тем, какие функции поиска (фильтрация, фасетная навигация, сортировка, полнотекстовый поиск и т. д.) применяются к каждому полю. Например, вполне вероятно, что людей, которые ищут гостиницы, будут интересовать совпадения по ключевым словам в поле "description", поэтому мы включаем для него полнотекстовый поиск, задав значение для свойства "Searchable".
	* Можно также задать [анализатор языка](https://msdn.microsoft.com/ru-RU/library/azure/dn879793.aspx) для каждого поля, щелкнув на вкладке "Анализатор" в верхней части колонки. Ниже можно увидеть, что мы выбрали анализатор французского языка для поля в нашем индексе, которое было предназначено для текста на французском языке.

3. Щелкните **ОК** в колонке "Поля", чтобы подтвердить определения полей.
4. Щелкните **ОК** в колонке "Добавить индекс", чтобы сохранить и создать только что определенный индекс.

На снимках экрана ниже вы увидите, как мы задали имена полей и определили поля для индекса "hotels".

# <a name="monitor-usage-and-query-metrics-in-an-azure-search-service"></a>Мониторинг метрик использования и статистики в службе поиска Azure
Поиск Azure собирает статистику о выполнении запросов на уровне службы, включая количество запросов в секунду (QPS), задержку и процент запросов, сброшенных при превышении емкости тома. Метрики отображаются на портале в колонке мониторинга.

   ![Снимок экрана. Число запросов в секунду][5]

В этой статье описываются общие метрики для службы. Для получения сведений о действии на уровне индекса включите аналитику поискового трафика и используйте Power BI для просмотра анализа. Посетите страницу [Включение и использование аналитики поискового трафика](search-traffic-analytics.md) , чтобы приступить к работе.

## <a name="view-query-throughput-statistics"></a>Просмотр статистики пропускной способности запросов
Щелкните плитку "Мониторинг" на панели мониторинга службы и откройте колонку мониторинга.

   ![Плитка "Мониторинг"][2]

Запрос записывается на уровне службы, чтобы получить сведения о выполнении, задержке и регулировании. Метрики собираются постоянно, но может потребоваться несколько минут, пока последние сведения отобразятся на портале. 

Выберите плитку метрик, чтобы просмотреть подробные сведения, добавить оповещения, включить диагностику или изменить диаграмму.

  ![Команды в колонке "Метрика"][4]

### <a name="set-up-alerts"></a>Настройка оповещений
На странице сведений метрики можно настроить оповещения, активирующие отправку уведомления по электронной почте, если показатели выполнения запроса, задержки или регулирования превышают определенные критерии.

### <a name="enable-diagnostics"></a>Включение диагностики
При включении диагностики можно настроить место хранения диагностических данных, то, следует ли включать журналы операций и метрики, и срок хранения данных.

### <a name="change-chart-type-and-data-collection-interval"></a>Изменение типа диаграммы и интервала сбора данных
Для каждой метрики щелкните **Изменить**, чтобы переключиться с линейного графика на линейчатую диаграмму или изменить ось Х для отображения другого диапазона дат.

  ![Настройка диапазона времени][3]

## <a name="view-counts-and-resource-usage-in-the-portal"></a>Просмотр счетчиков и использования ресурсов на портале
Отслеживание роста индексов и размера документа может помочь заранее настроить емкость перед достижением верхнего предела, установленного для службы. 

Для отслеживания использования ресурсов просматривайте счетчики и статистику для службы на [портале](https://portal.azure.com). Эти сведения также можно получать программным образом. Для этого нужно создать пользовательское средство администрирования службы.

1. Войдите на [портал](https://portal.azure.com). 
2. Откройте служебную панель службы поиска Azure. Плитки для службы можно найти на домашней странице или загрузить их в службу, нажав «Обзор» на панели быстрого перехода. 

Раздел «Использование» включает индикатор, показывающий, какая часть доступных в настоящее время ресурсов используется. Сведения об ограничениях индексов, документов и объема хранилища для каждой службы см. в статье [Ограничения поиска Azure](search-limits-quotas-capacity.md).

  ![Плитка "Использование"][1]

> [!NOTE]
> На снимке экрана представлена бесплатная служба, которая может содержать максимум одну реплику и одну секцию, только 3 индекса, 10 000 документов или 50 МБ данных в зависимости от того, какое событие произойдет первым. Службы, созданные на уровне "Базовый" или "Стандартный", предоставляют значительно больше возможностей. Дополнительные сведения о выборе ценовой категории см. в статье [Выбор SKU или ценовой категории для службы поиска Azure](search-sku-tier.md).
>

### <a name="get-index-statistics-using-the-rest-api"></a>Получение статистики индексов с помощью интерфейса REST API
И API REST и пакет SDK для .NET службы поиска Azure обеспечивают программный доступ к показателям обслуживания.  Если вы используете [индексаторы](https://msdn.microsoft.com/library/azure/dn946891.aspx) для загрузки индекса из базы данных SQL Azure или DocumentDB, для получения требуемых показателей имеется дополнительный API-интерфейс. 

* [Получение статистики индексов](https://msdn.microsoft.com/library/azure/dn798942.aspx)
* [Подсчет документов](https://msdn.microsoft.com/library/azure/dn798924.aspx)
* [Получение состояния индексатора](https://msdn.microsoft.com/library/azure/dn946884.aspx)

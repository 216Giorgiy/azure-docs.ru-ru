# <a name="global-data-distribution-with-azure-cosmos-db"></a>Глобальное распределение данных с помощью Azure Cosmos DB

Многие приложения, которые сейчас работают, претендуют на доступность в глобальном масштабе. Они доступны круглосуточно из любой точки мира. Управление глобальным распространением данных, которые используются в этих приложениях, во время обеспечения высокой производительности и высокого уровня доступности — задача непростая. Для ее решения и была создана с нуля Cosmos DB — служба глобально распределенной базы данных.

Cosmos DB является основной службой Azure, которая по умолчанию доступна во всех [регионах Azure](https://azure.microsoft.com/global-infrastructure/regions/). Корпорация Майкрософт управляет центрами обработки данных Azure в более чем 50 регионах по всему миру и продолжает расширяться, чтобы соответствовать растущим требованиям пользователей. Служба Cosmos DB доступна 24 часа в сутки и 7 дней в неделю, позволяя вам сосредоточиться на приложениях. Создавая учетную запись Cosmos DB, определите, в каком из регионов она должна развертываться.

Пользователи Cosmos DB могут настраивать базы данных для глобального распределения и доступности в одном, нескольких или во всех регионах Azure. Чтобы снизить задержку, необходимо переместить данные ближе к местоположению пользователя, поэтому решение о количестве регионов и о том, в каком из них выполнять приложение, зависит от глобальной доступности приложения и расположения пользователей. База данных Cosmos DB прозрачно реплицирует все данные учетной записи во всех настроенных регионах. Cosmos DB предоставляет один образ системы базы данных Cosmos и контейнеров, что позволяет приложению использовать их для локальных операций чтения и записи. С помощью Cosmos DB можно в любое время добавлять или удалять регионы, связанные с учетной записью. Нет нужды останавливать работу приложения или повторно его развертывать. Приложение продолжит быть высокодоступным, постоянно предоставляя данные. Это возможно благодаря множественной адресации, предоставляемой службой.

## <a name="key-benefits-of-global-distribution"></a>Ключевые преимущества глобального распределения

**Легко создавайте глобальные приложения по шаблону "активный — активный"**. Благодаря возможности поддержки нескольких источников каждый регион доступен для записи (в дополнение к тому, чтобы быть читаемым), обеспечивая неограниченную эластичную масштабируемость записи, доступность чтения и записи на 99,999 % по всему миру и гарантированное быстрое чтение и запись, обслуживаемое менее чем за 10 миллисекунд, на уровне 99-го процентиля.  

Благодаря API-интерфейсам Azure Cosmos DB со множественной адресацией приложение получает актуальные данные о расположении ближайшего региона, что позволяет ему отправлять запросы в этот регион. Ближайший регион будет определен автоматически, без изменения конфигурации. По мере добавления регионов из учетной записи Cosmos DB и удаления их оттуда вам не нужно повторно развертывать приложение. Оно будет оставаться высокодоступным.

**Создание быстродействующих приложений**. Приложение может быть легко спроектировано для выполнения операций чтения и записи в режиме реального времени с задержкой, измеряемой единицами во всех регионах, которые были выбраны для базы данных.  Azure Cosmos DB выполняет внутреннюю обработку репликации данных между регионами таким образом, чтобы гарантировать уровень согласованности, выбранный для учетной записи Cosmos.

Многие приложения получают преимущества от улучшений производительности, которые идут вместе с возможностью выполнения операции записи в нескольких регионах (или в одном регионе). Для некоторых приложений, например тех, которым требуется строгая согласованность, в одном регионе предпочтительней использовать воронку записи всех операций. В Cosmos DB поддерживаются оба варианта конфигурации, как для оного региона так и для нескольких.

**Создание приложений высокой доступности**. Запуск базы данных в нескольких регионах улучшает ее доступность. В случае, когда один из регионов недоступен, запросы приложений будут обработаны в других регионах автоматически. Azure Cosmos DB предлагает возможность чтения и записи на уровне 99,999 % для баз данных, распределенных по нескольким регионам.

**Непрерывность бизнес-процессов во время региональных сбоев**. Во время региональных сбоев Azure Cosmos DB поддерживает [автоматическую отработку отказа](how-to-manage-database-account.md#enable-automatic-failover-for-your-cosmos-account). Кроме того, во время регионального сбоя Cosmos DB буде продолжать управлять пропускной способностью, доступностью, согласованностью и соглашением об уровне обслуживания в части пропускной способности. Чтобы убедиться, что целое приложения обладает высокой доступностью, Azure Cosmos DB предлагает ручное API отработки отказа, с помощью которого можно имитировать региональный сбой. Если использовать эти API, то можно регулярно выполнять тренировки непрерывности бизнеса и всегда быть готовыми.

**Чтение и запись данных в глобальном масштабе**. Благодаря поддержке несколько источников можно выполнять гибкое масштабирование при чтения и записи по всему миру. Поддержка нескольких источников гарантирует, что приложение настроенное в базе данных Azure Cosmos DB или в контейнере будет распространено по всем регионам и защищено с помощью [соглашения об уровне обслуживания с финансовыми гарантиями](https://aka.ms/acdbsla).

**Несколько точно определенных моделей согласованности**. Протокол репликации Azure Cosmos DB разработан для обеспечения в каждой из пяти точно определенных, практических и интуитивно понятных моделей компромисса между согласованностью и производительностью. Эти модели согласованности позволяют с легкостью создавать правильные глобально распределенные приложения.

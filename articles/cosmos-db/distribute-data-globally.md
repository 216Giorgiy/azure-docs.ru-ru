# <a name="global-data-distribution-with-azure-cosmos-db"></a>Глобальное распределение данных с помощью Azure Cosmos DB

Многие из современных приложений работают в глобальном масштабе. Эти приложения доступны круглосуточно из любой точки мира. Управление глобальным распространением данных, которые используются в этих приложениях, во время обеспечения высокой производительности и высокого уровня доступности — задача непростая. Azure Cosmos DB — это глобальная служба базы данных, которая обеспечивает высокую производительность и высокий уровень доступности. По этих причинах Azure Cosmos DB лучше всего подходит для этих приложений реального времени.

Cosmos DB является основной службой Azure, которая по умолчанию доступна во всех [регионах Azure](https://azure.microsoft.com/global-infrastructure/regions/). Корпорация Майкрософт управляет центрами обработки данных Azure в более чем 54 регионах по всему миру и продолжает расширяться, чтобы соответствовать растущим требованиям пользователей. Создавая учетную запись Cosmos DB, определите, в каком из регионов она должна развертываться. Служба Cosmos DB доступна 24 часа в сутки и 7 дней в неделю, позволяя вам сосредоточиться на приложениях.

Можно настраивать базы данных для глобального распределения и доступности в любом из 50 (или более) регионов Azure. Чтобы уменьшить задержку, необходимо переместить данные ближе к расположению пользователя. Выбор необходимых регионов зависит от глобального охвата приложения и расположения пользователя. База данных Cosmos DB прозрачно реплицирует все данные об учетной записи во всех настроенных регионах. Она предоставляет единый системный образ базы данных Cosmos и контейнеров, с помощью которых приложение может локально считывать и записывать информацию. С помощью Cosmos DB можно в любое время добавлять или удалять регионы, связанные с учетной записью. Чтобы добавить или удалить регион, приложение не нужно приостанавливать или повторно развертывать. Оно все время высокодоступное из-за множественной возможностей, которые предоставляет служба.

## <a name="key-benefits-of-global-distribution"></a>Ключевые преимущества глобального распределения

**Создание глобального приложения "активный — активный"**. С возможностью нескольких источников каждый регион является регионом записи (кроме того, что доступен для чтения). Возможности нескольких источников также гарантируют неограниченную масштабируемость при записи, доступность для чтения и записи на 99,999% по всему миру, а также гарантированные операции чтения или записи, обслуживаемые менее чем за 10 миллисекунд в 99-м процентиле.  

Благодаря API-интерфейсам Azure Cosmos DB со множественной адресацией приложение получает данные о расположении ближайшего региона, что позволяет ему отправлять запросы в тот регион. Ближайший регион будет определен автоматически, без изменения конфигурации. По мере добавления регионов из учетной записи Cosmos DB и удаления их оттуда вам не нужно повторно развертывать приложение. Оно будет оставаться высокодоступным.

**Создание быстродействующих приложений**. Приложение может быть легко спроектировано для выполнения операций чтения и записи в режиме реального времени с задержкой, измеряемой единицами во всех регионах, которые были выбраны для базы данных.  Azure Cosmos DB выполняет внутреннюю обработку репликации данных между регионами таким образом, чтобы гарантировать уровень согласованности, выбранный для учетной записи Cosmos.

Многие приложения получают преимущества от улучшений производительности, которые идут вместе с возможностью выполнения операции записи в нескольких регионах (или в одном регионе). Для некоторых приложений, которым требуется строгая согласованность, в одном регионе предпочтительней использовать воронку записи всех операций. Чтобы поддержать это приложение, Cosmos DB поддерживает оба варианта конфигурации как для одного региона, так и для нескольких.

**Создание приложений высокой доступности**. Запуск базы данных в нескольких регионах улучшает ее доступность. В случае, когда один из регионов недоступен, запросы приложений будут обработаны в других регионах автоматически. Azure Cosmos DB предлагает возможность чтения и записи на уровне 99,999 % для баз данных, распределенных по нескольким регионам.

**Непрерывность бизнес-процессов во время региональных сбоев**. Во время региональных сбоев Azure Cosmos DB поддерживает [автоматическую отработку отказа](how-to-manage-database-account.md#enable-automatic-failover-for-your-cosmos-account). Кроме того, во время регионального сбоя Cosmos DB буде продолжать управлять пропускной способностью, доступностью, согласованностью и соглашением об уровне обслуживания в части пропускной способности. Чтобы убедиться, что целое приложение обладает высокой доступностью, Azure Cosmos DB предлагает API переход на другой ресурс вручную, с помощью которого можно имитировать региональный сбой. При использовании этого API можно регулярно выполнять тренировки непрерывности бизнеса.

**Чтение и запись данных в глобальном масштабе**. Благодаря поддержке несколько источников можно выполнять гибкое масштабирование при чтения и записи по всему миру. Поддержка нескольких источников гарантирует, что приложение настроенное в базе данных Azure Cosmos DB или в контейнере будет распространено по всем регионам и защищено с помощью [соглашения об уровне обслуживания с финансовыми гарантиями](https://aka.ms/acdbsla).

**Несколько точно определенных моделей согласованности**. Протокол репликации Azure Cosmos DB разработан для обеспечения в каждой из пяти точно определенных, практических и интуитивно понятных моделей согласованности. Каждая модель согласованности имеет компромисс между согласованностью и производительностью. Эти модели согласованности позволяют с легкостью создавать правильные глобально распределенные приложения.

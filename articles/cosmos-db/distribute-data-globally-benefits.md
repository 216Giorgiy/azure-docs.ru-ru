# <a name="distribute-data-globally-with-azure-cosmos-db"></a>Глобальное распределение данных с помощью Azure Cosmos DB

В этой статье описаны основные преимущества глобального распределения данных, а также некоторые сценарии с выполнением в режиме реального времени, для которых необходимо глобальное распределение данных.

## <a name="key-benefits"></a>Основные преимущества

Ниже приведены ключевые преимущества, доступные для учетных записей, которые используют возможности глобального распределения данных Azure Cosmos DB.

* **Задержка, измеряемая единицами**: Azure Cosmos DB обеспечивает задержку чтения и записи < 10 мс на уровне 99-го процентиля, гарантированную [Соглашениями об уровне обслуживания с финансовым обеспечением](https://azure.microsoft.com/support/legal/sla/cosmos-db/).

* **Доступность на уровне "пять девяток"**: Azure Cosmos DB предлагает высокую доступность на уровне 99,999 % для операций чтения и записи, гарантированную [Соглашениями об уровне обслуживания с финансовым обеспечением](https://azure.microsoft.com/support/legal/sla/cosmos-db/).

* **Гибкое разрешение конфликтов**: для клиентов, использующих возможности работы с несколькими источниками, Azure Cosmos DB предоставляет три режима обработки конфликтов для обеспечения глобальной целостности данных и согласованности.

* **Настраиваемая согласованность**: Azure Cosmos DB поддерживает 5 различных [уровней согласованности](consistency-levels.md) с глобальным распределением и строгую согласованность для учетных записей с поддержкой нескольких источников.

* **Неявная отказоустойчивость** : благодаря репликации данных в несколько регионов приложения могут обладать высокой отказоустойчивостью к региональным сбоям.

## <a name="use-cases"></a>Варианты использования

Вот лишь несколько примеров сценариев, в которых могут использоваться возможности глобального распределения Azure Cosmos DB.

* **Приложения социальных сетей**: приложения для социальных сетей требуют низкой задержки, высокой доступности и масштабируемости для обеспечения оптимальных возможностей для пользователей по всему миру.

* **Интернет вещей** : географически распределенные развертывания пограничных устройств часто требуют отслеживания данных временных рядов из нескольких расположений. Каждое устройство можно связать с ближайшим регионом. Когда устройства перемещаются в другие расположения, их можно переопределить для записи в ближайший доступный регион.

* **Электронная коммерция** : приложения электронной коммерции требуют очень низкой задержки и высокой доступности. Географическое распределение данных с операциями чтения и записи помещает данные в ближайшие к пользователям регионы, что увеличивает скорость реагирования приложений. Доступность для операций чтения и записи в нескольких регионах обеспечивает большую общую доступность.

* **Обнаружение мошенничества или неполадок** : часто приложения, наблюдающие за действиями пользователя или учетной записи, распределены глобально и должны отслеживать несколько событий одновременно для обновления показателей и сохранения встроенных метрик риска.

* **Измерение** : использование учета и регулирования (например, вызовов API, транзакций в секунду, использованных минут) можно легко и просто реализовать в глобальном масштабе за счет поддержки нескольких источников в Azure Cosmos DB. Встроенный механизм разрешения конфликтов обеспечивает как точность учета, так и регулирование в режиме реального времени.

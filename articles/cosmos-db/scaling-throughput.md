# <a name="scaling-throughput-in-azure-cosmos-db"></a>Масштабирование пропускной способности в Azure Cosmos DB

В Azure Cosmos DB подготовленная пропускная способность представлена в виде единиц запросов в секунду (ЕЗ). Эти единицы запросов измеряют стоимость операций чтения и записи в контейнере Cosmos.

![Единицы запросов](./media/scale-throughput/figure1.png)

Вы можете подготовить ЕЗ в контейнере Cosmos или базе данных Cosmos. ЕЗ, подготовленные в контейнере, доступны исключительно для операций, выполняемых в этом контейнере. ЕЗ, подготовленные в базе данных, распределяются между всеми контейнерами в этой базе данных (за исключением контейнеров с отдельно назначенными ЕЗ).

Для гибкого масштабирования пропускной способности вы можете увеличить или уменьшить подготовленные ЕЗ за секунду в любое время. Дополнительные сведения см. в статье "Настройка и получение пропускной способности контейнеров и базы данных Azure Cosmos DB". Для глобального масштабирования пропускной способности вы можете добавлять или удалять регионы в своей учетной записи Cosmos в любое время. Дополнительные сведения см. в разделе руководства, посвященному добавлению или удалению регионов в учетной записи Cosmos. Связывание нескольких регионов с учетной записью Cosmos важно во многих сценариях, чтобы обеспечить низкую задержку и [высокий уровень доступности](high-availability.md) по всему миру.

## <a name="throughput-scaling-details"></a>Сведения о масштабировании пропускной способности

Если вы подготавливаете R единиц запросов в контейнере Cosmos (или базе данных), Cosmos DB обеспечивает доступность этих единиц запросов в *каждом* регионе, связанном с учетной записью Cosmos. Каждый раз при добавлении в учетную запись Cosmos нового региона Cosmos DB автоматически подготавливает R единиц запросов в новом добавленном регионе. Операции, выполняемые в контейнере Cosmos, гарантированно получают R единиц запросов в каждом из регионов, связанном с вашей учетной записью Cosmos. Вы не можете выборочно назначать ЕЗ для определенного региона. ЕЗ, подготовленные для контейнера (или базы данных) Cosmos, подготавливаются для всех регионов, связанных с учетной записью Cosmos.

Предположим, что для контейнера Cosmos настроено R единиц запросов и есть N регионов, связанных с учетной записью Cosmos, тогда:

- Если учетная запись Cosmos настроена с одним регионом записи, общее количество глобально доступных единиц запросов в контейнере — R x N.
- Если учетная запись Cosmos настроена с несколькими регионами записи, общее количество глобально доступных единиц запросов в контейнере — R x (N+1). Дополнительные R единиц запросов автоматически предоставляются для обработки конфликтов обновления и антиэнтропийного трафика в регионах.

Выбор [модели согласованности](consistency-levels.md) также влияет на пропускную способность. Пропускная способность для чтения при согласованности на уровне сеанса, согласованности с постоянным префиксом и итоговой согласованности примерно в два раза выше той, которая предоставляется уровнями строгой согласованности и "Ограниченное устаревание".

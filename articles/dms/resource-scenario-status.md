# <a name="status-of-migration-scenarios-supported-by-the-azure-database-migration-service"></a>Состояние сценариев миграции, поддерживаемых службой Azure Database Migration Service
Azure Database Migration Service предназначена для поддержки различных сценариев миграции (пар исходных и целевых объектов): автономной (однократной) и с подключением к сети (непрерывная синхронизация). Сценарии использования, предоставляемые Azure Database Migration Service, расширяются со временем. Регулярно добавляются новые сценарии. В этой статье перечислены сценарии миграции, которые в настоящее время поддерживаются службой Azure Database Migration Service, и состояние каждого сценария (закрытая [или ограниченная] предварительная версия, общедоступная предварительная версия, общедоступная версия).

## <a name="offline-versus-online-migrations"></a>Автономная миграция и миграция с подключением к сети
При переносе баз данных в Azure с использованием Azure Database Migration Service операцию переноса можно выполнять автономно или с подключением к сети. При использовании *автономной* миграции простой приложения начинается в то же время, что и миграция. Для миграций *с подключением к сети* время простоя ограничено временем, необходимым для перехода в новую среду после завершения миграции. Рекомендуется выполнить тестирование автономной миграции, чтобы определить, допустим ли простой, и, если нет, выполнить миграцию с подключением к сети.

## <a name="migration-scenario-status"></a>Состояние сценария миграции
Состояние каждого сценария миграции, поддерживаемого Azure Database Migration Service, зависит от времени. Как правило, сценарии впервые появляются в **закрытой предварительной версии**, и для того, чтобы воспользоваться преимуществами возможностей, клиенту нужно отправить заявку через [веб-сайт предварительной версии DMS](https://aka.ms/dms-preview). Когда этап закрытой предварительной версии завершается, состояние сценария изменяется на **Общедоступная предварительная версия**. Все пользователи Azure Database Migration Service могут воспользоваться преимуществами сценариев миграции, доступных в общедоступной предварительной версии. Тем не менее сценарии миграции могут быть доступны не во всех регионах, а функциональные возможности могут быть подвержены дополнительным изменениям перед окончательным выпуском. Когда сценарий миграции становится **общедоступным** (последнее состояние выпуска), функциональные возможности готовы и доступны всем пользователям Azure Database Migration Service.

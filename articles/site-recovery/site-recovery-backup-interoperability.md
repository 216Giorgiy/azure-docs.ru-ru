# <a name="about-site-recovery-and-backup-interoperability"></a>О Site Recovery и резервного копирования взаимодействия

Эта статья содержит рекомендации для успешного использования резервного копирования виртуальных Машин IaaS Azure и виртуальных Машин Azure аварийного восстановления.

## <a name="azure-backup"></a>Служба архивации Azure

Служба Azure Backup помогает защитить данные для локальных серверов, виртуальных машин, виртуализованных рабочих нагрузок, серверов SQL Server, серверы SharePoint и многое другое. Azure Site Recovery организует и контролирует аварийное восстановление виртуальных машин Azure, локальных виртуальных машин и физических серверов.

## <a name="azure-site-recovery"></a>Azure Site Recovery

Это можно настроить Azure Backup и Azure Site Recovery для виртуальной Машины или группы виртуальных машин. Оба этих продукта с возможностью взаимодействия. Далее приведены несколько сценариев, где взаимодействие между резервным копированием и Azure Site Recovery становится важным.

### <a name="file-backuprestore"></a>Файл резервного копирования и восстановления

Если резервное копирование и репликация включено, и создается резервная копия, нет проблем с восстановлением любые файлы на стороне источника виртуальной Машины или группы виртуальных машин. Репликация будет продолжена с без изменений в состояние работоспособности репликации обычным образом.

### <a name="disk-backuprestore"></a>Диск резервного копирования и восстановления

Если восстановление диска из резервной копии защиты виртуальной машины не будет включить заново.

### <a name="vm-backuprestore"></a>Резервное копирование и восстановление виртуальных Машин

Резервное копирование и восстановление виртуальной Машины или группы виртуальных машин не поддерживается. Для его работы, необходимо будет снова включить защиту.

**Сценарий** | **Поддерживается службой Azure Site Recovery?** | **Чтобы избежать этого, если таковые имеются**  
--- | --- | ---
Архивация файлов и папок | Yes | Не применяется
Резервное копирование на диск | В настоящее время не | Отключите и включите защиту
Резервное копирование виртуальных Машин | Нет  | Отключите и включите защиту

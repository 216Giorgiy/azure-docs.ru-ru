# <a name="whats-new-in-site-recovery"></a>Новые возможности в Site Recovery

Служба [Azure Site Recovery](site-recovery-overview.md) постоянно обновляется и совершенствуется. Эта статья содержит сведения о последних выпусках, новых функциях и новом содержимом, чтобы помочь вам оставаться в курсе событий. Эта страница регулярно обновляется.

Если у вас есть предложения и отзывы по функциям Site Recovery, мы с радостью [ознакомимся с ними](https://feedback.azure.com/forums/256299-site-recovery).

## <a name="q1-2019"></a>1-й квартал 2019 г.

### <a name="linux-support"></a>Поддержка Linux

[Накопительный пакет обновления 32](https://support.microsoft.com/help/4485985/update-rollup-32-for-azure-site-recovery) предоставляет обновление для агентов и поставщиков Site Recovery. Обновления добавляют поддержку Linux следующим образом:

- **Аварийное восстановление с помощью виртуальных машин Azure**: RedHat Workstation 6/7, новые версии ядра для Ubuntu, Debian и SUSE.
- **Аварийное восстановление виртуальных машин VMware и физических серверов в Azure**: RedHat Enterprise Linux 7.6, RedHat Workstation 6/7, Oracle Linux 6.10/7.6, новые версии ядра для Ubuntu, Debian и SUSE.



## <a name="q4-2018"></a>4-й квартал 2018 г.

## <a name="pricing-calculator-for-azure-vm-disaster-recovery"></a>Калькулятор цен для аварийного восстановления виртуальных машин Azure.

Аварийное восстановление виртуальных машин Azure влечет за собой расходы на лицензирование виртуальных машин, а также расходы на сеть и хранилище. Azure [предоставляет калькулятор цен](https://aka.ms/a2a-cost-estimator), который поможет вам определить эти расходы. Теперь Site Recovery предоставляет [приблизительную стоимость](https://aka.ms/a2a-cost-estimator) примера развертывания на основе трехуровневого приложения с использованием шести виртуальных машин с 12 дисками HDD (цен. категория "Стандартный") и 6 дисками SSD (цен. категория "Премиум"). В примере предполагается изменение данных на 10 ГБ в день для дисков ценовой категории "Стандартный" и 20 ГБ для дисков ценовой категории "Премиум". Для конкретного развертывания можно изменить переменные для оценки затрат. Вы можете указать количество виртуальных машин, количество и тип управляемых дисков, а также ожидаемую общую скорость обмена данными на виртуальных машинах. Кроме того, можно применить коэффициент сжатия для оценки стоимости пропускной способности. [Прочтите объявление](https://azure.microsoft.com/blog/know-exactly-how-much-it-will-cost-for-enabling-dr-to-your-azure-vm/).

### <a name="support-for-azure-vms-in-zones"></a>Поддержка виртуальных машин Azure в зонах

Зоны доступности Azure — это уникальные физические расположения в пределах одного региона Azure. Каждая зона состоит из одного или нескольких центров обработки данных, оснащенных независимыми системами электроснабжения, охлаждения и сетевого взаимодействия. Теперь вы можете включить репликацию для виртуальных машин Azure и установить целевой регион для отработки отказа одного экземпляра виртуальной машины, виртуальной машины в группе доступности или виртуальной машины в зоне доступности. Этот параметр не влияет на репликацию. [Прочтите объявление](https://azure.microsoft.com/blog/disaster-recovery-of-zone-pinned-azure-virtual-machines-to-another-region/).
 
### <a name="disaster-recovery-for-encrypted-vms"></a>Аварийное восстановление зашифрованных виртуальных машин

Site Recovery поддерживает виртуальные машины Azure, зашифрованные с помощью шифрования дисков Azure (ADE) для приложений Azure AD. [Узнайте больше](azure-to-azure-how-to-enable-replication-ade-vms.md).

### <a name="disaster-recovery-for-vms-using-accelerated-networking"></a>Аварийное восстановление виртуальных машин с использованием функции ускоренной работы в сети

Функция ускоренной работы в сети позволяет использовать виртуализацию ввода-вывода с единым корнем (SR-IOV), повышая тем самым производительность сети виртуальной машины. При включении репликации для виртуальной машины Azure Site Recovery определяет, включена ли ускоренная работа в сети. Если да, то после отработки отказа Site Recovery автоматически настраивает ускоренную работу в сети на целевой реплике виртуальной машины Azure как для [Windows](https://docs.microsoft.com/azure/virtual-network/create-vm-accelerated-networking-powershell#enable-accelerated-networking-on-existing-vms), так и для [Linux](https://docs.microsoft.com/azure/virtual-network/create-vm-accelerated-networking-cli#enable-accelerated-networking-on-existing-vms). [Узнайте больше](azure-vm-disaster-recovery-with-accelerated-networking.md).

### <a name="automatic-updates-for-the-mobility-service-extension"></a>Автоматические обновления расширения Службы Mobility Service

В Site Recovery добавлена возможность автоматического обновления расширения Службы Mobility Service. Расширение Службы Mobility Service устанавливается на каждой виртуальной машине Azure, реплицированной с помощью Site Recovery. При включении репликации необходимо выбрать, разрешать ли Site Recovery управлять обновлениями расширения. Обновления не требуют перезагрузки виртуальной машины и не влияют на репликацию. [Узнайте больше](azure-to-azure-autoupdate.md).

### <a name="support-for-standard-ssd-disks"></a>Поддержка дисков SSD (цен. категория "Стандартный")

Диски Azure на [основе твердотельных накопителей (SSD) ценовой категории "Стандартный"](https://docs.microsoft.com/azure/virtual-machines/windows/disks-standard-ssd) — это экономически эффективный вариант хранилища для таких приложений, как веб-серверы, которым требуется стабильная производительность при более низких уровнях операций ввода-вывода в секунду. Они сочетают в себе элементы SSD (цен. категория "Премиум") и HDD (цен. категория "Стандартный"). Site Recovery обеспечивает аварийное восстановление виртуальных машин Azure с использованием диска SSD (цен. категория "Стандартный"). По умолчанию тип диска сохраняется после отработки отказа в целевой регион.

### <a name="support-for-azure-storage-firewall"></a>Поддержка брандмауэра для хранилища Azure

Вы можете защитить учетные записи хранения Azure в определенном наборе сетей, включив правила брандмауэра для учетной записи. Настройте учетные записи хранения для запрета трафика из внутренних сетей и Интернета по умолчанию, а затем предоставьте доступ к трафику из определенных виртуальных сетей. Site Recovery поддерживает репликацию виртуальных машин с неуправляемыми дисками в учетных записях хранения с поддержкой брандмауэра во вторичный регион. В целевом регионе для неуправляемых дисков можно выбрать учетные записи хранения с включенными брандмауэрами. Вы также можете ограничить доступ к учетной записи хранения кэша, ограничив сетевой доступ только той сетью, в которой расположены исходные виртуальные машины. Обратите внимание, что вы должны [разрешить доступ](https://docs.microsoft.com/azure/storage/common/storage-network-security#exceptions) для доверенных служб Майкрософт.

### <a name="linux-support"></a>Поддержка Linux

#### <a name="update-rollup-28"></a>Накопительный пакет обновления 28

[Накопительный пакет обновления 28](https://support.microsoft.com/help/4460079/update-rollup-28-for-azure-site-recovery) предоставляет обновление для агентов и поставщиков Site Recovery. Обновления добавляют поддержку Linux следующим образом:

- **Аварийное восстановление с помощью виртуальных машин Azure**: RedHat Enterprise Linux 6.10, CentOS 6.10. Теперь поддерживаются виртуальные машины на основе Linux, которые используют таблицы разделов GUID (GPT) в устаревшем режиме совместимости с BIOS.
- **Аварийное восстановление виртуальных машин VMware и физических серверов в Azure**: RedHat Enterprise Linux 6.10, CentOS 6.10. Теперь поддерживаются виртуальные машины на основе Linux, которые используют таблицы разделов GUID (GPT) в устаревшем режиме совместимости с BIOS.

#### <a name="update-rollup-27"></a>Накопительный пакет обновления 27

[Накопительный пакет обновления 28](https://support.microsoft.com/help/4460079/update-rollup-28-for-azure-site-recovery) предоставляет обновление для агентов и поставщиков Site Recovery. Обновления добавляют поддержку Linux следующим образом:

- **Аварийное восстановление с помощью виртуальных машин Azure**: Red Hat Enterprise Linux 7.5
- **Аварийное восстановление виртуальных машин VMware и физических серверов в Azure**: SUSE Linux Enterprise Server 12, Red Hat Enterprise Linux 7.5

### <a name="support-for-azure-vms-running-on-windows-server-2016"></a>Поддержка виртуальных машин Azure под управлением Windows Server 2016

Виртуальные машины Azure под управлением Windows Server 2016 можно реплицировать в регионах Azure с помощью Azure Site Recovery.

### <a name="support-for-azure-vms-running-storage-spaces-direct"></a>Поддержка виртуальных машин Azure, на которых используется функция "Локальные дисковые пространства"

[Локальные дисковые пространства](https://docs.microsoft.com/windows-server/storage/storage-spaces/storage-spaces-direct-overview) (доступные начиная с Windows Server 2016) группируют диски вместе в пул хранения, а затем используют емкость из пула для создания дисковых пространств. Дисковые пространства можно использовать на автономной виртуальной машине, в гостевом кластере [виртуальных машин Azure](https://docs.microsoft.com/windows-server/storage/storage-spaces/storage-spaces-direct-in-vm), используя локальное хранилище на каждом узле кластера или общее хранилище в кластере.

# <a name="manage-virtual-machine-network-interfaces-for-on-premises-to-azure-scenarios"></a>Управление виртуальной машины сетевых интерфейсов для локальной среды в Azure сценариев

Виртуальную машину в Azure необходимо иметь по крайней мере один сетевой интерфейс, подключенных к нему и может иметь любое количество сетевых интерфейсов прикрепленные к нему как поддерживает размер виртуальной Машины. По умолчанию первый сетевой интерфейс, подключенный к виртуальной машине Azure представляет собой первичный сетевой интерфейс. Все другие интерфейсы сети виртуальной машины являются вторичными сетевыми интерфейсами. По умолчанию весь исходящий трафик от виртуальной машины отправляется IP-адрес, назначенный основной IP-конфигурации первичный сетевой интерфейс.

В локальной среде виртуальных машин и серверы могут иметь несколько сетевых интерфейсов для разных сетей в среде. Различные сети обычно используются для выполнения конкретных операций, таких как обновление, обслуживание, доступ к Интернету, и т. д. При переносе или отказа в Azure в локальной среде, имейте в виду, что сетевые интерфейсы в той же виртуальной машине необходимо подключены к той же виртуальной сети.

По умолчанию Site Recovery создает большого количества сетевых интерфейсов на виртуальной машине Azure как подключенный к серверу в локальной среде. Можно избежать создания избыточных сетевых интерфейсов во время миграции или отработки отказа, изменив параметры сетевого интерфейса с настройками для реплицируемой виртуальной машины.

## <a name="select-the-target-network"></a>Выберите целевую сеть

Для VMware и физических компьютеров и виртуальных машин Hyper-V (без VMM) можно указать целевой виртуальной сети для отдельных виртуальных машин. Для виртуальных машин Hyper-V, управляемых с помощью VMM [сетевое сопоставление](site-recovery-network-mapping.md) используется для сопоставления сетей виртуальных Машин на исходном сервере VMM и целевыми сетями Azure.

1. В области «Реплицированные элементы» в хранилище служб восстановления щелкните реплицированных элемент, для доступа к параметрам для этого элемента реплицированные.

2. Откройте вкладку «Вычислительных и сетевых» для доступа к сетевые параметры для реплицированных элемента.

3. В разделе «Свойства сети» выберите виртуальную сеть из списка имеющихся сетевых интерфейсов.

    ![Вычисления и сеть](./media/site-recovery-manage-network-interfaces-on-premises-to-azure/compute-and-network.png)

Изменение целевой сети влияет на всех сетевых интерфейсов для этой конкретной виртуальной машины.

Для облаков VMM изменение сетевого сопоставления влияет на все виртуальные машины и их интерфейсы сети.

## <a name="select-the-target-interface-type"></a>Выберите целевой тип интерфейса

Колонка «Вычислительных и сетевых» в разделе «Сетевые интерфейсы» можно просмотреть и изменить параметры сетевого интерфейса и укажите тип целевого сетевого интерфейса.

- Объект **основной** сетевой интерфейс является обязательным для отработки отказа.
- Другие выбранные сетевые интерфейсы, если какой-либо, **получателя** сетевых интерфейсов.
- Выберите **используют** Запрет создания отказоустойчивой сетевой интерфейс.

По умолчанию при выборе Включение репликации, восстановление сайта все обнаруженные сетевые интерфейсы на локальном сервере, один — как «Основной», а все остальные пометки, как «Получателя» сетевые интерфейсы. Все последующие интерфейсов, добавленных на локальном сервере, помечаются «» по умолчанию не используется. При добавлении нескольких сетевых интерфейсов, должен быть выбран правильный виртуальной машины Azure целевой размер для размещения всех необходимых сетевых интерфейсов.

## <a name="modifying-network-interface-settings"></a>Изменение параметров сетевого интерфейса

Можно изменять подсети и IP-реплицированных элемента сетевых интерфейсов. Если IP-адрес не указан, Site Recovery назначить следующий доступный IP-адрес из подсети для сетевого интерфейса на переход на другой ресурс.

1. Щелкните любой доступной сетевой интерфейс, чтобы открыть колонку параметров сетевого интерфейса.

2. Выберите нужный подсети в списке доступных сетей.

3. Введите нужный IP-адрес (при необходимости).

    ![Параметры сетевого интерфейса](./media/site-recovery-manage-network-interfaces-on-premises-to-azure/network-interface-settings.png)

4. Нажмите кнопку "ОК" для завершения редактирования и возврата к колонке «Вычислений и сети».

5. Повторите шаги 1 – 4 для других сетевых интерфейсов.

6. Чтобы сохранить все изменения, нажмите кнопку «Сохранить».

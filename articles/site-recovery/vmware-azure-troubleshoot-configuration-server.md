# <a name="troubleshoot-configuration-server-issues"></a>Устранение неполадок, связанных с сервером конфигурации

Эта статья поможет устранить неполадки при развертывании сервера конфигурации [Azure Site Recovery](site-recovery-overview.md) и управлении им. При настройке аварийного восстановления локальных виртуальных машин VMware и физических серверов в Azure с помощью Azure Site Recovery используется сервер конфигурации, выступающий как сервер управления. В следующих разделах рассматриваются наиболее распространенные сбои, возникающие при добавлении нового сервера конфигурации и управлении сервером конфигурации.

## <a name="registration-failures"></a>Ошибки регистрации

Исходный компьютер регистрируется с сервером конфигурации во время установки агента мобильности. Любые сбои при выполнении этого шага можно отлаживать, следуя приведенным ниже рекомендациям.

1. Перейдите к файлу C:\ProgramData\ASR\home\svsystems\var\configurator_register_host_static_info.log. ProgramData может быть скрытой папкой. Если ее не удается найти, попробуйте отменить скрытие папки. Сбои могут возникать из-за нескольких проблем.
2. Найдите строку No Valid IP Address found (Не найден допустимый IP-адрес). Если вы ее нашли,
    - проверьте, совпадает ли идентификатор запрошенного узла с исходным компьютером.
    - Исходный компьютер должен иметь по крайней мере один IP-адрес, назначенный физическому сетевому адаптеру для успешной регистрации агента на сервере конфигурации.
    - Выполните команду на исходном компьютере `> ipconfig /all` (для ОС Windows) и `# ifconfig -a` (для ОС Linux) для получения всех IP-адресов исходного компьютера.
    - Обратите внимание, что для регистрации агента требуется допустимый IP-адрес v4, назначенный физическому сетевому адаптеру.
3. Если строка выше не найдена, выполните поиск строки "Reason"=>"NULL". Если она найдена,
    - эта ошибка возникает, если исходный компьютер использует пустой узел для регистрации на сервере конфигурации.
    - После устранения проблем выполните инструкции [здесь](vmware-azure-troubleshoot-configuration-server.md#register-source-machine-with-configuration-server), чтобы вручную повторить попытку регистрации.
4. Если строка выше не найдена, перейдите на исходный компьютер и проверьте журнал C:\ProgramData\ASRSetupLogs\UploadedLogs\* ASRUnifiedAgentInstaller.log. ProgramData может быть скрытой папкой. Если ее не удается найти, попробуйте отменить скрытие папки. Сбои могут возникать из-за нескольких проблем. Найдите строку post request: (7) - Couldn't connect to server (Запрос post: (7) — не удалось подключиться к серверу). Если она найдена,
    - устраните ошибки сети между исходным компьютером и сервером конфигурации. Убедитесь, что сервер конфигурации достижим с исходного компьютера, с помощью сетевых средств, таких как проверка связи, трассировка маршрута, веб-браузер и т. д. Убедитесь, что исходный компьютер может получить доступ к серверу конфигурации через порт 443.
    - Проверьте наличие правил брандмауэра на исходном компьютере, которые блокируют подключение между исходным компьютером и сервером конфигурации. Обратитесь к администратору сети, чтобы разблокировать проблемы с подключением.
    - Убедитесь, что папки, указанные [здесь](vmware-azure-set-up-source.md#azure-site-recovery-folder-exclusions-from-antivirus-program), исключены из антивирусного программного обеспечения.
    - После устранения проблем с сетью повторите регистрацию, следуя инструкциям [здесь](vmware-azure-troubleshoot-configuration-server.md#register-source-machine-with-configuration-server).
5. Если строка не найдена, в том же журнале найдите строку request: (60) - Peer certificate cannot be authenticated with given CA certificates. (запрос: (60) — одноранговый сертификат не может пройти проверку подлинности с помощью заданных сертификатов ЦС). Если она найдена, 
    - эта ошибка может возникать, если истек срок действия сертификата сервера конфигурации, или исходный компьютер не поддерживает протокол TLS 1.0 и протоколы SSL, или есть брандмауэр, который блокирует обмен данными по протоколу SSL между исходным компьютером и сервером конфигурации.
    - Чтобы устранить проблему, подключитесь к IP-адресу сервера конфигурации через веб-браузер на исходном компьютере с помощью URI https://<CSIPADDRESS>:443/. Убедитесь, что исходный компьютер может получить доступ к серверу конфигурации через порт 443.
    - Проверьте наличие правил брандмауэра на исходном компьютере, которые блокируют подключение между исходным компьютером и сервером конфигурации. Обратитесь к администратору сети, чтобы разблокировать проблемы с подключением.
    - Убедитесь, что папки, указанные [здесь](vmware-azure-set-up-source.md#azure-site-recovery-folder-exclusions-from-antivirus-program), исключены из антивирусного программного обеспечения.  
    - После устранения проблем повторите регистрацию, следуя инструкциям [здесь](vmware-azure-troubleshoot-configuration-server.md#register-source-machine-with-configuration-server).
6. В Linux, если значение платформы из <INSTALLATION_DIR>/etc/drscout.conf повреждено, регистрация завершается сбоем. Чтобы определить это, откройте журнал /var/log/ua_install.log. Вы найдете строку Aborting configuration as VM_PLATFORM value is either null or it is not VmWare/Azure (Прерывание конфигурации, поскольку значение VM_PLATFORM является NULL или не является VmWare/Azure). Платформа должна иметь значение VmWare или Azure. Так как файл drscout.conf поврежден, мы рекомендуем [удалить](vmware-physical-mobility-service-overview.md#uninstall-the-mobility-service) агент мобильности и повторно установить его. При сбое удаления выполните следующие действия.
    - Откройте файл Installation_Directory/uninstall.sh и закомментируйте вызов функции *StopServices*.
    - Откройте файл Installation_Directory/Vx/bin/uninstall и закомментируйте вызов функции `stop_services`.
    - Откройте файл Installation_Directory/Fx/uninstall и закомментируйте весь раздел, который пытается остановить службу Fx.
    - Теперь попробуйте [удалить](vmware-physical-mobility-service-overview.md#uninstall-the-mobility-service) агент мобильности. После успешного удаления перезагрузите систему и попробуйте снова установить агент.

## <a name="installation-failure---failed-to-load-accounts"></a>Сбой установки — не удалось загрузить учетные записи

Эта ошибка возникает, если службе не удается прочитать данные из транспортного соединения во время установки агента мобильности и регистрации на сервере конфигурации. Чтобы устранить проблему, включите протокол TLS 1.0 на исходном компьютере.

## <a name="change-ip-address-of-configuration-server"></a>Изменение IP-адреса сервера конфигурации

Настоятельно рекомендуется не изменять IP-адрес сервера конфигурации. Все IP-адреса, назначенные серверу конфигурации, должны быть статическими IP-адресами, а не IP-адресами DHCP.

## <a name="acs50008-saml-token-is-invalid"></a>"ACS50008: недопустимый токен SAML

Чтобы эта ошибка не возникала, системные часы не должны опережать местное время или отставать от него более чем на 15 минут. Для завершения регистрации перезапустите установщик.

## <a name="failed-to-create-certificate"></a>Не удалось создать сертификат

Не удалось создать сертификат, необходимый для проверки подлинности Site Recovery. Программу установки следует повторно запустить от имени локального администратора.

## <a name="register-source-machine-with-configuration-server"></a>Регистрация исходного компьютера на сервере конфигурации

### <a name="if-source-machine-has-windows-os"></a>Если на исходном компьютере установлена ОС Windows

Запустите следующую команду на исходном компьютере.

```
  cd C:\Program Files (x86)\Microsoft Azure Site Recovery\agent
  UnifiedAgentConfigurator.exe  /CSEndPoint <CSIP> /PassphraseFilePath <PassphraseFilePath>
  ```
**Параметр** | **Дополнительные сведения**
--- | ---
Использование | UnifiedAgentConfigurator.exe  /CSEndPoint <CSIP> /PassphraseFilePath <PassphraseFilePath>
Журналы конфигурации агента | Журналы находятся в папке %ProgramData%\ASRSetupLogs\ASRUnifiedAgentConfigurator.log.
/CSEndPoint | Обязательный параметр. Указывает IP-адрес сервера конфигурации. Используйте любой допустимый IP-адрес.
/PassphraseFilePath |  (Обязательный параметр.) Расположение файла с парольной фразой. Используйте любой допустимый локальный путь к файлу или UNC.

### <a name="if-source-machine-has-linux-os"></a>Если на исходном компьютере установлена ОС Linux

Запустите следующую команду на исходном компьютере.

```
  /usr/local/ASR/Vx/bin/UnifiedAgentConfigurator.sh -i <CSIP> -P /var/passphrase.txt
  ```
**Параметр** | **Дополнительные сведения**
--- | ---
Использование | cd /usr/local/ASR/Vx/bin<br/><br/> UnifiedAgentConfigurator.sh -i <CSIP> -P <PassphraseFilePath>
-i | Обязательный параметр. Указывает IP-адрес сервера конфигурации. Используйте любой допустимый IP-адрес.
-P |  (Обязательный параметр.) Полный путь к файлу, в котором хранится парольная фраза. Используйте любую допустимую папку

#  Что такое Site Recovery?

Вас приветствует служба Azure Site Recovery! В этой статье приведен краткий обзор службы Site Recovery. Также вы узнаете, как это решение помогает реализовать стратегию непрерывности бизнес-процессов и аварийного восстановления (BCDR).

В Azure предлагаются две модели развертывания для создания ресурсов и работы с ними: [модель диспетчера ресурсов и классическая модель](../resource-manager-deployment-model.md). Информация, приведенная в этой статье, относится к обеим моделям. Для большинства новых развертываний Майкрософт рекомендует использовать модель диспетчера ресурсов.

## Обзор

Важной частью корпоративной стратегии BCDR является решение о том, как будет обеспечиваться выполнение корпоративных рабочих нагрузок и приложений при возникновении запланированных и незапланированных простоев.

Служба Site Recovery помогает сделать это, координируя процессы репликации, отработки отказа и восстановления рабочих нагрузок и приложений. Кроме того, она обеспечивает их доступность из дополнительного расположения, если основное выходит из строя.

## Зачем использовать Site Recovery? 

Ниже перечислены преимущества использования службы Site Recovery для вашего бизнеса.

- **Упрощение стратегии BCDR**. Служба Site Recovery позволяет легко выполнять репликацию, отработку отказа и восстановление нескольких бизнес-приложений и рабочих нагрузок из одного расположения. Хотя служба Site Recovery координирует процессы репликации и отработки отказа, она не перехватывает данные приложения и не получает какие-либо сведения о нем.
- **Гибкая репликация**. С помощью Site Recovery можно реплицировать рабочие нагрузки, выполняющиеся на виртуальных машинах Hyper-V, виртуальных машинах VMware и физических серверах Windows и Linux. 
- **Простая отработка отказа и быстрое восстановление**. Служба Site Recovery позволяет выполнять тестовую отработку отказа для поддержки аварийного восстановления, не влияя на рабочие среды. Кроме того, вы можете запускать запланированные процедуры отработки отказа без потери данных при ожидаемых простоях, а также незапланированные процедуры отработки отказа с минимальной потерей данных (в зависимости от частоты репликации) на случай непредвиденных сбоев. После отработки отказа можно выполнить восстановление размещения на основные сайты. Служба Site Recovery предоставляет планы восстановления, которые могут включать скрипты и рабочие книги службы автоматизации Azure. С их помощью вы можете настраивать восстановление и отработку отказа многоуровневых приложений. 
- **Отсутствие необходимости в дополнительном центре обработки данных**. Вы можете выполнять репликацию на дополнительный локальный сайт или в Azure. Используя Azure в качестве целевого расположения при аварийном восстановлении, вы можете сократить расходы и затраты на обслуживание дополнительного сайта. При этом реплицированные данные будут храниться в хранилище Azure, обеспечивающем максимально возможную устойчивость.
- **Интеграция с существующими технологиями BCDR**. Служба Site Recovery взаимодействует с другими компонентами приложений в рамках стратегии BCDR. Например, вы можете использовать Site Recovery для защиты серверной части SQL Server корпоративных рабочих нагрузок, включая встроенную поддержку для SQL Server AlwaysOn для управления отработкой отказа групп доступности.

## Какие рабочие нагрузки я могу защитить?

Служба Site Recovery позволяет реализовать стратегию BCDR с учетом особенностей приложений, обеспечивая согласованное выполнение рабочих нагрузок и приложений при возникновении сбоев. Преимущества службы Site Recovery:

- **Моментальные снимки с согласованием приложений**. Репликация с использованием моментальных снимков с согласованием одноуровневых или n-уровневых приложений. **Практически синхронная репликация**. Частота репликации составляет всего 30 секунд для Hyper-V, а для VMware поддерживается непрерывная репликация. **Интеграции с SQL Server AlwaysOn**. Вы можете управлять отработкой отказа групп доступности с помощью планов восстановления Site Recovery. 
- **Гибкие планы восстановления**. Вы можете создавать и настраивать планы восстановления с помощью внешних скриптов, выполняемых вручную действий, а также модулей Runbook службы автоматизации Azure, которые позволяют быстро восстановить весь стек приложений.
- **Библиотека службы автоматизации**. Богатая библиотека службы автоматизации Azure предоставляет готовые скрипты с учетом особенностей приложений, которые можно скачать и интегрировать со службой Site Recovery. -** Простое управление сетью**. Дополнительные возможности управления сетью в Site Recovery и Azure упрощают требования к сети со стороны приложений, включая резервирование IP-адресов, настройку балансировщиков нагрузки и интеграцию диспетчера трафика Azure для эффективного выполнения сетевых переключений.

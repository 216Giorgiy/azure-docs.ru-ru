# <a name="what-is-site-recovery?"></a>Что такое Site Recovery?
Вас приветствует служба Azure Site Recovery! В этой статье содержится краткий обзор службы Site Recovery и сведения о роли этой службы для вашего бизнеса.

Вашей организации нужна стратегия обеспечения непрерывности бизнеса и аварийного восстановления (BCDR), которая обеспечит безопасность и доступность приложений, рабочих нагрузок и данных при запланированных и незапланированных простоях и поможет как можно быстрее возвращаться к нормальному режиму работы. Site Recovery — это служба Azure, которая помогает реализовать эту стратегию.

Site Recovery координирует репликацию рабочих нагрузок, выполняющихся на локальных физических серверах и виртуальных машинах. Серверы и виртуальные машины можно реплицировать из основного центра обработки данных в облако (Azure) или дополнительный центр обработки данных. При возникновении сбоев на основном сайте происходит отработка отказа на дополнительный сайт. Это обеспечивает доступность приложений и рабочих нагрузок. При восстановлении нормального режима работы исходного расположения происходит переключение на него.

## <a name="site-recovery-in-the-azure-portal"></a>Служба Site Recovery на портале Azure
В Azure предусмотрены две [модели развертывания](../resource-manager-deployment-model.md) для создания ресурсов и работы с ними: модель Azure Resource Manager и классическая модель управления службой. Кроме того, Azure предоставляет два портала — [классический портал Azure](https://manage.windowsazure.com/), поддерживающий классическую модель развертывания, и [портал Azure](https://portal.azure.com), поддерживающий классическую модель развертывания и модель развертывания Resource Manager.

* Служба Site Recovery доступна как на классическом портале, так и на портале Azure.
* На классическом портале Azure поддерживается классическая модель управления службой Site Recovery,
* а на портале Azure — классическая модель или модель Resource Manager. 

Информация, приведенная в этой статье, относится как к классической модели развертывания, так и к модели развертывания с помощью портала Azure. Отличия будут указаны.

## <a name="why-deploy-site-recovery?"></a>Зачем нужно развертывание Site Recovery?
Ниже перечислены преимущества использования службы Site Recovery для вашего бизнеса.

* **Упрощение BCDR.** Репликацию, отработку отказа и восстановление нескольких рабочих нагрузок можно выполнять в одном расположении на портале Azure. Хотя служба Site Recovery координирует процессы репликации и отработки отказа, она не перехватывает данные приложения и не получает какие-либо сведения о нем.
* **Гибкая репликация.** С помощью Site Recovery можно реплицировать рабочие нагрузки, выполняющиеся на поддерживаемых виртуальных машинах Hyper-V, виртуальных машинах VMware и физических серверах Windows и Linux.
* **Выполнение простого тестирования репликации.** Служба Site Recovery позволяет выполнять тестовые отработки отказа для поддержки аварийного восстановления, не влияя на рабочие среды.
* **Отработка отказа и восстановление.** Вы можете запускать запланированные процедуры отработки отказа без потери данных при ожидаемых простоях, а также незапланированные процедуры отработки отказа с минимальной потерей данных (в зависимости от частоты репликации) на случай непредвиденных сбоев. После отработки отказа можно выполнить восстановление размещения на основные сайты. Служба Site Recovery предоставляет планы восстановления, которые могут включать скрипты и рабочие книги службы автоматизации Azure. С их помощью можно настраивать восстановление и отработку отказа многоуровневых приложений.
* **Устранение дополнительного центра обработки данных.** Рабочие нагрузки можно реплицировать в Azure, а не на дополнительный сайт. Это позволит сократить затраты и отказаться от обременительного обслуживания дополнительного центра обработки данных. Реплицированные данные хранятся в отказоустойчивой службе хранилища Azure. При отработке отказа виртуальные машины создаются с реплицированными данными.
* **Интеграция с имеющимися технологиями BCDR.** Служба Site Recovery взаимодействует с другими компонентами стратегии BCDR. Например, можно использовать Site Recovery для защиты серверной части SQL Server корпоративных рабочих нагрузок, включая встроенную поддержку для SQL Server AlwaysOn для управления отработкой отказа групп доступности.

## <a name="what-workloads-can-i-protect?"></a>Какие рабочие нагрузки я могу защитить?
Служба Site Recovery позволяет реализовать стратегию BCDR с учетом особенностей приложений, обеспечивая согласованное выполнение рабочих нагрузок и приложений при возникновении сбоев. Преимущества службы Site Recovery:

* **Моментальные снимки, согласованные с приложением.** Компьютеры выполняют репликацию с использованием моментальных снимков с согласованием для одноуровневых или многоуровневых приложений. В дополнение к записи данных на диске с помощью моментальных снимков, согласованных с приложениями, записываются все данные в памяти и все выполняемые транзакции.
* **Практически синхронная репликация.** Служба Site Recovery обеспечивает частоту репликации, которая составляет всего 30 секунд для Hyper-V, а для VMware поддерживается непрерывная репликация.
* **Гибкие планы восстановления.** Планы восстановления можно создавать и настраивать с использованием внешних скриптов и действий, выполняемых вручную. Интеграция с модулями Runbook службы автоматизации Azure позволяет выполнять восстановление всего стека приложений одним щелчком.
* **Интеграция с SQL Server AlwaysOn.** Отработкой отказа групп доступности можно управлять с помощью планов восстановления Site Recovery.
* **Библиотека службы автоматизации.** Обширная библиотека службы автоматизации Azure содержит готовые к работе и учитывающие особенности приложений скрипты, которые можно скачать и интегрировать с помощью Site Recovery.
* **Простое управление сетью.** Дополнительные функции управления сетью в Site Recovery и Azure упрощают сетевые требования к приложениям, включая резервирование IP-адресов, настройку балансировщиков нагрузки и интеграцию диспетчера трафика Azure для эффективного переключения сетей.

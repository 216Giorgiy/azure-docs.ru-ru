#  Обзор Site Recovery

Служба Site Recovery помогает надежному решению по обеспечению непрерывности бизнес-процессов и аварийного восстановления, которое защищает ваши локальные физические серверы и виртуальные машины, выполняя оркестрацию и автоматизацию репликации и отработки отказа с использованием Azure или дополнительного локального центра обработки данных.

- **Простота** — Site Recovery помогает упростить стратегию BCDR, облегчая настройку репликации, а также отработку отказа и восстановление для локальных рабочих нагрузок и приложений.
- **Репликация** — вы можете реплицировать локальные рабочие нагрузки в хранилище Azure или вспомогательный центр обработки данных. 
- **Хранилище** — для управления репликацией настраивается хранилище Site Recovery в выбранном регионе Azure. Все метаданные остаются в этом регионе.
- **Метаданные** —в Azure не отправляются данные приложений. Для репликации и отработки отказа службе Site Recovery необходимы только метаданные, например имена виртуальных машин и облаков VMM. 
- **Подключения к Azure** — серверы управления взаимодействуют с Azure в зависимости от сценария развертывания. Например, при репликации виртуальных машин, расположенных в локальном облаке VMM, сервер VMM взаимодействует с Site Recovery по зашифрованному исходящему соединению HTTPS. Подключение от виртуальных машин и узлов Hyper-V не требуется.
- **Реплика Hyper-V** — Azure Site Recovery использует реплику Hyper-V для процесса репликации, а также может использовать репликацию SAN, если она выполняется между двумя локальными сайтами VMM. Site Recovery применяет смарт-репликацию, используя только блоки данных, а не весь виртуальный жесткий диск, для начальной репликации. Текущая репликация применяется только к разностным изменениям. Служба Site Recovery поддерживает автономную передачу данных и совместима с оптимизаторами WAN.
- **Цены** — вы можете [узнать больше](http://azure.microsoft.com/pricing/details/site-recovery/) о ценах на Site Recovery.

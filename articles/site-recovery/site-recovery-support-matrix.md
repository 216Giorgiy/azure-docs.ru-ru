# <a name="azure-site-recovery-support-matrix"></a>Таблица поддержки Azure Site Recovery

В этой статье перечислены операционные системы и компоненты, поддерживаемые для Azure Site Recovery. Список поддерживаемых и необходимых компонентов для каждого сценария развертывания указан в соответствующей статье о развертывании. В данном документе приводится сводка по ним.

## <a name="support-for-virtualization-server-operating-systems"></a>Поддерживаемые операционные системы для серверов виртуализации

### <a name="host-servers-replicate-to-azure"></a>Серверы узлов (репликация в Azure)

**Виртуальная машина VMware или физический сервер** | **Hyper-V (без VMM)** | **Hyper-V (с VMM)**
--- | --- | ---
vCenter 5.5 или vCenter 6.0 (поддержка только функций vCenter 5.5)  <br/><br/>  vSphere 6.0, vCenter 5.5 или vCenter 5.1 с последними обновлениями. | Windows Server 2012 R2 с последними обновлениями. | Windows Server 2012 R2 с последними обновлениями.

### <a name="host-servers-replicate-to-secondary-site"></a>Серверы узлов (репликация на дополнительный сайт)

**Виртуальная машина VMware или физический сервер** | **Hyper-V (с VMM)**
--- | --- | ---
vCenter 5.5 или vCenter 6.0 (поддержка только функций vCenter 5.5)  <br/><br/>  vSphere 6.0, vCenter 5.5 или vCenter 5.1 с последними обновлениями. | По крайней мере Windows Server 2012 с последними обновлениями


## <a name="support-for-replicated-machines"></a>Поддержка реплицируемых виртуальных машин

### <a name="machines-replicate-to-azure"></a>Виртуальные машины (репликация в Azure)

Виртуальные машины должны отвечать соответствующим [требованиям Azure](site-recovery-best-practices.md#azure-virtual-machine-requirements).

**Требование** | **VMware или физический сервер** | **Hyper-V (без VMM)** | **Hyper-V (с VMM)**
--- | --- | --- | ---
Что реплицируется | Любая рабочая нагрузка на виртуальных машинах Windows или Linux | Любая рабочая нагрузка. | Любая рабочая нагрузка.
ОС узла | 64-разрядная версия Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 как минимум с пакетом обновления 1 (SP1).<br/><br/> Red Hat Enterprise Linux 6.7, 7.1, 7.2. <br/><br/> Centos 6.5, 6.6, 6.7, 7.0, 7.1, 7.2 <br/><br/> Oracle Enterprise Linux 6.4, 6.5 с ядром, совместимым с Red Hat, или с ядром Unbreakable Enterprise Kernel Release 3 (UEK3). <br/><br/> SUSE Linux Enterprise Server 11 SP3<bbr/><br/> Требуются следующие типы хранилищ: файловые системы: EXT3, ETX4, ReiserFS, XFS; многомаршрутное подключение устройств (multipath); менеджер томов: LVM2. Физические серверы с хранилищем контроллера HP CCISS не поддерживаются. Файловая система ReiserFS поддерживается только в SUSE Linux Enterprise Server 11 с пакетом обновления 3 (SP3). | Любая гостевая ОС, [поддерживаемая Azure](https://technet.microsoft.com/library/cc794868.aspx) | Любая гостевая ОС, [поддерживаемая Azure](https://technet.microsoft.com/library/cc794868.aspx)

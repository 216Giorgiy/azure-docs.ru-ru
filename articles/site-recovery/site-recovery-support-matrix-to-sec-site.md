# <a name="azure-site-recovery-support-matrix-for-replicating-to-customer-owned-secondary-site"></a>Таблица поддержки Azure Site Recovery для репликации на принадлежащий клиенту вторичный сайт

> [!div class="op_single_selector"]
> * [Репликация в Azure](site-recovery-support-matrix-to-azure.md)
> * [Репликация на принадлежащий клиенту вторичный сайт](site-recovery-support-matrix-to-sec-site.md)

Список предварительных требований для Azure Site Recovery приведен [здесь](site-recovery-best-practices.md), а в данной статье перечислены поддерживаемые конфигурации и компоненты для Azure Site Recovery при репликации и восстановлении на принадлежащий клиенту вторичный сайт.

## <a name="support-for-deployment-options"></a>Поддержка вариантов развертывания

**Развертывание** | **VMware или физический сервер** | **Hyper-V (без VMM)** | **Hyper-V (с VMM)**
--- | --- | --- | ---
**портале Azure** | Из локальных виртуальных машин VMware на дополнительный сайт VMware.<br/><br/> Скачайте руководство пользователя InMage Scout (http://download.microsoft.com/download/E/0/8/E08B3BCE-3631-4CED-8E65-E3E7D252D06D/InMage_Scout_Standard_User_Guide_8.0.1.pdf). Недоступно на портале Azure. | Не поддерживается | Из локальных виртуальных машин Hyper-V (в облаках VMM) на дополнительный сайт VMM.<br/><br/> Только стандартная репликация Hyper-V, SAN не поддерживается.
**Классический портал.** | Доступен только режим обслуживания. Создать хранилища невозможно. | Не поддерживается | Доступен только режим обслуживания.
**PowerShell** | Не поддерживается. | Не поддерживается | Поддерживаются



## <a name="support-for-datacenter-management-servers"></a>Поддержка серверов управления центром обработки данных

### <a name="virtualization-management-entities"></a>Объекты управления виртуализацией

**Развертывание** | **Поддержка**
--- | ---
**Виртуальная машина VMware или физический сервер** | vSphere 6.0, 5.5 или 5.1 с последними обновлениями
**Hyper-V (с VMM)** | SCVMM 2016 и SCVMM 2012 R2

  >[!Note]
  > В настоящее время облако SCVMM 2016, сочетающее узлы Windows Server 2016 и 2012 R2, не поддерживается.

### <a name="host-servers"></a>Серверы узлов

**Развертывание** | **Поддержка**
--- | ---
**Виртуальная машина VMware или физический сервер** | vCenter 5.5 или vCenter 6.0 (поддержка только функций vCenter 5.5) 
**Hyper-V (без VMM)** | Конфигурация не поддерживается при репликации на вторичный сайт
**Hyper-V с VMM** | Windows Server 2016, Windows Server 2012 R2 с последними обновлениями.<br/><br/> Узлы Windows Server 2016 должны управляться сервером SCVMM 2016.

## <a name="support-for-replicated-machine-machine-os-versions"></a>Поддержка версий ОС для реплицируемых виртуальных машин
В таблице ниже представлены сведения о поддержке операционных систем в различных сценариях развертывания при использовании Azure Site Recovery. Эта поддержка **относится к любой рабочей нагрузке**, выполняемой в указанной ОС.

**VMware или физический сервер** | **Hyper-V (с VMM)**
--- | --- | ---
64-разрядная версия Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 как минимум с пакетом обновления 1 (SP1).<br/><br/> Red Hat Enterprise Linux 6.7, 7.1, 7.2. <br/><br/> Centos 6.5, 6.6, 6.7, 7.0, 7.1, 7.2 <br/><br/> Oracle Enterprise Linux 6.4, 6.5 с ядром, совместимым с Red Hat, или с ядром Unbreakable Enterprise Kernel Release 3 (UEK3). <br/><br/> SUSE Linux Enterprise Server 11 SP3 | Любая гостевая ОС, [поддерживаемая Hyper-V](https://technet.microsoft.com/library/mt126277.aspx)

>[!Note]
>**Поддержка хранилища для версий Linux** Файловая система (EXT3, ETX4, ReiserFS или XFS). Многомаршрутное подключение устройств (multipath). Диспетчер томов (LVM2). Физические серверы с хранилищем контроллера HP CCISS **не** поддерживаются.
>Файловая система ReiserFS поддерживается только в SUSE Linux Enterprise Server 11 с пакетом обновления 3 (SP3).

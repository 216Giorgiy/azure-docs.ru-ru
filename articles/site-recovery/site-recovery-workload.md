# Руководство по рабочим нагрузкам Site Recovery

Azure Site Recovery позволяет пользователям разворачивать решения доступности с учетом особенностей конкретного приложения. Будь то приложения на основе Windows Server или Linux, собственные корпоративные приложения Microsoft или предложения от других поставщиков, вы сможете использовать Azure Site Recovery для восстановления после сбоя, для сред разработки и тестирования по требованию или для миграции в облако.

Корпорация Microsoft обладает глубокими знаниями и опытом в разработке лучших приложений корпоративного класса, таких как SharePoint, Exchange, Dynamics, SQL Server и Active Directory. Корпорация Microsoft также тесно сотрудничает с ведущими производителями ПО, включая Oracle, SAP, IBM и Red Hat, чтобы гарантировать хорошую работу их приложений и сервисов на своих платформах, включая Azure и Hyper-V. Мы используем эти уникальные преимущества для глубокого понимания требований к доступности для каждого приложения, чтобы вы могли разворачивать лучшие в своем классе решения для аварийного восстановления и доступности, которые можно настроить для каждого приложения.


##Основные функции
Функции Azure Site Recovery разрабатывались с учетом защиты и восстановления на уровне приложения:

- Почти синхронная репликация с RPO (цель точки восстановления) всего в 30 секунд, что соответствует потребностям большинства критичных приложений.
- Согласованные моментальные снимки приложения для одноуровневых или n-уровневых приложений.
- Интеграция с репликацией уровня приложения. Использование лучших в своем классе предложений уровня приложения, в том числе репликации AD, SQL Always On, групп доступности баз данных Exchange и Oracle Data Guard
- Гибкие планы восстановления позволяют восстановить стек всего приложения одним щелчком мыши, включая выполнение внешних сценариев или даже действия вручную. 
- Улучшенное управление сетью в ASR и Azure позволяет автоматизировать все операции по настройке сети для вашего приложения: резервирование IP-адресов, настройка балансировщиков нагрузки или использование диспетчера трафика Microsoft для переключения сетей с низким RTO (целевое время восстановления).
- Богатая библиотека автоматизации, предоставляющая готовые к работе сценарии для конкретных приложений. Загрузите их и интегрируйте в свои решения на базе ASR.

##Active Directory и DNS

Правильная работа всех корпоративных приложений, таких как SharePoint, Dynamics AX и SAP зависят от AD и инфраструктуры DNS. При создании решения аварийного восстановления для таких приложений важно защитить и выполнить восстановление AD в случае сбоя до того, как с проблемами столкнутся другие компоненты приложения.

С Azure Site Recovery можно создать полностью автоматизированный план аварийного восстановления для вашей AD. Если AD используется в нескольких приложениях, например в SharePoint и SAP на основном сайте, и вы решили выполнять отработку отказа для всего сайта, можно сначала выполнить отработку отказа для AD с помощью плана восстановления AD, а затем выполнить отработку отказа для других приложений с помощью планов восстановления для этих приложений.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для AD](http://aka.ms/asr-ad).

##SQL Server
Microsoft SQL Server является основой для многих собственных, сторонних и пользовательских бизнес-приложений корпоративного уровня, расположенных в локальном центре обработки данных клиента. Такие приложения, как SharePoint, Dynamics и SAP, используют SQL Server для реализации служб данных. Azure Site Recovery и технологии высокой доступности и аварийного восстановления SQL Server дополняют друг друга и могут использоваться для обеспечения полной защиты многоуровневых приложений. Azure Site Recovery предлагает следующие преимущества для сред SQL Server:

- Легкая защита автономных или кластеризованных серверов SQL в Azure или на дополнительном сайте. Расширение простого и экономически выгодного решения аварийного восстановления для любой версии и выпуска SQL Server.
- Интеграция с группами доступности SQL Always On, лучшее в своем классе решение аварийного восстановления и управление отработкой отказа и восстановлением с планами восстановления ASR.
- Создание полных планов восстановления для стека всего приложения, включая базы данных SQL.
- Увеличение размера (масштабирование) SQL Server во время пиковых нагрузок с Azure Site Recovery за счет увеличения размеров виртуальных машин IaaS в Azure.
- Создание тестовых копий SQL Server в Azure или на дополнительном сайте. Использование этих копий для анализа и проверок соответствия данных без влияния на рабочую среду.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для SQL](http://aka.ms/asr-sql).

##SharePoint
SharePoint — это популярное приложение, позволяющее организациям работать совместно с использованием порталов интрасети, средств управления документами и файлами, социальных сетей, веб-сайтов и корпоративного поиска. Это также платформа приложений для простого развертывания пользовательских приложений и рабочих процессов. Используя Azure Site Recovery для SharePoint, вы можете:

- исключить необходимость и соответствующие расходы на инфраструктуру для дежурной фермы аварийного восстановления; с помощью ASR вы сможете выполнить защиту фермы (на уровнях веб-сайта, приложения и базы данных) в Azure или на дополнительном сайте;
- упростить развертывание приложения и управлением им. Обновления, развертываемые на основном сайте, автоматически реплицируются и доступны при восстановлении фермы на дополнительном сайте. Это упрощает управление, избавляя от необходимости поддержки дежурной фермы в обновленном состоянии, и снижает совокупную стоимость владения.
- Упростить разработку и тестирование приложений SharePoint путем создания по требованию копии приложения, аналогичной рабочей, для тестирования и отладки.
- Упростите свой путь в облако, выполнив миграцию развертываний SharePoint в Azure с помощью ASR.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для Sharepoint](http://aka.ms/asr-sharepoint).


##Dynamics AX
Microsoft Dynamics AX — ведущее решение для управления ресурсами предприятия (ERP). Благодаря его простоте в изучении и использовании вы сможете быстрее получить практическую пользу, воспользоваться бизнес-возможностями и стимулировать включение пользователя и инновации в организации.

- С помощью ASR вы сможете выполнить защиту всего решения Dynamics AX (на уровнях веб-сайта, AOS, базы данных и SharePoint) в Azure или на дополнительном сайте.
- Упростите свой путь в облако, выполнив миграцию развертываний Dynamics Ax в Azure с помощью ASR.
- Упростите разработку и тестирование приложений Dynamics путем создания по требованию копии приложения, аналогичной рабочей, для тестирования и отладки.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для Dynamics AX](http://aka.ms/asr-dynamics).

## RDS 
Службы удаленных рабочих столов Windows Server (RDS) ускоряют и расширяют развертывание рабочих столов и приложений на любых устройствах, повышая эффективность удаленного работника и в то же время способствуя защите важной интеллектуальной собственности и упрощая соответствие нормативным требованиям. Службы удаленных рабочих столов включают инфраструктуру виртуальных рабочих столов (VDI), рабочие столы на основе сеансов и приложения, позволяя пользователям работать в любом месте.

С помощью ASR можно включить защиту управляемых или неуправляемых пулов виртуальных рабочих столов на дополнительном сайте и удаленных приложений и сеансов на дополнительном сайте или в Azure.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для RDS/VDI](http://aka.ms/asr-rds).


## Exchange
Microsoft Exchange — программное обеспечение, которое предприятия предпочитают использовать для размещения своих служб обмена сообщениями и электронной почты. Exchange позволяет обмениваться информацией с компьютера, телефона или из браузера, обеспечивая непревзойденную надежность, управляемость и защиту данных.

Microsoft Exchange имеет встроенную поддержку решений высокой доступности корпоративного класса и аварийного восстановления. Технологии групп доступности баз данных Exchange и Azure Site Recovery дополняют друг друга.

- Рекомендуется развертывать группы доступности баз данных Exchange для обеспечения лучшего в своем классе аварийного восстановления для развертываний Exchange. Планы восстановления ASR могут включать группы доступности баз данных для управления отработками отказа групп доступности баз данных на различных сайтах.
- Для небольших развертываний, например для одного сервера или для некластеризованных серверов, Azure Site Recovery может обеспечить защиту и выполнить отработку отказа отдельных серверов в Azure или на дополнительный сайт.

Подробное руководство можно найти в документе [Развертывание Azure Site Recovery для Exchange](http://aka.ms/asr-exchange).

## SAP

Архитектура и функционирование таких приложений чаще всего являются очень сложными, и очень важно убедиться в выполнении требований BCDR (аварийное восстановление непрерывности бизнеса).

- С помощью ASR вы сможете выполнить защиту всего развертывания SAP на различных уровнях в Azure или на дополнительном сайте.

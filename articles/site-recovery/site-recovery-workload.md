# Какие рабочие нагрузки можно защитить с помощью службы Azure Site Recovery?

Служба Azure Site Recovery позволяет пользователям развертывать решения, обеспечивающие доступность с учетом особенностей приложений. Эти решения можно включать в корпоративную стратегию непрерывности бизнес-процессов и аварийного восстановления (BCDR). Служба Azure Site Recovery совместима с приложениями на основе Windows Server или Linux, корпоративными приложениями Майкрософт и предложениями других поставщиков. Этот продукт можно использовать для аварийного восстановления, при работе со средами разработки и тестирования по требованию или для миграции в облако.

Site Recovery интегрируется с приложениями Майкрософт, в том числе SharePoint, Exchange, Dynamics, SQL Server и Active Directory. Корпорация Майкрософт также тесно сотрудничает с ведущими производителями ПО, включая Oracle, SAP, IBM и Red Hat, обеспечивая слаженную работу сторонних приложений и служб на таких платформах, как Azure и Hyper-V. Вы можете указать параметры аварийного восстановления для каждого конкретного приложения.


##Основные функции
Ниже перечислены функции Azure Site Recovery, которые дополнят вашу стратегию защиты и восстановления на уровне приложений.

- Практически синхронная репликация — значение RPO (целевых точек восстановления) составляет всего 30 секунд, что соответствует потребностям большинства важных приложений.
- Моментальные снимки с согласованием одноуровневых или n-уровневых приложений.
- Интеграция с SQL Server AlwaysOn, взаимодействие с другими технологиями репликации на уровне приложения, включая репликацию Active Directory, SQL AlwaysOn, группы обеспечения доступности баз данных (DAG) Exchange и Oracle Data Guard.
- Гибкие планы восстановления, которые позволяют быстро и просто восстановить весь стек приложения. Поддерживается выполнение внешних сценариев и запуск действий вручную. 
- Дополнительные функции сетевого управления в Site Recovery и Azure упрощают сетевые требования к приложениям, такие как резервирование IP-адресов, настройка балансировщиков нагрузки или интеграция диспетчера трафика Azure для переключения сетей с низким значением RTO (целевого времени восстановления).
-  Обширная библиотека автоматизации, которая позволяет легко загрузить и интегрировать в Site Recovery готовые к работе сценарии, в которых учтена специфика приложений.

##Защита Active Directory и DNS

Все корпоративные приложения, такие как SharePoint, Dynamics AX и SAP, зависят от инфраструктуры Active Directory и DNS. Стратегия BCDR предполагает, что защита и восстановление этих компонентов инфраструктуры является более приоритетным заданием, чем восстановление рабочих нагрузок и приложений.

Azure Site Recovery позволяет создать полностью автоматизированный план аварийного восстановления для Active Directory и DNS. Например, на основном сайте вы используете Active Directory для нескольких приложений (таких как SharePoint и SAP) и хотите, чтобы отработка отказа включала весь сайт. В таком случае вы можете сначала выполнить отработку отказа Active Directory с помощью плана восстановления, а затем — отработку отказа приложений, которые зависят от Active Directory, с помощью связанных с приложениями планов восстановления.

##Защита SQL Server

SQL Server обеспечивает работу служб данных для множества бизнес-приложений в центре обработки данных. Служба Site Recovery дополняет технологии обеспечения высокой доступности и аварийного восстановления SQL Server. Эти решения могут использоваться совместно для обеспечения комплексной защиты многоуровневых приложений. Site Recovery предлагает следующие преимущества для сред SQL Server.

- Легко настраиваемые защита и репликация автономных или кластеризованных серверов SQL в Azure или на дополнительном сайте, а также простое и экономичное решение аварийного восстановления для разных выпусков и версий SQL Server.
- Интеграция с группами обеспечения доступности SQL AlwaysOn, управление процессами отработки отказа и восстановления размещения с использованием планов восстановления в Azure Site Recovery.
- Планы комплексного восстановления для всего стека приложения, включая базы данных SQL Server.
- Масштабирование SQL Server для обслуживания пиковых нагрузок с помощью Azure Site Recovery посредством переноса серверов на виртуальные машины IaaS Azure более крупного размера.
- Тестирование SQL Server в Azure или на дополнительном сайте с помощью Azure Site Recovery. Тестовые отработки отказа выполняются для проверки соответствия данных анализа без влияния на рабочую среду.

[Подробнее](http://aka.ms/asr-sql)

##Защита SharePoint

Azure Site Recovery помогает защитить развертывание SharePoint. Ниже перечислены возможности, связанные с использованием службы Site Recovery.

- исключить необходимость и соответствующие расходы на инфраструктуру для дежурной фермы аварийного восстановления; Site Recovery обеспечивает защиту фермы (на уровне веб-сайта, приложения и базы данных) в Azure или на дополнительном сайте.
- упростить развертывание приложения и управлением им. Обновления, развернутые на основном сайте, автоматически реплицируются, становясь доступными при восстановлении фермы на дополнительном сайте. Это упрощает управление обновлениями резервной фермы и сокращает расходы.
- Упрощение разработки и тестирования приложений SharePoint благодаря созданию реплики рабочей среды по запросу для тестирования и отладки.
- Упрощение перехода в облако благодаря использованию Site Recovery для переноса развертываний SharePoint в Azure.

[Подробнее](http://aka.ms/asr-sharepoint)

## Защита служб удаленных рабочих столов 
Службы удаленных рабочих столов включают инфраструктуру виртуальных рабочих столов (VDI), рабочие столы на основе сеансов и приложения, позволяя пользователям работать в любом месте. С помощью Site Recovery можно включить защиту управляемых или неуправляемых пулов виртуальных рабочих столов на дополнительном сайте, а также удаленных приложений и сеансов на дополнительном сайте или в Azure.

[Подробнее](http://aka.ms/asr-rds)


## Защита Exchange

Microsoft Exchange имеет встроенные функции поддержки высокой доступности и аварийного восстановления. Exchange DAG и Azure Site Recovery могут работать вместе.

- Exchange DAG рекомендуется в качестве основного решения для аварийного восстановления Exchange на уровне предприятия. Планы восстановления Site Recovery могут включать группы доступности баз данных для оркестрации отработок отказа групп доступности баз данных на различных сайтах.
- Для небольших развертываний, например отдельных или некластеризованных серверов, Site Recovery обеспечивает защиту и отработку отказа в Azure или на дополнительный сайт.

[Узнать больше](http://aka.ms/asr-exchange)

# <a name="replicate-azure-virtual-machines-to-another-azure-region"></a>Репликация виртуальных машин Azure в другой регион Azure



>[!NOTE]
>
> Репликация Azure Site Recovery для виртуальных машин Azure сейчас доступна в предварительной версии.

В этой статье описывается, как настроить репликацию виртуальных машин, работающих в одном регионе Azure в другой.

## <a name="prerequisites"></a>Предварительные требования

* В этой статье предполагается, что вы уже знаете о Site Recovery и хранилище служб восстановления. Хранилище служб восстановления необходимо предварительно создать.

    >[!NOTE]
    >
    > Мы советуем создать его в расположении, куда вы хотите реплицировать ваши виртуальные машины. Например, если целевое расположение — "Центральная часть США", создайте хранилище в этом регионе.

* Если вы используете правила группы безопасности сети или прокси-сервер брандмауэра для управления доступом к исходящим подключениям к Интернету на виртуальных машинах Azure, убедитесь, что необходимые URL- или IP-адреса добавлены в список разрешений. Дополнительные сведения см.в статье [Руководство по организации сети для репликации виртуальных машин Azure](./site-recovery-azure-to-azure-networking-guidance.md).

* Если у вас есть подключение ExpressRoute или VPN-подключение между локальным и исходным расположением в Azure, следуйте [рекомендациям Site Recovery для конфигурации подключения ExpressRoute или VPN из Azure в локальную среду](site-recovery-azure-to-azure-networking-guidance.md#guidelines-for-existing-azure-to-on-premises-expressroutevpn-configuration).

* Учетная запись пользователя Azure должна содержать определенные [разрешения](site-recovery-role-based-linked-access-control.md#permissions-required-to-enable-replication-for-new-virtual-machines) для включения репликации виртуальной машины Azure.

* Чтобы создать виртуальные машины в целевом расположении, которое будет использоваться как регион аварийного восстановления, должна быть включена подписка Azure. Вы можете связаться со службой поддержки, чтобы включить необходимые квоты.

## <a name="enable-replication-from-azure-site-recovery-vault"></a>Включение репликации из хранилища Azure Site Recovery
В этом примере мы реплицируем виртуальные машины, работающие в регионе Azure "Восточная Азия", в расположение "Юго-Восточная Азия". Для этого необходимо выполнить следующие шаги:

 Щелкните **+Replicate** (+Репликация) в хранилище, чтобы включить репликацию для виртуальных машин.

1. **Источник** — источник происхождения компьютеров, в этом случае это **Azure**.

2. **Исходное расположение** — регион Azure, в котором вы хотите защитить свои виртуальные машины. В этом примере исходным расположением будет "Восточная Азия".

3. **Модель развертывания** — модель развертывания Azure исходных компьютеров. Вы можете выбрать классическую модель или модель диспетчера ресурсов. После этого для защиты в следующем шаге будут указаны машины, принадлежащие к определенной модели.

      >[!NOTE]
      >
      > Классическую виртуальную машину можно реплицировать или восстанавливать только как классическую виртуальную машину, ее нельзя восстановить в качестве виртуальной машины диспетчера ресурсов.

4. **Группа ресурсов** — группа ресурсов, к которой принадлежат исходные виртуальные машины. Для защиты в следующем шаге будут перечислены все виртуальные машины в выбранной группе ресурсов.

    ![Включение репликации](./media/site-recovery-replicate-azure-to-azure/enabledrwizard1.png)

В области **Виртуальные машины > Выбор виртуальных машин**, выберите все компьютеры, которые нужно реплицировать. Можно выбрать только те компьютеры, для которых можно включить репликацию. Затем нажмите кнопку ОК.
    ![Включение репликации](./media/site-recovery-replicate-azure-to-azure/virtualmachine_selection.png)


В разделе "Параметры" можно настроить свойства целевого веб-сайта.

1. **Целевое расположение** — расположение, куда будут реплицированы данные исходных виртуальных машин. В зависимости от расположения выбранной машины Site Recovery предоставит список подходящих целевых регионов.

    > [!TIP]
    > Мы советуем оставить то же целевое расположение, что и у хранилища служб восстановления.

2. **Группа целевых ресурсов** — группа ресурсов, к которой будут принадлежать все реплицированные виртуальные машины. По умолчанию ASR создаст группу ресурсов в целевом регионе, в имени которого есть суффикс asr. Если группа ресурсов, создаваемая ASR, уже имеется, она будет использоваться повторно. Кроме того, ее можно настроить так, как показано в разделе ниже.    
3. **Target Virtual Network** (Целевая виртуальная сеть). По умолчанию ASR создаст виртуальную сеть в целевом регионе, в имени которого есть суффикс asr. Она будет сопоставлена с исходной сетью и будет использоваться для защиты в будущем.

    > [!NOTE]
    > [Ознакомьтесь с подробными сведениями о сети](site-recovery-network-mapping-azure-to-azure.md), чтобы узнать больше о сетевом сопоставлении.

4. **Target Storage accounts** (Целевые учетные записи хранения). По умолчанию ASR создаст целевую учетную запись хранения, копируя конфигурацию хранилища исходной виртуальной машины. Если учетная запись хранения, создаваемая ASR, уже имеется, она будет использоваться повторно.

5. **Cache Storage accounts** (Учетные записи хранения кэша). Для ASR необходима дополнительная учетная запись хранения, называемая учетной записью кэша, в исходном регионе. Все изменения, происходящие на исходных виртуальных машинах, отслеживаются и отправляются учетной записи хранения кэша перед репликацией в целевое расположение.

6. **Группа доступности.** По умолчанию ASR создаст группу доступности в целевом регионе, в имени которого есть суффикс asr. Если группа доступности, создаваемая ASR, уже имеется, она будет использоваться повторно.

7.  **Политика репликации** определяет параметры для журнала хранения точки восстановления и периодичность создания моментальных снимков, совместимых с приложениями. По умолчанию ASR создаст политику репликации с параметрами по умолчанию "24 часа" для хранения точки восстановления и "60 минут" — для периодичности создания моментальных снимков, совместимых с приложениями.

    ![Включение репликации](./media/site-recovery-replicate-azure-to-azure/enabledrwizard3.PNG)

## <a name="customize-target-resources"></a>Настройка целевых ресурсов

Если вы хотите изменить значения по умолчанию, используемые ASR, можно изменить параметры, исходя из своих потребностей.

1. Щелкните **Настройка**, чтобы изменить значения по умолчанию, используемые ASR.

2. **Группа целевых ресурсов.** Вы можете выбрать группу ресурсов из списка групп ресурсов в целевом расположении в пределах подписки.

3. **Target Virtual Network** (Целевая виртуальная сеть). Список всех виртуальных сетей в целевом расположении.

4. **Группа доступности.** Вы можете добавлять параметры групп доступности только на виртуальные машины, доступные в исходном регионе.

5. **Target Storage accounts** (Целевые учетные записи хранения):

![Включение репликации](./media/site-recovery-replicate-azure-to-azure/customize.PNG). Щелкните **Create target resource** (Создать целевой ресурс) и включите репликацию.


Теперь, когда виртуальные машины защищены, их работоспособность можно проверить в разделе **Реплицированные элементы**.

>[!NOTE]
>Во время выполнения начальной репликации есть вероятность того, что статус обновится через некоторое время и вы не будете видеть хода выполнения в течение этого времени. Щелкните "Обновить" в верхней части колонки, чтобы получить сведения о последнем состоянии.
>

# <a name="prepare-azure-resources-for-replication-of-on-premises-machines"></a>Подготовка ресурсов Azure для репликации локальных компьютеров

Служба [Azure Site Recovery](site-recovery-overview.md) помогает реализовать стратегию непрерывности бизнеса и аварийного восстановления (BCDR), обеспечивая работоспособность и доступность бизнес-приложений во время запланированных и незапланированных простоев. Site Recovery управляет аварийным восстановлением локальных компьютеров и виртуальных машин Azure, включая операции репликации, отработки отказа и восстановления.

Этот учебник описана процедура подготовки компонентов Azure для репликации виртуальных машин в локальной среде (Hyper-V или VMware) или физических серверов Windows и Linux в Azure. Из этого руководства вы узнаете, как выполнять такие задачи:

> [!div class="checklist"]
> * Убедитесь, что ваша учетная запись имеет разрешения репликации.
> * Создайте учетную запись хранения Azure.
> * Выберите сеть Azure. При создании виртуальных машин Azure после отработки отказа они присоединяются к этой сети.

Если у вас еще нет подписки Azure, [создайте бесплатную учетную запись Azure](https://azure.microsoft.com/pricing/free-trial/), прежде чем начинать работу.

## <a name="log-in-to-azure"></a>Вход в Azure

Войдите на портал Azure по адресу http://portal.azure.com.

## <a name="verify-account-permissions"></a>Проверка разрешений учетной записи

Если вы только что создали бесплатную учетную запись Azure, вы являетесь администратором подписки. Если вы не администратор подписки, свяжитесь с администратором, чтобы получить необходимые разрешения. Чтобы включить репликацию для новой виртуальной машины, необходимо иметь следующие разрешения.

- Разрешение на создание виртуальной Машины в выбранной группы ресурсов
- Разрешение на создание виртуальной Машины в выбранной виртуальной сети
- Разрешение на запись в выбранной учетной записи хранения

Встроенная роль "Участник виртуальных машин" обладает такими разрешениями. Кроме того, требуется разрешение на управление операциями Azure Site Recovery. Роль "Участник Site Recovery" имеет все разрешения, необходимые для управления операциями Azure Site Recovery в хранилище служб восстановления.

## <a name="create-a-storage-account"></a>Создайте учетную запись хранения.

Образы реплицированных компьютеров хранятся в службе хранилища Azure. Виртуальные машины Azure создаются из хранилища при отработке отказа с переходом из локальной среды в Azure.

1. В меню на [портале Azure](https://portal.azure.com) щелкните **Создать** -> **Хранилище** -> **Учетная запись хранения**.
2. Выберите имя для своей учетной записи хранения. В этих руководствах используется имя **contosovmsacct1910171607**. Имя должно быть уникальным в пределах Azure и находиться в диапазоне от 3 до 24 символов и содержать только строчные буквы и цифры.
3. Используйте модель развертывания **Resource Manager**.
4. Выберите **Общего назначения** > **Стандартный**.
5. Выберите значение по умолчанию **RA-GRS** для обеспечения избыточности хранилища.
6. Выберите подписку, в которой вы создаете учетную запись хранения.
7. Укажите новую группу ресурсов. Группа ресурсов Azure является логическим контейнером, в котором происходит развертывание ресурсов Azure и управление ими. Эти руководства мы используем имя **ContosoRG**.
8. Выберите географический регион для учетной записи хранения. Учетная запись хранения должна быть создана в том же регионе, что и хранилище служб восстановления. В этих учебниках мы будем использовать **Западной Европе** области.

   ![create-storageacct](media/tutorial-prepare-azure/create-storageacct.png)

9. Щелкните **Создать** , чтобы создать учетную запись хранения.

## <a name="create-a-vault"></a>Создание хранилища

1. В меню портала Azure щелкните **Создать** > **Monitoring & Management** (Мониторинг и управление) >
   **Backup and Site Recovery** (Служба архивации и Site Recovery).
2. В поле **Имя**укажите понятное имя для идентификации хранилища. В этом руководстве мы используем **ContosoVMVault**.
3. Выберите имеющуюся группу ресурсов с названием **contosoRG**.
4. Укажите регион Azure **Западной Европе**, который мы используем в этом наборе учебники.
5. Чтобы быстро получить доступ к хранилищу на панели мониторинга, щелкните **Закрепить на панели мониторинга** > **Создать**.

   ![Новое хранилище](./media/tutorial-prepare-azure/new-vault-settings.png)

   Новое хранилище появится в колонке **Панель мониторинга** > **Все ресурсы** и на основной странице **Хранилища служб восстановления**.

## <a name="set-up-an-azure-network"></a>Настроить сеть

При создании виртуальных машин Azure из хранилища после отработки отказа они присоединяются к сети.

1. В меню [портала Azure](https://portal.azure.com) щелкните **Создать**>**Сети**>
   **Виртуальная сеть**.
2. Оставьте **Resource Manager** в качестве модели развертывания. Resource Manager является предпочтительной моделью развертывания.
   - Укажите имя сети. Имя должно быть уникальным в пределах группы ресурсов Azure. Мы будем использовать имя **ContosoASRnet**.
   - Используйте имеющуюся группу ресурсов под названием **contosoRG**.
   - Укажите диапазон адресов сети 10.0.0.0/24.
   - В этом руководстве не понадобится подсеть.
   - Выберите подписку, в которой будет создана сеть.
   - Выберите расположение **Западная Европа**. Сеть должна располагаться в том же регионе, что и хранилище служб восстановления.
3. Нажмите кнопку **Создать**.

   ![create-network](media/tutorial-prepare-azure/create-network.png)

   Создание виртуальной сети занимает несколько секунд. После завершения этого процесса она отобразится на панели мониторинга портала Azure.

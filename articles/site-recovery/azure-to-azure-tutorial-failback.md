# <a name="fail-back-azure-vms-between-azure-regions"></a>Восстановление размещения виртуальных машин Azure между регионами Azure

Служба [Azure Site Recovery](site-recovery-overview.md) помогает реализовать стратегию аварийного восстановления за счет управления процессами репликации, отработки отказа и восстановления размещения локальных компьютеров и виртуальных машин Azure.

В этом учебнике содержатся сведения о восстановлении размещения одной виртуальной машины Azure. Восстановление размещения в основном регионе после отработки отказа происходит в тех ситуациях, где это возможно. Из этого руководства вы узнаете, как выполнять следующие задачи:

> [!div class="checklist"]
> 
> * Восстановление размещения дополнительной виртуальной машины
> * Повторная защита основной виртуальной машины, для которой выполнено восстановление размещения в дополнительном регионе.
> 
> [!NOTE]
> 
> Этот учебник содержит пошаговую инструкцию для пользователя по отработке отказа с переключением на целевой регион и обратно с минимальной настройкой. Если вам нужны дополнительные сведения о различных аспектах, связанных с отработкой отказа, включая требования к сети, автоматизацию или устранение неполадок, перейдите к статьям с руководствами для виртуальных машин Azure.

## <a name="prerequisites"></a>Предварительные требования

> * Убедитесь, что виртуальная машина находится в зафиксированном состоянии отработки отказа, проверьте, доступен ли основной регион, и вы можете создавать и использовать новые ресурсы в нем.
> * Убедитесь, что повторная защита включена.

## <a name="fail-back-to-the-primary-region"></a>Восстановление размещения в основном регионе

Когда вы повторно включите защиту виртуальной машины, можете при необходимости восстановить размещение в основном регионе.

1. Перейдите в хранилище Служб восстановления. Щелкните "Реплицированные элементы" и выберите виртуальную машину, которая повторно защищена.

2. Вы увидите приведенное ниже окно. Обратите внимание, что оно похоже на колонку для тестовой и обычной отработки отказа из основного региона.
![Восстановление размещения в основном регионе](./media/site-recovery-azure-to-azure-failback/azure-to-azure-failback.png)

3. Щелкните "Тестовая отработка отказа", чтобы выполнить тестовую отработку отказа обратно в основной регион. Выберите точку восстановления и виртуальную сеть для тестовой отработки отказа и нажмите кнопку "ОК". Вы увидите, что в основном регионе создана тестовая виртуальная машина, к которой можно получить доступ и которую можно изучить.

4. Если тестовая отработка отказа будет выполнена успешно, вы можете щелкнуть "Очистить тестовую отработку отказа", чтобы очистить ресурсы, созданные в исходном регионе для тестовой отработки отказа.

5. В разделе "Реплицированные элементы" выберите виртуальную машину, для которой требуется отработка отказа, а затем выберите "Отработка отказа".

6. В разделе "Отработка отказа" выберите точку восстановления, для которой будет выполнена отработка отказа. Можно выбрать один из следующих вариантов:
    1. "Последняя" (по умолчанию): выполняет обработку всех данных в службе Site Recovery и обеспечивает наименьшую целевую точку восстановления (RPO).
    2. "Последняя обработанная": отменяет изменения для виртуальной машины до последней точки восстановления из числа обработанных службой Site Recovery.
    3. "Специальная": выполняет отработку отказа до определенной точки восстановления. Этот параметр удобен для выполнения тестовой отработки отказа.

7. Выберите "Завершение работы виртуальной машины перед началом отработки отказа", чтобы служба Site Recovery попыталась завершить работу исходных виртуальных машин, прежде чем запускать отработку отказа. Отработка отказа продолжится, даже если их не удастся отключить. Обратите внимание, что Site Recovery не поддерживает очистку источника после отработки отказа.

8. На странице "Задания" можно следить за ходом выполнения отработки отказа.

9. После отработки отказа войдите на виртуальную машину, чтобы проверить ее работу. Если вы хотите использовать для виртуальной машины другую точку восстановления, выберите команду "Изменить точку восстановления".

10. Когда вы убедитесь, что после отработки отказа виртуальная машина работает правильно, можете зафиксировать отработку отказа. При фиксации удаляются все доступные в службе точки восстановления. Параметр "Изменить точку восстановления" больше не доступен.

![Виртуальная машина в основном и дополнительном регионах.](./media/site-recovery-azure-to-azure-failback/azure-to-azure-failback-vm-view.png)

Как показано на снимке экрана выше, для виртуальной машины ContosoWin2016 выполнена отработка отказа из региона "Центральная часть США" в регион "Восточная часть США", а затем восстановление из региона "Восточная часть США" в регион "Центральная часть США".

Обратите внимание, что виртуальные машины аварийного восстановления будут оставаться в отключенном состоянии. Это происходит намеренно, так как служба Azure Site Recovery сохраняет сведения о виртуальной машине, которые могут понадобиться при отработке отказа из основного региона в дополнительный регион. Вы не платите за отключенные виртуальные машины, поэтому их необходимо сохранить без изменений.

> Дополнительные сведения о том, как повторно включить защиту и что происходит во время повторного включения защиты, см. в [этом разделе](https://docs.microsoft.com/azure/site-recovery/azure-to-azure-how-to-reprotect#what-happens-during-reprotection).

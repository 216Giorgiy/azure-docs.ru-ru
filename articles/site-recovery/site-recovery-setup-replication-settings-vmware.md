# <a name="manage-replication-policy-for-vmware-to-azure"></a>Управление политикой репликации для VMware в Azure


## <a name="create-a-new-replication-policy"></a>Создание политики репликации

1. В меню слева щелкните "Управление" -> "Инфраструктура Site Recovery". 
2. В разделе "Для VMware и физических компьютеров" выберите "Политики репликации".
3. Вверху щелкните "Добавить политику репликации".

    ![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/createpolicy.png)

4. Введите имя политики.

5. В поле "Пороговое значение RPO" укажите предельное значение целевой точки восстановления (RPO). Если непрерывная репликация превышает это значение, выдаются оповещения.
6. В поле "Хранение точки восстановления" укажите продолжительность периода хранения для каждой точки восстановления (в часах). Защищенные компьютеры будут восстанавливаться до любой точки в пределах этого периода. 

    > [!NOTE] 
    > Для компьютеров, реплицируемых в хранилище класса Premium, этот период составляет 24 часа, а для компьютеров, реплицируемых в стандартное хранилище, — 72 часа.
    
    > [!NOTE] 
    > Политика репликации для восстановления размещения будет создана автоматически.

7. В поле "Периодичность создания моментальных снимков с согласованием приложений" укажите, с какой периодичностью (в минутах) будут создаваться точки восстановления, содержащие моментальные снимки, согласованные на уровне приложений.

8. Нажмите кнопку "ОК". Политика будет создана через 30–60 секунд.

![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Creating-Policy.png)

## <a name="associate-configuration-server-with-replication-policy"></a>Связывание сервера конфигурации с политикой репликации
1. Щелкните политику репликации, с которой вы хотите связать сервер конфигурации.
2. Вверху нажмите кнопку "Связать".
![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Associate-CS-1.PNG)

3. Выберите "Сервер конфигурации" в списке серверов.
4. Нажмите кнопку ОК. Сервер конфигурации будет связан с политикой через 1–2 минуты.

![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Associate-CS-2.png)

## <a name="edit-replication-policy"></a>Редактирование политики репликации
1. Щелкните политику репликации, параметры которой вы хотите изменить.
![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Select-Policy.png)

2. Вверху нажмите кнопку "Изменить параметры".
![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Edit-Policy.png)

3. Измените параметры по своему усмотрению.
4. Нажмите кнопку "Сохранить" вверху. Политика будет сохранена через 2–5 минут в зависимости от количества виртуальных машин, использующих эту политику репликации.

![Создание политики репликации](./media/site-recovery-setup-replication-settings-vmware/Save-Policy.png)

## <a name="dissociate-configuration-server-from-replication-policy"></a>Отмена связи сервера конфигурации с политикой репликации
1. Щелкните политику репликации, с которой вы хотите связать сервер конфигурации.
2. Вверху нажмите кнопку "Отменить связь".
3. Выберите "Сервер конфигурации" в списке серверов.
4. Нажмите кнопку ОК. Связь сервера конфигурации с политикой будет отменена через 1–2 минуты.
    
    > [!NOTE] 
    > Если хотя бы один реплицированный элемент использует политику, вы не сможете отменить ее связь с сервером конфигурации. Прежде чем отменять связь политики с сервером, убедитесь, что ни один реплицированный элемент не использует ее.

## <a name="delete-replication-policy"></a>Удаление политики репликации 

1. Щелкните политику репликации, которую вы хотите удалить.
2. Нажмите кнопку "Удалить". Политика будет удалена через 30–60 секунд.

    > [!NOTE] 
    > Если с политикой репликации связан хотя бы один сервер конфигурации, вы не сможете удалить ее. Прежде чем удалять политику, убедитесь, что ни один реплицированный элемент не использует ее, и удалите все связанные с ней серверы конфигурации.

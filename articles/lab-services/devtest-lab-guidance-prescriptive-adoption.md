# <a name="set-up-azure-devtest-labs-infrastructure-in-your-enterprise"></a>Настройка инфраструктуры DevTest Labs Azure на предприятии
Быстрое внедрение облака в предприятие происходит из-за его преимуществ в динамичности, гибкости и экономической перспективе. Обычно первым шагом перехода в облако является разработка и тестирование рабочих нагрузок. При переходе рабочих нагрузок в облако существует набор общих проблем. Примеры рабочих нагрузок приведены далее. 

- Обеспечение безопасности при разработке и тестировании ресурсов
- Четкое разделение между разработкой, тестированием и производством
- Разделение ресурсов между ИТ, приложениями и командами проектов
- Управление расходами
- Предоставление самообслуживания для настройки инфраструктуры разработки и тестирования без ущерба для безопасности

## <a name="intended-audience"></a>Целевая аудитория
Этот документ предназначен для проектировщиков ИТ-систем, архитекторов и менеджеров, которые отвечают за общее состояние разработки и анализа развертываний и контроля за рабочими процессами. Таким образом, в этом документе особое внимание уделяется общему процессу и рекомендуемым принципам по обеспечению безопасной и стабильной среды разработки, которая в конечном счете используется для внедрения DevTest Labs Azure в предприятие.

# <a name="manage-classroom-labs-in-azure-lab-services"></a>Управление лабораторией для аудитории в решении "Службы лабораторий Azure" 
В этой статье описываются процессы создания, настройки, просмотра и удаления лабораторий для аудитории.

## <a name="prerequisites"></a>Предварительные требования
Чтобы настроить лабораторию аудитории в учетной записи лаборатории, вы должны быть членом роли **Создатель лаборатории** в учетной записи лаборатории. Учетная запись, которую вы использовали для создания учетной записи лаборатории, автоматически добавляется в эту роль. См. дополнительные сведения о том, как [назначить владельцу лаборатории роль, позволяющую добавлять других пользователей в роль создателя лаборатории](tutorial-setup-lab-account.md#add-a-user-to-the-lab-creator-role).

## <a name="create-a-classroom-lab"></a>Создание лаборатории для аудитории

1. Перейдите на [веб-сайт Служб лабораторий Azure](https://labs.azure.com).
2. Выберите **Вход** и введите свои учетные данные. Службы лабораторий Azure поддерживают учетные записи организации и учетные записи Майкрософт.
3. В окне **New Lab** (Создание лаборатории) выполните следующие действия. 
    1. Укажите **имя** аудиторной лаборатории. 
    2. Выберите **размер** виртуальной машины, которую вы планируете использовать в аудитории.
    3. Выберите **образ** для создания виртуальной машины.
    4. Укажите **учетные данные по умолчанию**, которые необходимо использовать для входа в виртуальные машины в лаборатории.
    7. Щелкните **Сохранить**.

        ![Создание лаборатории для аудитории](../media/how-to-manage-classroom-labs/new-lab-window.png)
1. Откроется **панель мониторинга** лаборатории. 
    
    ![Панель мониторинга лаборатории для аудитории](../media/how-to-manage-classroom-labs/classroom-lab-home-page.png)

## <a name="configure-usage-policy"></a>Настройка политики использования

1. Выберите **Usage policy** (Политика использования). 
2. В разделе **Usage policy** (Политика использования) введите **количество пользователей**, которым разрешено использовать лабораторию.
3. Щелкните **Сохранить**. 

    ![Политика использования](../media/how-to-manage-classroom-labs/usage-policy-settings.png)

## <a name="set-up-the-template"></a>Настройка шаблона
На основе шаблона, в роли которого выступает базовый образ виртуальной машины, в лаборатории создаются все виртуальные машины пользователей. Настройте шаблон согласно параметрам, которые вы хотите предоставить пользователям лаборатории. Вы можете указать имя и описание шаблона, которое будет отображаться для пользователей лаборатории. Опубликуйте шаблон, чтобы сделать экземпляры шаблона виртуальной машины доступными для пользователей лаборатории.  

### <a name="set-template-title-and-description"></a>Настройка заголовка и описания шаблона
1. В разделе **Шаблон** выберите **Изменить** (значок карандаша) для шаблона. 
2. В окне **User view** (Пользовательское представление) введите **заголовок** шаблона.
3. Введите **описание** шаблона.
4. Щелкните **Сохранить**.

    ![Описание аудиторной лаборатории](../media/how-to-manage-classroom-labs/lab-description.png)

### <a name="set-up-the-template-vm"></a>Настройка шаблона виртуальной машины
 Перед тем как сделать шаблон виртуальной машины доступным для учащихся, необходимо подключиться к шаблону и установить на нем все необходимое программное обеспечение. 

1. Подождите, пока шаблон виртуальной машины будет готов. После этого включите кнопку **Запустить**. Чтобы запустить виртуальную машину, щелкните **Запустить**.

    ![Запуск шаблона виртуальной машины](../media/tutorial-setup-classroom-lab/start-template-vm.png)
1. Чтобы подключиться к виртуальной машине, нажмите кнопку **Подключиться** и следуйте инструкциям. 

    ![Подключение к шаблону виртуальной машины](../media/tutorial-setup-classroom-lab/connect-template-vm.png)
1. Установите программное обеспечение, необходимое для работы учащихся в лаборатории (например, Visual Studio, Обозреватель службы хранилища Azure и т. д.). 
2. Отключитесь от шаблона виртуальной машины (закрыв сеанс удаленного рабочего стола). 
3. Нажмите кнопку **Остановить**, чтобы **остановить** шаблон виртуальной машины. 

    ![Остановка шаблона виртуальной машины](../media/tutorial-setup-classroom-lab/stop-template-vm.png)


### <a name="publish-the-template"></a>Публикация шаблона 
Когда вы опубликуете шаблон, Службы лабораторий Azure создадут виртуальные машины в лаборатории с помощью этого шаблона. Число созданных при этом процессе виртуальных машин соответствует максимальному числу разрешенных в лаборатории пользователей, которое можно настроить в политике использования лаборатории. Все виртуальные машины имеют ту же конфигурацию, что и шаблон. 

1. В разделе **Шаблон** нажмите кнопку **Опубликовать**. 

    ![Публикация шаблона виртуальной машины](../media/tutorial-setup-classroom-lab/public-access.png)
1. В окне **Публикация** выберите параметр **Опубликовано**. 
2. Затем кнопку **Опубликовать**. Этот процесс может занять некоторое время в зависимости от числа виртуальных машин, которое соответствует числу разрешенных пользователей лаборатории.
    
    > [!IMPORTANT]
    > Отменить публикацию шаблона будет невозможно. 
4. Перейдите на страницу **Виртуальные машины** и проверьте, что на ней есть виртуальные машины в состоянии **Не назначено**. Эти виртуальные машины еще не назначены учащимся. 

    ![Виртуальные машины](../media/tutorial-setup-classroom-lab/virtual-machines.png)
5. Подождите, пока виртуальные машины будут созданы. Они должны находиться в состоянии **Остановлено**. На этой странице вы можете запустить виртуальную машину для учащегося, подключиться к ней, а также остановить или удалить ее. Виртуальные машины на этой странице можете запускать как вы, так и ваши учащиеся. 

    ![Виртуальные машины в остановленном состоянии](../media/tutorial-setup-classroom-lab/virtual-machines-stopped.png)


## <a name="send-registration-link-to-students"></a>Отправка ссылки для регистрации учащимся

1. Откройте представление **Панель мониторинга**. 
2. Щелкните плитку **User registration** (Регистрация пользователей).

    ![Ссылка для регистрации учащихся](../media/tutorial-setup-classroom-lab/dashboard-user-registration-link.png)
1. В диалоговом окне **User registration** (Регистрация пользователей) щелкните **Копировать**. Ссылка скопируется в буфер обмена. Вставьте ее в редактор электронной почты и отправьте электронное сообщение учащемуся. 

    ![Ссылка для регистрации учащихся](../media/tutorial-setup-classroom-lab/registration-link.png)
2. В диалоговом окне **Регистрация пользователей** нажмите кнопку **Закрыть**. 

## <a name="view-all-classroom-labs"></a>Просмотр всех лабораторий для аудитории
1. Перейдите на [портал Служб лабораторий Azure](https://labs.azure.com).
2. Убедитесь, что вам доступны все лаборатории в учетной записи выбранной лаборатории. 

    ![Все лаборатории](../media/how-to-manage-classroom-labs/all-labs.png)
3. Используйте раскрывающийся список в верхней части окна, чтобы выбрать учетную запись другой лаборатории. После этого отобразятся лаборатории выбранной учетной записи лаборатории. 

## <a name="delete-a-classroom-lab"></a>Удаление лаборатории для аудитории
1. В углу плитки лаборатории щелкните многоточие (...). 

    ![Выбор лаборатории](../media/how-to-manage-classroom-labs/select-three-dots.png)
2. Нажмите кнопку **Удалить**. 

    ![Кнопка "Удалить"](../media/how-to-manage-classroom-labs/delete-button.png)
3. В диалоговом окне **Delete lab** (Удалить лабораторию) щелкните **Удалить**. 

    ![Диалоговое окно удаления](../media/how-to-manage-classroom-labs/delete-lab-dialog-box.png)
 
## <a name="manage-student-vms"></a>Управление виртуальными машинами учащихся
После регистрации учащихся в Службах лабораторий Azure с помощью предоставленной вами ссылки для регистрации вы сможете просмотреть виртуальные машины, назначенные учащимся, на вкладке **Виртуальные машины**. 

![Виртуальные машины, назначенные учащимся](../media/how-to-manage-classroom-labs/virtual-machines-students.png)

Вы можете выполнить следующие действия с виртуальной машиной учащегося: 

- Остановить виртуальную машину, если она запущена. 
- Запустить виртуальную машину, если она остановлена. 
- Подключитесь к виртуальной машине. 
- Удалите виртуальную машину.

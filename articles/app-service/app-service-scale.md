# Масштабирование ценовой категории в службе приложения Azure

Чтобы повысить производительность и пропускную способность приложений в Microsoft Azure, можно на [портале Azure](https://portal.azure.com/) повысить уровень план обслуживания приложений от **Free** до **Shared**, **Basic**, **Standard** или **Premium**.

Уровень обслуживания вашего плана обслуживания приложений зависит от [*ценовой категории* этого плана](/pricing/details/app-service/). Более высокие ценовые категории, такие как **Standard** и **Premium**, предлагают повышенную надежность и гибкость в определении способов использования ресурсов в Azure. Изменение ценовой категории влияет на число ядер и объем памяти вашей службы, и это называется *увеличением масштаба* (или *уменьшением масштаба*).

Помимо повышения ценовой категории, можно увеличить количество экземпляров, имеющееся в вашей службе. Это называется *увеличением масштаба* или *уменьшением масштаба*. Дополнительные сведения об *увеличении масштаба* и *уменьшении масштаба* см. в статье [Масштабирование числа экземпляров вручную или автоматически](../insights-how-to-scale.md).

Сведения о планах обслуживания приложений (прежнее название — планы веб-размещения) см. в разделах [Что такое план обслуживания приложений?](../web-sites-web-hosting-plan-overview.md) и [Подробный обзор планов обслуживания приложений Azure](azure-web-sites-web-hosting-plans-in-depth-overview.md). Сведения о ценах и функциях отдельных планов обслуживания приложений см. в разделе [Сведения о ценах на службу приложений](/pricing/details/app-service/).

> [AZURE.NOTE]Прежде чем переключиться из режима **Free** в режим **Basic**, **Standard** или **Premium**, следует снять существующие ограничения на расходы, установленные для вашей подписки на службу приложений Azure. Сведения о просмотре или изменении параметров подписки на службу приложений Microsoft Azure см. в разделе [Подписки Microsoft Azure][azuresubscriptions].

## Выбор ценовой категории

1. Откройте в браузере [портал Azure][portal] и перейдите в приложение, которое требуется масштабировать.
	
2. В разделе **Основные компоненты** для вашего приложения щелкните ссылку **План обслуживания/ценовая категория приложения**.

3. Щелкните элемент **Ценовая категория**; появится список ценовых категорий, доступных для вашего плана. Для каждого уровня указана расчетная цена, чтобы вы могли представить себе среднюю стоимость для этого уровня.
	
	![Выбор плана](./media/app-service-scale/ChoosePricingTier.png)
	
4. После выбора уровня нажмите кнопку **Выбрать**.
	
	После завершения операции на вкладке **Уведомления** появится зеленая надпись **ВЫПОЛНЕНО**.
		
<a name="ScalingSQLServer"></a>
##Масштабирование связанных ресурсов
Если приложение зависит от других служб, таких как SQL или служба хранилища, их также можно масштабировать в зависимости от ваших потребностей.

1. В разделе **Основные компоненты** щелкните ссылку **Группа ресурсов**.

2. Затем в области **Сводка** колонки группы ресурсов щелкните одну из баз данных (или другой ресурс, который требуется масштабировать).

	![Связанная база данных](./media/app-service-scale/ResourceGroup.png)
	
3. В этой колонке связанных ресурсов щелкните область **Ценовая категория**, выберите одну из категорий в зависимости от требований к производительности и нажмите кнопку **Выбрать**.
	
	![Масштабирование базы данных SQL](./media/app-service-scale/ScaleDatabase.png)
	
4. Если приложение использует хранилище, георепликация настраивается автоматически при выборе ценовой категории, которая ее поддерживает. С другой стороны, для SQL необходимо вручную настроить георепликацию, чтобы увеличить возможности высокой доступности и аварийного восстановления базы данных SQL. Для этого щелкните область **Георепликация**.
	
	![Настройка георепликации для базы данных SQL](./media/app-service-scale/GeoReplication.png)
	
<a name="devfeatures"></a>
## Возможности разработчика
В зависимости от ценовой категории будут доступны следующие возможности для разработчиков.

### Разрядность ###

- Уровни **Basic**, **Standard** и **Premium** поддерживают 64-разрядные и 32-разрядные приложения.
- Уровни плана **Free** and **Shared** поддерживают только 32-разрядные приложения.

### Поддержка отладчика ###

- Поддержка отладчика доступна в режимах **Free**, **Shared** и **Basic** при 1 одновременном подключении на план обслуживания приложений.
- Поддержка отладчика доступна в режимах **Standard** и **Premium** при 5 одновременных подключениях на план обслуживания приложений.

# <a name="app-service-environment-management-addresses"></a>Адреса управления среды службы приложений

Среда службы приложений (ASE) — это развернутая служба приложений Azure в подсети виртуальной сети Azure.  Среда ASE должна быть доступна из плоскости управления, используемой службой приложений Azure.  Такой трафик управления ASE проходит через контролируемую пользователем сеть. Если этот трафик блокируется или неправильно маршрутизируется, работа ASE приостанавливается. Подробные сведения о зависимостях сетей ASE см. в статье [Рекомендации по работе с сетями в среде службы приложений][networking]. Для получения общей информации о среде ASE можно начать со статьи [Общие сведения о средах службы приложений][intro].

В этом документе перечислены адреса источников службы приложений, которые передают трафик управления в среду ASE и выполняют две важные задачи.  

1. С помощью этих адресов можно создать группы безопасности сети для блокировки входящего трафика.  
2. Маршруты с этими адресами вы можете создать для поддержки развертываний с принудительным туннелированием. Дополнительные сведения о настройке работы ASE в среде, где исходящий трафик передается локально, см. в статье [Настройка принудительного туннелирования в среде службы приложений][forcedtunnel].

Каждая среда ASE имеет общедоступный виртуальный IP-адрес, на который поступает трафик управления. Входящий трафик управления с этих адресов на поступает на порты 454 и 455 для общедоступного виртуального IP-адреса ASE.

## <a name="get-your-management-addresses-from-api"></a>Получение адресов управления из API ##

Доступен вызов API, который можно выполнить, чтобы получить список адресов управления, которые соответствуют вашей среде ASE.  Он приведен ниже.

    get /subscriptions/<subscription ID>/resourceGroups/<resource group>/providers/Microsoft.Web/hostingEnvironments/<ASE Name>/inboundnetworkdependenciesendpoints?api-version=2016-09-01

Этот вызов возвращает документ JSON, который содержит все входящие адреса для среды ASE, включая адреса управления, виртуальный IP-адрес, используемый ASE, и подсеть ASE.  

Для вызова API с помощью [armclient](http://github.com/projectkudu/ARMClient) используйте приведенные ниже команды, но укажите свои идентификатор подписки, группу ресурсов и имя среды ASE.  

    armclient login
    armclient get /subscriptions/<subscription ID>/resourceGroups/<resource group>/providers/Microsoft.Web/hostingEnvironments/<ASE Name>/inboundnetworkdependenciesendpoints?api-version=2016-09-01

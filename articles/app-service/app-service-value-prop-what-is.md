# Что такое служба приложений Azure?

*Служба приложений* — это решение на основе модели [платформа как служба](https://en.wikipedia.org/wiki/Platform_as_a_service) (PaaS) от Microsoft Azure, с помощью которого можно создавать веб- и мобильные приложения для любой платформы или любого устройства. Вы можете с легкостью интегрировать приложения с решениями SaaS (например, Office 365, Dynamics CRM, Salesforce, Twilio), подключаться к локальным приложениям (например, SAP, Oracle, Siebel) и автоматизировать бизнес-процессы. При этом решения SaaS удовлетворяют строгим требованиям к безопасности, надежности и масштабируемости.

Служба приложений обладает возможностями для работы в сетевой и мобильной среде, которые ранее предлагались в составе отдельных решений Веб-сайты Azure и мобильные службы Azure. Она также предусматривает новые возможности для автоматизации бизнес-процессов и размещения облачных API.

## Типы приложений в службе приложений

Ниже представлены типы приложений, которые можно создать в службе приложений для выполнения кода или рабочих процессов.

- [**Веб-приложения**](../app-service-web/app-service-web-overview.md) для размещения веб-сайтов и веб-приложений.

- [**Мобильные приложения**](../app-service-mobile/app-service-mobile-value-prop.md) для размещения серверной части мобильных приложений.
   
- [**Приложения API**](../app-service-api/app-service-api-apps-why-best-platform.md) для размещения облачных API.
 
- [**Приложения логики**](../app-service-logic/app-service-logic-what-are-logic-apps.md) для автоматизации доступа к данным и их использования в облаке без написания кода.

Интегрированная служба приложений позволяет легко создать единое решение из нескольких типов приложений.

## Планы и среды службы приложений

[Планы службы приложений](azure-web-sites-web-hosting-plans-in-depth-overview.md) представляют собой вычислительные ресурсы, которые используют приложения. План с низкой ценовой категорией предусматривает работу с приложениями на общих виртуальных машинах. При выборе более высокой ценовой категории приложения можно использовать на выделенных виртуальных машинах. Доступно несколько размеров виртуальных машин. Ценовую категорию можно изменить без риска простоев. Если вам требуется больше возможностей в отношении масштабируемости и сетевой изоляции, для работы с приложениями можно использовать [среду службы приложений](../app-service-web/app-service-app-service-environment-intro.md).

## Преимущества службы приложений

Ниже приведены некоторые основные функции и возможности службы приложений.

- **Полностью управляемая платформа.** Автоматическое исправление операционной системы и платформы, встроенная поддержка резервного копирования и аварийного восстановления. 

- **Возможность использования имеющихся навыков.** Работайте со знакомыми языками, платформами и средами разработки. Служба приложений поддерживает .NET, Node.js, Java, PHP и Python.

- **Быстрое развертывание.** Подготавливайте новые приложения к работе и развертывайте в них код за считанные секунды.

- **Непрерывная интеграция.** Настраивайте [непрерывную интеграцию и развертывание](../app-service-web/app-service-continous-deployment.md) в Visual Studio Team Services, GitHub или BitBucket.

- **Промежуточные и тестовые среды.** Реализуйте [промежуточное развертывание](../app-service-web/web-sites-staged-publishing.md) для проверки кода в подготовительной среде, которая идентична рабочей. А затем выпускайте новые версии приложения с нулевым временем простоя, выполняя операции замены.

- **Тестирование в рабочей среде.** Используйте промежуточные развертывания более эффективно, [выполняя A/B-тестирование](../app-service-web/app-service-web-test-in-production-get-start.md) для проверки нового кода с помощью настраиваемого объема реального трафика.

- **Проверка подлинности и авторизация.** Защитите свои приложения от несанкционированного доступа, не внося изменения в код. Службы встроенной проверки подлинности защищают ваши приложения, обеспечивая доступ для пользователей, клиентов, представляющих пользователей, и служб. В число поддерживаемых поставщиков удостоверений входят: Azure Active Directory, Facebook, Twitter, Google и учетная запись Майкрософт. Дополнительные сведения см. в статье [Проверка подлинности и авторизация в службе приложений Azure](app-service-authentication-overview.md).

- **Подключение к любой службе.** Подключайте приложение к корпоративным системам или платформам SaaS (программное обеспечение как услуга) всего за несколько минут с помощью встроенных [соединителей](../connectors/apis-list.md). Более 50 соединителей для корпоративных систем на выбор — SAP, Siebel и Oracle, популярные корпоративные службы SaaS, такие как Salesforce и Office 365, и популярные интернет-службы, такие как Facebook, Twitter и Dropbox.

- **Глобальное масштабирование.** [Увеличивайте](../app-service/app-service-scale.md) и [уменьшайте](../azure-portal/insights-how-to-scale.md) масштаб для обработки любой входящей пользовательской нагрузки. Вручную выбирайте количество и размер виртуальных машин или настраивайте автомасштабирование на основе нагрузки или по расписанию. Глобальная инфраструктура центров обработки данных Майкрософт позволяет размещать приложения, упрощая репликацию данных и размещение служб в нескольких расположениях.

- **Корпоративный уровень.** Служба приложений предназначена для создания и размещения безопасных и критически важных приложений. Создавайте интегрированные с Active Directory бизнес-приложения, которые безопасно подключаются к локальным ресурсам, и размещайте их на безопасной облачной платформе, совместимой со стандартами [ISO, SOC и PCI](https://www.microsoft.com/TrustCenter/). Все это возможно в рамках [соглашения об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/app-service/) корпоративного класса.

- **Azure Marketplace.** Выбирайте из постоянно пополняемого [списка шаблонов приложений](https://azure.microsoft.com/marketplace/). Используйте лучшие решения сообщества по приложениям OSS, одним щелчком мыши устанавливая такие пакеты, как WordPress, Joomla и Drupal.

- **Веб-задания.** [Запускайте любые программы или сценарии](../app-service-web/web-sites-create-web-jobs.md) на виртуальных машинах службы приложений. Выполнять задания можно по расписанию, непрерывно или с активацией событиями. [Пакет SDK для веб-заданий](../app-service-web/websites-dotnet-webjobs-sdk.md) Azure упрощает написание кода для интеграции с другими службами Azure и сторонних поставщиков.

- **Гибридные подключения.** Получайте доступ к локальным данным с помощью [гибридных подключений](../biztalk-services/integration-hybrid-connection-overview.md) и [виртуальных сетей Azure](../app-service-web/web-sites-integrate-with-vnet.md).

- **Интеграция с Visual Studio.** Специальные средства Visual Studio упрощают создание, развертывание, использование и отладку веб-приложений, мобильных приложений и приложений API, а также управление ими.

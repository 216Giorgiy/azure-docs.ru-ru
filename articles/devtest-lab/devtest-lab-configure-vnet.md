# Настройка виртуальной сети для лаборатории для разработки и тестирования

## Обзор

Как описано в статье [Добавление виртуальной машины с артефактами в лабораторию для разработки и тестирования Azure](devtest-lab-add-vm-with-artifacts.md), при создании виртуальной машины в лаборатории можно указать настроенную виртуальную сеть (и подсеть) для этой виртуальной машины. Это может потребоваться, например, когда необходим доступ к ресурсам корпоративной сети с виртуальных машин через виртуальную сеть, которая была настроена с помощью ExpressRoute или через VPN-подключение типа "сеть-сеть". В следующих разделах показано, как добавить существующую виртуальную сеть в настройках виртуальной сети лаборатории, чтобы она предлагалась для выбора при создании виртуальных машин.

## Настройка виртуальной сети для лаборатории для разработки и тестирования с помощью портала Azure
Далее пошагово описывается процедура добавления существующей виртуальной сети (и подсети) в лабораторию для разработки и тестирования, чтобы их можно было использовать при создании виртуальной машины в этой лаборатории.

1. Войдите на [портал Azure](https://portal.azure.com).

1. Выберите **Обзор**, а затем в списке выберите **Лаборатории для разработки и тестирования**.

1. В списке лабораторий найдите нужную лабораторию.

1. Появится колонка **Параметры** для выбранной лаборатории.

1. Выберите **Виртуальные сети**.

	![Виртуальные сети можно настраивать в колонке "Параметры" лаборатории](./media/devtest-lab-configure-vnet/lab-settings-vnet.png)
	
1. В колонке **Виртуальные сети** отобразится список виртуальных сетей, настроенных для текущей лаборатории, а также виртуальная сеть по умолчанию, созданная для лаборатории.

1. Выберите **+ Добавить**.

	![Добавление существующей виртуальной сети в лабораторию](./media/devtest-lab-configure-vnet/lab-settings-vnet-add.png)
	
1. В колонке **Виртуальная сеть** выберите **[Выбрать виртуальную сеть]**.

	![Выбор существующей виртуальной сети](./media/devtest-lab-configure-vnet/lab-settings-vnets-vnet1.png)
	
1. В колонке **Выбор виртуальной сети** выберите нужную виртуальную сеть. В колонке отображаются все виртуальные сети, которые находятся в том же регионе подписки, что и лаборатория.

1. После выбора виртуальной сети и возвращения к колонке **Виртуальная сеть** вы увидите, что несколько полей были включены.

	![Выбор существующей виртуальной сети](./media/devtest-lab-configure-vnet/lab-settings-vnets-vnet2.png)

1. Укажите описание для пары "виртуальная сеть-лаборатория".

1. Чтобы подсеть можно было использовать при создании лабораторной виртуальной машины, установите флажок **ИСПОЛЬЗОВАТЬ ПРИ СОЗДАНИИ ВИРТУАЛЬНОЙ МАШИНЫ**.

1. Чтобы разрешить использование общедоступных IP-адресов в подсети, установите флажок **РАЗРЕШИТЬ ОБЩЕДОСТУПНЫЕ IP-АДРЕСА**.

1. В поле **МАКСИМАЛЬНОЕ ЧИСЛО ВИРТУАЛЬНЫХ МАШИН НА ПОЛЬЗОВАТЕЛЯ** укажите максимальное число виртуальных машин на пользователя для каждой подсети. Чтобы разрешить использовать неограниченное число виртуальных машин, оставьте это поле пустым.

1. Нажмите **Сохранить**.

1. После настройки виртуальной сети ее можно выбирать при создании новой виртуальной машины. Об этом рассказывается в статье [Добавление виртуальной машины с артефактами в лабораторию для разработки и тестирования Azure](devtest-lab-add-vm-with-artifacts.md).

## Дальнейшие действия

После добавления нужных виртуальных сетей в лабораторию нужно [добавить виртуальную машину в лабораторию для разработки и тестирования](devtest-lab-add-vm-with-artifacts.md).

#Общие сведения о службах мультимедиа Azure

Службы мультимедиа Azure - это расширяемая облачная платформа, которая позволяет разработчикам создавать масштабируемые приложения для управления и доставки файлов мультимедиа. С помощью служб мультимедиа Azure можно безопасно передавать, сохранять, кодировать и упаковывать видео- или аудиосодержимое как локально, так и для потоковой трансляции разным клиентам (например, на ТВ, ПК и мобильные устройства).

Можно создавать сквозные рабочие процессы, полностью использующие службы мультимедиа. Также можно использовать сторонние компоненты в качестве некоторых частей рабочего процесса. Например, можно выполнять кодирование с помощью стороннего кодировщика. А затем отправить, защитить, упаковать и доставить содержимое с использованием служб мультимедиа.

Для сборки решений на базе служб мультимедиа можно использовать

- [REST API для службы мультимедиа](https://msdn.microsoft.com/library/azure/hh973617.aspx)
- Один из доступных клиентских пакетов SDK: [Пакет SDK служб мультимедиа Azure для .NET](https://github.com/Azure/azure-sdk-for-media-services), [Пакет SDK Azure для Java](https://github.com/Azure/azure-sdk-for-java), [службы мультимедиа Azure для Node.js](https://github.com/fritzy/node-azure-media), [Пакет SDK Azure для PHP](https://github.com/Azure/azure-sdk-for-php)
- Существующие средства: [портал управления Azure](http://manage.windowsazure.com/) или [Azure-Media-Services-Explorer](https://github.com/Azure/Azure-Media-Services-Explorer).

**Соглашение об уровне обслуживания (SLA)**: Службы мультимедиа гарантируют доступность транзакций через интерфейс API REST для кодирования с помощью служб мультимедиа на уровне 99,9%. Служба потоковой передачи по требованию будет успешно обслуживать запросы с доступностью на уровне 99,9% для существующего мультимедиа-контента, если приобретен по крайней мере один зарезервированный элемент потоковой передачи по требованию. Доступность вычисляется для полного месяца, за который выставляется счет. Для получения дополнительных сведений загрузите [документ соглашения об уровне обслуживания](https://www.microsoft.com/download/details.aspx?id=39302).

На следующем плакате показаны рабочие потоки служб мультимедиа Azure, начиная от создания медиаданных до их потребления. Загрузить плакат можно отсюда: [Плакат "Службы мультимедиа Azure"](http://www.microsoft.com/download/details.aspx?id=38195).

##Выбор опыта работы с медиаданными

Одним из первых шагов в распространении видеоконтента является выбор, какой тип опыта работы будут получать клиенты. Как клиенты будут просматривать видеоконтент? Какой вид подключения к Интернету они используют? Будут ли они просматривать содержимое на компьютере или на мобильном устройстве? Ожидают ли клиенты получения видео в высоком разрешении? Подобные вопросы смогут помочь клиентам получить максимально возможный опыт.

###Доступ к видео
 
Существуют четыре основных способа получения доступа к видео для клиентов:

- Просмотр в автономном режиме 
- Прогрессивное скачивание
- Потоковая передача (потоковая передача с адаптивной скоростью)

####Просмотр в автономном режиме

Чтобы просмотреть видео в автономном режиме, клиент должен полностью скачать видеофайл на персональный компьютер или мобильное устройство. Размеры видеофайлов имеют тенденцию к увеличению, поэтому скачивание может занять некоторое время. Преимущество просмотра в автономном режиме заключается в отсутствии необходимости наличия подключения к Интернету после его загрузки на устройство. 

####Прогрессивное скачивание

Прогрессивное скачивание требует наличия подключения к Интернету у клиента и позволяет ему начать просмотр видео до того, как оно было полностью загружено. Подходы просмотра в автономном режиме и прогрессивного скачивания требуют, чтобы на используемом клиентом для просмотра видео устройстве было достаточно свободного места для хранения видеофайла полностью.

####потоковая

Технологии потоковой передачи данных также требуют подключения к Интернету, но они скачивают небольшую часть видео за один раз и удаляют ее после отображения на экране. Для этого требуется совсем немного свободного места на устройстве. Пропускная способность сетевого соединения может варьироваться, но клиенты по-прежнему могут ожидать, что у них будет возможность смотреть видео независимо от пропускной способности сети. Технологии передачи с адаптивной скоростью позволяют видеопроигрывателю определять условия работы сети и выбрать одну из нескольких скоростей потока (битрейтов). Если условия в сети ухудшаются, клиент может выбрать более низкую скорость, позволяя видеопроигрывателю продолжать воспроизводить видео в более низком качестве. Если состояние в сети улучшается, то клиент может переключиться на более высокий битрейт с более высоким качеством видео. Службы мультимедиа Azure поддерживают следующие технологии потоковой передачи с переменным битрейтом: HTTP Live Streaming (HLS), Smooth Streaming, MPEG DASH и HDS.

###Используемые устройства

Еще одно решение, которое необходимо принять - тип используемых клиентом устройств для просмотра видео. Службы мультимедиа обеспечивают поддержку веб-браузеров, смартфонов, планшетов, игровых приставок XBOX, абонентских приставок и смарт-ТВ.

####Веб-браузеры

Веб-браузеры могут быть запущены на персональных компьютерах с ОС Windows, компьютерах Macintosh и смартфонах. При работе на ПК с ОС Windows или Macintosh можно воспользоваться преимуществами экрана с большим размером большим объемом места для хранения информации. Это позволяет организовать потоковую передачу видео высокого качества. На ПК с ОС Windows или Macintosh можно просматривать видео, доставленное службами мультимедиа, используя собственное приложение, либо HTML-совместимый веб-браузер. Собственные приложения могут поддерживать технологии Smooth Streaming, Apple HLS, прогрессивное скачивание или просмотр в автономном режиме. Веб-страницы HTML5 поддерживают прогрессивное скачивание.


####Смартфоны

У смартфонов меньше экран и меньшие возможности по хранению информации. Для подобных устройств лучшим выбором является потоковая передача. Поддерживаются iPhone, Windows Phone и телефоны Android. iPhone и телефоны Android поддерживают Smooth Streaming и HLS. Телефоны Windows Phone поддерживают Smooth Streaming.

###Планшеты

Планшеты имеют более большие размеры экрана, чем смартфоны, но как правило меньший объем места для хранения информации. Для планшетов лучшим выбором является потоковая передача. Планшеты с большим объемом места для хранения также могут использовать возможность просмотра в автономном режиме, а также прогрессивного скачивания.

####Игровая приставка XBox

Консоли XBox имеют возможность использования большого экрана и объема места для хранения информации, для них подходят просмотр в автономном режиме, прогрессивное скачивание и потоковая трансляция.
Абонентские приставки и смарт-ТВ
Эти устройства также имеют большие экраны, но минимальный объем места для хранения информации, для них подходит потоковая трансляция.

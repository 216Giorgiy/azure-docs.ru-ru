# <a name="how-to-use-cards-part-1-of-2"></a>Как использовать карты (часть 1 из 2)
В этом руководстве показано, как добавить в бот карту заполняемой формы. Также здесь описано, как поля форм перемещаются в сущности.

Средство для обучения ведению диалога предполагает, что файлы с определениями карт размещаются в каталоге с именем cards, созданном в каталоге, из которого запущен бот.

## <a name="requirements"></a>Требования
Для работы с этим руководством требуется запущенный бот обучения общего назначения.

    npm run tutorial-general

## <a name="details"></a>Сведения

Карты являются элементом пользовательского интерфейса, с помощью которых пользователь может выбрать вариант ответа в диалоге. 

### <a name="open-the-demo"></a>Запуск демонстрационного примера

В веб-интерфейсе откройте список приложений и щелкните элемент Tutorial-14-Cards-2. 

### <a name="the-card"></a>Карта

Определение карты находится по следующему адресу: C:\<каталог_установки\>\src\cards\shippingAddress.json.

Эта карта собирает данные для трех полей адреса доставки: город, улица и штат.

![](../media/tutorial14_card.PNG)

### <a name="actions"></a>Действия

Мы создали три действия. Как показано ниже, первое действие представляет собой карту.

![](../media/tutorial14_actions.PNG)

Давайте посмотрим, как был создан тип действия "Карта":

- Обратите внимание на сущность Address-Street, для которой указан тип Input.text и идентификатор.
- Также здесь есть аналогичная сущность Address-City и раскрывающийся список с идентификаторами Address-State.

Эти идентификаторы очень важны, так как это имена сущностей, которые при заполнении и отправки полей в боте получат свои значения.

## <a name="entities"></a>Сущности
Мы определили три сущности, соответствующие параметрам определенной выше карты.

![](../media/tutorial14_entities.PNG)

## <a name="actions"></a>Действия

Мы определили два действия.

![](../media/tutorial14_actions.PNG)

- Первое из них представляет собой карту адреса доставки. Оно имеет тип действия CARD, а в качестве шаблона в раскрывающемся списке выбран элемент shippingAddress.
- Второе — это простое действие для обратного чтения адреса доставки.

![](../media/tutorial14_sa_card.PNG)

### <a name="train-dialog"></a>Обучение диалогу

Давайте подробнее рассмотрим диалог для обучения.

1. Щелкните "Train Dialogs" (Обучение диалогам), а затем — "New Train Dialog" (Обучение новому диалогу).
1. Введите строку "Привет".
2. Выберите действие оценки.
3. Щелкните "Shipping Address" (Адрес доставки).
4. Заполните и отправьте карту.
    - Обратите внимание, что введенные значения теперь помещены в память сущности. Анализ данных не требуется, как форма уже разбила входные данные на разделы.
5. Щелкните "Score Actions" (Оценить действия).
3. Щелкните элемент Shipping to $Address... (Доставка на $Address).
4. Щелкните "Done Testing" (Завершить тестирование).

![](../media/tutorial14_train_dialog.PNG)

Теперь вы знаете, как получать значения из карты с заполняемыми полями и раскрывающимися списками, а также как записывать и собирать эти данные в сущностях для ботов.

# <a name="entity-resolvers"></a>Сопоставители сущностей

В этом руководстве содержатся сведения об использовании сопоставителей сущностей в Conversation Learner.

## <a name="video"></a>Видео

[![Предварительная версия руководства по использованию сопоставителей сущностей](https://aka.ms/cl_Tutorial_v3_EntityResolvers_Preview)](https://aka.ms/cl_Tutorial_v3_EntityResolvers)

## <a name="requirements"></a>Требования
Для работы с этим руководством требуется запущенный бот обучения общего назначения.

    npm run tutorial-general

## <a name="details"></a>Сведения

- Тип сопоставителя — это необязательное свойство настраиваемых сущностей.
- Сопоставители сущностей используют все возможности предварительно обученных распознавателей сущностей в LUIS, чтобы привнести дополнительные сведения и обеспечить ясность в отношении настраиваемых сущностей.

## <a name="steps"></a>Действия

### <a name="create-a-new-model"></a>Создание модели

1. В пользовательском веб-интерфейсе нажмите кнопку "Создать модель".
2. В поле "Имя" введите "Сопоставители сущностей", нажмите клавишу ВВОД или кнопку "Создать".

### <a name="create-a-pair-of-entities"></a>Создание пары сущностей

1. На левой панели щелкните "Сущности", а затем нажмите кнопку "Создать сущность".
2. В поле "Имя сущности" введите departure (отправление).
3. В раскрывающемся списке "Тип сопоставителя" выберите datetimeV2.
4. Нажмите кнопку "Создать".
5. Прочтите сведения во всплывающем информационном окне и нажмите кнопку OK.
6. Выполните те же шаги, чтобы создать другую сущность с именем return (возвращение) и типом сопоставителя datetimeV2.

### <a name="create-a-pair-of-actions"></a>Создание пары действий

1. На левой панели щелкните "Действия", а затем нажмите кнопку "Создать действие".
2. В поле "Ответ бота" введите You are leaving on $departure and returning on $return (Вы уезжаете $departure и возвращаетесь $return).
    - ВАЖНО! При вводе в поле $[entityName] нужно нажать клавишу ВВОД или выбрать сущность в раскрывающемся списке. В противном случае Conversation Learner будет считать эти данные текстом, а не сущностью.
    - Обратите внимание, что эти сущности также будут указаны в поле "Обязательные сущности" и удалить их невозможно. В этом случае действие будет доступно только после указания обеих обязательных сущностей.
3. Нажмите кнопку "Создать".
4. Еще раз нажмите кнопку "Создать действие", чтобы создать второе действие.
5. В поле "Ответ бота" введите When are you planning to travel? (Когда вы планируете уехать?).
6. В поле "Дисквалифицирующие сущности" введите departure (отправление) и return (возвращение).
    - Они указывают боту НЕ выполнять это действие, если какая-либо из этих сущностей содержит значение.
7. Нажмите кнопку "Создать".


### <a name="training"></a>Обучение

1. Подождите, пока индикатор "Обучение: [состояние]" в верхнем левом углу страницы не перейдет в состояние "Завершено".
    - Если смена состояния занимает слишком много времени, щелкните ссылку "Обновить".
    - Обучение должно находиться в состоянии "Завершено", чтобы сопоставители сущностей могли работать при обучении модели.
2. На левой панели щелкните "Обучающие диалоги" и нажмите кнопку "Создать обучающий диалог".
3. Введите первую фразу пользователя book me a flight (забронировать мне авиабилет). 
4. Нажмите кнопку "Оценить действия".
5. Выберите ответ When are you planning to travel? (Когда вы планируете уехать?).
6. Как пользователь, ответьте: "Leaving tomorrow and returning Sunday next week" (Уезжаю завтра и возвращаюсь в воскресенье на следующей неделе).
    - Обратите внимание, что Conversation Learner обнаружил две сущности "Предварительно обученная дата".
7. На панели "Обнаружение сущностей" выделите текст tomorrow (завтра) и пометьте его как departure (отправление).
8. Пометьте текст Sunday next week (в воскресенье на следующей неделе) как return (возвращение).
9. Нажмите кнопку "Оценить действия".
    - Обратите внимание, что на панели "Память" указаны даты отправления и возвращения.
    - Наведите указатель мыши на каждый из них и посмотрите, что сущности представляют собой объекты даты, которые четко отображают фактические календарные даты вместо Sunday (воскресенье) или tomorrow (завтра).
10. Выберите ответ бота You are leaving on... (Вы уезжаете...).
11. Нажмите кнопку "Сохранить".

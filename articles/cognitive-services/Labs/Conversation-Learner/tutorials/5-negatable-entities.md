# <a name="how-to-use-negatable-entities-with-a-conversation-learner-application"></a>Использование отрицаемых сущностей в приложении Conversation Learner.

В этом руководстве объясняется свойство negatable для сущностей.

## <a name="requirements"></a>Требования
Для работы с этим руководством требуется запущенный бот обучения общего назначения.

    npm run tutorial-general

## <a name="details"></a>Сведения
Действие следует обозначать как "отрицаемое", если пользователю нужна возможность "очистить" значение этой сущности, как например в таких контекстах: "мне не нужно $entity" или "я не буду использовать $entity". Конкретный пример: "меня не интересуют рейсы, вылетающие из Бостона".

На техническом уровне свойство negatable для сущности означает следующее:

- При назначении меток упоминаниям этой сущности вы можете отдельно обозначать как обычные (положительные), так и "отрицательные" экземпляры сущности.
- LUIS обучает две модели сущности: одну для положительных экземпляров и другую для отрицательных.
- Значение отрицательного экземпляра сущности заключается в удалении соответствующего значения (если оно существовало) из переменной сущности.

## <a name="steps"></a>Действия

### <a name="create-the-application"></a>Создание приложения

1. В пользовательском веб-интерфейсе щелкните "New App" (Новое приложение).
2. В поле имени введите NegatableEntity. Затем щелкните "Create" (Создать).

### <a name="create-an-entity"></a>Создание сущности

1. Щелкните "Entities" (Сущности), затем — "New Entity" (Новая сущность).
2. В поле "Entity Name" (Имя сущности) введите имя.
3. Установите флажок Negatable (Отрицаемая).
    - Это означает, что пользователь сможет не только назначить значение сущности, но и явным образом указать, что некоторое значение *не* входит в значение сущности. Во втором случае совпадающий элемент удаляется из значения сущности.
3. Щелкните Создать.

![](../media/tutorial5_entities.PNG)

### <a name="create-two-actions"></a>Создание двух действий

1. Нажмите "Actions" (Действия), затем "New Action" (Создать действие).
2. В поле "Response" (Ответ) введите "I don't know your name" (Я не знаю, как тебя зовут).
3. В поле "Disqualifying Entities" (Блокирующие сущности) введите name (имя).
3. Нажмите кнопку "Создать"

Теперь создайте второе действие.

1. Выберите "Actions" (Действия) и "New Action" (Создать действие), чтобы создать второе действие.
3. В поле "Response" (Ответ) введите "I know your name. It is $name" (Я знаю, как тебя зовут. Твое имя $name).
4. Нажмите кнопку "Создать"

Теперь у вас есть два действия.

![](../media/tutorial5_actions.PNG)

### <a name="train-the-bot"></a>Обучение бота

1. Щелкните "Train Dialogs" (Обучение диалогам), а затем — "New Train Dialog" (Обучение новому диалогу).
2. Введите "hello" (Привет).
3. Щелкните Score Actions (Оценка действий) и выберите вариант "I don't know your name" (Я не знаю, как тебя зовут).
    - Обратите внимание, что оценка имеет значение 100 %, так как это единственное допустимое действие.
2. Введите "my name is david" (Меня зовут Давид).
3. Выберите "david" (Давид) и щелкните метку "+name".
    - Обратите внимание, что эта метка представлена в двух вариантах: +name и -name.  Знак плюс здесь означает, что мы добавляем это значение. Знак минус сообщает системе, что выбранный вариант не является значением сущности.
5. Щелкните "Score Actions" (Оценить действия).
    - Вы увидите, что значение имени сохранилось в памяти бота.
    - Теперь "I know your name. It is $name" (Я знаю, как тебя зовут. Твое имя $name) становится единственным допустимым ответом. 
6. Выберите элемент "I know your name. It is $name" (Я знаю, как тебя зовут. Твое имя $name).

Давайте попробуем очистить значение отрицаемой сущности:

7. Введите "my name is not david" (мое имя не Давид).
    - Обратите внимание, что слово "not" (не) обозначается как имя, основываясь на структуре предыдущей фразы. Это неправильно.
2. Щелкните элемент "not" (не), а затем красный крестик. 
3. Теперь щелкните "david" (Давид).
    - Теперь система распознает отрицательную сущность, которая означает, что указанное имя не может являться значением сущности.
2. Выберите "-name".
3. Щелкните "Score Actions" (Оценить действия).
    - Обратите внимание, что выбранное значение удаляется из памяти.
2. Выберите действие "I don't know your name" (Я не знаю, как тебя зовут), которое снова стало единственным допустимым.

Далее мы продемонстрируем ввод нового значения имени.

3. Введите имя "john" (Иван). Теперь выберите элемент "john" (Иван) и щелкните name.
4. Щелкните "Score Actions" (Оценить действия).
5. Выберите элемент "I know your name. It is $name" (Я знаю, как тебя зовут. Твое имя $name).

Попробуйте заменить введенное имя.

6. Введите "my name is susan" (Меня зовут Светлана).
7. Выберите элемент "I know your name. It is $name" (Я знаю, как тебя зовут. Твое имя $name).
7. Щелкните "Score Actions" (Оценить действия).
8. Обратите внимание, что значение **susan** (Светлана) заменило имя **john** (Иван) в значений сущности.
9. Введите "my name is not susan" (Мое имя не Светлана).
    - Обратите внимание, что система распознала имя как отрицательный экземпляр.
2. Щелкните "Score Actions" (Оценить действия).
3. Выберите действие "I don't know your name" (Я не знаю, как тебя зовут), которое снова стало единственным допустимым.
7. Щелкните "Done Teaching" (Завершить обучение).

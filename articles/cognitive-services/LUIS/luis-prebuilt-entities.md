# <a name="prebuilt-entities"></a>Предварительно созданные сущности

Интеллектуальная служба распознавания речи (LUIS) включает набор предварительно созданных сущностей для распознавания общих типов информации, таких как даты, время, цифры, измерения и валюта. Поддержка предварительно созданной сущности зависит от языка и региональных параметров приложения LUIS. Полный список предварительно созданных сущностей, поддерживаемых LUIS, включая поддержку языка и региональных параметров, см. по ссылке [Entities per culture](./luis-reference-prebuilt-entities.md) (Язык и региональные параметры сущностей).

> [!NOTE]
> Сущность **builtin.datetime** не рекомендуется. Она заменяется на [**builtin.datetimeV2**](luis-reference-prebuilt-datetimev2.md), которая обеспечивает распознавание диапазонов дат и времени, а также улучшенное распознавание неоднозначных дат и времени.

## <a name="add-a-prebuilt-entity"></a>Добавить предварительно созданную сущность

1. Откройте приложение, щелкнув его имя на странице **Мои приложения**, а затем щелкните **Сущности** в левом углу. 
2. На странице **Сущности** щелкните **Управление предварительно созданными сущностями**.

    ![Страница "Сущности" — Управление предварительно созданными сущностями](./media/luis-use-prebuilt-entity/add-prebuilt-entity-button.png)
3. В диалоговом окне**Добавление предварительно созданных сущностей** щелкните на предварительно созданную сущность, которую необходимо добавить (например, "datetimeV2"). Нажмите кнопку **Сохранить**.

    ![Диалоговое окно добавления предварительно созданной сущности](./media/luis-use-prebuilt-entity/add-prebuilt-entity-dialog.png)

## <a name="use-a-prebuilt-number-entity"></a>Использование предварительно созданной сущности чисел
Если предварительно созданная сущность включена в приложение, ее прогнозы включатся в опубликованное приложение. Реакция на событие предварительно созданных сущностей предварительно подготовлена и **не может** быть изменена. Чтобы узнать, как работает предварительно созданная сущность, выполните следующие действия.

1. Добавьте сущность **чисел** в приложение, затем пройдите [Обучение](luis-interactive-test.md) и [опубликуйте](luis-how-to-publish-app.md) приложение.
2. Щелкните URL-адрес конечной точки на странице **Опубликовать приложение**, чтобы открыть конечную точку LUIS в веб-браузере. 
3. Добавьте к URL-адресу фразу, которая содержит числовое выражение. Например, можно ввести `buy two plane ticktets` и увидеть, что LUIS идентифицирует `two` как сущность `builtin.number` и `2` как ее значение в поле `resolution`. Поле `resolution`помогает вам работать с числами и датами в канонической форме, которая проще для использования клиентским приложением. 

    ![высказывание в браузере, содержащее сущность чисел](./media/luis-use-prebuilt-entity/browser-query.png)

LUIS может интеллектуально распознавать числа, которые не входят в нестандартную форму. Попробуйте использовать различные числовые выражения в фразах и посмотрите, что возвращает LUIS.

В следующем примере показан ответ LUIS в формате JSON, который содержит расшифровку высказывания "two dozen" (две дюжины) в значение 24.

```json
{
  "query": "order two dozen tickets for group travel",
  "topScoringIntent": {
    "intent": "BookFlight",
    "score": 0.905443209
  },
  "entities": [
    {
      "entity": "two dozen",
      "type": "builtin.number",
      "startIndex": 6,
      "endIndex": 14,
      "resolution": {
        "value": "24"
      }
    }
  ]
}
```
## <a name="use-a-prebuilt-datetimev2-entity"></a>Использование предварительно созданной сущности datetimeV2
Предварительно созданная сущность **datetimeV2** распознает даты, время, диапазоны дат и интервалы времени. Чтобы узнать, как `datetimeV2` работает предварительно созданная сущность, выполните следующие действия.

1. Добавьте сущность **datetimeV2** в приложение, затем выберите [Обучение](luis-interactive-test.md) и [опубликуйте](luis-how-to-publish-app.md) приложение.
2. Щелкните URL-адрес конечной точки на странице **Опубликовать приложение**, чтобы открыть конечную точку LUIS в веб-браузере. 
3. Добавьте фразу к URL-адресу, которая содержит диапазон дат. Например, можно ввести `book a flight tomorrow` и увидеть, что LUIS идентифицирует `tomorrow` как `builtin.datetimeV2.date`сущность и определяет завтрашнюю дату как свое значение в поле `resolution`. 

В следующем примере показан ответ LUIS в формате JSON, если сегодняшняя дата — 31 октября 2017 года.

```json
{
  "query": "book a flight tomorrow",
  "topScoringIntent": {
    "intent": "BookFlight",
    "score": 0.9063408
  },
  "entities": [
    {
      "entity": "tomorrow",
      "type": "builtin.datetimeV2.date",
      "startIndex": 14,
      "endIndex": 21,
      "resolution": {
        "values": [
          {
            "timex": "2017-11-01",
            "type": "date",
            "value": "2017-11-01"
          }
        ]
      }
    }
  ]
}
```

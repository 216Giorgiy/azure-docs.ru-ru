# <a name="versions"></a>Версии
Создавайте разные модели одного приложения с помощью [версий](luis-how-to-manage-versions.md). 

## <a name="version-id"></a>ИД версии
ИД версии состоит из символов, цифр или точки ".". Его длина не может превышать 10 символов.

## <a name="initial-version"></a>Исходная версия
Исходная версия (0.1) является заданной по умолчанию активной версией. 

## <a name="active-version"></a>Активная версия
[Установка версии](luis-how-to-manage-versions.md#set-active-version) в качестве активной означает, что в настоящее время она редактируется и тестируется на веб-сайте [LUIS](luis-reference-regions.md). Задайте версию в качестве активной, чтобы получать доступ к ее данным, вносить изменения, а также тестировать и публиковать ее.

Имя текущей активной версии отображается на верхней левой панели после имени приложения. 

[ ![Изменение активной версии](./media/luis-concept-version/version-in-nav-bar-inline.png) ](./media/luis-concept-version/version-in-nav-bar-expanded.png#lightbox)

## <a name="versions-and-publishing-slots"></a>Версии и слоты публикации
Публикация выполняется в промежуточный или рабочий слоты. У слотов могут быть разные или одинаковые версии. Это полезно для проверки изменений в версиях модели с помощью конечной точки, которая доступна ботам или другим вызывающим приложениям LUIS. 

## <a name="clone-a-version"></a>Клонирование версии
Клонируйте версию, чтобы создать копию существующей версии, и сохраните ее как новую версию. Клонируйте версию, чтобы использовать то же содержимое существующей версии в качестве отправной точки для новой версии. После клонирования версии новая версия становится **активной**. 

## <a name="import-and-export-a-version"></a>Импорт и экспорт версии
Версию можно импортировать на уровне приложения. Эта версия становится активной и использует идентификатор версии в свойстве "versionId" файла приложения. Можно также выполнять импорт в существующее приложение на уровне версии. Новая версия становится активной версией. 

Версию можно экспортировать на уровне приложения или на уровне версии. Единственное отличие заключается в том, что экспортированная на уровне приложения версии является текущей активной версией, тогда как на странице **[Параметры](luis-how-to-manage-versions.md)** на уровне версии можно экспортировать любую версию. 

Экспортированный файл не содержит сведения, прошедшие машинное обучение, так как после импорта приложение обучается повторно. Экспортированный файл не содержит участников совместной работы. Их необходимо добавить обратно после импорта версии в новое приложение.

## <a name="export-each-version-as-app-backup"></a>Экспорт каждой версии как резервной копии приложения
Чтобы создать резервную копию приложения LUIS, экспортируйте каждую версию на странице **[Параметры](luis-how-to-manage-versions.md)**.

## <a name="delete-a-version"></a>Удаление версии
Из списка версий на странице параметров можно удалить все версии, кроме активной. 

## <a name="version-availability-at-the-endpoint"></a>Доступность версии в конечной точке
Обученная версия недоступна автоматически в [конечной точке](luis-glossary.md#endpoint) приложения. Чтобы версия была доступна в конечной точке приложения, ее необходимо [опубликовать](luis-how-to-publish-app.md) или повторно опубликовать. При публикации в **промежуточную** и **рабочую** конечные точки у вас будет две доступные версии приложения. Если в конечной точке требуется больше версий приложения, следует экспортировать и повторно импортировать версию в новое приложение. Новое приложение имеет другой идентификатор.

## <a name="collaborators"></a>Участники совместной работы
Владелец и [участники совместной работы](luis-how-to-collaborate.md) имеют полный доступ ко всем версиям приложения.

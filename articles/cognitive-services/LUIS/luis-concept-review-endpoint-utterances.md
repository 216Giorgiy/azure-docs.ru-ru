# <a name="enable-active-learning-by-reviewing-endpoint-utterances"></a>Включение активного обучения путем проверки фрагментов речи конечной точки
Активное обучение является одной из трех стратегий повышения точности прогнозов, при этом самой простой в реализации. 

## <a name="what-is-active-learning"></a>Принципы активного обучения
Процесс активного обучения состоит из двух этапов. Во-первых, служба LUIS выбирает получаемые в конечной точке приложения фрагменты речи, для которых требуется проверка. Второй этап выполняется владельцем приложения или участником совместной работы и включает проверку допустимости выбранных фрагментов для [проверки](luis-how-to-review-endoint-utt.md), включая правильное намерение и сущности в намерении. После проверки фрагментов речи необходимо выполнить повторное обучение и публикацию приложения. 

## <a name="which-utterances-are-on-the-review-list"></a>Какие фрагменты входят в список проверки
Служба LUIS добавляет фрагменты речи в список проверки, если верхнее намерение срабатывания имеет низкую оценку или оценки двух верхних намерений слишком близки. 

## <a name="single-pool-for-utterances-per-app"></a>Один пул для высказывания на каждое приложение
Список **Просмотр высказываний конечной точки** не изменяется в зависимости от версии. Для проверки используется единственный пул фрагментов речи независимо от того, какую версию фрагмента речи вы активно редактируете или какая версия приложения опубликована в конечной точке. 

## <a name="where-are-the-utterances-from"></a>Откуда поступают фрагменты речи
Фрагменты речи конечной точки поступают из запросов конечных пользователей в конечной точке HTTP приложения. Если приложение не опубликовано или еще не получало совпадений, у вас нет фрагментов речи для проверки. Если совпадения конечной точки не получены для определенного намерения или сущности, у вас нет содержащих их фрагментов речи для проверки. 

## <a name="schedule-review-periodically"></a>Планирование периодической проверки
Проверку предлагаемых фрагментов речи необязательно выполнять ежедневно, однако эту задачу необходимо включить в регулярное обслуживание LUIS. 

## <a name="delete-review-items-programmatically"></a>Удаление элементов проверки программным способом
Если приложение имеет большой размер, вы можете проверять некоторые фрагменты речи и удалять остальные фрагменты из списка программным способом. Для этого сначала необходимо [получить](https://westus.dev.cognitive.microsoft.com/docs/services/5890b47c39e2bb17b84a55ff/operations/5890b47c39e2bb052c5b9c0a) список, а затем [удалить](https://westus.dev.cognitive.microsoft.com/docs/services/5890b47c39e2bb17b84a55ff/operations/58b6f32139e2bb139ce823c9) фрагменты речи по идентификатору.

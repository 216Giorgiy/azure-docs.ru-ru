# <a name="what-is-language-understanding-luis"></a>Что собой представляет служба Language Understanding (LUIS)?
Language Understanding (LUIS) — это облачная служба, которая применяет пользовательское машинное обучение к тексту пользователя в разговорном стиле и на естественном языке, чтобы предсказать общий смысл и извлечь соответствующую подробную информацию. 

Клиентское приложение для LUIS может быть любым диалоговым приложением, которое взаимодействует с пользователем на естественном языке для выполнения задачи. Примеры клиентских приложений включают в себя приложения для социальных сетей, чат-боты и классические приложения с поддержкой речи.  

![Концептуальное изображение 3-х приложений, передающих информацию в LUIS](./media/luis-overview/luis-entry-point.png)

## <a name="what-is-a-luis-app"></a>Что такое приложение LUIS?
Приложение LUIS содержит разработанную для вашего домена модель естественного языка. Вы можете запустить приложение LUIS с моделью предварительно созданного домена, создать собственную или объединить части готового домена с собственными пользовательскими данными.

[Модели предварительно созданного домена](luis-how-to-use-prebuilt-domains.md) включают в себя все эти части и являются отличным способом быстро начать использовать LUIS.

Приложение LUIS также содержит параметры интеграции, [участников совместной работы](luis-concept-collaborator.md) и [версии](luis-concept-version.md).

## <a name="using-a-luis-app"></a>Использование приложения LUIS
<a name="Accessing-LUIS"></a>После публикации вашего приложения LUIS клиентское приложение отправляет фразы в [API конечной точки][endpoint-apis] LUIS и получает результаты прогнозирования в виде ответов JSON.

На следующем рисунке сначала чат-бот вашего клиента отправляет текст, который человек говорит своими словами, в приложение LUIS в HTTP-запросе. Затем LUIS применяет вашу обученную модель к естественному языку, чтобы понять входные данные пользователя, и возвращает ответ в формате JSON. После этого чат-бот вашего клиента использует ответ JSON для выполнения запросов пользователя. 

![Концептуальное изображение приложения LUIS, работающего с чат-ботом](./media/luis-overview/luis-overview-process-2.png)

### <a name="example-of-json-endpoint-response"></a>Пример ответа конечной точки JSON

Ответ конечной точки JSON содержит по крайней мере фразу в запросе и намерение с высшим показателем. 

```JSON
{
  "query": "I want to call my HR rep.",
  "topScoringIntent": {
    "intent": "HRContact",
    "score": 0.921233
  },
  "entities": [
    {
      "entity": "call",
      "type": "Contact Type",
      "startIndex": 10,
      "endIndex": 13,
      "score": 0.7615982
    }
  ]
}
```

## <a name="what-is-a-natural-language-model"></a>Что такое модель естественного языка?
Модель начинается со списка общих преднамерений пользователя, называемых _намерениями_, например "забронировать рейс" или "связаться со справочной службой". Вы предоставите пример текста пользователя, называемый _примерами фраз_, для намерений, а затем отметите значимые слова или фразы в высказывании, называемые _сущностями_.


Модель включает в себя:

* **[Намерения](#intents)**: категории намерений пользователя (предполагаемое действие или результат).
* **[Сущности](#entities)**: конкретные типы данных в высказываниях, такие как число, адрес электронной почты или имя.
* **[Примеры фраз](#example-utterances)**: пример текста, который пользователь вводит в клиентское приложение.

### <a name="intents"></a>Намерения 
[Намерение](luis-how-to-add-intents.md) (сокращенно от _преднамерения_) является целью, выраженной в высказывании пользователя, например бронирование рейса, оплата счета или поиск новостной статьи. Намерение создается для каждого действия. Приложение для путешествий LUIS может определить намерение с именем "BookFlight". Ваше клиентское приложение может использовать намерение с самым высоким показателем для запуска действия. Например, когда LUIS возвращает намерение "BookFlight", ваше клиентское приложение может инициировать вызов API к внешней службе для бронирования авиабилета.

### <a name="entities"></a>Сущности
[Сущность](luis-how-to-add-entities.md) представляет собой подробную информацию, найденную в высказывании, которая относится к запросу пользователя. Например, во фразе "забронировать билет в Париж" запрашивается один билет, а "Париж" является местом назначения. Найдены две сущности: "билет", что обозначает один билет, и "Париж", что обозначает место назначения. 

После того как LUIS вернет сущности, найденные во фразе пользователя, клиентское приложение может использовать список сущностей в качестве параметров для запуска действия. Например, для бронирования рейса требуются такие сущности, как пункт назначения, дата и авиакомпания.

LUIS предоставляет несколько способов определения и категоризации сущностей.

* **Предварительно созданные сущности**. В LUIS имеется множество предварительно созданных моделей предметной области, включая намерения, фразы и [предварительно созданные сущности](luis-prebuilt-entities.md). Вы можете использовать эти сущности без намерений и фраз предварительно созданной модели. Предварительно созданные сущности помогают экономить время.

* **Пользовательские сущности**. LUIS предлагает несколько способов идентифицировать собственные [сущности](luis-concept-entity-types.md), включая сущности с машинным обучением, конкретные или литеральные сущности и комбинацию обоих типов.

### <a name="example-utterances"></a>Примеры высказываний
Пример [фразы](luis-how-to-add-example-utterances.md) — это предоставленный пользователем текст, который необходимо понять клиентскому приложению. Это может быть предложение, например "забронировать билет в Париж", или фрагмент предложения, например "бронирование" или "рейс в Париж". Фразы не всегда хорошо сформированы, и может быть много вариантов высказываний для конкретного намерения. Добавьте от 10 до 20 примеров фраз для каждого намерения и отметьте сущности в каждой фразе.

|Пример фразы пользователя|Намерение|Сущности|
|-----------|-----------|-----------|
|"Забронировать рейс до __Сиэтла__?"|BookFlight|Сиэтл;|
|"Когда __открыт__ ваш магазин?"|StoreHoursAndLocation|открыт|
|"Запланировать встречу в __13:00__ с __Бобом__ в Distribution"|ScheduleMeeting|13:00, Боб|

## <a name="improve-prediction-accuracy"></a>Повышение точности прогнозирования
Когда ваше приложение LUIS будет опубликовано и получит реальные пользовательские высказывания, в LUIS можно применить несколько методов для повышения точности прогноза: [активное обучение](#active-learning) фраз конечных точек, [список фраз](#phrase-lists) для включения слов в домен и [шаблоны](#patterns), чтобы уменьшить количество требуемых фраз.

### <a name="active-learning"></a>Активное обучение
В процессе [активного обучения](luis-how-to-review-endoint-utt.md) LUIS позволяет адаптировать ваше приложение LUIS к реальным фразам, выбирая для проверки фразы, полученные им в конечной точке. Вы можете принять или исправить предсказание конечной точки, выполнить переобучение и повторную публикацию. LUIS быстро обучается с помощью этого итеративного процесса, тратя минимальное количество времени и усилий. 

### <a name="phrase-lists"></a>Списки фраз 
LUIS предоставляет [списки фраз](luis-concept-feature.md), чтобы вы могли указать важные слова или фразы в домене модели. LUIS использует эти списки, чтобы добавить дополнительное значение к тем словам и фразам, которые в противном случае не были бы найдены в модели.

### <a name="patterns"></a>Шаблоны 
Шаблоны позволяют преобразовать коллекцию фраз намерений в общие [шаблоны](luis-concept-patterns.md) определенных слов и порядка. Благодаря этому LUIS может быстрее учиться с меньшим количеством примеров фраз для намерений. Шаблоны представляют собой гибридную систему регулярных выражений и выражений с машинным обучением.

## <a name="authoring-and-accessing-luis"></a>Разработка приложения LUIS и получение доступа к нему
Создайте свое приложение LUIS на веб-сайте LUIS программно с помощью API [разработки](https://aka.ms/luis-authoring-apis) или используйте оба варианта в зависимости от необходимости. Доступ к опубликованному приложению LUIS можно получить с помощью запроса [конечной точки](https://aka.ms/luis-endpoint-apis). 

LUIS предоставляет три веб-сайта по всему миру, в зависимости от вашего региона разработки. Регион разработки определяет регион Azure, где вы можете опубликовать свое приложение LUIS.

## <a name="what-technologies-work-with-luis"></a>Какие технологии работают в LUIS?
В LUIS работают следующие технологии Майкрософт:

* [API Bing для проверки орфографии](../bing-spell-check/proof-text.md) обеспечивает коррекцию текста перед предсказанием. 
* [Bot Framework][bot-framework] позволяет чат-боту разговаривать с пользователем с помощью ввода текста. Выберите пакет SDK [3.x](https://github.com/Microsoft/BotBuilder) или [4.x](https://github.com/Microsoft/botbuilder-dotnet) для эффективного использования бота.
* [QnA Maker][qnamaker] позволяет объединять несколько типов текста в базу знаний вопросов и ответов.
* Служба [Речь](../Speech/home.md) преобразует запросы на разговорном языке в текст, после чего их обрабатывает LUIS. Дополнительные сведения см. в [документации по службе "Речь" (предварительная версия)](https://aka.ms/csspeech).
* [Текстовая аналитика](../text-analytics/overview.md) обеспечивает анализ тональности и извлечение ключевых фраз.

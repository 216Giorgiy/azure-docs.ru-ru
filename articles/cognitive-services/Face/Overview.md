# <a name="what-is-the-azure-face-api"></a>Что такое API распознавания лиц Azure?

API распознавания лиц Azure — это когнитивная служба, предоставляющая алгоритмы для обнаружения, распознавания и анализа человеческих лиц на изображениях. Возможность обработки информации о лицах имеет важное значение в различных сценариях программного обеспечения, таких как обеспечение безопасности, анализ содержимого изображений и управление им, а также использование естественных пользовательских интерфейсов, мобильных приложений и робототехники.

API распознавания лиц предоставляет несколько разных функций, каждая из которых описана в следующих разделах. Ознакомьтесь с ними, чтобы получить дополнительные сведения о каждой из них и определить, подходят ли они для ваших задач.

## <a name="face-detection"></a>Обнаружение лиц

API распознавания лиц может выявлять лица на изображениях и возвращать координаты прямоугольника, в котором они расположены. При необходимости функция обнаружения лиц извлекает ряд атрибутов, связанных с лицом, например поза, пол, возраст, положение головы, волосяной покров лица и очки.

![Изображение женщины и мужчины с прямоугольниками, нарисованными вокруг их лиц, а также сведения о возрасте и поле](./Images/Face.detection.jpg)

Функция обнаружения лиц также доступна в [API компьютерного зрения](https://docs.microsoft.com/azure/cognitive-services/computer-vision/home). Однако если вы хотите выполнять дальнейшие операции с данными о лице, необходимо использовать API распознавания лиц (эта служба). Дополнительные сведения об обнаружении лиц см. в [этой статье](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395236).

## <a name="face-verification"></a>Проверка лиц

API проверки выполняет проверку идентичности двух обнаруженных лиц или одного обнаруженного лица по отношению к одному человеку. Практически оно оценивает, принадлежат ли два лица одному человеку. Это может быть полезно в сценариях безопасности. Дополнительные сведения см. в [этой статье.](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f3039523a)

## <a name="find-similar-faces"></a>Поиск похожих лиц

API поиска похожих лиц сравнивает целевое лицо и набор потенциальных лиц для поиска и находит небольшое количество лиц, очень похожих на целевое. Поддерживаются два режима работы: **matchPerson** и **matchFace**. Режим **matchPerson** возвращает похожие лица после фильтрации для одного пользователя (с помощью [API проверки](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f3039523a)). Режим **matchFace** игнорирует фильтр порогового значения для идентификации одного и того же человека и возвращает список потенциальных лиц, которые могут или не могут принадлежать тому же человеку.

В следующем примере это целевое лицо:

![Улыбающаяся женщина](./Images/FaceFindSimilar.QueryFace.jpg)

А это лица-кандидаты:

![Пять изображений улыбающихся людей. Изображения а) и б) — изображения одного человека](./Images/FaceFindSimilar.Candidates.jpg)

При поиске похожих лиц режим **matchPerson** вернет фотографии (а) и (б), на которых изображен тот же человек, что и на фотографии, использованной в качестве целевого лица. Режим **matchFace** вернет фотографии (a), (б), (в) и (г)&mdash;, т. е. четырех кандидатов, даже если некоторые из них не совпадают с целевым лицом или имеют низкое сходство. Дополнительные сведения см. в статье об [API поиска похожих лиц](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395237).

## <a name="face-grouping"></a>Группировка лиц

API группы делит неизвестные лица на несколько групп, основываясь на сходстве. Каждая группа является несвязанным подмножеством исходного набора лиц. Все лица в группе скорее всего принадлежат одному человеку, однако для одного человека может существовать несколько различных групп (разделенных по другому фактору, например по выражению). Дополнительные сведения см. в [статье об API группы](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395238).

## <a name="person-identification"></a>Идентификация личности

API идентификации может использоваться для идентификации обнаруженных лиц в базе данных. Это может быть полезно для автоматического добавления тегов в программном обеспечении для управления фотографиями. Эту базу данных можно создать заранее и редактировать по мере использования.

На изображении ниже показан пример базы данных с именем myfriends. Каждая группа может содержать до 1 000 000 объектов, соответствующих разным людям, и каждый объект, соответствующий одному человеку, может иметь до 248 зарегистрированных лиц.

![Таблица с 3 столбцами для разных людей, каждый с 3 записями изображений лиц.](./Images/person.group.clare.jpg)

Создав и обучив базу данных, вы можете идентифицировать новое обнаруженное лицо путем сравнения с группой. Если лицо определяется как принадлежащее человеку в группе, то возвращается объект, соответствующий этому человеку.

Дополнительные сведения об идентификации людей см. в [статье об API идентификации](https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395239).

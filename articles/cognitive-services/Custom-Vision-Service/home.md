# <a name="what-is-azure-custom-vision"></a>Пользовательская служба визуального распознавания Azure

API Пользовательской службы визуального распознавания — это служба Cognitive Service, которая позволяет создавать, развертывать и улучшать пользовательские классификаторы изображений. Классификатор изображений — это служба искусственного интеллекта, которая сортирует изображения на классы (теги) в соответствии с определенными характеристиками. В отличие от Службы [компьютерного зрения](https://docs.microsoft.com/azure/cognitive-services/computer-vision/home), Пользовательская служба визуального распознавания позволяет создавать собственные классификации.

## <a name="what-it-does"></a>Действие

Пользовательская служба визуального распознавания использует алгоритм машинного обучения для классификации изображений. Разработчику необходимо отправить группы изображений, которые содержат и не содержат теги для классификации. Во время отправки вы указываете правильные теги для изображений. Затем алгоритм обучается по этим данным и вычисляет собственную точность, проводя тесты на тех же данных. После обучения модели вы можете проверить ее, повторно обучить и, в конечном итоге, использовать для классификации новых изображений в соответствии с потребностями вашего приложения. Вы также можете экспортировать саму модель для автономного использования.

### <a name="classification-and-object-detection"></a>Классификация и обнаружение объектов

Функции Пользовательской службы визуального распознавания можно разделить на две части. **Классификация изображений** назначает распределение классификаций для каждого изображения. **Обнаружение объектов** действует как предыдущая функция, но также возвращает координаты в изображении, где можно найти примененный тег.

### <a name="optimization"></a>Оптимизация

В общем, Пользовательская служба визуального распознавания использует надежные методы распознавания отличий, поэтому вы можете создать прототип на основе небольшого объема данных. Обычно хорошим началом является 50 изображений на тег. Однако, это означает, что служба не подходит для обнаружения незначительных отличий в изображениях (например, мелкие трещины или сколы при контроле качества).

Кроме того, вы можете выбрать из нескольких различных алгоритмов Пользовательской службы визуального распознавания, которые оптимизированы для определенных субъектов&mdash;, например достопримечательностей или магазинов. Дополнительные сведения см. в статье [Как создать классификатор с помощью Пользовательской службы визуального распознавания](getting-started-build-a-classifier.md).

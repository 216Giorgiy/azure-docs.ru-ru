# <a name="web-api-interface"></a>Интерфейс веб-API

Файлы модели, созданной Службой поиска и распознавания данных, можно разместить и использовать с помощью набора веб-API.  Эти интерфейсы API можно разместить на локальном компьютере, используя команду [`host_service`](CommandLine.md#host_service-command), или развернуть в облачной службе Azure, используя команду [`deploy_service`](CommandLine.md#deploy_service-command).  В обоих случаях предоставляются следующие конечные точки API:

* [*interpret*](interpretMethod.md): интерпретирует строки пользовательских запросов, написанных на естественном языке. Возвращает аннотированные интерпретации, что дает возможность использовать автозавершение введенных в поле поиска запросов.
* [*evaluate*](evaluateMethod.md): вычисляет и возвращает результат структурированного выражения запроса.
* [*calchistogram*](calchistogramMethod.md): вычисляет гистограмму значений атрибутов для объектов, возвращаемых структурированным выражением запроса.

Сочетание этих методов API позволяет создать расширенные возможности семантического поиска.  На основании строки запроса на естественном языке метод *interpret* предоставляет аннотированные версии входного запроса со структурированными выражениями запросов, исходя из базовой грамматики и данных индекса.  Метод *evaluate* вычисляет структурированное выражение запроса и возвращает соответствующие объекты индекса для отображения.  Метод *calchistogram* вычисляет распределение значений атрибутов для применения фильтрации и уточнения.

**Пример**

Если в области научных публикаций пользователь вводит строку "latent s", метод *interpret* может предоставить ранжированный набор интерпретаций, предполагая, что пользователь ищет ключевую фразу "latent semantic analysis", научную работу "latent structure analysis" или другие выражения, начинающиеся со строки "latent s".  Эти сведения позволяют быстро предложить пользователю нужные результаты поиска.

Для этой области метод *evaluate* позволяет получить подходящие публикации из академической базы знаний, а метод *calchistogram* позволяет вычислить распределение значений атрибутов для этих публикаций, которое можно использовать для дополнительной фильтрации и уточнения результатов поиска.

Обратите внимание, чтобы для улучшения читаемости вызовы REST API в наших примерах могут содержать знаки (например, пробелы), не представленные в формате URL-адреса. В реальном коде необходимо применять соответствующую URL-кодировку.

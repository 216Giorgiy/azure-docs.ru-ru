# <a name="call-your-custom-search"></a>Обращение к пользовательскому поиску

Прежде чем создать первый вызов к API пользовательского поиска, который вернет результаты поиска для созданного экземпляра, вам необходимо получить ключ подписки Cognitive Services. Чтобы получить ключ для API пользовательского поиска, перейдите на страницу [Пробная версия Cognitive Services](https://azure.microsoft.com/try/cognitive-services/?api=bing-custom-search).


## <a name="try-it-out"></a>Попробуйте сейчас

После настройки интерфейса для пользовательского поиска вы можете проверить его конфигурацию через портал пользовательского поиска. 

1. Войдите в службу [Пользовательский поиск](https://customsearch.ai).
2. Выберите экземпляр службы в списке экземпляров.
3. Перейдите на вкладку **Рабочая среда**. 
4. На вкладке **Конечные точки** выберите конечную точку (например, веб-API). Отображаемые конечные точки зависят от подписки (варианты подписки см. в разделе [расценок](https://azure.microsoft.com/pricing/details/cognitive-services/bing-custom-search/)). 
5. Укажите значения параметров. 

    Ниже приведены возможные параметры, которые можно настроить (фактический список зависит от выбранной конечной точки). Дополнительные сведения об этих параметрах см. в разделе [API пользовательского поиска](https://docs.microsoft.com/rest/api/cognitiveservices/bing-custom-search-api-v7-reference#query-parameters).

    - **Query** (Запрос) — условие, по которому выполняется поиск. Оно доступно только для конечных точек веб-поиска, изображений, видео и автозаполнения.
    - **Custom Configuration ID** (Идентификатор пользовательской конфигурации) — идентификатор конфигурации для выбранного экземпляра пользовательского поиска. Это поле доступно только для чтения.
    - **Market** (Рынок) — обозначает рынок, по которому возвращаются результаты. Оно доступно только для конечных точек веб-поиска, изображений, видео и размещаемых интерфейсов.
    - **Subscription Key** (Ключ подписки) — ключ подписки для тестирования. Вы можете выбрать ключ в раскрывающемся списке или ввести его вручную.  
      
    Щелкнув **Дополнительные параметры**, вы увидите следующие параметры:  
      
    - **Safe Search** (Безопасный поиск) — фильтр, который позволяет убрать из результатов веб-страницы для взрослых. Доступен только для конечных точек веб-поиска, изображений, видео и размещаемых интерфейсов.
    - **User Interface Language** (Язык интерфейса пользователя) — язык, используемый для строк интерфейса пользователя. Например, если включить изображения и видео в размещенном интерфейсе, на вкладках **Изображение** и **Видео** будет использоваться указанный язык.
    - **Count** (Число) — количество возвращаемых в ответе результатов поиска. Доступен только для конечных точек веб-поиска, изображений и видео.
    - **Offset** (Смещение) — количество пропускаемых результатов перед возвращением результатов поиска. Доступен только для конечных точек веб-поиска, изображений и видео.

6. Завершив выбор всех необходимых параметров, щелкните **Call** (Вызов), чтобы просмотреть ответ в формате JSON в области справа. 

Если вы выбрали конечную точку размещенного пользовательского интерфейса, в нижней панели можно проверить работу поиска.

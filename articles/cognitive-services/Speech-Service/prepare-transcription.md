# <a name="transcription-guidelines-for-using-speech-service"></a>Рекомендации по транскрибированию для обучения службы "Речь"

Чтобы настроить **преобразование речи в текст** или **преобразование текста в речь**, необходимо предоставить текст, наряду с речью. Каждая строка текста соответствует одному фрагменту речи. Текст называется *расшифровкой*, и его необходимо создавать в определенном формате.

Служба речи выполняет некоторые нормализации в целях согласования текста. Перед отправкой текста для обучения необходимо выполнить другие задачи нормализации. 

В этой статье описываются оба типа нормализации. Рекомендации немного различаются для разных языков.

## <a name="us-english-en-us"></a>Английский, США (en-US)

Текстовые данные, отправленные в эту службу, должны быть записаны в виде обычного текста только с использованием кодировки ASCII. Каждая строка файла должна содержать текст для одного фрагмента речи.

Очень важно избегать использования расширенной кодировки (Latin-1) или знаков препинания Юникода. Эти символы могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных из веб-страниц. Замените эти символы соответствующими символами ASCII. Например: 

| Нерекомендуемые символы | Замена |
|----- | ----- |
| “Hello world” (открывающие и закрывающие двойные кавычки) | "Hello world" (двойные кавычки) |
| John’s day (правая одинарная кавычка) | John's day (апостроф) |
| it was good—no, it was great! (длинное тире) | it was good--no, it was great! (дефисы) |

### <a name="text-normalization-performed-by-the-service"></a>Нормализация текста, выполняемая службой

Служба речи выполняет следующую нормализацию текста для расшифровок текста:

*   перевод всего текста в нижний регистр;
*   удаление всех знаков пунктуации, кроме апострофов внутри слов;
*   расширение чисел в речевую форму, включая денежные суммы.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| Starbucks Coffee | starbucks coffee |
| "Holy cow!" said Batman. | holy cow said batman |
| "What?" said Batman's sidekick, Robin. | what said batman's sidekick robin |
| Go get - em! | go get em |
| I'm double-jointed | i'm double jointed |
| 104 Main Street | one oh four main street |
| Tune to 102.7 | tune to one oh two point seven |
| Pi is about 3.14 | pi is about three point one four |
| It costs $3.14 | it costs three fourteen |

### <a name="text-normalization-you-must-perform"></a>Нормализация текста, выполняемая пользователем

Примените следующую нормализацию к расшифровке текста.

*   Сокращения должны записываться словами, которые отражают речевую форму
*   Нестандартные числовые строки (например, некоторые даты или бухгалтерские формы) должны быть написаны словами
*   Слова с неалфавитными символами или смешанными буквенно-цифровыми символами должны записываться так, как произносятся
*   Сокращения, которые произносятся как слова, остаются без изменений. Например, radar, laser, RAM, NATO и Mr.
*   Сокращения, которые произносятся как отдельные буквы, записываются буквами, разделенными пробелами. Например, IBM, CPU, FBI, TBD, NaN. 

Ниже приведены некоторые примеры:

| Исходный текст | После нормализации |
|----- | ----- |
| 14 NE 3rd Dr. | fourteen northeast third drive |
| Dr. Strangelove | Doctor Strangelove |
| James Bond 007 | James Bond double oh seven |
| Ke$ha | Kesha |
| How long is the 2x4 | How long is the two by four |
| The meeting goes from 1-3pm | The meeting goes from one to three pm |
| my blood type is O+ | My blood type is O positive |
| water is H20 | water is H 2 O |
| play OU812 by Van Halen | play O U 8 1 2 by Van Halen |
| UTF-8 with BOM | U T F 8 with BOM |

## <a name="chinese-zh-cn"></a>Китайский (zh-CN)

Для текстовых данных, отправляемых в пользовательскую службу речи, следует использовать кодировку UTF-8 с метками порядка следования байтов. Каждая строка файла должна содержать текст для одного фрагмента речи.

Очень важно избегать использования знаков препинания половинной ширины. Эти символы могут быть включены случайно при подготовке данных в текстовом редакторе или импорте данных из веб-страниц. Замените их соответствующими знаками полной ширины. Например: 

| Нерекомендуемые символы | Замена |
|----- | ----- |
| "你好" (open and close double quotes) | "你好" (double quotes) |
| 需要什么帮助? (вопросительный знак) | 需要什么帮助？ |

### <a name="text-normalization-performed-by-the-service"></a>Нормализация текста, выполняемая службой

Служба речи выполняет следующую нормализацию текста для расшифровок текста:

*   удаление всех знаков препинания;
*   расширение чисел до речевой формы;
*   преобразование полноширинных букв в полуширинные буквы;
*   перевод всех слов на английском языке в верхний регистр.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| 3.1415 | 三 点 一 四 一 五 |
| ￥3.5 | 三 元 五 角 |
| w f y z | W F Y Z |
| 1992年8月8日 | 一 九 九 二 年 八 月 八 日 |
| 你吃饭了吗 ? | 你 吃饭 了 吗 |
| 下午5:00的航班 | 下午 五点 的 航班 |
| 我今年21岁 | 我 今年 二十 一 岁 |

### <a name="text-normalization-you-must-perform"></a>Нормализация текста, выполняемая пользователем

Примените следующую нормализацию к тексту перед импортом.

*   Сокращения должны записываться словами, которые отражают речевую форму
*   Эта служба не охватывает все числовые величины. Надежнее будет записывать числовые строки в речевой форме.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| 我今年21 | 我今年二十一 |
| 3号楼504 | 三号 楼 五 零 四 |

## <a name="other-languages"></a>Остальные языки

Для текстовых данных, отправляемых в службу **преобразования речи в текст**, следует использовать кодировку UTF-8 с метками порядка следования байтов. Каждая строка файла должна содержать текст для одного фрагмента речи.

> [!NOTE]
> В этих примерах используется немецкий язык. Однако эти правила применимы для всех языков, кроме английского или китайского.

### <a name="text-normalization-performed-by-the-service"></a>Нормализация текста, выполняемая службой

Служба речи выполняет следующую нормализацию текста для расшифровок текста:

*   перевод всего текста в нижний регистр;
*   удаление всех знаков препинания, включая различные виды кавычек ("test", 'test', "test„ или «test»);
*   исключение всех строк, содержащих специальные символы из кодировки ^ ¢? ¤ §... ¥ © ª ¬® ° ± ² µ × ÿ Ø¬¬;
*   расширение чисел в речевую форму, включая денежные суммы;
*   надстрочные знаки допустимы только для a, o, u; другие будут заменены "th" или исключены.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации |
|----- | ----- |
| Frankfurter Ring | frankfurter ring |
| "Hallo, Mama!" sagt die Tochter. | hallo mama sagt die tochter |
| ¡Eine Frage! | eine frage |
| wir, haben | wir haben |
| Das macht $10 | das macht zehn dollars |

### <a name="text-normalization-you-must-perform"></a>Нормализация текста, выполняемая пользователем

Примените следующую нормализацию к тексту перед импортом.

*   Десятичным разделителем должна быть запятая (","), а не точка ("."): 2,3%, а не 2.3%
*   Разделителем между часами и минутами должно быть двоеточие (":"), а не точка ("."): 12:00 Uhr.
*   Такие сокращения, как "ca.", "bzw." не заменяются. Мы рекомендуем использовать полную форму.
*   Пять основных математических операторов удаляются: +, -, \*, /. Мы рекомендуем заменить их речевыми формами: plus, minus, mal, geteilt.
*   То же относится к операторам сравнения (=, <, >,) — gleich, kleiner als, grösser als.
*   Используйте дроби, например 3/4, в словесной форме (например, drei viertel вместо ¾).
*   Замените символ € словом Euro.

Ниже приведены некоторые примеры.

| Исходный текст | После нормализации пользователя | После нормализации системы
|--------  | ----- | -------- |
| Es ist 12.23Uhr | Es ist 12:23Uhr | es ist zwölf uhr drei und zwanzig uhr |
| {12.45} | {12,45} | zwölf komma vier fünf |
| 3 < 5 | 3 kleiner als 5 | drei kleiner als vier |
| 2 + 3 - 4 | 2 plus 3 minus 4 | zwei plus drei minus vier|
| Das macht 12€ | Das macht 12 Euros | das macht zwölf euros |

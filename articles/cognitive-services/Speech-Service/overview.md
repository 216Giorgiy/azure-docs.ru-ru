# <a name="what-is-the-speech-service"></a>Что собой представляет служба "Речь"


Аналогично другим службам распознавания речи Azure служба "Речь" основана на технологиях распознавания речи, используемых в таких продуктах, как Кортана и Microsoft Office.

Служба "Речь" объединяет голосовые функции Azure, ранее доступные через [API распознавания речи Bing](https://docs.microsoft.com/azure/cognitive-services/speech/home), [API перевода речи](https://docs.microsoft.com/azure/cognitive-services/translator-speech/), [Пользовательскую службу распознавания речи](https://docs.microsoft.com/azure/cognitive-services/custom-speech-service/cognitive-services-custom-speech-home) и [службу пользовательских голосовых моделей](http://customvoice.ai/). Теперь одна подписка предоставляет доступ ко всем этим возможностям.

## <a name="main-speech-service-functions"></a>Основные функции службы "Речь"

К основным функциям службы "Речь" относятся преобразование речи в текст (также называемое распознаванием или транскрибированием), преобразование текста в речь (синтез речи) и перевод речи.

|Функция|Функции|
|-|-|
|[Преобразование речи в текст](speech-to-text.md)| <ul><li>Расшифровывает непрерывную речь в режиме реального времени в текстовый формат.<li>Может выполнять пакетное транскрибирование аудиозаписей. <li>Поддерживает промежуточные результаты, обнаружение окончания речи, автоматическое форматирование текста и маскировку нецензурной лексики. <li>Можно вызвать в [Интеллектуальной службе распознавания речи](https://docs.microsoft.com/azure/cognitive-services/luis/) (LUIS), чтобы получить намерение пользователя из расшифрованной речи.\*|
|[Преобразование текста в речь](text-to-speech.md)| <ul><li>Преобразует текст в естественно звучащую речь. <li>Предлагает голоса обеих полов и диалекты для множества поддерживаемых языков. <li>Поддерживает ввод простого текста или на языке разметки синтеза речи (Speech Synthesis Markup Language, SSML). |
|[Перевод речи](speech-translation.md)| <ul><li>Преобразует потоковое аудио в режиме, близком к реальному времени.<li> Может также обрабатывать записанную речь.<li>Предоставляет результаты в виде текста или синтезированной речи. |

\* *Для распознавания намерений требуется подписка LUIS.*

## <a name="customize-speech-features"></a>Настройка функций обработки речи

С помощью собственных данных вы можете обучать модели, используемые такими функциями службы "Речь", как преобразование речи в текст и текста в речь.

|Функция|Модель|Назначение|
|-|-|-|
|Преобразование речи в текст|[Акустическая модель](how-to-customize-acoustic-models.md)|Позволяет транскрибировать речь определенных говорящих и распознать речь в таких средах, как автомобили или фабрики.|
||[Языковая модель](how-to-customize-language-model.md)|Позволяет транскрибировать речь с профильными терминами и грамматикой, например медицинский или ИТ-жаргон.|
||[Модель произношения](how-to-customize-pronunciation.md)|Позволяет транскрибировать сокращения и акронимы, например "IOU" для "I owe you". |
|Преобразование текста в речь|[Настраиваемый голос](how-to-customize-voice-font.md)|Предоставляет приложению собственный голос, обучая модель на примерах человеческой речи.|

Пользовательские модели можно использовать везде, где в функциях преобразования речи в текст и текста в речь применялись стандартные модели.

## <a name="use-the-speech-service"></a>Использование службы "Речь"

Чтобы упростить разработку приложений с поддержкой речевых функций, корпорация Майкрософт предоставляет пакет SDK [службы "Речь"](speech-sdk.md). Пакет SDK для службы "Речь" предоставляет согласованные собственные API для преобразования текста в речь и речи в текст для C#, C++ и Java. Если вы разрабатываете с помощью одного из этих языков, пакет SDK для службы "Речь" упрощает разработку, обрабатывая сетевые сведения.

Служба "Речь" также имеет [REST API](rest-apis.md), который работает с любым языком, поддерживающим HTTP-запросы. Интерфейс REST не предоставляет функцию потоковой передачи в режиме реального времени для пакета SDK.

|<br>Метод|Речь<br>в текст|Текст в<br>Речь|Речь<br>Перевод|<br>ОПИСАНИЕ|
|-|-|-|-|-|
|[пакет SDK для службы "Речь"](speech-sdk.md);|Yes|Нет |Yes|Собственные API-интерфейсы для C#, C++ и Java для упрощения разработки.|
|[REST](rest-apis.md)|Yes|Да|Нет |Простой API на основе HTTP, который упрощает добавление речи в приложения.|

### <a name="websockets"></a>Подключения WebSocket

В службе "Речь" также есть протоколы WebSocket для потоковой передачи преобразования речи в текст и перевода речи. Пакеты SDK для службы "Речь" используют эти протоколы для обмена данными со службой. Используйте пакет SDK для службы "Речь", вместо того чтобы пытаться реализовать собственный обмен данными со службой по протоколу WebSocket.

При наличии кода, в котором запрограммировано использование API распознавания речи Bing или API перевода речи через подключение WebSocket, вы можете обновить его для использования службы "Речь". Протоколы WebSocket совместимы. Отличаются только конечные точки.

### <a name="speech-devices-sdk"></a>Пакет SDK для речевых устройств

[Пакет SDK для речевых устройств](speech-devices-sdk.md) — это интегрированная программная и аппаратная платформа для разработчиков устройств с поддержкой речи. Наш поставщик оборудования предоставляет примеры структуры и единицы разработки. Корпорация Майкрософт предоставляет оптимизированный под устройство пакет SDK, который использует все преимущества аппаратных возможностей.

## <a name="why-move-to-the-speech-service"></a>В чем преимущество использования службы "Речь"?

Служба "Речь" предоставляет все функциональные возможности и большинство функций API распознавания речи Bing и трех других служб речи Azure: Пользовательской службы распознавания речи, Пользовательской голосовой модели и Перевода речи. Мы рекомендуем пользователям этих служб начать использовать службу "Речь".

Служба "Речь" включает в себя множество обновлений для этих служб, в том числе:

* Более высокая точность распознавания речи. Мы регулярно совершенствуем модели, используемые в службе.

* Большая масштабируемость. Служба оптимизирована для обработки нескольких одновременных запросов, что уменьшает задержки.

* Служба "Речь" использует ценовую модель на основе времени. Дополнительные сведения см. [на странице цен службы "Речь"](https://azure.microsoft.com/pricing/details/cognitive-services/speech-services/).

* Служба "Речь" доступна в [нескольких регионах](regions.md) в соответствии с потребностями клиентов по всему миру. Для каждого региона, используемого приложением, вам потребуется [подписка Azure](https://docs.microsoft.com/azure/cognitive-services/welcome).

* Один ключ подписки службы "Речь" предоставляет доступ к следующим функциям. Каждый тарифицируется отдельно, поэтому плата взимается только за функции, которые вы используете.

    * [Преобразование речи в текст](speech-to-text.md)
    * [Настраиваемое преобразование речи в текст](https://cris.ai/CustomSpeech)
    * [Преобразование текста в речь](text-to-speech.md)
    * [Настраиваемое преобразование текста в речь](https://cris.ai/CustomVoice)
    * [Перевод речи](speech-translation.md) (не включает [перевод текста](https://docs.microsoft.com/azure/cognitive-services/translator/translator-info-overview))

* Функция преобразования речи в текст службы "Речь" интегрируется со [службой API распознавания речи](https://docs.microsoft.com/azure/cognitive-services/luis/) (LUIS) для распознавания намерения говорящего. Ключ конечной точки LUIS можно также использовать с помощью службы "Речь". Чтобы получить подробные сведения, ознакомьтесь со статьей [Распознавание намерений в речи с помощью пакета SDK службы "Речь" для C#](how-to-recognize-intents-from-speech-csharp.md).

* Для преобразования речи в текст больше не требуется указывать режим распознавания.

* Служба "Речь" поддерживает голоса с частотой дискретизации 24 кГц для преобразования текста в речь, повышая качество звука. На момент написания этой статьи есть две таких голосовых модели (только английский США): `Jessa24kRUS` и `Guy24kRUS`.

* [Пакетное транскрибирование](batch-transcription.md) службы "Речь" позволяет записывать большие объемы речи, например записи в колл-центрах, для эффективного преобразования в текст, быстрого поиска и анализа.

* При использовании пакета SDK для службы "Речь" потоковая передача транскрипции речи в текст не ограничена по времени.

* [Пакет SDK для службы "Речь"](speech-sdk.md) обеспечивает согласованный API для службы "Речь" на нескольких языках программирования и в нескольких средах выполнения (включая Windows 10, UWP и .NET Core), что упрощает разработку, особенно на нескольких платформах.

* Служба "Речь" совместима с REST API и протоколом WebSocket, используемыми другими службами речи Azure, что упрощает миграцию имеющихся клиентских приложений в службу "Речь".

## <a name="speech-scenarios"></a>Сценарии службы "Речь"

Варианты использования службы "Речь" включают:

> [!div class="checklist"]
> * Создание активируемых голосом приложений
> * Транскрибирование записей звонков в колл-центр
> * Реализация голосовых ботов

### <a name="voice-user-interface"></a>Голосовой пользовательский интерфейс

Голосовой ввод — это отличный способ сделать приложение гибким, автоматизированным и легким в использовании. В приложении с голосовым управлением пользователи могут просто запрашивать необходимую информацию.

Если приложение предназначено для общего доступа, можно использовать модели распознавания голоса по умолчанию. Они распознают голоса широкого спектра говорящих в общих средах.

Если приложение используется в определенной предметной области, к примеру, медицине и ИТ, можно создать [языковую модель](how-to-customize-language-model.md). С помощью этой модели можно обучать службу "Речь" специальной терминологии, используемой приложением.

Если приложение используется в шумной среде, например на фабрике, можно создать пользовательскую [акустическую модель](how-to-customize-acoustic-models.md). С помощью этой модели служба "Речь" сможет распознать речь среди шума.

Начать работу очень просто. Просто загрузите [пакет SDK службы "Речь"](speech-sdk.md) и выполните инструкции из статьи [Краткое руководство. Распознавание речи в приложении C# для .NET Framework в Windows с помощью пакета SDK для службы "Речь"](quickstart-csharp-dotnet-windows.md).

### <a name="call-center-transcription"></a>Транскрибирование звонков в колл-центр

Зачастую к записям звонков в колл-центр обращаются только при возникновении проблемы со звонком. С помощью службы "Речь" можно легко преобразовать любую запись в текст. Вы можете легко проиндексировать текст для [полнотекстового поиска](https://docs.microsoft.com/azure/search/search-what-is-azure-search) или применить [анализ текста](https://docs.microsoft.com/azure/cognitive-services/Text-Analytics/), чтобы обнаружить тональность, язык и ключевые фразы.

Если в записях звонков в колл-центр наблюдается специализированная терминология (такая, как имена продуктов или ИТ-жаргон), можно создать [языковую модель](how-to-customize-language-model.md), чтобы научить службу "Речь" этому словарю. Настраиваемая [звуковая модель](how-to-customize-acoustic-models.md) может помочь службе "Речь" понять неоптимальные телефонные соединения.

Чтобы получить дополнительные сведения об этом сценарии, узнайте больше о [пакетном транскрибировании](batch-transcription.md) с помощью службы "Речь".

### <a name="voice-bots"></a>Голосовые боты

[Чат-боты](https://dev.botframework.com/) — это популярный способ предоставления пользователям необходимой информации и помощи клиентам в подборе нужных продуктов и услуг. При добавлении пользовательского интерфейса для общения на свой веб-сайт или в приложение упрощается поиск и ускоряется доступ к функциям. С помощью службы "Речь" этот диалог приобретает новую степень глубины благодаря ответу на разговорные запросы с использованием аналогичной терминологии.

Чтобы сделать голос своего бота уникальным, ему можно дать собственный голос. Создание пользовательского голоса — это двухэтапный процесс. Сначала [запишите](record-custom-voice-samples.md) голос, который хотите использовать. Затем [отправьте эти записи](how-to-customize-voice-font.md) вместе с транскрипцией на [портал настройки голоса](https://cris.ai/Home/CustomVoice) службы "Речь", который делает все остальное. После создания пользовательского голоса нужно выполнить несколько простых действий для использования его в приложении.

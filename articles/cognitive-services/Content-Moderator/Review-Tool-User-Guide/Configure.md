# <a name="about-review-tool-settings"></a>О параметрах инструмента проверки #

С помощью вкладки "Settings" (Параметры) панели мониторинга инструмента проверки можно легко определить и изменить многие компоненты.

![Параметры проверки Content Moderator](images/settings-1.png)

## <a name="team-and-subteams"></a>Команды и подкоманды ## 

На этой вкладке можно управлять своей командой и подкомандами. У вас может быть только одна команда, но вы можете [создать несколько подкоманд](subteams.md) и отправить приглашения будущим участникам. После отправки приглашений вы можете отслеживать их, изменять разрешения для участников команды и приглашать дополнительных пользователей. После того, как участники команды примут приглашение, вы сможете назначить их в разные подкоманды. Можно назначить участникам команды роли администраторов или рецензентов. Администраторы могут приглашать других пользователей, а рецензенты — нет.

![Параметры команды Content Moderator](images/settings-2-team.png)

## <a name="tags"></a>Теги ##

Здесь вы можете [определить настраиваемые теги](tags.md), введя короткий код, имя и описание тегов. После создания теги будут доступны во время проверок. Разные теги можно использовать для разных проверок. Вы можете включить или отключить их видимость.

![Параметры тегов Content Moderator](images/settings-3-tags.png)

## <a name="connectors"></a>Соединители ##

Рабочие процессы добавляют функциональные возможности с помощью соединителей, используемых для обмена данными с инструментом проверки. Инструмент проверки вызывает интерфейсы API Content Moderator с рабочим процессом по умолчанию для модерации содержимого. При регистрации в инструменте проверки он автоматически подготавливает для вас учетные данные API Content Moderator. Он также поддерживает интеграцию интерфейсов API других соединителей, если такие соединители доступны. Мы добавили несколько стандартных соединителей.

Для управления соединителями используется вкладка [Connectors](connectors.md) (Соединители). Можно добавить или удалить соединители, а также найти ключ подписки для конкретного соединителя. Щелкните "Connect" (Подключиться), чтобы добавить их в пользовательские рабочие процессы. 

![Параметры соединители Content Moderator](images/settings-4-connectors.png)

## <a name="workflows"></a>Рабочие процессы ##

Для управления рабочими процессами используется вкладка "Workflows" (Рабочие процессы). Рабочие процессы можно проверить, передав пример файла. Можно также [определить пользовательские рабочие процессы](workflows.md) для изображения и текста с помощью доступных соединителей API (показанных на вкладке "Connectors" (Соединители)). 

![Параметры рабочего процесса Content Moderator](images/settings-5-workflows.png)

## <a name="credentials"></a>Учетные данные ##

Эта вкладка предоставляет быстрый доступ к ключу подписки Content Moderator, который понадобится для использования интерфейсов API в Content Moderator (API модерации изображений, API модерации текста, API управления списками, API рабочих процессов и API проверки).
 
![Учетные данные Content Moderator](images/settings-6-credentials.png)

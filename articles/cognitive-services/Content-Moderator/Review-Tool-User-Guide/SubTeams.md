# <a name="manage-review-teams-and-subteams"></a>Управление командами и подкомандами проверки

Перед использованием Content Moderator необходимо создать команду. При регистрации вы вводите имя команды, которая будет использоваться по умолчанию. В инструменте проверки можно использовать только одну команду. Тем не менее можно создать несколько подкоманд. Подкоманды удобно использовать для создания команд эскалации или отдельных команд для проверки содержимого определенных категорий. Например, можно отправлять содержимое для взрослых в другую команду для дополнительной проверки.

В этом разделе описывается, как создавать подкоманды и быстро назначать проверки в режиме реального времени. Тем не менее, можно использовать вкладку [Workflows](workflows.md) (Рабочие процессы) для назначения проверок на основе заданных критериев.

## <a name="go-to-the-teams-setting"></a>Переход к параметрам команды

Чтобы приступить к созданию подгруппы, выберите **Teams** (Команды) в меню параметров.

![Параметры команды](images/0-teams-1.png)

## <a name="create-subteams"></a>Создание подкоманд

Команда по умолчанию содержит всех потенциальных рецензентов. Подкоманды являются подмножествами команды по умолчанию. Невозможно назначить пользователя в подкоманду, если он не является участником команды по умолчанию. Поэтому сейчас нужно добавить рецензентов в команду по умолчанию Нажмите кнопку "Invite" (Пригласить) на странице "Team" (Команда).

![Приглашение пользователей](images/invite-users.png)

### <a name="create-a-subteam"></a>Создание подкоманды
Прокрутите страницу "Team" (Команда) вниз до раздела "Subteam" (Подкоманда) Нажмите кнопку "Add Subteam" (Добавить подкоманду). 

![Добавление подкоманды](images/1-teams-1.png)

### <a name="name-your-subteam"></a>Назначение имени подкоманде
Введите имя подкоманды в диалоговом окне, а затем нажмите кнопку "Save" (Сохранить).

![Имя подкоманды](images/1-Teams-2.PNG)

### <a name="assign-members-from-your-default-team"></a>Назначьте участников команды по умолчанию.
Нажмите кнопку "Add Member" (Добавить участника), чтобы назначить участника команды по умолчанию в одну или несколько подкоманд. В подкоманду можно добавить только существующих пользователей. Чтобы добавить новых пользователей, не зарегистрированных в инструменте проверки, пригласите их с помощью кнопки "Invite" (Пригласить) на странице "Team Setting" (Параметры команды).

![Назначение участников подкоманды](images/1-Teams-3.PNG)

## <a name="assign-reviews-to-your-subteams"></a>Назначение проверок подкомандам

Создав подкоманды и назначив участников команды, вы можете начать назначение проверок изображений и текста этим подкомандам. Для этого используется окно "Review" (Проверка).
Если вы хотите назначить подкоманде отдельное изображение, щелкните многоточие в правом верхнем углу этого изображения, выберите "Move to" (Переместить) и выберите подкоманду.

![Назначение подкоманде проверки изображений](images/3-review-image-subteam-1.png)

## <a name="switch-between-subteams-to-review-assigned-content"></a>Переключение между подкомандами для проверки назначенного содержимого

Если вы являетесь участником одной или нескольких подкоманд, то вы можете переключаться между ними на панели мониторинга инструментов проверки. Чтобы просмотреть все текущие ожидающие проверки, назначенные подкоманде, на вкладке "Image" (Изображение) щелкните "Choose Subteam" (Выбрать подкоманду).

![Переключение между подкомандами](images/3-review-image-subteam-2.png)

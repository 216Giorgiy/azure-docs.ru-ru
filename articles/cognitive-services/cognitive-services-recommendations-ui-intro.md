# <a name="building-a-model-with-the-recommendations-ui"></a>Создание модели с помощью пользовательского интерфейса рекомендаций
Этот документ содержит пошаговое руководство. Мы покажем, какие шаги нужно выполнить, чтобы обучить модель с помощью [пользовательского интерфейса рекомендаций](https://recommendations-portal.azurewebsites.net/).
Это упражнение позволяет ознакомиться с процессом создания модели до его выполнения программным способом. Вы также ознакомитесь с удобным пользовательским интерфейсом для начала работы со службой.

Это упражнение занимает около 30 минут.

<a name="Step1"></a>

## <a name="step-1---sign-in-to-the-recommendations-ui"></a>Шаг 1. Вход в пользовательский интерфейс рекомендаций
1. [Зарегистрируйтесь](https://portal.azure.com/#create/Microsoft.CognitiveServices/apitype/Recommendations/pricingtier/S1) для получения новой подписки [API рекомендаций](https://www.microsoft.com/cognitive-services/en-us/recommendations-api), если вы еще этого не сделали. Чтобы зарегистрироваться в службе, перейдите на **портал Azure** по адресу [http://portal.azure.com/](https://portal.azure.com/#create/Microsoft.CognitiveServices/apitype/Recommendations/pricingtier/S1) и выполните вход со своей учетной записью Azure. Подробные инструкции по процессу регистрации приведены в разделе *Задача 1* [руководства по началу работы](cognitive-services-recommendations-quick-start.md). 
2. Получив **ключ** для подписки API рекомендаций, перейдите к [пользовательскому интерфейсу рекомендаций](https://recommendations-portal.azurewebsites.net/). 
3. Войдите на портал, введя ключ в поле **Ключ учетной записи**, а затем нажмите кнопку **Вход**.
   
    ![Пользовательский интерфейс рекомендаций. Диалоговое окно входа.][reco_signin]

<a name="Step2"></a>

## <a name="step-2---lets-gather-some-training-data"></a>Шаг 2. Сбор данных для обучения
Перед созданием сборки подсистеме требуется два типа сведений — файл каталога и набор файлов использования. 

Файл каталога содержит информацию об элементах, которые вы предлагаете клиентам. Файл об использовании содержит информацию о том, как используются эти элементы, или транзакции вашего предприятия.

Обычно эти сведения запрашиваются из базы данных хранилища. Мы предоставили некоторые примеры данных, которые помогут узнать, как использовать API рекомендаций.

Данные можно скачать по адресу [http://aka.ms/RecoSampleData](http://aka.ms/RecoSampleData). Скопируйте и распакуйте файл **Books.Zip** в папку на локальном компьютере. Например, **C:\data**.

Подробные сведения о схеме каталога и файлах использования см. в статье [Сбор данных для обучения модели](cognitive-services-recommendations-collecting-data.md).

В этом упражнении мы будем работать с небольшим файлом, чтобы ожидание обучения не длилось слишком долго. Если вы хотите использовать более реалистичный файл, мы также предоставили **MsStoreData.zip**, содержащий пример транзакций из Microsoft Store, в том же [расположении](http://aka.ms/RecoSampleData).

<a name="Step3"></a>

## <a name="step-3---create-a-project-and-upload-catalog-and-usage-data"></a>Шаг 3. Создание проекта и отправка данных каталога и данных об использовании
После входа в [пользовательский интерфейс рекомендаций](https://recommendations-portal.azurewebsites.net/) отобразится страница "Проекты". Все раннее созданные проекты отобразятся здесь.
Проект (также известный как *модель* в справочнике по API) представляет собой контейнер для данных каталога и данных об использовании. В проекте можно создать несколько *сборок*. Следующие действия помогут вам выполнить этот процесс.

1. Чтобы создать проект, введите имя в текстовом поле (подойдет нечто вроде MyFirstModel) и нажмите кнопку **Добавить проект**.
   
    ![Пользовательский интерфейс рекомендаций. Страница "Проекты".][reco_projects]
2. Создав проект в разделе **Add a Catalog File** (Добавьте файл каталога), нажмите кнопку **Поиск файла**. 
   Отправьте каталог, полученный на шаге 2. Если вы сохранили его в папке *C:\data*, перейдите в нее.
   
     ![Пользовательский интерфейс рекомендаций. Добавление данных в проект.][reco_firstmodel]
3. После отправки каталога в разделе **Add Usage Files** (Добавьте файлы использования) нажмите кнопку **Поиск файла**. Добавьте файл usage_large.txt.

> **Что делать, если файл данных об использовании большой?**
> 
> Можно отправить только файлы использования размером не более 200 МБ. С другой стороны, система может хранить до 2 ГБ данных транзакций, поэтому при необходимости можно отправить несколько файлов.
> Для создания эффективной модели может потребоваться гораздо меньше данных. Важен не только объем данных, но и их качество. Чаще всего в данных об использовании представлены транзакции самых популярных элементов и совсем немного транзакций других элементов.
> 
> 

<a name="Step4"></a>

## <a name="step-4---lets-do-some-training"></a>Шаг 4. Приступая к обучению
Отправив данные каталога и данные об использовании, мы готовы к обучению подсистемы с помощью шаблонов данных.

1. Нажмите кнопку **New Build** (Новая сборка).
2. Выберите **Рекомендации** в качестве типа сборки. Обратите внимание, что также обеспечивается поддержка таких типов сборки, как сборка ранжирования и сборка FBT ("Часто покупают вместе").
   
   ![Пользовательский интерфейс рекомендаций. Диалоговое окно сборки.][reco_build_dialog.png]

    Сборка FBT позволяет выявить закономерности для продуктов, которые обычно приобретаются или используются в пределах одной транзакции.
    Сборка ранжирования позволяет определить возможности, которые вас интересуют. 
    Мы не будем подробно рассматривать сборки в этом семинаре. Однако если вы хотите узнать о них, см. [страницу с документацией по типам сборок и качеству модели](cognitive-services-recommendations-buildtypes.md).

1. Нажмите кнопку **Сборка**. При использовании данных из файла Books процесс сборки занимает около пяти минут. При использовании более крупных наборов данных процесс выполняется дольше.

<a name="Step5"></a>

## <a name="step-5---lets-find-out-what-the-machine-learned"></a>Шаг 5. Результаты обучения компьютера
После того, как сборка завершена, она появится в списке сборок. К ней можно отправлять запросы, чтобы получить рекомендации для элементов и пользователей.

1. После завершения сборки нажмите кнопку **Оценка**. Таким образом вы сможете настроить модель и просмотреть рекомендованные элементы.
   
    ![Пользовательский интерфейс рекомендаций. Кнопка "Оценка".][reco_score_button]
2. Выберите элемент, чтобы увидеть, какие элементы возвращаются в качестве рекомендаций для него. Обратите внимание, что если транзакций для прогнозирования этих рекомендаций недостаточно, система не возвратит рекомендации для выбранного элемента.  Если по какой-то причине у вас есть элемент, для которого не возвращаются рекомендации, попробуйте оценить другие элементы.

<a name="Step6"></a>

## <a name="step-6---next-steps"></a>Шаг 6. Дальнейшие действия
Поздравляем! Вы выполнили обучение модели и получили рекомендации на ее основе.  Пользовательский интерфейс рекомендаций — это полезный инструмент, позволяющий просматривать состояние проектов и сборок. 

Теперь, когда у вас есть модель, вы можете узнать, как выполнить все описанные выше действия программным способом. Чтобы узнать, как вызывать API программным способом, ознакомьтесь со [справочником по API рекомендаций](http://go.microsoft.com/fwlink/?LinkId=759348) и скачайте [пример приложения рекомендаций](http://go.microsoft.com/fwlink/?LinkID=759344).

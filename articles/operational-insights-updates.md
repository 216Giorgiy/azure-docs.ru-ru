# Установка системных обновлений на серверах

Пакет аналитики "Системные обновления" в Microsoft Azure Operational Insights позволяет применять пропущенные обновления к серверам в инфраструктуре. Пакет аналитики устанавливается для обновления агента Operations Manager и модуля базовой конфигурации для службы оперативной аналитики. Сведения об обновлениях считываются в отслеживаемых службах, а затем данные обновлений отправляются в службу оперативной аналитики в облако для обработки. К данным обновлений применяется логика, и облачная служба записывает данные. Отсутствующие обновления отображаются на панели мониторинга **Обновления**. На панели мониторинга **Обновления** можно работать с отсутствующими обновлениями и разработать план, применимый ко всем серверам, которые нужны для них.

## Установка системных обновлений на серверах

Перед тем, как вы сможете установить системные обновления в службе оперативной аналитики Microsoft Azure, необходимо установить пакет аналитики. Дополнительные сведения об установке пакетов аналитики см. в разделе [Добавление и удаление пакетов аналитики с помощью коллекции](operational-insights-add-intelligence-packs.md). После установки можно просмотреть обновления, отсутствующие на отслеживаемых серверах, с помощью плитки **Оценка системных обновлений** на странице **Обзор** в службе оперативной аналитики. 

![изображение плитки "Оценка системных обновлений"](./media/operational-insights-updates/overview-update.png)

Плитка откроет панель мониторинга **Обновления**, на которой можно просмотреть общую сводку отсутствующих обновлений. На странице приводятся следующие категории:

- серверы, на которых отсутствуют обновления безопасности;

- серверы, которые давно не обновлялись;

- обновления, которые должны применяться к определенным серверам;

- тип отсутствующих обновлений.

Чтобы получить дополнительные сведения об отсутствующем обновлении, можно щелкнуть любую плитку или элемент и просмотреть его сведения на странице **Поиск**. 

![изображение панели мониторинга "Обновления"](./media/operational-insights-updates/gallery-sysupdate-01.png)

![изображение панели мониторинга "Обновления"](./media/operational-insights-updates/gallery-sysupdate-02.png)

## Результаты поиска##
В результаты поиска обновлений входит следующее:

- сервер;

- плитка "Обновления";

- идентификатор базы знаний;

- продукт, для которого предназначено обновление;

- важность обновления;

- дата публикации.

В результаты поиска по слову *Сервер* входит следующее:

- имя сервера;

- версия операционной системы;

- метод включения автоматических обновлений;

- число дней с момента последнего обновления;

- версия агента обновления Windows.

## Работа с обновлениями

1. На странице **Обзор** щелкните плитку **Оценка системных обновлений**.

2. На панели мониторинга **Обновления** просмотрите категории обновлений и выберите одну из них.

3. Щелкните плитку или любой элемент, чтобы просмотреть подробную информацию о нем на странице **Поиск**.

4. Используя найденную информацию, можно создать план для установки отсутствующих обновлений.

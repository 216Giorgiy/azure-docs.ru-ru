# Приступая к работе с предварительно настроенными решениями IoT

## Введение

Предварительно настроенные решения набора IoT Azure включают ряд служб IoT Azure для иллюстрации полного решения, удовлетворяющего бизнес-сценарию Интернета вещей.

В этом учебнике показано, как подготовить предварительно настроенное решение удаленного мониторинга. В нем также показано, как просмотреть базовые компоненты этого решения.

Для работы с этим учебником вам потребуется следующее.

-   Активная подписка Azure.

    Если ее нет, можно создать бесплатную пробную учетную запись всего за несколько минут. Дополнительные сведения см. в разделе [Бесплатная пробная версия Azure][].

## Подготовка предварительно настроенного решения удаленного мониторинга

1.  Войти в <https://azureiotsuite.com> и нажмите кнопку **+** для создания нового решения.

2.  Выберите тип решения **Удаленный мониторинг**.

3.  Введите **Имя решения** для предварительно настроенного решения удаленного мониторинга.

4.  Проверьте **Регион** и **Подписку**, в которых вы хотите подготовить это решение.

5.  Щелкните **Создать решение**.

## Просмотр панели мониторинга для решения удаленного мониторинга

1.  Когда подготовка завершится и на элементе для вашего предварительно настроенного решения появится надпись **Готово**, щелкните **Запустить**, чтобы открыть панель мониторинга вашего решения на новой вкладке.

2.  По умолчанию в меню слева выбрана **Панель мониторинга**. Это панель мониторинга вашего решения.

## Просмотр списка устройств решения

1.  Щелкните **Устройства** в меню слева для перехода к списку устройств решения.

2.  После окончания подготовки вы увидите 4 виртуальных устройства.

3.  Щелкните на **отдельном устройстве** в списке, чтобы просмотреть сведения об этом устройстве.

## Отправка команды устройству

1.  Щелкните **Отправить команду** в области сведений об устройстве для выбранного виртуального устройства.

2.  Выберите **Проверка связи** из раскрывающегося списка команд.

3.  Щелкните **Отправить команду**.

4.  Просмотрите состояние команды при его появлении в журнале команд.

## Добавление нового виртуального устройства

1.  Щелкните **←** (стрелку назад) для возврата в список устройств.

2.  Щелкните **+ Добавить устройство** в левом нижнем углу для добавления нового устройства.

3.  Щелкните **Добавить новое** для **Виртуального устройства**.

4.  Выберите **Задать идентификатор устройства самостоятельно** и укажите имя уникального идентификатора устройства.

5.  Щелкните **Создать**.

6.  Щелкните **←** (стрелку назад) для возврата в список устройств.

7.  Убедитесь, что состояние вашего устройства — **Работает**.

## Просмотр и изменение правил решения

1.  Обратите внимание на таблицу **История оповещений** на **панели мониторинга решения**.

2.  Эти предупреждения вызываются выводом правила **AlarmTemp**, которое указано в разделе **Правила**.

3.  Щелкните **Правила** в меню слева для перехода к списку правил решения.

4.  После подготовки правило 1 будет уже включено.

5.  Щелкните **правило** в списке правил, чтобы просмотреть свойства соответствующего правила.

6.  Щелкните **Изменить** на панели свойств правила.

7.  Измените **Пороговое значение** на 30, оставив остальные свойства без изменений.

8.  Щелкните **Сохранить и просмотреть правила**.

9.  Вернитесь к таблице **История оповещений** на **панели мониторинга** и обратите внимание на изменение условия активации в результате изменения правила.

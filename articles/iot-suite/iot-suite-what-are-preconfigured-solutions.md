# Что такое предварительно настроенные решения Azure IoT Suite?

Предварительно настроенные решения Azure IoT Suite представляют собой реализацию стандартных шаблонов IoT, которые вы можете развернуть в Microsoft Azure в рамках своей подписки Azure. Предварительно настроенные решения можно использовать:

- в качестве отправной точки для собственных решений IoT;
- для ознакомления с распространенными шаблонами, которые используются при проектировании и разработке решений IoT.

Каждое предварительно настроенное решение реализует распространенный сценарий IoT и представляет собой завершенную комплексную реализацию, в которой для создания данных телеметрии используются виртуальные устройства.

Наряду с развертыванием и запуском предварительно настроенных решений в Azure можно загрузить полный исходный код, а затем настраивать и расширять решение в соответствии с конкретными требованиями IoT.

Доступны следующие предварительно настроенные решения:

- [Удаленный мониторинг][lnk-remote-monitoring]
- [Диагностическое обслуживание][lnk-predictive-maintenance]

В следующей таблице показано соответствие этих предварительно настроенных решений конкретным функциям IoT.

|------------------------|-----|-----|-----|-----|-----|

## Устройства

Развертывание предварительно настроенного решения для удаленного мониторинга включает экземпляры программного эмулятора устройства, которые имитируют аппаратное охлаждающее устройство. Виртуальные устройства отправляют телеметрические данные о температуре и влажности в конечную точку центра IoT. Виртуальные устройства также реагируют на следующие команды, отправляемые с портала решения через центр IoT:

- проверка связи с устройством;
- начало сбора данных телеметрии;
- завершение сбора данных телеметрии;
- изменение заданной температуры;
- диагностические данные телеметрии;
- изменение состояния устройства.

## Центр IoT

Центр IoT получает данные телеметрии из охлаждающего устройства в одной конечной точке. Центр IoT также поддерживает отдельные конечные точки устройства, через которые возможно получение отправляемых ему команд, например команды **Проверить связь с устройством**.

Центр IoT предоставляет полученные данные телеметрии конечной точке группы потребителей.

В этом предварительно настроенном решении экземпляр центра IoT соответствует *облачному шлюзу* в стандартной [архитектуре решения IoT][lnk-what-is-azure-iot].

## Azure Stream Analytics

Предварительно настроенное решение использует три задания службы [Azure Stream Analytics][lnk-asa] (ASA) для фильтрации потока данных телеметрии от охлаждающих устройств.

- Задание 1 отправляет все данные телеметрии в хранилище BLOB-объектов Azure для хранения в автономном неструктурированном защищенном хранилище.
- Задание 2 фильтрует поток данных телеметрии, определяя отправляемые устройствами сообщения с ответами на команды и сообщения об обновлении состояния устройств. Затем эти сообщения отправляются в конечную точку концентратора событий Azure.
- Задание 3 выбирает из потока данных телеметрии значения, которые активируют аварийные сигналы. Если значение активирует аварийный сигнал, решение отображает уведомление в таблице журнала аварийных сигналов, которая отображается на панели мониторинга на портале решения.

В этом предварительно настроенном решении задания ASA формируют *серверную часть решения IoT* в стандартной [архитектуре решения IoT][lnk-what-is-azure-iot].

## Обработчик событий

Экземпляр [EventPocessorHost][lnk-event-processor], выполняющийся в [веб-задании][lnk-web-job], обрабатывает ответы на команды и сообщения о состоянии устройства, определенные в задании 2, и сохраняет эту информацию в базе данных [Azure DocumentDB][lnk-document-db].

В этом предварительно настроенном решении обработчик событий формирует *серверную часть решения IoT* в стандартной [архитектуре решения IoT][lnk-what-is-azure-iot].

## Портал решения

Портал решения представляет собой веб-интерфейс, разворачиваемый в облаке в составе предварительно настроенного решения. Он позволяет:

- просматривать данные телеметрии и историю оповещений на панели мониторинга;
- подготавливать новые устройства;
- осуществлять управление и мониторинг устройств;
- отправлять команды на определенные устройства;
- управлять правилами и действиями.

> [AZURE.NOTE] На портале решения также выполняется синхронизация [реестра удостоверений устройств][lnk-identity-registry] центра IoT с хранилищем расширенных сведений о состоянии устройств в базе данных DocumentDB этого решения.

В данном предварительно настроенном решении портал образует *серверную часть решения IoT*, а также часть *обработки и бизнес-подключений* в стандартной [архитектуре решения IoT][lnk-what-is-azure-iot].

# <a name="troubleshoot-system-center-data-protection-manager"></a>Устранение неполадок при работе с System Center Data Protection Manager

Последние заметки о выпуске SC DPM см. [здесь](https://docs.microsoft.com/en-us/system-center/dpm/dpm-release-notes?view=sc-dpm-2016).
Таблицу поддержки защиты см. [здесь](https://docs.microsoft.com/en-us/system-center/dpm/dpm-protection-matrix?view=sc-dpm-2016).

## <a name="replica-is-inconsistent"></a>Несогласованная реплика.

Эта ошибка может возникать по различным причинам, таким как сбой создания реплики, проблемы с журналом изменений, ошибки битовой карты для фильтра на уровне тома, неожиданное отключение исходного компьютера, переполнение журнала синхронизации или несогласованность реплики. Проблему можно устранить следующим способом.
- Чтобы устранить несогласованное состояние, выполните проверку согласованности вручную или настройте ежедневную проверку согласованности по расписанию.
- Убедитесь, что используется последняя версия сервера MAB или System Center DPM.
- Убедитесь, что включена автоматическая проверка согласованности.
- Попытайтесь перезапустить службы из командной строки (введите "net stop dpmra", а затем "net start dpmra").
- Убедитесь, что соблюдаются требования к сетевым подключениям и пропускной способности.
- Проверьте, не был ли исходный компьютер неожиданно выключен.
- Проверьте, исправен ли диск и достаточно ли места для реплики.
- Проверьте, нет ли выполняющегося параллельно аналогичного задания резервного копирования.

## <a name="online-recovery-point-creation-failed"></a>Сбой при создании точки восстановления в сети

Проблему можно устранить следующим способом.
- Убедитесь, что вы используете последнюю версию агента Azure Backup.
- Попробуйте вручную создать точку восстановления в области задач защиты.
- Обязательно проверьте согласованность источника данных.
- Убедитесь, что соблюдаются требования к сетевым подключениям и пропускной способности.
- Если данные реплики находятся в несогласованном состоянии, создайте точку восстановления диска для этого источника данных.
- Убедитесь, что реплика существует.
- Убедитесь, что в реплике достаточно места для создания журнала USN.

## <a name="unable-to-configure-protection"></a>Не удалось настроить защиту.

Эта ошибка возникает, когда серверу DPM не удается связаться с защищенным сервером. Проблему можно устранить следующим способом.
- Убедитесь, что вы используете последнюю версию агента Azure Backup.
- Убедитесь в наличии подключения (по сети, через брандмауэр или прокси-сервер) между сервером DPM и защищенным сервером.
- Если вы хотите защитить SQL Server, убедитесь, что для пользователя NT AUTHORITY\SYSTEM в разделе "Свойства имени входа" включена роль sysadmin.

## <a name="this-server-is-not-registered-to-the-vault-specified-by-the-vault-credential"></a>Этот сервер не зарегистрирован в хранилище, заданном с помощью учетных данных хранилища

Эта ошибка возникает, когда выбранный файл учетных данных хранилища не принадлежит к хранилищу служб восстановления, связанному с сервером System Center DPM или Azure Backup Server, на котором выполняется попытка восстановления. Проблему можно устранить следующим способом.
- Скачайте файл учетных данных из хранилища служб восстановления, в котором зарегистрирован сервер System Center DPM или Azure Backup Server.
- Попробуйте зарегистрировать сервер в хранилище с помощью последнего скачанного файла учетных данных хранилища.

## <a name="either-the-recoverable-data-is-not-available-or-the-selected-server-is-not-a-dpm-server"></a>Восстанавливаемые данные недоступны или выбранный сервер не является сервером DPM
Эта ошибка возникает, если другие серверы System Center DPM или Azure Backup Server, зарегистрированные в хранилище служб восстановления, отсутствуют, на серверы еще не загрузились метаданные или выбранный сервер не является сервером System Center DPM либо Azure Backup Server.
- Если в хранилище служб восстановления есть другие зарегистрированные серверы System Center DPM или Azure Backup Server, убедитесь, что установлена последняя версия агента Azure Backup.
- Если в хранилище служб восстановления есть другие зарегистрированные серверы System Center DPM или Azure Backup Server, подождите один день после установки. Затем можно приступить к процессу восстановления. Задания, запланированные на ночь, отправят метаданные для всех защищенных файлов резервных копий в облако. Данные будут доступны для восстановления.

## <a name="the-encryption-passphrase-provided-does-not-match-with-passphrase-associated-with-the-following-server"></a>Указанная парольная фраза для шифрования не соответствует парольной фразе, связанной со следующим сервером

> [!NOTE]
>Если вы забыли или потеряли парольную фразу для шифрования, восстановить данные нельзя. Единственный вариант — повторно создать парольную фразу и использовать ее в дальнейшем для шифрования данных резервного копирования.
>
>

Эта ошибка возникает, если указанная парольная фраза не соответствует парольной фразе для шифрования, использованной при шифровании восстанавливаемых данных System Center DPM или Azure Backup Server. Агенту не удалось расшифровать данные. Поэтому восстановление завершается неудачно. Проблему можно устранить следующим способом.
- Укажите именно ту парольную фразу, которая связана с сервером System Center DPM или Azure Backup Server, данные для которого восстанавливаются.

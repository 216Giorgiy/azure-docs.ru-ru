# Резервное копирование и восстановление c сервера Windows или клиентского компьютера Windows
В этой статье описывается процедура резервного копирования с использованием сервера Windows и клиентского компьютера Windows. Здесь также рассматриваются шаги, необходимые для восстановления файлов из резервной копии на том же и любом другом компьютере.

## Резервное копирование файлов

1. После регистрации компьютера откройте оснастку MMC в службе архивации Microsoft Azure.

    ![Результат поиска](./media/backup-azure-backup-and-recover/result.png)

2. Выберите команду **Создать расписание архивации**.

    ![Создание расписания архивации](./media/backup-azure-backup-and-recover/schedulebackup.png)

3. Выберите элементы, для которых необходимо создать резервные копии. Служба архивации Azure на сервере или клиенте Windows (т. е. без System Center Data Protection Manager) позволяет защитить файлы и папки.

    ![Элементы для резервного копирования](./media/backup-azure-backup-and-recover/items.png)

4. Укажите расписание резервного копирования и политику хранения, как описано в [этой статье](backup-azure-backup-cloud-as-tape.md).

5. Выберите метод отправки начальной резервной копии. Выбор параметров для начального заполнения зависит от объема данных, для которых необходимо выполнить резервное копирование, и от скорости вашего интернет-соединения. Если вы планируете создать резервную копию нескольких гигабайт или терабайт данных через сеть с высокой задержкой и низкой пропускной способностью, рекомендуется выполнить начальное резервное копирование, отправив диск в ближайший центр обработки данных Azure. Это называется автономным резервным копированием, о чем подробно описано в [этой статье](backup-azure-backup-import-export.md). При наличии сети с достаточной пропускной способностью рекомендуется выполнять начальное резервное копирование по сети.

    ![Начальное резервное копирование](./media/backup-azure-backup-and-recover/initialbackup.png)

6. После завершения процесса вернитесь к оснастке консоли MMC и выберите команду **Выполнить моментальную архивацию**, чтобы завершить начальное заполнение по сети.

    ![Команда «Выполнить моментальную архивацию»](./media/backup-azure-backup-and-recover/backupnow.png)

7. После завершения начального заполнения в консоли службы архивации Azure в представлении **Задания** отобразится соответствующее состояние.

    ![Первоначальное восстановление завершено](./media/backup-azure-backup-and-recover/ircomplete.png)

## Восстановление данных на том же компьютере
Если вы случайно удалили файл или том и хотите восстановить его с того же компьютера (на котором создана резервная копия), выполните указанные ниже действия.

1. Откройте оснастку **Служба архивации Microsoft Azure**.

2. Выберите действие **Восстановить данные**, чтобы запустить рабочий процесс.

    ![Восстановление данных](./media/backup-azure-backup-and-recover/recover.png)

3. Выберите параметр **Этот сервер (\*имя\_компьютера\*)**, так как вы собираетесь восстановить файл из резервной копии на том же компьютере.

    ![Тот же компьютер](./media/backup-azure-backup-and-recover/samemachine.png)

4. Далее можно выбрать **Обзор файлов** или **Поиск файлов**. Если вы планируете восстановить один или несколько файлов с известным путем, оставьте вариант по умолчанию. Если вы не знаете структуру папок и хотите найти файл, выберите вариант **Поиск файлов**. В этом разделе мы будем использовать параметр по умолчанию.

    ![Обзор файлов](./media/backup-azure-backup-and-recover/browseandsearch.png)

5. Выберите том, в котором хранится резервная копия файла. Этот экран позволяет восстановить состояние данные на любой момент времени. Даты, которые в календаре выделены **полужирным**, означают доступность точки восстановления. После выбора даты в расписании резервного копирования (которое было завершено успешно) в раскрывающемся списке **Время** можно выбрать момент времени.

    ![Том и дата](./media/backup-azure-backup-and-recover/volanddate.png)

6. Выберите элементы, которые нужно восстановить. Для восстановления можно выбрать сразу несколько папок и файлов.

    ![Выбор файлов](./media/backup-azure-backup-and-recover/selectfiles.png)

7. Укажите параметры восстановления.

    ![Варианты восстановления](./media/backup-azure-backup-and-recover/recoveroptions.png)

  - Данные можно восстановить в исходное расположение (в котором папка или файл будут перезаписаны) или в другое расположение на том же компьютере.
  - Если восстанавливаемый файл (или папка) существует в целевом расположении, вы можете создать копию (будет две версии одного файла) или перезаписать файлы в целевом расположении. Также можно пропустить восстановление файлов, которые существуют в целевом расположении.
  - Настоятельно рекомендуем оставить параметр восстановление ACL для восстанавливаемых файлов (выбран по умолчанию).

8. После указания этих параметров запускается рабочий процесс восстановления файлов на этом компьютере.

## Восстановление файлов на другой компьютер
При потере всего содержимого сервера вы можете восстановить файл или том на другой компьютер. Соответствующий рабочий процесс описан ниже.

В описанных шагах используется следующая номенклатура: *Исходный компьютер* — машина, с которой была создана резервная копия и которая на данный момент недоступна. *Целевой компьютер* — компьютер, на котором извлекаются данные. *Образец хранилища* — хранилище архивации, в котором зарегистрированы *исходный* и *целевой компьютер*.<br/>

> [AZURE.NOTE]Резервные копии, созданные с компьютера с более поздней версией ОС, невозможно восстановить на компьютер с более ранней версией ОС. Например, если резервные копии созданы с компьютера под управлением Windows 7, их можно восстановить на компьютер под управлением Windows 8 или более поздних версий ОС, но не наоборот.

1. Откройте на *целевом компьютере* оснастку **Служба архивации Microsoft Azure**.

2. Убедитесь, что и *целевой*, и *исходный компьютер* восстанавливаются в одно и то же хранилище архивации.

3. Выберите действие **Восстановить данные**, чтобы запустить рабочий процесс.

    ![Восстановление данных](./media/backup-azure-backup-and-recover/recover.png)

4. Выберите вариант **Другой сервер**.

    ![Другой сервер](./media/backup-azure-backup-and-recover/anotherserver.png)

5. Укажите файл с учетными данными хранилища, который соответствует *образцу хранилища*. Если файл с учетными данными хранилища недействителен (или просрочен), загрузите новый файл из *образца хранилища* на портале Azure. Когда файл с учетными данными будет указан, отобразится соответствующее хранилище службы архивации.

6. Выберите в предложенном списке *Исходный компьютер*.

    ![Список компьютеров](./media/backup-azure-backup-and-recover/machinelist.png)

7. Как и раньше, выберите вариант **Поиск файлов** или **Обзор файлов**. В этом разделе мы будем использовать параметр **Поиск файлов**.

    ![Поиск](./media/backup-azure-backup-and-recover/search.png)

8. На следующем экране выберите том и дату. Найдите имя папки или файла, который нужно восстановить.

    ![Поиск элементов](./media/backup-azure-backup-and-recover/searchitems.png)

9. Выберите расположение, в которое необходимо восстановить файлы.

    ![Расположение для восстанавливаемых данных](./media/backup-azure-backup-and-recover/restorelocation.png)

10. Укажите парольную фразу для шифрования, заданную во время регистрации *исходного компьютера* в *образце хранилища*.

    ![Шифрование](./media/backup-azure-backup-and-recover/encryption.png)

11. После ввода данных нажмите кнопку **Восстановить**, после чего начнется восстановление файлов из резервных копий в указанное расположение.

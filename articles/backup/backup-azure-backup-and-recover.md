# Резервное копирование и восстановление c сервера Windows или клиентского компьютера Windows
В этой статье описывается процедура резервного копирования с использованием сервера Windows и клиентского компьютера Windows. Здесь также рассматриваются шаги, необходимые для восстановления файлов из резервной копии на том же и любом другом компьютере.

## Резервное копирование файлов
1. После регистрации компьютера откройте оснастку MMC в службе архивации Microsoft Azure. <br/> ![Результат поиска][1]

2. Выберите команду **Создать расписание архивации** <br/> ![Создание расписания архивации][2]

3. Выберите элементы, для которых необходимо создать резервные копии. Служба архивации Azure на сервере или клиенте Windows (т. е. без System Center Data Protection Manager) позволяет защитить файлы и папки. <br/> ![Элементы для резервного копирования][3]

4. Укажите расписание резервного копирования и политику хранения, как описано в [этой статье](backup-azure-backup-cloud-as-tape.md).

5. Выберите метод отправки начальной резервной копии. Выбор параметров для начального заполнения зависит от объема данных, для которых необходимо выполнить резервное копирование, и от скорости вашего интернет-соединения. Если вы планируете создать резервную копию нескольких гигабайт или терабайт данных через сеть с высокой задержкой и низкой пропускной способностью, рекомендуется выполнить начальное резервное копирование, отправив диск в ближайший центр обработки данных Azure. Это называется автономным резервным копированием, о чем подробно описано в [этой статье](https://msdn.microsoft.com/library/azure/dn894419.aspx). При наличии сети с достаточной пропускной способностью рекомендуется выполнить начальное резервное копирование по сети. <br/> ![Начальное резервное копирование][4]

6. После завершения рабочего процесса вернитесь к оснастке консоли MMC и выберите команду **Выполнить моментальную архивацию**, чтобы завершить начальное заполнение по сети. <br/> ![Команда «Выполнить моментальную архивацию»][5]

7. После завершения начального заполнения в консоли службы архивации Azure в представлении **Задания** отобразится соответствующее состояние. <br/> ![Первоначальное восстановление завершено][6]

## Восстановление данных на том же компьютере
Если вы случайно удалили файл или том и хотите восстановить его с того же компьютера (на котором создана резервная копия), выполните следующие действия.

1. Откройте оснастку **Служба архивации Microsoft Azure**.

2. Выберите действие **Восстановить данные**, чтобы запустить рабочий процесс. <br/> ![Восстановление данных][7]

3. Выберите параметр **Этот сервер (*имя_компьютера*)**, так как вы собираетесь восстановить файл из резервной копии на том же компьютере. <br/> ![Тот же компьютер][8]

4. Далее можно выбрать **Обзор файлов** или **Поиск файлов**. Если вы планируете восстановить один или несколько файлов с известным путем, оставьте вариант по умолчанию. Если вы не знаете структуру папок и хотите найти файл, выберите вариант **Поиск файлов**. В этом разделе мы будем использовать параметр по умолчанию. <br/> ![Обзор файлов][9]

5. На следующем экране выберите том, на котором хранится резервная копия файла. Этот экран позволяет восстановить состояние данные на любой момент времени. Даты, которые в календаре выделены **полужирным**, означают доступность точки восстановления. После выбора даты в расписании резервного копирования (которое было завершено успешно) в раскрывающемся списке **Время** можно выбрать момент времени. <br/> ![Том и дата][10]

6. На следующем экране выберите элементы, которые необходимо восстановить. Для восстановления можно выбрать сразу несколько папок и файлов. <br/> ![Выбор файлов][11]

7. На следующем экране укажите параметры восстановления. <br/> ![Варианты восстановления][12]
  + Данные можно восстановить в исходное расположение (в котором папка или файл будут перезаписаны) или в другое расположение на том же компьютере.

  + Если восстанавливаемый файл (или папка) существует в целевом расположении, вы можете создать копию (будет две версии одного файла) или перезаписать файлы в целевом расположении. Также можно пропустить восстановление файлов, которые существуют в целевом расположении.

  + Настоятельно рекомендуем оставить параметр восстановление ACL для восстанавливаемых файлов (выбран по умолчанию).

8. После указания этих параметров запускается рабочий процесс восстановления файлов на этом компьютере.

## Восстановление файлов на другой компьютер
При потере всего содержимого сервера вы можете восстановить файл или том на другой компьютер. Соответствующий рабочий процесс описан ниже.

В этом разделе используется такая терминология: + *Исходный компьютер* — компьютер, для данных на котором была создана резервная копия и который в настоящий момент недоступен.

  + *Целевой компьютер* — компьютер, на который восстанавливаются данные.

  + *Образец хранилища* — хранилище службы архивации, в котором зарегистрированы *исходный* и *целевой* компьютеры. <br/>

> [AZURE.NOTE]Резервные копии, созданные с компьютера с более поздней версией ОС, невозможно восстановить на компьютер с более ранней версией ОС. Например, если резервные копии созданы с компьютера под управлением Windows 7, их можно восстановить на компьютер под управлением Windows 8 или более поздних версий ОС, но не наоборот.

1. Откройте на *целевом компьютере* оснастку **Служба архивации Microsoft Azure**.

2. Убедитесь, что для *целевого компьютера* и *исходного компьютера* восстановление выполняется в одно и то же хранилище службы архивации (в этой статье — *образец хранилища*).

3. Выберите действие **Восстановить данные**, чтобы запустить рабочий процесс. <br/> ![Восстановление данных][7]

4. Выберите вариант **Другой сервер** <br/> ![Другой сервер][13]

5. Укажите файл с учетными данными хранилища, который соответствует *образцу хранилища*. Если файл с учетными данными хранилища недействителен (или просрочен), загрузите новый файл из *образца хранилища* на портале Azure. Когда файл с учетными данными будет указан, отобразится соответствующее хранилище службы архивации.

6. Выберите в предложенном списке *исходный компьютер*. <br/> ![Список компьютеров][14]

7. Как и раньше, выберите вариант **Поиск файлов** или **Обзор файлов**. В этом разделе мы будем использовать параметр **Поиск файлов**. <br/> ![Поиск][15]

8. На следующем экране выберите том и дату. Найдите имя папки (или файла), которую нужно восстановить. <br/> ![Поиск элементов][16]

9. Выберите расположение, в которое необходимо восстановить файлы. <br/> ![Расположение для восстанавливаемых данных][17]

10. Укажите парольную фразу для шифрования, которая была указана во время регистрации *исходного компьютера* в *образце хранилища*. <br/> ![Шифрование][18]

После ввода данных нажмите кнопку **Восстановить**, после чего начнется восстановление резервных копий файлов в указанное расположение.

# Подготовка среды к резервному копированию компьютеров Windows в Azure
В этой статье перечислены задачи, которые необходимо выполнить, чтобы подготовить среду для резервного копирования компьютера Windows в Azure.

| Шаг | Имя | Сведения |
| :------: | ---- | ------- |
| 1 | [Создание хранилища](#create-a-backup-vault) | Создайте хранилище на [портале управления службы архивации Azure](http://manage.windowsazure.com). |
| 2 | [Скачивание учетных данных хранилища](#download-the-vault-credential-file) | Скачайте учетные данные хранилища, которые будут использоваться для регистрации компьютера Windows в хранилище службы архивации. |
| 3 | [Установка агента службы архивации Azure](#download-install-and-register-the-azure-backup-agent) | Установите агент и зарегистрируйте сервер в хранилище службы архивации с помощью учетных данных хранилища. |

Если вы уже выполнили все основные действия, указанные выше, можно приступать к [резервному копированию компьютеров Windows](backup-azure-backup-windows-server.md). В противном случае выполните подробные инструкции ниже, чтобы подготовить свою среду.

## Перед началом работы
Для подготовки среды к резервному копированию компьютеров Windows требуется учетная запись Azure. Если ее нет, можно создать [бесплатную пробную учетную запись](https://azure.microsoft.com/pricing/free-trial/) всего за несколько минут.

## Создание хранилища службы архивации
Для резервного копирования файлов и данных с компьютера Windows или из Data Protection Manager (DPM) в Azure либо для резервного копирования виртуальных машин IaaS в Azure необходимо создать хранилище службы архивации в том географическом регионе, в котором необходимо хранить данные.

**Чтобы создать хранилище службы архивации, сделайте следующее.**

1. Войдите на [портал управления](https://manage.windowsazure.com/).

2. Последовательно выберите пункты **Создать**, **Службы данных**, **Службы восстановления**, **Хранилище архивации** и **Быстрое создание**.

    ![Создать хранилище](./media/backup-configure-vault/createvault1.png)

3. В поле **Имя** укажите понятное имя для идентификации хранилища. Это имя должно быть уникальным для каждой подписки.

    В поле **Регион** выберите географический регион хранилища. Этот выбор определяет географический регион, в который будут отправляться резервные копии данных. Выбрав географический регион, который находится недалеко от вашего расположения, можно уменьшить задержки в сети при архивации данных в Azure.

    Для завершения процесса выберите команду **Создать хранилище**.

    Для создания хранилища службы архивации может потребоваться некоторое время. Для проверки состояния можно отслеживать уведомления в нижней части портала.

    ![Создание хранилища](./media/backup-configure-vault/creatingvault1.png)

    После создания хранилища службы архивации отобразится сообщение о том, что оно успешно создано. Кроме того, это хранилище отобразится в списке ресурсов служб восстановления с состоянием **Активное**. ![Состояние создания хранилища](./media/backup-configure-vault/backupvaultstatus1.png)

> [AZURE.IMPORTANT] Наилучшее время для определения параметров избыточности хранилища — сразу после его создания и перед регистрацией в нем компьютеров. После регистрации элемента в хранилище параметр избыточности хранилища блокируется и в последствии не может быть изменен.
>
> **Дополнительные сведения о выборе вариантов избыточности хранилища см. в этом [обзоре](backup-azure-storage-redundancy-options.md).**

## Скачивание файла учетных данных хранилища
Перед архивированием данных в Azure локальный сервер (клиент Windows, Windows Server или сервер Data Protection Manager) должен пройти проверку подлинности в хранилище архивации. Проверка подлинности осуществляется с помощью учетных данных хранилища. Файл учетных данных хранилища загружается через безопасный канал с портала Azure. При этом служба архивации Azure не знает о существовании закрытого ключа сертификата, который не сохраняется на портале или в службе.

Дополнительные сведения об [использовании учетных данных хранилища для проверки подлинности в службе архивации Azure](backup-introduction-to-azure-backup.md#what-is-the-vault-credential-file).

**Чтобы скачать файл учетных данных хранилища на локальный компьютер, выполните указанные ниже действия.**

1. Войдите на [портал управления](https://manage.windowsazure.com/).

2. В области навигации слева щелкните пункт **Службы восстановления** и выберите созданное хранилище службы архивации.

3.  Щелкните значок облака, чтобы перейти к представлению *быстрого запуска* хранилища службы архивации.

    ![Быстрый просмотр](./media/backup-configure-vault/quickview.png)

4. На странице **быстрого запуска** щелкните **Скачать учетные данные хранилища**.

    ![Скачивание](./media/backup-configure-vault/downloadvc.png)

    Портал создаст учетные данные хранилища с помощью сочетания имени хранилища и текущей даты. Файл учетных данных хранилища используется только во время регистрации. Срок его действия истекает через 48 часов.

    Файл учетных данных хранилища можно скачать с портала.

5. Щелкните **Сохранить**, чтобы скачать учетные данные хранилища в папку скачиваемых файлов локальной учетной записи, или выберите пункт **Сохранить как** в меню *Сохранить*, чтобы указать расположение учетных данных хранилища.

    Открывать учетные данные хранилища в данный момент не требуется.

    Убедитесь, что учетные данные хранилища сохраняются в расположении, к которому у вашего компьютера есть доступ. Если они сохраняются на файловый ресурс, доступ к которому осуществляется по протоколу SMB, убедитесь, что у вас есть необходимые разрешения на доступ к этому ресурсу.

## Скачивание, установка и регистрация агента службы архивации Azure
После создания хранилища службы архивации Azure на каждом компьютере под управлением Windows (Windows Server, клиент Windows, сервер System Center Data Protection Manager и компьютер с сервером службы архивации Azure) необходимо установить агент, который позволяет архивировать данные и приложения в Azure.

**Чтобы скачать, установить и зарегистрировать агент, сделайте следующее.**

1. Войдите на [портал управления](https://manage.windowsazure.com/).

2. Щелкните **Службы восстановления**, затем выберите хранилище архивации, которое необходимо зарегистрировать на сервере.

3. На странице быстрого запуска щелкните **Агент для Windows Server, System Center Data Protection Manager или клиента Windows > Сохранить**.

    ![Сохранить агент](./media/backup-configure-vault/agent.png)

4. После завершения загрузки *MARSagentinstaller.exe* нажмите кнопку **запуска** (или дважды щелкните файл **MARSAgentInstaller.exe** в папке, где он был сохранен). Выберите *папку установки* и *папку кэша* для агента, а затем щелкните **Далее**.

    В указанном расположении кэша должно быть свободное место, равное по крайней мере 5 % размера резервной копии данных.

    ![Папка временных файлов и кэша](./media/backup-configure-vault/recovery-services-agent-setup-wizard-1.png)

5. При использовании прокси-сервера для подключения к Интернету на экране **Конфигурация прокси-сервера** введите данные прокси-сервера. Если используется проверенный прокси-сервер, введите имя пользователя и пароль и щелкните **Далее**.

    В конце процесса установки агент службы архивации Azure установит .NET Framework 4.5 и Windows PowerShell (если эти компоненты еще не установлены).

6. После установки агента нажмите кнопку **Перейти к регистрации**.

    ![Зарегистрировать](./media/backup-configure-vault/register.png)

7. На экране **Идентификация хранилища** найдите и выберите ранее скачанный *файл учетных данных хранилища*.

    ![Учетные данные хранилища](./media/backup-configure-vault/vc.png)

    Файл учетных данных хранилища действителен только в течение 48 часов после загрузки с портала. Если на этом экране отобразится ошибка (например, "Срок действия файла учетных данных хранилища истек"), войдите на портал Azure и загрузите файл учетных данных хранилища еще раз.

    Убедитесь, что файл учетных данных хранилища находится в расположении, к которому может получить доступ программа установки. При возникновении ошибок, связанных с доступом, скопируйте файл учетных данных хранилища во временное расположение на этом компьютере и повторите операцию.

    Если возникает ошибка недопустимых учетных данных хранилища (например, "Указаны недопустимые учетные данные хранилища"), это означает, что файл поврежден или в нем нет последних учетных данных, связанных со службой восстановления. Скачайте новый файл учетных данных хранилища на портале, а затем повторите операцию. Эта ошибка обычно возникает, если несколько раз подряд щелкнуть *Загрузить учетные данные хранилища*. В этом случае действительным является только последний файл учетных данных хранилища.

8. На экране **Настройка шифрования** можно *создать* парольную фразу или *ввести* уже существующую (длиной не менее 16 символов). Не забудьте сохранить парольную фразу в безопасном месте.

    ![Шифрование](./media/backup-configure-vault/encryption.png)

    > [AZURE.WARNING] Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не сможет помочь вам восстановить резервную копию данных. Парольная фраза принадлежит конечному пользователю, и корпорация Майкрософт не имеет возможности ее просматривать. Сохраните файл в безопасном месте, так как он потребуется при выполнении операции восстановления.

9. Нажмите кнопку **Готово**

    Теперь компьютер зарегистрирован в хранилище. Все готово для резервного копирования в Microsoft Azure.

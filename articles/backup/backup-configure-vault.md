# Резервное копирование Windows Server или клиентского компьютера Windows в Azure
В данной статье рассматриваются процедуры, которые необходимо выполнить для подготовки среды и архивации сервера Windows Server (или клиентского компьютера Windows) в Azure. Здесь также рассматриваются вопросы развертывания решения для архивации. Если вы хотите попробовать службу архивации Azure в первый раз, обратитесь к [этой статье](backup-configure-vault.md).

![Создать хранилище](./media/backup-configure-vault/initial-backup-process.png)

## Перед началом работы
Для архивации Windows Server (или клиентского компьютера Windows) в Azure вам потребуется учетная запись Azure. Если ее нет, можно создать [бесплатную учетную запись](https://azure.microsoft.com/free/) всего за несколько минут.

## Шаг 1. Создание хранилища архивации
Для архивации файлов и папок с сервера или клиента Windows необходимо создать хранилище архивации в том географическом регионе, в котором вы хотите хранить данные.

### Создание службы архивации

1. Войдите на [портал управления](https://manage.windowsazure.com/).

2. Последовательно выберите пункты **Создать**, **Службы данных**, **Службы восстановления**, **Хранилище архивации** и **Быстрое создание**.

3. В поле **Имя** укажите понятное имя для идентификации хранилища. Введите имя длиной от 2 до 50 знаков. Имя должно начинаться с буквы, оно может содержать только буквы, цифры и дефисы. Это имя должно быть уникальным для каждой подписки.

    В поле **Регион** выберите географический регион хранилища. Этот выбор определяет географический регион, в который будут отправляться резервные копии данных. Если вы выберите географический регион, который находится недалеко от вашего расположения, вы уменьшите задержки в сети во время резервного копирования в Azure.

    Щелкните **Создать хранилище**.

    ![Создать хранилище](./media/backup-configure-vault/demo-vault-name.png)

    Для создания резервного хранилища может потребоваться некоторое время. Для проверки состояния можно отслеживать уведомления в нижней части портала.

    После создания хранилища службы архивации отобразится сообщение о том, что оно успешно создано. Кроме того, это хранилище отобразится в списке ресурсов служб восстановления с состоянием **Активное**.

    ![Состояние создания хранилища](./media/backup-configure-vault/recovery-services-select-vault.png)

4. Выберите параметры **избыточности хранилища**.

    >[AZURE.IMPORTANT] Наилучшее время для определения параметров избыточности хранилища — сразу после его создания и перед регистрацией в нем компьютеров. После регистрации элемента в хранилище параметр избыточности хранилища блокируется и в последствии не может быть изменен.

    Если Azure используется как основная конечная точка для хранилища архивации (например: архивация в Azure выполняется с Windows Server), рекомендуем выбрать вариант [геоизбыточное хранилище](../storage/storage-redundancy.md#geo-redundant-storage) (значение по умолчанию).

    Если вы используете Azure как третьестепенную конечную точку хранилища службы архивации (например: SCDPM используется для локальной архивации, а Azure — для длительного хранения), следует выбрать [локально избыточное хранилище](../storage/storage-redundancy.md#locally-redundant-storage). Это снизит затраты на хранение данных в Azure, но при этом также снизится уровень устойчивости данных, что может быть допустимо для копий третьего уровня.

    **Выбор параметров избыточности хранилища:**

    а. Щелкните хранилище, которое вы только что создали.

    b. На **странице быстрого запуска** щелкните **Настроить**.

    ![Состояние настройки хранилища](./media/backup-configure-vault/configure-vault.png)

    c. Выберите соответствующий параметр избыточности хранилища.

    Если вы выбрали параметр **Локально избыточное**, нажмите кнопку **Сохранить**, так как по умолчанию используется параметр **Геоизбыточное**.

    г) На панели навигации слева щелкните **Службы восстановления**, чтобы вернуться к списку ресурсов **служб восстановления**.

## Шаг 2. Скачивание файла учетных данных хранилища
Перед архивацией данных в Azure локальный компьютер (с сервером Windows Server или клиентом Windows) должен пройти проверку подлинности в хранилище службы архивации. Проверка подлинности выполняется с помощью *учетных данных хранилища*. Файл учетных данных хранилища скачивается с портала Azure через безопасный канал. Служба архивации Azure не знает о существовании закрытого ключа сертификата, а сам закрытый ключ не хранится ни на портале, ни в службе.

Узнайте больше об [использовании учетных данных хранилища для проверки подлинности в службе архивации Azure](backup-introduction-to-azure-backup.md#what-is-the-vault-credential-file).

### Скачивание файла учетных данных хранилища на локальный компьютер

1. В области навигации слева щелкните **Службы восстановления** и выберите созданное хранилище архивации.

    ![Первоначальное восстановление завершено](./media/backup-configure-vault/rs-left-nav.png)

2.  На странице **быстрого запуска** щелкните **Скачать учетные данные хранилища**.

    Портал создаст учетные данные хранилища с помощью сочетания имени хранилища и текущей даты. Файл учетных данных хранилища используется только во время регистрации. Срок его действия истекает через 48 часов.

    Файл учетных данных хранилища можно скачать с портала.

3. Щелкните **Сохранить**, чтобы скачать учетные данные хранилища в папку скачиваемых файлов локальной учетной записи, или выберите пункт **Сохранить как** в меню **Сохранить**, чтобы указать расположение учетных данных хранилища.

    >[AZURE.NOTE] Убедитесь, что учетные данные хранилища сохраняются в расположении, которое доступно с вашего компьютера. Если они сохраняются на файловый ресурс, доступ к которому осуществляется по протоколу SMB, убедитесь, что у вас есть необходимые разрешения на доступ к этому ресурсу.

## Шаг 3. Скачивание, установка и регистрация агента службы архивации Azure
После создания хранилища Azure Backup и скачивания учетных данных хранилища необходимо установить агент на всех компьютерах Windows (с серверами Windows Server или клиентами Windows).

### Скачивание, установка и регистрация агента

1. Щелкните **Службы восстановления**, затем выберите хранилище архивации, которое необходимо зарегистрировать на сервере.

2. На странице быстрого запуска щелкните **Агент для Windows Server, System Center Data Protection Manager или клиент Windows > Сохранить**.

    ![Сохранить агент](./media/backup-configure-vault/agent.png)

3. После завершения загрузки *MARSagentinstaller.exe* нажмите кнопку **Запустить** (или дважды щелкните файл **MARSAgentInstaller.exe** в папке, в которой он был сохранен).

4. 	Выберите *папку установки* и *папку кэша* для агента, а затем щелкните **Далее**.

    В указанном расположении кэша должно быть свободное место, равное по крайней мере 5 % размера резервной копии данных.

5. Для подключения к Интернету можно и дальше использовать настройки прокси-сервера по умолчанию. При необходимости изменить их в разделе **Конфигурация прокси-сервера** установите флажок **Использовать пользовательские настройки прокси-сервера** и введите нужные параметры сервера.

    Если вы используете проверенный прокси-сервер, введите имя пользователя и пароль.

    Нажмите кнопку **Далее**.

6. Чтобы приступить к установке агента, нажмите кнопку **Установить**.

    В конце процесса установки агент службы архивации Azure установит .NET Framework 4.5 и Windows PowerShell (если эти компоненты еще не установлены).

7. После установки агента нажмите кнопку **Перейти к регистрации**, чтобы продолжить.

8. В разделе **Идентификация хранилища** найдите и выберите ранее скачанный *файл учетных данных хранилища*.

    Файл учетных данных хранилища действителен только в течение 48 часов после загрузки с портала. Если на этом этапе отобразится ошибка (например: "Срок действия файла учетных данных истек"), войдите на портал Azure и загрузите файл учетных данных хранилища еще раз.

    Убедитесь, что файл учетных данных хранилища находится в расположении, к которому может получить доступ программа установки. При возникновении ошибок, связанных с доступом, скопируйте файл учетных данных хранилища во временное расположение на этом компьютере и повторите операцию.

    Если возникает ошибка недопустимых учетных данных хранилища (например: "Указаны недопустимые учетные данные хранилища"), это означает, что либо файл поврежден, либо в нем нет последних учетных данных, сопоставленных со службой восстановления. Скачайте новый файл учетных данных хранилища на портале, а затем повторите операцию. Эта ошибка обычно возникает, если несколько раз подряд щелкнуть *Загрузить учетные данные хранилища*. В этом случае действительным является только последний файл учетных данных хранилища.

9. На экране **Параметры шифрования** можно *создать* парольную фразу или *ввести* уже имеющуюся (длиной не менее 16 символов). Не забудьте сохранить парольную фразу в безопасном месте.

    Нажмите кнопку **Готово**

    > [AZURE.WARNING] Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не сможет помочь вам восстановить резервную копию данных. Парольная фраза принадлежит конечному пользователю, и корпорация Майкрософт не имеет возможности ее просматривать. Сохраните файл в безопасном месте, так как он потребуется при выполнении операции восстановления.

    **Мастер регистрации сервера** зарегистрирует сервер в службе архивации Microsoft Azure.

10. Когда **ключ шифрования** будет настроен, оставьте установленным флажок **Запустить агент служб восстановления Microsoft Azure** и нажмите кнопку **Закрыть**.

## Шаг 4. Выполните начальную архивацию

Начальная архивация состоит из двух основных задач: **создание расписания архивации** и **первая архивация файлов и папок**. После выполнения первоначальной архивации на основании заданного расписания политика архивации создает точки архивации, которые можно использовать для восстановления данных.

### Планирование архивации

1. Откройте **агент архивации Microsoft Azure** (он откроется автоматически, если оставить флажок **Запустить агент служб восстановления Microsoft Azure** установленным при закрытии **мастера регистрации серверов**; также его можно найти на своем компьютере, набрав в строке поиска *Служба архивации Microsoft Azure*).

    ![Планирование архивации Windows Server](./media/backup-configure-vault/snap-in-search.png)

2. В **агенте архивации Microsoft Azure** щелкните **Планирование архивации**.

    ![Планирование архивации Windows Server](./media/backup-configure-vault/schedule-backup-close.png)

3. В окне **Приступая к работе к мастером планирования архивации** щелкните **Далее**.

4. На экране **Выбор элементов для архивации** щелкните **Добавить элементы**.

5. Выберите файлы и папки, которые вы хотите архивировать, и нажмите кнопку **ОК**.

6. Нажмите кнопку **Далее**.

7. В окне **Укажите расписание архивации** выберите желаемое **расписание архивации** и нажмите кнопку **Далее**.

    Вы можете запланировать ежедневное (не более трех раз в день) или еженедельное резервное копирование.

    ![Элементы для архивации Windows Server](./media/backup-configure-vault/specify-backup-schedule-close.png)

    >[AZURE.NOTE] Дополнительные сведения об указании расписания для архивации см. в [этой статье](backup-azure-backup-cloud-as-tape.md).

8. В окне **Выберите политику хранения** выберите **Политику хранения** архива.

    Политика хранения определяет время, в течение которого должен храниться архив. Вместо того, чтобы настраивать одну политику для всех точек архивации, вы можете настроить разные политики хранения на основе времени создания архива. Вы можете выбрать ежедневную, еженедельную, ежемесячную или ежегодную политику хранения с учетом своих потребностей.

     Нажмите кнопку **Далее**.

9. В окне **Выбор типа первоначальной архивации** выберите тип первоначальной архивации**.

    Резервное копирование можно выполнять по сети или в автономном режиме. В оставшейся части этой статьи описан процесс автоматической архивации. Если вы предпочитаете автономную архивацию, см. статью [Автономная архивация в службе архивации Azure](backup-azure-backup-import-export.md).

    Оставьте флажок **Автоматически по сети** выбранным и нажмите кнопку **Далее**.

10. Проверьте сведения в окне **Подтверждение** и нажмите кнопку **Готово**.

11. Когда мастер завершит создание **расписания архивации**, нажмите кнопку **Закрыть**.

### Включить регулирование сети (необязательно)

Агент архивации Azure обеспечивает регулирование сети. Регулирование управляет тем, как пропускная способность сети используется во время передачи данных. Это полезно, если вам нужно выполнить архивацию в рабочее время так, чтобы операция копирования не мешала другим процессам, связанным с обработкой интернет-трафика. Регулирование передачи данных применяется при резервном копировании и восстановлении.

1. В **агенте архивации** щелкните **Изменить свойства**.

    ![Изменить свойства](./media/backup-configure-vault/change-properties.png)

2. На вкладке **Регулирование** установите флажок **Включить регулирование пропускной способности Интернета для операций архивации**.

    ![Регулирование сети](./media/backup-configure-vault/throttling-dialog.png)

3. После включения регулирования укажите допустимую пропускную способность для передачи данных во время архивации в **рабочие часы** и в **нерабочие часы**.

    Значения пропускной способности начинаются с 512 килобит в секунду (Кбит/с) и могут перейти до 1023 мегабит в секунду (Мбит/с). Также можно назначить время начала и окончания для **рабочих часов** и выбрать, какие дни недели считаются рабочими. Время вне назначенных рабочих часов считается нерабочим.

4. Нажмите кнопку **ОК**.

### Создать резервную копию

1. В **агенте службы архивации** нажмите кнопку **Выполнить архивацию сейчас**, чтобы завершить начальное заполнение по сети.

    ![Моментальная архивация Windows Server](./media/backup-configure-vault/backup-now.png)

2. На экране **Подтверждение** проверьте параметры, которые мастер будет использовать для архивации данных на компьютере, и нажмите кнопку **Выполнить архивацию**.

3. Нажмите кнопку **Закрыть**, чтобы закрыть мастер. Вы можете закрыть мастер до завершения **процесса архивации**. Мастер продолжит работу в фоновом режиме.

После завершения начальной архивации состояние *Задание завершено* появится в **консоли службы архивации Azure**.

![Первоначальное восстановление завершено](./media/backup-configure-vault/ircomplete.png)

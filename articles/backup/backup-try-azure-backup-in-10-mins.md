# Первое знакомство: резервное копирование файлов и папок из Windows Server или клиента Windows в Azure

Всего за несколько шагов вы можете создать резервную копию компьютера Windows (клиента Windows или Windows Server) в Azure.

Для этого нужно сделать следующее.

![Шаг 1](./media/backup-try-azure-backup-in-10-mins/step-1.png) Оформить подписку Azure (при необходимости).<br> ![Шаг 2](./media/backup-try-azure-backup-in-10-mins/step-2.png) Создать хранилище службы архивации и скачать необходимые компоненты.<br> ![Шаг 3.](./media/backup-try-azure-backup-in-10-mins/step-3.png) Подготовить сервер Windows Server или клиент Windows, установив и зарегистрировав эти компоненты.<br> ![Шаг 4.](./media/backup-try-azure-backup-in-10-mins/step-4.png) Обеспечить защиту данных.


![Процесс резервного копирования для сервера Windows Server и клиента Windows](./media/backup-try-azure-backup-in-10-mins/windows-machine-backup-process.png)

## Шаг 1. Оформление подписки Azure

Если у вас нет подписки Azure, вы можете создать [бесплатную учетную запись](https://azure.microsoft.com/free/), которая предоставляет доступ ко всем службам Azure.

>[AZURE.NOTE] Если у вас уже есть подписка Azure, этот шаг можно пропустить.

## Шаг 2. Создание хранилища службы архивации и загрузка необходимых компонентов.

Для резервного копирования файлов и данных с компьютера Windows в Azure необходимо создать хранилище резервных копий в том географическом регионе, в котором вы хотите хранить данные.

1. Используя подписку Azure, войдите на [портал Azure](https://portal.azure.com/), если вы еще этого не сделали.

2. Щелкните элементы **Создать > Гибридная интеграция > Резервная копия**.

    ![Создать хранилище](./media/backup-try-azure-backup-in-10-mins/second-blade-backup.png)

    а. На появившемся экране в поле **Имя** введите понятное имя для идентификации хранилища службы архивации. Введите имя длиной от 2 до 50 знаков. Имя должно начинаться с буквы, оно может содержать только буквы, цифры и дефисы.

    b. В поле **Регион** выберите географический регион для хранилища службы архивации. Если вы выберите географический регион, который находится недалеко от вашего расположения, вы уменьшите задержки в сети во время резервного копирования в Azure.

    c. Щелкните **Создать хранилище**.

    ![Создать хранилище](./media/backup-try-azure-backup-in-10-mins/demo-vault-name.png)

    Для проверки состояния отслеживайте уведомления в нижней части портала.

    ![Создание хранилища](./media/backup-try-azure-backup-in-10-mins/creatingvault1.png)

    Созданное хранилище будет отображаться в списке ресурсов служб восстановления со статусом **Активно**.

    ![Состояние создания хранилища](./media/backup-try-azure-backup-in-10-mins/recovery-services-select-vault.png)

3. Выберите параметры **избыточности хранилища**.

    Лучше выбирать параметры избыточности хранилища сразу после его создания и до регистрации в нем компьютеров. После регистрации элемента в хранилище параметр избыточности хранилища блокируется и в последствии не может быть изменен.

    Если Azure используется как основная конечная точка хранилища службы архивации (например, резервное копирование выполняется с Windows Server в Azure), попробуйте выбрать [геоизбыточное хранилище](../storage/storage-redundancy.md#geo-redundant-storage) (значение по умолчанию).

    Если вы используете Azure как третьестепенную конечную точку хранилища службы архивации (например, SCDPM используется для хранения резервных копий локально, а Azure — для длительного хранения), следует выбрать [локально избыточное хранилище](../storage/storage-redundancy.md#locally-redundant-storage). Это снизит затраты на хранение данных в Azure, но при этом также снизится уровень устойчивости данных, что может быть допустимо для копий третьего уровня.

    а. Щелкните хранилище, которое вы только что создали.

    b. На странице быстрого запуска щелкните **Настроить**.

    ![Состояние настройки хранилища](./media/backup-try-azure-backup-in-10-mins/configure-vault.png)

    c. Выберите соответствующий параметр избыточности хранилища.

    Если вы выбрали параметр **Локально избыточное**, нажмите кнопку **Сохранить**, так как по умолчанию используется параметр **Геоизбыточное**.

    ![GRS](./media/backup-try-azure-backup-in-10-mins/geo-redundant.png)

    г) На панели навигации слева щелкните **Службы восстановления**, чтобы вернуться к списку ресурсов **служб восстановления**.

    ![Выбор хранилища резервных копий](./media/backup-try-azure-backup-in-10-mins/rs-left-nav.png)

    Теперь компьютер Windows должен пройти проверку подлинности с помощью хранилища резервных копий, которое вы только что создали. Проверка подлинности выполняется с помощью учетных данных хранилища.

4.  Щелкните хранилище, которое вы только что создали.

    ![Состояние создания хранилища](./media/backup-try-azure-backup-in-10-mins/demo-vault-active-full-screen.png)

5. На странице **быстрого запуска** щелкните **Скачать учетные данные хранилища**.

    ![Загрузить](./media/backup-try-azure-backup-in-10-mins/downloadvc.png)

    Портал создаст учетные данные хранилища с помощью сочетания имени хранилища и текущей даты.

    >[AZURE.NOTE] Файл учетных данных хранилища используется только во время регистрации. Срок его действия истекает через 48 часов.

6. Щелкните **Сохранить**, чтобы скачать учетные данные хранилища в папку локальной учетной записи **Загрузки**, или выберите в меню **Сохранить** пункт **Сохранить как**, чтобы указать расположение для учетных данных хранилища.

    ![Выбор хранилища резервных копий](./media/backup-try-azure-backup-in-10-mins/save.png)

    Убедитесь, что учетные данные хранилища сохраняются в расположении, к которому у вашего компьютера есть доступ. Если файл хранится в общей папке или SMB, убедитесь, что у вас есть необходимые разрешения на доступ.

    >[AZURE.NOTE] Открывать учетные данные хранилища в данный момент не требуется.

    Теперь необходимо загрузить агент резервного копирования.

7. В области навигации слева щелкните **Службы восстановления** и выберите хранилище службы архивации, которое нужно зарегистрировать на сервере.

    ![Выбор хранилища резервных копий](./media/backup-try-azure-backup-in-10-mins/recovery-services-select-vault.png)

8. На странице быстрого запуска щелкните **Агент для Windows Server, System Center Data Protection Manager или клиента Windows > Сохранить** (либо выберите в меню **Сохранить** пункт **Сохранить как**, чтобы указать расположение для агента).

    ![Сохранить агент](./media/backup-try-azure-backup-in-10-mins/agent.png)

Вы создали хранилище резервных копий и загрузили необходимые компоненты. Теперь вы установите агент службы архивации.

## Шаг 3. Подготовка сервера Windows Server или клиента Windows путем установки и регистрации этих компонентов

1. По завершении скачивания дважды щелкните файл **MARSagentinstaller.exe** в папке, в которой вы его сохранили (или сразу щелкните **Запустить**, вместо того чтобы сохранять файл на предыдущем шаге).

2. Выберите **папку установки** и **папку кэша** для агента.

    В указанном расположении кэша должно быть свободное место, равное по крайней мере 5 % размера резервной копии данных.

    Нажмите кнопку **Далее**.

    ![Папка временных файлов и кэша](./media/backup-try-azure-backup-in-10-mins/recovery-services-agent-setup-wizard-1.png)

3. Для подключения к Интернету можно и дальше использовать настройки прокси-сервера по умолчанию. Если эти настройки нужно изменить, на экране **Конфигурация прокси-сервера** установите флажок **Использовать пользовательские настройки прокси-сервера** и введите нужные данные.

    Если вы используете проверенный прокси-сервер, введите имя пользователя и пароль.

    Нажмите кнопку **Далее**.

    ![Настройка прокси-сервера](./media/backup-try-azure-backup-in-10-mins/proxy-configuration.png)

4. Щелкните **Install** (Установить).

    ![Настройка прокси-сервера](./media/backup-try-azure-backup-in-10-mins/agent-installation.png)

    В конце установки агент службы архивации Azure установит .NET Framework 4.5 и Windows PowerShell (если эти компоненты еще не установлены).

5. После установки агента нажмите кнопку **Перейти к регистрации**, чтобы продолжить.

    ![Зарегистрировать](./media/backup-try-azure-backup-in-10-mins/register.png)

6. На экране **Идентификация хранилища** найдите и выберите ранее скачанный **файл учетных данных хранилища**. Убедитесь, что файл учетных данных хранилища находится в расположении, к которому может получить доступ программа установки.

    Нажмите кнопку **Далее**.

    ![Учетные данные хранилища](./media/backup-try-azure-backup-in-10-mins/vault-identification-with-creds.png)

7. На экране **Параметры шифрования** можно создать парольную фразу или ввести уже имеющуюся (длиной не менее 16 символов). Не забудьте сохранить парольную фразу в безопасном месте.

    > [AZURE.WARNING] Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не сможет помочь вам восстановить резервную копию данных. Парольная фраза принадлежит конечному пользователю, и корпорация Майкрософт не имеет возможности ее просматривать. Сохраните файл в безопасном месте, так как он потребуется при выполнении операции восстановления.

    Нажмите кнопку **Готово**

    ![Шифрование](./media/backup-try-azure-backup-in-10-mins/encryption.png)

    **Мастер регистрации сервера** зарегистрирует сервер в службе архивации Microsoft Azure.

    ![Шифрование](./media/backup-try-azure-backup-in-10-mins/registering-server.png)

8. Когда **ключ шифрования** будет настроен, оставьте установленным флажок **Запустить агент служб восстановления Microsoft Azure** и нажмите кнопку **Закрыть**.

    ![Шифрование](./media/backup-try-azure-backup-in-10-mins/close-server-registration.png)

    Теперь компьютер зарегистрирован в хранилище, и вы можете настроить и запланировать параметры резервного копирования.

## Шаг 4. Защита данных

1. В **агенте службы архивации** (открывается автоматически, если установлен флажок **Запустить агент служб восстановления Microsoft Azure**) щелкните элемент **Создать расписание архивации**.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/snap-in-schedule-backup-closeup.png)

2. На экране **Начало работы** нажмите кнопку **Далее**.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/getting-started.png)

3. На экране **Выбор элементов для резервного копирования** щелкните **Добавить элементы**.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/add-items.png)

    Выберите элементы для резервного копирования и нажмите кнопку **ОК**. Служба архивации Azure на Windows Server или клиенте Windows позволяет защитить файлы и папки.

    ![Элементы для архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/added-items.png)

    Нажмите кнопку **Далее**.

    ![Элементы для архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/selected-items.png)

4. Настройте **расписание резервного копирования** и нажмите кнопку **Далее**.

    Вы можете запланировать ежедневное (не более трех раз в день) или еженедельное резервное копирование.

    ![Элементы для архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/specify-backup-schedule.png)

5. Выберите **политику хранения** резервных копий. Вы можете выбрать ежедневную, еженедельную, ежемесячную или ежегодную политику хранения с учетом своих потребностей.

    >[AZURE.NOTE] Дополнительные сведения о расписании резервного копирования см. в [этой статье](backup-azure-backup-cloud-as-tape.md).

     Нажмите кнопку **Далее**.

    ![Элементы для архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/select-retention-policy.png)

6. Выберите тип начального резервного копирования.

    Резервное копирование можно выполнять по сети или в автономном режиме. В оставшейся части этой статьи описан процесс автоматического резервного копирования. Если вы предпочитаете автономное резервное копирование, см. статью [Автономное резервное копирование в службе архивации Azure](backup-azure-backup-import-export.md).

    Нажмите кнопку **Далее**.

    ![Начальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/choose-initial-backup-type.png)

7. На экране **Подтверждение** проверьте информацию и нажмите кнопку **Готово**.

    ![Начальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/confirmation.png)

8. Когда мастер завершит создавать **расписание резервного копирования**, нажмите кнопку **Закрыть**.

    ![Начальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/backup-schedule-created.png)

9. В **агенте службы архивации** нажмите кнопку **Выполнить моментальную архивацию**, чтобы завершить начальное заполнение по сети.

    ![Моментальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/snap-in-backup-now.png)

10. На экране **Подтверждение** проверьте параметры, которые мастер будет использовать для резервного копирования данных на компьютере, и нажмите кнопку **Создать резервную копию**.

    ![Моментальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/backup-now-confirmation.png)

11. Нажмите кнопку **Закрыть**, чтобы закрыть мастер. Вы можете закрыть мастер до завершения **резервного копирования**. Мастер продолжит работу в фоновом режиме.

    ![Моментальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/backup-progress.png)

12. После завершения начального резервного копирования в консоли службы архивации Azure в представлении **Задания** отобразится состояние "Задание выполнено".

    ![Первоначальное восстановление завершено](./media/backup-try-azure-backup-in-10-mins/ircomplete.png)

Поздравляем, вы успешно выполнили резервное копирование своих файлов и папок в службу архивации Azure.

# Первое знакомство: резервное копирование файлов и папок из Windows Server или клиента Windows в Azure

В этой статье описывается, как выполнить резервное копирование файлов и папок Windows Server или рабочей станции Windows в Azure с помощью службы архивации Azure. В этом руководстве приведены общие сведения, и с помощью него вы сможете опробовать службу архивации Azure.

Дополнительные сведения о службе архивации Azure см. в этом [обзоре](backup-introduction-to-azure-backup.md).

Чтобы создать резервную копию файлов и папок, нужно сделать следующее:

![Шаг 1](./media/backup-try-azure-backup-in-10-mins/step-1.png) Оформить подписку Azure (при необходимости).<br> ![Шаг 2](./media/backup-try-azure-backup-in-10-mins/step-2.png) Создать хранилище службы архивации и скачать необходимые компоненты.<br> ![Шаг 3.](./media/backup-try-azure-backup-in-10-mins/step-3.png) Установить и зарегистрировать агент службы архивации.<br> ![Шаг 4.](./media/backup-try-azure-backup-in-10-mins/step-4.png) Выполнить резервное копирование файлов и папок.


![Резервное копирование компьютеров Windows с помощью службы архивации Azure](./media/backup-try-azure-backup-in-10-mins/windows-machine-backup-process.png)

## Шаг 1. Оформление подписки Azure

Если у вас нет подписки Azure, вы можете создать [бесплатную учетную запись](https://azure.microsoft.com/free/), которая предоставляет доступ ко всем службам Azure.

## Шаг 2. Создание хранилища службы архивации и скачивание необходимых компонентов

Для резервного копирования файлов и папок нужно создать хранилище архивации в том регионе, в котором вы планируете хранить данные. Вам также необходимо определить параметры репликации хранилища и скачать учетные данные и агент службы архивации.

### Создание хранилища службы архивации

1. Используя подписку Azure, войдите на [портал Azure](https://portal.azure.com/) (если вы еще этого не сделали).

2. Последовательно выберите **Создать > Гибридная интеграция > Резервная копия**.

    ![Подготовка к резервному копированию файлов и папок](./media/backup-try-azure-backup-in-10-mins/second-blade-backup.png)

3. В поле **Имя** введите понятное имя, чтобы идентифицировать хранилище службы архивации.

4. В поле **Регион** выберите регион, который расположен к вам ближе всего. Таким образом файлы будут передаваться быстрее.

5. Щелкните **СОЗДАТЬ ХРАНИЛИЩЕ**.

    ![Создание хранилища](./media/backup-try-azure-backup-in-10-mins/demo-vault-name.png)

    Созданное хранилище будет отображаться в списке ресурсов служб восстановления с состоянием **Активно**.

    ![Хранилище в активном состоянии](./media/backup-try-azure-backup-in-10-mins/recovery-services-select-vault.png)

После этого необходимо выбрать способ репликации хранилища.

>[AZURE.NOTE] Лучше выбирать параметры репликации хранилища сразу после его создания и до регистрации в нем компьютеров. После регистрации компонента в хранилище параметр репликации хранилища блокируется и впоследствии не может быть изменен.

### Выбор параметров репликации хранилища

1. Щелкните хранилище, которое вы только что создали.
2. На странице "Быстрый запуск" щелкните **Настроить**.

    ![Настройка хранилища](./media/backup-try-azure-backup-in-10-mins/configure-vault.png)

3. Выберите соответствующий тип для хранилища.

    Если вы используете Azure в качестве основного хранилища архивации, выберите параметр [Геоизбыточное](../storage/storage-redundancy.md#geo-redundant-storage). Если вы используете Azure в качестве третичного хранилища архивации, выберите параметр [Локально избыточное](../storage/storage-redundancy.md#locally-redundant-storage).

    ![Выбор параметров репликации хранилища](./media/backup-try-azure-backup-in-10-mins/geo-redundant.png)

4. Если вы выбрали параметр **Локально избыточное**, нажмите кнопку **Сохранить**, так как по умолчанию используется параметр **Геоизбыточное**.

Проверка подлинности компьютера в хранилище службы архивации выполняется с помощью учетных данных хранилища. В разделе ниже описано, как скачать эти учетные данные.

### Скачивание учетных данных хранилища
Файл учетных данных хранилища используется только во время регистрации. Срок его действия истекает через 48 часов.

1. Чтобы вернуться на страницу **Быстрый запуск** для своего хранилища, щелкните ![Выбор нового хранилища](./media/backup-try-azure-backup-in-10-mins/quick-start-icon.png).

2. Щелкните **Скачать учетные данные хранилища > Сохранить**.

Теперь необходимо скачать агент службы архивации.

### Скачивание агента службы архивации

Щелкните **Агент для Windows Server, System Center Data Protection Manager или клиента Windows > Сохранить**.

![Сохранение агента службы архивации](./media/backup-try-azure-backup-in-10-mins/agent.png)

Вы создали хранилище и скачали необходимые компоненты. Теперь нужно установить и зарегистрировать агент службы архивации.

## Шаг 3. Установка и регистрация агента службы архивации

1. Дважды щелкните файл **MARSagentinstaller.exe** в папке, куда он был сохранен.
2. Выполните шаги мастера настройки агента служб восстановления Microsoft Azure. Для этого необходимо:
    - Выбрать папку установки и папку кэша.
    - Если для подключения к Интернету используется прокси-сервер, указать данные прокси-сервера.
    - Если используется прокси-сервер, прошедший проверку подлинности, ввести имя пользователя и пароль.
    - Сохранить парольную фразу шифрования в безопасном месте.

    >[AZURE.NOTE] Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не сможет помочь вам восстановить резервную копию данных. Сохраните файл в безопасном месте, так как он потребуется для восстановления резервной копии.

Теперь, когда агент установлен и компьютер зарегистрирован в хранилище, вы можете настроить параметры резервного копирования, в том числе расписание.

## Шаг 4. Резервное копирование файлов и папок
Чтобы открыть агент службы архивации, найдите его, введя "Служба архивации Microsoft Azure".

1. В **агенте службы архивации** щелкните **Планирование архивации**.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/snap-in-schedule-backup-closeup.png)

2. Выполните шаги мастера планирования архивации. Для этого необходимо:

    - Выбрать файлы и папки, которые вы хотите архивировать.
    - Указать расписание архивации (ежедневно или еженедельно).
    - Определить политику хранения.
    - Выбрать способ выполнения для начальной архивации (по сети или в автономном режиме).

    Дополнительные сведения о завершении автономной начальной архивации см. [здесь](backup-azure-backup-import-export.md). <br><br>

3. После выполнения шагов мастера в **агенте службы архивации** нажмите кнопку **Архивировать**, чтобы завершить начальную архивацию по сети.

    ![Моментальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/backup-now.png)

4. На экране **подтверждения** нажмите кнопку **Архивировать**. Вы можете закрыть мастер до завершения архивации. Тогда он продолжит работу в фоновом режиме.

    Когда начальная архивация будет завершена, в консоли в представлении **Задания** отобразится состояние "Задание выполнено".

    ![Завершение начального резервного копирования](./media/backup-try-azure-backup-in-10-mins/ircomplete.png)

Поздравляем, вы успешно выполнили резервное копирование своих файлов и папок в службу архивации Azure.

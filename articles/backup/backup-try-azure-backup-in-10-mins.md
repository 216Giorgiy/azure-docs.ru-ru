# Первое знакомство: резервное копирование файлов и папок из Windows Server или клиента Windows в Azure

В этой статье описывается, как выполнить резервное копирование файлов и папок Windows Server или рабочей станции Windows в Azure с помощью службы архивации Azure. Вы увидите, что использовать службу архивации Azure на самом деле просто.

Чтобы создать резервную копию файлов и папок, нужно выполнить следующие действия.

![Шаг 1](./media/backup-try-azure-backup-in-10-mins/step-1.png) Оформить подписку Azure (при необходимости).<br> ![Шаг 2](./media/backup-try-azure-backup-in-10-mins/step-2.png) Создать хранилище службы архивации и скачать необходимые компоненты.<br> ![Шаг 3.](./media/backup-try-azure-backup-in-10-mins/step-3.png) Подготовить сервер Windows Server или клиент Windows, установив и зарегистрировав эти компоненты.<br> ![Шаг 4.](./media/backup-try-azure-backup-in-10-mins/step-4.png) Создать резервную копию данных.


![Резервное копирование компьютеров Windows с помощью службы архивации Azure](./media/backup-try-azure-backup-in-10-mins/windows-machine-backup-process.png)

## Шаг 1. Оформление подписки Azure

Если у вас нет подписки Azure, вы можете создать [бесплатную учетную запись](https://azure.microsoft.com/free/), которая предоставляет доступ ко всем службам Azure.

>[AZURE.NOTE] Если у вас уже есть подписка Azure, этот шаг можно пропустить.

## Шаг 2. Создание хранилища службы архивации и загрузка необходимых компонентов.

Для резервного копирования файлов и папок вам нужно создать хранилище архивации в том географическом регионе, в котором вы планируете хранить данные.

1. Используя подписку Azure, войдите на [портал Azure](https://portal.azure.com/), если вы еще этого не сделали.

2. Щелкните элементы **Создать > Гибридная интеграция > Резервная копия**.

    ![Подготовка к резервному копированию файлов и папок](./media/backup-try-azure-backup-in-10-mins/second-blade-backup.png)

    а. На появившемся экране в поле **Имя** введите понятное имя для идентификации хранилища службы архивации. Введите имя длиной от 2 до 50 знаков. Имя должно начинаться с буквы, оно может содержать только буквы, цифры и дефисы.

    b. В поле **Регион** выберите географический регион для хранилища службы архивации. Если вы выберите географический регион, который находится недалеко от вашего расположения, вы уменьшите задержки в сети во время резервного копирования в Azure.

    c. Щелкните **Создать хранилище**.

    ![Создание хранилища](./media/backup-try-azure-backup-in-10-mins/demo-vault-name.png)

    Для проверки состояния отслеживайте уведомления в нижней части портала.

    ![Процесс создания хранилища](./media/backup-try-azure-backup-in-10-mins/creatingvault1.png)

    Созданное хранилище будет отображаться в списке ресурсов служб восстановления со статусом **Активно**.

    ![Хранилище в активном состоянии](./media/backup-try-azure-backup-in-10-mins/recovery-services-select-vault.png)

3. Выберите параметры **избыточности хранилища**.

    Лучше выбирать параметры избыточности хранилища сразу после его создания и до регистрации в нем компьютеров. После регистрации элемента в хранилище параметр избыточности хранилища блокируется и в последствии не может быть изменен.

    Если вы используете Azure в качестве конечной точки основного хранилища службы архивации, мы рекомендуем выбрать [Геоизбыточное хранилище](../storage/storage-redundancy.md#geo-redundant-storage) (параметр по умолчанию).

    Если вы используете Azure в качестве конечной точки третичного хранилища службы архивации, выберите [Локально избыточное хранилище](../storage/storage-redundancy.md#locally-redundant-storage). Это снизит затраты на хранение данных в Azure, но при этом также снизится уровень устойчивости данных, что может быть допустимо для копий третьего уровня.

    а. Щелкните хранилище, которое вы только что создали.

    b. На странице быстрого запуска щелкните **Настроить**.

    ![Настройка хранилища](./media/backup-try-azure-backup-in-10-mins/configure-vault.png)

    c. Выберите соответствующий параметр избыточности хранилища.

    ![Выбор варианта избыточности для хранилища](./media/backup-try-azure-backup-in-10-mins/geo-redundant.png)

    Если вы выбрали параметр **Локально избыточное**, нажмите кнопку **Сохранить**, так как по умолчанию используется параметр **Геоизбыточное**.

    г) На панели навигации слева щелкните **Службы восстановления**, чтобы вернуться к списку ресурсов **служб восстановления**.

    ![Службы восстановления](./media/backup-try-azure-backup-in-10-mins/rs-left-nav.png)

    Теперь компьютер Windows должен пройти проверку подлинности с использованием хранилища резервных копий, которое вы только что создали. Проверка подлинности выполняется с помощью учетных данных хранилища.

4.  Щелкните хранилище, которое вы только что создали.

    ![Выбор нового хранилища](./media/backup-try-azure-backup-in-10-mins/demo-vault-active-full-screen.png)

5. На странице **быстрого запуска** щелкните **Скачать учетные данные хранилища**.

    ![Скачивание учетных данных хранилища](./media/backup-try-azure-backup-in-10-mins/downloadvc.png)

    Портал создаст учетные данные хранилища, используя сочетание имени хранилища и текущей даты.

    >[AZURE.NOTE] Файл учетных данных хранилища используется только во время регистрации. Срок его действия истекает через 48 часов.

6. Щелкните **Сохранить**, чтобы скачать учетные данные хранилища в локальную папку **загрузок**. В меню **Сохранить** можно также выбрать пункт **Сохранить как**, чтобы указать расположение учетных данных хранилища.

    Убедитесь, что учетные данные хранилища сохраняются в расположении, к которому у вашего компьютера есть доступ. Если файл хранится в общей папке или SMB, убедитесь, что у вас есть необходимые разрешения на доступ.

    >[AZURE.NOTE] Открывать учетные данные хранилища сейчас не нужно.

    Теперь необходимо загрузить агент резервного копирования.

7. В области навигации слева щелкните **Службы восстановления** и выберите хранилище службы архивации, которое нужно зарегистрировать на сервере.

    ![Выбор хранилища резервных копий](./media/backup-try-azure-backup-in-10-mins/recovery-services-select-vault.png)

8. На странице быстрого запуска щелкните **Агент для Windows Server, System Center Data Protection Manager или клиент Windows > Сохранить**.

    ![Сохранение агента службы архивации](./media/backup-try-azure-backup-in-10-mins/agent.png)

Вы создали хранилище резервных копий и загрузили необходимые компоненты. Теперь нужно установить агент службы архивации.

## Шаг 3. Подготовка сервера Windows Server или рабочей станции Windows путем установки и регистрации этих компонентов

1. По завершении скачивания дважды щелкните файл **MARSagentinstaller.exe** в целевой папке (или сразу щелкните **Запустить**, чтобы не сохранять файл на предыдущем шаге).

2. Выберите **папку установки** и **папку кэша** для агента.

    В указанном расположении кэша должно быть свободное место, равное по крайней мере 5 % размера резервной копии данных.

    Нажмите кнопку **Далее**.

    ![Установка и папка кэша](./media/backup-try-azure-backup-in-10-mins/recovery-services-agent-setup-wizard-1.png)

3. (Необязательное действие) При использовании прокси-сервера для подключения к Интернету на экране **Конфигурация прокси-сервера** установите флажок **Использовать настраиваемые параметры прокси-сервера** и введите данные прокси-сервера.

    Если вы используете проверенный прокси-сервер, введите имя пользователя и пароль.

    Нажмите кнопку **Далее**.

    ![Настройка прокси-сервера](./media/backup-try-azure-backup-in-10-mins/proxy-configuration.png)

4. Щелкните **Install** (Установить).

    В конце установки агент службы архивации Azure установит .NET Framework 4.5 и Windows PowerShell (если эти компоненты еще не установлены).

5. После установки агента нажмите кнопку **Перейти к регистрации**, чтобы продолжить.

    ![Зарегистрировать](./media/backup-try-azure-backup-in-10-mins/register.png)

6. В разделе **Идентификация хранилища** найдите и выберите ранее скачанный **файл учетных данных хранилища**. Убедитесь, что файл учетных данных хранилища находится в расположении, к которому может получить доступ программа установки.

    Нажмите кнопку **Далее**.

    ![Определение хранилища](./media/backup-try-azure-backup-in-10-mins/vault-identification-with-creds.png)

7. На экране **Параметры шифрования** можно создать парольную фразу или ввести уже имеющуюся (длиной не менее 16 символов). Не забудьте сохранить парольную фразу в безопасном месте.

    > [AZURE.WARNING] Если вы потеряли или забыли парольную фразу, корпорация Майкрософт не поможет вам восстановить резервную копию данных. Парольная фраза принадлежит пользователю, и корпорация Майкрософт не имеет возможности ее просматривать. Сохраните файл в безопасном месте, так как он потребуется при выполнении операции восстановления.

    Нажмите кнопку **Готово**

    ![Параметры шифрования](./media/backup-try-azure-backup-in-10-mins/encryption.png)

    **Мастер регистрации сервера** зарегистрирует сервер в службе архивации Microsoft Azure.

8. Когда **ключ шифрования** будет настроен, оставьте установленным флажок **Запустить агент служб восстановления Microsoft Azure** и нажмите кнопку **Закрыть**.

    Теперь компьютер зарегистрирован в хранилище, и вы можете настроить и запланировать параметры резервного копирования.

## Шаг 4. Резервное копирование данных

1. В **агенте службы архивации** (открывается автоматически, если установлен флажок **Запустить агент служб восстановления Microsoft Azure**) щелкните элемент **Создать расписание архивации**.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/snap-in-schedule-backup-closeup.png)

2. На экране **Начало работы** нажмите кнопку **Далее**.

3. На экране **Выбор элементов для архивации** щелкните **Добавить элементы**. Служба архивации Azure на Windows Server или рабочей станции Windows позволяет защитить файлы и папки.

    ![Планирование архивации Windows Server](./media/backup-try-azure-backup-in-10-mins/add-items.png)

    Выберите элементы для резервного копирования и нажмите кнопку **ОК**.

    Нажмите кнопку **Далее**.

4. Настройте **расписание резервного копирования** и нажмите кнопку **Далее**.

    Вы можете запланировать ежедневное (не более трех раз в день) или еженедельное резервное копирование.

    ![Выбор расписания резервного копирования](./media/backup-try-azure-backup-in-10-mins/specify-backup-schedule.png)

5. Выберите **политику хранения** резервных копий. Вы можете выбрать ежедневную, еженедельную, ежемесячную или ежегодную политику хранения с учетом своих потребностей.

    >[AZURE.NOTE] Дополнительные сведения о расписании резервного копирования см. в [этой статье](backup-azure-backup-cloud-as-tape.md).

     Нажмите кнопку **Далее**.

    ![Выбор политики хранения](./media/backup-try-azure-backup-in-10-mins/select-retention-policy.png)

6. Выберите тип начального резервного копирования.

    Резервное копирование можно выполнять по сети или в автономном режиме. В оставшейся части этой статьи описан процесс автоматического резервного копирования. Если вы предпочитаете автономное резервное копирование, см. статью [Автономное резервное копирование в службе архивации Azure](backup-azure-backup-import-export.md).

    Нажмите кнопку **Далее**.

    ![Тип начального резервного копирования](./media/backup-try-azure-backup-in-10-mins/choose-initial-backup-type.png)

7. Проверьте сведения в окне **Подтверждение** и нажмите кнопку **Готово**.

8. Когда мастер завершит создание **расписания резервного копирования**, нажмите кнопку **Закрыть**.

9. В **агенте службы архивации** нажмите кнопку **Архивировать**, чтобы завершить начальное заполнение по сети.

    ![Моментальная архивация Windows Server](./media/backup-try-azure-backup-in-10-mins/snap-in-backup-now.png)

10. На экране **Подтверждение** проверьте параметры, которые мастер будет использовать для резервного копирования данных на компьютере, и нажмите кнопку **Архивировать**.

11. Нажмите кнопку **Закрыть**, чтобы закрыть мастер. Вы можете закрыть мастер до завершения резервного копирования. Мастер продолжит работу в фоновом режиме.

12. Когда начальное резервное копирование будет завершено, в консоли службы архивации Azure в представлении **Задания** отобразится состояние "Задание выполнено".

    ![Завершение начального резервного копирования](./media/backup-try-azure-backup-in-10-mins/ircomplete.png)

Поздравляем, вы успешно выполнили резервное копирование своих файлов и папок в службу архивации Azure.

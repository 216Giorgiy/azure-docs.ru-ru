# Резервное копирование виртуальных машин Azure
В этой статье описываются процедуры резервного копирования существующих виртуальных машин Azure для их защиты в соответствии с корпоративными политиками резервного копирования и хранения.

Есть несколько моментов, которые необходимо уладить до выполнения резервного копирования виртуальной машины Azure. Удовлетворите [необходимые требования](backup-azure-vms-prepare.md) (если вы еще не сделали это), чтобы подготовить среду к резервному копированию виртуальных машин.

Если вам нужна информация о [планировании инфраструктуры резервного копирования виртуальных машин в Azure](backup-azure-vms-introduction.md) или о [виртуальных машинах Azure](https://azure.microsoft.com/documentation/services/virtual-machines/), перейдите по следующим ссылкам.

Резервное копирование виртуальных машин Azure состоит из трех основных шагов.

![Три этапа резервного копирования виртуальной машины Azure](./media/backup-azure-vms/3-steps-for-backup.png)

>[AZURE.NOTE]Резервное копирование виртуальных машин выполняется локально. Создать резервную копию виртуальных машин из одного региона в хранилище службы архивации в другом регионе нельзя. Поэтому для каждого региона Azure с виртуальными машинами, для которых требуется резервное копирование, необходимо создать как минимум одно хранилище службы архивации в этом регионе.

## Шаг 1. Обнаружение виртуальных машин Azure
Сначала следует запустить процесс обнаружения, чтобы убедиться в идентификации всех недавно добавленных в подписку виртуальных машин. В ходе этого процесса в Azure отправляется запрос на список виртуальных машин в подписке, а также на дополнительные сведения, включая имя облачной службы и регион.

1. Перейдите на портале Azure в хранилище службы архивации в разделе **Службы восстановления**, а затем откройте вкладку **Зарегистрированные элементы**.

2. В раскрывающемся меню выберите пункт **Виртуальная машина Azure**.

    ![выбор рабочей нагрузки](./media/backup-azure-vms/discovery-select-workload.png)

3. Щелкните **ОБНАРУЖИТЬ** в нижней части страницы.![кнопка обнаружения](./media/backup-azure-vms/discover-button-only.png)

    Процесс обнаружения может длиться несколько минут, в течение которых будет создаваться таблица со списком виртуальных машин. После запуска процесса в нижней части экрана отобразится уведомление о ходе его выполнения.

    ![обнаружение виртуальных машин](./media/backup-azure-vms/discovering-vms.png)

    Уведомление изменится, когда процесс завершится.

    ![обнаружение выполнено](./media/backup-azure-vms/discovery-complete.png)

##  Шаг 2. Регистрация виртуальных машин Azure
Чтобы связать виртуальную машину со службой резервного копирования Azure, выполняется ее регистрация. Как правило, это разовая операция.

1. Перейдите на портале Azure в хранилище службы архивации в разделе **Службы восстановления**, а затем откройте вкладку **Зарегистрированные элементы**.

2. В раскрывающемся меню выберите пункт **Виртуальная машина Azure**.

    ![выбор рабочей нагрузки](./media/backup-azure-vms/discovery-select-workload.png)

3. В нижней части страницы нажмите кнопку **ЗАРЕГИСТРИРОВАТЬ**. ![кнопка регистрации](./media/backup-azure-vms/register-button-only.png)

4. В контекстном меню **Регистрация элементов** выберите виртуальные машины, которые нужно зарегистрировать. Если в меню отображаются виртуальные машины с одинаковыми именами, используйте облачную службу, чтобы их различить.

    >[AZURE.TIP]В ходе одной процедуры можно зарегистрировать несколько виртуальных машин.

    Для каждой выбранной виртуальной машины будет создано задание.

5. В уведомлении щелкните ссылку **Просмотр задания**, чтобы перейти на страницу **Задания**.

    ![задание регистрации](./media/backup-azure-vms/register-create-job.png)

   Виртуальная машина отображается в списке зарегистрированных элементов с указанием состояния операции регистрации.

    ![Registering status 1](./media/backup-azure-vms/register-status01.png)

    When the operation completes, the status will change to reflect the *registered* state.

    ![Registration status 2](./media/backup-azure-vms/register-status02.png)

## Шаг 3. Защита виртуальных машин Azure
Теперь вы можете настроить для виртуальной машины политику резервного копирования и хранения. Одним действием можно защитить несколько виртуальных машин.

Хранилища службы архивации Azure, созданные после мая 2015 г., доступны со встроенной политикой по умолчанию. Эта политика определяет не только стандартный срок хранения на 30 дней, но и расписание ежедневного резервного копирования.

1. Перейдите в хранилище службы архивации в разделе **Службы восстановления** на портале Azure, а затем откройте вкладку **Зарегистрированные элементы**.
2. В раскрывающемся меню выберите пункт **Виртуальная машина Azure**.

    ![Выберите рабочую нагрузку на портале](./media/backup-azure-vms/select-workload.png)

3. В нижней части страницы щелкните **ЗАЩИТИТЬ**.

    Откроется **Мастер защиты элементов**. В этом мастере перечислены только незарегистрированные и незащищенные виртуальные машины. Выберите виртуальные машины, которые необходимо защитить.

    При наличии двух или нескольких виртуальных машин с одинаковым именем используйте облачную службу, чтобы их различать.

    >[AZURE.TIP]Можно защитить несколько виртуальных машин одновременно.

    ![Настройка защиты нужного уровня](./media/backup-azure-vms/protect-at-scale.png)

4. Выберите **расписание резервного копирования**, чтобы создать резервную копию выбранных виртуальных машин. Выберите политику из существующего набора или создайте новую.

    У каждой политики резервного копирования может быть несколько связанных виртуальных машин. Но в определенный момент времени виртуальная машина может быть связана только с одной политикой.

    ![Защита с помощью новой политики](./media/backup-azure-vms/policy-schedule.png)

    >[AZURE.NOTE]Политика резервного копирования включает схему хранения плановых резервных копий. Выбрав существующую политику резервного копирования, вы не сможете впоследствии изменить параметры хранения.

5. Выберите **диапазон хранения**, который нужно связать с резервными копиями.

    ![Защита с помощью гибких возможностей хранения](./media/backup-azure-vms/policy-retention.png)

    Политика хранения определяет продолжительность хранения резервной копии. Вы можете указывать разные политики хранения в зависимости от времени создания резервной копии. Например, для точки резервного копирования, которая создается в конце каждого квартала, может потребоваться более продолжительное хранения (для проведения аудита). При этом точку резервного копирования, создаваемую ежедневно (для оперативного восстановления), можно хранить 90 дней.

    ![Резервное копирование виртуальной машины с точкой восстановления](./media/backup-azure-vms/long-term-retention.png)

    Описание политик, приведенных на изображении выше:

    - **Политика хранения ежедневных резервных копий**. Резервные копии, создаваемые ежедневно, хранятся в течение 30 дней.
    - **Политика хранения еженедельных резервных копий**. Резервные копии, создаваемые каждое воскресенье, хранятся в течение 104 недель.
    - **Политика хранения ежемесячных резервных копий**. Резервные копии, создаваемые в последнее воскресенье каждого месяца, хранятся в течение 120 месяцев.
    - **Политика хранения ежегодных резервных копий**. Резервные копии, создаваемые каждое первое воскресенье января, хранятся в течение 99 лет.

    Чтобы настроить политику защиты и связать ее с виртуальными машинами, для каждой выбранной виртуальной машины создается задание.

6. Щелкните вкладку **Задание** и выберите нужный фильтр, чтобы просмотреть список заданий **Настройка защиты**.

    ![Настройка задания защиты](./media/backup-azure-vms/protect-configureprotection.png)

## Начальное резервное копирование
Виртуальная машина, защищенная политикой, появится на вкладке **Защищенные элементы** со статусом *Защищено (в ожидании начального резервного копирования)*. По умолчанию *начальным резервным копированием* является первое запланированное резервное копирование.

Чтобы активировать начальное резервное копирование сразу же после настройки, выполните следующие действия.

1. Нажмите кнопку **Создать резервную копию** в нижней части страницы **Защищенные элементы**.

    Служба архивации Azure создаст задание резервного копирования для операции начального резервного копирования.

2. Чтобы просмотреть список заданий, щелкните вкладку **Задания**.

    ![Выполняется резервное копирование](./media/backup-azure-vms/protect-inprogress.png)

>[AZURE.NOTE]В ходе резервного копирования служба архивации Azure дает команду расширению для резервного копирования на каждой виртуальной машине сохранять на диск все данные операций записи и делать согласованный моментальный снимок.

Когда начальное резервное копирование будет завершено, на вкладке **Защищенные элементы** отобразится состояние виртуальной машины *Защищено*.

![Резервное копирование виртуальной машины с точкой восстановления](./media/backup-azure-vms/protect-backedupvm.png)

## Просмотр состояния резервного копирования и подробностей
После обеспечения защиты число виртуальных машин также увеличивается на странице сводки **Панели мониторинга**. На странице **Панель мониторинга** отображается количество *успешных*, *завершившихся сбоем* и *выполняющихся* заданий за последние 24 часа. Если вы щелкните одну из категорий, для нее на странице **Задания** отобразятся подобные сведения.

![Состояние резервного копирования на странице панели мониторинга](./media/backup-azure-vms/dashboard-protectedvms.png)

Значения на панели мониторинга обновляются каждые 24 часа.

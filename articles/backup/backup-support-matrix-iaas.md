# <a name="support-matrix-for-azure-vm-backup"></a>Матрица поддержки для резервного копирования виртуальных машин Azure
С помощью [службы Azure Backup](backup-overview.md) можно выполнять резервное копирование локальных компьютеров, рабочих нагрузок и виртуальных машин Azure. В этой статье приведены параметры поддержки и ограничения, относящиеся к резервному копированию виртуальных машин Azure с помощью службы Azure Backup.

Другие матрицы поддержки:

- [Общая матрица поддержки](backup-support-matrix.md) для Azure Backup.
- [Матрица поддержки](backup-support-matrix-mabs-dpm.md) для резервного копирования с помощью Microsoft Azure Backup Server или DPM.
- [Матрица поддержки](backup-support-matrix-mars-agent.md) для резервного копирования с использованием агента MARS.



## <a name="supported-scenarios"></a>Поддерживаемые сценарии использования.

Вот как вы можете выполнить резервное копирование и восстановление виртуальных машин с помощью службы Azure Backup.


**Сценарий** | **Архивация** | **Агент** |**Restore** 
--- | --- | --- | ---
**Прямое резервное копирование виртуальных машин Azure** | Резервное копирование всей виртуальной машины  | Агент на виртуальной машине Azure не требуется. Azure Backup устанавливает расширение для [агента виртуальной машины Azure](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-windows), который работает на этой виртуальной машине, и использует его. | Восстановление состоит из следующих шагов.<br/><br/> **Создание базовой виртуальной машины**. Это полезно, если на виртуальной машине нет специальной конфигурации, например несколько IP-адресов.<br/><br/> **Восстановление диска виртуальной машины**. Восстановите диск. Затем присоедините его к существующей виртуальной машине или создайте новую виртуальную машину с диска с помощью PowerShell.<br/><br/> **Замена диска виртуальной машины**. Если виртуальная машина существует и использует управляемые диски (без шифрования), вы можете восстановить диск и заменить ним существующий диск на виртуальной машине.<br/><br/> **Восстановление определенных файлов или папок**. Вы можете восстановить файлы и папки из виртуальной машины, а не всю виртуальную машину.  
**Прямое резервное копирование виртуальных машин Azure (только для Windows)** | Создание резервной копии определенных файлов, папок или тома | Установите [агент Служб восстановления Microsoft Azure (MARS)](backup-azure-file-folder-backup-faq.md).<br/><br/> Агент MARS можно запустить вместе с расширением резервного копирования для агента виртуальной машины Azure, чтобы выполнить резервное копирование виртуальной машины на уровне файла или папки. | Восстановление определенных папок или файлов.
**Резервное копирование виртуальной машины Azure на сервер резервного копирования** |  Резервное копирование файлов, папок или томов, файлов состояния системы или исходного состояния, данных приложений в System Center DPM или на сервер Microsoft Azure Backup Server (MAB).<br/><br/> Затем MABS или DPM создает резервную копию в хранилище резервного копирования | Установите агент защиты MABS или DPM на виртуальной машине. Агент MARS установлен на DPM или MABS.| Восстановление файлов, папок и томов; файлов состояния системы и исходного состояния; данных приложений. 

Дополнительные сведения о создании резервных копий с использованием сервера резервного копирования (backup-architecture.md#architecture-back-up-to-dpmmabs) и [требованиях к поддержке](backup-support-matrix-mabs-dpm.md).


## <a name="supported-backup-actions"></a>Поддерживаемые действия резервного копирования

**Действие** | **Поддержка**
--- | ---
Включение резервного копирования при создании виртуальной машины Azure Windows | Поддерживается для:  Windows Server 2016 (Datacenter/Datacenter Core); Windows Server 2012 R2 DataCenter; Windows Server 2008 R2 с пакетом обновления 1
Включение резервного копирования при создании виртуальной машины Linux | Поддерживается для:<br/><br/> — Ubuntu Server: 1710, 1704, 1604 (LTS), 1404 (LTS);<br/><br/> — Red Hat: RHEL 6.7, 6.8, 6.9, 7.2, 7.3, 7.4;<br/><br/> — SUSE Linux Enterprise Server: 11 с пакетом обновления 4 (SP4), 12 с пакетом обновления 2 (SP2), 12 с пакетом обновления 3 (SP3);<br/><br/> — Debian: 8, 9<br/><br/> — CentOS: 6.9, 7.3;<br/><br/> Oracle Linux: 6.7, 6.8, 6.9, 7.2, 7.3.
Резервное копирование виртуальной машины, которая отключена, находится вне сети, для которой выполняется поиск |  Поддерживается.<br/><br/> Можно создать только отказоустойчивый моментальный снимок, но не согласованный на уровне приложений.
Резервное копирование дисков после перехода на управляемые диски |  Поддерживается.<br/><br/> Резервное копирование продолжит работу. Никаких действий не требуется.
Резервное копирование управляемых дисков после включения блокировки группы ресурсов | Не поддерживается.<br/><br/> Резервное копирование не может удалить старые точки ресурсов. Сбои в резервном копировании начинаются по достижении максимального предельного числа точек восстановления.
Изменение политики резервного копирования для виртуальной машины |  Поддерживается.<br/><br/> Резервная копия виртуальной машины создается в соответствии с параметрами расписания и хранения в новой политике. Если срок хранения увеличен, существующие точки восстановления будут помечены и сохранены. Если срок хранения уменьшен, существующие точки восстановления будут урезаны во время следующего задания очистки, а затем удалены.
Отмена задания резервного копирования | Поддерживается во время процесса создания моментального снимка.<br/><br/> Не поддерживается, если моментальный снимок переносится в хранилище. 
Резервное копирование виртуальной машины в другой регион или подписку |  Не поддерживается.
Операций резервного копирования в день (с использованием расширения виртуальных машин Azure) | Одно запланированное резервное копирование в день.<br/><br/> Можно выполнить до четырех операций резервного копирования по запросу в день.
Операций резервного копирования в день (с использованием агента MARS) | Три запланированных операции резервного копирования в день. 
Операций резервного копирования в день (с использованием DPM или MABS) | Две запланированных операции резервного копирования в день.
Ежемесячное или ежегодное резервное копирование   | Не поддерживается при создании резервных копий с использованием расширения виртуальной машины Azure. Поддерживается только ежедневное и еженедельное.<br/><br/> Вы можете настроить политику, чтобы хранить ежедневные или еженедельные резервные копии в течение месяца или года.
Автоматическая настройка часов | Не поддерживается.<br/><br/> Azure Backup не учитывает настройку автоматического перехода на летнее время для виртуальной машины, для которой создается резервная копия.<br/><br/>  Измените политику вручную, если это необходимо.
[Функции безопасности для гибридных резервных копий](https://docs.microsoft.com/azure/backup/backup-azure-security-feature) |  Отключение функций не поддерживается.

## <a name="operating-system-support-linux"></a>Поддержка операционной системы (Linux)

Для резервного копирования виртуальных машин Azure под управлением Linux служба Azure Backup поддерживает весь [список дистрибутивов Linux, рекомендуемых для использования в Azure](https://docs.microsoft.com/azure/virtual-machines/linux/endorsed-distros). Обратите внимание на следующее.

- Azure Backup не поддерживает CoreOS Linux.
- Azure Backup не поддерживает 32-разрядные операционные системы.
- Вы можете использовать и другие дистрибутивы Linux, если на них установлен [агент виртуальной машины для Linux](https://docs.microsoft.com/azure/virtual-machines/extensions/agent-linux) и включена поддержка Python.

Максимальное количество точек восстановления на защищенный экземпляр (компьютер или рабочую нагрузку) | 9999

Максимальная частота резервного копирования в хранилище (расширение виртуальной машины Azure) | один раз в день
Максимальная частота резервного копирования в хранилище (агент MARS) | Три операции резервного копирования в день.
Максимальная частота резервного копирования в DPM или MABS | Каждые 15 минут для SQL Server<br/><br/> Раз в час для других рабочих нагрузок.
Период удержания точки восстановления | Ежедневно, еженедельно, ежемесячно, ежегодно.
Максимальный срок хранения | Зависит от частоты резервного копирования.
Точки восстановления на диске DPM или MABS | 64 для файловых серверов, 448 для серверов приложений.<br/><br/> Точки восстановления ленты не имеют ограничений для локального DPM.


## <a name="supported-restore-methods"></a>Поддерживаемые методы восстановления

**Метод восстановления** | **Дополнительные сведения**
--- | ---
**Создание новой виртуальной машины** | Виртуальную машину можно создать во время восстановления. <br/><br/> Выбрав этот вариант, вы получите базовую виртуальную машину, запущенную и готовую к работе. Вы можете указать имя виртуальной машины, группу ресурсов, виртуальную сеть, подсеть и хранилище.  
**Восстановление диска** | Вы можете восстановить диск и использовать его для создания виртуальной машины.<br/><br/> Если вы выберите этот вариант, Azure Backup копирует данные из хранилища в выбранную учетную запись хранения. Задание восстановления создает шаблон. Этот шаблон можно скачать и использовать для указания пользовательских параметров виртуальной машины и ее создания.<br/><br/> Этот вариант позволяет указать больше параметров для создаваемой виртуальной машины, чем предыдущий вариант.<br/><br/>
**Замена существующего диска** | Вы можете восстановить диск, а затем использовать его для замены диска, который в настоящий момент используется на виртуальной машине.
**Восстановление файлов** | Вы можете восстановить файлы из выбранной точки восстановления. Вы скачаете сценарий для подключения диска виртуальной машины из точки восстановления. Затем вы просмотрите тома диска, чтобы найти файлы и папки, которые нужно восстановить, а когда все будет готово, отключите диск.

## <a name="support-for-file-level-restore"></a>Поддержка восстановления на уровне файлов

**Restore** | **Поддерживаются**
--- | --- 
Восстановление файлов во всех операционных системах | Вы можете восстановить файлы на любом компьютере, операционная система на котором совпадает или совместима с операционной системой виртуальной машины, для которой выполнено резервное копирование. Дополнительные сведения см. в [таблице совместимых ОС](backup-azure-restore-files-from-vm.md#system-requirements).
Восстановление файлов на классических виртуальных машинах | Не поддерживается. 
Восстановление файлов из зашифрованных виртуальных машин | Не поддерживается.
Восстановление файлов из учетных записей хранения с ограниченным доступом из сети | Не поддерживается.
Восстановление файлов на виртуальных машинах с помощью дисковых пространств Windows | Восстановление на той же виртуальной машине не поддерживается.<br/><br/> Вместо этого нужно восстановить файлы на совместимой виртуальной машине.
Восстановление файлов на виртуальной машине Linux с использованием LVM или RAID-массивов | Восстановление на той же виртуальной машине не поддерживается.<br/><br/> Выполните восстановление на совместимой виртуальной машине.
Восстановление файлов со специальными сетевыми параметрами | Восстановление на той же виртуальной машине не поддерживается. <br/><br/> Выполните восстановление на совместимой виртуальной машине.

## <a name="support-for-vm-management"></a>Поддержка управления виртуальными машинами

В таблице изложена поддержка резервного копирования во время выполнения задач управления виртуальной машиной, таких как добавление или замена дисков виртуальной машины.

**Restore** | **Поддерживаются** 
--- | --- 
Восстановление между подписками, регионами или зонами | Не поддерживается 
Восстановление на существующей виртуальной машине | Используйте вариант замены диска.
Восстановление диска с помощью учетной записи хранения, в которой включено Шифрование службы хранилища Azure | Не поддерживается.<br/><br/> Выполните восстановление в учетной записи, в которой отключено SSE.
Восстановление в смешанных учетных записях хранения | Не поддерживается.<br/><br/> В зависимости от типа учетной записи хранения все восстановленные диски будут класса Premium или Standard, но не смешанными. 
Восстановление в учетной записи хранения, использующей ZRS | Не поддерживается
Восстановление виртуальной машины напрямую в группе доступности | Что касается управляемых дисков, можно восстановить диск и использовать параметр группы доступности в шаблоне.<br/><br/> Эта возможность недоступна для неуправляемых дисков. Что касается неуправляемых дисков, восстановите диск, а затем создайте виртуальную машину в группе доступности.
Восстановление резервной копии неуправляемых виртуальных машин после обновления их до управляемых |  Поддерживается.<br/><br/> Вы можете восстановить диски, а затем создать управляемую виртуальную машину.
Восстановление виртуальной машины до точки восстановления до переноса виртуальной машины на управляемые диски |  Поддерживается.<br/><br/> Нужно выполнить восстановление на неуправляемых дисках (по умолчанию), преобразовать восстановленные диски в управляемые диски и создать виртуальную машину с управляемыми дисками.
Восстановление удаленной виртуальной машины |  Поддерживается.<br/><br/> Вы можете восстановить виртуальную машину из точки восстановления.
Восстановление виртуальной машины контроллера домена, которая входит в конфигурацию с несколькими контроллерами домена, с помощью портала | Поддерживается, если восстановить диск и создать виртуальную машину с помощью PowerShell.
Восстановление виртуальной машины в другой виртуальной сети |   Поддерживается.<br/><br/> Эта виртуальная сеть должна относиться к той же подписке и тому же региону.




## <a name="vm-compute-support"></a>Поддержка вычислений виртуальной машины

**Среда выполнения приложений** | **Поддержка** 
--- | ---

## <a name="vm-securityencryption-support"></a>Поддержка шифрования и безопасности виртуальной сети

Azure Backup поддерживает шифрование передаваемых и хранящихся данных.

Сетевой трафик в Azure:
- Трафик файла резервной копии с серверов в хранилище Служб восстановления подвергается шифрованию с использованием AES 256.
- Данные файла резервной копии передаются через защищенное соединение HTTPS.
- Данные файла резервной копии также хранятся в зашифрованном виде в хранилище Служб восстановления.
- Только у вас есть парольная фраза для расшифровки этих данных. Корпорация Майкрософт не может расшифровывать данные резервных копий.
    > [!WARNING]
    > После настройки хранилища только у вас есть доступ к ключу шифрования. Корпорация Майкрософт никогда не управляет копированием и не имеет доступа к ключу. При утрате ключа Майкрософт не сможет восстановить данные резервных копий.
Защита данных:
- Для создания резервной копии виртуальных машин Azure вам необходимо настроить шифрование *в рамках* виртуальной машины.
- Служба Azure Backup поддерживает шифрование дисков Azure, которое использует BitLocker на виртуальных машинах Windows и **dm-crypt** на виртуальных машинах Linux.
- В серверной части Azure Backup использует [шифрование службы хранилища Azure](../storage/common/storage-service-encryption.md), что обеспечивает защиту неактивных данных.

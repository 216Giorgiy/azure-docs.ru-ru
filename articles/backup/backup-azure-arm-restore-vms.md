# Восстановление виртуальных машин с помощью портала Azure

> [AZURE.SELECTOR]
- [Восстановление виртуальных машин на классическом портале Azure](backup-azure-restore-vms.md)
- [Восстановление виртуальных машин на портале Azure](backup-azure-arm-restore-vms.md)


Служба архивации позволяет защитить ваши данные, создавая моментальные снимки данных через определенные интервалы времени. Эти моментальные снимки называются точками восстановления. Они хранятся в хранилищах служб восстановления. Если вам нужно исправить или повторно создать виртуальную машину, вы можете восстановить ее из любой сохраненной точки восстановления. После восстановления из точки восстановления виртуальная машина вернется в то же состояние, в котором она находилась на момент создания этой точки. В этой статье описывается процесс восстановления виртуальной машины.

> [AZURE.NOTE] В Azure предусмотрены две модели развертывания, позволяющие создавать ресурсы и работать с ними: [модель Resource Manager и классическая модель](../resource-manager-deployment-model.md). Данная статья содержит сведения о восстановлении виртуальных машин, развернутых с использованием модели Resource Manager.



## Восстановление из точки восстановления

1. Войдите на [портал Azure](http://ms.portal.azure.com/).

2. В меню Azure щелкните элемент **Обзор**, а затем введите в поле текст **службы восстановления**, чтобы отобразить список ресурсов. Список служб будет отображаться в соответствии с вводимым текстом. Когда вы увидите пункт **Хранилища служб восстановления**, щелкните его.

    ![Открытие хранилища служб восстановления](./media/backup-azure-arm-restore-vms/open-recovery-services-vault.png)

    Появится список хранилищ, доступных в подписке.

    ![Список хранилищ служб восстановления](./media/backup-azure-arm-restore-vms/list-of-rs-vaults.png)

3. В этом списке выберите хранилище, связанное с восстанавливаемой виртуальной машиной. Щелкните это хранилище, чтобы открыть панель мониторинга.

    ![Список хранилищ служб восстановления](./media/backup-azure-arm-restore-vms/select-vault-open-vault-blade.png)

4. На панели мониторинга выполните следующие действия. На плитке **Элементы службы архивации** щелкните **Виртуальные машины Azure**, чтобы увидеть связанные с этим хранилищем виртуальные машины.

    ![Панель мониторинга хранилища](./media/backup-azure-arm-restore-vms/vault-dashboard.png)

    Откроется колонка **Элементы службы архивации**, в которой вы увидите список виртуальных машин Azure.

    ![Список виртуальных машин в хранилище](./media/backup-azure-arm-restore-vms/list-of-vms-in-vault.png)

5. Выберите в этом списке нужную виртуальную машину, чтобы открыть панель мониторинга.

    ![Список виртуальных машин в хранилище](./media/backup-azure-arm-restore-vms/list-of-vms-in-vault-selected.png)

    В области мониторинга откроется панель мониторинга виртуальной машины с плиткой "Точки восстановления".

    ![Список виртуальных машин в хранилище](./media/backup-azure-arm-restore-vms/vm-blade.png)

6. В меню панели мониторинга виртуальной машины щелкните **Восстановить**.

    ![Список виртуальных машин в хранилище](./media/backup-azure-arm-restore-vms/vm-blade-menu-restore.png)

    Откроется колонка "Восстановление".

    ![Колонка восстановления](./media/backup-azure-arm-restore-vms/restore-blade.png)

7. В колонке **Восстановление** щелкните **Точка восстановления**, чтобы открыть колонку **Выбор точки восстановления**.

    ![Колонка восстановления](./media/backup-azure-arm-restore-vms/recovery-point-selector.png)

    По умолчанию в этом диалоговом окне показаны все точки восстановления, созданные за последние 30 дней. Вы можете произвольно изменять параметры **Период**, **Год**, и **Согласованность точки восстановления**. Дополнительные сведения о типах точек восстановления см. в разделе [Согласованность данных](./backup-azure-vms-introduction.md#data-consistency).
    - **Период**: можно выбрать любой месяц или последние 30 дней.
    - **Год**: можно выбрать год.
    - **Согласованность точки восстановления**. Можно выбрать следующие варианты:
        - точки восстановления с файлами, согласованными на момент времени;
        - точки восстановления с согласованным состоянием приложений;
        - точки восстановления с согласованным состоянием файловой системы;
        - все точки восстановления.  

8. Выберите точку восстановления и нажмите кнопку **ОК**.

    ![Выбор точки восстановления](./media/backup-azure-arm-restore-vms/select-recovery-point.png)

    В колонке **Восстановление** отобразится выбранная точка восстановления.

    ![Выбранная точка восстановления](./media/backup-azure-arm-restore-vms/recovery-point-set.png)

9. В колонке **Восстановление** щелкните **Конфигурация восстановления**, чтобы открыть соответствующую колонку.

    ![Выбранный мастер настройки восстановления](./media/backup-azure-arm-restore-vms/recovery-configuration-wizard.png)

## Выбор конфигурации восстановления для виртуальной машины

Теперь, когда вы выбрали точку восстановления, следует выбрать учетную запись хранения и метод настройки восстанавливаемой виртуальной машины. Настроить восстановление виртуальной машины можно с помощью портала Azure или PowerShell. Учетную запись хранения следует выбрать такую, которая находится в одном расположении с хранилищем служб восстановления. Учетные записи хранения, избыточные в пределах зоны, не поддерживаются. Если у вас нет учетной записи хранения в одном расположении с хранилищем служб восстановления, ее следует создать до запуска восстановления.

1. Перейдите к колонке **Восстановление**, если она еще не открыта. Убедитесь, что вы выбрали **точку восстановления**, а затем щелкните **Конфигурация восстановления**, чтобы открыть колонку **Конфигурация восстановления**.

    ![Выбранный мастер настройки восстановления](./media/backup-azure-arm-restore-vms/recovery-configuration-wizard.png)

2. В колонке **Конфигурация восстановления** щелкните **Учетная запись хранения**. Откроется список учетных записей хранения в том же расположении, что и хранилище служб восстановления.

3. Выберите в этом списке нужную учетную запись хранения и нажмите кнопку ОК.

    ![Список учетных записей хранения](./media/backup-azure-arm-restore-vms/list-of-storage-accounts.png)

    В колонке **Конфигурация восстановления** выбранная учетная запись хранения появится в разделе **Учетная запись хранения**.

    ![Выбор учетной записи хранения](./media/backup-azure-arm-restore-vms/selected-storage-account.png)

4. Для восстановления на портале вы можете выбрать виртуальную машину ARM или классическую виртуальную машину. Если вы хотите восстановить сложную конфигурацию (см. пример ниже), для настройки виртуальной машины с диска следует использовать PowerShell. Если конфигурация не будет сложной, переходите к шагу 5.

    ![Список сложных конфигураций виртуальных машин](./media/backup-azure-arm-restore-vms/complex-vm-configurations.png)

    Если конфигурация *будет* сложной, щелкните **OK** в колонке **Конфигурация восстановления**. Колонка **Конфигурация восстановления** закроется, а рядом с элементом "Конфигурация восстановления" появится галочка. Щелкните **Восстановить**, чтобы запустить задание восстановления диска. Не выполняйте шаг 5; вместо этого перейдите к разделу [Восстановление виртуальных машин с помощью специальных конфигураций сети](#restoring-vms-with-special-network-configurations).

    ![Настроенное восстановление](./media/backup-azure-arm-restore-vms/restore-configured.png)

5. В колонке **Конфигурация восстановления** щелкните элемент **Конфигурация виртуальной машины**, чтобы открыть колонку **Создание виртуальной машины**.

    ![Открытие колонки создания виртуальной машины](./media/backup-azure-arm-restore-vms/recovery-configuration-create-vm.png)

6. В колонке **Создание виртуальной машины** введите или выберите значения для каждого из следующих полей.

    - **Имя виртуальной машины** — введите имя виртуальной машины. Имя должно быть уникальным в пределах группы ресурсов (для виртуальной машины ARM) или облачной службы (для классической виртуальной машины). Если вы планируете заменить существующую виртуальную машину одноименной новой, следует сначала удалить старую виртуальную машину и диски данных, а затем восстановить данные из службы архивации Azure.
    - **Группа ресурсов** — выберите существующую группу ресурсов или создайте новую. При восстановлении классической виртуальной машины укажите в этом поле имя новой облачной службы. Имя новой группы ресурсов или облачной службы должно быть глобально уникальным. Обычно имя облачной службы связано с общедоступным URL-адресом, например: [облачная\_служба].cloudapp.net. Если вы попытаетесь использовать уже существующее имя группы ресурсов или облачной службы, Azure назначит группе ресурсов или облачной службе имя виртуальной машины. Azure отображает группы ресурсов, облачные службы и виртуальные машины, не связанные с территориальными группами. Подробную информацию см. в статье [Переход от территориальных групп к региональной виртуальной сети](../virtual-network/virtual-networks-migrate-to-regional-vnet.md).
    - **Виртуальная сеть** — выберите нужную виртуальную сеть для создания виртуальной машины. В этом раскрывающемся списке перечислены все виртуальные сети, связанные с подпиской.

    > [AZURE.NOTE] Если вы восстанавливаете виртуальную машину на основе ARM, вам нужно обязательно указать виртуальную сеть. Для классической виртуальной машины виртуальную сеть можно не указывать.

    - **Подсеть** — если виртуальная сеть содержит подсети, по умолчанию будет выбрана первая из них. Если подсетей несколько, выберите нужную.

7. В колонке **Создание виртуальной машины** щелкните **ОК** для завершения настройки.

    Колонка **Создание виртуальной машины** закроется, а в колонке **Конфигурация восстановления** появится имя новой виртуальной машины.

    ![Завершенная настройка восстановления](./media/backup-azure-arm-restore-vms/recovery-configuration-complete.png)

8. В колонке **Конфигурация восстановления** щелкните **ОК**, чтобы подтвердить конфигурацию восстановления.

9. В колонке **Восстановление** щелкните **Восстановить**, чтобы запустить восстановление.

    ![Завершенная настройка восстановления](./media/backup-azure-arm-restore-vms/trigger-restore-operation.png)

## Отслеживание восстановления

Когда вы запускаете восстановление, служба архивации создает задание для отслеживания выполнения операции. Служба архивации также создает и временно отображает уведомление.

![Запущенное восстановление](./media/backup-azure-arm-restore-vms/restore-triggered.png)

Если вы пропустите это уведомление, вы всегда сможете щелкнуть соответствующий значок для просмотра уведомлений.

![Запущенное восстановление](./media/backup-azure-arm-restore-vms/restore-notification.png)

Чтобы просмотреть состояние выполняющейся или завершенной операции, откройте список заданий службы архивации.

1. В меню Azure щелкните элемент **Обзор**, а затем введите в поле текст **службы восстановления**, чтобы отобразить список ресурсов. Список служб будет отображаться в соответствии с вводимым текстом. Когда вы увидите пункт **Хранилища служб восстановления**, щелкните его.

    ![Открытие хранилища служб восстановления](./media/backup-azure-arm-restore-vms/open-recovery-services-vault.png)

    Появится список хранилищ, доступных в подписке.

    ![Список хранилищ служб восстановления](./media/backup-azure-arm-restore-vms/list-of-rs-vaults.png)

2. В этом списке выберите хранилище, связанное с восстанавливаемой виртуальной машиной. Щелкните это хранилище, чтобы открыть панель мониторинга.

    ![Список хранилищ служб восстановления](./media/backup-azure-arm-restore-vms/select-vault-open-vault-jobs.png)

3. На панели мониторинга хранилища в колонке **Задания службы архивации** щелкните **Виртуальные машины Azure**, чтобы увидеть связанные с этим хранилищем задания.

    ![Панель мониторинга хранилища](./media/backup-azure-arm-restore-vms/vault-dashboard-jobs.png)

    Откроется колонка **Задания службы архивации**, где содержится список заданий.

    ![Список виртуальных машин в хранилище](./media/backup-azure-arm-restore-vms/restore-job-in-progress.png)




## Восстановление виртуальных машин с помощью специальных конфигураций сети
Вы можете выполнять резервное копирование и восстановление виртуальных машин, используя следующие специальные конфигурации сети. Но восстановление с использованием таких конфигураций предполагает соблюдение следующих требований.

- Виртуальные машины с балансировщиком нагрузки (внутренним или внешним).
- Виртуальные машины с несколькими зарезервированными IP-адресами
- Виртуальные машины с несколькими сетевыми картами

>[AZURE.IMPORTANT] Если вы создаете специальную конфигурацию сети для виртуальных машин, следует использовать PowerShell для восстановления с диска.


Вот как можно повторно создать виртуальные машины после восстановления на диск:

1. Восстановите диски из хранилища служб восстановления с помощью [средства PowerShell службы архивации Azure](../backup-azure-vms-automation.md#restore-an-azure-vm).

2. Создайте конфигурацию виртуальной машины для сценариев с использованием балансировщика нагрузки, нескольких сетевых адаптеров или зарезервированных IP-адресов с помощью командлетов PowerShell. Затем создайте с помощью этой конфигурации виртуальную машину с нужными параметрами.
	- Создайте виртуальную машину в облачной службе с [внутренним балансировщиком нагрузки](https://azure.microsoft.com/documentation/articles/load-balancer-internal-getstarted/).
	- Создайте виртуальную машину для подключения к [внутреннему балансировщику нагрузки для Интернета](https://azure.microsoft.com/documentation/articles/load-balancer-internet-getstarted/).
	- Создайте виртуальную машину с [несколькими сетевыми адаптерами](https://azure.microsoft.com/documentation/articles/virtual-networks-multiple-nics/).
	- Создайте виртуальную машину с [несколькими зарезервированными IP-адресами](https://azure.microsoft.com/documentation/articles/virtual-networks-reserved-public-ip/).

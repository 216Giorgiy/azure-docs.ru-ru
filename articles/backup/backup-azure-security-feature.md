# <a name="security-features-for-protecting-hybrid-backups-using-azure-backup"></a>Инструменты безопасности для защиты гибридных резервных копий с помощью службы архивации Azure
Все больше пользователей сталкиваются с проблемами безопасности, такими как вредоносные программы, программы-шантажисты, атаки и т. д. Эти проблемы приводят к потере данных, а расходы из-за нарушения безопасности постоянно растут. Для обеспечения защиты от таких атак служба архивации Azure предоставляет инструменты безопасности для защиты гибридных резервных копий. В этой статье рассказывается о том, как включить и использовать эти инструменты с помощью агента служб восстановления Microsoft Azure и Сервера резервного копирования Microsoft Azure. Эти инструменты созданы на основе трех фундаментальных компонентов безопасности:

1. **Предотвращение.** При выполнении критически важных операций, таких как изменение парольной фразы, добавляется дополнительный уровень аутентификации. Эта проверка гарантирует, что такие операции могут выполнять только пользователи с действительными учетными данными Azure. 
2. **Оповещение.** При выполнении критически важных операций, таких как удаление данных резервных копий, администратору подписки отправляется уведомление по электронной почте. Это сообщение гарантирует своевременное уведомление пользователя о таких действиях.
3. **Восстановление.** Удаленные данные резервных копий сохраняются в течение дополнительных 14 дней с даты удаления. Это гарантирует возможность восстановления данных в течение заданного периода времени, поэтому данные не будут утеряны даже в случае атаки. Кроме того, поддерживается больше точек минимального восстановления для защиты от повреждения данных. 

> [!NOTE]
> Инструменты безопасности следует включить только при использовании следующих компонентов: <br/>
> * **Агент службы архивации Microsoft Azure** версии не менее 2.0.9052. Включив эти инструменты, следует обновить агент до этой версии для выполнения критически важных операций, таких как изменение парольной фразы и прекращение архивации с удалением данных. <br/>
> * **Сервер резервного копирования Azure.** Следует использовать агент службы архивации Microsoft Azure версии не менее 2.0.9052 с Сервером резервного копирования Azure с обновлением 1. <br/>
> * **DPM.** Не включайте эти инструменты для DPM. Их скоро добавят в следующие выпуски. Поэтому если включить их, то это никак не скажется на существующих функциях. <br/>
> * **Архивация виртуальных машин IaaS.** Не включайте эти инструменты для архивации виртуальных машин IaaS. Эти инструменты еще не доступны для архивации виртуальных машин IaaS. Поэтому если включить их, то это никак не скажется на этой возможности.
> * Включив их, вы получаете инструменты безопасности для всех компьютеров агента служб восстановления Azure и Серверов резервного копирования Azure, зарегистрированных в хранилище. <br/>
> * Включать эти инструменты нужно только один раз. Включив их, вы не сможете их отключить. <br/>
> * Эти инструменты доступны только для хранилища служб восстановления.
> 
> 

## <a name="enabling-security-features"></a>Включение инструментов безопасности
Создавая хранилище служб восстановления, пользователи смогут воспользоваться всеми инструментами безопасности. Чтобы включить инструменты безопасности в существующем хранилище служб восстановления, сделайте следующее:

1. Войдите на портал Azure, используя учетные данные Azure.
2. В меню концентратора введите "службы восстановления", чтобы перейти к списку этих служб. 
   
    ![Создание хранилища служб восстановления — шаг 1](./media/backup-azure-security-feature/browse-to-rs-vaults.png) <br/>
   
    После этого отобразится список хранилищ служб восстановления, Выберите хранилище в списке. 
   
    Затем откроется панель мониторинга выбранного хранилища.
3. В открывшемся списке элементов хранилища в разделе **Параметры** щелкните **Свойства**.
   
    ![Открытие свойств хранилища](./media/backup-azure-security-feature/vault-list-properties.png)
4. В разделе **Параметры безопасности** щелкните **Обновить**.
   
    ![Открытие параметров безопасности](./media/backup-azure-security-feature/security-settings-update.png)
   
    После выбора ссылки "Обновить" откроется колонка "Параметры безопасности", в которой можно включить эти инструменты и просмотреть сводку о них.
5. В раскрывающемся списке **Have you configured Azure Multi-Factor Authentication?** (Настроена ли Многофакторная идентификация Azure?) выберите соответствующее значение, чтобы подтвердить включение [Многофакторной идентификации Azure](../multi-factor-authentication/multi-factor-authentication.md). Если она включена, вам будет предложено пройти аутентификацию на другом устройстве (например, на мобильном телефоне) при входе на портал Azure. 
   
   При выполнении критически важных операций в службе архивации Azure инструменты безопасности требуют введения ПИН-кода безопасности, доступного на портале Azure. Включение Многофакторной идентификации добавляет дополнительный уровень безопасности, гарантируя, что только авторизованные пользователи, использующие допустимые учетные данные Azure и прошедшие аутентификацию на втором устройстве, могут получить доступ к порталу Azure, а также выполнять такие критически важные операции.
6. Воспользуйтесь переключателем для **включения** и нажмите кнопку **Сохранить** в верхней части колонки, чтобы сохранить параметры безопасности, как показано на рисунке. Вы можете нажать кнопку **Включить**, только выбрав значение в списке Have you configured Azure Multi-Factor Authentication? (Настроена ли Многофакторная идентификация Azure?) .
   
    ![Включение параметров безопасности](./media/backup-azure-security-feature/enable-security-settings.png)

## <a name="recovering-deleted-backup-data"></a>Восстановление удаленных данных резервных копий
Из соображений безопасности служба архивации Azure сохраняет удаленные данные резервных копий в течение дополнительных 14 дней и не удаляет их немедленно после выполнения операции прекращения архивации с удалением данных резервных копий. Чтобы восстановить эти данные в течение 14 дней, сделайте следующее:

Для пользователей **агента служб восстановления Майкрософт**:

1. Если компьютер, на котором выполнялась архивация, все еще доступен, воспользуйтесь [восстановлением данных на том же компьютере](backup-azure-restore-windows-server.md#recover-data-to-the-same-machine) в агенте служб восстановления Майкрософт для восстановления из всех старых точек восстановления.
2. Если этот компьютер недоступен, используйте [восстановление на другой компьютер](backup-azure-restore-windows-server.md#recover-to-an-alternate-machine), чтобы получить эти данные, используя другой компьютер агента служб восстановления Майкрософт.

Для пользователей **Сервера службы архивации Azure**:

1. Если сервер, на котором выполнялась архивация, все еще доступен, выполните повторную защиту удаленных источников данных и воспользуйтесь возможностью восстановления данных для восстановления из всех старых точек восстановления.
2. Если этот компьютер недоступен, воспользуйтесь [восстановлением данных с другого Сервера резервного копирования Azure](backup-azure-alternate-dpm-server.md#recover-data-from-another-azure-backup-server), чтобы получить эти данные, используя другой сервер резервного копирования Azure.

## <a name="preventing-attacks"></a>Предотвращение атак
В этих инструментах предусмотрена проверка, гарантирующая, что только действительные пользователи могут выполнять различные операции.

### <a name="authentication-to-perform-critical-operations"></a>Аутентификация для выполнения критически важных операций
При добавлении дополнительного уровня аутентификации для критически важных операций вам будет предложено ввести ПИН-код безопасности, выполняя операции прекращения защиты с удалением данных и изменения парольной фразы. 

Чтобы получить ПИН-код безопасности, сделайте следующее:

1. Войдите на портал Azure.
2. Перейдите в хранилище службы восстановления и выберите "Параметры > Свойства".
3. В разделе "ПИН-код безопасности" щелкните **Создать**. При выборе ссылки "Создать" откроется колонка, содержащая ПИН-код безопасности, который следует ввести в пользовательском интерфейсе агента служб восстановления Azure. 
    Этот ПИН-код действителен только в течение 5 минут и автоматически создается по истечении этого периода.

### <a name="maintaining-minimum-retention-range"></a>Поддержка минимального периода удержания
Чтобы обеспечить постоянную доступность допустимого числа точек восстановления, добавлены следующие проверки:

1. Для ежедневного сохранения необходимо настроить период удержания в **семь** дней.
2. Для еженедельного сохранения необходимо настроить период удержания в **четыре** недели.
3. Для ежемесячного сохранения необходимо настроить период удержания в **три** месяца.
4. Для ежегодного сохранения необходимо настроить период удержания в **год**.

## <a name="notifications-for-critical-operations"></a>Уведомления для критически важных операций
При выполнении некоторых критически важных операций администратору подписки отправляются уведомления по почте со сведениями об операции. Чтобы настроить дополнительные идентификаторы электронной почты для получения уведомлений, можно воспользоваться порталом Azure.

Инструменты безопасности, упоминаемые в этой статье, предоставляют механизмы защиты от направленных атак, не позволяя злоумышленникам выполнять действия с резервными копиями. Что более важно, эти инструменты позволяют восстановить данные в случае атаки.

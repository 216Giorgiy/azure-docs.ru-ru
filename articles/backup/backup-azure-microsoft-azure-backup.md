# Подготовка к резервному копированию рабочих нагрузок в Microsoft Azure

> [AZURE.SELECTOR]
- [SCDPM](backup-azure-dpm-introduction.md)
- [Azure Backup](backup-azure-microsoft-azure-backup.md)

В этой статье рассматривается подготовка среды для резервного копирования рабочих нагрузок в Azure. Используя сервер архивации Microsoft Azure, вы можете создавать резервные копии рабочих нагрузок приложения, таких как виртуальные машины Hyper-V, Microsoft SQL Server, SharePoint Server, Microsoft Exchange и клиенты Windows, в следующих расположениях:

- **Диск** (D2D), обеспечивая высокие показатели целевого времени восстановления (RTO) для рабочих нагрузок уровня 1.
- **Azure** (D2D2C) для долгосрочного хранения.

Вы можете управлять защитой различных объектов (серверов и клиентов) с помощью единого локального пользовательского интерфейса.

Можно развернуть сервер архивации Microsoft Azure как:

- Изолированный физический сервер.
- Виртуальную машину Hyper-V. Вы можете запустить DPM как виртуальную машину, размещенную на хост-сервере Hyper-V, для резервного копирования локальных данных. Важную информацию об этой среде см. в разделе [Установка DPM в качестве виртуальной машины Hyper-V на локальном сервере](https://technet.microsoft.com/library/dn440941.aspx).
- Виртуальную машину Windows в VMWare. Вы можете развернуть DPM для защиты рабочих нагрузок Майкрософт, выполняющихся на виртуальных машинах Windows в VMWare. В этом сценарии DPM можно развернуть как отдельный физический сервер, как виртуальную машину Hyper-V или как виртуальную машину Windows в VMWare.
- Виртуальную машину Azure. Вы можете запустить DPM как виртуальную машину в Azure для резервного копирования облачных рабочих нагрузок, выполняющихся на виртуальных машинах Azure. Сведения о развертывании см. в статье [Установка DPM в качестве виртуальной машины Azure](https://technet.microsoft.com/library/hh758176.aspx).

Кроме того:

- Сервер архивации Microsoft Azure можно установить на Windows Server 2008 R2 с пакетом обновления 1 (SP1), 2012 и 2012 R2.
- Сервер архивации Microsoft Azure нельзя установить на компьютере, где установлен агент SCDPM или SCDPM RA.
- Сервер архивации Microsoft Azure нельзя установить на компьютере, где установлен агент архивации Microsoft Azure, зарегистрированный в хранилище архивации Azure.

>[AZURE.NOTE]Сервер архивации Microsoft Azure наследует функциональные возможности Data Protection Manager (DPM) для резервного копирования рабочих нагрузок. Однако он не обеспечивают защиту на ленте и интеграцию с System Center.

После того как вы выберете сервер, на котором будет установлена служба архивации Microsoft Azure:

- Убедитесь, что выполняются необходимые условия для сервера архивации Microsoft Azure.
- Создайте хранилище архивации.
- Загрузите учетные данные хранилища.
- Загрузите файлы установки сервера архивации Microsoft Azure.

Ниже подробно рассмотрено каждое из этих действий.

## Перед началом работы

В настоящее время сервер архивации Microsoft Azure доступен только на английском языке. Службы восстановления Microsoft Azure доступны во всех регионах, где доступен сервис Azure, за исключением государственного центра обработки данных Microsoft Azure и Microsoft Azure в Китае.

Во избежание проблем с установкой сервера архивации Microsoft Azure убедитесь, что выполняются следующие условия.

- На сервере должно быть настроено подключение к Интернету. У сервера должен быть доступ к Microsoft Azure.
- К серверу установки службы архивации Microsoft Azure применяются те же требования, что и к DPM. См. дополнительные сведения о [требованиях к оборудованию](https://technet.microsoft.com/library/dn726764.aspx).
- Сервер архивации Microsoft Azure должен быть включен в домен.
- На сервере архивации Microsoft Azure должны быть установлены компоненты .NET 3.5, .NET 4.0 и .NET 3.5 с пакетом обновления 1 (SP1). Дополнительные сведения о том, как включить .NET Framework, см. [здесь](https://technet.microsoft.com/library/dn482071.aspx).
- На сервере архивации Microsoft Azure должен быть установлен компонент Windows Management Framework 4.0. Его можно скачать [здесь](http://www.microsoft.com/download/details.aspx?id=40855).
- В качестве выделенного хранилища данных DPM рекомендуется использовать пул носителей, размер которого в 1,5 раза превышает объем защищаемых данных. Дополнительную информацию см. в разделе «Диски и хранилище» [этой статьи](https://technet.microsoft.com/library/hh758176.aspx#DPM).

Подготовьте сервер архивации Microsoft Azure для резервного копирования данных. Для этого:

1. **Создайте хранилище архивации**. Создайте хранилище в консоли службы архивации Azure.
2. **Загрузите учетные данные хранилища**. В службе архивации Azure загрузите в хранилище созданный вами сертификат управления.
3. **Загрузите сервер архивации Microsoft Azure**. Вы можете загрузить сервер архивации Microsoft Azure для рабочих нагрузок приложения на странице быстрого запуска хранилища архивации.

## Загрузка сервера архивации Microsoft Azure
Как и в случае с учетными данными хранилища, вы можете загрузить службу архивации Microsoft Azure для рабочих нагрузок приложения на **странице быстрого запуска** хранилища архивации.

1. Щелкните **Для рабочих нагрузок приложения (диск — диск — облако)**.

    ![Резервное копирование Microsoft Azure: экран приветствия](./media/backup-azure-microsoft-azure-backup/dpm-venus1.png)

2. Щелкните элемент **Загрузить**.

    ![Центр загрузки 1](./media/backup-azure-microsoft-azure-backup/downloadcenter1.png)

3. Выберите все файлы и нажмите кнопку **Далее**. ![Центр загрузки 1](./media/backup-azure-microsoft-azure-backup/downloadcenter.png)

    >[AZURE.NOTE]Поскольку общий размер файлов превышает 3 ГБ, при скорости подключения 10 Мбит/с загрузка может занять до одного часа.

4. Поместите все файлы в одну папку.

5. Загрузите все файлы со страницы загрузки сервера архивации Microsoft Azure.

## Установка сервера архивации Microsoft Azure
Перед запуском программы установки убедитесь, что необходимые условия, перечисленные в предыдущем разделе, выполнены.

1. После загрузки всех файлов щелкните **MicrosoftAzureBackupInstaller.exe**. Откроется **Мастер настройки сервера архивации Microsoft Azure**.

    ![Резервное копирование Microsoft Azure: установщик](./media/backup-azure-microsoft-azure-backup/installer-click.png)

2. Выберите серверную папку, в которую следует извлечь файлы, и нажмите кнопку **Далее**.

    ![Мастер настройки резервного копирования в Microsoft Azure](./media/backup-azure-microsoft-azure-backup/extract-to.png)

3. Щелкните **Извлечь**, чтобы начать извлечение файлов установки.

    ![Мастер настройки резервного копирования в Microsoft Azure](./media/backup-azure-microsoft-azure-backup/ready-to-extract.png)

4. Установите флажок для запуска setup.exe, чтобы приступить к установке сервера архивации Microsoft Azure, и нажмите кнопку **Готово**.

    ![Мастер настройки резервного копирования в Microsoft Azure](./media/backup-azure-microsoft-azure-backup/launch-setup.png)

5. Щелкните **Сервер архивации Microsoft Azure**, чтобы запустить мастер установки.

    ![Мастер настройки резервного копирования в Microsoft Azure](./media/backup-azure-microsoft-azure-backup/launch-screen2.png)

6. На экране приветствия щелкните **Далее**.

    ![Резервное копирование Microsoft Azure: экран приветствия](./media/backup-azure-microsoft-azure-backup/welcome-screen.png)

7. Щелкните **Проверить**, чтобы определить, выполняются ли требования к оборудованию и программному обеспечению для сервера архивации Microsoft Azure.

    ![Microsoft Azure Backup PreReq1](./media/backup-azure-microsoft-azure-backup/prereq-screen1.png)

8. Если все обязательные условия выполняются, появится соответствующее сообщение. Нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/prereq-screen2.png)

9. Для сервера архивации Microsoft Azure требуется SQL Server Standard. Выберите существующий локальный экземпляр SQL Server Standard 2014 или установите SQL Server Standard с помощью мастера. Щелкните **Проверить и установить**, чтобы убедиться, что необходимые компоненты SQL Server установлены.

    ![Служба архивации Microsoft Azure - SQL1](./media/backup-azure-microsoft-azure-backup/sql-setup1.png)

    В случае сбоя и появления рекомендации перезагрузить компьютер, выполните инструкцию на экране и нажмите кнопку **Проверить снова**.

    ![Сбой фильтра SiS](./media/backup-azure-microsoft-azure-backup/sis-filter.png)

10. После успешной установки необходимых компонентов нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/sql-setup3.png)

11. Укажите расположение для установки файлов сервера архивации Microsoft Azure и нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/space-screen.png)

    >[AZURE.NOTE]Папка для временных файлов является обязательной для резервного копирования в Azure. Убедитесь, что размер папки для временных файлов составляет по крайней мере 5 % объема данных, для которых планируется создать резервную копию в облаке. После завершения установки необходимо настроить защиту отдельных дисков. Дополнительные сведения о пулах носителей см. в статье [Настройка пулов носителей и дисковых хранилищ](https://technet.microsoft.com/library/hh758075.aspx).

12. Введите надежный пароль для локальных учетных записей пользователей с ограниченными правами и нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/security-screen.png)

    >[AZURE.NOTE]Надежный пароль — неотъемлемый компонент обеспечения безопасности вашей системы. Надежным считается пароль, который состоит как минимум из шести символов, не содержит части имени учетной записи пользователя и включает по крайней мере три из этих четырех категорий символов: прописные и строчные буквы, основные 10 цифр и специальные знаки (!, @, #).

13. Укажите, следует ли проверять обновления с помощью *Центра обновления Майкрософт*, и нажмите кнопку **Далее**.

    >[AZURE.NOTE]Мы рекомендуем выполнять перенаправление из Центра обновления Windows в Центр обновления Майкрософт, который предлагает обновления безопасности и другие важные обновления для Windows и других продуктов, таких как сервер архивации Microsoft Azure.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/update-opt-screen2.png)

14. Просмотрите *Сводку параметров* и нажмите кнопку **Установить**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/summary-screen.png)

    Установка выполняется поэтапно. На первом этапе выполняется установка агента на сервере. Необходимо зарегистрировать сервер с действительными учетными данными хранилища архивации Azure, которые загружаются вместе с продуктом. Далее мастер проверяет наличие прямого подключения к Интернету. При наличии подключения к Интернету можно продолжить установку. В противном случае потребуется указать сведения о прокси-сервере.

15. Щелкните **Далее**, чтобы перейти к этапу установки агента служб восстановления Microsoft Azure.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/proxy.png)

    Мастер проверяет наличие необходимых компонентов и при необходимости устанавливает недостающие.

16. Щелкните **Установить**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/mars-prereq.png)

17. После завершения установки агента щелкните **Далее**, чтобы зарегистрировать сервер в хранилище архивации Azure.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/mars-successful.png)

18. Укажите учетные данные хранилища архивации Azure, чтобы зарегистрировать сервер, и нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/vault-reg-screen.png)

19. Укажите парольную фразу, используемую для шифрования и расшифровки данных, отправляемых в Azure, а также место, где она будет храниться. Вы можете сгенерировать парольную фразу автоматически или ввести ее вручную. Она должна включать не менее 16 символов. Нажмите кнопку **Далее**.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/encrypt1.png)

    >[AZURE.NOTE]В целях обеспечения конфиденциальности сервер архивации Microsoft Azure не отправляет парольную фразу вместе с данными в Azure. Необходимо сохранить парольную фразу в безопасном месте. Она понадобится при восстановлении данных из Azure.

    После успешного завершения регистрации сервера архивации Microsoft Azure установка продолжается: выполняется установка и настройка SQL Server Standard 2014 (если этот компонент выбран).

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/sql-install-start-screen.png)

    C установкой SQL Server завершается установка компонентов сервера архивации Microsoft Azure.

    ![Microsoft Azure Backup PreReq2](./media/backup-azure-microsoft-azure-backup/venus-installation-screen.png)

20. После установки продукта и создания ярлыков на рабочем столе дважды щелкните соответствующий ярлык для запуска продукта.

    ![Значки](./media/backup-azure-microsoft-azure-backup/icons.png)

## Добавление диска в пул носителей

Чтобы выполнить резервное копирование рабочих нагрузок на диске и в Azure, необходимо сначала добавить диск в пул носителей. Дополнительные сведения о добавлении дисков см. в статье [Настройка пулов носителей и дисковых хранилищ](https://technet.microsoft.com/library/hh758075.aspx).

После настройки пулов носителей можно создать резервные копии рабочих нагрузок приложений как на диске, так и в Azure.

## Устранение неполадок

Если на этапе установки, резервного копирования или восстановления происходит сбой сервера архивации Microsoft Azure с указанием кода ошибки, см. информацию о кодах ошибок [в этом документе](https://support.microsoft.com/kb/3041338). Вы также можете изучить [часто задаваемые вопросы, связанные с резервным копированием в Azure](backup-azure-backup-faq.md).

## Часто задаваемые вопросы

### Установка и настройка

Вопрос. Поддерживают ли диски и тома сервера архивации Microsoft Azure, используемые для резервного копирования дисков, сжатие NTFS? <br>Ответ. NTFS **не** поддерживается на дисках и томах, подключенных к серверу архивации Microsoft Azure.

Вопрос. Можно ли переместить сервер архивации Microsoft Azure в новый домен после развертывания? <br>Ответ. Нет, перемещение сервера архивации Microsoft Azure в новый домен после развертывания **не** поддерживается.

Вопрос. Какие ресурсы может защитить сервер архивации Microsoft Azure защитить, если он установлен на контроллере домена? <br>Ответ. Сервер архивации Microsoft Azure может защитить только локальные источники данных на сервере, который выполняет роль контроллера домена. Чтобы он мог защищать рабочие нагрузки на других серверах, не устанавливайте его на компьютере, который является контроллером домена.

Вопрос. Может ли сервер архивации Microsoft Azure использовать удаленный экземпляр SQL Server? <br>Ответ. Нет, сервер архивации Microsoft Azure может использовать только локальный экземпляр SQL Server.

### Пул носителей для сервера архивации Microsoft Azure

Вопрос. Можно ли добавить виртуальный диск (VHD/HDx) в пул носителей сервера архивации Microsoft Azure? <br>Ответ. Да.

Вопрос. Можно ли убрать дубликаты из пула носителей сервера архивации Microsoft Azure? <br>Ответ. Да, дедупликация доступна для пула носителей сервера архивации Microsoft Azure. Все пользовательские операции с сервером архивации Microsoft Azure подробно описаны в [этом блоге по DPM](http://blogs.technet.com/b/dpm/archive/2015/01/06/deduplication-of-dpm-storage-reduce-dpm-storage-consumption.aspx).

Вопрос. Можно ли добавить в пул носителей сервера архивации Microsoft Azure съемные и USB-диски? <br>Ответ. Нет.

### Подключение сервера архивации Microsoft Azure к службе Azure

Вопрос. Каким образом подключение сервера архивации Microsoft Azure к службе Azure и состояние подписки Azure влияют на резервное копирование и восстановление? <br>Ответ. Для успешной работы серверу архивации Microsoft Azure необходимо подключение к службе Azure. При этом подписка Azure должна быть активна.

В таблице ниже перечислены функциональные возможности сервера архивации Microsoft Azure в зависимости от наличия подключения и состояния подписки Azure.

| Состояние подключения | Подписка Azure | Резервное копирование в Azure| Резервное копирование на диск | Восстановление из Azure | Восстановление с диска |
| -------- | ------- | --------------------- | ------------------- | --------------------------- | ----------------------- |
| Подключено | Активна | Разрешено | Разрешено | Разрешено | Разрешено |
| Подключено | Срок действия истек | Остановлено | Остановлено | Разрешено | Разрешено |
| В обычном режиме | Отозвана | Остановлено | Остановлено | Остановлено; точки восстановления Azure удалены | Остановлено |
| Подключение потеряно на 15 и более дней | Активна | Остановлено | Остановлено | Разрешено | Разрешено |
| Подключение потеряно на 15 и более дней | Срок действия истек | Остановлено | Остановлено | Разрешено | Разрешено |
| Подключение потеряно на 15 и более дней | Отозвана | Остановлено | Остановлено | Остановлено; точки восстановления Azure удалены | Остановлено |

Вопрос. Что произойдет, если соединение с сервером архивации Microsoft Azure будет восстановлено, а подписка будет оставаться *активной*? <br>Ответ. Если подписка *активна*, как только соединение между сервером архивации Microsoft Azure и службой Azure будет восстановлено, будут разрешены все операции сервера архивации Microsoft Azure. Резервное копирование на диск и в Azure будет выполняться в обычном режиме.

Вопрос. Что произойдет, если состояние подписки изменится с *Истек срок действия* на *Активна*? <br>Ответ. Если соединение между сервером архивации Microsoft Azure и службой Azure осуществляется в штатном режиме, после перехода подписки Microsoft Azure в состояние *Активна* будут разрешены все операции сервера архивации Microsoft Azure. Резервное копирование на диск и в Azure будет выполняться в обычном режиме.

Вопрос. Что произойдет, если состояние подписки изменится с *Отозвана* на *Активна*? <br>Ответ. Если соединение между сервером архивации Microsoft Azure и службой Azure осуществляется в штатном режиме, после перехода подписки Microsoft Azure в состояние *Активна* будут разрешены все операции сервера архивации Microsoft Azure. Однако точки восстановления Azure удаляются без возможности восстановления. Точки восстановления диска могут быть восстановлены, если срок хранения резервных копий не истек.

Вопрос. Какие исключения необходимо создать, чтобы разрешить серверу архивации Microsoft Azure подключаться к открытой службе Azure? <br>Ответ. Необходимо разрешить следующие домены в профиле брандмауэра: www.msftncsi.com, *.Microsoft.com, *.WindowsAzure.com, *.microsoftonline.com, *.windows.net.

Вопрос. Как проверить подключение сервера архивации Microsoft Azure к службе Azure? <br>Ответ. Выполните следующий командлет PowerShell в консоли сервера архивации Microsoft Azure.

```
Get-DPMCloudConnection
Output:
TRUE - Connected to Azure service
FALSE - Not connected to Azure service
```

Вопрос. Почему состояние моей подписки отличается от *Активна*? Как восстановить состояние *Активна*? <br>Ответ. Для управления подпиской войдите на [портал подписки](https://account.windowsazure.com/Subscriptions).

### Выставление счетов

Вопрос. Какая модель выставления счетов используется для сервера архивации Microsoft Azure? <br>Ответ. Счета пользователям выставляются на основе модели защищенных экземпляров. Дополнительные сведения см. в вопросах и ответах на странице [Цены](http://azure.microsoft.com/pricing/details/backup/).

Вопрос. Какая модель выставления счетов используется, если резервное копирование выполняется только на диске? <br>Ответ. Используется та же модель выставления счетов, что и для модели защищенных экземпляров. Поскольку данные защищаются с помощью локального хранилища, плата за хранение резервных копий в хранилище Azure не взимается. В этом случае клиенту выставляется счет только за защищенные экземпляры. Дополнительные сведения о параметрах, определяющих экземпляр, и о размере платы за каждый экземпляр, см. в вопросах и ответах на странице [Цены](http://azure.microsoft.com/pricing/details/backup/).

Вопрос. Какова плата за защищенный экземпляр? <br>См. информацию на странице [Цены](http://azure.microsoft.com/pricing/details/backup/).

Вопрос. Где можно найти примеры различных рабочих нагрузок приложений и расчета платы за защищенные экземпляры? <br>Ответ. См. раздел «Примеры» на странице [Цены](http://azure.microsoft.com/pricing/details/backup/)

Вопрос. Как выставляются счета на использование сервера архивации Microsoft Azure, если источник данных защищен как на диске, так и в облаке? <br>A. Независимо от того, где именно выполняется резервное копирование, на диске или в облаке, сервер архивации Microsoft Azure будет начислять плату по защищенным экземплярам. Размер защищаемого экземпляра рассчитывается на основе размера интерфейсной части источника данных. При резервном копировании данных в хранилище Azure также начисляется плата за пользование хранилищем Azure.

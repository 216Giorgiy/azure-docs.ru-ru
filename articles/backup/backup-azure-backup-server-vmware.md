# <a name="back-up-vmware-server-to-azure"></a>Резервное копирование сервера VMware в Azure

В этой статье описывается, как настроить Сервер резервного копирования Azure для защиты рабочей нагрузки сервера VMware. Чтобы приступить к работе с этим руководством, вам необходимо установить сервер резервного копирования Azure. Если у вас установлен Сервер резервного копирования Azure, прочитайте статью [Подготовка к резервному копированию рабочих нагрузок с использованием сервера службы архивации Azure](backup-azure-microsoft-azure-backup.md).

Сервер резервного копирования Azure может архивировать (защищать) серверы VMware vCenter версии 6.0 и 5.5.


## <a name="create-a-secure-connection-to-the-vcenter-server"></a>Создание безопасного подключения к серверу vCenter

По умолчанию Сервер резервного копирования Azure взаимодействует с серверами vCenter через канал HTTPS. Для обеспечения безопасного взаимодействия рекомендуется установить сертификат центра сертификации (ЦС) VMware на Сервер резервного копирования Azure. Если безопасный обмен данными не требуется и вы хотите отключить требование HTTPS, ознакомьтесь с разделом [Отключение протокола безопасной связи](backup-azure-backup-server-vmware.md#disable-secure-communication-protocol). Чтобы создать безопасное подключение между Сервером резервного копирования Azure и сервером vCenter, импортируйте доверенный сертификат на Сервер резервного копирования Azure.

Обычно для подключения к серверу vCenter через веб-клиент vSphere используется браузер на компьютере Сервера резервного копирования Azure. При первом использовании браузера на Сервере резервного копирования Azure для подключения к серверу vCenter устанавливается небезопасное подключение. На следующем рисунке показано незащищенное подключение.

![Пример незащищенного подключения к серверу VMware](./media/backup-azure-backup-server-vmware/unsecure-url.png)

Чтобы устранить эту проблему и создать безопасное подключение, скачайте сертификаты доверенного корневого ЦС.

1. В браузере на Сервере резервного копирования Azure введите URL-адрес веб-клиента vSphere.

  Откроется страница входа веб-клиента vSphere.

  ![Веб-клиент vSphere](./media/backup-azure-backup-server-vmware/vsphere-web-client.png)

  В нижней части сведений для администраторов и разработчиков указана ссылка для **скачивания сертификатов доверенного корневого ЦС**.

  ![Ссылка для скачивания доверенных корневых сертификатов ЦС](./media/backup-azure-backup-server-vmware/vmware-download-ca-cert-prompt.png)

  Если страница входа веб-клиента vSphere не отображается, проверьте параметры прокси-сервера в браузере.

2. Щелкните **Download trusted root CA certificates** (Скачать сертификаты доверенного корневого ЦС).

  Сервер vCenter скачает файл на локальный компьютер. Имя этого файла — **download**. В зависимости от браузера появится сообщение с предложением открыть или сохранить файл.

  ![Сообщение после скачивания сертификатов](./media/backup-azure-backup-server-vmware/download-certs.png)

3. Сохраните файл в папку на Сервере резервного копирования Azure. При сохранении файла добавьте к его имени расширение ZIP.

  Это ZIP-файл, содержащий сведения о сертификатах. Добавление расширения ZIP позволит использовать инструменты извлечения.

4. Щелкните правой кнопкой мыши файл **download.zip** и выберите "Извлечь все", чтобы извлечь его содержимое.

  Содержимое сжатого ZIP-файла будет извлечено в папку **certs**. В папке certs содержатся файлы двух типов. Файл корневого сертификата имеет расширение 0, 1, или *число*. Файл списка отзыва сертификатов имеет расширение, которое начинается с r0, r1 и т. д. Файл списка отзыва сертификатов связан с сертификатом.

  ![Извлечение скачанного файла на локальный компьютер ](./media/backup-azure-backup-server-vmware/extracted-files-in-certs-folder.png)

5. В папке **certs** щелкните правой кнопкой мыши файл корневого сертификата и выберите **Переименовать**.

  ![Переименование корневого сертификата ](./media/backup-azure-backup-server-vmware/rename-cert.png)

  Измените расширение файла корневого сертификата на CRT. Если отобразится запрос подтверждения того, что вы хотите изменить расширение, так как это может привести к изменению его функционирования, нажмите кнопку "Да" или "ОК". Значок файла изменится на значок корневого сертификата.

6. Щелкните правой кнопкой мыши этот корневой сертификат и во всплывающем меню выберите **Установить сертификат**.

  Откроется мастер импорта сертификатов.

7. В мастере импорта сертификатов выберите **Локальный компьютер** в качестве места назначения для сертификата, а затем щелкните **Далее**, чтобы продолжить.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-import-wizard1.png)

  Если отобразится запрос разрешения на внесение изменений в компьютер, нажмите кнопку "Да" или "ОК" для всех изменений.

8. На экране хранилища сертификатов выберите **Поместить все сертификаты в следующее хранилище**, а затем щелкните **Обзор**, чтобы выбрать хранилище сертификатов.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-import-wizard-local-store.png)

  Откроется диалоговое окно "Выбор хранилища сертификата".

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-store.png)

9. В качестве папки назначения для сертификатов выберите **Trusted Root Certification Authorities** и нажмите кнопку **ОК**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-store-selected.png)

  Выбранное хранилище сертификатов появится в **мастере импорта сертификатов**. Щелкните **Далее**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-import-wizard2.png)

10. На экране "Завершение мастера импорта сертификатов" проверьте, находится ли сертификат в нужной папке, а затем нажмите кнопку **Готово**, чтобы завершить работу мастера.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-wizard-final-screen.png)

  В случае успешного импорта появится соответствующее диалоговое окно.

11. Войдите на сервер vCenter, чтобы проверить, защищено ли подключение.

  Если при импорте сертификата возникли проблемы и вам не удается установить безопасное подключение, обратитесь к документации по VMware vSphere в разделе [Obtaining Server Certificates](http://pubs.vmware.com/vsphere-60/index.jsp#com.vmware.wssdk.dsg.doc/sdk_sg_server_certificate_Appendixes.6.4.html) (Получение сертификатов сервера).

  Вы можете отключить безопасное подключение, следуя приведенным ниже инструкциям, если в организации защищены границы или вы не хотите использовать протокол HTTPS.

### <a name="disable-secure-communication-protocol"></a>Отключение протокола безопасной связи

Если вашей организации не требуется протокол безопасной связи (HTTPS), отключите протокол HTTPS, как описано ниже. Чтобы отключить поведение по умолчанию, создайте раздел реестра, который игнорирует поведение по умолчанию.

1. Скопируйте приведенный ниже текст и вставьте его в TXT-файл.

  ```
Windows Registry Editor Version 5.00
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft Data Protection Manager\VMWare]
"IgnoreCertificateValidation"=dword:00000001
```
2. Сохраните файл с именем **DisableSecureAuthentication.reg** на Сервере резервного копирования Azure.

3. Дважды щелкните этот файл, чтобы активировать запись реестра.


## <a name="create-a-role-and-user-account-on-the-vcenter-server"></a>Создание роли и учетной записи пользователя на сервере vCenter

На сервере vCenter роль — это предварительно определенный набор привилегий. Администратор сервера vCenter создает роли и связывает учетные записи пользователей с ролями, назначая им определенные разрешения. Чтобы настроить необходимые учетные данные для архивации сервера vCenter, создайте роль с определенными привилегиями и свяжите с ней учетную запись пользователя.

Сервер резервного копирования Azure использует имя пользователя и пароль для аутентификации на сервере vCenter. Сервер резервного копирования Azure использует эти учетные данные пользователя для аутентификации всех операций архивации.

Вот как можно добавить роль сервера vCenter и привилегии для администратора архивации.

1. Войдите на сервер vCenter и на панели **Navigator** (Навигатор) щелкните **Administration** (Администрирование).

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-navigator-panel.png)

2. Чтобы создать роль, в разделе **Administration** (Администрирование) выберите **Roles** (Роли). В области **Roles** (Роли) щелкните значок добавления роли (знак "+").

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-define-new-role.png)

  Откроется диалоговое окно **Создать роль**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-define-new-role-priv.png)

3. В поле **Role name** (Имя роли) диалогового окна **Create Role** (Создание роли) введите *BackupAdminRole*. Это может быть любое имя, описывающее назначение этой роли.

4. Выберите привилегии для соответствующей версии vCenter и нажмите кнопку **ОК**. В следующей таблице перечислены привилегии, необходимые для vCenter 6.0 и vCenter 5.5.

  При выборе привилегий щелкните значок шеврона на родительской метке, чтобы развернуть список дочерних привилегий. Чтобы выбрать необходимые привилегии VirtualMachine, нужно будет развернуть несколько уровней. Вам не нужно выбирать все дочерние привилегии, входящие в родительскую.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-add-privilege-expand.png)

  После нажатия кнопки **OK** новая роль появится в списке в области "Roles" (Роли).

## <a name="create-vcenter-server-user-account-and-permissions"></a>Создание учетной записи пользователя сервера vCenter и разрешений

Создав роль с привилегиями, создайте учетную запись пользователя. Имя и пароль учетной записи пользователя являются учетными данными, используемыми для проверки подлинности.

1. Чтобы создать учетную запись пользователя, на панели "Navigator" (Навигатор) сервера vCenter щелкните **Users and Groups** (Пользователи и группы).

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-userandgroup-panel.png)

  Появится панель "Пользователи и группы".

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/usersandgroups.png)

2. На панели "vCenter Users and Groups" (Пользователи и группы vCenter) выберите вкладку **Users** (Пользователи) и щелкните значок добавления пользователей (знак "+").

  Откроется диалоговое окно "Новый пользователь".

3. В диалоговом окне "Новый пользователь" заполните необходимые поля и нажмите кнопку **ОК**. Например, введите имя пользователя **BackupAdmin**. Необходимо указать надежный пароль.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-new-user-account.png)

  Новая учетная запись пользователя появится в списке.

4. Чтобы связать учетную запись пользователя с ролью на панели "Navigator" (Навигатор), щелкните **Global Permissions** (Глобальные разрешения). На панели "Global Permissions" (Глобальные разрешения) выберите вкладку **Manage** (Управление), а затем щелкните значок добавления (знак "+").

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-add-new-perms.png)

  Откроется диалоговое окно корневой папки глобальных разрешений для добавления разрешения.

5. В диалоговом окне **корневой папки глобальных разрешений для добавления разрешения** нажмите кнопку **Добавить**, чтобы выбрать пользователя или группу.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-add-new-global-perm.png)

  Откроется диалоговое окно для выбора пользователей или групп.

6. В диалоговом окне **Select Users/Groups** (Выбор пользователей или групп) выберите **BackupAdmin** и щелкните **Add** (Добавить).

  Учетная запись пользователя в поле "Users" (Пользователи) отображается в формате *домен*`\`*имя пользователя*. Если вы хотите использовать другой домен, выберите его из списка доменов.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-assign-account-to-role.png)

  Нажмите кнопку **ОК**, чтобы добавить выбранных пользователей в диалоговое окно "Добавление разрешения".

7. Теперь, когда вы определили пользователя, вы можете назначить его роли. В области "Assigned Role" (Назначенная роль) в раскрывающемся меню выберите **BackupAdminRole** и нажмите кнопку **ОК**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-choose-role.png)

  На вкладке управления глобальных разрешений в списке появятся новая учетная запись пользователя и связанная роль.


## <a name="establish-vcenter-server-credentials-on-azure-backup-server"></a>Настройка учетных данных сервера vCenter на Сервере резервного копирования Azure

Прежде чем добавить сервер VMware на Сервер резервного копирования Azure, установите [обновление 1 для Сервера резервного копирования Microsoft Azure](https://support.microsoft.com/help/3175529/update-1-for-microsoft-azure-backup-server).

1. Чтобы открыть Сервер резервного копирования Azure, дважды щелкните соответствующий значок на рабочем столе Сервера резервного копирования Azure.

  ![Значок сервера резервного копирования Azure](./media/backup-azure-backup-server-vmware/mabs-icon.png)

  Если не удается найти значок на рабочем столе, откройте Сервер резервного копирования Azure из списка установленных приложений. Имя приложения Сервера резервного копирования Azure — "Служба архивации Microsoft Azure".

2. На консоли сервера резервного копирования Azure щелкните раздел **Управление**, затем выберите **Рабочие серверы** и на ленте инструментов щелкните **Управление VMware**.

  ![Консоль MABS](./media/backup-azure-backup-server-vmware/add-vmware-credentials.png)

  Откроется диалоговое окно "Управление учетными данными".

  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/mabs-manage-credentials-dialog.png)

3. В диалоговом окне управления учетными данными нажмите кнопку **Добавить**, чтобы открыть диалоговое окно добавления учетных данных.

4. В диалоговом окне "Add Credentials" (Добавление учетных данных) введите имя и описание для новых учетных данных. Затем укажите имя пользователя и пароль. Имя учетных данных (в примере — *Contoso Vcenter credential*) позволит идентифицировать учетные данные в следующей процедуре. Используйте те же имя пользователя и пароль, которые использовались для сервера vCenter. Если сервер vCenter и Сервер резервного копирования Azure размещены в разных доменах, укажите домен в имени пользователя.

  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/mabs-add-credential-dialog2.png)

  Нажмите кнопку **Добавить**, чтобы добавить новые учетные данные на сервер резервного копирования Azure. Новые учетные данные появятся в списке диалогового окна управления учетными данными.
  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/new-list-of-mabs-creds.png)

5. Чтобы закрыть диалоговое окно "Manage Credentials" (Управление учетными данными), нажмите кнопку **X** в правом верхнем углу.


## <a name="add-the-vcenter-server-to-azure-backup-server"></a>Добавление сервера vCenter на Сервер резервного копирования Azure

Чтобы открыть мастер добавления рабочего сервера, сделайте следующее.

1. На консоли сервера резервного копирования Azure щелкните раздел **Управление**, затем выберите **Рабочие серверы** и щелкните **Добавить**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vcenter-to-mabs.png)

  Откроется мастер добавления рабочего сервера.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/production-server-add-wizard.png)

2. На экране выбора типа рабочего сервера выберите серверы VMware и нажмите кнопку **Далее**.

3. В поле "Имя или IP-адрес сервера:" укажите полное доменное имя сервера VMware или его IP-адрес. Если всеми серверами ESXi управляет один сервер vCenter, то вы можете ввести имя сервера vCenter.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vmware-server-provide-server-name.png)

4. В поле **SSL-порт** введите порт, используемый для взаимодействия с сервером VMware. Если вам не нужно вводить какой-то определенный порт, используйте порт 443, который является портом по умолчанию.

5. В диалоговом окне **Укажите учетные данные** выберите созданные учетные данные.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/identify-creds.png)

6. Нажмите кнопку **Добавить**, чтобы добавить сервер VMware в список **Добавлены серверы VMware**, а затем нажмите кнопку **Далее**, чтобы перейти к следующему экрану мастера.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vmware-server-credentials.png)

7. На экране **Задачи** нажмите кнопку **Добавить**, чтобы добавить указанный сервер VMware на сервер резервного копирования Azure.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/tasks-screen.png)

  Так как резервное копирование сервера VMware происходит без агента, вы можете добавить новый сервер за несколько секунд. На экране окончания показаны результаты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/summary-screen.png)

  Чтобы добавить несколько серверов vCenter на Сервер резервного копирования Azure, повторно выполните описанные выше действия.

После добавления сервера VMware на Сервер резервного копирования Azure следует создать группу защиты. Группа защиты указывает различные детали для краткосрочного или долгосрочного хранения. В этой группе вы определяете и применяете политику архивации. С помощью политики архивации вы определяете, какие данные подлежат резервному копированию, а также когда оно будет происходить.


## <a name="configure-a-protection-group"></a>Настройка группы защиты

Если вы прежде не использовали сервер резервного копирования Azure или System Center Data Protection Manager, ознакомьтесь со статьей [Планирование резервного копирования на диски](https://technet.microsoft.com/library/hh758026.aspx), чтобы подготовить аппаратную среду. Если у вас есть подходящее хранилище, используйте мастер создания групп защиты, чтобы добавить виртуальные машины VMware.

1. На консоли сервера резервного копирования Azure щелкните раздел **Защита**, затем выберите на ленте инструментов **Создать**, чтобы открыть мастер создания групп защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/open-protection-wizard.png)

  Откроется мастер создания групп защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/protection-wizard.png)

  Нажмите кнопку **Далее**, чтобы перейти к экрану **Выбор типа группы защиты**.

2. На экране выбора типа группы защиты выберите **Серверы** и нажмите кнопку **Далее**.

3. На экране выбора элементов группы вы увидите список доступных и выбранных элементов. Выберите элементы, которые вы хотите защитить, и нажмите кнопку **Далее**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/server-add-selected-members.png)

  Если при выборе элемента вы выберете папку, содержащую другие папки или виртуальные машины, эти элементы также будут выбраны. Включение папок и виртуальных машин в родительскую папку называется защитой на уровне папки. Вы можете исключить любую папку или виртуальную машину, убрав соответствующие флажки.

  Если виртуальная машина или папка, содержащая виртуальную машину, уже защищена в Azure, вы не сможете выбрать эту виртуальную машину снова. Таким образом, если виртуальная машина защищена в Azure, для нее нельзя включить повторную защиту, что предотвращает создание повторяющихся точек восстановления для одной виртуальной машины. Если вы хотите увидеть, какой сервер резервного копирования Azure уже защищает элемент, наведите указатель мыши на нужный элемент.

4. На экране выбора метода защиты данных введите имя для группы защиты. Выбраны краткосрочная защита (на диск) и оперативная защита. Если вы хотите использовать оперативную защиту (в Azure), необходимо использовать краткосрочную защиту на диск. Нажмите кнопку **Далее**, чтобы перейти к диапазону краткосрочной защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/name-protection-group.png)

5. На экране указания краткосрочных целей в **диапазоне хранения** укажите число дней, в течение которых необходимо хранить точки восстановления *на диске*. Чтобы изменить время и дни создания точек восстановления, нажмите кнопку **Изменить**. Точки восстановления краткосрочного хранения представляют собой полные резервные копии. Для них не выполняется добавочное резервное копирование. Если вы задали необходимые параметры для краткосрочных целей, нажмите кнопку **Далее**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/short-term-goals.png)

6. На экране проверки выделения места на диске просмотрите объем места на диске для виртуальных машин и при необходимости измените его. Рекомендуемые выделения дискового пространства основаны на диапазоне хранения, указанном на предыдущем экране, типе рабочей нагрузки и размере защищенных данных (определенных на шаге 3).  

  - Размер данных — размер данных в группе защиты.
  - Место на диске — объем дискового пространства, рекомендуемый для группы защиты. Если вы хотите изменить этот параметр, вы должны выделить общее место, объем которого будет немного превышать рассчитанный вами объем для каждого источника данных.
  - Совместное размещение данных — если вы включите совместное размещение, несколько источников данных в защите могут сопоставиться с отдельной репликой и томом точек восстановления. Совместное размещение поддерживается не для всех рабочих нагрузок.
  - Автоматическое увеличение — если вы включите этот параметр, то в случае, когда объем данных в защищенной группе превысит первоначальный выделенный объем, DPM попытается увеличить размер диска на 25 %.
  - Дополнительные сведения о пуле носителей — этот параметр отображает текущее состояние пула носителей, включая общий размер диска и оставшийся.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/review-disk-allocation.png)

  Если вас устраивает выделенное пространство, нажмите кнопку **Далее**.

7. На экране выбора метода создания реплики укажите, каким образом нужно создавать начальную копию или реплику защищенных данных на сервере резервного копирования Azure.

  Значения по умолчанию — **Автоматически по сети** и **Сейчас**. Если вы используете значения по умолчанию, укажите время низкой нагрузки. Выберите параметр **Позже** и укажите день и время.

  Для больших объемов данных или неоптимальных условий сети лучше выбрать автономную репликацию данных с использованием съемных носителей.

  Задав параметры по своему усмотрению, нажмите кнопку **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/replica-creation.png)

8. На экране **параметров проверки согласованности** выберите способ проверок на согласованность и время их автоматизации. В случае несогласованности данных реплики вы можете запустить проверку на согласованность, либо эта проверка может выполняться по расписанию.

  Если настраивать автоматическую проверку согласованности не требуется, можно выполнить проверку вручную. В области "Защита" консоли Сервера резервного копирования Azure щелкните правой кнопкой мыши группу защиты и выберите **Выполнить проверку согласованности**.

  Чтобы перейти к следующему экрану, нажмите кнопку **Далее**.

9. На экране **указания данных для оперативной защиты** выберите источники данных, которые необходимо защитить. Вы можете выбрать элементы по отдельности или же щелкнуть **Выделить все**. Выбрав необходимые элементы, нажмите кнопку **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/select-data-to-protect.png)

10. На экране **указания расписания архивации в сети** укажите расписание для создания точек восстановления из резервной копии диска. Созданная точка восстановления передается в хранилище служб восстановления в Azure. Когда вы зададите параметры расписания оперативного резервного копирования, нажмите кнопку **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/online-backup-schedule.png)

11. На экране указания политики хранения в сети укажите необходимое время хранения резервных копий данных в Azure. После определения политики щелкните **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/retention-policy.png)

  Временных ограничений для хранения данных в Azure нет. Если вы храните данные точек восстановления в Azure, единственным ограничением является количество точек восстановления — не более 9999 на защищенный экземпляр. В этом примере защищенным экземпляром является сервер VMware.

12. На странице сводки вы можете просмотреть сведения для группы защиты. Обратите внимание на элементы группы и параметры. Когда вы зададите все необходимые параметры, нажмите кнопку "Создать группу".

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/protection-group-summary.png)

# <a name="back-up-vmware-server-to-azure"></a>Резервное копирование сервера VMware в Azure

В этой статье описывается, как подключить сервер VMware к серверу резервного копирования Azure, чтобы выполнять резервное копирование содержимого сервера VMware в облако. Чтобы приступить к работе с этим руководством, вам необходимо установить сервер резервного копирования Azure.

## <a name="create-secure-connection-to-vmware-server"></a>Создание безопасного подключения к серверу VMware

Чтобы защитить сервер VMware, необходимо установить к нему безопасное подключение от сервера резервного копирования Azure. Чтобы обеспечить безопасное подключение, установите допустимый сертификат на сервере VMware и сервере резервного копирования Azure.

Если при подключении к серверу VMware URL-адрес является небезопасным, тогда вам необходимо экспортировать сертификат, чтобы обеспечить безопасное подключение к сайту.
![Пример незащищенного подключения к серверу VMware](./media/backup-azure-backup-server-vmware/unsecure-url.png)

1. Щелкните https (с зачеркиванием), а затем во всплывающем меню щелкните ссылку подробных сведений.

  В зависимости от браузера вам может потребоваться щелкнуть **Параметры** > **Дополнительные средства** > **Инструменты разработчика**, а затем выбрать вкладку "Безопасность".

  ![Пример сообщения об ошибке о незащищенном подключении](./media/backup-azure-backup-server-vmware/security-tab.png)

2. В разделе сведений на вкладке "Безопасность" щелкните **Просмотр сертификата**.

  ![Пример сообщения об ошибке о незащищенном подключении](./media/backup-azure-backup-server-vmware/security-tab-view-certificate.png)

  Откроется диалоговое окно сертификата.

3. В этом диалоговом окне щелкните вкладку пути сертификации.  

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-certification-path.png)

  Выделенный выше сертификат не является доверенным, так как не найден издатель. Сертификат может быть недоверенным и по другим причинам.

4. Чтобы экспортировать сертификат на локальный компьютер, щелкните вкладку "Подробности", а затем щелкните "Копировать в файл".

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-details-tab.png)

  Откроется мастер экспорта сертификатов.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-wizard1.png)

  Щелкните **Далее** для перехода к следующим страницам мастера.

5. На экране "Формат экспортируемого файла" укажите предпочитаемый формат файла сертификата. Если он не важен, примите для сертификата формат файла по умолчанию, а затем щелкните **Далее**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-wizard1b.png)

6. На экране "Имя экспортируемого файла" присвойте сертификату имя, а затем щелкните "Обзор", чтобы выбрать расположение, где будет сохранен сертификат на локальном компьютере. Сохраните сертификат в удобном расположении.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/certificate-wizard2.png)

7. После экспорта сертификата перейдите в то расположение, где он был сохранен, щелкните его правой кнопкой мыши и выберите в меню **Установить сертификат**.

  Откроется мастер импорта сертификатов.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-import-wizard1.png)

8. В мастере импорта сертификатов выберите **Локальный компьютер** в качестве места назначения для сертификата, а затем щелкните **Далее**, чтобы продолжить.

9. На экране хранилища сертификатов выберите **Поместить все сертификаты в следующее хранилище**, а затем щелкните **Обзор**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-import-wizard-local-store.png)

  Откроется диалоговое окно "Выбор хранилища сертификата".

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-store.png)

  Выберите конечную папку для сертификатов и нажмите кнопку **ОК**. Если вы не знаете, какую папку использовать, выберите доверенные корневые центры сертификации. В диалоговом окне выбора хранилища сертификата появится выбранная конечная папка. Щелкните **Далее**, чтобы импортировать сертификат.

10. На экране "Завершение мастера импорта сертификатов" проверьте, находится ли сертификат в конечной папке, а затем нажмите кнопку "Готово", чтобы завершить работу мастера.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-wizard-final-screen.png)

  В случае успешного импорта появится соответствующее диалоговое окно.

11. Выполните вход на виртуальную машину VMware, чтобы убедиться в безопасном подключении к серверу VMware. Сервер резервного копирования Azure подключится к серверу VMware через защищенный канал HTTPS. Вы можете отключить безопасное подключение через реестр, если в организации защищены границы или вы не хотите использовать протокол HTTPS. Тем не менее мы рекомендуем установить сертификаты на сервере резервного копирования Azure и сервере VMware, чтобы включить безопасное подключение.


## <a name="create-role-and-user-account-on-vmware-server"></a>Создание роли и учетной записи пользователя на сервере VMware

Сервер резервного копирования Azure взаимодействует с удаленным сервером VMware, выполняя аутентификацию указанных учетных данных пользователя VMware. Сервер резервного копирования Azure выполняет проверку подлинности учетных данных пользователя VMware для всех операций резервного копирования. Используйте мастер добавления рабочего сервера на сервере резервного копирования Azure, чтобы обеспечить безопасное взаимодействие с сервером VMware. Настройка связи между сервером резервного копирования Azure и сервером VMware выполняется в три этапа. 

- Создание роли пользователя с назначенными привилегиями.
- Создание учетной записи пользователя с учетными данными — именем пользователя и паролем.
- Добавление учетной записи пользователя сервера VMware на сервер резервного копирования Azure

при работе с мастером добавления рабочего сервера. Имя пользователя и пароль сохраняются как учетные данные.


### <a name="create-user-role-and-add-privileges"></a>Создание роли пользователя и добавление привилегий
Учетной записи пользователя VMware, указанной в учетных данных сервера резервного копирования Azure, должны быть назначены определенные связанные привилегии. Но так как привилегии связаны с ролью пользователя, нам сначала необходимо создать эту роль, а затем добавить для нее определенные привилегии. Привилегии, связанные с ролью пользователя, назначаются администратору резервного копирования.

1. Чтобы создать новую роль пользователя VMware, выполните вход на сервер VMware, а затем в области **навигатора** выберите раздел **Администрирование**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-navigator-panel.png)

2. Чтобы создать новую роль, выберите в разделе **администрирования** **Роли** и щелкните в **этой** области значок добавления роли (знак "+").

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-define-new-role.png)

  Откроется диалоговое окно **Создать роль**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-define-new-role-priv.png)

3. Введите имя роли в поле **Имя роли** диалогового окна **Создание роли**. Для этого примера мы будем использовать имя *BackupAdminRole*. Это может быть любое имя, распознаваемое для этой роли.

4. Выберите привилегии, чтобы применить их к роли пользователя. Выберите привилегии из списка ниже. При выборе привилегий щелкните значок шеврона на родительской метке, чтобы раскрыть список дочерних привилегий. Вам не нужно выбирать все дочерние привилегии, входящие в родительскую.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/cert-add-privilege-expand.png)

  - Privilege.Datastore.AllocateSpace.label
  - Privilege.Global.ManageCustomerFields.label
  - privilege.Network.Assign.label
  - Privilege.VirtualMachine.Config.AddNewDisk.label
  - Privilege.VirtualMachine.Config.AdvanceConfig.label
  - Privilege.VirtualMachine.Config.ChangeTracking.label
  - Privilege.VirtualMachine.Config.HostUSBDevice.label
  - Privilege.VirtualMachine.Config.SwapPlacement.label  
  - Privilege.VirtualMachine.Interact.PowerOff.label
  - Privilege.VirtualMachine.Inventory.Create.label
  - Privilege.VirtualMachine.Provisioning.DiskRandomRead.summary
  - Privilege.VirtualMachine.State.CreateSnapshot.label
  - Privilege.VirtualMachine.State.RemoveSnapshot.label
</br>

  После того как вы выберете привилегии, нажмите кнопку **ОК**. В списке в области ролей появится новая роль.

### <a name="create-a-user-account-and-assign-permissions"></a>Создание учетной записи пользователя и назначение разрешений

Определив роль пользователя с привилегиями, вы можете создать учетную запись пользователя. Во время создания учетной записи пользователя вам необходимо назначить ее определенной роли пользователя, которая предоставляет этой учетной записи связанные привилегии. Имя и пароль учетной записи пользователя являются учетными данными, используемыми для проверки подлинности.

1. Чтобы создать новую учетную запись пользователя, в области навигатора на сервере VMware щелкните **Пользователи и группы**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-userandgroup-panel.png)

  Появится панель "Пользователи и группы".

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/usersandgroups.png)

2. На вкладке "Пользователи" панели "Пользователи и группы" на сервере VMware щелкните значок добавления пользователей (знак "+").

  Откроется диалоговое окно "Новый пользователь".

3. В созданной учетной записи пользователя пара "имя пользователя и пароль" будет использоваться как учетные данные. В диалоговом окне "Новый пользователь" заполните необходимые поля и нажмите кнопку **ОК**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-new-user-account.png)

  Новая учетная запись пользователя появится в списке.

4. После того как вы создали учетную запись пользователя, необходимо связать ее с ролью пользователя (у которой есть необходимые разрешения). В области навигатора щелкните раздел **Глобальные разрешения**. На панели глобальных разрешений щелкните вкладку **Управление**, а затем щелкните значок добавления (знак "+").

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-add-new-perms.png)

  Откроется диалоговое окно корневой папки глобальных разрешений для добавления разрешения.

5. В диалоговом окне **корневой папки глобальных разрешений для добавления разрешения** нажмите кнопку **Добавить**, чтобы выбрать пользователя или группу.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-add-new-global-perm.png)

  Откроется диалоговое окно для выбора пользователей или групп.

6. В диалоговом окне **Select Users/Groups** (Выбор пользователей или групп) выберите созданную учетную запись пользователя и нажмите кнопку **Добавить**. Выбранная учетная запись пользователя появится в поле "Пользователи". Имя пользователя появится в поле "Пользователи" в формате *домен*`\`*имя пользователя*.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-assign-account-to-role.png)

  Нажмите кнопку **ОК**, чтобы добавить выбранных пользователей в диалоговое окно "Добавление разрешения".

7. Теперь, когда вы определили пользователя, вы можете назначить его роли. В области назначения роли в раскрывающемся меню выберите роль и нажмите кнопку **ОК**.

  ![Диалоговое окно сертификата с ошибкой ](./media/backup-azure-backup-server-vmware/vmware-choose-role.png)

  На вкладке управления глобальных разрешений в списке появятся новая учетная запись пользователя и связанная роль.


### <a name="add-the-vmware-user-account-credentials-to-azure-backup-server"></a>Добавление учетных данных учетной записи пользователя VMware на сервер резервного копирования Azure

Перед тем как добавить сервер VMware на сервер резервного копирования Azure, установите [обновление 1 для сервера службы архивации Microsoft Azure](https://support.microsoft.com/help/3175529/update-1-for-microsoft-azure-backup-server).

1. Чтобы открыть консоль сервера резервного копирования Azure, щелкните значок ниже (расположенный на рабочем столе сервера).

  ![Значок сервера резервного копирования Azure](./media/backup-azure-backup-server-vmware/mabs-icon.png)

  Если не удается найти значок на рабочем столе, вы можете открыть сервер резервного копирования Azure из списка установленных приложений. В этом списке приложений этот сервер называется службой архивации Microsoft Azure.

2. На консоли сервера резервного копирования Azure щелкните раздел **Управление**, затем выберите **Рабочие серверы** и на ленте инструментов щелкните **Управление VMware**.

  ![Консоль MABS](./media/backup-azure-backup-server-vmware/add-vmware-credentials.png)

  Откроется диалоговое окно "Управление учетными данными".

  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/mabs-manage-credentials-dialog.png)

3. В диалоговом окне управления учетными данными нажмите кнопку **Добавить**, чтобы открыть диалоговое окно добавления учетных данных.

4. В диалоговом окне добавления учетных данных введите имя и описание для новых учетных данных. Имя пользователя и пароль должны совпадать с именем и паролем, указанными при создании учетной записи пользователя на сервере VMware.

  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/mabs-add-credential-dialog.png)

  Нажмите кнопку **Добавить**, чтобы добавить новые учетные данные на сервер резервного копирования Azure. Новые учетные данные появятся в списке диалогового окна управления учетными данными.
  ![Диалоговое окно управления учетными данными MABS](./media/backup-azure-backup-server-vmware/new-list-of-mabs-creds.png)

5. Чтобы закрыть диалоговое окно управления учетными данными, нажмите кнопку **X** в правом верхнем углу.


## <a name="add-vmware-server-to-azure-backup-server"></a>Добавление сервера VMware на сервер резервного копирования Azure

Чтобы открыть мастер добавления рабочего сервера:

1. На консоли сервера резервного копирования Azure щелкните раздел **Управление**, затем выберите **Рабочие серверы** и щелкните **Добавить**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vcenter-to-mabs.png)

  Откроется мастер добавления рабочего сервера.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/production-server-add-wizard.png)

2. На экране выбора типа рабочего сервера выберите серверы VMware и нажмите кнопку **Далее**.

3. В поле "Имя или IP-адрес сервера:" укажите полное доменное имя сервера VMware или его IP-адрес. Если все серверы ESXi находятся под управлением одного vCenter, вы можете ввести имя VMware.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vmware-server-provide-server-name.png)

4. В поле **SSL-порт** введите порт, используемый для взаимодействия с сервером VMware. Если вам не нужно вводить какой-то определенный порт, используйте порт 443, который является портом по умолчанию.

5. В поле **Укажите учетные данные** вы можете создать новые учетные данные или же выбрать существующие. В разделе выше вы создали учетные данные. Выберите их.

  Если у вас нет доступных учетных данных или вам нужно создать новые, щелкните **Добавить новые учетные данные**, а затем после их создания нажмите кнопку **ОК**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/specify-new-cred.png)

6. Нажмите кнопку **Добавить**, чтобы добавить сервер VMware в список **добавленных серверов VMware**, а затем нажмите кнопку "Далее", чтобы перейти к следующему экрану мастера.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/add-vmware-server-credentials.png)

7. На экране **Задачи** нажмите кнопку **Добавить**, чтобы добавить указанный сервер VMware на сервер резервного копирования Azure.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/tasks-screen.png)

  Так как резервное копирование сервера VMware происходит без агента, вы можете добавить новый сервер за несколько секунд. Вы можете добавить несколько серверов VMware на сервер резервного копирования Azure, повторно выполнив описанные выше действия.

Теперь, когда вы добавили сервер VMware на сервер резервного копирования Azure, необходимо создать группу защиты. Группа защиты указывает различные детали для краткосрочного или долгосрочного хранения. В этой группе вы определяете и применяете политику архивации. С помощью политики архивации вы определяете, какие данные подлежат резервному копированию, а также когда оно будет происходить.


## <a name="configure-a-protection-group-to-back-up-vmware-server"></a>Настройка группы защиты для резервного копирования сервера VMware

Если вы прежде не использовали сервер резервного копирования Azure или System Center Data Protection Manager, ознакомьтесь со статьей [Планирование резервного копирования на диски](https://technet.microsoft.com/library/hh758026.aspx), чтобы подготовить аппаратную среду. Если у вас есть необходимое хранилище, используйте мастер создания групп защиты, чтобы добавить определенные виртуальные машины.

1. На консоли сервера резервного копирования Azure щелкните раздел **Защита**, затем выберите на ленте инструментов **Создать**, чтобы открыть мастер создания групп защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/open-protection-wizard.png)

  Откроется мастер создания групп защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/protection-wizard.png)

  Нажмите кнопку **Далее**, чтобы перейти к экрану **Выбор типа группы защиты**.

2. На экране выбора типа группы защиты выберите **Серверы** и нажмите кнопку **Далее**.

3. На экране выбора элементов группы вы увидите список доступных и выбранных элементов. Выберите элементы, которые вы хотите защитить, и нажмите кнопку **Далее**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/server-add-to-selected-members.png)

  Если при выборе элемента вы выберете папку, содержащую другие папки или виртуальные машины, эти элементы также будут выбраны. Включение папок и виртуальных машин в родительскую папку называется защитой на уровне папки. Вы можете исключить любую папку или виртуальную машину, убрав соответствующие флажки.

  Если виртуальная машина или папка, содержащая виртуальную машину, уже защищена в Azure, вы не сможете выбрать эту виртуальную машину снова. Таким образом, если виртуальная машина защищена в Azure, для нее нельзя включить повторную защиту, что предотвращает создание повторяющихся точек восстановления для одной виртуальной машины. Если вы хотите увидеть, какой сервер резервного копирования Azure уже защищает элемент, наведите указатель мыши на нужный элемент.

4. На экране выбора метода защиты данных введите имя для группы защиты. Затем для **метода защиты** выберите расположение для резервного копирования данных. Для обеспечения краткосрочной защиты данные копируются на диск. Для обеспечения долгосрочной защиты данные копируются в облако (Azure). Нажмите кнопку **Далее**, чтобы перейти к диапазону краткосрочной защиты.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/name-protection-group.png)

5. На экране указания краткосрочных целей в **диапазоне хранения** укажите число дней, в течение которых необходимо хранить точки восстановления *на диске*. Чтобы изменить время и дни создания точек восстановления, нажмите кнопку **Изменить**. Точки восстановления краткосрочного хранения представляют собой полные резервные копии. Для них не выполняется добавочное резервное копирование. Если вы задали необходимые параметры для краткосрочных целей, нажмите кнопку **Далее**.

  ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/short-term-goals.png)

6. На экране проверки выделения места на диске просмотрите объем места на диске для виртуальных машин и при необходимости измените его. Рекомендуемые выделения дискового пространства основаны на диапазоне хранения, указанном на предыдущем экране, типе рабочей нагрузки и размере защищенных данных (определенных на шаге 3).  

  - Размер данных — размер данных в группе защиты.
  - Место на диске — объем дискового пространства, рекомендуемый для группы защиты. Если вы хотите изменить этот параметр, вы должны выделить общее место, объем которого будет немного превышать рассчитанный вами объем для каждого источника данных.
  - Совместное размещение данных — если вы включите совместное размещение, несколько источников данных в защите могут сопоставиться с отдельной репликой и томом точек восстановления. Совместное размещение поддерживается не для всех рабочих нагрузок.
  - Автоматическое увеличение — если вы включите этот параметр, то в случае, когда объем данных в защищенной группе превысит первоначальный выделенный объем, DPM попытается увеличить размер диска на 25 %.
  - Дополнительные сведения о пуле носителей — этот параметр отображает текущее состояние пула носителей, включая общий размер диска и оставшийся.

    ![Мастер добавления рабочего сервера](./media/backup-azure-backup-server-vmware/review-disk-allocation.png)

  Когда вы зададите все необходимые параметры, нажмите кнопку **Далее**.

7. На экране выбора метода создания реплики укажите, каким образом нужно создавать начальную копию или реплику защищенных данных на сервере резервного копирования Azure.

  Значения по умолчанию — **Автоматически по сети** и **Сейчас**. Если вы используете значения по умолчанию, укажите время низкой нагрузки. Выберите параметр **Позже** и укажите день и время.

  Для больших объемов данных или неоптимальных условий сети лучше выбрать автономную репликацию данных с использованием съемных носителей.

  Задав параметры по своему усмотрению, нажмите кнопку **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/replica-creation.png)

8. На экране **параметров проверки согласованности** выберите способ проверок на согласованность и время их автоматизации. В случае несогласованности данных реплики вы можете запустить проверку на согласованность, либо эта проверка может выполняться по расписанию.

  Если вы не хотите настраивать автоматическую проверку согласованности, вы можете запустить проверку вручную в любое время. Для этого щелкните правой кнопкой мыши группу защиты в области защиты на консоли сервера резервного копирования Azure и выберите выполнение проверки согласованности.

  Чтобы перейти к следующему экрану, нажмите кнопку **Далее**.

9. На экране **указания данных для оперативной защиты** выберите источники данных, которые необходимо защитить. Вы можете выбрать элементы по отдельности или же щелкнуть **Выделить все**. Выбрав необходимые элементы, нажмите кнопку *Далее*.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/select-data-to-protect.png)

10. На экране **указания расписания архивации в сети** укажите расписание для создания точек восстановления из резервной копии диска. Созданная точка восстановления передается в хранилище служб восстановления в Azure. Когда вы зададите параметры расписания оперативного резервного копирования, нажмите кнопку **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/online-backup-schedule.png)

11. На экране указания политики хранения в сети укажите необходимое время хранения резервных копий данных в Azure. После определения политики щелкните **Далее**.

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/retention-policy.png)

  Временных ограничений для хранения данных в Azure нет. Если вы храните данные точек восстановления в Azure, единственным ограничением является количество точек восстановления — не более 9999 на защищенный экземпляр. В этом примере защищенным экземпляром является сервер VMware.

12. На странице сводки вы можете просмотреть сведения для группы защиты. Обратите внимание на элементы группы и параметры. Когда вы зададите все необходимые параметры, нажмите кнопку "Создать группу".

  ![Создание мастера создания новой группы защиты](./media/backup-azure-backup-server-vmware/protection-group-summary.png)

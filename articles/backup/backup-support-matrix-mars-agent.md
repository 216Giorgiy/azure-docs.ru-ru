# <a name="support-matrix-for-backup-with-the-microsoft-azure-recovery-services-mars-agent"></a>Матрица поддержки для резервного копирования с помощью агента Служб восстановления Microsoft Azure (MARS)

С помощью [службы Azure Backup](backup-overview.md) можно выполнять резервное копирование локальных компьютеров и приложений, а также виртуальных машин Azure. В этой статье приведены сведения о параметрах и ограничениях поддержки для резервного копирования компьютеров с использованием агента Служб восстановления Microsoft Azure (MARS).

## <a name="about-the-mars-agent"></a>Общие сведения об агенте MARS

Агент MARS используется в службе Azure Backup для резервного копирования данных с локальных компьютеров и виртуальных машин Azure в резервное хранилище Служб восстановления в Azure. Агент MARS можно использовать следующим образом.
- Запустить агент на локальных компьютерах Windows, чтобы выполнять резервное копирование непосредственно в резервное хранилище Служб восстановления в Azure.
- Запустить агент на виртуальных машинах Windows Azure, чтобы выполнять резервное копирование непосредственно в хранилище.
- Запустить агент на сервере Microsoft Azure Backup Server (MABS) или на сервере System Center Data Protection Manager (DPM). В этом сценарии резервные копии данных компьютеров и рабочих нагрузок создаются на сервере MABS или DPM, а затем MABS или DPM выполняет резервное копирование в хранилище Azure с помощью агента MARS. 

Целевой объект резервного копирования зависит от того, где установлен агент.

- [Дополнительные сведения](backup-architecture.md#architecture-direct-backup-of-on-premises-windows-machinesazure-vm-filesfolders) об архитектуре резервного копирования при использовании агента MARS.
- [Дополнительные сведения]() об [архитектуре резервного копирования](backup-architecture.md#architecture-back-up-to-dpmmabs) с использованием сервера MABS или DPM и [требования](backup-support-matrix-mabs-dpm.md) к нему.


## <a name="supported-installation"></a>Поддерживаемая установка

**Установка** | **Дополнительные сведения**
--- | ---
**Скачать последнюю версию агента MARS** | Последнюю версию агента можно скачать из хранилища или [напрямую](https://aka.ms/azurebackup_agent).
**Установить непосредственно на компьютере** | Вы можете установить агент MARS непосредственно на локальном сервере Windows или на виртуальной машине Microsoft Azure под управлением любой из [поддерживаемых операционных систем]().
**Установить на сервере резервного копирования** | При настройке сервера DPM или MABS для резервного копирования в Azure нужно скачать и установить агент MARS на этом сервере. Агент можно установить в одной из [поддерживаемых операционных систем](backup-support-matrix-mabs-dpm.md#supported-mabs-and-dpm-operating-systems), указанных в матрице поддержки сервера резервного копирования.

> [!NOTE]
> По умолчанию на виртуальные машины Azure, для которых включено резервное копирование, установлено расширение Azure Backup. Это расширение создает резервную копию всей виртуальной машины. Вы можете установить и запустить агент MARS на виртуальной машине Azure вместе с расширением, если хотите выполнить резервное копирование определенных папок или файлов, а не всей виртуальной машины.
> При запуске агента MARS на виртуальной машине Azure он создает резервные копии файлов и папок, находящихся во временном хранилище виртуальной машины. Резервное копирование завершится ошибкой, если файлы и папки будут удалены из временного хранилища или если само временное хранилище будет удалено.


## <a name="cache-folder-support"></a>Поддержка папок кэширования

При выполнении резервного копирования с помощью агента MARS агент делает моментальный снимок данных и сохраняет его в локальной папке кэширования перед отправкой в Azure. Для папки кэширования (временных файлов) существует ряд требований.

**Кэш.** | **Дополнительные сведения**
--- | ---
**Объем кэша** |  Свободное место в папке кэширования должно составлять не менее 5–10 % от общего размера данных резервного копирования. 
**Расположение кэша** | Папка кэширования должна храниться локально на компьютере, для которого выполняется резервное копирование, и должна быть подключена к сети.<br/><br/> Папка кэширования не должна находиться в общей сетевой папке, на съемном носителе или в автономном томе. 
**Папка кэширования** | Папка кэширования должна быть зашифрована на дедуплицированном томе или в папке, которая является сжатой, разреженной или точкой повторного анализа.
**Изменение расположения кэша** | Вы можете изменить расположение кэша, остановив работу модуля резервного копирования (net stop bengine) и скопировав папку кэширования на новый диск (убедитесь, что на нем достаточно места). После этого нужно изменить две записи реестра в файле HKLM\SOFTWARE\Microsoft\Windows Azure Backup (Config/ScratchLocation и Config/CloudBackupProvider/ScratchLocation), указав значение нового расположения, и перезапустить модуль.

Windows Server 2008 с пакетом обновления 2 (SP2) | 1700 ГБ
Windows 8 или более поздняя версия  | 54 400 ГБ
Windows 7   | 1700 ГБ

## <a name="support-for-initial-offline-backup"></a>Поддержка начального автономного резервного копирования

Azure Backup поддерживает "автономное заполнение" для передачи начальных резервных копий данных в Azure с использованием дисков. Это полезно, если ваша начальная резервная копия имеет размер в несколько терабайт (ТБ). Автономное резервное копирование поддерживается для:

- прямого резервного копирования файлов и папок локальных компьютеров, на которых запущен агент MARS;
- резервного копирования рабочих нагрузок и файлов с сервера DPM или MABS;
- автономного резервного копирования, которое не может использоваться для файлов состояния системы.


## <a name="support-for-restore"></a>Поддержка восстановления

- Новая версия [мгновенного восстановления](/backup-instant-restore-capability.md) Azure Backup позволяет восстанавливать данные до их копирования в хранилище.<br/><br/> Чтобы использовать эту функцию, на компьютере, на котором выполняется резервное копирование, должна быть запущена платформа .NET Framework 4.5.2 или более поздняя версия.
- Резервные копии невозможно восстановить на целевой компьютер с более ранней версии операционной системы. Например, резервную копию, созданную на компьютере Windows 7, можно восстановить на компьютере Windows 8 или более поздней версии, а резервную копию, созданную на компьютере Windows 8, невозможно восстановить на компьютере Windows 7.

# <a name="troubleshooting-azure-backup-agent-configurationregistration-issues"></a>Устранение неполадок с настройкой и регистрацией агента Microsoft Azure Backup
## <a name="recommended-steps"></a>Рекомендуемые действия
Описанные ниже рекомендации позволяют устранить соответствующие ошибки, с которыми вы можете столкнуться при настройке или регистрации агента Microsoft Azure Backup.

## <a name="error-invalid-vault-credentials-provided-the-file-is-either-corrupted-or-does-not-have-the-latest-credentials-associated-with-recovery-service"></a>Ошибка. Указаны недопустимые учетные данные хранилища. Файл либо поврежден, либо не содержит последних учетных данных, связанных со службой восстановления.
| Сведения об ошибке | Возможные причины | Рекомендуемые действия |
| ---     | ---     | ---    |
| **Ошибка** </br> *Указаны недопустимые учетные данные хранилища. Файл либо поврежден, либо не содержит последних учетных данных, связанных со службой восстановления. (ID: 34513)* | <ol><li> Недопустимые учетные данные хранилища (например, эти учетные данные были скачаны более чем за 48 часов до регистрации).<li>   Агент Microsoft Azure Backup не может скачать временный файл во временную папку Windows. <li>Учетные данные хранилища находятся в сетевой папке. <li>Отключен протокол TLS 1.0.<li> Настроенный прокси-сервер блокирует подключения. <br> |  <ol><li>Скачайте новые учетные данные хранилища.<li>Перейдите к разделу "Обозреватель" > "Безопасность" > "Интернет", затем щелкните "Другой...", прокрутите вниз до раздела скачивания файла и выберите действие "Включить".<li>Возможно, узел следует добавить в список [надежных сайтов](https://docs.microsoft.com/en-us/azure/backup/backup-try-azure-backup-in-10-mins#network-and-connectivity-requirements).<li>Измените параметры прокси-сервера и предоставьте правильные сведения о нем. <li> Убедитесь, что значения даты и времени (UTC) совпадают с настроенными на компьютере.<li>Перейдите к папке C:/Windows/Temp, и если там скопилось более 60 000 или 65 000 файлов (с расширением .tmp), удалите эти файлы.<li>Для проверки запустите на сервере пакет SDP. Если появится ошибка с сообщением о запрете скачивать файлы, это, скорее всего, означает превышение допустимого количества файлов в каталоге C:/Windows/Temp.<li>Убедитесь, что вы используете платформу .NET Framework 4.6.2. <li>Если TLS 1.0 отключен для соблюдения стандартов PCI, используйте рекомендации по устранению неполадок, представленные по [этой ссылке](https://support.microsoft.com/help/4022913). <li>Если на сервере установлена антивирусная программа, исключите из проверки в ней следующие файлы. <ol><li>CBengine.exe.<li>CSC.exe (связан с .NET Framework. Для каждой версии .NET, установленной на сервере, существует отдельный файл CSC.exe. Нужно исключить все файлы CSC.exe, которые привязаны к версиям платформы .NET, установленным на целевом сервере). <li>Временная папка или расположение кэша. <br>*По умолчанию временная папка или расположение кэша находятся по адресу C:\Program Files\Microsoft Azure Recovery Services Agent\Scratch*.

## <a name="error-the-microsoft-azure-recovery-service-agent-was-unable-to-connect-to-microsoft-azure-backup"></a>Ошибка. Агенту служб восстановления Microsoft Azure не удалось подключиться к службе архивации Microsoft Azure.

| Сведения об ошибке | Возможные причины | Рекомендуемые действия |
| ---     | ---     | ---    |
| **Ошибка** </br><ol><li>*Агенту служб восстановления Microsoft Azure не удалось подключиться к службе архивации Microsoft Azure. (ID: 100050) Проверьте параметры сети и наличие подключения к Интернету*.<li>*(407) Требуется аутентификация прокси-сервера* |Прокси-сервер блокирует подключения* |  <ol><li>Перейдите к разделу "Обозреватель" > "Безопасность" > "Интернет", затем щелкните "Другой...", прокрутите вниз до раздела скачивания файла и выберите действие "Включить".<li>Возможно, узел следует добавить в список [надежных сайтов](https://docs.microsoft.com/en-us/azure/backup/backup-try-azure-backup-in-10-mins#network-and-connectivity-requirements).<li>Измените параметры прокси-сервера и предоставьте правильные сведения о нем. <li>Если на сервере установлена антивирусная программа, исключите из проверки в ней следующие файлы. <ol><li>CBengine.exe.<li>(вместо dpmra.exe)<li>CSC.exe (связан с .NET Framework. Для каждой версии .NET, установленной на сервере, существует отдельный файл CSC.exe. Нужно исключить все файлы CSC.exe, которые привязаны к версиям платформы .NET, установленным на целевом сервере). <li>Временная папка или расположение кэша. <br>*По умолчанию временная папка или расположение кэша находятся по адресу C:\Program Files\Microsoft Azure Recovery Services Agent\Scratch*.

## <a name="error-failed-to-set-the-encryption-key-for-secure-backups"></a>Ошибка. Не удалось настроить ключ шифрования для защиты резервных копий.

| Сведения об ошибке | Возможные причины | Рекомендуемые действия |
| ---     | ---     | ---    |      
| **Ошибка** </br>*Не удалось настроить ключ шифрования для защиты резервных копий. Сбой текущей операции из-за внутренней ошибки в службе: "Неправильные входные данные". Повторите операцию через некоторое время. Если проблема будет повторяться, обратитесь в службу поддержки Майкрософт*. |Сервер уже зарегистрирован в другом хранилище.| Отмените регистрацию сервера и зарегистрируйте его снова.

## <a name="error-the-activation-did-not-complete-successfully-the-current-operation-failed-due-to-an-internal-service-error-0x1fc07"></a>Ошибка. Активация не выполнена. Сбой текущей операции из-за внутренней ошибки в службе [0x1FC07].

| Сведения об ошибке | Возможные причины | Рекомендуемые действия |
| ---     | ---     | ---    |          
| **Ошибка** </br><ol><li>*Активация не выполнена. Сбой текущей операции из-за внутренней ошибки в службе [0x1FC07]. Повторите операцию через некоторое время. Если проблема будет повторяться, обратитесь в службу поддержки Майкрософт*. <li>*Ошибка 34506. Парольная фраза для шифрования, сохраненная для этого компьютера, настроена неправильно*. | <li> Недостаточно свободного места на томе, где расположена временная папка. <li> Временная папка некорректно перемещена в другое расположение. <li> Отсутствует файл OnlineBackup.KEK. | <li>Переместите временную папку или расположение кэша в том, имеющий достаточно свободного пространства (не менее 5–10 % от общего размера данных резервных копий). Правильный процесс перемещения кэша описан [в этой статье](https://docs.microsoft.com/azure/backup/backup-azure-file-folder-backup-faq#backup).<li> Убедитесь, что присутствует файл OnlineBackup.KEK. <br>*По умолчанию временная папка или расположение кэша находятся по адресу C:\Program Files\Microsoft Azure Recovery Services Agent\Scratch*.

# Управление виртуальными машинами


## Управление защищенными виртуальными машинами

1. Для просмотра параметров резервного копирования виртуальной машины и управления ими перейдите на вкладку **Защищенные элементы**.

  - Щелкните имя защищенного элемента, чтобы открыть вкладку **Сведения об архивации**. Здесь отображается информация о последней резервной копии.

        ![Virtual machine backup](./media/backup-azure-manage-vms/backup-vmdetails.png)

2. Для просмотра параметров политики резервного копирования виртуальной машины и управления ими перейдите на вкладку **Политики**.

  - На вкладке **Политики резервного копирования** отображается существующая политика. Ее параметры можно изменить при необходимости. Если необходимо создать новую политику, щелкните **Создать** на **политики** страницы. Обратите внимание, что если вы хотите удалить политику, с ней не должна быть связана ни одна виртуальная машина.

        ![Virtual machine policy](./media/backup-azure-manage-vms/backup-vmpolicy.png)

3. Дополнительные сведения о действиях и состоянии виртуальной машины можно найти на странице **Задания**. Щелкните задание в списке, чтобы получить дополнительные сведения или отфильтровать задания для конкретной виртуальной машины.

    ![Задания](./media/backup-azure-manage-vms/backup-job.png)

## Резервное копирование по запросу виртуальной машины.
По запросу можно выполнить резервное копирование виртуальной после настройки защиты. Если исходный резервного копирования находится в состоянии ожидания для виртуальной машины, резервное копирование по запросу создается полная копия виртуальной машины в резервном хранилище Azure. Если первое резервное копирование завершено, резервное копирование по требованию будет отправлять изменения из предыдущей резервной копии резервное хранилище Azure.

Чтобы выполнить резервное копирование виртуальной машины по запросу:

1. Перейдите к **защищенные элементы** и выберите **Виртуальная машина Azure** как **тип** (если он еще не выбран) и выберите команду **выберите** кнопки.

    ![Тип виртуальной Машины](./media/backup-azure-manage-vms/vm-type.png)

2. Выберите виртуальную машину, на котором требуется выполнить резервное копирование по требованию и выберите команду **теперь резервного копирования** расположенную в нижней части страницы.

    ![Создать резервную копию](./media/backup-azure-manage-vms/backup-now.png)

    Это создаст задание резервного копирования на выбранной виртуальной машиной. Диапазон хранения точки восстановления, созданные с помощью этого задания будут совпадать, указанный в политике, связанных с виртуальной машиной.

    ![Создание задания резервного копирования](./media/backup-azure-manage-vms/creating-job.png)

    >[AZURE.NOTE]Чтобы просмотреть политику, связанную с виртуальной машиной, детализировать виртуальной машины в **защищенные элементы** страницы и перейдите на вкладку политику резервного копирования.

3. После создания задания можно щелкнуть **Просмотр задания** » в строке «всплывающее уведомление для просмотра соответствующего задания на странице «задания».

    ![Задание резервного копирования создано](./media/backup-azure-manage-vms/created-job.png)

4. После успешного завершения задания точки восстановления будет создан, который можно использовать для восстановления виртуальной машины. Это будет увеличивать значение столбца точки восстановления на 1 в **защищенные элементы** страницы.

## Отключение защиты виртуальных машин
Вы можете остановить будущие резервные копии виртуальной машины со следующими параметрами:

- Сохранить резервного копирования данных, связанных с виртуальной машины в Azure резервного хранилища
- Удаление резервных копий данных, связанных с виртуальной машины

Если выбран первый вариант, можно использовать данные резервной копии для восстановления виртуальной машины. Дополнительные сведения для таких виртуальных машин о ценах, нажмите кнопку [здесь](http://azure.microsoft.com/pricing/details/backup/).

Чтобы остановить защиту для виртуальной машины:

1. Перейдите к **защищенные элементы** и выберите **Виртуальная машина Azure** тип фильтра (если он еще не выбран) и нажмите кнопку на **выберите** кнопки.

    ![Тип виртуальной Машины](./media/backup-azure-manage-vms/vm-type.png)

2. Выберите виртуальную машину и щелкните **Остановить защиту** в нижней части страницы.

    ![Остановить защиту](./media/backup-azure-manage-vms/stop-protection.png)

3. По умолчанию резервного копирования Azure не удаляет резервного копирования данных, связанных с виртуальной машиной.

    ![Подтверждение остановки защиты](./media/backup-azure-manage-vms/confirm-stop-protection.png)

    Если вы хотите удалить резервного копирования данных, установите флажок.

    ![Флажок](./media/backup-azure-manage-vms/checkbox.png)

    Выберите причину остановки резервного копирования. Хотя это необязательно, предоставления причины поможет резервного копирования Azure для работы на отзывы и определения приоритетов клиентских сценариев.

4. Щелкните **Отправить** кнопки для отправки **Остановить защиту** задания. Щелкните **Просмотр задания** для просмотра соответствующего задания в **заданий** страницы.

    ![Остановить защиту](./media/backup-azure-manage-vms/stop-protect-success.png)

    Если вы не выбрали **удалить связанные данные резервного копирования** во время **Остановить защиту** мастера, а затем post завершения задания, изменения состояния защиты **остановки защиты**.

    ![Остановки защиты](./media/backup-azure-manage-vms/protection-stopped-status.png)

    При выборе **удалить связанные данные резервного копирования** параметр, виртуальная машина не будет частью **защищенные элементы** страницы.

## Вновь включить защиту виртуальной машины
Если вы не выбрали **удаления связывание резервного копирования данных** параметр **Остановить защиту**, можно повторно защитить виртуальную машину, выполнив действия, аналогично резервное копирование виртуальных машин, зарегистрированных. После защищен, эта виртуальная машина будет иметь сохраняются до остановки защиты резервных копий данных и создания точек восстановления после вновь включить защиту.

После повторного защитить, состояние защиты виртуальной машины будет изменено на **защищенные** при наличии точки восстановления до **Остановить защиту**.

  ![Виртуальная машина защищена повторно](./media/backup-azure-manage-vms/reprotected-status.png)

>[AZURE.NOTE]При заново защиту виртуальной машины, можно выбрать различные политики, чем политики, с которой изначально был защищен виртуальной машины.

## Отмена регистрации виртуальных машин

Чтобы удалить виртуальную машину из хранилища службы архивации, выполните следующие действия.

1. Нажмите в нижней части страницы кнопку **ОТМЕНИТЬ РЕГИСТРАЦИЮ**.

    ![Отключение защиты](./media/backup-azure-manage-vms/unregister-button.png)

    В нижней части экрана отобразится всплывающее уведомление с запросом на подтверждение. Чтобы продолжить, щелкните **ДА**.

    ![Отключение защиты](./media/backup-azure-manage-vms/confirm-unregister.png)

## Удаление данных резервной копии
Можно удалить резервного копирования данных, связанных с виртуальной машины, как:

- Во время остановки задания защиты
- После остановки защиты завершения задания на виртуальной машине

Для удаления резервного копирования данных на виртуальной машине, в «Защиты остановлено» состояние учета успешного завершения **остановки резервного копирования** задания:

1. Перейдите к **защищенные элементы** и выберите **Виртуальная машина Azure** как тип и выберите команду **выберите** кнопки.

    ![Тип виртуальной Машины](./media/backup-azure-manage-vms/vm-type.png)

2. Выберите виртуальную машину. Виртуальные машины будут находиться в **остановки защиты** состояния.

    ![Защита остановлена](./media/backup-azure-manage-vms/protection-stopped-b.png)

3. Щелкните **Удаление** расположенную в нижней части страницы.

    ![Удаление резервной копии](./media/backup-azure-manage-vms/delete-backup.png)

4. В **удаления резервного копирования данных** мастера выберите причину Удаление резервных копий данных (настоятельно рекомендуется) и щелкните **Отправить**.

    ![Удаление архивных данных](./media/backup-azure-manage-vms/delete-backup-data.png)

5. Создается задание для удаления данных резервной копии выбранной виртуальной машины. Щелкните **Просмотр задания** для просмотра соответствующего задания на странице заданий.

    ![Удаление данных выполнено](./media/backup-azure-manage-vms/delete-data-success.png)

    После завершения задания операции, соответствующей виртуальной машины будут удалены из **защищенные элементы** страницы.

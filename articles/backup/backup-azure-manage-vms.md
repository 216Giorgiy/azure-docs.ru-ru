# <a name="manage-azure-vm-backups"></a>Управление резервными копиями виртуальных машин Azure

В этой статье описывается, как управлять процессами резервного копирования виртуальных машин Azure в [службе Azure Backup](backup-overview.md), а также приводятся сведения об оповещениях резервного копирования, которые доступны на панели мониторинга на портале.


На панели мониторинга хранилища служб восстановления на портале Azure предоставляются следующие сведения:

* последняя резервная копия, которая одновременно является последней точкой восстановления;
* политика резервного копирования;
* общий размер всех моментальных снимков резервных копий;
* количество виртуальных машин, для которых настроено резервное копирование.

Вы можете управлять резервным копированием на панели мониторинга или разворачивая узлы до отдельных виртуальных машин. Для начала откройте элемент хранилища на панели мониторинга. 

![Полное представление с ползунком](./media/backup-azure-manage-vms/bottom-slider.png)

## <a name="view-vms-in-the-dashboard"></a>Просмотр виртуальных машин на панели мониторинга

1. Войдите на [портале Azure](https://portal.azure.com/).
2. В главном меню щелкните **Обзор**, а затем в списке ресурсов введите **Службы восстановления**. Как только вы начнете вводить символы, список отфильтруется соответствующим образом. Щелкните **Хранилище служб восстановления**. 
    ![Создание хранилища Служб восстановления — шаг 1](./media/backup-azure-manage-vms/browse-to-rs-vaults.png)

3. Чтобы упростить доступ к хранилищу, щелкните его имя правой кнопкой мыши в списке хранилищ и выберите действие **Закрепить на панели мониторинга**.
3. Откройте панель мониторинга хранилища. 
    ![Открытые панели мониторинга хранилища и колонки "Параметры"](./media/backup-azure-manage-vms/full-view-rs-vault.png)

4. На плитке **Архивные элементы** щелкните **Виртуальные машины Azure**.

    ![Открытие элемента "Архивируемые элементы"](./media/backup-azure-manage-vms/contoso-vault-1606.png)

5. В колонке **Архивные элементы** отображаются последние задания резервного копирования для каждого элемента. В этом примере приведена одна виртуальная машина (demovm-markgal), защищенная с помощью этого хранилища.  

    ![Элемент "Архивируемые элементы"](./media/backup-azure-manage-vms/backup-items-blade-select-item.png)

 
6. На панели мониторинга элемента хранилища вы можете создать или изменить политики резервного копирования, просмотреть точки восстановления, запустить резервное копирование по запросу, остановить или возобновить защиту виртуальных машин, удалить точки восстановления или запустить восстановление.

    ![Панель мониторинга архивируемого элемента с колонкой "Параметры"](./media/backup-azure-manage-vms/item-dashboard-settings.png)



## <a name="manage-backup-policies"></a>Управление политиками резервного копирования
1. На [панели мониторинга элемента хранилища](backup-azure-manage-vms.md#open-a-vault-item-dashboard) щелкните **Все параметры**.

    ![Колонка "Политика архивации"](./media/backup-azure-manage-vms/all-settings-button.png)
2. В разделе **Параметры** щелкните **Политика архивации**.
3. Используйте меню **Выбор политики архивации** , чтобы выполнить следующие действия.

   * Чтобы изменить политику, выберите необходимую политику и нажмите кнопку **Сохранить**. Новая политика будет немедленно применена к хранилищу.
   * Чтобы создать политику, щелкните **Создать**. [Подробнее](backup-azure-arm-vms-prepare.md#configure-a-backup-policy)

     ![Резервное копирование виртуальной машины](./media/backup-azure-manage-vms/backup-policy-create-new.png)


## <a name="run-an-on-demand-backup"></a>Выполнение резервного копирования по запросу
Резервное копирование виртуальной машины по запросу можно выполнять, когда для нее настроена защита.
- Если начальное резервное копирование находится в состоянии ожидания, процесс резервного копирования по запросу создает полную копию виртуальной машины в хранилище служб восстановления.
- Если начальное резервное копирование уже завершено, процесс резервного копирования по запросу отправит в хранилище служб восстановления только те изменения, которые произошли со времени создания предыдущего моментального снимка. Таким образом, все последующие резервные копии являются добавочными.
- Для диапазона хранения резервных копий, созданных по запросу, используется значение, указанное в политике для ежедневной точки резервного копирования. Если значение ежедневного сохранения не выбрано, используется значение еженедельного сохранения.


Чтобы запустить резервное копирование по запросу, сделайте следующее:

1. На [панели мониторинга элемента хранилища](backup-azure-manage-vms.md#open-a-vault-item-dashboard)щелкните **Создать резервную копию**.

    ![Кнопка "Создать резервную копию"](./media/backup-azure-manage-vms/backup-now-button.png)

 2. Чтобы запустить задание резервного копирования, нажмите кнопку **Да** .

    ![Кнопка "Создать резервную копию"](./media/backup-azure-manage-vms/backup-now-check.png)

 
 Задание резервного копирования создает точку восстановления. Период хранения созданной точки восстановления указан в политике, связанной с этой виртуальной машиной. Чтобы отслеживать выполнение задания, на панели мониторинга хранилища щелкните плитку **Задания архивации** .  

## <a name="stop-protecting-a-vm"></a>Отключение защиты для виртуальной машины

У вас есть два способа отключить защиту для виртуальных машин:

- Можно остановить все будущие задания резервного копирования и удалить все точки восстановления. В этом случае вы не сможете восстановить виртуальную машину.
- Можно остановить все будущие задания резервного копирования, но сохранить точки восстановления. Сохранение точек восстановления в хранилище сопровождается определенными затратами. Но зато вы сможете позднее восстановить эту виртуальную машину, если потребуется. [Дополнительные сведения](https://azure.microsoft.com/pricing/details/backup/) о ценах.

Чтобы остановить защиту виртуальной машины, сделайте следующее:

1. На [панели мониторинга элемента хранилища](backup-azure-manage-vms.md#open-a-vault-item-dashboard)щелкните **Остановить резервное копирование**.

    ![Кнопка прекращения резервного копирования](./media/backup-azure-manage-vms/stop-backup-button.png
2. Выберите, будут ли данные резервного копирования сохранены или удалены. Подтвердите, если появится запрос, и введите комментарий (необязательно). Если вы не помните имя элемента, чтобы просмотреть его, наведите указатель мыши на восклицательный знак.

    ![остановка защиты;](./media/backup-azure-manage-vms/retain-or-delete-option.png)

 После остановки отобразится сообщение с соответствующим уведомлением.


## <a name="resume-protection-of-a-vm"></a>Возобновление защиты виртуальной машины

Если при остановке виртуальной машины вы сохранили данные резервного копирования, защиту можно возобновить. Если данные резервного копирования удалены, возобновить защиту не удастся.

Te

1. На [панели мониторинга элемента хранилища](backup-azure-manage-vms.md#open-a-vault-item-dashboard)щелкните **Возобновить резервное копирование**.

2. Чтобы назначить политику для виртуальной машины, следуйте инструкциям в разделе [Управление политиками резервного копирования](backup-azure-manage-vms.md#manage-backup-policies). Вы можете выбрать другую политику, отличную от политики, с помощью которой виртуальная машина была защищена изначально.
3. После применения политики резервного копирования к виртуальной машине вы увидите следующее сообщение:

    ![Защищенные виртуальные машины](./media/backup-azure-manage-vms/success-message.png)

## <a name="delete-backup-data"></a>Удаление данных резервных копий

Данные резервного копирования, связанные с виртуальной машиной, можно удалить при **остановке резервного копирования** или в любое время после завершения этого задания.

- Возможно, стоит подождать несколько дней или недель, прежде чем удалять точки восстановления.
- В отличие от восстановления точек восстановления при удалении данных резервного копирования нельзя выбрать определенные точки для удаления. При удалении данных резервного копирования удаляются все точки восстановления, связанные с этим элементом.

В этой процедуре предполагается, что задание резервного копирования для виртуальной машины остановлено или отключено.


1. На [панели мониторинга элемента хранилища](backup-azure-manage-vms.md#open-a-vault-item-dashboard)щелкните **Delete backup**(Удалить резервную копию).

    ![Тип виртуальной машины](./media/backup-azure-manage-vms/delete-backup-buttom.png)

2. Чтобы подтвердить удаление точек восстановления, введите имя элемента.

    ![Остановить проверку](./media/backup-azure-manage-vms/item-verification-box.png)

4. Чтобы удалить данные резервных копий для текущего элемента, нажмите кнопку ![Остановить архивацию](./media/backup-azure-manage-vms/delete-button.png).

    После удаления отобразится сообщение с соответствующим уведомлением.

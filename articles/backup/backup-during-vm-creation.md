# <a name="enable-backup-when-you-create-an-azure-virtual-machine"></a>Включить резервное копирование, при создании виртуальной машины Azure

Используйте службу архивации Azure для резервного копирования виртуальных машин (ВМ) Azure. В соответствии с расписанием, заданным в политику резервного копирования резервного копирования виртуальных машин, и создания точек восстановления из резервных копий. Точки восстановления хранятся в хранилищах служб восстановления.

В этой статье подробно описано, как включить архивацию во время создания виртуальной машины (VM) на портале Azure.  

## <a name="sign-in-to-azure"></a>Вход в Azure

Если вы еще не вошли учетную запись, войдите в [портала Azure](https://portal.azure.com).
 
## <a name="create-a-vm-with-backup-configured"></a>Создание виртуальной Машины с настроенной архивацией 

1. В левом верхнем углу портала Azure, выберите **New**.

1. Выберите **вычислений**, а затем выберите образ виртуальной Машины.

1. Введите сведения для виртуальной Машины. Имя пользователя и пароля будет использоваться для входа в виртуальную Машину. Когда вы закончите, выберите **ОК**. 

1. Выберите размер виртуальной машины.  

1. В разделе **параметры** > **резервного копирования**выберите **включено** чтобы открыть параметры конфигурации резервного копирования.

   - Чтобы принять значения по умолчанию, выберите **ОК** на **параметры** страницы. Вы перейдете к **Сводка** страницы, чтобы создать виртуальную Машину. Пропустите шаги 6 – 8.
   - Для изменения значений конфигурации архивации, выполните следующие шаги.  

1. Создайте или выберите хранилище служб восстановления для хранения резервных копий виртуальной машины. При создании хранилища служб восстановления, можно выбрать группу ресурсов для хранилища.  

    ![Параметры конфигурации резервного копирования на странице создания виртуальной Машины](./media/backup-during-vm-creation/create-vm-backup-config.png) 

    > [!NOTE] 
    > Группу ресурсов для хранилища служб восстановления может отличаться от группы ресурсов для виртуальной Машины.  

1. По умолчанию политику резервного копирования выбирается защитить виртуальную Машину. Политика архивации указывает частоту создания резервных моментальных снимков и срок их хранения этих резервных копий. 

   - Вы можете принять политику по умолчанию либо создать или выбрать другую политику архивации. 
   - Чтобы изменить политику резервного копирования, выберите **политика архивации** и измените значения.  

1. Завершив настройку значений конфигурации архивации, выберите **ОК** на **параметры** страницы.  

1. На **Сводка** странице после успешно прошел проверку, выберите **создать** для создания виртуальной Машины, который использует настроенные параметры резервного копирования. 

## <a name="start-a-backup-after-creating-the-vm"></a>Начать резервное копирование после создания виртуальной Машины 

Несмотря на то, что вы настроили политику резервного копирования для виртуальной Машины, рекомендуется создать начальный архив. 

После завершения создания шаблона виртуальной Машины, перейдите к **операций** в меню слева и выберите **резервного копирования** Чтобы просмотреть сведения об архивации для виртуальной машины. Можно использовать эту страницу, чтобы:

- Активируйте резервное копирование по запросу.
- Восстановление полной виртуальной Машины или всех дисков.
- Восстановление файлов из резервной копии виртуальной Машины.
- Изменение политики архивации, связанные с виртуальной Машиной.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы 

### <a name="which-vm-images-support-backup-configuration-during-vm-creation"></a>Какие образы виртуальных Машин поддерживают конфигурации резервного копирования во время создания виртуальной Машины?

Включение архивации во время создания виртуальной Машины, поддерживаются следующие основные образы, опубликованные корпорацией Майкрософт. Для других виртуальных машин архивацию можно включить после создания виртуальной Машины. Дополнительные сведения см. в разделе [включить резервное копирование после создания виртуальной Машины](quick-backup-vm-portal.md).

- **Windows** -Windows Server 2016 Datacenter, основных компонентов Windows Server 2016 Datacenter, Windows Server 2012 Datacenter, Windows Server 2012 R2 Datacenter, Windows Server 2008 R2 SP1 
- **Ubuntu** — сервер Ubuntu 17.10, сервер Ubuntu 17.04, Ubuntu Server 16.04 (LTS), Ubuntu Server 14.04 (LTS) 
- **Red Hat** — RHEL 6.7, 6.8, 6.9, 7.2, 7.3, 7.4 
- **SUSE**. SUSE Linux Enterprise Server 11 с пакетом обновления 4 (SP4), 12 (SP2), 12 (SP3). 
- **Debian**. Debian 8, Debian 9. 
- **CentOS**. CentOS 6.9, CentOS 7.3. 
- **Oracle Linux**. Oracle Linux 6.7, 6.8, 6.9, 7.2, 7.3. 
 
### <a name="is-the-backup-cost-included-in-the-vm-cost"></a>— Это расходы на резервное копирование включает в себя стоимость виртуальной Машины? 

№ Стоимость резервного копирования отделены от затрат на виртуальной Машине. Дополнительные сведения о ценах на архивацию, см. в разделе [ценами на архивацию Azure](https://azure.microsoft.com/pricing/details/backup/).
 
### <a name="which-permissions-are-required-to-enable-backup-on-a-vm"></a>Какие разрешения необходимы, чтобы включить архивацию на виртуальной машине? 

Если вы участник виртуальной Машины, вы можете включить резервное копирование на виртуальной Машине. При использовании пользовательской роли, вам потребуются следующие разрешения для включения архивации виртуальной Машины: 

- Microsoft.RecoveryServices/Vaults/write 
- Microsoft.RecoveryServices/Vaults/read 
- Microsoft.RecoveryServices/locations/* 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/*/read 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/read 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/write 
- Microsoft.RecoveryServices/Vaults/backupFabrics/backupProtectionIntent/write 
- Microsoft.RecoveryServices/Vaults/backupPolicies/read 
- Microsoft.RecoveryServices/Vaults/backupPolicies/write 
 
Если хранилище служб восстановления и виртуальной Машины имеют разные группы ресурсов, убедитесь, что у вас есть разрешения на запись в группу ресурсов для хранилища служб восстановления.

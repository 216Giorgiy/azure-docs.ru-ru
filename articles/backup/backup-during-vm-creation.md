# <a name="enable-backup-during-azure-virtual-machine-creation"></a>Включение архивации виртуальной машины Azure во время создания 

Служба Azure Backup предоставляет интерфейс для создания и настройки резервных копий в облаке. Защитите свои данные, создавая точки восстановления с регулярными интервалами. Служба Azure Backup создает точки восстановления, которые могут храниться в геоизбыточных хранилищах служб восстановления. В этой статье подробно описывается, как включить архивацию при создании виртуальной машины на портале Azure.  

## <a name="log-in-to-azure"></a>Вход в Azure 

Если вы еще не вошли в свою учетную запись, войдите на [портал Azure](http://portal.azure.com).
 
## <a name="create-virtual-machine-with-backup-configured"></a>Создание виртуальной машины с настроенной архивацией 

1. В верхнем левом углу портала Azure щелкните **Создать**. 

2. Выберите **Среда выполнения приложений**, а затем — образ виртуальной машины.   

3. Введите сведения о виртуальной машине. Предоставленные имя пользователя и пароль понадобятся для входа на виртуальную машину. По завершении нажмите кнопку **ОК**. 

4. Выберите размер виртуальной машины.  

5. В разделе **Параметры > Резервное копирование**, щелкните **Включено**, чтобы открыть параметры конфигурации архивации. Вы можете принять значения по умолчанию и щелкнуть **ОК** на странице параметров, чтобы перейти на страницу "Сводка" для создания виртуальной машины. Если вы хотите изменить значения, выполните следующие шаги.  

6. Создайте или выберите хранилище служб восстановления, в котором содержатся резервные копии виртуальной машины. При создании хранилища служб восстановления можно выбрать группу ресурсов для него.  

    ![Настройка архивации на странице создания виртуальной машины](./media/backup-during-vm-creation/create-vm-backup-config.png) 

    > [!NOTE] 
    > Группа ресурсов хранилища служб восстановления может отличаться от группы ресурсов виртуальной машины.  
    > 
    > 

7. По умолчанию политика архивации выбирается, чтобы быстро защитить виртуальную машину. Политика архивации указывает частоту создания резервных моментальных снимков и их срок хранения. Вы можете принять политику по умолчанию либо создать или выбрать другую политику архивации. Чтобы изменить политику архивации, выберите **Политика архивации** и измените имеющуюся политику.  

8. После ввода значений конфигурации архивации на странице параметров нажмите кнопку **ОК**.  

9. На странице сводки после прохождения проверки щелкните **Создать**, чтобы создать виртуальную машину, использующую параметры резервного копирования. 

## <a name="initiate-a-backup-after-creating-the-vm"></a>Архивация после создания виртуальной машины 

Несмотря на то, что была создана политика архивации, рекомендуем создать первоначальную резервную копию. Чтобы просмотреть сведения об архивации виртуальной машины после создания шаблона виртуальной машины, в области **Операции** в меню слева щелкните **Backup** (Архивация). Это можно использовать для запуска архивации по требованию, восстановления полной виртуальной машины или всех дисков, восстановления файлов из резервной копии виртуальной машины или изменения политики архивации, связанной с виртуальной машиной.  

## <a name="using-a-resource-manager-template-to-deploy-a-protected-vm"></a>Развертывание защищенной виртуальной машины с помощью шаблона Resource Manager

Ранее было описано, как использовать портал Azure для создания виртуальной машины и ее защиты в хранилище служб восстановления. Если вы хотите быстро развернуть одну или несколько виртуальных машин и защитить их в хранилище служб восстановления, ознакомьтесь с шаблоном из раздела [Deploy a Windows VM and enable backup using Azure Backup](https://azure.microsoft.com/resources/templates/101-recovery-services-create-vm-and-configure-backup/) (Развертывание виртуальной машины Windows и включение резервного копирования с помощью Azure Backup).

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы 

### <a name="which-vm-images-enable-backup-at-the-time-of-vm-creation"></a>Какие образы виртуальных машин активируют архивацию во время создания виртуальной машины? 

Основные образы, опубликованные корпорацией Майкрософт, в списке ниже поддерживают включение архивации во время создания виртуальной машины. Для других виртуальных машин архивацию можно включить после создания. Дополнительные сведения см. в статье [Резервное копирование виртуальной машины в Azure](quick-backup-vm-portal.md). 

- **Windows**. Windows Server 2016 Datacenter, ядро Windows Server 2016 Datacenter, Windows Server 2012 Datacenter, Windows Server 2012 R2 Datacenter, Windows Server 2008 R2 с пакетом обновления 1. 
- **Ubuntu**. Сервер Ubuntu 1710, сервер Ubuntu 1704, сервер Ubuntu 1604 (LTS), сервер Ubuntu 1404 (LTS). 
- **RedHat**. RHEL 6.7, 6.8, 6,9, 7.2, 7.3, 7.4. 
- **SUSE**. SUSE Linux Enterprise Server 11 с пакетом обновления 4 (SP4), 12 (SP2), 12 (SP3). 
- **Debian**. Debian 8, Debian 9. 
- **CentOS**. CentOS 6.9, CentOS 7.3. 
- **Oracle Linux**. Oracle Linux 6.7, 6.8, 6.9, 7.2, 7.3. 
 
### <a name="is-backup-cost-included-in-the-vm-cost"></a>Включены ли расходы на архивацию в стоимость виртуальной машины? 

Нет, расходы на архивацию — это отдельные расходы, не связанные со стоимостью виртуальной машины. Дополнительные сведения о ценах на резервное копирование см. на [этой странице](https://azure.microsoft.com/pricing/details/backup/).
 
### <a name="which-permissions-are-required-to-enable-backup-on-a-vm"></a>Какие разрешения необходимы, чтобы включить архивацию на виртуальной машине? 

Если вам назначена роль "Участник", вы можете включить архивацию на виртуальной машине. При использовании пользовательской роли вам потребуются следующие разрешения для включения архивации виртуальной машины. 

- Microsoft.RecoveryServices/Vaults/write 
- Microsoft.RecoveryServices/Vaults/read 
- Microsoft.RecoveryServices/locations/* 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/*/read 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/read 
- Microsoft.RecoveryServices/Vaults/backupFabrics/protectionContainers/protectedItems/write 
- Microsoft.RecoveryServices/Vaults/backupFabrics/backupProtectionIntent/write 
- Microsoft.RecoveryServices/Vaults/backupPolicies/read 
- Microsoft.RecoveryServices/Vaults/backupPolicies/write 
 
Если хранилище служб восстановления и виртуальная машина имеют разные группы ресурсов, убедитесь, что у вас есть права на запись в группе ресурсов хранилища служб восстановления.

#Ограничения DocumentDB для предварительного выпуска
В следующей таблице указаны ограничения и квоты, применяемые к DocumentDB на этапе предварительной версии. В большинстве случаев ограничения применяются либо с намерением получить ваши отзывы или на основании текущих ограничений емкости ресурсов. Если у вас возникнет необходимость увеличить указанные пределы, позвоните нам, и мы сделаем все от нас зависящее в рамках ограничений публичной оферты.    

|Сущность |Квота (Стандартная оферта для выпуска предварительной версии)|
|-------|--------|
|Учетные записи базы данных     |5
|Количество баз данных в учетной записи базы данных     |100
|Количество пользователей в учетной записи базы данных (итоговое для всех баз данных) |500 000
|Количество разрешений в учетной записи базы данных (итоговое для всех баз данных)   |2 000 000
|Объем хранилища вложений в учетной записи базы данных      |2 ГБ
|Максимальное количество единиц емкости в учетной записи базы данных       |50
|Количество коллекций в единице емкости      |3
|Минимальный размещенный объем хранилища для коллекции хотя бы с одним документом    |3,3 ГБ
|Минимальная размещенная пропускная способность для коллекции хотя бы с одним документом |667 RU
|Гибкость коллекции    |0-10 ГБ
|Максимальное количество единиц запросов в секунду для одной коллекции   |2000
|Количество хранимых процедур, триггеров и пользовательских функций для одной коллекции       |25 на каждую
|Максимальное время выполнения хранимой процедуры или триггера     |5 с
|Подготовленный размер хранилища/единицы емкости |10 ГБ
|Подготовленные единицы запросов в секунду/единицы емкости     |2000
|Максимальный объем хранилища документов базы данных (5 единиц емкости)    |500 ГБ
|Максимальная длина свойства идентификатора    |255 символов
|Количество элементов на одной странице по умолчанию     |100
|Максимальное количество элементов на одной странице        |1000
|Максимальный размер запроса документа и вложения       |512 ГБ
|Максимальный размер запроса хранимой процедуры, триггера и пользовательской функции        |256KB
|Максимальный размер ответа |1 МБ
|Максимальное количество отдельных путей в коллекции       |100
|Строка |Все строки должны соответствовать кодировке UTF-8. Так как UTF-8 представляет собой кодирование с переменной шириной, размеры строк определяются с использованием байт UTF-8.
|Максимальная длина свойства или значения  |Без ограничений
|Максимальное количество пользовательских функций в запросе     |1
|Максимальное количество соединений в запросе    |2
|Максимальное количество предложений AND в запросе      |5
|Максимальное количество предложений OR в запросе       |5
